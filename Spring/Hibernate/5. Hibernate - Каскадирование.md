Каскадирование - механизм, который определяет как поступить с данными при удалении одних данных связанными с другими

```java
public class App {
	public static void main(String[] args) {
		Configuration myConfigHiber = new Configuration();
		myConfigHiber.addAnnotatedClass(Client.class);
		myConfigHiber.addAnnotatedClass(Item.class);
		
		SessionFactory mySessionFact = myConfigHiber.buildSessionFactory();
		Session mySession = mySessionFact.getCurrentSession();
		
		try {
			mySession.beginTransaction();
			
			Client pers = new Client("Tom", 25);
			Item item = new Item(pers, "Apple phone");
			
			pers.setListItems(new ArrayList<>(Collections.singletonList(item)));
			
			mySession.save(pers);
			
			mySession.getTransaction().commit();
		}finally {
			mySessionFact.close();
		}
	}
}
```

В данном случае ==мы создали два объекта двух классов== `Client` и `Item`, а также ==связали их== двусторонней связью (указали в конструкторе `Item` объект `Client`, и добавили товар в список товаров клиента)

***
# 1. Настройка каскадирования в `@OneToMany`
Обычно ==каскадирование настраивается в родительском классе==

> [!NOTE] `@OneToMany` и параметр `cascade`
> Для  определения каскадирование в аннотации используется параметр `cascade`, в котором можно выбрать каким будет каскадирование
> 

```java
. . .
@OneToMany(mappedBy = "clientRef", cascade = CascadeType.PERSIST)
private List<Item> listItems;
. . .
```

***
# 2. Метод `persist()` класса `Session`

> [!NOTE] `persist()`
> Для использования каскадирования ВМЕСТЕ С ПАРАМЕТРОМ `cascade` АННОТАЦИИ `@OneToMany` НЕОБХОДИМО использовать метод `persist`, а не `save()`
> 
> Если использовать метод `save()` каскадирование не будет использоваться

# 4. Настройка каскадирование в `save()` с помощью аннотации `@Cascade`

> [!NOTE] Каскадирование метода `save()`
>
> Чтобы при использовании метода `save()` выполнялось каскадирование, необходимо использовать аннотацию `@Cascade`

```java
@OneToMany(mappedBy = "clientRef")
@Cascade(org.hibernate.annotations.CascadeType.SAVE_UPDATE)
private List<Item> listItems; 
```

Теперь вызов метода `save()` будет также КАСКАДИРОВАТЬСЯ
```java
. . .
try {
	mySession.beginTransaction();
	
	Client pers = new Client("Jery", 25);
	Item item = new Item(pers, "Apple phone");
	
	pers.setListItems(new ArrayList<>(Collections.singletonList(item)));
	mySession.save(pers);
	mySession.getTransaction().commit();
. . .
```
