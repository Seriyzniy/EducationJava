# 1. Добавление зависимости в `pom.xml`

Добавим в зависимости проекта `spring-jdbc`, которая предоставит нам класс `JdbcTemplate`

```xml
<dependency>
	<groupId>org.springframework</groupId>
	<artifactId>spring-jdbc</artifactId>
	<version>${spring.version}</version>
</dependency>
```

# 2. Создание `Bean` JdbcTemplate в контексте

Чтобы класс `JdbcTemplate` смог работать с данными, ему необходимо передать в конструктор источник данных `DataSource`

```java
@Bean
public DataSource dataSource() {
	DriverManagerDataSource driverManagerDataSource = new DriverManagerDataSource();
	
	driverManagerDataSource.setDriverClassName("org.postgresql.Driver");
	driverManagerDataSource.setUrl("jdbc:postgresql://localhost:5432/myFirstDB");
	driverManagerDataSource.setUsername("postgres");
	driverManagerDataSource.setPassword("12345");
	
	return driverManagerDataSource;
}
```
Для этого ==необходимо создать профиль соединения в Java-коде==, и указать ДРАЙВЕР для подключения к PostgreSQL, URL, имя пользователя и пароль

После этого создадим бин класса `JdbcTemplate`
```java
@Bean
public JdbcTemplate jdbcTemplate() {
	return new JdbcTemplate(dataSource());
}
```

# 3. Изменение метода `getList()`

Теперь мы можем использовать наш объект `JdbcTemplate`.
```java
public List<Person> getList(){
	return myJdbcTemplate.query("SELECT * FROM persons", new PersonMapper());
}
```


> [!NOTE] Метод `query()`
> ==Метод ВОЗВРАЩАЕТ== `List<>`
> 
> Метод `query()` имеет следующие параметры:
> - 1-й параметр это SQL запрос
> - 2-й параметр это класс, реализующий интерфейс `RowMapper`


> [!NOTE] `RowMapper`
> `RowMapper` - интерфейс, который ==отображает строки из таблицы== в нужные нам сущности(==в объекты==)
> 
> Содержит метод `mapRow`, который и переводит данные из таблицы в объект необходимого класса
> 
> ==Класс==, реализующий интерфейс ==мы пишем сами==

***
# 4. Класс реализующий интерфейс `RowMapper`

```java
public class PersonMapper implements RowMapper<Person> {
	@Override
	public Person mapRow(ResultSet rs, int rowNum) throws SQLException {
		Person myNewPerson = new Person();
		
		myNewPerson.setName(rs.getString("personname"));
		myNewPerson.setEmail(rs.getString("email"));
		myNewPerson.setAge(rs.getInt("age"));
		myNewPerson.setId(rs.getInt("id"));
		
		return myNewPerson;
	}
}
```

В этом классе мы ==один раз пишем код==, который ==преобразует строки из таблицы в объект== класса `Person`. И далее просто вызываем этот объект

# 5. УПРОЩЕНИЕ `RowMapper`, `BeanPropertyRowMapper<>`

Если в таблице НАЗВАНИЯ СТОБЛЦА соответствует определенному ПОЛЮ класса, то можно воспользоваться готовый классом `BeanPropertyRowMapper`, реализующим интерфейс `RowMapper`

Тогда код будет выглядеть так:
```java
public List<Person> getList(){
	return myJdbcTemplate.query("SELECT * FROM persons", BeanPropertyRowMapper<>(Person.class));
}
```

> [!NOTE] Класс `BeanPropertyRowMapper`
> Класс автоматически сопоставит СТОЛБЦЫ таблицы ПОЛЯМ класса Person, которые совпадают по имени

Но поскольку у нас в таблице столбец с именем называется не просто `name`, а `personname`, данный метод нам не подходит (хотя можно изменить в классе `Person` название поля)

# 6. Изменение метода `getPerson()`

```java
public Person getPerson(int id) 
{
	return myJdbcTemplate.query("SELECT * FROM persons WHERE id=?", new Object[]{id}, new PersonMapper()).stream().findAny().orElse(null);
}
```

Поскольку `JdbcTemplate` использует ПО УМОЛЧАНИЮ `PreparedStatement`, мы можем использовать возможности этого класса.

Метод `query` ==ожидает, что вместо знака== `?` мы подставим ==массив значений== (т.к `?` может быть несколько).
Поэтому мы пишем
```java
new Object[]{id}
```

Так как метод `query` возвращает `List<>` нам необходимо ==с помощью лямбда выражений== ==выбрать== из списка один объект (==первый попавшийся==) или `null`

# 7. Изменение метода `saved()`

```java
public void saved(Person pers) {
	myJdbcTemplate.update("INSERT INTO persons VALUES(1,?,?,?)", pers.getName(), pers.getAge(), pers.getEmail());
}
```

Для вставки данных в таблицу используется метод `update` класса`JdbcTemplate`, у которого ==второй параметр может принимать любое количество параметров==
