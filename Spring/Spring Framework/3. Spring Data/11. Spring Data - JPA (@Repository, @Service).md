# 1. Аннотация`@Repository` - Ключевое понятие в JPA

==Репозиторий== - ==интерфейс== помеченный аннотацией `@Repository`, который можно определить для получения доступа к данным. JPA-запросы создаются автоматически на основе имен методов

Благодаря использованию `JpaRepository`, мы получим автоматически СГЕНЕРИРОВАННЫЕ МЕТОДЫ для работы с сущностью Person

![[Pasted image 20241219102316.png]]

> [!NOTE] Класс `JpaRepository`
> `JpaRepository<Person, Integer>` в качестве первого параметра указываем ==для какой сущности создаем репозиторий==, а в вторым параметром ==указываем ТИП данных первичного ключа==

#                           Спецификация JPA
***
Ранее ==мы использовали понятия== `Session` и  `SessionFactory` ==ПРИСУЩИЕ ТОЛЬКО== `Hibernate`

> [!NOTE] Особенности JPA
> В спецификации JPA ТАКИХ ПОНЯТИЙ НЕТ.
> Вместо этого там используются такие понятия как:
> - `EntityManager`
> - `EntityManagerFactory`
> 
> Эти понятия аналогичны `Session`  `SessionFactory`

Для создание сессии из `entityManager` используется метод `unwrap(Session.class)`
