# 3. Изменение `SessionFactory` на `EntityManagerFactory`

Поскольку в JPA используется понятие `EntityManagerFactory` а не `SessionFactroy` используемое только в `Hibernate` (см теории раздела 11), то необходимо изменить функцию `sessionFactory`

```java
@Bean
public LocalContainerEntityManagerFactoryBean entityManagerFactory() {
	final LocalContainerEntityManagerFactoryBean em = new LocalContainerEntityManagerFactoryBean();
	
	em.setDataSource(dataSource());
	em.setPackagesToScan("ru.my.spring.crud.myModel");
	
	
	final HibernateJpaVendorAdapter vendorAdapter = new HibernateJpaVendorAdapter();
	
	em.setJpaVendorAdapter(vendorAdapter);
	em.setJpaProperties(hibernateProperties());
	
	return em;
}
```

> [!NOTE] `LocalContainerEntityManagerFactoryBean`
> `LocalContainerEntityManagerFactoryBean` — это Spring-обертка для настройки `EntityManagerFactory`, используемого в JPA. Он облегчает настройку JPA, добавляя в конфигурацию Spring-friendly свойства.


> [!NOTE] Title
> ```java
> final HibernateJpaVendorAdapter vendorAdapter = new HibernateJpaVendorAdapter();
> em.setJpaVendorAdapter(vendorAdapter);
> ```
> Этот адаптер сообщает Spring, что Hibernate будет использоваться в качестве провайдера JPA

# 4. Изменение функции для работы с транзакциями
```java
@Bean
public PlatformTransactionManager hibernateTransactionManager() {
	JpaTransactionManager transactionManager = new JpaTransactionManager();
	transactionManager.setEntityManagerFactory(entityManagerFactory().getObject());
	
	return transactionManager;
}
```

В целом, эта ==функция похожа на ранее используемую функцию в Hibernate+Spring==
```java
@Bean
public PlatformTransactionManager hibernateTransactionManager() {
	HibernateTransactionManager transactionManager = new HibernateTransactionManager();
	
	transactionManager.setSessionFactory(sessionFactory().getObject());
	
	return transactionManager;
}
```

# 5. Создание СЕРВИСА

Для создания сервиса, так же создадим пакет `services` `src/main/java/ru.my.spring.crud.services`

В этой пакете создадим СЕРВИСНЫЙ КЛАСС
```java
@Service
@Transactional(readOnly = true)
public class PeopleService {
	private final PersonRepository personRepository;
	
	@Autowired
	public PeopleService(PersonRepository personRepository) {
		this.personRepository = personRepository;
	}
	
	public List<Person> findAll(){
		return personRepository.findAll();
	}
	public Person findOne(int id) {
		Optional<Person> optional = personRepository.findById(id);
		return optional.orElse(null);
	}
	
	@Transactional
	public void save(Person person) {
		personRepository.save(person);
	}
	@Transactional
	public void update(int id, Person person) {
		person.setId(id);
		personRepository.save(person);
	}
	@Transactional
	public void delete(int id) {
		personRepository.deleteById(id);
	}
}
```

В методе `update()` для новых введенных данных мы присваиваем `id`, и после этого наш `PersonRepository` поймет, что человек с таким `id` уже есть в таблице и обновит существующие данные

`@Transactional(readOnly = true)` на уровне класса говорит о том, что всем публичным методам ПО УМОЛЧАНИЮ будет соответствовать такая транзакция, кроме тех где мы эту аннотацию не укажем самостоятельно
