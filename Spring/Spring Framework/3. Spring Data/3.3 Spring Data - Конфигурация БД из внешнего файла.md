Указывать свойства БД в коде, плохое решение, поскольку если кто-нибудь получит исходный код приложения, то он сможет подключиться к БД

Для решения проблемы, все данные для подключения должны храниться в скрытом файле

![[Pasted image 20241127131809.png]]
# 1. Создание СКРЫТОГО файла `.properties` в папке `src/main/resources`

```
driver=org.postgresql.Driver
url=jdbc:postgresql://localhost:5432/myFirstDB
username_value=postgres
userpass_value=12345
```

Данные в файле хранятся в виде `ключ=значение` без кавычек

***
# 2. Создание ПУБЛИЧНОГО файла `.origin` в папке `src/main/resources`

```
driver=
url=
username_value=
userpass_value=
```

Данный ==файл определяет структуру НУЖНЫХ свойств==, необходимых ==для запуска приложения==
А также служит подсказкой для других пользователей, которые используют наш код, какие параметры необходимо определить

***
# 3. Чтение свойств из файла

Для того чтобы прочитать значение из формы, используется аннотация `@PropertySource`

```java
@PropertySource("classpath:database.properties")
public class AppContext implements WebMvcConfigurer{
	private final ApplicationContext applicationContext;
	private final Environment environment;
	
	@Autowired
	public AppContext(ApplicationContext applicationContext, Environment environment) {
		this.applicationContext = applicationContext;
		this.environment = environment;
	}
	. . .
```

> [!NOTE] Интерфейс `Enviroment`
> Данный интерфейс позволяет получить доступ к файлам со СВОЙСТВАМИ (`.properties`), которые подключились к нашему приложению 

```java
@Bean
public DataSource dataSource() {
	DriverManagerDataSource driverManagerDataSource = new DriverManagerDataSource();
	
	driverManagerDataSource.setDriverClassName(environment.getProperty("driver"));
	driverManagerDataSource.setUrl(environment.getProperty("url"));
	driverManagerDataSource.setUsername(environment.getProperty("username_value"));
	driverManagerDataSource.setPassword(environment.getProperty("userpass_value"));
	
	return driverManagerDataSource;
}
```

И теперь мы можем из переменной `enviroment` получить доступ к значениям из файла `database.properties`


> [!NOTE] Проверка поля на отсутствие
> Можно уставить проверку, если в файле отсутствует ключ `driver`, то получим ошибку
> ```java
> Objects.requireNonNull(environment.getProperty("driver"))
> ```
