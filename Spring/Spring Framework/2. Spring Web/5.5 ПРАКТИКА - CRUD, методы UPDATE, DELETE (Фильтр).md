# 1. Создание метода возвращающего форму для редактирования

Обсудим идею. Пользователь введет в браузере URL `/people/:id/edit`, данный GET запрос придет на сервер. 

> [!NOTE] Title
> Контроллер, отвечающий за этот запрос, вернет HTML-страницу с формой для ввода новый данных для старой записи. 
> Чтобы найти старую запись, необходимо считать `id` из URL.
> 
> После этого необходимо передать данные из БД в представление, для этого ==поместим в модель редактируемую запись (объект)==
> ```java
> @GetMapping("/{id}/edit")
> public String getEditPersonForm(Model model, @PathVariable("id") int id) {
> 	model.addAttribute("persForId", myPersonDAO.getPerson(id));
> 	return "people/edit";
> }
> ```

# 3. Принятие запроса PATCH
После того как пользователь отправит данные с формы на сервер, их необходимо принять.

> [!NOTE] Метод принятия PATCH запроса
> Данные (объект) с формы мы получим с помощью аннотации `@ModelAttribute` 
> После принятия данных, передадим их объекту `myPersonDAO`, отвечающего за работу с данными
> ```java
> @PatchMapping("/{id}") 
> public String updatePersonInDateBase(@ModelAttribute("updatePerson") Person person, @PathVariable("id") int id)
> {
> 	myPersonDAO.update(id, person);
> 	return "redirect:/people";
> }
> ```

#### 1. ServletContext и ServletConfig

> [!NOTE] `ServletConfig`
> `ServletConfig` — это интерфейс в **Servlet API**, который предоставляет конфигурационные данные для конкретного сервлета. В отличие от `ServletContext`, который является глобальной конфигурацией для всего веб-приложения, `ServletConfig` содержит настройки, специфичные для одного сервлета.

`ServletConfig` — это интерфейс, который предоставляет конфигурационную информацию для сервлета. Каждый сервлет имеет свой собственный экземпляр `ServletConfig`, который создается и передается контейнером сервлетов (например, Tomcat или Jetty)

**Основные задачи `ServletConfig`:**
- Хранить параметры инициализации, которые определяются для конкретного сервлета в файле конфигурации (`web.xml`) или с использованием аннотаций.
- Обеспечивать доступ к объекту `ServletContext` через метод `getServletContext()`.

`ServletConfig` позволяет сервлету получить доступ к параметрам инициализации, которые могут быть специфичны для него.

> [!NOTE] `ServletContext`
> 
> `ServletContext` — это объект, который контейнер сервлетов создает для каждого веб-приложения. Этот объект доступен во всей жизни приложения и предоставляет несколько методов для настройки его работы.
> 
> `ServletContext` - ==интерфейс==, который ==предоставляет методам сервлета== доступ к ==информации о веб-приложении==, включая его инициализацию и ==параметры конфигурации.== 
> 
> Он предоставляет возможности взаимодействия сервлетов с контейнером сервлетов (например, Tomcat, Jetty).
> > 
> **Использование**: `ServletContext` используется для регистрации и управления компонентами веб-приложения, такими как фильтры, слушатели и сервлеты.

Мы добавлям новый фильтр в **контекст веб-приложения**, представленный объектом `ServletContext`. Хотя `ServletContext` — это интерфейс, его реализацию предоставляет контейнер сервлетов (например, Tomcat или Jetty).
#### 2. Класс HiddenHttpMethodFilter

> [!NOTE] `HiddenHttpMethodFilter`
> `HiddenHttpMethodFilter` — это фильтр в Spring MVC, который позволяет обрабатывать скрытые поля формы, чтобы изменять HTTP-метод запроса.
> 
> Фильтр ищет специальное скрытое поле в теле HTTP-запроса с именем `_method`. Если оно найдено, фильтр заменяет метод запроса на тот, который указан в этом поле.

Метод `addMappingForUrlPatterns()` используется для указания, к каким URL-паттернам должен применяться фильтр.
- **`null` (dispatcherTypes)**:
    - Это первый параметр, который задает типы диспетчеризации (`DispatcherType`). Значение `null` означает, что фильтр будет применяться ко всем типам диспетчеризации по умолчанию (например, `REQUEST`, `FORWARD`, `INCLUDE`, `ERROR`, `ASYNC`).
    
- **`true` (isMatchAfter)**:
    - Это второй параметр, который определяет, будет ли фильтр добавлен после (если `true`) или перед (если `false`) фильтрами, определенными в `web.xml`. В данном случае `true` означает, что фильтр будет применяться после всех фильтров, которые уже определены.
    
- **`"/*"` (urlPatterns)**:
    - Третий параметр указывает, какие URL-паттерны будут обрабатываться фильтром. Паттерн `"/*"` означает, что фильтр будет применяться ко всем URL-адресам в приложении, т.е. ко всем входящим HTTP-запросам.

#### 3.  Запуск фильтра
Хотя мы и реализовали ==метод для добавления фильтра==, ==этот метод  необходимо вызвать== при старте сервера (диспетчер сервлета)
```java
@Override
public void onStartup(ServletContext aServletContext) throws ServletException {
	super.onStartup(aServletContext);
	registerHiddenFieldFilter(aServletContext);
}
```
