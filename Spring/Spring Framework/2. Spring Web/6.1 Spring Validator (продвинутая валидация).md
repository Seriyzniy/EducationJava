# 1. Создание класса Spring Validator

Для того чтобы создать класс Sprint Validator, ==необходимо создать пакет== `util`

> [!NOTE] Пакет `util`
> Всякого рода ВСПОМОГАТЕЛЬНЫЕ классы должны лежать в пакете `util`

![[Pasted image 20241128094346.png]]

В данном пакете создадим ==класс== `PersonValidator`. 

==Чтобы использовать функционал== `Spring Validator`, НЕОБХОДИМО чтобы наш класс ==РЕАЛИЗОВАЛ интерфейс== `Validator` из пакета `org.springframework.validation`

```java
@Component
public class PersonValidator implements Validator{
	private PersonDAO persDAO;
	
	@Autowired
	public PersonValidator(PersonDAO personDao) {
		this.persDAO = personDao;
	}
	
	@Override
	public boolean supports(Class<?> clazz) {
		return Person.class.equals(clazz);
	}
		  
	@Override
	public void validate(Object target, Errors errors) {
		Person person = (Person) target;
		if(persDAO.showPersonWithSameEmail(person.getEmail()) != null) {
			errors.rejectValue("email", "", "Person with that email is already exist!");
		}
	}
}
```

> [!NOTE] Интерфейс `Validator` из пакета `org.springframework.validation`
> У данного интерфейса есть два метода:
> - `supports()` необходим для указания КЛАССА, который должен подвергаться валидации
> - `validate()` указываем правила валидации
> 

Для вывода ошибки используем ==объект класса== `Errors`, у которого ==есть метод== `rejectValue`, в который указываем:
- Поле в котором случилась ошибка
- ==КОД ОШИБКИ== (обязательное поле для вывода ошибки)
- Сообщение ошибки

Для того чтобы понять, есть ли человек с введенным email, необходимо обратиться к БД. Для этого создадим поле `PersonDAO`, и реализуем в нем метод `showPersonWithSameEmail()`

# 2. Реализация метода `showPersonWithSameEmail()`

```java
public Person showPersonWithSameEmail(String email) 
{
	return myJdbcTemplate.query("SELECT * FROM persons WHERE email=?", new Object[] {email}, new PersonMapper()).stream().findAny().orElse(null);
}
```

# 3. Использование `Validator`
==Чтобы использовать== описанный ==класс== `PersonValidator`, его необходимо ==ВНЕДРИТЬ в КОНТРОЛЛЕР==

```java
@Controller
@RequestMapping("/people")
public class PeopleController {
	private PersonDAO myPersonDAO;
	
	private PersonValidator persValidator;
	
	@Autowired
	public PeopleController(PersonDAO PersonDAO, PersonValidator persValid) {
		this.myPersonDAO = PersonDAO;
		this.persValidator = persValid;
	}
	. . .
```

Теперь можем АКТИВИРОВАТЬ валидацию
```java
@PostMapping()
public String addPersonInDateBase(@ModelAttribute("newPerson") @Valid Person person, BindingResult bindingResult) {
	//Вызываем валидацию
	persValidator.validate(person, bindingResult);
	
	if(bindingResult.hasErrors())
		return "people/new";
	
	myPersonDAO.saved(person);
	return "redirect:/people";
}
```
В метод `validate()` мы передаем ОБЪЕКТ для валидации и ошибки вызванные простой валидацией (ОБЪЕДИНЯЕМ ошибки из простой валидации и добавляем из продвинутой )
