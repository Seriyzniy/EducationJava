# 3. Создание классов под данным из таблицы

Возьмем ранее созданные классы для работы с таблицей `Persons` из БД `myHibernateDB`
#### Класс Person
```java
@Entity
@Table(name = "Persons")
public class Person {
	@Id
	@Column(name = "p_id")
	@GeneratedValue(strategy = GenerationType.IDENTITY)
	private int id;
	
	@Column(name = "p_name")
	@NotEmpty(message = "Name can`t not be empty!")
	@Size(min = 2, max = 30, message = "Name can be length between 2 and 30 character!")
	private String name;
	
	@Column(name = "p_email")
	@NotEmpty(message = "Email can`t not be empty!")
	@Email(message = "Email should be valid format!")
	private String email;
	
	@Column(name = "p_age")
	@Min(value = 0, message = "Age can`t be negative value!")
	private int age;
	
	@Column(name = "p_adress")
	@Pattern(regexp = "[A-Z]\\w+, [A-Z]\\w+, \\d{6}", message = "Format will be 'Country, City, Index(6 numbers)'")
	private String adress;
	
	@Column(name = "date_of_birthday")
	@Temporal(TemporalType.DATE)
	@DateTimeFormat(pattern = "yyyy-MM-dd")
	private Date dateBirth;
	
	@Column(name="created_at")
	@Temporal(TemporalType.TIMESTAMP)
	private Date createAt;
	
	@Enumerated(EnumType.ORDINAL)
	private Mood mood;
	
	public Person() {};
	public Person(String name, int age, String email, String adress) {
		this.name = name;
		this.age = age;
		this.email = email;
		this.adress = adress;
	}
	public int getId() {
		return id;
	}
	public String getName() {
		return name;
	}
	public String getEmail() {
		return email;
	}
	public int getAge() {
		return age;
	}
	public String getAdress() {
		return adress;
	}
	public Date getDateBirth() {
		return dateBirth;
	}
	public Date getCreateAt() {
		return createAt;
	}
	
	@Enumerated(EnumType.ORDINAL)
	public Mood getMood() {
		return mood;
	}
	
	public void setId(int id) {
		this.id = id;
	}
	public void setName(String name) {
		this.name = name;
	}
	public void setEmail(String em) {
		this.email = em;
	}
	public void setAge(int age) {
		this.age = age;
	}
	public void setAdress(String adr) {
		this.adress = adr;
	}
	public void setCreateAt(Date createAt) {
		this.createAt = createAt;
	}
	public void setDateBirth(Date dateBirth) {
		this.dateBirth = dateBirth;
	}
	public void setMood(Mood mood) {
		this.mood = mood;
	}
}
```

# 4. Создание классов `Repository`  и `Service`

Точно также возьмем классы из предыдущего проекта
#### Интерфейс Репозитория
```java
package ru.learn.RestApi.FirstRestApp.Repository;

import org.springframework.data.jpa.repository.JpaRepository;
import org.springframework.stereotype.Repository;

import ru.learn.RestApi.FirstRestApp.Model.Person;

@Repository
public interface PersonRepository extends JpaRepository<Person, Integer>{
}
```
#### Класс Сервис
```java
public class PersonService {
	private final PersonRepository personRepository;

	@Autowired
	public PersonService(PersonRepository p) {
		this.personRepository = null;
	}
	
	public List<Person> findAll(){
		return personRepository.findAll();
	}
	
	public Person findOne(int id) {
		Optional<Person> optional = personRepository.findById(id);
		return optional.orElse(null);
	}
	
	@Transactional
	public void save(Person person) {
		person.setCreateAt(new Date());
		person.setMood(Mood.HAPPY);
		personRepository.save(person);
	}
	
	@Transactional
	public void update(int id, Person person) {
		person.setId(id);
		person.setCreateAt(new Date());
		personRepository.save(person);
	}
	
	@Transactional
	public void delete(int id) {
		personRepository.deleteById(id);
	}
}
```

# 5. Создание контроллера

Теперь создадим контроллер, который будет получать данные из БД и возвращать их клиенту в виде JSON файла

## 1. Теория
> [!NOTE] Аннотация `@ResponseBody` на методе
> Данная аннотация говорит, что возвращаемое значение ДОЛЖНО быть конвертировано в ФОРМАТ JSON
> 
> Если метод возвращает `String` - строка будет конвертирована в данные (JSON)
> Если метод возвращает другой объект - данные JSON

Так например, если мы пометим этой аннотацией метод, ==Spring не будет искать представление с этим именем, а вернет данные==
```java
@Controller
@RequestMapping("/rest")
public class RestContoller {
	
	@ResponseBody
	@GetMapping()
	public String welcome() {
		return "Hello to rest Controller";
	}
}
```

 ***
 
> [!NOTE] Аннотация `@RestController`
> Данная аннотация ПРИМЕНЯЕТСЯ НА уровне КЛАССА.
> 
> С ее помощью ВСЕ методы класса, будут возвращать ДАННЫЕ (все методы будут автоматические считаться методами с аннотацией `@ResponseBody`)

```java
@RestController
@RequestMapping("/rest")
public class RestContoller {
	
	@GetMapping()
	public String welcome() {
		return "Hello to rest Controller";
	}
}
```

## 2. Практика
Теперь сделаем так, чтобы при разных запросах наш контроллер отправлял данные в формате JSON

```java
@RestController
@RequestMapping("/rest")
public class RestContoller {
	private final PersonService service;
	
	@Autowired
	public RestContoller(PersonService s) {
		this.service = s;
	}
	
	@GetMapping("/all")
	public List<Person> findAll(){
		return service.findAll();
	}
}
```

Теперь по запросы по адресу `localhost:8080/rest/all` получим такую страницу

# 6. Обработка исключения
Однако, если попытаться получить данные по id, которого нет в БД, вернется пустой объект. 

Для вывода сообщения о том, что человек с таким id не существует, необходимо СОЗДАТЬ КЛАСС, который будет возвращен в качестве сообщения об ошибке

## 1. Создадим объект "ошибка"
Создадим объект, который будет возвращаться клиенту при неверном запросе к БД
```java
public class PersonErrorResponse {
	private String message;
	
	public PersonErrorResponse(String mess) {
		this.message = mess;
	}
	
	public void setMessage(String message) {
		this.message = message;
	}
	
	public String getMessage() {
		return message;
	}
}
```
## 2. Создание исключения
Для того чтобы приложение могло обрабатывать именно это исключение (человек не найден) необходимо создать это исключение
```java
public class PersonNotFoundException extends RuntimeException{
}
```

## 3. Создание ОБРАБОТЧИКА исключения
Для того чтобы исключение было выброшено, ==изменим класс== `PersonService`, чтобы в случае получения пустого объекта из БД, метод вызывал исключение
```java
public Person findOne(int id) {
	Optional<Person> optional = personRepository.findById(id);
	return optional.orElseThrow(PersonNotFoundException::new);
}
```

Теперь в контроллере, когда пользователь будет пытаться получить человека по неверному `id`, вызовется метод сервиса, который в случае неудачного поиска записи в БД, вызовет исключение

Для того чтобы поймать вызванное исключение используется аннотация
> [!NOTE] Аннотация `@ExceptionHandler`
> Аннотация `@ExceptionHandler` в **Spring** используется для обработки исключений в контроллерах. Она позволяет задать метод, который будет автоматически вызываться при возникновении определенных исключений.

```java
@RestController
@RequestMapping("/rest")
public class RestContoller {
	
	. . .
		
	@GetMapping("/{id}")
	public Person findOne(@PathVariable("id") int id) {
		return service.findOne(id);
	}
	
	@ExceptionHandler
	private ResponseEntity<PersonErrorResponse> personNotFoundHandler(PersonNotFoundException e) 
	{
		PersonErrorResponse personError = new PersonErrorResponse(
			"Person with this id wasn`t found"
		);
		return new ResponseEntity<PersonErrorResponse>(personError, HttpStatus.NOT_FOUND);
	}
}
```

Теперь при вводе id, которого нет в БД, метод `findOne()` в классе `PersonService` вызовет исключение `PersonNotFoundException`
Это исключение будет ПОЙМАНО методом `personNotFoundHeandler()` при помощи аннотации `@ExceptionHeandler`
