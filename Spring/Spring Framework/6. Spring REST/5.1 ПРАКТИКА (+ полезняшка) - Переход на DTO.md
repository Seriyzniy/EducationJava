# 1. Создание класса DTO
Поскольку наш DTO объект это тот же класс Person, только с определенными полями, то создадим класс с именем `PersonDTO`. Так же создадим отдельный пакет для класса

Класс DTO будет содержать поля:
- имя
- возраст
- емаил
- дата рождения

Также будем проверять поля на валидность
```java
public class PersonDTO {
	@NotEmpty(message = "Name can`t not be empty!")
	@Size(min = 2, max = 30, message = "Name can be length between 2 and 30 character!")
	private String name;
	
	@NotEmpty(message = "Email can`t not be empty!")
	@Email(message = "Email should be valid format!")
	private String email;

	@Min(value = 0, message = "Age can`t be negative value!")
	private int age;

	@Pattern(regexp = "[A-Z]\\w+, [A-Z]\\w+, \\d{6}", message = "Format will be 'Country, City, Index(6 numbers)'")
	private String adress;

	@Temporal(TemporalType.DATE)
	@DateTimeFormat(pattern = "yyyy-MM-dd")
	private Date dateBirth;

	public void setName(String name) {
		this.name = name;
	}
	public String getName() {
		return name;
	}
	
	public void setAge(int age) {
		this.age = age;
	}
	public int getAge() {
		return age;
	}
	public void setEmail(String email) {
		this.email = email;
	}
	public String getEmail() {
		return email;
	}
	public void setDateBirth(Date dateBirth) {
		this.dateBirth = dateBirth;
	}
	public Date getDateBirth() {
		return dateBirth;
	}
	public void setAdress(String adress) {
		this.adress = adress;
	}
	public String getAdress() {
		return adress;
	}
}
```

# 2. Переход на DTO

## 1. Создание конверторов `PersonDTO` в `Person` и наоборот
Поскольку `PersonService`  при помощи `PersonRepository` получает данные из БД и формирует из них объекты класса `Person`, то для перехода на `PersonDTO` необходимо написать конвертор, который будет преобразовывать объекты в `PersonDTO`

Методы в контроллере
```java
private PersonDTO convertToPersonDTO(Person person) {
		PersonDTO personDTO = new PersonDTO();
		
		personDTO.setName(person.getName());
		personDTO.setAge(person.getAge());
		personDTO.setEmail(person.getEmail());
		personDTO.setAdress(person.getAdress());
		personDTO.setDateBirth(person.getDateBirth());
		
		return personDTO;
	}
```

(Для этого еще добавили в конструктор класса Person параметр День Рождения)
```java
private Person convertToPerson(PersonDTO personDTO) {
		return new Person(
				personDTO.getName(),
				personDTO.getAge(),
				personDTO.getEmail(),
				personDTO.getAdress(),
				personDTO.getDateBirth());	
	}
```

> [!NOTE] Класс `ModelMapper`
> Однако такая реализация не удобная, поскольку если у нас будет МНОГО полей, то вот так вручную конвертировать каждое поле будет не удобно
> 
> Поэтому будем использовать готовое решение - класс `ModelMapper`

Для этого подключим зависимость
```xml
<dependency>
    <groupId>org.modelmapper</groupId>
    <artifactId>modelmapper</artifactId>
    <version>3.2.1</version>
</dependency>
```

Теперь чтобы не создавать объект этого класса, заставим Spring внедрять его за нас. Для этого создадим Bean этого класса в классе конфигураторе

```java
@SpringBootApplication
public class FirstRestAppApplication {

	public static void main(String[] args) {
		SpringApplication.run(FirstRestAppApplication.class, args);
	}
	
	@Bean
	public ModelMapper modelMapper() {
		return new ModelMapper();
	}
}
```

Теперь внедрим его в КОНТРОЛЛЕР
```java
@RestController
@RequestMapping("/rest")
public class RestContoller {
	private final PersonService service;
	private final ModelMapper modelMapper;
	
	@Autowired
	public RestContoller(PersonService s, ModelMapper modelMapper) {
		this.service = s;
		this.modelMapper = modelMapper;
	}
	. . .
```

#### Методы конвертации
Теперь мы можем переписать методы конвертации `PersonDTO` в `Person`
В методе `map()` в 1-ом параметре указываем КОГО хотим преобразовать, а 2-ым В КАКОЙ КЛАСС
```java
private Person convertToPerson(PersonDTO personDTO) {
	return modelMapper.map(personDTO, Person.class);
}
	
private PersonDTO convertToPersonDTO(Person person) {
	return modelMapper.map(person, PersonDTO.class);
}
```

### 2. Изменение метода получения данных ВСЕХ пользователей
Ранее метод, при получении GET запроса, получал данные из БД и возвращал их клиенту
```java
@GetMapping("/all")
public List<Person> getPeople(){
	return service.findAll();
}
```

Теперь же нам нужно чтобы этот метод возвращал список `List<PersonDTO>`. Для этого используем Stream API
```java
@GetMapping("/all")
public List<PersonDTO> getPeople(){
	return service.findAll().stream()
		.map(p -> convertToPersonDTO(p))
		.collect(Collectors.toList());
}
```

### 3. Изменение метода получения данных ОДНОГО пользователя
```java
@GetMapping("/{id}")
public Person getPerson(@PathVariable("id") int id) {
	return service.findOne(id);
}
```

Теперь сделаем чтобы возвращал ОДИН объект `PersonDTO`
```java
@GetMapping("/{id}")
public PersonDTO getPerson(@PathVariable("id") int id) {
	return convertToPersonDTO(service.findOne(id));
}
```

***
#### 4. Изменение метода ДОБАВЛЕНИЯ пользователя
Изменили параметр `Person` на `PersonDTO`, и конвертировали объект перед сохранением
```java
@PostMapping
	public ResponseEntity<HttpStatus> savePerson(@RequestBody @Valid PersonDTO personDto, BindingResult bindingResult) {
		
		if(bindingResult.hasErrors()) {
			
			StringBuilder errorMsg = new StringBuilder();
			List<FieldError> errors = bindingResult.getFieldErrors();
			
			for(FieldError e : errors) {
				errorMsg.append(e.getField()).append("-").append(e.getDefaultMessage());
			}
			throw new PersonCreateException(errorMsg.toString());
		};
		service.save(convertToPerson(personDto));
		return ResponseEntity.ok(HttpStatus.OK);
	}
```
