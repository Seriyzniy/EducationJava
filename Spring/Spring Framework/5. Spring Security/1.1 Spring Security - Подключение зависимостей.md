Для работы с Spring Security необходимо подключить зависимости:
#### Spring-Security-Core -  Базовый функционал безопасности
```xml
<dependency>
    <groupId>org.springframework.security</groupId>
    <artifactId>spring-security-core</artifactId>
    <version>6.2.1</version>
</dependency>
```
🔹 **Что содержит?**
- Основные классы и интерфейсы Spring Security.
- Механизмы аутентификации и авторизации.
- Объекты `UserDetails`, `GrantedAuthority`, `Authentication`, `SecurityContext`.
- Алгоритмы шифрования паролей (`BCryptPasswordEncoder`, `Pbkdf2PasswordEncoder` и др.).
- Инструменты управления сессиями и токенами.

🔹 **Пример:**
- Определение пользователя:
```java
UserDetails user = User.withUsername("user")
        .password("password")
        .roles("USER")
        .build();
```

- Создание `UserDetailsService`:
```java
@Bean
public UserDetailsService userDetailsService() {
    return new InMemoryUserDetailsManager(user);
}
```

#### Spring-Security-Web - Фильтры и интеграция с веб-приложением
```xml
<dependency>
    <groupId>org.springframework.security</groupId>
    <artifactId>spring-security-config</artifactId>
    <version>6.2.1</version>
</dependency>
```
🔹 **Что содержит?**
- Фильтры безопасности, включая `SecurityFilterChain`, `UsernamePasswordAuthenticationFilter`.
- Обработку запросов через `HttpSecurity`.
- Поддержку CSRF, CORS, защиты от XSS.
- Классы для работы с аутентификацией через сессию, cookies и headers.

🔹 **Пример:**
- Настройка фильтров в Spring Security:
```java
@Bean
public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
    http
        .authorizeHttpRequests(auth -> auth.anyRequest().authenticated())
        .formLogin(login -> login.loginPage("/login").permitAll())
        .logout(logout -> logout.permitAll());
    return http.build();
}
```
***
#### Spring-Security-Config – Позволяет настраивать безопасность
```xml
<dependency>
    <groupId>org.springframework.security</groupId>
    <artifactId>spring-security-web</artifactId>
    <version>6.2.1</version>
</dependency>
```
🔹 **Что содержит?**
- Классы и аннотации для Java-конфигурации (`@EnableWebSecurity`, `@EnableGlobalMethodSecurity`).
- Поддержку конфигурации через XML (`<http>`, `<authentication-manager>`).
- Упрощение настройки безопасности (без него придется вручную прописывать `SecurityFilterChain`).

🔹 **Пример:**
- Подключение Java-конфигурации:
```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    // Код настройки безопасности
}
```
