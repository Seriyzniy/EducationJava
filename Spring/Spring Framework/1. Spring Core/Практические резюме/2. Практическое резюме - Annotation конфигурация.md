Чтобы НЕ конфигурировать бины вручную в файле XML, можно использовать конфигурацию XML + Annotation
# 1. Проверка подключения ПРОСТРАНСТВА имен context
Чтобы `context:component-scan` работал, НУЖНО чтобы в `applicationContext.xml` было:

1. Пространство имён `context`:
```xml
xmlns:context="http://www.springframework.org/schema/context"
```

2. **Актуальная ссылка в `xsi:schemaLocation`**:
```xml
xsi:schemaLocation="http://www.springframework.org/schema/beans
	http://www.springframework.org/schema/context
	http://www.springframework.org/schema/context/spring-context.xsd">
```

```xml
<beans  xmlns="http://www.springframework.org/schema/beans"
        xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
        xmlns:context="http://www.springframework.org/schema/context"
        xsi:schemaLocation="http://www.springframework.org/schema/beans
        http://www.springframework.org/schema/beans/spring-beans.xsd
        http://www.springframework.org/schema/context
        http://www.springframework.org/schema/context/spring-context.xsd">

</beans>
```

***
# 2. Использование `Component-Scan`
Чтобы использовать аннотации над классами, чтобы Spring их обнаруживал, необходимо в файле конфигурации .xml:

==(Сразу же после ШАПКИ xml)==
```xml
<context:component-scan base-package="ru.springcore.tasks.springCore.task1"/>
```

Теперь файл конфигурации applicationContext.xml:
```xml
<?xml version="1.0" encoding="UTF-8"?>
<beans  xmlns="http://www.springframework.org/schema/beans"
        xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
        xmlns:context="http://www.springframework.org/schema/context"
        xsi:schemaLocation="http://www.springframework.org/schema/beans
        http://www.springframework.org/schema/beans/spring-beans.xsd
        http://www.springframework.org/schema/context
        http://www.springframework.org/schema/context/spring-context.xsd">
	
	<context:component-scan base-package="ru.springcore.tasks.springCore.task1"/>

</beans>
```

# 3. ЕСЛИ СТАРАЯ ВЕРСИЯ Spring 3.0 и ниже
==Для успешного сканирования классов==, необходимо использовать следующий тег

**`<context:annotation-config />`**:
- Этот элемент активирует поддержку аннотаций, таких как `@Autowired`, `@Controller`, `@Service`, и т.д., и позволяет Spring обрабатывать их в бинах, объявленных в конфигурации.

***
# 4. Использование Аннотаций над классами
```java
package ru.springcore.tasks.springCore.task1;

import org.springframework.stereotype.Component;

@Component
public class MessageService {
	public String getMessage() { 
		return "Messge Service";
	}
}

@Component
public class UserService {
	private final MessageService messageService;
	
	public UserService(MessageService msgService) {
		this.messageService = msgService;
	}
	
	public void printMessage() {
		System.out.println("User service " + messageService.getMessage());
	}
}
```

***
# 5. Запуск контекста с XML+Annotation конфигурацией
```java
public class App 
{
    public static void main( String[] args )
    {
        ClassPathXmlApplicationContext context = new ClassPathXmlApplicationContext("applicationContext.xml");
        
        UserService user = context.getBean("userService",UserService.class);
        
        user.printMessage();
        
        context.close();
    }
}
```

Также можно использовать такую конструкцию
```java
ApplicationContext context = new ClassPathXmlApplicationContext("applicationContext.xml");
```
