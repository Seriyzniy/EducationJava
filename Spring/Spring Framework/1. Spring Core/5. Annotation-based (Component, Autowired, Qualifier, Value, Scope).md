Java-Аннотация - специальный тип комментариев, позволяющих:
- Передавать какие-либо инструкции Java-компилятору (@Override)
	
- Передавать какие-либо инструкции для анализатора исходного кода
	
- ==Передавать метаданные==, которые могут быть использованы 
	- либо в Java-приложении (с помощью рефлексии)
	- либо другим приложениям или фреймворкам (Spring)

# Преимущества аннотаций
- Короче чем XML-конфигурация
	
- Удобнее чем XML-конфигурация
	
- Лучшая читабельность

# Как работает конфигурация с аннотациями

Spring сканирует все классы, находит классы с аннотациями и создает бины из этих классов

Вместо создания бина в XML используется аннотация `@Component`
```java
@Component
public class ClassicMusic implements GenreMusic{
	@Override
	public String plaing() {
		return "Classic Music";
	}
}
```

Для того чтобы включить сканирование классов, необходимо в файле XML (где ранее конфигурировались Beans), удалить ранее созданные bean и написать одну строчку
```xml
 <?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
	xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
	xmlns:context="http://www.springframework.org/schema/context"
	xsi:schemaLocation="http://www.springframework.org/schema/beans
	http://www.springframework.org/schema/beans/spring-beans.xsd
	http://www.springframework.org/schema/context
	http://www.springframework.org/schema/context/spring-context.xsd">

	<context:component-scan base-package="ru.studyspring.myproject"/>

</beans>
```

Данная строчка включает сканирование классов
```xml
<context:component-scan base-package="ru.studyspring.myproject"/>
```

Теперь используем эту аннотацию
```java
package ru.studyspring.myproject;

import org.springframework.stereotype.Component; //Импорт аннотации

@Component("SomeClassMusic") //Использование аннотации
public class ClassicMusic implements GenreMusic{
	@Override
	public String plaing() {
		return "Classic Music";
	}
}
```

```java
public class ApplicationRunner {
	public static void main(String[] args) {
		ClassPathXmlApplicationContext appContext = new ClassPathXmlApplicationContext("applicationcontext.xml");
	
	GenreMusic gmusic = appContext.getBean("SomeClassMusic", ClassicMusic.class);
	MusicPlayer music = new MusicPlayer(gmusic);
	music.PlaingMusic();
	
	appContext.close();
	}
}
```
Нам также необходимо создать контекст приложения используя XML файл, но теперь в этом файле одна строчка.

Поскольку класс ClassMusic содержит аннотацию `@Component` Spring создаст бин этого класса.
# @Component

Данной аннотацией помечаем класс, который должен стать Bean. Spring так и сделает.

Именно эту аннотацию ищет Spring когда сканирует все классы

> [!NOTE] Примичание!
> Также можно сразу же после аннотации в круглых скобках указать `id` класса, если же этого не делать, то создастся Bean с `id` равным названию класса с маленькой буквы

# @Autowired

Данная аннотация позволяет ВНЕДРЯТЬ Bean АВТОМАТИЧЕСКИИ. То есть Spring сам будет внедрять зависимости

```java
@Component("player")
public class MusicPlayer {
	private GenreMusic music;
	
	@Autowired
	public MusicPlayer(GenreMusic gmusic) {
		this.music = gmusic;
	}
	public void PlaingMusic() {
		System.out.println("Playing: " + music.plaing());
	}
}
```

В данном примере в бин musicPlayer (@Component создаст бин с ID равным названию класса написанного с маленькой буквы) необходимо внедрить бин, реализующий интерфейс GenreMusic

Spring сканирует все классы с аннотацией `@Component` и создает для них bean

> [!NOTE] !
>  Spring сканирует все созданные бины и проверяет, подходит ли хотя бы один бин в качестве зависимости там, где есть аннотация `@Autowired`
>  
>  Если находится один такой бин, то он внедряется в качестве зависимости
>  Если не находится ни одного - ошибка
>  Если находится больше чем один - неоднозначность

Аннотацию `@Autowired` можно использовать на:
- полях
	 Внедряет даже в поле `private`, даже если нет конструктора / set-метода. Это аннотация делает с помощью Рефлексии
- set-методах
- конструкторах

#### @Autowired в конструкторе
```java
package ru.studyspring.myproject;

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Component;

@Component
public class MusicPlayer {
	private ClassicMusic music;
	@Autowired
	public MusicPlayer(ClassicMusic cmusic) {
		this.music = cmusic;
	}
	public void PlaingMusic() { System.out.println("Playing: " + music.plaing()); }
}
```

С помощью аннотации `@Autowired` внедрим зависимость `ClassicMusic`  в класс `MusicPlayer` и получим этот бин из контекста
```java
package ru.studyspring.myproject;

import org.springframework.context.support.ClassPathXmlApplicationContext;

public class ApplicationRunner {
	public static void main(String[] args) {
		ClassPathXmlApplicationContext appContext = new СlassPathXmlApplicationContext("applicationcontext.xml");

	MusicPlayer music = appContext.getBean("musicPlayer", MusicPlayer.class);
	music.PlaingMusic();
	appContext.close();
	}
}
```

#### @Autowired в методе
В данном случае аннотацией `@Autowired` помечен метод set (имя метода может быть любым), а конструктора вообще нет. Но программа все так же работает
```java
package ru.studyspring.myproject;

import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Component;

@Component
public class MusicPlayer {
	private GenreMusic music;
	@Autowired
	public void setMusic(GenreMusic music) {
		this.music = music;
	}
	public void PlaingMusic() { System.out.println("Playing: " + music.plaing()); }
}
```
#### @Autowired в поле
```java
@Component
public class MusicPlayer {
	@Autowired
	private GenreMusic music;
	public void PlaingMusic() { System.out.println("Playing: " + music.plaing()); }
}
```
Программа выполняется так же как и в предыдущих 2-х случаях

# @Qualifier
При внедрении зависимостей с помощью аннотации `@Autowired` могла возникнуть следующая ошибка:
![[Pasted image 20241021141534.png]]

==Данная ошибка возникала когда несколько бинов подходят для автоматического внедрения зависимости.==

> [!NOTE] @Qualifier
> Qualifier (уточнитель) - указывает на `id` бина, который мы хотим внедрить

```java
@Component
public class MusicPlayer {
	@Autowired
	@Qualifier("rockMusic")
	private GenreMusic music;
	
	public void PlaingMusic() { System.out.println("Playing: " + music.plaing()); }
}
```

Аннотацию `@Qualifier` можно использовать на:
- Конструкторах
- Set-методах
- Полях

#### @Qualifier в конструкторах 
```java
@Autowired
public MusicPlayer(@Qualifier("rockMusic") GenreMusic music) {
	this.music = music;
}
```
Аннотацию необходимо использовать рядом с аргументами конструктора
