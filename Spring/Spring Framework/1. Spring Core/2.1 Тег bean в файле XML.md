# **1. Атрибуты тега `<bean>`**

|Атрибут|Описание|
|---|---|
|`id`|Уникальный идентификатор бина (используется для получения бина из контекста).|
|`name`|Альтернативное имя бина (может быть несколько имен через запятую или пробел).|
|`class`|Полное имя Java-класса, который используется для создания бина.|
|`scope`|Определяет область видимости (`singleton`, `prototype`, `request`, `session`, `application`, `websocket`).|
|`lazy-init`|Если `true`, бин создаётся только при первом запросе, а не сразу при старте контекста.|
|`init-method`|Имя метода, который вызывается после создания бина (для инициализации).|
|`destroy-method`|Имя метода, который вызывается перед уничтожением бина (для освобождения ресурсов).|
|`autowire`|Автоматическое связывание зависимостей (`byType`, `byName`, `constructor`, `no`).|
|`depends-on`|Указывает, что этот бин зависит от другого бина (тот должен быть создан раньше).|
|`factory-method`|Если класс создаётся не через конструктор, а через статический метод-фабрику.|
|`factory-bean`|Если бин создаётся через другой бин-фабрику.|
***
## **1.1 Зависящие бины (`depends-on`)**
Говорит Spring, что перед этим бином должны быть загружены другие бины.
```xml
<bean id="serviceA" class="com.example.ServiceA" depends-on="serviceB"/>
<bean id="serviceB" class="com.example.ServiceB"/>
```
***
## **1.2 Область видимости (`scope`)**

Определяет, сколько экземпляров бина создаётся:

|`scope`|Описание|
|---|---|
|`singleton`|Один объект на весь контекст (по умолчанию).|
|`prototype`|Новый объект создаётся при каждом запросе.|
|`request`|Новый объект для каждого HTTP-запроса (Spring Web).|
|`session`|Новый объект на каждую HTTP-сессию (Spring Web).|
|`application`|Один объект на всё веб-приложение (Spring Web).|
|`websocket`|Один объект на WebSocket-сессию.|
```xml
<bean id="myService" class="com.example.MyService" scope="prototype"/>
```
***
## **1.3 Ленивый запуск (`lazy-init`)**
Если `lazy-init="true"`, ==**бин создаётся только при первом обращении к нему.**==
```xml
<bean id="heavyBean" class="com.example.HeavyBean" lazy-init="true"/>
```
***
## **1.4 Инициализация и уничтожение (`init-method`, `destroy-method`)**

Spring может вызвать методы при создании и перед уничтожением бина.
```java
public class MyService {
    public void init() {
        System.out.println("Бин инициализирован!");
    }

    public void destroy() {
        System.out.println("Бин уничтожен!");
    }
}
```

```xml
<bean id="myService" class="com.example.MyService" init-method="init" destroy-method="destroy"/>
```
***
## **1.5 Фабричные бины (`factory-method` и `factory-bean`)**

Если объект создаётся через ==статический метод-фабрику==:
```xml
<bean id="myBean" class="com.example.MyFactory" factory-method="createInstance"/>
```

Если объект создаётся ==через метод экземпляра другого бина==:
```xml
<bean id="factory" class="com.example.BeanFactory"/>
<bean id="myBean" factory-bean="factory" factory-method="createBean"/>
```

# **2. ВЛОЖЕННЫЕ теги для `<bean>`**
***
## **2.1 Внедрение зависимостей (DI)**

##### **а) Внедрение через сеттер (`<property>`)**

Используется, если в классе есть `set`-метод.
```xml
<bean id="userService" class="com.example.UserService">
    <property name="messageService" ref="messageService"/>
</bean>

<bean id="messageService" class="com.example.MessageService"/>
```
- `name="messageService"` — имя сеттера `setMessageService()`.
- `ref="messageService"` — ссылка на другой бин.
##### **б) Внедрение через конструктор (`<constructor-arg>`)**

Используется, если зависимость передаётся через конструктор.
```xml
<bean id="userService" class="com.example.UserService">
    <constructor-arg ref="messageService"/>
</bean>
```

Если у конструктора несколько параметров, можно уточнить их с `index` или `type`:
```xml
<bean id="userService" class="com.example.UserService">
    <constructor-arg index="0" ref="messageService"/>
    <constructor-arg index="1" value="Admin"/>
</bean>
```

***
## **2.2 Внедрение простых значений (`value`)**

Простые типы (`String`, `int`, `boolean`, `double` и т. д.) можно передавать через `<value>`:
```xml
<bean id="databaseConfig" class="com.example.DatabaseConfig">
    <property name="url" value="jdbc:mysql://localhost:3306/mydb"/>
    <property name="username" value="root"/>
    <property name="password" value="1234"/>
</bean>
```

***
## **2.3 Внедрение коллекций (List, Set, Map, Properties)**

Если у класса есть поле типа `List`, `Set`, `Map` или `Properties`, можно передавать значения так:
#### **а) `List`**
```xml
<bean id="userService" class="com.example.UserService">
    <property name="users">
        <list>
            <value>John</value>
            <value>Jane</value>
        </list>
    </property>
</bean>
```
##### б) `Set`
```xml
<property name="roles">
    <set>
        <value>ADMIN</value>
        <value>USER</value>
    </set>
</property>
```
##### в) `Map`
```xml
<property name="settings">
    <map>
        <entry key="timeout" value="30"/>
        <entry key="theme" value="dark"/>
    </map>
</property>
```
##### г) `Properties`
```xml
<property name="config">
    <props>
        <prop key="url">http://example.com</prop>
        <prop key="port">8080</prop>
    </props>
</property>
```
