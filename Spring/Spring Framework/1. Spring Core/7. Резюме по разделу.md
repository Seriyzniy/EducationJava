# Dependency Injection

==Dependency Injection== - одна из реализаций IoC, посредством которой созданием объекта и компоновкой его зависимостей занимается другой объект (фреймворк)

При создании классов могут возникать зависимости, когда один класс использует функционал другого класса
![[Img_Example1.png]]
## Способы внедрения зависимостей
 
<p style="text-align: center;"> 
<span style="font-size: 22px;">Конструктор</span> 
</p>
 ==**XML based**==
```xml
	<bean id="musicBean"
		class="ru.tudyspring.myproject.ClassicMusic">
	</bean>
	
	<bean id="musicPlayer"
		class="ru.studyspring.myproject.MusicPlayer">
		<constructor-arg ref="musicBean"/>
	</bean>
```

==**Annotation based** (+ XML based)==
Данная строчка включает сканирование классов
```xml
<context:component-scan base-package="ru.studyspring.myproject"/>
```

```java
@Component("player")
public class MusicPlayer {
	private GenreMusic music;
	
	@Autowired
	public MusicPlayer(GenreMusic gmusic) {
		this.music = gmusic;
	}
	public void PlaingMusic() {
		System.out.println("Playing: " + music.plaing());
	}
}
```

==**Java based** (+ Annotation based)==
```java
@Configuration
@ComponentScan("ru.studyspring.myproject")
public class SpringConfig {
	@Bean
	public ClassicMusic classicMusic() {
		return new ClassicMusic();
	}
	@Bean
	public MusicPlayer musicPlayer() {
		return new MusicPlayer(classicMusic());
	}
}
```
Можно в классах помеченных `@Component` использовать аннотацию `@Autowired`

<p style="text-align: center;"> 
<span style="font-size: 22px;">Set-методы</span> 
</p>
 ==**XML based**==
Этот способ использует тот же принцип что и внедрение зависимости через конструктор, но в файле XML где хранятся BeanDefinitions появляется новый тег
Необходимо создать Сеттер (set-method начинается с "set")
```java
public class MusicPlayer {
	private GenreMusic music;
	//Метод set
	public void setMusic(GenreMusic gmusic) {
		this.music = gmusic;
	}
	//Конструктор по умолчанию
	public MusicPlayer() {};
	public MusicPlayer(GenreMusic gmusic) {
		this.music = gmusic;
	}
	public void PlaingMusic() {
		System.out.println("Playing: " + music.plaing());
	}
}
```

 Затем через тег `<property` указать в параметре `name` имя set-метода без "set"
```xml
<bean id="musicBean"
	class="ru.studyspring.myproject.ClassicMusic">
</bean>

<bean id="musicPlayer"
	class="ru.studyspring.myproject.MusicPlayer">
	<property name="music" ref="musicBean"/>
</bean>
```
Если необходимо внедрить:
 - Объект - `ref`
 - Значение - `value`

==**Java based (+ Annotation based)**==
```java
// Сеттер для внедрения зависимости 
@Autowired 
public void setUserRepository(UserRepository userRepository) {
	this.userRepository = userRepository;
}
```

<p style="text-align: center;"> 
<span style="font-size: 22px;">Внешние файлы</span> </p>

Для этого необходимо создать файл с расширением `.properties` или `.yml`
==**XML Based**==
![[Pasted image 20241102193147.png]]
Необходимо ***указать путь*** к внешнему файлу

Добавить в файл .xml где хранятся Beans если используется Spring >=3.1 :
```xml
<bean id="propertyConfigurer"
	class="org.springframework.beans.factory.config.PropertyPlaceholderConfigurer">
	<property name="location" value ="classpath:myProperty.properties"/>
</bean>
```
В более старых версиях можно так (или вместо classpath указать путь к файлу)
```xml
<context:property-placeholder location="classpath:myProperty.properties"/>
```

Для чтения значения из файла `${musicPlayer.namePlayer}` 
```xml
<bean id="musicPlayer"
	class="ru.studyspring.myproject.MusicPlayer">
	<property name="music" ref="musicBean"/>
	<property name="namePlayer" value="${musicPlayer.namePlayer}"/>
</bean>
```

# Inversion Of Control

**Inversion of Control** - принцип программирования, при котором ==управление выполнением программы передается фреймворку==, а не программисту.

Происходит потеря контроля над выполнением кода: фреймворк управляет кодом программиста, а не программист управляет фреймворком

Inversion of Control Spring подразумевает Dependency Injection, то есть фреймворк управляет внедрением зависимостей

### Структура Spring приложения
![[Img_SpringApp.png]]

**IoC Container** - объект, который занимается созданием (==***через Reflection API***==) других объектов и внедрением в них зависимостей (DI).
Контейнер представлен в виде ассоциативного массива (Map). Где ключом является String (имя класса), а значение объект ==Bean==

Реализует один из двух интерфейсов:
	 - BeanFactory
	 - ==ApplicationContext ==(наиболее используемый, реализует интерфейс BeanFactory)

**_Bean_** - объект со всеми необходимыми зависимостями, который был создан с помощью IoC Container (Controller, Service, Repository)

Metadata - метаинформация 
	- Metadata представляет собой совокупность объектов (==Bean Definitions== ). которые говорят IoC Container как создавать Beans, конфигурировать и внедрять их зависимости
	- Bean - объект
	- Bean Definition - сам класс (чертеж)

### Жизненный цикл бина

> [!NOTE] Жизненный цикл 
> Когда Spring запускается, он создает экземпляр `ApplicationContext`, который служит корневым контейнером для вашего приложения.
> 
> Далее используя один из способов конфигурации (XML, Annotation, Java) мы предоставляем фреймворку информацию о бинах

Есть несколько типов ==Scope==
> [!Singleton]
> По умолчанию создается ОДИН bean-объект (создается до вызова метода getBean())

> [! Prototype] 
> Prototype каждый раз создает новый bean при вызове метода getBean()

# XML-based

Для реализации XML-based ==необходимо создать файл XML== в папке src/main/resources. 
Именно его будет читать IoC Container для внедрения зависимостей.
```xml
<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
		xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
		xmlns:context="http://www.springframework.org/schema/context"
		xsi:schemaLocation="http://www.springframework.org/schema/beans
		http://www.springframework.org/schema/beans/spring-beans.xsd
		http://www.springframework.org/schema/context
		http://www.springframework.org/schema/context/spring-context.xsd">
		
	<bean id="musicBean"
		class="ru.tudyspring.myproject.ClassicMusic">
	</bean>
</beans>
```

> [!NOTE] IoC контейнер
> ```java
ClassPathXmlApplicationContext myContext =
new ClassPathXmlApplicationContext("applicationcontext.xml");
> ```
Здесь с помощью класса ClassPathXmlApplicationContext мы создаем IoC контейнер и указываем ему где лежат BeanDefinitions, необходимые для создания бинов.

# Annotation-based

> [!NOTE] ВАЖНО
> Данный способ использует XML (для включения сканирования компонентов) и Аннотации

Для того чтобы включить сканирование классов, необходимо в файле XML (где ранее конфигурировались Beans), удалить ранее созданные bean и написать одну строчку
```xml
 <?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
	xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
	xmlns:context="http://www.springframework.org/schema/context"
	xsi:schemaLocation="http://www.springframework.org/schema/beans
	http://www.springframework.org/schema/beans/spring-beans.xsd
	http://www.springframework.org/schema/context
	http://www.springframework.org/schema/context/spring-context.xsd">

	<context:component-scan base-package="ru.studyspring.myproject"/>

</beans>
```

Данная строчка включает сканирование классов
```xml
<context:component-scan base-package="ru.studyspring.myproject"/>
```

==Классы==, из которых необходимо ==сделать Bean==, помечаются ==аннотацией== `@Component`

> [!NOTE] IoC контейнер
>Такой же как и в XML-based

# Java-based
> [!NOTE] ВАЖНО
> Данный способ использует Java-код и Аннотации

```java
package ru.studyspring.myproject;

import org.springframework.context.annotation.Configuration;

@Configuration
public class SpringConfig {
}
```

==Пустой класс с аннотацией== `@Configuration` ==равен по функциональности== пустому ==XML файлу==
```xml
<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
	xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
	xmlns:context="http://www.springframework.org/schema/context"
	xsi:schemaLocation="http://www.springframework.org/schema/beans
	http://www.springframework.org/schema/beans/spring-beans.xsd
	http://www.springframework.org/schema/context
	http://www.springframework.org/schema/context/spring-context.xsd">
</beans>
```
### Аннотации для тегов
Для ==каждого тега== в xml файле ==есть аннотация==
Для этого:
```xml
<context:component-scan base-package="ru.studyspring.myproject"/>
```
Существует это:
```java
@Configuration
@ComponentScan("ru.studyspring.myproject") //Аналог тега <context:component-scan>
public class SpringConfig {
}
```

==! ! !== Также для тега `<bean>` существует аннотация
```xml
<bean id="musicPlayer"
	  class="ru.studyspring.myproject.MusicPlayer"/>
</bean>
```
Заменяется на:
```java
@Configuration
@ComponentScan("ru.studyspring.myproject")
public class SpringConfig {
	@Bean                              // Аннотация @Bean
	public MusicPlayer musicPlayer() {
		return new MusicPlayer();
	}
}
```

> [!NOTE] IoC контейнер
> ```java
AnnotationConfigApplicationContext context = new  AnnotationConfigApplicationContext(SpringConfig.class);
> ```
Здесь с помощью класса ClassPathXmlApplicationContext мы создаем IoC контейнер и указываем ему где лежат BeanDefinitions, необходимые для создания бинов.
