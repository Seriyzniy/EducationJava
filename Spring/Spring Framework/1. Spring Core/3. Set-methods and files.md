Как говорилось в предыдущей главе, внедрять зависимость можно через set-методы и выгружать данные из определенного файла

## 1. Set-methods
Этот способ использует тот же принцип что и внедрение зависимости через конструктор, но в файле XML где хранятся BeanDefinitions появляется новый тег

```java
package ru.studyspring.myproject;

public class MusicPlayer {
	private GenreMusic music;
	//Метод set
	public void setMusic(GenreMusic gmusic) {
		this.music = gmusic;
	}
	//Конструктор по умолчанию
	public MusicPlayer() {};
	public MusicPlayer(GenreMusic gmusic) {
		this.music = gmusic;
	}
	public void PlaingMusic() {
		System.out.println("Playing: " + music.plaing());
	}
}
```

Теперь опишем зависимость в applicationcontext.xml
```xml
<bean id="musicBean"
	class="ru.studyspring.myproject.ClassicMusic">
</bean>

<bean id="musicPlayer"
	class="ru.studyspring.myproject.MusicPlayer">
	<property name="music" ref="musicBean"/>
</bean>
```

Для того что бы внедрять зависимости через методы set, необходимо:
1. Создать метод set с именем переменной, в которую будет внедряться зависимость
2. Создать конструктор по умолчанию (т.к внедрение через тег конструктора не используем)
3. Использовать тег `<property` где:
	- `name` = "имя_set_метода без set"
	- `value` = "значение строковое или числовое";

## 2. Внедрение через внешний файл

```java
package ru.studyspring.myproject;

public class MusicPlayer {
	private GenreMusic music;
	
	private String namePlayer;
	private int volume;
	
	//SET методы полей
	public void setNamePlayer(String name) {
		this.namePlayer = name;
	}
	public void setVolume(int v) {
		this.volume = v;
	}
	//Get методы
	public void getNamePlayer() {
		System.out.println(namePlayer);
	}
	public void getVolume() {
		System.out.println(volume);
	}
	
	public void setMusic(GenreMusic gmusic) {
		this.music = gmusic;
	}
	public MusicPlayer() {};
	public MusicPlayer(GenreMusic gmusic) {
		this.music = gmusic;
	}
	public void PlaingMusic() {
		System.out.println("Playing: " + music.plaing());
	}
}
```
У нас есть два поля, теперь нам нужно внедрить в них значения для этих полей через файл.
Для этого:
##### 1. Создадим файл с расширением properties в папке src/main/resources
![[Img_Result.png]]
##### 2 Добавить в файл .xml где хранятся Beans если используется Spring >=3.1 :
```xml
<bean id="propertyConfigurer"
	class="org.springframework.beans.factory.config.PropertyPlaceholderConfigurer">
	<property name="location" value ="classpath:myProperty.properties"/>
</bean>
```
В более старых версиях можно так (или вместо classpath указать путь к файлу)
```xml
<context:property-placeholder location="classpath:myProperty.properties"/>
```

Теперь необходимо изменить BeanDefinition musicPlayer:
```xml
<bean id="musicPlayer"
	class="ru.studyspring.myproject.MusicPlayer">
	<property name="music" ref="musicBean"/>
	<property name="namePlayer" value="${musicPlayer.namePlayer}"/>
</bean>
```
Для чтения переменной из файла используется конструкция ${. . .}

## 3. Внедрение в коллекции
### Map
```java
package ru.studyspring.myproject;

import java.util.Map;

public class MusicPlayer {
	private GenreMusic music;
	//Коллекция map
	private Map<String, GenreMusic> myMap;
	//Set метод
	public void setMyMap(Map<String, GenreMusic> map) {
		this.myMap = map;
	}
	//Вывод map
	public void display() {
		for(Map.Entry<String, GenreMusic> i : myMap.entrySet()) {
			System.out.println(i.getKey() + " " + i.getValue().plaing());
		}
	}
	public MusicPlayer(GenreMusic gmusic) {
		this.music = gmusic;
	}
	public void PlaingMusic() {
		System.out.println("Playing: " + music.plaing());
	}
}```

Теперь файл BeanDefiniion
```xml
<bean id="musicBean"
	class="ru.studyspring.myproject.ClassicMusic">
</bean>

<bean id="musicRock"
	class="ru.studyspring.myproject.RockMusic">
</bean>

<bean id="musicPlayer"
	class="ru.studyspring.myproject.MusicPlayer">
	<constructor-arg ref="musicBean"/>
	<property name="MyMap">
		<map>
			<entry key = "Classical" value-ref = "musicBean"/>
			<entry key = "Rock" value-ref = "musicRock"/>
		</map>
	</property>
</bean>
```
Опять, через `<property name` указывается имя метода set. Далее через тег `<map>` и внутренние теги `<entry key` указывается ключ поля и значение-ссылка на объект `value-ref` Если необходимо вставить просто значение то используется `value` 

### List
```java
public class MusicPlayer {
	private GenreMusic music;
	//Коллекция List
	private List<String> myList;
	//Set метод
	public void setMyList(List<String> list) {
		this.myList = list;
	}
	//Вывод list
	public void disp() {
		for(int i = 0; i < myList.size(); i++) {
			System.out.println(myList.get(i));
		}
	}
	
	public MusicPlayer(GenreMusic gmusic) {
		this.music = gmusic;
	}
	public void PlaingMusic() {
		System.out.println("Playing: " + music.plaing());
	}
}
```
Вот файл xml
```xml
<bean id="musicPlayer"
	class="ru.studyspring.myproject.MusicPlayer">
	<constructor-arg ref="musicBean"/>
	<property name="MyList">
		<list>
			<value>Hello</value>
			<value>World</value>
		</list>
	</property>
</bean>
```

Если вместо строки будет объект то необходимо заменить тег `<value>` на `<ref bean = бин/>`
