В зависимости от состояние блокировки объекта, его MarkWord изменяется динамически!

## 🔎 Как JVM "понимает", что сейчас в `Mark Word`?
JVM **смотрит на младшие 2–3 бита (`lock bits`)** → **интерпретирует остальную часть `Mark Word` соответствующим образом.**
##### Пример:
- Если lock bits = `01` → значит это обычный объект → читаем hashCode, age, flags.
    
- Если lock bits = `10` → значит `Mark Word` содержит указатель на `ObjectMonitor`.

***
## 🚦 Жизненный цикл Mark Word
**Создание объекта**:
- JVM инициализирует `Mark Word` со значением по умолчанию (`Unlocked`) → `01`.
	
- Хранит `hashCode` (если вычислен), `GC age`, и другие служебные биты.

**Захват блокировки (lightweight)**:
- Поток пытается захватить lock с помощью **CAS**.
	
- `Mark Word` заменяется на указатель на **Lock Record** из стека → lock bits `00`.

**Эскалация (heavyweight)**:
- Если CAS не прошёл → JVM создаёт `ObjectMonitor` в куче.
	
- `Mark Word` теперь содержит указатель на `ObjectMonitor` → lock bits `10`.

**При необходимости**:
- Если ==hashCode== раньше хранился в `Mark Word`, но место занял `ObjectMonitor`, то:
	
	- ==JVM перемещает хэш в структуру== `ObjectMonitor`.

# Практика
Чтобы убедиться в этом, создадим проект на основе Maven и загрузим зависимость JOL (Java Object Layout)
```xml
<dependency>
	<groupId>org.openjdk.jol</groupId>
	<artifactId>jol-core</artifactId>
	<version>0.17</version>
</dependency>
```

Теперь напишем код, показывающий содержимое заголовка объекта
```java
import org.openjdk.jol.info.ClassLayout;

public class App {
    public static void main(String[] args) {
    	Foobar o = new Foobar();
        System.out.println(ClassLayout.parseInstance(o).toPrintable());

        synchronized (o) {
            System.out.println(ClassLayout.parseInstance(o).toPrintable());
        }
    }
}

class Foobar {
	int x;
	boolean y;
}
```

После блока `synchronized` мы захватываем объект, в результате чего MarkWord теперь полностью содержит ссылку на объект ObjectMonitor (небольшое колдунство есть, так как мы не видим биты говорящие о типе блокировки, а видим сразу ссылку на ObjectMonitor)
