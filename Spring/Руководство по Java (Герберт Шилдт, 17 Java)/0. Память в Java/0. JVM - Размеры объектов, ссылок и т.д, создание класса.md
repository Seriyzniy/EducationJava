# 1. Из чего же состоит объект класса?
  JVM для каждого объекта хранит:  
- 🟥Заголовок объекта;
- 🔹Память для примитивных типов (поля класса);
- 🔹Память для ссылочных типов;
- 🔹Смещение/выравнивание — по сути, это несколько неиспользуемых байт, что размещаются после данных самого объекта. Это сделано для того, чтобы адрес в памяти всегда был кратным машинному слову, для ускорения чтения из памяти + уменьшения количества бит для указателя на объект + предположительно для уменьшения фрагментации памяти. Стоит также отметить, что в java размер любого объекта кратен 8 байтам!

# 🔷 2. Заголовка объекта 🔷

Заголовок объекта — это служебная информация, которую JVM использует для внутренних нужд: управления памятью, синхронизации, виртуального вызова и т.д. 

> [!NOTE] Title
> Каждый экземпляр класса содержит заголовок.
>  
> Каждый ==заголовок== для большинства JVM(Hotspot, openJVM) ==состоит из двух машинных слов==.
> 
> Если речь идет о 32-х разрядной системе, то размер заголовка — 8 байт, если речь о 64-х разрядной системе, то соответственно — 16 байт.  (32 бита это 4 байт, 4 байта - одно машинное слово)

## Структура заголовка

##### ==Маркировочное слово (mark word)== 
📌Размер: 8 байт (64 бита)  == два машинных слова в зависимости от архитектуры JVM

> [!NOTE] 🛑 ЖЕСТЬ КАК ВАЖНО 🛑
> 📌 Mark Word — это **переиспользуемое поле**, ==НЕ ИМЕЮЩАЯ фиксированной структуры.==
> 
> Его ==**содержимое** и **интерпретация**== зависят от **состояния блокировки объекта**, которое закодировано ==в его **младших битах**== (обычно 2 или 3).

##### 🔁 Типичные СОСТОЯНИЯ `Mark Word` и что в нём хранится 

|Состояние|Lock Bits|Что содержится в Mark Word|
|---|---|---|
|**Unlocked (default)**|`01`|✅ hashCode, ✅ GC age, ✅ flags|
|**Biased lock**|`101`|✅ Thread ID, epoch, biasable info|
|**Lightweight lock**|`00`|➡️ Указатель на стековой lock record|
|**Heavyweight lock**|`10`|➡️ Указатель на `ObjectMonitor` в куче|
|**Marked for GC**|спец.|Служебные данные сборщика мусора (может быть особый шаблон)|

Это битовая структура, в которой хранятся:

| Назначение                  | Описание                                                                                 |
| --------------------------- | ---------------------------------------------------------------------------------------- |
| 🔴**HashCode**              | Если объекту был вызван `hashCode()`, сюда может быть записан его хэшированный результат |
| 🔴**Бит блокировки (lock)** | Состояние блокировки при `synchronized`                                                  |
| 🔴**GC-флаги**              | Пометки для сборщика мусора                                                              |
| **Эпоха (age)**             | Для "поколенческой" сборки мусора                                                        |
| **Бит biased locking**      | Быстрое сопоставление потока с объектом без накладных расходов                           |

> [!NOTE] `Для массива`
> Array Length — если объект — массив, то заголовок расширяется 4 байтами для хранения длины массива.

##### ==**Klass Pointer (или Type Pointer / Klass Word)**==

> [!NOTE] Klass
> В HotSpot метаданные классов описываются с помощью структуры `Klass`, которая представляет собой ==**внутренний С++-класс JVM**, описывающий **тип Java-объекта**.==
> 
> `Klass Pointer` — это **указатель на структуру `Klass`**, и он находится в **заголовке каждого объекта**.

📌 Размер: 4 или 8 байт (в зависимости от настроек JVM)

Это ==**указатель на метаданные** о классе объекта.==
Эти метаданные включают:
- ссылку на таблицу виртуальных методов (vtable),    
- имя класса,    
- информацию об интерфейсах,    
- информацию о полях и методах,    
- сведения для RTTI (RunTime Type Identification),    
- вспомогательные структуры для `instanceof` и вызовов через интерфейсы.

> [!NOTE] **Виртуальные методы в Java**
> Виртуальные методы — это ==методы, которые поддерживают **полиморфизм**== во время выполнения (runtime). 
> 
> Их вызов определяется не по типу ссылки, а по **реальному типу объекта**.

# 2. Размер ссылки

В принципе, размер ссылки в JVM зависит от ее разрядности, подозреваю, что для оптимизации. Поэтому в 32-х разрядных JVM размер ссылки обычно 4 байта, а в 64-х разрядных — 8 байт. Хотя это условие и не обязательно.

# 🔶 3. Методы класса 🔶

> [!NOTE] ❗️Ключевой момент:
> **Методы в Java НЕ хранятся внутри каждого объекта!**
## 🧠 Где хранятся методы Java-классов

🔷 Методы — это часть **метаданных класса**, которые хранятся в `Metaspace`

Когда класс загружается JVM:
- Байткод методов считывается из `.class` файла
- Он сохраняется в **Metaspace**
- Методы размещаются в структуре `Klass` → `Method[]` и связаны с **таблицей виртуальных методов (vtable)**

То есть:
- Методы существуют **один раз** на весь класс
- ==Объекты **не дублируют** методы== — они просто **"знают", где их найти** через `klass pointer`

==Пример:==
```java
class Foo {
    void hello() {
        System.out.println("Hi!");
    }
}
```

- Объект `Foo foo = new Foo();` содержит:
    - Заголовок (`MarkWord`, `KlassPointer`)
    - Никакой информации о методе `hello()` внутри самого объекта!
        
- Когда вызывается `foo.hello();`, JVM:
    - Следует по `klass pointer` → `Klass`
    - Использует таблицу методов (`vtable`) или поиск по `Method`-таблице
    - Находит байткод метода `hello()`, который уже размещён в Metaspace
