
> [!NOTE] **Future** — это интерфейс из пакета `java.util.concurrent`, представляющий `РЕЗУЛЬТАТ` `асинхронной операции`.
> Он используется для управления задачами, выполняющимися в других потоках, и позволяет:
> - **проверить статус выполнения**, 
> - **отменить задачу**, 
> - **дождаться результата** 
> - или **получить исключение**, если выполнение завершилось с ошибкой.

❗❗Обычно объекты `Future` создаются при отправке задачи через `ExecutorService` (метод `submit()`).

Реализуется такими классами, как:
- `FutureTask`
- внутренние реализации в `ThreadPoolExecutor`, `ForkJoinTask` и т.д.

---
## Интерфейс
```java
public interface Future<V>
```

## Методы
### Управление выполнением задачи
```java
// Отменяет выполнение задачи.
// mayInterruptIfRunning = true — если задача уже выполняется, попытаться прервать поток.
// Возвращает true, если задача была отменена успешно.
boolean cancel(boolean mayInterruptIfRunning)

// Проверяет, была ли задача отменена до завершения.
boolean isCancelled()

// Проверяет, завершено ли выполнение задачи (успешно, с ошибкой или отменой).
boolean isDone()
```

### Получение результата
```java
// Возвращает результат выполнения задачи, если он готов.
// Если задача ещё не завершена, поток блокируется до её завершения.
// Может выбросить:
// - InterruptedException — если поток ожидания прерван
// - ExecutionException — если задача завершилась с исключением
V get() throws InterruptedException, ExecutionException

// Возвращает результат, если он готов, или ожидает не дольше указанного времени.
// timeout — время ожидания
// unit — единицы измерения (TimeUnit)
// Может выбросить TimeoutException, если время ожидания истекло.
V get(long timeout, TimeUnit unit)
        throws InterruptedException, ExecutionException, TimeoutException
```

### (начиная с Java 19) Управление завершением (default методы)

Появились в JDK 19 (в рамках обновления `Structured Concurrency`),  
позволяют **реагировать на завершение задачи** или **обрабатывать исключения**.
```java
// (Java 19+) Возвращает состояние завершения задачи (SUCCESS, RUNNING, FAILED, CANCELLED и т.п.)
// Может быть реализован подклассом. По умолчанию возвращает null.
default State state() {
    throw new UnsupportedOperationException();
}

// (Java 19+) Блокирует поток до завершения задачи, но не возвращает результат.
// Полезен, если нужно дождаться окончания без получения значения.
default void awaitDone() throws InterruptedException {
    get();
}
```

### Вложенный enum `Future.State` (Java 19+)
```java
public static enum Future.State {
    RUNNING,     // Задача выполняется
    SUCCESS,     // Завершена успешно
    FAILED,      // Завершилась с исключением
    CANCELLED;   // Была отменена
}
```
Этот enum введён в **Java 19** для улучшенной диагностики состояния задач  
и используется некоторыми новыми API (`StructuredTaskScope`).

## 💡 Особенности и подводные камни 💡

1️⃣ **==Одноразовый результат==** — после завершения задачи метод `get()` всегда возвращает один и тот же результат.  

2️⃣ ❗❗**Блокирующее ожидание** — вызов `get()` блокирует поток до завершения задачи, если результат ещё не готов.  

3️⃣ 💡**Не поддерживает обратную связь (callback)** — для этого нужен `CompletableFuture`.  

4️⃣ **Отмена** может быть неэффективной, если задача не проверяет состояние прерывания.  

5️⃣ `Future` сам **не запускает задачу** — только управляет её состоянием.  

6️⃣ Если вызвать `get()` после `cancel(true)`, будет `CancellationException`.  

7️⃣ Методы `isDone()` и `isCancelled()` полезны для опросного контроля выполнения.

### ⚠️ Подводные камни Future

1. ❗==Метод== **`get()` ==блокирует поток** ==— если задача долгая, основной поток зависнет.
    - Используй `get(timeout)` чтобы ограничить время ожидания.

2.  ❗**Исключения "пропадают", если не вызвать `get()`**
Если `Callable` бросил исключение, оно НЕ "упадёт" в поток, который вызвал `submit()`. Исключение сохраняется внутри `Future` и будет выброшено только при `get()`.  
**Следствие:** нужно обрабатывать `Future.get()` (логировать/реагировать), иначе ошибки скрываются.

4.  ❗ **Timeouts и retry**
Всегда подумай о таймаутах при `get(timeout)`. При долгих/ненадёжных операциях применяй таймауты и стратегию повторов/отказа.

5. ❗==**Исключения оборачиваются в== `ExecutionException
```java
try {
    future.get();
} catch (ExecutionException e) {
    Throwable cause = e.getCause();
    ...
}
```

6. ❗**Нет нотификации о завершении** — если не опрашивать `isDone()`, не узнаешь о готовности без `get()`.

7. ❗Особенности при получении исключений

| Событие в задаче (`Callable`) | Что делает `Future.get()`                   |
| ----------------------------- | ------------------------------------------- |
| Всё прошло успешно            | Возвращает результат                        |
| Брошено исключение            | Бросает `ExecutionException` с `getCause()` |
| Задача отменена               | Бросает `CancellationException`             |
| Поток ожидания прерван        | Бросает `InterruptedException`              |

### Отмена задач (`Future.cancel(...)`)
```java
boolean cancel(boolean mayInterruptIfRunning)
```
- `mayInterruptIfRunning = true` — при отмене будет послан `interrupt()` потоку, выполняющему задачу (если он ещё выполняется).
    
- 🛑**Не гарантируется**, что задача остановится. Прерывание — кооперативный механизм: если задача игнорирует `Thread.interrupt()` и не проверяет/не блокируется, она продолжит работать.
    
- Если задача ещё не стартовала и находится в очереди — отмена просто пометит её как отменённую.
    
- `isCancelled()` показывает, была ли отмена запрошена; `isDone()` — true для завершённых, успешно или нет (включая отмены).

## 💎 Best Practices 💎

✅ 💡Используйте `Future` с `ExecutorService.submit()` для асинхронных задач.  

✅ Проверяйте `isDone()` перед `get()`, если не хотите блокироваться.  

✅ При необходимости отмены используйте `cancel(true)` и убедитесь, что задача обрабатывает прерывания (`Thread.interrupted()`).  

✅ Для цепочек задач или callback-логики используйте `CompletableFuture`.  

✅ 💡Используйте `get(timeout, unit)` вместо `get()` для предотвращения зависания.

## Пример `Callable` + `Future` с `ExecutorService`
```java
import java.util.concurrent.*;

public class Example {
    public static void main(String[] args) throws Exception {
        ExecutorService executor = Executors.newSingleThreadExecutor();

        Callable<String> task = () -> {
            Thread.sleep(1000);
            return "Результат задачи";
        };

        Future<String> future = executor.submit(task);

        // Блокирует поток до завершения задачи
        String result = future.get();
        System.out.println("Результат: " + result);

        executor.shutdown();
    }
}
```
