# 📕 0. Стратегия "Разделяй и властвуй"

Как правило, при работе с инфраструктурой Foгk/Join Fгamewoгk используется стратегия "==разделяй и властвуй==", основанная на рекурсии

Стратегия "разделяй и властвуй" ==базируется на рекурсивном разделении задачи на более мелкие подзадачи== до тех пор, пока размер подзадачи не станет достаточно малым для последовательной обработки.

Одним из условий наилучшего использования стратегии "разделяй и властвуй" является корректный выбор порога, при котором применяется последовательная обработка (а не дальнейшее разделение). 

Оптимальный порог обычно получается в результате профилирования характеристик выполнения. Тем не менее, даже при использовании порога ниже оптимального все равно будет наблюдаться значительное ускорение

# 📕 1. Что такое Fork/Join Framework

`ForkJoinPool` — это специализированный пул потоков, появившийся в **Java 7**, предназначенный для:
- выполнения ==задач, которые **можно рекурсивно разделить на подзадачи (fork)**==,
- ==и затем **объединить результаты (join)**==.

Это реализация паттерна **"разделяй и властвуй"** (**Divide and Conquer**) с поддержкой **work-stealing**

Используется для задач:
- обработки больших массивов / коллекций,
- рекурсивных вычислений (например, параллельный `merge sort`),
- поиска (поиск элемента в дереве, файле, структуре).

# 📕 2. Основные классы

1. `ForkJoinPool`
    - 🛑 аналог `ExecutorService`, но оптимизирован под рекурсивные задачи.
    - управляет пулом потоков и балансировкой нагрузки (work-stealing). Выполняет задачу `ForkJoinTask`
    
2. `ForkJoinTask<V>`
    - базовый абстрактный класс для задач.
    - два подкласса:
        - 🔴`RecursiveAction` — задача **без возвращаемого результата**.
            
        - 🟢`RecursiveTask<V>` — задача **с возвращаемым результатом**.
