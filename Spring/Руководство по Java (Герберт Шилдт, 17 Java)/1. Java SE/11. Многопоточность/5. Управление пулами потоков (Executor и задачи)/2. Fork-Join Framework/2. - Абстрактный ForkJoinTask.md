
> [!NOTE] `ForkJoinTask<V>` — это абстрактный базовый `класс для задач`, выполняемых `в рамках Fork/Join Framework`.
> Fork/Join — это механизм для ==параллельного разделения вычислений== на подзадачи (`fork`) и последующего их **объединения** (`join`).
> 
> Определяет задачу, которую может выполнить `ForkJoinPool`
> 
> ⭐**Реализует интерфейс:** `java.util.concurrent.Future<V>`

Класс `ForkJoinTask` лежит в основе таких конкретных реализаций, как:
- `RecursiveTask<V>` — возвращает результат.
- `RecursiveAction` — не возвращает результата.
    
- `CountedCompleter<T>` — поддерживает асинхронное завершение


## **Методы**

Поскольку это абстрактный базовый класс, многие методы предназначены для переопределения в подклассах, а часть — для управления задачами.
### Основные абстрактные методы
```java
// Определяет ОСНОВНУЮ ЛОГИКУ вычислений.
// Метод вызывается при выполнении задачи.
protected abstract V compute();
```
💬 В наследниках этот метод реализуется как основная точка вычислений (например, в `RecursiveTask`).

### Управление выполнением (Fork/Join API)
```java
// 💡Делит задачу💡 — ОТПРАВЛЯЕТ её В ПУЛЛ для выполнения другим потоком.
// Возвращает саму задачу для цепочек вызовов.
public final ForkJoinTask<V> fork()

// 💡Ожидает завершения задачи и возвращает результат💡 (если есть).
public final V join()



// ВЫПОЛНЯЕТ ЗАДАЧУ синхронно (ожидает заврешения выполнения), если она ещё не выполнена.
// Возвращает результат.
public final V invoke()
```
💬 Комбинация `fork()` и `join()` позволяет распараллеливать вычисления:
- `fork()` — отправить подзадачу;
- `join()` — дождаться результата.

## 💡 **Особенности и подводные камни** 💡

1️⃣ 💡`ForkJoinTask` — ==**не поток**==, а _единица работы_, исполняемая внутри `ForkJoinPool`.  

2️⃣❗❗❗ **Нельзя запускать задачи напрямую** через `new Thread()`. Используй `ForkJoinPool.invoke()`.  

3️⃣ 💡Методы `fork()` и `join()` должны использоваться сбалансированно — перекос вызывает блокировки.  

4️⃣ Использование `join()` в неправильном месте может привести к **deadlock**.  

5️⃣ 💡Метод `compute()` НЕ должен быть длительным или блокирующим** — это снижает производительность пула.  

6️⃣ Исключения не пробрасываются напрямую — нужно проверять `isCompletedAbnormally()` и `getException()`.

---
## 💎 **Best Practices** 💎

✅ Разделяй задачу рекурсивно до минимальных подзадач (divide & conquer). 

✅ ❗❗❗Не выполняй в `compute()` операций ввода/вывода (I/O) — только CPU-bound задачи.  

✅ Используй `ForkJoinPool.commonPool()` для большинства сценариев.  

✅ При необходимости создавать собственный пул — указывай оптимальное количество потоков = числу ядер.  

✅ Убедись, что `fork()` всегда сопровождается `join()`, иначе будет потеря результатов.

## Пример
```java
import java.util.concurrent.RecursiveTask;
import java.util.concurrent.ForkJoinPool;

class SumTask extends RecursiveTask<Long> {
    private final long[] arr;
    private final int start, end;
    private static final int THRESHOLD = 10_000;

    public SumTask(long[] arr, int start, int end) {
        this.arr = arr;
        this.start = start;
        this.end = end;
    }

    @Override
    protected Long compute() {
        int length = end - start;
        if (length <= THRESHOLD) {
            long sum = 0;
            for (int i = start; i < end; i++) sum += arr[i];
            return sum;
        } else {
            int mid = start + length / 2;
            SumTask left = new SumTask(arr, start, mid);
            SumTask right = new SumTask(arr, mid, end);
            left.fork();                 // Запустить левую часть в фоне
            long rightResult = right.compute(); // Вычислить правую часть в текущем потоке
            long leftResult = left.join();      // Дождаться левой части
            return leftResult + rightResult;
        }
    }
}

public class ForkJoinExample {
    public static void main(String[] args) {
        long[] data = new long[100_000];
        for (int i = 0; i < data.length; i++) data[i] = i;

        ForkJoinPool pool = new ForkJoinPool();
        SumTask task = new SumTask(data, 0, data.length);

        long result = pool.invoke(task); // Запуск задачи в пуле
        System.out.println("Сумма = " + result);
    }
}
```
