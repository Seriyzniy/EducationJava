# 📕Losk Striping (мелкозернистая блокировка)
Это стратегия, при которой ==**состояние объекта разбивается на независимые части**==, и **каждая ЧАСТЬ ЗАЩИЩАЕТСЯ СВОЕЙ БЛОКИРОВКОЙ**.

## 🚫 Противоположность: **грубая (монолитная) синхронизация**
```java
public class SynchronizedList<T> {
    private final List<T> list = new ArrayList<>();

    public synchronized void add(T item) {
        list.add(item);
    }

    public synchronized T get(int index) {
        return list.get(index);
    }
}
```
📌 Здесь ==**вся структура** блокируется целиком==, даже если потоки работают с **разными частями** списка.

## ✅ Мелкозернистая синхронизация: идея

Вместо одной блокировки — **несколько**. Каждый поток может работать с **разной частью данных параллельно**, не мешая другим.

### 💡 Пример — простая реализация striping map:
```java
public class StripedMap<K, V> {
    private static final int N_SEGMENTS = 16;
    private final Map<K, V>[] segments;
    private final Object[] locks;

    public StripedMap() {
        segments = new Map[N_SEGMENTS];
        locks = new Object[N_SEGMENTS];
        for (int i = 0; i < N_SEGMENTS; i++) {
            segments[i] = new HashMap<>();
            locks[i] = new Object(); //Заполняем массив объектами для блокировки
        }
    }

    public void put(K key, V value) {
        int index = Math.abs(key.hashCode() % N_SEGMENTS);
        synchronized (locks[index]) { // Захватываем монитор определенного объекта из массива
            segments[index].put(key, value);
        }
    }

    public V get(K key) {
        int index = Math.abs(key.hashCode() % N_SEGMENTS);
        synchronized (locks[index]) {
            return segments[index].get(key);
        }
    }
}
```
📌 Здесь `put` и `get` **работают независимо**, если попали в разные сегменты.

# 🧠 Вопросы на собеседования 🧠
## ❓ Зачем нужен lock striping?

Чтобы ==потоки **могли одновременно модифицировать разные участки данных**, не блокируя друг друга.== Это снижает конкуренцию и повышает пропускную способность.

---
## ❓ Почему `CopyOnWriteArrayList` не использует striping?

Потому что она вообще **не синхронизирует операции записи**, а **копирует массив** на каждую модификацию. Это другая модель: **оптимизация под многократное чтение** и **редкие записи**.
