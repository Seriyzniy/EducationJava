
> [!NOTE] **ConcurrentSkipListSet** — это **потокобезопасное, упорядоченное множество (Set)**, реализованное на основе `ConcurrentSkipListMap`.
> Элементы в нем `УНИКАЛЬНЫ` и `ОТСОРТИРОВАНЫ` по **естественному порядку** (`Comparable`) или по **заданному компаратору**.  
> 
> 👉 Потокобезопасные версии `TreeSet`
> `ConcurrentSkipListSet` = **==отсортированное множество== (только ключи)**
> 
> ⭐Реализует интерфейсы: `NavigableSet<E>`, `Serializable`, `Cloneable`.

🧠Под капотом `ConcurrentSkipListSet` 
- хранит элементы как **ключи** в `ConcurrentSkipListMap`, 
- а значения — фиктивные (`Boolean.TRUE`).  

Использует **lock-free skip list** для конкурентного доступа без блокировок.

## Конструкторы
```java
// Создает пустое множество, где порядок элементов определяется их естественным порядком.
public ConcurrentSkipListSet()

// Создает пустое множество, где порядок элементов определяется указанным компаратором.
public ConcurrentSkipListSet(Comparator<? super E> comparator)

// Создает множество, содержащее все элементы из указанной коллекции, упорядоченные по естественному порядку.
public ConcurrentSkipListSet(Collection<? extends E> c)

// Создает множество, содержащее все элементы из указанного SortedSet, с тем же порядком сортировки.
public ConcurrentSkipListSet(SortedSet<E> s)
```

## 💡 Особенности и подводные камни 💡

1️⃣ Основан на **ConcurrentSkipListMap**, то есть элементы — это ключи lock-free карты.  

2️⃣ 💡Не допускает `null`-элементы (ни при вставке, ни при сравнении).  

3️⃣ **Итераторы слабосогласованные** — они не бросают `ConcurrentModificationException`, но могут не отражать все изменения в реальном времени.  

4️⃣ ⭐⭐Гарантирует **упорядоченность элементов** даже при конкурентных изменениях.  

5️⃣ 💡Методы диапазонов (`subSet`, `headSet`, `tailSet`) возвращают **живые представления**, отражающие изменения оригинального множества.  

6️⃣ ❗❗❗Элементы должны быть **взаимосравнимыми** — либо реализовывать `Comparable`, либо использовать заданный `Comparator`.  

7️⃣ ❗❗❗Не подходит для задач, где требуется частое массовое добавление (bulk insert) — так как каждая вставка `O(log n)`.❗❗❗

---

## 💎 Best Practices 💎
✅ Используйте, если нужно **упорядоченное, потокобезопасное множество**.  

✅ 💡Отлично подходит для **регистрирования уникальных событий, логов, отсортированных кэшей**.  

✅ Если порядок не важен — используйте `ConcurrentHashMap.newKeySet()` (быстрее).  

✅ Не используйте `null`-значения.  

✅ Итераторы можно безопасно использовать в многопоточной среде — они не падают при изменениях множества.

## Пример
```java
import java.util.*;
import java.util.concurrent.*;

public class ConcurrentSkipListSetExample {
    public static void main(String[] args) {
        ConcurrentSkipListSet<Integer> set = new ConcurrentSkipListSet<>();

        set.add(5);
        set.add(1);
        set.add(3);
        set.add(2);

        System.out.println("Элементы (в порядке возрастания):");
        set.forEach(System.out::println);

        System.out.println("\nПервый элемент: " + set.first());
        System.out.println("Последний элемент: " + set.last());
        System.out.println("Ceiling(3): " + set.ceiling(3));
        System.out.println("Higher(3): " + set.higher(3));

        NavigableSet<Integer> head = set.headSet(3, true);
        System.out.println("\nHeadSet (<=3): " + head);

        // Потокобезопасная итерация
        ExecutorService executor = Executors.newFixedThreadPool(2);
        executor.submit(() -> set.add(6));
        executor.submit(() -> set.remove(1));

        executor.shutdown();
        
        System.out.println("\nСостояние множества после многопоточной работы:");
        set.forEach(System.out::println);
    }
}
Элементы (в порядке возрастания):
1
2
3
5

Первый элемент: 1
Последний элемент: 5
Ceiling(3): 3
Higher(3): 5

HeadSet (<=3): [1, 2, 3]

Состояние множества после многопоточной работы:
1
2
3
5
6
```
