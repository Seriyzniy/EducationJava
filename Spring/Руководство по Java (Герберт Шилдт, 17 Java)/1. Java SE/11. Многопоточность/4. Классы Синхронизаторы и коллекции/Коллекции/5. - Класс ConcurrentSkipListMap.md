
> [!NOTE] **ConcurrentSkipListMap** — это потокобезопасная, `упорядоченная по ключам` коллекция
>  🧠Представляет собой реализацию интерфейсов `ConcurrentNavigableMap` и `NavigableMap`, основанная на **skip list (списке с пропусками)**.
> 
> 💡Она обеспечивает **логарифмическое время доступа** (`O(log n)`) для операций поиска, вставки и удаления, аналогично `TreeMap`, но при этом **без блокировок всей структуры** — используется **lock-free алгоритм** на основе CAS.
> 
> 👉 Потокобезопасные версии `TreeMap`
> `ConcurrentSkipListMap` = **==отсортированный словарь== (ключ → значение)**.



⭐Поддерживает **естественный порядок ключей** (через `Comparable`) или **заданный `Comparator`**.  

❗❗Не допускает `null` в ключах или значениях.

## Конструкторы
```java
// Создает пустую карту, где порядок ключей определяется их естественным порядком.
public ConcurrentSkipListMap()

// Создает пустую карту, порядок ключей определяется переданным компаратором.
public ConcurrentSkipListMap(Comparator<? super K> comparator)

// Создает карту, содержащую все отображения из указанной Map, упорядоченные по естественному порядку ключей.
public ConcurrentSkipListMap(Map<? extends K, ? extends V> m)

// Создает карту, содержащую все отображения из указанной SortedMap, упорядоченные так же, как и в исходной карте.
public ConcurrentSkipListMap(SortedMap<K, ? extends V> m)\
```

## 💡 Особенности и подводные камни 💡

1️⃣ 💡**Lock-free** реализация — большинство операций используют CAS, что делает их неблокирующими и высокопроизводительными.  

2️⃣ 💡Гарантируется **слабая согласованность** итераторов — итератор отражает текущее состояние, но не выбрасывает `ConcurrentModificationException`.  

3️⃣ Не допускает `null` в ключах и значениях (в отличие от `HashMap`).  

4️⃣ Использует **skip list**, обеспечивая упорядоченность и быстрое двоичное "приближение" к нужному элементу.  

5️⃣ ❗❗Производительность обычно выше, чем у `ConcurrentHashMap`, если требуется упорядоченность.  

6️⃣  💡Итераторы — **не fail-fast**, но и не snapshot (т.е. не фиксируют момент состояния, а "видят" обновления).

---
## 💎 Best Practices 💎
✅ Используйте, если нужно **потокобезопасное упорядоченное отображение**.

✅ Подходит для **реализации приоритетных структур**, логов, кэшей с диапазонами ключей.  

✅ Не используйте, если порядок не нужен — в этом случае `ConcurrentHashMap` быстрее.  

✅ ⭐⭐Если порядок нестандартный — передавайте `Comparator` в конструктор.  

✅ Для итерации — можно безопасно использовать `for-each`, т.к. итератор слабосогласованный.

## Пример
```java
ConcurrentSkipListMap<Integer, String> map = new ConcurrentSkipListMap<>();

// Поток записи
Thread writer = new Thread(() -> {
	for (int i = 1; i <= 10; i++) {
		map.put(i, "Value-" + i);
		System.out.println("Writer added: " + i);
		try {
			Thread.sleep(100); // задержка для наглядности
		} catch (InterruptedException e) {
			Thread.currentThread().interrupt();
		}
	}
});

// Поток чтения
Thread reader = new Thread(() -> {
	while (true) {
		if (!map.isEmpty()) {
			Integer firstKey = map.firstKey();
			Integer lastKey = map.lastKey();
			System.out.println("Reader sees: first=" + firstKey + ", last=" + lastKey);
		}
		try {
			Thread.sleep(150); // задержка для наглядности
		} catch (InterruptedException e) {
			Thread.currentThread().interrupt();
			break;
		}
	}
});

writer.start();
reader.start();

writer.join();
Thread.sleep(2000); // подождать немного, чтобы reader "догнал"
reader.interrupt();
```
