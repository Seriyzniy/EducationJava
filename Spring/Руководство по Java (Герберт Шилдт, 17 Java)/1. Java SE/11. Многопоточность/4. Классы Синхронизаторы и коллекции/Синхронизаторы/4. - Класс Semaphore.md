
> [!NOTE] `Semaphore` — это синхронизатор из пакета `java.util.concurrent`,
> Который `управляет количеством` "`разрешений`" (permits), доступных для выполнения потоками.  
> 
> Он `ограничивает` `одновременный доступ` `к ресурсу` определённым числом потоков.
> 
> Реализует интерфейс: `Serializable`

Используется для реализации пулов ресурсов, ограничения количества активных задач и синхронизации потоков.

---
## Конструкторы
```java
// Создает семафор с заданным количеством разрешений (permits).
// fair = false — порядок получения разрешений не гарантируется (поведение по умолчанию, выше производительность).
public Semaphore(int permits)

// Создает семафор с заданным количеством разрешений и режимом справедливости.
// fair = true — потоки получают разрешения в порядке запроса (FIFO).
public Semaphore(int permits, boolean fair)
```
**Пояснения к параметрам:**
- `permits` — ==начальное количество== доступных ==разрешений== (если 0 — все потоки будут блокироваться при acquire).
    
- `fair` — ==режим справедливости==: если `true`, приоритет отдаётся потокам, ожидающим дольше всего.

## Методы
### Получение разрешения
```java
// Запрашивает одно разрешение, блокируясь, пока не станет доступно.
// Если поток прерван — выбрасывается InterruptedException.
public void acquire() throws InterruptedException

// Запрашивает указанное количество разрешений.
// Если не хватает — блокируется, пока не освободятся.
// При прерывании выбрасывает InterruptedException.
public void acquire(int permits) throws InterruptedException

// Пытается получить разрешение, не блокируясь.
// Возвращает true, если удалось, иначе false.
public boolean tryAcquire()

// Пытается получить разрешение в течение заданного времени.
// timeout — время ожидания, unit — единицы времени.
// Возвращает true, если удалось получить разрешение, иначе false.
// При прерывании выбрасывает InterruptedException.
public boolean tryAcquire(long timeout, TimeUnit unit) throws InterruptedException

// Пытается получить указанное количество разрешений в течение заданного времени.
public boolean tryAcquire(
	int permits, 
	long timeout, 
	TimeUnit unit)
        throws InterruptedException
```

### Освобождение разрешений
```java
// Освобождает одно разрешение, увеличивая счётчик доступных.
public void release()

// Освобождает указанное количество разрешений.
public void release(int permits)
```

### Диагностика и служебные методы
```java
// Возвращает количество доступных (не занятых) разрешений.
public int availablePermits()

// Устанавливает количество разрешений равным 0 (экстренное "закрытие").
public void drainPermits()

// Возвращает true, если используется справедливый режим.
public boolean isFair()

// Возвращает количество потоков, ожидающих разрешения.
public int getQueueLength()

// Проверяет, есть ли ожидающие потоки.
public boolean hasQueuedThreads()
```

## 💡 Особенности и подводные камни 💡

1️⃣ `Semaphore` **==НЕ ПРИВЯЗАН К конкретным ПОТОКАМ==** — поток, который вызвал `release()`, не обязан быть тем, кто делал `acquire()`.  
⭐Даже если один поток захватил семафор, то если он попытается сделать это еще, то не выйдет - счетчик семафора на 0.

2️⃣ Если разрешений нет — поток блокируется до тех пор, пока кто-то не вызовет `release()`.  

3️⃣ **Fair-mode (`true`)** гарантирует FIFO, но снижает производительность.  

4️⃣ Если вызвать `release()` больше раз, чем `acquire()` — количество разрешений превысит изначальное (никаких исключений не будет).  

5️⃣ Важно корректно обрабатывать `InterruptedException` при `acquire()`.  

6️⃣ 💡Может использоваться как **мьютекс**, если количество разрешений = 1.  

7️⃣ Не путать с `CountDownLatch`: семафор ==можно **переиспользовать** многократно.  ==

8️⃣ Внутри реализован на базе `AbstractQueuedSynchronizer` (AQS).

---

## 💎 Best Practices 💎

✅ Используйте `Semaphore(1)` как **мьютекс**, если требуется только один поток в критической секции.  

✅ Для управления пулом ресурсов (например, соединений) создавайте семафор с количеством разрешений, равным размеру пула.  

✅ Всегда вызывать `release()` в `finally` после успешного `acquire()`.  

✅ Используйте `tryAcquire(timeout, unit)` вместо бесконечного ожидания.  

✅ В большинстве случаев не включайте "справедливость" (`fair=false`), если нет строгих требований по порядку.  

✅ Помните: `Semaphore` — инструмент координации, а не блокировки доступа к объекту.

## Пример
```java
import java.util.concurrent.Semaphore;

public class MyMain  {
	public static void main(String[] args){
		Q q = new Q();
        new Thread(new Consumer(q), "Consumer").start();
        new Thread(new Producer(q), "Producer").start();
	}
}


class Q {
    int n;
    // Начать с недоступным семафором потребителя.
    static Semaphore semCon = new Semaphore(0);
    static Semaphore semProd = new Semaphore(1);
    
    void get() {
        try {
            semCon.acquire();
        } catch(InterruptedException e) {
            System.out.println("Перехвачено исключение InterruptedException");
        }
        System.out.println("Получено: " + n);
        semProd.release();
    }
    
    void put(int n) {
        try {
            semProd.acquire();
        } catch(InterruptedException e) {
            System.out.println("Перехвачено исключение InterruptedException");
        }
        this.n = n;
        System.out.println("Отправлено: " + n);
        semCon.release();
    }
}

class Producer implements Runnable {
    Q q;
    
    Producer(Q q) {
        this.q = q;
    }
    
    public void run() {
        for (int i = 0; i < 20; i++) {
            q.put(i);
        }
    }
}

class Consumer implements Runnable {
    Q q;
    
    Consumer(Q q) {
        this.q = q;
    }
    
    public void run() {
        for (int i = 0; i < 20; i++) {
            q.get();
        }
    }
}
```
