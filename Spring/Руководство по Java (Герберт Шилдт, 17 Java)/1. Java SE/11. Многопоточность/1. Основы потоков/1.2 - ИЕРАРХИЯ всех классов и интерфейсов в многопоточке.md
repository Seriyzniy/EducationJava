# 📕 **3. Интерфейсы для СИНХРОНИЗАЦИИ

![[Pasted image 20250712165542.png]]
## 🔶 `Lock` (интерфейс)
**Назначение:** Абстракция взаимного исключения, более гибкая альтернатива `synchronized`.
##### Основные методы:
- `lock()` — блокирует поток (может ждать вечно)
    
- `tryLock()` — пытается захватить замок, **не блокируя** (возвращает `true/false`)
    
- `lockInterruptibly()` — как `lock()`, но можно прервать через `Thread.interrupt()`
    
- `unlock()` — освобождает замок

---
## 🔶`ReentrantLock`
**Назначение:** Стандартная реализация `Lock` с возможностью **повторного входа** (reentrancy).
##### Особенности:
- Один поток может многократно захватывать замок
    
- Может быть **справедливым** (`fair=true`) или нет (по умолчанию)
##### Методы:
- `lock()`, `tryLock()`, `unlock()`
    
- `isHeldByCurrentThread()` — удерживает ли текущий поток замок
    
- `getHoldCount()` — сколько раз замок был захвачен текущим потоком

---
## 🔶`ReadWriteLock`
**Назначение:** Разделяет доступ на **чтение и запись**.
- Одновременное чтение — допускается
    
- Запись — эксклюзивна
    
---
## 🔶 `ReentrantReadWriteLock`
**Назначение:** Реализация `ReadWriteLock`.
##### Методы:
- `readLock()` — возвращает `Lock` для чтения
    
- `writeLock()` — возвращает `Lock` для записи

# 📕 **4. Некоторые коллекции и другие дополнительные классы

## 📙 1. BlockingQueue
![[Pasted image 20250716103731.png]]

🛑🛑==Интерфейс== `BlockingQueue` — это ==**очередь с возможностью блокировки**==, которая блокирует операции `put()` и `take()`, если очередь заполнена или пуста соответственно.
### Общие методы:
- `put(E e)` — добавляет элемент, блокирует, если очередь заполнена
- `take()` — извлекает элемент, блокирует, если очередь пуста
    
- `offer(E e)` / `poll()` — неблокирующие аналоги

***
### 🔶`ArrayBlockingQueue`
- 🛑**Ограниченная** очередь (фиксированный размер)
- Реализована на **массиве**

- Потокобезопасна через **ReentrantLock**

***
### 🔶 `LinkedBlockingQueue`
- Может быть **ограниченной или неограниченной**
- Реализована на **двусвязном списке**

- Использует **отдельные замки для put и take**, лучше масштабируется

***
### 🔶 `DelayQueue`
- ==Элементы== становятся ==доступны== только ==**после истечения задержки**==
- Работает с объектами, реализующими интерфейс `Delayed`
    
- Часто ==используется для **таймеров, отложенной отправки**== и т.д.

***
### 🔶 `PriorityBlockingQueue`
- **Приоритетная очередь** (не FIFO, а по Comparator’у или `Comparable`)
- 🛑🛑**Неблокирующая** при добавлении, но блокирующая при извлечении
    
- Не имеет размера, может расти бесконечно

***
### 🔶 `SynchronousQueue`
- 🛑**Нет хранилища вообще** — каждая `put()` блокирует до `take()`
- 🛑Используется для **“рукопожатий” между потоками**
    
- Часто используется в `Executors.newCachedThreadPool()`

***
## 📙 2. ThreadFactory
Позволяет управлять **созданием потоков** в `ExecutorService`.
```java
ExecutorService pool = Executors.newFixedThreadPool(5, new ThreadFactory() {
    public Thread newThread(Runnable r) {
        Thread t = new Thread(r);
        t.setName("Worker-" + count.incrementAndGet());
        return t;
    }
});
```

### 💡 Используется для:
- Задания имён потокам
- Управления приоритетами
    
- Применения `UncaughtExceptionHandler`

***
## 📙3. `ThreadLocal<T>`

Переменные, **уникальные для каждого потока**.
### 💡 Особенности: 💡
- Значение видно **только потоку, который его установил**

- Удобно использовать для:
    - Безопасного хранения `SimpleDateFormat`
    - Хранения соединений с БД (connection per thread)

```java
private static final ThreadLocal<Connection> conn =
    ThreadLocal.withInitial(() -> DriverManager.getConnection(...));
```

## 📙 4.`Atomic*`классы (`java.util.concurrent.atomic`)

Предоставляют ==**потокобезопасные атомарные операции**==, реализованы ==через **CAS== (Compare-And-Swap)**.

### Основные:
- `AtomicInteger` / `AtomicLong` — атомарные числовые счётчики
    - `incrementAndGet()`, `addAndGet()`, `compareAndSet(...)`
        
- `AtomicBoolean` — булев флаг
    
- `AtomicReference<T>` — атомарная ссылка
    
- `AtomicStampedReference<T>` — защита от ABA-проблемы с помощью “штампов”
    
- `AtomicMarkableReference<T>` — дополнительный булев флаг (true/false)
