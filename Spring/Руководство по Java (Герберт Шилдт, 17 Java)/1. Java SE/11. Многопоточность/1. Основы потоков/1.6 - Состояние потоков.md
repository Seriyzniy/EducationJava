В Java каждый поток может находиться в одном из **состояний жизненного цикла**, определённых в перечислении `java.lang.Thread.State`.

Для получения состояние потока используется метод:
```java
Thread.State getState();
```
# 💡 Основные состояния потоков (`Thread.State`)

| Состояние       | Описание                                                                             |
| --------------- | ------------------------------------------------------------------------------------ |
| `NEW`           | Поток создан (`new Thread()`), но ещё не запущен (`start()` не вызван)               |
| `RUNNABLE`      | Поток готов к выполнению или выполняется                                             |
| `BLOCKED`       | Поток ожидает **входа в `synchronized` блок/метод**, занятый другим потоком          |
| `WAITING`       | Поток ждёт другого потока (например, вызов `join()` или `wait()`)                    |
| `TIMED_WAITING` | То же самое, что `WAITING`, но с таймером (например, `sleep(1000)`)                  |
| `TERMINATED`    | ==Поток завершил выполнение== (естественно или через исключение) (по старому `DEAD`) |

# ⚠️ Подводные камни

## 🔸`RUNNABLE` ≠ "работает прямо сейчас"
- Поток может быть в `RUNNABLE`, но **ожидать планировщика**.
    
## 🔸`BLOCKED` ≠ `WAITING`
- `BLOCKED` — поток ждёт входа в `synchronized`.
    
- `WAITING` — поток **осознанно ожидает** (например, `wait()`), не блокируя монитор.
