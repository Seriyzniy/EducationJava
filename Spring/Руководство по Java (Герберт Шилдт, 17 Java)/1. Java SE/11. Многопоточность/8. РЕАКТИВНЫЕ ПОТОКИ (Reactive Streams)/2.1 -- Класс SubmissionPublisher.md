
> [!NOTE] `SubmissionPublisher<T>` — это `стандартная реализация интерфейса` `Flow.Publisher<T>`, добавленная в Java 9.
> Он служит `асинхронным источником данных`
> 
> `Публикует элементы` подписчикам (`Flow.Subscriber`), используя:
> - **пул потоков (ForkJoinPool.commonPool())** 
> - указанный `Executor`.

🔸Реализует интерфейсы:
- `Flow.Publisher<T>` — чтобы публиковать элементы подписчикам
    
- ⭐⭐⭐`AutoCloseable` — чтобы можно было безопасно завершить публикацию через `close()`⭐⭐⭐

## Методы
### Публикация данных в поток
```java
// 💡Публикует элемент всем подписчикам АСИНХРОННО.💡
// Возвращает оценку задержки до передачи (в наносекундах).
public int submit(T item);

// 💡Отправляет элемент СИНХРОННО💡 (блокирует, если буфер заполнен).
// Возвращает оценку задержки.
public int offer(
	T item, 
	long timeout, 
	TimeUnit unit, 
	BiPredicate<Flow.Subscriber<? super T>, ? super T> onDrop);

// Отправляет элемент, если возможно, иначе возвращает -1 без блокировки.
public int offer(
	T item, 
	BiPredicate<Flow.Subscriber<? super T>, ? super T> onDrop);
```

- `submit()` — стандартный способ публикации.
    
- `offer()` — гибкий способ, позволяющий контролировать поведение при переполнении буфера.
    
- Параметр `onDrop` позволяет задать, что делать с "лишними" элементами (например, отбрасывать или ждать).

### Управление подписчиками
```JAVA
// Регистрирует нового подписчика, который будет получать элементы.
public void subscribe(Flow.Subscriber<? super T> subscriber);

// Возвращает текущее количество подписчиков.
public int getNumberOfSubscribers();

// Возвращает true, если у издателя есть хотя бы один подписчик.
public boolean hasSubscribers();
```
Обычно вызывается `subscribe()` — при этом `SubmissionPublisher` сам создаёт и передаёт подписчику `Flow.Subscription`.

### Управление жизненным циклом
```java
// Завершает публикацию: подписчики получают onComplete()
public void close();

// Завершает публикацию с ошибкой: подписчики получают onError()
public void closeExceptionally(Throwable error);

// Проверяет, завершена ли публикация
public boolean isClosed();
```

### Буфер и производительность
```java
// Возвращает текущий размер буфера (в элементах)
public int getMaxBufferCapacity();

// Возвращает среднюю задержку при публикации элементов
public long estimateMinimumDemand();
```

## 💡 Особенности и подводные камни 💡

1️⃣ 💡**Асинхронная доставка элементов.**  
Каждый подписчик получает элементы независимо через собственный поток (используя Executor).

2️⃣ **Буферизация.**  
У каждого подписчика есть отдельный буфер. Если он заполнен, `submit()` может блокировать поток или отбрасывать элементы (при `offer()`).

3️⃣ **Backpressure (управление потоком).**  
Подписчики регулируют количество получаемых элементов через `Subscription.request(n)`.

4️⃣ 💡**Thread-safety.**  💡
Методы потокобезопасны, можно публиковать элементы из разных потоков.

5️⃣ ❗❗❗**Необходимо закрывать издателя.**  
После публикации всех данных обязательно вызывать `close()`, иначе подписчики не получат сигнал `onComplete()`.

6️⃣ 🧠**Проблема с медленными подписчиками.**  
Если один из подписчиков обрабатывает элементы медленно, буфер может переполниться — нужно использовать `offer()` с обработчиком.

7️⃣ 💡**По умолчанию используется ForkJoinPool.commonPool().**  
Если требуется контроль, лучше передать собственный `Executor`.

---

## 💎 Best Practices 💎

✅ Всегда закрывай издателя методом `close()` после завершения отправки данных.  

✅ Используй `offer()` при высоконагруженных потоках, чтобы избежать блокировки.  

✅ 💡Для долгих или тяжёлых подписчиков — используй отдельный `Executor`.  

✅ 💡Храни ёмкость буфера не слишком большой (обычно 
`Flow.defaultBufferSize()` подходит).  

✅ Обрабатывай исключения через `handler` в конструкторе — это повышает устойчивость системы.  

✅ Для сложных конвейеров используй `Processor` между издателем и подписчиками.

## Пример
```java
import java.util.concurrent.Flow;
import java.util.concurrent.SubmissionPublisher;

public class SubmissionPublisherExample {
    public static void main(String[] args) throws InterruptedException {
        // Создаём издателя
        try (SubmissionPublisher<String> publisher = new SubmissionPublisher<>()) {
            // Создаём подписчика
            Flow.Subscriber<String> subscriber = new Flow.Subscriber<>() {
                private Flow.Subscription subscription;

                @Override
                public void onSubscribe(Flow.Subscription subscription) {
                    this.subscription = subscription;
                    subscription.request(1); // запрос первого элемента
                }

                @Override
                public void onNext(String item) {
                    System.out.println("Получено: " + item);
                    subscription.request(1); // запрос следующего
                }

                @Override
                public void onError(Throwable throwable) {
                    System.out.println("Ошибка: " + throwable.getMessage());
                }

                @Override
                public void onComplete() {
                    System.out.println("Поток завершён!");
                }
            };

            // Подписываемся
            publisher.subscribe(subscriber);

            // Публикуем данные
            System.out.println("Публикация элементов...");
            publisher.submit("Java");
            publisher.submit("Flow");
            publisher.submit("API");

            // Завершаем поток
            publisher.close();
        }

        // Дадим немного времени для завершения асинхронных задач
        Thread.sleep(500);
    }
}

//Вывод
Публикация элементов...
Получено: Java
Получено: Flow
Получено: API
Поток завершён!
```
