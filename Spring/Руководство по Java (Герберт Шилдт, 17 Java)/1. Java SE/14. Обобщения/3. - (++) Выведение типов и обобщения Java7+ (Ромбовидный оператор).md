### Ромбовидный оператор `<>` (JDK 7+)

Сокращает синтаксис создания экземпляров:
```java
MyClass<Integer, String> mcOb = new MyClass<>(98, "Строка");
```

Также можно использовать `var`, но с указанием обобщения в конструкторе
```java
var mcOb = new MyClass<Integer, String>(98, "Строка");
```

Выведение типов можно также применять к передаче параметров

### Выведение типов локальных переменных (JDK 10+)
```java
import java.io.FileNotFoundException;

interface MyFunc<T> {
    boolean func(T v1, T v2);
}

class HighTemp {
    private int hTemp;
    
    HighTemp(int ht) { 
        hTemp = ht; 
    }
    
    boolean sameTemp(HighTemp ht2) {
        return hTemp == ht2.hTemp;
    }
    
    boolean lessThanTemp(HighTemp ht2) {
        return hTemp < ht2.hTemp;
    }
}

public class MyMain {
	// ❓❓
	// Вопрос, как компилятор определит тип T?
	// ❓❓
	static <T> int counter(T[] vals, MyFunc<T> f, T v) {
        int count = 0;
        for(int i = 0; i < vals.length; i++)
            if(f.func(vals[i], v))
                count++;
        return count;
    }
	
    public static void main(String[] args) throws FileNotFoundException  {
    	int count;
    	
        // Создать массив объектов HighTemp.
        HighTemp[] weekDayHighs = { 
            new HighTemp(89), new HighTemp(82),
            new HighTemp(90), new HighTemp(89),
            new HighTemp(89), new HighTemp(91),
            new HighTemp(84), new HighTemp(83) 
        };
        
        // 🔑КЛЮЧЕВОЙ МОМЕНТ🔑
        count = counter(weekDayHighs, HighTemp::sameTemp, new HighTemp(89));
        
        
        System.out.println(
	        "Количество суток, когда самая высокая температура была 89 градусов: "
	        + count
	    );
    }
}
```

Чтобы правильно ВЫВЕСТИ тип `T` в вызываемом методе `counter` в `main`, компилятор:
1. Смотрит тип 1-го параметра `weekDayHighs` => массив типа `HighTemp`
2. Смотрит на тип 3-го параметра `new HighTemp(89)` => `HighTemp`
3. Значит вместо `T` подставится `HighTemp`

Второй параметр не дает ясного понимания. (Хотя возможно тип метода - тип класса, то есть тоже `HighTemp`)
