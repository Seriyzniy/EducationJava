Возникают когда СТИРАНИЕ делает РАЗНЫЕ объявления ОДИНАКОВЫМИ:
```java
void set(T o) { ... }    // После стирания: void set(Object o)
void set(V o) { ... }    // После стирания: void set(Object o) - конфликт!
```

Поэтому так не делать!
Если нужны такие методы, то стоит ограничить обобщения, чтобы они относились к разным типам. 
Например
```java
class MyGenClass<T, V extends Nubmer>
```
