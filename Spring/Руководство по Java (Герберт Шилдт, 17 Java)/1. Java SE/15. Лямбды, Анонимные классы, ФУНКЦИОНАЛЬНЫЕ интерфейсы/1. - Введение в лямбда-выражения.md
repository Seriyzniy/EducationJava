Ключевым аспектом для понимания реализации лямбда-выражений в Java являются две конструкции: собственно лямбда-выражение и функциональный интерфейс.

***
## **Ключевые понятия:**

⭐**Лямбда-выражение** - ==анонимный метод==, который не выполняется самостоятельно, а используется ==для реализации метода функционального интерфейса==

⭐**Функциональный интерфейс** - ==интерфейс с ОДНИМ АБСТРАКТНЫМ методом== (SAM - Single Abstract Method)
> [!NOTE] ⭐⭐Методы из `Object` (`equals`, `hashCode`, `toString`) НЕ СЧИТАЮТСЯ.⭐⭐
> 

***
# 📕 1. Основы лямбда-выражений

Лямбда-выражение ==состоит из двух частей==, разделенных оператором `->`:
- 🔶Левая часть: параметры (если есть)
- 🔶Правая часть: тело выражения

## Тип лямбды - выводится
> [!NOTE] 🛑`ТИП ПАРАМЕТРА ЛЯМБДЫ` будет `ВЫВОДИТСЯ ЦЕЛЕВОГО контекста`. Если в интерфейсе `int[]` (массив), то в лямбде писать `n[]` НЕ НУЖНО, просто `n` 🛑
> ```java
> interface MyNumЬer { 
> 	int getValue(int[] array);
> }
> //Main
> MyNumber myNum = (n) -> {
> 	int result = 0;
> 	for(int i = 0; i < n.length; i++){
> 		result += n[i];
> 	}
> 	return result;
> }
> ```
> ⭐В этом случае целевой контекст - `int[]` и 💡потому типом `n` окажется `int[]`. 
> 
> Указывать тип параметра в виде `n[]` не нужно, да и недопустимо. 
> Было бы законно явно объявить параметр как `int [] n`, но здесь это ничего не даст.

## Основы синтаксиса и return

- `(params) -> expression` или `(params) -> { block }`.
    
- 🟣Если тело одно выражение → ❗❗❗`return` НЕЛЬЗЯ ПИСАТЬ. ❗❗❗
- 🟣Если блок → ❗❗❗`return` обязателен.❗❗❗
    
- Скобки вокруг одного параметра можно опустить (`n -> n % 2 == 0`).
    
- 🟣Типы параметров чаще всего выводятся, но если один явно указал, то все должны быть указаны

## Несколько примеров
```java
// Это лямбда-выражение получает псевдослучайное значение от Math.random () , умножает его на 100 и возвращает результат. 
// Оно не требует параметра.
() -> Math.random() * 100

//Такое лямбда-выражение возвращает true, если значение параметра n оказывается четным.
(n) -> (n % 2) == О
```

# 📕 2. Функциональные интерфейсы

Функциональный интерфейс должен содержать ровно один абстрактный метод:
```java
interface MyNumЬer { 
	double getValue();
}
```

В результате лямбда-выражение ==может быть указано только в контексте, ГДЕ ОПРЕДЕЛН ЦЕЛЕВОЙ ТИП. ==

Один из таких контекстов создается, когда лямбда-выражение присваивается ссылке на функциональный интерфейс.
```java
MyNumber myNum = () -> 123.45;

System.out.println(myNum.getValue()); // 123.45
```

# 📕3. Блочные лямбда-выражения
Когда нужно выполнить ==несколько операторов, используются блочные лямбда-выражения==:
```java
interface NumericFunc {
    int func(int n);
}

class BlockLambdaDemo {
    public static void main(String[] args) {
        // Вычисление факториала
        NumericFunc factorial = (n) -> {
            int result = 1;
            for(int i = 1; i <= n; i++)
                result = i * result;
            return result;
        };
        
        System.out.println("Факториал 5: " + factorial.func(5));
    }
}
```
