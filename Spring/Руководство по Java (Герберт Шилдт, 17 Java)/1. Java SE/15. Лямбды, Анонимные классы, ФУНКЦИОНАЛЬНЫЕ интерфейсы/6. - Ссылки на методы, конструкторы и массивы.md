С лямбда-выражениями связана одна важная возможность, которая называется `ссылкой на метод`. 

Ссылка на метод предлагает ==способ обращения к методу, не инициируя его выполнение==

> [!NOTE] ✅Синтаксис получения ссылки на метод `::`
> ```java
> className::method
> ```
> Разделитель `::` был добавлен к языку в JDK 8 специально для этой цели.

***
# 📕 1. Ссылки на STATIC-МЕТОДЫ
```JAVA
// Функциональный интерфейс
interface StringFunc {
    String func(String n);
}

// Класс, ссылка на статический метод которого будет получена
class MyStringOps {
    static String strReverse(String str) {
        String result = "";
        for(int i = str.length()-1; i >= 0; i--)
            result += str.charAt(i);
        return result;
    }
}

class MethodRefDemo {
	// Метод, в аргумент которого передается лямбда
    static String stringOp(StringFunc sf, String s) {
        return sf.func(s);
    }
    
    public static void main(String[] args) {
	    // ✅Получение ссылки на метод при помощи ::
        String outStr = stringOp(MyStringOps::strReverse, "Hello");
    }
}
```

# 📕 2. Ссылки на МЕТОДЫ экземпляров (`НЕ static-методы`)

```java
class MyStringOps {
	// Не static-метод класса
    String strReverse(String str) {
        String result = "";
        for(int i = str.length()-1; i >= 0; i--)
            result += str.charAt(i);
        return result;
    }
}

class MethodRefDemo2 {
    public static void main(String[] args) {
	    // Получение ссылки на метод только через объект этого класса
        MyStringOps strOps = new MyStringOps();
        String outStr = stringOp(strOps::strReverse, "Hello");
    }
}
```

Возможна также ситуация, когда желательно указать метод экземпляра, который можно применять с любым объектом заданного класса, а не только с указанным объектом. 

В таком случае ссылку на метод необходимо создать, точно также как для статического метода
```java
className::method
```

```java
import java.io.FileNotFoundException;

public class MyMain {
	static <T> int counter(T[] vals, MyFunc<T> f, T v) {
        int count = 0;
        for(int i = 0; i < vals.length; i++)
            if(f.func(vals[i], v))
                count++;
        return count;
    }
	
    public static void main(String[] args) throws FileNotFoundException  {
    	int count;
    	
    	// ✅Пример ссылки на НЕ СТАТИЧЕСКИЙ метод
    	MyFunc<HighTemp> f = HighTemp::sameTemp; // закометить для работы кода
    	
        // Создать массив объектов HighTemp.
        HighTemp[] weekDayHighs = { 
            new HighTemp(89), new HighTemp(82),
            new HighTemp(90), new HighTemp(89),
            new HighTemp(89), new HighTemp(91),
            new HighTemp(84), new HighTemp(83) 
        };
        
        // на метод экземпляра sameTemp().
        count = counter(weekDayHighs, HighTemp::sameTemp, new HighTemp(89));
        System.out.println("Количество суток, когда самая высокая температура была 89 градусов: " + count);
        
        
        
        // Создать и использовать еще один массив элементов типа HighTemp.
        HighTemp[] weekDayHighs2 = { 
            new HighTemp(32), new HighTemp(12),
            new HighTemp(24), new HighTemp(19),
            new HighTemp(18), new HighTemp(12),
            new HighTemp(-1), new HighTemp(13) 
        };
        
        count = counter(weekDayHighs2, HighTemp::sameTemp, new HighTemp(12));
        System.out.println("Количество суток, когда самая высокая температура была 12 градусов: " + count);
        
        // Использовать lessThanTemp() для нахождения суток, когда температура
        // была ниже указанного значения.
        count = counter(weekDayHighs, HighTemp::lessThanTemp, new HighTemp(89));
        System.out.println("Количество суток, когда самая высокая температура была меньше 89 градусов: " + count);
        
        count = counter(weekDayHighs2, HighTemp::lessThanTemp, new HighTemp(19));
        System.out.println("Количество суток, когда самая высокая температура была меньше 19 градусов: " + count);
    }
}

interface MyFunc<T> {
    boolean func(T v1, T v2);
}

class HighTemp {
    private int hTemp;
    
    HighTemp(int ht) { 
        hTemp = ht; 
    }
    
    boolean sameTemp(HighTemp ht2) {
        return hTemp == ht2.hTemp;
    }
    
    boolean lessThanTemp(HighTemp ht2) {
        return hTemp < ht2.hTemp;
    }
}
```

# 📕 3. Ссылки на ОБОБЩЕННЫЕ МЕТОДЫ
```java
// Демонстрация использования ссылки на обобщенный метод,
// объявленный внутри необобщенного класса.
// Функциональный интерфейс, который работает с массивом
// и значением и возвращает результат int.
interface MyFunc<T> {
    int func(T[] vals, T v);
}

// В этом классе определен метод по имени countMatching(), который
// возвращает количество элементов в массиве, равных указанному значению.
// 💡 Обратите внимание, что метод countMatching() является обобщенным 💡,
// но класс MyArrayOps - нет.
class MyArrayOps {
    static <T> int countMatching(T[] vals, T v) {
        int count = 0;
        for(int i = 0; i < vals.length; i++)
            if(vals[i] == v)
                count++;
        return count;
    }
}

class GenericMethodRefDemo {
    // Первый параметр этого метода имеет тип функционального интерфейса MyFunc.
    // В остальных двух параметрах он принимает массив и значение, оба типа T.
    static <T> int myOp(MyFunc<T> f, T[] vals, T v) {
        return f.func(vals, v);
    }
    
    public static void main(String[] args) {
        Integer[] vals = { 1, 2, 3, 4, 2, 3, 4, 4, 5 };
        String[] strs = { "One", "Two", "Three", "Two" };
        int count;
        
        // ✅ Ссылка на ОБОБЩЕННЫЙ метод НЕОБОБЩЕННОГО класса ✅
        count = myOp(MyArrayOps::<Integer>countMatching, vals, 4);
        System.out.println("Количество элементов 4, содержащихся в vals: " + count);
        
        count = myOp(MyArrayOps::<String>countMatching, strs, "Two");
        System.out.println("Количество элементов Two, содержащихся в strs: " + count);
    }
}
```

# Пример использования ссылок на методы в CollectionsFramework
```java
// Использование ссылки на метод при поиске максимального значения в коллекции.
import java.util.*;

class MyClass {
    private int val;
    
    MyClass(int v) { 
        val = v; 
    }
    
    int getVal() { 
        return val; 
    }
}

class UseMethodRef {
    // Метод compareMC(), совместимый с методом compare(),
    // который определен в Comparator<T>.
    static int compareMC(MyClass a, MyClass b) {
        return a.getVal() - b.getVal();
    }
    
    public static void main(String[] args) {
        ArrayList<MyClass> al = new ArrayList<MyClass>();
        al.add(new MyClass(1));
        al.add(new MyClass(4));
        al.add(new MyClass(2));
        al.add(new MyClass(9));
        al.add(new MyClass(3));
        al.add(new MyClass(7));
        
        // Найти максимальное значение в al, используя метод compareMC().
        MyClass maxValObj = Collections.max(al, UseMethodRef::compareMC);
        System.out.println("Максимальное значение равно: " + maxValObj.getVal());
    }
}
```

Для поиска максимума, используется метод `Collections.max()`, который принимает объкт коллекции, а вторым параметром реализацию `Comparator`.

Лямбды упрощают работу тем, что можно написать просто метод, который сравнивает объекты (должен совпадать по сигнатуре с методом `Comarator.compareTo()`) и передать его по ссылке в метод `max`. 

Тем самым не нежно создавать отдельную реализацию `Comparator`

# 📕 4. Ссылки на конструкторы
Подобно ссылкам на методы можно создавать ссылки на конструкторы.

```java
className::new;
```
Такую ссылку можно присваивать ==любой ссылке на функциональный интерфейс==, в котором определен ==метод, совместимый с конструктором==.

```java
interface MyFunc {
	// ✅ Совпадает по параметрам с одним из конструкторов MyClass
    MyClass func(int n); 
}

class MyClass {
    private int val;
    MyClass(int v) { val = v; }
    int getVal() { return val; }
}

class ConstructorRefDemo {
    public static void main(String[] args) {
        MyFunc myClassCons = MyClass::new;
        
        MyClass mc = myClassCons.func(100);
        System.out.println("Значение: " + mc.getVal());
    }
}
```

## Ссылка на конструктор обобщенного класса
```java
// Демонстрация использования ссылки на конструктор обобщенного класса.
// Теперь MyFunc - обобщенный функциональный интерфейс.
interface MyFunc<T> {
    MyClass<T> func(T n);
}

class MyClass<T> {
    private T val;
    
    // Конструктор, принимающий аргумент.
    MyClass(T v) { 
        val = v; 
    }
    
    // Стандартный конструктор.
    MyClass() { 
        val = null; 
    }
    
    T getVal() { 
        return val; 
    }
}

class ConstructorRefDemo2 {
    public static void main(String[] args) {
        // Создать ссылку на конструктор MyClass<T>.
        MyFunc<Integer> myClassCons = MyClass<Integer>::new;
        
        // Создать экземпляр MyClass<T> через эту ссылку на конструктор.
        MyClass<Integer> mc = myClassCons.func(100);
        
        // Использовать только что созданный экземпляр MyClass<T>.
        System.out.println("val в mc равно " + mc.getVal());
    }
}
```

## Ссылки на массив
Важно упомянуть о существовании второй формы синтаксиса ссылок на конструкторы, которая предназначена для массивов. 

Чтобы создать ссылку на конструктор для массива, используйте такую конструкцию:
```java
тип[]::new
```

```java
import java.util.function.IntFunction;

public class ArrayConstructorReferenceExample {
    public static void main(String[] args) {
        // Ссылка на конструктор массива строк
        IntFunction<String[]> stringArrayCreator = String[]::new;
        
        // Создание массива строк длиной 3
        String[] stringArray = stringArrayCreator.apply(3);
        stringArray[0] = "Hello";
        stringArray[1] = "World";
        stringArray[2] = "Java";
        
        System.out.println("Массив строк: " + java.util.Arrays.toString(stringArray));
        
        
        
        // Ссылка на конструктор массива целых чисел
        IntFunction<Integer[]> intArrayCreator = Integer[]::new;
        
        // Создание массива целых чисел длиной 5
        Integer[] intArray = intArrayCreator.apply(5);
        intArray[0] = 10;
        intArray[1] = 20;
        intArray[2] = 30;
        intArray[3] = 40;
        intArray[4] = 50;
        
        System.out.println("Массив целых чисел: " + java.util.Arrays.toString(intArray));
        
        
        
        // Ссылка на конструктор массива собственного класса
        IntFunction<Person[]> personArrayCreator = Person[]::new;
        
        // Создание массива Person длиной 2
        Person[] personArray = personArrayCreator.apply(2);
        personArray[0] = new Person("Alice", 25);
        personArray[1] = new Person("Bob", 30);
        
        System.out.println("Массив Person: " + java.util.Arrays.toString(personArray));
    }
}

class Person {
    private String name;
    private int age;
    
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
    
    @Override
    public String toString() {
        return name + "(" + age + ")";
    }
}
```
