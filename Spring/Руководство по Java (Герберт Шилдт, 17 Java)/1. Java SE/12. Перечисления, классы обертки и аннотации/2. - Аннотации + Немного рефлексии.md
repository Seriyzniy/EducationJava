Язык Java предлагает средство, позволяющее ==встраивать дополнительную информацию в файл== исходного кода. 

Такая информация, ==называемая аннотацией==, НЕ МЕНЯЕТ действия программы, оставляя семантику программы неизменной. (также называется *метаданные*)

# 📕 1. Основы аннотаций
==Аннотация создается== с помощью механизма, ==основанного на интерфейсе.==
```java
// Простой тип аннотации. 
@interface MyAnno { 
	String str (); 
	int val(); 
}
```

Первым делом обратите внимание ==на символ @ перед ключевым словом== `interface`. Он ==сообщает компилятору== о том, ==что объявляется тип аннотации==. 

⭐⭐⭐Далее обратите внимание на два члена, `str()` и `val()`. ==Все аннотации состоят исключительно из объявлений методов. ==
- Тем не менее, вы НЕ ПРЕДОСТАВЛЯЕТЕ ТЕЛА для этих методов. 
- Взамен их реализует компилятор Java. 
- Более того, как вы увидите, методы во многом похожи на поля.

> [!NOTE] 💡💡Установка - как поле, Получение значение - как метод💡💡
> Для установки значений: имяЧлена = значение
> Для получения значений (как метод): имяЧлена() 

### Аннотации автоматически расширяют интерфейс `Annotation`
Аннотация НЕ МОЖЕТ содержать конструкцию `extends`.
Однако ==все типы аннотаций== автоматически ==расширяют интерфейс== `Annotation`, который является суперинтерфейсом для всех аннотаций. 

Он объявлен в пакете `java.lang.annotation` и переопределяет методы `hashCode()` , `equals()` и `toString()` , определенные в классе `Object`. 

Интерфейс Annotation также задает `annotationType()` , который возвращает объект Class, представляющий вызывающую аннотацию.

### Использование аннотаций
Скажем, можно аннотировать классы, методы, поля, параметры и константы перечислений. Аннотировать допускается даже саму аннотацию. 
Во всех ситуациях аннотация предшествует остальной части объявления

==В случае применения аннотации ее элементам присваиваются значения==. 
Например, вот пример применения аннотации MyAnno к объявлению метода:
```java
//Аннотировать метод. 
@MyAnno (str = "Пример аннотации", val = 100) 
public static void myMeth () ( //...
```
Здесь аннотация `MyAnno` связывается с методом `myMeth()`. 

Внимательно взгляните на синтаксис аннотации. За именем аннотации, которому предшествует символ @, находится заключенный в круглые скобки список инициализаций членов. 

==Чтобы предоставить члену значение, его понадобится присвоить имени члена.== Следовательно, в приведенном выше примере члену `str` аннотации `MyAnno` присваивается строка `"Пример аннотации".`

## 📙 1.1 ⭐⭐⭐Указание политики хранения⭐⭐⭐
Перед дальнейшим исследованием аннотаций необходимо обсудить политики хранения аннотаций. 

==Политика хранения устанавливает момент, когда аннотация ОТБРАСЫВАЕТСЯ.== 
В Java определены ТРИ ПОЛИТИКИ, которые инкапсулированы внутри перечисления `java.lang.annotation.RetentionPolicy` 
1. Аннотация с политикой хранения ==SOURCE== удерживается ==только в файле исходноrо кода и на этапе компиляции отбрасывается==. 
	
2. Аннотация с политикой хранения ==CLASS== ==на этапе компиляции сохраняется в файле . class.== Однако она ==НЕ ДОСТУПНА== через машину JVM ==во время выполнения==. 
	
3. Аннотация с политикой хранения ==RUNTIME== на этапе ==компиляции сохраняется в файле . class и доступна через машину JVM во время выполнения==. Таким образом, политика RUNTIME обеспечивает наивысшее по стоянство аннотаций.

==Политика хранения== для аннотации ==указывается с применением== одной из встроенных аннотаций Java: `@Retention`. Вот ее общая форма: `@Retention(retention-policy)`
```java
@Retention(RetentionPolicy.RUNTIME) 
@interface MyAnno(
	String str(); 
	int val();
}
```

## 📙 1.2 Получение аннотаций во время выполнения с использованием РЕФЛЕКСИИ

> [!NOTE] Аннотации в основном для других инструментов разработки
> Аннотации предназначены rлавным образом для использования друrими инструментами разработки или развертывания. 

Тем не менее, если для аннотаций определена политика хранения `RUNТIME`, тоrда их ==можно запрашивать во время выполнения с помощью любой проrраммы на Java== через `рефлексию`

> [!NOTE] Рефлексия - средство получения инфы о классе во время выполнения
> Рефлексия представляет собой средство, позволяющее получать информацию о классе во время выполнения. 
> 
АРI-интерфейс рефлексии содержится в пакете `java.lang.reflect`. 

Существует несколько способов работы с рефлексией, но все они в rлаве рассматриваться не будут. Однако мы обсудим ряд приме ров, применимых к аннотациям.

### Использование рефлексии
==Первым шагом== к использованию рефлексии ==является получение объекта== `Class`, представляющего класс, аннотации которого вы хотите получить. 

`Class` - один из встроенных классов Java, определенный в пакете `java.lang`

Один из самых простых способов предусматривает вызов метода `getClass()`, определенный в `Object` со следующей общей формой: 
```java
final Class getClass() 
```

Метод getClass () возвращает объект Class, который представляет вызывающий объект.

После того, как объект `Class` получен, ==с помощью его методов можно просматривать информацию о различных элементах==, объявленных в классе, в том числе об аннотациях. 

При желании извлечь аннотации, которые ассоциированы с конкретным элементом, объявленным в классе, сначала потребуется получить объект, представляющий данный элемент. 

Например, `Class` ==предлагает== (помимо прочих) ==методы== `getMethod ()`,` getField ()` и `getConstructor ()`, предназначенные для получения информации о методе, поле и конструкторе соответственно. 
Эти методы возвращают объекты типа `Method`, `Field` и `Constructor`.

### Пример получения аннотации метода
1. 🔶Первым делом нужно получить объект `Class` при помощи `getClass()`, представляющий класс, а затем вызвать для этого объекта `Class` метод `getMethod()`, указав имя метода.

2. 🔶Метод `getMethod()` имеет следую щую общую форму:
```java
Method getMethod(String methName, Class ... paramTypes)
```
Метод `getMethod()` возвращает объект `Method`, представляющий метод. Если метод не может быть найден, тогда генерируется исключение `NoSuchМethodException`.

- Имя метода передается в аргументе `methName`. 
- Если у метода есть аргументы, то объекты `Class`, представляющие типы аргументов, также должны быть указаны в аргументе `paramTypes`. 

Обратите внимание, что `paramTypes` **является параметром с переменным числом аргументов**, т.е. можно задавать столько типов параметров, сколько необходимо, включая ноль

3. 🔶 Вызывая метод `getAnnotation()` , из объекта `Class`, `Method`, `Field` или `Constructor` можно получить специфическую аннотацию, ассоциированную с этим объектом. 

Вот общая форма `getAnnotation()` :
```java
<А extends Annotation> getAnnotation(Class annoType)
```

#### Пример с одним параметром метода
```java
import java.lang.annotation.*;
import java.lang.reflect.Method;

// Объявление аннотации типа.
@Retention(RetentionPolicy.RUNTIME)
@interface MyAnno {
    String str();
    int val();
}

class Meta {
    // Аннотировать метод.
    @MyAnno(str = "Пример аннотации", val = 100)
    public static void myMeth() {
        Meta ob = new Meta();
        // Получить аннотацию для этого метода
        // и отобразить значения ее членов.
        try {
            // Для начала получить объект Class,
            // который представляет данный класс.
            Class<?> c = ob.getClass();
            
            // Теперь получить объект Method,
            // который представляет данный метод.
            Method m = c.getMethod("myMeth");
            
            // Далее получить аннотацию для этого класса.
            //🛑🛑🛑Внимание, в аргумент передается тип Class🛑🛑🛑
            MyAnno anno = m.getAnnotation(MyAnno.class);  
            
            // В заключение вывести значения.
            //🛑🛑🛑Внимание,получение значений как ВЫЗОВ метода🛑🛑🛑
            System.out.println(anno.str() + " " + anno.val());
            
        } catch (NoSuchMethodException exc) {
            System.out.println("Метод не найден.");
        }
    }
    
    public static void main(String[] args) {
        myMeth();
    }
}
```

#### Пример с ДВУМЯ параметрами метода
```java
import java.lang.annotation.*;
import java.lang.reflect.Method;

@Retention(RetentionPolicy.RUNTIME)
@interface MyAnno {
    String str();
    int val();
}

class Meta {
    // Метод myMeth теперь принимает два аргумента.
    @MyAnno(str = "Два параметра", val = 19)
    public static void myMeth(String str, int i) {
        Meta ob = new Meta();
        try {
            Class<?> c = ob.getClass();
            // Здесь указываются типы параметров.
            Method m = c.getMethod("myMeth", String.class, int.class);
            
            MyAnno anno = m.getAnnotation(MyAnno.class);
            System.out.println(anno.str() + " " + anno.val());
        } catch (NoSuchMethodException exc) {
            System.out.println("Метод не найден.");
        }
    }
    
    public static void main(String[] args) {
        myMeth("тест", 10);
    }
}
```

### Получение всех аннотаций
Чтобы получить ВСЕ АННОТАЦИИ с политикой хранения RUNТIME, ассоциированные с элементом, можно вызвать на данном элементе метод `getAnnotationS()` , который имеет приведенную ниже общую форму: 
```java
Annotation[] getAnnotations() // S на конце метода 
```

Метод getAnnotations() ==возвращает массив аннотаций== и может быть вы зван на объектах типа `Class`, `Method`, `Constructor` и `Field`
```java
// Отображение всех аннотаций для класса и метода.
import java.lang.annotation.*;
import java.lang.reflect.*;

@Retention(RetentionPolicy.RUNTIME)
@interface MyAnno {
    String str();
    int val();
}

@Retention(RetentionPolicy.RUNTIME)
@interface What {
    String description();
}

@What(description = "Аннотация класса")
@MyAnno(str = "Meta2", val = 99)
class Meta2 {
    @What(description = "Аннотация метода")
    @MyAnno(str = "Testing", val = 100)
    public static void myMeth() {
        Meta2 ob = new Meta2();
        try {
            Annotation[] annos = ob.getClass().getAnnotations();
            // Отобразить все аннотации для Meta2.
            System.out.println("Все аннотации для класса Meta2:");
            for(Annotation a : annos)
                System.out.println(a);
            System.out.println();
            
            // Отобразить все аннотации для myMeth.
            Method m = ob.getClass().getMethod("myMeth");
            annos = m.getAnnotations();
            System.out.println("Все аннотации для метода myMeth:");
            for(Annotation a : annos)
                System.out.println(a);
        } catch(NoSuchMethodException exc) {
            System.out.println("Метод не найден.");
        }
    }
    
    public static void main(String[] args) {
        myMeth();
    }
}

Все аннотации для класса Meta2: 
@What(description=Aннoтaция класса) 
@MyAnno (str=Meta2, val=99) 

Все аннотации для метода myMeth: 
@What(description=Aннoтaция метода) 
@MyAnno (str=Testing, val=lOO)
```

## 📙 1.3 Интepфeйc AnnotatedElement

Интерфейс `AnnotatedElement` (пакет `java.lang.reflect`) предоставляет методы для работы с аннотациями через рефлексию. Его реализуют классы `Method`, `Field`, `Constructor`, `Class`, `Package`.

```java
// Возвращает все аннотации элемента (включая унаследованные)
Annotation[] getAnnotations();

// Возвращает все НЕУНАСЛЕДОВАННЫЕ аннотации элемента  
Annotation[] getDeclaredAnnotations();

// Проверяет наличие конкретной аннотации
boolean isAnnotationPresent(Class<? extends Annotation> annoType);

// Возвращает конкретную аннотацию (с JDK 8)
<T extends Annotation> T getDeclaredAnnotation(Class<T> annoType);

// Возвращает аннотации по типу (работают с повторяющимися аннотациями)
<T extends Annotation> T[] getAnnotationsByType(Class<T> annoType);
<T extends Annotation> T[] getDeclaredAnnotationsByType(Class<T> annoType);
```

## 📙 1.4 Использование стандартных значений
Для членов аннотации ==можно задавать значения по умолчанию== с помощью ключевого слова `default`.
```java
import java.lang.annotation.*;
import java.lang.reflect.*;

@Retention(RetentionPolicy.RUNTIME)
@interface MyAnno {
    String str() default "Тест";
    int val() default 9000;
}

class Meta3 {
    // Аннотировать метод с использованием стандартных значений
    @MyAnno()
    public static void myMeth() {
        Meta3 ob = new Meta3();
        try {
            Class<?> c = ob.getClass();
            Method m = c.getMethod("myMeth");
            MyAnno anno = m.getAnnotation(MyAnno.class);
            System.out.println(anno.str() + " " + anno.val());
        } catch (NoSuchMethodException exc) {
            System.out.println("Метод не найден.");
        }
    }
    
    public static void main(String[] args) {
        myMeth();
    }
}
```

## 📙1.5 Маркерные аннотации
**Маркерные аннотации** - это специальный тип аннотаций, которые ==не содержат членов== (параметров). Их единственная ==цель - пометить элемент==, чтобы указать на его особое ==состояние или поведение.==

```java
import java.lang.annotation.*;
import java.lang.reflect.*;

// Маркерная аннотация
@Retention(RetentionPolicy.RUNTIME)
@interface MyMarker {}

class Marker {
    // Аннотировать метод, используя маркерную аннотацию
    // Скобки () не требуются
    @MyMarker
    public static void myMeth() {
        Marker ob = new Marker();
        try {
            Method m = ob.getClass().getMethod("myMeth");
            
            // Проверка наличия аннотации
            if(m.isAnnotationPresent(MyMarker.class))
                System.out.println("MyMarker присутствует.");
                
        } catch (NoSuchMethodException exc) {
            System.out.println("Метод не найден.");
        }
    }
    
    public static void main(String[] args) {
        myMeth();
    }
}

MyMarker присутствует.
```

💡 **Ключевые моменты:**
- Маркерные аннотации не имеют параметров
- Для проверки используется `isAnnotationPresent()`
    
- Скобки `()` не обязательны: `@MyMarker` вместо `@MyMarker()`

## 📙1.6 Однозлементные аннотации

**Однозлементные аннотации** ==содержат только один член с именем== `value`, что позволяет использовать сокращенный синтаксис.

```java
import java.lang.annotation.*;
import java.lang.reflect.*;

// Однозлементная аннотация
@Retention(RetentionPolicy.RUNTIME)
@interface MySingle {
    int value(); // Именем члена должно быть value
}

class Single {
    // Аннотировать метод, используя однозлементную аннотацию
    @MySingle(100) // 🛑value = не обязательно
    public static void myMeth() {
        Single ob = new Single();
        try {
            Method m = ob.getClass().getMethod("myMeth");
            MySingle anno = m.getAnnotation(MySingle.class);
            System.out.println(anno.value()); // выводит 100
        } catch (NoSuchMethodException exc) {
            System.out.println("Метод не найден.");
        }
    }
    
    public static void main(String[] args) {
        myMeth();
    }
}
```

# 📕2. Встроенные аннотации Java
Java предоставляет несколько встроенных аннотаций общего назначения:
### 1. Аннотации из пакета java.lang.annotation:
- **`@Retention`** - определяет политику хранения аннотации
- **`@Documented`** - указывает, что аннотация должна быть документирована
    
- **`@Target`** - задает типы элементов, к которым может применяться аннотация.
Она спроектирована ==для использования только в качестве аннотации к другой аннотации.== Аннотация @Target принимает один аргумент, представляющий собой массив констант перечисления ElementType 👇👇👇

|Константа|Где применяется|Пояснение|
|---|---|---|
|`ANNOTATION_TYPE`|Другая аннотация|Аннотации для аннотаций (мета-аннотации).|
|`CONSTRUCTOR`|Конструктор|Аннотация для конструкторов.|
|`FIELD`|Поле|Поля класса (включая enum-константы).|
|`LOCAL_VARIABLE`|Локальная переменная|Переменные внутри методов и блоков.|
|`METHOD`|Метод|Методы класса или интерфейса.|
|`MODULE`|Модуль|Используется для модулей (`module-info.java`), добавлено в JDK 9.|
|`PACKAGE`|Пакет|Аннотация для пакетов (в `package-info.java`).|
|`PARAMETER`|Параметр|Параметры методов или конструкторов.|
|`RECORD_COMPONENT`|Компонент записи|Специально для `record` (с JDK 16).|
|`TYPE`|Класс, интерфейс или перечисление|Любые типы: классы, интерфейсы, перечисления.|
|`TYPE_PARAMETER`|Параметр типа|Аннотации для параметров обобщений (`<T>`).|
|`TYPE_USE`|Использование типа|Аннотации для использования типа (например, `@Nonnull String`).|

- **`@Inherited`** - позволяет наследование аннотации в подклассах
### 2. Аннотации из пакета java.lang:
- **`@Override`** - гарантирует, что метод переопределяет метод суперкласса
- **`@Deprecated`** - указывает, что элемент устарел
    
- **`@FunctionalInterface`** - маркирует функциональный интерфейс
    
- **`@SafeVarargs`** - подавляет предупреждения для параметров переменной длины
- **`@SuppressWarnings`** - подавляет указанные предупреждения компилятора

# 📕 3. Повторяющиеся аннотации

Начиная с JDK 8, аннотации можно применять многократно к одному элементу.

Чтобы аннотацию можно было повторять, ее потребуется снабдить аннотацией `@Repeatabe`, определенной в пакете java. lang.annotation, и указать в ее поле value тип контейнера для повторяющейся аннотации.

# 📕 4. Ограничения аннотаций

1. Аннотации не могут наследоваться от других аннотаций
    
2. Методы аннотаций не могут принимать параметры
    
3. Методы должны возвращать один из допустимых типов:
    
    - Примитивные типы (int, double и т.д.)
        
    - String или Class
        
    - Enum
        
    - Другая аннотация
        
    - Массив допустимых типов
        
4. Аннотации не могут быть обобщенными
    
5. Методы аннотаций не могут объявлять исключения (throws)
