`FileWriter` — это наследник `OutputStreamWriter`, специализированный ==для записи **символов в файл**.==

Под капотом он создаёт `FileOutputStream` и оборачивает его в `OutputStreamWriter`.

Особенности:
- Работает **только с файлами** (в отличие от `OutputStreamWriter`, который можно прикрутить к любому `OutputStream`).
    
- По умолчанию использует **системную кодировку** (⚠️ то же самое, что и `OutputStreamWriter` без указания charset).
    
- 💡Может открываться в режиме **append** (дозапись в конец файла).

***
## Конструкторы
```java
FileWriter(String fileName) throws IOException
// Создаёт новый FileWriter для файла fileName.
// Если файл существует — он будет перезаписан.

FileWriter(String fileName, boolean append) throws IOException
// fileName — имя файла.
// append = true → запись будет добавляться в конец файла.


FileWriter(File file) throws IOException
// То же самое, но принимает объект File.

FileWriter(File file, boolean append) throws IOException
// File + режим дозаписи.


FileWriter(FileDescriptor fd)
// fd — файловый дескриптор (например, можно писать прямо в System.out).
```
⚠️ Все эти конструкторы используют **кодировку по умолчанию системы** (Locale/JVM).
## Методы
Наследуются от `OutputStreamWriter` и `Writer`:
```java
void write(int c) throws IOException
// Записывает один символ.

void write(char[] cbuf, int off, int len) throws IOException
// Записывает часть массива символов.

void write(String str) throws IOException
// Записывает строку.

void write(String str, int off, int len) throws IOException
// Записывает подстроку строки.


void flush() throws IOException
// Сбрасывает буфер (если есть) в файл.

void close() throws IOException
// Закрывает FileWriter и связанный FileOutputStream.
```

***
## Особенности

- ✅ ⭐Очень удобен, если просто нужно писать текст в файл «по-быстрому».
    
- ✅ Поддерживает режим дозаписи (`append`).
    
- ⚠️ Всегда использует **системную кодировку** → если важно, чтобы текст был в `UTF-8`, лучше использовать `OutputStreamWriter`.
    
- ⚠️ При большом объёме мелких записей желательно оборачивать в `BufferedWriter` для повышения производительности.
***
## Пример
```java
import java.io.FileWriter;
import java.io.IOException;

public class FileWriterExample {
    public static void main(String[] args) {
        // try-with-resources автоматически закроет поток
        try (FileWriter fw = new FileWriter("example.txt", true)) { // append = true
            fw.write("Hello, world!\n");
            fw.write("Привет, мир!\n");
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

***
## Best-Practices

- ✅ Если нужен **UTF-8 или другая кодировка** → использовать `OutputStreamWriter` вместо `FileWriter`.
    
- ✅ Для **частых мелких записей** → комбинировать с `BufferedWriter`:
```java
new BufferedWriter(new FileWriter("file.txt"))
```

- ✅ Для удобного построчного вывода → `PrintWriter` поверх `FileWriter`.
    
- ❌ Не использовать `FileWriter` для бинарных данных (он работает только с символами).
    
- ❌ Не полагаться на системную кодировку в продакшене.
