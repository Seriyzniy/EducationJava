`BufferedWriter` — это наследник `Writer`, который ==добавляет **буферизацию символов при записи**. == 

Вместо того чтобы писать каждый символ сразу в целевой поток (например, файл), `BufferedWriter` сначала накапливает данные во внутреннем буфере, и только когда буфер заполнен или вызывается `flush()`, данные реально записываются в выходной поток.

Это уменьшает количество обращений к диску/сети и **существенно повышает производительность** при работе с текстовыми данными.

***
## Конструкторы
```java
BufferedWriter(Writer out)
// out — целевой Writer, в который будут записываться данные.
// Размер буфера по умолчанию = 8192 символов.

BufferedWriter(Writer out, int sz)
// out — целевой Writer.
// sz — размер внутреннего буфера (в символах).
```

## Методы
```java
void write(int c) throws IOException
// Записывает один символ (в буфер).
// Запись в реальный поток произойдет только при переполнении буфера или flush().

void write(char[] cbuf, int off, int len) throws IOException
// Записывает подмассив символов из cbuf начиная с позиции off, длиной len.

void write(String s)
// ✅ Записывает строку

void write(String s, int off, int len) throws IOException
// Записывает подстроку строки s начиная с off, длиной len.


void newLine() throws IOException
// 💡Записывает символ новой строки.💡
// Использует 💡системно-зависимый разделитель строк (System.lineSeparator()).

void flush() throws IOException
// 💡Принудительно сбрасывает данные из буфера в целевой поток.

void close() throws IOException
// Сбрасывает буфер и закрывает поток.
// После этого Writer использовать нельзя.
```

***
## Особенности

- ✅ Уменьшает количество вызовов `write()` в базовом Writer (например, `FileWriter`).
    
- ✅ Позволяет удобно писать строки построчно через `newLine()`.
    
- ⚠️ Обязательно вызывать `flush()` перед `close()`, если важно, чтобы данные точно были записаны.
    
- ⚠️ Если размер записываемых данных превышает размер буфера, `BufferedWriter` сам выполнит `flush()`.

***
## Пример
```java
import java.io.BufferedWriter;
import java.io.FileWriter;
import java.io.IOException;

public class BufferedWriterExample {
    public static void main(String[] args) {
        try (BufferedWriter bw = new BufferedWriter(new FileWriter("output.txt"))) 
        {
            bw.write("Hello");
            bw.newLine(); // перенос строки
            bw.write("World!");
            // Данные пока в буфере, реально они в файле появятся только после flush/close
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

***
## Best-Practices

- ✅ Всегда использовать `BufferedWriter`, если пишете **много символов или строк в файл** — это в разы быстрее, чем голый `FileWriter`.
    
- ✅ Использовать `newLine()` вместо `"\n"`, чтобы код был кроссплатформенным.
    
- ✅ Всегда закрывать поток (`try-with-resources` решает проблему).
    
- ⚠️ Если пишете очень большие куски текста — можно увеличить буфер (например, 64KB).
    
- ❌ Не забывать, что вызов `close()` автоматически вызывает `flush()`.
