`FilterReader` — это ==**абстрактный класс** ==из `java.io`, который реализует паттерн _Decorator_ для `Reader`.  

Он ==нужен== не для того, чтобы его использовать напрямую, а ==как **базовый класс для фильтрующих ридеров**==. 
То есть он оборачивает другой `Reader` и делегирует ему вызовы, позволяя при этом "вставить" дополнительную логику.

Прямо его ты никогда не создашь (конструктор `protected`). Он — чисто **скелет**, который переиспользует`PushbackReader` и кастомные наследники, если тебе нужно сделать свой фильтр (например, преобразовывать символы на лету).
## Конструкторы
```java
protected FilterReader(Reader in)
// in — базовый Reader, который будет обёрнут этим фильтром.
// Все операции чтения делегируются этому объекту.
```

## Методы
```java
// Читает один символ, возвращает код символа (0–65535) или -1 при EOF.
// Делегирует вызов в wrapped Reader.
public int read() throws IOException

// Читает данные в массив символов cbuf.
// Возвращает количество реально прочитанных символов, либо -1 при EOF.
public int read(char[] cbuf, int off, int len) throws IOException

// Проверяет, готов ли поток для чтения (не блокирующий вызов).
// Делегирует вызов во внутренний Reader.
public boolean ready() throws IOException

// Закрывает внутренний Reader.
public void close() throws IOException

// Маркирует текущую позицию в потоке (если поддерживается).
// Делегирует вызов wrapped Reader.
public void mark(int readAheadLimit) throws IOException

// Возвращает true, если mark/reset поддерживаются.
public boolean markSupported()

// Возвращает текущую позицию (не символ, а просто откат к mark()).
// Делегирует wrapped Reader.
public void reset() throws IOException

// Пропускает n символов вперёд. Возвращает количество реально пропущенных.
public long skip(long n) throws IOException
```

***
## Особенности

- **Абстрактный** — не используется напрямую.
    
- Всегда делегирует вызовы во внутренний `Reader`.
    
- ⭐Это "скелет-декоратор": его смысл — **писать наследников**.
    
- Именно через него построены все "умные" ридеры (`BufferedReader`, `PushbackReader`, `LineNumberReader`).
    
- Без него тебе пришлось бы вручную копировать делегирование.

***

## 💡 Пример (⭐разобрать как работает Декоратор)
Пример кастомного фильтра на основе `FilterReader`:
```java
import java.io.*;

// Фильтр, который преобразует все буквы в верхний регистр
class UpperCaseReader extends FilterReader {
    public UpperCaseReader(Reader in) {
        super(in);
    }

    @Override
    public int read() throws IOException {
        int c = super.read();
        return (c == -1 ? -1 : Character.toUpperCase((char) c));
    }
}

public class FilterReaderExample {
    public static void main(String[] args) throws IOException {
        try (Reader reader = new UpperCaseReader(new StringReader("Hello world"))) {
            int ch;
            while ((ch = reader.read()) != -1) {
                System.out.print((char) ch); // HELLO WORLD
            }
        }
    }
}

```

***

## Best-Practices

- ✅**Не используй `FilterReader` напрямую** — только через наследников.
    
- ✅Если нужен буфер и удобные методы (`readLine`) — используй `BufferedReader`.
    
- Если нужен возврат символов — `PushbackReader`.
    
- Если надо трекать номера строк — `LineNumberReader`.
    
- ⭐⭐⭐Для кастомной фильтрации (`шифрование`, `транслитерация`, `uppercase/lowercase`) — **наследуй `FilterReader`**.⭐⭐⭐
    
- Не забывай закрывать поток (`try-with-resources`).
