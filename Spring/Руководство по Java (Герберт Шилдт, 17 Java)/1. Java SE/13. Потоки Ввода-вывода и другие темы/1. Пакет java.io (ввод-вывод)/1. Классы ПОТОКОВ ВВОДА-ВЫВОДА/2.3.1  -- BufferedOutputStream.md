`BufferedOutputStream` — это ==**декоратор над другим ПОТОКОМ== `OutputStream`**, который ==добавляет **буферизацию при записи**==.  

> [!NOTE] Данные копятся в массиве, перед записью
> То есть: данные сначала копятся во внутреннем массиве (буфере), и только когда он заполняется или вызывается `flush()`/`close()`, данные реально уходят во вложенный поток (например, в файл или сокет).

Зачем нужен:
- 📈 Увеличивает производительность — вместо множества мелких вызовов `write()` в ОС делает меньше обращений большими порциями.
    
- Особенно эффективен при записи в **файлы** или **сети**, где системные вызовы дорогие.

***
## Конструкторы
```java
// Создаёт буферизированный поток с 💡буфером 8192 байта (по умолчанию)
// param out — поток для записи (например, FileOutputStream)
public BufferedOutputStream(OutputStream out)

// Создаёт буферизированный поток с указанным размером буфера
// 💡 param out — поток для записи
// 💡 param size — размер буфера (в байтах)
public BufferedOutputStream(OutputStream out, int size)
```

## Методы
```java
// Записывает один байт в буфер.
// Если буфер заполнен → сбрасывает его в out.
public synchronized void write(int b) throws IOException

// Записывает массив байтов (целый).
// Если размер больше буфера → сбрасывает напрямую в out.
public synchronized void write(byte[] b, int off, int len) throws IOException

// Принудительно сбрасывает буфер в out.
// Нужно вызывать, чтобы данные точно оказались в файле/сокете.
public synchronized void flush() throws IOException

// Закрывает поток: сначала flush(), потом закрывает out.
public void close() throws IOException
```

***
##  💡 Особенности 💡
- Размер буфера по умолчанию — **8192 байта (8 КБ)**, но можно задать свой.
    
- Если писать **мелкими кусками (например, по 1 байту)** → разница с `FileOutputStream` огромна (буфер снижает накладные расходы).
    
- Если писать большими массивами (сотни килобайт за раз) → выгода от буфера минимальна.
    
- ❗ `flush()` обязателен, если нужно гарантировать, что данные уже на диске/в сокете.
    
- ❗ После `close()` буфер автоматически сбрасывается.

***
## Пример
```java
import java.io.*;

public class BufferedOutputStreamExample {
    public static void main(String[] args) {
        String text = "Hello, BufferedOutputStream!\n";

        try (FileOutputStream fos = new FileOutputStream("output.txt");
             BufferedOutputStream bos = new BufferedOutputStream(fos)) 
        {

            // Записываем строку байтами
            bos.write(text.getBytes("UTF-8"));

            // Принудительно сбрасываем (хотя close() тоже это сделает)
            bos.flush();

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

```

***
## ✅ Best-Practices

- ✅ Оборачивай «медленные» потоки (`FileOutputStream`, `Socket.getOutputStream()`) в `BufferedOutputStream`.
    
- ✅ Используй **flush()** там, где важно сбросить данные немедленно (например, при логгировании).
    
- ✅ Для текста часто используют `BufferedWriter` (тот же принцип, только над символьными потоками).
    
- ❌ Не забывай про размер буфера:
    - маленький буфер = частые flush → низкая производительность;
        
    - слишком большой буфер = лишний расход памяти.
        
- ✅ Хорошая практика: всегда использовать `BufferedOutputStream` поверх «сырых» потоков, если нет специфической причины писать напрямую.

***
## Отличия от FileOutputStream 
### 🔶FileOutputStream

- Это **низкоуровневый поток**, который сразу пишет данные в файл (через системные вызовы ОС).
- Каждый вызов `write(int b)` → это **системный вызов** `write()` в ОС (дорогая операция).
    
- 💡Да, ты можешь вручную сначала писать в `byte[]`, а потом вызвать `fos.write(myByteArray)`, и это будет эффективно, НО ТОГДА:
    - тебе нужно самому управлять буфером;
        
    - следить за его размером и наполнением;
        
    - самому решать, когда сбрасывать.
        
> [!NOTE] То есть вся `логика «буферизации» ложится на твой код`.
> 

```java
FileOutputStream fos = new FileOutputStream("data.bin");

// ❌ каждый вызов — сразу в ОС (очень медленно для больших циклов)
for (int i = 0; i < 1000000; i++) {
    fos.write(65); // 'A'
}
fos.close();
```

***
### 🔶 BufferedOutputStream

- Это **декоратор над FileOutputStream**.
    
- ⭐Он **сам управляет внутренним буфером** (по умолчанию 8 КБ):
    - при вызове `write(1 байт)` он не сразу пишет в файл, а складывает в массив;
        
    - ==когда массив заполняется== или вызывается `flush()`/`close()` → ==данные пишутся в== `FileOutputStream` ==одним блоком==.
        
- Ты можешь спокойно писать хоть миллион раз по 1 байту — реально в файл уйдёт гораздо меньше системных вызовов.

```java
FileOutputStream fos = new FileOutputStream("data.bin");
BufferedOutputStream bos = new BufferedOutputStream(fos);

// ✅ складывается в буфер и пишется блоками
for (int i = 0; i < 1000000; i++) {
    bos.write(65); // 'A'
}
bos.close();
```

## Итог:
- Если ты **пишешь большими массивами** → `FileOutputStream` достаточно.
    
- Если ты **пишешь мелкими порциями** (например, цикл, логгирование, сериализация) → всегда лучше `BufferedOutputStream`
