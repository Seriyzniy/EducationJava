📌 `PrintStream` — это ==расширение== `FilterOutputStream`, ==предоставляющее удобные методы для **печати данных различных типов**== в поток вывода.  

Главные особенности:
- 💡Поддерживает вывод **примитивных типов** (`int`, `long`, `float`, `double`, `boolean`, `char`) и 💡объектов (`Object`) в удобной форме.
    
- Имеет методы **`print()`** и **`println()`**, которые автоматически преобразуют данные в строковое представление.
    
- Может работать с **автоочисткой буфера (autoFlush)**.
    
- В отличие от `Writer` (который работает с символами), `PrintStream` работает с **байтами**, но ориентирован на вывод **человекочитаемого текста**.

***
## Конструкторы
```java
// ✅ ЗАПИСЬ В ПОТОК ✅
// Создает PrintStream с выводом в указанный OutputStream
PrintStream(OutputStream out);

// Создает PrintStream с управлением автоочисткой буфера
PrintStream(OutputStream out, boolean autoFlush);

// Создает PrintStream с выбором кодировки
PrintStream(OutputStream out, boolean autoFlush, String charsetName);


// ✅ ЗАПИСЬ В ФАЙЛ ✅
// Создает PrintStream для записи в файл по имени
PrintStream(String fileName);

// То же самое, но с указанием кодировки
PrintStream(String fileName, String charsetName);

// Создает PrintStream для записи в File
PrintStream(File file);

// То же самое, но с кодировкой
PrintStream(File file, String charsetName);
```

> [!NOTE] 💡**`autoFlush`** — если `true`, то `буфер очищается автоматически` при каждом вызове `println`, `printf` или при записи символа новой строки.
> 

## Методы
```java
// Печатает 🛑СТРОКУ🛑 без перевода строки
void print(String s);

// Печатает строку с переводом строки
void println(String s);

// Перегруженные варианты print/println для всех типов
void print(int i);
void print(double d);
void print(boolean b);
void print(Object obj); // вызывает obj.toString()

// Проверка ошибок вывода
boolean checkError();

// Форматированный вывод (аналог printf в C)
PrintStream printf(String format, Object... args);
PrintStream format(String format, Object... args);

// Запись байта (унаследовано от OutputStream)
void write(int b);
void write(byte[] buf, int off, int len);

// Закрыть поток
void close();

// Принудительно очистить буфер
void flush();
```

***
## 💡 Особенности
1. `PrintStream` **никогда не выбрасывает `IOException` напрямую** — вместо этого ошибки сохраняются внутри объекта, а проверить их можно через `checkError()`.
    
2. Удобен для **логирования и отладки**, так как позволяет легко выводить любые данные.
    
3. В отличие от `FileOutputStream`, ориентирован на **текстовый вывод**, а не на "сырые" байты.
    
4. Чаще всего используется совместно с **`System.out`** или **`System.err`**.

***
## Пример
```java
import java.io.*;

public class PrintStreamExample {
    public static void main(String[] args) {
        try (PrintStream ps = new PrintStream("output.txt")) {
            
            // Запись разных типов
            ps.println("Hello, world!");   // строка с переводом строки
            ps.print(123);                 // число без перевода строки
            ps.println(45.67);             // дробное число
            ps.println(true);              // булево
            ps.println(new Object());      // вызовется toString()
            
            // Форматированный вывод
            ps.printf("Name: %s, Age: %d%n", "Alex", 25);

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

📄 После выполнения в файле `output.txt` будет:
```css
Hello, world!
12345.67
true
java.lang.Object@6d06d69c
Name: Alex, Age: 25
```


## Ключевые отличия от PrintWriter

|Характеристика|PrintStream|PrintWriter|
|---|---|---|
|Наследуется от|`OutputStream`|`Writer`|
|Работает с|**байтами**|**символами**|
|Кодировка|Указывается при создании, строка конвертируется в байты|Работает сразу с символами, кодировка задаётся через `Writer` (например, `OutputStreamWriter`)|
|Частое применение|`System.out`, низкоуровневые байтовые потоки|Работа с текстовыми файлами, где важна кодировка|
|Ошибки|Никогда не кидает `IOException` напрямую, есть `checkError()`|Может кидать `IOException`|
