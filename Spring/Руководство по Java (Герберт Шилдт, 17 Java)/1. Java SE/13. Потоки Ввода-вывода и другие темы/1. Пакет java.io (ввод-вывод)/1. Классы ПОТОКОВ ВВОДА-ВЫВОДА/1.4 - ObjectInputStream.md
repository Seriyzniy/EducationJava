`ObjectInputStream` — это поток, который ==умеет читать==:
- **примитивные типы данных** (`int`, `double`, `boolean` и т.п.),
- **объекты**, которые были сериализованы через `ObjectOutputStream`.

По сути — это ==десериализатор: восстанавливает объекты в памяти из бинарного представления.==

> [!NOTE] ВАЖНО
> ⚠️ Класс работает в связке с `ObjectOutputStream`: что один записал — другой может прочитать.  
> ⚠️ Поддерживает механизм **сериализации Java**, включая сохранение графа объектов с циклами, transient-поля, serialVersionUID и т.д.

## Конструкторы
```java
// Создаёт ObjectInputStream, который читает объекты из InputStream.
// 💡 Обычно InputStream — это FileInputStream или BufferedInputStream.
public ObjectInputStream(InputStream in) throws IOException
```

## Методы
```java
// Читает объект из потока и возвращает его.
// ⭐Объект должен быть сериализован⭐ через ObjectOutputStream.
// Может бросить ClassNotFoundException, если класс объекта отсутствует в classpath.
public final Object readObject() throws IOException, ClassNotFoundException

// ✅⭐Читает примитивы — парные методы к ObjectOutputStream.writeXXX(...).⭐✅
public boolean readBoolean() throws IOException
public byte readByte() throws IOException
public int readUnsignedByte() throws IOException
public short readShort() throws IOException
public int readUnsignedShort() throws IOException
public char readChar() throws IOException
public int readInt() throws IOException
public long readLong() throws IOException
public float readFloat() throws IOException
public double readDouble() throws IOException

// Читает массив байтов в b[off..off+len].
public int read(byte[] b, int off, int len) throws IOException

// Читает строку в формате writeUTF() (модифицированная UTF-8).
public String readUTF() throws IOException

// Закрывает поток и освобождает ресурсы.
public void close() throws IOException


// === ✅Специальные методы для управления сериализацией✅ ===

// Возвращает дескриптор текущего класса (метаданные сериализации).
protected ObjectStreamClass readClassDescriptor() throws IOException, ClassNotFoundException

// Позволяет настроить замену объектов при десериализации.
// Например, можно подменить singleton вместо нового экземпляра.
protected Object resolveObject(Object obj) throws IOException

// Загружает класс по имени.
// Можно переопределить для кастомного ClassLoader-а.
protected Class<?> resolveClass(ObjectStreamClass desc) throws IOException, ClassNotFoundException

```

***
## 💡❌Особенности
- 📌 **Поддерживает сериализацию целого графа объектов** (ссылки, циклы, повторяющиеся объекты → сохраняются корректно).
    
- 📌 Если поле `transient` — оно не будет записано в поток. При чтении → установится в значение по умолчанию.
    
- 📌 Проверяет `serialVersionUID` класса. Если он не совпадает — будет `InvalidClassException`.
    
- 📌 Можно кастомизировать сериализацию через `readObject()` / `writeObject()` внутри самих классов.
    
- ❌ Не потокобезопасен.
    
- ❌ Считается небезопасным для данных из недоверенных источников → возможны уязвимости RCE (remote code execution).

***
## Пример
```java
import java.io.*;

class Person implements Serializable {
    private static final long serialVersionUID = 1L;//⭐контроль версии⭐
    String name;
    int age;

    Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    @Override
    public String toString() {
        return "Person{name='" + name + "', age=" + age + "}";
    }
}

public class ObjectInputStreamExample {
    public static void main(String[] args) {
        String filename = "person.bin";

        // Сериализация
        try (ObjectOutputStream oos = new ObjectOutputStream(new FileOutputStream(filename))) {
            Person p = new Person("Alice", 30);
            oos.writeObject(p);
        } catch (IOException e) {
            e.printStackTrace();
        }

        // Десериализация
        try (ObjectInputStream ois = new ObjectInputStream(new FileInputStream(filename))) {
            Person p = (Person) ois.readObject();
            System.out.println("Прочитан объект: " + p);
        } catch (IOException | ClassNotFoundException e) {
            e.printStackTrace();
        }
    }
}
```

## Best-Practices

- ✅ Всегда явно объявляй `private static final long serialVersionUID`, чтобы контролировать совместимость версий классов.
    
- ✅ Оборачивай `FileInputStream` в `BufferedInputStream` → быстрее.
    
- ✅ Используй `try-with-resources` для автоматического закрытия.
    
- ✅ Если читаешь данные из внешних (недоверенных) источников → **не используй ObjectInputStream** (уязвимость RCE). Лучше применяй JSON, Protocol Buffers, Avro, XML.
    
- ✅ Для чувствительных данных помни про `transient` (например, пароли).
    
- ❌ Никогда не меняй сигнатуры сериализуемых классов без учёта `serialVersionUID`.
