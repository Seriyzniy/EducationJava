`FilterWriter` — это **абстрактный класс** из пакета `java.io`, который расширяет `Writer` и ==служит **обёрткой (декоратором)** для других ==`Writer`-ов.  

Он реализует _паттерн Decorator_: ==позволяет перехватывать вызовы методов== записи (`write`, `flush`, `close`) и ==добавлять дополнительное поведение==.

> [!NOTE] Только наследоваться от него
> Сам по себе `FilterWriter` не очень полезен — от него нужно наследоваться и реализовывать собственные фильтрующие/обрабатывающие механизмы.

***
## Конструктор
```java
protected FilterWriter(Writer out)
// out – целевой Writer, к которому делегируются все вызовы
```

## Методы
```java
public void write(int c) throws IOException
// Записывает один символ
// По умолчанию вызывает out.write(c)

public void write(char[] cbuf, int off, int len) throws IOException
// Записывает часть массива символов
// cbuf – массив символов
// off – смещение
// len – количество символов
// По умолчанию вызывает out.write(cbuf, off, len)


public void write(String str) throws IOException
// Записывает строкy
// str – строка

public void write(String str, int off, int len) throws IOException
// Записывает часть строки
// str – строка
// off – смещение
// len – количество символов
// По умолчанию вызывает out.write(str, off, len)


public void flush() throws IOException
// Сбрасывает буфер, делегирует вызов out.flush()

public void close() throws IOException
// Закрывает поток, делегирует вызов out.close()
```

***
## Особенности
- Является **==абстрактным== классом** — ==нельзя использовать напрямую.==
    
- Работает по принципу "обёртки": ==все вызовы методов делегируются внутреннему== `Writer`.
    
- Чтобы ==извлечь пользу, нужно создавать собственные наследники== (например, `UpperCaseWriter`, `CensorWriter`).
    
- ==Можно комбинировать несколько фильтров==, оборачивая их друг в друга.

***
## Пример
```java
import java.io.*;

// Определили свой собственный фильтр
class UpperCaseWriter extends FilterWriter {
    protected UpperCaseWriter(Writer out) {
        super(out);
    }

    @Override
    public void write(int c) throws IOException {
        super.write(Character.toUpperCase(c));
    }

    @Override
    public void write(char[] cbuf, int off, int len) throws IOException {
        for (int i = off; i < off + len; i++) {
            cbuf[i] = Character.toUpperCase(cbuf[i]);
        }
        super.write(cbuf, off, len);
    }

    @Override
    public void write(String str, int off, int len) throws IOException {
        super.write(str.toUpperCase(), off, len);
    }
}


public class Demo {
    public static void main(String[] args) {
        try (Writer writer = new UpperCaseWriter(
		        new FileWriter("output.txt"))
		) {
            writer.write("Hello, world! Привет, Java!");
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

***
## Best-Practices

- ✅**Всегда вызывай `super.close()` и `super.flush()`** при переопределении, чтобы не потерять данные.
    
- Используй наследование для конкретных задач (цензура текста, логирование, преобразования).
    
- ✅Не нарушай контракт `Writer`: фильтр должен вести себя как обычный поток записи, просто с добавленной логикой.
    
- Комбинируй фильтры — это позволяет строить цепочку преобразований (пример: `new UpperCaseWriter(new CensorWriter(new FileWriter("out.txt")))`).
