
> [!NOTE] Класс `java.io.File`
> Класс `java.io.File` -  это не поток и не сам файл, а ==абстракция «пути» к файлу== или каталогу. 
> 
> Он **описывает путь** в файловой системе и позволяет узнать информацию о нём (существует ли он, его размер, время модификации, права и т.д.), а также создавать/удалять файлы и каталоги.

⚠️ Важно: сам `File` ==**НЕ открывает** файл== для чтения/записи — для этого нужны `FileInputStream`, `FileOutputStream`, `RandomAccessFile`, `Files.newBufferedReader` и т.д.


⭐Поскольку класс `File` реализует интерфейс `Comparable`, также поддерживается метод `compareTo()` .⭐
## Конструкторы
```java
// Создаёт объект File по строковому пути
public File(String pathname)

// Создаёт объект File, комбинируя каталог и имя файла
public File(String parent, String child)

// То же самое, но parent задаётся как File
public File(File parent, String child)

// Создаёт объект File на основе URI (обычно file://)
public File(URI uri)
```
Комментарии:
- `pathname` — строка пути (может быть относительной или абсолютной).
- `parent` — путь к директории.
- `child` — имя файла или подкаталога.
    
- `uri` — должен быть **иерархическим URI с протоколом `file`**, иначе `IllegalArgumentException`.

## 💡 Особенности и подводные камни💡

1. 💡❗**`File` ≠ файл.** Он ==не держит файловый дескриптор, не читает и не пишет данные.== Это лишь «обёртка» вокруг пути. Многие новички думают, что `new File("x.txt")` создаёт файл — нет, это просто объект в памяти.
    
2. 💡**Методы возвращают `boolean`, а не кидают исключения.** Например, `delete()`, `mkdir()` — если что-то пошло не так, ты получаешь `false`, а не `IOException`. Это делает отладку сложнее, если не проверять результат.
    
3. **`exists()` не даёт гарантий.** Между проверкой и операцией файл может исчезнуть или поменяться. Это называется TOCTTOU (Time of Check to Time of Use) баг.
    
4. 💡❗**`length()` не всегда надёжно.** Для директорий возвращает 0, для файлов может кэшироваться ОС. На сетевых FS — ещё веселее.
    
5. **Права и операции зависят от ОС.** Например, `setExecutable` на Windows часто бесполезен.
    
6. 💡❗**Кодировки и локали.** `File` хранит путь как `String`, интерпретация зависит от файловой системы и её кодировки. В старых системах Windows возможны сюрпризы с кириллицей.
    
7. **`deleteOnExit()`** — классический memory leak: JVM хранит список таких файлов в статике, и очистить его нельзя. Если использовать массово — OutOfMemoryError.

***
## ✅ Best Practices ✅
- 💡Используйте `File` для **метаданных и навигации**, но для работы с содержимым — `Files` из `java.nio.file`.
    
- Проверяйте возвращаемые `boolean` у `delete()`, `renameTo()`, `mkdir()` и всегда логируйте неудачи.
    
- Для переносимости используйте `File.separator` или `Paths.get(...)`, не конкатенируйте `"/"` руками.
    
- 💡Для новых проектов — **предпочитайте `java.nio.file.Path/Files`**: они дают больше информации и кидают осмысленные исключения вместо `boolean`.
    
- Не используйте `deleteOnExit()` для больших файлов — вместо этого планируйте нормальную уборку.
    
- 💡Если нужно обойти каталог с рекурсией — лучше `Files.walkFileTree()`, а не ручная рекурсия через `listFiles()`.

## Пример
```java
import java.io.File;
import java.io.IOException;

public class FileExample {
    public static void main(String[] args) throws IOException {
        File dir = new File("data");
        if (!dir.exists()) {
            dir.mkdirs(); // создаём папку data/
        }

        File file = new File(dir, "test.txt");

        if (file.createNewFile()) {
            System.out.println("Файл создан: " + file.getAbsolutePath());
        } else {
            System.out.println("Файл уже существует");
        }

        System.out.println("Размер: " + file.length());
        System.out.println("Можно читать? " + file.canRead());
        System.out.println("Абсолютный путь: " + file.getAbsolutePath());

        for (File f : dir.listFiles()) {
            System.out.println("-> " + f.getName());
        }

        if (file.delete()) {
            System.out.println("Файл удалён");
        }
    }
}
```
