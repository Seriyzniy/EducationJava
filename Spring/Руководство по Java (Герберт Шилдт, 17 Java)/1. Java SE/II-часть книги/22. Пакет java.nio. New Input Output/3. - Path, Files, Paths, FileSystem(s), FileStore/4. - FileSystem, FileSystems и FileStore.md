# 📕 1. Абстрактный класс FileSystem


> [!NOTE] Класс `FileSystem` — это абстрактное представление файловой системы
> Он ==описывает, как Java взаимодействует с файловыми системами==: локальными (NTFS, ext4) или виртуальными (например, ZIP или in-memory).  
> 
> Каждый `Path` всегда связан с конкретным `FileSystem`.

## Конструктор
Нельзя создать `FileSystem` напрямую, только через `FileSystems.getDefault()` или `FileSystems.newFileSystem()`
Это абстрактный класс

## Методы
```java
//✅ Получение корневых директорий
// Возвращает Iterable<Path> — список корневых каталогов (например, "C:\", "/")
// @return Iterable с корневыми директориями
public abstract Iterable<Path> getRootDirectories();

//✅ Получение доступных хранилищ
// Возвращает Iterable<FileStore> — список устройств или томов
// @return Iterable с FileStore
public abstract Iterable<FileStore> getFileStores();

//✅ Получение разделителя пути
// @return String — символ, который разделяет элементы пути (например, "\" или "/")
public abstract String getSeparator();

//✅ Получение провайдера файловой системы
// @return FileSystemProvider — объект, предоставляющий реализацию (обычно sun.nio.fs.WindowsFileSystemProvider)
public abstract FileSystemProvider provider();

//✅ Проверка, открыта ли файловая система
// @return boolean — true, если FileSystem ещё активна
public abstract boolean isOpen();

//✅ Проверка, только для чтения ли файловая система
// @return boolean — true, если доступна только для чтения
public abstract boolean isReadOnly();

//✅ Получение Path по строковому пути
// @param first — первый элемент пути
// @param more — дополнительные элементы пути
// @return Path — созданный объект пути
public abstract Path getPath(String first, String... more);

//✅ Получение набора поддерживаемых атрибутов
// @return Set<String> — типы поддерживаемых атрибутов ("basic", "posix" и т.д.)
public abstract Set<String> supportedFileAttributeViews();

//✅ Закрытие файловой системы
// Закрывает FileSystem и освобождает ресурсы
public abstract void close() throws IOException;
```

## 💡 Особенности и подводные камни 💡
- 💡Объекты `FileSystem` редко создаются напрямую — чаще через `FileSystems.getDefault()` (локальная ОС) или `FileSystems.newFileSystem()` (например, ZIP-архив).
    
- После вызова `close()` использовать пути из этой системы больше нельзя.
    
- 💡`FileSystem` может быть только для чтения (например, архив).
    
- 💡Разделитель пути (`getSeparator()`) различается на разных ОС.

## Пример
```java
import java.nio.file.*;

public class FileSystemExample {
    public static void main(String[] args) {
        FileSystem fs = FileSystems.getDefault();

        System.out.println("Separator: " + fs.getSeparator());
        System.out.println("Root directories:");
        for (Path root : fs.getRootDirectories()) {
            System.out.println("  " + root);
        }

        System.out.println("FileStores:");
        for (FileStore store : fs.getFileStores()) {
            System.out.println("  " + store.name() + " (" + store.type() + ")");
        }
    }
}

Separator: \
Root directories:
  C:\
  D:\
  E:\
FileStores:
   (NTFS)
  Локальный диск (NTFS)
  Games (NTFS)
```

# 📕 2. Утилитарный класс FileSystems

> [!NOTE] `FileSystems` — это `утилитарный класс` для создания и получения объектов `FileSystem`
> 
> Он управляет доступом к файловым системам, включая стандартную (локальную) и пользовательские (например, ZIP, network, custom provider).

## Методы
```java
//✅ Получение стандартной файловой системы
// Возвращает FileSystem, связанную с текущей ОС (обычно Windows, Linux или macOS)
// @return FileSystem — основная файловая система JVM
public static FileSystem getDefault();

//✅ Создание новой файловой системы на основе URI
// Например, для работы с ZIP-архивом как с файловой системой
// @param uri — URI схемы ("jar:file:///C:/archive.zip")
// @param env — дополнительные параметры (например, {"create": "true"})
// @return FileSystem — новая виртуальная файловая система
public static FileSystem newFileSystem(URI uri, Map<String, ?> env) throws IOException;

//✅ Создание новой файловой системы по пути к ZIP/JAR
// Упрощённый вариант: создаёт виртуальную FS из ZIP/JAR
// @param path — путь к файлу архива
// @param env — параметры (например, {"create": "true"})
// @return FileSystem — объект файловой системы архива
public static FileSystem newFileSystem(Path path, Map<String, ?> env) throws IOException;

//✅ Получение списка провайдеров файловых систем
// @return List<FileSystemProvider> — список доступных провайдеров (обычно один — OS)
public static Iterable<FileSystemProvider> installedFileSystems();
```

---
## 💡 Особенности и подводные камни 💡
- 💡Основной метод: `getDefault()`.
    
- 💡Метод `newFileSystem()` часто используется для монтирования ZIP как виртуальной FS.
    
- Если попытаться открыть один и тот же архив дважды без закрытия, будет `FileSystemAlreadyExistsException`.
    
- ❓❓Провайдеры файловых систем загружаются через SPI (Service Provider Interface).

---

## Пример
```java
import java.nio.file.*;
import java.net.URI;
import java.util.*;

public class FileSystemsExample {
    public static void main(String[] args) throws Exception {
        FileSystem defaultFs = FileSystems.getDefault();
        
        System.out.println("Default FS separator: " + defaultFs.getSeparator());

        URI zipUri = URI.create("jar:file:///C:/data/archive.zip");
        Map<String, String> env = Map.of("create", "true");

        try (FileSystem zipFs = FileSystems.newFileSystem(zipUri, env)) {
            Path newFile = zipFs.getPath("/new.txt");
            Files.writeString(newFile, "Hello ZIP FS!");
            System.out.println("Created inside ZIP: " + newFile);
        }
    }
}
```

# 📕 3. Класс FileStore

> [!NOTE] Класс `FileStore` представляет ==физическое или логическое хранилище данных== (диск, раздел, сетевой ресурс).  
> Он предоставляет информацию о размере, типе и доступных атрибутах.
> 

## Конструктор
Конструктор защищённый (protected)
Нельзя создавать вручную — ==объекты== `FileStore` ==предоставляются== `FileSystem`

## Методы
```java
//✅ Имя хранилища
// @return String — имя (например, "C:" или "/")
public abstract String name();

//✅ Тип файловой системы
// @return String — тип (например, "NTFS", "ext4")
public abstract String type();

//✅ Проверка на только чтение
// @return boolean — true, если диск доступен только для чтения
public abstract boolean isReadOnly();

//✅ Общий размер хранилища
// @return long — байты общего объёма
public abstract long getTotalSpace() throws IOException;

//✅ Доступное свободное место
// @return long — байты, доступные текущему пользователю
public abstract long getUsableSpace() throws IOException;

//✅ Физически свободное место
// @return long — байты физически свободного пространства
public abstract long getUnallocatedSpace() throws IOException;

//✅ Поддержка определённого вида атрибутов
// @param attribute — имя типа атрибутов ("basic", "posix")
// @return boolean — поддерживается ли данный тип
public abstract boolean supportsFileAttributeView(Class<? extends FileAttributeView> type);

//✅ Получение атрибута хранилища
// @param attribute — имя атрибута
// @return Object — значение атрибута
public abstract Object getAttribute(String attribute) throws IOException;
```

## Особенности и подводные камни
- Значения свободного места могут отличаться из-за кеширования ОС.
    
- Некоторые типы атрибутов могут быть недоступны в зависимости от платформы.
- Проверяй `supportsFileAttributeView()` перед запросом атрибутов.
- Избегай частого вызова `getTotalSpace()` — это может быть дорого.
    
- На Windows `type()` часто возвращает `"NTFS"`, на Linux — `"ext4"` или `"tmpfs"`.

---
## Пример
```java
import java.nio.file.*;
import java.io.IOException;

public class FileStoreExample {
    public static void main(String[] args) throws IOException {
        Path path = Paths.get("C:/");
        FileStore store = Files.getFileStore(path);

        System.out.println("FileStore name: " + store.name());
        System.out.println("Type: " + store.type());
        System.out.println("Total: " + store.getTotalSpace() / 1_000_000 + " MB");
        System.out.println("Usable: " + store.getUsableSpace() / 1_000_000 + " MB");
        System.out.println("Read-only: " + store.isReadOnly());
    }
}
```
