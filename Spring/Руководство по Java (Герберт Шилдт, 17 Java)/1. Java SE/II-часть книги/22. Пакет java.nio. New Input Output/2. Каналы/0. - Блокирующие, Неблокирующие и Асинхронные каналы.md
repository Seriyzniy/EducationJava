## 🔹 **Блокирующие каналы**

> Работают синхронно (вызов `read()` / `write()` блокирует поток, пока операция не завершится).  
> Используются в основном для работы с файлами и потоками, где нет смысла в неблокирующем режиме.

- `FileChannel`
    
- `Pipe.SourceChannel`
    
- `Pipe.SinkChannel`
    
- `ReadableByteChannel` (базовый интерфейс, если реализация не поддерживает неблокирующий режим)
    
- `WritableByteChannel` (аналогично)
    
- `SeekableByteChannel`
    
- `GatheringByteChannel`
    
- `ScatteringByteChannel`
    

---

## 🔹 **Неблокирующие каналы**

> ==Поддерживают режим== `non-blocking` через метод `configureBlocking(false)`  
> (обычно используются совместно с `Selector`).

- `SocketChannel`
    
- `ServerSocketChannel`
    
- `DatagramChannel`
    

_(эти три — единственные стандартные каналы, поддерживающие неблокирующий режим и работу с селектором)_

---

## 🔹 **Асинхронные каналы (Java NIO.2, начиная с Java 7)**

> Работают ==на основе callback-модели== (`CompletionHandler`) или `Future`.  
> Не блокируют поток вообще — операции выполняются в фоне с уведомлением по завершении.

- `AsynchronousFileChannel`
    
- `AsynchronousSocketChannel`
    
- `AsynchronousServerSocketChannel`
    
- `AsynchronousChannelGroup` _(группа, управляющая асинхронными каналами)_
