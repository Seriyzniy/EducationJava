
> [!NOTE] `AsynchronousSocketChannel` — это абстрактный класс, из пакета java.nio.channels.  
> Он реализует интерфейсы:
> - `AsynchronousByteChannel`
> - `NetworkChannel`
> - `Multichannel` (косвенно через `Channel`)
> 
> Этот класс представляет собой `двусторонний TCP-канал`, поддерживающий ==асинхронные операции ПОДКЛЮЧЕНИЯ, чтения и записи.==

## Назначение
Позволяет:
- 💡Асинхронно **подключаться** к серверу (`connect()`);
    
- 💡Асинхронно **читать и писать** данные (`read()`, `write()`);
    
- Использовать **Future** или **CompletionHandler** API;
    
- Работать **без блокировки** основного потока — все операции выполняются через системные async механизмы.

## Конструкторы

Как и другие асинхронные каналы, `AsynchronousSocketChannel` **не имеет публичных конструкторов**.  
Создаётся через статические фабричные методы:
```java
// Открывает новый асинхронный сокет
public static AsynchronousSocketChannel open() throws IOException;

// Открывает сокет, используя определённую группу каналов (thread pool)
public static AsynchronousSocketChannel open(AsynchronousChannelGroup group) throws IOException;
```
📌 **`AsynchronousChannelGroup`** — это `пул потоков`, где обрабатываются события ввода-вывода (если не указан, используется системный общий пул).

## Методы
```java
// ✅ Подключение
// Асинхронно подключается к указанному адресу (Future API)
public abstract Future<Void> connect(SocketAddress remote);

// Асинхронно подключается, вызывая CompletionHandler по завершении
public abstract <A> void connect(
	SocketAddress remote, 
	A attachment,
    CompletionHandler<Void, ? super A> handler);



// ✅ ЧТЕНИЕ
// Асинхронно читает данные в буфер, возвращая Future<Integer>
@Override
public abstract Future<Integer> read(ByteBuffer dst);

// Асинхронное чтение с callback
@Override
public abstract <A> void read(
	ByteBuffer dst, 
	A attachment,
    CompletionHandler<Integer, ? super A> handler);



// ✅ ЗАПИСЬ
// Асинхронно записывает данные из буфера, возвращая Future<Integer>
@Override
public abstract Future<Integer> write(ByteBuffer src);

// Асинхронная запись с callback
@Override
public abstract <A> void write(
	ByteBuffer src, 
	A attachment,
    CompletionHandler<Integer, ? super A> handler);



// ✅ Настройки TCP
// Возвращает локальный адрес сокета
@Override
public abstract SocketAddress getLocalAddress() throws IOException;

// Возвращает удалённый адрес (если подключён)
@Override
public abstract SocketAddress getRemoteAddress() throws IOException;

// Настраивает параметр сокета (например, TCP_NODELAY, SO_KEEPALIVE)
@Override
public abstract <T> AsynchronousSocketChannel setOption(
	SocketOption<T> name, 
	T value) throws IOException;

// Возвращает значение TCP-опции
@Override
public abstract <T> T getOption(SocketOption<T> name) throws IOException;

// Возвращает список поддерживаемых опций
@Override
public abstract Set<SocketOption<?>> supportedOptions();



// ✅ Управление состоянием
@Override
public abstract boolean isOpen();

@Override
public abstract void close() throws IOException;
```

## 💡 Особенности и подводные камни 💡

### 1. Асинхронное подключение
- Метод `connect()` **не блокирует** поток.
    
- При использовании `CompletionHandler` событие `completed()` вызывается после установления TCP-соединения.

### 2. Полностью неблокирующие операции
- Все методы `read()` и `write()` немедленно возвращают `Future`,  
    или запускают callback через `CompletionHandler`.
    
### 3. Буфер обязан жить до завершения операции
- Пока асинхронная операция выполняется, нельзя менять содержимое `ByteBuffer` (даже позицию и лимит).

### ❗❗4. Не потокобезопасен
- В отличие от `AsynchronousFileChannel`, нельзя одновременно вызывать несколько `read()` или `write()` на одном и том же канале.  
    — Ожидай завершения предыдущей операции прежде чем вызывать новую.
    
### 5. TCP-опции
- Поддерживаются стандартные опции:
    - `SO_SNDBUF`, `SO_RCVBUF`,
        
    - `SO_KEEPALIVE`, `TCP_NODELAY`,
        
    - `SO_REUSEADDR`, `SO_LINGER`.

### 💡6. Привязка (bind)
- Можно вызвать `bind()` до подключения, если нужно слушать определённый локальный порт.

### 7. Работа с группами каналов
- При создании можно передать `AsynchronousChannelGroup` (thread pool).
    
- Это позволяет гибко управлять производительностью и изоляцией задач.

***
## ✅Best Practices ✅
- 💡Используй `CompletionHandler` для event-driven логики (реактивный стиль).
- 💡Используй `Future` для простых одноразовых операций (например, тестовых клиентов).
    
- 🔒 Не вызывай параллельно несколько `read()` / `write()` на одном сокете.
    
- Не забудь `flip()` перед `write()`, `clear()` после `read()`.
    
- При долгих операциях — создавай свой `AsynchronousChannelGroup` с кастомным `ExecutorService`.
    
- Не забывай закрывать канал (`close()`), иначе соединение останется открытым в системе.

## Пример 1 — Асинхронный TCP-клиент (CompletionHandler)
```java
import java.net.*;
import java.nio.*;
import java.nio.channels.*;

public class AsyncTCPClient {
    public static void main(String[] args) throws Exception {
        AsynchronousSocketChannel client = AsynchronousSocketChannel.open();
        
        InetSocketAddress serverAddr = new InetSocketAddress("localhost", 9000);

        client.connect(serverAddr, null, new CompletionHandler<Void, Void>() {
            @Override
            public void completed(Void result, Void att) {
                System.out.println("Connected to server!");

                ByteBuffer buffer = ByteBuffer.wrap(
	                "Hello from client!".getBytes()
	            );
	            
                client.write(buffer, buffer, new CompletionHandler<Integer, ByteBuffer>() {
                    @Override
                    public void completed(
	                    Integer bytesWritten, 
	                    ByteBuffer buf) 
	                {
                        System.out.println(
	                        "Sent " + bytesWritten + " bytes");
	                        
                        client.close();
                    }

                    @Override
                    public void failed(Throwable exc, ByteBuffer buf) {
                        System.err.println("Write failed: " + exc);
                        try { 
	                        client.close(); 
	                    } catch (Exception ignored) {}
                    }
                });
            }

            @Override
            public void failed(Throwable exc, Void att) {
                System.err.println("Connection failed: " + exc);
            }
        });

        Thread.sleep(2000);
    }
}

```

## Пример 2 — Асинхронное чтение через Future
```java
import java.net.*;
import java.nio.*;
import java.nio.channels.*;
import java.util.concurrent.Future;

public class AsyncSocketFutureExample {
    public static void main(String[] args) throws Exception {
        AsynchronousSocketChannel client = AsynchronousSocketChannel.open();
        client.connect(new InetSocketAddress("example.com", 80)).get();

        String request = "GET / HTTP/1.1\r\nHost: example.com\r\n\r\n";
        ByteBuffer out = ByteBuffer.wrap(request.getBytes());
        client.write(out).get();

        ByteBuffer in = ByteBuffer.allocate(1024);
        client.read(in).get();

        in.flip();
        System.out.println("Response: " + new String(in.array(), 0, in.limit()));
        client.close();
    }
}
```
