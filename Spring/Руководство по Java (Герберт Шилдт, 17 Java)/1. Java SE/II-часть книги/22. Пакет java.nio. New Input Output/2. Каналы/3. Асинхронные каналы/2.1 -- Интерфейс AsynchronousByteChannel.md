
> [!NOTE] `AsynchronousByteChannel` — это **интерфейс** из пакета `java.nio.channels`,  наследующий `AsynchronousChannel`
> `Добавляет методы` для **асинхронного чтения и записи байтов**.
> 

Он — **основа для всех байтовых асинхронных каналов**, таких как:
- `AsynchronousSocketChannel`
- `AsynchronousFileChannel`
    
- `AsynchronousPipe.SinkChannel` / `SourceChannel`

## Интерфейс
```java
public interface AsynchronousByteChannel
    extends AsynchronousChannel
{
    // ✅ Асинхронное чтение
	// Асинхронно читает данные в буфер.
	// Возвращает Future, через который можно проверить статус операции.
	Future<Integer> read(ByteBuffer dst);
	
	// Асинхронно читает данные и вызывает handler по завершении.
	// attachment — объект, который передаётся в handler (например, контекст).
	<A> void read(
		ByteBuffer dst, 
		A attachment, 
		CompletionHandler<Integer, ? super A> handler);
	
	
	// ✅ Асинхронная запись
	// Асинхронно записывает данные из буфера.
	// Возвращает Future, который завершится, когда запись закончится.
	Future<Integer> write(ByteBuffer src);
	
	// Асинхронно записывает данные, вызывая handler при завершении.
	<A> void write(
		ByteBuffer src, 
		A attachment, 
		CompletionHandler<Integer, ? super A> handler);
}
```

## Создание объектов
Объекты создаются через фабричные методы конкретных реализаций, например:
```java
// Создание асинхронного TCP-канала
AsynchronousByteChannel ch = AsynchronousSocketChannel.open();

// Создание асинхронного файлового канала
AsynchronousByteChannel file = AsynchronousFileChannel.open(path, StandardOpenOption.READ);
```

## 💡Особенности и подводные камни 💡

1. ❗❗ **Асинхронность ≠ параллельность**
    - Методы не создают поток сами по себе.
        
    - Они регистрируют операцию в `AsynchronousChannelGroup`, где её выполняет пул потоков или нативный I/O API (например, IOCP).
        
2. **Буфер остаётся “в живых”**
    - Пока операция не завершена, нельзя менять содержимое `ByteBuffer`.
        
    - Иначе в поток, выполняющий I/O, попадут некорректные данные.
        
3. 💡**Future и CompletionHandler — два стиля работы**
    - `Future` — удобен для простого ожидания (pull-модель).
        
    - `CompletionHandler` — подходит для реактивных / событийных моделей (push-модель).
        
4. 💡💡**Не потокобезопасен по умолчанию**
    - Нельзя вызывать два `read()` или два `write()` одновременно на одном и том же канале,  
        если документация конкретной реализации не указывает обратное (например, `AsynchronousFileChannel` — потокобезопасен).
        
5. **Ограничения ОС**
    - На Windows — реализуется через IOCP (эффективно).
        
    - На Linux/macOS — через пул потоков (чуть медленнее, но всё ещё асинхронно).
        
6. **Завершение операций при закрытии канала**
    - Если канал закрыт до завершения операции, `CompletionHandler.failed()` будет вызван с `AsynchronousCloseException`.

## ✅ Best Practices ✅

- 🟠Используй `CompletionHandler`, если хочешь **немедленно** обработать результат без блокировки.
- 🟢Используй `Future`, если нужно просто **дождаться** выполнения, например в фоне.
    
- Для сетевых серверов — комбинируй с `AsynchronousSocketChannel` и `Selector`-подобной логикой.
    
- ⚠️ Не переиспользуй `ByteBuffer`, пока операция не завершена.
    
- 🚨 Не вызывай несколько `read()` подряд без ожидания завершения предыдущего.
    
- При работе с файлами — предпочитай `AsynchronousFileChannel`, он потокобезопасен.

## Пример использования (CompletionHandler)
```java
import java.nio.*;
import java.nio.channels.*;
import java.nio.file.*;
import java.util.concurrent.*;

public class AsyncByteChannelExample {
    public static void main(String[] args) throws Exception {
        Path path = Path.of("async-data.txt");
        AsynchronousFileChannel channel = AsynchronousFileChannel.open(
                path, StandardOpenOption.READ);

        ByteBuffer buffer = ByteBuffer.allocate(128);

        channel.read(buffer, 0, buffer, new CompletionHandler<>() {
            @Override
            public void completed(Integer bytesRead, ByteBuffer attachment) {
                attachment.flip();
                System.out.println("Read " + bytesRead + " bytes: " +
                        new String(attachment.array(), 0, bytesRead));
                try { channel.close(); } catch (Exception e) { e.printStackTrace(); }
            }

            @Override
            public void failed(Throwable exc, ByteBuffer attachment) {
                System.err.println("Read failed: " + exc);
                try { channel.close(); } catch (Exception e) { e.printStackTrace(); }
            }
        });

        System.out.println("Main thread is free to do other work...");
        Thread.sleep(2000);
    }
}

```

## Пример использования (Future)
```java
import java.nio.*;
import java.nio.channels.*;
import java.nio.file.*;
import java.util.concurrent.*;

public class AsyncByteChannelFutureExample {
    public static void main(String[] args) throws Exception {
        Path path = Path.of("async-write.txt");
        AsynchronousFileChannel channel = AsynchronousFileChannel.open(
                path, StandardOpenOption.CREATE, StandardOpenOption.WRITE);

        ByteBuffer buffer = ByteBuffer.wrap("Hello from Future API!".getBytes());
        Future<Integer> writeOp = channel.write(buffer, 0);

        while (!writeOp.isDone()) {
            System.out.println("Writing in background...");
            Thread.sleep(50);
        }

        System.out.println("Write complete, " + writeOp.get() + " bytes written.");
        channel.close();
    }
}

```
