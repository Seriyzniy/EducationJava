
> [!NOTE] `AsynchronousFileChannel` — это **финальный класс** из пакета `java.nio.channels`.  
> Он реализует интерфейс `AsynchronousChannel` и `AsynchronousByteChannel`
> 
> 💡Поддерживает `позиционное чтение и запись`, то есть можно работать с разными участками файла параллельно.
> 

Позволяет выполнять операции чтения и записи **без блокировки вызывающего потока**,  используя 
- **асинхронную модель I/O** на уровне ОС 
- или thread-pool JVM.

## Назначение
Позволяет выполнять операции ==чтения и записи **без блокировки вызывающего потока**,  ==
используя **асинхронную модель I/O** на уровне ОС или thread-pool JVM.

Идеально подходит для:
- высоконагруженных файловых серверов;
- журналирования (логгеры, аудиты);
- систем, читающих большие бинарные файлы (видео, БД, индексы);
    
- асинхронных фреймворков (Netty, Vert.x, Spring Reactor и т. п.).

## Конструкторы

У `AsynchronousFileChannel` **нет публичных конструкторов** —  
объекты создаются через статические методы `open()`:
```java
// Открывает асинхронный файл с набором опций
public static AsynchronousFileChannel open(Path file, OpenOption... options) throws IOException;

// С указанием группы каналов (thread pool)
public static AsynchronousFileChannel open(Path file,
        Set<? extends OpenOption> options,
        ExecutorService executor,
        FileAttribute<?>... attrs) throws IOException;
```
**Параметры:**
- `file` — путь к файлу;
    
- `options` — флаги (`READ`, `WRITE`, `CREATE`, `TRUNCATE_EXISTING`, и т.д.);
    
- 💡`executor` — пул потоков, в котором будут выполняться асинхронные задачи (если `null` → используется общий системный пул);
    
- `attrs` — атрибуты создаваемого файла.

## Методы
```java
// ✅ Асинхронное ЧТЕНИЕ
// Читает данные в буфер, начиная с указанной позиции, возвращая Future

// Получает эксклюзивную блокировку (блокирующий вызов)

public final boolean isOpen();

public final Set<OpenOption> options();

// Закрывает канал и отменяет незавершённые операции

public abstract void close() throws IOException;
```

## Особенности и подводные камни

### 1. Позиционное чтение и запись
- В отличие от обычного `FileChannel`, `AsynchronousFileChannel` **не имеет общей позиции**.  
    Каждая операция указывает **позицию вручную** (`position`),  
    что позволяет выполнять **параллельные** операции чтения/записи в разных участках файла.
### 2. Многопоточность
- Канал **потокобезопасен**: можно запускать несколько `read()` и `write()` одновременно —  
    это одно из главных отличий от `AsynchronousSocketChannel`.
    
### 3. Асинхронность через `CompletionHandler` или `Future`
- Поддерживает **оба стиля**:
    - колбэки — реактивный подход (`CompletionHandler`);
        
    - `Future` — удобен для фоновых задач.
        
### 🔒 4. Блокировки файлов
- Блокировки (`FileLock`) также выполняются асинхронно;
- Можно заблокировать **весь файл или диапазон байтов**;
    
- Работает через тот же механизм, что и у `FileChannel`.
    
### 5. Производительность
- На Windows — основан на IOCP, реальная асинхронность без потоков.
    
- На Linux — реализован через пул потоков (`AsynchronousChannelGroup`),  
    поэтому эффективность зависит от числа потоков и размера задач.
    
### ⚠️ 6. Взаимодействие с ByteBuffer
- Пока операция не завершена — **нельзя модифицировать `ByteBuffer`**.
    
- После завершения `completed()` буфер должен быть “перевернут” (`flip()`).

## ✅ Best Practices ✅

- Указывай **позицию** в каждом `read()` / `write()`, чтобы избежать гонок.
    
- 💡Используй `CompletionHandler` для реактивных сценариев (серверы, пайплайны).
- 💡Для фоновых задач с ожиданием — `Future`.
    
- Для файлов большого размера — читай участками (параллельно по позициям).
    
- 🔒 Всегда освобождай `FileLock` после использования.
    
- Закрывай канал (`try-with-resources`), иначе останутся открытые дескрипторы.
    
- ❗❗❗ Не используйте одновременно `FileChannel` и `AsynchronousFileChannel` для одного файла — возможны рассинхронизации.

## Пример 1 — Асинхронное чтение файла (CompletionHandler)
```JAVA
import java.nio.*;
import java.nio.channels.*;
import java.nio.file.*;

public class AsyncFileReadExample {
    public static void main(String[] args) throws Exception {
        Path path = Path.of("example.txt");
        AsynchronousFileChannel channel = AsynchronousFileChannel.open(path, StandardOpenOption.READ);


        ByteBuffer buffer = ByteBuffer.allocate(128);
        
        channel.read(buffer, 0, buffer, new CompletionHandler<>() {
            @Override
            public void completed(
	            Integer bytesRead, 
	            ByteBuffer attachment) 
	        {
                attachment.flip();
                String data = new String(
	                attachment.array(), 
	                0, 
	                bytesRead);
	                
                System.out.println("Async read completed: " + data);
                try { channel.close(); } catch (Exception ignored) {}
            }

            @Override
            public void failed(Throwable exc, ByteBuffer attachment) {
                System.err.println("Read failed: " + exc);
                try { channel.close(); } catch (Exception ignored) {}
            }
        });

        System.out.println("Main thread continues doing other work...");
        Thread.sleep(2000);
    }
}

```

## Пример 2 — Асинхронная запись с Future API
```java
import java.nio.*;
import java.nio.channels.*;
import java.nio.file.*;
import java.util.concurrent.Future;

public class AsyncFileWriteExample {
    public static void main(String[] args) throws Exception {
        Path path = Path.of("async-output.txt");
        AsynchronousFileChannel channel = AsynchronousFileChannel
	        .open(
		        path, 
		        StandardOpenOption.WRITE, 
		        StandardOpenOption.CREATE);

        ByteBuffer buffer = ByteBuffer.wrap(
	        "Hello from AsynchronousFileChannel!".getBytes()
	    );
	    
        Future<Integer> result = channel.write(buffer, 0);

        while (!result.isDone()) {
            System.out.println("Writing asynchronously...");
            Thread.sleep(50);
        }

        System.out.println("Write complete, bytes written: " + result.get());
        channel.close();
    }
}
```

## Пример 3 — Параллельное чтение разных участков файла
```java
import java.nio.*;
import java.nio.channels.*;
import java.nio.file.*;
import java.util.concurrent.*;

public class ParallelReadExample {
    public static void main(String[] args) throws Exception {
        Path path = Path.of("bigfile.bin");
        AsynchronousFileChannel channel = AsynchronousFileChannel.open(
	        path, 
	        StandardOpenOption.READ);

        ByteBuffer part1 = ByteBuffer.allocate(256);
        ByteBuffer part2 = ByteBuffer.allocate(256);

        Future<Integer> f1 = channel.read(part1, 0);
        Future<Integer> f2 = channel.read(part2, 256);

        f1.get();
        f2.get();

        System.out.println("Part 1: " + new String(part1.array(), 0, part1.position()));
        System.out.println("Part 2: " + new String(part2.array(), 0, part2.position()));

        channel.close();
    }
}

```
