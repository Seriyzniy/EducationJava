
> [!NOTE] `ServerSocketChannel` — это **абстрактный класс**, наследующий `AbstractSelectableChannel` и реализующий `NetworkChannel`
> Используется ==для **принятия входящих TCP-соединений**.==  
> 
> 🧠Метод `accept()` возвращает новый `SocketChannel` для взаимодействия с клиентом.

## Конструктор
У `ServerSocketChannel` нет публичных конструкторов.  
Создание происходит через фабричный метод:
```java
// Открытие нового ServerSocketChannel
public static ServerSocketChannel open() throws IOException;
```

## Методы
```java
// ✅ Управление блокировкой / неблокировкой (унаследовано из SelectableChannel)
// Переключение blocking/non-blocking
public final ServerSocketChannel configureBlocking(boolean block) throws IOException;

// Проверка режима
public final boolean isBlocking();

// Возвращает объект блокировки (используется внутри NIO для синхронизации)
public final Object blockingLock();



// ✅ Связывание с локальным адресом (унаследовано из NetworkChannel)
// Привязка к локальному адресу (например, порт TCP)
public final ServerSocketChannel bind(SocketAddress local) throws IOException;

// Привязка с указанием размера очереди ожидающих соединений (backlog)
public final ServerSocketChannel bind(SocketAddress local, int backlog) throws IOException;

// Получение локального адреса (IP + порт)
public final SocketAddress getLocalAddress() throws IOException;



// ✅ Настройка опций сокета (унаследовано из NetworkChannel)
// Установка опции (например, SO_REUSEADDR)
public final <T> ServerSocketChannel setOption(SocketOption<T> name, T value) throws IOException;

// Получение значения опции
public final <T> T getOption(SocketOption<T> name) throws IOException;

// Возвращает список поддерживаемых опций (SO_RCVBUF, SO_REUSEADDR, и т.д.)
public final Set<SocketOption<?>> supportedOptions();



// ✅ Приём подключений
// Принимает входящее соединение
// В blocking-режиме ждёт клиента, в non-blocking — возвращает null, если клиентов нет
public abstract SocketChannel accept() throws IOException;



// ✅ Доступ к java.net.ServerSocket
// Возвращает связанный java.net.ServerSocket (для совместимости и настройки)
public abstract java.net.ServerSocket socket();



// ✅ СЕЛЕКТОР, Регистрация с Selector (унаследовано из SelectableChannel)
// Регистрирует канал в Selector (например, для OP_ACCEPT)
public final SelectionKey register(Selector sel, int ops, Object att) throws ClosedChannelException;

// Регистрирует канал в Selector (без attachment)
public final SelectionKey register(Selector sel, int ops) throws ClosedChannelException;

// Маска поддерживаемых операций (только OP_ACCEPT)
public final int validOps();

// 💡 Возвращает ключ регистрации для указанного селектора💡
public abstract SelectionKey keyFor(Selector sel);



// ✅ Закрытие (унаследовано из AbstractInterruptibleChannel / Channel)
// Закрывает канал (все операции станут недоступны)
public abstract void close() throws IOException;

// Проверка: открыт ли канал
public final boolean isOpen();
```

## 💡 Особенности и подводные камни 💡

1. **Blocking vs Non-blocking**
    - 💡В blocking-режиме `accept()` ждёт клиента.
        
    - 💡В non-blocking-режиме `accept()` возвращает `null`, если нет подключений.
        
2. **Selector**
    - Поддерживает только одно событие — `OP_ACCEPT`.
        
    - После срабатывания `OP_ACCEPT` нужно вызвать `accept()`, чтобы получить `SocketChannel`.
        
3. **Опции**
    - Поддерживает стандартные опции TCP-сокета:
        - `SO_RCVBUF` (размер приёмного буфера),
            
        - `SO_REUSEADDR` (разрешает повторное использование адреса),
            
        - `SO_REUSEPORT` (на некоторых ОС).
            
4. 💡**Привязка (bind)**
    - Канал нужно привязать к адресу перед использованием (`bind(new InetSocketAddress(port))`).
        
    - Если не привязать → `accept()` выбросит исключение.
        
5. 💡**Связь с java.net.ServerSocket**
    - Через `socket()` можно настраивать опции, как в старом API.
        
    - Но в реальных NIO-приложениях лучше использовать `setOption()`.

## ✅Best Practices ✅

- 🧠Используй non-blocking режим + `Selector` для масштабируемых серверов.
    
- 💡Всегда обрабатывай `accept()` → `null` (non-blocking).
    
- Закрывай канал корректно (`try-with-resources` или shutdown hook).
    
- 💡Для настройки сокета предпочитай `setOption()` вместо `socket().setSoTimeout(...)`.
    
-  Если сервер многопоточный, управляй доступом к каналу синхронизацией.

## Пример
Сервер в blocking-режиме (один клиент)
```java
import java.net.InetSocketAddress;
import java.nio.ByteBuffer;
import java.nio.channels.ServerSocketChannel;
import java.nio.channels.SocketChannel;

public class BlockingServerExample {
    public static void main(String[] args) throws Exception {
        try (ServerSocketChannel server = ServerSocketChannel.open()) {
            server.bind(new InetSocketAddress(8080));

            System.out.println("Server started on port 8080...");

            try (SocketChannel client = server.accept()) { // блокируется, пока клиент не подключится
                System.out.println("Client connected: " + client.getRemoteAddress());

                ByteBuffer buffer = ByteBuffer.wrap("Hello client!\n".getBytes());
                client.write(buffer);
            }
        }
    }
}
```

Сервер в non-blocking-режиме с Selector
```java
import java.net.InetSocketAddress;
import java.nio.ByteBuffer;
import java.nio.channels.*;
import java.util.Iterator;

public class NonBlockingServerExample {
    public static void main(String[] args) throws Exception {
        ServerSocketChannel server = ServerSocketChannel.open();
        server.bind(new InetSocketAddress(8080));
        server.configureBlocking(false);

        Selector selector = Selector.open();
        server.register(selector, SelectionKey.OP_ACCEPT);

        System.out.println("Non-blocking server started on port 8080...");

        while (true) {
            selector.select(); // ждём события
            Iterator<SelectionKey> it = selector.selectedKeys().iterator();

            while (it.hasNext()) {
                SelectionKey key = it.next();
                it.remove();

                if (key.isAcceptable()) {
                    SocketChannel client = server.accept();
                    if (client != null) {
                        client.configureBlocking(false);
                        client.register(selector, SelectionKey.OP_WRITE, ByteBuffer.wrap("Hello async client!\n".getBytes()));
                        System.out.println("Client connected: " + client.getRemoteAddress());
                    }
                } else if (key.isWritable()) {
                    SocketChannel client = (SocketChannel) key.channel();
                    ByteBuffer buffer = (ByteBuffer) key.attachment();
                    client.write(buffer);
                    if (!buffer.hasRemaining()) {
                        client.close();
                    }
                }
            }
        }
    }
}

```
