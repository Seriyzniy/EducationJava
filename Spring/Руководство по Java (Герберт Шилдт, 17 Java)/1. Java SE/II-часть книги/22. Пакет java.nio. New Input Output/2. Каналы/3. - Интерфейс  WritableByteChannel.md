
> [!NOTE] `WritableByteChannel` — это **подинтерфейс** `Channel`, 
> Интерфейс представляющий приёмник байтов (==аналог **OutputStream** в NIO==).  
> 
> Он определяет метод `write(ByteBuffer)`, который ==записывает данные из буфера в канал.==
> 

Используется для записи данных в:
- **файлы** (`FileChannel`),
- **TCP-сокеты** (`SocketChannel`),
- **UDP** (`DatagramChannel`),
- **пайпы** (`Pipe.SinkChannel`).

## Интерфейс
```java
package java.nio.channels;

public interface WritableByteChannel extends Channel {

    /**
     * Записывает последовательность байтов из указанного буфера в канал.
     *
     * src - буфер, из которого будут взяты данные
     * @return количество записанных байтов (0, если ничего не записано)
     * @throws IOException - при ошибке ввода-вывода
     */
    public int write(ByteBuffer src) throws IOException;
}
```

## 💡 Особенности и подводные камни 💡

1. 💡**Аналог OutputStream.write()**  
    Только работает не с массивами, а с `ByteBuffer`.
    
2. **Возвращаемое значение**
    - `> 0` → количество записанных байтов,
    - `0` → в неблокирующем режиме данные пока не могут быть записаны (например, сокетный буфер переполнен).
        
3. 💡**Буферное состояние**
    - 💡 Запись идёт из `position` до `limit`.
    - После записи `position` сдвигается вперёд на число записанных байтов.
        
4. **Неблокирующий режим**
    - В blocking режиме метод ждёт, пока данные не будут записаны.
        
    - В non-blocking режиме может вернуться `0`, если в сокет пока ничего нельзя записать.
        
5. **Ограничение**
    - `WritableByteChannel` умеет только писать.
        
    - Для чтения — `ReadableByteChannel`.
        
    - Для чтения и записи — `ByteChannel` (расширяет оба).

## ✅ Best Practices ✅

- Используй `try-with-resources` для автоматического закрытия канала.
    
- Перед вызовом `write()` убедись, что буфер в режиме чтения (`flip()` после `put()`).
    
- 🔄 Если пишешь в цикле, учитывай, что `write()` может записать меньше данных, чем есть в буфере → проверяй `buffer.hasRemaining()`.
    
- ❗❓Для больших файлов применяй `FileChannel.transferTo/transferFrom` вместо ручного цикла `write()`.
    
- ⚠️ В non-blocking I/O (например, с `Selector`) всегда обрабатывай ситуацию `write() == 0`.

## Пример
Запись в файл
```java
import java.io.RandomAccessFile;
import java.nio.ByteBuffer;
import java.nio.channels.FileChannel;
import java.nio.channels.WritableByteChannel;

public class WritableByteChannelFileExample {
    public static void main(String[] args) throws Exception {
        try (RandomAccessFile raf = new RandomAccessFile("output.txt", "rw");
             WritableByteChannel channel = raf.getChannel()) {

            String text = "Hello from WritableByteChannel!\n";
            ByteBuffer buffer = ByteBuffer.wrap(text.getBytes());

            // Записываем все данные
            while (buffer.hasRemaining()) {
                channel.write(buffer);
            }
        }
    }
}
```

Запись в сокет
```java
import java.net.InetSocketAddress;
import java.nio.ByteBuffer;
import java.nio.channels.SocketChannel;
import java.nio.channels.WritableByteChannel;

public class WritableByteChannelSocketExample {
    public static void main(String[] args) throws Exception {
        try (SocketChannel socketChannel = SocketChannel.open(
	        new InetSocketAddress("example.com", 80))
	    ) {
            WritableByteChannel channel = socketChannel;

            String request = "GET / HTTP/1.1\r\nHost: example.com\r\n\r\n";
            ByteBuffer buffer = ByteBuffer.wrap(request.getBytes());

            // Отправляем HTTP-запрос
            while (buffer.hasRemaining()) {
                channel.write(buffer);
            }
        }
    }
}
```
