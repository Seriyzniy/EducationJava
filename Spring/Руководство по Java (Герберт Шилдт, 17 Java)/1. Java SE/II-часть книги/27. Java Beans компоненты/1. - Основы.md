# 1. Что это такое?

**JavaBeans** — это **специальные многократно используемые программные компоненты Java**, которые:
- инкапсулируют данные (свойства),
- предоставляют доступ к ним через **геттеры и сеттеры**,
    
- могут использоваться в визуальных средах разработки (например, в IDE вроде NetBeans, IntelliJ, Eclipse),
    
- поддерживают механизмы **сериализации**, **событий** и **редактирования свойств**.

> [!NOTE] ⭐ Java Bean = Класс Java ⭐
> Проще говоря, **JavaBean — это обычный Java-класс**, который следует определённым соглашениям (naming conventions) и реализует интерфейс `Serializable`.

# 2. Самоанализ (Проектирование JavaBean)
В основе программирования Веаn-компонентов лежит самоанализ - процесс анализа Веаn-компонента с целью выяснения его возможностей. 

Разработчик компонента может указать, ==какие из его== `свойств`, `событий` и `методов` ==должны быть доступны==, ==двумя способами==.

1) Использование простых ==соглашений об именовании==, которые позволяют механизмам самоанализа выводить информацию о Веаn-компоненте.
2) Созданием ==дополнительного класса==, расширяющего интерфейс `BeanInfo`, который `явно предоставляет такую информацию`.

## Требования к JavaBean

Чтобы класс считался JavaBean, он должен соответствовать **четырём правилам**:

1. **Публичный конструктор без параметров**  
    → нужен для того, чтобы IDE и фреймворки могли автоматически создавать экземпляры (например, через `Class.newInstance()`).
    
2. **Инкапсулированные поля (private)**  
    → все свойства класса должны быть закрыты и доступны только через методы.
    
3. **Публичные геттеры и сеттеры** для доступа к свойствам:
    - `getPropertyName()` и `setPropertyName(value)` — для обычных свойств;
        
    - `isPropertyName()` — для булевых свойств.  
        Названия методов строго следуют JavaBeans Naming Convention.
        
4. **Реализация интерфейса `Serializable`**  
    → чтобы экземпляры Bean можно было сохранять и восстанавливать в потоки (например, при передаче по сети или хранении в файле).

## Свойства JavaBean
Свойство — это логическая пара геттер/сеттер.  

Например, для свойства `name`:
- `getName()` — возвращает значение свойства;
- `setName(String value)` — устанавливает значение.

### Типы свойств:
1. **Простые (simple)** — одно значение (`String name`).
    
2. **Индексированные (indexed)** — списки или массивы (`List<String> phones`).
    
3. **Связанные (bound)** — при изменении свойства рассылается событие (например, через `PropertyChangeSupport`).
    
4. **Ограниченные (constrained)** — изменение можно отклонить слушателем (`VetoableChangeListener`).

# 3. Событийная модель JavaBeans

JavaBeans поддерживают **модель слушателей событий (Event Listener Model)**, аналогичную AWT/Swing:
- Bean ==может **генерировать события**== (например, изменение свойства);
    
- другие объекты (слушатели) могут **подписываться** и реагировать на эти события.
    

Для этого используют:
- 💡классы типа `PropertyChangeEvent`;
    
- 💡поддержку через `PropertyChangeSupport` и `VetoableChangeSupport`.

#### **VetoableChangeSupport**
Расширение предыдущего механизма: слушатель может **отклонить** изменение свойства, выбросив `PropertyVetoException`.


Пример:
```java
import java.beans.*;

public class TemperatureBean {
    private int temperature;
    private PropertyChangeSupport support = new PropertyChangeSupport(this);

    public void addPropertyChangeListener(PropertyChangeListener listener) {
        support.addPropertyChangeListener(listener);
    }

    public void setTemperature(int newTemp) {
        int oldTemp = this.temperature;
        this.temperature = newTemp;
        support.firePropertyChange("temperature", oldTemp, newTemp);
    }
}

```
