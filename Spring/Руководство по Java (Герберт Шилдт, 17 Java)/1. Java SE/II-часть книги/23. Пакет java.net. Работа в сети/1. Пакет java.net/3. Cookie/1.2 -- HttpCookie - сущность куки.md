
> [!NOTE] Класс **`java.net.HttpCookie`** представляет **`один HTTP-cookie`**
> То есть `пару имя–значение`, передаваемую между клиентом и сервером в HTTP-заголовках (`Set-Cookie` / `Cookie`).
> 
> ⭐Реализует интерфейсы: `Cloneable, Serializable`

Он инкапсулирует все параметры cookie — ==имя, значение, домен, путь, срок действия, флаг безопасности== и т.д.  

Используется `CookieManager` и `CookieStore` для хранения и управления cookies.

### Пару слов о назначении
- `HttpCookie` — это **модель данных** для хранения информации о cookie.
    
- Может быть создан вручную и добавлен в `CookieStore`.
    
- Применяется в API `CookieManager`, `HttpURLConnection`, а также в пользовательских HTTP-клиентах.
    
- Поддерживает все атрибуты cookie, определённые в RFC 6265 (и частично RFC 2109 / 2965).

## Конструкторы
```java
//✅ Создаёт cookie с заданным именем и значением.
// Параметры:
//  name  — имя cookie (не может содержать пробелы, запятые, точки с запятой и знаки =);
//  value — значение cookie.
public HttpCookie(String name, String value)
```
💬 После создания cookie можно дополнительно задать свойства —  
домен, путь, время жизни, secure-флаг и т.д.

## Методы
### Получение и изменение основных свойств
```java
//✅ Возвращает имя cookie.
public String getName()

//✅ Возвращает значение cookie.
public String getValue()

//✅ Устанавливает новое значение cookie.
public void setValue(String newValue)

//✅ Возвращает true, если cookie помечен как "HttpOnly" (недоступен JavaScript’у).
public boolean isHttpOnly()

//✅ Устанавливает флаг HttpOnly (true — только для HTTP, false — доступен в скриптах).
public void setHttpOnly(boolean httpOnly)
```

### Домен, путь и порт
```java
//✅ Возвращает домен, к которому привязан cookie (например, ".example.com").
public String getDomain()

//✅ Устанавливает домен.
public void setDomain(String pattern)

//✅ Возвращает путь, для которого cookie действует (например, "/api/").
public String getPath()

//✅ Устанавливает путь.
public void setPath(String uriPath)

//✅ Возвращает строку с портами, для которых cookie допустим (может быть null).
public String getPortlist()

//✅ Устанавливает список портов в виде строки ("80,8080").
public void setPortlist(String ports)
```

### Время жизни и истечение
```java
//✅ Возвращает время жизни cookie в секундах.
// Если <0 → cookie является сессионным (удаляется при закрытии соединения).
public long getMaxAge()

//✅ Устанавливает срок жизни cookie (в секундах).
// Пример: setMaxAge(3600) → cookie живёт 1 час.
public void setMaxAge(long expiry)

//✅ Проверяет, истёк ли срок действия cookie.
public boolean hasExpired()
```

### Безопасность и флаги
```java
//✅ Возвращает true, если cookie доступен только по HTTPS (secure-флаг).
public boolean getSecure()

//✅ Устанавливает secure-флаг.
public void setSecure(boolean flag)

//✅ Возвращает версию cookie (0 — Netscape, 1 — RFC 2965).
public int getVersion()

//✅ Устанавливает версию cookie.
public void setVersion(int v)
```

### Преобразование и сравнение
```java
//✅ Возвращает строковое представление cookie (имя=значение; атрибуты).
public String toString()

//✅ Проверяет, эквивалентны ли два cookie (по имени, домену, пути).
public boolean equals(Object obj)

//✅ Создаёт копию cookie.
public Object clone()
```

### Парсинг cookie из HTTP-заголовков
```java
//✅ Разбирает заголовок "Set-Cookie" и возвращает список HttpCookie.
public static List<HttpCookie> parse(String header)
```
💬 Удобно для ручного парсинга ответов сервера без использования `CookieManager`.  
Пример:
```java
List<HttpCookie> cookies = HttpCookie
	.parse("session=abc123; Path=/; HttpOnly");
```

## 💡 Особенности и подводные камни 💡

- 💡`HttpCookie` сам по себе **не хранится** — его нужно добавить в `CookieStore`.
    
- 🧠 Флаг `HttpOnly` запрещает доступ cookie из JavaScript (повышает безопасность).
    
- 💡Флаг `Secure` требует HTTPS-соединение для передачи cookie.
    
- 💡Время жизни (`Max-Age`) определяет, будет ли cookie постоянным или сессионным.
    
- ❗❗❗ Поле `Domain` должно соответствовать имени сервера, иначе cookie не отправится.
    
- 🧠В `equals()` сравниваются не только имя, но и домен/путь — два cookie с одинаковым именем, но разными доменами, считаются разными.
    
- ❓Версия 0 соответствует старым Netscape-cookie (более простым). Версия 1 — современный RFC-стандарт.

## 💎 Best Practices 💎

✅ ❗❗❗**Создавай cookie через конструктор и сразу задавай домен и путь:**
```java
HttpCookie cookie = new HttpCookie("session", "abc123");

cookie.setDomain(".example.com");
cookie.setPath("/");
```

✅ **Добавляй cookie в хранилище через CookieManager:**\
```java
CookieManager manager = new CookieManager();
manager.getCookieStore().add(new URI("https://example.com"), cookie);
```

✅ **Используй короткое время жизни для чувствительных cookie:**
```java
cookie.setMaxAge(300); // 5 минут
```

✅ **Ограничивай область cookie:**
- `setDomain()` → только нужный сайт;
- `setPath()` → только нужный раздел.
    

✅ **Проверяй срок действия:**
```java
if (cookie.hasExpired()) 
	manager.getCookieStore().remove(uri, cookie);
```

✅ **Для отладки**:
```java
System.out.println(cookie.toString());
```

## Пример
```java
import java.net.*;
import java.util.*;

public class HttpCookieDemo {
    public static void main(String[] args) throws Exception {
        // 1️⃣ Создаём cookie вручную
        HttpCookie cookie = new HttpCookie("user", "Seriy");
        
        cookie.setDomain(".httpbin.org");
        cookie.setPath("/");
        cookie.setMaxAge(3600); // 1 час
        cookie.setSecure(false);
        cookie.setHttpOnly(true);


        // 2️⃣ Создаём CookieManager и добавляем cookie в хранилище
        CookieManager manager = new CookieManager();
        CookieStore store = manager.getCookieStore();
        store.add(new URI("https://httpbin.org/"), cookie);

        // 3️⃣ Просмотр сохранённых cookies
        System.out.println("--- Cookies в хранилище ---");
        for (HttpCookie c : store.getCookies()) {
            System.out.println("Имя: " + c.getName() +
                               ", значение: " + c.getValue() +
                               ", домен: " + c.getDomain());
        }

        // 4️⃣ Проверка срока действия
        System.out.println("\nИстёк ли cookie? " + cookie.hasExpired());
    }
}

```
