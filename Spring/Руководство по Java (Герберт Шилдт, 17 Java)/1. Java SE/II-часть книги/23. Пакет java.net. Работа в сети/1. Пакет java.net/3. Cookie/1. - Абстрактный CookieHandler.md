
> [!NOTE] Класс `java.net.CookieHandler` — это абстрактный механизм `для управления HTTP-куки` 
> Используется вместе с API `URLConnection` и `HttpURLConnection`.
> 
> Он служит **`централизованным обработчиком`** `cookies`:  
> принимает cookies из ответов сервера и добавляет их к последующим запросам, поддерживая _сессионность_ (аналогично браузеру).
> 
> ⭐Реализует интерфейс: `Serializable`

### 🔸 Связанные классы Cookie API 🔸

|Класс|Назначение|
|---|---|
|**`CookieHandler`**|Абстрактный базовый класс для работы с cookies.|
|**`CookieManager`**|Реализация `CookieHandler`, сохраняющая cookies в `CookieStore`.|
|**`CookieStore`**|Интерфейс для хранения cookies (в памяти или на диске).|
|**`CookiePolicy`**|Определяет, какие cookies можно сохранять.|
|**`HttpCookie`**|Модель одного cookie (данные и флаги).|

### Пару слов о назначении
- `CookieHandler` является **глобальной точкой доступа** для всех cookie-операций.
    
- ⭐Конкретная реализация — класс `CookieManager`.
    
- Используется прозрачным образом — при установке обработчика cookies  
    все объекты `HttpURLConnection` автоматически используют его.

## Методы
### Управление глобальным обработчиком cookie
```java
//✅ Устанавливает глобальный обработчик cookie (например, CookieManager).
// Параметр:
//  cHandler — экземпляр CookieHandler (или null, чтобы отключить обработку cookie).
static void setDefault(CookieHandler cHandler)

//✅ Возвращает текущий глобальный обработчик cookie (или null, если не установлен).
static CookieHandler getDefault()
```

💬 ⭐⭐⭐Обычно приложение вызывает:⭐⭐⭐
```java
CookieHandler.setDefault(new CookieManager());
```
⭐⭐⭐После этого все `HttpURLConnection` будут автоматически использовать cookie-хранилище.

### Обработка cookies вручную
```java
//✅ Вызывается внутренними реализациями URLConnection.
// Возвращает заголовки, которые нужно добавить к исходящему запросу (например, Cookie: ...).
// Параметры:
//  uri — адрес ресурса;
//  requestHeaders — исходные заголовки запроса.
// Возвращает: Map<String, List<String>> с cookie-заголовками.
public abstract Map<String, List<String>> get(
	URI uri, 
	Map<String, List<String>> requestHeaders) throws IOException


//✅ Вызывается при получении ответа от сервера.
// Добавляет cookie-данные из заголовков в хранилище.
// Параметры:
//  uri — адрес ресурса;
//  responseHeaders — заголовки ответа (Set-Cookie, Set-Cookie2 и др.).
public abstract void put(
	URI uri, 
	Map<String, List<String>> responseHeaders) throws IOException
```
💬 ❗❗❗Эти методы реализуются в `CookieManager`,  
где cookies сохраняются в `CookieStore` и фильтруются через `CookiePolicy`.

## 💡 Особенности и подводные камни 💡

- ❗❗❗`CookieHandler` — **абстрактный**, используется только через `CookieManager`
    
- ⭐⭐Работает автоматически: при каждом HTTP-запросе `HttpURLConnection`
    добавляет cookies из хранилища, а при ответе сохраняет новые.
    
- 💡Cookie-обработка зависит от **политики (`CookiePolicy`)**,  
    которая определяет, какие cookies разрешено сохранять.
    
- 🚫 Если не вызвать `CookieHandler.setDefault(...)`, cookies **НЕ БУДУТ СОХРАНЯТЬСЯ между запросами**.
    
- 💥 В многопоточных приложениях важно использовать **один общий CookieManager**  
    для всего приложения, чтобы обеспечить согласованное поведение.


---
## 💎 Best Practices 💎

✅ **Устанавливай `CookieManager` один раз** при старте приложения:
```java
CookieManager manager = new CookieManager();
CookieHandler.setDefault(manager);
```

✅ **Используй политику и собственное хранилище:**
```java
CookieManager manager = new CookieManager(
    new InMemoryCookieStore(), CookiePolicy.ACCEPT_ALL);
```

✅ **Для постоянного хранения** (например, логин-сессии) реализуй свой `CookieStore`,  
который записывает cookies в файл или базу данных.

✅ **Для отладки** можно получить все cookies:
```java
CookieStore store = manager.getCookieStore();
for (HttpCookie c : store.getCookies())
    System.out.println(c);
```

## Пример использования
```java
import java.net.*;
import java.io.*;
import java.util.*;

public class CookieDemo {
    public static void main(String[] args) throws Exception {
        // 1️⃣ Создаём CookieManager и регистрируем как глобальный обработчик
        CookieManager manager = new CookieManager(
	        null, 
	        CookiePolicy.ACCEPT_ALL);
        CookieHandler.setDefault(manager);


        // 2️⃣ Делаем первый запрос (сервер установит cookie)
        URL url = new URL("https://httpbin.org/cookies/set?session=12345");
        try (InputStream in = url.openStream()) {
            in.readAllBytes();
        }

        // 3️⃣ Делаем второй запрос — cookie автоматически добавятся
        URL url2 = new URL("https://httpbin.org/cookies");
        try (BufferedReader reader = new BufferedReader(
                new InputStreamReader(url2.openStream()))) {
            System.out.println("--- Ответ сервера ---");
            reader.lines().forEach(System.out::println);
        }

        // 4️⃣ Просматриваем cookies, сохранённые в CookieStore
        System.out.println("\n--- Сохранённые cookies ---");
        CookieStore store = manager.getCookieStore();
        for (HttpCookie cookie : store.getCookies())
            System.out.println(cookie);
    }
}
```
