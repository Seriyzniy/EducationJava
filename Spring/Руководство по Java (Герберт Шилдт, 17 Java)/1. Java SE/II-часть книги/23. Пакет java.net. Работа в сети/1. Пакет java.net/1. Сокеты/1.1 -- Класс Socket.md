
> [!NOTE] Класс **`java.net.Socket`** используется `для создания клиентских TCP/IP-сокетов` — то есть точек подключения к удалённым хостам.
> Он ==обеспечивает **надёжную, ДВУНАПРАВЛЕННУЮ, потоковую связь**== между двумя программами через сеть.
> 
> 💡Реализует ИНТЕРФЕЙС: `AutoCloseable`


`Socket` — это клиентская сторона TCP-соединения.  
Для серверной стороны используется класс `ServerSocket`

## Конструкторы
```JAVA
//✅ СОЗАДЕТ сокет и УСТАНАВЛИВАЕТ TCP-соединение с указанным хостом и портом.
// Если хост не найден → генерируется UnknownHostException.
// Если не удалось установить соединение → IOException.
Socket(String host, int port)
    throws UnknownHostException, IOException

//✅ То же самое, но принимает объект InetAddress вместо строки с именем.
// Полезно, когда адрес уже получен через InetAddress.getByName().
Socket(InetAddress address, int port)
    throws IOException

```

🔹 Создание объекта `Socket` **автоматически устанавливает соединение** с удалённым хостом.  
Если соединение не удалось — будет выброшено исключение.

## Методы
### Получение информации о соединении
```JAVA
//✅ Возвращает объект InetAddress, связанный с удалённым хостом.
// Если сокет не подключен — возвращает null.
InetAddress getInetAddress()

//✅ Возвращает номер удалённого порта, к которому подключен сокет.
// Если сокет не подключен — возвращает 0.
int getPort()

//✅ Возвращает номер локального порта, к которому привязан сокет.
// Если сокет не привязан — возвращает -1.
int getLocalPort()
```

### Работа с потоками ввода/вывода
```java
//✅ Возвращает объект InputStream, СВЯЗАННЫЙ С СОКЕТОМ.
// Через него можно читать данные, поступающие от сервера.
InputStream getInputStream()
    throws IOException

//✅ Возвращает объект OutputStream, связанный с сокетом.
// Через него можно отправлять данные серверу.
OutputStream getOutputStream()
    throws IOException
```
Эти потоки используются так же, как обычные потоки ввода-вывода в Java (см. `java.io`).
ПИСАТЬ/ЧИТАТЬ сокет можно через ПОТОКИ ВВОДА/ВЫВОДА

### Управление состоянием сокета
```java
//✅ Возвращает true, если сокет успешно подключен к серверу.
boolean isConnected()

//✅ Возвращает true, если сокет привязан к адресу (т.е. порт выделен).
boolean isBound()

//✅ Возвращает true, если сокет был закрыт.
boolean isClosed()

//✅ Закрывает сокет и связанные с ним потоки ввода/вывода.
// После закрытия дальнейшая работа с сокетом невозможна.
void close()
    throws IOException
```

### Подключение и проверка соединения (дополнительные)
```java
//✅ Устанавливает новое соединение с указанным адресом.
void connect(SocketAddress endpoint)
    throws IOException

//✅ Устанавливает соединение с возможностью указать тайм-аут (в миллисекундах).
void connect(SocketAddress endpoint, int timeout)
    throws IOException
```

## 💡 Особенности и подводные камни 💡

- 💡**Создание `Socket` = автоматическое подключение.**  
    Это может заблокировать поток, если сервер не отвечает.
    
- ⏱️ При работе с `connect()` важно указывать **тайм-аут**, иначе поток зависнет.
    
- 💡Потоки ввода/вывода **автоматически закрываются** при закрытии сокета.
    
- ❗❗❗ Каждый вызов `read()` может блокировать выполнение до получения данных.
(или см [[2.1.2 --- Абстрактный AsynchronousSocketChannel]])
    
- ❗❗❗ После вызова `close()` — сокет и его потоки становятся **недействительными**.
    
- Использовать один и тот же `Socket` из разных потоков без синхронизации опасно.
    
- 🧠 При обмене двоичными данными важно учитывать **кодировку** и границы сообщений.

## 💎 Best Practices 💎

✅ Используй `try-with-resources` для автоматического закрытия сокета:
```java
try (Socket socket = new Socket("example.com", 80)) {
    // работа с сокетом
}
```

✅ Проверяй соединение перед отправкой данных:
```java
if (socket.isConnected() && !socket.isClosed()) {
    // можно писать в поток
}
```

✅ При долгих операциях задавай **тайм-аут**:
```java
socket.connect(new InetSocketAddress("example.com", 80), 3000);
```

✅ Всегда **обрабатывай исключения**, особенно `UnknownHostException` и `IOException`.

✅ Используй **буферизацию** (`BufferedReader`, `BufferedWriter`) при работе с текстовыми потоками.

## Пример
```java
import java.io.*;
import java.net.*;

public class WhoisClient {
    public static void main(String[] args) {
        String domain = args.length > 0 ? args[0] : "example.com";

        // try-with-resources для автоматического закрытия сокета
        try (Socket socket = new Socket("whois.internic.net", 43)) {

            // Получаем потоки
            InputStream in = socket.getInputStream();
            OutputStream out = socket.getOutputStream();

            // Отправляем запрос
            out.write((domain + "\n").getBytes());

            // Читаем ответ
            int c;
            while ((c = in.read()) != -1)
                System.out.print((char) c);

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```
