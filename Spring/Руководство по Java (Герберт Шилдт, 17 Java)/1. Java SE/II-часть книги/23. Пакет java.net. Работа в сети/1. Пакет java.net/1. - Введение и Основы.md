Java изначально создавалась с акцентом на сетевое программирование.  
Главная сила — пакет **`java.net`**, а начиная с **JDK 11** — новый модуль **`java.net.http`**, обеспечивающий работу с современными HTTP-клиентами (HTTP Client API).

---

# 📕 1. Основы работы в сети

**Ключевая идея — сокеты.**  
Сокет — это конечная точка сетевого соединения.  
Программа может слушать порт (сервер) или подключаться к нему (клиент).  
Порт — это просто номер (0–65535), через который идентифицируется конкретное приложение на машине.

**Протоколы:**
- **IP** — маршрутизация пакетов (ненадежный).
- **TCP** — надежная доставка (контроль порядка, повторная отправка).
- **UDP** — быстрая, но ненадежная доставка без установления соединения.
    
**Порты:**

|Протокол|Порт|
|---|---|
|FTP|21|
|Telnet|23|
|SMTP (почта)|25|
|HTTP|80|
|HTTPS|443|

**IP-адреса:**
- IPv4 — 32 бита (например `192.168.0.1`)
- IPv6 — 128 бит (например `2001:db8::1`)

DNS преобразует имена (`google.com`) в IP-адреса
