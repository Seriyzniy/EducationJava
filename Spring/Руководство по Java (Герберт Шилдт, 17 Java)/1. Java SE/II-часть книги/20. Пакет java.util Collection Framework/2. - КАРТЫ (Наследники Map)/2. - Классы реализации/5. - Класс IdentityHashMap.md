
> [!NOTE] Класс `IdentityHashMap` расширяет класс `AbstractMap` и реализует интерфейс `Мар`.
>  Он похож на HashMap, но ==ПРИ СРАВНЕНИИ элементов использует РАВЕНСТВО ССЫЛОК.==

## Под капотом

Основа — **обычный массив `Object[]` таблица**, в котором ключ и значение хранятся **парами в соседних ячейках**
```java
[key1, value1, key2, value2, ..., null, null, ...]
```

В отличие от `HashMap`, здесь **нет Entry-объектов** (экономия памяти).
    
Поиск и вставка работают через:
- `System.identityHashCode(key)` → ==вычисляется хэш по ссылке== (адресу объекта в памяти), а не через `key.hashCode()`.
	
- Далее идёт поиск свободного места методом **линейного пробинга** (перебор соседних слотов).

Проверка равенства ключей всегда по **`==`** (сравнение ссылок), а не по `equals()`.

## Конструкторы
```java
// Создаёт пустую карту с дефолтной ёмкостью (32 бакета)
IdentityHashMap()

// Создаёт карту, рассчитанную на указанное количество элементов
IdentityHashMap(int expectedMaxSize)

// Создаёт карту и копирует все пары из другой Map
IdentityHashMap(Map<? extends K, ? extends V> m)
```

## Методы
```JAVA
int size()                     // количество пар
boolean isEmpty()              // пустая ли карта
V get(Object key)              // получить значение по ссылке (==)
V put(K key, V value)          // добавить/заменить по ссылке
V remove(Object key)           // удалить по ссылке

boolean containsKey(Object k)  // есть ли ключ (==)
boolean containsValue(Object v)// есть ли значение (equals)

void clear()                   // очистить карту

Set<K> keySet()                // множество ключей
Collection<V> values()         // коллекция значений
Set<Map.Entry<K,V>> entrySet() // множество пар

```

***
## 💡 Особенности и подводные камни 💡
- 🔑 Два разных объекта с одинаковым содержимым (`new String("x")` и `"x"`) → **разные ключи**.

***
## ✅ Best Practices ✅

- ✅ Применять, если нужно хранить данные **строго по ссылке**, а не по логическому содержимому.
    
- ❌ Не использовать в бизнес-коде, где ключи должны быть "логически равны".
    
- Хорошо подходит для:
    - интернирования строк,
    - графов объектов,
    - кэшей низкоуровневых библиотек.

***
## Пример
```java
import java.util.IdentityHashMap;

public class IdentityHashMapDemo {
    public static void main(String[] args) {
        IdentityHashMap<String, String> map = new IdentityHashMap<>();

        String a1 = new String("key");
        String a2 = new String("key");

        map.put(a1, "Value1");
        map.put(a2, "Value2");

        System.out.println(map.size()); // 2 (разные ссылки)

        HashMap<String, String> regular = new HashMap<>();
        regular.put(a1, "Value1");
        regular.put(a2, "Value2");

        System.out.println(regular.size()); // 1 (equals true)
    }
}

```
