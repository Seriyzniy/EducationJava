В инфраструктуре Collections Framework определено несколько основных интерфейсов. Далее представлен обзор каждого интерфейса. 

Начинать с интерфейсов коллекций необходимо из-за того, что они определяют фундаментальную природу классов коллекций.

| Интерфейс / Класс | Наследование (Расширяет)          | Основное назначение                                                                                  |
| ----------------- | --------------------------------- | ---------------------------------------------------------------------------------------------------- |
| **Collection**    | - (Находится на вершине иерархии) | Позволяет работать с группами объектов.                                                              |
| **List**          | Collection                        | Поддержка последовательностей (списков объектов).                                                    |
| **Set**           | Collection                        | Поддержка наборов, которые должны содержать уникальные элементы.                                     |
| **SortedSet**     | Set                               | Поддержка сортированных наборов.                                                                     |
| **NavigableSet**  | SortedSet                         | Расширяет SortedSet для поддержки извлечения элементов на базе поиска с наиболее точным совпадением. |
| **Queue**         | Collection                        | Поддержка специальных типов списков, в которых элементы удаляются только из головы.                  |
| **Deque**         | Queue                             | Расширяет Queue для поддержки двусторонней очереди.                                                  |
Помимо интерфейсов коллекций в коллекциях также используются интерфейсы `Comparator`, `RandomAccess`, `Iterator`, `Listiterator` и `Spliterator`, которые подробно описаны далее в главе.

***
# 📕 1. Интерфейс Collection

> [!NOTE] Интерфейс `Collection`
>  Это ==основа, на которой построена инфраструктура Collections Framework==, т.к. он `должен быть реализован любым классом, определяющим коллекцию`. 


Collection представляет собой обобщенный интерфейс с таким объявлением:
```java
interface Collection<E>
```
С помощью `Е` указывается тип объектов, которые будут храниться в кол лекции. 

⭐⭐Интерфейс `Collection` ==расширяет== `Iterable`, т.е. по всем коллекциям можно проходить в цикле for стиля "`for-each`'

## Генерируемые исключения в методах
- `UnsupportedOperationException` - если коллекцию изменять нельзя. 
	
- `ClassCastException` - когда один объект несовместим с друrим, скажем, при попытке добавления в коллекцию несовместимого объекта. 
	
- `NullPointerException` - если предпринимается попытка сохранить объект `null`, а элементы `null` в коллекции не разрешены. 
	
- `IllegalArgumentException` -  в случае применения недопустимого аргумента. 
	
- `IllegalStateException` - при попытке добавления элемента в заполненную коллекцию фиксированной длины.

## Методы
### ⭐ Самые важные методы:
- `add(E obj)` - базовое добавление элемента
    
- `contains(Object obj)` - проверка наличия элемента
    
- `iterator()` - получение итератора для обхода
    
- `stream()` - интеграция со Stream API
    
- `toArray()` - преобразование в массив


```java
// ИНТЕРФЕЙС COLLECTION<E>
// ✅ Базовые операции добавления ✅
/**
 * Добавляет obj в вызывающую коллекцию. 
 * Возвращает true в случае успешного добавления obj. 
 * Возвращает false, если obj уже является членом коллекции, а дубликаты в коллекции не разрешены.
 */
boolean add(E obj);

/**
 * Добавляет все элементы коллекции c в вызывающую коллекцию.
 * Возвращает true, если коллекция изменилась (т.е. элементы были добавлены), или false в противном случае.
 */
boolean addAll(Collection<? extends E> c);



// ✅ Базовые операции удаления✅ 
/**
 * Удаляет из вызывающей коллекции все элементы.
 */
void clear();

/**
 * Удаляет из вызывающей коллекции один экземпляр obj.
 * Возвращает true, если элемент был удален, или false в противном случае.
 */
boolean remove(Object obj);

/**
 * Удаляет из вызывающей коллекции все элементы коллекции c.
 * Возвращает true, если коллекция изменилась (т.е. элементы были удалены), или false в противном случае.
 */
boolean removeAll(Collection<?> c);

/**
 * Удаляет из вызывающей коллекции элементы, которые удовлетворяют условию, указанному в predicate.
 */
default boolean removeIf(Predicate<? super E> predicate);

/**
 * Удаляет из вызывающей коллекции все элементы кроме тех, что находятся в коллекции c.
 * Возвращает true, если коллекция изменилась (т.е. элементы были удалены), или false в противном случае.
 */
boolean retainAll(Collection<?> c);



// ✅ Операции проверки✅ 
/**
 * Возвращает true, если obj является элементом вызывающей коллекции, или false в противном случае.
 */
boolean contains(Object obj);

/**
 * Возвращает true, если вызывающая коллекция содержит все элементы коллекции c, или false в противном случае.
 */
boolean containsAll(Collection<?> c);

/**
 * Возвращает true, если вызывающая коллекция пуста, или false в противном случае.
 */
boolean isEmpty();

/**
 * Возвращает количество элементов, содержащихся в вызывающей коллекции.
 */
int size();



// ✅ Операции сравнения и хеширования✅ 
/**
 * Возвращает true, если вызывающая коллекция и obj равны. В противном случае возвращает false.
 */
boolean equals(Object obj);

/**
 * Возвращает хеш-код для вызывающей коллекции.
 */
int hashCode();



// ✅ Операции итерации ✅
/**
 * Возвращает итератор для вызывающей коллекции.
 */
Iterator<E> iterator();

/**
 * Возвращает сплитератор для вызывающей коллекции.
 */
default Spliterator<E> spliterator();



// ✅ Операции преобразования в массив✅ 
/**
 * Возвращает массив элементов из вызывающей коллекции.
 */
Object[] toArray();

/**
 * Возвращает массив элементов из вызывающей коллекции.
 * Если размер массива равен количеству элементов в коллекции, то эти элементы возвращаются в массиве.
 * Если размер массива меньше количества элементов в коллекции, тогда выделяется память под новый массив нужного размера, который затем возвращается.
 * Если размер массива больше количества элементов в коллекции, тогда элементы массива, следующие после последнего элемента коллекции, устанавливаются в null.
 * Если любой элемент коллекции имеет тип, который не совместим с типом массива, тогда генерируется исключение ArrayStoreException.
 */
<T> T[] toArray(T[] array);

/**
 * Возвращает массив элементов из вызывающей коллекции.
 * Возвращаемый массив создается функцией, указанной в arrayGen.
 * Если любой элемент коллекции имеет тип, который не совместим с типом массива, тогда генерируется исключение ArrayStoreException.
 */
default <T> T[] toArray(IntFunction<T[]> arrayGen);



// ✅ Операции работы с потоками (Stream API) ✅
/**
 * Возвращает поток данных, который использует вызывающую коллекцию в качестве своего источника элементов.
 * Поток данных является последовательным.
 */
default Stream<E> stream();

/**
 * Возвращает поток данных, который использует вызывающую коллекцию в качестве своего источника элементов.
 * Если возможно, то поток данных поддерживает параллельные операции.
 */
default Stream<E> parallelStream();
```
