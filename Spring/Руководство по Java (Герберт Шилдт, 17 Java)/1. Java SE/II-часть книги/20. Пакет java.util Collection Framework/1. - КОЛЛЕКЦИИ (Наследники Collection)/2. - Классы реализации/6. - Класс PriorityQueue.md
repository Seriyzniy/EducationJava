
> [!NOTE] `PriorityQueue<E>` — это реализация интерфейса `Queue`, представляющая `ОЧЕРЕДЬ С ПРИОРИТЕТОМ`.
> Класс `PriorityQueue` расширяет `AbstractQueue` и реализует интерфейс `Queue`. Он создает ==очередь с приоритетом на основе компаратора очереди==

Под капотом используется **==минимальная двоичная куча (min-heap)==**, которая обеспечивает:
- 💡Элемент с наименьшим приоритетом (`compareTo` или `Comparator`) всегда    💡находится в голове очереди. (извлекается последним, если поступают элементы с большим приоритетом)
    
- Порядок извлечения не зависит от времени вставки (в отличие от `LinkedList` или `ArrayDeque`), а от приоритета.

⚠️ Важно: `PriorityQueue` **не гарантирует полного порядка при итерации** — только порядок "головы". То есть `iterator()` не даёт отсортированный список.

***

> [!NOTE] ⭐`Стандартный компаратор` упорядочивает очередь `в порядке возрастания.`⭐
> 

## Конструкторы
```java
// Создаёт пустую очередь с начальной ёмкостью 11 и "естественным" порядком элементов
public PriorityQueue()

// Создаёт пустую очередь с указанной ёмкостью
public PriorityQueue(int initialCapacity)

// Создаёт пустую очередь с указанным компаратором
public PriorityQueue(Comparator<? super E> comparator)

// Создаёт пустую очередь с указанной ёмкостью и компаратором
public PriorityQueue(int initialCapacity, Comparator<? super E> comparator)

// 💡Создаёт очередь, содержащую элементы другой коллекции (натуральный порядок)
public PriorityQueue(Collection<? extends E> c)

// Создаёт очередь, содержащую элементы другого PriorityQueue (копия кучи)
public PriorityQueue(PriorityQueue<? extends E> c)

// Создаёт очередь, содержащую элементы другого SortedSet (порядок сохранён)
public PriorityQueue(SortedSet<? extends E> c)
```

## Методы
```java
// ✅ДОБАВЛЕНИЕ✅
// Добавляет элемент в очередь (если есть место или динамически расширяется)
public boolean add(E e)

// То же самое, что add(), но предпочтительно использовать offer() для очередей
public boolean offer(E e)

// ✅Получение и удаление/Получение и null✅
// Возвращает и удаляет "голову" очереди (элемент с наименьшим приоритетом)
// Если очередь пуста — выбрасывает NoSuchElementException
public E remove()

// То же самое, но возвращает null, если очередь пуста (более безопасный вариант)
public E poll()

// ✅Получение и Exception/Получение и null✅
// Возвращает, но не удаляет голову очереди
// Если очередь пуста — выбрасывает NoSuchElementException
public E element()

// То же самое, но возвращает null, если очередь пуста
public E peek()

// ✅Специальные методы✅
// Проверка наличия элемента
public boolean contains(Object o)

// Размер очереди
public int size()

// Пустая ли очередь
public boolean isEmpty()

// Удалить все элементы
public void clear()

// ✅Преобразование в массив✅
// Возвращает массив со всеми элементами очереди (порядок не гарантирован как сортированный!)
public Object[] toArray()
public <T> T[] toArray(T[] a)

// ✅Итератор✅
// Возвращает итератор (элементы в произвольном порядке, НЕ по приоритету)
public Iterator<E> iterator()

// ✅Компаратор✅
// Получить ссылку на компаратор
Comparator<? super E> comparator()
```

## 💡Особенности и подводные камни💡

- 🔶Под капотом: **минимальная двоичная куча** на базе массива.
    - Индекс `0` — минимальный элемент.
        
    - `offer` вставляет в конец массива и "поднимает" элемент вверх (heapify up).
        
    - `poll` меняет голову с последним элементом и "опускает" вниз (heapify down).
        
- 🔶⭐**Уникальность НЕ ГАРАНТИРУЕТСЯ** → можно хранить дубликаты.
    
- 🔶**Null запрещён** → попытка добавить `null` выбросит `NullPointerException`.
    
- 🔶⭐**Итератор не отсортирован** → нельзя полагаться на `for-each` для упорядоченного обхода.
    
- 🔶💡**Не потокобезопасна** → если нужна многопоточность, используйте `PriorityBlockingQueue`.
    
- 🔶💡**Оценка сложности**:
    - `offer` / `add` / `poll` / `remove` → `O(log n)`
        
    - `peek` → `O(1)`
        
    - `contains` → `O(n)` (так как поиск по массиву)

***
## ✅ Best Practices ✅
- Используйте `PriorityQueue`, когда важен ==**приоритетный доступ к минимальному/максимальному элементу**==, а не порядок вставки.
    
- 🧠Для извлечения в **отсортированном порядке** лучше перегонять в `ArrayList` и вызывать `Collections.sort()`, либо использовать `TreeSet`.
    
- Если нужен **максимум**, а не минимум — используйте `Comparator.reverseOrder()`.
    
- ⭐⭐Для многопоточной обработки — `PriorityBlockingQueue`.

***
## Пример
```JAVA
import java.util.*;

public class PriorityQueueExample {
    public static void main(String[] args) {
        // Очередь по умолчанию: минимальный элемент первый
        PriorityQueue<Integer> pq = new PriorityQueue<>();

        pq.add(10);
        pq.add(5);
        pq.add(20);
        pq.add(15);

        System.out.println("Очередь (неупорядоченный вывод через toString): " + pq);
        // Порядок может быть [5, 10, 20, 15], но это не сортировка — структура кучи

        System.out.println("Голова очереди (peek): " + pq.peek()); // 5

        while (!pq.isEmpty()) {
            System.out.println("poll(): " + pq.poll());
        }
        // Элементы выйдут в порядке возрастания: 5, 10, 15, 20

        // Очередь по приоритету "максимум первый"
        PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Comparator.reverseOrder());
        maxHeap.addAll(Arrays.asList(10, 5, 20, 15));

        System.out.println("Максимальный элемент первый (peek): " + maxHeap.peek()); // 20
    }
}
```
