
> [!NOTE] `LinkedList<E>` — это `двусвязный список`
> Реализующий интерфейсы:
> - `List` (индексированный доступ к элементам), 
> - `Deque` (двусторонняя очередь — можно работать и как очередь, и как стек).
> 
> В отличие от `ArrayList`, ==хранит **узлы== (`Node`)**, где каждый элемент знает **предыдущего и следующего**.

### Характеристики:
- **Порядок сохраняется**.
    
- **Допускает дубликаты и `null`**.
    
- **Не синхронизирован**.
    
- Быстрая вставка/удаление в середину (**O(1)** при известном узле, но **O(n)** при поиске по индексу).
    
- Медленный `get(index)` (**O(n)**), так как нужно пройти по связям.

***
## Конструкторы
```java
// Создаёт пустой список.
public LinkedList()

// 💡Создаёт список, содержащий элементы другой коллекции.
// Элементы копируются в том порядке, в котором они итерируются в коллекции.
public LinkedList(Collection<? extends E> c)
```

***
## Методы
`LinkedList` поддерживает все методы `List`, но дополнительно реализует методы `Deque` — работа с головой/хвостом.
```java
// 📙==== Методы List ====📙
// ✅Добавление✅
// Добавляет элемент в конец.
public boolean add(E e)

// Вставляет элемент по индексу.
// O(n), так как нужно дойти до index.
public void add(int index, E element)

// Добавляет все элементы коллекции c в конец.
public boolean addAll(Collection<? extends E> c)

// Вставляет коллекцию c начиная с index.
public boolean addAll(int index, Collection<? extends E> c)

// ✅Удаление✅
// Удаляет элемент по индексу.
// O(n), нужно найти элемент.
public E remove(int index)

// Удаляет первый встреченный элемент по equals().
public boolean remove(Object o)

// ✅Получение и замена элемента✅
// Возвращает элемент по индексу.
// O(n).
public E get(int index)

// Заменяет элемент по индексу.
public E set(int index, E element)


// ✅Инфа о списке✅
// Возвращает количество элементов.
public int size()

// Проверка на пустоту.
public boolean isEmpty()

// Проверяет, содержится ли элемент.
public boolean contains(Object o)

// Возвращает индекс первого вхождения.
public int indexOf(Object o)

// Возвращает индекс последнего вхождения.
public int lastIndexOf(Object o)

// ✅Итераторы✅
// Возвращает итератор.
public Iterator<E> iterator()

// Возвращает ListIterator.
public ListIterator<E> listIterator()


// 📙==== Методы Deque ====📙
// ✅Добавление✅
// Добавляет элемент в начало.
public void addFirst(E e)

// Добавляет элемент в конец (эквивалент add()).
public void addLast(E e)

// ✅Получение✅
// Возвращает первый элемент (не удаляет).
// Бросает NoSuchElementException, если пусто.
public E getFirst()

// Возвращает последний элемент (не удаляет).
public E getLast()

// ✅Удаление✅
// Удаляет и возвращает первый элемент.
public E removeFirst()

// Удаляет и возвращает последний элемент.
public E removeLast()


// 📙==== Методы как Queue ====📙
// ✅Добавление✅
// Добавляет элемент в конец (эквивалент offerLast).
public boolean offer(E e)

// Добавляет в начало.
public boolean offerFirst(E e)

// Добавляет в конец.
public boolean offerLast(E e)

// Возвращает первый элемент или null, если пусто (peek).
public E peek()

// Возвращает первый элемент или null.
public E peekFirst()

// Возвращает последний элемент или null.
public E peekLast()

// Удаляет и возвращает первый элемент или null.
public E poll()

// Удаляет и возвращает первый элемент или null.
public E pollFirst()

// Удаляет и возвращает последний элемент или null.
public E pollLast()


// 📙==== Методы как Stack ====📙 
// Добавляет элемент наверх стека (эквивалент push).
public void push(E e)

// Удаляет и возвращает элемент сверху стека.
public E pop()
```

***
## 💡 Особенности и подводные камни💡

1. 🔶**Сложность доступа**
    - `get(index)` — **O(n)** (нужно пройти список).
        
    - Но оптимизировано: если индекс ближе к началу, идёт с головы, если ближе к концу — с хвоста.
        
2. 🔶**Вставка/удаление**
    - При известном узле (через итератор) — **O(1)**.
        
    - При вставке по индексу всё равно надо искать узел → **O(n)**.
        
3. 🔶**Использование памяти**
    - Каждый элемент хранит 2 ссылки (`prev`, `next`) → `LinkedList` расходует больше памяти, чем `ArrayList`.
        
4. 🔶**Итератор**
    - ⭐В `LinkedList` удаление через итератор быстрее, чем через `ArrayList` (нет сдвига массива).
        
5. 🔶**Потокобезопасность**
    - Как и `ArrayList`, не синхронизирован.

***
## ✅ Best Practices ✅
- Используй `LinkedList`, если:
    - много **вставок/удалений** в середине списка;
    - нужен **Deque/Queue/Stack** (FIFO, LIFO).
        
- Не используй для частых `get(index)` — лучше `ArrayList`.
    
- Для многопоточности — либо `Collections.synchronizedList(new LinkedList<>())`, либо `ConcurrentLinkedDeque`.
    
- Если нужна очередь — предпочитай специализированные классы (`ArrayDeque`) вместо `LinkedList` (лучше по производительности).

## Пример
```java
import java.util.*;

public class LinkedListExample {
    public static void main(String[] args) {
        // Создаём LinkedList
        LinkedList<String> list = new LinkedList<>();

        // Добавление элементов
        list.add("Java");
        list.addFirst("C++"); // в начало
        list.addLast("Python"); // в конец

        // Вставка по индексу
        list.add(1, "Rust");

        // Доступ к элементам
        System.out.println("Первый: " + list.getFirst()); // C++
        System.out.println("Последний: " + list.getLast()); // Python
        System.out.println("По индексу [2]: " + list.get(2)); // Java

        // Удаление
        list.remove("Rust");
        list.removeFirst();
        list.removeLast();

        // 💡Использование как стек
        list.push("Go");
        list.push("Kotlin");
        System.out.println("pop: " + list.pop()); // Kotlin

        // 💡Использование как очередь
        list.offer("Scala");
        list.offer("Elixir");
        System.out.println("poll: " + list.poll()); // Go

        // Итерация
        for (String lang : list) {
            System.out.println(lang);
        }
    }
}
```
