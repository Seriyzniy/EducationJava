
> [!NOTE] `ArrayDeque<E>` - `двусторонняя очередь` (double-ended queue), *реализованная на* динамическом массиве (`циклическом буфере`)
> Класс `ArrayDeque` расширяет `AbstractCollection` и реализует интерфейс `Deque`.

`ArrayDeque` создает ==динамический массив и не имеет ограничений по емкости==. (Интерфейс `Deque` поддерживает реализации, которые ограничивают емкость, но не требует таких ограничений.)

🧠Используется в случаях, когда нужна **очередь FIFO**, **стек LIFO** или комбинация этих сценариев.

***
## Конструкторы
```java
// Создаёт пустую очередь с начальной ёмкостью по умолчанию (16 элементов)
public ArrayDeque()

// Создаёт пустую очередь с указанной ёмкостью
public ArrayDeque(int numElements)

// Создаёт очередь, содержащую элементы указанной коллекции,
// элементы добавляются в порядке итератора этой коллекции
public ArrayDeque(Collection<? extends E> c)
```

## Методы
💡Никакие собственные методы не добавляются

`ArrayDeque` реализует **Deque**, поэтому есть полный набор методов для работы с концами очереди.
```java
// ✅ Добавление элементов ✅
// Добавляет элемент в хвост (как offer в Queue)
public boolean add(E e)

// Добавляет элемент в хвост, возвращает false при ошибке (но в ArrayDeque всегда true)
public boolean offer(E e)

// Добавляет элемент в голову (передний конец)
public void addFirst(E e)

// Добавляет элемент в хвост (задний конец)
public void addLast(E e)

// Аналог addFirst, но возвращает false при ошибке
public boolean offerFirst(E e)

// Аналог addLast, но возвращает false при ошибке
public boolean offerLast(E e)


// ✅ Удаление элементов ✅
// Удаляет и возвращает голову (первый элемент), выбрасывает исключение если пусто
public E remove()

// Аналог remove, но возвращает null при пустой очереди
public E poll()

// Удаляет и возвращает первый элемент
public E removeFirst()

// Удаляет и возвращает последний элемент
public E removeLast()

// Аналог removeFirst, но возвращает null если пусто
public E pollFirst()

// Аналог removeLast, но возвращает null если пусто
public E pollLast()


// ✅ Просмотр элементов (без удаления) ✅
// Возвращает голову (первый элемент), исключение если пусто
public E element()

// Аналог element, но возвращает null при пустоте
public E peek()

// Возвращает первый элемент
public E getFirst()

// Возвращает последний элемент
public E getLast()

// Аналог getFirst, но возвращает null если пусто
public E peekFirst()

// Аналог getLast, но возвращает null если пусто
public E peekLast()


// ✅ Прочее ✅
// Удаляет первое вхождение объекта
public boolean removeFirstOccurrence(Object o)

// Удаляет последнее вхождение объекта
public boolean removeLastOccurrence(Object o)

// Размер очереди
public int size()

// Проверка пустоты
public boolean isEmpty()

// Очистка
public void clear()

// Итератор от головы к хвосту
public Iterator<E> iterator()

// Итератор от хвоста к голове
public Iterator<E> descendingIterator()

// Преобразование в массив
public Object[] toArray()
public <T> T[] toArray(T[] a)
```

***
## Особенности и подводные камни
- 🔶💡**Под капотом:** реализован как **циклический массив**, размер которого удваивается при переполнении.
    
- 🔶**null ЗАПРЕЩЕН** — попытка вставить `null` выбросит `NullPointerException`.
    
- 🔶💡**Быстрее `LinkedList`** почти всегда, потому что `ArrayDeque` хранит данные в массиве, а `LinkedList` требует аллокации узлов и кеш хуже.
    
- 🔶💡**Не потокобезопасен** — при многопоточности используйте `ConcurrentLinkedDeque`.
    
- 🔶💡💡**Нет ограничений по размеру** кроме доступной памяти.
    
- 🔶Итерация не гарантирует стабильность при изменениях в процессе обхода.

***

## Пример
```java
import java.util.*;

public class ArrayDequeExample {
    public static void main(String[] args) {
        ArrayDeque<Integer> deque = new ArrayDeque<>();

        // Используем как очередь FIFO
        deque.offer(10);
        deque.offer(20);
        deque.offer(30);

        System.out.println("Очередь: " + deque); // [10, 20, 30]
        System.out.println("peek(): " + deque.peek()); // 10
        System.out.println("poll(): " + deque.poll()); // 10
        System.out.println("После poll: " + deque); // [20, 30]

        // Используем как стек LIFO
        deque.push(100); // addFirst()
        deque.push(200);
        System.out.println("Стек: " + deque); // [200, 100, 20, 30]

        System.out.println("pop(): " + deque.pop()); // 200
        System.out.println("После pop: " + deque); // [100, 20, 30]

        // Доступ к концам
        deque.addFirst(5);
        deque.addLast(40);
        System.out.println("После addFirst и addLast: " + deque); // [5, 100, 20, 30, 40]

        // Итерация в обратном порядке
        Iterator<Integer> descIt = deque.descendingIterator();
        System.out.print("Обратный порядок: ");
        while (descIt.hasNext()) {
            System.out.print(descIt.next() + " ");
        }
        // 40 30 20 100 5
    }
}

```
