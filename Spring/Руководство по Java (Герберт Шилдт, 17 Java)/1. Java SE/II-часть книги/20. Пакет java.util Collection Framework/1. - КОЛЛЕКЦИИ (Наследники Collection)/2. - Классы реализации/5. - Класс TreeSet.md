
> [!NOTE] `TreeSet` — это реализация интерфейса `NavigableSet`, основанная на `красно-чёрном дереве` (self-balancing binary search tree).
>  Класс `TreeSet` расширяет `AbstractSet` и реализует интерфейс `NavigableSet`. Он создает коллекцию, которая в качестве хранилища использует дерево.

⭐Она хранит ==элементы в **ОТСОРТИРОВАННОМ порядке**== (согласно естественному порядку элементов или `Comparator`, переданному в конструктор).


> [!NOTE] Быстрый поиск в отсортированном объеме данных 
> TreeSet превосходным вариантом для организации хранения больших объемов отсортированной информации, в которой необходим быстрый поиск.


Главная особенность: **элементы уникальны и упорядочены**.  

🧠Фактически, `TreeSet` — это "обёртка" над `TreeMap`, где значения не хранятся, а в качестве ключей выступают элементы множества.

***
## Конструкторы
```java
// Создаёт пустое множество, элементы будут упорядочены по "естественному порядку" (Comparable) ⭐⭐в порядке возрастания⭐⭐
public TreeSet()

// Создаёт пустое множество, используя переданный компаратор для упорядочивания
public TreeSet(Comparator<? super E> comparator)

// Создаёт множество, содержащее все элементы указанной коллекции,
// элементы будут отсортированы в естественном порядке
public TreeSet(Collection<? extends E> c)

// Создаёт множество, содержащее все элементы указанного SortedSet,
// порядок будет соответствовать этому сету
public TreeSet(SortedSet<E> s)
```

## Методы
```java
//✅Добавление✅
// Добавляет элемент в множество. Возвращает true, если элемент был добавлен,
// и false, если он уже существовал.
public boolean add(E e)

//✅Удаление✅
// Удаляет элемент из множества. Возвращает true, если элемент был удалён.
public boolean remove(Object o)

//✅Проверка содержимого✅
// Проверяет, содержится ли элемент в множестве
public boolean contains(Object o)

//✅Получениеэелементов✅
// Возвращает первый (наименьший) элемент множества
public E first()

// Возвращает последний (наибольший) элемент множества
public E last()

// Возвращает наименьший элемент >= e или null, если такого нет
public E ceiling(E e)

// Возвращает наибольший элемент <= e или null, если такого нет
public E floor(E e)

// Возвращает наименьший элемент > e или null
public E higher(E e)

// Возвращает наибольший элемент < e или null
public E lower(E e)

// Удаляет и возвращает первый элемент множества или null, если пусто
public E pollFirst()

// Удаляет и возвращает последний элемент множества или null, если пусто
public E pollLast()

//✅Подмножество✅
// Возвращает "живое" представление элементов от fromElement (включительно/исключительно)
// до toElement (включительно/исключительно)
public NavigableSet<E> subSet(E fromElement, boolean fromInclusive,
                              E toElement, boolean toInclusive)

// Возвращает "живое" представление элементов меньше указанного
public NavigableSet<E> headSet(E toElement, boolean inclusive)

// Возвращает "живое" представление элементов больше (или равно) указанного
public NavigableSet<E> tailSet(E fromElement, boolean inclusive)

//✅Специальные методы✅
// Возвращает компаратор, используемый этим множеством, или null (если используется Comparable)
public Comparator<? super E> comparator()

// Размер множества
public int size()

// Пустое ли множество
public boolean isEmpty()

// Очистить множество
public void clear()
```

## 💡Особенности и подводные камни💡
- 🔶**Уникальность:** хранит только уникальные элементы. 
	⭐⭐Проверка основана на `compareTo()` или `Comparator.compare()`, а не на `equals()`/`hashCode()`.  
    
    Если компаратор считает два объекта равными (`compare() == 0`), то в множестве будет только один из них.
    
- 🔶**Нельзя хранить `null`:**
    - Если коллекция пустая и компаратор не нужен — `add(null)` вызовет `NullPointerException`.
        
    - 🧠Если передан свой `Comparator`, допускающий `null`, можно хранить `null`, но в стандартной реализации — нет.
        
- 🔶**Производительность:**
    - 💡`add`, `remove`, `contains` — `O(log n)` (за счёт дерева).
        
    - 💡`first`, `last` — `O(log n)`.
        
    - 💡Итерация — `O(n)`, но в отсортированном порядке.
        
- 🔶**Порядок элементов гарантирован:** всегда сортировка.

***
## ✅Best Practices✅
- 💡Используйте `TreeSet`, если ==нужно **упорядоченное множество** с возможностью искать элементы по "ближайшему значению"== (`ceiling`, `floor`, `higher`, `lower`).
    
- Если нужен просто уникальный набор без порядка — используйте `HashSet`.
    
- Если нужен порядок вставки — используйте `LinkedHashSet`.
    
- 💡Для работы с "диапазонами" элементов лучше `TreeSet` (например, `subSet`

***
## Пример
```java
import java.util.*;

public class TreeSetExample {
    public static void main(String[] args) {
        // Создание TreeSet с естественным порядком (Integer -> возрастание)
        TreeSet<Integer> numbers = new TreeSet<>();

        numbers.add(10);
        numbers.add(5);
        numbers.add(20);
        numbers.add(15);

        System.out.println("TreeSet: " + numbers); 
        // [5, 10, 15, 20] (автоматически отсортирован)

        System.out.println("Первый элемент: " + numbers.first());  // 5
        System.out.println("Последний элемент: " + numbers.last()); // 20

        System.out.println("ceiling(12): " + numbers.ceiling(12)); // 15
        System.out.println("floor(12): " + numbers.floor(12));     // 10
        System.out.println("higher(15): " + numbers.higher(15));   // 20
        System.out.println("lower(15): " + numbers.lower(15));     // 10

        // Подмножество [10, 20)
        NavigableSet<Integer> subset = numbers.subSet(10, true, 20, false);
        System.out.println("Подмножество [10,20): " + subset); // [10, 15]
    }
}

```
