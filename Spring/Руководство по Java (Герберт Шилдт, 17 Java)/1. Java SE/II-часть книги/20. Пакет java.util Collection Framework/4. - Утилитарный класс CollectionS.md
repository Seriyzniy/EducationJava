 
> [!NOTE] `java.util.Collections` — это `final utility class`, 
> Содержит `набор СТАТИЧЕСКИХ методов для работы с коллекциями` (`List`, `Set`, `Map`, `Queue`). 

Он ==НЕ имеет публичного конструктора== (вызвать `new Collections()` нельзя). 

Используется как "инструментальный ящик": сортировка, поиск, обёртки для синхронизации, создание неизменяемых коллекций и т.д.

## Методы проверки совместимости типов
Методы `checkedXXX()` (например, `checkedCollection()`, `checkedSet()`, `checkedList()`, `checkedMap()`) возвращают **типобезопасные представления коллекций**. 
Такие обёртки проверяют вставляемые элементы на совместимость типов **во время выполнения**. 

При попытке добавить неподходящий объект генерируется `ClassCastException`. 

Это особенно полезно при **отладке**, чтобы гарантировать, что коллекция содержит только допустимые элементы.

## 💡 Особенности и подводные камни 💡
1. **`binarySearch` работает только на отсортированных списках** — иначе результат непредсказуем.
    
2. **`copy(dest, src)` требует, чтобы размер `dest >= src`**. Иначе `IndexOutOfBoundsException`.
    
3. **`unmodifiableList` — не полноценный "immutable"**: если изменишь исходную коллекцию, изменится и обёртка.
    
4. **`synchronizedList` даёт только базовую потокобезопасность** — но итерироваться всё равно нужно в `synchronized` блоке.

***
## ✅Best Practices ✅
- Используй `Collections.unmodifiableList()` для возврата коллекций наружу, если не хочешь, чтобы их меняли.
    
- ❗❗❗Для многопоточности лучше применять `ConcurrentHashMap` или `CopyOnWriteArrayList`, а не `synchronizedXXX`.
    
- 💡💡Вместо `Collections.sort()` в Java 8+ 💡💡используй `List.sort()` (удобнее и читабельнее).
    
- 💡💡Для инициализации маленьких коллекций лучше использовать методы `of()` (`List.of()`, `Set.of()` (Java 9+)), а не `Collections.singleton()`.

## Пример
```java
import java.util.*;

public class CollectionsExample {
    public static void main(String[] args) {
        List<Integer> numbers = new ArrayList<>(Arrays.asList(5, 3, 8, 1, 2));

        // Сортировка, НО ЛУШЧЕ numbers.sort()
        Collections.sort(numbers);
        System.out.println("Отсортировано: " + numbers);

        // Перемешивание
        Collections.shuffle(numbers);
        System.out.println("Перемешано: " + numbers);

        // Разворот
        Collections.reverse(numbers);
        System.out.println("Разворот: " + numbers);

        // Минимум и максимум
        System.out.println("min = " + Collections.min(numbers));
        System.out.println("max = " + Collections.max(numbers));

        // Поиск
        Collections.sort(numbers);
        int idx = Collections.binarySearch(numbers, 5);
        System.out.println("Индекс числа 5: " + idx);

        // Неподвижный список
        List<Integer> unmodifiable = Collections.unmodifiableList(numbers);
        // unmodifiable.add(10); // Ошибка: UnsupportedOperationException

        // Синхронизированный список
        List<Integer> syncList = Collections.synchronizedList(new ArrayList<>(numbers));
    }
}
```
