
> [!NOTE] `TextEvent` — это событие высокого уровня,  которое генерируется, `когда текст в компоненте` `изменяется`.
> 

🧠 Оно применяется для компонентов, реализующих интерфейс `TextComponent` - например:
- `TextField` (однострочное текстовое поле),
- `TextArea` (многострочная область текста).

## Иерархия
```java
java.lang.Object
   └── java.util.EventObject
        └── java.awt.AWTEvent
             └── java.awt.event.TextEvent
```

`TextEvent` наследуется напрямую от `AWTEvent`,  
то есть НЕ ИМЕЕТ ОТНОШЕНИЯ к `InputEvent` (так как не связан с клавиатурой напрямую).  

Он отражает **изменение состояния содержимого**, а не конкретные действия пользователя.

### 🧭 Назначение
`TextEvent` сообщает, что **текст в компоненте был изменён**.  
Это может быть вызвано:
- набором текста пользователем,
- вставкой через буфер обмена,
- программным изменением методом `setText()`.

Для обработки таких событий используется интерфейс:
```java
public interface TextListener {
    void textValueChanged(TextEvent e);
}
```
> Когда содержимое текстового компонента изменяется, AWT вызывает метод `textValueChanged()` слушателя.

## Конструктор
```java
// Создаёт новое событие изменения текста
// src — компонент, вызвавший событие (обычно TextField или TextArea)
// id — идентификатор события (TextEvent.TEXT_VALUE_CHANGED)
public TextEvent(Object src, int id)
```

### Константы класса
|Константа|Значение|Описание|
|---|---|---|
|`TEXT_FIRST`|900|Первый идентификатор событий текста|
|`TEXT_LAST`|900|Последний идентификатор событий текста|
|`TEXT_VALUE_CHANGED`|900|Содержимое текстового компонента изменено|

### Основные поля
|Поле|Тип|Описание|
|---|---|---|
|`source`|`Object`|Компонент-источник (обычно `TextField` или `TextArea`)|
|`id`|`int`|Идентификатор события (`TEXT_VALUE_CHANGED`)|

## Методы
Класс `TextEvent` содержит всего несколько публичных методов,  
так как его ==задача== — ==просто сообщить о факте изменения текста==.

### Работа с источником события
```java
// Возвращает компонент, в котором изменился текст
// Источник всегда реализует интерфейс TextComponent
public Object getSource()
```

> [!NOTE] Чтобы безопасно использовать источник, обычно приводят тип
> ```java
> TextComponent src = (TextComponent) e.getSource();
> ```

### Отладочная информация
```java
// Возвращает строковое описание события
public String paramString()
```

## 💡 Особенности и подводные камни 💡

1️⃣ 💡 **`TextEvent` НЕ СОДЕРЖИТ сам ТЕКСТ.**  
Чтобы получить текущее значение — нужно вызвать `getText()` у источника:
```java
((TextComponent)e.getSource()).getText();
```

2️⃣ **Вызывается при любом изменении содержимого**,  
даже если оно произошло программно (через `setText()`).

3️⃣ **Не различает “набор” и “удаление”.**  
`TextEvent` просто сообщает: _“текст изменился”_,  
без указания характера изменения.

4️⃣ 💡**Для реакции на отдельные символы**  
(например, при вводе пароля или автоформатировании)  
следует использовать `KeyEvent`, а не `TextEvent`.

5️⃣ **Если внутри обработчика ты снова вызываешь `setText()`**,  
это может вызвать **рекурсивное событие**.  
Поэтому такие вызовы стоит блокировать флагом:
```java
if (!isUpdating) {
    isUpdating = true;
    textField.setText(...);
    isUpdating = false;
}
```

---
## 💎 Best Practices 💎

✅ 💡Используй `TextListener`, если нужно реагировать на **изменение содержимого**, а не на нажатие клавиш.

✅ Для работы с символами — используй `KeyListener`, а для готового текста — `TextListener`.

✅ Проверяй, не вызывает ли обработчик новых изменений текста (чтобы избежать зацикливания).

✅ 💡Если нужно проверять корректность введённых данных в реальном времени (например, email или цифры) — делай это в `textValueChanged()`.

✅ Для Swing-компонентов (`JTextField`, `JTextArea`) вместо `TextEvent` используется **`DocumentEvent`** (в пакете `javax.swing.event`).

## Пример
```java
import java.awt.*;
import java.awt.event.*;

public class TextEventDemo extends Frame implements TextListener {
    private TextField input;
    private Label info;

    public TextEventDemo() {
        setLayout(new FlowLayout());

        input = new TextField(20);
        input.addTextListener(this);

        info = new Label("Введите что-нибудь...");

        add(input);
        add(info);

        setSize(300, 120);
        setVisible(true);

        addWindowListener(new WindowAdapter() {
            public void windowClosing(WindowEvent e) {
                System.exit(0);
            }
        });
    }

    @Override
    public void textValueChanged(TextEvent e) {
        String text = ((TextField)e.getSource()).getText();
        info.setText("Текущий ввод: " + text);
    }

    public static void main(String[] args) {
        new TextEventDemo();
    }
}

```
🧠 **Как работает:**
- Каждый раз, когда пользователь изменяет содержимое `TextField`, вызывается `textValueChanged()`.
    
- Обработчик получает новое значение текста и отображает его в `Label`.
