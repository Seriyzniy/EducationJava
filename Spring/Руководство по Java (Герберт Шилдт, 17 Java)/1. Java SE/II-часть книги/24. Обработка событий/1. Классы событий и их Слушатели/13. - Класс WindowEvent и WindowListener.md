
> [!NOTE] `WindowEvent` — это событие, которое генерируется `при изменении состояния окна` (`Frame`, `Dialog`, `Window` и их потомков).
> С его помощью можно реагировать на `открытие`, `закрытие`, `активацию`, `деактивацию`, `сворачивание`, восстановление и `фокус окна`.

## Иерархия
```java
java.lang.Object
   └── java.util.EventObject
        └── java.awt.AWTEvent
             └── java.awt.event.ComponentEvent
                  └── java.awt.event.WindowEvent
```

🧠 `WindowEvent` наследует `ComponentEvent`,  
==поэтому содержит координаты==, компонент-источник и базовую информацию о GUI-элементе.

### 🧭 Назначение
`WindowEvent` сообщает о **любом изменении состояния окна**.  

Для обработки таких событий используются два интерфейса:
```java
public interface WindowListener {
    void windowOpened(WindowEvent e);
    void windowClosing(WindowEvent e);
    void windowClosed(WindowEvent e);
    void windowIconified(WindowEvent e);
    void windowDeiconified(WindowEvent e);
    void windowActivated(WindowEvent e);
    void windowDeactivated(WindowEvent e);
}

public interface WindowFocusListener {
    void windowGainedFocus(WindowEvent e);
    void windowLostFocus(WindowEvent e);
}

public interface WindowStateListener {
    void windowStateChanged(WindowEvent e);
}
```
> То есть `WindowEvent` — универсальный контейнер,  
> используемый разными типами слушателей для разных аспектов жизненного цикла окна.

## Конструкторы
```java
// Создаёт событие окна
// src — окно, вызвавшее событие (Frame, Dialog, Window)
// id — тип события (например, WINDOW_OPENED, WINDOW_CLOSING)
// opposite — противоположное окно, если применимо (например, при переключении фокуса)
// oldState, newState — старое и новое состояние (используются при изменении состояния окна)
public WindowEvent(
	Window src, 
	int id, 
	Window opposite, 
	int oldState, 
	int newState)

// Упрощённый конструктор (для старых событий, без состояний)
public WindowEvent(Window src, int id)
```

## Методы

Класс `WindowEvent` наследует методы от `ComponentEvent` (`getComponent()`, `paramString()`),  
и добавляет свои специфичные методы для окон.

### Информация о связях окон
```java
// Возвращает другое окно, связанное с этим событием
// Например, при переключении фокуса между окнами
public Window getOppositeWindow()
```

### Изменение состояния
```java
// Возвращает предыдущее состояние окна (например, ICONIFIED, NORMAL, MAXIMIZED)
public int getOldState()

// Возвращает новое состояние окна
public int getNewState()
```

> Используется только для `WINDOW_STATE_CHANGED`,  
> чтобы определить, что произошло (например, окно было развернуто или свернуто).


### Отладочная информация
```java
// Возвращает текстовое описание события
public String paramString()
```

## 💡 Особенности и подводные камни 💡

1️⃣ 💡**`windowClosing` не закрывает окно автоматически.**  
Это ==просто уведомление==. Чтобы реально закрыть окно, нужно вызвать:
```java
dispose();
```

2️⃣ 💡**`windowClosed` вызывается ==после закрытия==**,  
а `windowClosing` — _до него_ (момент, когда пользователь нажал [X]).

3️⃣ **`WINDOW_STATE_CHANGED`** используется в связке с `WindowStateListener`  
и позволяет узнавать, развернул ли пользователь окно, свернул и т.п.

4️⃣ **События фокуса (`WINDOW_GAINED_FOCUS` / `WINDOW_LOST_FOCUS`)**  
приходят только при смене активного окна.

5️⃣ 💡**Некоторые события не приходят в модальных диалогах**,  
так как фокус передаётся не напрямую, а через систему блокировки окон.

6️⃣ 💡**Проверяй `getOppositeWindow()`**, если нужно узнать, между какими окнами происходила смена фокуса.

## 💎 Best Practices 💎

✅ Используй `windowClosing()` для сохранения данных или вывода диалога подтверждения перед выходом.

✅ Если приложение многооконное, регистрируй отдельный `WindowListener` для каждого окна.

✅ ⭐⭐Используй `WindowAdapter`, если нужно обрабатывать только часть событий:
```java
frame.addWindowListener(new WindowAdapter() {
    public void windowClosing(WindowEvent e) {
        System.exit(0);
    }
});
```

✅ При работе с `WINDOW_STATE_CHANGED` проверяй `getNewState()`:
```java
if ((e.getNewState() & Frame.ICONIFIED) != 0) ...
```

✅ ⭐Не выполняй тяжёлые операции (запись в БД, сетевые вызовы) прямо в `windowClosing()` —  
лучше вызывать асинхронно.

## Пример
```java
import java.awt.*;
import java.awt.event.*;

public class WindowEventDemo extends Frame implements WindowListener {
    private TextArea info;

    public WindowEventDemo() {
        setLayout(new BorderLayout());
        info = new TextArea();
        add(info, BorderLayout.CENTER);

        addWindowListener(this);
        setSize(400, 300);
        setVisible(true);
    }

    @Override
    public void windowOpened(WindowEvent e) {
        info.append("Окно открыто\n");
    }

    @Override
    public void windowClosing(WindowEvent e) {
        info.append("Окно закрывается...\n");
        System.exit(0);
    }

    @Override
    public void windowClosed(WindowEvent e) {
        info.append("Окно закрыто\n");
    }

    @Override
    public void windowIconified(WindowEvent e) {
        info.append("Окно свернуто\n");
    }

    @Override
    public void windowDeiconified(WindowEvent e) {
        info.append("Окно восстановлено\n");
    }

    @Override
    public void windowActivated(WindowEvent e) {
        info.append("Окно активно\n");
    }

    @Override
    public void windowDeactivated(WindowEvent e) {
        info.append("Окно деактивировано\n");
    }

    public static void main(String[] args) {
        new WindowEventDemo();
    }
}
```

🧠 **Как работает:**
- При открытии окна → `windowOpened()`.
    
- При нажатии [X] → `windowClosing()` и затем `windowClosed()`.
    
- При сворачивании → `windowIconified()`,  
    при восстановлении — `windowDeiconified()`.
    
- При активации/деактивации окна — соответствующие методы.
