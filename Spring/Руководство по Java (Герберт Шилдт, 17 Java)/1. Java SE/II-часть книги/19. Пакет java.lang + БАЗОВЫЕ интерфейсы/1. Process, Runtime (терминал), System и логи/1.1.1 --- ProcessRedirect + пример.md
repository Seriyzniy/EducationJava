`ProcessBuilder.Redirect` — это **вложенный статический класс** внутри `ProcessBuilder`, появившийся в JDK 7.

Его задача — ==**управлять перенаправлением стандартных потоков**== (`stdin`, `stdout`, `stderr`) внешнего процесса.

## Зачем нужен

==Когда вы запускаете внешний процесс== через `ProcessBuilder`, у него есть:
- **стандартный ввод** (куда можно писать);
- **стандартный вывод** (откуда можно читать);
- **поток ошибок**.
    
Без `Redirect` вам пришлось бы вручную работать с `InputStream` и `OutputStream` (`process.getInputStream()`, и т.п.).  

С `Redirect` можно **напрямую привязать их (потоки) к файлам или каналам**.

### Методы Redirect
```java
// 🟢--- Методы создания Redirect ---🟢
public static ProcessBuilder.Redirect to(File file);      
// Перенаправление stdout/stderr в файл (перезаписывает файл)

public static ProcessBuilder.Redirect appendTo(File file); 
// Перенаправление stdout/stderr в файл (добавляет в конец файла)

public static ProcessBuilder.Redirect from(File file);     
// Перенаправление stdin из файла (чтение данных для процесса)


// 🟢--- Методы экземпляра ---🟢
public File file();                                        
// Возвращает File, связанный с данным перенаправлением

public ProcessBuilder.Redirect.Type type();                
// Возвращает тип перенаправления (APPEND, READ, WRITE, PIPE, INHERIT)


// 🟢--- Константы Redirect ---🟢
public static final ProcessBuilder.Redirect INHERIT;       
// Использовать те же потоки ввода/вывода, что и у текущего процесса

public static final ProcessBuilder.Redirect PIPE;          
// Использовать канал (по умолчанию, доступ через getInputStream(), getOutputStream())

public static final ProcessBuilder.Redirect DISCARD;       
// Отбросить вывод (stdout/stderr уходит "в никуда")


// 🟢--- Вложенное перечисление Redirect.Type ---🟢
public static enum ProcessBuilder.Redirect.Type {
    APPEND,  // дозапись в файл
    INHERIT, // наследование потоков родителя
    PIPE,    // использование канала
    READ,    // ввод из файла
    WRITE    // вывод в файл (перезапись)
}

```

### Перечисление ТИПОВ перенапрявления `Redirect.Type`

Описывает, какой именно вид перенаправления применяется. 

Варианты:
- `APPEND` → дозапись в файл;
    
- `INHERIT` → использовать потоки текущего процесса (например, выводить прямо в консоль);
    
- `PIPE` → создать канал (по умолчанию, когда вы используете `getInputStream()` и т.п.);
    
- `READ` → ввод из файла;
    
- `WRITE` → вывод в файл (перезапись).

### Предопределённые константы

`Redirect.INHERIT` → использовать консоль текущего процесса.
    
`Redirect.PIPE` → стандартное поведение с потоками.
    
`Redirect.DISCARD` → просто отбросить вывод (будет "в никуда").

## Пример

- Вызовется команда `ipconfig`.
    
- Её **stdout** попадёт в файл `process_output.txt` (перезапишет содержимое).
    
- Её **stderr** (ошибки) будут **добавляться** в тот же файл.
    
- После завершения программа выведет код выхода процесса

```java
import java.io.File;
import java.io.IOException;

public class RedirectDemo {
    public static void main(String[] args) {
        try {
            // Файл, куда будем писать вывод процесса
            File log = new File("process_output.txt");

            // Создаём ProcessBuilder
            ProcessBuilder pb = new ProcessBuilder("ipconfig");

            // Перенаправляем стандартный вывод в файл
            pb.redirectOutput(ProcessBuilder.Redirect.to(log));

            // Перенаправляем ошибки тоже в файл (можно в другой)
            pb.redirectError(ProcessBuilder.Redirect.appendTo(log));

            // Запускаем процесс
            Process process = pb.start();

            // Ждём завершения
            int exitCode = process.waitFor();
            System.out.println("Процесс завершился с кодом: " + exitCode);

        } catch (IOException | InterruptedException e) {
            e.printStackTrace();
        }
    }
}
```
