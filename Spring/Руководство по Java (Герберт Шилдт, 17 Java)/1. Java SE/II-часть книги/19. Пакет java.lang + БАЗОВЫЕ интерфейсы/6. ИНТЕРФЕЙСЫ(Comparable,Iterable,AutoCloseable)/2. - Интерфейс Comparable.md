
> [!NOTE] Декларация
> «Объекты этого класса можно сравнивать друг с другом естественным образом»

Объекты классов, реализующих интерфейс `Comparable`, можно упорядочивать. 

Другими словами, реализующие `Comparable` классы содержат объекты, которые можно сравнивать каким-то осмысленным образом. 

Интерфейс `Comparable` является обобщенным и объявлен следующим образом:
```java
interface Comparable<T>
```

## Метод `compareTo()`

> [!NOTE] ⭐Главный метод для РЕАЛИЗАЦИИ в классе⭐
>

В интерфейсе `Comparable` объявлен один метод, который ==применяется для определения== того, что Java называет ==естественным порядком следования== экземпляров класса

Метод `compareTo()` сравнивает вызываемый объект с `obj`.
```java
int compareTo(T obj)
```
- `< 0` → меньше
- `0` → равно
- `> 0` → больше


> [!NOTE] Связь `equals` и `compareTo`
> Нарушение контракта `compareTo()` и `equals()` — классика. Если два объекта равны по `equals`, они должны возвращать `0` в `compareTo()`

# Пример
Пример класса «Сотрудник», который сравнивается по зарплате:
```java
class Employee implements Comparable<Employee> {
    private final String name;
    private final int salary;

    public Employee(String name, int salary) {
        this.name = name;
        this.salary = salary;
    }

    @Override
    // ✅Собственная реализация ТОГО, ЧТО СРАВНИВАЕМ И КАК✅
    public int compareTo(Employee other) {
        return Integer.compare(this.salary, other.salary);
    }

    @Override
    public String toString() {
        return name + " (" + salary + ")";
    }
}

public class TestComparable {
    public static void main(String[] args) {
        List<Employee> list = List.of(
                new Employee("Alice", 50000),
                new Employee("Bob", 40000),
                new Employee("Charlie", 70000)
        );
        list.stream().sorted().forEach(System.out::println);
    }
}
```
