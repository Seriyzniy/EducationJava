# 📕 1. `Date` — класс для хранения даты/времени (классический API)

Хранит точку времени в миллисекундах с эпохи (1 янв 1970). 

==Большая часть старых методов устарела== — более тонкие операции выполняются через 
- `Calendar`
- новый API `java.time` (в книге отмечено перенаправление функционала в `Calendar`/`DateFormat`)

> [!NOTE] ⭐⭐ Рекомендации ⭐⭐
> Для новой разработки предпочитайте `java.time` (JSR-310): 
> - `Instant`, 
> - `LocalDateTime`, 
> - `ZonedDateTime` и т. п. (об этом подробнее в главе, посвящённой `java.time`)

## Конструкторы
```java
Date()
// Создаёт объект с текущим временем.

Date(long millis)
// Создаёт дату, соответствующую времени в миллисекундах с 1 янв. 1970 UTC.
```

## Методы
```java
long getTime()
// Возвращает время в миллисекундах с 1 янв. 1970 UTC.

void setTime(long time)
// Устанавливает новое время.



boolean before(Date when)
// true, если текущая дата раньше заданной.

boolean after(Date when)
// true, если текущая дата позже заданной.



int compareTo(Date anotherDate)
// Сравнивает две даты.

boolean equals(Object obj)
// Проверяет равенство по времени.



int hashCode()
// Хэш-код (по времени).

String toString()
// Строковое представление (в локали).

```

## Пример
```java
import java.util.Date;

public class DateExample {
    public static void main(String[] args) {
        Date now = new Date();
        System.out.println("Текущее время: " + now);

        Date past = new Date(0);
        System.out.println("Эпоха: " + past);

        System.out.println("Сейчас после эпохи? " + now.after(past));
    }
}

```

# 📕 2. `Calendar` (абстрактный) - разложение времени на компоненты

Cлужит ==для получения/установки частей даты/времени== (год, месяц, день, час и т.д.) и интерпретации в локали/часовом поясе. 

- `Calendar` — абстрактный; 
- конкретная реализация — `GregorianCalendar`.

> [!NOTE] ⭐Примечание ⭐
> - API гибкое, но довольно громоздкое; 
> - ⭐Для современной разработки есть `java.time` (замена `Calendar`/`Date`).

## Конструкторы
```java
// ⭐⭐Calendar — АБСТРАКТНЫЙ класс, напрямую не создаётся.⭐⭐

```

## Пример
```java
import java.util.Calendar;

public class CalendarExample {
    public static void main(String[] args) {
        Calendar cal = Calendar.getInstance();
        System.out.println("Сейчас: " + cal.getTime());

        cal.set(Calendar.YEAR, 2000);
        cal.set(Calendar.MONTH, Calendar.JANUARY);
        cal.set(Calendar.DAY_OF_MONTH, 1);

        System.out.println("Установленная дата: " + cal.getTime());

        cal.add(Calendar.DAY_OF_MONTH, 30);
        System.out.println("Плюс 30 дней: " + cal.getTime());
    }
}

```

# 📕 3. `GregorianCalendar` - реализация календаря

Метод `getinstance()` класса Calendar обычно возвращает GregorianCalendar

Реализация григорианского календаря, предоставляющая те же возможности, что и `Calendar`, а также дополнительные (например, `isLeapYear(int year)`). 

Конструкторы позволяют инициализировать год/месяц/день и время, либо задавать `TimeZone`/`Locale`.
