
> [!NOTE] `GenericServlet` — это `абстрактный базовый класс`, реализующий интерфейсы `Servlet`, `ServletConfig` и `Serializable`.
> Он предназначен` для упрощения написания сервлетов`, `НЕ ЗАВИСЯЩИХ `от конкретного `протокола` (HTTP, FTP, SMTP и т.д.).  
> 
> Обычно его расширяет `HttpServlet`, добавляющий HTTP-специфику (`doGet()`, `doPost()` и т.д.).

`GenericServlet` берет на себя рутину
- хранит `ServletConfig`, 
- реализует базовые методы жизненного цикла (`init()`, `destroy()`, `getServletConfig()`, `getServletContext()` и т.п.), 

💡а разработчику остается лишь реализовать метод `service()`.

## Конструктор
```java
package jakarta.servlet;

import java.io.Serializable;

public abstract class GenericServlet implements Servlet, ServletConfig, Serializable {

    /**
     * Создает экземпляр GenericServlet.
     * Конструктор по умолчанию не делает ничего — инициализация происходит в методе init().
     */
    public GenericServlet() {
        super();
    }
}
```

## Методы
Класс **наследует** методы из интерфейсов `Servlet` и `ServletConfig`,  
а также **реализует** часть из них сам, чтобы упростить жизнь потомкам.  

Ниже перечислены **методы, реализованные или определённые в GenericServlet**.

### Методы жизненного цикла
```java
// Инициализация сервлета с конфигурацией.
// Контейнер вызывает этот метод один раз при создании сервлета.
public void init(ServletConfig config) throws ServletException

// Упрощённый метод инициализации (без параметров).
// Можно переопределить его вместо полного init(config).
public void init() throws ServletException

// Метод уничтожения сервлета. Вызывается контейнером при выгрузке.
public void destroy()
```

### Основная логика обработки
```java
// Абстрактный метод, который должен реализовать каждый потомок.
// Именно здесь происходит обработка запроса.
public abstract void service(
	ServletRequest req, 
	ServletResponse res) throws ServletException, IOException;
```

### Работа с конфигурацией и контекстом
```java
// Возвращает объект конфигурации сервлета (ServletConfig),
// переданный контейнером во время инициализации.
public ServletConfig getServletConfig()

// Возвращает объект ServletContext, связанный с данным сервлетом.
public ServletContext getServletContext()
```

### Работа с параметрами и метаданными
```java
// Возвращает параметр инициализации по имени (из web.xml или аннотации @WebInitParam).
public String getInitParameter(String name)

// Возвращает перечисление всех параметров инициализации.
public Enumeration<String> getInitParameterNames()

// Возвращает имя сервлета (обычно совпадает с именем в конфигурации).
public String getServletName()
```

### Утилитарные методы
```java
// Записывает объект инициализации конфигурации в приватное поле.
// Обычно вызывается контейнером, не используется вручную.
public void log(String msg)

// Логирует сообщение об ошибке с объектом Throwable.
public void log(String message, Throwable t)
```

## 💡 Особенности и подводные камни 💡

1️⃣ `GenericServlet` **==не знает о протоколе==** — это чисто абстрактная заготовка для любых типов запросов. Для HTTP лучше использовать `HttpServlet`.  

2️⃣ ⭐⭐Метод `service()` **обязателен к реализации** — без него класс останется абстрактным.  

3️⃣ Метод `init()` в двух вариантах: контейнер вызывает `init(ServletConfig)`, а ты можешь переопределить `init()` без параметров, чтобы не париться с конфигурацией вручную.  

4️⃣ 💡В методе `init()` можно получить `ServletContext` через `getServletContext()` и сохранить необходимые ресурсы.  

5️⃣ 💡`log()` использует логгер контейнера, А НЕ `System.out`, что делает его безопасным для production.  

6️⃣ `GenericServlet` сериализуем (`implements Serializable`), что важно для некоторых контейнеров, поддерживающих персистентность сессий.

---
## 💎 Best Practices 💎

✅ Используй `init()` без параметров для своей инициализации — это чище и безопаснее.  

✅ ❗❗❗Никогда не создавай экземпляры сервлета вручную — это делает контейнер.  

✅ Если тебе нужен доступ к ресурсам приложения — всегда используй `getServletContext()`.  

✅ Используй `log()` вместо `System.out.println()` для сообщений и ошибок.  

✅ ❗❗❗Освобождай ресурсы (например, соединения с БД) в `destroy()`.

## Пример
```java
package servlets;

import java.io.IOException;
import java.io.PrintWriter;
import java.util.Date;

import jakarta.servlet.GenericServlet;
import jakarta.servlet.ServletException;
import jakarta.servlet.ServletRequest;
import jakarta.servlet.ServletResponse;
import jakarta.servlet.http.HttpServlet;

public class HelloServlet extends GenericServlet {
	
	@Override
    public void init() throws ServletException {
        log("MyGenericServlet initialization");
    }
	
	public void service(
	    ServletRequest req, 
	    ServletResponse res) throws ServletException, IOException
	{
		res.setContentType("text/plain;charset=UTF-8");
        PrintWriter out = res.getWriter();

        out.println("Hello from GenericServlet!");
        out.println("Servlet name: " + getServletName());
        out.println("Parametr example: " + getInitParameter("example"));
    }
	
	@Override
    public void destroy() {
        log("MyGenericServlet destroyed!!");
    }
}
```
