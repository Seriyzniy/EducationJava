
> [!NOTE] `ServletContext` — это **`глобальный объект контекста` веб-приложения**
> предоставляемый контейнером (например, Tomcat, Jetty).
> 
> ❗❗❗Каждое `веб-приложение` (WAR-файл) имеет ровно `один экземпляр` `ServletContext`.  
> 
> ⭐`Все сервлеты` внутри ОДНОГО ПРИЛОЖЕНИЯ` делят один и тот же контекст`, что делает его удобным для обмена данными между ними.

Он используется для:
- 💡доступа ==к общим ресурсам приложения== (файлы, атрибуты, параметры);
- 🧠взаимодействия между сервлетами;
- ❗❗❗логирования и получения информации о сервере.

---
## Интерфейс
```java
package jakarta.servlet;

import java.io.InputStream;
import java.net.URL;
import java.util.Enumeration;
import java.util.Set;

/**
 * Defines a set of methods that a servlet uses to communicate with
 * its servlet container and to share information with other servlets.
 */
public interface ServletContext {

    // ==============================
    // 🟢 Основная информация о приложении 🟢
    // ==============================
    /**
     * ⭐⭐Возвращает имя контекста (обычно — ❗путь приложения❗).
     * ❗Например, "/myapp" для приложения, ❗развернутого как myapp.war❗.
     *
     * @return контекстный путь приложения
     */
    String getContextPath();

    /**
     * Возвращает версию спецификации Servlet API, поддерживаемую контейнером.
     *
     * @return major версия
     */
    int getMajorVersion();

    /**
     * Возвращает минорную версию спецификации Servlet API.
     *
     * @return minor версия
     */
    int getMinorVersion();

    /**
     * Возвращает название и версию контейнера сервлетов (например, "Apache Tomcat/10.1").
     *
     * @return строка с информацией о сервере
     */
    String getServerInfo();



    // ==============================
    // 🟢 Доступ к параметрам инициализации 🟢
    // ==============================
    /**
     * 💡Возвращает значение глобального параметра инициализации (<context-param> в web.xml).
     *
     * @param name имя параметра
     * @return значение параметра или null, если параметр не найден
     */
    String getInitParameter(String name);

    /**
     * Возвращает имена всех глобальных параметров инициализации.
     *
     * @return Enumeration имён параметров
     */
    Enumeration<String> getInitParameterNames();



    // ==============================
    // 🟢 Работа с атрибутами контекста 🟢
    // ==============================
    /**
     * ⭐⭐⭐СОХРАНЯЕТ ОБЪЕКТ как АТРИБУТ контекста⭐⭐⭐ (глобальная область видимости).
     * ❗Все сервлеты и фильтры могут получить доступ к этому объекту❗
     *
     * ✅ МОЖНО ЗАПИХНУТЬ В КОНТЕКСТ ПУЛ СОЕДИНЕНИЙ, ❗предварительно создав объект пула❗
     * @param name имя атрибута
     * @param object объект для хранения
     */
    void setAttribute(String name, Object object);

    /**
     * Возвращает объект по имени атрибута.
     *
     * @param name имя атрибута
     * @return объект или null, если атрибут не найден
     */
    Object getAttribute(String name);

    /**
     * Удаляет атрибут контекста по имени.
     *
     * @param name имя атрибута
     */
    void removeAttribute(String name);

    /**
     * Возвращает имена всех установленных атрибутов.
     *
     * @return Enumeration имён атрибутов
     */
    Enumeration<String> getAttributeNames();



    // ==============================
    // 🟢 Работа с ресурсами приложения 🟢
    // ==============================
    /**
     * 💡Возвращает абсолютный путь в файловой системе для ресурса внутри веб-приложения.
     * Например, "/WEB-INF/config.xml" → "C:/tomcat/webapps/myapp/WEB-INF/config.xml"
     *
     * @param path виртуальный путь
     * @return физический путь или null, если ресурс недоступен
     */
    String getRealPath(String path);

    /**
     * ⭐⭐Возвращает URL для указанного ресурса внутри приложения.
     *
     * @param path виртуальный путь (например, "/images/logo.png")
     * @return URL ресурса или null
     */
    URL getResource(String path);

    /**
     * 💡Возвращает InputStream для чтения ресурса.
     *
     * @param path путь к ресурсу
     * @return поток для чтения или null
     */
    InputStream getResourceAsStream(String path);

    /**
     * Возвращает набор путей ко всем ресурсам внутри приложения,
     * начиная с указанного пути.
     *
     * @param path путь, например "/WEB-INF/"
     * @return множество путей или null
     */
    Set<String> getResourcePaths(String path);



    // ==============================
    // 🟢 Работа с другими контекстами 🟢
    // ==============================
    /**
     * Возвращает ServletContext другого приложения (если разрешено контейнером).
     *
     * @param uripath контекстный путь другого приложения
     * @return ServletContext другого приложения или null
     */
    ServletContext getContext(String uripath);



    // ==============================
    // 🟢 Логирование 🟢
    // ==============================
    /**
     * ⭐⭐⭐Логирует сообщение в системный лог контейнера (например, Tomcat logs).
     *
     * @param msg сообщение
     */
    void log(String msg);

    /**
     * Логирует сообщение с исключением (stack trace).
     *
     * @param message сообщение
     * @param throwable исключение
     */
    void log(String message, Throwable throwable);
}
```

## 💡 Особенности и подводные камни 💡

1️⃣ `ServletContext` — **==общий для всего приложения==**, в отличие от `ServletConfig`, который уникален для каждого сервлета.

2️⃣ ⭐⭐Хранение mutable-объектов в `ServletContext` (через `setAttribute`) требует **потокобезопасности** — к контексту обращаются все сервлеты одновременно.

3️⃣ 🧠❗**Метод `getRealPath()`** может вернуть `null`, если приложение развёрнуто не в виде файлов, а, например, как архив `.war`.

4️⃣ 💡Параметры, заданные через `<context-param>`, доступны **только для чтения** — их нельзя изменить через `ServletContext`.

5️⃣ **в `ServletContext` ==можно помещать НЕ ТОЛЬКО строки==, но и любые Java-объекты**, в том числе, например, **пул соединений с базой данных**, сервисы, DAO, кэш и т.д.
```java
@WebListener
public class AppContextListener implements ServletContextListener {

    @Override
    public void contextInitialized(ServletContextEvent sce) {
        ServletContext context = sce.getServletContext();

        // Создаём и настраиваем пул (например, HikariCP)
        HikariDataSource ds = new HikariDataSource();
        ds.setJdbcUrl("jdbc:postgresql://localhost:5432/mydb");
        ds.setUsername("user");
        ds.setPassword("password");
        ds.setMaximumPoolSize(10);

        // Сохраняем в контекст под ключом
        context.setAttribute("dataSource", ds);
        context.log("HikariCP DataSource создан и помещён в ServletContext");
    }

    @Override
    public void contextDestroyed(ServletContextEvent sce) {
        // Корректно закрываем пул при завершении приложения
        HikariDataSource ds = (HikariDataSource) sce.getServletContext().getAttribute("dataSource");
        if (ds != null) {
            ds.close();
            sce.getServletContext().log("HikariCP DataSource закрыт");
        }
    }
}
```


6️⃣ 💡❗Доступ к другому контексту (`getContext("/otherApp")`) возможен не всегда — контейнер может запретить это из соображений безопасности.

## 💎 Best Practices 💎

✅ Используй `ServletContext` для:
- хранения общих ресурсов (пулы соединений, кэш, конфигурация);
- обмена объектами между сервлетами.
    

✅ 💡Для логирования ошибок или информации пользуйся `context.log(...)`, а не `System.out.println()`.

✅ Извлекай параметры конфигурации через `getInitParameter()` вместо хардкода.

✅ ❗❗❗При работе с файлами внутри WAR используй `getResourceAsStream()` — он работает даже при развёртывании из архива.

✅ Если нужно хранить глобальные данные, синхронизируй доступ к ним (например, через `ConcurrentHashMap` или `synchronized`).

### В коде сервлета
```java
package servlets;

import java.io.IOException;
import java.io.PrintWriter;
import java.util.Optional;

import jakarta.servlet.GenericServlet;
import jakarta.servlet.ServletContext;
import jakarta.servlet.ServletException;
import jakarta.servlet.ServletRequest;
import jakarta.servlet.ServletResponse;

public class HelloServlet extends GenericServlet   {
	
	private ServletContext context;
	
	@Override
	public void service(ServletRequest arg0, ServletResponse arg1) throws ServletException, IOException {
		context = getServletContext();
		arg1.setContentType("text/html; charset=UTF-8");
        PrintWriter out = arg1.getWriter();

        // Получаем глобальный параметр
        String message = context.getInitParameter("globalMessage");

        // Сохраняем общий атрибут
        context
	        .setAttribute(
		        "visits", 
		        ((Integer)getAttributeOrDefault("visits", 0)) + 1
		    );

        out.println("<h2>" + message + "</h2>");
        out.println("<p>Count visit: " + context.getAttribute("visits") + "</p>");

        // Логирование в системный лог
        context.log("Servlet ContextExampleServlet called!!!");
	}
	
	private Object getAttributeOrDefault(
		String attribute, 
		int defValue) 
	{
		Optional<Object> valueAttribute = Optional
			.ofNullable(context.getAttribute(attribute));
		
		return valueAttribute.isPresent() ? 
			valueAttribute.get() : defValue;
	}
}
```
