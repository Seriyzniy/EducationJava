
> [!NOTE] `ServletInputStream` — это абстрактный класс, `расширяющий` `java.io.InputStream`, предназначенный `для чтения тела запроса` (body) в сервлетах.
> Он используется контейнером сервлетов `для передачи двоичных данных` запроса в метод `service()`.  
> 
> 💡Например, когда клиент отправляет `JSON, XML или файл` (multipart/form-data), эти` данные можно прочитать` именно `через` `ServletInputStream`
> 
> ⭐Реализуется контейнерами (например, Tomcat, Jetty) и возвращается методом `ServletRequest.getInputStream()`.

## Конструктор
```java
package jakarta.servlet;

import java.io.IOException;
import java.io.InputStream;

public abstract class ServletInputStream extends InputStream {

    // 🔹 Конструктор по умолчанию
    /**
     * Конструктор без параметров.
     * Обычно вызывается контейнером сервлетов, а не пользователем напрямую.
     */
    protected ServletInputStream() {
        super();
    }
}
```

## Методы

> Класс **наследует все методы** из `java.io.InputStream`  
> (`read()`, `available()`, `close()`, `mark()`, `reset()` и т.д.),  
> а также **добавляет** несколько дополнительных методов для интеграции с==сервлетами и асинхронным вводом.==

###  **Состояние потока (неблокирующее чтение)**
```java
// Проверяет, можно ли прочитать данные без блокировки.
// Возвращает true, если чтение возможно сразу.
public abstract boolean isFinished();

// Проверяет, доступен ли поток для чтения (например, не закрыт).
// Возвращает true, если данные готовы к чтению.
public abstract boolean isReady();

// Устанавливает слушателя для неблокирующего ввода.
// Используется при асинхронной обработке запросов.
public abstract void setReadListener(ReadListener readListener);
```

### ⭐⭐Наследуемые методы из InputStream (Базовое чтение данных)
```java
// Читает следующий байт из входного потока.
public abstract int read() throws IOException;

// Читает несколько байт данных в массив.
public int read(byte[] b, int off, int len) throws IOException

// Возвращает количество доступных для чтения байт без блокировки.
public int available() throws IOException;

// Закрывает поток.
public void close() throws IOException;
```

### Управление позицией чтения
```java
// Навигация по потоку (унаследовано из InputStream)
long skip(long n)           // пропуск байтов
void mark(int readlimit)    // отметка позиции
void reset()                // возврат к отметке
boolean markSupported()     // поддерживает ли отметки?
```

## 💡 Особенности и подводные камни 💡

1️⃣ ❗❗❗**Нельзя одновременно использовать `getReader()` и `getInputStream()`** — они оба читают тело запроса. Контейнер выбросит `IllegalStateException`, если вызвать оба.  

2️⃣ `ServletInputStream` — это **абстрактный класс**, экземпляр создаёт контейнер. Разработчик ==НЕ ДОЛЖЕН СОЗДАВАТЬ его вручную==.  

3️⃣ ⭐Методы `isFinished()`, `isReady()` и `setReadListener()` — часть **асинхронного API Servlet 3.1+**. Они позволяют ==читать== тело запроса ==неблокирующим образом==.  

4️⃣ ⭐Поток можно прочитать **только один раз** — после этого тело запроса “исчерпывается”.  

5️⃣ 🧠Если тело запроса большое, то при чтении напрямую может быть задействовано буферизованное чтение — не стоит вызывать `read()` в цикле без буфера.  

6️⃣ `setReadListener()` требует, чтобы запрос обрабатывался асинхронно (через `AsyncContext`), иначе возникнет исключение.

---
`ServletInputStream` — ==**не просто копия**== `InputStream`, а его **специализированная версия**, которую возвращает контейнер сервлетов (Tomcat, Jetty, Undertow и др.) при вызове:
```java
ServletInputStream in = request.getInputStream();
```

Ты **не создаёшь** этот объект сам — его **создаёт контейнер**, который:
- управляет сетевым сокетом (HTTP-соединением);
- буферизует входящие данные;
- следит за состоянием запроса (например, асинхронные операции);
- реализует специфику протокола HTTP (chunked, multipart и т. д.);
- синхронизирует чтение тела запроса с другими компонентами (фильтрами, слушателями).

если бы ты создал свой 
- `FileInputStream` или 
- `BufferedInputStream` 
и “отдал” его контейнеру — он бы не понял, как из него читать данные HTTP-запроса (потому что ==это уже не сетевой поток==, а просто ==байты из файла==).

---
## 💎 Best Practices 💎

✅ 💡Для небольших JSON/XML — лучше использовать `getReader()` (удобнее работать с текстом).  

✅ ❗❗❗==Для бинарных данных== (файлов, потоков) — всегда используй `getInputStream()`.  

✅ Если читаешь тело запроса вручную — не используй фильтры, которые тоже читают тело (иначе оно “пропадет”). 

✅ При асинхронной обработке всегда проверяй `isReady()` перед чтением.  

✅ Не забудь закрыть поток (`close()`), чтобы освободить ресурсы.  

✅ ⭐⭐⭐Если тело запроса большое — читай его частями в буфер, не всё сразу в память.

## Пример
```java
package servlets;

import java.io.IOException;
import java.io.PrintWriter;

import jakarta.servlet.ServletException;
import jakarta.servlet.ServletInputStream;
import jakarta.servlet.ServletRequest;
import jakarta.servlet.ServletResponse;
import jakarta.servlet.http.HttpServlet;

public class HelloServlet extends HttpServlet {
	
	public void service(
	    ServletRequest req, 
	    ServletResponse res) throws ServletException, IOException
	{
		res.setContentType("text/plain;charset=UTF-8");
        PrintWriter out = res.getWriter();

        ServletInputStream inputStream = req.getInputStream();
        StringBuilder body = new StringBuilder();

        byte[] buffer = new byte[1024];
        int bytesRead;
        while ((bytesRead = inputStream.read(buffer)) != -1) {
            body.append(new String(buffer, 0, bytesRead, req.getCharacterEncoding()));
        }

        out.println("Тело запроса:");
        out.println(body.toString());
    }
}
```

## Пример асинхронного чтения (Servlet 3.1+)
### В `web.xml`
```xml
<servlet>
    <servlet-name>HelloServlet</servlet-name>
    <servlet-class>servlets.HelloServlet</servlet-class>
    
    <!-- Включаем поддержку асинхронных операций -->
    <async-supported>true</async-supported>
</servlet>

<servlet-mapping>
    <servlet-name>HelloServlet</servlet-name>
    <url-pattern>/async-read</url-pattern>
</servlet-mapping>
```


```java
package servlets;

import java.io.IOException;

import jakarta.servlet.AsyncContext;
import jakarta.servlet.ReadListener;
import jakarta.servlet.ServletException;
import jakarta.servlet.ServletInputStream;

import jakarta.servlet.http.HttpServlet;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;

public class HelloServlet extends HttpServlet {
	
	@Override
    protected void doPost(HttpServletRequest req, HttpServletResponse resp)
            throws ServletException, IOException {

        AsyncContext asyncContext = req.startAsync();
        ServletInputStream input = req.getInputStream();

        input.setReadListener(new ReadListener() {
            private final StringBuilder sb = new StringBuilder();
            private final byte[] buffer = new byte[1024];

            @Override
            public void onDataAvailable() throws IOException {
                int len;
                while (input.isReady() && (len = input.read(buffer)) != -1) {
                    sb.append(new String(buffer, 0, len, req.getCharacterEncoding()));
                }
            }

            @Override
            public void onAllDataRead() throws IOException {
                resp.getWriter().write("Body of request readed:\n" + sb);
                asyncContext.complete();
            }

            @Override
            public void onError(Throwable t) {
                t.printStackTrace();
                asyncContext.complete();
            }
        });
    }
}
```
