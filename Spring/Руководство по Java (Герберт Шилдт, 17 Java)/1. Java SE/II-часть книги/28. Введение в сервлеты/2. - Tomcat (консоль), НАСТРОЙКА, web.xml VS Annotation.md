## Батник для запуска
После установки Tomcat:
- Запускается с помощью файла `startup.bat` `в подкаталоге bin` внутри каталога apache-tomcat-10.1.28 . 
- Останавливается Tomcat посредством файла `shutdown.bаt` `в подкаталоге bin`. 

## API сервлетов в Tomcat
==Классы и интерфейсы==, необходимые ==для создания сервлетов==, содержатся в архиве `servlet-api.jar`, который находится в  каталоге `lib`
![[Pasted image 20251016091414.png]]


Чтобы сделать `servlet-api.jar` доступным, нужно обновить переменную среды CLASSPATH для включения в нее следующего пути к файлу

В качестве альтернативы данный файл можно указать при компиляции сервлетов.
```bash
javac HelloServlet.java -classpath "...\lib\servlet-api.jar"
```

## Передача сервлета Tomcat'у
После компиляции сервлета необходимо позволить Tomcat найти его. 

Для этого сервлет понадобится поместить в подкаталог внутри каталога `webapps` контейнера Tomcat и ввести его имя в файле `web.xml`. 

Ради простоты в примерах применяется каталог и файл `web.xml`, которые Tomcat предоставляет для собственных примеров сервлетов.

- 🔶Первым делом необходимо скопировать файл класса сервлета в каталог 
`apache-tomcat-10.1.28\webapps\examples\WEB-INF\classes`

- 🔶Затем понадобится ==добавить имя и сопоставление сервлета== в файл web. xml, находящийся в показанном ниже каталоге
`apache-tomcat-10.1.28\webapps\examples\WEB-INF`


> [!NOTE] ⭐В папке `classes` структура пакетов сервлета должна быть одинаковая 
> При этом, СТРУТУРА пакетов должна быть одинаковой.
> Если сервлет лежит в папке `myservlets` то и в `calsses` ДОЛЖНА быть такая же папка, в которой должен лежать скомпилированный файл

## Web.xml VS Annotation
> [!NOTE] ⭐⭐`web.xml` VS `Annotation` ⭐⭐
> **`web.xml` всегда ==имеет приоритет над аннотациями==.**
> 
> То есть:
> - Если **сервлет описан и в аннотации, и в `web.xml`**, контейнер **игнорирует аннотацию** для этого сервлета.
> - Если **в `web.xml` нет упоминания этого сервлета**, тогда контейнер использует данные из аннотации.

```xml
<web-app xmlns="https://jakarta.ee/xml/ns/jakartaee"
  xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
  xsi:schemaLocation="https://jakarta.ee/xml/ns/jakartaee
                      https://jakarta.ee/xml/ns/jakartaee/web-app_6_0.xsd"
  version="6.0"
  metadata-complete="true">
```

`metadata-complete="true"`
⚠️ Очень важный флаг! Указывает контейнеру ==не сканировать аннотации== (@WebServlet, @WebFilter, @WebListener) и использовать только web.xml для регистрации компонентов.

---

Скажем, для первого примера под названием `HelloServlet` в раздел определения сервлетов будут добавлены следующие строки:
```xml
 <servlet>
	<servlet-name>HelloServlet</servlet-name>
	<servlet-class>servlets.HelloServlet</servlet-class>
</servlet>
```

А в раздел сопоставления (МАППИНГА) сервлетов - такие строки:
```xml
<servlet-mapping>
	<servlet-name>HelloServlet</servlet-name>
	<url-pattern>/servlets/servlet/HelloServlet</url-pattern>
</servlet-mapping>
```
Та же самая общая процедура используется во всех примерах.
