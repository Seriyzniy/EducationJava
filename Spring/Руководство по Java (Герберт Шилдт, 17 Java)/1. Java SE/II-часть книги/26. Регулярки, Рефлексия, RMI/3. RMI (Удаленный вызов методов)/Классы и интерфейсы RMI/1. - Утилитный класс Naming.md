
> [!NOTE] `Naming` — это `вспомогательный класс` из пакета **`java.rmi`**
> предоставляющий высокоуровневые `статические методы` `для взаимодействия с RMI-реестром` (Remote Method Invocation Registry).  
> 
> Он позволяет 
> - **регистрировать, 
> - удалять, 
> - искать и получать список удалённых объектов**, доступных по RMI.
> 
> 🧠Можно сказать, что `Naming` — это аналог “каталога” или “службы имён” для RMI, связывающей строковые имена с объектами, реализующими интерфейс `Remote`.

## Методы
### Работа с удалёнными объектами (bind / rebind / unbind)
```java
// 💡Регистрирует (привязывает) удалённый объект к имени в RMI-реестре.
// Если имя уже занято — выбрасывается AlreadyBoundException.
public static void bind(
	String name, 
	Remote obj) throws RemoteException, AlreadyBoundException, MalformedURLException;

// Привязывает объект, 💡заменяя старую привязку💡, если она уже существует.
public static void rebind(
	String name, 
	Remote obj) throws RemoteException, MalformedURLException;


// Удаляет привязку удалённого объекта из реестра по имени.
public static void unbind(String name)
    throws RemoteException, NotBoundException, MalformedURLException;
```

⚙️ **`name`** — это URL-строка формата `rmi://<host>:<port>/<objectName>`  
⚙️ **`obj`** — объект, реализующий интерфейс `java.rmi.Remote`  
⚙️ **Порты:** если порт не указан, по умолчанию используется `1099`.

### Поиск и получение удалённых объектов
```java
// 💡Получает ссылку на удалённый объект💡 по имени из RMI-реестра.
public static Remote lookup(String name)
    throws RemoteException, NotBoundException, MalformedURLException;


// 🧠Возвращает список всех имён (привязок), зарегистрированных в реестре.
public static String[] list(String name)
    throws RemoteException, MalformedURLException;

```

⚙️ Метод `lookup()` ==возвращает объект типа== `Remote`, который затем обычно ==приводится к конкретному== удалённому интерфейсу.  

⚙️ Метод `list()` возвращает массив строк — имена всех зарегистрированных удалённых объектов на указанном хосте.

---
## 💡 Особенности и подводные камни 💡

1️⃣ Все методы `Naming` — статические, то есть нет необходимости создавать экземпляр класса.  

2️⃣ 💡Методы работают **через RMI-реестр** (обычно ==запущенный== на порту `1099`), поэтому если реестр не запущен — произойдёт `ConnectException`.  

3️⃣ 💡Метод `lookup()` возвращает **stub-объект**, а не сам удалённый объект. Stub — это прокси, перенаправляющий вызовы по сети.  

4️⃣ ⭐⭐В имени `"rmi://host:port/name"` часть `rmi://` **обязательна**, иначе будет `MalformedURLException`.  

5️⃣ `bind()` нельзя вызывать повторно для того же имени — для перезаписи используется `rebind()`.  

6️⃣ Объекты должны быть **экспортированы** (`UnicastRemoteObject.exportObject()`), прежде чем их можно зарегистрировать.

---
## 💎 Best Practices 💎

✅ Всегда используйте `rebind()` вместо `bind()`, если не уверены, что имя свободно.  

✅ 💡Проверяйте, что RMI-реестр запущен до регистрации объекта (`rmiregistry` в отдельном процессе).  

✅ 💡Оборачивайте вызовы `Naming.lookup()` в `try-catch` с обработкой `NotBoundException` и `MalformedURLException`.  

✅ Используйте логгирование (например, `java.util.logging`) для отслеживания регистрации/удаления.  

✅ Храните имена объектов в виде констант, чтобы избежать опечаток при lookup.

## Синтетический пример
```java
import java.rmi.*;
import java.rmi.registry.*;
import java.rmi.server.*;

// Удалённый интерфейс
interface HelloService extends Remote {
    String sayHello(String name) throws RemoteException;
}

// Реализация удалённого интерфейса
class HelloServiceImpl extends UnicastRemoteObject implements HelloService {
    protected HelloServiceImpl() throws RemoteException { super(); }
    public String sayHello(String name) { \
	    return "Привет, " + name + "!"; 
	}
}

// Серверная часть
public class HelloServer {
    public static void main(String[] args) {
        try {
            // Создаем и экспортируем объект
            HelloService service = new HelloServiceImpl();

            // Регистрируем объект в RMI-реестре под именем "HelloService"
            Naming.rebind("rmi://localhost:1099/HelloService", service);
            System.out.println("Сервис зарегистрирован в реестре!");
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}


// Клиентская часть
class HelloClient {
    public static void main(String[] args) {
        try {
            // Получаем удаленный объект по имени
            HelloService service = (HelloService)
		            Naming.lookup("rmi://localhost:1099/HelloService");
		            
            System.out.println(service.sayHello("Сергей"));
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```
