Пакет `java.util.regex` поддерживает обработку регулярных выражений. 
Начиная с версии JDK 9, пакет `java.util.regex` находится в модуле `java.base`.

## Введение в регулярки

> [!NOTE] **Регулярное выражение** — это `строка-шаблон`, `описывающая` п`оследовательность символов`. 
> Шаблон может содержать групповые символы, наборы символов и квантификаторы, что позволяет задать общую форму для поиска совпадений в других последовательностях символов.

==Обработка== регулярных выражений осуществляется ==двумя классами==:

- 🔸**Pattern** — ==определяет== регулярное выражение;
	
- 🔸**Matcher** — ==выполняет сопоставление== шаблона с последовательностью символов.


Построенный объект `Pattern` будет использоваться для создания объекта `Matcher`, для чего вызывается метод `matcher()`, определенный в `Pattern`

---
# 📕 1. Класс Pattern

> [!NOTE] Класс `Pattern` представляет `СКОМПИЛИРОВАННОЕ` `регулярное выражение.`
> Он используется для поиска и сопоставления шаблонов строк в тексте совместно с классом `Matcher`
> 

Класс **НЕ реализует интерфейсы** и **является неизменяемым** (==immutable==).  
Это означает, что один объект `Pattern` можно безопасно использовать из разных потоков.

## Фабричные методы
```java
// 💡Компилирует регулярное выражение в объект Pattern.
// regex — строка с регулярным выражением.
public static Pattern compile(String regex)


// Компилирует регулярное выражение с указанными флагами (например, CASE_INSENSITIVE, MULTILINE и т.д.)
// regex — строка с регулярным выражением.
// flags — битовая комбинация флагов поведения.
public static Pattern compile(String regex, int flags)

```

## Методы
### Работа с сопоставлением
Основные методы для создания Matcher и проверки строк на соответствие
```java
// 💡Создает Matcher для данного входного текста.
// input — строка, по которой выполняется сопоставление.
public Matcher matcher(CharSequence input)


// 💡Проверяет, полностью ли строка соответствует шаблону.
// regex — регулярное выражение.
// input — проверяемая строка.
public static boolean matches(String regex, CharSequence input)
```

### Разделение строк
Разделяет строку по совпадениям регулярного выражения
```java
// Разделяет строку input на подстроки, используя шаблон как разделитель.
public String[] split(CharSequence input)

// То же, но ограничивает количество подстрок.
// limit — максимальное количество элементов в массиве.
public String[] split(CharSequence input, int limit)
```

### Получение информации о шаблоне
Методы для доступа к исходным данным шаблона
```java
// Возвращает исходную строку регулярного выражения.
public String pattern()

// Возвращает флаги, с которыми был скомпилирован шаблон.
public int flags()
```

### Утилиты для работы с регулярками
Преобразование и обработка специальных символов
```java
// 💡Экранирует метасимволы в строке так, чтобы она воспринималась как литерал.
public static String quote(String s)
```


## Если нужно один раз выполнить сопоставление
Можно просто вызвать метод `matches()` класса `Pattern`

---
## 💡 Особенности и подводные камни 💡
1️⃣ ⭐`Pattern` **==компилирует выражение один раз==**, и повторное использование — значительно эффективнее, чем вызов `matches()` с новой строкой каждый раз.  

2️⃣💡`Pattern.compile()` может выбросить **PatternSyntaxException**, если регулярка некорректна.  

3️⃣ Некоторые флаги (`MULTILINE`, `DOTALL`, `UNICODE_CASE`, `COMMENTS` и т.д.) **существенно меняют поведение** шаблона — нужно использовать с пониманием.  

4️⃣ 💡Символы `\` в Java-строках нужно экранировать двойным слэшем (`"\\d+"` вместо `"\d+"`).  

5️⃣ Метод `split()` ведёт себя по-другому при `limit=0` — убирает пустые строки в конце результата.

---
## 💎 Best Practices 💎
✅ 💡Повторно используйте один и тот же `Pattern` для множества проверок — это **экономит CPU**.  

✅ ❗❗❗Оборачивайте вызовы `Pattern.compile()` в `try-catch` для отлова `PatternSyntaxException`.  

✅ Используйте `Pattern.quote()` для безопасного поиска литералов, а не шаблонов.  

✅ ⭐⭐⭐Храните часто используемые паттерны как **константы** 
(`static final Pattern`).  

✅ 💡Для простых проверок на совпадение одной строки с регуляркой можно использовать `String.matches()`, но для многократных вызовов предпочтителен `Pattern`.

---
## Пример
```java
import java.util.regex.*;

public class PatternExample {
    public static void main(String[] args) {
        // Компиляция шаблона для поиска email
        Pattern emailPattern = Pattern.compile("\\b[\\w.%+-]+@[\\w.-]+\\.[A-Za-z]{2,6}\\b");

        // Создание Matcher
        Matcher matcher = emailPattern.matcher("Свяжитесь с нами по адресу support@example.com");

        // Проверка наличия совпадений
        if (matcher.find()) {
            System.out.println("Найден email: " + matcher.group());
        }

        // Разделение строки по пробелам
        Pattern space = Pattern.compile("\\s+");
        String[] words = space.split("Java   Stream   API");
        System.out.println("Слова: " + java.util.Arrays.toString(words));

        // Экранирование строки
        String safe = Pattern.quote("file(name).txt");
        System.out.println("Экранированная строка: " + safe);
    }
}

//Вывод
Найден email: support@example.com
Слова: [Java, Stream, API]
Экранированная строка: \Qfile(name).txt\E
```

# 📕 2. Класс Mathcer

> [!NOTE] Класс `Matcher` — это `механизм сопоставления` `текста` с `шаблоном`, созданным классом `Pattern`.
> Он управляет процессом поиска, позволяет получать совпадения, группы, выполнять замены и навигацию по тексту.

Каждый объект `Matcher` **связан с одним объектом `Pattern`** и **одним входным текстом** (`CharSequence`).  

Класс ❗**не является потокобезопасным**❗, но ==может быть переиспользован== для разных входных строк с помощью `reset()`.

## Конструктор
Конструктор **пакетный (package-private)** — напрямую создать `Matcher` нельзя.  

⭐Объекты создаются ==через ФАБРИКУ в классе `Patterrn`== `Pattern.matcher()`:
```java
// Создает новый Matcher для указанной входной последовательности.
// input — текст, в котором будет выполняться поиск.
public Matcher matcher(CharSequence input)
```

## Методы

### Поиск совпадений
Управляют процессом поиска и навигацией по найденным подстрокам
```java
// 💡Проверяет, соответствует ли ❗ВСЯ входная последовательность❗ символов шаблону.
// Возвращает true, если совпадение охватывает всю строку.
public boolean matches()

// 💡Проверяет, есть ли ❗ПОДСТРОКА❗, соответствующая шаблону.
// Можно вызывать многократно для итерации по всем совпадениям.
public boolean find()

// Начинает поиск совпадений с указанной позиции.
// start — индекс, с которого начать поиск.
public boolean find(int start)

// Проверяет, совпадает ли начало строки (prefix) с шаблоном.
public boolean lookingAt()
```


### Доступ к результатам совпадений
Методы для извлечения найденных фрагментов и их позиций
```java
// 💡Возвращает всю найденную подстроку (последнее совпадение).
public String group()

// Возвращает содержимое конкретной захваченной группы.
// group — номер группы (0 — вся строка, 1..n — подгруппы).
public String group(int group)

// Возвращает количество групп в шаблоне.
public int groupCount()

// Возвращает индекс начала последнего совпадения.
public int start()

// Возвращает индекс начала указанной группы.
public int start(int group)

// Возвращает индекс конца последнего совпадения.
public int end()

// Возвращает индекс конца указанной группы.
public int end(int group)
```

### Замена текста
Методы для подстановок, замены и построения нового текста
```java
// 💡Заменяет ПЕРВОЕ совпадение💡 шаблона на replacement.
public String replaceFirst(String replacement)

// 💡Заменяет ВСЕ совпадения💡 шаблона на replacement.
public String replaceAll(String replacement)

// Гибкий механизм: записывает часть строки в буфер до найденного совпадения,
// затем вставляет replacement. Обычно используется в цикле.
public Matcher appendReplacement(StringBuffer sb, String replacement)

// Завершает процесс замены и дописывает остаток строки в буфер.
public StringBuffer appendTail(StringBuffer sb)
```

### Работа с группами и именованными подгруппами
```java
// Возвращает содержимое именованной группы.
// name — имя группы, заданное в шаблоне (?<name>...).
public String group(String name)
```

### Управление входными данными
Методы ==для переиспользования Matcher== с новым текстом
```java
// Сбрасывает состояние и устанавливает новый входной текст.
public Matcher reset(CharSequence input)

// Сбрасывает состояние для текущего текста (начать поиск заново).
public Matcher reset()
```

### Информация о состоянии
Диагностические методы для понимания текущего состояния поиска
```java
// Возвращает строку, с которой работает Matcher.
public String toString()

// Проверяет, завершен ли поиск совпадений.
public boolean hitEnd()

// Проверяет, может ли шаблон соответствовать продолжению входа.

```

---

## 💡 Особенности и подводные камни 💡

1️⃣ ❌`Matcher` **не потокобезопасен** — каждый поток должен иметь свой экземпляр.  

2️⃣ После успешного вызова `find()` методы `group()`, `start()`, `end()` действительны, 
но **до** первого `find()` вызов `group()` вызовет `IllegalStateException`. 

3️⃣ Если шаблон содержит **захватывающие группы**, `groupCount()` покажет их количество, но `group(0)` — всегда вся строка совпадения.  

4️⃣ При использовании `replaceAll()` подстроки вида `$1`, `$2` заменяются на содержимое соответствующих групп.  

5️⃣ `Matcher` можно **переиспользовать** с помощью `reset()` — это быстрее, чем создавать новый объект.

---
## 💎 Best Practices 💎

✅ 💡Всегда вызывай `find()` перед `group()`.  

✅ При множественных заменах используйте `appendReplacement()` и `appendTail()` для максимальной гибкости.  

✅ 💡Если нужна только проверка полного совпадения, — `matches()`;  
если поиск внутри строки, — `find()`.  

✅ ⭐⭐Для многократных проверок компилируй `Pattern` один раз и используй один `Matcher` с `reset()`.  

✅ ⭐⭐⭐Не держи `Matcher` в static-полях — он хранит состояние и не потокобезопасен.

## Пример
```java
import java.util.regex.*;

public class MatcherExample {
    public static void main(String[] args) {
        String text = "User: admin, Email: admin@example.com; User: guest, Email: guest@mail.com";

        // Компилируем шаблон
        Pattern pattern = Pattern.compile("Email:\\s*(\\S+)");
        Matcher matcher = pattern.matcher(text);

        // Поиск всех совпадений
        while (matcher.find()) {
            System.out.println("Найден email: " + matcher.group(1)
                    + " (позиция: " + matcher.start(1) + ")");
        }

        // Пример замены
        String masked = matcher.replaceAll("Email: [hidden]");
        System.out.println("После замены: " + masked);
    }
}

Найден email: admin@example.com; (позиция: 20)
Найден email: guest@mail.com (позиция: 59)
После замены: User: admin, Email: [hidden] User: guest, Email: [hidden]
```
