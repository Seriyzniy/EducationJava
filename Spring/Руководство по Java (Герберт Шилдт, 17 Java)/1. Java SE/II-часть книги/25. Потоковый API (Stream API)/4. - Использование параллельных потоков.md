Одно из преимуществ потокового API связано с возможностью простой и надежной параллельной обработки определенных операций.

Параллельную обработку потока запросить довольно просто: достаточно воспользоваться параллельным потоком.

### Порядок элементов
В общем случае, если источник данных упорядочен, то и поток будет упорядоченным. 

Однако при использовании параллельного потока иногда можно увеличить производительность, позволив потоку быть неупорядоченным.

В случаях, когда порядок операций не имеет значения, можно установить неупорядоченное поведение, вызвав метод unordered() :
```java
stream.unordered()    // снять ограничения упорядоченности
```
### Сохранение порядка в параллельных потоках
```java
// Порядок не гарантируется:
parallelStream.forEach(...)

// Порядок сохраняется:
parallelStream.forEachOrdered(...)
```

### Управление режимом потока
```java
stream.parallel()     // переключить на параллельный
stream.sequential()   // переключить на последовательный  
```

### Требования к операциям в параллельных потоках
Операции должны быть:

1. **Без состояния** - не зависеть от порядка обработки
    
2. **Без побочных эффектов** - не изменять исходные данные
    
3. **Ассоциативные** - порядок операций не влияет на результат

### Создание параллельных потоков
```java
// 🟢Из коллекции
Stream<T> parallelStream = collection.parallelStream();

// 🟠Из последовательного потока  
Stream<T> parallelStream = stream.parallel();
```

Метод `parallel()` ==возвращает параллельный поток==, основанный на последовательном потоке, для которого он был вызван.
(Если он вызывается на потоке, который уже является параллельным, тогда возвращается вызывающий поток.)

После получения параллельного потока потоковые операции могут выполняться параллельно, при условии, что параллелизм поддерживается средой.

> [!NOTE] ⭐Параллельный поток делит список на части (chunks).⭐
> 
### Третья форма reduce() для параллельных потоков
```java
<U> U reduce(U identityVal, 
             BiFunction<U,? super T,U> accumulator,
             BinaryOperator<U> combiner)
```

**Пример с разными функциями:**
```java
ArrayList<Integer> myList = new ArrayList<>();
myList.add(2); myList.add(3); myList.add(10); 
myList.add(2); myList.add(5); myList.add(2);


// Произведение квадратных корней (параллельный поток)
double productOfSqrRoots = myList.parallelStream()
		.reduce(1.0,
			(a,b) -> a * Math.sqrt(b),    // аккумулятор
			(a,b) -> a * b                // комбинатор
		);

System.out.println(productOfSqrRoots);

// Вывод
34.64
```

**Часть 1 `[2, 3, 10]`:**
```java
// identity = 1.0
result1 = 1.0 * sqrt(2) = 1.414  // (a,b) -> a * sqrt(b)
result1 = 1.414 * sqrt(3) = 1.414 * 1.732 = 2.449
result1 = 2.449 * sqrt(10) = 2.449 * 3.162 = 7.746
```

**Часть 2 `[2, 5, 2]:`**
```java
// identity = 1.0  
result2 = 1.0 * sqrt(2) = 1.414
result2 = 1.414 * sqrt(5) = 1.414 * 2.236 = 3.162
result2 = 3.162 * sqrt(2) = 3.162 * 1.414 = 4.472
```

Комбинирование результатов
```java
finalResult = result1 * result2  // (a,b) -> a * b
finalResult = 7.746 * 4.472 = 34.641
```
