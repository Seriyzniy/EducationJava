# 📕1. IntStream
Интерфейс для работы с последовательностями примитивных значений типа `int`

Позволяет выполнять потоковые операции без автобоксинга (`int → Integer`), повышая производительность.  

Предназначен для **массовых вычислений, статистики, фильтрации, агрегации и преобразования числовых данных**.
```java
public interface IntStream extends BaseStream<Integer, IntStream> {

    /* =======================
       Фабричные методы (static)
       ======================= */

    // Создает поток из указанных значений
    static IntStream of(int... values)

    // Создает пустой поток
    static IntStream empty()

    // Создает бесконечный поток, значения которого генерируются через IntSupplier
    static IntStream generate(IntSupplier s)

    // Создает бесконечный поток, начиная с seed, применяя к нему функцию f
    static IntStream iterate(int seed, IntUnaryOperator f)

    // Создает конечный поток, где hasNext контролирует условие продолжения
    static IntStream iterate(int seed, IntPredicate hasNext, IntUnaryOperator next)

    // Возвращает поток от startInclusive до endExclusive
    static IntStream range(int startInclusive, int endExclusive)

    // Возвращает поток от startInclusive до endInclusive
    static IntStream rangeClosed(int startInclusive, int endInclusive)

    // Объединяет два IntStream в один
    static IntStream concat(IntStream a, IntStream b)

    // Создает билдер для ручного добавления элементов
    static IntStream.Builder builder()



    /* =======================
       Промежуточные операции (Intermediate)
       ======================= */

    // Фильтрует элементы по условию
    IntStream filter(IntPredicate predicate)


    // Преобразует каждый элемент
    IntStream map(IntUnaryOperator mapper)

    // Преобразует каждый элемент в объектный поток
    <U> Stream<U> mapToObj(IntFunction<? extends U> mapper)

    // Преобразует поток int → long
    LongStream mapToLong(IntToLongFunction mapper)

    // Преобразует поток int → double
    DoubleStream mapToDouble(IntToDoubleFunction mapper)


    // "Разворачивает" потоки (аналог flatMap)
    IntStream flatMap(IntFunction<? extends IntStream> mapper)

    // Возвращает элементы без дубликатов
    IntStream distinct()

    // Сортирует элементы
    IntStream sorted()

    // Позволяет "подсмотреть" элементы без изменения потока
    IntStream peek(IntConsumer action)

    // Ограничивает количество элементов
    IntStream limit(long maxSize)

    // Пропускает первые n элементов
    IntStream skip(long n)

    // Убирает упорядоченность (для оптимизации параллельных потоков)
    IntStream unordered()



    /* =======================
       Терминальные операции (Terminal)
       ======================= */
       
    // Итерация по элементам
    void forEach(IntConsumer action)
    void forEachOrdered(IntConsumer action)

    // Агрегирующие операции
    int sum()
    OptionalInt min()
    OptionalInt max()
    long count()
    OptionalDouble average()
    IntSummaryStatistics summaryStatistics()

    // Редукция
    int reduce(int identity, IntBinaryOperator op)
    OptionalInt reduce(IntBinaryOperator op)

    // Преобразование в коллекции
    <R> R collect(Supplier<R> supplier,
                  ObjIntConsumer<R> accumulator,
                  BiConsumer<R, R> combiner)

    // Проверки
    boolean anyMatch(IntPredicate predicate)
    boolean allMatch(IntPredicate predicate)
    boolean noneMatch(IntPredicate predicate)

    // Поиск
    OptionalInt findFirst()
    OptionalInt findAny()

    // Преобразование в массив
    int[] toArray()


    /* =======================
       🔁 Конвертация в другие потоки
       ======================= */
       
    // Преобразует int-поток в Stream<Integer>
    Stream<Integer> boxed()

    // Преобразует в LongStream
    LongStream asLongStream()

    // Преобразует в DoubleStream
    DoubleStream asDoubleStream()



    /* =======================
       ❗❗❗ Вложенный интерфейс Builder
       ======================= */
    interface Builder extends IntConsumer {
        // Добавляет элемент
        Builder add(int t)

        // Строит поток из добавленных элементов
        IntStream build()
    }
}
```

## Особенности и подводные камни 💡

1️⃣ Потоки примитивов **не допускают null**, что избавляет от NPE.  

2️⃣ Методы `average()`, `min()`, `max()` возвращают `OptionalDouble` / `OptionalInt` — нужно проверять наличие значения.  

3️⃣ У `IntStream.generate()` и `iterate()` — **бесконечные потоки**, обязательно ограничивай `.limit(n)`.  

4️⃣ После терминальной операции поток нельзя использовать повторно.  

5️⃣ При конвертации `boxed()` возможен **автобоксинг**, снижающий производительность.

## 💎 Best Practices 💎

✅ 💡Используй `IntStream.range()` вместо обычного цикла `for` — чище и функциональнее.  

✅ 💡Применяй `summaryStatistics()` для сбора статистики (`count`, `min`, `max`, `avg`, `sum`).  

✅ Для безопасного извлечения значений из `OptionalInt` используй `.orElse(defaultValue)`.  

✅ ❗❗❗При генерации бесконечных потоков всегда ограничивай `.limit(n)`.  

✅ Избегай `forEach()` с изменением внешнего состояния — это антипаттерн в функциональном стиле.  

✅ Для больших диапазонов можешь использовать `.parallel()`, если обработка независимая и CPU-bound.

## Пример
```java
import java.util.*;
import java.util.stream.*;

public class IntStreamFullExample {
    public static void main(String[] args) {
        // Создание потока чисел от 1 до 100
        IntStream stream = IntStream.rangeClosed(1, 100);

        // Обработка: фильтрация, преобразование, агрегирование
        IntSummaryStatistics stats = stream
                .filter(x -> x % 2 == 0)          // оставить только чётные
                .map(x -> x * x)                  // возвести в квадрат
                .limit(10)                        // взять первые 10
                .summaryStatistics();             // собрать статистику

        System.out.println("Count = " + stats.getCount());
        System.out.println("Sum = " + stats.getSum());
        System.out.println("Average = " + stats.getAverage());
        System.out.println("Min = " + stats.getMin());
        System.out.println("Max = " + stats.getMax());

        // Пример generate + limit
        IntStream.generate(() -> (int)(Math.random() * 10))
                 .limit(5)
                 .forEach(System.out::println);
    }
}
```

# 📕2. LongStream и DoubleStream

Эти два потока идентичны IntStream по методам. 

## ✅ Сравнение - Одинаковые методы

```java
// IntStream
static IntStream of(int... values)
static IntStream range(int start, int end)
static IntStream generate(IntSupplier s)

// LongStream  
static LongStream of(long... values)
static LongStream range(long start, long end)
static LongStream generate(LongSupplier s)

// DoubleStream
static DoubleStream of(double... values)
// НЕТ range() для double!
static DoubleStream generate(DoubleSupplier s)
```

### Промежуточные операции
```java
// IntStream
IntStream map(IntUnaryOperator mapper)
IntStream filter(IntPredicate predicate)

// LongStream
LongStream map(LongUnaryOperator mapper) 
LongStream filter(LongPredicate predicate)

// DoubleStream
DoubleStream map(DoubleUnaryOperator mapper)
DoubleStream filter(DoublePredicate predicate)
```

### Терминальные операции
```java
// IntStream
int sum()
OptionalInt min()
OptionalDouble average()

// LongStream  
long sum()
OptionalLong min()
OptionalDouble average()

// DoubleStream
double sum()
OptionalDouble min()
OptionalDouble average()
```

## ❌ Сравнение - РАЗЛИЧАЮЩИЕСЯ методы
### Range методы (только для целых чисел)
```java
// ЕСТЬ в IntStream и LongStream
IntStream.range(1, 5)        // 1, 2, 3, 4
LongStream.range(1L, 5L)     // 1, 2, 3, 4

// НЕТ в DoubleStream!
// DoubleStream.range(1.0, 5.0) - ❌ Не существует
```

### Особенности DoubleStream
```java
// Дополнительные методы для работы с NaN/бесконечностью
boolean isNaN(double value) - только в DoublePredicate

// Особенности вычислений
DoubleStream.sum() // Может быть NaN или бесконечностью
```
