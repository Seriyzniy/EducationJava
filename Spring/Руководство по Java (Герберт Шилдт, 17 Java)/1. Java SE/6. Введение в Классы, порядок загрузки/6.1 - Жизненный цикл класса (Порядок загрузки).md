==Создание объекта== в Java (`new Object()`) ==**не атомарно**==. 

Оно ==распадается на шаги==:
1. Выделение памяти.
2. Конструирование объекта (вызов конструктора).
3. Присвоение ссылки в переменную `obj`.

# 🧠 Краткий обзор:
Выводы из этого примера такие:

1. **🔶Статическая инициализация** (статические поля + статические блоки):
    - ==выполняется **только один раз**== при первой инициализации класса;
        
    - выполняется в порядке, в котором они записаны в коде;
        
    - всегда **до начала работы `main()`**, если в классе есть `main`.
        
2. 🔶**Нестатическая инициализация** (обычные поля + нестатические блоки):
    - выполняется **каждый раз при создании объекта**;
        
    - порядок: сначала инициализируются нестатические поля, потом выполняются нестатические блоки, и только потом вызывается конструктор.
        
3. 🔶**Конструктор**:
    - ==всегда идёт **после инициализации всех нестатических членов**==;
        
    - выполняется при каждом создании объекта.
        
4. ☢**Общий порядок**:
    - Статические члены → один раз при загрузке класса.
        
    - Нестатические члены → при каждом `new`.
        
    - Конструктор → всегда последний шаг при создании объекта.

# 📕1. Жизненный цикл класса (Class Lifecycle)

JVM проходит 3 основные стадии при работе с классом:

## 📙Loading (загрузка)
- 🔶Выполняется **ClassLoader’ом**.
    
- 🔶Находит байт-код класса (`.class`), загружает его в память, создаёт объект `java.lang.Class`.
    
- 🔶При загрузке учитывается **иерархия загрузчиков**:
    - Bootstrap ClassLoader (ядро JDK: `java.*`)
    - Extension (Platform) ClassLoader (`javax.*`, модули)
    - Application (System) ClassLoader (классы приложения, classpath)
    - Пользовательские (custom) ClassLoader’ы

- Принцип делегирования: сначала ищет родитель, потом сам.

## 📙Linking (связывание)
 Проходит в три подэтапа:
1. 🔶**Verification** – проверка байт-кода на корректность и безопасность (типобезопасность, формат).
    
2. 🔶**Preparation** – ==выделение памяти под **статические поля** и инициализация их **значениями по умолчанию**== (0, null, false и т. д.).
> [!NOTE] 🛑На этом этапе НЕТ ПОЛЬЗОВАТЕЛЬСКИХ значений из кода!🛑
> 

3. 🔶**Resolution** – подстановка реальных ссылок вместо символьных (методы, поля, классы). Может быть выполнено отложенно (**lazy resolution**)
        
## 📙Initialization (инициализация)
- 🔶Выполняется первый раз при **первом обращении** к классу (например, `new`, `static field/method`, `Class.forName()`).
    
- 🔶Последовательность:
    1. Выполняются **статические инициализаторы** (`static { ... }`) инициализации в порядке объявления.
        
    2. Присваиваются значения статическим полям, заданные явно.
        
- 🔶Инициализация **сначала у родителя**, потом у текущего класса.

# 2. 📕⭐ Вложенные классы (inner classes)

## 1) **Статический вложенный класс (static nested class)**
- ==Загружается и инициализируется **только ПРИ ПЕРВОМ ОБРАЩЕНИИ к нему**.==
    
- Не зависит от создания объекта внешнего класса.
    
- Может содержать `static` поля и блоки.  
    Пример:
```java
class Outer {
    static class Inner {
        static int x = init();
        static int init() { System.out.println("Inner loaded"); return 1; }
    }
}

public static void main(String[] args) {
    System.out.println("Main start");
    Outer.Inner.x;   // только тут загрузится Inner
}
```

## 2) **Нестатический вложенный класс (inner class)**
- Всегда ==связан с объектом== внешнего ==класса==.
    
- ==Его загрузка и инициализация ТОЖЕ происходят ПРИ ПЕРВОМ создании/обращении:==
```java
class Outer {
    class Inner {
        Inner() { System.out.println("Inner created"); }
    }
}

public static void main(String[] args) {
    Outer o = new Outer();
    Outer.Inner in = o.new Inner(); // Inner загрузится только тут
}
```

> [!NOTE] ⭐ИТОГО⭐
> ==Вложенные классы грузятся **отдельно от внешнего**:==
> - static nested — при первом обращении,
> - inner/anonymous/local — при первом создании, и только через объект внешнего класса.

***
# 💡 Важные детали
- Класс загружается **лениво** (по факту необходимости).
    
- Повторная загрузка невозможна — класс в ClassLoader загружается единожды.
    
- Несколько ClassLoader’ов могут загрузить один и тот же класс → будут разные объекты `Class`.
    
- Инициализация запускается **только один раз** на класс/loader-комбинацию.
    
- Если класс **не используется** → он может так и остаться незагруженным.
    
- `Class.forName("MyClass")` вызывает **инициализацию**,  
    а `ClassLoader.loadClass("MyClass")` только загружает.

# Пример
```java
public class MyMain {
	// 1. Статическое поле
    static int staticVar;

    // 2. Статический блок
    static {
    	System.out.println("======= STATIC INIT =======");
    	System.out.println("Before init:  staticVar = " + staticVar);
    	staticVar = initStaticVar();
    	System.out.println("After init: staticVar = " + staticVar + "\n");
    }

    // 3. Нестатическое поле
    int instanceVar;
    int secondInstanceVar = initInstanceVar();

    // 4. Нестатический блок
    {
    	System.out.println("\n======= NO_STATIC INIT =======");
    	System.out.printf("Before init: instanceVar = %d, secondInstanceVar = %d \n", instanceVar, secondInstanceVar);
    	instanceVar = initInstanceVar();
    	System.out.printf("After init: instanceVar = %d, secondInstanceVar = %d \n", instanceVar, secondInstanceVar);
    }

    // 5. Конструктор
    public MyMain() {
        System.out.println("Конструктор выполнен");
    }

    private static int initStaticVar() {
        System.out.println("Статическое поле инициализировано");
        return 1;
    }

    private int initInstanceVar() {
        System.out.println("Нестатическое поле инициализировано");
        return 2;
    }

    public static void main(String[] args) {
        System.out.println("===== Начало main ======");
        
        System.out.println("Before create obj");
        new MyMain();
        
        
        System.out.println("\n===== Второе создание того же класса =====");
        new MyMain();
    }
}
```

```java
public class MyMain {
	public static void main(String[] args) {
		try {
			System.out.println("=== Начало работы программы ===");

	        // 1. Только загружаем класс через ClassLoader (без инициализации!)
	        Class<?> clazz = MyMain.class.getClassLoader().loadClass("MyClass");
	        System.out.println("Класс MyClass загружен (но не инициализирован) \n---");

	        // 2. Явное инициирование (инициализация запускается при Class.forName)
	        Class.forName("MyClass");
	        System.out.println("Class.forName вызвал инициализацию \n---");

	        // 3. Обращение к статическому полю (инициализация запускается автоматически)
	        System.out.println("Статическое поле: " + MyClass.STATIC_VALUE + "\n---");

	        // 4. Создание объекта (инициализация родителя + потомка)
	        new MyClass();

	        System.out.println("=== Конец программы ===");
		} catch (Exception e) {
			e.printStackTrace();
		}
		
    }
}
class ParentClass {
    static int PARENT_STATIC = initParentStatic();

    static {
        System.out.println("ParentClass: статический блок выполнен");
    }

    {
        System.out.println("ParentClass: обычный блок выполнен");
    }

    ParentClass() {
        System.out.println("ParentClass: конструктор выполнен");
    }

    private static int initParentStatic() {
        System.out.println("ParentClass: статическое поле инициализировано");
        return 100;
    }
}

class MyClass extends ParentClass {
    static int STATIC_VALUE = initStatic();

    static {
        System.out.println("MyClass: статический блок выполнен");
    }

    {
        System.out.println("MyClass: обычный блок выполнен");
    }

    MyClass() {
        System.out.println("MyClass: конструктор выполнен");
    }

    private static int initStatic() {
        System.out.println("MyClass: статическое поле инициализировано");
        return 42;
    }
}
```
