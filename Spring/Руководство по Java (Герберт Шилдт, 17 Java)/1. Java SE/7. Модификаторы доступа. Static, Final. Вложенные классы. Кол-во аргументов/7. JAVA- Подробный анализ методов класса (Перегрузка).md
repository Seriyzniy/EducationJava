# 📕1. Перегрузка методов

**==Перегрузка метода==** означает, что в одном классе могут быть **несколько методов с ==одинаковым именем**==, но **разным списком параметров**.

Это позволяет вызывать один и тот же метод с разными аргументами — компилятор сам выберет подходящий вариант.

> [!NOTE] Компилятор и перегрущка
> При перегрузке КОМПИЛЯТОР ==ищет соответствие между аргументами==, используемыми для вызова метода, ==и параметрами метода==

***
> [!NOTE] Если перегруженный метод не найден, но возможно повышение типа
> Если **метода с точно подходящей сигнатурой нет**, компилятор будет искать **наиболее подходящий вариант**, используя **автоматическое приведение типов**
```java
class Example {
    void test(double x) {
        System.out.println("double version: " + x);
    }
}

class Main {
    public static void main(String[] args) {
        Example ex = new Example();
        ex.test(5);  // int → double
    }
}
```
## ❌ Перегрузка только по ВОЗВРАЩАЕМОМУ типу — НЕВОЗМОЖНА

Пример **==неработающего==** кода:
```java
int test(int a) { return a; }
double test(int a) { return a * 1.0; } // Ошибка компиляции!
```

## ✅ Корректные примеры перегрузки:
```java
class MyClass {
    void print() {
        System.out.println("Без параметров");
    }

    void print(int a) {
        System.out.println("С целым: " + a);
    }

    void print(String s) {
        System.out.println("С строкой: " + s);
    }

    void print(int a, String s) {
        System.out.println("С двумя аргументами: " + a + ", " + s);
    }

    double print(double d) {
        System.out.println("С double: " + d);
        return d * 2;
    }
}
```

## Заключение

| Разница в...       | Перегрузка разрешена? |
| ------------------ | --------------------- |
| Кол-во параметров  | ✅ Да                  |
| Тип параметров     | ✅ Да                  |
| Порядок параметров | ✅ Да                  |
| Возвращаемый тип   | ❌ Нет                 |
