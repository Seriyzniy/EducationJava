# 📕 2. Автоупаковка
Современные версии Java включают два важных средства: автоупаковку и автораспаковку.

```java
Integer iOb = 100; // 🔶автоупаковка значения int

int i = iOb; // 🔶автораспаковка
```

## 3.1 Автоупаковка и методы
Автоупаковка / автораспаковка может быт инициирована при передаче аргумента методу или при возвращении методом значения.
```java
class AutoBox2 {
    // Принимает параметр типа Integer и возвращает значение int.
    static int m(Integer v) {
        return v; // автораспаковка в int
    }
    
    public static void main(String[] args) {
        // Передать значение int в m() и присвоить возвращаемое значение
        // объекту Integer. 
        
        //Здесь аргумент 100 автоупаковывается в объект Integer. 
        //Возвращаемое значение тоже автоупаковывается в объект Integer.
        Integer iOb = m(100);
        System.out.println(iOb);
    }
}
```

> [!NOTE] ⭐⭐⭐ЛИШНИЕ ПРЕОБРАЗОВАНИЯ из объекта в примитив и обратно 
> Для различных операторов, класс обертка распаковывается в примитив, а после вычислений заново упаковывается в объект

## 3.2 ⭐Автоупаковка/автораспаковка помогает предотвратить ошибки
```java
class UnboxingError {
    public static void main(String[] args) {
        Integer iOb = 1000; // автоматически упаковать значение 1000
        int i = iOb.byteValue(); // вручную распаковать как byte!!!
        System.out.println(i); // выводится не 1000!
    }
}
```

Вместо ожидаемого значения 1000 программа выводит -24! Дело в том, что значение внутри iOb распаковывается вручную путем вызова byteValue ( ) , ==вызывая усечение значения==, которое хранится в iOb и равно 1000. 
В итоге i присваивается "урезанное" значение -24. 

Автораспаковка предотвращает ошибки такого типа, поскольку значение в iOb будет всегда автоматически распаковываться в значение, совместимое с int.

> [!NOTE] ВСЕ РАБОТАЕТ ПРАВИЛЬНО
> Биты из числа "урезаются", поэтому внимательно нужно смотреть какие именно биты урезаются
> Никакого мусорного значения на выходе не получаем!

# 📕 3. Особенности
## 📙Сравнение ==размера==: примитив vs обёртка

| Тип       | Примитивный тип (stack)  | Обёртка (heap)             | Примерный размер                  |
| --------- | ------------------------ | -------------------------- | --------------------------------- |
| `int`     | 4 байта                  | `Integer` → ~16–24 байт    | + метаданные JVM                  |
| `long`    | 8 байт                   | `Long` → ~24–32 байта      |                                   |
| `double`  | 8 байт                   | `Double` → ~24–32 байта    |                                   |
| `boolean` | 1 байт (JVM выравнивает) | `Boolean` → ~16 байт       | 2 объекта `true/false` кэшируются |
| `char`    | 2 байта                  | `Character` → ~16–24 байта |                                   |
### Заголовок объекта (object header)
Каждый объект в Java имеет **служебный заголовок**, который содержит:

| Компонент         | Назначение                                 | Размер (на 64-бит JVM с CompressedOops) |
| ----------------- | ------------------------------------------ | --------------------------------------- |
| **Mark Word**     | Хэш-код, флаги GC, информация о блокировке | 8 байт                                  |
| **Class Pointer** | Ссылка на метаинформацию класса (`Klass`)  | 4 байта (сжатые указатели)              |

## 📙 Зачем нужны обертки?

1. ✅ **Использование в обобщениях (generics)**  
```java
List<Integer> list = new ArrayList<>();
```
  
2. ✅ **Методы и функциональность**  
Обёртки имеют методы: 
- преобразование в строки, 
- парсинг, 
- сравнение и т.д.
```java
Integer.parseInt("123");
Boolean.valueOf("true");
```

2. ✅ **Работа с `null`**  
Примитивы не могут быть `null`, обёртки — могут. Это важно при взаимодействии с базами данных.
***
## 📙 Автоупаковка и автораспаковка
С Java 5, Java автоматически:
- **упаковывает (autoboxing)**: примитив → обёртка
- **распаковывает (unboxing)**: обёртка → примитив

```java
Integer x = 10;      // autoboxing: int → Integer
int y = x;           // unboxing: Integer → int
```

## 💡 Важные нюансы

### 1. **Кэширование значений**
Обёртки кэшируют часто используемые значения (от -128 до 127 для `Integer`):
```java
Integer a = 127;
Integer b = 127;
System.out.println(a == b);  // true (один и тот же объект)

Integer c = 128;
Integer d = 128;
System.out.println(c == d);  // false (разные объекты)
```

### 2. **Сравнение объектов**
Для сравнения используйте `equals()`, а не `==`:
```java
Integer x = 10;
Integer y = 10;
System.out.println(x.equals(y));  // true
```

### **3. Проблемы производительности**
1. Создание лишних объектов
```java
long sum = 0;
for (int i = 0; i < 1_000_000; i++) {
    sum += i;          // Быстро (примитивы)
}

Long sumWrapper = 0L;  // Опасность!
for (int i = 0; i < 1_000_000; i++) {
    sumWrapper += i;   // Медленно: автоупаковка i -> Long + распаковка sumWrapper -> long
}
```
