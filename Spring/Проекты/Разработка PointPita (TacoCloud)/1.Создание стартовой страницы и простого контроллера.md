## 1.1 Обработка веб-запросов
В состав Spring входит мощная веб-платформа, известная как Spring MVC. В основе Spring MVS лежит идея ***==контроллера==*** - класса, обрабатывающего запросы и возвращающего некоторую информацию.

Наше приложение будет работать на встроенном сервере ==Tomcat== (при использовании ==Dashboard==)

Напишем простой класс контроллера, который обрабатывает запросы с корневым путем (например /) и передает их представление домашней страницы без заполнения модели данных.

Для этого в главном пакете `ru.point`  создадим новый пакет, который будет хранить в себе контроллеры `Controllers` и в нем создадим новый класс `HomeController`
![[Pasted image 20241127094226.png]]

Теперь опишем созданный класс
```java
package ru.point.Controllers;

import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.GetMapping;

@Controller
public class HomeController {
	@GetMapping("/")
	public String home() {
		return "home";
	}
}
```

`@Controller` - Определяет класс как контроллер в MVC-приложении
`@GetMapping` - указывает что метод будет обрабатывать HTTP запросы GET с корневым путем.

## 1.2 Определение представления(домашней страницы)
В проекте будет использоваться шаблон Thymeleaf — **это современный шаблонизатор Java на стороне сервера**. Он делает упор на естественные HTML-шаблоны, которые можно предварительно просмотреть в браузере двойным щелчком мыши

Имя шаблона состоит из имени логического представления, к которому добавляются префикс пути /templates/ и расширение .html.

В результате полный путь к шаблону будет иметь вид: /templates/home.html

Это означает, что шаблон должен находиться в папке проекта src/main/resources/templates/home.html

Домашняя страница будет простой и содержать лишь приветствие посетителя сайта.

```html
<!DOCTYPE html>

<html xmlns="https://www.w3.org/1999/xhtml" xmlns:th="https://www.thymeleaf.org">
<head>
	<title>Taco Cloud</title>
</head>

<body>
	<h1>Welcome to ... </h1>
	<img th:src="@{/images/PointPita.jpg}"/>
</body>
</html>
```

Это стандартная страница html выводящая hello world. Единственное что здесь примечательно - строка с тегом ==<img==  отображающая логотип.

В ней присутствует Thymeleaf-атрибут `th:src` и выражение `@{. . .}`, описывающая ссылку на изображение.

Как описывалось в теории, все изображения, таблицы, стили должны находиться в папке /src/main/resources/static. 

Значит изображение PointPita.jpg должно находиться в этой папке (мы создадим подпапку images и поместим туда изображение).

## 1.3 Тестирование контроллера
У Spring есть мощные средства для тестирования MVCTest.

Он основан на использовании класса `MockMvc`, который предоставляет средства для тестирования веб-приложений на уровне контроллеров без необходимости поднимать полноценный сервер. 
Это означает, что вы можете проверять работу контроллеров, не создавая и не разворачивая целое приложение

```java
package ru.point;

import static org.hamcrest.CoreMatchers.containsString;
import static org.springframework.test.web.servlet.request.MockMvcRequestBuilders.get;
import static org.springframework.test.web.servlet.result.MockMvcResultMatchers.content;
import static org.springframework.test.web.servlet.result.MockMvcResultMatchers.status;
import static org.springframework.test.web.servlet.result.MockMvcResultMatchers.view;

import org.junit.jupiter.api.Test;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.test.autoconfigure.web.servlet.WebMvcTest;
import org.springframework.test.web.servlet.MockMvc;

import ru.point.Controllers.HomeController;

@WebMvcTest(HomeController.class)
public class HomeControllerTest {
	@Autowired
	private MockMvc mockMvc;
	
	@Test
	public void testHomaPage() throws Exception{
		mockMvc.perform(get("/"))
		.andExpect(status().isOk())
		.andExpect(view().name("home"))
		.andExpect(content().string(containsString("Welcome to ...")));
	}
}

```

Аннотация `@WebMvcTest` поднимает контекст приложения, необходимый только для тестирования слоя веба, инициализируя только те компоненты, которые связаны с вебом (например, контроллеры, фильтры и пр.).

`MockMvc` используется для отправки запроса к `/` и проверки:
- что ответ имеет статус HTTP 200 OK 
- представление имеет имя home
- получившаяся страница содержит текст "Welcome to . . . ".
