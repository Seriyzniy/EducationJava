# 1. Подключение зависимостей
### Подключение драйвера PostgreSQL
Поскольку наше приложение будет работать в БД PostgreSQL, нам необходимо добавить в зависимости драйвер PostgreSQL

```xml
<dependency>
    <groupId>org.postgresql</groupId>
    <artifactId>postgresql</artifactId>
    <version>42.7.3</version>
</dependency>
```
### Подключение Hibernate
Будем использовать версию Hibernate, который используется пространство имен `javax`
(более новые используют `jakatra`)
```xml
<dependency>
    <groupId>org.hibernate</groupId>
    <artifactId>hibernate-core</artifactId>
    <version>5.4.28.Final</version>
</dependency>
```

# 2. Конфигурация Hibernate
Перед использованием фреймворка Hibernate необходимо настроить его конфигурацию.
А именно необходимо настроить конфигурацию для:
- Источника данных (Драйвер, URL до БД, Пользователь, Пароль)
- Конфигурация самого Hibernate

1) Для этого необходимо создать в папке `src/main/resources` файл `hibernate.properties`. Именно файл с таким названием Hibernate будет искать
![[Pasted image 20241130134921.png]]

Поскольку Hibernate будет подключаться к PostgreSQL, ==необходимо указать данные для подключения к БД==
```
# Конфигурация источника данных (Data source)
hibernate.driver_class=org.postgresql.Driver
hibernate.connection.url=jdbc:postgresql://localhost:5432/hibernate_demo_db
hibernate.connection.username=postgres
hibernate.connection.password=postgres
```
Здесь мы указываем Драйвер, URL, Пользователя и Пароль для подключения к БД

2) Теперь, поскольку Hibernate может работать с разными БД, ==необходимо указать диалект== той БД, к которой он будет обращаться, поскольку у разных БД разный синтаксис для написания запросов
```
# Конфигурация самого Hibernate
hibernate.dialect=org.hibernate.dialect.PostgreSQLDialect
hibernate.show_sql=true
hibernate.current_session_context_class=thread
```
Также можно включить дополнительную функцию - отображения собранного SQL запроса в консоль `hibernate.show_sql=true`

`hibernate.current_session_context_class=thread` необходима для описания доступа к данным (ТРАНЗАКЦИЯМ) (многопоточно / или нет) 

> [!NOTE] При использовании `Spring` свойства `thread`
> Это свойство НЕ нужно указывать, поскольку Spring сам создает свою реализацию `SpringSessionContext`

# 3. Создание связи между таблицей и классом
Перед тем как считать данные с БД из таблицы `Persons` необходимо добавить данные в таблицу
```sql
INSERT INTO persons VALUES(1, 'Tom', 24);
```
![[Pasted image 20241130142050.png]]

> [!NOTE] Title
> Как и рассматривалось в теории - Hibernate позволяет работать с записями из таблицы как с объектами в программировании при помощи технологии ORM. 
> 
> Поэтому ==необходимо создать класс с полями, таким же как и столбцы в БД==.
> Такие классы ОБЫЧНО помещаются в пакет `model`

```java
@Entity
@Table(name="persons")
public class Person {
	@Id
	@Column(name="id")
	private int id;
	
	@Column(name="p_name")
	private String name;
	
	@Column(name="age")
	private int age;
	
	public Person() {}
	
	public int getId() {
		return id;
	}
	public int getAge() {
		return age;
	}
	public String getName() {
		return name;
	}
	
	public void setId(int id) {
		this.id = id;
	}
	public void setName(String name) {
		this.name = name;
	}
	public void setAge(int age) {
		this.age = age;
	}
}
```

> [!NOTE] Аннотации `Hibernate`
> `@Entity` - используется для аннотирования класса который будет связан с сущностью (таблицей) в БД
> Классы с такой аннотацией должны:
> - Содержать хотя бы ОДНО поле с аннотацией `@Id`
> - Иметь пустой конструктор
> 
>`@Table` - используется для явного указания таблицы БД для реализуемого класса(сущности)
> - Не обязательная аннотация, В СЛУЧАИ если ИМЯ ТАБЛИЦЫ СПОВПАДАЕТ с ИМЕНЕМ КЛАССА
> 
> `@Сolumn` - используется для указания имени СТОЛБЦА, из которого нужно взять значение для текущего поля (устанавливает соответствие СТОЛБЕЦ=ПОЛЕ класса)
> 
> `@Id` - специальная аннотация для указания что данное поле является первичным ключом из таблицы в БД

# 4. Считывание данных при помощи Hibernate


> [!NOTE] НАЧАЛО РАБОТЫ
> Перед тем как начать работу с Hibernate, необходимо прочитать данные из конфигурационного файла `hibernate.properties`, для установки соединения и настройки hibernate

Для этого ==необходимо создать объект класса== `Configuration` из пакета `org.hibernate.cfg`

После этого необходимо в этот класс добавить объект класса `Class`, в качестве аннотируемого (чтобы Hibernate понял, какая таблица и какой класс связаны между собой).
Это делается при помощи метода `addAnnotatedClass(Person.class)`

Далее необходимо создать сессию из класса `Configuration`. Для этого сначала необходимо создать `SessionFactory`, и потом из нее создать сессию `Session`

После получения сессии мы создаем(начинаем) транзакцию, указывает что нам нужно сделать:
- `get()` получает два аргумента, класс объекта в который нужно считать данные, и номер записи (`id`), которую нужно считать

И потом выполняем транзакцию методом `commit()`

После этого ОБЯЗАТЕЛЬНО нужно закрыть `SessionFactory`

```java
package ru.myProj.Hibernate.HibernateApp;
  
import org.hibernate.Session;
import org.hibernate.SessionFactory;
import org.hibernate.cfg.Configuration;
  
import ru.myProj.Hibernate.HibernateApp.model.Person;
  
public class App {
	public static void main(String[] args) {
		Configuration myConfigHiber = new Configuration();
		myConfigHiber.addAnnotatedClass(Person.class);
	
		SessionFactory mySessionFact = myConfigHiber.buildSessionFactory();
		Session mySession = mySessionFact.getCurrentSession();
		
		try {
			mySession.beginTransaction();
			Person pers = mySession.get(Person.class, 1);
			System.out.println(pers.getId() + " " + pers.getName() + " " + pers.getAge());
			mySession.getTransaction().commit();
		}finally {
			mySessionFact.close();
		}
	}
}
```
