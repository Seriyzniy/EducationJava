Для сохранения текущей даты и времени добавим в таблицу Persons соответствующие столбцы

```sql
ALTER TABLE Persons ADD COLUMN date_of_birthday DATE;
ALTER TABLE Persons ADD COLUMN created_at TIMESTAMP;
```

Также добавим соответствующие поля в java-классе, и СОЗДАДИМ Set/get методы
```java
@Column(name = "date_of_birthday")
@Temporal(TemporalType.DATE)
@DateTimeFormat(pattern = "dd/MM/yyyy")
private Date dateBirth;

@Column(name="created_at")
@Temporal(TemporalType.TIMESTAMP)
private Date createAt;
```

> [!NOTE] `@Temporal`
> Аннотация используется в JPA для преобразования поля даты и времени в объекте в тип SQL (DATE, TIME или TIMESTAMP) при сопоставлении с базой данных

> [!NOTE] `@DateTimeFormat`
> Указывает формат вводимых данных (можно самим реализовать валидацию и вывод ошибки на экран пользователя),
> Указываем `MM` так как `mm` это МИНУТЫ

Изменим представление для СОЗДАНИЯ НОВОГО человека для ввода даты 
```html
<br/>
<label for="Date">Enter date of birthday person</label>
<input type="text" th:field="*{dateBirth}" id="Date"/>
<div style="color:red" th:if="${#fields.hasErrors('dateBirth')}" th:errors="*{dateBirth}">Date error</div>
```

То же самое добавим в представление для ИЗМЕНЕНИЯ ДАННЫХ человека для ввода даты

Теперь в СЕРВИСНОМ КЛАССЕ будем сохранять ВРЕМЯ в БД.
Для этого просто вызовет set-метод и передадим в качестве параметра конструктор класса `Date()` из пакета `util` 
```java
@Transactional
public void save(Person person) {
	person.setCreateAt(new Date());
	personRepository.save(person);
}

@Transactional
public void update(int id, Person person) {
	person.setId(id);
	person.setCreateAt(new Date());
	personRepository.save(person);
}
```
