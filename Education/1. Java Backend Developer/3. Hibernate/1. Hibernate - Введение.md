# 1. Технология ORM
По причине сильной схожести классов и таблиц БД было придумано понятие ORM
> [!NOTE] ORM
> `ORM (Object Relational Mapping, объектно-реляционное отображение)` — это ==технология==, которая ==позволяет работать с базами данных== так, ==как== если бы это были ==объекты из языков программирования.==

![[Pasted image 20241130115411.png]]

При помощи технологии ORM, нам не нужно будет писать SQL-запрос для сохранения данных, мы просто будем вызывать для необходимого объекта метод `save()`, который реализует весь тот код, который мы делали раньше

### Связи между сущностями
Ранее рассматривалась БД, в которой были сущности: Клиент и заказ. 

У одного клиента может быть несколько заказов, но у одного заказа НЕ может быть несколько клиентов. Поэтому таблица (класс) `Order` хранит во внешнем ключе (поле) `id` (ссылку на объект) человека из таблицы `Person`

А в классе `Person` есть список заказов, и при помощи `Hibernate` мы можем вывести все заказы определенного человека, при этом `Hibernate` напишет код SQL самостоятельно
![[Pasted image 20241130120124.png]]

> [!NOTE] Аннотации `Hibernate`
> `@Entity` - используется для аннотирования класса который будет связан с сущностью (таблицей) в БД
> Классы с такой аннотацией должны:
> - Содержать хотя бы ОДНО поле с аннотацией `@Id`
> - Иметь пустой конструктор
> 
>`@Table` - используется для явного указания таблицы БД для реализуемого класса(сущности)
> - Не обязательная аннотация, В СЛУЧАИ если ИМЯ ТАБЛИЦЫ СПОВПАДАЕТ с ИМЕНЕМ КЛАССА
> 
> `@Сolumn` - используется для указания имени СТОЛБЦА, из которого нужно взять значение для текущего поля (устанавливает соответствие СТОЛБЕЦ=ПОЛЕ класса)
> 
> `@Id` - специальная аннотация для указания что данное поле является первичным ключом из таблицы в БД 
> Если поле класса, помеченное этой аннотацией не является числовым, то класс должен реализовать интерфейс `Serializable`

## 2. Транзакции (Transaction)

> [!NOTE] Транзакция
> Транзакция - единица для работы с БД. 
> 
> - Нужны для поддержания согласованности данных в таблице
> - Могут блокировать доступ к данным до тех пор пока данные не освободятся другим пользователем
> - Транзакции можно выполнять (COMMIT) и откатывать (ROLLBACK)
