# 1. Настройка pom.xml
## Подключение зависимостей
Теперь, подключим зависимости для проекта.
Будем разрабатывать на Spring MVC, и запускать в Tomcat, поэтому понадобятся еще и сервлеты. 

Чтобы было проще, добавил Jackson для де/сериализации, и SLF4J как API для логирования, а logback как логгер

## Сборка в war архив
Чтобы приложение можно было развернуть в Tomcat, проект должен собираться в .war файл.

Поэтому добавим в шапке pom.xml указание на сборку
```xml
<groupId>org.example</groupId>
<artifactId>KeyCloakProject</artifactId>
<version>1.0-SNAPSHOT</version>

<packaging>war</packaging>
```

И чтобы был нормальный naming проекта, в конце pom.xml добавим раздел build, в котором укажем имя финального .war архива
```xml
<build>
	<finalName>eventapp</finalName>
</build>
```

# 2. Docker-compose
Для volumes создадим отдельную папку в проекте, и в ней подпапки для каждого сервиса-контейнера
## Tomcat
Чтобы было проще развертывать проект, будем использовать Docker, в котором будет запущен контейнер Tomcat.

Чтобы было можно добавлять в Tomcat приложение, добавим `volumes` при описании сервиса в docker-compose
```yaml
services:
  tomcat:  
    image: tomcat:10.1.50-jdk17-temurin-jammy  
    container_name: tomcat_container  
    ports:  
      - "8081:8080"  
    volumes:  
      - ./volumes/tomcat/webapps:/usr/local/tomcat/webapps  
  
  
volumes:  
  postgres_data:  
  webapps:  
```

## KeyCloak и Postgres
Теперь, так как проект про KeyCloak, то добавим описание для развертывания KeyCloak с собственной БД в виде Postgres.
