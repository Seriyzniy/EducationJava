# 2. Добавление авторизации через KeyCloak в проект
После того как настроили KeyCloak, и изменили структуру проекта на Monorepo (Монорепозиторий - [[1.1 -- ИЗМЕНЕНИЕ СТРУКТУРЫ ПРОЕКТА]]), приступим к добавлению авторизации при помощи KeyCloak

Для этого добавим пакет `config.security`. И в этом пакете опишем конфигурацию `SecurityConfig`

## 1. SecurityConfig
В этом классе будет описана конфигурация `securityFilterChain` - цепочки фильтров безопасности, а также указано как и при помощи чего проверять токены

```java
package eventapp.config.security;  
  
import eventapp.controller.EventController;  
import org.slf4j.Logger;  
import org.slf4j.LoggerFactory;  
import org.springframework.context.annotation.Bean;  
import org.springframework.context.annotation.ComponentScan;  
import org.springframework.context.annotation.Configuration;  
import org.springframework.security.config.annotation.web.builders.HttpSecurity;  
import org.springframework.security.config.annotation.web.configuration.EnableWebSecurity;  
import org.springframework.security.oauth2.server.resource.authentication.JwtAuthenticationConverter;  
import org.springframework.security.web.SecurityFilterChain;  
  
@Configuration  
@EnableWebSecurity  
@ComponentScan("eventapp.config.security")  
public class SecurityConfig {  

    @Bean  
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception 
    {  
        http  
            .authorizeHttpRequests(auth -> auth  
				.requestMatchers("/eventapp/hello/**").permitAll()  
				.requestMatchers("/eventapp/event/**").authenticated()  
				.anyRequest().authenticated()  
			)  
			.oauth2ResourceServer(oauth2 -> oauth2  
				.jwt(jwt -> {  
					jwt.jwtAuthenticationConverter(
						jwtAuthenticationConverter()
					);  
					jwt.jwkSetUri(
						"http://localhost:9090/realms/eventrealm"
					);  
				})  
            );  
        return http.build();  
    }  
  
    private JwtAuthenticationConverter jwtAuthenticationConverter() {  
        JwtAuthenticationConverter converter = 
	        new JwtAuthenticationConverter();  
  
        converter.setJwtGrantedAuthoritiesConverter(
	        new KeycloakJwtAuthenticationConverter()
	    );  
	    
        return converter;  
    }  
}
```

> [!WARNING] Указание мэтчинга запросов
> Здесь указывается полный путь запроса включая название приложения, которое лежит в Tomcat, а не только относительно enpoint-ов в контроллере

### Аннотация @EnableWebSecurity
**`@EnableWebSecurity`**
- Включает механизм Spring Security в приложении.
    
- Создаёт базовую инфраструктуру Security: бины типа `springSecurityFilterChain`, обработку фильтров, менеджеры аутентификации и т.д.
    
- Без этой аннотации твои настройки `SecurityFilterChain` не будут применяться.

### Подключение OAuth2.0
Важной частью здесь является блок 
```java
.oauth2ResourceServer(oauth2 -> oauth2  
	.jwt(jwt -> {  
		jwt.jwtAuthenticationConverter(
			jwtAuthenticationConverter()
		);  
		jwt.jwkSetUri(
			"http://localhost:9090/realms/eventrealm"
		);  
	})  
);  
```

**`oauth2ResourceServer()`**
- Настройка Spring Security как **ресурсного сервера OAuth2**.
	
- То есть сервер не будет хранить пользователей сам, а будет проверять токены JWT, которые выдаёт Keycloak.
    
**`jwt(jwt -> {...})`**
- Настройка проверки JWT.

1. **`jwtAuthenticationConverter(jwtAuthenticationConverter())`**
    - Подставляем кастомный конвертер JWT → Spring Security `GrantedAuthority`.
        
    - Нужно, чтобы роли из токена Keycloak корректно преобразовывались в `SimpleGrantedAuthority` для Spring Security.
        
2. 🔥🔥`jwkSetUri("http://localhost:9090/realms/eventrealm")`🔥🔥
    - ==Адрес, откуда Spring Security будет брать== **публичные ключи для проверки подписи JWT**.
        
    - Keycloak хранит их в формате JWK.
        
    - Без этого Spring Security не сможет проверить токен и аутентификация будет невозможна.

### Метод jwtAuthenticationConverter()
```java
private JwtAuthenticationConverter jwtAuthenticationConverter() {
    JwtAuthenticationConverter converter = 
	    new JwtAuthenticationConverter();
	    
    converter.setJwtGrantedAuthoritiesConverter(
	    new KeycloakJwtAuthenticationConverter()
	);
	
    return converter;
}
```
- Создаёт объект **`JwtAuthenticationConverter`**, который превращает JWT в объект Spring Security `Authentication`.
    
- **`KeycloakJwtAuthenticationConverter`** — твоя кастомная реализация, которая:
    - Берёт список ролей из токена Keycloak (`roles`)
        
    - Превращает их в `SimpleGrantedAuthority`, которые Spring Security понимает для проверки авторизации.
        
- Без кастомного конвертера роли Keycloak (`realm roles`, `client roles`) не попадут в `Authentication`, и `@PreAuthorize` или `authenticated()` не будут работать корректно.

## 2. KeycloakJwtAuthenticationConverter - кастомный конвертер для JWT
Теперь нам нужно достать из JWT роли. Для этого нужно написать конвертер, и передать его в конфигурацию `SecurityFilterChain`

> [!NOTE] Claims
> Cliams это поля в токене JWT

```java
package eventapp.config.security;  
  
import org.springframework.core.convert.converter.Converter;  
import org.springframework.security.core.authority.SimpleGrantedAuthority;  
import org.springframework.security.core.GrantedAuthority;  
import org.springframework.security.oauth2.jwt.Jwt;  
  
import java.util.Collection;  
import java.util.Collections;  
import java.util.List;  
import java.util.stream.Collectors;  
  
public class KeycloakJwtAuthenticationConverter implements Converter<Jwt, Collection<GrantedAuthority>> {  
  
    @Override  
    public Collection<GrantedAuthority> convert(Jwt source) {  
        List<String> roles = source.getClaimAsStringList("roles");  
  
        if (roles == null) {  
            return Collections.emptyList();  
        }  
  
        return roles.stream()  
            .map(SimpleGrantedAuthority::new)  
            .collect(Collectors.toList());  
    }  
}
```

- Класс ==реализует== **`Converter<Jwt, Collection<GrantedAuthority>>`**.
    
- Это значит, что он **берёт JWT токен** (`Jwt`) и **превращает его в список прав (ролей) Spring Security** (`Collection<GrantedAuthority>`).
    
- `GrantedAuthority` — это объект Spring Security, который представляет **право или роль пользователя**, например `"ROLE_ADMIN"` или `"ROLE_USER"`.

### Метод convert
```java
@Override
public Collection<GrantedAuthority> convert(Jwt source) {
```
- `source` — это объект `Jwt`, который Spring Security получил от Keycloak и уже проверил его подпись через JWK.
    
- В нём содержатся **все данные токена**: `sub`, `iss`, `exp`, а также кастомные поля (`roles`, `realm_access` и т.п.).

```java
return roles.stream()
    .map(SimpleGrantedAuthority::new)
    .collect(Collectors.toList());
```
- Берём список строк ролей (`"USER"`, `"ADMIN"`) и превращаем каждую в объект **`SimpleGrantedAuthority`**.
    
- `SimpleGrantedAuthority` — стандартный объект Spring Security для ролей.
    
- В итоге получаем `Collection<GrantedAuthority>` — это **то, что Spring Security использует для проверки `authenticated()`, `hasRole()` и `@PreAuthorize`**.

# 3. Изменение конфигурации Tomcat
Теперь необходимо добавить конфигурацию Security в контекст DicpatherServlet, а также зарегистрировать фильтр `securityFilterChain` в контекст сервлетов и указать регистрацию при старте

> [!WARNING] Помечать класс как @Configuration не нужно
> Этот класс конфигурации DispathcerServlet не обязательно помечать как `@Configuration`, так как Tomcat и так будет искать конфигурацию 
> - либо в web.xml
> - либо искать класс, реализующий `WebApplicationInitializer` (или его дочерний `AbstractAnnotationConfigDispatcherServletInitializer`)

```java
package eventapp.config.tomcat;  
  
import eventapp.EventAppApplication;  
import eventapp.config.security.SecurityConfig;  
import jakarta.servlet.ServletContext;  
import jakarta.servlet.ServletException;  
import org.springframework.web.filter.DelegatingFilterProxy;  
import org.springframework.web.servlet.support.AbstractAnnotationConfigDispatcherServletInitializer;  
  
public class TomcatServerConfig extends AbstractAnnotationConfigDispatcherServletInitializer {  
    @Override  
    protected Class<?>[] getRootConfigClasses() {  
        return new Class[0];  
    }  
  
    @Override  
    protected Class<?>[] getServletConfigClasses() {  
        return new Class[]{
	        EventAppApplication.class, 
	        SecurityConfig.class};  
    }  
  
    @Override  
    protected String[] getServletMappings() {  
        return new String[]{"/"};  
    }  
  
    //Добавление фильтра  
    private void registerSecurityFilter(ServletContext servletContext) {  
        servletContext.addFilter(
	        "springSecurityFilterChain", 
	        new DelegatingFilterProxy("springSecurityFilterChain")
	    )  
        .addMappingForUrlPatterns(null, true, "/*");  
    }  
  
    //Добавление фильтра в запуск  
    @Override  
    public void onStartup(ServletContext servletContext) throws ServletException {  
        super.onStartup(servletContext);  
        registerSecurityFilter(servletContext);  
    }  
}
```

> [!NOTE] МОЖНО Объединить классы с аннотациями в один
> Можно классы конфигурации MVC и Security объединить в однин:
> ```java
> @Configuration
> @ComponentScan("eventapp")
> @EnableWebMvc
> @EnableWebSecurity
> public class EventAppApplication {
> }
> ```
> И тогда использовать только один класс в конфигурации DispatcherServlet

Но мы разнесем логику в разные классы для наглядности
