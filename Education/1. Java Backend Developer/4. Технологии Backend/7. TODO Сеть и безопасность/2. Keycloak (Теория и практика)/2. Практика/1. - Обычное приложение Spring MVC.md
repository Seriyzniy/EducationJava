# 2. Создание класса конфигурации Tomcat и настройка логирования
Вместо web.xml будем использовать Java-based конфигурацию
(см [[2. Практическое резюме - Java конфигурация]])

Для этого наследуемся от `AbstractAnnotationConfigDispatcherServletInitializer` и реализуем методы для настройки контекста приложения, и маппинга Dicpatcher-сервлета 

```java
package eventapp.config.tomcat;  
  
import eventapp.EventAppApplication;  
import org.springframework.context.annotation.Configuration;  
import org.springframework.web.servlet.support.AbstractAnnotationConfigDispatcherServletInitializer;  
  
@Configuration  
public class TomcatServerConfig extends AbstractAnnotationConfigDispatcherServletInitializer {  
    @Override  
    protected Class<?>[] getRootConfigClasses() {  
        return new Class[0];  
    }  
  
    @Override  
    protected Class<?>[] getServletConfigClasses() {  
        return new Class[]{EventAppApplication.class};  
    }  
  
    @Override  
    protected String[] getServletMappings() {  
        return new String[]{"/"};  
    }  
}
```

> [!WARNING]
> Важно, чтобы в конфигурацию (`getServletConfigClasses()`) был передан контекст (EventAppAppliaction), иначе сервлет не увидит контроллер

## Логирование
Настроем логирование так, чтобы было видно их на сервере Tomcat в консоли

Для этого будем использовать SLF4j и Logback, поэтому для Logback настроем файл `logback.xml` по пути `src/main/resources/logback.xml`
```xml
<?xml version="1.0" encoding="UTF-8"?>
<configuration>
    <appender name="STDOUT"
              class="ch.qos.logback.core.ConsoleAppender">
        <encoder>
            <pattern>
                %d{yyyy-MM-dd HH:mm:ss.SSS} %-5level [%thread] %logger{40} - %msg%n
            </pattern>
        </encoder>
    </appender>

    <root level="INFO">
        <appender-ref ref="STDOUT"/>
    </root>

</configuration>
```

После чего, в коде можем использовать 
```java
private static final Logger log =  
        LoggerFactory.getLogger(EventController.class);
```

# 4. Описание сервиса
Опишем сервис, который будет генерировать случайные события, и возвращать данные по определенной странице.

Также опишем заглушку метод, который будет сохранять событие, и отправлять событие в другой сервис
```java
package eventapp.service;  
  
import eventapp.dto.EventDto;  
import eventapp.dto.EventPageDto;  
import eventapp.dto.EventType;  
import org.springframework.stereotype.Service;  
  
import java.util.List;  
import java.util.Map;  
import java.util.Random;  
import java.util.UUID;  
import java.util.stream.IntStream;  
  
@Service  
public class EventService {  
    private final Random random = new Random();  
  
    private static final int TOTAL_EVENTS = 1000;  
  
    public EventPageDto getRandomGeneratedEvents(int page, int size) {  
        int fromIndex = page * size;  
        int toIndex = Math.min(fromIndex + size, TOTAL_EVENTS);  
  
        int totalPages = (int) Math.ceil((double) TOTAL_EVENTS / size);  
  
        List<EventDto> content = IntStream.range(fromIndex, toIndex)  
                .mapToObj(this::generateEvent)  
                .toList();  
  
        return new EventPageDto(
	        content, 
	        page, 
	        size, 
	        TOTAL_EVENTS, 
	        totalPages);  
    }  
  
    // Заглушка на дальнейший функционал  
    public EventDto save(EventDto dto) {  
        EventDto eventDto = new EventDto(
	        UUID.randomUUID().toString(), 
	        dto.type(), 
	        System.currentTimeMillis(), 
	        System.currentTimeMillis(), 
	        dto.description());  
  
        String message = "event: " + eventDto.uid() + " received";  
  
        Map<String, Object> notification = Map.of(  
                "type", "SYSTEM_ALERT",  
                "recipient", "admin@company.com",  
                "message", message  
        );  
  
        return eventDto;  
    }  
  
    private EventDto generateEvent(int index) {  
        return new EventDto(  
            UUID.randomUUID().toString(),  
            randomEventType(),  
            System.currentTimeMillis() - random.nextInt(1_000_000),  
            System.currentTimeMillis(),  
            "Событие №" + index);  
    }  
  
    private EventType randomEventType() {  
        EventType[] types = EventType.values();  
        return types[random.nextInt(types.length)];  
    }  
}
```

# 5. Создание контроллера
## Rest контроллер
Опишем контроллер, который будет принимать запросы пользователя на получение страницы событий, и на создание одного события
```java
package eventapp.controller;  
  
import eventapp.dto.EventDto;  
import eventapp.dto.EventPageDto;  
import eventapp.service.EventService;  
import org.slf4j.Logger;  
import org.slf4j.LoggerFactory;  
import org.springframework.beans.factory.annotation.Autowired;  
import org.springframework.web.bind.annotation.*;  
  
@RestController  
@RequestMapping("/event")  
public class EventController {  
    private static final Logger log =  
            LoggerFactory.getLogger(EventController.class);  
    private final EventService eventService;  
  
    @Autowired  
    public EventController(EventService eventService) {  
        this.eventService = eventService;  
    }  
  
    @GetMapping("/hello")  
    public String hello() {  
        return "hello";  
    }  
  
    @GetMapping  
    public EventPageDto getEventPageDto(  
            @RequestParam(name = "page", defaultValue = "0") int page,  
            @RequestParam(name = "size", defaultValue = "10") int size  
    ) {  
        return eventService.getRandomGeneratedEvents(page, size);  
    }  
  
    @PostMapping
    public EventDto createEvent(@RequestBody EventDto dto) {  
        return eventService.save(dto);  
    }  
}
```


> [!NOTE] `@RequestParam`
> Лучше указывать имя параметра явно, иначе необходимо указывать при сборке проекта некоторые настройки maven - иначе будет ошибка:
> "Нельзя определить эндпоинт, так как не определены параметры. Попробуйте запустить проект с параметром -parametrs".
> Этот -parametrs и задается в сборке Maven


## ExceptionHandlerController
Также полезно будет написать контроллер, который будет перехватывать ошибки при работе обычного контроллера

> [!IMPORTANT] `@ExceptionHandler`
> Для этого используется аннотация над методом `@ExceptionHandler`, которая помечает метод, который перехватывает ошибки эндпоинтов

```java
package eventapp.controller;  
  
import org.slf4j.Logger;  
import org.slf4j.LoggerFactory;  
import org.springframework.http.HttpStatus;  
import org.springframework.stereotype.Controller;  
import org.springframework.web.bind.annotation.ExceptionHandler;  
import org.springframework.web.bind.annotation.ResponseStatus;  
  
import java.util.Map;  
  
@Controller  
public class ExceptionHandlerController {  
    private static final Logger log =  
            LoggerFactory.getLogger(ExceptionHandlerController.class);  
  
    @ExceptionHandler(Exception.class)  
    @ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)  
    public Map<String, String> handleException(Exception e) {  
        log.error("Unhandled exception", e);  
  
        return Map.of(  
                "error", "Internal server error",  
                "message", e.getMessage()  
        );  
    }  
}
```
