# Без Docker
![[Pasted image 20251020144453.png]]

Программист: 
- пишет код и собирает в проект (war/ exe)
- прописывает документацию (какие порты нужны, какие программы и зависимости, и что нужно для запуска)
- отправляет в Git репозиторий

Админ:
- Получает задание на установку и запуск программы
- Скачивает программу с git
- Изучает документацию о запуске программы
- Подготавливает сервер
- Переносит программу на сервер, устанавливает зависимости, прокидывает порты

## Возможные проблемы
Если админ что-то забыл или не так понял, или программист добавил библиотеку и не добавил ее в документацию, ТО ПРИЛОЖЕНИЕ НЕ ЗАПУСТИТСЯ

Потом опять, что-то забыли, локально у программиста все работает. а на сервере нет. И так эта цепочка продолжает выполняться

# С Docker
![[Pasted image 20251020145355.png]]

Программист: 
- пишет код и собирает в проект (war/ exe)
- прописывает документацию (какие порты нужны, какие программы и зависимости, и что нужно для запуска)
- 💡ПИШЕТ Dokcer File - описание того, что необходимо для запуска и работы приложения (порты, зависимости, библиотеки)
- 💡При СБОРКЕ Docker File получает Docker Image (образ приложения)
- 💡ПУШИТ в DockerHub (Docker Registry) 

Админ:
- Получает задание на установку и запуск программы
- Подготавливает сервер для развертывания контейнеров Docker
- Скачивает образ Docker Image
- Запускает на сервере контейнер
