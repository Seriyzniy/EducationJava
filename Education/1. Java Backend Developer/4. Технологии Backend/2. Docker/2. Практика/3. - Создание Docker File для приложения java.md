# 📕1. Создание программы на Java
Для начала напишем программу на Java
```java
import java.util.List;
import java.util.Random;

public class MyMain {
	static List<String> PHRASES = List.of(
			"Freedom and peace",
			"Forever young",
			"Dare to dream",
			"Beauty and the Beast",
			"Blind rage",
			"Aspire to inspire"
			);
			
	
    public static void main(String[] args){
    	while(true) {
    		Random rand = new Random();
        	System.out.println(PHRASES.get((rand.nextInt(100) % PHRASES.size())));
        	try {
        		Thread.sleep(1000);
        	} catch (Exception e) {
        		
        	}
    	}
    }
}
```

Выводит рандомно заготовленные фразы и стопается на 1 секнуду

## Сборка проекта в файл jar
Теперь, нам НУЖНО (но не необходимо) собрать проект в файл jar.

Можно было бы скомпилировать файл в .class, и собрать на основе этого файла контейнер, но "как говорят", запуск `.jar` практичнее, так как не нужно сохранять структуру файлов

### Компиляция класса
```bash
javac -d ./out MyMain.java
```

### Сборка в jar файл
```bash
jar --create --file myJarApp.jar --main-class MyMain -C . MyMain.class
```

> [!NOTE] `-C` требует два параметра, первый директория, а второй имя файлов
> 

### Пробный запуск файла jar
```bash
java -jar myJarApp.jar
```

## 📙 Определение базового слоя
Теперь опишем ЧТО НЕОБХОДИМО ДЛЯ ЗАПУСКА ПРИЛОЖЕНИЯ
- Мы уже знаем что Docker работает на Linux, нам нужен базовый слой Linux
- Так как приложении на Java нам необходим JDK определённой версии. Так как мы писали приложение на Java 17, то и JDK нам нужен 17 версии.

Так как образ JDK уже собирался, значит он 100% СОБИРАЛСЯ НА УЖЕ КАКОМ-ТО БАЗОВОМ ОБРАЗЕ Linux, а значит можно просто указать что базовым слоев для нашего образа является Open JDK 17.

Чтобы найти правильное имя JDK, зайдем на DockerHub

После этого ищем любой подходящий Image (выбирал по наименьшему размеру)

## 📙Создание каталога приложения, где будут храниться все файлы приложения (рабочая директория)

Создадим на этом образе рабочую директорию, откуда будут выполняться все последующие инструкции
```dockerfile
FROM openjdk:21-ea-17-jdk-oracle
WORKDIR /app
```

## 📙Помещение файла приложения (.jar / .class и др) в рабочую директорию базового образа
Далее поместим наше приложение в рабочую директорию.

Путь до собранного проекта/класса собирается 
- относительно нашего Dockerfile
- абсолютный путь

![[Pasted image 20251021154114.png]]

Так как наш Dockerfile лежит внутри нашего проекта, то выберем относительный путь.

Используя команду COPY, скопируем файл проекта в нашу рабочую директорию, с указанием нового имени файла проекта
```dockerfile
FROM openjdk:21-ea-17-jdk-oracle
WORKDIR /app
COPY /src/out/myJarApp.jar /app/myJarAppCopy.jar
```

## 📙Определение команды запуска
Далее необходимо указать, как запускать наш проект ИЗ КОНСОЛИ

Можно сделать при помощи CMD или ENTRYPOINT
### Вариант 1 - CMD
В этой команде указываем команды(параметры) для запуска приложения.

Если в консоли мы обычно запускаем приложение при помощи
```bash
java -jar myJarApp.jar
```

То команда будет состоять из 3-х команд
- java
- -jar
- ПУТЬ ДО ПРОЕКТА относительно Linux (мы же запускаем в контейнере) 

При этом ==путь до рабочей папки указывать не нужно!== 
Однако, если была бы внутренняя папка 
(допустим `/app/myproject/myJarAppCopy.jar`), 
тогда бы указали `/myproject/myJarAppCopy.jar`

Поэтому получим команду
```dockerfile
CMD ["java", "-jar", "myJarAppCopy.jar"]
```

### Вариант 2 - ENTRYPOINT
Здесь тоже самое, указываются команды в консоли 
```dockerfile
ENTRYPOINT ["java", "-jar", "myJarAppCopy.jar"]
```

## 📙 Результат
```dockerfile
FROM openjdk:21-ea-17-jdk-oracle

ENTRYPOINT ["java", "-jar", "myJarAppCopy.jar"]
```

## Сборка Dockerfile в Image
Теперь необходимо выполнить сборку

Для этого используется команда 
```bash
docker build -t myfirstjarcontainer .
```
`-t` - указывает имя образа
> [!NOTE] ❗❗❗ Имя ДОЛЖНО БЫТЬ В `lowercase` (нижний регистр) ❗❗❗

`.` - указывает ==НА ПУТЬ к Dockerfile==, так как мы заранее перешли в папку с файлом, то ставим `.`
