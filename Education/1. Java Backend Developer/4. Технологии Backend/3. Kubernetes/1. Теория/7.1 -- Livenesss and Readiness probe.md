## ⭐ Liveness и Readiness-пробы ⭐

> [!NOTE] Указываются в манифестах `.yaml`
> Deployment → создаёт ReplicaSet → создаёт Pods 
> А ❗❗**пробы определяются именно в Pod’ах**❗❗.
> 
> То есть ты можешь указать их:
> - **В Deployment** — как часть шаблона Pod’а (чаще всего так и делают).
> - **В Pod-манифесте напрямую**, если ты создаёшь Pod без Deployment. 
> - Также пробы можно указать в **DaemonSet**, **StatefulSet**, **Job** — везде, где есть `PodSpec`.
> 

| Тип проверки                      | Назначение                                          | Что делает kubelet ==при сбое==                                     |
| --------------------------------- | --------------------------------------------------- | ------------------------------------------------------------------- |
| **Liveness Probe**                | Проверяет, «жив» ли контейнер.                      | Перезапускает контейнер, если он не отвечает.                       |
| **Readiness Probe**               | Проверяет, «готов ли контейнер обслуживать трафик». | Убирает Pod из Service (не отправляет трафик), но не перезапускает. |
| **Startup Probe** _(опционально)_ | Используется для медленно стартующих приложений.    | Пока она не завершится — другие проверки не выполняются.            |

```yaml
livenessProbe:
  httpGet:
    path: /healthz           # путь для проверки "живости"
    port: 8080
    
  initialDelaySeconds: 10 # ⭐время ожидания перед первой проверкой⭐
  periodSeconds: 5        # частота проверок
  failureThreshold: 3     # после скольких неудач контейнер перезапустится


readinessProbe:
  httpGet:
    path: /ready              # путь для проверки "готовности"
    port: 8080
  initialDelaySeconds: 5
  periodSeconds: 3
  failureThreshold: 2
```

### 💡 Виды проб (способы проверки) 💡
1. **httpGet** — проверка по HTTP-запросу.
2. **exec** — выполнение команды внутри контейнера.
3. **tcpSocket** — проверка доступности TCP-порта.

---
### Пример 1 — Deployment (через шаблон Pod’а)
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: myapp
spec:
  replicas: 2
  selector:
    matchLabels:
      app: myapp
      
  template: # 👇 Здесь начинается PodSpec
    metadata:
      labels:
        app: myapp
    spec:
      containers:
        - name: myapp
          image: myapp:latest
          livenessProbe:
            httpGet:
              path: /healthz
              port: 8080
```

### Пример 2 — Pod напрямую
```yaml
apiVersion: v1
kind: Pod
metadata:
  name: myapp-pod
spec:
  containers:
    - name: myapp
      image: myapp:latest
      readinessProbe:
        httpGet:
          path: /ready
          port: 8080
```

### Итог

|Где указывается|Почему работает|Типичные случаи|
|---|---|---|
|**Deployment**|В шаблоне Pod’а (`spec.template.spec.containers`)|99% случаев — приложения|
|**Pod**|Прямо в `spec.containers`|Тесты, отладка|
|**DaemonSet / StatefulSet / Job / CronJob**|В их `spec.template.spec.containers`|Системные демоны, базы, фоновые задачи|
