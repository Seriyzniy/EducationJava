### Пояснения к ключевым параметрам:
- **num.partitions** и **default.replication.factor** - влияют на масштабируемость и отказоустойчивость по умолчанию
    
- **min.insync.replicas** + **acks=all** на стороне продюсера дают гарантию надежной записи
    
- **log.retention** параметры контролируют политику хранения данных
    
- **replica.** параметры управляют репликацией и отказоустойчивостью
    
- **transaction.** параметры важны для транзакционных продюсеров
    
- **socket.** параметры влияют на сетевую производительность
    
- **[connections.max.idle.ms](https://connections.max.idle.ms/)** помогает контролировать ресурсы соединений

## Базовые настройки топиков
```yaml
# Сколько партиций создаётся по умолчанию для новых топиков
num.partitions=1

# 🔥 Фактор репликации по умолчанию для новых топиков
default.replication.factor=1

# Минимальное число реплик, которые должны быть “in-sync” (ISR) для записи
min.insync.replicas=2

# 🔥Разрешено ли авто-создание топиков при обращении
auto.create.topics.enable=false

# Разрешено ли удаление топиков
delete.topic.enable=true

# 💡Максимальный размер сообщения, принимаемого брокером
message.max.bytes=1048576
```

## Настройки логов и хранения
```yaml
# Путь к директории журналов (логов) брокера
log.dirs=/var/lib/kafka/logs

# Максимальный размер сегмента логов в байтах
log.segment.bytes=1073741824   # 1 GB

# Максимальное время хранения логов (в часах) до удаления
log.retention.hours=168   # 7 дней

# Максимальный размер логов в байтах (если правило retention по размеру)
log.retention.bytes=-1   # -1 означает без ограничения по размеру

# Интервал проверки условий удаления логов (ms)
log.retention.check.interval.ms=300000   # 5 мин

# Время перед удалением логов после их закрытия
log.segment.delete.delay.ms=60000

# Класс для политики удаления логов (по времени, размеру и т.д.)
log.cleanup.policy=delete

# Время между проверками необходимости компактизации логов
log.cleaner.backoff.ms=15000
```

# Настройки сети: число потоков для обработки сетевых запросов (listener)

num.io.threads=8

# Размер TCP буфера приёма

listeners=PLAINTEXT://:9092

## Replication и High Availability
```yaml
# Максимальное количество реплик для топиков
offsets.topic.replication.factor=3

# Таймаут для репликации между брокерами
replica.socket.timeout.ms=30000

# Максимальное отставание реплики в сообщениях
replica.lag.time.max.ms=30000

# Размер буфера для репликации
replica.socket.receive.buffer.bytes=65536

# Частота отправки fetch-запросов ведомыми репликами
replica.fetch.wait.max.ms=500

# Минимальное количество байт для fetch-запроса реплики
replica.fetch.min.bytes=1

# Максимальный размер данных для одного fetch-запроса реплики
replica.fetch.max.bytes=1048576

# Частота обновления метаданных реплик
replica.fetch.response.max.bytes=10485760
```
