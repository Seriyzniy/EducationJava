## Основные параметры воркера (Worker Configuration)
```properties
# Адрес(а) брокеров Kafka, с которыми Connect-воркер будет взаимодействовать.
bootstrap.servers=broker1:9092,broker2:9092

# Уникальный идентификатор группы воркеров в режиме распределённой (distributed) работы.
group.id=connect-cluster-1

# Путь(и) на файловой системе, где ищутся плагины (connectors, transforms, converters).
plugin.path=/usr/local/share/kafka/plugins

# Класс преобразователя (converter) для ключей сообщений.
key.converter=org.apache.kafka.connect.json.JsonConverter

# Класс преобразователя для значений сообщений.
value.converter=org.apache.kafka.connect.json.JsonConverter

# Включает или выключает хранение схемы (schema) в ключах.
key.converter.schemas.enable=false

# Включает или выключает хранение схемы (schema) в значениях.
value.converter.schemas.enable=false
```

**Пояснение**:
- `bootstrap.servers` необходим для подключения к кластеру Kafka.
    
- `group.id` важен для режима распределённых воркеров: все воркеры с одним `group.id` образуют кластер Connect. [Confluent Docs+1](https://docs.confluent.io/platform/current/connect/userguide.html?utm_source=chatgpt.com)
    
- `plugin.path` позволяет подключать сторонние коннекторы и преобразователи. [Confluent Docs+1](https://docs.confluent.io/platform/current/connect/userguide.html?utm_source=chatgpt.com)
    
- `key.converter` / `value.converter` отвечают за преобразование данных из/в Kafka (например JSON, Avro, Protobuf).
    
- `*.schemas.enable` контролируют, сохранить ли схему данных вместе с нагрузкой — важно для типов данных с схемой.

## Параметры внутренних тем (Internal Topics)
```properties
# Топик, где хранится конфигурация коннекторов.
config.storage.topic=connect-configs

# Репликация топика конфигураций.
config.storage.replication.factor=3

# Топик, где хранится оффсеты задач коннекторов.
offset.storage.topic=connect-offsets

# Число разделов (partitions) для топика оффсетов.
offset.storage.partitions=25

# Репликация топика статусов коннекторов.
status.storage.topic=connect-status
status.storage.replication.factor=3
```

## Переопределение Producer/Consumer настроек внутри Connect
```properties
# Переопределение параметра consumer.max.poll.records для всех коннекторов.
consumer.max.poll.records=1000

# Переопределение параметра producer.compression.type для всех source-коннекторов.
producer.compression.type=lz4
```

## Клиентские политики переопределения и безопасности
```properties
# Политика переопределения клиентских настроек (producer/consumer/admin) через коннекторы.
connector.client.config.override.policy=None

# Уровень изоляции транзакций для consumer внутри Connect (read_committed/read_uncommitted).
consumer.isolation.level=read_committed
```

## Таймауты, ребалансировка и планировщики
```properties
# Максимальная задержка перед выполнением планового ребаланса (ms).
scheduled.rebalance.max.delay.ms=300000

# Таймаут сессии управления группой для воркеров (ms).
session.timeout.ms=10000

# Интервал проверки состояния задач или соединения (ms).
heartbeat.interval.ms=3000
```
