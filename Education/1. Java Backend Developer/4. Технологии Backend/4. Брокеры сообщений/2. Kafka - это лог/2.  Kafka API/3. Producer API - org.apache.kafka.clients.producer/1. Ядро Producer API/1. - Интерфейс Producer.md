
> [!NOTE] `Producer<K, V>`
> Интерфейс Kafka-продюсера, определяющий **базовые операции для отправки сообщений** (`send`), управления буферами, фиксации транзакций и закрытия соединения.
> 
> ⭐⭐Реализуется основным классом — **`KafkaProducer`**.
> 
> Не наследует других интерфейсов напрямую, но определяет контракт для асинхронной, потокобезопасной работы с Kafka.

## Методы

### Отправка сообщений
```java
// Отправляет запись в Kafka асинхронно, возвращает Future с метаданными записи
Future<RecordMetadata> send(ProducerRecord<K, V> record);

// Отправляет запись и вызывает указанный callback после завершения (успешно или с ошибкой)
Future<RecordMetadata> send(
	ProducerRecord<K, V> record, 
	Callback callback);
```
**Пояснение:**
- `ProducerRecord<K, V>` — объект с ключом, значением, топиком и партицией.
    
- `Callback` — интерфейс, вызываемый **при завершении отправки** (например, для логирования или ретраев).
    
- Возвращаемое значение — `Future<RecordMetadata>`, из которого можно получить метаданные (offset, partition и т.д.) после завершения.

### Управление жизненным циклом
```java
// Принудительно сбрасывает буфер сообщений (отправляет всё накопленное)
void flush();

// Закрывает продюсер и освобождает ресурсы
void close();

// Закрывает продюсер с ожиданием заданного времени до завершения всех запросов
void close(Duration timeout);
```

**Пояснение:**
- `flush()` блокирует поток до отправки всех сообщений.
    
- `close()` закрывает сетевые соединения и освобождает память.
    
- `close(Duration timeout)` — безопасно завершает работу в пределах указанного таймаута.

### Работа с транзакциями
```java
// Инициализирует транзакцию для данного продюсера
void initTransactions();

// Начинает новую транзакцию
void beginTransaction();

// Фиксирует (commit) текущую транзакцию
void commitTransaction();

// Отменяет (abort) текущую транзакцию
void abortTransaction();

// Отправляет смещения потребителя в рамках транзакции
void sendOffsetsToTransaction(
	Map<TopicPartition, OffsetAndMetadata> offsets, 
	String consumerGroupId);

// Отправляет смещения с метаданными группы потребителей в рамках транзакции
void sendOffsetsToTransaction(
	Map<TopicPartition, OffsetAndMetadata> offsets,
	ConsumerGroupMetadata groupMetadata);
```
**Пояснение:**
- ❗❗❗Методы используются при **Exactly-Once Semantics (EOS)** — когда нужно гарантировать, что сообщения не будут продублированы при сбоях.
    
- `initTransactions()` вызывается один раз перед началом транзакционной работы.
    
- `sendOffsetsToTransaction()` связывает offset'ы потребителя с записью транзакции, чтобы избежать двойной обработки сообщений.

---

## 💡 Особенности и подводные камни 💡

1️⃣ ❗❗❗`send()` — асинхронный метод, возвращает `Future`, но ошибки могут произойти позже (например, при ретраях или сетевых сбоях).  

2️⃣ 💡Не стоит вызывать `flush()` слишком часто — это блокирующая операция, уменьшающая производительность. 

3️⃣ ⭐⭐Для транзакций, `enable.idempotence=true` и `transactional.id` — обязательны.  

4️⃣ При использовании транзакций нельзя использовать один и тот же продюсер из разных потоков одновременно.

5️⃣ После `close()` объект продюсера больше не может быть использован.

---
## 💎 Best Practices 💎

✅ 💡Создавай **один экземпляр продюсера** на приложение или поток, а не на каждое сообщение.  

✅ 💡Настраивай `acks=all` и `retries>0` для надёжной доставки.  

✅ Используй `linger.ms` и `batch.size` для оптимизации производительности.  

✅ При Exactly-Once обработке всегда связывай продюсер и консьюмер через `sendOffsetsToTransaction`.  

✅ 💡Не забудь вызвать `flush()` перед завершением приложения, чтобы не потерять сообщения.

## Пример
```java
import org.apache.kafka.clients.producer.*;
import java.util.Properties;

public class SimpleProducerExample {
    public static void main(String[] args) {
        Properties props = new Properties();
        props.put(
	        "bootstrap.servers", 
	        "localhost:9092");
        props.put(
	        "key.serializer", 
	"org.apache.kafka.common.serialization.StringSerializer");
        props.put(
	        "value.serializer", 
	"org.apache.kafka.common.serialization.StringSerializer");


        // 1️⃣Создаём продюсер
        Producer<String, String> producer = new KafkaProducer<>(props);

        try {
            // 2️⃣Подготовка сообщения
            ProducerRecord<String, String> record = new ProducerRecord<>("test-topic", "key1", "Hello Kafka!");
            
            // 3️⃣Отправляем сообщение
            producer.send(record, (metadata, exception) -> {
                if (exception == null)
                    System.out.printf(
	                    "Сообщение отправлено в %s-%d,"
	                    "offset=%d%n",
                        metadata.topic(), 
                        metadata.partition(), 
                        metadata.offset());
                else
                    exception.printStackTrace();
            });
        } finally {
            // Гарантированная отправка всех сообщений и закрытие ресурсов
            producer.flush();
            producer.close();
        }
    }
}
```
