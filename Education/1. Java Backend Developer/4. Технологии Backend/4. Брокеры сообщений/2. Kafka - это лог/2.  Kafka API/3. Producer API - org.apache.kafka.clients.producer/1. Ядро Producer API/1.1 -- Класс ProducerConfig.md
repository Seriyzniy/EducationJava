
> [!NOTE] `ProducerConfig` — это **класс-конфигурация** Kafka продюсера, содержащий:
> - **все доступные параметры** (в виде `public static final String` ключей);
> - их **описания**; 
> - **умолчания** 
> - и **валидацию значений** при создании `KafkaProducer`.
> 

Он расширяет базовый класс
```java
org.apache.kafka.common.config.AbstractConfig
```
и предоставляет удобный способ конфигурирования продюсера через `Properties`, `Map` или файл настроек.

---
## Конструкторы - НЕ ИСПОЛЬЗУЕТСЯ 
```java
// Создает объект конфигурации продюсера из Properties.
public ProducerConfig(Properties props)

// Создает объект конфигурации продюсера из карты.
public ProducerConfig(Map<?, ?> props)
```
💬 Пояснение:
- **`props`** — коллекция пар ключ/значение, задающая настройки продюсера (например, bootstrap-сервера, сериализаторы, стратегия партиционирования и т. д.).
    
- При создании экземпляра происходит проверка корректности параметров (валидация типов, диапазонов и зависимостей).

> [!NOTE] ⭐⭐ИСПОЛЬЗУЮТСЯ его КОНСТАНТЫ⭐⭐
> 
### Доступ к ключам конфигурации

Класс содержит множество **публичных статических строковых констант**, представляющих **имена параметров конфигурации**.

```java
// Список брокеров Kafka, к которым подключается продюсер.
public static final String BOOTSTRAP_SERVERS_CONFIG = "bootstrap.servers";

// Класс сериализатора ключа.
public static final String KEY_SERIALIZER_CLASS_CONFIG = "key.serializer";

// Класс сериализатора значения.
public static final String VALUE_SERIALIZER_CLASS_CONFIG = "value.serializer";

// Количество подтверждений (acks) от брокеров.
public static final String ACKS_CONFIG = "acks";

// Максимальный размер буфера для неотправленных сообщений.
public static final String BUFFER_MEMORY_CONFIG = "buffer.memory";

// Максимальное количество попыток повторной отправки.
public static final String RETRIES_CONFIG = "retries";

// Максимальный размер одной отправки (в байтах).
public static final String MAX_REQUEST_SIZE_CONFIG = "max.request.size";

// Таймаут ожидания подтверждения (в миллисекундах).
public static final String REQUEST_TIMEOUT_MS_CONFIG = "request.timeout.ms";

// Интервал между попытками повторной отправки.
public static final String RETRY_BACKOFF_MS_CONFIG = "retry.backoff.ms";
```

## 💡 Особенности и подводные камни 💡

1️⃣ ⭐⭐⭐**Обязательные параметры:**⭐⭐⭐
- `bootstrap.servers` — список брокеров Kafka, например `"localhost:9092"`.
    
- `key.serializer` и `value.serializer` — классы сериализаторов для ключа и значения.
    

2️⃣ **ACKS управляет надежностью доставки:**
- `acks=0` — не ждать подтверждения (наиболее быстрая, но ненадежная отправка).
    
- `acks=1` — подтверждение лидером партиции.
    
- `acks=all` — подтверждение всеми репликами (надежно, но медленнее).
    

3️⃣ **Batching и производительность:**
- `batch.size` — размер пакета сообщений, отправляемого за один раз.
    
- `linger.ms` — сколько ждать, прежде чем отправить неполный батч.
    
- 💡`compression.type` — тип сжатия (`none`, `gzip`, `snappy`, `lz4`, `zstd`).
    

4️⃣ **Идемпотентность и транзакции:**
- `enable.idempotence=true` — гарантирует отсутствие дубликатов сообщений.
    
- `transactional.id` — включает transactional API для exactly-once доставки.
    

5️⃣ **Безопасность:**
- Поддержка SSL и SASL через параметры `security.protocol`, `ssl.*`, `sasl.*`.
    

6️⃣ **Память и контроль потока:**
- Если буфер (`buffer.memory`) переполнен — отправка блокируется максимум на `max.block.ms`.

## 💎 Best Practices 💎

✅ ==Минимальный рабочий набор параметров:==
```bash
bootstrap.servers=localhost:9092

key.serializer=org.apache.kafka.common.serialization.StringSerializer
value.serializer=org.apache.kafka.common.serialization.StringSerializer
```

✅ **Для высокой производительности:**
```bash
acks=1
batch.size=32768
linger.ms=10
compression.type=lz4
```

✅ **Для надежной доставки (финансовые данные, критичные события):**
```bash
acks=all
enable.idempotence=true
max.in.flight.requests.per.connection=1
retries=5
```

✅ **Для exactly-once semantics (EOS):**
```bash
enable.idempotence=true
transactional.id=my-tx-id
```

✅ **Для дебага:**  
Используйте `ProducerConfig.configDef()` или включайте логирование на уровне `DEBUG` для пакета `org.apache.kafka.clients.producer`.

## Пример
```java
import org.apache.kafka.clients.producer.*;
import java.util.Properties;

public class ProducerConfigExample {
    public static void main(String[] args) {
        // ⭐⭐Создание конфигурации продюсера
        Properties props = new Properties();
        props.put(
	        ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, 
	        "localhost:9092");
        props.put(
	        ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, "org.apache.kafka.common.serialization.StringSerializer");
        props.put(
	        ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, "org.apache.kafka.common.serialization.StringSerializer");
        
        props.put(ProducerConfig.ACKS_CONFIG, "all");
        props.put(ProducerConfig.LINGER_MS_CONFIG, "10");
        props.put(ProducerConfig.BATCH_SIZE_CONFIG, "32768");

        // Создание продюсера
        try (KafkaProducer<String, String> producer = new KafkaProducer<>(props)) {
            ProducerRecord<String, String> record =
                    new ProducerRecord<>(
	                    "orders", 
	                    "user123", 
	                    "Order #1001 completed");

            // Асинхронная отправка
            producer.send(record, (metadata, exception) -> {
                if (exception == null) {
                    System.out.printf(
	                    "Сообщение записано в %s-%d с"
	                    + "offset=%d%n",
                        metadata.topic(), 
                        metadata.partition(), 
                        metadata.offset());
                } else {
                    exception.printStackTrace();
                }
            });
        }
    }
}
```
