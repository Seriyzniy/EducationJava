# Базовые классы и интерфейсы

| Класс / Интерфейс      | Описание                                                                                                                                                  |
| ---------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `Producer<K,V>`✅       | Интерфейс Kafka-продюсера, определяющий основные методы для отправки записей (`send`), закрытия соединения (`close`) и управления транзакциями.           |
| `KafkaProducer<K,V>`✅  | Основная реализация интерфейса `Producer`, используется для отправки сообщений (records) в Kafka. Управляет буферами, сериализацией и сетевыми запросами. |
| `ProducerRecord<K,V>`✅ | Объект, представляющий запись, отправляемую в Kafka — содержит тему (`topic`), ключ (`key`), значение (`value`) и необязательный номер партиции.          |
| `RecordMetadata`✅      | Метаданные, возвращаемые брокером после успешной отправки записи. Включает `topic`, `partition`, `offset` и `timestamp`.                                  |

---

# Конфигурация и настройки

| Класс / Интерфейс | Описание                                                                                                                    |
| ----------------- | --------------------------------------------------------------------------------------------------------------------------- |
| `ProducerConfig`✅ | Класс конфигурации продюсера. Определяет все настройки (bootstrap.servers, acks, retries, buffer.memory, linger.ms и т.д.). |

---

# Партиционирование

|Класс / Интерфейс|Описание|
|---|---|
|`Partitioner`|Интерфейс, определяющий стратегию выбора партиции для записи на основе ключа, значения или других факторов.|
|`RoundRobinPartitioner`|Реализация `Partitioner`, распределяющая записи по партициям циклически (равномерно).|

---

# Интерцепторы и обратные вызовы

| Класс / Интерфейс          | Описание                                                                                                                                             |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| `Callback`✅                | Интерфейс обратного вызова, вызывается после завершения отправки сообщения (успешно или с ошибкой). Используется для логирования и обработки ошибок. |
| `ProducerInterceptor<K,V>` | Интерфейс-плагин, позволяющий ==перехватывать и изменять записи перед== их отправкой в кластер, а также реагировать на подтверждение отправки.       |

---

# Тестирование и эмуляция

|Класс / Интерфейс|Описание|
|---|---|
|`MockProducer<K,V>`|Тестовая реализация `Producer`, не взаимодействует с настоящим кластером Kafka, а сохраняет отправленные записи в памяти. Используется для unit-тестов.|

---

# Транзакции

|Класс / Интерфейс|Описание|
|---|---|
|`PreparedTxnState`|Класс, содержащий состояние транзакции после подготовки к двухфазному коммиту (Two-Phase Commit). Используется внутри механизма транзакций Kafka.|

---

# ⚠️Исключения

|Класс / Интерфейс|Описание|
|---|---|
|`BufferExhaustedException`|Исключение, выбрасываемое, если продюсер не может выделить память для записи в течение времени, указанного в `max.block.ms`, из-за переполнения буфера.|
