
> [!NOTE] `KafkaStreams` - главный класс в Kafka Streams API.
> **Класс `KafkaStreams`** — это основной клиент Kafka Streams API. 
> 
> Он связывает **описанную топологию (`Topology`)** с **реальной обработкой данных**: 
> - запускает потоки, 
> - распределяет задачи по потокам (stream threads),
> - отслеживает состояние
> - управляет жизненным циклом приложения.

##### Реализует интерфейсы
```java
public class KafkaStreams implements AutoCloseable
```
👉 `AutoCloseable` — для корректного закрытия ресурсов (`close()` в try-with-resources).  

Интерфейс `AutoCloseable` реализован, чтобы гарантировать, что Kafka Streams закроет все свои внутренние consumers, producers и state stores.

##### Назначение
`KafkaStreams`:
- управляет потоковой обработкой, основанной на `Topology` или `StreamsBuilder`;
    
- запускает обработку данных из Kafka-топиков в реальном времени;
    
- поддерживает отказоустойчивость и восстановление состояния;
    
- управляет внутренними потоками (`StreamThread`);
    
- предоставляет API для интерактивных запросов (interactive queries);
    
- позволяет масштабировать приложение горизонтально.

## Конструктор
```java
// Создаёт новый Kafka Streams экземпляр, используя заданную топологию и конфигурацию.
// topology — граф обработки данных (узлы и связи между ними).
// props — конфигурация Streams (StreamsConfig или обычные Properties).
public KafkaStreams(Topology topology, Properties props)


// Создаёт новый Kafka Streams экземпляр с пользовательским поставщиком клиентов Kafka.
// topology — граф обработки данных.
// props — конфигурация Streams.
// clientSupplier — поставщик клиентов Kafka (producers, consumers, admin), если нужно кастомизировать поведение.
public KafkaStreams(
	Topology topology, 
	Properties props, 
	KafkaClientSupplier clientSupplier)
```
💬 Обычно используется первый конструктор.  
Второй применяется, если нужно, например, подменить клиента Kafka для тестов или мониторинга.

## Методы
### Управление жизненным циклом
```java
// ❗❗❗Запускает обработку потоков.❗❗❗  
// 💡После вызова start() Kafka Streams начинает читать данные из входных топиков и выполнять топологию.
public void start()

// Асинхронно закрывает приложение и останавливает все потоки.
// Обычно вызывается при завершении работы приложения.
public void close()

// Закрывает приложение с дополнительными параметрами закрытия.
// options — объект CloseOptions, позволяющий задать таймаут и поведение при остановке.
public void close(KafkaStreams.CloseOptions options)



// Возвращает текущее состояние Kafka Streams (CREATED, RUNNING, REBALANCING, ERROR и т. д.).
public KafkaStreams.State state()

// Регистрирует слушателя для отслеживания изменений состояния приложения.
// listener — объект, реализующий интерфейс KafkaStreams.StateListener.
public void setStateListener(KafkaStreams.StateListener listener)
```

### Метаданные и информация о задачах
```java
// 💡Возвращает информацию о всех потоковых задачах (tasks) текущего приложения.
public Set<TaskMetadata> metadataForLocalThreads()

// Возвращает полные метаданные для всех задач (во всех инстансах приложения).
public Collection<StreamsMetadata> allMetadata()

// Возвращает метаданные для задач, обрабатывающих указанный ключ.
// key — ключ для поиска.
// serializer — сериализатор ключа.
public <K> KeyQueryMetadata queryMetadataForKey(
	String storeName, 
	K key, 
	Serializer<K> serializer)
```

### Работа с хранилищами состояния (State Stores)
```java
// Позволяет получить доступ к локальному хранилищу состояния (State Store).
// storeQueryParameters — параметры запроса (тип и имя стора).
public <T> T store(StoreQueryParameters<T> storeQueryParameters)
```
⚠️ Работает только, если стора существует локально.  
Для запросов к удалённым сторам нужно использовать `queryMetadataForKey`.

### Мониторинг и отладка
```java
// Возвращает объект StreamsMetrics для сбора пользовательских метрик.
public StreamsMetrics metrics()

// Возвращает уникальный идентификатор инстанса Kafka Streams (используется для распределения задач).
public ClientInstanceIds clientInstanceIds()
```

### Сброс состояния
```java
// Сбрасывает локальное состояние стора.  
// Используется редко, обычно при восстановлении после критических ошибок.
public void cleanUp()
```

## 💡 Особенности и подводные камни 💡

1️⃣ ❗❗❗**Одно приложение — одна топология**  
`KafkaStreams` управляет одной `Topology`. Если нужно запустить несколько топологий — создавай отдельные экземпляры `KafkaStreams`.

2️⃣ **Потокобезопасность**  
Только некоторые методы (например, `state()`) потокобезопасны. Нельзя одновременно менять состояние приложения и обращаться к стору без синхронизации.

3️⃣ **State Stores — локальные!**  
Если запросить данные из стора, который находится в другом инстансе приложения, ты не получишь результат — нужно обращаться через интерактивный запрос (interactive query routing).

4️⃣ ❗❗❗**Обязательно закрывай приложение**  
Если не вызвать `close()`, consumers и producers не будут корректно завершены, и Kafka будет считать инстанс "живым".

5️⃣ **Реакция на ошибки**  
Если внутри топологии произойдёт ошибка сериализации или записи, приложение может перейти в состояние `ERROR`. Для этого нужно обработать ошибки через `setUncaughtExceptionHandler`.

---
## 💎 Best Practices 💎

✅ ⭐⭐⭐**Используй `StreamsBuilder` для построения топологии**, а не вручную создавай `Topology`.  

✅ **Регулярно вызывай `close()` при завершении работы** — например, через shutdown hook. 

✅ **Настраивай `commit.interval.ms` и `processing.guarantee`** для балансировки между производительностью и надёжностью.  

✅ **Используй `stateListener` и `uncaughtExceptionHandler`** для отслеживания проблем.  

✅ 💡**Мониторь метрики через JMX или API `StreamsMetrics`**. 

✅ **Для интеграционных тестов** — применяй `TopologyTestDriver`, а не реальную Kafka.

## Пример
```java
import org.apache.kafka.streams.*;
import org.apache.kafka.streams.kstream.*;
import java.util.Properties;

public class WordCountApp {
    public static void main(String[] args) {
        Properties props = new Properties();
        props.put(
	        StreamsConfig.APPLICATION_ID_CONFIG, 
	        "wordcount-app");
	        
        props.put(
	        StreamsConfig.BOOTSTRAP_SERVERS_CONFIG, 
	        "localhost:9092");
	        
        props.put(
	        StreamsConfig.DEFAULT_KEY_SERDE_CLASS_CONFIG,
	        Serdes.String().getClass());
        props.put(
	        StreamsConfig.DEFAULT_VALUE_SERDE_CLASS_CONFIG, 
	        Serdes.String().getClass());



        StreamsBuilder builder = new StreamsBuilder();

        KStream<String, String> textLines = builder
	        .stream("input-topic");

        textLines
            .flatMapValues(value -> List.of(value.toLowerCase().split(" ")))
            .groupBy((key, word) -> word)
            .count()
            .toStream()
            .to("output-topic", Produced.with(Serdes.String(), Serdes.Long()));


        KafkaStreams streams = new KafkaStreams(
	        builder.build(), 
	        props);

        // Регистрация корректного завершения
        Runtime.getRuntime().addShutdownHook(new Thread(streams::close));
        streams.start();
    }
}
```
