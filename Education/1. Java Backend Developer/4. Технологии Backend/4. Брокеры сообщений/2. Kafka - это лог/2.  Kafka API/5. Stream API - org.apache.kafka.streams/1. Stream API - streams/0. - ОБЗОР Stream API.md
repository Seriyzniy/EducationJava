### Сводка по используемым классам
Из всей огромной кучи классов, которые есть в пакете `org.apache.kafka.streams`, **в реальной работе ты напрямую используешь только 3**:

|Класс|Используется вручную|Назначение|
|---|---|---|
|✅ `StreamsBuilder`|**Да, всегда**|Построение топологии|
|✅ `StreamsConfig`|**Да, всегда**|Настройка приложения|
|✅ `KafkaStreams`|**Да, всегда**|Запуск и управление обработкой|
|⚙️ `Topology`|Иногда|При низкоуровневом доступе (Processor API)|
|⚙️ `TopologyTestDriver`|Только в тестах|Тестирование топологий|
|❌ Остальные|Нет|Внутренние или служебные классы|

## Основные классы/интерфейсы
| Класс / Интерфейс     | Описание                                                                                                                  |
| --------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| `KafkaStreams`✅       | Клиент Kafka Streams, выполняющий непрерывную обработку данных из входных топиков и запись результатов в выходные топики. |
| `StreamsBuilder`✅     | Высокоуровневый DSL для построения топологии потоков данных (графа обработки).                                            |
| `StreamsConfig`✅      | Конфигурация экземпляра KafkaStreams (application.id, bootstrap.servers, и т.д.).                                         |
| `Topology`            | Логическое представление топологии — граф процессоров (узлов обработки данных).                                           |
| `TopologyDescription` | Метаданные, описывающие структуру топологии в читаемом виде.                                                              |

## Конфигурация и управление
Класс / Интерфейс | Описание
------------------|----------
`AutoOffsetReset` | Определяет политику auto.offset.reset при добавлении источников или создании KStream/KTable через StreamsBuilder.
`KafkaClientSupplier` | Позволяет предоставлять собственные Kafka-клиенты (продюсер, консьюмер, админ) для экземпляра KafkaStreams.
`StreamsConfig.InternalConfig` | Внутренние параметры конфигурации, не предназначенные для прямого использования.
`TopologyConfig` | Конфигурационные параметры, применяемые на уровне отдельной топологии.
`TopologyConfig.TaskConfig` | Конфигурация, применяемая на уровне отдельных задач внутри топологии.
`KafkaStreams.CloseOptions` | Опции для корректного завершения или масштабирования вниз экземпляра KafkaStreams.

## Состояние и жизненный цикл
Класс / Интерфейс | Описание
------------------|----------
`KafkaStreams.State` | Возможные состояния экземпляра Kafka Streams (CREATED, RUNNING, REBALANCING, ERROR и т.д.).
`KafkaStreams.StateListener` | Слушатель событий изменения состояния Kafka Streams (переходы RUNNING → REBALANCING и т.д.).

## Метаданные и мониторинг
Класс / Интерфейс | Описание
------------------|----------
`StreamsMetadata` | Метаданные клиента Kafka Streams (информация о задачах и хранилищах).
`StreamsMetrics` | Интерфейс для добавления пользовательских метрик и доступа к системным метрикам Kafka Streams.
`ClientInstanceIds` | Инкапсулирует идентификаторы клиентских экземпляров (для метрик продюсеров, консьюмеров и админ-клиентов).
`KeyQueryMetadata` | Метаданные о расположении ключа (в каком инстансе и партиции он хранится).
`LagInfo` | Содержит информацию о лаге (отставании) у активных или резервных реплик стора.
`TaskMetadata` | Метаданные о задаче (task) — идентификатор, топики, состояние, позиция и т.п.
`ThreadMetadata` | Метаданные о потоке выполнения Kafka Streams (stream thread).

## Интерактивные запросы и вспомогательные структуры
Класс / Интерфейс | Описание
------------------|----------
`KeyValue<K,V>` | Пара ключ–значение, используемая для представления записей в потоках Kafka Streams.
`StoreQueryParameters<T>` | Параметры запроса к стору при выполнении интерактивных запросов (store(...)).

## Тестирование
Класс / Интерфейс | Описание
------------------|----------
`TopologyTestDriver` | Класс для тестирования топологий, созданных с помощью Topology или StreamsBuilder, без запуска реального Kafka-кластера.
`TestInputTopic<K,V>` | Используется для подачи тестовых записей во входной топик внутри TopologyTestDriver.
`TestOutputTopic<K,V>` | Используется для чтения выходных записей из топика внутри TopologyTestDriver.

## Описание топологии (узлы)
| Класс / Интерфейс                 | Описание                                                                       |
| --------------------------------- | ------------------------------------------------------------------------------ |
| `TopologyDescription.Node`        | Базовый интерфейс, представляющий узел в топологии.                            |
| `TopologyDescription.Source`      | Узел-источник данных (читает из Kafka topic).                                  |
| `TopologyDescription.Processor`   | Узел-обработчик, применяющий функции преобразования данных.                    |
| `TopologyDescription.Sink`        | Узел-приёмник данных, отправляющий результат в Kafka topic.                    |
| `TopologyDescription.Subtopology` | Подграф, представляющий взаимосвязанные узлы одной топологии.                  |
| `TopologyDescription.GlobalStore` | Узел, представляющий глобальный стор (доступный всем инстансам Kafka Streams). |
