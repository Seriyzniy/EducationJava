`org.apache.kafka.streams.kstream` — это **сердце DSL Kafka Streams**, где определяются все основные типы потоков (`KStream`, `KTable`, `GlobalKTable`) и функциональные интерфейсы для трансформаций, агрегаций, окон, объединений и т.д.

# 1. Основные классы и интерфейсы

| Класс / Интерфейс   | Описание                                                                                                                  |
| ------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| `KStream<K,V>`✅     | Поток записей (key–value). Каждая запись — независимое событие. Основной тип для работы с потоковыми данными. ✅           |
| `KTable<K,V>`✅      | Табличное представление (changelog stream) — хранит последнее состояние по ключу. Используется для join’ов и агрегатов. ✅ |
| `GlobalKTable<K,V>` | Глобальная таблица, реплицируемая на все инстансы приложения. Применяется для join без партиционирования. ✅               |

> 💡 Эти три класса — **основа DSL**.  
> Почти всё в Kafka Streams строится вокруг них.

---

# 2. Конфигурация и вспомогательные параметры

| Класс / Интерфейс                        | Описание                                                                                                |
| ---------------------------------------- | ------------------------------------------------------------------------------------------------------- |
| `Consumed<K,V>`💡                        | Опции при создании `KStream`, `KTable`, `GlobalKTable`: `Serde`, `TimestampExtractor`, `resetPolicy`. ✅ |
| `Produced<K,V>`                          | Опции при выводе данных в Kafka через `to()`: `Serde`, `Partitioner`, `topicName`. ✅                    |
| `Grouped<K,V>`                           | Опции при `groupBy`, `groupByKey` — задаёт Serde и имя внутреннего репартиционного топика. ✅            |
| `Joined<K,VLeft,VRight>`                 | Опции для join операций (Kafka Streams выбирает внутренние сторы и топики). ✅                           |
| `StreamJoined<K,V1,V2>`                  | Настройки join’ов между двумя потоками (`KStream–KStream`). ✅                                           |
| `TableJoined<K,KO>`                      | Опции для foreign-key join между таблицами (`KTable–KTable`).                                           |
| `Repartitioned<K,V>`                     | Настройки для внутреннего переразбиения (repartition topic).                                            |
| `Materialized<K,V,S extends StateStore>` | Указывает, как и где хранить состояние (например, `Materialized.as("store-name")`). ✅                   |
| `Named`💡                                | Позволяет задать пользовательские имена процессоров внутри топологии (для отладки и мониторинга). ✅     |

> 💡 Все эти классы — **DSL-конфигурационные обёртки**, часто применяются напрямую в коде.  
> Например:

```java
stream.groupByKey(
	Grouped.with(Serdes.String(), 
	Serdes.Long())
).count(Materialized.as("count-store"));
```

# 3. Агрегации и редукции

|Класс / Интерфейс|Описание|
|---|---|
|`Aggregator<K,V,VAgg>`|Интерфейс для агрегирования значений (используется в `aggregate()`). ✅|
|`Initializer<VAgg>`|Создаёт начальное значение для агрегатов. ✅|
|`Reducer<V>`|Функция для объединения двух значений в одно (`reduce()`). ✅|
|`Merger<K,V>`|Объединяет результаты агрегатов для `SessionWindows`. ✅|

> 💡 Применяются при `count()`, `aggregate()`, `reduce()` и т.п.  
> Обычно передаются через **лямбды**, но могут быть и отдельными классами.

# 4. Оконные операции (Windows)

|Класс / Интерфейс|Описание|
|---|---|
|`Windows<W extends Window>`|Базовый интерфейс спецификаций окон (определяет границы и grace period). ✅|
|`TimeWindows`|Фиксированные окна (например, каждые 5 минут). ✅|
|`SlidingWindows`|Скользящие окна (перекрывающиеся интервалы). ✅|
|`SessionWindows`|Сессионные окна, определяются активностью событий. ✅|
|`UnlimitedWindows`|Бесконечное окно, без ограничений по времени.|
|`JoinWindows`|Определяет временные рамки для stream–stream join. ✅|
|`Window`|Представляет отдельное окно с временем начала и конца.|
|`Windowed<K>`|Ключ с привязкой к окну (`Windowed<String>`). ✅|
|`SessionWindowedKStream<K,V>`|KStream, сгруппированный по ключу и разделённый на сессии. ✅|
|`TimeWindowedKStream<K,V>`|KStream, сгруппированный по временным окнам. ✅|
|`SessionWindowedCogroupedKStream<K,V>`|То же самое, но для нескольких KStreams.|
|`TimeWindowedCogroupedKStream<K,V>`|Cogroup с временными окнами.|
|`WindowedSerdes`|Serdes для сериализации ключей с окнами. ✅|
|`WindowedSerdes.TimeWindowedSerde<T>`|Serde для `TimeWindows`.|
|`WindowedSerdes.SessionWindowedSerde<T>`|Serde для `SessionWindows`.|

> 💡 Эти классы активно применяются при агрегациях во временных рамках, например:

```java
stream
	.groupByKey()       
	.windowedBy(
		TimeWindows.ofSizeWithNoGrace(Duration.ofMinutes(5))
	).count();
```

# 5. Маппинг, фильтрация, действия

|Класс / Интерфейс|Описание|
|---|---|
|`KeyValueMapper<K,V,VR>`|Функция `(key, value) -> newValue`, используется в `map()`, `groupBy()`. ✅|
|`ValueMapper<V,VR>`|Преобразует значение (без ключа). ✅|
|`ValueMapperWithKey<K,V,VR>`|Преобразует значение с доступом к ключу. ✅|
|`Predicate<K,V>`|Булев фильтр (используется в `filter()`, `branch()`). ✅|
|`ForeachAction<K,V>`|Действие для `foreach()`. ✅|
|`ForeachProcessor<K,V>`|Устарел — больше не используется. 🚫|
|`ValueTransformerWithKey<K,V,VR>`|Stateful mapping с доступом к состоянию. ✅|
|`ValueTransformerWithKeySupplier<K,V,VR>`|Создаёт экземпляры `ValueTransformerWithKey`. ✅|

> 💡 Реально используемые: `KeyValueMapper`, `Predicate`, `ValueMapper`, `ForeachAction`.  
> Stateful-трансформеры применяются, если нужно доступаться к состоянию стора.

# 6. Join’ы и CoGroup’ы

|Класс / Интерфейс|Описание|
|---|---|
|`ValueJoiner<V1,V2,VR>`|Определяет, как объединяются значения при join. ✅|
|`ValueJoinerWithKey<K,V1,V2,VR>`|То же, но с доступом к ключу. ✅|
|`CogroupedKStream<K,VOut>`|Для объединения нескольких KStream с общей логикой агрегации. ✅|
|`BranchedKStream<K,V>`|Для разветвления потока (branching). ✅|
|`Branched<K,V>`|Опции для отдельных ветвей `branch()`. ✅|

> 💡 Применяются в:

 ```java
 stream
	 .leftJoin(table, (value1, value2) -> ...)

stream.branch((k,v) -> v > 10, (k,v) -> true)
 ```

# 7. Продвинутые механизмы и буферизация

|Класс / Интерфейс|Описание|
|---|---|
|`EmitStrategy`|Контролирует стратегию выдачи результатов в процессоре.|
|`Suppressed<K>`|Позволяет подавлять промежуточные результаты в окнах (`suppress()`). ✅|
|`Suppressed.BufferConfig`|Конфигурация буфера подавления.|
|`Suppressed.EagerBufferConfig`|Быстрая, но нестрогая конфигурация (частично может дублировать результаты).|
|`Suppressed.StrictBufferConfig`|Строгая, полностью соблюдает ограничения по времени.|

> 💡 Используется в оконных агрегациях для управления "всплытием" результатов:

```java
.suppress(
	Suppressed.untilWindowCloses(
		Suppressed.BufferConfig.unbounded()
	)
)
```

# 8. Сериализация и десериализация

|Класс / Интерфейс|Описание|
|---|---|
|`TimeWindowedSerializer<T>`|Сериализатор ключей с временными окнами.|
|`TimeWindowedDeserializer<T>`|Десериализатор ключей с временными окнами.|
|`SessionWindowedSerializer<T>`|Сериализатор для сессионных окон.|
|`SessionWindowedDeserializer<T>`|Десериализатор для сессионных окон.|

> 💡 Эти классы редко используются напрямую — Kafka Streams применяет их **внутренне** при работе с оконными сторями.
