**Каждая ветка `branch()` НЕ СОЗДАЕТ отдельный Java-поток.**  
Все ветки внутри одной топологии выполняются **внутри тех же потоков**, которые обрабатывают исходный `KStream`.

Удобный вариант ветвления, когда нужно РАЗДЕЛЯТЬ поток данных на разные потоки обработки, В ЗАВИСИМОСТИ от значения какого-то объекта потока

# Простое слияние
![[Pasted image 20251110125804.png]]
Два ПОТОКА МОЖНО ОБЪЕДИНИТЬ,
ЕСЛИ они ОДИНАКОВО ТИПИЗИРОВАНЫ 
```java
KStream<String, Integer> foo = ...
KStream<String, Integer> bar = ...

KStream<String, Integer> merge = foo.merge(bar)
```
В данном случае потоки одинаково типизированы `KStream<String, Integer>`

> [!NOTE] ⭐ ПОРЯДОК не гарантируется ⭐
> Нет никаких гарантий как именно элементы двух потоков будут объединены
