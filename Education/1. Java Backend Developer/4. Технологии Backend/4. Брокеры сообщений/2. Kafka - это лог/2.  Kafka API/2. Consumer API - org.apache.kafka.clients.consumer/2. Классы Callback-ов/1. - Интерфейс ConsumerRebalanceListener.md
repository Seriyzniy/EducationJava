
> [!NOTE] `ConsumerRebalanceListener`
> **интерфейс** из пакета `org.apache.kafka.clients.consumer`,  
> который позволяет ==выполнять пользовательские действия== **при изменении распределения партиций (ребалансировке)** в группе потребителей.

Он используется совместно с методом:
```java
consumer.subscribe(
	Collections.singletonList("my-topic"), 
	new MyRebalanceListener());
```

### Что такое ребалансировка и когда она происходит
==Ребалансировка (rebalance)== — это **процесс перераспределения партиций** между потребителями **в одной consumer-группе**.  

Kafka стремится, чтобы каждая партиция обрабатывалась **только одним** consumer-ом в группе,  
поэтому при изменении состава группы (или метаданных топика) брокер **координатор группы** запускает процесс перераспределения.

Ребалансировка происходит, когда:  
1️⃣ В группу добавляется новый consumer.  
2️⃣ Какой-то consumer отключился или "завис" (heartbeat не поступает).  
3️⃣ Изменилось число партиций в топике.  
4️⃣ Consumer изменил список подписанных топиков (`subscribe()` с другими темами).

Во время ребалансировки все участники **временно приостанавливают обработку сообщений**,  
и **получают новый список партиций**, за которые теперь отвечают.

## Интерфейс
Реализация выполняется через пользовательский класс, реализующий его методы:
```java
public class MyRebalanceListener 
	implements ConsumerRebalanceListener 
{
    // Реализация методов интерфейса
}
```

## Методы
Интерфейс `ConsumerRebalanceListener` содержит **два метода**, которые вызываются Kafka при изменении распределения партиций
### Колбэки ребалансировки
```java
// ❗❗Вызывается 💡ПЕРЕД💡 тем, как партиции будут отозваны❗❗ у текущего consumer-а во время ребалансировки.
// Обычно здесь выполняют commit оффсетов для безопасного завершения обработки.
void onPartitionsRevoked(Collection<TopicPartition> partitions);

// ❗❗Вызывается 💡ПОСЛЕ💡 того, как consumer получил новые партиции❗❗ после ребалансировки.
// Обычно здесь восстанавливают состояние (например, позицию seek) или читают оффсеты из БД.
void onPartitionsAssigned(Collection<TopicPartition> partitions);
```

---
## 💡 Особенности и подводные камни 💡

1️⃣ 💡**Ребалансировка = пауза чтения.**  
Пока происходит ребалансировка, метод `poll()` НЕ возвращает записи.  
Это может привести к временной остановке обработки.

2️⃣ ❗❗❗**onPartitionsRevoked() — важен для commit.**  
Если consumer работает с ручной фиксацией (`enable.auto.commit=false`),  
то **необходимо вручную коммитить оффсеты** внутри `onPartitionsRevoked()`.  
Иначе вы рискуете потерять или повторно обработать сообщения.
(Так как некоторые партиции могут уйти от Consumer, то необходимо обозначить, какие сообщения уже обработаны)

3️⃣ **onPartitionsAssigned() — важен для восстановления.**  
Здесь обычно выполняют `seek()` к нужным оффсетам (например, из внешнего хранилища, если нужно восстановление состояния).

4️⃣ ❗❗❗**Асинхронный commit внутри ребалансировки — плохая идея.**  
Используйте **синхронный commit**, т.к. асинхронный может не успеть завершиться до конца ребалансировки.

5️⃣ **Следи за временем выполнения методов.**  
Эти методы вызываются в основном потоке клиента — не блокируй их долгими операциями (вроде запросов к БД без таймаута).

---
## 💎 Best Practices 💎

✅ ❗❗❗**Сохраняй оффсеты при отзыве партиций (`onPartitionsRevoked`).**  

✅ **Восстанавливай состояние (`seek()`) при назначении новых партиций (`onPartitionsAssigned`).**  
 
✅ **Минимизируй логику внутри методов — только commit/seek, ничего лишнего.**  

✅ **Логируй события ребалансировки**, чтобы отслеживать стабильность работы группы.

## Пример
```java
import org.apache.kafka.clients.consumer.*;
import org.apache.kafka.common.TopicPartition;
import java.util.*;

public class MyRebalanceListener 
	implements ConsumerRebalanceListener 
{
    private final KafkaConsumer<String, String> consumer;

    public MyRebalanceListener(
	    KafkaConsumer<String, String> consumer) 
	{
        this.consumer = consumer;
    }

    @Override
    public void onPartitionsRevoked(Collection<TopicPartition> partitions) {
        // Вызывается перед отзывом партиций
        System.out.println(
	        "Ребалансировка началась. Партиции будут отозваны: " 
	        + partitions);
	          
        // Сохраняем оффсеты для безопасного завершения обработки
        consumer.commitSync();
    }

    @Override
    public void onPartitionsAssigned(Collection<TopicPartition> partitions) {
        // Вызывается после назначения новых партиций
        System.out.println(
	        "Назначены новые партиции: " + partitions
	    );
	    
        // ❗❗❗Восстанавливаем позицию чтения (если нужно)
        for (TopicPartition partition : partitions) {
	        //пример: загрузка из внешнего хранилища
            long offset = loadOffsetFromDB(partition); 
             
            consumer.seek(partition, offset);
        }
    }

    private long loadOffsetFromDB(TopicPartition partition) {
        // фиктивный пример
        return 0L;
    }
}

// Использование
KafkaConsumer<String, String> consumer = new KafkaConsumer<>(props);

consumer.subscribe(
	Collections.singletonList("orders"), 
	new MyRebalanceListener(consumer));
```
