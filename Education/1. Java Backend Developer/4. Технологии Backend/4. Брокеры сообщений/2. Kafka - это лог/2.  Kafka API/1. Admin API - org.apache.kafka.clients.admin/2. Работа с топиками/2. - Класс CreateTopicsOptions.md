
> [!NOTE] `CreateTopicsOptions`
> **final класс**, унаследованный от `AbstractOptions<CreateTopicsOptions>`.  
> 
> Он предназначен для настройки параметров при вызове 
> ```java
> Admin.createTopics(
> 	Collection<NewTopic>, 
> 	CreateTopicsOptions
> )
> ```

Позволяет указать:
- Таймаут операции (`timeoutMs`);
    
- Поведение при попытке создать уже существующие топики (`validateOnly`, `retryOnQuotaViolation`);
    
- Нужно ли только проверить корректность параметров, не создавая топики на самом деле.

---
## Конструктор
```java
// Создает новый объект настроек для операции создания топиков.
// Конструктор без параметров — все настройки по умолчанию.
public CreateTopicsOptions() { }
```

## Методы
Класс **унаследует** часть базовых методов из `AbstractOptions` (например, `timeoutMs()`, `timeoutMs(Integer timeoutMs)`), но также добавляет **специализированные методы для управления логикой создания тем**.

### Настройка режима "только валидация"
```java
// Устанавливает флаг: только проверить параметры без создания тем.
// Если true — проверяется корректность NewTopic, но топики не создаются.
public CreateTopicsOptions validateOnly(boolean validateOnly)

// Возвращает true, если включён режим только валидации.
public boolean validateOnly()
```

### Обработка ситуации, когда топик уже существует
```java
// Устанавливает флаг: если топик уже существует — не выбрасывать ошибку, а ❗❗просто игнорировать.❗❗
// По умолчанию — false, т.е. ошибка будет выброшена.
public CreateTopicsOptions ignoreIfExists(boolean ignoreIfExists)

// Проверяет, установлен ли режим игнорирования существующих топиков.
public boolean shouldIgnoreIfExists()
```

### Поведение при превышении квоты
```java
// Устанавливает поведение: если превышена квота (quota violation), повторить попытку.
// Если true — клиент может автоматически повторить операцию после превышения квоты.
public CreateTopicsOptions retryOnQuotaViolation(
	boolean retryOnQuotaViolation)

// Проверяет, включено ли повторение операции при нарушении квоты.
public boolean shouldRetryOnQuotaViolation()
```

### Таймаут операции
```java
// Устанавливает таймаут (в миллисекундах) для операции создания топиков.
@Override
public CreateTopicsOptions timeoutMs(Integer timeoutMs)

// Возвращает текущий таймаут операции.
@Override
public Integer timeoutMs()
```

## 💡 Особенности и подводные камни 💡

1️⃣ 💡`CreateTopicsOptions` **не создаёт топики сам по себе** — он лишь конфигурирует поведение💡 `AdminClient#createTopics()`

2️⃣ Если `validateOnly(true)`, Kafka **не создаёт** топики, даже если всё корректно — это полезно для dry-run.  

3️⃣ При `ignoreIfExists(true)` операция не будет проваливаться при дублирующемся топике, ❗❗но **ошибка всё же может возникнуть** при конфликте конфигурации.❗❗  

4️⃣ Метод `retryOnQuotaViolation(true)` ❗❗может привести к дополнительным задержкам, если брокеры перегружены. ❗❗

5️⃣ ❗❗Параметр `timeoutMs()` управляет **временем ожидания ответа от брокеров**❗❗, а не временем создания самих топиков на всех узлах.

---
## 💎 Best Practices 💎

✅ 💡Используй `validateOnly(true)` перед массовым созданием топиков — это поможет выявить ошибки в конфигурации заранее.  

✅ 💡Для CI/CD или автоматизированных систем лучше включать `ignoreIfExists(true)` — 💡чтобы не падать при повторном запуске деплоя.💡

✅ Явно задавай `timeoutMs()` в зависимости от размера кластера (по умолчанию может быть слишком мал).  

✅ ❗❗Не включай `retryOnQuotaViolation(true)` без необходимости — это может скрывать проблемы с настройками квот.

## Пример
```java
import org.apache.kafka.clients.admin.*;

import java.util.List;
import java.util.Properties;

public class CreateTopicExample {
    public static void main(String[] args) throws Exception {
        Properties props = new Properties();
        props.put(
	        AdminClientConfig.BOOTSTRAP_SERVERS_CONFIG, 
	        "localhost:9092");

        try (AdminClient admin = AdminClient.create(props)) {
            // Описание новой темы
            NewTopic newTopic = new NewTopic(
	            "events", 
	            3, 
	            (short) 2
	        );

            // Настройки операции
            CreateTopicsOptions options = new CreateTopicsOptions()
                    .validateOnly(false)
                    .ignoreIfExists(true)
                    .timeoutMs(10_000);

            // Создание топиков
            admin.createTopics(
	            List.of(newTopic), 
	            options
	        ).all().get();

            System.out.println(
	            "Топик успешно создан (или уже существовал)."
	        );
        }
    }
}
```
