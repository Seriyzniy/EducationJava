## 2. Класс конфигурации Producer

```java
package producer.config;  
  
import model.CustomHeader;  
import model.UserInfo;  
import org.apache.kafka.clients.producer.ProducerConfig;  
import org.apache.kafka.common.header.Headers;  
import org.apache.kafka.common.serialization.StringSerializer;

import org.springframework.context.annotation.Configuration;

import org.springframework.kafka.core.KafkaTemplate;  
import org.springframework.kafka.core.ProducerFactory;  
import org.springframework.kafka.support.KafkaHeaderMapper;  
import org.springframework.kafka.support.converter.MessagingMessageConverter;  
import org.springframework.messaging.MessageHeaders;  
import util.CustomUserInfoSerializer;  
  
import java.util.HashMap;  
import java.util.Map;  
  
@Configuration  
public class KafkaProducerConfig {  
    private Map<String, Object> producerConfig(){  
        Map<String, Object> props = new HashMap<String, Object>();  
        props.put(  
            ProducerConfig.BOOTSTRAP_SERVERS_CONFIG,  
			"localhost:29092,localhost:39092,localhost:49092"  
        );  
        props.put(  
            ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG,  
            StringSerializer.class  
        );  
        // 🔥Кастомный сериализатор сообщения
        props.put(  
            ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG,  
            CustomUserInfoSerializer.class  
        );  
        return props;  
    }  
  
    @Bean  
    public ProducerFactory<String, UserInfo> producerFactory(){  
        return 
	        new DefaultKafkaProducerFactory<>(producerConfig());  
    }  
  
    @Bean  
    public KafkaTemplate<String, UserInfo> kafkaTemplate(  
            ProducerFactory<String, UserInfo> producerFactory){  
  
        KafkaTemplate<String, UserInfo> template = 
	        new KafkaTemplate<>(producerFactory);  
  
        MessagingMessageConverter converter = 
	        new MessagingMessageConverter(); 
	        
	    // Описание класса, 🔥для конвертации заголовков🔥 
        converter.setHeaderMapper(new KafkaHeaderMapper() {  
            @Override  
            public void fromHeaders(
	            MessageHeaders headers, 
	            Headers target)

				headers.entrySet()

	                    entry.getKey(), 
	                    entry.getValue()
		                    .toString()
		                    .getBytes());

            }  
  
            @Override  
            public void toHeaders(
	            Headers source, 
	            Map<String, Object> target) { }

        template.setMessageConverter(converter);  
  
        return template;  
    }  
}
```

### Конвертация заголовков Spring Kafka -> Apache Kafka
Так как мы при отправке сообщения будем ОТПРАВЛЯТЬ КАСТОМНЫЙ СЛОЖНЫЙ ЗАГОЛОВОК, то там необходимо указать, **как КОНВЕРТИРОВАТЬ заголовки из MessageHeaders** (используемые в MessageBuilder) **в заголовки Headers Apache Kafka**

Для этого в Spring Kafka есть интерфейс KafkaHeaderMapper
(см [Message Headers :: Spring Kafka](https://docs.spring.io/spring-kafka/reference/kafka/headers.html))

У этого интерфейса есть два важных метода
```java
public interface KafkaHeaderMapper {
	//Spring Kafka MessageHeaders -> Apache Kafka Headers
    void fromHeaders(MessageHeaders headers, Headers target);

	//Apache Kafka Headers -> Spring Kafka MessageHeaders
    void toHeaders(Headers source, Map<String, Object> target);
}
```

Конвертор хедеров настраивается в `MessagingMessageCOnvertor`
```java
MessagingMessageConverter converter = 
	new MessagingMessageConverter();

// Описание класса, 🔥для конвертации заголовков🔥 
converter.setHeaderMapper(new KafkaHeaderMapper() {  
	@Override  
	public void fromHeaders(
		MessageHeaders headers, 
		Headers target) 
	{  
		// Перебираем все заголовки, которые добавили в MessageBuilder
		// и добавляем в Headers Apache Kafka
		for(Map.Entry<String, Object> entry: headers.entrySet()){
			target.add(
				entry.getKey(), 
				entry.getValue()
					.toString()
					.getBytes());  
		}  
	}  

	@Override  
	public void toHeaders(
		Headers source, 
		Map<String, Object> target) { }  
}); 
```

## 3. Класс для отправки данных
После конфигурации Producer, опишем класс для отправки данных с КАСТОМНЫМ заголовком
```java
package producer.service;  
  
import model.CustomHeader;  
import model.UserInfo;  
import org.springframework.kafka.core.KafkaTemplate;  
import org.springframework.kafka.support.KafkaHeaders;  
import org.springframework.messaging.support.MessageBuilder;  
import org.springframework.stereotype.Component;  
  
import java.util.UUID;  
import java.util.concurrent.TimeUnit;  
  
@Component  
public class ProducerService {  
    private final KafkaTemplate<String, UserInfo> kafkaTemplate;  
  
    public ProducerService(KafkaTemplate<String, UserInfo> kafkaTemplate) 
    {  
        this.kafkaTemplate = kafkaTemplate;  
    }  
  
    public void produceMessage(){  
        System.out.println("[PRODUCER] In produceMessage");  
  
        try {  
            kafkaTemplate.send(
	            MessageBuilder  
                    .withPayload(new UserInfo("Tom Kruz", 10))  
                    .setHeader(KafkaHeaders.TOPIC, "topic-spring-1")  
                    .setHeader("Simple_Header", "Header 1") 
                    // 🔥КАСТОМНЫЙ сложный заголовок🔥
                    .setHeader(  
                            "Custom_Header",  
                            new CustomHeader(  
								"Inner Header",  
								UUID.randomUUID(),  
								17))  
                    .build())  
                .get(10, TimeUnit.SECONDS);  
  
            System.out.println("Message sent (sync).");  
        } catch (Exception e) {  
            e.printStackTrace();  
        }  
    }  
}
```

И поскольку, при отправке сообщения происходят дополнительные действия (см [[6. - Отправка данных, Producer (at most-least-exactly once)]])

то чтобы JVM не завершился раньше, чем сообщения действительно отправятся, БУДЕМ ЖДАТЬ ответа от брокера Kafka при помощи вызова метода get() - этот метод вызывается на CompletablFuture 
(см [[4. - Класс CompletableFuture]])

### Пояснения принципа отправки
После того, как мы запустим приложение, создадутся бины(в результате создания которых мы получим KafkaTemplate).

Затем, мы получим бин класса отправки сообщения, и явно вызовем метод отправки сообщения.

В методе отправки, мы создаем сообщение при помощи MessageBuilder, и вызываем метод send().

Но поскольку мы работаем с API Spring Kafka, а не с API Apache Kafka напрямую, то мы не можем отправить этот созданный Message.
Поэтому, Spring вызываем MessaggeConvertor, который мы указали в конфигурации KafkaTemplate.

Далее Spring увидит, что в конверторе, так же настроен KafkaHeaderMapper, и он понимает, как преобразовывать заголовки Spring Kafka, в заголовки Apache Kafka

# 3. Создание Consumer

## 1. Описание Десериализаторов
Поскольку к нам поступает НЕ ОБЫЧНОЕ строковое сообщение, а КАСТОМНОЕ, так еще и со сложным заголовком, то нам необходимы специальные сериализаторы
### Десериализатор сообщения
Опишем сериализатор сообщения. Так как нам, поступает сообщение в виде объекта UserInfo, с соответствующими полями, в виде 
```java
"username:name, uuid:231asd48mx2, x:17"
```
и в виде массива байт, то нам необходимо преорбазовать этот массив байт в строку, и распарсить строку

Напишем десериализатор сообщения
```java
package util;  
  
import model.UserInfo;  
import org.apache.kafka.common.serialization.Deserializer;  
  
public class CustomUserInfoDeserializer implements Deserializer<UserInfo> {  
    @Override  
    public UserInfo deserialize(String s, byte[] bytes) {  
        UserInfo userInfo = new UserInfo();  
  
        String json = new String(bytes);  
        String[] parts = json.split(",\\s*"); // ["userName: Alex", "age: 25"]  
  
        for (String part : parts) {  
            String[] keyValue = part.split(":\\s*");  
  
            String key = keyValue[0].trim();  
            String value = keyValue[1].trim();  
  
            switch (key) {  
                case "userName":  
                    userInfo.setUserName(value);  
                    break;  
                case "age":  
                    userInfo.setAge(Integer.parseInt(value));  
                    break;  
            }  
        }  
        return userInfo;  
    }  
}
```

Чтобы кастомный десериализатор можно было использовать в параметрах конфигурации, реализация должна наследовать класс Deserializer
(см [[2. - Интерфейс Deserializer]])

### Десиализатор заголовка
Теперь опишем десериализатор заголовка, аналогичным образом, как мы писали десериализатор тела сообщения (из массива байт получаем строку, а из строки достаем данные и собираем объект)

```java
package util;  
  
import model.CustomHeader;  
import java.util.UUID;  
  
public class CustomHeaderDeserializer {  
    public static CustomHeader deserialize(byte[] bytes){  
        CustomHeader customHeader = new CustomHeader();  
  
        String json = new String(bytes); 
        //"header:header1, uuid:7123asd71, x:17;" 
         
        String[] parts = json.split(",\\s*"); 
        // [header:header1],[uuid:7123asd71], [x:17];  
  
        for (String part : parts) {  
            String[] keyValue = part.split(":\\s*");  
  
            String key = keyValue[0].trim();  
            String value = keyValue[1].trim();  
  
            switch (key) {  
                case "header":  
                    customHeader.setHeader(value);  
                    break;  
                case "uuid":  
                    customHeader.setUuid(UUID.fromString(value));  
                    break;  
                case "x":  
                    customHeader.setX(Integer.parseInt(value));  
            }  
        }  
        return customHeader;  
    }  
}
```

## 2. Описание Конфигурации Consumer
Теперь так как у нас есть десериализаторы данных, опишем Consumer. 

Как говорилось в Spring Kafka 
(см [[0. - Концепция Cosnumer в Spring Kafka и принцип работы с ним]])
нам нужны следующие компоненты
- Параметры конфигурации Consumer
- ConsumerFactory
- Фабрика контейнеров слушателей
- Слушатель (через @KafkaListener или реализация MessageListener)

```java
package consumer.config;  
  
import model.CustomHeader;  
import model.UserInfo;  
import org.apache.kafka.clients.consumer.ConsumerConfig;  
import org.apache.kafka.common.header.Header;  
import org.apache.kafka.common.header.Headers;  
import org.apache.kafka.common.serialization.StringDeserializer;  
import org.springframework.context.annotation.Bean;  
import org.springframework.context.annotation.Configuration;  
import org.springframework.kafka.annotation.EnableKafka;  
import org.springframework.kafka.config.ConcurrentKafkaListenerContainerFactory;  
import org.springframework.kafka.core.ConsumerFactory;  
import org.springframework.kafka.core.DefaultKafkaConsumerFactory;  
import org.springframework.kafka.support.KafkaHeaderMapper;  
import org.springframework.kafka.support.converter.MessagingMessageConverter;  
import org.springframework.messaging.MessageHeaders;  
import util.CustomHeaderDeserializer;  
import util.CustomUserInfoDeserializer;  
  
import java.util.HashMap;  
import java.util.Map;  
  
@Configuration  
@EnableKafka  
public class KafkaConsumerConfig {  
    @Bean  
    public Map<String, Object> consumerConfig() {  
        Map<String, Object> props = new HashMap<String, Object>();  
        props.put(  
            ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG,  
            "localhost:29092,localhost:39092,localhost:49092"  
        );  
        props.put(  
            ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG,  
            StringDeserializer.class  
        );  
        props.put(  
            ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG,  
            CustomUserInfoDeserializer.class  
        );  
        props.put(  
            ConsumerConfig.AUTO_OFFSET_RESET_CONFIG,  
            "earliest"  
        );  
        props.put(  
            ConsumerConfig.GROUP_ID_CONFIG,  
            "custom_consumer"  
        );  
        return props;  
    }  
    @Bean  
    public ConsumerFactory<String, UserInfo> consumerFactory() {  
        return new DefaultKafkaConsumerFactory<String, UserInfo>(consumerConfig());  
    }  
  
    @Bean  
    public 
    ConcurrentKafkaListenerContainerFactory<String, UserInfo> 
    kafkaListenerContainerFactory() 
    {  
        ConcurrentKafkaListenerContainerFactory<String, UserInfo> 
        factory = 
	        new ConcurrentKafkaListenerContainerFactory<>();  
  
        factory.setConsumerFactory(consumerFactory());  
  
        MessagingMessageConverter converter = 
	        new MessagingMessageConverter();  
        converter.setHeaderMapper(new KafkaHeaderMapper() {  
            @Override  
            public void fromHeaders(MessageHeaders headers, Headers target) {}  
  
            @Override  
            public void toHeaders(
	            Headers source, 
	            Map<String, Object> target) 
	        {  
                CustomHeader customHeader = new CustomHeader();  
                for(Header header : source){  
                    if(header.key().equals("Custom_Header")){  
                        customHeader = 
	                        CustomHeaderDeserializer
		                        .deserialize(header.value());  
		                        
                        target.put(header.key(), customHeader);  
                        continue;  
                    }  
                    target.put(
	                    header.key(), 
	                    new String(header.value())
	                );  
                }  
            }  
        });  
  
        factory.setRecordMessageConverter(converter);  
        return factory;  
    }  
}
```

> [!NOTE] @EnableKafka
> Важная аннотация, благодаря которой работает @KafkaListener

Здесь, мы также добавляем MessageConvertor, чтобы из ConsumerRecord, получить Message. 

А также чтобы правильно смапились заголовки, добавляем KafkaHeaderMapper в MessagingMesssageConvertor.
Только чтобы получить объект КАСТОМНОГО ЗАГОЛОВКА, мы вызываем метод нашего десериализатора заголовка, который вернет из массива байт нужный объект.
