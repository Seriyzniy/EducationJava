```java
private final String DB_URL = "jdbc:postgresql://localhost:5532/testdb";
```

Для подключения и работы с БД, нам нужен драйвер `org.postgresql.Driver` от Postgres. Его необходимо загрузить в память. Для этого вопользуемся рефлексией
```java
try {

    System.out.println("Driver not found");  
    throw new RuntimeException(e);  
}
```

От этого драйвера, мы можем открыть соединение с БД
```java
try{

	    .getConnection(
		    DB_URL,
		    DB_USER,
		    DB_PASSWORD);  
		    
} catch (SQLException e) {  
    System.out.println("Get connection to Postgres error");  
    throw new RuntimeException(e);  
}
```

И после чего, выполнять свой код по сохранению данных
```java
try {  
    PreparedStatement preparedStatement =

    throw new RuntimeException(e);

```

## Удаление не нужных полей 
Как мы видим, в сообщение положился еще и id записи из БД. Это нам не нужно, нам нужно оставить только ключевые поля

Эту проблему можно решить при помощи еще одного SMT, который мы добавим в свойство `transforms`

В Apache Kafka есть специальный класс, для исключения/добавления полей в итоговое сообщение:
- **[ReplaceField](https://kafka.apache.org/documentation/#org.apache.kafka.connect.transforms.ReplaceField)** – Фильтрует или переименовывает поля.

Чтобы его добавить, нам нужно добавить еще один `transforms` в обновленный конфиг
```json
"transforms": "unwrap,remove_id",
  
"transforms.unwrap.type": "io.debezium.transforms.ExtractNewRecordState",  

"transforms.remove_id.type": 
"org.apache.kafka.connect.transforms.ReplaceField$Value",  

"transforms.remove_id.exclude": "id",
```

В `"transforms": "unwrap,remove_id"` добавляем новое имя для SMT

И уже для него определяем тип нашего SMT:
`"transforms.remove_id.type": "org.apache.kafka.connect.transforms.ReplaceField$Value",`

После чего, используя свойство `exclude` (описан в документации), убираем не нужные поля
`"transforms.remove_id.exclude": "id",`

# 7. Описание Consumer
Чтобы не мудрить, возьмем готовый Consumer, который мы ранее описывали в прошлых проектах для приема этого же типа данных (UserInfo)
## 1. Десериализатор тела
Но чтобы принять данные, их нужно десериализовать, поэтому напишем десериализатор:
```java
package util;  
  
import com.fasterxml.jackson.databind.ObjectMapper;  
import model.UserInfo;  
import org.apache.kafka.common.serialization.Deserializer;  
  
import java.io.IOException;  
  
public class CustomUserInfoDeserializer implements Deserializer<UserInfo> {  
    private final ObjectMapper objectMapper = new ObjectMapper();  
  
    @Override  
    public UserInfo deserialize(String s, byte[] bytes) {  
        try {  
            return objectMapper.readValue(bytes, UserInfo.class);  
        } catch (IOException e) {  
            throw new RuntimeException(e);  
        }  
    }  
}
```

## 2. Десериализатор ключа
Так как ключ нам передается как JSON, с полем ID, готовые десериализаторы по типу `IntegerDeserializator` на мне подойдут, так как передается не просто число, а JSON с полем id

Поэтому нужно написать кастомный десериализатор ключа.

Но и создавать отдельный класс с get/set методами под одну переменную не хочется.

Поэтому в Jasckon есть способ прочитать JSON объект, и получить определенное поле - `JsonNode`
```java
package util;  
  
import com.fasterxml.jackson.databind.JsonNode;  
import com.fasterxml.jackson.databind.ObjectMapper;  
import org.apache.kafka.common.serialization.Deserializer;  
  
import java.io.IOException;  
  
public class CustomKeyDeserializer implements Deserializer<Integer> {  
    private final ObjectMapper mapper = new ObjectMapper(); 
     
    @Override  
    public Integer deserialize(String s, byte[] bytes) {  
        try {  
            JsonNode node = mapper.readTree(bytes);  
            int id = node.get("id").asInt();  
            
            return id;  
        } catch (IOException e) {  
            throw new RuntimeException(e);  
        }  
    }  
}
```

В результате получим Integer значение

## 3. Кастомный консюмер
```java
package consumer;  
  
import model.UserInfo;  
import org.apache.kafka.clients.consumer.ConsumerConfig;  
import org.apache.kafka.clients.consumer.ConsumerRecord;  
import org.apache.kafka.clients.consumer.ConsumerRecords;  
import org.apache.kafka.clients.consumer.KafkaConsumer;  
import util.CustomKeyDeserializer;  
import util.CustomUserInfoDeserializer;  
  
import java.time.Duration;  
import java.util.Collections;  
import java.util.Properties;  
  
public class CustomKafkaConsumer {  
    private final Properties props;  
  
    public static void main(String[] args) {  
        CustomKafkaConsumer consumer = new CustomKafkaConsumer();  
  
        consumer.consumeUserInfo();  
    }  
  
    public CustomKafkaConsumer() {  
        props = new Properties();  
        props.put(  
                ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG,  
                "localhost:29092,localhost:39092,localhost:49092"  
        );  
        props.put(  
                ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG,  
                CustomKeyDeserializer.class  
        );  
        props.put(  
                ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG,  
                CustomUserInfoDeserializer.class  
        );  
        props.put(  
                ConsumerConfig.AUTO_OFFSET_RESET_CONFIG,  
                "earliest"  
        );  
        props.put(  
                ConsumerConfig.GROUP_ID_CONFIG,  
                "custom_consumer"  
        );  
    }  
  
    public void consumeUserInfo(){  
        try(KafkaConsumer<Integer, UserInfo> consumer = 
	        new KafkaConsumer<>(props)
	    ){  
            consumer.subscribe(
	            Collections.singleton("pg-dev.public.userinfo"));  
  
            while(true){  
                ConsumerRecords<Integer, UserInfo> records = 
	                consumer.poll(Duration.ofSeconds(1000));  
  
                for(ConsumerRecord<Integer, UserInfo> record : records){  
                    System.out.println(  
						"Key = " + record.key() +  
						"\nValue =" +  record.value().toString());  
                }  
            }  
        }  
    }  
}
```

## 3. Проблема с именем полей в модели и БД
В БД определены поля:
- uname - для поля в Java коде userName
- age - age в Java коде

А Debezium отправляет данные исходя из столбцов в БД.
![[Pasted image 20251205094822.png]]

Так как у нас разные названия переменных для полей имени пользователя, то чтобы успешно десериализовать данные, нужно как-то их смапить:
- Вручную десериализуем данные
- Изменить имя в модели 
- Изменить столбце в Postgres

Самое простое это изменить имя в модели
```java
package model;  
  
public class UserInfo {  
    private String uname;  
    private int age;  
  
    public UserInfo() {}  
    public UserInfo(String userName, int age) {  
        this.uname = userName;  
        this.age = age;  
    }  
  
    public String getUname() {return uname;}
```
