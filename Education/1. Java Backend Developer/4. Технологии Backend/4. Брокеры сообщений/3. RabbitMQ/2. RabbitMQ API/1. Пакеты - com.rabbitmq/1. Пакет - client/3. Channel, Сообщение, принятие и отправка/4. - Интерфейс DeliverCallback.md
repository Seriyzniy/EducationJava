
> [!NOTE] Интерфейс `DeliverCallback`
> `DeliverCallback` — это **функциональный интерфейс** (Java 8+) ==для обработки полученных сообщений== от брокера RabbitMQ. 
> 
> Он используется в **адаптированном коллбек-стиле**, когда вы хотите указать только **логику обработки сообщений**, ==без необходимости реализовывать весь интерфейс== `Consumer`.  
> 
> 💡Типично применяется вместе с методом `Channel.basicConsume(queue, autoAck, DeliverCallback, CancelCallback)`.

В Javadoc **не указано**, что интерфейс реализует `AutoCloseable` или `Closeable`, — это **обычный функциональный интерфейс**, который **не управляет временем жизни** ресурса.

Он **не содержит подинтерфейсов** и **немедленно обрабатывает только доставку сообщений**.  

Его можно реализовать с помощью лямбда-выражения благодаря единственному абстрактному методу.

# Интерфейс
```java
@FunctionalInterface
public interface DeliverCallback
```
💬 **Комментарий:**  
Функциональный интерфейс, представляющий обработчик для доставки сообщений в basicConsume.  
(Комментируется в Javadoc RabbitMQ Java Client)

---
# Методы
## Основной метод
```java
// Called when a message is delivered.
// Вызывается при доставке сообщения.
void handle(String consumerTag,
            Delivery message)
            throws IOException
```

💬 **Комментарий:**  
Метод **вызывается для каждого доставленного сообщения**. Реализация должна обработать тело (`message.getBody()`), свойства и метаданные. В данном интерфейсе есть только этот метод — его задача **обрабатывать доставленные сообщения**.


## ## 💬 Комментарий по методам
🔹 `DeliverCallback` определяет только **один метод `handle`**, что делает его удобным для использования в **лямбда-стиле**.  

🔹 Он отвечает **только за доставку сообщений**, в отличие от интерфейса `Consumer`, который имеет несколько методов жизненного цикла (cancel, shutdown и т. д.).  

🔹 Если нужны **подробные события жизненного цикла**, лучше использовать `Consumer` или `DefaultConsumer`.

---
# 💎 Best Practices 💎
✅ Использовать **лямбда-выражение** для `DeliverCallback`, чтобы сделать обработчик компактным.  

✅ Обрабатывать исключения внутри `handle`, чтобы не приводить к закрытию канала по не обработанному исключению.  

✅ Комбинировать с `CancelCallback` для полного набора событий при подписке `basicConsume`.  

✅ Если нужна логика жизненного цикла (отмена, shutdown), использовать `Consumer`/`DefaultConsumer`.

# Пример использования
```java
Channel channel = connection.createChannel();

// deliverCallback обрабатывает только доставку сообщений
DeliverCallback deliverCallback = (consumerTag, delivery) -> {
    String message = new String(
	    delivery.getBody(), 
	    StandardCharsets.UTF_8);
	    
    System.out.println("Received: " + message);
    // при ручном подтверждении
    channel.basicAck(delivery.getEnvelope().getDeliveryTag(), false);
};


// cancelCallback обрабатывает отмену доставки
CancelCallback cancelCallback = consumerTag -> {
    System.out.println("Cancelled: " + consumerTag);
};

channel.basicConsume("my.queue", false,
                     deliverCallback,
                     cancelCallback);
```
