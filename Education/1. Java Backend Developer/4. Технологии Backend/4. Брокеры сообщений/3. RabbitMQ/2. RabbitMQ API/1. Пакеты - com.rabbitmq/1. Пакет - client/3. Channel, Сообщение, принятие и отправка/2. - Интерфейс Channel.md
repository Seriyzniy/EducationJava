
> [!NOTE] Интерфейс `Channel`
> `Channel` представляет собой **виртуальный канал AMQP 0-9-1 внутри соединения (`Connection`)**. 
> 
> Через него осуществляется почти **всё взаимодействие с брокером**: отправка сообщений, получение, декларация очередей/обменников, управление подтверждениями, транзакции и пр.

Интерфейс `Channel` покрывает **почти все AMQP-методы**, представленные на уровне приложения, поэтому API довольно объёмный. [Javadoc](https://javadoc.io/doc/com.rabbitmq/amqp-client/5.7.3/com/rabbitmq/client/Channel.html?utm_source=chatgpt.com)

### **Реализуемые интерфейсы:**
- `AutoCloseable` (с Java 7+) - позволяет использовать в try-with-resources
    
- `ShutdownNotifier` - для получения уведомлений о завершении работы
    

**Не реализует напрямую:** `Recoverable` - но он является суперинтерфейсом для `RecoverableChannel`

### **Подинтерфейсы:**
- `RecoverableChannel` - предоставляет дополнительные методы для работы с автоматически восстанавливаемыми каналами. Наследует от `Channel` и `Recoverable`.
    

### **Основные реализации:**
- `ChannelN` (в пакете `com.rabbitmq.client.impl`) - стандартная реализация канала
    
- `RecoveryAwareChannelN` (в `com.rabbitmq.client.impl`) - реализация с поддержкой восстановления
    
- `AutorecoveringChannel` (в пакете `com.rabbitmq.client.impl.recovery`) - автоматически восстанавливаемый канал

# Методы
Не все методы перечислены выше (например, дополнительные перегрузки декларации exchange/queue), но это — **основные и наиболее часто используемые**.
## Основные операции AMQP
```java
// Publish a message.
// Опубликовать сообщение.
void basicPublish(String exchange,
                  String routingKey,
                  AMQP.BasicProperties props,
                  byte[] body)
                  throws IOException;

// Publish a message with flags.
// Опубликовать сообщение с дополнительными флагами.
void basicPublish(String exchange,
                  String routingKey,
                  boolean mandatory,
                  boolean immediate,
                  AMQP.BasicProperties props,
                  byte[] body)
                  throws IOException;

// Acknowledge one or more delivered messages.
// Подтвердить (ack) одно или несколько сообщений.
void basicAck(long deliveryTag, boolean multiple) throws IOException;

// Reject a single message.
// Отклонить сообщение.
void basicReject(long deliveryTag, boolean requeue) throws IOException;

// Negative acknowledgement.
// Негативное подтверждение с возможностью requeue.
void basicNack(long deliveryTag, boolean multiple, boolean requeue)
                  throws IOException;
```
💬 Эти методы используются **для публикации и подтверждения сообщений**.

## Потребление и получение сообщений
```java
// Start a non-nolocal, non-exclusive consumer.
// Начать потребление (basicConsume).
String basicConsume(String queue,
                    boolean autoAck,
                    Consumer callback)
                    throws IOException;

// Cancel a consumer.
// Отменить потребителя.
void basicCancel(String consumerTag) throws IOException;

// Pull a single message (synchronous).
// Получить одно сообщение (poll).
GetResponse basicGet(String queue, boolean autoAck) throws IOException;
```
💬 `basicConsume` асинхронно вызывает колбэки Consumer при доставке сообщений; `basicGet` — синхронный “pull”

# 💡 Особенности и подводные камни 💡

1️⃣ **Канал НЕ ПОТОКОБЕЗОПАСЕН для параллельных операций** — почти все вызовы должны быть синхронизированы, если используются из нескольких потоков. [RabbitMQ](https://www.rabbitmq.com/client-libraries/java-api-guide?utm_source=chatgpt.com)  

2️⃣ **Закрытие канала делает его полностью нерабочим** — дальнейшие вызовы методов приведут к ошибкам. [RabbitMQ](https://www.rabbitmq.com/docs/channels?utm_source=chatgpt.com)  

3️⃣ **Channel работает в контексте Connection** — при закрытии соединения автоматически закрываются все каналы. [RabbitMQ](https://www.rabbitmq.com/docs/channels?utm_source=chatgpt.com)

4️⃣ **Не разделяет состояние между каналами** — каждая QoS/confirm/transaction настройка локальна. [Javadoc](https://javadoc.io/doc/com.rabbitmq/amqp-client/5.7.3/com/rabbitmq/client/Channel.html?utm_source=chatgpt.com)  

5️⃣ **Используйте confirmSelect() вместо транзакций** для высокой пропускной способности. [Javadoc](https://javadoc.io/doc/com.rabbitmq/amqp-client/5.7.3/com/rabbitmq/client/Channel.html?utm_source=chatgpt.com)

# 💎 Best Practices 💎
✅ Использовать **отдельный канал на поток** — избегать одновременного доступа из разных нитей. [RabbitMQ](https://www.rabbitmq.com/client-libraries/java-api-guide?utm_source=chatgpt.com)  

✅ Для гарантий доставки использовать **Publisher Confirms** (`confirmSelect`) вместо транзакций. [Javadoc](https://javadoc.io/doc/com.rabbitmq/amqp-client/5.7.3/com/rabbitmq/client/Channel.html?utm_source=chatgpt.com)  

✅ Декларировать очередь/обменники до публикации/потребления. [RabbitMQ](https://www.rabbitmq.com/client-libraries/java-api-guide?utm_source=chatgpt.com)  

✅ Корректно закрывать канал после завершения работы. [RabbitMQ](https://www.rabbitmq.com/docs/channels?utm_source=chatgpt.com)

# Пример использования
```java
Connection connection = factory.newConnection();
Channel channel = connection.createChannel();

// declare
channel.queueDeclare("task_queue", true, false, false, null);

// publish
channel.basicPublish(
		"", 
		"task_queue",
        MessageProperties.PERSISTENT_TEXT_PLAIN,
        "Hello".getBytes(StandardCharsets.UTF_8));

// consume
DeliverCallback deliver = (tag, delivery) -> {
    System.out.println(new String(delivery.getBody(), StandardCharsets.UTF_8));
    channel.basicAck(delivery.getEnvelope().getDeliveryTag(), false);
};
channel.basicConsume("task_queue", false, deliver, consumerTag -> {});

// close when done
channel.close();
```
