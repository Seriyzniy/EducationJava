
> [!NOTE] Интерфейс `Connection`
> представляет собой **основное API для AMQP-соединения** между клиентским приложением и брокером RabbitMQ. 
> Он позволяет:
> - открывать **каналы** (`Channel`) для публикации и подписки сообщений;
> - управлять состоянием соединения (закрывать, аварийно прерывать);
> - получать свойства и параметры соединения;
> - регистрировать слушателей блокировки/разблокировки.
>
>Это фундаментальный элемент работы с RabbitMQ на Java — без него клиент не может общаться с брокером

### **Реализует интерфейсы:**
- `AutoCloseable` — позволяет использовать try-with-resources для закрытия соединения.
    
- `Closeable` — расширяет `AutoCloseable`, добавляя возможность закрытия с IOException.
    
- `ShutdownNotifier` — позволяет регистрировать слушателей событий завершения работы соединения. [RabbitMQ](https://rabbitmq.github.io/rabbitmq-java-client/api/current/com/rabbitmq/client/Connection.html?utm_source=chatgpt.com)

### **Подинтерфейсы / реализации:**
- Подинтерфейс: `RecoverableConnection` (для автоматически восстанавливаемых соединений). [RabbitMQ](https://rabbitmq.github.io/rabbitmq-java-client/api/current/com/rabbitmq/client/Connection.html?utm_source=chatgpt.com)
    
- Основные реализации: `AMQConnection`, `AutorecoveringConnection`, `RecoveryAwareAMQConnection`

# 2. Методы интерфейса
## 1. Управление соединением
```java
// Закрывает соединение и все каналы с кодом AMQP.REPLY_SUCCESS ('OK')
void close() throws IOException;

// То же, что close(), но с таймаутом ожидания (мс)
void close(int timeout) throws IOException;

// Закрывает с указанным кодом/сообщением
void close(int closeCode, String closeMessage) throws IOException;

// Закрывает с кодом/сообщением и таймаутом
void close(int closeCode, String closeMessage, int timeout) throws IOException;



// Прерывает соединение и каналы (не кидает исключения)
void abort();

// Прерывает с указанным кодом/сообщением
void abort(int closeCode, String closeMessage);

// Прерывает с таймаутом
void abort(int timeout);

// Прерывает с кодом/сообщением и таймаутом
void abort(int closeCode, String closeMessage, int timeout);
```
**Что это делает:**  
Закрытие соединения — нормальное или аварийное, с возможностью задать код и сообщение AMQP. При `abort` любые ошибки игнорируются.

## 2. Каналы
```java
// Создать новый канал, автоматически назначая номер
Channel createChannel() throws IOException;

// Создать новый канал с заданным номером
Channel createChannel(int channelNumber) throws IOException;


// То же, но возвращает Optional (начиная с 5.6.0)
default Optional<Channel> openChannel() throws IOException;

// То же с указанием номера
default Optional<Channel> openChannel(int channelNumber) throws IOException;
```
**Смысл:**  
Канал — лёгкая виртуальная сессия внутри соединения. ==Они должны быть созданы через соединение.==

## 3. Информация о соединении
```java
// Получить адрес брокера
InetAddress getAddress();

// Получить порт брокера
int getPort();

// Максимальный номер канала, согласованный с брокером
int getChannelMax();

// Максимальный размер кадров (frame size)
int getFrameMax();

// Heartbeat (интервал проверок подключения)
int getHeartbeat();

// Свойства клиентов, отправленные брокеру
Map<String, Object> getClientProperties();

// Имя соединения, заданное клиентом (не уникально)
String getClientProvidedName();

// Свойства брокера
Map<String, Object> getServerProperties();
```
**Пояснение:**  
Методы дают мета-информацию о соединении и брокере, полезную для логирования, мониторинга или диагностики.

## 4. Слушатели блокировки
```java
// Добавить слушателя блокировки/разблокировки
BlockedListener addBlockedListener(BlockedCallback blockedCallback,
                                   UnblockedCallback unblockedCallback);

// Добавить слушателя блокировки
void addBlockedListener(BlockedListener listener);


// Удалить слушателя
boolean removeBlockedListener(BlockedListener listener);

// Очистить всех слушателей
void clearBlockedListeners();
```

**Когда это нужно:**  
RabbitMQ может блокировать соединение по ресурсам (например, если брокер испытывает давление). Эти методы позволяют узнать, когда это происходит.

# 3. 💡 Особенности и подводные камни 💡
1️⃣ `Connection` сам по себе не предназначен для многократного частого открытия/закрытия — **соединение дорого по ресурсам**. Открывайте его однажды и переиспользуйте. [RabbitMQ](https://www.rabbitmq.com/client-libraries/java-api-guide?utm_source=chatgpt.com)  

2️⃣ Каналы — **лёгкие и дешёвые**, их можно создавать по необходимости внутри одного соединения. [RabbitMQ](https://www.rabbitmq.com/client-libraries/java-api-guide?utm_source=chatgpt.com)  

3️⃣ `close()` бросает IOException, `abort()` — **нет**; последний используется при авариях, но может скрыть ошибки. [RabbitMQ](https://rabbitmq.github.io/rabbitmq-java-client/api/current/com/rabbitmq/client/Connection.html?utm_source=chatgpt.com)

4️⃣ `getClientProvidedName()` — удобно для логов и мониторинга, но **не подходит как уникальный идентификатор**. [RabbitMQ](https://rabbitmq.github.io/rabbitmq-java-client/api/current/com/rabbitmq/client/Connection.html?utm_source=chatgpt.com)  

5️⃣ Восстанавливаемые соединения реализуют `RecoverableConnection`, но для их полного использования нужно включить автоматическое восстановление в `ConnectionFactory`. [RabbitMQ](https://www.rabbitmq.com/client-libraries/java-api-guide?utm_source=chatgpt.com)

# 💎 Best Practices 💎
✅ Открывать **одно глобальное соединение** на приложение/worker. [RabbitMQ](https://www.rabbitmq.com/client-libraries/java-api-guide?utm_source=chatgpt.com)  

✅ Все операции AMQP выполнять через **каналы**, а не через новое соединение. [RabbitMQ](https://www.rabbitmq.com/client-libraries/java-api-guide?utm_source=chatgpt.com)  

✅ Устанавливать **наименование соединения** (ЧЕРЕЗ ФАБРИКУ) для облегчения диагностики. [RabbitMQ](https://www.rabbitmq.com/client-libraries/java-api-guide?utm_source=chatgpt.com)  

✅ Обрабатывать события блокировки через listeners для устойчивых систем. [RabbitMQ](https://rabbitmq.github.io/rabbitmq-java-client/api/current/com/rabbitmq/client/Connection.html?utm_source=chatgpt.com)  

✅ Использовать `try-with-resources` для гарантированного закрытия соединения. [RabbitMQ](https://rabbitmq.github.io/rabbitmq-java-client/api/current/com/rabbitmq/client/Connection.html?utm_source=chatgpt.com)

# Пример
```java
ConnectionFactory factory = new ConnectionFactory();
factory.setHost("rabbitmq.example.com");
factory.setUsername("user");
factory.setPassword("password");
factory.setVirtualHost("/");

try (Connection connection = factory.newConnection("my-app-conn")) {
    Channel channel = connection.createChannel();
    
    // публикация сообщения
    channel.basicPublish(
	    "", 
	    "queue.name",
        MessageProperties.PERSISTENT_TEXT_PLAIN,
        "Hello RabbitMQ".getBytes(StandardCharsets.UTF_8));

    // потребление
    DeliverCallback deliverCallback = (consumerTag, delivery) -> {
        String msg = new String(delivery.getBody(), StandardCharsets.UTF_8);
        System.out.println("Received: " + msg);
    };
    channel.basicConsume("queue.name", true, deliverCallback, consumerTag -> { });
}

```
