
> [!NOTE] Класс `Delivery`
> **Delivery** - это простой ==контейнерный класс== (data holder), который ==инкапсулирует все данные== о доставленном ==сообщении==. 
> 
> Он **передается в обработчики (callback)** при получении сообщений от RabbitMQ и содержит как само тело сообщения, так и метаданные: свойства сообщения, информацию о доставке (envelope) и тег потребителя.

**Зачем нужен:** Для получения полной информации о сообщении в обработчиках (`DeliverCallback`, `Consumer.handleDelivery()`). Служит единой точкой доступа ко всем данным сообщения.


# Конструктор
```java
/**
 * Construct a new {@link Delivery} with the specified envelope, properties and body.
 * @param envelope message envelope
 * @param properties message properties
 * @param body message body
 */
public Delivery(Envelope envelope, 
                AMQP.BasicProperties properties, 
                byte[] body) {
    this.envelope = envelope;
    this.properties = properties;
    this.body = body;
    this.consumerTag = null;
}
// Создает новый Delivery с указанным конвертом, свойствами и телом.

/**
 * Construct a new {@link Delivery} with the specified envelope, properties, body and consumer tag.
 * @param envelope message envelope
 * @param properties message properties
 * @param body message body
 * @param consumerTag consumer tag
 */
public Delivery(Envelope envelope, 
                AMQP.BasicProperties properties, 
                byte[] body, 
                String consumerTag) {
    this.envelope = envelope;
    this.properties = properties;
    this.body = body;
    this.consumerTag = consumerTag;
}
// Создает новый Delivery с указанным конвертом, свойствами, телом и тегом потребителя.
```
💬**Комментарий:** Конструкторы пакетно-приватные (package-private), что означает, что обычные пользователи API не могут создавать экземпляры `Delivery` напрямую. ==Эти объекты создаются только ВНУТРИ КЛИЕНТА RABBITMQ и передаются в обработчики==. Это обеспечивает целостность данных доставки.

## **Поля класса (package-private):**
- `Envelope envelope` - информация о доставке (exchange, routingKey, deliveryTag)
    
- `BasicProperties properties` - свойства сообщения (headers, contentType и др.)
    
- `byte[] body` - тело сообщения в бинарном виде
    
- `String consumerTag` - тег потребителя, который получил сообщение

# Методы
## Получение данных сообщения
```java
/**
 * Retrieve the message envelope.
 * @return the message envelope
 */
public Envelope getEnvelope() {
    return envelope;
}
// Возвращает конверт сообщения (информация о доставке).

/**
 * Retrieve the message properties.
 * @return the message properties
 */
public AMQP.BasicProperties getProperties() {
    return properties;
}
// Возвращает свойства сообщения.

/**
 * Retrieve the message body.
 * @return the message body
 */
public byte[] getBody() {
    return body;
}
// Возвращает тело сообщения.

/**
 * Retrieve the consumer tag.
 * @return the consumer tag
 * @since 4.0.0
 */
public String getConsumerTag() {
    return consumerTag;
}
// Возвращает тег потребителя.
```
💬**Небольшой комментарий по методам:** Все методы являются простыми геттерами (getters) без логики. 

Класс иммутабелен (неизменяем) после создания, что обеспечивает безопасность при многопоточном использовании в обработчиках.

# 💡 Особенности и подводные камни 💡
1️⃣ **Иммутабельность:** Объект `Delivery` неизменяем после создания. Все поля объявлены как `final`, что гарантирует потокобезопасность при передаче между потоками.

2️⃣ **Прямой доступ к массиву байтов:** Метод `getBody()` возвращает ссылку на оригинальный массив байтов, а не копию. 
Это может привести к неожиданным побочным эффектам, если изменять содержимое массива:
```java
// ❌ ПЛОХО - изменяем оригинальный массив
byte[] body = delivery.getBody();
body[0] = 0; // Изменяет оригинальные данные!

// ✅ ХОРОШО - создаем копию при необходимости
byte[] bodyCopy = Arrays.copyOf(
	delivery.getBody(), 
	delivery.getBody().length);
```

3️⃣ **Проверка на null:** Тело сообщения (`body`) может быть `null`, если сообщение было отправлено без тела. Всегда проверяйте:
```java
if (delivery.getBody() != null) {
    String message = new String(
	    delivery.getBody(), 
	    StandardCharsets.UTF_8);
}
```

4️⃣ **Свойства могут быть null:** `BasicProperties` также может быть `null`, если сообщение было отправлено без свойств. Используйте безопасный доступ:
```java
String contentType = delivery.getProperties() != null 
    ? delivery.getProperties().getContentType() 
    : null;
```

5️⃣ **Тег потребителя доступен не всегда:** Метод `getConsumerTag()` был добавлен в версии 4.0.0. В старых версиях или в некоторых сценариях он может быть `null`.

6️⃣ **Envelope содержит критическую информацию:** Через `getEnvelope()` можно получить `deliveryTag`, который необходим для подтверждения сообщения:
```java
long deliveryTag = delivery.getEnvelope().getDeliveryTag();

channel.basicAck(deliveryTag, false);
```

7️⃣ **Нет методов для сериализации/десериализации:** Класс не предоставляет встроенных методов для преобразования тела сообщения в строку или другие форматы. Это нужно делать вручную:
```java
// Для текстовых сообщений
String text = new String(delivery.getBody(), StandardCharsets.UTF_8);

// Для JSON
ObjectMapper mapper = new ObjectMapper();
MyObject obj = mapper.readValue(delivery.getBody(), MyObject.class);
```

## 💎 Best Practices 💎

✅ **Всегда копируйте тело сообщения** при необходимости его модификации:
```java
byte[] originalBody = delivery.getBody();

byte[] safeCopy = originalBody != null 
    ? Arrays.copyOf(originalBody, originalBody.length) 
    : new byte[0];
```

✅ **Используйте безопасное приведение типов** при работе с телом сообщения:
```java
try {
    String message = new String(
	    delivery.getBody(), 
        delivery.getProperties() != null && 
	        delivery.getProperties().getContentEncoding() != null
	            ? delivery.getProperties().getContentEncoding()
	            : StandardCharsets.UTF_8.name()
	);
            
} catch (UnsupportedEncodingException e) {
    // fallback to UTF-8
    String message = new String(delivery.getBody(), StandardCharsets.UTF_8);
}
```

✅ **Извлекайте deliveryTag для подтверждений** как можно раньше:
```java
DeliverCallback deliverCallback = (consumerTag, delivery) -> {
    long deliveryTag = delivery.getEnvelope().getDeliveryTag();
    // Обработка сообщения
    channel.basicAck(deliveryTag, false);
};
```

✅ **Проверяйте свойства перед использованием**:
```java
AMQP.BasicProperties props = delivery.getProperties();

if (props != null) {
    Map<String, Object> headers = props.getHeaders();
    String messageId = props.getMessageId();
    // работа с заголовками и свойствами
}
```

✅ **Логируйте ключевую информацию** для отладки:
```java
logger.debug("Received message: deliveryTag={}, exchange={}, routingKey={}, bodySize={}",
    delivery.getEnvelope().getDeliveryTag(),
    delivery.getEnvelope().getExchange(),
    delivery.getEnvelope().getRoutingKey(),
    delivery.getBody() != null ? delivery.getBody().length : 0);
```

✅ **Используйте try-with-resources для обработки потоков** при работе с бинарными данными:
```java
if (delivery.getBody() != null) {
    try (ByteArrayInputStream bis = 
	    new ByteArrayInputStream(delivery.getBody()
	);
         ObjectInputStream ois = new ObjectInputStream(bis)) {
        Object obj = ois.readObject();
        // обработка объекта
    }
}
```

# Пример использования
```java
// Пример обработчика доставки сообщений
DeliverCallback deliverCallback = (consumerTag, delivery) -> {
    try {
        // 1. Получаем основную информацию о сообщении
        Envelope envelope = delivery.getEnvelope();
        AMQP.BasicProperties properties = delivery.getProperties();
        
        System.out.println("Received message:");
        System.out.println("  Consumer tag: " + delivery.getConsumerTag());
        System.out.println("  Delivery tag: " + envelope.getDeliveryTag());
        System.out.println("  Exchange: " + envelope.getExchange());
        System.out.println("  Routing key: " + envelope.getRoutingKey());
        
        // 2. Проверяем наличие тела сообщения
        if (delivery.getBody() == null) {
            System.out.println("  Message has no body");
            channel.basicAck(envelope.getDeliveryTag(), false);
            return;
        }
        
        // 3. Обрабатываем свойства сообщения
        if (properties != null) {
            System.out.println("  Content type: " + properties.getContentType());
            System.out.println("  Message ID: " + properties.getMessageId());
            System.out.println("  Timestamp: " + properties.getTimestamp());
            
            // Работа с заголовками
            Map<String, Object> headers = properties.getHeaders();
            if (headers != null && headers.containsKey("custom-header")) {
                System.out.println("  Custom header: " + headers.get("custom-header"));
            }
        }
        
        // 4. Декодируем тело сообщения
        String contentEncoding = properties != null && properties.getContentEncoding() != null
            ? properties.getContentEncoding()
            : StandardCharsets.UTF_8.name();
        
        String messageBody;
        try {
            messageBody = new String(delivery.getBody(), contentEncoding);
        } catch (UnsupportedEncodingException e) {
            // Fallback to UTF-8
            messageBody = new String(delivery.getBody(), StandardCharsets.UTF_8);
        }
        
        System.out.println("  Body: " + messageBody);
        System.out.println("  Body size: " + delivery.getBody().length + " bytes");
        
        // 5. Обработка бизнес-логики
        processMessage(messageBody, properties);
        
        // 6. Подтверждаем обработку сообщения
        channel.basicAck(envelope.getDeliveryTag(), false);
        
    } catch (Exception e) {
        System.err.println("Error processing message: " + e.getMessage());
        
        // 7. В случае ошибки - отклоняем сообщение с возвратом в очередь
        channel.basicNack(delivery.getEnvelope().getDeliveryTag(), false, true);
    }
};

// Регистрация обработчика
channel.basicConsume("my_queue", false, deliverCallback, consumerTag -> {
    System.out.println("Consumer cancelled: " + consumerTag);
});

// Вспомогательный метод для обработки
private void processMessage(String body, AMQP.BasicProperties properties) {
    // Реализация бизнес-логики
    System.out.println("Processing: " + body);
    
    // Пример: работа с разными типами контента
    if (properties != null && "application/json".equals(properties.getContentType())) {
        try {
            ObjectMapper mapper = new ObjectMapper();
            Map<String, Object> json = mapper.readValue(body, Map.class);
            // Обработка JSON
        } catch (JsonProcessingException e) {
            throw new RuntimeException("Invalid JSON", e);
        }
    }
}
```
