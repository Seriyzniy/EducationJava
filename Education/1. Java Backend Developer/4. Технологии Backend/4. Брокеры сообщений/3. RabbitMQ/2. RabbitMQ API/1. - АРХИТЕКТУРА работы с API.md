# 1. Базовая конструкция подключения
Для подключения к RabbitMQ, в любом из случаев происходит цепочка
```java
// 1. ConnectionFactory
ConnectionFactory cf = new ConnectionFactory();
cf.setHost("localhost");  
cf.setPort(5672);  
cf.setUsername("username");  
cf.setPassword("userPassword");


// 2. Connection
Connection connection = cf.newConnection()

// 3. Channel
Channel channel = connection.createChannel()
```

### Важные примечания
1. **Channel не thread-safe** — используйте отдельный channel на поток
    
2. **Connection thread-safe** — можно использовать в нескольких потоках
    
3. **Автоматическое восстановление** — нужно реализовывать вручную или использовать `AutomaticRecovery`
    
4. **Ресурсы нужно закрывать** — всегда закрывайте channel и connection в finally-block
    
5. **Default Exchange** — работает только с существующими очередями (routingKey = имя очереди)

## 1. Declaring components
После создания `Channel` можно декларировать инфраструктуру RabbitMQ. 

Декларация **идемпотентна** — повторный вызов с теми же параметрами не вызовет ошибки, но с разными параметрами приведет к исключению.

### 1.1 Exchange (Обменник)
```java
// Декларация обменника
channel.exchangeDeclare(
    "orders_exchange",  // exchange: имя обменника
    "direct",           // type: тип (direct, fanout, topic, headers)
    true,               // durable: сохранится после перезагрузки брокера
    false,              // autoDelete: удалится, когда отвяжутся все очереди
    false,              // internal: нельзя публиковать напрямую
    null                // arguments: дополнительные аргументы (Map)
);

// Пассивная проверка существования (не создает, если не существует)
channel.exchangeDeclarePassive("orders_exchange");
```

### 1.2 Queue (Очередь)
```java
// Декларация очереди
channel.queueDeclare(
    "orders_queue",     // queue: имя очереди
    true,               // durable: устойчивая очередь
    false,              // exclusive: эксклюзивная для соединения
    false,              // autoDelete: удалится при отписке потребителя
    null                // arguments: Map с дополнительными параметрами
);

// С автогенерацией имени (сервер вернет сгенерированное имя)
AMQP.Queue.DeclareOk result = channel.queueDeclare();
String generatedQueueName = result.getQueue(); // "amq.gen-XXXX"

// Пассивная проверка существования
channel.queueDeclarePassive("orders_queue");
```

#### 1.2.1 Аргументы очереди (arguments)
```java
Map<String, Object> args = new HashMap<>();
args.put("x-message-ttl", 60000);           // TTL сообщений: 60 секунд
args.put("x-expires", 7200000);             // TTL очереди: 2 часа
args.put("x-max-length", 10000);            // Макс. количество сообщений
args.put("x-max-length-bytes", 104857600);  // Макс. размер: 100MB

args.put("x-dead-letter-exchange", "dlx");  // Dead Letter Exchange
args.put("x-dead-letter-routing-key", "dead.orders");

args.put("x-max-priority", 10);             // Поддержка приоритетов (0-255)
args.put("x-queue-mode", "lazy");           // Ленивый режим (диск > RAM)


channel.queueDeclare("orders_queue", true, false, false, args);
```

### 1.3 Binding (Привязка очереди к обменнику)
```java
// Привязка очереди к обменнику с routing key
channel.queueBind(
    "orders_queue",     // queue: имя очереди
    "orders_exchange",  // exchange: имя обменника
    "order.created"     // routingKey: ключ маршрутизации
);

// Привязка с аргументами (редко используется)
channel.queueBind("orders_queue", "orders_exchange", "order.created", null);

// Отвязка очереди от обменника
channel.queueUnbind("orders_queue", "orders_exchange", "order.created");


// Для fanout exchange ROUTING KEY ИГНОРИРУЕТСЯ (можно использовать "")
channel.queueBind("queue1", "fanout_exchange", "");
channel.queueBind("queue2", "fanout_exchange", "");
```

### 1.4 Удаление компонентов
```java
// Удаление очереди (только если пустая/не используется)
channel.queueDelete("orders_queue");

// Удаление очереди с флагами
channel.queueDelete("orders_queue", false, false);
// ifUnused: true - удалить только если нет потребителей
// ifEmpty: true - удалить только если пустая

// Удаление обменника
channel.exchangeDelete("orders_exchange");

// Удаление обменника с флагом ifUnused
channel.exchangeDelete("orders_exchange", true);
```

## 2. Publishing messages (Публикация сообщений)
```java
// Базовая публикация
channel.basicPublish(
    "orders_exchange",  // exchange: имя обменника
    "order.created",    // routingKey: ключ маршрутизации
    null,               // props: свойства сообщения (BasicProperties)
    "Order data".getBytes() // body: тело сообщения (byte[])
);

// Публикация в Default Exchange (прямо в очередь)
channel.basicPublish(
    "",                 // пустая строка = Default Exchange
    "orders_queue",     // routingKey = имя очереди (ОБЯЗАТЕЛЬНО СУЩЕСТВУЮЩЕЙ!)
    null,
    messageBody
);
```

### 2.1 BasicProperties (Свойства сообщения)
```java
// Создание свойств сообщения
AMQP.BasicProperties props = new AMQP.BasicProperties.Builder()
    .contentType("application/json")     // тип контента
    .contentEncoding("UTF-8")            // кодировка
    .deliveryMode(2)                     // 1=non-persistent, 2=persistent
    .priority(5)                         // приоритет (0-9)
    .correlationId("corr-12345")         // для RPC
    .replyTo("reply_queue")              // очередь для ответа (RPC)
    .expiration("60000")                 // TTL сообщения в мс
    .messageId("msg-12345")              // ID сообщения
    .timestamp(new Date())               // временная метка
    .type("order.created")               // тип сообщения
    .userId("producer-app")              // ID пользователя
    .appId("order-service")              // ID приложения
    .headers(Map.of(                     // произвольные заголовки
        "version", "1.0",
        "source", "mobile-app"
    ))
    .build();


channel.basicPublish("exchange", "routing.key", props, body);
```

### 2.2 Подтверждения публикации
```java
// Включение подтверждений (Publisher Confirms)
channel.confirmSelect();
```

#### СИНХРОННОЕ ожидание
```java
// Синхронное ожидание подтверждения
boolean confirmed = channel.waitForConfirms(); // throws InterruptedException

if (confirmed) {
    System.out.println("Сообщение подтверждено брокером");
}
```

#### АСИНХРОННОЕ ожидание
```java
// Асинхронные callback'и
channel.addConfirmListener(new ConfirmListener() {
    @Override
    public void handleAck(long deliveryTag, boolean multiple) {
        // Сообщение получено брокером
    }
    
    @Override
    public void handleNack(long deliveryTag, boolean multiple) {
        // Брокер не смог принять сообщение
    }
});
```

## 3. Consuming messages (Потребление сообщений)

### 3.1 Базовое потребление
```java
// Создание потребителя с callback
DeliverCallback deliverCallback = (consumerTag, delivery) -> {
    String message = new String(delivery.getBody(), "UTF-8");
    String routingKey = delivery.getEnvelope().getRoutingKey();
    
    System.out.println(
	    "Получено: " + message + " [key: " + routingKey + "]");
    
    // Подтверждение обработки (если autoAck=false)
    channel.basicAck(delivery.getEnvelope().getDeliveryTag(), false);
};

CancelCallback cancelCallback = consumerTag -> {
    System.out.println("Потребление отменено: " + consumerTag);
};

// Подписка на очередь
String consumerTag = channel.basicConsume(
    "orders_queue",     // queue: имя очереди
    false,              // autoAck: автоматическое подтверждение
    deliverCallback,
    cancelCallback
);


// Отмена подписки
channel.basicCancel(consumerTag);
```

### 3.2 QoS (Качество обслуживания, prefetch)
```java
// Настройка prefetch count (сколько сообщений отправлять без подтверждения)
channel.basicQos(
    10,     // prefetchCount: максимум неподтвержденных сообщений
    false   // global: true=на всё соединение, false=на канал
);

// Для строгой очередности (1 сообщение за раз)
channel.basicQos(1, false);
```

### 3.3 Подтверждения и отказы
```java
// Ручное подтверждение (ack)
channel.basicAck(
    deliveryTag,    // тег доставки из envelope
    false          // multiple: подтвердить все до этого тега
);


// Отказ с возвратом в очередь (nack)
channel.basicNack(
    deliveryTag,
    false,         // multiple
    true           // requeue: вернуть в очередь
);

// Отказ без multiple (reject)
channel.basicReject(
    deliveryTag,
    false          // requeue
);
```

# 2. Добавление СЛУШАТЕЛЕЙ
| Компонент              | Зачем нужен                        | Когда использовать            |
| ---------------------- | ---------------------------------- | ----------------------------- |
| **ShutdownListener**   | Отслеживание неожиданных разрывов  | Всегда, для graceful shutdown |
| **BlockedListener**    | Реакция на блокировку брокера      | Production с high-throughput  |
| **ReturnListener**     | Обработка недоставленных сообщений | Когда mandatory=true          |
| **ConfirmListener**    | Гарантия доставки до брокера       | Транзакционные сообщения      |
| **Consumer listeners** | Расширенная обработка consumer     | Сложная бизнес-логика         |

## 6.1 Connection Listeners (Слушатели соединения)

### 6.1.1 ShutdownListener (Слушатель блокировки/закрытия)
Отслеживание неожиданных разрывов соединения для graceful shutdown или переподключения.
```java
// Добавление слушателя блокировки соединения или канала
connection.addShutdownListener(cause -> {
    System.out.println("Соединение закрыто!");
    System.out.println("Причина: " + cause.getMessage());
    System.out.println("Инициировано приложением: " + cause.isInitiatedByApplication());
    System.out.println("Hard ошибка: " + cause.isHardError());
});

channel.addShutdownListener(cause -> {
    // Канал закрыт (например, protocol exception)
    System.out.println("Канал закрыт: " + cause.getReason());
});

// Получение shutdown cause
ShutdownSignalException shutdownCause = connection.getCloseReason();
```

### 6.1.2 BlockedListener (Слушатель блокировки ресурсов)
**RabbitMQ блокирует producers при нехватке ресурсов**. 
Этот listener позволяет приостановить отправку, избегая ошибок.
```java
// Слушатель блокировки соединения при нехватке памяти/диска
connection.addBlockedListener(new BlockedListener() {
    @Override
    public void handleBlocked(String reason) {
        // Брокер заблокировал соединение (Flow Control активирован)
        System.out.println("⚠️ Соединение заблокировано. Причина: " + reason);
        // Приостановить отправку сообщений
    }
    
    @Override
    public void handleUnblocked() {
        // Блокировка снята
        System.out.println("✅ Соединение разблокировано");
        // Возобновить отправку сообщений
    }
});
```

## 6.2 ReturnListener (Слушатель возвращаемых сообщений)
*Отслеживание сообщений*, **которые не смогли быть маршрутизированы ни в одну очередь** (например, неверный routing key).
```java
// Включение mandatory флага для отслеживания невозможных к доставке сообщений
channel.addReturnListener((replyCode, replyText, exchange, routingKey, 
                          properties, body) -> {
    System.out.println("Сообщение возвращено брокером!");
    System.out.println("Exchange: " + exchange);
    System.out.println("Routing key: " + routingKey);
    System.out.println(
	    "Причина: " + replyText + " (код: " + replyCode + ")");
    
    // Обработка недоставленного сообщения
    // - записать в dead letter queue
    // - отправить уведомление
    // - залогировать ошибку
});

// Публикация с mandatory = true (сообщение вернется, если не доставлено)
channel.basicPublish(
	"exchange", 
	"routing.key", 
	true, // ← mandatory=true
    new AMQP.BasicProperties(),
    "Сообщение".getBytes());
```

## 6.3 ConfirmListener (Слушатель подтверждений от брокера)
**Гарантия доставки сообщений до брокера** (==at-least-once==). 

Критически важно для финансовых транзакций, заказов.
```java
// Включение publisher confirms
channel.confirmSelect();
```

### Асинхронный слушатель подтверждений
```java
// Добавление асинхронного слушателя подтверждений
channel.addConfirmListener(new ConfirmListener() {
    @Override
    public void handleAck(long deliveryTag, boolean multiple) {
        // Брокер успешно принял сообщение на диск (persistent)
        System.out.println("✅ Подтверждено deliveryTag: " + deliveryTag);
        if (multiple) {
            System.out.println("   (и все предыдущие до " + deliveryTag + ")");
        }
        // Можно удалить из локальной очереди на отправку
    }
    
    @Override
    public void handleNack(long deliveryTag, boolean multiple) {
        // Брокер НЕ смог принять сообщение (например, нет места на диске)
        System.out.println("❌ Отклонено deliveryTag: " + deliveryTag);
        // Повторить отправку или обработать ошибку
    }
});
```

### С использованием CompletableFuture
```java
// Альтернатива: использование CompletableFuture
CompletableFuture<Boolean> future = new CompletableFuture<>();

channel.addConfirmListener((deliveryTag, multiple) -> {
    future.complete(true);
}, (deliveryTag, multiple) -> {
    future.completeExceptionally(new Exception("Nack received"));
});

// Ожидание подтверждения с таймаутом
try {
    boolean confirmed = channel.waitForConfirms(5000); // 5 секунд
} catch (InterruptedException | TimeoutException e) {
    // Обработка таймаута
}
```

# 3. Flow Control (Управление потоком)
**Flow Control — это механизм защиты брокера от перегрузки**, когда производители (producers) отправляют сообщения быстрее, чем потребители (consumers) могут их обрабатывать или быстрее, чем брокер может сохранять на диск.

## Настройка Flow Control на клиенте
```java
// 1. Ограничение размера канала (frame max)
ConnectionFactory factory = new ConnectionFactory();
factory.setRequestedFrameMax(131072); // 128KB max frame size
factory.setRequestedHeartbeat(60);    // heartbeat 60 секунд

// 2. Ограничение количества каналов
factory.setRequestedChannelMax(100);  // макс 100 каналов на соединение

// 3. Включение automatic recovery (рекомендуется)
factory.setAutomaticRecoveryEnabled(true);
factory.setNetworkRecoveryInterval(5000); // попытки каждые 5 секунд

// 4. Топология восстановления
factory.setTopologyRecoveryEnabled(true); // восстанавливает очереди/обменники

// 5. Настройка thread pool для обработки
factory.setSharedExecutor(Executors.newFixedThreadPool(10));
```

## Ручное управление потоком на канале
```java
// Приостановка канала (остановка доставки сообщений)
channel.flow(false); // пауза
// ...
channel.flow(true);  // возобновление


// Проверка статуса
boolean isActive = channel.isFlowActive();


// Обработка уведомлений о flow control
channel.getConnection().addBlockedListener(new BlockedListener() {
    @Override
    public void handleBlocked(String reason) {
        // Автоматически приостанавливаем все каналы
        for (Channel ch : getAllChannels()) {
            if (ch.isOpen()) {
                ch.flow(false);
            }
        }
    }
    
    @Override  
    public void handleUnblocked() {
        // Возобновляем каналы
        for (Channel ch : getAllChannels()) {
            if (ch.isOpen()) {
                ch.flow(true);
            }
        }
    }
});
```

## Backpressure на Producer
```java
// Реализация backpressure при помощи confirms
private final ConcurrentNavigableMap<Long, Message> outstandingConfirms = 
    new ConcurrentSkipListMap<>();


channel.confirmSelect();


channel.addConfirmListener(
	(deliveryTag, multiple) -> {
	    // Удаляем подтвержденные сообщения
	    if (multiple) {
	        outstandingConfirms.headMap(deliveryTag, true).clear();
	    } else {
	        outstandingConfirms.remove(deliveryTag);
	    }
	    
	    // Проверяем лимит неподтвержденных сообщений
	    if (outstandingConfirms.size() < 1000) { // наш лимит
	        resumeSending(); // возобновляем отправку
	    }
	}, 
	(deliveryTag, multiple) -> {
	    // Обработка nack
	    Message failed = outstandingConfirms.get(deliveryTag);
	    // retry логика
});

// Отправка с отслеживанием
public void sendWithBackpressure(Message msg) {
    if (outstandingConfirms.size() >= 1000) {
        pauseSending(); // приостанавливаем до confirm
        return;
    }
    
    long deliveryTag = channel.getNextPublishSeqNo();
    outstandingConfirms.put(deliveryTag, msg);
    channel.basicPublish(...);
}
```
