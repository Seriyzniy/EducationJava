
> [!NOTE] `Удалённый репозиторий (remote)` — это `копия твоего локального репозитория`,  
> расположенная **на другом сервере** (например, GitHub, GitLab, Bitbucket, или на корпоративном сервере). 
> 
> Git хранит не «подключение к серверу», а просто **ссылку (URL)** на другой репозиторий.
>   
> То есть Git не работает в режиме «клиент–сервер» — это **распределённая система**.
> 

> [!NOTE] ⭐НИКАКОЙ автоматической СИНХРОНИЗАЦИИ НЕТ⭐
>

# 📕 1. Клонирование репозитория
Если В ЛОКАЛЬНОМ репозитории НЕТ УДАЛЕННОГО репозитория, его МОЖНО СКАЧАТЬ 
```bash
git clone <url>
```

> [!NOTE] **Скачать можно по протоколу**:
> - `https`
> - `ssh`

При этом Git
- создает папку, куда помещаются все файлы рабочей директории
-  создает подпапку `.git`
- создает связь между локальным и удаленным репозиторием

## 📙 1. Установка связи с удаленным репозиторием 

> [!NOTE] `ORIGIN` - имя удаленного репозитория по умолчанию
> 💡 `origin` — это стандартное имя, которое Git присваивает удалённому репозиторию по умолчанию.

> [!NOTE] ⭐⭐ `ПЕРВОЕ связывание веток` ⭐⭐
> Чтобы установить связь между ветками, НЕОБХОДИМО ВЫПОЛНИТЬ 
> ```bash
> git push -u origin <branch>
> ```
> Чтобы СВЯЗАТЬ локальную ветку с удаленной
> 
> ❗❗❗А после этого, можно использовать обычные `push` и `pull`❗❗❗

### Основные команды

#### 🔶 1.  ⭐**Добавление удалённого репозитория**⭐
```bash
git remote add <имя_для_обращения> <url>
```
Пример:

```bash
git remote add origin https://github.com/user/project.git
```

---

#### 🔶 2.  Проверка списка удалённых репозиториев
```bash
git remote -v
```
Показывает список всех удалённых репозиториев и их URL.  
Пример вывода:
```bash
origin  https://github.com/user/project.git (fetch)
origin  https://github.com/user/project.git (push)
```

---

#### 🔶 3. Удаление удалённого репозитория
```bash
git remote remove <имя>
```

или устаревший вариант:
```bash
git remote rm <имя>
```

---

#### 🔶 4.  Переименование
```bash
git remote rename <старое> <новое>
```

---

#### 🔶 5.  Получение изменений с сервера
```bash
git fetch [remote]
```

📘 Пример:
```bash
git fetch origin
```

Эта команда:
- загружает все **новые коммиты**, **ветки** и **теги** с удалённого репозитория;
    
- ❗❗❗**не объединяет** их с твоими локальными ветками.

После `fetch` ветки с сервера сохраняются как **удалённые ветки**:
```bash
origin/main origin/feature-1
```

---

#### 🔶 6. 💡 Слияние (PULL) изменений с сервера 💡
После `fetch` можно объединить изменения вручную:

```bash
git merge origin/main
```

или одной командой — **автоматически**:
```bash
git pull
```
❗❗`git pull` = `git fetch` + `git merge`❗❗

---

#### 🔶 7. ⭐ **Отправка (PUSH) изменений** на сервер⭐
```bash
git push [remote] [branch]
```

Пример:
```bash
git push origin main
```

> [!NOTE] ⭐Отправляет твою локальную ветку `main` в удалённый репозиторий `origin`.⭐
> 

---

#### 🔶 8. Удаление ветки на сервере
```bash
git push origin --delete <branch>
```

---

#### 🔶 9.  Получение тегов
```bash
git fetch --tags
```

или сразу всё:
```bash
git pull --tags
```

---

#### 🔶 10. Отправка тегов
```bash
git push origin --tags
```


---
# 💡 Полезные приёмы

## 📙Проверить, какие ветки есть локально и на сервере
```bash
git branch -a
```

## 📙Обновить все ссылки на серверные ветки

```bash
git fetch --all
```

## 📙Настроить upstream (связать локальную ветку с удалённой)

```bash
git push -u origin main
```

Теперь можно просто писать:

```bash
git push 
git pull
```
