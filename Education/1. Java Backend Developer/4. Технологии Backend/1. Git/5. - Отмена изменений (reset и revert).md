| Команда          | Что делает                                                 | Меняет историю | Применение                                                     |
| ---------------- | ---------------------------------------------------------- | -------------- | -------------------------------------------------------------- |
| **`git reset`**  | «Отматывает» указатель ветки назад                         | ✅ Да           | Отменить локальные коммиты (ещё не пушенные)                   |
| **`git revert`** | Создаёт новый коммит, который «отменяет» изменения другого | 🚫 Нет         | Безопасно отменить изменения, уже попавшие в общий репозиторий |

# 📕 1. `git reset`

`git reset` **перемещает указатель HEAD** (а значит и ветку) на другой коммит,  
и _в зависимости от режима_ — может менять staging area (индекс) и рабочие файлы.

```css
A --- B --- C --- D (HEAD -> main)
```

Ты хочешь “откатиться” к коммиту `B`.

```bash
git reset [--soft | --mixed | --hard] <commit>
```

|Опция|HEAD|Индекс (staging)|Рабочие файлы|Типичный случай|
|---|---|---|---|---|
|`--soft`|Перемещается|❌ Не трогает|❌ Не трогает|Объединение коммитов|
|`--mixed`|Перемещается|✅ Сбрасывает|❌ Не трогает|Пересоздание коммитов|
|`--hard`|Перемещается|✅ Сбрасывает|✅ Стирает|Полный откат|

---
# 📕 2. `git revert`

`git revert` **создаёт новый коммит**, который _отменяет изменения указанного коммита_,  
вместо того чтобы удалять его из истории.

```bash
git revert <commit>
```

```css
A --- B --- C --- D (HEAD -> main)
```

Хочешь отменить изменения из `C`.
```bash
git revert C
```


Результат:
```css
A --- B --- C --- D --- R (HEAD -> main)
```

- `R` — новый коммит, который «обратен» коммиту `C`;
    
- История **остаётся неизменной**, то есть `C` по-прежнему в истории, но его изменения «перекрыты»;
    
- Подходит для **общих веток** (`main`, `develop` и т.п.), где уже был `push`.
    

📘 Пример: ты залил неудачный релиз в main — можно `revert`-нуть его, не ломая историю.

---
# 💡 Пример из практики

✅ **До push’а** (работа локально):
```bash
git reset --hard HEAD~1
```
— просто удаляешь последний коммит.


✅ **После push’а** (работа в общей ветке):
```bash
git revert HEAD
```
— создаёшь новый коммит, который отменяет изменения, но сохраняет историю.
