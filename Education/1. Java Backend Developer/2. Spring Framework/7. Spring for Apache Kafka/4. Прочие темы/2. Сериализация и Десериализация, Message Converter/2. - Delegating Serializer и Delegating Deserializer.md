# 1. Использование заголовков (Using Headers)
В версии **2.3** были введены `DelegatingSerializer` и `DelegatingDeserializer`, которые позволяют производить (producer) и потреблять (consumer) записи с **разными типами ключей и/или значений**.

Продюсеры должны установить заголовок:
- `DelegatingSerializer.VALUE_SERIALIZATION_SELECTOR` — для выбора сериализатора значения
    
- `DelegatingSerializer.KEY_SERIALIZATION_SELECTOR` — для выбора сериализатора ключа

Значение этого заголовка — это «селектор», который используется для выбора, какой именно сериализатор применять.  
Если соответствие не найдено — выбрасывается `IllegalStateException`.

Для входящих записей (consumer side) десериализатор использует те же заголовки (выше), чтобы выбрать десериализатор.  
Если соответствие не найдено или заголовок отсутствует, возвращается исходный `byte[]`.

Вы можете настроить **карту селектор → Serializer/Deserializer** через конструктор, или через свойства Kafka producer/consumer, используя ключи:
- `DelegatingSerializer.VALUE_SERIALIZATION_SELECTOR_CONFIG`
    
- `DelegatingSerializer.KEY_SERIALIZATION_SELECTOR_CONFIG`


Для сериализатора, свойство producer может быть: 
`Map<String, Object>`:  
    ключ — селектор  
    значение — экземпляр Serializer, класс Serializer или полное имя класса

Также может быть строка, содержащая элементы карты, разделённые запятыми, как показано ниже.

Для десериализатора consumer также может использовать `Map<String, Object>` или строку такого же формата.
### Синтаксис настройки через свойства:
```java
"ключ:Полное.Имя.Класса.Сериализатора, ключ:Другой.Сериализатор"
```

```java
producerProps.put(
	DelegatingSerializer.VALUE_SERIALIZATION_SELECTOR_CONFIG,
    "thing1:com.example.MyThing1Serializer, thing2:com.example.MyThing2Serializer")


consumerProps.put(
	DelegatingDeserializer.VALUE_SERIALIZATION_SELECTOR_CONFIG,
    "thing1:com.example.MyThing1Deserializer, thing2:com.example.MyThing2Deserializer")
```
- Если в сообщении есть header с ключом `thing1` → используется `MyThing1Serializer`
    
- Если есть header с ключом `thing2` → используется `MyThing2Serializer`


Продюсеры затем должны установить заголовок  
`DelegatingSerializer.VALUE_SERIALIZATION_SELECTOR`  
со значением `thing1` или `thing2`.

Этот подход поддерживает отправку **разных типов в один и тот же топик (или в разные топики)**.

# 2. По типу (By Type)
В версии **2.8** был введён `DelegatingByTypeSerializer`.
```java
@Bean
public ProducerFactory<Integer, Object> producerFactory(Map<String, Object> config) {
    return new DefaultKafkaProducerFactory<>(
		    config,
            null, 
            new DelegatingByTypeSerializer(Map.of(
				byte[].class, new ByteArraySerializer(),
				Bytes.class, new BytesSerializer(),
				String.class, new StringSerializer())));
}
```

Начиная с версии **2.8.3**, можно настроить сериализатор так, чтобы он проверял, является ли ключ карты «назначаемым» (`assignable from`) для целевого объекта.  
Это полезно, когда один сериализатор способен сериализовать подклассы.

В этом случае, если есть несколько подходящих совпадений, следует предоставить **упорядоченную Map**, например `LinkedHashMap`.

# 3. По топику (By Topic)
Начиная с версии **2.8**, `DelegatingByTopicSerializer` и `DelegatingByTopicDeserializer` позволяют выбирать сериализатор/десериализатор на основании **имени топика**.

Для поиска соответствующего экземпляра используются **Regex-паттерны**.

Карта может быть сконфигурирована через конструктор или через свойства (строка вида `pattern:serializer`, разделённая запятыми).
```java
producerConfigs.put(
	DelegatingByTopicSerializer.VALUE_SERIALIZATION_TOPIC_CONFIG,
	"topic[0-4]:" + ByteArraySerializer.class.getName()
	+ ", topic[5-9]:" + StringSerializer.class.getName());
...
consumerConfigs.put(DelegatingByTopicDeserializer.VALUE_SERIALIZATION_TOPIC_CONFIG,
            "topic[0-4]:" + ByteArrayDeserializer.class.getName()
        + ", topic[5-9]:" + StringDeserializer.class.getName());
```

## Пример с бином:
```java
@Bean
public ProducerFactory<Integer, Object> producerFactory(Map<String, Object> config) {
    return new DefaultKafkaProducerFactory<>(
		    config,
            new IntegerSerializer(),
            new DelegatingByTopicSerializer(
	            Map.of(
                    Pattern.compile("topic[0-4]"), 
                    new ByteArraySerializer(),
                    Pattern.compile("topic[5-9]"), 
                    new StringSerializer())
                ),
                new JsonSerializer<Object>());  // default
}
```

Вы можете указать **серилизатор/десериализатор по умолчанию**, когда нет совпадений:
- `DelegatingByTopicSerialization.KEY_SERIALIZATION_TOPIC_DEFAULT`
    
- `DelegatingByTopicSerialization.VALUE_SERIALIZATION_TOPIC_DEFAULT`
    

Дополнительное свойство:

- `DelegatingByTopicSerialization.CASE_SENSITIVE` (по умолчанию `true`)
    

Если установить его в `false`, поиск топика станет **регистронезависимым**.
