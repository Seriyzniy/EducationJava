Начиная с версии 3.2, методы `@KafkaListener` (и `@KafkaHandler`) могут быть объявлены с асинхронными типами возврата, позволяя отправлять ответ асинхронно. 

Поддерживаемые типы включают `CompletableFuture<?>`, `Mono<?>` и Kotlin suspend-функции.

> Асинхронные возвраты в `@KafkaListener` используются **исключительно** для request-reply сценариев и требуют явного указания топика для ответа через `@SendTo` или настройки `ReplyingKafkaTemplate`.

# Примеры
## С CompletableFuture:
```java
@KafkaListener(id = "myListener", topics = "myTopic")
public CompletableFuture<String> listen(String data) {
    ...
    CompletableFuture<String> future = new CompletableFuture<>();
    future.complete("done");
    return future;
}
```

## Ключевые особенности:

**Автоматическая настройка AckMode:**
- При обнаружении асинхронных типов возврата `AckMode` автоматически устанавливается в `MANUAL`
    
- Включаются **out-of-order commits** (неупорядоченные подтверждения)
    
- Подтверждение происходит **асинхронно** при завершении асинхронной операции
    

**Обработка ошибок:**
- При завершении асинхронного результата с ошибкой - восстановление сообщения зависит от error handler'а контейнера
    
- Если исключение возникает **до создания** асинхронного объекта - вы **ДОЛЖНЫ** перехватить его и вернуть соответствующий объект
    
- Если настроен `KafkaListenerErrorHandler` - он вызывается после сбоя

## ⚠️ **Критически важные моменты:**

Обязательная обработка ошибок:
```java
@KafkaListener(id = "asyncListener", topics = "myTopic")
public CompletableFuture<String> listen(String data) {
    try {
        // Может выбросить исключение ДО создания Future
        SomeObject obj = parseData(data);
        
        CompletableFuture<String> future = processAsync(obj);
        return future;
    } catch (Exception e) {
        // ОБЯЗАТЕЛЬНО перехватываем исключения на синхронной стадии
        CompletableFuture<String> failed = new CompletableFuture<>();
        failed.completeExceptionally(e);
        return failed;
    }
}
```
