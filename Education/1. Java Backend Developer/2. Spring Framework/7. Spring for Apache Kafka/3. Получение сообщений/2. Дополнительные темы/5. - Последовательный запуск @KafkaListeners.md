## Общий use-case:
Запуск listener'а **после того**, как другой listener обработал все записи в топике. Например: загрузка содержимого compacted-топиков в память перед обработкой записей из других топиков.

## Решение: ContainerGroupSequencer (версия 2.7.3+)
Компонент группирует контейнеры и запускает их последовательно, когда все контейнеры в текущей группе становятся idle.

### Пример 
```java
@KafkaListener(
	id = "listen1", 
	topics = "topic1", 
	containerGroup = "g1", 
	concurrency = "2")
public void listen1(String in) {
}

@KafkaListener(
	id = "listen2", 
	topics = "topic2", 
	containerGroup = "g1", 
	concurrency = "2")
public void listen2(String in) {
}

@KafkaListener(
	id = "listen3", 
	topics = "topic3", 
	containerGroup = "g2", 
	concurrency = "2")
public void listen3(String in) {
}

@KafkaListener(
	id = "listen4", 
	topics = "topic4", 
	containerGroup = "g2", 
	concurrency = "2")
public void listen4(String in) {
}

@Bean
ContainerGroupSequencer sequencer(
	KafkaListenerEndpointRegistry registry) 
{
    return new ContainerGroupSequencer(
	    registry, 
	    5000, 
	    "g1", 
	    "g2");
}
```

#### Как это работает:
1. **4 listener'а в двух группах**: g1 и g2
    
2. **При инициализации** контекста:
    - Устанавливает `autoStartup = false` для всех контейнеров в указанных группах
        
    - Устанавливает `idleEventInterval` (5000ms в примере)
        
3. **При запуске sequencer'а**:
    - Запускаются контейнеры **первой группы** (g1)
        
    - При получении `ListenerContainerIdleEvent` останавливаются отдельные child-контейнеры
        
    - Когда **все контейнеры в группе остановлены** → запускается **следующая группа** (g2)
        

#### Дополнительные настройки:
- **`stopLastGroupWhenIdle`** - остановить последнюю группу при idle (по умолчанию `false`)
    
- **Нет ограничений** на количество групп или контейнеров в группе

### Важное замечание:
Ранее создавались бины типа `Collection<MessageListenerContainer>` с именем группы. 

Теперь создаются бины типа `ContainerGroup` с суффиксом `.group` (например, `g1.group`, `g2.group`). Старые коллекции **deprecated** и будут удалены.
