В основе работы с Consumer в Spring Kafka лежит немного иная концепция чем в обычном Kafka

# 0. Концепция Consumer в Spring Kafka

## 1. Что такое Consumer в Kafka?
**Consumer** - это клиент Kafka, который:
- Подключается к Kafka-кластеру
- Читает сообщения из топиков
- Отслеживает свою позицию (offset) в каждом partition
- Участвует в rebalance при изменении количества потребителей в группе

**На низком уровне (чистый Kafka API):**
```java
// Вручную создаем и управляем consumer
Properties props = new Properties();
props.put(
	"bootstrap.servers", "localhost:9092");
props.put(
	"group.id", "test");

KafkaConsumer<String, String> consumer = 
	new KafkaConsumer<>(props);


consumer.subscribe(Arrays.asList("topic"));

while (true) {
    ConsumerRecords<String, String> records = 
	    consumer.poll(Duration.ofMillis(100));
    
    for (ConsumerRecord<String, String> record : records) {
        // Обрабатываем каждое сообщение
        processRecord(record);
    }
    // Вручную коммитим offsets
    consumer.commitSync();
}
```


## 2. Listener/Container в Spring Kafka
> [!NOTE] Обертка над Consumer
> Spring Kafka не заменяет Consumer, а ОБЕРТЫВАЕТ его!
### Аналогия:
- **Consumer** = Двигатель автомобиля
    
- **Listener Container** = Весь автомобиль с коробкой передач, рулем, тормозами
    
- **Message Listener** = Водитель, который только говорит "куда ехать"

### Принцип работы
#### 1. Spring создает Consumer через ConsumerFactory
```java
@Bean
public ConsumerFactory<String, String> consumerFactory() {
    // Spring создает настоящие KafkaConsumer-ы
    return new DefaultKafkaConsumerFactory<>(consumerConfigs());
}
```

#### 2. 🔥Container управляет жизненным циклом🔥 Consumer:
```java
@Bean
public ConcurrentKafkaListenerContainerFactory<String, String> containerFactory() 
{
    ConcurrentKafkaListenerContainerFactory<String, String> 
	    factory = 
		    new ConcurrentKafkaListenerContainerFactory<>();
    
    factory.setConsumerFactory(consumerFactory());
    factory.setConcurrency(3); // Создает 3 Consumer-а!
    return factory;
}
```

`ConcurrentKafkaListenerContainerFactory<K, V>` — это дженерик-класс из Spring Kafka, который параметризуется:
- **K** — тип _ключа сообщения_
- **V** — тип _значения сообщения_

#### 3. Вы пишете только бизнес-логику
```java
@Component
public class MyBusinessLogic {
    
    @KafkaListener(
	    topics = "my-topic", 
	    containerFactory = "containerFactory")
    public void handleMessage(String message) {
        // Spring сам создал Consumer, подписался на топик,
        // получает сообщения и вызывает этот метод
        System.out.println("Received: " + message);
    }
}
```

### Под капотом
Когда Spring видит `@KafkaListener` или предоставленный слушатель:
1. **Создает реальные KafkaConsumer-ы** через вашу ConsumerFactory
    
2. 🔥**Запускает контейнер**🔥, который в фоновом потоке:
    - ❗❗Вызывает `consumer.poll()`❗❗
    - Обрабатывает rebalance
    - Управляет коммитами offsets
    - Обрабатывает ошибки
        
3. **Когда приходят сообщения** → вызывает ваш метод `handleMessage()`

#### Как Kafka распределяет партиции
- число consumer-потоков ≤ число партиций
    
- один поток = один consumer instance
    
- одна партиция может быть обработана **только одним consumer** в группе
    
- если потоков больше, чем партиций, — некоторые простаивают

# Принцип работы при получении сообщений в Spring

Здесь чтобы принять сообщения мы описываем:
- Параметры конфигурации для Consumer
- Фабрику Consumer
- **Контейнер слушателей**, которые используют фабрику Consumer
- **Слушатель** - класс или лямбда, который мы передаем в контейнер слушателей, и который отвечает ТОЛЬКО за бизнес логику обработки сообщения


Spring Kafka предоставляет **ДВА способа реализации** этой концепции для приятия сообщений
- Ручное создание всех выше описанных компонентов
- Использование `@KafkaListener` и фабрики контейнеров

Рассмотрим оба

# 📕 1. Конфигурация Consumer
Оба способа начинаются с конфигурации Consumer, и создании фабрики Consumer'ов
## 1. Настройка свойств Consumer и создание фабрики 
Для успешной работы, минимально нужны следующие параметры:
- `bootstrap.servers` — адрес брокера Kafka
- `key.deserializer` и `value.deserializer` — десериализация ключа и значения
	>В параметры передаются классы, которые реализуют `Deserializer` из Kafka API
	
- `group-id` — группа Conusumer (иначе нужно вручную указывать партицию)
- `auto_offset_reset` - политика чтения сообщений "earliest" или "latest"

Пример кода
```java
@Bean  
    public ConsumerFactory<String, UserInfo> consumerFactory() {
	    // 1️⃣ Создание параметров конфигурации
		Map<String, Object> props = new HashMap<String, Object>();  
        props.put(  
            ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG,  
            "localhost:29092,localhost:39092,localhost:49092"  
        );  
        props.put(  
            ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG,  
            StringDeserializer.class  
        );  
        props.put(  
            ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG,  
            CustomUserInfoDeserializer.class  
        );  
        props.put(  
            ConsumerConfig.AUTO_OFFSET_RESET_CONFIG,  
            "earliest"  
        );  
        props.put(  
            ConsumerConfig.GROUP_ID_CONFIG,  
            "custom_consumer"  
        );  
		
		//  2️⃣ Создание фабрики
        return new DefaultKafkaConsumerFactory<String, UserInfo>(props);  
    } 
```

## 2. Способы конфигурации контейнера и слушателей
После конфигурации фабрики Consumer, нужно создать контейнер и слушателя

---
### 1. Контейнер слушателей и слушатель
(низкоуровневый, полный контроль)

Для создания контейнеров используются классы
- Concurrent**MessageListenerContainer**
- Kafka**MessageListenerContainer**

При создании контейнера необходимо передать ему СВОЙСТВА, для этого используется класс `ContainerProperties`. 
- параметры топика
- слушатель

```java
@Bean
public ConcurrentMessageListenerContainer<String, UserInfo> listenerContainer(ConsumerFactory<String, UserInfo> consumerFactory) 
{
	// 1. Создание класса свойств для контейнера
    ContainerProperties containerProps =
            new ContainerProperties("topic-spring-1");
	
	// 2. Добавление слушателя
    containerProps.setMessageListener(
	    new MessageListener<String, UserInfo>() 
	{
        @Override
        public void onMessage(ConsumerRecord<String, UserInfo> record) {
            System.out.println("RECEIVED: " + record);
        }
    });

	// 3. Создание контейнера используя
	// - описанные свойства ContainerProperties
	// - фабрику Consumer
    ConcurrentMessageListenerContainer<String, UserInfo> container =
            new ConcurrentMessageListenerContainer<>(
	            consumerFactory, 
	            containerProps);

    container.setConcurrency(3); // количество Consumer потоков

    return container;
}
```

`ConcurrentMessageListenerContainer`:
- создаёт указанное количество Consumer потоков
    
- каждый поток подписывается на те же топики
    
- Kafka балансирует партиции между Consumer потоками
    
- каждый Consumer вызывает твой Listener при получении сообщения
    
- ackMode управляет тем, когда Consumer отправляет offset commit


Listener вызывается напрямую через callback.

### 2. @KafkaListener и фабрика для контейнера слушателей
(высокоуровневый, удобный)

> [!NOTE] НЕОБХОДИМАЯ аннотация `@EnableKafka`
> Чтобы `@KafkaListener` работал, необходима аннотация `@EnableKafka` над классом конфигурации

В этом случае ты:
1. Создаёшь только Concurrent**KafkaListener**ContainerFactory
    
2. Помечаешь методы `@KafkaListener`
    
3. Spring автоматически создаёт контейнеры в фоне

Этот способ используется в 99% проектов, т.к. он удобный и упрощает код.

#### 1. Создание фабрики контейнера слушателя для @KafkaListegner
```java
@Bean
public ConcurrentKafkaListenerContainerFactory<String, UserInfo> kafkaListenerContainerFactory(
	ConsumerFactory<String, UserInfo> consumerFactory) {

    ConcurrentKafkaListenerContainerFactory<String, UserInfo> factory =
            new ConcurrentKafkaListenerContainerFactory<>();

    factory.setConsumerFactory(consumerFactory);

    factory.setConcurrency(3); // как и в контейнере вручную
    factory.getContainerProperties().setAckMode(AckMode.MANUAL);

    return factory;
}
```

#### 2. Использования @KafkaListener
Этой аннотацией помечаем метод, который служит слушателем. Он выполняет бизнес-логику по обработке сообщения

В аннотации мы указываем:
- топик
- фабрику контейнера слушателя

```java
@KafkaListener(
        topics = "topic-spring-1",
        groupId = "group-1",
        containerFactory = "kafkaListenerContainerFactory"
)
public void receive(UserInfo userInfo) {
    System.out.println("Listener received: " + userInfo);
}
```

Spring:
1. Находит метод с `@KafkaListener`
2. Создаёт для него `ConcurrentMessageListenerContainer`
3. Устанавливает фабрику из `containerFactory`
    
4. Создаёт `MessageListenerAdapter`, который вызывает твой метод
    
5. Запускает контейнер при старте приложения


Listener контейнер работает абсолютно так же, как и в способе 1.

# 📕 2. Принятие сообщений
Так как самый распространенный способ принятия сообщений именно через @KafkaListener, рассмотрим его

## 1. Где можно использовать аннотацию @KafkaListener
После конфигурации, мы должны описать бизнес-логику, как будет обрабатываться сообщение.

`@KafkaListener` может аннотировать:
- класс
- метод

В случае аннотирования класса, мы говорим, что весь класс является слушателем сообщений. 
НО тогда методы обработки сообщений должны быть помечены как `@KafkaHandler`
> Обычно используют `@KafkaListener` над методом

Важными параметрами аннотации являются
- topic
- containerFactory

> [!NOTE] Метод `poll()` вызывается АВТОМАТИЧЕСКИ
> 

## 2. Принятие сообщений
В случае, если аннотация `@KafkaListener` находится над методом, то **метод должен иметь одну из допустимых сигнатур**.
(см [[3. - Аннотация KafkaListener]])

Spring Kafka автоматически выбирает тип Listener-а в зависимости от сигнатуры метода.

### 1. **RecordListener (одиночные сообщения)**
Самый распространённый — один вызов = одно сообщение:
```java
public void listen(UserInfo message)
```

### 2. **BatchListener (пакет сообщений)**
Для чтения сразу нескольких сообщений (poll возвращает пачку):
```java
@KafkaListener(topics = "topic", batch = "true")
public void listen(List<UserInfo> messages)
```
❗ batch-режим должен быть включён:
```java
factory.setBatchListener(true);
```
Используется когда:
- высокая производительность
- важна скорость обработки

- хотите делать commit-ы реже

### 3. **AcknowledgingMessageListener (ручной ack)**
Если `AckMode.MANUAL`:
```java
public void listen(UserInfo message, Acknowledgment ack) {
    // обработали
    ack.acknowledge(); // вручную коммитим offset
}
```

| AckMode          | Что делает                          |
| ---------------- | ----------------------------------- |
| RECORD           | commit после каждого сообщения      |
| BATCH            | commit после каждой пачки poll()    |
| TIME             | commit через каждые N секунд        |
| COUNT            | commit после N сообщений            |
| MANUAL           | commit вручную через Acknowledgment |
| MANUAL_IMMEDIATE | commit вручную и сразу              |

Твой код должен корректно выставлять:
```java
factory
	.getContainerProperties()
		.setAckMode(AckMode.MANUAL);
```

### Десериализация сообщений
*Поскольку из Kafka поступают сырые байты*, то нам **необходимо**, в зависимости от сигнатуры методы, **преобразовать байты в тип, указанный в качестве параметра метода**, который принимает сообщение

Для этого используются десериализаторы, которые мы указываем при конфигурации фабрики Consumer.

Классы десериализации ТЕЛ сообщений должны реализовывать интерфейс `Deserializer<T>` от Apache Kafka.

```java
public class CustomUserInfoDeserializer implements Deserializer<UserInfo> {  
    private final ObjectMapper mapper = new ObjectMapper();  
  
    @Override  
    public UserInfo deserialize(String s, byte[] bytes) {  
        try {  
            return mapper.readValue(bytes, UserInfo.class);  
        } catch (IOException e) {  
            throw new RuntimeException(e);  
        }  
    }  
}
```

В целом все, сообщение получено!

## 3. Принятие сообщений с заголовками
Если же, вместе с сообщением поступает заголовок, его также необходимо десериализовать.

Для этого необходимо указать как десериализовать заголовки сообщений.

Это можно сделать двумя способами
### 1.KafkaHeaderMapper
Для этого, как и в Producer используется `KafkaHeaderMapper` который передается в `MessagingMessageConverter`
```java
@Bean  
    public 
    ConcurrentKafkaListenerContainerFactory<String, UserInfo> 
    kafkaListenerContainerFactory() 
    {  
        ConcurrentKafkaListenerContainerFactory<String, UserInfo> 
        factory = 
	        new ConcurrentKafkaListenerContainerFactory<>();  
  
        factory.setConsumerFactory(consumerFactory());  
  
        MessagingMessageConverter converter = 
	        new MessagingMessageConverter();  
	        
        converter.setHeaderMapper(new KafkaHeaderMapper() {  
            @Override  
            public void fromHeaders(
	            MessageHeaders headers, 
	            Headers target) {}  
  
            @Override  
            public void toHeaders(
	            Headers source, 
	            Map<String, Object> target) 
	        {  
                CustomHeader customHeader = new CustomHeader();  
                for(Header header : source){  
                    if(header.key().equals("Custom_Header")){  
                        customHeader = 
	                        CustomHeaderDeserializer
		                        .deserialize(header.value());  
		                        
                        target.put(header.key(), customHeader);  
                        continue;  
                    }  
                    target.put(
	                    header.key(), 
	                    new String(header.value())
	                );  
                }  
            }  
        });  
  
        factory.setRecordMessageConverter(converter);  
        return factory;  
    } 
```

### 2. DefaultKafkaHeaderMapper
DefaultKafkaHeaderMapper по умолчанию использует Jakson для десериализации заголовков, то есть десериализаует заголовок, который был записан в формате JSON
```JAVA
@Bean  
    public ConcurrentKafkaListenerContainerFactory<String, UserInfo> kafkaListenerContainerFactory() {  
        ConcurrentKafkaListenerContainerFactory<String, UserInfo>  
            factory = new ConcurrentKafkaListenerContainerFactory<>();  
  
        factory.setConsumerFactory(consumerFactory());  
		
		// 🔥Использование готового маппера заголовков🔥
		/*
		Приходит заголовок вида
		"spring_json_header_types": 
			"{\"Simple_Header\":\"java.lang.String\",
			\"Custom_Header\":\"model.CustomHeader\"}",
		
		DefaultKafkaHeaderMapper видит, что для заголовка 
		Custom_Header установлен тип model.CustomHeader.
		
		А значит дальнейший парсинг данных, он будет 
		осуществлять относительно этого класса, 
		ТОЛЬКО ЕСЛИ этот класс находится в доверенном пакете
		
		И так как ранне этот объект был записан при помощи 
		ObjcetMapper, который использует 
			- название полей в качестве ключа, 
			- а значение переменной в качестве значения, 
		то десериализация проходит успешно
		*/
        DefaultKafkaHeaderMapper headerMapper = 
	        new DefaultKafkaHeaderMapper(); 
	        
	    // По умолчанию, десериализации подлежат классы из java.lang
	    // поэтому чтобы добавить десераилизацию других классов,
	    // указываем доверенные пакеты 
        headerMapper.addTrustedPackages("*");  
  
        MessagingMessageConverter converter = 
	        new MessagingMessageConverter(); 
	    // И передаем маппер хедеров в конвертор сообщений 
        converter.setHeaderMapper(headerMapper);  
  
        factory.setRecordMessageConverter(converter);  
        return factory;  
    }
```

### Использование заголовков в lister()
После того, как мы описали как десериализовывать заголовки, мы можем использовать заголовки в методе

Здесь есть два метода использования
#### @Header
При помощи этой аннотации указываем параметр метода, в который смапится полученный заголовок сообщения

```java
@KafkaListener(
        topics = "topic-spring-1",
        containerFactory = "kafkaListenerContainerFactory"
)
public void receiveMessage(
        @Payload UserInfo userInfo,
        @Header("StringHeader") String stringHeader,
        @Header("CustomHeader") CustomHeader customHeader,
        @Header(KafkaHeaders.RECEIVED_PARTITION_ID) int partition,
        @Header(KafkaHeaders.OFFSET) long offset
) {
    System.out.println("Received: " + userInfo);
    System.out.println("StringHeader = " + stringHeader);
    System.out.println("CustomHeader = " + customHeader);
    System.out.println("Partition = " + partition);
    System.out.println("Offset = " + offset);
}
```

При использовании аннотации `@Header` ВАЖНО ПОМЕНИТЬ ТЕЛО СООБЩЕНИЯ аннотацией `@Payload`

🔥 Важно:
- `@Payload` — тело сообщения
- `@Header` — заголовки
    
- можно указывать свои заголовки и системные (`KafkaHeaders.*`)

#### Ручное получение из Message
Либо можно получить сообщение из Message, усли не хочешь описывать параметры через `@Header`, можно получить всё вручную:

```java
@KafkaListener(
        topics = "topic-spring-1",
        containerFactory = "kafkaListenerContainerFactory"
)
public void receiveMessage(Message<UserInfo> message) {

    UserInfo payload = message.getPayload();
    MessageHeaders headers = message.getHeaders();

    System.out.println("Payload = " + payload);

    // Получение конкретного хедера
    CustomHeader custom = headers.get(
	    "CustomHeader", 
	    CustomHeader.class);
    System.out.println("CustomHeader = " + custom);

    // Получение системных хедеров Kafka
    int partition = headers.get(
	    KafkaHeaders.RECEIVED_PARTITION_ID, Integer.class);
    long offset = headers.get(KafkaHeaders.OFFSET, Long.class);

    System.out.println("Partition = " + partition);
    System.out.println("Offset = " + offset);

    // Перебор всех заголовков
    headers.forEach((k, v) -> {
        System.out.println("Header: " + k + " = " + v);
    });
}
```

## 4. Приостановка-возобновления чтения, и откат offset
Spring Kafka позволяет **приостановить** чтение сообщений и потом **возобновить**:

```java
@KafkaListener(id = "myListener", topics = "topic")
public void listen(
	String msg, 
	Acknowledgment ack, 
    Consumer<?,?> consumer) 
{
    consumer.pause(
	    Collections.singleton(new TopicPartition("topic", 0)));
    ...
    consumer.resume(...);
}
```

А также можно откатить offset 
```java
consumer.seek(tp, offset);
```
Используется при парсинге невалидных сообщений.
