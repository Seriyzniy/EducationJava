Иногда вам **может понадобиться использовать одну и ту же конфигурацию для нескольких слушателей.** 

Чтобы *уменьшить шаблонный код конфигурации, вы можете использовать* **мета-аннотации** для создания собственной аннотации слушателя. 

Следующий пример показывает, как это сделать:
```java
@Target(
	{
		ElementType.TYPE, 
		ElementType.METHOD, 
		ElementType.ANNOTATION_TYPE
	}
)
@Retention(RetentionPolicy.RUNTIME)
@RabbitListener(bindings = @QueueBinding(
	value = @Queue,
	exchange = @Exchange(
		value = "metaFanout", 
		type = ExchangeTypes.FANOUT)
	)
)
public @interface MyAnonFanoutListener {
}



public class MetaListener {

    @MyAnonFanoutListener
    public void handle1(String foo) {
        ...
    }

    @MyAnonFanoutListener
    public void handle2(String foo) {
        ...
    }

}
```

> В предыдущем примере каждый слушатель, созданный с помощью аннотации `@MyAnonFanoutListener`, привязывает анонимную, автоматически удаляемую очередь к обменнику типа "fanout" с именем `metaFanout`. 

Начиная с версии 2.2.3, поддерживается `@AliasFor`, позволяющий переопределять свойства мета-аннотированной аннотации. 

Кроме того, пользовательские аннотации теперь могут быть `@Repeatable`, что позволяет создавать несколько контейнеров для одного метода.

```java
@Component
static class MetaAnnotationTestBean {
    @MyListener("queue1")
    @MyListener("queue2")
    public void handleIt(String body) {
    }

}


@RabbitListener
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
@Repeatable(MyListeners.class)
static @interface MyListener {

    @AliasFor(annotation = RabbitListener.class, attribute = "queues")
    String[] value() default {};

}

@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
static @interface MyListeners {

    MyListener[] value();

}
```
