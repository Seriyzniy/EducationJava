## Включение аннотации
Чтобы включить поддержку аннотаций `@RabbitListener`, ==вы можете добавить== `@EnableRabbit` в один из ваших классов с аннотацией `@Configuration`. 

Следующий пример показывает, как это сделать:
```java
@Configuration
@EnableRabbit
public class AppConfig {

    @Bean
    public SimpleRabbitListenerContainerFactory rabbitListenerContainerFactory() 
    {
        SimpleRabbitListenerContainerFactory factory = 
	        new SimpleRabbitListenerContainerFactory();
	        
        factory.setConnectionFactory(connectionFactory());
        factory.setConcurrentConsumers(3);
        factory.setMaxConcurrentConsumers(10);
        
        factory.setContainerCustomizer(container -> /* настроить контейнер */);
        return factory;
    }
}
```

## Фабрики контейнеров слушателей
==По умолчанию используется бин== с именем **`rabbitListenerContainerFactory`**. В примере выше методы слушателей будут выполняться с базовым пулом в **3 потока** и максимальным в **10 потоков**.

Начиная с версии **2.0**, доступна также **`DirectRabbitListenerContainerFactory`**, **создающая** `DirectMessageListenerContainer`. 

Выбор между 
- `SimpleRabbitListenerContainerFactory` 
- и `DirectRabbitListenerContainerFactory` описан в разделе _Choosing a Container_.

С версии **2.2.2** можно задавать **`ContainerCustomizer`** для дополнительной настройки контейнера после его создания. 
С версии **2.4.8** доступен **`CompositeContainerCustomizer`** для применения нескольких кастомизаторов.

==Фабрику контейнеров== можно:
- указать явно в `@RabbitListener`,
- или задать ==фабрику по умолчанию, реализовав ==**`RabbitListenerConfigurer`** (нужно, если хотя бы один listener не указывает фабрику). 

**Фабрики также позволяют добавлять** **`MessagePostProcessor`**, применяемые после получения сообщений и перед отправкой ответов. 
> С версии **2.0.6** можно подключить **`RetryTemplate`** и **`RecoveryCallback`**, которые используются при отправке ответов.

Если используется XML-конфигурация, применяется элемент `<rabbit:annotation-driven>`.

Пример для `SimpleRabbitListenerContainerFactory`:
```xml
<rabbit:annotation-driven/>

<bean id="rabbitListenerContainerFactory"
      class="org.springframework.amqp.rabbit.config.SimpleRabbitListenerContainerFactory">
    <property name="connectionFactory" ref="connectionFactory"/>
    <property name="concurrentConsumers" value="3"/>
    <property name="maxConcurrentConsumers" value="10"/>
</bean>

```

Пример для `DirectRabbitListenerContainerFactory`:
```xml
<rabbit:annotation-driven/>

<bean id="rabbitListenerContainerFactory"
      class="org.springframework.amqp.rabbit.config.DirectRabbitListenerContainerFactory">
    <property name="connectionFactory" ref="connectionFactory"/>
    <property name="consumersPerQueue" value="3"/>
</bean>
```

С версии **2.0** аннотация `@RabbitListener` ==поддерживает свойство== **`concurrency`**, которое может использовать **SpEL (`#{…}`)** и **property placeholders (`${…}`)**.
- Для **`DirectMessageListenerContainer`** задаётся одно число → `consumersPerQueue`.
    
- Для **`SimpleMessageListenerContainer`** число устанавливает `concurrentConsumers`, или форма `m-n` задаёт 
	- `concurrentConsumers = m` 
	- и `maxConcurrentConsumers = n`.

Это позволяет переопределять настройки фабрики для отдельных слушателей без создания разных фабрик.

Начиная с версии **2.2**, аннотация также позволяет переопределять **`autoStartup`**, **`executor`** и **`ackMode`**:
```java
@RabbitListener(
	id = "manual.acks.1", 
	queues = "manual.acks.1", 
	ackMode = "MANUAL")
public void manual1(
	String in, 
	Channel channel,
    @Header(AmqpHeaders.DELIVERY_TAG) long tag) throws IOException 
{
	...
    channel.basicAck(tag, false);
}
```

Использование разных **executors** помогает легче идентифицировать потоки слушателей в логах и дампах потоков.
