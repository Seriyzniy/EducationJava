# 📕 1. Конфигурация Producer в Spring AMQP

## **1️⃣ Настройка ConnectionFactory**
Как было указано в архитектуре RabbitMQ API, для подключения к брокеру, необходимо соединение - Connection, которое можно получить через фабрику `ConnectionFactory`

Для подключения к RabbitMQ нужны основные свойства:
- `host`, `port` — адрес брокера RabbitMQ
    
- `username`, `password` — аутентификация
    
- `virtualHost` — виртуальный хост (по умолчанию "/") 

**Пример конфигурации через properties:**
```properties
# application.properties

spring.rabbitmq.host=localhost
spring.rabbitmq.port=5672
spring.rabbitmq.username=guest
spring.rabbitmq.password=guest
spring.rabbitmq.virtual-host=/
```

**Пример кода через Java Config:**
```java
@Bean
public ConnectionFactory connectionFactory() {
    CachingConnectionFactory factory = new CachingConnectionFactory();
    
    factory.setHost("localhost");
    factory.setPort(5672);
    factory.setUsername("guest");
    factory.setPassword("guest");
    factory.setVirtualHost("/");
    
    // Кэширование соединений и каналов
    factory.setCacheMode(CachingConnectionFactory.CacheMode.CHANNEL);
    factory.setChannelCacheSize(25); // размер кэша каналов
    
    return factory;
}
```

## **2️⃣ Создание RabbitTemplate**
`RabbitTemplate` — **основной класс Spring AMQP для отправки** сообщений - реализация `AmqpTemplate`
(см [[1. - Интерфейс AmqpTemplate и реализация RabbitTemplate]])

> В этот "шаблон" необходимо передать `ConnectionFactory`

```java
@Bean
public RabbitTemplate rabbitTemplate(ConnectionFactory connectionFactory) {
    RabbitTemplate template = new RabbitTemplate(connectionFactory);
    
    // Настройка конвертации сообщений
    template.setMessageConverter(jsonMessageConverter());
    
    // Включение подтверждений (Publisher Confirms)
    template.setConfirmCallback((correlationData, ack, cause) -> {
        if (ack) {
            System.out.println("Сообщение подтверждено брокером");
        } else {
            System.out.println("Ошибка доставки: " + cause);
        }
    });
    
    // Включение ReturnCallback для недоставленных сообщений
    template.setMandatory(true); // обязательный для ReturnCallback
    
    template.setReturnsCallback(returned -> {
        System.out.println("Сообщение возвращено: " + 
            returned.getReplyText() + " для exchange: " + 
            returned.getExchange() + ", routingKey: " + 
            returned.getRoutingKey());
    });
    
    return template;
}
```

## **3️⃣** ДОПОЛНИТЕЛЬНО - **Настройка MessageConverter**
Сообщения в RabbitMQ — это всегда `byte[]`. 

Конвертеры преобразуют объекты (см [[1. - Message Converters (Конвертеры сообщений )]])

| Конвертер                                 | Описание                                                |
| ----------------------------------------- | ------------------------------------------------------- |
| **SimpleMessageConverter**                | По умолчанию. Конвертирует String, Serializable, byte[] |
| **Jackson2JsonMessageConverter**          | JSON через ==Jackson==                                  |
| **MarshallingMessageConverter**           | XML через JAXB                                          |
| **ContentTypeDelegatingMessageConverter** | Выбор конвертера по Content-Type                        |

### Jackson2JsonMessageConverter
В документации указано, что рекомендуется использовать `Jackson2JsonMessageConverter` (подключи Jackson в pom.xml)

```java
@Bean
public MessageConverter jsonMessageConverter() {
    Jackson2JsonMessageConverter converter = 
	    new Jackson2JsonMessageConverter();
    
    // Настройка ObjectMapper
    ObjectMapper mapper = new ObjectMapper();
    mapper.registerModule(new JavaTimeModule());
    mapper.disable(SerializationFeature.WRITE_DATES_AS_TIMESTAMPS);
    
    converter.setJsonObjectMapper(mapper);
    
    // Настройка типа для десериализации (опционально)
    // можно через addTrustPackage или как то так
    DefaultJackson2JavaTypeMapper typeMapper = 
	    new DefaultJackson2JavaTypeMapper();
    typeMapper.setTrustedPackages("com.example.dto");
    converter.setJavaTypeMapper(typeMapper);
    
    return converter;
}
```

либо
```java
rabbitTemplate.setMessageConverter(
	new Jackson2JsonMessageConverter(
		jsonObjectMapper,
		"*"));
```
(см [[2. - Пример-1 (Hello world)]])

### КАСТОМНЫЙ маппер
Можно создать кастомный маппер, реализовав интерфейс `MessageCOnvertet`
```java
public class CustomMessageConverter implements MessageConverter {
    private final ObjectMapper objectMapper = new ObjectMapper();
    
    @Override
    public Message toMessage(
	    Object object, 
	    MessageProperties messageProperties) 
	{
        try {
            byte[] body = objectMapper.writeValueAsBytes(object);
            
            messageProperties.setContentType("application/json");
            messageProperties.setContentEncoding("UTF-8");
            
            return new Message(body, messageProperties);
            
        } catch (JsonProcessingException e) {
            throw new MessageConversionException("Ошибка конвертации", e);
        }
    }
    
    @Override
    public Object fromMessage(Message message) {
        try {
            return objectMapper.readValue(message.getBody(), Map.class);
        } catch (IOException e) {
            throw new MessageConversionException("Ошибка чтения", e);
        }
    }
}
```

# 📕 2. Процесс БАЗОВОЙ отправки сообщения
## **1️⃣ Типы сообщений в Spring AMQP**
### **Message (родной объект Spring AMQP)**
```java
Message message = MessageBuilder
    .withBody("Hello Rabbit".getBytes())
    .setContentType(MessageProperties.CONTENT_TYPE_TEXT_PLAIN)
    .setMessageId(UUID.randomUUID().toString())
    .setTimestamp(new Date())
    .setHeader("custom-header", "value")
    .build();
```

### **Отправка объектов (с конвертацией)**
```java
// При наличии MessageConverter
rabbitTemplate.convertAndSend("exchange", "routing.key", userObject);
// Автоматически конвертирует User в JSON/byte[]
```

Spring используя установленный MessageConverter преобразует объект в массив байт

## **2️⃣ Основные методы отправки**
### **Отправка в конкретный exchange с routing key**
```java
// 1. Базовый вариант
rabbitTemplate.convertAndSend(
    "orders-exchange",    // exchange
    "order.created",      // routing key
    order                 // объект (конвертируется)
);

// 2. С дополнительными свойствами
rabbitTemplate.convertAndSend(
    "orders-exchange",
    "order.created",
    order,
    message -> {
        // Модификация MessagePostProcessor
        message.getMessageProperties().setPriority(5);
        message.getMessageProperties().setExpiration("60000"); // TTL 60s
        message.getMessageProperties().setHeader("source", "api");
        return message;
    }
);
```

### **Отправка напрямую в очередь (через Default Exchange)**
```java
rabbitTemplate.convertAndSend(
    "",                   // пустая строка = Default Exchange
    "orders-queue",       // routing key = имя очереди
    order                 // очередь должна существовать!
);
```

### **Синхронная отправка с ожиданием ответа (RPC)**
```java
Object response = rabbitTemplate.convertSendAndReceive(
    "rpc-exchange",
    "rpc.key",
    requestObject,
    message -> {
        message.getMessageProperties().setReplyTo("reply-queue");
        message.getMessageProperties().setCorrelationId("corr-123");
        return message;
    }
);
```

## **3️⃣ДОПОЛНИТЕЛЬНО
### MessagePostProcessor - модификация сообщений
```java
// Кастомный MessagePostProcessor
MessagePostProcessor processor = new MessagePostProcessor() {
    @Override
    public Message postProcessMessage(Message message) throws AmqpException {
        // Добавление заголовков
        message.getMessageProperties().setHeader("trace-id", "12345");
        message.getMessageProperties().setHeader("version", "1.0");
        
        // Установка свойств доставки
        message.getMessageProperties()
	        .setDeliveryMode(MessageDeliveryMode.PERSISTENT);
        message.getMessageProperties().setPriority(10);
        message.getMessageProperties().setExpiration("30000"); // 30 секунд TTL
        
        // Установка типа контента
        message.getMessageProperties().setContentType("application/json");
        message.getMessageProperties().setContentEncoding("UTF-8");
        
        return message;
    }
};

// Использование
rabbitTemplate.convertAndSend("exchange", "key", payload, processor);
```

### CorrelationData - отслеживание сообщений
```java
// Для подтверждений (Publisher Confirms)
CorrelationData correlationData = new CorrelationData("order-12345");

rabbitTemplate.convertAndSend(
    "orders-exchange",
    "order.created",
    order,
    correlationData
);

// Получение подтверждения (можно в отдельном потоке)
Confirm confirm = correlationData.getFuture().get(5, TimeUnit.SECONDS);
if (confirm.isAck()) {
    System.out.println("Подтверждено: " + correlationData.getId());
}
```

### Batch Sending
```java
// Использование BatchingRabbitTemplate для пакетов
@Bean
public BatchingRabbitTemplate batchingRabbitTemplate(
        ConnectionFactory connectionFactory) {
    
    BatchingStrategy batchingStrategy = new SimpleBatchingStrategy(
        100, // размер батча
        10000, // timeout (10 секунд)
        1024 * 1024 // 1MB размер
    );
    
    return new BatchingRabbitTemplate(
        connectionFactory, 
        batchingStrategy
    );
}

// Отправка
batchingRabbitTemplate.convertAndSend("exchange", "key", message);
// Сообщение буферизуется и отправляется пачкой
```

# 3. Обеспечение надежности
## Publisher Confirms (ConfirmCallback)

**В ConnectionFactory**
```java
CachingConnectionFactory factory = new CachingConnectionFactory();
factory.setPublisherConfirmType(
	CachingConnectionFactory.ConfirmType.CORRELATED);
```

**В RabbitTemplate**
```java
rabbitTemplate.setConfirmCallback((correlationData, ack, cause) -> {
    if (ack) {
        log.info("Сообщение {} доставлено в брокер", 
            correlationData != null 
	            ? correlationData.getId() 
	            : "unknown");
    } else {
        log.error("Ошибка доставки сообщения {}: {}", 
            correlationData != null 
	            ? correlationData.getId() 
	            : "unknown", 
            cause);
        // Повторная отправка или сохранение в Dead Letter
    }
});
```

## Returned Messages (ReturnsCallback)
```java
// Включение mandatory режима
rabbitTemplate.setMandatory(true);

// Обработка недоставленных сообщений
rabbitTemplate.setReturnsCallback(returned -> {
    log.warn("""
        Сообщение не доставлено!
        ReplyCode: {}
        ReplyText: {}
        Exchange: {}
        RoutingKey: {}
        Message: {}
        """,
        returned.getReplyCode(),
        returned.getReplyText(),
        returned.getExchange(),
        returned.getRoutingKey(),
        new String(returned.getMessage().getBody())
    );
    
    // Сохранение в Dead Letter Queue или повторная отправка
});
```

## Retry Template (повторные попытки)
```java
@Bean
public RabbitTemplate rabbitTemplate(ConnectionFactory connectionFactory) {
    RabbitTemplate template = new RabbitTemplate(connectionFactory);
    
    // Настройка RetryTemplate
    RetryTemplate retryTemplate = new RetryTemplate();
    
    // Политика повторов
    SimpleRetryPolicy retryPolicy = new SimpleRetryPolicy();
    retryPolicy.setMaxAttempts(3); // максимум 3 попытки
    retryTemplate.setRetryPolicy(retryPolicy);
    
    // Бэкофф политика (экспоненциальная задержка)
    ExponentialBackOffPolicy backOffPolicy = new ExponentialBackOffPolicy();
    backOffPolicy.setInitialInterval(1000); // 1 секунда
    backOffPolicy.setMultiplier(2.0); // удваивается каждый раз
    backOffPolicy.setMaxInterval(10000); // максимум 10 секунд
    retryTemplate.setBackOffPolicy(backOffPolicy);
    
    template.setRetryTemplate(retryTemplate);
    
    return template;
}
```
