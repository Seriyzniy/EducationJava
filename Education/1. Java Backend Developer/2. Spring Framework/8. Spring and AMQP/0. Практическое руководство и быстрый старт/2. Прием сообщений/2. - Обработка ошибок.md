# 0. Группы ошибок в RabbitMQ Consumer

## Группа 1 — Ошибки ДО вызова Listener (до уровня твоего кода)
Consumer **не успел попасть в @RabbitListener**:
### 1.1. **Ошибка десериализации сообщения (Message Conversion Error)**
- JSON parse error
- Неверный формат данных
- Несоответствие типа данных

### 1.2. **Ошибки соединения**
- Connection refused
- Channel closed
- Heartbeat timeout

### 1.3. **Ошибки ресурсов**
- Память закончилась
- File descriptors exhausted

### ПОВЕДЕНИЕ ПО УМОЛЧАНИЮ
❌ _По умолчанию контейнер перезапускается с экспоненциальной задержкой._  
Spring AMQP пытается десериализовать `Message` → вызывает MessageConverter.

Если exception → контейнер:
1. Закрывает Channel
    
2. Перезапускается через `SimpleRabbitListenerContainerFactory`
    
3. Если ошибки продолжаются → увеличивает задержку между перезапусками

⚠️ Это может привести к **зависанию в бесконечном цикле перезапусков**.

## Группа 2 — Ошибки ВНУТРИ Listener (уровень твоего кода)

### 2.1. **Ошибки бизнес-логики**
- NPE, IllegalStateException, RuntimeException
- Ошибки валидации

### 2.2. **Ошибки вызова внешних сервисов**
- RestTemplate timeout
    
- Database connection failed
    
- HTTP 5xx ошибки

### 2.3. **Ошибки в транзакциях**
- @Transactional.rollbackFor исключения
    
- TransactionTimedOutException

### ПОВЕДЕНИЕ ПО УМОЛЧАНИЮ
Если Listener бросил исключение, поведение зависит от `AcknowledgeMode`:

**AcknowledgeMode.AUTO** (по умолчанию):
- Исключение → сообщение **возвращается в очередь**
    
- Consumer продолжает читать следующее сообщение

**AcknowledgeMode.MANUAL**:
- Исключение → сообщение **не подтверждается**
    
- При `basicNack(requeue=true)` → возврат в очередь
    
- При `basicNack(requeue=false)` → ==отправка в DLX ==(Dead Letter Exchange)

⚠️ Это приводит к **бесконечному циклу обработки одного сообщения**:
```
msg → exception → requeue → exception → requeue → ...
```

# 📕 1. Обработка ошибок десериализации

## 1. **MessageConverter с обработкой ошибок**
Создаем кастомный MessageConverter, который не падает при ошибках:
```java
@Component
public class SafeJackson2JsonMessageConverter extends Jackson2JsonMessageConverter {
    
    @Override
    public Object fromMessage(Message message, Object conversionHint) 
            throws MessageConversionException {
        try {
            return super.fromMessage(message, conversionHint);
            
        } catch (MessageConversionException e) {
            // Создаем объект-обертку с ошибкой
            DeserializationError error = new DeserializationError(
                message.getMessageProperties().getMessageId(),
                e.getMessage(),
                message.getBody()
            );
            return error;
        }
    }
}

// Обертка для сообщений с ошибкой десериализации
public class DeserializationError {
    private String messageId;
    private String error;
    private byte[] originalBody;
    // конструкторы, геттеры, сеттеры
}
```

## 2. **Использование в Listener**
```java
@Bean
public SimpleRabbitListenerContainerFactory rabbitListenerContainerFactory(
	ConnectionFactory connectionFactory,
	MessageConverter messageConverter
) {
    SimpleRabbitListenerContainerFactory factory = 
        new SimpleRabbitListenerContainerFactory();
        
    factory.setConnectionFactory(connectionFactory);
    
    factory.setMessageConverter(messageConverter);
    
    return factory;
}

@Component
public class MessageConsumer {
    
    @RabbitListener(
        queues = "main-queue",
        containerFactory = "rabbitListenerContainerFactory"
    )
    // Как только сообщение придет, Spring будет пытаться десериализовать его при помощи MessageConverter, чтобы в аргумент метода попал объект
    public void handleMessage(Object payload) {
	    
	    //Если объект - обертка над ошибкой
        if (payload instanceof DeserializationError) {
            DeserializationError error = (DeserializationError) payload;
            log.error("Ошибка десериализации сообщения {}: {}", 
                error.getMessageId(), error.getError());
            
            // Можно отправить в Dead Letter Queue
            // или сохранить для дальнейшего анализа
            return;
        }
        
        // Нормальная обработка
        processMessage(payload);
    }
}
```

# 📕 2. ГЛОБАЛЬНЫЙ обработчик ошибок

## 1. **ErrorHandler в Spring AMQP**
Spring AMQP предоставляет `ErrorHandler` для обработки исключений:
```java
@Bean
public SimpleRabbitListenerContainerFactory rabbitListenerContainerFactory(
	ConnectionFactory connectionFactory
) {
    SimpleRabbitListenerContainerFactory factory = 
        new SimpleRabbitListenerContainerFactory();
    factory.setConnectionFactory(connectionFactory);
    
    // Настраиваем ErrorHandler
    factory.setErrorHandler(new ConditionalRejectingErrorHandler(
        new CustomExceptionStrategy()
    ));
    
    // Или с Retry
    factory.setAdviceChain(retryOperationsInterceptor());
    
    return factory;
}
```
(см подробнее про `ConditionalRejectingErrorHandler` в [[4. - Exception Handling]])

## 2. **ConditionalRejectingErrorHandler с кастомной стратегией**
```java
public class CustomExceptionStrategy extends DefaultExceptionStrategy {
    
    @Override
    public boolean isFatal(Throwable t) {
        // Определяем, какие исключения фатальны
        if (t instanceof MessageConversionException) {
            return true; // Фатальная ошибка десериализации
        }
        
        if (t instanceof IllegalArgumentException) {
            return false; // Не фатальная, можно ретраить
        }
        
        return super.isFatal(t);
    }
}
```

## 3. **RetryOperationsInterceptor для повторных попыток**
```java
@Bean
public RetryOperationsInterceptor retryOperationsInterceptor() {
    return RetryInterceptorBuilder.stateless()
        .maxAttempts(3) // Максимум 3 попытки
        .backOffOptions(1000, 2.0, 10000) // initialInterval, multiplier, maxInterval
        .recoverer(new MessageRecoverer() {
            @Override
            public void recover(Message message, Throwable throwable) {
                // После исчерпания попыток
                log.error(
	                "Не удалось обработать сообщение после 3 попыток",
	                throwable);
                
                // Отправляем в Dead Letter Queue
                sendToDLQ(message, throwable);
            }
        })
        .build();
}
```

## 4. Настройка DLQ в RabbitMQ
```java
@Configuration
public class RabbitMQConfig {
    
    @Bean
    public Queue mainQueue() {
        Map<String, Object> args = new HashMap<>();
        args.put("x-dead-letter-exchange", "dlx.exchange");
        args.put("x-dead-letter-routing-key", "dlq.routing.key");
        args.put("x-message-ttl", 10000); // TTL 10 секунд
        
        return new Queue("main.queue", true, false, false, args);
    }
    
    @Bean
    public DirectExchange dlxExchange() {
        return new DirectExchange("dlx.exchange");
    }
    
    @Bean
    public Queue deadLetterQueue() {
        return new Queue("dead.letter.queue", true);
    }
    
    @Bean
    public Binding dlqBinding() {
        return BindingBuilder.bind(deadLetterQueue())
            .to(dlxExchange())
            .with("dlq.routing.key");
    }
}
```

### Consumer с обработкой DLQ
```java
@Component
public class ConsumerWithDLQ {
    
    private static final int MAX_RETRIES = 3;
    
    @RabbitListener(queues = "main.queue")
    public void processWithRetry(
            Message message,
            @Header(AmqpHeaders.RECEIVED_ROUTING_KEY) String routingKey,
            Channel channel,
            @Header(AmqpHeaders.DELIVERY_TAG) long deliveryTag,
            @Header("x-death") List<Map<String, Object>> deaths) {
        
        // Проверяем количество попыток обработки
        int retryCount = getRetryCount(deaths);
        
        if (retryCount >= MAX_RETRIES) {
            // Превышено количество попыток - отправляем в DLQ
            channel.basicReject(deliveryTag, false);
            return;
        }
        
        try {
            // Пытаемся обработать
            processMessage(message);
            channel.basicAck(deliveryTag, false);
        } catch (Exception e) {
            // Ошибка - отправляем в DLQ для повторной попытки
            channel.basicNack(
	            deliveryTag, 
	            false, //multiple, 
	            false // requeue=false -> сообщение в DLQ
	        ); 
        }
    }
    
    private int getRetryCount(List<Map<String, Object>> deaths) {
        if (deaths == null || deaths.isEmpty()) {
            return 0;
        }
        Map<String, Object> death = deaths.get(0);
        return (int) death.getOrDefault("count", 0);
    }
}
```
