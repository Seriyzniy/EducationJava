В файле pom.xml находится привычная нам конфигурация
```xml
<parent>
	<groupId>org.springframework.boot</groupId>
	<artifactId>spring-boot-starter-parent</artifactId>
	<version>3.4.1</version>
	<relativePath/> <!-- lookup parent from repository -->
</parent>
```
Благодаря этому коду, Spring Boot настраивает Maven необходимым образом, благодаря чему, не нужно указывать версию в зависимостях

# 1. Главный класс Spring Boot
```java
package ru.vestramas.springBoot;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class FirstSpringBootAppApplication {

	public static void main(String[] args) {
		SpringApplication.run(FirstSpringBootAppApplication.class, args);
	}

}
```
Этот класс запускает Spring Boot приложение


> [!NOTE] Аннотация `@SpringBootApplication`
> Данная аннотация включает в себе все необходимые аннотации для конфигурации приложения
> - `@ComponentScan`
> - `@Configuration`
> - и д.р.
> 
> ==Класс помеченный данной аннотацией, ДОЛЖЕН НАХОДИТЬСЯ В КОРНЕ ПРОЕКТА==
> Все другие классы должны находиться либо НА ОДНОМ УРОВНЕ с этим классом, либо находиться в подпапке корневой папки

 При запуске метода `main` будет запускаться ВСТРОЕННЫЙ веб-сервер, на котором и будет дальше запускаться НАШЕ SpringBoot приложение

# 2. Важный файл `application.properties`

Не смотря на то, что Spring Boot САМ КОНФИГУРИРУЕТ зависимости, некоторые из них необходимо конфигурировать вручную.

> [!NOTE] Ручная конфигурация
> АБСОЛЮТНО ВСЯ КОНФИГУРАЦИЯ должна находиться в файле `src/main/resources/application.properties`
> 
> Конфигурация должна храниться в формате `ключ=значение`
> Также можно определять свои ключи

# Практика

***
Определим собственную переменную в файле `application.properties`
```
spring.application.name=firstSpringBootApp
myHello=Hello world
```

Создадим контроллер
```java
@Controller
public class HelloController {
	@Value("${myHello}")
	private String hello;
	
	@GetMapping("/hello")
	public String sayHello() {
		System.out.println(hello);
		return "hello";
	}
}
```

Создадим `html` страницу
```html
<!DOCTYPE html>
<html>
	<head>
		<meta charset="UTF-8">
		<title>Insert title here</title>
	</head>
<body>
	Hello world!
</body>
</html>
```

После этого запускаем приложение как Spring Boot App

### Запуск через консоль
Одно из преимуществ Spring Boot ЭТО ВСТРОЕННЫЙ веб-сервер.
Теперь НЕ НУЖНО запускать отдельно сервер и на него загружать `.war` файл

С помощью этой команды Maven соберет `Jar файл`

После этого у нас будет `Jar-файл`, который нужно просто запустить, так как в этом файле уже есть встроенный веб-сервер
