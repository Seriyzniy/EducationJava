# 1. Создание бина JdbcTemplate
После подключения зависимостей, необходимо СОЗДАТЬ БИН `JdbcTemplate`

> [!NOTE] Бин `JdbcTemplate`
> Для создания бина `JdbcTemplate`необходимо
🟢Метод, который вернет бин `DataSource` из пакета `javax.sql.DataSource`;
🟢Метод, который вернет бин `JdbcTemplate`

Для этого создадим отдельный класс конфигуратор - `JdbcConfig`
#### 1. `DataSource`
```java
@Bean
	public DataSource dataSource() {
		DriverManagerDataSource driverManagerDataSource = new DriverManagerDataSource();
		
		driverManagerDataSource.setDriverClassName("org.postgresql.Driver");
		driverManagerDataSource.setUrl("jdbc:postgresql://localhost:5432/myFirstDB");
		driverManagerDataSource.setUsername("postgres");
		driverManagerDataSource.setPassword("12345");
		
		return driverManagerDataSource;
	}
```

> [!NOTE] Класс `DriverManagerDataSource`
> Класс `DriverManagerDataSource` является реализацией стандартного интерфейса `DataSource`, который конфигурирует обычный драйвер JDBC через свойства бина и каждый раз возвращает новое `Connection`.
> 
> Используется ==ТОЛЬКО ДЛЯ ТЕСТИРОВАНИЯ СОЕДИНЕНИЯ==

> [!NOTE] Класс `HikariDataSource`
> HikariCP переиспользует открытые соединения, уменьшая накладные расходы

#### 2. `JdbcTemplate`
```java
@Bean
	public JdbcTemplate jdbcTemplate() {
		return new JdbcTemplate(dataSource());
	}
```

#### 3. РЕЗУЛЬТАТ
```java
@Configuration
@ComponentScan("mvcInAction.MvcApplication.DAO")
public class JdbcConfig {
	@Bean
	public DataSource dataSource() {
		DriverManagerDataSource driverManagerDataSource = new DriverManagerDataSource();
		
		driverManagerDataSource.setDriverClassName("org.postgresql.Driver");
		driverManagerDataSource.setUrl("jdbc:postgresql://localhost:5432/myFirstDB");
		driverManagerDataSource.setUsername("postgres");
		driverManagerDataSource.setPassword("12345");
		
		return driverManagerDataSource;
	}
	@Bean
	public JdbcTemplate jdbcTemplate() {
		return new JdbcTemplate(dataSource());
	}
}
```

# 2. Создание модели данных
Создадим класс (`Model`) , который будет "связан" с таблицей в БД в отдельном пакете `Models`
Этот класс будет использоваться для возвращения результата
```java
package mvcInAction.MvcApplication.Model;

public class Person {
	private int id;
	private String name;
	private int age;
	private String email;
	private String address;
	
	public Person() {};
	public Person(int id, String name, int age, String email, String addres) {
		this.id = id;
		this.name = name;
		this.age = age;
		this.email = email;
		this.address = addres;
	}
	
	public int getId() {
		return id;
	}
	public void setId(int id) {
		this.id = id;
	}
	
	public String getName() {
		return name;
	}
	public void setName(String name) {
		this.name = name;
	}
	
	public int getAge() {
		return age;
	}
	public void setAge(int age) {
		this.age = age;
	}
	
	public String getEmail() {
		return email;
	}
	public void setEmail(String email) {
		this.email = email;
	}
	
	public String getAddress() {
		return address;
	}
	public void setAddress(String address) {
		this.address = address;
	}
}
```

# 3. Создание класса DAO
Теперь наш JdbcTmplate готов к использованию. Создадим класс, который будет пользоваться JdbcTemplate, для работы с БД

В отдельном пакете `DAO` создадим класс `PersonDao`
```java
@Component
public class PersonDAO {
	private final JdbcTemplate jdbcTemplate;
	
	@Autowired
	public PersonDAO(JdbcTemplate jdbcTemplate) {
		this.jdbcTemplate = jdbcTemplate;
	}
	
	public List<Person>getAll() {
		return jdbcTemplate.query("SELECT * FROM persons", new PersonMapper());
	}
	
	public Person getOne(int id) {
		return jdbcTemplate.queryForObject("SELECT * FROM persons WHERE id=?", new PersonMapper(), id);
	}
}
```

# 4. Реализация `RowMapper`
Чтобы `PersonDAO` работал успешно, необходимо реализовать класс, который будет преобразовывать СТРОКИ ТАБЛИЦЫ в объект КЛАССА `Person`

В отдельном пакете `util` создадим класс, реализующий интерфейс `RowMapper`
```java
public class PersonMapper implements RowMapper<Person>{
	@Override
	public Person mapRow(ResultSet rs, int rowNum) throws SQLException {
		Person person = new Person();
		
		person.setId(rs.getInt("id"));
		person.setName(rs.getString("p_name"));
		person.setAge(rs.getInt("age"));
		person.setEmail(rs.getString("email"));
		person.setAddress(rs.getString("adress"));
		
		return person;
	}
}
```

# 5. Использование DAO в Контроллере
Теперь контроллер может пользоваться услугами `PersonDAO`

Создадим метод, который будет возвращать JSON строку пользователю
```java
@GetMapping("/all")
@ResponseBody
public List<Person> getAll() {
	return dao.getAll();
}
```
Чтобы `@ResponseBody` успешно конвертировал `List<Person>` в JSON формат, НЕОБХОДИОМ ПОДКЛЮЧИТЬ ЗАВИСИМОСТЬ

```xml
<dependency>
    <groupId>com.fasterxml.jackson.core</groupId>
    <artifactId>jackson-databind</artifactId>
    <version>2.18.1</version>
</dependency>
```

А также не забудем внедрить `PersonDAO` в контроллер
```java
@Controller
@RequestMapping("/mvc")
public class HelloController {
	private final PersonDAO dao;
	
	@Autowired
	public HelloController(PersonDAO dao) {
		this.dao = dao;
	}
	
	@GetMapping("/all")
	@ResponseBody
	public List<Person> getAll() {
		return dao.getAll();
	}

	@GetMapping("/one")
	@ResponseBody
	public Person getOne(@RequestParam("id") int id) {
		return dao.getOne(id);
	}
	
	@GetMapping
	public String sayHello() {
		return "home";
	}
}
```
