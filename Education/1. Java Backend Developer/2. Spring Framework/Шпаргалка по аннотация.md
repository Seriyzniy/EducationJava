Аннотации Spring представляют собой форму метаданных, которые предоставляют данные о программе. 
Аннотации используются для предоставления дополнительной информации о программе.
## Типы аннотаций Spring Framework

В принципе, во всем spring Framework доступно 6 типов аннотаций.
- Аннотации Spring Core
- Веб-аннотации Spring
- Аннотации к загрузке Spring
- Spring Scheduling Annotations
- Аннотации данных Spring
- Аннотации Spring Bean
## **Тип 1:** Аннотации Spring Core
Аннотации Spring, присутствующие в пакетах _**org.springframework.beans.factory.annotation**_ и _**org.springframework.context.annotation**_, обычно известны как аннотации ядра Spring. 

Эти аннотации используются в основном для основной конфигурации и управления зависимостями.
- `@Component` - Общая аннотация для создания бинов.
- `@Service` - Специализированная версия `@Component` для сервисов.
- `@Repository` - Специализированная версия `@Component` для DAO (работа с базой данных).
- `@Autowired` - Внедрение зависимостей через поиск по типу.
- `@Qualifier` - Уточнение, какой бин должен быть внедрен.
- `@Value` - Внедрение значений из файлов конфигурации.
- `@Configuration` - Указывает, что класс содержит определения бинов.
- `@Bean` - Определяет метод, который создает и возвращает бин.
- `@Scope` - Определяет область видимости бина (`singleton`, `prototype`, и т.д.).
- `@Profile` - Позволяет активировать бины только для определенных профилей.
- `@Lazy` - Откладывает создание бина до момента его первого использования.
- `@DependsOn` - Указывает, что бин зависит от других бинов.
## **Тип 2**: Веб-аннотации Spring
Эти аннотации используются для создания веб-приложений и обработки HTTP-запросов.

- `@Controller` - Определяет класс как контроллер в MVC-приложении.
- `@RestController` - Комбинация `@Controller` и `@ResponseBody`, упрощает создание RESTful сервисов.
- `@RequestMapping` - Назначает маршруты для методов контроллера или самого класса контроллера.
- `@GetMapping`, `@PostMapping`, `@PutMapping`, `@DeleteMapping`, `@PatchMapping` - Упрощенные версии `@RequestMapping` для конкретных HTTP-методов.
- `@RequestParam` - Внедряет параметры запроса в методы контроллера.
- `@PathVariable` - Извлекает переменные из URL.
- `@RequestBody` - Связывает тело HTTP-запроса с объектом Java.
- `@ResponseBody` - Возвращает объект в формате HTTP-ответа.
- `@SessionAttributes` - Определяет атрибуты сессии.
- `@ModelAttribute` - Добавляет объекты в модель данных.
- `@CrossOrigin` - Настройка CORS (разделение ресурсов между источниками).
## **Тип 3**: Аннотации к загрузке Spring (Spring Boot Initialization and Configuration)
Эти аннотации помогают управлять конфигурацией приложения и загрузкой контекста.

- `@ComponentScan` - Указывает пакеты, которые нужно сканировать для поиска бинов.
- `@PropertySource` - Подключает внешний файл свойств для использования в приложении.
- `@Import` - Импортирует дополнительные классы конфигурации.
- `@EnableAspectJAutoProxy` - Включает поддержку прокси для аспектно-ориентированного программирования (AOP).
- `@EnableTransactionManagement` - Включает управление транзакциями.
- `@EnableScheduling` - Включает поддержку планировщика задач.
- `@EnableCaching` - Включает кеширование в приложении.
## **Тип 4**: Spring Scheduling Annotations
Эти аннотации используются для выполнения задач по расписанию.

- `@Scheduled` - Задает расписание для выполнения метода. Можно использовать `cron`, `fixedRate`, `fixedDelay`.
- `@EnableScheduling` - Включает функциональность планировщика задач.
## **Тип 5**: Аннотации данных Spring (Spring Data Annotations)
Аннотации для работы с данными и базами данных, включая управление транзакциями.

- `@Transactional` - Управляет транзакциями (например, rollback, read-only и т.д.).
- `@Repository` - Определяет, что класс является репозиторием данных, и добавляет функционал обработки исключений.
- `@Entity` - (из JPA) Определяет класс как сущность для базы данных.
- `@Id` - (из JPA) Помечает поле как идентификатор сущности.
- `@GeneratedValue` - (из JPA) Указывает стратегию генерации значения для идентификатора.
- `@Table` - (из JPA) Устанавливает настройки для таблицы в базе данных, к которой привязана сущность.
- `@Column` - (из JPA) Определяет свойства столбца в таблице базы данных.
## **Тип 6**: Аннотации Spring Bean
Аннотации, которые влияют на создание и жизненный цикл бинов в Spring.

- `@Bean` - Определяет метод для создания бина в классе конфигурации.
- `@Primary` - Указывает, что бин должен использоваться в качестве основного кандидата для внедрения.
- `@DependsOn` - Задает зависимость одного бина от другого, гарантируя, что бин будет инициализирован только после указанных зависимостей.
- `@PostConstruct` - Метод, который будет выполнен после создания бина (аналог `init`-метода).
- `@PreDestroy` - Метод, который будет выполнен перед уничтожением бина (аналог `destroy`-метода).
- `@Scope` - Определяет область видимости бина (например, `singleton`, `prototype`, `request`, `session`).

# Типы аннотаций в SpringBoot
В Spring Boot аннотации можно разделить на несколько категорий в зависимости от их предназначения. Вот наиболее распространённые аннотации с разделением по группам:
### 1. **Аннотации для конфигурации**

- `@SpringBootApplication` — объединяет три аннотации: `@Configuration`, `@EnableAutoConfiguration`, и `@ComponentScan`. Используется для обозначения основной конфигурации приложения.
- `@Configuration` — указывает, что класс содержит одну или несколько методов, которые определяют бины (beans).
- `@Bean` — используется для определения метода, который создаёт и управляет бином.
- `@ComponentScan` — указывает Spring, где искать компоненты, конфигурации и сервисы (включает классы, помеченные `@Component`, `@Service`, `@Repository`, и т. д.).
- `@PropertySource` — добавляет источник свойств к окружению Spring.
- `@EnableAutoConfiguration` — включает автоматическую конфигурацию компонентов Spring на основе зависимостей.

### 2. **Аннотации для компонентов и бинов**

- `@Component` — используется для создания и регистрации бина в контексте Spring.
- `@Service` — специальный тип компонента, который указывает, что класс выполняет бизнес-логику.
- `@Repository` — используется для компонентов, которые взаимодействуют с базой данных.
- `@Controller` — обозначает контроллер в MVC-приложениях.
- `@RestController` — сочетает `@Controller` и `@ResponseBody`. Применяется для REST API.

### 3. **Аннотации для веб-контроллеров**

- `@RequestMapping` — используется для задания URL маршрута на уровне класса или метода.
- `@GetMapping`, `@PostMapping`, `@PutMapping`, `@DeleteMapping`, `@PatchMapping` — сокращенные версии `@RequestMapping`, задающие конкретные HTTP методы (GET, POST, PUT, DELETE, PATCH).
- `@PathVariable` — извлекает переменные пути из URI.
- `@RequestParam` — связывает параметр метода с параметром запроса.
- `@RequestBody` — связывает тело HTTP запроса с параметром метода.
- `@ResponseBody` — указывает, что возвращаемое значение метода должно быть записано непосредственно в HTTP ответ (обычно используется для JSON).
- `@ModelAttribute` — связывает данные из HTTP запроса с моделью.

### 4. **Аннотации для обработки исключений**

- `@ExceptionHandler` — определяет метод для обработки исключений, выбрасываемых в контроллере.
- `@ControllerAdvice` — позволяет объявить методы `@ExceptionHandler`, которые будут использоваться глобально для всех контроллеров.
- `@ResponseStatus` — устанавливает HTTP статус для ответа.

### 5. **Аннотации для управления транзакциями**

- `@Transactional` — управляет транзакциями, определяя, что метод или класс должен быть выполнен в транзакционном контексте.
- `@EnableTransactionManagement` — включает поддержку управления транзакциями через аннотацию `@Transactional`.

### 6. **Аннотации для безопасности**

- `@EnableWebSecurity` — включает поддержку веб-безопасности.
- `@Secured` — определяет роли безопасности для методов.
- `@PreAuthorize`, `@PostAuthorize` — позволяет настроить выражения безопасности для авторизации.
- `@RolesAllowed` — указывает, что метод доступен только для определённых ролей.

### 7. **Аннотации для валидации**

- `@Valid` — указывает, что объект должен быть валидирован.
- `@NotNull`, `@NotEmpty`, `@Size`, `@Min`, `@Max`, и другие аннотации из пакета `javax.validation.constraints` — используются для валидации полей объекта.

### 8. **Аннотации для работы с данными и JPA**

- `@Entity` — указывает, что класс является сущностью JPA.
- `@Table` — задаёт имя таблицы в базе данных, с которой связана сущность.
- `@Id` — определяет первичный ключ сущности.
- `@GeneratedValue` — задаёт стратегию генерации значения для первичного ключа.
- `@Column` — указывает, что поле связано с колонкой в таблице.
- `@OneToOne`, `@OneToMany`, `@ManyToOne`, `@ManyToMany` — задают типы связей между сущностями.
- `@Repository` — помечает класс как репозиторий для работы с данными (например, для работы с базой данных через JPA).

### 9. **Аннотации для тестирования**

- `@SpringBootTest` — загружает контекст приложения для тестов.
- `@MockBean` — создаёт mock-объект для бина в тестах.
- `@TestConfiguration` — создаёт конфигурацию, которая будет использоваться только в тестах.
- `@WebMvcTest` — загружает контекст приложения, связанный только с веб-слоем.
- `@DataJpaTest` — загружает контекст приложения, связанный с репозиториями JPA для тестирования.
