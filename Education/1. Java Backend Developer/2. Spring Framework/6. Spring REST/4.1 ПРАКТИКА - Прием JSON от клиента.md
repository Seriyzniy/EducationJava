# 2. Обработка исключения

Теперь, нужно обработать исключение, когда пользователь ПЕРЕДАСТ ошибочные данные пользователя в JSON.

Для этого используя аннотацию @Valid и объект класса BuildengResult полученный объект из файла JSON пользователя, пройдет валидацию, и ошибки поместятся в BuildingResult

#### Контроллер
```java
@PostMapping
	public ResponseEntity<HttpStatus> savePerson(@RequestBody @Valid Person person, BindingResult bindingResult) {
		if(bindingResult.hasErrors()) {
			StringBuilder errorMsg = new StringBuilder();
			List<FieldError> errors = bindingResult.getFieldErrors();
			errors.stream().map(e -> errorMsg.append(e));
			
			throw new PersonCreateException(errorMsg.toString());
		}
		service.save(person);
		return ResponseEntity.ok(HttpStatus.OK);
		
	}
```

Теперь, при помощи метода `getFieldErrors()` получим все строковые сообщения о неверной валидации. Эти ошибки поместим в список ошибок, и при помощи `Stream API` поместим все ошибки в строку. 
После сбора ошибок, вызовем собственное исключение и передадим эти ошибки
#### Исключение
Исключение получит в конструктор сообщение об ошибке
```java
public class PersonCreateException extends RuntimeException {
	public PersonCreateException(String msg) {
		super(msg);
	}
}
```

#### Обработчик исключения
Обработчик увидит выброшенное исключение и перехватит его. 

Затем создастся ОБЪЕКТ ОШИБКИ (который определили ранее) и в него поместится сообщение об ошибке и вернется пользователю  
```java
@ExceptionHandler
	private ResponseEntity<PersonErrorResponse> personNotCreateHandler(PersonCreateException e) 
	{
		PersonErrorResponse personError = new PersonErrorResponse(e.getMessage());
		return new ResponseEntity<PersonErrorResponse>(personError, HttpStatus.BAD_REQUEST);
	}	
```
