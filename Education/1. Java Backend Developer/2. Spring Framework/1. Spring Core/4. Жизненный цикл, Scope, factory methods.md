***Scope*** определяет каким создастся бин
## Singleton

Singleton - scope который создается ==по умолчанию==
```xml
<bean id="musicBean"
	  class="ru.tudyspring.myproject.ClassicMusic"
	  scope="singleton">
</bean>
```

> [!Singleton]
> По умолчанию создается ОДИН bean-объект (создается до вызова метода getBean())

То есть при вызове метода getBean возвращается одна и та же ссылка

![[Pasted image 20241016190011.png]]

Этот Scope используется тогда, когда у bean НЕТ ИЗМЕНЯЕМЫХ СОСТОЯНИЙ. Потому что бы если мы изменим по ссылке объект, он изменится для всех ссылающегося на него. 
## Prototype

> [! Prototype] 
> Prototype каждый раз создает новый bean при вызове метода getBean()

Используется тогда, когда объект ИМЕЕТ изменяемые состояния

# Scope используемые в MVC

## Request
Бин с областью видимости `request` создается для каждого HTTP-запроса.

Это значит, что каждый раз при поступлении нового запроса создается новый экземпляр бина. Когда запрос завершится, бин будет уничтожен.

## Session
Бин с областью видимости `session` создается один раз на **HTTP-сессию**. 

Это значит, что объект будет доступен в течение всей сессии пользователя. Если пользователь начинает новую сессию, бин будет создан заново.

## Init-method

==Init-method== запускается в ходе инициализации бина (инициализация ресурсов, обращение к внешним файлам, запуск БД). И лишь потом он передается пользователю.

## Destroy-method
==Destroy-method== вызывается в ходе уничтожения бина, при завершении приложения (очищение ресурсов, закрытие потоков ввода/вывода, закрытие доступа к БД)

Выглядят оба метода в коде следующим образом:
```xml
<bean id="musicBean"
	  class="ru.studyspring.myproject.ClassicMusic"
	  init-method="doMyInit"
	  destroy-method="doMyDestroy">
</bean>
```
==Методы ==doMyInit() и doMyDestroy() ==создаются к классе бина== ClassicMusic

## Особенности init и destroy методов

- Модификатор доступа - может быть любым
	
- Тип возвращаемого значения - может быть любым, но чаще используется void, поскольку нет возможности получить значения из этих методов
	
- Название - может быть любым
	
- Аргументы методы - ==НЕ должны принимать никаких аргументов==

Для бинов использующих scope `prototype` Spring НЕ вызывает destroy-методов

# 3. Factory-method

Factory-method - паттерн программирования

> [!NOTE] Factory-method
> Паттерн предлагает создавать объекты не на прямую через оператор `new`, а через вывоз особого ФАБРИЧНОГО МЕТОДА.

Объекты всё равно будут создаваться через оператор `new`, но это будет делать фабричный метод - иногда это бывает полезно

Для того чтобы сказать Springу что данный бин необходимо создавать через фабричный метод, то необходимо сделать следующие:

```xml
<bean id="musicBean"
	  class="ru.studyspring.myproject.ClassicMusic"
	  factory-method="myFactoryMethod">
</bean>
```


> [!NOTE] Особенности
> Factory-method должен быть СТАТИЧЕСКИМ и быть определенным в классе
> 
> Даже если бин определен как singleton, а для создания используется фабричный метод, который при вызове создает новый объект, объект все равно будет SINGLETON. 
> То есть если два раза вызвать метод getBean, то ссылки будут указывать на один и тот же объект
