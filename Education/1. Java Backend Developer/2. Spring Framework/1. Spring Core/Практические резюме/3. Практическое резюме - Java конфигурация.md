# 2. Создание класса КОНФИГУРАТОРА 

> [!NOTE] Аннотация `@Configuration`
> Класс конфигуратор должен быть помечен аннотацией `@Configuration`
> 
> Если нужно сканировать другие классы с аннотацией `@Component` необходимо использовать также аннотацию `@ComponentScan(путь_до_пакета)`

```java
@Configuration
@ComponentScan("ru.springcore.tasks.springCore.task1")
public class SpringCoreConfigurator {

}
```

Данный класс РАВНОСИЛЕН пустому файлу xml конфигурации:
```xml
<beans  xmlns="http://www.springframework.org/schema/beans"
        xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
        xmlns:context="http://www.springframework.org/schema/context"
        xsi:schemaLocation="http://www.springframework.org/schema/beans
        http://www.springframework.org/schema/beans/spring-beans.xsd
        http://www.springframework.org/schema/context
        http://www.springframework.org/schema/context/spring-context.xsd">

		<!-- Здесь создаем бины -->
</beans>
```

# 2.1 Специальная конфигурация бинов
Для специфичной конфигурации бинов, можно использовать аннотацию `@Bean` ДЛЯ МЕТОДОВ создающий бины

```java
@Configuration
@ComponentScan("ru.springcore.tasks.springCore.task1")
public class SpringCoreConfigurator {
	//Специфичная конфигурация
	@Bean
	public MessageService messageService() {
		return new MessageService();
	}
}
```

# 3. Запуск контекста 
Для запуска контекста используется класс `AnnotationConfigApplicationContext`

> [!NOTE] Класс `AnnotationConfigApplicationContext`
> ```java
> ApplicationContext context = new AnnotationConfigApplicationContext(SpringCoreConfigurator.class);
> ```
> 
> В параметре указываем КЛАСС КОНФИГУРАТОР, который мы создали

```java
public class App 
{
    public static void main( String[] args )
    {
        ApplicationContext context = new AnnotationConfigApplicationContext(SpringCoreConfigurator.class);
        
        UserService user = context.getBean("userService",UserService.class);
        
        user.printMessage();
        
    }
}
```
