## --------------------**HTTP-запрос (request)**--------------------

У HTTP-запроса как и у любого другого протокола есть своя структура пакета.

![[Pasted image 20241104093932.png]]

В `Host` указывается адрес сервера, к которому мы делаем запрос

В протоколе HTTP блок =="Тело"== (`Body`) используется ==для передачи основного содержимого HTTP-запроса== или ответа.
Он несет в себе данные, которые отправляются от клиента к серверу (например, при загрузке файлов) или от сервера к клиенту (например, при передаче HTML-кода страницы или других ресурсов).

==В ответах на запросы==, особенно при запросах `GET`, ==тело содержит запрашиваемый контент==, такой как ==HTML-страница==, изображение, JSON-данные, аудио и видео файлы. ==Сервер вставляет этот контент в тело ответа==, чтобы клиент мог его использовать.

## **Методы HTTP**

Методы позволяют понять серверу, какое действие хочет сделать клиент

### **Get-запрос**
Самый используемый запрос отвечающий за:
- Переход по адресу
- Переход по ссылке
- Поисковой запрос
- Открытие видео на YouTube


> [!NOTE] Get-запрос
> Тело  Get-запроса ПУСТОЕ (никакие данные не передаются)
> Идемпотентный - ничего не меняет на сервере

Иногда все же необходимо передать некоторые данные на сервер. 
Для этого ==данные отправляются== в самом адресе ==через параметры==

![[Pasted image 20241104102002.png]]
### **Post-запрос**
Post (от англ. Публиковать)

Post запрос отвечает за:
- Изменение данных на сервере
- Добавление данных на сервер

Часто используется для отправки данных с веб-форм, но не только для этого (создать новую учетку, добавить запись в сообщество и т.д)

> [!NOTE] Post-запрос
> Все параметры хранятся в ТЕЛЕ запроса
> Тип данных, передаваемых в ТЕЛЕ запроса, указывается в ЗАГОЛОВКЕ `Content-Type`

![[Pasted image 20241104102621.png]]

==Content-Type может быть:==
###### application/x-www-form-urlencoded
- Используется для передачи данных формы, когда данные кодируются в URL-подобном формате (пары "ключ=значение").
###### multipart/form-data
- Используется для отправки данных формы, включая файлы.
- Позволяет передавать как текстовые данные (например, текстовые поля формы), так и файлы, такие как изображения или документы.
- Данные в теле запроса отправляются в виде частей (part), каждая из которых имеет собственные заголовки.
###### application/json
- Используется для передачи данных в формате JSON.
- Это популярный формат для обмена данными между клиентом и сервером, особенно в REST API.
###### application/xml
- Используется для передачи данных в формате XML.
- XML-формат применяется в некоторых API, особенно в более старых или специфичных для бизнеса системах.
###### text/plain
- Применяется, если данные передаются как обычный текст.
###### application/octet-stream
- Используется для передачи бинарных данных, если формат данных точно не известен или сервер ожидает двоичные данные.
- Это может быть файл любого формата: изображения, видео, сжатые файлы и т.д.
###### application/pdf, image/png, image/jpeg, и другие специфические типы
- Эти значения применяются для передачи данных определенного типа. Например, `application/pdf` указывает, что в теле запроса находится PDF-файл, а `image/png` или `image/jpeg` — изображение в соответствующем формате.
