Файл **`web.xml`** — это ==конфигурационный файл для **web-приложений** на основе **Servlet API**==. Он используется для настройки и развертывания приложения в веб-контейнерах (например, **Tomcat**, **Jetty** и других), а также для указания конфигураций, таких как **серверные ресурсы**, **фильтры**, **сервлеты** и **слушатели**.

Используется для:
1. **Настройка сервлетов**: Файл **`web.xml`** используется для настройки **сервлетов**, включая их регистрацию и сопоставление с URL. В нем можно указать, какие сервлеты обрабатывают запросы на определенные URL.
```xml
<servlet>
    <servlet-name>dispatcher</servlet-name>
    <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
    <load-on-startup>1</load-on-startup>
</servlet>

<servlet-mapping>
    <servlet-name>dispatcher</servlet-name>
    <url-pattern>/</url-pattern>
</servlet-mapping>
```

2. **Настройка фильтров**: В **`web.xml`** можно определять **фильтры** для обработки запросов и ответов, например, для логирования, аутентификации или сжатия данных. Фильтры работают до или после того, как запрос будет передан сервлету.
```xml
<filter>
    <filter-name>encodingFilter</filter-name>
    <filter-class>org.springframework.web.filter.CharacterEncodingFilter</filter-class>
    <init-param>
        <param-name>encoding</param-name>
        <param-value>UTF-8</param-value>
    </init-param>
</filter>

<filter-mapping>
    <filter-name>encodingFilter</filter-name>
    <url-pattern>/*</url-pattern>
</filter-mapping>
```

3. **Настройка слушателей**: В **`web.xml`** можно настроить **слушателей** (listeners), которые реагируют на события в жизни веб-приложения, такие как стартап приложения, уничтожение приложения, сессии пользователей и т.д.
```xml
<listener>
    <listener-class>org.springframework.web.context.ContextLoaderListener</listener-class>
</listener>
```

4. **Настройка настроек сессии и безопасности**: Также можно указать настройки, такие как **максимальная продолжительность сессии**, **параметры безопасности**, **кеширование** и другие параметры

# 2. Теги в web.xml

### 1. **`<web-app>`**
Это корневой элемент в файле **`web.xml`**. Все остальные элементы конфигурации веб-приложения должны быть вложены в него.

**Атрибуты**:
- `xmlns`: Пространство имен для схемы XML, которое указывает на версию спецификации.
- `xmlns:xsi`: Пространство имен для схемы XML, которое используется для проверки схемы.
- `xsi:schemaLocation`: Определяет расположение схемы XML.
- `version`: Указывает версию спецификации Servlet, например, `3.0`, `4.0`, и т.д.

```xml
<?xml version="1.0" encoding="UTF-8"?>
<web-app xmlns="http://java.sun.com/xml/ns/javaee"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://java.sun.com/xml/ns/javaee http://java.sun.com/xml/ns/javaee/web-app_3_0.xsd"
         version="3.0">

<!-- Другой вариант (использовался ранее) -->
<?xml version="1.0" encoding="UTF-8"?>
<web-app xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" 
	xmlns="http://xmlns.jcp.org/xml/ns/javaee" 
	xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee 
	http://xmlns.jcp.org/xml/ns/javaee/web-app_3_1.xsd" 
	id="WebApp_ID" version="3.1">
```
***
### 2. **`<servlet>`**
Определяет сервлет, который будет обрабатывать HTTP-запросы.

Атрибуты:
- `servlet-name`: Уникальное имя сервлета.
- `servlet-class`: Полное имя класса сервлета (например, `org.springframework.web.servlet.DispatcherServlet`).
- `init-param`: Параметры инициализации сервлета (необязательные параметры, которые передаются при запуске сервлета).
- `load-on-startup`: Указывает, должен ли сервлет быть загружен при запуске веб-приложения (число от 1 до N).

```xml
<servlet>
    <servlet-name>dispatcher</servlet-name>
    <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
    <load-on-startup>1</load-on-startup>
</servlet>
```
***
### 3. **`<servlet-mapping>`**
Связывает сервлет с определенным URL-паттерном.

Атрибуты:
- `servlet-name`: имя сервлета, которое связывается с этим URL.
- `url-pattern`: шаблон URL, который будет обрабатываться данным сервлетом.

```xml
<servlet-mapping>
    <servlet-name>dispatcher</servlet-name>
    <url-pattern>/</url-pattern>
</servlet-mapping>
```
***
### 4. **`<filter>`**
Определяет фильтр, который будет обрабатывать запросы до и/или после того, как они будут переданы сервлету.

Атрибуты:
- `filter-name`: Имя фильтра.
- `filter-class`: Полное имя класса фильтра (например, `org.springframework.web.filter.CharacterEncodingFilter`).
- `init-param`: Инициализационные параметры для фильтра.

```xml
<filter>
    <filter-name>encodingFilter</filter-name>
    <filter-class>org.springframework.web.filter.CharacterEncodingFilter</filter-class>
    <init-param>
        <param-name>encoding</param-name>
        <param-value>UTF-8</param-value>
    </init-param>
</filter>
```
***
### 5. **`<filter-mapping>`**
Указывает, на какие URL-шаблоны должен реагировать фильтр.

**Атрибуты**:
- `filter-name`: Имя фильтра, который будет применяться.
- `url-pattern`: Шаблон URL, который будет фильтроваться.
- `servlet-name`: (Необязательный) Имя сервлета, к которому фильтр применяется.
- `dispatcher`: (Необязательный) Указывает типы запросов, для которых фильтр будет применяться (`REQUEST`, `FORWARD`, `INCLUDE`, `ERROR`).

```xml
<filter-mapping>
    <filter-name>encodingFilter</filter-name>
    <url-pattern>/*</url-pattern>
</filter-mapping>
```
***
### 6. **`<listener>`**
Определяет слушателя, который будет реагировать на события жизненного цикла веб-приложения (например, стартап или уничтожение контекста).

Атрибуты:
    - `listener-class`: класс слушателя.

```xml
<listener>
    <listener-class>org.springframework.web.context.ContextLoaderListener</listener-class>
</listener>
```
***
### 7. **`<welcome-file-list>`**
Содержит список **приветственных файлов** (файлы, которые веб-приложение будет искать при обращении к корневому URL).
```xml
<welcome-file-list>
    <welcome-file>index.html</welcome-file>
    <welcome-file>index.jsp</welcome-file>
</welcome-file-list>
```
***
### 8. **`<error-page>`**
 Указывает страницу для отображения в случае возникновения ошибки.

**Атрибуты**:
- `error-code`: HTTP код ошибки, для которой будет отображаться эта страница (например, `404`).
- `exception-type`: Тип исключения (например, `java.lang.NullPointerException`).
- `location`: Путь к странице для отображения при ошибке.

```xml
<error-page>
    <error-code>404</error-code>
    <location>/WEB-INF/errors/404.html</location>
</error-page>
```
***
### 9. **`<context-param>`**
Определяет параметры контекста для всего веб-приложения.==(БД и прочее)==

Атрибуты:
- `param-name`: имя параметра.
- `param-value`: значение параметра.

```xml
<context-param>
    <param-name>configLocation</param-name>
    <param-value>/WEB-INF/applicationContext.xml</param-value>
</context-param>
```
***
### 10. **`<security-constraint>`**
Используется для определения ограничений безопасности для определенных URL.

**Атрибуты**:
- `web-resource-collection`: Вложенный элемент, который определяет коллекцию ресурсов, к которым применяется ограничение.
- `auth-constraint`: Вложенный элемент, который указывает роли, имеющие доступ к ресурсам.
- `user-data-constraint`: Указывает на требуемый тип безопасности (например, SSL).

```xml
<security-constraint>
    <web-resource-collection>
        <web-resource-name>Admin Section</web-resource-name>
        <url-pattern>/admin/*</url-pattern>
    </web-resource-collection>
    <auth-constraint>
        <role-name>admin</role-name>
    </auth-constraint>
</security-constraint>
```
***
### 11. **`<login-config>`**
Определяет способ аутентификации (например, базовую аутентификацию или форму входа).

**Атрибуты**:
- `auth-method`: Метод аутентификации (например, `BASIC`, `FORM`).
- `realm-name`: Название области безопасности (например, `MyRealm`).

```xml
<login-config>
    <auth-method>BASIC</auth-method>
    <realm-name>MyRealm</realm-name>
</login-config>
```
***
### 12. **`<mime-mapping>`**
Связывает расширения файлов с типами MIME для правильной обработки контента.

**Атрибуты**:
- `extension`: Расширение файла (например, `json`).
- `mime-type`: MIME-тип, соответствующий расширению (например, `application/json`).

```xml
<mime-mapping>
    <extension>json</extension>
    <mime-type>application/json</mime-type>
</mime-mapping>
```
***
### 13. **`<session-config>`**
Определяет параметры сессии.
**Атрибуты**:
- `session-timeout`: Время в минутах до истечения сессии (например, `30` минут).

```xml
<session-config>
    <session-timeout>30</session-timeout>
</session-config>
```

***
# Пример конфигурации

```xml
<web-app xmlns="http://java.sun.com/xml/ns/javaee"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://java.sun.com/xml/ns/javaee http://java.sun.com/xml/ns/javaee/web-app_3_0.xsd"
         version="3.0">

    <!-- Настройка сервлета -->
    <servlet>
        <servlet-name>dispatcher</servlet-name>
        <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
        <load-on-startup>1</load-on-startup>
    </servlet>

    <!-- Настройка маппинга сервлета -->
    <servlet-mapping>
        <servlet-name>dispatcher</servlet-name>
        <url-pattern>/</url-pattern>
    </servlet-mapping>

    <!-- Настройка фильтра -->
    <filter>
        <filter-name>encodingFilter</filter-name>
        <filter-class>org.springframework.web.filter.CharacterEncodingFilter</filter-class>
        <init-param>
            <param-name>encoding</param-name>
            <param-value>UTF-8</param-value>
        </init-param>
    </filter>

    <filter-mapping>
        <filter-name>encodingFilter</filter-name>
        <url-pattern>/*</url-pattern>
    </filter-mapping>

    <!-- Настройка слушателя -->
    <listener>
        <listener-class>org.springframework.web.context.ContextLoaderListener</listener-class>
    </listener>

</web-app>

```
