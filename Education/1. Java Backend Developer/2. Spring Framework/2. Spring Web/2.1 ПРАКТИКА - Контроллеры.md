==Используем проект== который был создан ==в практике 1.1==

## Создание нового контроллера
Удалим старый контроллер и создадим новый
![[Pasted image 20241102183318.png]]
```java
@Controller
public class MyWelcomeController {
	@GetMapping("/hello")
	public String sayHello() {
		return "myfirst/hellopage";
	}
	@GetMapping("/bye")
	public String sayBye() {
		return "myfirst/byepage";
	}
}
```
В контроллере есть два метода, каждый из которых возвращает определенную страницу html

## Создание представлений
Теперь необходимо создать представления, который возвращают методы. Представления должны храниться в папке WEB-INF

Создадим подпапки views/myfirst в WEB-INF и создадим два представления
![[Pasted image 20241102183803.png]]

Представление hellopage
```html
<!DOCTYPE html>
<html>
<head>
	<meta charset="UTF-8">
	<title>Hello my fried</title>
</head>
<body>
	Hello!
	<a href="/simpleMVC/bye"> Say good bye!</a>
</body>
</html>
```

Представление byepage
```html
<!DOCTYPE html>
<html>
<head>
	<meta charset="UTF-8">
	<title>Good bye</title>
</head>
<body>
	Good bye!
	<a href="/simpleMVC/hello"> Say hello!</a>
</body>
</html>
```

==Для переадресации== с страницы на страницу используется тег `<a>` с параметром `href`
## Использование `@RequestMapping`
Пометим класс контроллера аннотацией `@RequestMapping`
```java
@Controller
@RequestMapping("/home")
public class MyWelcomeController {
. . .
```
Теперь все методы которые находились по адресу `/simpleMVC/hello` будут находиться по адресу `/simpleMVC/home/hello`

Однако и ссылки теперь не будут работать, поскольку в коде они ссылаются на `/bye` и `/hello`, а не на `/home/. . .`
