# 📕 1. Утилита javac (Java Compiler)


> [!NOTE] **Утилита javac** 
> Это **компилятор**, который `транслирует исходный код в байт-код`. 
> - На `вход` компилятор принимает `файл с расширением` `.java`, 
> - На `выходе` `выдаёт файл` с расширением `.class`, содержащий `байт-код`, необходимый `для исполнения программы виртуальной машиной.`
> 


```bash
# Использование:
# javac <опции> <исходные файлы>

@<filename>                  
    Считать список опций и имена файлов из указанного файла

-Akey[=value]                
    Передать параметры аннотационным процессорам

--add-modules <module>(,<module>)* 
    Добавить модули при сборке; можно указать ALL-MODULE-PATH для всех модулей на пути

--boot-class-path <path>, -bootclasspath <path> 
    Переопределить расположение базовых (bootstrap) классов

--class-path <path>, -classpath <path>, -cp <path> 
    Указать путь, где искать пользовательские классы и аннотационные процессоры

-d <directory>               
    Указать каталог, куда сохранять скомпилированные .class файлы

-deprecation                 
    Показать предупреждения о использовании устаревших API

--enable-preview             
    Включить экспериментальные (preview) возможности языка

-encoding <encoding>         
    Указать кодировку исходных файлов (например, UTF-8)

-endorseddirs <dirs>         
    Переопределить путь к "утверждённым" стандартам Java (устарело)

-extdirs <dirs>              
    Переопределить путь к установленным расширениям Java (устарело)

-g                           
    Сгенерировать всю отладочную информацию

-g:{lines,vars,source}       
    Сгенерировать только выбранную отладочную информацию (строки, переменные, исходники)

-g:none                      
    Не создавать отладочную информацию

-h <directory>               
    Указать каталог для размещения сгенерированных нативных заголовочных файлов (JNI)

--help, -help, -?            
    Показать справку

--help-extra, -X             
    Показать дополнительные (внутренние) опции компилятора

-implicit:{none,class}       
    Указать, нужно ли генерировать .class файлы для неявно упомянутых файлов

-J<flag>                     
    Передать флаг напрямую JVM, которая выполняет javac

--limit-modules <module>(,<module>)* 
    Ограничить список наблюдаемых модулей

--module <module>(,<module>)*, -m <module>(,<module>)* 
    Компилировать только указанные модули

--module-path <path>, -p <path> 
    Указать путь, где искать модули приложения

--module-source-path <module-source-path> 
    Путь к исходникам, если проект содержит несколько модулей

--module-version <version>   
    Указать версию модуля, который компилируется

-nowarn                      
    Не выводить предупреждения

-parameters                  
    Добавить метаданные о параметрах методов (для reflection API)

-proc:{none,only,full}       
    Контролировать выполнение аннотационной обработки:
        none — не обрабатывать аннотации
        only — только обработать аннотации, без компиляции
        full — полная компиляция с обработкой

-processor <class1>[,<class2>,...] 
    Указать конкретные аннотационные процессоры

--processor-module-path <path> 
    Указать путь, где искать модули с аннотационными процессорами

--processor-path <path>, -processorpath <path> 
    Указать путь, где искать классы аннотационных процессоров

-profile <profile>           
    Проверить, что используемые API доступны в заданном профиле (Java EE / SE)

--release <release>, -source <release> 
    Установить совместимость исходного кода с конкретной версией Java (например, 8, 11, 17)

--source-path <path>, -sourcepath <path> 
    Указать путь к исходным файлам

--system <jdk>|none          
    Переопределить расположение системных модулей

--target <release>, -target <release> 
    Сгенерировать .class файлы, совместимые с указанной версией Java

--upgrade-module-path <path> 
    Указать путь к "обновляемым" системным модулям

-verbose                     
    Выводить подробные сообщения о процессе компиляции

--version, -version          
    Показать версию компилятора

-Werror                      
    Прервать компиляцию, если есть предупреждения
```

## Топ флагов утилиты
|№|Флаг|Назначение|Пример использования|
|---|---|---|---|
|1️⃣|`-d <directory>`|Куда поместить `.class`-файлы после компиляции|`javac -d out src/com/example/*.java`|
|2️⃣|`-classpath` или `-cp`|Путь, где искать сторонние библиотеки и зависимости|`javac -cp lib/mysql-connector.jar -d out src/*.java`|
|3️⃣|`--release <версия>`|Компиляция под конкретную версию Java API (начиная с JDK 9)|`javac --release 11 Main.java`|
|4️⃣|`-source <версия>` и `-target <версия>`|Поддержка исходников и совместимость байткода со старой JVM|`javac -source 8 -target 8 Main.java`|
|5️⃣|`-encoding <кодировка>`|Задает кодировку исходных файлов (особенно важно для кириллицы)|`javac -encoding UTF-8 Main.java`|
|6️⃣|`-verbose`|Подробный вывод — показывает, какие файлы компилируются|`javac -verbose Main.java`|
|7️⃣|`-nowarn`|Отключает предупреждения|`javac -nowarn Main.java`|
|8️⃣|`-Xlint`|Включает подробные предупреждения о потенциальных проблемах|`javac -Xlint:all Main.java`|
|9️⃣|`-Werror`|Превращает предупреждения в ошибки (например, на CI/CD)|`javac -Werror -Xlint:all Main.java`|
|🔟|`-J<option>`|Передает аргументы непосредственно JVM, на которой запускается `javac`|`javac -J-Xmx512m Main.java` (ограничивает память компилятора)|

### Дополнительные 

|Флаг|Назначение|Пример|
|---|---|---|
|`-parameters`|Добавляет в байткод имена параметров методов (используется при reflection, напр. в Spring)|`javac -parameters -d out Main.java`|
|`-proc:none`|Отключает аннотационную обработку (ускоряет сборку, если не нужна)|`javac -proc:none Main.java`|
|`-h <directory>`|Генерирует заголовочные файлы для JNI (native) методов|`javac -h headers com/example/NativeLib.java`|
|`--add-modules`|Явно указывает, какие модули подключить при сборке|`javac --add-modules java.sql Main.java`|
|`--module-path`|Путь для поиска модулей (аналог classpath, но для модульной системы)|`javac --module-path mods -d out module-info.java Main.java`|

## 💡 Частые комбинации флагов

### 💬 Пример 1. Компиляция всего проекта в папку `out`:

`javac -d out -encoding UTF-8 src/com/example/*.java`

### 💬 Пример 2. Компиляция с зависимостью:

`javac -cp lib/gson.jar -d out src/com/example/*.java`

### 💬 Пример 3. Компиляция под Java 8:

`javac -source 8 -target 8 -d out src/com/example/*.java`

### 💬 Пример 4. Проверка кода на предупреждения:

`javac -Xlint:all -Werror -d out src/com/example/*.java`

### 💬 Пример 5. Использование модулей:

`javac --module-path mods -d out --module my.module`

# 📕 2. Утилита jar

> [!NOTE] **Утилита jar**
> Это `утилита архивирования`, которая используется для `создания сжатого архивного файла` - JAR/WAR файл. 
> 
> Jar также может пригодиться, когда нужно сравнить одну версию выпуска кода с другой или указать точку входа для встроенных JAR-файлов или других исполняемых файлов
> 

```bash
jar --create --file myJarApp.jar --main-class MyMain -C . MyMain.class
```
> [!NOTE] `-C` требует два параметра, первый директория, а второй имя файлов
> 

```bash
# 🟢 Основные режимы работы (взаимоисключающие):🟢

-c, --create
    Создать новый архив (JAR/WAR-файл)

-i, --generate-index=FILE
    Создать индексный файл (ускоряет загрузку классов из нескольких JAR)

-t, --list
    Показать список содержимого архива (таблицу файлов)

-u, --update
    Обновить существующий архив (добавить или заменить файлы)

-x, --extract
    Извлечь указанные файлы (или все) из архива

-d, --describe-module
    Показать модульный дескриптор (module-info.class) или имя автомодуля

--validate
    Проверить целостность архива (особенно для multi-release JAR — JAR с версиями для разных Java)

---------------------------------------------------------
# 🟢Модификаторы, работающие в любом режиме:🟢

-C DIR
    💡 Перейти в указанную директорию и добавить файлы оттуда 💡
    (аналог pushd в Linux; удобно при сборке)
    
-f, --file=FILE
    💡Указать имя создаваемого/читаемого архива.💡
    Если не указано — используется стандартный ввод/вывод.

--release VERSION
    Поместить все следующие файлы в версионную папку META-INF/versions/<VERSION>/
    (для multi-release JAR — разные реализации для разных версий Java)

-v, --verbose
    Подробный вывод (отображает, какие файлы добавляются или извлекаются)

---------------------------------------------------------
# 🟢Модификаторы только для create / update:🟢

-e, --main-class=CLASSNAME
    ✅Указать главный класс (Main-Class - ❗❗Класс с main()❗❗), который будет в манифесте.✅
    ⭐⭐Делает JAR исполняемым⭐⭐ (`java -jar app.jar`).

-m, --manifest=FILE
    Добавить в архив указанный файл манифеста (META-INF/MANIFEST.MF)

-M, --no-manifest
    Не создавать манифест вообще

--module-version=VERSION
    Установить версию модуля (для модульного JAR)

--hash-modules=PATTERN
    Вычислить и записать хэши модулей, зависящих от этого JAR
    (для обеспечения безопасности зависимостей)

-p, --module-path
    Указать путь, где искать модули при создании хэшей (вместе с --hash-modules)

---------------------------------------------------------
# 🟢Модификаторы для create / update / generate-index:🟢

-0, --no-compress
    Не использовать сжатие ZIP — просто сохранять файлы «как есть»

--date=TIMESTAMP
    Указать дату и время (в формате ISO-8601) для файлов в архиве

---------------------------------------------------------
# 🟢Прочие опции:🟢

-?, -h, --help[:compat]
    Показать справку (или совместимую старую версию помощи)

--help-extra
    Показать дополнительные опции

--version
    Показать версию утилиты jar

```

## 🔟 Топ реально используемых команд `jar` 💡
| №    | Команда                                     | Что делает                                                         | Пример                                        |
| ---- | ------------------------------------------- | ------------------------------------------------------------------ | --------------------------------------------- |
| 1️⃣⭐ | `jar --create --file app.jar -C out .`      | Создаёт JAR-файл из всех `.class` в папке `out`                    | `jar --create --file app.jar -C out .`        |
| 2️⃣  | `jar cfe app.jar com.example.Main -C out .` | Создаёт **исполняемый JAR**, указывая главный класс (`Main-Class`) | `jar cfe myapp.jar com.example.Main -C out .` |
| 3️⃣  | `jar --list --file app.jar`                 | Показывает содержимое архива                                       | `jar --list --file app.jar`                   |
| 4️⃣  | `jar tf app.jar`                            | Короткая версия команды выше (`t` = list, `f` = file)              | `jar tf app.jar`                              |
| 5️⃣  | `jar xf app.jar`                            | Извлекает все файлы из архива                                      | `jar xf app.jar`                              |
| 6️⃣  | `jar xf app.jar META-INF/MANIFEST.MF`       | Извлекает только манифест                                          | `jar xf app.jar META-INF/MANIFEST.MF`         |
| 7️⃣  | `jar uf app.jar com/example/NewClass.class` | Добавляет (обновляет) файл в существующий JAR                      | `jar uf app.jar com/example/NewClass.class`   |
| 8️⃣  | `jar cmf manifest.txt app.jar -C out .`     | Создаёт JAR с собственным манифестом                               | `jar cmf manifest.txt app.jar -C out .`       |
| 9️⃣  | `jar --describe-module --file app.jar`      | Показывает информацию о модуле (`module-info.class`)               | `jar --describe-module --file app.jar`        |
| 🔟   | `jar --validate --file app.jar`             | Проверяет целостность и правильность multi-release JAR             | `jar --validate --file app.jar`               |

## Минимальный практический пример

1️⃣ Скомпилировать код:

`javac -d out src/com/example/Main.java`

2️⃣ Создать исполняемый JAR:

`jar cfe app.jar com.example.Main -C out .`

3️⃣ Запустить:

`java -jar app.jar`

# 📕 3. Утилита java


> [!NOTE] **Утилита java** - запускает программу
> Это **утилита**, которая `загружает .class файлы` и `интерпретирует байт-код в машинные инструкции.` 
> 
> С её помощью можно `запускать JAR-файл на выполнение`
> 

```bash
# Команда: java --help
# Назначение: запуск программ на Java и управление параметрами JVM

Usage:
  java [options] <mainclass> [args...]
      # Запуск класса с методом main()

  java [options] -jar <jarfile> [args...]
      # Запуск исполняемого JAR-файла

  java [options] -m <module>[/<mainclass>] [args...]
  java [options] --module <module>[/<mainclass>] [args...]
      # Запуск главного класса в модуле (Java 9+)

  java [options] <sourcefile> [args]
      # Запуск одиночного исходного файла (Java 11+)


# После основного класса, файла или JAR идут аргументы, которые передаются методу main().



# 🟢--- Основные параметры ---🟢

-cp, -classpath, --class-path <path>
    # Пути поиска классов и библиотек (через ; на Windows, : на Linux)
    # Пример: java -cp .;lib/* MainClass

-p, --module-path <path>
    # Пути поиска модулей

--upgrade-module-path <path>
    # Пути к модулям, заменяющим стандартные системные (для обновления)

--add-modules <name>[,<name>...]
    # Указывает дополнительные модули, которые нужно подключить
    # Возможные значения:
    #   ALL-DEFAULT — стандартные модули
    #   ALL-SYSTEM  — все системные модули
    #   ALL-MODULE-PATH — все на module-path

--enable-native-access <modules>
    # Разрешает определённым модулям доступ к нативным операциям

--list-modules
    # Вывести список всех доступных модулей и завершить работу

-d, --describe-module <module>
    # Показать описание конкретного модуля и завершить работу

--dry-run
    # Создать JVM, загрузить главный класс, но не вызывать main()
    # Используется для проверки аргументов и конфигурации

--validate-modules
    # Проверить корректность всех модулей и выйти
    # Полезно для обнаружения конфликтов в module-path

-D<имя>=<значение>
    # Установить системное свойство (например: -Dfile.encoding=UTF-8)

-verbose:[class|module|gc|jni]
    # Подробный вывод (какие классы, модули, GC или JNI используются)

-version, --version
    # Показать версию JVM и выйти

-showversion, --show-version
    # Показать версию JVM, но продолжить выполнение

--show-module-resolution
    # Показать, как JVM выбирает и разрешает модули при старте

-?, -h, -help, --help
    # Вывести справку

-X, --help-extra
    # Показать расширенные (внутренние) параметры JVM



# 🟢--- Управление утверждениями (assertions) ---🟢

-ea[:<пакет>|:<класс>], -enableassertions[:<пакет>|:<класс>]
    # Включить утверждения (assert)

-da[:<пакет>|:<класс>], -disableassertions[:<пакет>|:<класс>]
    # Отключить утверждения

-esa, -enablesystemassertions
    # Включить системные утверждения

-dsa, -disablesystemassertions
    # Отключить системные утверждения



# 🟢--- Отладка и агенты ---🟢

-agentlib:<libname>[=<options>]
    # Подключить нативную библиотеку агента
    # Пример: -agentlib:jdwp=transport=dt_socket,server=y,address=5005

-agentpath:<pathname>[=<options>]
    # Загрузить библиотеку агента по полному пути

-javaagent:<jarpath>[=<options>]
    # Подключить Java агент (через java.lang.instrument API)



# 🟢--- Графика ---🟢 

-splash:<imagepath>
    # Показать изображение-заставку при запуске программы
    # Автоматически выбирает изображение подходящего разрешения



# 🟢--- Прочее ---🟢

@argument files
    # Файл с аргументами командной строки (можно передавать опции через файл)

-disable-@files
    # Запретить использование @файлов для аргументов

--enable-preview
    # Разрешить использование preview-функций языка Java



# 🟢--- Формат указания аргументов ---🟢
--<name>=<value> или --<name> <value>
    # Оба варианта допустимы

```
## Примеры

| Категория                           | Ключ                             | Описание                                                                            |
| ----------------------------------- | -------------------------------- | ----------------------------------------------------------------------------------- |
| 🎯 **Запуск приложения**            | `java <MainClass>`               | Запуск программы, где `<MainClass>` — имя класса с методом `main()`                 |
|                                     | `java -jar file.jar`             | Запуск приложения, упакованного в JAR-файл (если в манифесте прописан `Main-Class`) |
|                                     | `--module-path path`             | Указывает путь к модулям (используется в Java 9+ при модульной структуре)           |
|                                     | `--module module/class`          | Запускает модульное приложение (Java 9+)                                            |
|                                     |                                  |                                                                                     |
| 🧱 **Класспуть (ClassPath)**        | `-cp path` или `-classpath path` | Указывает, где искать классы и пакеты (папки, JAR-файлы)                            |
|                                     | `--class-path path`              | Аналог `-cp`, но в длинной форме                                                    |
|                                     |                                  |                                                                                     |

|                                     | `-XX:MaxMetaspaceSize=<size>`    | Лимит памяти для метапространства классов                                           |
|                                     |                                  |                                                                                     |
| 🧵 **Отладка и диагностика**        | `-verbose:class`                 | Показывает загружаемые классы                                                       |

|                                     | `--list-modules`                 | Показать список доступных модулей                                                   |

## Часто используемые команды `java` с примерами
| №   | Команда                                                                                        | Что делает                                                              |
| --- | ---------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------- |
| 1️⃣ | `java -version`                                                                                | Показывает версию JVM и JRE                                             |
| 2️⃣ | `java -jar app.jar`                                                                            | Запускает приложение из JAR-файла                                       |
| 3️⃣ | `java -cp . MainClass`                                                                         | Запускает класс из текущей папки                                        |
| 4️⃣ | `java -cp lib/*;bin MainClass`                                                                 | Запускает с указанием путей к зависимостям (Windows, `;` — разделитель) |
| 5️⃣ | `java -Xms512m -Xmx2g -jar server.jar`                                                         | Ограничивает выделяемую память для JVM                                  |
| 6️⃣ | `java -Dspring.profiles.active=prod -jar app.jar`                                              | Передаёт системное свойство приложению (часто в Spring)                 |
| 7️⃣ | `java --module-path mods --module com.app/module.Main`                                         | Запуск модульного приложения (Java 9+)                                  |
| 8️⃣ | `java -verbose:gc -jar app.jar`                                                                | Показывает логи сборки мусора                                           |
| 9️⃣ | `java -agentlib:jdwp=transport=dt_socket`,<br>`server=y,suspend=n,address=*:5005 -jar app.jar` | Запуск с возможностью отладки по порту 5005                             |
| 🔟  | `java --enable-preview -jar future.jar`                                                        | Запуск программы, использующей preview-функции языка                    |
