# ☢Особенности:
- Работает **одним потоком**
    
- Подходит для **одноядерных** или **небольших приложений**
    
- Полная остановка приложения (`stop-the-world`) на сборку. В это время программа полностью останавливается, и ожидает когда GC закончит работу 
    
- Использует: **Copying GC** для Young и **Mark-Compact** для Old

***
# 1. Принцип работы
Память разделяется на младшее и старшее поколение

![[Pasted image 20250622125031.png]]

🟢 Eden - молодые объекты, только созданные
🟠Survivor S0 и S1 - выжившие объекты после первой сборки
🔴Tenured - объекты которые долго не удаляются

## 1.1 Малая сборка - Eden Space заполнен
##### Первая сборка
Все созданные объекты помещаются в Eden, как только память в Eden заканчивается, начинается сборка мусора.

Все выжившие объекты помещаются в Survivor Zero (S0)

![[Pasted image 20250622125831.png]]
##### Вторая сборка
Когда заполняется Eden и SO, то выжившие объекты помещаются в S1
![[Pasted image 20250622135803.png]]

## 1.2 Большая сборка
• Выполняется когда в старшем поколении не хватает места
• ==Совмещается с малой сборкой==
• В старшем поколении происходит сборка типа Mark-Sweep-Compact: помечаются живые объекты, удаляются неиспользуемые, оставшиеся уплотняются
