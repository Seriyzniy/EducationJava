☢Особенности:
- ==Малая сборка аналогична Parallel GC==
- **Минимизирует паузы** (работает параллельно с приложением)
- Старое поколение очищается **без полной остановки**
- Малые и старшие сборки разнесены во времени (==не выполняются одновременно==)
- 🔴Минус: может приводить к **фрагментации памяти**
- Устарел с Java 9, удалён в Java 14

# 1. Большая сборка
Выполняется постоянно в фоновом режиме, не дожидаясь заполнения старшего поколения

Называется старшей для данного сборщика, так как ==не совмещена с малой сборкой==

Допускает наличие "плавающего мусора" - неиспользуемых объектов, не найденных в процессе сборки

## 1.1 Этапы большой сборки

##### Snapshot-At-The-Beginning (SATB)
> [!NOTE] Любой новый объект == живой объект
> В список живых попадают все объекты, которые были таковыми на момент начала работы алгоритмы, плюс все объекты созданные за время его выполнения также считаются живыми


1. **Остановка** основных потоков приложения и **пометки всех объектов**, напрямую доступных из корней
2. **Приложение возобновляет свою работу**, а ==сборщик параллельно с ним производит поиск всех живых== объектов, ==доступных по  ссылкам== из первых помеченных объектов
3. Повторная установка, поиск упущенных объектов
4. Удаление мусора в фоновом режиме
