**Интерфейс** — это контракт, определяющий **набор методов**, которые должен реализовать класс.  По сути это полностью абстрактный класс, который одержит только абстрактные методы

Интерфейсы — важнейший ==механизм **абстракции** и **многократного наследования== типа** в Java.

Это способ указать: "Класс поддерживает такое поведение".

***
# 💡 ОСОБЕННОСТИ интерфейсов 💡
- ==Методы по умолчанию== — `public abstract`
	
- ==РЕАЛИЗАЦИЯ метода интерфейса, ДОЛЖНА БЫТЬ== `public`
    
- ==Поля== — `public static final` (==всегда константы!==) и их НЕОБХОДИМО сразу же инициализировать
    
- Нельзя создавать экземпляры интерфейсов напрямую
    
- Класс может реализовывать **несколько интерфейсов**
    
- Интерфейс можно использовать как **тип переменной**
	
- Интерфейсы могут расширяться при помощи `extends` (при реализации необходимо реализовать все методы в цепочке наследования)
	
- Если класс не реализует все методы интерфейса, то он должен быть объявлен как абстрактный 
### 🧠 Модификаторы доступа по умолчанию в интерфейсе 

| Член интерфейса | Модификаторы по умолчанию | Примечание                                            |
| --------------- | ------------------------- | ----------------------------------------------------- |
| Поле            | `public static final`     | Только константы                                      |
| Метод           | `public abstract`         | ==Обязательно реализовать== в классе                  |
| Default метод   | `public default`          | Содержит реализацию (с Java 8)                        |
| Static метод    | `public static`           | Доступен только через имя интерфейса                  |
| Private метод   | `private`                 | Только для использования внутри интерфейса (с Java 9) |
***
### 1. Объявление интерфейса
```java
interface Flyable {       // Создание интерфейса 
    void fly();
}
```

***
### 2. Реализация интерфейса
Используется ключевое слово `implements`
```java
class Bird implements Flyable { //Реалзиация интерфейса
    public void fly() {
        System.out.println("Bird is flying");
    }
}
```

***
#### 3. Переменные интерфейса
Интерфейсы могут использоваться для импортирования общих констант в несколько классов

При реализации интерфейса, такие переменные будут доступны в области видимости как КОСНТАНТЫ

```java
interface A {
	int x = 20; //Т.к поля константы, их необходимо сразу же инициализировать
}
class MyClass implements A {
    public void greet() {
        x = 15; // Ошибка, поля интерфейа КОНСТАНТНЫ
    }
}
```

# 📕 Интерфейсы можно расширять

С помощью ключевого слова `extends` один интерфейс может быть унаследован от другого. 

Синтаксис используется такой же как и при наследовании классов

> [!NOTE] Реализация интерфейса, который наследуется от другого
> Когда класс реализует интерфейс, унаследованный от другого, ОН ДОЛЖЕН ПРЕДОСТАВИТЬ РЕАЛЗИАЦИИ ДЛЯ ВСЕХ МЕТОДОВ В ЦЕПОЧКЕ НАСЛЕДОВАНИЯ

При наследовании интерфейсов, в классе реализации обоих интерфейсов, вызовется метод ПОСЛЕДЕНГО ДОЧЕРНЕГО ИНТЕРФЕЙСА
```java
interface X {
	default void doIt() {
		System.out.println("Dafault method X");
	}
}
interface Y extends X{
	default void doIt() {
		System.out.println("Dafault method Y");
	}
}

class Test implements X, Y{
	
}


// Main
public class MyMain {
	public static void main(String[] args) {
		Test t = new Test();
		t.doIt();
	}
}

// Вывод
Dafault method Y
```

# 📕 Default методы интерфейсов  
## Что такое `default`-метод?
Это ==**метод с реализацией** внутри интерфейса==.

До Java 8 интерфейсы могли содержать **только абстрактные методы** — без тела. Это ограничивало возможности эволюции API: при добавлении нового метода во все реализации приходилось его добавлять вручную.

С Java 8 в интерфейсах можно писать методы с ключевым словом `default`, **вместе с телом метода**:
```java
interface MyInterface {
    default void greet() {
        System.out.println("Hello from interface!");
    }
}
```


> [!NOTE] ❌`ВЫЗЫВАТЬ` `default` метод `НАПРЯМУЮ` у интерфейса `НЕЛЬЗЯ`, только через класс реализатор❌
> 


> [!NOTE] 🛑НЕОБЯЗАТЕЛЬНО РЕАЛИЗОВЫВАТЬ `default` метод 🛑
> 

## 💡 Особенности `default` методов 💡
### 1. default-методы могут быть ПЕРЕОПРЕДЕЛЕНЫ в классе
```java
interface Logger {
    default void log(String message) {
        System.out.println("LOG: " + message);
    }
}

class CustomLogger implements Logger {
    public void log(String msg) {
        System.out.println("CUSTOM: " + msg);
    }
}
```

### 2. ==Вызов default-метода РОДИТЕЛЬСКОГО== интерфейса:
```java
Logger.super.log();
```

### 3. Конфликт с ДВУМЯ одинаковыми`default` - необходимо переопределить один из методов
```java
interface A {
    default void hello() {
        System.out.println("A");
    }
}

interface B {
    default void hello() {
        System.out.println("B");
    }
}

class C implements A, B {
    // ❌ Ошибка компиляции: конфликт default-методов, 
    // НЕОБХОДИМО переопределить default метод
    
    // Если убрать из наследования один из интерфейсов, или убрать метод 
    // в одном интерфейсе, ТО ОШИБКИ НЕ БУДЕТ
}


//РЕШЕНИЕ
class Test implements X,Y{
	@Override
	public void doIt() {
		X.super.doIt(); //Или Y.super.doIt()
	}
}
```

> [!NOTE] 🛑НЕОЖИДАННЫЙ КОНФЛИКТ с ДВУМЯ `default` методами🛑
> **Только в случае конфликта между default-методами двух (или более) интерфейсов с одинаковыми сигнатурами** КОМПИЛЯТОР ТРЕБУЕТ ПЕРЕОПРЕДЕЛЕНИЕ

|Ситуация|Нужно ли переопределить?|Почему|
|---|---|---|
|Один интерфейс с `default`-методом|❌ Нет|Нет конфликта|
|Один `default`, один `abstract`|✅ Да|Нужно реализовать `abstract`|
|Два интерфейса с одинаковыми `default`-методами|✅ Да|Конфликт — Java не знает, что выбрать|
|Класс сам реализует метод|❌ Нет|Конфликт снимается|

# 📕 Static методы интерфейсов
**Статический метод в интерфейсе** — это метод, который:
- ==принадлежит **самому интерфейсу**, а **не объекту**, который его реализует;==
    
- ==НЕ НАСЛЕДУЮТСЯ== ни реализующим классом, ни интерфейсом;
	
- НЕЛЬЗЯ ПЕРЕОПРЕДЕЛИТЬ
    
- ==вызывается **по имени интерфейса**==, а не по ссылке на объект класса реализации интерфейса
- ==ДОЛЖНО БЫТЬ ТЕЛО==

```java
interface MathUtil {
    static int square(int x) {
        return x * x;
    }
}

public class Demo {
    public static void main(String[] args) {
        int result = MathUtil.square(5); //Вызов статического метода интерфейса
        System.out.println(result); // 25
    }
}
```

##### Вызов static-метода
```java
Имя_интерфейса.метод()
```

# 📕 Private методы интерфейсов
Это ==**вспомогательные методы**==, доступные ==**ТОЛЬКО ВНУТРИ самого интерфейса**==, и **не видимые снаружи** — ни в классах, ни в других интерфейсах.
```java
interface MyInterface {
    default void greet() {
        log("Hello");
    }

    private void log(String msg) {
        System.out.println("[LOG]: " + msg);
    }
}
```
