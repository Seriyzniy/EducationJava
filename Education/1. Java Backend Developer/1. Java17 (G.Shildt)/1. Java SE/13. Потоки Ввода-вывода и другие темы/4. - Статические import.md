## Что такое `static import`?

Обычный `import` подтягивает **класс или интерфейс**:
```java
import java.util.Arrays;
```
→ после этого можно писать `Arrays.sort(...)` вместо полного `java.util.Arrays.sort(...)`.

А ==**static import** подтягивает **статические члены класса**== — константы (`static final`) и методы (`static`).
```java
import static java.lang.Math.*; // импорт всех статических членов

public class Demo {
    public static void main(String[] args) {
        double r = sqrt(4);   // 💡🟢💡 вместо Math.sqrt(4)
        double pi = PI;       // 💡🟢💡 вместо Math.PI
        System.out.println(r + " " + pi);
    }
}
```

## Подводные камни
**Конфликты имён**  
    Если импортировать два метода с одинаковым именем из разных классов → компилятор не поймёт, что использовать.
```java
import static java.lang.Integer.*;
import static java.lang.Long.*;

// valueOf() есть и там, и там
Integer i = valueOf("123"); // ❌ ошибка: ambiguous method
```

## Удобный прикол
```java
import static java.lang.System.out;

public class MyMain {
    public static void main(String[] args) throws InterruptedException {
    	out.println("Hello");
    }
}
```
НЕ НУЖНО писать `System.out.prinln()`, а просто `out.prinln()`
