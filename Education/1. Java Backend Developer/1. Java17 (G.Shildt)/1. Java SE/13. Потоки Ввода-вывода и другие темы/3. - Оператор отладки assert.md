`assert` — это ==**оператор для отладки**, встроенный в язык Java.  ==
Он проверяет некоторое **булево условие**, и если оно ложно, то выбрасывает исключение `AssertionError`. 🛑🛑Да, да- `Error`🛑🛑

> [!NOTE] 🧠Идея:  
> 👉 «Я, как разработчик, **гарантирую**, что это условие _всегда_ истинно. Если нет — значит, в программе баг, и она должна упасть прямо здесь».

# 📕 1. Синтаксис
Есть две формы:
1. **Простая проверка**
```java
assert condition;
```
Если `condition == false`, то выбрасывается `AssertionError` без сообщения.

2. С сообщением
```java
assert condition : expression;
```
Тогда при падении в `AssertionError` попадёт результат `expression.toString()`.

## Пример
```java
int x = -5;
assert x >= 0 : "x должно быть неотрицательным, а сейчас " + x;
```

***
# 📕 2. Как включаются / отключаются `assert`

==По умолчанию `assert` в Java **выключен**.  ==

Чтобы включить, нужно явно запустить JVM с флагом:
- `-ea` или `-enableassertions` → включить проверки
    
- `-da` или `-disableassertions` → отключить проверки
    

Можно включать/выключать для отдельных пакетов/классов:
```bash
java -ea:com.myapp... -da:com.myapp.util.Main MyApp
```

> [!NOTE] 🛑🛑🛑То есть `assert` — это инструмент **только для разработки и тестирования**, а не для боевого кода.🛑🛑🛑
>

# Где использовать (best practices)

1. **Проверка инвариантов внутри кода**  
    Например, в алгоритме сортировки массив **не должен** содержать null:
```java
for (String s : array) {
	assert s != null : "Элемент массива не должен быть null"; 
}
```
    
2. **Документация намерений разработчика**  
    Ты явно фиксируешь, что считаешь правильным состоянием программы.
    
3. **При разработке библиотек**  
    Чтобы ловить неправильное использование API.
    
4. **В тестах и отладке**  
    Можно вставить `assert` прямо в код, и он будет сигнализировать о логических ошибках.

# Где **НЕ** использовать

❌ Для проверки пользовательского ввода (нужно кидать `IllegalArgumentException` или `ValidationException`).  

❌ Для логики, которая **обязана** работать==в продакшене== (assert можно отключить, и твоя проверка исчезнет).  

❌ Для бизнес-валидации (например, «сумма перевода > 0»).
