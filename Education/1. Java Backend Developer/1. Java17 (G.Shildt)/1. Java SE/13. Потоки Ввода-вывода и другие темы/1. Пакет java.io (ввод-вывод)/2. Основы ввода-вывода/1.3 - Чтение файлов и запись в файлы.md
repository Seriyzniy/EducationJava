# 1. FileInputStream и FileOutputStream
Двумя наиболее часто используемыми классами потоков являются `FileInputStream` и `FileOutputStream`, которые создают потоки байтовых данных, связанные с файлами. 

Для открытия файла нужно просто создать объект одного из этих классов, указывая имя файла в качестве аргумента конструктора.

```java
FileInputStream(String fileName) throws FileNotFoundException 

FileOutputStream(String fileName) throws FileNotFoundException
```

В аргументе `fileName` указывается имя файла, который необходимо от крыть. Если при создании входного потока файл не существует, тогда генерируется исключение FileNotFoundException

> [!NOTE] ⭐⭐⭐Интерфейсы`Closeable` и `AutoCloseable`⭐⭐⭐
> Важный момент — `FileInputStream` и `FileOutputStream` реализуют **`Closeable`** (а значит и `AutoCloseable`), поэтому их можно использовать в **try-with-resources**.
> 
> ==Метод== `close()`==определен в интерфейсе== `AutoCloseable` внутри пакета java. lang. 
> Интерфейс `AutoCloseable` унаследован интерфейсом `Closeable` в jаva.io.


> [!NOTE] 🛑🛑`FileInputStream` и `FileOutputStream` **не потокобезопасны**.🛑🛑
> Решение: либо синхронизировать, либо каждому потоку давать свой экземпляр.

## 📙 1.1 FileInputStream

Класс `FileInputStream` — это ==**байтовый поток ввода**==, предназначенный ==для чтения **файлов в двоичном виде**.==
- Используется для чтения изображений, видео, звуков, архивов, а также текстов в байтовом виде.
- Работает посимвольно или блоками (в массив байтов).

#### Конструкторы
```java
FileInputStream(String name) throws FileNotFoundException
FileInputStream(File file) throws FileNotFoundException
FileInputStream(FileDescriptor fdObj)
```
- `String name` — путь к файлу.
- `File file` — объект `File`, представляющий путь.
    
- `FileDescriptor fdObj` — низкоуровневый файловый дескриптор (редко используется напрямую).
    
👉 Если файл не найден → выбрасывается `FileNotFoundException`.

#### Основные методы
```java
//Читает один байт (0–255). Возвращает `-1`, если достигнут конец файла или сам байт
int read() throws IOException

//Читает данные В МАССИВ `b`, возвращает количество реально прочитанных байт или `-1` при конце файла.
int read(byte[] b) throws IOException

//Читает до `len` байт в массив `b`, НАЧИНАЯ С ПОЗИЦИИ `off`. Возвращает количество прочитанных байт или `-1`.
int read(byte[] b, int off, int len) throws IOException

//Пропускает `n` байт во входном потоке.
long skip(long n) throws IOException

//Возвращает количество доступных для чтения байт (без блокировки).
int available() throws IOException

//Закрывает поток и освобождает ресурсы.
void close() throws IOException
```

#### Пример
```java
import java.io.*;

class FileInputStreamDemo {
    public static void main(String[] args) {
        FileInputStream fin = null;
        try {
            fin = new FileInputStream("test.txt");
            int i;
            while ((i = fin.read()) != -1) {
                System.out.print((char) i);
            }
        } catch (IOException e) {
            System.out.println("Ошибка ввода-вывода: " + e);
        } finally {
            // Закрываем поток в блоке finally
            // 🛑if используется чтобы ЛИШНИЙ РАЗ не вызывать close()
            if (fin != null) {
                try {
                    fin.close();
                } catch (IOException e) {
                    System.out.println("Ошибка при закрытии файла: " + e);
                }
            }
        }
    }
}
```

## 📙 1.2 FileOutputStream
Класс `FileOutputStream` — это ==**байтовый поток вывода**==, предназначенный для записи данных в файлы.
- Используется ==для записи== любых типов данных (==обычно в бинарном виде==).
    
- ⭐Может дописывать в конец файла или перезаписывать.

#### Конструкторы
```java
FileOutputStream(String name) throws FileNotFoundException
FileOutputStream(String name, boolean append) throws FileNotFoundException
FileOutputStream(File file) throws FileNotFoundException
FileOutputStream(File file, boolean append) throws FileNotFoundException
FileOutputStream(FileDescriptor fdObj)
```
- `String name` / `File file` — путь к файлу.
- `boolean append` — если `true`, данные добавляются в конец файла, иначе файл перезаписывается.
    
- `FileDescriptor fdObj` — файловый дескриптор (низкоуровневый доступ).
    
👉 Если файла нет → он создаётся автоматически.

#### Основные методы
```java
//Записывает один байт (младшие 8 бит из `int`).
void write(int b) throws IOException

//Записывает массив байтов `b`.
void write(byte[] b) throws IOException

//Записывает часть массива `b`, начиная с `off` и длиной `len`.
void write(byte[] b, int off, int len) throws IOException

//Принудительно сбрасывает буфер (если есть) в файл.
void flush() throws IOException

//Закрывает поток и освобождает ресурсы.
void close() throws IOException
```

#### Пример
```java
import java.io.*;

class FileOutputStreamDemo {
    public static void main(String[] args) {
        FileOutputStream fout = null;
        
        try {
            fout = new FileOutputStream("output.txt");
            String text = "Hello, FileOutputStream!";
            fout.write(text.getBytes());
            System.out.println("Файл успешно записан.");
        } catch (IOException e) {
            System.out.println("Ошибка записи: " + e);
        } finally {
            // Всегда закрываем поток
            // 🛑if используется чтобы ЛИШНИЙ РАЗ не вызывать close()
            if (fout != null) {
                try {
                    fout.close();
                } catch (IOException e) {
                    System.out.println("Ошибка при закрытии файла: " + e);
                }
            }
        }
    }
}

```

# 📕⭐⭐2. Два подхода к закрытию ресурса⭐⭐

Существуют ==два основных подхода==, которые можно применять ==для закрытия файла по завершении== работы с ним. 
1. 🔶Явный вызов метода `close()`. 
Такой подход использовался во всех версиях Java перед выходом JDK 7 и потому встречается во всем унаследованном коде до JDK 7. 

2. 🔶Использование `try with resources`, который АВТОМАТИЧЕСКИ закрывает ресурс когда в нем отпадает необходимость
