
> [!NOTE] `RandomAccessFile`
> Позволяет ==**читать и записывать данные в произвольном порядке**== — можно перемещаться по файлу, изменять отдельные участки, не переписывая всё содержимое.  
> 
> Подходит для реализации файловых БД, индексов, журналов и т.д.

> [!NOTE] ⭐Реализует интерфейсы `DataInput` и `DataOutput`⭐
> 

## Конструкторы
```java
// Конструктор, принимающий путь к файлу в виде строки
// name — путь к файлу
// mode — ⭐режим открытия ("r", "rw", "rws", "rwd")⭐
// @throws FileNotFoundException — если файл не найден (для режима "r")
public RandomAccessFile(String name, String mode) throws FileNotFoundException

// Конструктор, принимающий объект File
// file — объект File, указывающий на файл
// mode — режим открытия ("r", "rw", "rws", "rwd")
// @throws FileNotFoundException — если файл не найден (для режима "r")
public RandomAccessFile(File file, String mode) throws FileNotFoundException
```

## Методы
```java
//✅ Управление позицией
// Перемещает "указатель файла" на позицию pos (в байтах)
// pos — новая позиция указателя
//
// @throws IOException — при ошибке ввода-вывода
public void seek(long pos) throws IOException

// Возвращает текущую позицию указателя (в байтах от начала файла)
// @return текущее смещение
// @throws IOException — при ошибке ввода-вывода
public long getFilePointer() throws IOException

// Возвращает длину файла в байтах
// @return размер файла
// @throws IOException — при ошибке ввода-вывода
public long length() throws IOException



//✅ ЧТЕНИЕ БАЙТОВ и массивов байт
// Считывает один байт и возвращает его как int (0–255), либо -1 при достижении конца файла
// @return считанное значение байта или -1
// @throws IOException — при ошибке ввода-вывода
public int read() throws IOException

// Считывает данные в массив байт
// b — буфер для хранения данных
//
// @return количество реально прочитанных байт, или -1, если достигнут конец файла
// @throws IOException — при ошибке ввода-вывода
public int read(byte[] b) throws IOException

// Считывает часть данных в массив байт
// b — буфер для хранения данных
// off — смещение в массиве, с которого начинать запись
// len — количество байт для чтения
//
// @return количество реально прочитанных байт
// @throws IOException — при ошибке ввода-вывода
public int read(byte[] b, int off, int len) throws IOException



//✅ ЗАПИСЬ БАЙТОВ и массивов байт
// Записывает один байт в текущую позицию файла
// b — байт для записи
//
// @throws IOException — при ошибке ввода-вывода
public void write(int b) throws IOException

// Записывает массив байт
// b — массив байт для записи
//
// @throws IOException — при ошибке ввода-вывода
public void write(byte[] b) throws IOException

// Записывает часть массива байт
// b — массив байт
// off — смещение, с которого начать запись
// len — количество байт для записи
//
// @throws IOException — при ошибке ввода-вывода
public void write(byte[] b, int off, int len) throws IOException



//✅ ЧТЕНИЕ ПРИМИТИВОВ
// Считывает boolean (1 байт)
// @return значение типа boolean
// @throws IOException — при ошибке ввода-вывода
public final boolean readBoolean() throws IOException

// Считывает byte
public final byte readByte() throws IOException

// Считывает unsigned byte (0–255) как int
public final int readUnsignedByte() throws IOException

// Считывает short (2 байта, Big Endian)
public final short readShort() throws IOException

// Считывает unsigned short (0–65535)
public final int readUnsignedShort() throws IOException

// Считывает char (2 байта, UTF-16)
public final char readChar() throws IOException

// Считывает int (4 байта)
public final int readInt() throws IOException

// Считывает long (8 байт)
public final long readLong() throws IOException

// Считывает float (4 байта, IEEE 754)
public final float readFloat() throws IOException

// Считывает double (8 байт, IEEE 754)
public final double readDouble() throws IOException

// Считывает строку в формате модифицированного UTF-8
public final String readUTF() throws IOException


//✅ ЗАПИСЬ ПРИМИТИВОВ
// Записывает boolean (1 байт)
public final void writeBoolean(boolean v) throws IOException

// Записывает byte
public final void writeByte(int v) throws IOException

// Записывает short (2 байта, Big Endian)
public final void writeShort(int v) throws IOException

// Записывает char (2 байта)
public final void writeChar(int v) throws IOException

// Записывает int (4 байта)
public final void writeInt(int v) throws IOException

// Записывает long (8 байт)
public final void writeLong(long v) throws IOException

// Записывает float (4 байта)
public final void writeFloat(float v) throws IOException

// Записывает double (8 байт)
public final void writeDouble(double v) throws IOException

// Записывает строку в формате модифицированного UTF-8
public final void writeUTF(String str) throws IOException


//✅ Прочее
// Закрывает файл и освобождает ресурсы
// @throws IOException — при ошибке закрытия файла
public void close() throws IOException
```

## 💡 Особенности и подводные камни 💡
1. 💡**Не буферизует ввод/вывод** → работает напрямую с файловой системой.  
    Для больших объемов данных производительность может быть ниже, чем у потоков с буферизацией.
    
2. **Работа с указателем**: чтение/запись всегда происходят из текущей позиции, поэтому нужно аккуратно использовать `seek()`.
    
3. 💡**Файл не обнуляется при открытии** — запись поверх старых данных может оставить «мусор», если не перезаписать полностью.
    
4. 💡**НЕ Потокобезопасный**: `RandomAccessFile` не является потокобезопасным.  
    При одновременном доступе из нескольких потоков нужно использовать `synchronized`.
    
5. 💡**Методы `readUTF` и `writeUTF`** используют **модифицированный UTF-8**, несовместимый с обычным UTF-8 в `StandardCharsets`.
    
6. **Расширение файла**: если вызвать `seek()` за пределами конца файла, промежуточная область будет заполнена нулями (`0x00`).

---
## ✅ Best Practices ✅

- Используйте **`try-with-resources`** для автоматического закрытия:
```java
try (RandomAccessFile raf = new RandomAccessFile("data.bin", "rw")) {
    // работа с файлом
}
```

- Для **частых операций** или **больших файлов** лучше применять `FileChannel` (из `java.nio`) — быстрее и безопаснее при многопоточности.
    
- После записи вызывайте `getFilePointer()` для контроля текущей позиции.
    
- Используйте `"rwd"` или `"rws"`, если требуется гарантированная запись данных на диск (например, для журналирования).

## Пример
```java
import java.io.*;

public class RandomAccessFileExample {
    public static void main(String[] args) {
        try (RandomAccessFile raf = new RandomAccessFile("example.dat", "rw")) {

            // Запишем данные разных типов
            raf.writeInt(12345);
            raf.writeDouble(3.14159);
            raf.writeUTF("Hello, world!");

            System.out.println("Позиция после записи: " + raf.getFilePointer());

            // Переместимся в начало и прочитаем
            raf.seek(0);
            int i = raf.readInt();
            double d = raf.readDouble();
            String s = raf.readUTF();

            System.out.println("Прочитано: " + i + ", " + d + ", " + s);

            // Изменим часть файла: перепишем int в начале
            raf.seek(0);
            raf.writeInt(99999);

            // Проверим обновлённое значение
            raf.seek(0);
            System.out.println("Новое значение: " + raf.readInt());

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

```
