📌 `ByteArrayInputStream` — это наследник `InputStream`, который ==позволяет читать байты ИЗ МАССИВА БАЙТОВ (`byte[]`) как из входного потока**==.  
То есть мы эмулируем источник данных, который уже есть в памяти, вместо чтения из файла, сокета и т.п.

## Конструкторы
```java
// ⭐⭐buf — массив байтов, из которого будут читаться данные.
// Начальная позиция: `pos = 0`.
// Количество доступных байтов = `buf.length`.
public ByteArrayInputStream(byte[] buf)


// `buf` — исходный массив.
// `offset` — начальная позиция чтения (индекс в массиве).
// `length` — сколько байтов из массива будет доступно для чтения.
public ByteArrayInputStream(byte[] buf, int offset, int length)
```
⚠️ **Важно**: `offset + length` ≤ `buf.length`, иначе `IndexOutOfBoundsException`.

## Методы
```java
// Читает один байт (возвращает `int` в диапазоне `0–255`).
// Если конец массива достигнут → возвращает `-1`.
public synchronized int read()

// Читает в массив b, возвращает количество реально прочитанных байтов
public int read(byte[] b)

// Читает до `len` байтов в массив `b`, начиная с позиции `off`.
// Возвращает фактическое количество прочитанных байтов, или `-1`, если достигнут конец потока.
public synchronized int read(byte[] b, int off, int len)

// Пропускает до `n` байтов.
// Возвращает фактическое количество пропущенных байтов.
public synchronized long skip(long n)

// Возвращает количество оставшихся байтов для чтения.
public synchronized int available()

// Устанавливает «закладку» в текущей позиции (`pos`).
// `readAheadLimit` для этого класса игнорируется.
public synchronized void mark(int readAheadLimit)

// Возвращает курсор (`pos`) обратно на позицию, сохранённую вызовом `mark()`.
// Если `mark()` не вызывался → вернёт в начало.
public synchronized void reset()

//Возвращает `true` если `mark()` и `reset()` поддерживаются.
public boolean markSupported()

// Здесь ничего не делает (в отличие от файловых потоков).
public void close()
```

## Пример
```java
import java.io.ByteArrayInputStream;
import java.io.IOException;

public class ByteArrayInputStreamDemo {
    public static void main(String[] args) {
        // Исходные данные - просто строка в байтах
        byte[] data = "Hello, ByteArrayInputStream!".getBytes();
        
        try (ByteArrayInputStream bais = new ByteArrayInputStream(data)) {
            // Читаем по одному байту (старая школа)
            System.out.println("Чтение по одному байту:");
            int singleByte;
            while ((singleByte = bais.read()) != -1) {
                System.out.print((char) singleByte);
            }
            System.out.println("\n");
            
            // Сбрасываем позицию и читаем блоками
            bais.reset();
            
            System.out.println("Чтение блоками:");
            byte[] buffer = new byte[10];
            int bytesRead;
            while ((bytesRead = bais.read(buffer)) != -1) {
                System.out.println("Прочитано " + bytesRead + " байт: " 
                    + new String(buffer, 0, bytesRead));
            }
            
        } catch (IOException e) {
            // На практике с ByteArrayInputStream исключения почти никогда не возникают
            e.printStackTrace();
        }
    }
}

Чтение по одному байту:
Hello, ByteArrayInputStream!

Чтение блоками:
Прочитано 10 байт: Hello, Byt
Прочитано 10 байт: eArrayInpu
Прочитано 8 байт: tStream!
```

## ✅Best Practices
1. **Использовать `ByteArrayInputStream` только для данных, уже находящихся в памяти.**  
    Если массив слишком большой → лучше работать с `FileInputStream` или `BufferedInputStream`.
    
2. **`close()` можно не вызывать**, но если используешь `try-with-resources` — это не повредит (повышает читаемость и унифицированный подход).
    
3. **`mark/reset` удобно использовать**, если нужно временно «заглянуть вперёд» и потом вернуться назад.
    
4. Для работы с текстом поверх байтов **лучше использовать `InputStreamReader` + `BufferedReader`**, а не напрямую `ByteArrayInputStream`. Это позволит учитывать кодировку.
