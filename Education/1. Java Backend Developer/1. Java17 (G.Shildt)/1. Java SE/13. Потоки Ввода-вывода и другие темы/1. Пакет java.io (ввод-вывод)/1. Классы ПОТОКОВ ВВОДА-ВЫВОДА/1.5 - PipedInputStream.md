`PipedInputStream` — это особый вид потока в Java, который ==используется **для организации канала связи между двумя потоками** (в смысле `Thread`)==, в связке с `PipedOutputStream`.

🔑 Основная идея: один поток пишет данные в `PipedOutputStream`, а другой поток может читать эти же данные через `PipedInputStream`. 

> [!NOTE] Обмен сообщениями между `Thread` "НА ЛЕТУ"
> Таким образом, можно организовать обмен сообщениями или потоковую передачу данных "на лету", без промежуточных файлов или буферов. 
> 💡(Данные НЕ ХРАНЯТСЯ во внешнем ресурсе (например, во временном файле на диске или отдельной очереди в ОС).  )

Они проходят **через внутренний кольцевой буфер в памяти** (по умолчанию 1024 байта).

***
## 💡Основные особенности
1. ✅ PipedInputStream всегда работает в паре с PipedOutputStream.
Либо через конструктор:
```java
PipedInputStream in = new PipedInputStream(out);
```
Либо методом connect:
```java
in.connect(out);
```

2. ✅ Размер внутреннего буфера по умолчанию — 1024 байта.
Можно указать свой размер через конструктор:
```java
PipedInputStream in = new PipedInputStream(4096);
```

3. ✅ Если буфер заполнен, то поток-запись (PipedOutputStream) блокируется, пока кто-то не прочитает данные.
Если буфер пуст, то поток-чтение (PipedInputStream) блокируется, пока кто-то не запишет данные.

4. ✅ Потоки должны использоваться разными потоками выполнения (Thread).
Использовать PipedInputStream и PipedOutputStream в одном и том же потоке нельзя — это приведет к блокировке.

5. ❌❌Не потокобезопасен сам по себе (но работает корректно при использовании из разных потоков).
***
## Конструкторы

- `PipedInputStream()` — создаёт входной поток без подключения (нужно подключить вручную).
    
- `PipedInputStream(int pipeSize)` — создаёт входной поток с буфером заданного размера.
    
- `PipedInputStream(PipedOutputStream src)` — сразу подключает к указанному выходному потоку.
    
- `PipedInputStream(PipedOutputStream src, int pipeSize)` — то же самое, но с буфером указанного размера.

## Основные методы
- `int read()` — читает один байт, блокируется если данных нет.
- `int read(byte[] b, int off, int len)` — читает в массив.
    
- `int available()` — показывает количество байт, доступных для чтения без блокировки.
    
- `void connect(PipedOutputStream src)` — подключает выходной поток.
    
- `void close()` — закрывает поток.

## Пример использования
```java
import java.io.*;

public class PipedExample {
    public static void main(String[] args) throws IOException {
        PipedOutputStream out = new PipedOutputStream();
        PipedInputStream in = new PipedInputStream(out);

        // Поток-писатель
        Thread writer = new Thread(() -> {
            try {
                String message = "Привет из PipedStream!";
                out.write(message.getBytes());
                out.close(); // закрываем после записи
            } catch (IOException e) {
                e.printStackTrace();
            }
        });

        // Поток-читатель
        Thread reader = new Thread(() -> {
            try {
                int data;
                while ((data = in.read()) != -1) {
                    System.out.print((char) data);
                }
                in.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
        });

        writer.start();
        reader.start();
    }
}

```
