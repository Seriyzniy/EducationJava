`ObjectOutputStream` — это поток вывода, который ==позволяет **сериализовать объекты Java**== и ==записывать их в поток в БИНАРНОМ формате==.

> [!NOTE] Работает в паре с `ObjectInputStream`
> Он работает поверх любого `OutputStream` (например, `FileOutputStream`) и сохраняет состояние объекта, включая его поля, чтобы потом их можно было восстановить через `ObjectInputStream`.

📌 Основная цель: **сохранение объектов в поток и передача по сети или в файл**.

⭐⭐⭐Особенности:⭐⭐⭐
- Записывает не только данные, но и метаданные классов.
    
- Поддерживает сериализацию как отдельных объектов, так и графа объектов.
    
- ⭐⭐⭐Объекты должны реализовывать интерфейс `Serializable`, иначе будет `NotSerializableException`.
    
- Можно использовать `transient` для исключения полей из сериализации.
    
- Поддерживает механизм контроля версий класса через `serialVersionUID`

***
## Конструкторы
```java
// Создаёт ObjectOutputStream, оборачивая другой OutputStream.
// 💡out — поток для записи объектов.
public ObjectOutputStream(OutputStream out) throws IOException
```

## Методы
```java
// Записывает объект в поток (сериализация).
// obj — объект, который реализует Serializable
public final void writeObject(Object obj) throws IOException

// Записывает один байт.
public void write(int b) throws IOException

// Записывает массив байтов.
public void write(byte[] b) throws IOException

// Записывает часть массива байтов.
public void write(byte[] b, int off, int len) throws IOException


// Записывает буферизированно массив байтов.
protected void drain() throws IOException


// Записывает заголовок потока (вызвается при создании потока).
protected void writeStreamHeader() throws IOException

// Записывает массив объектов
public void writeObjects(Object[] objs) throws IOException



// Закрывает поток и освобождает ресурсы.
public void close() throws IOException

// Принудительно сбрасывает буфер в поток.
public void flush() throws IOException

```

***
## 💡 Особенности 💡

- **Сериализация:** объект и все объекты, на которые он ссылается (если они тоже сериализуемы), будут записаны в поток.
    
- ⭐**serialVersionUID:** рекомендуется явно задавать в классе, чтобы избежать проблем при изменении структуры класса.
    
- При чтении объекта через `ObjectInputStream` необходимо, чтобы класс объекта был доступен в класспате.
    
- ⭐`ObjectOutputStream` создаёт специальный **заголовок потока** при первом создании, чтобы потом `ObjectInputStream` мог правильно его прочитать.

- Можно создавать собственные методы сериализации, переопределяя `writeObject` и `readObject` в классе объекта.

***
## Пример
```java
import java.io.*;

class Person implements Serializable {
    private static final long serialVersionUID = 1L;
    private String name;
    private transient int age; // transient поле не будет сериализовано

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    @Override
    public String toString() {
        return "Person{name='" + name + "', age=" + age + "}";
    }
}

public class ObjectOutputStreamExample {
    public static void main(String[] args) {
        Person person = new Person("Alex", 30);

        try (ObjectOutputStream oos = new ObjectOutputStream(
                new BufferedOutputStream(
	                new FileOutputStream("person.dat")
	            )
	        )
	    ){

            oos.writeObject(person); // сериализуем объект
            System.out.println("Объект записан в файл");

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

```

## Best-Practices

- ✅ Оборачивать в `BufferedOutputStream` для производительности.
    
- ✅ Явно задавать `serialVersionUID` в классах для контроля версий.
    
- ✅ Реализовывать интерфейс `Serializable` для всех сериализуемых классов.
    
- ✅ Использовать `transient` для полей, которые не должны сериализоваться.
    
- ✅ Переопределять методы `writeObject` / `readObject` при необходимости особой логики сериализации.
    
- ❌ Не использовать сериализацию для передачи объектов между ненадёжными источниками без проверки — возможна угроза безопасности.
    
- ✅ Закрывать поток через try-with-resources.

## Что такое ЗАГОЛОВОК ПОТОКА в `ObjectOutputStream`

Когда ты создаёшь `ObjectOutputStream`, он **пишет специальный «заголовок» в поток данных**.  
Этот ==заголовок содержит метаинформацию==, которая необходима `ObjectInputStream` при чтении, чтобы корректно восстановить объекты.

**По сути:** это служебная информация, которая:
- Помогает определить, что поток действительно содержит сериализованные объекты.
- Содержит версию формата сериализации.
    
- Служит для обеспечения совместимости между `ObjectOutputStream` и `ObjectInputStream`.
    

> [!NOTE] 📌 Этот заголовок `пишется один раз при создании потока`, автоматически в конструкторе `ObjectOutputStream`.
> 

### Формат заголовка

==Заголовок потока== — это ==**фиксированная последовательность байтов**==, которая включает:
1. **Магическое число** — чтобы распознать поток как объектный сериализованный поток.
    
2. **Версию формата** сериализации.
    

Пример:
- Магическое число: `0xAC ED`
- Версия: `0x00 05`
    
То есть, первые байты файла, записанного `ObjectOutputStream`, выглядят примерно так:
```css
AC ED 00 05
```

### Где используется заголовок
- **При чтении**: `ObjectInputStream` первым делом читает заголовок потока, чтобы проверить, что он получает корректный поток сериализованных объектов.
    
- Если заголовок отсутствует или повреждён → `StreamCorruptedException`.
```java
ObjectInputStream ois = new ObjectInputStream(new FileInputStream("somefile.txt"));
```
