`PipedOutputStream` — это **байтовый выходной поток**, который работает в паре с `PipedInputStream`.  

Он используется для передачи данных между потоками **в памяти**, создавая механизм типа «трубы» (pipe).  

> [!NOTE] Механизм трубы 
> То есть: один поток пишет в `PipedOutputStream`, другой поток читает эти данные из связанного `PipedInputStream`.

📌 Это способ организации **межпоточного взаимодействия** без использования файлов или сетевых соединений.  

Чаще используется для:
- 💡 Передачи данных между потоками в реальном времени.
    
- 💡Симуляции ввода-вывода между разными частями программы.

***
## Конструкторы
```java
// Создаёт PipedOutputStream, который позже будет соединён с PipedInputStream.
public PipedOutputStream()

// Создаёт PipedOutputStream, который СРАЗУ СОЕДИНЯЕТСЯ с заданным PipedInputStream.
public PipedOutputStream(PipedInputStream snk) throws IOException
```

## Методы
```java
// Записывает один байт.
public void write(int b) throws IOException

// Записывает массив байтов целиком.
public void write(byte[] b, int off, int len) throws IOException

// Закрывает поток.
public void close() throws IOException

// Принудительно сбрасывает поток.
public void flush() throws IOException
```

***
## 💡 Особенности 💡

- 💡**Работает только в паре с `PipedInputStream`** — иначе запись вызовет `IOException`.
    
- Буфер по умолчанию **размером 1024 байта** (в Java).
    
- 💡Если буфер переполнен → запись блокируется до тех пор, пока читающий поток не освободит место.
    
- 🛑💡Если читающий поток не подключён или закрыт → будет `IOException`.
    
- ✅Используется для **синхронизации потоков** без внешних файлов или сетевых операций.
    
- При работе с `PipedOutputStream` и `PipedInputStream` нужно быть осторожным: неверная организация потоков может привести к **deadlock** (взаимной блокировке).

***
## Пример
```java
import java.io.*;

public class PipedStreamExample {
    public static void main(String[] args) throws IOException {
        PipedOutputStream pos = new PipedOutputStream();
        PipedInputStream pis = new PipedInputStream(pos);

        // Поток записи
        Thread writer = new Thread(() -> {
            try {
                String message = "Hello from PipedOutputStream!";
                pos.write(message.getBytes());
                pos.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
        });

        // Поток чтения
        Thread reader = new Thread(() -> {
            try {
                int data;
                while ((data = pis.read()) != -1) {
                    System.out.print((char) data);
                }
                pis.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
        });

        writer.start();
        reader.start();
    }
}
```

## Best-Practices

- ✅ Использовать `PipedOutputStream` **только** при обмене данными между потоками в рамках одного JVM.
    
- ✅ ==Всегда соединять== с `PipedInputStream` перед записью.
    
- ✅ Следить за тем, чтобы читающий поток был активен, иначе запись заблокируется.
    
- ✅ Не использовать в однопоточном режиме (нет смысла — проще использовать буфер).
    
- ✅ Оборачивать в `BufferedOutputStream`, если требуется высокая производительность.
```java
import java.io.*;

public class BufferedPipedExample {
    public static void main(String[] args) throws IOException {
        // Создаём PipedOutputStream
        PipedOutputStream pos = new PipedOutputStream();

        // Буферизуем его для повышения производительности
        BufferedOutputStream bos = new BufferedOutputStream(pos, 1024);

        // Соединяем с PipedInputStream
        PipedInputStream pis = new PipedInputStream(pos);

        // Поток записи
        Thread writer = new Thread(() -> {
            try {
                String message = "Hello from Buffered PipedOutputStream!";
                bos.write(message.getBytes()); // записываем в буфер
                bos.flush(); // обязательно сбрасываем буфер, иначе данные не уйдут в поток
                bos.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
        });

        // Поток чтения
        Thread reader = new Thread(() -> {
            try {
                int data;
                while ((data = pis.read()) != -1) {
                    System.out.print((char) data);
                }
                pis.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
        });

        writer.start();
        reader.start();
    }
}

```

- ⚠️ Следить за размером буфера — переполнение приведёт к блокировке.
    
- ⚠️ Избегать deadlock — чётко организовывать порядок запуска потоков записи и чтения.
