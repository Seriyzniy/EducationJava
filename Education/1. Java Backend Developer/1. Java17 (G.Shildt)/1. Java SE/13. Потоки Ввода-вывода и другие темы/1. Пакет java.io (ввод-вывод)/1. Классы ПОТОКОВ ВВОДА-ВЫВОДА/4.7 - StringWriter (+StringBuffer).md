`StringWriter` — это наследник `Writer`, который ==пишет данные **НЕ В ФАЙЛ или поток**, А ВО **ВНУТРЕННИЙ БУФЕР== 👉`StringBuffer`** (👈⭐потокобезопасный⭐, в отличие от `StringBuilder`).

Используется:
- 💡Используется, когда нужно «накопить» текст в памяти, а потом получить его как одну строку.
    
- Часто применяется для тестирования, отладки, промежуточных преобразований, например, когда API требует `Writer`, но нужно работать со строками.
    
- 💡 В отличие от `CharArrayWriter`, внутри хранит `StringBuffer`, а не `char[]`.

***
## Конструкторы
```java
public StringWriter()
// Создает writer с буфером по умолчанию (емкость = 16 символов)

public StringWriter(int initialSize)
// Создает writer с буфером указанной начальной емкости
// initialSize – количество символов, под которое сразу выделяется память
```

## Методы
```java
public void write(int c)
// Записывает один символ

public void write(char[] cbuf)
// Записывает массив символов

public void write(char[] cbuf, int off, int len)
// Записывает часть массива символов

public void write(String str)
// Записывает строку целиком

public void write(String str, int off, int len)
// Записывает часть строки


public StringWriter append(CharSequence csq)
// ⭐ Добавляет CharSequence (например, StringBuilder, String, StringBuffer)
// Если csq == null → добавляет "null"

public StringWriter append(CharSequence csq, int start, int end)
// Добавляет часть CharSequence (с позиции start до end)

public StringWriter append(char c)
// Добавляет один символ



public StringBuffer getBuffer()
// Возвращает сам StringBuffer (можно работать напрямую, но обычно не рекомендуется для инкапсуляции)

public String toString()
// Возвращает накопленное содержимое как String

public void flush()
// Ничего не делает (метод пустой, так как буфер в памяти)

public void close()
// Тоже ничего не делает (ресурсы не выделяются)
```

***
## Особенности

- ⭐⭐⭐Писать можно сколько угодно: ==буфер== `StringBuffer` автоматически ==расширяется==.⭐⭐⭐
    
- ⭐⭐⭐`close()` и `flush()` — заглушки, их вызов не нужен.
    
- Используется в случаях, когда API требует `Writer`, а нужно писать в строку.
    
- В отличие от `CharArrayWriter`, хранит данные в `StringBuffer`, а значит:
    
    - ⭐⭐⭐`StringBuffer` потокобезопасный (синхронизированный).
        
    - ⭐⭐⭐Для МНОГОПОТОЧНЫХ сценариев `StringWriter` предпочтительнее.


***
## Пример
```JAVA
import java.io.*;

public class Demo {
    public static void main(String[] args) {
        StringWriter sw = new StringWriter();

        sw.write("Hello, ");
        sw.append("world");
        sw.write("!");
        
        System.out.println("Buffer content: " + sw.toString());
        
        // ⭐⭐Можно получить доступ напрямую⭐⭐
        StringBuffer buf = sw.getBuffer();
        buf.append(" Added via StringBuffer.");

        System.out.println("Final: " + sw.toString());
    }
}
```

***
## Best-Practices
- Используй `StringWriter`, если:
    - Нужно имитировать работу с `Writer`, но писать в строку (например, при тестах).
        
    - Нужно накапливать текст в памяти с возможностью получить `String`.
        
- Если работаешь в **однопоточной среде** и важна производительность — можно использовать `CharArrayWriter` (он быстрее).
    
- Не вызывай `close()` и `flush()`: это лишнее.
    
- ⭐Для **потокобезопасных операций** лучше оставить `StringWriter`, так как он использует `StringBuffer`.
