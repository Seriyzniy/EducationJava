`DataOutputStream` — это ==**обёртка над байтовым потоком**== (`OutputStream`), которая ==позволяет записывать не только «сырые» байты, но и **ПРИМИТИВНЫЕ типы Java**== (`int`, `long`, `double`, `char`, `boolean`) в 💡**бинарном виде**💡.

Используется:
- 💡Используется вместе с `DataInputStream`, чтобы сохранить данные так, чтобы их можно было потом корректно считать обратно.
    
- Не заботится о кодировке строк (в отличие от `PrintWriter`). Для строк есть методы `writeUTF` / `readUTF`.
    
-==Главная цель: сериализация примитивов **в точном формате**==, независимо от платформы.

***
## Конструкторы
```java
// Создаёт DataOutputStream, оборачивая другой OutputStream.
// 💡out — базовый поток, в который будет идти запись.
public DataOutputStream(OutputStream out)
```

## Методы
```java
// Записывает 1 байт (младшие 8 бит от int b).
public synchronized void write(int b) throws IOException

// Записывает массив байтов целиком.
public synchronized void write(byte[] b, int off, int len) throws IOException


// // ✅ ЗАПИСЬ ПРИМИТИВЫ КАК БАЙТЫ ✅
// Записывает boolean как 1 байт (1 = true, 0 = false).
public final void writeBoolean(boolean v) throws IOException

// Записывает byte (просто 1 байт).
public final void writeByte(int v) throws IOException

// Записывает short (2 байта, Big-Endian).
public final void writeShort(int v) throws IOException

// Записывает char (2 байта, Big-Endian, UTF-16 код юникода).
public final void writeChar(int v) throws IOException

// Записывает int (4 байта, Big-Endian).
public final void writeInt(int v) throws IOException

// Записывает long (8 байт, Big-Endian).
public final void writeLong(long v) throws IOException

// Записывает float (4 байта, IEEE 754).
public final void writeFloat(float v) throws IOException

// Записывает double (8 байт, IEEE 754).
public final void writeDouble(double v) throws IOException



// Записывает строку в формате модифицированного UTF-8.
// Сначала 2 байта длины, затем байты символов.
public final void writeUTF(String str) throws IOException

// Возвращает, сколько байтов уже записано через этот поток.
public final int size()
```

***
## 💡 Особенности
- **Формат фиксированный**: всегда Big-Endian (старший байт первым). Это гарантирует кроссплатформенность.
    
- `writeUTF` использует **модифицированный UTF-8**, который отличается от стандартного UTF-8 (например, null-символ `\u0000` кодируется как `0xC0 0x80`).
    
- 💡Не буферизует данные. Обычно оборачивается в `BufferedOutputStream` для производительности.

***
## Пример
```java
import java.io.*;

public class DataOutputStreamExample {
    public static void main(String[] args) {
        try (DataOutputStream dos = new DataOutputStream(
                new BufferedOutputStream(
	                new FileOutputStream("data.bin")
	            )
	        )
	    ) {
            
            dos.writeInt(42);           // 4 байта
            dos.writeDouble(3.14);      // 8 байт
            dos.writeBoolean(true);     // 1 байт
            dos.writeUTF("Привет");     // длина (2 байта) + строка в UTF
            
            System.out.println("Данные записаны в data.bin");
            
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

## ✅ Best-Practices

- ✅ **Комбинировать с `DataInputStream`** — иначе будет сложно правильно распарсить данные.
    
- ✅ **Оборачивать в `BufferedOutputStream`** для повышения производительности (иначе будет много системных вызовов `write`).
    
- ⚠️ **Не путать с PrintStream**: `DataOutputStream` ==пишет **бинарно**==, а не человекочитаемо.
    
- ⚠️ Следить за порядком чтения/записи: если записал `int + UTF + boolean`, то читать нужно точно в том же порядке.
    
- ⚠️ Для строк помнить про **модифицированный UTF-8** — он несовместим с обычными UTF-парсерами.
