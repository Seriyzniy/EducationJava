`StringReader` — это реализация `Reader`, которая позволяет ==читать символы **из строки== (`String`) как из потока**.  

> [!NOTE] 💡Полезен, когда нужно протестировать логику работы с потоками ввода, но без создания реальных файлов или сетевых соединений.
>

## Конструкторы
```java
StringReader(String s)
// Создает поток чтения, который читает символы из строки s.
// s — исходная строка, из которой будут поступать данные.
```

## Методы
```java
int read()
// Читает один символ из строки. Возвращает -1, если строка закончилась.

int read(char[] cbuf, int off, int len)
// Читает символы в массив cbuf, начиная с позиции off, максимум len символов.
// Возвращает количество реально прочитанных символов или -1, если строка закончилась.

boolean ready()
// Возвращает true, если есть ещё непрочитанные символы.

long skip(long n)
// Пропускает n символов и возвращает количество реально пропущенных.

void close()
// Закрывает поток. Фактически ничего не делает, но нужен для совместимости с Reader API.

void mark(int readAheadLimit)
// Помечает текущую позицию в строке. readAheadLimit игнорируется (не имеет смысла).

void reset()
// Возвращает чтение к ранее установленной позиции mark().
```

***
## Особенности

- ✅Работает **только с текстовыми данными** (символами).
    
- ✅Не требует внешних ресурсов (в отличие от `FileReader` или `InputStreamReader`).
    
- `close()` формально ничего не освобождает, но по контракту `Reader` должен быть вызван.
    
- Поддерживает `mark()` и `reset()` для возврата к ранее отмеченной позиции.

***

## Пример
```java
import java.io.StringReader;

public class StringReaderExample {
    public static void main(String[] args) throws Exception {
        String text = "Hello, world!";
        StringReader reader = new StringReader(text);

        int ch;
        while ((ch = reader.read()) != -1) {
            System.out.print((char) ch);
        }

        reader.close();
    }
}
```

***
## Best-Practices

- ✅ Использовать для **тестов и отладки**, когда нужно эмулировать поток ввода без файлов/сети.
    
- ✅ Хорошо подходит для **парсеров**, где данные уже есть в виде строки.
    
- ⚠️ Помнить, что `close()` ничего не делает, но вызывать его всё равно стоит ради чистоты кода.
    
- ⚠️ Если данные большие, лучше использовать `StringReader` с буферизацией (`BufferedReader`) для ускорения работы.
