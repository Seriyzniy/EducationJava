`BufferedInputStream` —  один из самых полезных и часто используемых наследников `FilterInputStream`.

- Наследуется от FilterInputStream.
- ==Хранит внутренний буфе==р (массив байтов), ==чтобы уменьшить количество обращений к реальному источнику== (файлу, сокету и т.д.).
- Это значительно ускоряет операции ввода.

## Конструкторы
```java
// Создаёт поток с буферизацией, 💡буфер по умолчанию = 8192 байта.💡
public BufferedInputStream(InputStream in) {
	this(in, 8192);
}

// Создаёт поток с буферизацией, размер буфера задаётся явно.
// size > 0, иначе IllegalArgumentException.
public BufferedInputStream(InputStream in, int size)
```

## Методы
```java
// Читает **1 байт** (возвращает `0–255`).
// Возвращает `-1`, если достигнут конец файла (EOF).
// 
//🟢Работает через внутренний буфер: сначала пытается взять из него,
// ✅ если буфер пуст — подгружает данные из обёрнутого потока.✅
public int read() throws IOException { /* ... */ }

// Читает данные в массив `b`.
// Возвращает количество реально прочитанных байтов, либо `-1` при EOF.
// 
// 🟢Сначала копирует байты из буфера, если он пуст — читает из источника.
public int read(byte[] b) throws IOException { /* ... */ }

// Читает в массив `b`, начиная с позиции `off`, до `len` байтов.
// Возвращает фактическое количество прочитанного или `-1`.
// Позволяет эффективно читать кусками.
public int read(byte[] b, int off, int len) throws IOException { /* ... */ }

// Пропускает до `n` байтов.
// 
// 🟢 Если байты есть в буфере — смещает позицию.
// Если нет — вызывает skip() у обёрнутого потока.
// Возвращает фактическое количество пропущенных байтов.
public long skip(long n) throws IOException { /* ... */ }

// Возвращает число доступных байтов для чтения без блокировки.
// 🟢 Суммирует (count - pos) из буфера и available() у обёрнутого потока.
// ❌ Не использовать для определения размера файла.
public int available() throws IOException { /* ... */ }

// Закрывает поток и освобождает ресурсы.
// Автоматически вызывает close() у обёрнутого потока.
public void close() throws IOException { /* ... */ }


// === MARK/RESET ===

// Ставит "маркер" на текущую позицию в потоке.
// readlimit — сколько байтов можно безопасно прочитать, сохраняя возможность reset().
public synchronized void mark(int readlimit) { /* ... */ }

// Возвращается к позиции, где ранее был установлен mark().
// Если mark не был установлен или превысили readlimit → IOException.
public synchronized void reset() throws IOException { /* ... */ }

// Возвращает true → класс поддерживает mark/reset.
public boolean markSupported() { /* ... */ }
```

## Пример
```java
import java.io.*;

public class BufferedDemo {
    public static void main(String[] args) {
        try (BufferedInputStream bis = new BufferedInputStream(
			        new FileInputStream("test.txt"), 4096)) 
		{

            byte[] buffer = new byte[128];
            int bytesRead;
            while ((bytesRead = bis.read(buffer)) != -1) {
                System.out.print(new String(buffer, 0, bytesRead));
            }

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```
📌 Здесь `BufferedInputStream` подгружает кусками по 4 KB, а мы читаем по 128 байт. Это **гораздо быстрее**, чем читать по 1 байту.

## ✅ Best Practices✅

- ✅ Всегда оборачивай `FileInputStream` в `BufferedInputStream`, если читаешь много байтов.
    
- ✅ Используй размер буфера в зависимости от ситуации (по умолчанию 8192 байт → обычно норм).
    
- ✅ Для текста лучше использовать `BufferedReader` (`Reader` API), чтобы сразу работать с символами и строками.
    
- ✅ Если используешь `mark()` → всегда проверяй `markSupported()`.
    
- ❌ Не делай `bis.read()` в цикле без буфера — теряешь производительность.
