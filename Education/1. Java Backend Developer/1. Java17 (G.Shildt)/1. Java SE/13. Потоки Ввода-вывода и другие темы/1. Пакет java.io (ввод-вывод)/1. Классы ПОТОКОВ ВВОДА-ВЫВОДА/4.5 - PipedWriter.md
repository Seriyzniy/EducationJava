`PipedWriter` — это специализированный `Writer`, который ==используется для **организации канала связи между потоками** ЧЕРЕЗ СИМВОЛЫ.  ==

Работает в паре с `PipedReader`: один поток пишет данные в `PipedWriter`, а другой читает их через `PipedReader`.

Таким образом, это способ сделать **межпоточный обмен текстовыми данными** без использования файлов или сетевых сокетов.  
Внутри реализован буфер (размер обычно 1024 символа), который хранит промежуточные данные.

***
## Конструкторы
```JAVA
public PipedWriter()
// Создает "отвязанный" писатель, который можно потом соединить с PipedReader.connect()

public PipedWriter(PipedReader snk) throws IOException
// 💡Создает PipedWriter и сразу соединяет его с указанным PipedReader
// snk – приёмник (читатель), который будет связан с этим писателем
```

## Методы
```java
public void connect(PipedReader snk) throws IOException
// ✅ Соединяет PipedWriter с указанным PipedReader
// snk – приёмник для чтения данных
// Если уже соединён – будет IOException

public void write(int c) throws IOException
// Записывает один символ в поток

public void write(char[] cbuf) throws IOException
// Записывает массив символов
// cbuf – массив символов

public void write(char[] cbuf, int off, int len) throws IOException
// Записывает часть массива символов
// cbuf – массив символов
// off – смещение
// len – количество символов

public void flush() throws IOException
// Обычно ничего не делает (буферизация реализована внутри канала)

public void close() throws IOException
// Закрывает поток, сигнализируя PipedReader об окончании данных

```

***
## 💡Особенности

- 💡**Работает только в связке с `PipedReader`**.
    
- 💡Используется для **передачи текста между потоками** в пределах одного процесса.
    
- Размер внутреннего буфера ограничен (~1024 символа). Если буфер заполнен, `write()` блокируется до тех пор, пока другой поток не прочитает данные.
    
- `read()` в `PipedReader` блокируется, если данных нет.
    
- 💡 Подходит для простого обмена сообщениями или построения конвейеров обработки данных.

***
## Пример
```java
import java.io.*;

public class Demo {
    public static void main(String[] args) throws IOException {
        PipedReader reader = new PipedReader();
        PipedWriter writer = new PipedWriter(reader);

        Thread producer = new Thread(() -> {
            try {
                writer.write("Hello from writer!\n");
                writer.write("Another line\n");
                writer.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
        });

        Thread consumer = new Thread(() -> {
            try {
                int ch;
                while ((ch = reader.read()) != -1) {
                    System.out.print((char) ch);
                }
            } catch (IOException e) {
                e.printStackTrace();
            }
        });

        producer.start();
        consumer.start();
    }
}
```

***
## Best-Practices
- **Обязательно** связывай `PipedWriter` и `PipedReader` до начала записи/чтения.
    
- Используй в разных потоках: писать и читать в одном потоке нельзя — возникнет deadlock.
    
- Помни про блокировки: `write()` ждёт, пока читатель освободит буфер, а `read()` ждёт новых данных.
    
- Закрывай `PipedWriter`, чтобы `PipedReader` корректно завершил чтение (иначе `read()` может висеть в ожидании).
    
> [!NOTE] Для сложной многопоточной коммуникации ==лучше использовать== ==очереди== из `java.util.concurrent` (например, `LinkedBlockingQueue`).
