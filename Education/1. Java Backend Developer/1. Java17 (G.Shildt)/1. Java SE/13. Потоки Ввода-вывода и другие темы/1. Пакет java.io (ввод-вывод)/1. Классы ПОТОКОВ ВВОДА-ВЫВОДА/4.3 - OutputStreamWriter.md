`OutputStreamWriter` — это наследник `Writer`, который ==пишет **символы** (char) и преобразует их в **байты**== согласно заданной кодировке.  

> [!NOTE] Это обратная пара к `InputStreamReader`. 
> `InputStreamReader` - из байтов в символ
> ⭐`OutputStreamWriter` - из символов в байт

Особенности:
- Работает поверх любого `OutputStream` (например, `FileOutputStream`, `Socket.getOutputStream()`).
    
- Обеспечивает корректное преобразование текста в байты по кодировке (`UTF-8`, `Cp1251`, `ISO-8859-1` и т. д.).
    
- Если кодировка не указана явно, используется **системная по умолчанию** (`Charset.defaultCharset()`) — и это может быть источником багов при переносе кода.

***
## Конструкторы
```java
OutputStreamWriter(OutputStream out)
// out — целевой байтовый поток.
// Используется кодировка по умолчанию (небезопасно!).

OutputStreamWriter(OutputStream out, String charsetName) throws UnsupportedEncodingException
// out — целевой поток.
// charsetName — имя кодировки (например, "UTF-8").

OutputStreamWriter(OutputStream out, Charset cs)
// out — целевой поток.
// ❓⭐cs — объект Charset (java.nio.charset).

OutputStreamWriter(OutputStream out, CharsetEncoder enc)
// out — целевой поток.
// ❓⭐❓enc — CharsetEncoder, даёт тонкий контроль над процессом кодирования.
```

## Методы
```java
void write(int c) throws IOException
// Записывает один символ (char).
// Преобразует его в байты согласно кодировке и пишет в поток.

void write(char[] cbuf, int off, int len) throws IOException
// Записывает часть массива символов (от off до len).

void write(String str) throws IOException
// Записывает строку.

void write(String str, int off, int len) throws IOException
// Записывает подстроку строки.


void flush() throws IOException
// Сбрасывает внутренний буфер кодировщика и данные в целевой поток.

void close() throws IOException
// Закрывает поток, предварительно вызвав flush().


String getEncoding()
// Возвращает имя используемой кодировки.
// Может быть null, если кодировка по умолчанию инициализировалась динамически.
```

***
## Особенности

- ✅ Позволяет писать **символы в любой `OutputStream`** (например, в файл через `FileOutputStream`).
    
- ✅ Позволяет явно задать кодировку → критично для мультиплатформенных приложений.
    
- ⚠️ При `close()` автоматически вызывается `flush()`.
    
- ⚠️ Если не вызвать `flush()`, часть данных может остаться в буфере.
    
- ❌ Если не указать кодировку, результат зависит от ОС и настроек JVM → потенциальные баги.

***
## Пример
```java
import java.io.FileOutputStream;
import java.io.OutputStreamWriter;
import java.io.IOException;
import java.nio.charset.StandardCharsets;

public class OutputStreamWriterExample {
    public static void main(String[] args) {
        try (OutputStreamWriter osw = new OutputStreamWriter(
		        new FileOutputStream("output.txt"),
		        StandardCharsets.UTF_8)
		) {
            osw.write("Привет, мир!");
            osw.write("\nHello, world!");
            // при close() произойдет flush()
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

***
## Best-Practices

- ✅ Всегда указывать кодировку явно (`UTF-8` — стандарт де-факто).
    
- ✅ Для удобства оборачивать `OutputStreamWriter` в `BufferedWriter`, если идёт много мелких записей:
```java
new BufferedWriter(
	new OutputStreamWriter(out, StandardCharsets.UTF_8)
)
```

- ✅ ⭐Использовать в связке с `PrintWriter`, если нужен удобный `println`.
    
- ❌ Не полагаться на `Charset.defaultCharset()`, если важно межплатформенное поведение.
    
- ❌ Не забывать `flush()`, если поток живёт долго (например, при записи в сокет).
