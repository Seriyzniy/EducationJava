`CharArrayReader` — это **символьный поток ввода**, который читает символы из **массивa `char[]` в памяти**.  

> [!NOTE] Фактически, это `аналог` `ByteArrayInputStream`, `но для символов`.
> 

Особенности:
- Позволяет читать символы, как будто они поступают из файла или сети, но фактически берёт их из массива.
    
- Полезен для тестирования или парсинга строк, которые уже загружены в память.

***
## Конструкторы
```java
// Создаёт CharArrayReader для всего массива символов.
public CharArrayReader(char[] buf)

// Создаёт CharArrayReader для ПОДМАССИВА символов.
// buf — массив символов
// offset — начальная позиция в массиве
// length — количество символов для чтения
public CharArrayReader(char[] buf, int offset, int length)
```

## Методы
```java
// Читает один символ и возвращает его как int (0–65535), или -1 при EOF.
public int read() throws IOException

// Читает массив символов.
// cbuf — массив для записи символов
// off — смещение в массиве cbuf
// len — количество символов для чтения
public int read(char[] cbuf, int off, int len) throws IOException

// Пропускает n символов.
public long skip(long n) throws IOException

// Проверяет, готов ли поток к чтению (не блокирует)
public boolean ready() throws IOException

// Закрывает поток
public void close()

// Устанавливает маркер текущей позиции чтения
public void mark(int readAheadLimit) throws IOException

// Проверяет поддержку mark/reset
public boolean markSupported()

// Возвращает в маркер
public void reset() throws IOException
```

***
## 💡 Особенности 💡

- 💡 Работает **только с уже загруженными в память данными**.
    
- 💡Не выполняет никакой внешней загрузки — чтение максимально быстрое.
    
- `mark()` и `reset()` работают идеально, потому что данные находятся в памяти.
    
- 💡Размер «потока» фиксирован при создании.
    
- Очень полезен для тестирования кода, который ожидает `Reader`.

***
## Пример

```java
import java.io.*;

public class CharArrayReaderExample {
    public static void main(String[] args) {
        char[] data = "Hello, CharArrayReader!".toCharArray();

        try (CharArrayReader car = new CharArrayReader(data)) {
            int ch;
            while ((ch = car.read()) != -1) {
                System.out.print((char) ch);
            }
        }
    }
}
```

***
## ✅ Best-Practices

- ✅ Использовать `CharArrayReader` для тестов и парсинга строк в памяти.
    
- ✅ Использовать `mark()`/`reset()` при необходимости повторного чтения.
    
- ✅ Закрывать поток через try-with-resources.
    
- ⚠️ Не использовать для больших массивов символов — это потребует много памяти.
    
- ⚠️ Не применять, если нужно читать данные из внешнего источника — для этого лучше использовать `BufferedReader` или `FileReader`.
