`PrintWriter` — это подкласс `Writer`, предназначенный ==для удобного вывода **текстовых данных**== (символов, строк, чисел, булевых значений, объектов) в человеко-читаемом виде.  

> [!NOTE] ⭐Фактически это аналог `PrintStream`, но для символов.⭐
> 

Особенности:
- 💡Работает с символьными потоками (а не байтовыми, как `PrintStream`).
    
- Имеет методы `print(...)` и `println(...)` для любых типов данных.
    
- 💡Может автоматически вызывать `flush()` после каждой записи (если включен `autoFlush`).
    
- 💡==Не выбрасывает== `IOException` ==напрямую== в большинстве методов — вместо этого ==нужно проверять метод== `checkError()`.

***
## Конструкторы
```java
public PrintWriter(Writer out)
// out – базовый Writer, в который будет происходить запись

public PrintWriter(Writer out, boolean autoFlush)
// out – базовый Writer
// autoFlush – если true, то после каждого вызова println(), printf(), format() будет вызываться flush()

public PrintWriter(OutputStream out)
// out – байтовый поток, будет обернут в OutputStreamWriter с кодировкой по умолчанию

public PrintWriter(OutputStream out, boolean autoFlush)
// out – байтовый поток
// autoFlush – автоочистка буфера при println/printf/format


// ✅ ЗАПИСЬ В ФАЙЛ ✅
public PrintWriter(String fileName) throws FileNotFoundException
// fileName – имя файла для записи (с использованием кодировки по умолчанию)

public PrintWriter(String fileName, String csn) throws FileNotFoundException, UnsupportedEncodingException
// fileName – имя файла
// csn – кодировка

public PrintWriter(File file) throws FileNotFoundException
// file – файл для записи (кодировка по умолчанию)

public PrintWriter(File file, String csn) throws FileNotFoundException, UnsupportedEncodingException
// file – файл
// csn – кодировка
```

***
## 💡 Особенности 💡

- ==Подходит для текстового вывода, особенно когда нужно писать строки в файл или в консоль.==
    
- 💡Если требуется контроль над кодировкой — лучше использовать конструкторы с `OutputStream` или `File` + указание charset.
    
- Не выбрасывает `IOException` из методов `print/println` → ошибки нужно проверять через `checkError()`.
    
- Может использоваться с `autoFlush`, что удобно для логирования.

***
## Пример
```java
import java.io.*;

public class Demo {
    public static void main(String[] args) {
        try (PrintWriter pw = new PrintWriter(
		        new FileWriter("output.txt"), 
		        true)
	    ) {
            pw.println("Hello, world!");
            pw.print(123);
            pw.println(" <- это число");
            pw.printf("Форматированный вывод: %.2f%n", 3.14159);
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

***
## Best-Practices

- ⭐⭐⭐Для **текстовых файлов** чаще выбирай `PrintWriter`, чем `PrintStream`.⭐⭐⭐
    
- Всегда учитывай **кодировку** (используй конструкторы с `OutputStreamWriter` или `PrintWriter(File, Charset)`).
    
- ⭐⭐⭐Включай `autoFlush`, если важно, чтобы данные сразу попадали в файл/консоль (например, при логировании).
    
- Для бинарных данных используй `DataOutputStream` или `ObjectOutputStream`, а не `PrintWriter`.
    
- Если требуется обработка ошибок — не забывай проверять `checkError()`
