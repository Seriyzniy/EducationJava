`FilterOutputStream` — это **базовый класс-декоратор для других потоков вывода**, наследник `OutputStream`.  

Он принимает в конструкторе **другой `OutputStream`** и ==просто перенаправляет== (`delegate`) ==все вызовы методов туда==.


💡 Используется как **основа** для таких классов, как:
- `BufferedOutputStream` (добавляет буферизацию),
- `DataOutputStream` (пишет примитивы в бинарном виде),
- `PrintStream` (удобная печать, например `System.out`),
- и другие.
    

> [!NOTE] ⭐`FilterOutputStream` НЕ используется напрямую
> Сам по себе `FilterOutputStream` редко используется напрямую. Его роль — быть «скелетом» для написания собственных фильтров/обёрток.

***
## Конструктор
```java
// Создаёт фильтр, который пишет в указанный поток
// ⭐⭐param out — поток, в который будут перенаправляться все операции
public FilterOutputStream(OutputStream out)
```

## Методы
```java
// Записывает 1 байт в подлежащий поток.
// param b — байт (0–255)
public void write(int b) throws IOException

// Записывает массив байтов целиком.
// param b — массив данных
public void write(byte[] b) throws IOException

// Записывает часть массива байтов.
// param b — массив данных
// param off — смещение
// param len — сколько байтов писать
public void write(byte[] b, int off, int len) throws IOException

// Принудительно сбрасывает буфер подлежащего потока (если есть).
public void flush() throws IOException

// Закрывает поток: сначала flush(), затем закрывает out.
public void close() throws IOException

```

***
## ⭐⭐Особенности

- Все методы просто **делегируют вызовы** во внутренний поток (`out`).
    
- Метод `close()` вызывает `flush()`, а потом закрывает `out`.
    
- ⭐Часто используется как **база для расширения**: можно переопределить `write(...)` и добавить собственную логику 
⭐(например, логгирование, шифрование, сжатие).
    
- Поток **не добавляет никакой дополнительной функциональности**, если использовать его напрямую.
***
## Пример
```java
import java.io.*;

public class FilterOutputStreamExample {
    public static void main(String[] args) {
	    //ДВА открытых РЕСУРСА
        try (OutputStream fos = new FileOutputStream("output.txt");
             FilterOutputStream fosFilter = new FilterOutputStream(fos)) 
        
        {

            // Запишем строку
            String text = "Hello via FilterOutputStream!\n";
            fosFilter.write(text.getBytes("UTF-8"));

            fosFilter.flush(); // Сброс в файл
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

## Best-Practices

- ✅ Не использовать напрямую — он сам по себе бесполезен. Используй его наследников (`BufferedOutputStream`, `DataOutputStream`, `PrintStream`).
    
- ✅ Используй его как **базовый класс**, если нужно реализовать собственный поток-обёртку.
    
- ❌ Не забывать, что `close()` закроет и вложенный поток → если поток разделяется между несколькими декораторами, нужно аккуратно управлять жизненным циклом.
    
- ✅ При написании кастомных фильтров стоит переопределять только `write(int b)` (остальные методы уже реализованы через него).
