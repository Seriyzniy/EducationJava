# 📕 1. Интерфейс ObjectInputFilter

> [!NOTE] `java.io.ObjectInputFilter` — это интерфейс ==для **ФИЛЬТРАЦИИ ОБЪЕКТОВ при ДЕСЕРИАЛИЗАЦИИ**.== 
>  
>  Когда мы читаем объект через `ObjectInputStream.readObject()`, JVM вызывает фильтр, чтобы спросить: «А можно ли десериализовать вот этот класс, вот такого размера?»

⭐Цель — **предотвратить атаки через вредоносные данные** (gadget chains, huge object graphs → DoS, OutOfMemory).

## Интерфейс ObjectInputFilter
```java
@FunctionalInterface
public interface ObjectInputFilter {
	
	// МЕТОД для реализации
    /**
     * Проверяет десериализацию объекта.
     * filterInfo информация о десериализуемом объекте
     * @return решение фильтра
     */
    ObjectInputFilter.Status checkInput(ObjectInputFilter.FilterInfo filterInfo);

    // Возможные статусы
    enum Status {
        UNDECIDED, // ❓ фильтр не уверен → JVM спросит дальше
        ALLOWED,   // ✅ разрешить
        REJECTED   // ❌ запретить
    }

    interface FilterInfo {
        Class<?> serialClass();   // класс, который пытаются десериализовать
        long arrayLength();       // длина массива (если массив)
        long depth();             // глубина графа объектов
        long references();        // количество ссылок в графе
        long streamBytes();       // сколько байт уже прочитано
    }
}
```

## 💡 Особенности и подводные камни ObjectInputFilter 💡

1. 💡**Фильтры можно задавать глобально и локально.**
    - Локально: `ois.setObjectInputFilter(filter)` на конкретный `ObjectInputStream`.
        
    - Глобально: системное свойство `jdk.serialFilter` или через API `ObjectInputFilter.Config.setSerialFilter(...)`.
        
2. **UNDECIDED — значит, JVM ищет другой фильтр.**  
    Если все фильтры undecided → десериализация разрешается.
    
3. **Можно строить цепочки фильтров.** Например, один фильтр проверяет только типы, другой — глубину графа.
    
4. **Белые списки и чёрные списки.** Обычно делают whitelist: разрешаем только конкретные типы (`String`, `MyMessage`, `ArrayList` и т.д.).

## Пример
```java
import java.io.*;

class Message implements Serializable {
    private static final long serialVersionUID = 1L;
    
    String text;
    Message(String t) { text = t; }
}

public class InputFilterExample {
    public static void main(String[] args) throws Exception {
        // сериализуем
        try (ObjectOutputStream out = new ObjectOutputStream(
	        new FileOutputStream("msg.bin"))
	    ){
            out.writeObject(new Message("Hello!"));
        }

        // десериализуем с фильтром
        try (ObjectInputStream in = new ObjectInputStream(
	        new FileInputStream("msg.bin"))
	    ){
		    // Установка фильтра
            in.setObjectInputFilter(info -> {
                if (info.serialClass() == null) {
	                return ObjectInputFilter.Status.UNDECIDED;
	            }
                if (info.serialClass() == Message.class){
	                return ObjectInputFilter.Status.ALLOWED;
                } 
                return ObjectInputFilter.Status.REJECTED;
            });

            Message msg = (Message) in.readObject();
            System.out.println("Message: " + msg.text);
        }
    }
}
```

# 📕 2.  Интерфейсы ObjectOutputFilter

> [!NOTE] `java.io.ObjectOutputFilter` появился позже (`Java 17`).  
> 
> Он нужен для ==**фильтрации объектов при СЕРИАЛИЗАЦИИ**== (то есть при `ObjectOutputStream.writeObject()`).

Это симметричная защита: мы можем не только ==контролировать==, что читаем, но и ==что пишем== (например, не допустить утечки секретных типов или слишком больших структур).

## Интерфейс ObjectOutputFilter
```JAVA
@FunctionalInterface
public interface ObjectOutputFilter {
    /**
     * Проверяет сериализуемый объект.
     * @param filterInfo информация о сериализуемом объекте
     * @return решение фильтра
     */
    ObjectOutputFilter.Status checkOutput(ObjectOutputFilter.FilterInfo filterInfo);

    enum Status {
        UNDECIDED, // не знаем, пусть решает другой фильтр
        ALLOWED,   // разрешить сериализацию
        REJECTED   // запретить
    }

    interface FilterInfo {
        Class<?> serialClass();   // класс объекта
        long arrayLength();       // длина массива (если массив)
        long depth();             // глубина графа объектов
        long references();        // количество ссылок
        long streamBytes();       // сколько байтов уже записано
    }
}

```

## 💡 Особенности и подводные камни ObjectOutputFilter 💡

1. 💡**Похож на InputFilter**, но срабатывает при записи.
    
2. 💡**Редко нужен в обычном приложении.** Чаще применяют на границах (💡например, сервис пишет данные в сокет, и мы хотим гарантировать, что наружу не уйдёт что-то лишнее).
    
3. **Полезен для аудита.** Можно логировать, какие типы объектов уходят наружу.
    
4. **Также бывают глобальные и локальные фильтры.** Настраиваются похожим образом.

## Пример
```java
import java.io.*;

class Secret implements Serializable {
    private static final long serialVersionUID = 1L;
    String data = "SECRET";
}

public class OutputFilterExample {
    public static void main(String[] args) throws Exception {
        try (ObjectOutputStream out = new ObjectOutputStream(
			new FileOutputStream("out.bin"))
		) {
            out.setObjectOutputFilter(info -> {
                if (info.serialClass() == Secret.class) {
                    System.out.println("Запрещаем сериализацию Secret!");
                    return ObjectOutputFilter.Status.REJECTED;
                }
                return ObjectOutputFilter.Status.ALLOWED;
            });

            out.writeObject("Normal string");
            // out.writeObject(new Secret()); // ❗❗❗тут будет REJECTED
        }
    }
}
```

# Best Practices

- **ObjectInputFilter**
    - Всегда используйте при десериализации данных извне (сеть, файлы от пользователя).
        
    - Делайте whitelist допустимых типов.
        
    - Ограничивайте `depth`, `references`, `arrayLength`, чтобы не словить DoS.
        
- **ObjectOutputFilter**
    - Используйте для контроля и аудита, что вы сериализуете.
        
    - Запрещайте чувствительные классы уходить наружу.
        
    - Хорошо подходит для библиотек и middleware.
