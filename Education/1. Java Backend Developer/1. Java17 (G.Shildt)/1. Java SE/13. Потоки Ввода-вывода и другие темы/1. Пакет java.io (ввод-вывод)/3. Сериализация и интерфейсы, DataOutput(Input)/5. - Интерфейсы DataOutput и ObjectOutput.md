# 📕 1. Интерфейс DataOutput

> [!NOTE] `java.io.DataOutput` — это ==интерфейс для записи **примитивов и строк** в поток== в _машино-независимом ==бинарном формате_==.  
> 
> ⭐Его основная реализация — `DataOutputStream`.


⭐То есть вместо того чтобы вручную кодировать числа в байты, можно вызвать `writeInt(42)` и быть уверенным, что оно запишется в 4 байта Big Endian (по спецификации Java).

## Интерфейс DataOutput и методы
```java
public interface DataOutput {
    // Запись одного байта (только младшие 8 бит int)
    void write(int b) throws IOException;

    // Запись массива байтов
    void write(byte[] b) throws IOException;

    // Запись части массива
    void write(byte[] b, int off, int len) throws IOException;

    // Запись boolean (true = 1, false = 0)
    void writeBoolean(boolean v) throws IOException;

    // Запись одного байта (символ как int 0-255)
    void writeByte(int v) throws IOException;

    // Запись 2 байт (Big Endian)
    void writeShort(int v) throws IOException;

    // Запись 2 байт (Big Endian, беззнаковое значение)
    void writeChar(int v) throws IOException;

    // Запись 4 байт (Big Endian)
    void writeInt(int v) throws IOException;

    // Запись 8 байт (Big Endian)
    void writeLong(long v) throws IOException;

    // Запись 4 байт IEEE 754
    void writeFloat(float v) throws IOException;

    // Запись 8 байт IEEE 754
    void writeDouble(double v) throws IOException;

    // Запись строки в "modified UTF-8" формате (специфичная версия UTF-8)
    void writeUTF(String s) throws IOException;

    // Запись массива байтов длиной 2 (Big Endian, unsigned short) + строка UTF-8
    void writeBytes(String s) throws IOException;

    // Запись каждого символа строки как 16-битный char (Big Endian)
    void writeChars(String s) throws IOException;
}
```

## 💡 Особенности и подводные камни DataOutput 💡

1. 💡**Формат фиксирован.** Все методы обязаны писать данные в ==Big Endian==. То есть файлы будут переносимы между архитектурами.
    
2. **writeUTF — не настоящий UTF-8.** Это так называемый _modified UTF-8_ (например, `null`-символ кодируется в 2 байта). Такой формат используется в Java для совместимости с `DataInput.readUTF`.
    
3. **writeBytes vs writeChars.**
    - 🔸`writeBytes(String)` — каждый символ обрезается до ==одного байта== (опасно для Unicode).
        
    - 🔸`writeChars(String)` — каждый символ пишется ==как 2 байта== (Java char).  
        В обоих случаях лучше использовать `writeUTF` для строк.
        
1. **Низкоуровневая вещь.** Почти всегда используешь через `DataOutputStream`, редко напрямую.

## Пример
```java
import java.io.*;

public class DataOutputExample {
    public static void main(String[] args) throws IOException {
        try (DataOutputStream out = new DataOutputStream(new FileOutputStream("data.bin"))) {
            out.writeInt(42);
            out.writeDouble(3.14);
            out.writeUTF("Привет");
        }

        try (DataInputStream in = new DataInputStream(new FileInputStream("data.bin"))) {
            System.out.println(in.readInt());    // 42
            System.out.println(in.readDouble()); // 3.14
            System.out.println(in.readUTF());    // Привет
        }
    }
}

```

# 📕 2. Интерфейс ObjectOutput


> [!NOTE] `java.io.ObjectOutput` расширяет `DataOutput` и ==добавляет возможность **записывать объекты**.==
> 
> 💡Его `основная реализация` — `ObjectOutputStream`.

То есть это «DataOutput + сериализация объектов».
## Интерфейс ObjectOutput
```java
public interface ObjectOutput extends DataOutput, AutoCloseable {
    // Записывает объект (должен быть Serializable или Externalizable)
    void writeObject(Object obj) throws IOException;

    // Остальные методы приходят от DataOutput:
    // writeInt, writeUTF, writeDouble и т.д.

    // Управление потоком
    void flush() throws IOException;
    void close() throws IOException;
}
```

## 💡 Особенности и подводные камни ObjectOutput 💡

1. 💡 **writeObject требует Serializable.**  
    Если объект не реализует `Serializable` → `NotSerializableException`.
    
2. 💡 **Целый граф объектов.**  
    Записывается не только сам объект, но и все объекты, на которые он ссылается.
    
3. 💡**Кеширование объектов.**  
    `ObjectOutputStream` кеширует объекты и может писать ссылки вместо дубликатов. Чтобы «сбросить» кеш и записывать заново — используйте `reset()`.
    
4. **Совместимость версий.**  
    Как и с Serializable: без `serialVersionUID` есть риск сломать обратную совместимость.

## Пример
```java
import java.io.*;

class Person implements Serializable {
    private static final long serialVersionUID = 1L;
    String name;
    int age;
    Person(String n, int a) { name = n; age = a; }
}

public class ObjectOutputExample {
    public static void main(String[] args) throws IOException, ClassNotFoundException {
    
        // Запись
        try (ObjectOutputStream out = new ObjectOutputStream(
	        new FileOutputStream("person.bin"))
	    ) {
            out.writeObject(new Person("Alice", 30));
        }

        // Чтение
        try (ObjectInputStream in = new ObjectInputStream(
	        new FileInputStream("person.bin"))
	    ) {
            Person p = (Person) in.readObject();
            System.out.println(p.name + " " + p.age); // Alice 30
        }
    }
}

```

# Best Practices

- **DataOutput**
    - Используйте `writeUTF` для строк → совместимость с `DataInput.readUTF`.
        
    - Не используйте `writeBytes` для Unicode-строк (обрежет символы).
        
    - Всегда оборачивайте в буфер (`BufferedOutputStream`) для скорости.
        
- **ObjectOutput**
    - Всегда определяйте `serialVersionUID` в сериализуемых классах.
        
    - Не пишите в поток объекты, если не уверены в совместимости версий.
        
    - Если нужно несколько раз писать один и тот же объект в файл, вызывайте `ObjectOutputStream.reset()` (иначе будут ссылки, а не новые копии).
        
    - Для сетевых протоколов чаще лучше придумать свой формат (JSON, protobuf, Avro) — стандартная сериализация небезопасна и тяжеловесна.
