Ключевое слово `final` в Java используется ==для **создания неизменяемых элементов**.== 

Оно может применяться к:
1. **переменным**,
2. **методам**,
3. **классам**.

# 💡 Ключевые особенности `final` 💡

| Где используется    | Что означает                                                              |
| ------------------- | ------------------------------------------------------------------------- |
| `final` переменная  | Нельзя изменить значение после присваивания                               |
| `final` поле класса | Должно быть инициализировано один раз (в конструкторе или при объявлении) |
| `final` метод       | Нельзя переопределить в подклассе                                         |
| `final` класс       | Нельзя расширить (наследовать)                                            |
| `final` ссылка      | НЕЛЬЗЯ поменять ссылку на другой объект                                   |

В каждом случае поведение `final` немного отличается, но ==общее правило: **нельзя изменить, переопределить или повторно присвоить**.==

## ⭐⭐⭐ Гарантия безопасной публикации (`final field semantics`)

> [!NOTE] ⭐Работает ТОЛЬКО при использовании конструктора⭐
> Логично, так как `final` необходимо инициализировать в конструкторе либо сразу при инициализации
> 
> ⭐Однако, если поля типа final ссылаются на изменяемые объекты, ⭐синхронизация по-прежнему требуется⭐ для доступа к состоянию объектов, на которые они ссылаются.

Когда объект создаётся:
1. Все **записи в `final` поля**, сделанные конструктором:
    - становятся _замороженными_ (frozen) в момент выхода из конструктора;
        
    - никакой поток не увидит «старое» значение, даже без `synchronized` или `volatile`.
        
2. Это распространяется **транзитивно**:
    - Если `final` поле ссылается на другой объект, то все записи внутри этого объекта, сделанные конструктором, тоже «замораживаются».

## Почему так работает
- В JMM для `final` полей есть **специальные правила**:
    - ==Записи в `final` не могут «утечь» до завершения конструктора==.
        
    - ==Запрещены reorder-оптимизации,== которые могли бы выставить ссылку наружу до инициализации `final` полей.
        
- То есть:
    - `final` = не только «однократное присваивание», но и ==**барьер памяти** на этапе публикации.==

# 📕 1. `final` переменные
## 🔸 Локальные переменные и параметры
Если переменная объявлена с `final`, ей ==**можно присвоить значение ТОЛЬКО ОДИН РАЗ**==.
```java
void example(final int x) {
    // x = 5; // Ошибка — нельзя изменить final-параметр
}

//----------------

final int a = 10;
a = 20; // Ошибка — нельзя изменить
```

> [!NOTE] `final` при объявлении ссылки (объекта класса)
> ✅ `final` **не означает**, что объект неизменен. Это означает, что **ссылка** не может указывать на другой объект.
```java
final List<String> list = new ArrayList<>();
list.add("Hello"); // OK — объект изменяем

list = new ArrayList<>(); // Ошибка — нельзя изменить ссылку

```

## 🔸 `final` поля в классе

- Для **полей экземпляра**: `final` нужно инициализировать **при объявлении** или **в конструкторе**.
- Для **`static final`**: обязательно задать значение при объявлении или в `static` блоке.
```java
class MyClass {
    final int id; // можно задать только в конструкторе
    static final double PI = 3.1415; // либо при объявлении, либо в static-блоке

    MyClass(int id) {
        this.id = id; // OK
    }
}
```

# 📕 2. `final` методы

Метод, объявленный как `final`, **не может быть переопределён в подклассе**.
```java
class Base {
    final void show() {
        System.out.println("Base");
    }
}

class Derived extends Base {
    // void show() {} // Ошибка: нельзя переопределить final-метод
}
```

***
# 📕 3. `final` классы

==Класс==, объявленный как `final`, ==**не может иметь подклассов*==.
```java
final class MathUtils {
    public static int square(int x) {
        return x * x;
    }
}

class ExtendedMathUtils extends MathUtils {} // Ошибка
```

# 🧠 Часто задаваемые вопросы:

### ❓ Можно ли инициализировать `final` переменную позже?
Да, если это:
- **поле экземпляра** → в конструкторе.
- **локальная переменная** → до первого использования.

```java
void test() {
    final int x;
    if (someCondition()) {
        x = 10;
    } else {
        x = 20;
    }
    System.out.println(x); // OK — x инициализирован
}
```

### ❓ Чем отличается `final` от `const` в других языках?
==В Java нет== `const`. Вместо этого используют `final`, а **неизменяемость объекта** реализуется через:
- неизменяемые классы (`String`, `List.of(...)`)
- отсутствие `set`-методов
- `Collections.unmodifiableList(...)` и аналоги.
