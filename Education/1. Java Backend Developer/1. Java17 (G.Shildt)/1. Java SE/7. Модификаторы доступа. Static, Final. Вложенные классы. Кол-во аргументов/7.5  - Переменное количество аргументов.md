Аргументы переменной длины (varargs) в Java позволяют методам принимать **произвольное количество аргументов** одного типа.

# 💡 Синтаксис varargs 💡
```java
void printNumbers(int... numbers) {
    for (int n : numbers) {
        System.out.println(n);
    }
}
```

***
# 💡 Что происходит "под капотом"? 💡

Когда вы пишете `int... numbers`, компилятор ==**преобразует это в массив**==:
```java
void printNumbers(int[] numbers) // фактически то же самое
```

> [!NOTE] `int... numbers`
> Varargs — это **синтаксический сахар** для массива.

***
# ⚠️ Особенности и ограничения ⚠️

1. **Varargs-параметр ==должен быть последним**== в списке параметров метода
```java
void example(String label, int... values) {}  // ✅ правильно
void wrong(int... values, String label) {}    // ❌ ошибка компиляции
```

2. **Только один параметр varargs в методе**
```java
void doSomething(int... a, String... b) {}  // ❌ нельзя два varargs
```

3. **Можно передать явно массив**
```java
printNumbers(new int[]{10, 20, 30}); // тоже работает
```

4. Можно перегружать (это же массив, просто меняем тип массива)
```java
class VarAgs{
	static void vaTest(int ... x){
		return 0;
	}
	
	static void vaTest(float ... x){
		return 1;
	}
}
```
