Для того чтобы один класс унаследовал `public` и `protected` поля и методы, используется ключевое слово `extends`
```java
class Base{
	int b1 = 1;
	void print() {
		System.out.println("Base = " + b1);
	}
}

class SecondBase extends Base1{
	int b2 = 2;
	void print() {
		System.out.println("SecondBase = " + b2);
	}
}
```

# 📕 Что подлежит наследованию 📕

| Члены суперкласса             | Наследуются?                | Можно использовать?               | Можно переопределить?                          |
| ----------------------------- | --------------------------- | --------------------------------- | ---------------------------------------------- |
| **public поля**               | ✅ Да                        | ✅ Да                              | ❌ Нет (==переопределять можно только методы==) |
| **public методы**             | ✅ Да                        | ✅ Да                              | ✅ Да (если не `final`)                         |
| ==-------------------==       |                             |                                   |                                                |
| **protected поля**            | ✅ Да                        | ✅ Да (внутри подкласса)           | ❌ Нет                                          |
| **protected методы**          | ✅ Да                        | ✅ Да                              | ✅ Да                                           |
| ==-------------------==       |                             |                                   |                                                |
| **default (package-private)** | ✅ Да (если в том же пакете) | ✅ Да                              | ✅ Да                                           |
| **private поля/методы**       | ❌ Нет                       | ❌ Нет                             | ❌ Нет                                          |
| ==-------------------==       |                             |                                   |                                                |
| **конструкторы**              | ❌ Нет                       | ❌ Нет                             | ❌ Нет (но можно вызывать `super(...)`)         |
| ==-------------------==       |                             |                                   |                                                |
| **static поля**               | ✅ Да                        | ✅ Да (но не через `this`)         | ❌ Нет (не переопределяются)                    |
| **static методы**             | ✅ Да                        | ✅ Да                              | ❌ Нет (можно скрыть, но не переопределить)     |
| ==-------------------==       |                             |                                   |                                                |
| **final методы**              | ✅ Да                        | ✅ Да                              | ❌ Нет                                          |
| **abstract методы**           | ✅ Да (если не реализованы)  | ❌ Нет напрямую — надо реализовать | ❌ Нет                                          |

## 2. Ссылке на базовый класс, присваивается ссылка на производный  
> [!NOTE] Переменная типа суперкласса может ссылаться на объект подкласса
> При этом, когда ссылочная переменная типа суперкласса присваивается ссылка на объект подкласса, то ==доступ имеется только к тем частям объекта, которые определены в суперклассе==
```java
public static void main(String[] args) {
	Base b1 = new SecondBase();
	b1.print();
}

//Вывод: SecondBase = 2
```

## 3. Порядок вызова переопределенных методов при наследовании
> [!NOTE] Полиморфизм обеспечивается во время выполнения, а не компиляции (Динамический полиморфизм)
> Именно ==тип объекта==, на которые делается ссылка (а не тип ссылочной переменной), ==определяет== какая ==версия ПЕРЕОПРЕДЕЛЕННОГО== метода будет выполняться

```java
class Base{
	int b1 = 1;
	void print() {
		System.out.println("Base = " + b1);
	}
}

class SecondBase extends Base{
	int b2 = 2;
	void print() {
		System.out.println("SecondBase = " + b2);
	}
}

class ThirdBase extends SecondBase{
	int b3 = 3;
	void print() {
		System.out.println("ThirdBase = " + b3);
	}
}


// Функция Main
public class MyMain {
	public static void main(String[] args) {
		Base b1 = new ThirdBase();
		b1.print();
	}
}

//Вывод: ThirdBase = 3
```

При этом, если в классе `ThirdBase` убрать метод print, то вызовется метод print из класса SecondBase
```java
class Base{
	int b1 = 1;
	void print() {
		System.out.println("Base = " + b1);
	}
}

class SecondBase extends Base{

		System.out.println("SecondBase = " + b2);
	}
}

class ThirdBase extends SecondBase{

}

//Main
public class MyMain {

		Base b1 = new ThirdBase();
		b1.print();
	}

//Вывод: SecondBase = 2
```

## 4. ==Полиморфны ТОЛЬКО методы==, а поля нет
```java
class A { int x = 5; }
class B extends A { int x = 10; }

A ref = new B();
System.out.println(ref.x); // выведет 5, а не 10
```

# 📕 2. Ключевое слово `super` 
Позволяет обращаться к членам суперкласса:
- `super.field` — доступ к полю родителя (==используется если подкласс "скрывает" родительскую переменную или метод)==
- `super.method()` — вызов метода родителя
    
- `super()` — вызов конструктора ==ближайшего== суперкласса (==обязательно в первой строке конструктора подкласса==)

```java
class A {
	private int x;
    
    A (int x) {
	    this.x = x;
    }
}

class B extends A {
	private float y; 
	
    B (int x, float y) {
        super(x); // обязательно
        this.y = y;
    }
}
```
