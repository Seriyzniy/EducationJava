# 📕 1. Модуль java.base
Начиная с JDK 9, ==API Java организован в **модули платформы**== (их имена начинаются с `java.`, например, `java.base`, `java.desktop`).

- **`java.base`**: Самый главный модуль. Содержит фундаментальные пакеты (`java.lang`, `java.util`, `java.io` и др.).
    
- 💡**Автоматическая доступность**: Модуль `java.base` автоматически требуется всем другим модулям. Его не нужно объявлять в `requires`. Именно поэтому классы вроде `System` и `Math` доступны без дополнительных объявлений.

# 📕 2. Немодульный "старый" код
 Объясняется, как Java сохраняет обратную совместимость с кодом, написанным до JDK 9 (немодульным кодом).

- **Неименованный модуль**: Весь код, не входящий в именованные модули, автоматически помещается в специальный "неименованный модуль".
    - Все пакеты в неименованном модуле автоматически экспортируются.
    - Неименованный модуль имеет доступ ко всем другим модулям.
        
- **Путь к классам (classpath)**: Для немодульных программ продолжает использоваться механизм classpath, как в старых версиях Java.
    
**Вывод:** Благодаря этому старый код продолжает работать без изменений в новых версиях Java.
