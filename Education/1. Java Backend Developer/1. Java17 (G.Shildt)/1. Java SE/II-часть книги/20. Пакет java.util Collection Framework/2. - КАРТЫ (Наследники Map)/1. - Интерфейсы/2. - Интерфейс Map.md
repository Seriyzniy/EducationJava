
> [!NOTE] `Map` — это интерфейс для структур данных, хранящих ==пары **ключ → значение**.==
> 
> ⭐В отличие от `Collection`, `Map` **НЕ НАСЛЕДУЕТСЯ от `Collection`**.  ⭐
> 
> Каждый **ключ уникален**, а значение может быть любым (в т.ч. дублироваться).


> [!NOTE] ⭐❗❗❗`null` ЗАПРЕЩЕН!!!❗❗❗⭐
> При попытке использования объекта `null`, (а значения `null` в карте не разрешены) генерируется исключение `NullPointerException`.

## Главные методы `get()` и `put()`, и остальные
В `К` указывается тип ключей, а в `V` - тип значений. (`Key` = `Value`)

```java
public interface Map<K, V> {
	// ✅ Статические операции  ✅
	// 💡Создает неизменяемую и основанную на значении карту💡 с элементами, указанными в списке параметров.
	// Ключи или значения null не разрешены.
	static <К,V> Мар<К,V> of(список-параметров)
	
	// Возвращает неизменяемую и основанную на значении карту с парами 
	// "ключ-значение" описанными с помощью элементов, которые переданы в 
	// entries. 
	// Ключи или значения null не разрешены
	static <К,V> Map<К,V> ofEntries(
		Map.Entry<? extends К, ? extend V> ... entries)

    // ✅ Базовые операции  ✅
    // Возвращает количество пар (ключ-значение) в карте.
    int size();                         
    
	// true, если карта пустая (нет элементов).
    boolean isEmpty();                  

	// Возвращает значение по ключу.
    // Если ключа нет – вернёт null.
    V get(Object key);                  
    
	// Добавляет новую пару (ключ-значение).
    // 💡🧠Если ключ уже есть — заменяет старое значение и возвращает его.
    V put(K key, V value);              
    
	// Помещает все элементы из m в вызывающую карту
	void putAll(Map<? extends K, ? extends V> m)
	
    V remove(Object key);               
    // Удаляет пару по ключу.
    // Возвращает старое значение, если ключ был, иначе null.


    // ✅ Проверки ✅ 
	// true, если карта содержит указанный ключ.
    boolean containsKey(Object key);    
    
	// true, если карта содержит хотя бы одно указанное значение.
    boolean containsValue(Object value);
    
    // ✅ Очистка ✅
	// Полностью очищает карту (удаляет все пары).
    void clear();                       
    

    //  ✅ Коллекции-представления ✅
    // 💡Возвращает множество всех КЛЮЧЕЙ💡 (Set<K>).
    Set<K> keySet();                    
    
	// 💡Возвращает коллекцию всех ЗНАЧЕНИЙ💡 (допускает дубликаты).
    Collection<V> values();             
    
	// 💡Возвращает множество всех ПАР💡 (ключ-значение).
    // Map.Entry<K,V> — вложенный интерфейс для работы с парой.
    Set<Map.Entry<K, V>> entrySet();    
    
    // ✅ Новые методы с Java 8 ✅ 
    // 💡 Возвращает значение по ключу 
    // или defaultValue, если ключ отсутствует.
    default V getOrDefault(Object key, V defaultValue);
    
    // 💡 Выполняет ❗❗действие для каждой пары❗❗ (ключ, значение).
    default void forEach(
	    BiConsumer<? super K, ? super V> action);
    
	// Заменяет все значения в карте, применяя функцию:
    // новое значение = function(ключ, старое значение).
    default void replaceAll(
	    BiFunction<? super K, ? super V, ? extends V> function);
    
	// 💡 Добавляет пару только если ключ ещё не существует.
    // Если уже есть — ничего не делает.
    default V putIfAbsent(K key, V value);
    
    // Удаляет пару только если совпадают и ключ, и значение.
    default boolean remove(Object key, Object value);
    
	// Заменяет значение по ключу на новое, если текущее == oldValue.
    // Возвращает true/false — удалось ли заменить.
    default boolean replace(K key, V oldValue, V newValue);
    
	// Заменяет значение по ключу на новое, если ключ существует.
    // Возвращает старое значение или null, если ключа не было.
    default V replace(K key, V value);
    
    // 💡🟢 ЕСЛИ КЛЮЧ ОТСУТСТВУЕТ, вычисляет значение функцией и 
    // добавляет в карту.
    // Если ключ есть — просто возвращает его значение.
    default V computeIfAbsent(
	    K key, 
	    Function<? super K, ? extends V> mappingFunction);
   
	// 💡🟣 ЕСЛИ КЛЮЧ ПРИСУТСТВУЕТ, вычисляет новое значение по функции 
	// и обновляет его.
    // Если функция вернёт null — пара удаляется.
    default V computeIfPresent(
	    K key, 
	    BiFunction<? super K, ? super V, ? extends V> remappingFunction);
    
    // Всегда пересчитывает значение для ключа (есть он или нет).
    // Если функция вернула null → пара удаляется.
    default V compute(
	    K key, 
	    BiFunction<? super K, ? super V, ? extends V> remappingFunction);
    

	// Если ключа нет — просто добавляет (key, value).
    // Если есть — объединяет старое и новое значение функцией.
    default V merge(
	    K key, 
	    V value, 
	    BiFunction<? super V, ? super V, ? extends V> remappingFunction);
    
}
```

***
## 💡 Особенности и подводные камни💡 
- 🔶 💡Ключи должны быть **уникальны**, но значения могут повторяться.
    
- 🔶Некоторые реализации (`TreeMap`) требуют, чтобы ключи были **сравнимы**.
    
- 🔶💡💡В `HashMap` ключ может быть `null` (только один), а значение — любое количество `null`.
- 🔶💡💡В `TreeMap` ключ `null` запрещён (так как он сравнивается).
    
- 🔶Итерация по `HashMap` не гарантирует порядок.
    
- 🔶У `LinkedHashMap` порядок = порядок вставки или порядок доступа.
    
- 🔶 💡`ConcurrentModificationException` может прилететь при итерации по `Map.entrySet()` если карта модифицируется (кроме ConcurrentHashMap).

***
## ✅⭐Best Practices⭐✅
- Если важна **скорость** → `HashMap`.
    
- Если важен **порядок вставки** → `LinkedHashMap`.
    
- Если важна **сортировка по ключу** → `TreeMap`.
    
- Для многопоточности → `ConcurrentHashMap` (не `Hashtable`).
    
- Для enum-ключей → `EnumMap` (очень быстрый и компактный).

## Пример
```java
import java.util.*;

public class MapDemo {
    public static void main(String[] args) {
        Map<String, Integer> map = new HashMap<>();

        // Добавление элементов
        map.put("Java", 1);
        map.put("C++", 2);
        map.put("Python", 3);

        // Получение значения
        System.out.println("Python -> " + map.get("Python")); // 3

        // Проверки
        System.out.println("Содержит ключ 'Java'? " + map.containsKey("Java"));
        System.out.println("Содержит значение 2? " + map.containsValue(2));

        // Обход
        for (Map.Entry<String, Integer> entry : map.entrySet()) {
            System.out.println(entry.getKey() + " = " + entry.getValue());
        }

        // Новые методы (Java 8+)
        map.putIfAbsent("Go", 4); // добавит только если ключа нет
        map.merge("Python", 10, Integer::sum); // Python = 3+10 = 13

        System.out.println("Обновлённый map: " + map);
    }
}

```
