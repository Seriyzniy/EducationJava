
> [!NOTE] Класс `EnumМap` расширяет класс `AЬstractMap` и реализует интерфейс `Мар`.
> Предназначен ==специально ДЛЯ ИСПОЛЬЗОВАНИЯ С КЛЮЧАМИ перечислимоrо типа== (enum)

```java
class EnumМap<K extends Enum<K>, V>

В K указывается тип ключей, а в V - тип значений
```
## Что под капотом
- В основе — **обычный массив значений** (`Object[]`), где индекс соответствует **`ordinal()`** ключа (позиции в перечислении).
    
- 💡Ключи хранятся **ссылкой на сам enum-класс**, поэтому работать ==можно только с одним фиксированным== `enum` для данной карты.
    
- 💡Нет хэширования💡, нет коллизий — доступ идёт напрямую по индексу → **O(1)** операции.
    
- Порядок обхода всегда соответствует **естественному порядку enum (ordinal)**.

***
## Конструкторы
```java
// Создаёт пустую карту для указанного класса enum
EnumMap(Class<K> keyType)

// Создаёт карту, копируя все пары из другой EnumMap того же типа ключа
EnumMap(EnumMap<K, ? extends V> m)

// Создаёт карту, копируя все пары из другой Map
// (ключи должны быть совместимы с типом enum)
EnumMap(Map<K, ? extends V> m)
```

## Методы
Никаких собственных методов класс `EnumМap` не определяет.

***

## 💡 Особенности и подводные камни 💡

- 💡 Очень компактная реализация: ==ключи не хранятся==, только значения.
    
- Быстрее и экономичнее памяти, чем `HashMap<Enum, V>`.
    
- Нельзя использовать ключи вне указанного класса `enum`.
    
- `null` ключ не допускается (будет `NullPointerException`).

***
## Пример
```java
import java.util.EnumMap;

enum Day { MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY }

public class EnumMapDemo {
    public static void main(String[] args) {
        EnumMap<Day, String> schedule = new EnumMap<>(Day.class);

		// ⭐⭐Из ключей ТОЛЬКО enum-значения⭐⭐
        schedule.put(Day.MONDAY, "Gym");
        schedule.put(Day.TUESDAY, "Work");
        schedule.put(Day.WEDNESDAY, "Study");

        System.out.println(schedule); 
        // {MONDAY=Gym, TUESDAY=Work, WEDNESDAY=Study}
        
        System.out.println(schedule.get(Day.MONDAY)); // Gym
    }
}
```
