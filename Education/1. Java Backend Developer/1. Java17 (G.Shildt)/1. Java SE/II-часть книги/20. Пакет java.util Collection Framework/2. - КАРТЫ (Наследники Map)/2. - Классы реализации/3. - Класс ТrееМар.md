
> [!NOTE] Класс `TreeMap` расширяет класс `AbstractMap` и реализует интерфейс `NavigableMap`.
> Хранит элементы ==в **отсортированном порядке ключей**== (по естественному порядку `Comparable` или по переданному `Comparator`)

🧠Под капотом используется ==**красно-чёрное дерево**== (самобалансирующееся бинарное дерево поиска).  
⭐Поэтому операции поиска, вставки и удаления имеют сложность **O(log n)**.

***
## Конструкторы
```java
public TreeMap();  
// Конструктор по умолчанию — сортировка по естественному порядку ключей (ключи должны реализовывать Comparable)

public TreeMap(Comparator<? super K> comparator);  
// Сортировка по заданному компаратору

public TreeMap(Map<? extends K, ? extends V> m);  
// Конструктор на основе другой карты (копирование всех элементов)

public TreeMap(SortedMap<K, ? extends V> m);  
// Конструктор на основе SortedMap (с сохранением порядка)
```

## Методы
`TreeMap` наследует методы `Map`, `SortedMap`, `NavigableMap`.
```java
// ✅ Из NavigableMap ✅
Map.Entry<K, V> firstEntry();      // первая (минимальная) пара
Map.Entry<K, V> lastEntry();       // последняя (максимальная) пара
K firstKey();                      // первый ключ
K lastKey();                       // последний ключ

Map.Entry<K, V> lowerEntry(K key); // < key
K lowerKey(K key);

Map.Entry<K, V> floorEntry(K key); // <= key
K floorKey(K key);

Map.Entry<K, V> ceilingEntry(K key); // >= key
K ceilingKey(K key);

Map.Entry<K, V> higherEntry(K key); // > key
K higherKey(K key);

NavigableMap<K, V> headMap(K toKey, boolean inclusive);  
NavigableMap<K, V> tailMap(K fromKey, boolean inclusive);  
NavigableMap<K, V> subMap(K fromKey, boolean fromInclusive, K toKey, boolean toInclusive);  
// части карты по диапазонам

NavigableSet<K> navigableKeySet(); // ключи в порядке возрастания
NavigableSet<K> descendingKeySet(); // ключи в порядке убывания

// ✅ Методы из Map ✅
```

***
## 💡 Особенности и подводные камни 💡

- 🔶💡Все ключи должны быть **сравнимыми** (`Comparable`) или при создании `TreeMap` нужно передать `Comparator`.
    
- 🔶💡`null`-ключи запрещены (в отличие от `HashMap`), но **null-значения допустимы**.
    
- 🔶🧠Все операции занимают **O(log n)** (медленнее чем у `HashMap` с O(1) в среднем, но зато упорядочено).
    
- 🔶Итерация идёт в **отсортированном порядке** ключей.
    
- 🔶`TreeMap` не потокобезопасен. Для многопоточности → `Collections.synchronizedSortedMap()` или `ConcurrentSkipListMap`.

***
## ✅ Best Practices ✅
- Использовать `TreeMap`, когда важен **порядок ключей** (например, словари, интервалы, ранжирование).
    
- Для задач, где порядок не важен, лучше брать `HashMap` (быстрее).

***
## Пример
```java
import java.util.*;

public class TreeMapDemo {
    public static void main(String[] args) {
        TreeMap<String, Integer> scores = new TreeMap<>();

        // Добавление элементов
        scores.put("Charlie", 85);
        scores.put("Alice", 90);
        scores.put("Bob", 75);

        // Автоматически сортирует по ключу (по алфавиту)
        System.out.println("Все элементы: " + scores);

        // Получение первого и последнего ключа
        System.out.println("Первый ключ: " + scores.firstKey());
        System.out.println("Последний ключ: " + scores.lastKey());

        // Получение диапазона (от A до C)
        System.out.println("Диапазон [A;C): " + scores.subMap("A", true, "C", false));

        // Итерация по ключам в порядке возрастания
        for (Map.Entry<String, Integer> entry : scores.entrySet()) {
            System.out.println(entry.getKey() + " -> " + entry.getValue());
        }

        // Итерация по ключам в порядке убывания
        for (String key : scores.descendingKeySet()) {
            System.out.println(key + " -> " + scores.get(key));
        }
    }
}
```
