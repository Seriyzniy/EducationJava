# 📕 1. Iterator

Интерфейс `Iterator<E>` — это ==**универсальный курсор для обхода коллекции**== (обычно вперёд, от начала до конца).  
Появился ещё в Java 2 (замена устаревшему `Enumeration`).

```java
public interface Iterator<E> {
	// 💡true, если есть ещё элементы
    boolean hasNext(); 
    
    // 💡возвращает следующий элемент, сдвигая курсор
    E next();
    
    
    // удаляет последний возвращённый элемент (опционально)
    void remove(); 
    
    // вызывает action.accept(e) для всех оставшихся элементов
    default void forEachRemaining(Consumer<? super E> action);
}
```

### Особенности:
- 🔶**Однонаправленный** — только вперёд.
    
- 🔶**Fail-fast**: большинство реализаций бросают `ConcurrentModificationException`, если коллекция была изменена не через сам `Iterator`.
    
- 🔶Метод `remove()` — часто забывают: он удаляет **тот элемент, который вернул `next()`**

# 📕 2. ListIterator
`ListIterator<E>` — это ==**усовершенствованный курсор для списков**== (`List`), появился в Java 2.  
Он расширяет `Iterator` и добавляет двунаправленную навигацию.

```java
public interface ListIterator<E> extends Iterator<E> {
    boolean hasNext();        // есть ли следующий элемент
    E next();                 // вернуть следующий элемент
    
    boolean hasPrevious();    // есть ли предыдущий элемент
    E previous();             // вернуть предыдущий элемент


    int nextIndex();          // индекс следующего элемента
    int previousIndex();      // индекс предыдущего элемента

    void remove();            // удалить элемент (как в Iterator)
    void set(E e);            // заменить последний возвращённый элемент
    void add(E e);            // вставить новый элемент перед курсором
}
```

***
# ✅Best Practices✅

- Используй `for-each` (`enhanced for`) вместо ручного `Iterator`, когда нужен только обход.
    
- Используй `ListIterator`, когда нужно идти **назад** или **модифицировать список во время обхода**.
    
- Если нужен безопасный многопоточный обход → смотри на `CopyOnWriteArrayList` или `ConcurrentLinkedQueue`.
