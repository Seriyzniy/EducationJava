
> [!NOTE] ⭐Каждая коллекция ДАЕТ СВОЮ РЕАЛИЗАЦИЮ `Iterator` или `ListItrator`⭐
> Чтобы появилась возможность доступа к коллекции через итератор, его необходимо получить. 
> 
> Каждый из классов коллекций ==предоставляет метод== `iterator()`, который ==возвращает итератор== для начала коллекции.

## Общий порядок действий
1. ==Получить итератор== для начала коллекции, вызвав метод `iterator()` коллекции. 
2. ==Организовать цикл==, внутри которого вызывается метод `hasNext()` . Обеспечить выполнение цикла, пока `hasNext()` возвращает `true`. 
3. ==Получать в цикле элементы== с помощью вызова `next(`).

> [!NOTE] 💡Для коллекций, реализующих интерфейс `List`, получить итератор можно также, вызвав метод `listIterator()`
> 
> В остальном `ListIterator` применяется точно так же, как `Iterator`.
> 

***
# Пример `Iterator`
```java
List<String> list = Arrays.asList("Java", "C++", "Python");

// ✅Получили итератор
Iterator<String> it = list.iterator();

// ✅Организация цикла, пока hasNext == true
while (it.hasNext()) {
    String lang = it.next(); //✅Получение элемента
    
    System.out.println(lang);
    // it.remove(); // удалить можно, но только если список поддерживает remove()
}
```

# Пример `ListIterator`
```java
List<String> list = new ArrayList<>(Arrays.asList("Java", "C++", "Python"));

// ✅Получили итератор
ListIterator<String> it = list.listIterator();

// ✅Организация цикла, пока hasNext == true
while (it.hasNext()) {
    String lang = it.next();//✅Получение элемента
    
    if (lang.equals("C++")) {
        it.set("C#");   // заменить C++ на C#
        it.add("Go");   // добавить Go сразу после C#
    }
}

// ✅ Обходим в обратном порядке
while (it.hasPrevious()) {
    System.out.println(it.previous());
}
```
