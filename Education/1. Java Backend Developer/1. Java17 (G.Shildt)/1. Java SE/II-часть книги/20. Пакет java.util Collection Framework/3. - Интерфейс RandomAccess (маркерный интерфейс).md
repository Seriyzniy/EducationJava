`RandomAccess` — это ==**маркерный интерфейс**== (в нём нет методов), введённый в Java 2 (JDK 1.4).  

Он ==сигнализирует==, что коллекция (`List`) ==поддерживает **быстрый произвольный доступ**== к элементам по индексу (т.е. `list.get(i)` работает за **O(1)**).

```java
public interface RandomAccess { }
```

## Какие коллекции его реализуют?
- `ArrayList` ✅
- `Vector` ✅
- `CopyOnWriteArrayList` ✅

❌ Не реализуют:
- `LinkedList` (потому что `get(i)` там **O(n)**).

## 💡Особенности и подводные камни💡
- 🔶 Это **маркер**, поэтому ==сам по себе он ничего не делает==.
    
- 🔶 Нужен НЕ программисту, а **алгоритмам в JDK**, чтобы понимать, как эффективнее работать с коллекцией.
    
- 🔶 Если ты пишешь свою коллекцию на основе массива → стоит реализовать `RandomAccess`.
    
- 🔶 Если коллекция на основе списка (например, `LinkedList`) → реализовывать `RandomAccess` нельзя, иначе алгоритмы будут работать неэффективно (подумают, что доступ O(1), а он O(n)).

***
## ✅ Best Practices ✅
- Когда пишешь метод, принимающий `List`, и планируешь часто обращаться по индексу → проверяй:
```java
if (list instanceof RandomAccess) {
	// ✅Если есть доступ по индексу
    // Быстрый доступ по индексу
    for (int i = 0; i < list.size(); i++) {
        process(list.get(i));
    }
} else {
    // ✅❗Медленный список → лучше использовать Iterator
    for (Iterator<?> it = list.iterator(); it.hasNext();) {
        process(it.next());
    }
}
```
