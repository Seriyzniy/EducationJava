
> [!NOTE] `EnumSet<E extends Enum<E>>` — это специализированная реализация интерфейса `Set` ==для работы **только с перечислениями (enum)**==.
> Класс `EnumSet` расширяет `AbstractSet` и реализует интерфейс `Set`. 
> Он предназначен для использования с объектами типа enum

⭐⭐Под капотом используется **битовая маска** (bit vector), что делает её чрезвычайно быстрой и компактной по памяти.  

Операции выполняются за **O(1)** (практически за одно машинное слово), так как каждый элемент соответствует одному биту.

## Зачем нужен, если есть enum?
1. **Наборы значений, а не одно значение**  
`enum` сам по себе — это одна константа. Например:
```java
enum Day { MONDAY, TUESDAY, WEDNESDAY }

Day today = Day.MONDAY; // хранит только одно значение
```

Но что если нужно ==хранить **несколько дней сразу**== (например, расписание работы)?  
Можно было бы использовать `Set<Day>`:
```java
Set<Day> workDays = new HashSet<>();
workDays.add(Day.MONDAY);
workDays.add(Day.WEDNESDAY);
```

> [!NOTE] ⭐Но `EnumSet` делает то же самое, **но гораздо быстрее и эффективнее**.⭐
> 

## Битовая маска
Допустим есть такой enum
```java
enum Day { MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY }
```

👉 Здесь **7 элементов**, значит, `EnumSet<Day>` может быть представлен **одним числом `long` (64 бита)**, где:
- `MONDAY` → бит №0
- `TUESDAY` → бит №1
- `WEDNESDAY` → бит №2
- …
- `SUNDAY` → бит №6

Добавляем один элемент
```java
EnumSet<Day> set = EnumSet.of(Day.MONDAY);
```
В памяти (`long`):
```java
0000001
```

Добавляем ещё элемент
```java
set.add(Day.WEDNESDAY);
```
Теперь в `long`:
```java
0000101
```

## Конструкторы

⚠️ `EnumSet` — ==абстрактный класс==. Непосредственно через `new` его создать нельзя, нужно использовать **фабричные методы** (`of`, `range`, `allOf`, `noneOf`, `copyOf`).
```java
// Создаёт 💡пустое множество💡 для указанного enum-типа
public static <E extends Enum<E>> EnumSet<E> noneOf(Class<E> elementType)

// Создаёт множество, содержащее все элементы указанного 💡enum-типа💡
public static <E extends Enum<E>> EnumSet<E> allOf(Class<E> elementType)

// Создаёт множество, содержащее элементы 💡в указанном диапазоне💡 (от first до last включительно)
public static <E extends Enum<E>> EnumSet<E> range(E from, E to)

// Создаёт множество, 💡содержащее те же элементы💡, что и указанный EnumSet
public static <E extends Enum<E>> EnumSet<E> copyOf(EnumSet<E> s)

// Создаёт 💡множество из другой коллекции💡 (коллекция должна содержать 💡элементы одного enum-типа💡)
public static <E extends Enum<E>> EnumSet<E> copyOf(Collection<E> c)

// Создаёт множество, содержащее указанные элементы
public static <E extends Enum<E>> EnumSet<E> of(E e)
public static <E extends Enum<E>> EnumSet<E> of(E e1, E e2)
public static <E extends Enum<E>> EnumSet<E> of(E e1, E e2, E e3)
public static <E extends Enum<E>> EnumSet<E> of(E e1, E e2, E e3, E e4)
public static <E extends Enum<E>> EnumSet<E> of(E e1, E e2, E e3, E e4, E e5)
public static <E extends Enum<E>> EnumSet<E> of(E first, E... rest)
```

## Методы

`EnumSet` наследует методы от `AbstractSet`, `Set`, `Collection`.
```java
// Добавление элемента
public boolean add(E e)

// Добавление всех элементов другой коллекции
public boolean addAll(Collection<? extends E> c)

// Удаление элемента
public boolean remove(Object o)

// Проверка наличия
public boolean contains(Object o)

// Удаляет все элементы
public void clear()

// Размер множества
public int size()

// Пусто ли множество
public boolean isEmpty()

// Возвращает массив
public Object[] toArray()
public <T> T[] toArray(T[] a)

// Итератор по элементам (в порядке объявления в enum)
public Iterator<E> iterator()

// Дополнительно от самого EnumSet (операции над множествами):
public void complementOf(EnumSet<E> s) // дополнение множества (инверсия битовой маски)

```

***
## 💡 Особенности и подводные камни 💡

- 🔶💡**Только для enum** — нельзя использовать с другими типами.
    
- 🔶💡**Под капотом битовая маска**:
    - если в enum ≤ 64 элементов — хранится в `long` (SingleEnumSet),
        
    - если > 64 — используется массив `long[]` (JumboEnumSet).
        
- 🔶**Невероятно быстрая работа**: операции `add`, `remove`, `contains` работают через битовые операции.
    
- 🔶**Null запрещён** — при попытке вставки `null` будет `NullPointerException`.
    
- 🔶**Порядок хранения = порядок объявления в enum**.
    
- 🔶**Не потокобезопасен**, как и большинство коллекций.

***
## ✅Best Practices✅ 

- Используйте `EnumSet`, если нужно работать с множеством значений одного enum. Это **самый эффективный Set** по памяти и скорости.
    
- Никогда не используйте `HashSet<Enum>` — он медленнее и прожорливее.
    
- Хорош для флагов и состояний (альтернатива `Enum.ordinal()` + битовые маски руками).
    
- Для потокобезопасности можно обернуть через `Collections.synchronizedSet(EnumSet.of(...))`.
    
- Для хранения всех значений — используйте `EnumSet.allOf(MyEnum.class)`.

***

## Пример

```java
import java.util.*;

enum Day {
    MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY
}

public class EnumSetExample {
    public static void main(String[] args) {
        // Создать множество рабочих дней
        EnumSet<Day> workDays = EnumSet.range(Day.MONDAY, Day.FRIDAY);
        System.out.println("Рабочие дни: " + workDays); // [MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY]

        // Выходные
        EnumSet<Day> weekend = EnumSet.of(Day.SATURDAY, Day.SUNDAY);
        System.out.println("Выходные: " + weekend); // [SATURDAY, SUNDAY]

        // Все дни
        EnumSet<Day> allDays = EnumSet.allOf(Day.class);
        System.out.println("Все дни: " + allDays);

        // Проверка принадлежности
        System.out.println("Пятница рабочая? " + workDays.contains(Day.FRIDAY)); // true

        // Удаление
        workDays.remove(Day.FRIDAY);
        System.out.println("Рабочие без пятницы: " + workDays); // [MONDAY, TUESDAY, WEDNESDAY, THURSDAY]

        // Дополнение (инверсия)
        EnumSet<Day> notWorkDays = EnumSet.complementOf(workDays);
        System.out.println("Не рабочие: " + notWorkDays);
    }
}

```
