
> [!NOTE] ⭐`Spliterator<E>` (splitable iterator) — `интерфейс`, похожий на `Iterator`
> Но `с возможностью РАЗДЕЛЯТЬ (split) КОЛЛЕКЦИЮ на части` для параллельной обработки (например, в `Stream API`).

```java
public interface Spliterator<T> {
    // ✅выполняет действие для следующего элемента✅, двигаясь вперёд, 
    // возвращает false если элементов больше нет
    boolean tryAdvance(Consumer<? super T> action); 
    

	 // пытается разделить элементы ❗❗пополам❗❗, возвращает новый Spliterator для второй части 
    // или null, если больше делить некуда
    Spliterator<T> trySplit(); 
   

	// оценка количества оставшихся элементов
    long estimateSize(); 
    

	// 💡битовые флаги характеристик (ORDERED, DISTINCT, SORTED,SIZED,...)
    int characteristics(); 
    
	// применяет действие ко всем оставшимся элементам
    default void forEachRemaining(Consumer<? super T> action); 
    
    
    // Возвращает компаратор, используемый вызывающим сплитератором, или null в случае применения естественного упоря дочения.
    // 💡Если последовательность не упорядочена💡, тогда генерируется исключение 💡IllegalStateException💡
    default Comparator<? super T> getComparator()
    
    // 💡Возвращает количество элементов, которые осталось подвергнуть итерации
    // Если для вызывающего сплитератора установлен размер, или -1 в противном случае
    default long getExactSizeIfKnown()
    
    // Возвращает true, если вызывающий сплите ратор имеет характеристики, переданные в val, или false в противном случае
    default boolean hasCharacteristics int val)
}
```

Тем не менее, реализацию интерфейса `Spliterator` можно применять, даже если не планируется использовать параллельное выполнение. 

Одна из причин заключается в том, что `Spliterator` ==предлагает оптимизированный подход==, который объединяет операции, реализуемые методами `hasNext()` и `next()`, ==в один метод.==

## Характеристики (bitmask)

`Spliterator` возвращает `int characteristics()`, где каждая константа — это **битовый флаг**:
- `ORDERED` – порядок элементов фиксирован
- `DISTINCT` – элементы уникальны
- `SORTED` – элементы отсортированы
    
- `SIZED` – известен точный размер
    
- `NONNULL` – элементы не равны `null`
    
- `IMMUTABLE` – коллекция не изменяема
- `CONCURRENT` – безопасно для многопоточного обхода
    
- `SUBSIZED` – если `Spliterator` разбить, то все части тоже знают свой размер

Пример: `ArrayList.spliterator()` → вернёт `ORDERED | SIZED | SUBSIZED`.

## Вложенные интерфейсы для работы с примитивами
Есть несколько вложенных интерфейсов `Spliterator`, которые предназначены для использования с примитивными типами `double`, `int` и `long` и называются `Spliterator.OfDouble`, `Spliterator.OfInt` и `Spliterator.OfLong`.

***
## 💡 Особенности и подводные камни💡

- В отличие от `Iterator`, `Spliterator` может **параллельно обрабатывать коллекцию** (ключ для `parallelStream()`).
    
- `trySplit()` может возвращать `null`, если делить больше нельзя (например, уже один элемент).
    
- Работает лучше всего, когда коллекцию можно делить на равные части (например, массив или `ArrayList`).
    
- ⭐⭐Для ==связных структур== (`LinkedList`) ==деление неэффективно== → параллельные стримы будут медленнее.

***
## ✅ Best Practices ✅

- ⭐⭐Используй `spliterator()` вместо `iterator()`, если работаешь со `Stream API`.⭐⭐
    
- Для своих коллекций можно реализовать кастомный `Spliterator`, чтобы эффективно делить данные для параллельных стримов.
    
- ⭐🧠Если нужна просто итерация → бери `Iterator`, не усложняй.

***
## Пример
### Последовательный обход
```java
List<String> list = Arrays.asList("Java", "C++", "Python");

Spliterator<String> spliterator = list.spliterator();

while (spliterator.tryAdvance(System.out::println)) {
    // печатает по одному элементу
}
```

### Разделение
```java
List<Integer> numbers = Arrays.asList(1,2,3,4,5,6,7,8);

// ✅ Новый Spliterator на каждое разделение ✅
Spliterator<Integer> spl1 = numbers.spliterator();
Spliterator<Integer> spl2 = spl1.trySplit(); // делим коллекцию

System.out.println("Первая часть:");
spl1.forEachRemaining(System.out::println);

System.out.println("Вторая часть:");
spl2.forEachRemaining(System.out::println);
```
