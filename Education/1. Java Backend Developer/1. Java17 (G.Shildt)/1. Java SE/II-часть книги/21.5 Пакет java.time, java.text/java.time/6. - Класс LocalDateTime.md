
> [!NOTE] Представляет `дату и время` `без часового пояса`.
> Иммутабельный, потокобезопасный.  
> Реализует:  `Temporal`, `TemporalAdjuster`, `Comparable<LocalDateTime>`, `Serializable`.

## Фабричные методы:
```java
public static LocalDateTime now();
public static LocalDateTime now(ZoneId zone);
public static LocalDateTime of(int year, int month, int day, int hour, int minute);
public static LocalDateTime of(LocalDate date, LocalTime time);
public static LocalDateTime from(TemporalAccessor temporal);
```

---
## 💡 Особенности и подводные камни 💡

1️⃣ Не хранит информации о часовом поясе.  

2️⃣ ⭐⭐Для преобразования в `Instant` нужен `ZoneOffset`.  

3️⃣ Потокобезопасный.  

4️⃣ ⭐Не путай с `ZonedDateTime` — тот хранит часовой пояс.

---

## 💎 Best Practices 💎
✅ Используй для бизнес-логики, где важна дата/время, но не часовой пояс.  

✅ Для глобальных систем (UTC) используй `Instant`.  

✅ Для локализации времени — `ZonedDateTime`.

---
## Пример
```java
LocalDateTime meeting = LocalDateTime.of(2025, 10, 8, 14, 0);
System.out.println("Встреча: " + meeting);

Instant utc = meeting.toInstant(ZoneOffset.ofHours(3));
System.out.println("UTC-время: " + utc);
```
