Мощный инструмент ==для форматирования чисел, строк, дат и времени== (аналог `printf` в C/C++)

Он ==сохраняет форматированный текст в буфере==, содержимое которого может быть в любой момент получено

Поддерживает 
- локали, 
- флаги форматирования, 
- выравнивание, 
- ширины полей, 
- спецификаторы для даты/времени (`%t` + суффиксы) и пр.

> [!NOTE] ✅ Советы ✅
> - `Formatter` очень гибок; если нужно локализованное представление — передайте `Locale`.
> - Для простых задач часто хватает `String.format(...)` (внутри использует `Formatter`).

## Конструкторы

В параметре `buf` указывается `буфер для форматированного вывода`. 
Если `buf` имеет значение `null`, тогда `Formatter` автоматически выделяет память под `StringBuilder`

```java
Formatter()
// Создаёт объект, выводящий в StringBuilder.

Formatter(Appendable buf)
// Создаёт объект для вывода в указанный Appendable (например, StringBuilder).

Formatter(Appendable buf, Locale l)
// То же, но с указанием локали.

Formatter(String fileName) throws FileNotFoundException
// Создаёт объект для записи в файл.

Formatter(String fileName, String charset) throws FileNotFoundException, UnsupportedEncodingException
// Создаёт объект для записи в файл с указанной кодировкой.

Formatter(String fileName, String charset, Locale l)
// Создаёт объект для записи в файл с локалью.
```

## Методы
```java
Formatter format(String format, Object... args)
// Форматирует строку по шаблону и добавляет в поток.

Formatter format(Locale l, String format, Object... args)
// Форматирует с учётом локали.



void flush()
// Сбрасывает буфер.



void close()
// ❗❗Закрывает Formatter.



IOException ioException()
// Возвращает последнюю ошибку ввода/вывода.

Locale locale()
// Возвращает текущую локаль.

Appendable out()
// Возвращает объект, куда выводит Formatter.

String toString()
// Возвращает строку, если вывод был в StringBuilder.
```

## Указание минимальной ширины поля 
Целое число, помещенное между знаком `%` и кодом преобразования формата, действует как спецификатор минимальной ширины поля

## Закрытие объекта Formatter 
Как правило, после окончания эксплуатации объект ==Formatter должен быть закрыт==, что освободит все ресурсы, которые он использовал. 

Это особенно важно при форматировании во время записи в файл, но может быть важно и в других случаях.

Тем не менее, класс Formatter также реали зует интерфейс AutoCloseaЫe, т.е. поддерживает оператор try с ресурсами.

## Пример
```java
import java.util.Formatter;
import java.util.Locale;

public class FormatterExample {
    public static void main(String[] args) {
        Formatter f = new Formatter();
        f.format("Число: %d, Дробное: %.2f", 42, 3.14159);
        System.out.println(f.toString());
        f.close()

        // Локализованный вывод
        Formatter f2 = new Formatter(Locale.FRANCE);
        f2.format("%,.2f", 12345.67);
        System.out.println("Французский формат: " + f2.toString());
        f2.close();
    }
}

```
