# 📕 1. Абстрактный класс TimeZone

`TimeZone` — абстрактный тип для смещения относительно UTC (GMT) и вычисления летнего времени;

## Конструкторы
```java
// TimeZone — абстрактный, используется фабрика:
static TimeZone getTimeZone(String id)
// Возвращает временную зону по ID (например, "Europe/Moscow").

static TimeZone getDefault()
// Возвращает временную зону по умолчанию.

static String[] getAvailableIDs()
// Все доступные ID временных зон.
```

## Методы
```java
int getRawOffset()
// Смещение от UTC в миллисекундах (без учёта летнего времени).

boolean inDaylightTime(Date date)
// true, если для этой даты действует летнее время.



int getOffset(long date)
// Смещение UTC с учётом летнего времени.

boolean useDaylightTime()
// true, если в этой зоне используется переход на летнее время.



String getID()
// Возвращает строковый ID зоны.

void setID(String ID)
// Устанавливает ID.



static void setDefault(TimeZone zone)
// Устанавливает временную зону по умолчанию.
```

## Пример
```java
import java.util.TimeZone;
import java.util.Date;

public class TimeZoneExample {
    public static void main(String[] args) {
        TimeZone tz = TimeZone.getTimeZone("Europe/Moscow");
        
        System.out.println("ID: " + tz.getID());
        System.out.println("Смещение (часы): " + tz.getRawOffset() / 3600000);

        Date now = new Date();
        System.out.println("Летнее время сейчас? " + tz.inDaylightTime(now));
    }
}

```

# 📕 2. `SimpleTimeZone` -  подкласс

`SimpleTimeZone` — удобный подкласс для григорианского календаря, позволяющий задать правила DST.

## Конструкторы
```java
// SimpleTimeZone:
SimpleTimeZone(int rawOffset, String ID)
// Создаёт зону с фиксированным смещением от UTC.

SimpleTimeZone(
	int rawOffset, 
	String ID, 
    int startMonth, 
    int startDay, 
    int startDayOfWeek, 
    int startTime,
    int endMonth, 
    int endDay, 
    int endDayOfWeek, 
    int endTime)
// Создаёт зону с правилами перехода на летнее время.
```
