Представляет локаль (язык, страна, вариант), ==используется другими классами== (`Calendar`, `DateFormat`, `ResourceBundle`, `Formatter`) ==для выбора форматов и переводов==.

> [!NOTE] ⭐ Примечание ⭐
> С JDK 7–9 расширены возможности (BCP 47, LDML, `Builder`, `toLanguageTag()` и т.д.).

## Конструкторы
```java
Locale(String language)
// Локаль по коду языка (например, "en").

Locale(String language, String country)
// Локаль по коду языка и страны (например, "en", "US").

Locale(String language, String country, String variant)
// Локаль с вариантом (например, для особых региональных различий).
```

## Методы
```java
String getLanguage()
// Код языка ("en", "ru").

String getCountry()
// Код страны ("US", "RU").

String getVariant()
// Вариант.



String getDisplayLanguage()
// Название языка на языке текущей локали.

String getDisplayCountry()
// Название страны на языке текущей локали.

String getDisplayVariant()
// Название варианта.



String toLanguageTag()
// Возвращает тег BCP 47 (например, "en-US").

static Locale getDefault()
// Локаль по умолчанию.

static void setDefault(Locale newLocale)
// Устанавливает локаль по умолчанию.
```

## Пример
```java
import java.util.Locale;

public class LocaleExample {
    public static void main(String[] args) {
        Locale locale = new Locale("ru", "RU");
        
        System.out.println("Язык: " + locale.getLanguage());
        System.out.println("Страна: " + locale.getCountry());
        System.out.println("Полное: " + locale.toLanguageTag());

        Locale us = Locale.US;
        System.out.println("US язык: " + us.getDisplayLanguage());
        System.out.println("US страна: " + us.getDisplayCountry());
    }
}
```
