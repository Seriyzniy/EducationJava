
> [!NOTE] `Optional`
> Безопасная ==работа с возможным отсутствием значения==
> - `Optional`, 
> - `OptionalDouble`, `OptionalInt`, `OptionalLong`
> 
> Может возникнуть ситуация, когда вместо объекта, получили `null`, и если попытаться вызвать метод/поле этого объекта - получим Исключение
> `Optional` предлагает элегантное решение этой проблемы

Обёртки-«`optional`» для значений, позволяющие избегать распространённых `null`-проверок. 

- ⭐`Optional<T>` — общий вариант; 
- ⭐Cпециализированные версии ==для примитивов== (`int`, `long`, `double`) избегают бокса
	- Таким образом, в них определены методы `getAsDouble`() , `getAsInt`() и `getAsLong` , а не `get` . 
	- Кроме того, они не поддерживают методы `filter`() , `ofNullable`( ) ,`map`() , `flatMap`( ) и `or`( ) .

## Замечания и практики:
- `Optional` ==хорош для возвращаемых значений API== (явно сигнализирует о том, что значение может отсутствовать).
    
- ==Не== рекомендуется ==хранить== `Optional` ==в полях класса или коллекциях== (в книге не запрещают, но обычно это считается анти-паттерном). Для примитивов используйте специализированные `OptionalInt`/`OptionalLong`/`OptionalDouble` (они дают `getAsInt()` и т.п.).

## Конструкторы
В классе `Optional` конструкторы не определены, но есть несколько методов, позволяющих работать с объектами Optional
```java
// ⭐⭐Конструкторов напрямую нет⭐⭐, используются фабричные методы:
static <T> Optional<T> empty()
// Возвращает пустой Optional.

static <T> Optional<T> of(T value)
// Создаёт Optional с непустым значением value (если value == null → NPE).

static <T> Optional<T> ofNullable(T value)
// Создаёт Optional, допускающий null (если value == null → Optional.empty()).
```

## Методы
```java
boolean isPresent()
// 💡 true, если значение присутствует.

boolean isEmpty()
// 💡 true, если значение отсутствует.


T get()
// Возвращает значение. Если пусто → NoSuchElementException.

Optional<T> or(Supplier<? extends Optional<? extends T> func)
// Если в вызывающем объекте отсутствует ка кое-либо значение, тогда вызывает func

T orElse(T other)
// Возвращает значение, если есть, иначе other.

T orElseGet(Supplier<? extends T> supplier)
// Возвращает значение, если есть, иначе supplier.get().

<T extends Throwable> T orElseThrow()
// Бросает NoSuchElementException, если пусто.



<U> Optional<U> map(Function<? super T, ? extends U> mapper)
// Преобразует значение, если оно присутствует.

<U> Optional<U> flatMap(Function<? super T, Optional<U>> mapper)
// Как map(), но mapper возвращает Optional (избегает вложенности).

Optional<T> filter(Predicate<? super T> predicate)
// Возвращает Optional, если условие выполняется.



void ifPresent(Consumer<? super T> action)
// Выполняет действие, если значение присутствует.

void ifPresentOrElse(Consumer<? super T> action, Runnable emptyAction)
// Выполняет действие, если значение есть, иначе выполняет emptyAction.



Stream<T> stream()
// Возвращает Stream из 0 или 1 элементов.
```

## Пример
```java
import java.util.Optional;

public class OptionalExample {
    public static void main(String[] args) {
        Optional<String> opt = Optional.ofNullable(null);

        String result = opt.orElse("Значение по умолчанию");
        System.out.println(result); // "Значение по умолчанию"

        Optional<String> name = Optional.of("Java");
        name.ifPresent(n -> System.out.println("Привет, " + n));
    }
}
```
