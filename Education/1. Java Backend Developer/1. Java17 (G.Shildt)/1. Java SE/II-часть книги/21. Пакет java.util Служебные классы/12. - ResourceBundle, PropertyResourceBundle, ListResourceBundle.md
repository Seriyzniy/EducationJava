
> [!NOTE] ⭐Позволяют управлять коллекцией `чувствительных к лакали ресурсов`⭐
> `ResourceBundle`, `PropertyResourceBundle`, `ListResourceBundle` -  комплекты ресурсов и локализация

Поддерживает загрузку локализованных ресурсов (строки/объекты) по семейству имён и `Locale`. 
Позволяет иметь разные локализованные версии (например, `SampleRB` и `SampleRB_de`).

***
# 📕 1. ResourceBundle
## Конструкторы
❗❗❗ Абстрактный класс
```java
// Экземпляры создаются фабричными методами.

static ResourceBundle getBundle(String baseName)
// Загружает ресурсный пакет по базовому имени и локали по умолчанию.

static ResourceBundle getBundle(String baseName, Locale locale)
// Загружает для указанной локали.
```

## Методы
```java
/**
 * КЛАСС ResourceBundle
 * Основные методы для работы с ресурсными пакетами (интернационализация)
 */

// ==================✅ УПРАВЛЕНИЕ КЕШЕМ ✅ =================
/**
 * Удаляет из кеша все комплекты ресурсов, которые были загружены загрузчиком классов.
 * Начиная с JDK 9, удаляет все комплекты ресурсов, загруженные модулем, из которого вызывается метод
 */
static final void clearCache()

/**
 * Удаляет из кеша все комплекты ресурсов, которые были загружены указанным загрузчиком классов
 */
static final void clearCache(ClassLoader ldr)


// =============✅ ПОЛУЧЕНИЕ РЕСУРСНЫХ ПАКЕТОВ ✅=============
/**
 * Загружает комплект ресурсов с указанным именем семейства, используя стандартную локаль
 * @throws MissingResourceException если нет доступных комплектов ресурсов с указанным именем
 */
static final ResourceBundle getBundle(String familyName)

/**
 * Загружает комплект ресурсов с указанным именем семейства для указанного модуля, используя стандартную локаль
 * @throws MissingResourceException если нет доступных комплектов ресурсов с указанным именем
 */
static ResourceBundle getBundle(String familyName, Module mod)

/**
 * Загружает комплект ресурсов с указанным именем семейства для указанной локали
 * @throws MissingResourceException если нет доступных комплектов ресурсов с указанным именем
 */
static final ResourceBundle getBundle(String familyName, Locale loc)

/**
 * Загружает комплект ресурсов с указанным именем семейства для указанной локали и модуля
 * @throws MissingResourceException если нет доступных комплектов ресурсов с указанным именем
 */
static ResourceBundle getBundle(String familyName, Locale loc, Module mod)

/**
 * Загружает комплект ресурсов с указанным именем семейства для указанной локали и загрузчика классов
 * @throws MissingResourceException если нет доступных комплектов ресурсов с указанным именем
 */
static ResourceBundle getBundle(String familyName, Locale loc, ClassLoader ldr)

/**
 * Загружает комплект ресурсов с указанным именем семейства, используя стандартную локаль и указанный контроллер
 * @throws MissingResourceException если нет доступных комплектов ресурсов с указанным именем
 */
static final ResourceBundle getBundle(String familyName, ResourceBundle.Control cntl)

/**
 * Загружает комплект ресурсов с указанным именем семейства для указанной локали и контроллера
 * @throws MissingResourceException если нет доступных комплектов ресурсов с указанным именем
 */
static final ResourceBundle getBundle(String familyName, Locale loc, ResourceBundle.Control cntl)

/**
 * Загружает комплект ресурсов с указанным именем семейства для указанной локали, загрузчика классов и контроллера
 * @throws MissingResourceException если нет доступных комплектов ресурсов с указанным именем
 */
static ResourceBundle getBundle(String familyName, Locale loc, ClassLoader ldr, ResourceBundle.Control cntl)


// ===========✅ ИНФОРМАЦИЯ О РЕСУРСНОМ ПАКЕТЕ ✅============
/**
 * Возвращает базовое имя комплекта ресурсов, если оно доступно, или null в противном случае
 */
String getBaseBundleName()

/**
 * Возвращает локаль, поддерживаемую комплектом ресурсов
 */
Locale getLocale()

/**
 * Проверяет, является ли указанная строка ключом внутри вызывающего комплекта ресурсов (или его родителя)
 */
boolean containsKey(String k)


// =============✅ ПОЛУЧЕНИЕ РЕСУРСОВ ✅==============
/**
 * Возвращает объект, ассоциированный с указанным ключом
 * @throws MissingResourceException если ключ отсутствует в комплекте ресурсов
 */
final Object getObject(String k)

/**
 * Возвращает строку, ассоциированную с указанным ключом
 * @throws MissingResourceException если ключ отсутствует в комплекте ресурсов
 * @throws ClassCastException если объект, ассоциированный с ключом, не является строкой
 */
final String getString(String k)

/**
 * Возвращает строковый массив, ассоциированный с указанным ключом
 * @throws MissingResourceException если ключ отсутствует в комплекте ресурсов
 * @throws ClassCastException если объект, ассоциированный с ключом, не является строковым массивом
 */
final String[] getStringArray(String k)


// ====================✅  РАБОТА С КЛЮЧАМИ ✅ ====================
/**
 * Возвращает ключи комплекта ресурсов в виде перечисления строк (включая ключи родителя)
 */
abstract Enumeration<String> getKeys()

/**
 * Возвращает ключи комплекта ресурсов в виде набора строк (включая ключи родителя)
 */
Set<String> keySet()

/**
 * Возвращает ключи комплекта ресурсов в виде набора строк (без ключей родителя)
 */
protected Set<String> handleKeySet()


// ================✅АБСТРАКТНЫЕ И ЗАЩИЩЕННЫЕ МЕТОДЫ ✅=================

/**
 * Возвращает объект, ассоциированный с указанным ключом (защищенный абстрактный метод)
 * Возвращает null, если ключ отсутствует в комплекте ресурсов
 */
protected abstract Object handleGetObject(String k)

/**
 * Устанавливает родительский комплект для текущего комплекта ресурсов
 * При поиске ключа будет выполняться поиск в родителе, если ключ не найден в текущем комплекте
 */
protected void setParent(ResourceBundle parent)
```

# 📕 2. PropertyResourceBundle
## Конструкторы
```java
PropertyResourceBundle(InputStream stream) throws IOException
// Создаёт из InputStream (❗❗файл *.properties❗❗).

PropertyResourceBundle(Reader reader) throws IOException
// Создаёт из Reader.
```

## Методы
Наследует методы `ResourceBundle` (`getString`, `getObject` и т.д.).

## Пример
Создадим файл `Messages.properties`:
```properties
greeting=Привет
farewell=Пока
```

```java
import java.util.*;

public class ResourceBundleExample {
    public static void main(String[] args) {
        ResourceBundle bundle = ResourceBundle.getBundle("Messages", new Locale("ru", "RU"));

        System.out.println(bundle.getString("greeting")); // "Привет"
        System.out.println(bundle.getString("farewell")); // "Пока"
    }
}
```

# 📕 3. ListResourceBundle
## Конструкторы
❗❗❗ Абстрактный класс,
```java
//  необходимо наследовать.
protected ListResourceBundle()
```

## Методы
```java
protected abstract Object[][] getContents()
// Возвращает двумерный массив ключ-значение вида:
// { {"key1", "value1"}, {"key2", "value2"} }

Object handleGetObject(String key)
// Возвращает объект по ключу.

Enumeration<String> getKeys()
// Все ключи.
```

## Пример
```java
import java.util.*;

public class MyResources extends ListResourceBundle {
    @Override
    protected Object[][] getContents() {
        return new Object[][] {
            {"greeting", "Hello"},
            {"farewell", "Goodbye"}
        };
    }
}

class ListRBExample {
    public static void main(String[] args) {
        ResourceBundle bundle = ResourceBundle.getBundle("MyResources", Locale.getDefault());
        System.out.println(bundle.getString("greeting")); // "Hello"
    }
}
```
