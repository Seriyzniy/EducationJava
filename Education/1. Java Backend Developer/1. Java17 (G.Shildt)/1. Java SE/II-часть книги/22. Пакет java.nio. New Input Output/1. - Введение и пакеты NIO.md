Начиная с версии 1.4, в Java предлагается вторая система ввода-вывода под названием NIO (New I/0 - новый ввод-вывод). 

Она ==поддерживает ориентированный на БУФЕРЫ и основанный на КАНАЛАХ подход== к выполнению операций ввода-вывода.

В выпуске JDK 7 система NIO значительно расширилась, обеспечив усовершенствованную поддержку функциональных средств для обработки файлов и взаимодействия с файловой системой. 
На самом деле изменения были настолько значительными, что часто применяется тер мин NIO.2.


> [!NOTE] 🧠🧠 Мост между `java.io` и `java.nio` 🧠🧠
> Когда в Java добавили NIO (Java 1.4), старые классы из `java.io` не стали переписывать с нуля. 
> 
> Вместо этого сделали «мост»:
> - К некоторым объектам ввода-вывода (`FileInputStream`, `FileOutputStream`, `RandomAccessFile`, `Socket`, `ServerSocket`, `DatagramSocket`) ==добавили метод== **`getChannel()`**.
> - Этот метод возвращает объект из пакета `java.nio.channels` (`FileChannel`, `SocketChannel`, `ServerSocketChannel`, `DatagramChannel`).

```java
// Старый добрый java.io.RandomAccessFile
FileOutputStream fos = new FileOutputStream("D:\\test.txt");

// Новый объект NIO — FileChannel
FileChannel channel = fos.getChannel();

// Работаем уже с буферами
ByteBuffer buffer = ByteBuffer.allocate(32);
buffer.put("Hello NIO".getBytes());
buffer.flip();

channel.write(buffer);

channel.close();
fos.close();
```

# Основные отличия между Java IO и Java NIO
| IO                                  | NIO                                    |
| ----------------------------------- | -------------------------------------- |
| Потокоориентированный               | Буфер-ориентированный                  |
| Блокирующий (синхронный) ввод/вывод | Неблокирующий (асинхронный) ввод/вывод |
|                                     | Селекторы                              |

# 📕 ⭐1. Основы NIO⭐

> [!NOTE] Система NIO построена на `ДВУХ` фундаментальных `элементах`
> - **Buffer** хранит данные; 
> - **Channel** — открытое подключение к файлу/сокету
> 
> ⭐В общем случае для использования системы NIO понадобится `получить канал к устройству` ввода-вывода `и создать буфер` для хранения данных. 
> 
> Затем нужно `работать с буфером`, по мере необходимости `выполняя ввод или вывод данных.`

# 📕 2. Пакеты NIO

Классы NIO содержатся в пакетах, кратко описанных в табл. 23.1. Начиная с версии JDK 9, все они находятся в модуле `java.base`.

| Пакет                       | Описание                                                                                                                                                 |
| --------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **java.nio**                | Классы верхнего уровня системы NIO. Инкапсулирует различные типы буферов, содержащих данные, с которыми работает система NIO.                            |
|                             |                                                                                                                                                          |
| **java.nio.channels**       | ==Поддерживает каналы==, которые по существу открывают подключения ввода-вывода.                                                                         |
| **java.nio.channels.spi**   | Поддерживает ==поставщики служб для каналов.==                                                                                                           |
|                             |                                                                                                                                                          |
| **java.nio.charset**        | Инкапсулирует ==наборы символов==. Также поддерживает кодировщики и декодировщики, которые преобразуют соответственно символы в байты и байты в символы. |
| **java.nio.charset.spi**    | Поддерживает ==поставщики служб для наборов символов==.                                                                                                  |
|                             |                                                                                                                                                          |
| **java.nio.file**           | Предоставляет ==поддержку для файлов==.                                                                                                                  |
| **java.nio.file.attribute** | Предоставляет поддержку для ==файловых атрибутов==.                                                                                                      |
| **java.nio.file.spi**       | Поддерживает ==поставщики служб для файловой системы==.                                                                                                  |
