
> [!NOTE] `IntBuffer` — это буфер для хранения и обработки 32-битных целых (`int`)
> - Наследуется от `Buffer`.
> - Предоставляет API ==для чтения/записи INT-значений.==
> - Может использоваться как _view buffer_ над `ByteBuffer` (`asIntBuffer()`), что часто применяется при парсинге бинарных протоколов или работе с нативными структурами.

## Конструкторы - нет конструкторов
У `IntBuffer` **нет публичных конструкторов**. Создание идёт через статические методы:
```java
// Создаёт новый буфер на куче (backed array int[])
public static IntBuffer allocate(int capacity)

// Оборачивает существующий массив int[]
public static IntBuffer wrap(int[] array)

// Оборачивает часть массива
public static IntBuffer wrap(int[] array, int offset, int length)
```

## 💡 Особенности и подводные камни 💡

1. **View Buffer**
    - Создать `IntBuffer` напрямую можно через `allocate`/`wrap`.
        
    - Но часто его делают из `ByteBuffer`:
```java
ByteBuffer bb = ByteBuffer.allocate(16).order(ByteOrder.LITTLE_ENDIAN);
IntBuffer ib = bb.asIntBuffer();
```
	- Здесь один и тот же массив байтов можно читать как ints.
	
2.  💡**Endianness**
    - При использовании `asIntBuffer()` важно учитывать порядок байт (`ByteOrder`).
        
    - По умолчанию — BIG_ENDIAN. Для кроссплатформенного кода лучше явно вызывать `.order(ByteOrder.nativeOrder())`.
        
3. **compact()**
    - Сдвигает все непрочитанные ints к началу → удобно при частичном чтении.
        
    - После compact нужно снова `flip()` перед чтением.
        
4. **slice()/duplicate()**
    - Разделяют данные, но имеют свои независимые `position/limit/mark`.
        
    - Изменения в данных одного — видны в другом.
5. ==РАЗМЕР В int'ах==
```java
ByteBuffer bb = ByteBuffer.allocate(16);

System.out.println(bb.capacity()); // 16 байт

System.out.println(bb.asIntBuffer().capacity()); // 4 int
```

## ✅ Best Practices ✅

- 💡Используй `allocateDirect()` через `ByteBuffer` → `asIntBuffer()` для высокопроизводительного I/O.
    
- 💡Не забывай про `ByteOrder`, особенно при работе с бинарными файлами и сетевыми протоколами.
    
- Проверяй `remaining()` перед записью или чтением массивов.
    
- Если нужно «склеивать» данные из кусков — используй `compact()`.
    
- ⚠️ Помни: `equals` и `compareTo` сравнивают **содержимое**, а не ссылку.

## Пример
```java
import java.nio.ByteBuffer;
import java.nio.IntBuffer;
import java.nio.ByteOrder;

public class IntBufferExample {
    public static void main(String[] args) {
        // Создаём IntBuffer напрямую
        IntBuffer intBuf = IntBuffer.allocate(5);

        // Записываем числа
        intBuf.put(10);
        intBuf.put(20);
        intBuf.put(30);

        // Готовим к чтению
        intBuf.flip();

        while (intBuf.hasRemaining()) {
            System.out.println("Value: " + intBuf.get());
        }

        // --- Теперь через ByteBuffer ---
        ByteBuffer bb = ByteBuffer
	        .allocate(16)
	        .order(ByteOrder.LITTLE_ENDIAN);
	        
        IntBuffer ib = bb.asIntBuffer();

        ib.put(100).put(200).put(300);
        ib.flip();

        while (ib.hasRemaining()) {
            System.out.println("Int from view: " + ib.get());
        }
    }
}
```
