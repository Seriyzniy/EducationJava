
> [!NOTE] Буфер для хранения `double` (64-бит с плавающей точкой).
> 

## Конструктор - нет конструкторов
```java
// Создаёт новый буфер в куче
public static DoubleBuffer allocate(int capacity)

// Оборачивает существующий массив double[]
public static DoubleBuffer wrap(double[] array)

// Оборачивает часть массива
public static DoubleBuffer wrap(double[] array, int offset, int length)
```

public final int remaining()

// проверяет, является ли буфер доступным только для чтения

// проверяет, поддерживает ли буфер доступ к базовому массиву
public abstract boolean hasArray()

public abstract double[] array()

// возвращает смещение в базовом массиве
public abstract int arrayOffset()

// проверяет, является ли буфер прямым (direct)
public abstract boolean isDirect()



// ✅ ЧТЕНИЕ
// читает double с текущей позиции и увеличивает позицию

public abstract double get(int index)

// копирует оставшиеся элементы в указанный массив
public DoubleBuffer get(double[] dst)

// копирует часть элементов в указанный массив с заданного смещения
public DoubleBuffer get(double[] dst, int offset, int length)



// ✅ ЗАПИСЬ
// записывает double в текущую позицию и увеличивает позицию

public DoubleBuffer put(double[] src)

// записывает часть элементов из массива с заданного смещения

## 💡 Особенности и подводные камни 💡

- `NaN` и `0.0/-0.0` при сравнении ведут себя так же, как в `FloatBuffer`.
	-❗❗`NaN` не равен **ничему**, даже самому себе!
    
- Каждый `double` занимает **8 байт**, поэтому `ByteBuffer.capacity()/8 == DoubleBuffer.capacity()`.
    
- `ByteOrder` критичен при создании через `asDoubleBuffer()`.
    

## ✅ Best Practices ✅
- Для научных вычислений: лучше direct + native order.
    
- Проверяй `remaining()` перед чтением/записью массивов.

## Пример
```java
DoubleBuffer db = DoubleBuffer.allocate(3);
db.put(3.14).put(2.71).flip();

while (db.hasRemaining()) {
    System.out.println(db.get());
}
```
