
> [!NOTE] ⭐БАЗОВЫЙ класс `Buffer`⭐
> Классы буферов определены в пакете `java.nio`. 
> 
> `Все буферы являются ПОДКЛАССАМИ` класса `Buffer`, который определяет основную функциональность, общую для всех буферов: 
> - текущую позицию, 
> - предел 
> - емкость.

🔸**Текущая позиция** - это ==индекс== в буфере, ==в котором будет выполняться следующая операция== чтения или записи. Текущая позиция перемещается после выполнения большинства операций чтения или записи. 

🔸**Пpедел** - представляет собой `значение индекса` ==за последним допустимым местоположением== в буфере. 

🔸**Емкость** -  задает ==количество элементов==, которые ==способен хранить буфер==. Часто предел равен емкости буфера.

# 📕 1. Абстрактный класс Buffer


> [!NOTE] `Buffer` — это `абстрактный класс` в пакете `java.nio`
> Лежит в основе всех буферов (`ByteBuffer`, `CharBuffer`, `IntBuffer` и т.д.). 
> 
> Он инкапсулирует **состояние позиции, предела (limit), ёмкости (capacity)** и дополнительные механизмы (`mark`) для работы с массивом примитивных данных.  
> 
> ==Используется вместе с **каналами== (`Channel`)**: вы читаете данные из канала в буфер или пишете из буфера в канал. 
> 
> 💡💡Главная идея: **вместо потоков (stream-oriented API) NIO ==оперирует буферами== (buffer-oriented API)**.

## Конструктор - нет конструкторов
`Buffer` — ==**абстрактный класс**==, напрямую его не создают.  
Конструктор `protected`, используется только подклассами (`ByteBuffer`, `CharBuffer`, …).
```java
// Protected конструктор, доступен только наследникам

protected Buffer(int mark, int pos, int lim, int cap) 
// mark  – сохранённая позиция (обычно -1, если не задана)
// pos   – текущая позиция (начинается с 0)
// lim   – предел (граница, до которой можно читать/писать)
// cap   – вместимость буфера (capacity), не меняется после создания
```

## Методы
Методы `Buffer` управляют состоянием (position, limit, capacity, mark) и позволяют переключаться между режимами записи/чтения.
```java
//✅ Основные свойства буфера
//Возвращает вместимость буфера (capacity) — максимальное количество элементов
public final int capacity()

//true, если буфер только для чтения (например, read-only slice)
public abstract boolean isReadOnly()

//true, если буфер direct (off-heap), иначе false
public abstract boolean isDirect()


//✅ Управление позицией и лимитом
//Возвращает текущую позицию
public final int position()

//Устанавливает текущую позицию (0 <= newPosition <= limit)
public final Buffer position(int newPosition)

//Возвращает текущий предел (limit)
public final int limit()

//Устанавливает предел (0 <= newLimit <= capacity)
public final Buffer limit(int newLimit)


//✅ Навигация по буферу
//Устанавливает "метку" в текущей позиции
public final Buffer mark()

//Возвращает позицию на ранее установленную mark (или бросает InvalidMarkException)
public final Buffer reset()

//position = 0, mark = -1 (переход к началу без изменения limit)
public final Buffer rewind()


//✅ Подготовка буфера к операциям
//position = 0, limit = capacity ❗❗(буфер готов к записи capacity элементов)❗❗
public final Buffer clear()

//limit = position, position = 0 (буфер готов к чтению)
public final Buffer flip()

//Переносит оставшиеся элементы к началу буфера, position = число оставшихся, limit = capacity
public abstract Buffer compact()


//✅ Проверка состояния
//Кол-во элементов между position и limit
public final int remaining()

//Есть ли ещё элементы (position < limit)?
public final boolean hasRemaining()


//✅ Работа с массивом
//true, если буфер основан на массиве (heap buffer)
public abstract boolean hasArray()

//Возвращает ссылку на массив (если есть, иначе UnsupportedOperationException)
public abstract Object array()

//Смещение начала данных в массиве (если есть backing array)
public abstract int arrayOffset()


//✅ Создание производных буферов
//Новый буфер поверх данных, начиная от текущей позиции до limit
public abstract Buffer slice()

//Дублирует буфер, разделяя данные, но с независимым состоянием position/limit/mark
public abstract Buffer duplicate()

//Создает новый буфер только для чтения, разделяющий данные
public abstract Buffer asReadOnlyBuffer()


//✅ Сравнение и идентификация
//Сравнивает содержимое буферов
public abstract boolean equals(Object ob)

//Возвращает хеш-код буфера
public abstract int hashCode()

//Возвращает строковое представление буфера
public abstract String toString()


//✅ Порядок байт (для ByteBuffer)
//Получить порядок байт буфера
public abstract ByteOrder order()

//Установить порядок байт буфера
public abstract Buffer order(ByteOrder bo)


//✅ Позиционирование и получение данных
//Возвращает объект для итерации по элементам буфера
public abstract Object array()

//Проверяет, доступен ли буфер в текущий момент
public abstract boolean isAccessible()

//Закрывает буфер и освобождает ресурсы
public abstract void close()
```

## 💡Особенности и подводные камни💡

1. 💡 **`capacity` НЕИЗМЕНЕН** → после создания буфера ==вместимость изменить нельзя.==
    
2. 💡 **`flip()` — ключевой метод**: без него вы часто будете получать «пустой» буфер при записи/чтении.
    - 🟢Перед чтением `flip()`.
    - 🟠Перед записью `clear()`.
        
3. 💡`flip()` vs `rewind()` vs `clear()`:
	- ❗`flip()` ставит `limit = position; position = 0` — используйте при переходе: запись в буфер → чтение из буфера.
	    
	- ❗`rewind()` только `position = 0` — полезно, когда limit уже установлен (например, при повторной записи тех же байт).
	    
	- ❗`clear()` делает `position = 0; limit = capacity` — готовит буфер к новой записи.
        
4. **Direct buffer (`isDirect() == true`)** — данные в off-heap памяти. Быстрее при I/O, но:
    - дороже при создании,
        
    - может удерживать память дольше (зависит от GC).

## ✅ Best Practices ✅

- 💡 **Flip after write, clear before write again**:
    - После записи в буфер вызывай `flip()`.
        
    - После чтения из буфера и подготовки к новой записи — `clear()`.
        
- Используй `compact()` для сохранения оставшихся данных при повторном чтении/дозаписи.
    
- 💡 Для большого числа операций — пулить/==переиспользовать буферы ==(особенно direct).
    
- ❗❗❗ Для многопоточности: `Buffer` **НЕ потокобезопасен**. Делайте защиту снаружи.
    
- Всегда проверяй `remaining()` перед `get()`/`put()`.

## Пример
```java
import java.nio.ByteBuffer;

public class BufferExample {
    public static void main(String[] args) {
        // Создаём буфер на 16 байт
        ByteBuffer buffer = ByteBuffer.allocate(16);

        // Записываем данные (put сдвигает position)
        buffer.put((byte) 10);
        buffer.put((byte) 20);
        buffer.put((byte) 30);

        // Подготовка к чтению: flip (limit = position, position = 0)
        buffer.flip();

        while (buffer.hasRemaining()) {
            byte value = buffer.get();
            System.out.println("Value: " + value);
        }

        // Очистка для повторного использования
        buffer.clear();

        // Запись новых данных
        buffer.put((byte) 42);
        buffer.flip();
        System.out.println("First byte after clear: " + buffer.get());
    }
}

```
