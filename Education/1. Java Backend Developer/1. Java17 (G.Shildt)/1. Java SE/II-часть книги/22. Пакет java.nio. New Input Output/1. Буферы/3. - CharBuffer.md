
> [!NOTE] `CharBuffer` — это буфер для хранения и обработки `char`.
> 🔸Основан на `Buffer` (унаследованы `capacity`, `position`, `limit`, `mark`, `flip`, `clear`, `rewind` и т.д.).
> 
> 🔸❗❗Реализует интерфейсы: ❗❗
> - `Comparable<CharBuffer>`, 
> - `Appendable`, 
> - `CharSequence`, 
> - `Readable`. 
> 
> 
> 🔸Может использоваться как `StringBuilder`-подобная структура, но с ограниченным API.

## Конструкторы - нет конструкторов
У `CharBuffer` **нет публичных конструкторов**. Создание идёт через статические фабрики:
```java
// Выделяет новый буфер на куче
public static CharBuffer allocate(int capacity)

// Оборачивает существующий массив char[]
public static CharBuffer wrap(char[] array)

// Оборачивает часть массива
public static CharBuffer wrap(char[] array, int offset, int length)

// Оборачивает CharSequence (например, String)
public static CharBuffer wrap(CharSequence seq)

// Оборачивает подстроку CharSequence
public static CharBuffer wrap(CharSequence seq, int start, int end)
```

## Особенности и подводные камни

1. 💡**CharBuffer = CharSequence**  
    Можно передавать его в методы, где ожидается `CharSequence` (например, в `Pattern.matcher(CharSequence)`).
    
2. **toString()**  
    Возвращает только часть буфера от `position` до `limit`, а не весь массив.
    
3. **append() и put()**
    - `append()` двигает позицию.
        
    - `put(String)` может бросить `BufferOverflowException`, если не хватает места.
        
4. **slice() / duplicate()**
    - Разделяют одни и те же данные.
        
    - Изменения в исходном буфере отражаются в slice/duplicate.
        
5. 💡**codePoints()**  
    Важный метод: даёт поток **Unicode code points**, а не просто `char`. Это критично для работы с суррогатными парами.


## Best Practices

- Используй `wrap(String)` для работы с подстроками без копирования.
    
- Перед записью всегда проверяй `remaining()`.
    
- 💡Для преобразований byte ↔ char всегда применяй `CharsetDecoder`/`Encoder`, а не просто `(char)byte`.
    
- ⚠️ Помни, что `char` в Java = UTF-16 кодовое слово, не всегда 1 символ Unicode. Для полного Unicode используй `codePoints()`.
    
- 🔄 Для чтения строк из каналов — комбинируй `ByteBuffer` + `CharsetDecoder` → `CharBuffer`.

## Пример
```java
import java.nio.CharBuffer;

public class CharBufferExample {
    public static void main(String[] args) {
        // Создаём буфер на 16 символов
        CharBuffer buffer = CharBuffer.allocate(16);

        // Записываем строку
        buffer.put("Hello");
        buffer.put(' ');
        buffer.put("World");

        // Готовим к чтению
        buffer.flip();

        // Чтение по символам
        while (buffer.hasRemaining()) {
            System.out.print(buffer.get());
        }
        System.out.println();

        // Повторная запись
        buffer.clear();
        buffer.put("Java NIO");
        
        buffer.flip();
        System.out.println("Buffer content: " + buffer.toString());
    }
}
```
