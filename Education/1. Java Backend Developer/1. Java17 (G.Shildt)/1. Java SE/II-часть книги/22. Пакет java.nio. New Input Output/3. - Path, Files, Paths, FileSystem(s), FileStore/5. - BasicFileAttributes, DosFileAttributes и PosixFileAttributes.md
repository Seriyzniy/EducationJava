# 📕 1. Интерфейс BasicFileAttributes

Интерфейс `BasicFileAttributes` описывает ==**базовые атрибуты файла**== — общие для всех файловых систем.  

Он предоставляет метаинформацию: время создания, последнего доступа, тип файла и размер.

## Интерфейс
```java
public interface BasicFileAttributes {

    //✅ Проверка, является ли файл директорией
    // @return boolean — true, если объект — директория
    boolean isDirectory();

    //✅ Проверка, является ли файл обычным файлом
    // @return boolean — true, если это обычный файл
    boolean isRegularFile();

    //✅ Проверка, является ли файл символической ссылкой
    // @return boolean — true, если это симлинк
    boolean isSymbolicLink();

    //✅ Проверка, является ли файл другим типом (например, сокет, устройство)
    // @return boolean — true, если файл не подходит под стандартные категории
    boolean isOther();

    //✅ Получение размера файла
    // @return long — размер в байтах
    long size();

    //✅ Получение времени создания файла
    // @return FileTime — время создания
    FileTime creationTime();

    //✅ Получение времени последнего доступа
    // @return FileTime — время последнего чтения
    FileTime lastAccessTime();

    //✅ Получение времени последнего изменения
    // @return FileTime — время последней модификации
    FileTime lastModifiedTime();

    //✅ Уникальный ключ файла
    // @return Object — уникальный идентификатор (может быть null)
    Object fileKey();
}
```

## 💡 Особенности и подводные камни 💡

- Это **основной интерфейс**, на котором базируются `DosFileAttributes` и `PosixFileAttributes`.
- 💡При кроссплатформенной разработке полагайся только на этот интерфейс.
    
- 💡Время (`FileTime`) можно преобразовывать в миллисекунды или `Instant`.
    
- `fileKey()` не гарантированно уникален на всех платформах — зависит от ОС.
    
- ⭐⭐Доступ осуществляется через:
```java
BasicFileAttributes attr = Files.readAttributes(
	path, 
	BasicFileAttributes.class);
```

## Пример
```java
import java.nio.file.*;
import java.nio.file.attribute.*;
import java.io.IOException;

public class BasicFileAttributesExample {
    public static void main(String[] args) throws IOException {
        Path path = Paths.get("example.txt");

        BasicFileAttributes attrs = Files.readAttributes(path, BasicFileAttributes.class);

        System.out.println("Size: " + attrs.size());
        System.out.println("Created: " + attrs.creationTime());
        System.out.println("Last Modified: " + attrs.lastModifiedTime());
        System.out.println("Is Directory: " + attrs.isDirectory());
    }
}
```

# 📕 2. Интерфейс DosFileAttributes

`DosFileAttributes` расширяет `BasicFileAttributes`, ==добавляя **атрибуты DOS/Windows**== (скрытый, системный, архивный, только для чтения).  

Используется на платформах с файловыми системами типа FAT/NTFS.

## Интерфейс
```java
public interface DosFileAttributes extends BasicFileAttributes {

    //✅ Проверка, установлен ли флаг "read-only"
    // @return boolean — true, если файл нельзя изменять
    boolean isReadOnly();

    //✅ Проверка, установлен ли флаг "hidden"
    // @return boolean — true, если файл скрыт
    boolean isHidden();

    //✅ Проверка, установлен ли флаг "system"
    // @return boolean — true, если файл системный
    boolean isSystem();

    //✅ Проверка, установлен ли флаг "archive"
    // @return boolean — true, если файл отмечен для архивации
    boolean isArchive();
}
```

Можно читать и изменять через:
```java
DosFileAttributes attr = Files.readAttributes(
	path, 
	DosFileAttributes.class);

Files.setAttribute(path, "dos:hidden", true);
```

## Пример
```java
import java.nio.file.*;
import java.nio.file.attribute.*;
import java.io.IOException;

public class DosFileAttributesExample {
    public static void main(String[] args) throws IOException {
        Path path = Paths.get("C:/example.txt");

        DosFileAttributes attrs = Files.readAttributes(path, DosFileAttributes.class);

        System.out.println("Read-only: " + attrs.isReadOnly());
        System.out.println("Hidden: " + attrs.isHidden());
        System.out.println("System: " + attrs.isSystem());
        System.out.println("Archive: " + attrs.isArchive());
    }
}
```

# 📕 3. Интерфейс PosixFileAttributes

`PosixFileAttributes` расширяет `BasicFileAttributes` и ==добавляет **POSIX-атрибуты**==, используемые в ==Unix/Linux-системах==.  

Они включают владельца файла, группу и права доступа (`rwx`).

## Интерфейс
```java
public interface PosixFileAttributes extends BasicFileAttributes {

    //✅ Получение владельца файла
    // @return UserPrincipal — владелец файла
    UserPrincipal owner();

    //✅ Получение группы файла
    // @return GroupPrincipal — группа владельца
    GroupPrincipal group();

    //✅ Получение прав доступа (rwx)
    // @return Set<PosixFilePermission> — набор разрешений (например, OWNER_READ, GROUP_WRITE)
    Set<PosixFilePermission> permissions();
}
```

## 💡 Особенности и подводные камни 💡
- Поддерживается **только на POSIX-системах** (Linux, macOS, Unix).
    
- При использовании в Windows выбрасывается `UnsupportedOperationException`.
    
- 💡Для изменения прав используется:
```java
Files.setPosixFilePermissions(path, perms);
```
💡Права (`PosixFilePermission`) можно получить и изменить через `Files.getPosixFilePermissions()` и `Files.setPosixFilePermissions()`.

## Пример
```java
import java.nio.file.*;
import java.nio.file.attribute.*;
import java.io.IOException;
import java.util.Set;

public class PosixFileAttributesExample {
    public static void main(String[] args) throws IOException {
        Path path = Paths.get("/home/user/example.txt");

        PosixFileAttributes attrs = Files.readAttributes(path, PosixFileAttributes.class);

        System.out.println("Owner: " + attrs.owner().getName());
        System.out.println("Group: " + attrs.group().getName());
        System.out.println("Permissions: " + attrs.permissions());
    }
}
```
