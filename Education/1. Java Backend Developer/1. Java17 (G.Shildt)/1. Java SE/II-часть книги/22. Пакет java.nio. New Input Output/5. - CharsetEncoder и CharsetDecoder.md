Эти классы управляют процессом 
- **кодирования СИМВОЛОВ в БАЙТЫ** (`CharsetEncoder`) 
- и **декодирования БАЙТОВ в СИМВОЛЫ** (`CharsetDecoder`).

Они работают **поверх NIO-буферов** (`ByteBuffer` и `CharBuffer`),  
позволяя эффективно обрабатывать текстовые данные при потоковом I/O,  
не создавая временных строк и массивов.

## Абстрактный класс Charset

> [!NOTE] `Charset` — это **абстракция кодировки символов**,  
> Знает, **как преобразовать `char` ↔ `byte`** в соответствии с конкретным стандартом (UTF-8, ISO-8859-1, Windows-1251 и т.д.).

Эземпляры создаются через фабричные методы:
```JAVA
Charset utf8 = Charset.forName("UTF-8");
Charset utf16 = Charset.forName("UTF-16");
Charset win1251 = Charset.forName("windows-1251");
```

## **Утилитный класс** StandardCharsets

> [!NOTE] `StandardCharsets` — это **утилитный класс**, содержащий **предопределённые, стандартные Charsets** (Java 7+)
> Его цель — **избавить тебя от `UnsupportedEncodingException` и ошибок в названии кодировок.**

```java
public final class StandardCharsets {
    public static final Charset US_ASCII;
    public static final Charset ISO_8859_1;
    public static final Charset UTF_8;
    public static final Charset UTF_16;
    public static final Charset UTF_16BE;
    public static final Charset UTF_16LE;
}
```

### Пример
```java
String text = "Привет";

byte[] bytes = text.getBytes(StandardCharsets.UTF_8); // encode
String decoded = new String(bytes, StandardCharsets.UTF_8); // decode

System.out.println(decoded); // Привет

```

## Отличие Charset vs StandardCharsets

|Свойство|`Charset`|`StandardCharsets`|
|---|---|---|
|Тип|Класс|Утилитный класс|
|Создание|`Charset.forName("UTF-8")`|`StandardCharsets.UTF_8`|
|Возможность кастомных кодировок|✅ Да|❌ Нет|
|Проверка доступности|Нужно|Не нужно|
|Исключения|Может бросить|Не бросает|
|Производительность|Одинаковая|Одинаковая|
|Назначение|Общая работа с кодировками|Быстрый доступ к стандартным кодировкам|

# 📕 1. CharsetEncoder
## Конструктор
Экземпляры создаются через объект `Charset`:
```JAVA
CharsetEncoder encoder = StandardCharsets.UTF_8.newEncoder();
```

## Основные методы
```java
// Кодирует все символы из входного CharBuffer в выходной ByteBuffer
// in - входной буфер символов для кодирования
// out - выходной буфер для записи закодированных байтов  
// endOfInput - флаг, указывающий что это последняя порция входных данных
// @return результат операции кодирования (переполнение, недополнение или ошибка)
public final CoderResult encode(
	CharBuffer in, 
	ByteBuffer out, 
	boolean endOfInput);

// Завершает кодирование (должен вызываться после последней порции данных)
// out - выходной буфер для записи оставшихся байтов
// @return результат операции завершения
public final CoderResult flush(ByteBuffer out);

// Сбрасывает внутреннее состояние энкодера (например, BOM и т.д.)
// @return этот энкодер для цепочки вызовов
public final CharsetEncoder reset();

// Кодирует весь буфер символов в байты за один шаг (удобно, но неэффективно для больших потоков)
// in - входной буфер символов для кодирования
// @return буфер с закодированными байтами
public final ByteBuffer encode(CharBuffer in);
```

## Важные параметры энкодера
```java
public Charset charset();             // Кодировка, с которой связан энкодер
public float averageBytesPerChar();   // Среднее число байт на символ
public float maxBytesPerChar();       // Максимум байт на символ
```
Например, для UTF-8:
- `averageBytesPerChar()` ≈ 1.1
- `maxBytesPerChar()` = 4

## Обработка ошибок
Энкодер может быть настроен, как реагировать на невалидные символы:
```java
public CharsetEncoder onMalformedInput(CodingErrorAction newAction);
public CharsetEncoder onUnmappableCharacter(
	CodingErrorAction newAction);
```
`CodingErrorAction`:
- `REPORT` → выбрасывает исключение (`MalformedInputException`)
- `REPLACE` → заменяет символ на `?` (по умолчанию)
- `IGNORE` → просто пропускает

## Пример — потоковое кодирование UTF-8
```java
CharsetEncoder encoder = StandardCharsets.UTF_8.newEncoder();

CharBuffer input = CharBuffer.wrap("Привет, NIO!");
ByteBuffer output = ByteBuffer.allocate(64);

CoderResult result = encoder.encode(input, output, true);
encoder.flush(output);
output.flip();

System.out.println("Encoded bytes: " + output.remaining());
```

# 📕 2. CharsetDecoder
## Конструктор
```java
CharsetDecoder decoder = StandardCharsets.UTF_8.newDecoder();
```

## Основные методы
```java
// Декодирует байты в символы
public final CoderResult decode(ByteBuffer in, CharBuffer out, boolean endOfInput);

// Завершает декодирование (аналог flush)
public final CoderResult flush(CharBuffer out);

// Сбрасывает внутреннее состояние
public final CharsetDecoder reset();

// Упрощённый способ: декодирует весь ByteBuffer в CharBuffer за раз
public final CharBuffer decode(ByteBuffer in);
```

## Важные параметры
```java
public Charset charset();               // Связанная кодировка
public float averageCharsPerByte();     // Среднее число символов на байт
public float maxCharsPerByte();         // Максимальное число символов на байт
```

## Настройка обработки ошибок
```java
public CharsetDecoder onMalformedInput(CodingErrorAction newAction);
public CharsetDecoder onUnmappableCharacter(CodingErrorAction newAction);
```
`CodingErrorAction`:
- `REPORT` → выбрасывает исключение (`MalformedInputException`)
- `REPLACE` → заменяет символ на `?` (по умолчанию)
- `IGNORE` → просто пропускает

## Пример
```java
CharsetDecoder decoder = StandardCharsets.UTF_8.newDecoder();

ByteBuffer input = ByteBuffer.wrap("Привет, NIO!".getBytes(StandardCharsets.UTF_8));
CharBuffer output = CharBuffer.allocate(64);

decoder.decode(input, output, true);
decoder.flush(output);
output.flip();

System.out.println("Decoded: " + output.toString());
```

# Детали и подводные камни

### ⚠️ 1. Потоковый режим
`encode()` / `decode()` не обязательно кодируют весь буфер за раз.  
Они могут вернуть `CoderResult.OVERFLOW` (если `out` мал) или `UNDERFLOW` (если `in` пуст).  
→ Поэтому нужно **в цикле вызывать encode/decode** до тех пор, пока не закончится ввод или вывод.

```java
while (in.hasRemaining()) {
    CoderResult result = encoder.encode(in, out, false);
    if (result.isOverflow()) {
        out = enlarge(out);
    }
}
encoder.flush(out);
```

### ⚠️ 2. Нельзя использовать один энкодер/декодер из нескольких потоков
Они **не потокобезопасны** — создавай отдельный экземпляр на поток.

### ⚠️ 3. 💡Performance💡
`CharsetEncoder/Decoder` 💡быстрее💡 `String.getBytes()` и `new String(byte[])` при больших потоках,  
потому что они ==работают **поверх NIO-буферов**== без промежуточных массивов.
