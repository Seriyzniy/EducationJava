
> [!NOTE] `AsynchronousChannel` — это **базовый интерфейс**, который расширяет `Channel` 
> Задаёт поведение для ==**асинхронных операций ввода-вывода**==, которые `НЕ блокируют поток`, вызвавший метод.
> 
> ❗❗❗НАПРЯМУЮ от него НИКТО НЕ РЕАЛИЗУЕТСЯ❗❗❗
> Наследуются от `AsynchronousByteChannel`
> 

Он является **фундаментом** для таких реализаций, как:

| Класс                             | Назначение                                            |
| --------------------------------- | ----------------------------------------------------- |
| `AsynchronousFileChannel`         | Асинхронное чтение/запись файлов                      |
| `AsynchronousSocketChannel`       | Асинхронное чтение/запись по TCP                      |
| `AsynchronousServerSocketChannel` | Асинхронное принятие подключений TCP                  |
| `AsynchronousByteChannel`         | Расширение с методами `read()` / `write()` (см. ниже) |
## Назначение

 ❗❗❗==Обычные каналы== (`FileChannel`, `SocketChannel`, и т.д.) выполняют операции `read()` и `write()` **синхронно**  
(то есть ==вызывающий поток ждёт завершения операции==),  

 💡💡А`AsynchronousChannel` выполняет их **в фоновом потоке**,  
а результат передаётся через:
- **`Future<V>`**, или
    
- **`CompletionHandler<V, A>`** (колбэк).

## Интерфейс
```java
package java.nio.channels;

public interface AsynchronousChannel extends Channel {
    // ✅ Закрывает канал и отменяет все ожидающие операции
    @Override
    void close() throws IOException;

    // ✅ Проверяет, открыт ли канал
    @Override
    boolean isOpen();
}
```
