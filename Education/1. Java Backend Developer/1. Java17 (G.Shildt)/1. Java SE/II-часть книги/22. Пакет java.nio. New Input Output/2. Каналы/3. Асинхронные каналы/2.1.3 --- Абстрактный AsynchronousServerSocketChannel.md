
> [!NOTE] `AsynchronousServerSocketChannel` — это **финальный класс** из пакета `java.nio.channels`,  
> Реализующий интерфейсы:
> - `AsynchronousChannel`
> - `NetworkChannel`
> 
> Он используется для ==ПРИНЯТИЯ асинхронных TCP-подключений==, возвращая объекты `AsynchronousSocketChannel`,  с которыми затем можно взаимодействовать (читать/писать данные).
> 

## Назначение
- Принимает **TCP-клиентов асинхронно** (не блокирует поток).
    
- Работает через **CompletionHandler** или **Future**.
    
- Может быть создан внутри конкретного `AsynchronousChannelGroup` (thread pool).
    
- Поддерживает установку **сетевых опций** (`SO_REUSEADDR`, `SO_RCVBUF` и т. д.)

Используется для написания полностью неблокирующих **асинхронных серверов**.

## Конструкторы
У `AsynchronousServerSocketChannel` **нет публичных конструкторов**.  
Создание канала производится через статические методы:
```java
// Создаёт новый асинхронный серверный сокет (в системной группе потоков)
public static AsynchronousServerSocketChannel open() throws IOException;

// Создаёт серверный сокет, использующий указанную группу каналов
public static AsynchronousServerSocketChannel open(AsynchronousChannelGroup group) throws IOException;
```
📌 **Параметры:**
- `group` — объект `AsynchronousChannelGroup`, определяющий пул потоков, где выполняются операции `accept()`.  
    Если `null`, используется общий системный пул.

## Методы
```java
// ✅ Привязка к адресу
/**
 * Привязывает сервер к локальному адресу (например, порту).
 * local — адрес (например, new InetSocketAddress(8080))
 * @return этот же канал (для chaining)
 */
@Override
public abstract AsynchronousServerSocketChannel bind(
	SocketAddress local) throws IOException;

/**
 * Привязывает сервер с указанием размера очереди ожидающих клиентов.
 * local — локальный адрес (порт/IP)
 * backlog — размер очереди ожидающих подключений
 * @return этот же канал
 */
public abstract AsynchronousServerSocketChannel bind(
	SocketAddress local, 
	int backlog) throws IOException;



// ✅ Получение локального адреса
/**
 * Возвращает локальный адрес, к которому привязан сервер.
 * @return объект SocketAddress (IP + порт)
 */
@Override
public abstract SocketAddress getLocalAddress() throws IOException;



// ✅ Асинхронное ПРИНЯТИЕ подключений
/**
 * Асинхронно принимает подключение, возвращая Future.
 * Завершается, когда клиент подключился, возвращая AsynchronousSocketChannel.
 * @return Future, который завершится новым клиентским каналом
 */
public abstract Future<AsynchronousSocketChannel> accept();

/**
 * Асинхронно ПРИНИМАЕТ подключение с callback.
 * <A> — тип объекта, передаваемого в handler
 * attachment — объект, доступный в CompletionHandler (контекст, например сервер)
 * handler — обработчик завершения, вызывается при успешном или неудачном подключении
 */
public abstract <A> void accept(
	A attachment, 
	CompletionHandler<AsynchronousSocketChannel, ? super A> handler);



// ✅ Управление TCP-параметрами
/**
 * Устанавливает опцию сокета (например, SO_REUSEADDR).
 * <T> — тип значения
 * name — имя опции (из StandardSocketOptions)
 * value — значение опции
 * @return этот же канал
 */
@Override
public abstract <T> AsynchronousServerSocketChannel setOption(
	SocketOption<T> name, 
	T value) throws IOException;

/**
 * Возвращает значение указанной опции.
 * <T> — тип значения
 * name — имя опции
 * @return текущее значение
 */
@Override
public abstract <T> T getOption(SocketOption<T> name) throws IOException;

/**
 * Возвращает набор поддерживаемых опций (SO_RCVBUF, SO_REUSEADDR и т. д.)
 */
@Override
public abstract Set<SocketOption<?>> supportedOptions();



// ✅ Управление состоянием
/**
 * Проверяет, открыт ли канал.
 * @return true, если открыт
 */
@Override
public abstract boolean isOpen();

/**
 * Закрывает канал и отменяет все ожидающие операции accept().
 */
@Override
public abstract void close() throws IOException;
```

## 💡Особенности и подводные камни 💡

### 1. Асинхронный accept()
- `accept()` не блокирует — возвращает `Future` или сразу планирует `CompletionHandler`.
    
- 💡Когда клиент подключается, вызывается `completed()`.
    
- После каждого успешного `accept()` нужно **вручную вызвать `accept()` снова**, чтобы слушать новые соединения.  
    Это очень важно — сервер **не слушает** автоматически!

### 2. Backlog
- `backlog` задаёт размер очереди TCP-подключений, ожидающих обработки.
    
- Если очередь переполнена, новые подключения могут быть отклонены ОС.
    
### 3. Опции сокета
- Можно настраивать:
    - `SO_RCVBUF` — размер приёмного буфера;
        
    - `SO_REUSEADDR` — разрешает повторно использовать адрес (типично для серверов);
        
    - `SO_REUSEPORT` — для балансировки на некоторых ОС.
        

### 💡4. Non-blocking всегда
- В отличие от `ServerSocketChannel`, этот канал **всегда неблокирующий**.
    
- Даже вызов `accept()` не ждёт клиента, он лишь планирует событие.
    
###  5. Группы каналов
- `AsynchronousServerSocketChannel` выполняет операции в рамках `AsynchronousChannelGroup` (thread pool).
    
- Можно использовать **один пул для нескольких серверов**.
    
### 6. Исключения
- Если сервер закрыт во время `accept()` → `AsynchronousCloseException`.
    
- Если ошибка сети → `IOException` или `ClosedChannelException`.

## ✅ Best Practices ✅

- ❗❗❗После каждого `accept()` **немедленно вызывать `accept()` снова**, иначе сервер перестанет принимать клиентов.
    
- Для реактивного кода — всегда используй `CompletionHandler`, а не `Future`.
    
- 🔒 Обязательно закрывай клиентские каналы после завершения сессии.
    
- Настраивай `SO_REUSEADDR` до `bind()` для перезапуска без задержки.
    
- Создавай собственную `AsynchronousChannelGroup` с фиксированным пулом потоков при большом числе соединений.
    
- ❗❗❗Помни: `AsynchronousSocketChannel` (клиент) нужно обслуживать в `completed()` — чтение/запись выполняются там.

## Пример 1 — Простой асинхронный echo-сервер (CompletionHandler)
```java
import java.net.*;
import java.nio.*;
import java.nio.channels.*;

public class AsyncEchoServer {
    public static void main(String[] args) throws Exception {
        AsynchronousServerSocketChannel server =
                AsynchronousServerSocketChannel.open().bind(new InetSocketAddress(9000));

        System.out.println("Async server started on port 9000...");

        server.accept(null, new CompletionHandler<>() {
            @Override
            public void completed(
	            AsynchronousSocketChannel client, 
	            Object att) 
	        {
                // ВАЖНО: снова вызвать accept(), 
                // чтобы слушать новые подключения
                server.accept(null, this);

                try {
                    System.out.println(
	                    "Client connected: " + 
	                    client.getRemoteAddress());
	                    
                } catch (Exception ignored) {}


                ByteBuffer buffer = ByteBuffer.allocate(256);


                // Асинхронное чтение
                client.read(
	                buffer, 
	                buffer, 
	                new CompletionHandler<Integer, ByteBuffer>() 
	            {
                    @Override
                    public void completed(
	                    Integer bytesRead, 
	                    ByteBuffer buf) 
	                {
                        if (bytesRead == -1) {
                            try { 
	                            client.close(); 
	                        } catch (Exception ignored) {}
                            return;
                        }

                        buf.flip();
                        client.write(
	                        buf, 
	                        buf, 
	                        new CompletionHandler<Integer, ByteBuffer>
	                    () {
                            @Override
                            public void completed(
	                            Integer result, 
	                            ByteBuffer b) 
	                        {
                                b.clear();
                                client.read(b, b, this); // читаем снова
                            }

                            @Override
                            public void failed(Throwable exc, ByteBuffer b) {
                                try { client.close(); } catch (Exception ignored) {}
                            }
                        });
                    }

                    @Override
                    public void failed(Throwable exc, ByteBuffer buf) {
                        try { client.close(); } catch (Exception ignored) {}
                    }
                });
            }

            @Override
            public void failed(Throwable exc, Object att) {
                System.err.println("Accept failed: " + exc);
            }
        });

        Thread.currentThread().join(); // держим сервер живым
    }
}
```

## Пример 2 — Принятие подключения через Future API
```java
import java.net.*;
import java.nio.channels.*;
import java.util.concurrent.Future;

public class AsyncServerFutureExample {
    public static void main(String[] args) throws Exception {
        AsynchronousServerSocketChannel server =
                AsynchronousServerSocketChannel
                .open()
                .bind(new InetSocketAddress(8080));

        System.out.println("Waiting for connection...");

        Future<AsynchronousSocketChannel> future = server.accept();
        
        AsynchronousSocketChannel client = future.get(); // блокируемся только здесь

        System.out.println("Client connected: " + client.getRemoteAddress());
        server.close();
        client.close();
    }
}
```
