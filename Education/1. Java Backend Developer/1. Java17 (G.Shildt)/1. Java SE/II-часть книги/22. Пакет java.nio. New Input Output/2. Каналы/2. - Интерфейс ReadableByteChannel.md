
> [!NOTE] `ReadableByteChannel` — это **подинтерфейс** `Channel`
> Интерфейс представляющий источник байтов (==аналог **InputStream** в NIO==). 
>  
> Он определяет метод `read(ByteBuffer)`, который ==загружает данные из канала в буфер.==

Используется для чтения данных из:
- **файлов** (`FileChannel`),
- **сетевых сокетов** (`SocketChannel`, `DatagramChannel`),
- **пайпов** (`Pipe.SourceChannel`),
    
- и любых других источников, поддерживающих NIO.

## Интерфейс
```java
package java.nio.channels;

public interface ReadableByteChannel extends Channel {

    /**
     * СЧИТЫВАЕТ последовательность байтов из этого канала В УКАЗАННЫЙ БУФЕР.
     *
     * dst - буфер, куда будут записаны данные
     * @return количество прочитанных байтов, 
     *         либо -1 если достигнут конец потока
	 *
     * @throws IOException - при ошибке ввода-вывода
     */
    public int read(ByteBuffer dst) throws IOException;
}
```

## Особенности и подводные камни
1. 💡**Аналог InputStream.read()** 
    Только работает не с массивами, а с `ByteBuffer`.
    
2. **Возвращаемое значение**
    - `> 0` → число прочитанных байт,
    - `0` → в неблокирующем режиме данные недоступны,
    - `-1` → конец потока (EOF).
        
3. 💡**Неблокирующий режим**
    - Для каналов, которые могут работать неблокирующе (`SocketChannel`, `DatagramChannel`), вызов `read()` может вернуть `0`, ==если данных пока нет==.
        
    - Для `FileChannel` всегда блокирующий (чтение до конца доступного куска или EOF).
        
4. 💡**Буферное состояние**
    - После вызова `read()` `position` буфера сдвигается на количество записанных байтов.
        
    - Перед чтением данных из буфера не забудь вызвать `flip()`.
        
5. 💡**Ограничение**
    - `ReadableByteChannel` определяет только чтение.
    - Для записи существует `WritableByteChannel`.
        
    - 💡Для работы и туда, и туда — `ByteChannel` (расширяет оба).

## ✅ Best Practices ✅
- Используй `try-with-resources` для автоматического закрытия канала.
    
- Перед передачей буфера в `read()` убедись, что в нём есть место (`buffer.hasRemaining()`).
    
- После чтения — обязательно вызывай `flip()` перед обработкой содержимого.
    
- При работе с сетью (non-blocking) проверяй `0` и `-1`.


## Пример - Чтение из файла

```JAVA
import java.io.RandomAccessFile;
import java.nio.ByteBuffer;
import java.nio.channels.FileChannel;
import java.nio.channels.ReadableByteChannel;

public class ReadableByteChannelFileExample {
    public static void main(String[] args) throws Exception {
        try (RandomAccessFile raf = new RandomAccessFile("example.txt", "r");
             ReadableByteChannel channel = raf.getChannel()) {

            ByteBuffer buffer = ByteBuffer.allocate(64);

            while (channel.read(buffer) != -1) {
                buffer.flip(); // подготовка к чтению
                while (buffer.hasRemaining()) {
                    System.out.print((char) buffer.get());
                }
                buffer.clear(); // готовим для новой порции
            }
        }
    }
}
```

Чтение из сокета
```java
import java.net.InetSocketAddress;
import java.nio.ByteBuffer;
import java.nio.channels.ReadableByteChannel;
import java.nio.channels.SocketChannel;

public class ReadableByteChannelSocketExample {
    public static void main(String[] args) throws Exception {
        try (SocketChannel socketChannel = SocketChannel.open(
	        new InetSocketAddress("example.com", 80))
	    ) {
            String request = "GET / HTTP/1.1\r\nHost: example.com\r\n\r\n";
            
            socketChannel.write(ByteBuffer.wrap(request.getBytes()));

            ReadableByteChannel channel = socketChannel;

            ByteBuffer buffer = ByteBuffer.allocate(1024);
            int bytesRead = channel.read(buffer);

            buffer.flip();
            while (buffer.hasRemaining()) {
                System.out.print((char) buffer.get());
            }
        }
    }
}
```
