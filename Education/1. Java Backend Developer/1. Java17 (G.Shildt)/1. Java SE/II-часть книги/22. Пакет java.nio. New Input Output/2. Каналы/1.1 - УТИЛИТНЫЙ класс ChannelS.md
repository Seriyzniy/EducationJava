
> [!NOTE] `Channels` - **финальный утилитарный класс** (все методы `static`)
> Предоставляет методы для **создания потоков(`Reader`/`Writer`/`InputStream`/`OutputStream`) из каналов (`Channel`, `ReadableByteChannel`, `WritableByteChannel`)** ==и наоборот==.
> 
> 💡Проще говоря: `Channels` связывает старый мир `java.io.*` и новый `java.nio.*`.

## Методы 
### Преобразование ПОТОКИ -> КАНАЛЫ
```java
//✅Создание канала для чтения из InputStream
//Параметр: inputStream — поток ввода
//Возвращает ReadableByteChannel, читающий байты из данного InputStream
public static ReadableByteChannel newChannel(InputStream inputStream);


//✅Создание канала для записи в OutputStream
//Параметр: outputStream — поток вывода
//Возвращает WritableByteChannel, записывающий байты в данный OutputStream
public static WritableByteChannel newChannel(OutputStream outputStream);
```
Эти методы оборачивают потоки в легкие адаптеры (`ReadableByteChannelImpl`, `WritableByteChannelImpl`).  

Их удобно использовать при интеграции старого кода (IO) с новым (NIO).

### Преобразование  КАНАЛЫ -> ПОТОКИ
```JAVA
//✅Создание InputStream из канала
//Параметр: ch — канал для чтения байт
//Возвращает InputStream, который читает из указанного ReadableByteChannel
public static InputStream newInputStream(ReadableByteChannel ch);


//✅Создание OutputStream из канала
//Параметр: ch — канал для записи байт
//Возвращает OutputStream, который пишет в указанный WritableByteChannel
public static OutputStream newOutputStream(WritableByteChannel ch);
```

### Преобразование КАНАЛЫ -> СИМВОЛЬНЫЕ ПОТОКИ (с кодировкой)
```java
//✅Создание Reader из канала с кодировкой
//Параметры:
// ch — ReadableByteChannel источник байт
// decoder — CharsetDecoder для декодирования байтов в символы
// Возвращает Reader, который читает символы из байтов канала
public static Reader newReader(ReadableByteChannel ch, CharsetDecoder decoder, int minBufferCap);


//✅Создание Writer из канала с кодировкой
//Параметры:
// ch — WritableByteChannel, куда будут записываться байты
// encoder — CharsetEncoder для кодирования символов
// Возвращает Writer, который преобразует символы в байты и пишет в канал
public static Writer newWriter(WritableByteChannel ch, CharsetEncoder encoder, int minBufferCap);


// Упрощённые ПЕРЕГРУЗКИ с Charset
//✅Создание Reader с кодировкой
//Параметры:
// ch — ReadableByteChannel источник байт
// charset — Charset, определяющий кодировку
// Возвращает Reader, читающий символы с автоматическим декодированием
public static Reader newReader(ReadableByteChannel ch, Charset charset);

//✅Создание Writer с кодировкой
//Параметры:
// ch — WritableByteChannel получатель байт
// charset — Charset, по которому будут кодироваться символы
// Возвращает Writer, который записывает символы с автоматическим кодированием
public static Writer newWriter(WritableByteChannel ch, Charset charset);
```

💬 ⭐⭐Эти методы позволяют, например, писать строки в `SocketChannel` напрямую через `Writer`,⭐⭐

(реже используется напрямую, но встраивается в `Files.newAsynchronousChannelGroup()`)

## 💡 Особенности и подводные камни 💡

### ⚠️ 1. Потоки, созданные через Channels, не всегда поддерживают `mark()`/`reset()`
— Т.к. они работают поверх каналов, где произвольное позиционирование не всегда возможно (например, в `SocketChannel`).

### ⚠️ 2. `Channels.newChannel()` не делает буферизацию
Если хочешь эффективность — оборачивай результат в `BufferedInputStream` или `BufferedReader`.
```java
BufferedReader reader = new BufferedReader(
	Channels.newReader(ch, StandardCharsets.UTF_8)
);
```

### ⚠️ 3. ⭐⭐Закрытие⭐⭐
==Закрытие потока закрывает и канал, и наоборот.  ==
Это поведение часто становится неожиданностью при работе с сетевыми сокетами:
```java
InputStream in = Channels.newInputStream(socketChannel);
in.close(); // ⭐также закрывает socketChannel!⭐
```

### ⚠️ 4. Разные буферы для char и byte
Методы с `Charset` создают внутренние `ByteBuffer` и `CharBuffer`,  
поэтому работа с большими текстами должна учитывать ёмкость буфера (`minBufferCap`).

## Пример
```java
import java.io.*;
import java.nio.*;
import java.nio.channels.*;
import java.nio.charset.*;

public class ChannelsExample {
    public static void main(String[] args) throws IOException {
        // Создаём FileChannel
        FileChannel fileChannel = FileChannel.open(
                java.nio.file.Path.of("example.txt"),
                java.nio.file.StandardOpenOption.READ);

        // Создаём Reader с указанием кодировки UTF-8
        Reader reader = Channels.newReader(fileChannel, StandardCharsets.UTF_8);

        // Читаем символы
        BufferedReader bufferedReader = new BufferedReader(reader);
        String line;
        while ((line = bufferedReader.readLine()) != null) {
            System.out.println("Line: " + line);
        }

        bufferedReader.close(); // закроет и Reader, и FileChannel
    }
}

```
