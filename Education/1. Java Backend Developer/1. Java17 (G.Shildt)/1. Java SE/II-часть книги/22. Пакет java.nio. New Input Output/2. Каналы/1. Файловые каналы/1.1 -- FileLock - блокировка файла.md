
> [!NOTE] `FileLock` — это **абстрактный класс** из пакета `java.nio.channels`, 
> 
> Класс ==представляющий блокировку участка файла==, полученную через
> - `FileChannel.lock()` или 
> - `FileChannel.tryLock()`.  
> 
> Он позволяет **предотвращать одновременную запись или чтение** файла из нескольких процессов или потоков.
> 

❗❗❗Блокировка может быть:
- 🟢**эксклюзивной (exclusive)** — только ОДИН ПРОЦЕСС может читать/писать,
    
- 🟠**разделяемой (shared)** — НЕСКОЛЬКО процессов могут ЧИТАТЬ, НО НЕ ПИСАТЬ

## Конструктор
У `FileLock` **нет публичного конструктора**.  

❗❗❗Объекты создаются только методами `FileChannel.lock()` или `tryLock()`.
```java
try (FileChannel channel = FileChannel.open(Path.of("data.txt"), StandardOpenOption.WRITE)) {
    FileLock lock = channel.lock();  // 💡блокировка всего файла
}
```

## Методы
```java
// ✅ Информация о блокировке
// Канал, которому принадлежит блокировка
public abstract FileChannel channel();

// Канал (в виде абстрактного Channel)
public Channel acquiredBy();

// Начальная позиция заблокированного участка
public final long position();

// Размер заблокированного участка
public final long size();

// Разделяемая ли блокировка (true = shared)
public final boolean isShared();

// Проверяет, действительна ли блокировка
public final boolean isValid();

// Освобождает блокировку
public abstract void release() throws IOException;



// ✅ Проверка пересечения (overlap)
// Проверяет, пересекается ли данный диапазон с другим
public final boolean overlaps(long position, long size);
```

## 💡 Особенности и подводные камни 💡

1. **Блокировки зависят от ОС**
    - На большинстве систем блокировки являются **advisory** — т.е. носят рекомендательный характер.  
        Если другой процесс не использует NIO, он может всё равно изменить файл.
        
2. 💡 **Диапазон блокировки**
    - Можно заблокировать **весь файл** или **его часть**, передав позицию и длину.
        
    - `position()` и `size()` возвращают границы.
        
3. 🟢**Разделяемая (shared) блокировка**
    - Позволяет нескольким процессам читать файл, но запрещает запись.
        
    - Доступна только при открытии канала в `READ`-режиме.
        
4. 🟠**Эксклюзивная (exclusive) блокировка**
    
    - Полностью блокирует доступ другим процессам (ни чтение, ни запись).
        
5. **Время жизни**
    - `FileLock` автоматически освобождается при закрытии `FileChannel`.
        
    - Но **лучше вызывать `release()` явно**, чтобы не держать дескрипторы открытыми.
        
6. **Неблокирующие блокировки**
    - `tryLock()` возвращает `null`, если участок уже заблокирован.
        
    - `lock()` блокируется, пока не сможет установить блокировку.
        
7. **Кроссплатформенные отличия**
    - На Windows `lock()` блокирует доступ даже через другие процессы.
        
    - На UNIX-системах блокировки — только на уровне приложений, использующих NIO/FILE API.

## ✅ Best Practices ✅
- Всегда освобождай блокировку (`release()` или try-with-resources).
    
- Используй `tryLock()` для неблокирующих попыток.
    
- 💡Если нужно ограничить доступ только к части файла — указывай позицию и размер.
    
- 💡Для синхронизации внутри одного процесса — проще использовать `synchronized` или `ReentrantLock`.
    
- 🚨 **Механизм `FileLock` НЕ ЯВЛЯЕТСЯ абсолютной гарантией защиты файла на уровне всей системы**.

## Пример
Пример 1 — эксклюзивная блокировка всего файла
```java
import java.io.IOException;
import java.nio.channels.FileChannel;
import java.nio.channels.FileLock;
import java.nio.file.Path;
import java.nio.file.StandardOpenOption;

public class FileLockExample {
    public static void main(String[] args) throws IOException {
        Path path = Path.of("example.txt");

        try (FileChannel channel = FileChannel.open(path, 
                StandardOpenOption.CREATE, 
                StandardOpenOption.WRITE)) {

            System.out.println("Attempting to acquire lock...");
            
            FileLock lock = channel.lock(); // блокирующий вызов
            
            System.out.println("Lock acquired: " + lock);

            // Работа с файлом
            Thread.sleep(3000);

            lock.release(); // освобождаем
            System.out.println("Lock released.");
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

```

Пример 2 — частичная блокировка файла
```java
import java.io.IOException;
import java.nio.channels.FileChannel;
import java.nio.channels.FileLock;
import java.nio.file.Path;
import java.nio.file.StandardOpenOption;

public class PartialFileLockExample {
    public static void main(String[] args) throws IOException {
        Path path = Path.of("data.bin");

        try (FileChannel channel = FileChannel.open(path, 
                StandardOpenOption.CREATE, 
                StandardOpenOption.READ, 
                StandardOpenOption.WRITE)) {

            // Блокируем байты с 100 по 200
            FileLock lock = channel.lock(100, 100, false); // false = exclusive
            System.out.println("Locked region: position=" + lock.position() + ", size=" + lock.size());

            Thread.sleep(2000);

            lock.release();
            System.out.println("Region unlocked.");
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```
