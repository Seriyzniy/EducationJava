
> [!NOTE] `FileChannel` — это канал `для работы с файлами`
> Расширяет `AbstractInterruptibleChannel` и реализует `SeekableByteChannel`, `GatheringByteChannel`, `ScatteringByteChannel`.


> [!NOTE] ❗❗❗Реализации `FileChannel`❗❗❗
> Реализацию `FileChannel` `ПРЕДОСТАВЛЯЕТ САМА JVM` через фабричные методы
> - `FileChannel.open()`
> - `RandomAccessFile.getChannel()`).


Он ==объединяет возможности старого== `RandomAccessFile`, но работает через **буферы** и поддерживает NIO-фичи:
- ==чтение и запись== через `ByteBuffer`,
- работа с позиционированием (`position`, `truncate`, `size`),
    
- **memory-mapped files** (`map`),
    
- 💡блокировки файлов (`lock`, `tryLock`),
    
- 💡прямое копирование (`transferTo`, `transferFrom`).

## Конструктор
У `FileChannel` **нет публичных конструкторов**.  
Экземпляры создаются через фабричный метод:
```JAVA
// ✅Метод open()✅
static FileChannel open(Path path, OpenOption... options)
static FileChannel open(
	Path path, 
	Set<? extends OpenOption options,
	FileAttribute<?>... attrs)
```

Пример
```java
// ✅ Создание FileChannel через open() ✅
FileChannel channel = FileChannel.open(Path.of("file.txt"), StandardOpenOption.READ);


// ✅Метод getChannel()✅
// Через RandomAccessFile (java.io)
RandomAccessFile raf = new RandomAccessFile("file.txt", "rw");
FileChannel channel = raf.getChannel();

// Через FileInputStream / FileOutputStream
FileInputStream fis = new FileInputStream("file.txt");
FileChannel channel = fis.getChannel();
```

## Методы
```java
// ✅ Чтение
// Читает данные из канала в указанный буфер
// Возвращает количество прочитанных байт или -1 при EOF
public abstract int read(ByteBuffer dst) throws IOException;

// Scatter read: читает данные в массив буферов последовательно
// offset - начальный индекс в массиве буферов, length - количество буферов
// Возвращает общее количество прочитанных байт
public abstract long read(ByteBuffer[] dsts, int offset, int length) throws IOException;

// Читает данные из указанной позиции файла, не изменяя текущую позицию канала
// position - позиция в файле для чтения, dst - буфер назначения
public abstract int read(ByteBuffer dst, long position) throws IOException;



// ✅ Запись
// Записывает данные из буфера в канал
// Возвращает количество записанных байт
public abstract int write(ByteBuffer src) throws IOException;

// Gather write: записывает данные из массива буферов последовательно
// offset - начальный индекс в массиве буферов, length - количество буферов
// Возвращает общее количество записанных байт
public abstract long write(ByteBuffer[] srcs, int offset, int length) throws IOException;

// Записывает данные в указанную позицию файла, не изменяя текущую позицию канала
// position - позиция в файле для записи, src - исходный буфер
public abstract int write(ByteBuffer src, long position) throws IOException;



// ✅ Позиционирование и размер
// Возвращает текущую позицию в файле
public abstract long position() throws IOException;

// Устанавливает новую позицию в файле
// Возвращает этот FileChannel для цепочки вызовов
public abstract FileChannel position(long newPosition) throws IOException;

// Возвращает текущий размер файла в байтах
public abstract long size() throws IOException;

// Обрезает файл до указанного размера
// Если размер меньше текущего, данные после усечения теряются
public abstract FileChannel truncate(long size) throws IOException;



// ✅ Memory-mapped files (отображение в память)
// Создает memory-mapped регион файла
// mode - режим доступа (READ_ONLY, READ_WRITE, PRIVATE)
// position - начальная позиция в файле, size - размер региона
public abstract MappedByteBuffer map(
	MapMode mode, 
	long position, 
	long size) throws IOException;



// ✅ Zero-copy операции
// Передает данные из этого канала в целевой канал без копирования в пользовательское пространство
// position - позиция в исходном файле, count - количество байт, target - целевой канал
public abstract long transferTo(
	long position, 
	long count, 
	WritableByteChannel target) throws IOException;

// Передает данные из исходного канала в этот канал без копирования в пользовательское пространство
// src - исходный канал, position - позиция в целевом файле, count - количество байт
public abstract long transferFrom(
	ReadableByteChannel src, 
	long position, 
	long count) throws IOException;



// ✅ Блокировки файла
// Блокирует регион файла (блокирующий вызов)
// position - начальная позиция, size - размер региона, shared - разделяемая блокировка
public abstract FileLock lock(
	long position, 
	long size, 
	boolean shared) throws IOException;

// Пытается заблокировать регион файла (неблокирующий вызов)
// Возвращает null если блокировка недоступна
public abstract FileLock tryLock(
	long position, 
	long size, 
	boolean shared) throws IOException;

// Блокирует весь файл (блокирующий вызов)
public final FileLock lock() throws IOException;

// Пытается заблокировать весь файл (неблокирующий вызов)
public final FileLock tryLock() throws IOException;



// ✅ Сброс изменений
// Принудительно записывает все изменения файла на диск
// metaData - true если нужно синхронизировать метаданные (время модификации и т.д.)
public abstract void force(boolean metaData) throws IOException;



// ✅ Закрытие
// Закрывает канал и освобождает все связанные ресурсы
@Override
public abstract void close() throws IOException;
```

## 💡Особенности и подводные камни 💡
- Это **абстрактный класс**, прямого конструктора нет → экземпляры 💡создаются через фабрики (`open`, `getChannel`).💡
    
- **Поддерживает позиционированное чтение/запись** — можно читать/писать в произвольное место файла.
    
- `map()` даёт **MappedByteBuffer** для доступа к файлу как к памяти.
    
- 💡`transferTo/transferFrom` — очень быстрые операции zero-copy (например, для отправки файлов через сокет).
    
- 💡`lock()` и `tryLock()` → **блокировки на уровне ОС** (часто advisory, а не обязательные).
    
- На Windows **MappedByteBuffer блокирует файл** до тех пор, пока буфер не будет собран GC.
    
- ❗❗❗Не потокобезопасен — требуется синхронизация при доступе из разных потоков.

## ✅Best Practices ✅

- Используй `map()` для больших файлов (БД, логи, индексы).
    
- 💡Для копирования файлов — `transferTo`/`transferFrom` вместо ручных циклов.
    
- ⭐⭐⭐Закрывай канал (`try-with-resources`), чтобы избежать утечки дескрипторов.⭐⭐⭐
    
- 💡При критичных данных (журналы, транзакции) → вызывай `force(true)` после записи.
    
-  💡Для параллельного доступа нескольких процессов → применяй `lock()`.

## Пример
```JAVA
import java.io.IOException;
import java.nio.ByteBuffer;
import java.nio.channels.FileChannel;
import java.nio.file.Path;
import java.nio.file.StandardOpenOption;

public class FileChannelExample {
    public static void main(String[] args) throws IOException {
        Path path = Path.of("data.txt");

        try (FileChannel channel = FileChannel.open(path, 
                StandardOpenOption.CREATE,
                StandardOpenOption.READ,
                StandardOpenOption.WRITE)) {

            // Записываем строку
            ByteBuffer buffer = ByteBuffer.wrap("Hello FileChannel!\n".getBytes());
            channel.write(buffer);

            // Читаем с начала
            buffer.clear();
            channel.position(0);
            channel.read(buffer);

            buffer.flip();
            while (buffer.hasRemaining()) {
                System.out.print((char) buffer.get());
            }
        }
    }
}

```

### Zero-copy копирование
```JAVA
try (FileChannel in = FileChannel
	.open(Path.of("source.dat"), StandardOpenOption.READ);
     
     FileChannel out = FileChannel
     .open(Path.of("target.dat"), StandardOpenOption.CREATE, StandardOpenOption.WRITE)
) {

    long size = in.size();
    in.transferTo(0, size, out); // копирование напрямую, без буферов
}
```
