До JDK 9 у нас был один способ пройти по стеку:
- Вызвать `Thread.currentThread().getStackTrace()`.
    
- Или перехватить исключение и вызвать `Throwable.getStackTrace()`.
    
Обе эти штуки возвращают массив `StackTraceElement[]`.  

Но есть проблема:
- **Тяжело** — возвращается целый массив, даже если тебе нужно только пару фреймов сверху.
    
- **Неудобно** — нельзя просто «прогуляться» по стеку потоково, приходится работать с массивом.
    
- **Медленно** — вся информация собирается сразу, даже если нужна только часть.
    
- **Гибкости ноль** — ты не можешь сказать JVM: «Дай мне только имена методов без номеров строк» или «дай максимум 3 фрейма».
    

👉 ⭐Поэтому в JDK 9 придумали **`StackWalker`**. 
Он решает всё это:
- ленивый обход (не тащит сразу весь стек),
    
- гибкость через `Option`,
    
- аккуратный API в стиле стримов/функционального программирования.

***
# 📕 1. StackWalker
`StackWalker` создаётся через фабричный метод:
```java
StackWalker walker = StackWalker.getInstance();
```

Дальше у него есть главный метод — `walk(Function<Stream<StackFrame>, T> action)`:
- он даёт тебе `Stream<StackFrame>`,
    
- ты работаешь с ним как с обычным стримом,
    
- результат возвращается в `T` (например, список строк).

```java
StackWalker walker = StackWalker.getInstance();
List<String> top3 = walker.walk(stream ->
    stream.limit(3)
          .map(StackFrame::getMethodName)
          .toList()
);

System.out.println(top3);

// Вывод
[main, myMethod, helper]
```

# 📕 2. StackFrame
`StackWalker.StackFrame` — это «обёртка» над одним фреймом стека.  

У него есть методы:
- `getClassName()`
    
- `getMethodName()`
    
- `getLineNumber()`
    
- `getByteCodeIndex()`
    
- `isNativeMethod()`
    
- `getDeclaringClass()`

Это типа расширенный аналог `StackTraceElement`, но гибче и удобнее.


# 📕 3. Перечисление `StackWalker.Option`

Это вообще главная фишка. 

При создании `StackWalker` ты можешь указать, что именно тебе нужно:
- `SHOW_HIDDEN_FRAMES` — включить «скрытые» фреймы (например, из лямбд и анонимных классов).
    
- `RETAIN_CLASS_REFERENCE` — вместо имён классов возвращать настоящие `Class<?>`. Очень полезно, если ты хочешь работать с рефлексией, а не строками.
    
- `SHOW_REFLECT_FRAMES` — включить вызовы через рефлексию (обычно они скрыты).

```java
StackWalker walker = StackWalker.getInstance(
        Set.of(StackWalker.Option.RETAIN_CLASS_REFERENCE));

walker.forEach(frame -> {
    Class<?> cls = frame.getDeclaringClass();
    System.out.println(cls.getName() + " :: " + frame.getMethodName());
});

```
