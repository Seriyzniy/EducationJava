
> [!NOTE] Интерфейс **`Cloneable`** — это ==_маркерный интерфейс_== из `java.lang`, не содержащий методов. 
> 

***

> [!NOTE] ⭐`Клонировать можно только` `классы`, `реализующие` интерфейс `Cloneable`.

Он служит ==индикатором для метода== `Object.clone()`, что данный класс ==**разрешает копирование объектов побитовым копированием**==. 

❗❗❗Если класс НЕ реализует `Cloneable`, вызов `clone()` приведёт к `CloneNotSupportedException`.

## Интерфейс
```JAVA
package java.lang;

/**
 * Marker interface - указывает, что объект поддерживает клонирование.
 */
public interface Cloneable {
    // ❗❗Нет методов!❗❗
    // Используется методом Object.clone() для проверки:
    // если объект реализует Cloneable — разрешаем клонирование,
    // иначе бросаем CloneNotSupportedException.
}
```

> [!NOTE] ⭐При создании `клона конструктор` клонируемого объекта `НЕ вызывается`

## 💡 Особенности 💡
☢❌**Клонирование в Java — опасная операция**, потому что:
- При поверхностном (shallow) клонировании копируются только **примитивы**, а ==ссылки указывают на те же объекты==. Изменения через клон повлияют и на оригинал.
    
- Если объект владеет ресурсами (например, поток ввода-вывода), то после клонирования оба объекта будут работать с одним и тем же ресурсом → возможны ошибки.
    

Поэтому:
- `clone()` в `Object` объявлен как **protected** и напрямую недоступен.
    
- Чтобы клонировать, класс должен:
    1. Реализовать `Cloneable`.
    2. Переопределить `clone()` (или вызвать его через свой метод).

***

## ✅ Best Practices ✅
- Переопределяйте `clone()` с модификатором `public`.
	
- Вызывайте `super.clone()` для получения "базовой копии".
	
- Для mutable-полей вручную делайте глубокое копирование.

***
## Пример
### Примитивные поля
Если у класса только `int`, `double` и т.п. → достаточно обычного `super.clone()`.

```java
class Point implements Cloneable {
    int x, y;

    @Override
    public Point clone() {
        try {
            return (Point) super.clone(); // поверхностное клонирование достаточно
        } catch (CloneNotSupportedException e) {
            throw new AssertionError(); // не должно происходить
        }
    }
}
```
### Ссылочные поля (объекты внутри объекта)
Тут нужен **глубокий клон**: клонировать вложенные объекты вручную.
```java
class Address implements Cloneable {
    String city;

    @Override
    public Address clone() {
        try {
            return (Address) super.clone();
        } catch (CloneNotSupportedException e) {
            throw new AssertionError();
        }
    }
}

class Person implements Cloneable {
    String name;
    Address address; // ссылочный объект

    @Override
    public Person clone() {
        try {
            Person copy = (Person) super.clone();
            copy.address = address.clone(); // ⭐глубокое копирование⭐
            return copy;
            
        } catch (CloneNotSupportedException e) {
            throw new AssertionError();
        }
    }
}
```

### **Альтернативные подходы (часто лучше `clone()`)**

- 🔶**Конструктор копирования**
```java
class Person {
    String name;
    Address address;

    Person(Person other) {
        this.name = other.name;
        this.address = new Address(other.address);
    }
}
```

- 🔶**Статический фабричный метод** (`Person.copyOf(original)`).
    
- 🔶**Сериализация/библиотеки** (например, `SerializationUtils.clone()` из Apache Commons Lang).
