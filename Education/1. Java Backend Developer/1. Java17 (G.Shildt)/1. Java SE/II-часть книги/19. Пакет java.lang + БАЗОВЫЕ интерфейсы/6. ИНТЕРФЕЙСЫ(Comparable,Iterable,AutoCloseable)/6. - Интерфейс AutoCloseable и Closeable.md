
> [!NOTE] Контракт
> «Я ресурс, меня нужно закрывать после использования».

### Closeable
В интерфейсе `Closeable` тоже определен метод `close()` . 
Объекты класса, реализующие `Closeable`, могут быть закрыты.
```java
public interface Closeable extends AutoCloseable {
    public void close() throws IOException;
}
```

### AutoCloseable
```java
public interface AutoCloseable {
    void close() throws Exception;
}
```
Интерфейс `AutoCloseable` обеспечивает ==поддержку оператора== `try с ресурсами`, который реализует то, что иноrда называют автоматическим управлением ресурсами

Использовать в операторе `try с ресурсами` можно только объекты классов, которые реализуют `AutoCloseable`. 

В интерфейсе `AutoCloseable` определен ==единственный метод== `close()` :
```java
void close() throws Exception
```

Метод `close()` закрывает вызывающий объект, ==освобождая все ресурсы, которые тот моr удерживать.== Он автоматически вызывается в конце оператора try с ресурсами, что устраняет необходимость в явном вызове `close()`. 

Интерфейс `AutoCloseable` реализуется несколькими классами, в том числе всеми классами ввода-вывода, открывающими поток, который можно закрыть.

### ❗❗Разница между `AutoCloseable` и `Closeable`:

- `Closeable.close()` может только `IOException`.
	
- `AutoCloseable.close()` — любой `Exception`.  
	Поэтому для API рекомендуется использовать `AutoCloseable`.

***
# Пример
```java
class MyResource implements AutoCloseable {
    public void work() {
        System.out.println("Работаю с ресурсом...");
    }

    @Override
    public void close() {
        System.out.println("Ресурс закрыт!");
    }
}

public class TestAutoCloseable {
    public static void main(String[] args) {
        try (MyResource r = new MyResource()) {
            r.work();
        }
    }
}

```
