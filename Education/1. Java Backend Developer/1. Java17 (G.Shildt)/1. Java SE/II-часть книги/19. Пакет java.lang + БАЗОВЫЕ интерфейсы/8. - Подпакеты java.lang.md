## 1. `java.lang.annotation`

- Поддержка аннотаций.
    
- Здесь лежат:
    - `Annotation` (базовый интерфейс для всех аннотаций),
        
    - `ElementType` (где можно использовать аннотацию — класс, метод, поле…),
        
    - `RetentionPolicy` (жизненный цикл аннотации — `SOURCE`, `CLASS`, `RUNTIME`),
        
    - предопределённые аннотации (`@Target`, `@Retention`, `@Inherited`).
        
- Используется повсеместно в фреймворках (Spring, JUnit).
    

---

## 2. `java.lang.constant` (JDK 12+)
- Поддержка **дескрипторов констант** (==API для работы с байткодом==).
    
- Используется в низкоуровневых задачах — инструменты анализа, JVM-интерпретация, генерация кода.
    
- Пример: `ClassDesc`, `MethodTypeDesc`.
    
- Обычному прикладному разработчику почти не нужен.
    
---

## 3. `java.lang.instrument`

- ==API для **инструментирования** байткода во время загрузки==.
    
- Можно модифицировать классы через `ClassFileTransformer`.
    
- Используется в профайлерах, APM (например, в Java Agent для мониторинга).
    
- Находится в модуле `java.instrument`.

---

## 4. `java.lang.invoke`

- ==Поддержка **динамических вызовов**==.
    
- Ключевые классы:
    - `MethodHandle` (альтернатива рефлексии, быстрее и гибче),
        
    - `CallSite`,
        
    - `MethodType`.
        
- ✅Используется для лямбд и динамических языков на JVM.
    
- По сути, это сердце `invokedynamic`.
    

---

## 5. `java.lang.management`

- ==API для **управления JVM**==.
    
- Классы типа `MemoryMXBean`, `ThreadMXBean`, `RuntimeMXBean`.
    
- ✅Можно мониторить память, потоки, загрузку классов.
    
- Используется в мониторинге (например, JConsole, VisualVM).
    
- Находится в модуле `java.management`.
    

---

## 6. `java.lang.module` (JDK 9+)

- ==API для работы с **модульной системой JPMS**.==
    
- Классы: `ModuleDescriptor`, `ModuleReference`, интерфейсы `ModuleFinder`, `ModuleReader`.
    
- 💡❓Позволяет программно читать и анализировать `module-info.class`.
    
- В реальной жизни — ==редкая штука==, нужна в контейнерах/системах плагинов.
    

---

## 7. `java.lang.ref`

- ==Поддержка **особых типов ссылок**:==
    - `SoftReference`,
        
    - `WeakReference`,
        
    - `PhantomReference`,
        
    - `ReferenceQueue`.
        
- Используются для кешей, финализации объектов и управления памятью.
    
- Пример: `WeakHashMap` использует `WeakReference` для ключей.

---

## 8. `java.lang.reflect`

- ⭐==Рефлексия==.⭐
    
- ✅Классы: `Field`, `Method`, `Constructor`, `Array`.
    
- Интерфейсы: `AnnotatedElement`, `Member`, `Type`.
    
- Используется для динамического анализа и изменения классов во время выполнения.
    
- 🧠Основа для фреймворков (`Spring`, `Hibernate`).
