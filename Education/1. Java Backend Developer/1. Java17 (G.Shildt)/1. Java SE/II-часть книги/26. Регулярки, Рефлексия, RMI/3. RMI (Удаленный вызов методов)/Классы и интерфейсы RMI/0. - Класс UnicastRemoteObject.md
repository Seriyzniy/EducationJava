
> [!NOTE] `UnicastRemoteObject` — это `базовый класс` для `удалённых объектов`, 
> которые взаимодействуют через **одноадресные (point-to-point)** TCP-соединения.  
> 
> Он обеспечивает:
> - ❗`создание слушающего сокета` на сервере;
> - экспорт объекта (т.е. `регистрацию его в RMI runtime`, А НЕ РЕГИСТРАЦИЯ В Реестре RMI);
> - ❗`генерацию stub’а `**(заглушки)** для клиента.
>
>💡Используется, когда нужно создать объект, доступный удалённо через RMI без участия активационной системы (в отличие от старого `Activatable`).
>

## Конструктор - вызывается в наследниках
```java
// Создаёт и экспортирует удалённый объект на случайно выбранном TCP-порту.
protected UnicastRemoteObject() throws RemoteException

// Создаёт и экспортирует удалённый объект, привязанный к указанному TCP-порту.
protected UnicastRemoteObject(int port) throws RemoteException

// Создаёт и экспортирует удалённый объект, используя пользовательские фабрики сокетов.
protected UnicastRemoteObject(int port,
                              RMIClientSocketFactory csf,
                              RMIServerSocketFactory ssf)
        throws RemoteException
```

📝 **Пояснение:**
- `port` — локальный порт, на котором сервер будет слушать входящие вызовы (0 означает автоматический выбор порта).
    
- `csf` — клиентская фабрика сокетов (`RMIClientSocketFactory`), используется для настройки клиентских соединений.
    
- `ssf` — серверная фабрика сокетов (`RMIServerSocketFactory`), используется для создания серверных сокетов (например, SSL).

## Методы

### Экспорт объектов

```java
//Экспортирует существующий удалённый объект, делая его доступным для вызовов по сети.
public static Remote exportObject(
	Remote obj, 
	int port) throws RemoteException
	
	
// Экспортирует объект с указанием пользовательских фабрик сокетов.
public static Remote exportObject(Remote obj,
                                  int port,
                                  RMIClientSocketFactory csf,
                                  RMIServerSocketFactory ssf)
        throws RemoteException

// Отменяет экспорт объекта, делая его больше недоступным для удалённых вызовов.
public static boolean unexportObject(
	Remote obj, 
	boolean force) throws NoSuchObjectException

// Возвращает stub (заглушку), соответствующую данному удалённому объекту.
public static RemoteStub toStub(Remote obj) throws NoSuchObjectException
```

### Методы экземпляра
```java
// Возвращает порт, к которому привязан данный `UnicastRemoteObject`.
```

## 💡 Особенности и подводные камни 💡

1️⃣ 💡**Автоматическая регистрация слушателя.**  
Конструктор `UnicastRemoteObject` сразу экспортирует объект — он начинает слушать порт немедленно. (см [[1. - Введение в RMI]])

2️⃣ ⭐⭐**Необходимость вызова `super()` в конструкторе.**  
Если вы наследуете класс от `UnicastRemoteObject`, нужно обязательно вызвать `super()` в конструкторе, чтобы объект стал доступным по RMI.

3️⃣ 💡❗**Альтернатива через `UnicastRemoteObject.exportObject()`.**  
Если класс **не может наследовать `UnicastRemoteObject`** (например, уже есть родитель), можно вручную экспортировать экземпляр через статический метод `exportObject()`.

4️⃣ **Деэкспорт с `unexportObject()`.**  
При завершении работы нужно корректно снимать объект с регистрации, чтобы освободить порт.

5️⃣ 💡**Не сериализуется напрямую.**  
Сами `UnicastRemoteObject` не передаются по сети — передаются их stub’ы, которые сериализуются.

6️⃣ **Сокет-фабрики сериализуемы.**  
Если указаны `RMIClientSocketFactory` и `RMIServerSocketFactory`, они должны быть сериализуемыми, т.к. заглушка может передаваться клиенту по сети.

---
## 💎 Best Practices 💎

✅ 💡**Используй `exportObject()`**, если нужно сделать удалённым объект, унаследованный от другого класса.  

✅ ❗❗❗**Вызывай `unexportObject()`** ==при завершении работы== сервера или в `shutdown hook`.  

✅ ❗❗❗**Явно указывай порт** для предсказуемости (особенно при firewall).  

✅ **Не держи `UnicastRemoteObject` в статике** без необходимости — это может блокировать завершение JVM.  

✅ **Используй SSL-фабрики (`SslRMIClientSocketFactory` и `SslRMIServerSocketFactory`)** для безопасного RMI. (javax / jakatra)

## Пример
```java
import java.rmi.*;
import java.rmi.registry.LocateRegistry;
import java.rmi.registry.Registry;
import java.rmi.server.UnicastRemoteObject;

// Интерфейс
public interface HelloService extends Remote {
    String sayHello(String name) throws RemoteException;
}

// Реализация
public class HelloServiceImpl 
	extends UnicastRemoteObject 
	implements HelloService 
{

    protected HelloServiceImpl() throws RemoteException {
        super(1099); // экспорт и привязка к порту 1099
    }

    @Override
    public String sayHello(String name) throws RemoteException {
        return "Hello, " + name + "!";
    }

    public static void main(String[] args) throws Exception {
        HelloService service = new HelloServiceImpl();
        Registry registry = LocateRegistry.createRegistry(1099);
        registry.rebind("HelloService", service);
        System.out.println("Server ready");
    }
}
```

### 📍 **Клиент:**
```java
import java.rmi.registry.LocateRegistry;
import java.rmi.registry.Registry;

public class HelloClient {
    public static void main(String[] args) throws Exception {
        Registry registry = LocateRegistry
	        .getRegistry(
		        "localhost", 
		        1099);
		        
        HelloService service = (HelloService) registry.lookup("HelloService");
        System.out.println(service.sayHello("Seriy"));
    }
}
```
