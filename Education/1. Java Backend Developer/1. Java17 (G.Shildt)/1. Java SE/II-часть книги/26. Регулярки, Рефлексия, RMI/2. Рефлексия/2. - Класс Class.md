
> [!NOTE] Класс `Class` `представляет` метаинформацию (`метаданные`) `о типе`
> класс, интерфейс, массив, примитив или аннотацию.
> 
> Это ключевой элемент механизма **рефлексии** в Java — позволяет исследовать структуру и поведение классов во время выполнения (runtime).

Реализует интерфейсы:
- `Serializable`
- `GenericDeclaration`
- `Type`
- `AnnotatedElement`

## Фабричные методы

```java
// Возвращает объект Class для данного типа (в том числе примитивов, интерфейсов, массивов).
public static Class<?> forName(String className) throws ClassNotFoundException

// Пояснение:
// 💡className — полное имя класса с пакетом (например, "java.util.ArrayList").
// Загружает класс с помощью системного загрузчика (ClassLoader).



// Возвращает объект Class для заданного типа с указанием загрузчика и параметров инициализации.
public static Class<?> forName(
	String name, 
	boolean initialize, 
	ClassLoader loader) throws ClassNotFoundException

// Пояснение:
// 💡name — имя класса,
// 💡❓initialize — нужно ли инициализировать класс при загрузке,
// loader — загрузчик классов, который будет использоваться.
```

## Методы
### Получение информации о классе
```java
// Возвращает имя класса (включая пакет).
public String getName()

// Возвращает простое имя класса без пакета.
public String getSimpleName()

// Возвращает каноническое имя класса (например, "java.util.List").
public String getCanonicalName()


// Возвращает имя пакета, которому принадлежит класс.
public Package getPackage()

// Возвращает суперкласс данного класса (или null, если это Object или интерфейс).
public Class<? super T> getSuperclass()

// Возвращает интерфейсы, которые реализует данный класс.
public Class<?>[] getInterfaces()



// Проверяет, является ли этот класс интерфейсом.
public boolean isInterface()

// Проверяет, является ли этот класс примитивом.
public boolean isPrimitive()

// Проверяет, является ли этот класс массивом.
public boolean isArray()

// Проверяет, является ли этот класс аннотацией.
public boolean isAnnotation()

// Проверяет, является ли этот класс enum-типом.
public boolean isEnum()



// Проверяет, является ли класс абстрактным, публичным, приватным и т.д.
public int getModifiers()
```

### Работа с конструкторами, методами и полями
```java
// Возвращает публичные конструкторы класса (включая унаследованные).
public Constructor<?>[] getConstructors()

// Возвращает ВСЕ конструкторы, включая приватные.
public Constructor<?>[] getDeclaredConstructors()

// Возвращает конструктор с указанными типами параметров.
public Constructor<T> getConstructor(Class<?>... parameterTypes)

// Возвращает конструктор (в т.ч. приватный) с указанными типами параметров.
public Constructor<T> getDeclaredConstructor(Class<?>... parameterTypes)



// Возвращает все публичные методы (включая унаследованные).
public Method[] getMethods()

// Возвращает все методы, включая приватные.
public Method[] getDeclaredMethods()

// Возвращает метод с конкретным именем и параметрами.
public Method getMethod(String name, Class<?>... parameterTypes)

// Возвращает приватный или любой метод с именем и параметрами.
public Method getDeclaredMethod(String name, Class<?>... parameterTypes)



// Возвращает публичные поля класса (включая унаследованные).
public Field[] getFields()

// Возвращает все поля, включая приватные.
public Field[] getDeclaredFields()

// Возвращает конкретное поле по имени.
public Field getField(String name)

// Возвращает приватное или любое поле по имени.
public Field getDeclaredField(String name)
```

### Работа с объектами и экземплярами
```java
// 💡Создает новый экземпляр класса, используя конструктор без параметров.
public T newInstance() throws InstantiationException, IllegalAccessException

// Проверяет, является ли объект экземпляром данного класса или его подкласса.
public boolean isInstance(Object obj)

// Проверяет, можно ли присвоить объект одного типа переменной другого типа.
public boolean isAssignableFrom(Class<?> cls)
```

## 💡 Особенности и подводные камни 💡

1️⃣ 💡Объекты `Class` ==существуют в единственном экземпляре== 
ДЛЯ КАЖДОГО ==загруженного типа== (singleton per type).  (НЕ ОБЪЕКТА, А КЛАССА)

2️⃣ ❗❗❗Метод `newInstance()` вызывает только **пустой конструктор**; если его нет — возникнет `InstantiationException`.  

3️⃣ ❗❗❗Метод `forName()` может инициализировать класс — при этом вызовется статический блок `static {}`.  

4️⃣ ⭐⭐Рефлексия нарушает инкапсуляцию и может быть небезопасной, особенно при использовании `setAccessible(true)`.  

5️⃣ ⭐⭐Использование ==рефлексии СНИЖАЕТ производительность==, т.к. обходится без прямой компиляции методов.  

6️⃣ В Java 9+ модульная система (JPMS) ограничивает доступ через рефлексию к закрытым членам.

---
## 💎 Best Practices 💎

✅ 💡Использовать `ClassName.class` или `obj.getClass()` вместо `forName()`, если класс известен на этапе компиляции.  

✅ 💡Проверять наличие конструктора перед вызовом `newInstance()`.  

✅ 🧠Кэшировать результаты вызова рефлексии, если операция повторяется часто.

✅ ❗❗❗Не использовать рефлексию для обычных задач — только если нужно обрабатывать неизвестные заранее типы.  

✅ При работе с аннотациями — использовать интерфейсы `AnnotatedElement` и `getAnnotationsByType()`.

## Пример
```java
class Team {
	private String type;
	private List<Student> students;

	public Team() {}
	
	public Team(String type, List<Student> students) {
		super();
		this.type = type;
		this.students = students;
	}

	public String getType() {
		return type;
	}

	public void setType(String type) {
		this.type = type;
	}

	public List<Student> getStudents() {
		return students;
	}

	public void setStudents(List<Student> students) {
		this.students = students;
	}
}

public class MyMain {
	public static void main(String[] args){
		try {
			// Получаем объект Class по имени
	        Class<?> clazz = Class.forName("Team");

	        // Выводим имя класса
	        System.out.println("Имя класса: " + clazz.getName());

	        // Создаем экземпляр через newInstance()
	        Object list = clazz.getDeclaredConstructor().newInstance();

	        // Проверяем тип
	        System.out.println("Это List? " + List.class.isAssignableFrom(clazz));

	        // Выводим все методы
	        for (Method m : clazz.getDeclaredMethods()) {
	            System.out.println("Метод: " + m.getName());
	        }
		} catch (Exception e) {
			e.printStackTrace();
		}
	}
}
```
