# 📕 1. Класс MethodHandle

Является **абстрактным классом** (внутри JVM имеет подклассы, например `DirectMethodHandle`).

> [!NOTE] `MethodHandle` — это **низкоуровневый `объект-ссылка` на: `метод`, `конструктор` или `поле`**, 🧠`аналогичный указателю на функцию` в C.  
> 
> Он позволяет `вызвать метод напрямую`, `БЕЗ РЕФЕЛКСИИ` (`Method.invoke()`), с **более высокой производительностью** и **строгой типизацией**.
> 

Используется как часть API `java.lang.invoke`, введённого в **Java 7** для поддержки invokedynamic и улучшенной оптимизации JVM.

## Фабричные методы для создания
```java
// Получает дескриптор метода экземпляра или статического метода
MethodHandle findVirtual(Class<?> refc, String name, MethodType type);

// Получает дескриптор статического метода
MethodHandle findStatic(Class<?> refc, String name, MethodType type);

// Получает дескриптор конструктора (аналог newInstance)
MethodHandle findConstructor(Class<?> refc, MethodType type);



// Получает дескриптор поля для чтения (геттер)
MethodHandle findGetter(Class<?> refc, String name, Class<?> type);

// Получает дескриптор поля для записи (сеттер)
MethodHandle findSetter(Class<?> refc, String name, Class<?> type);



// Для доступа к приватным или защищённым членам используется lookup
MethodHandles.Lookup lookup = MethodHandles.lookup();
```

💬 _Все эти методы возвращают `MethodHandle` и требуют точного описания типа через `MethodType` (аргументы и возвращаемое значение)._

---
## Методы
### Вызов метода
Основные способы вызвать метод, на который указывает handle
```java
// Универсальный способ вызова
Object invoke(Object... args) throws Throwable;

// Типобезопасный вызов, если сигнатура известна на этапе компиляции
<R> R invokeExact(Object... args) throws Throwable;

// Приведение к другой сигнатуре с автоматическим преобразованием типов
MethodHandle asType(MethodType newType);
```

### Манипуляции с аргументами
Позволяют создавать новые handle с изменёнными параметрами
```java
// Подставляет фиксированные значения для части аргументов
MethodHandle bindTo(Object x);

// Изменяет порядок или тип аргументов
MethodHandle asSpreader(Class<?> arrayType, int arrayLength);
MethodHandle asCollector(Class<?> arrayType, int arrayLength);
MethodHandle dropArguments(int pos, List<Class<?>> valueTypes);
MethodHandle insertArguments(int pos, Object... values);
```

### Информация о типе
Позволяют узнать сигнатуру метода
```java
// Возвращает объект MethodType — сигнатуру метода (типы параметров и возвращаемого значения)
MethodType type();
```
### Прочее
Вспомогательные методы
```java
// Проверяет, эквивалентен ли данный handle другому
boolean equals(Object obj);

// Возвращает hashCode для handle
int hashCode();
```

---
## 💡 Особенности и подводные камни 💡

1️⃣ 💡`MethodHandle` — **быстрее**, чем `Method.invoke()`, потому что компилятор JIT может его **==инлайнить==**.  

2️⃣ ❗❗❗В отличие от рефлексии, `invokeExact()` требует **точного совпадения типов** (`MethodType`).  

3️⃣ Ошибки сигнатур (`WrongMethodTypeException`) возникают **в runtime**, если `MethodType` не совпадает.  

4️⃣ 🧠Нельзя напрямую создать `MethodHandle`; нужен `MethodHandles.Lookup`.  

5️⃣ При доступе к приватным методам нужно использовать `privateLookupIn()`.

---
## 💎 Best Practices 💎

✅ Используй `invokeExact()` при известной сигнатуре для максимальной производительности.  

✅ Для универсальных вызовов — `invoke()`.  

✅ Не злоупотребляй `MethodHandle`, если можно обойтись обычным вызовом метода — он сложен для отладки.  

✅ Используй `bindTo()` для частичного применения аргументов (аналог каррирования).  

✅ Проверяй сигнатуру через `MethodType` перед вызовом.

# 📕 2. Класс `MethodType`

Является **финальным классом** (final), наследуется от `Object`.

> [!NOTE] `MethodType` `описывает` `сигнатуру метода` — то есть `тип возвращаемого` значения и `типы всех параметров`.
> Используется вместе с `MethodHandle`, чтобы JVM знала точное соответствие аргументов и возвращаемого типа при вызове метода.

## Фабричные методы
```java
// Создает MethodType из возвращаемого типа и ТИПОВ параметров
static MethodType methodType(Class<?> rtype, Class<?>... ptypes);

// Создает MethodType БЕЗ параметров
static MethodType methodType(Class<?> rtype);

// Создает MethodType С ОДНИМ параметром
static MethodType methodType(Class<?> rtype, Class<?> ptype);


// Создает MethodType, аналогичный другому, но с другим возвращаемым типом
MethodType changeReturnType(Class<?> nrtype);
```

## Методы
### Получение информации
```java
// Возвращает количество параметров
int parameterCount();

// Возвращает тип параметра по индексу
Class<?> parameterType(int num);

// Возвращает тип возвращаемого значения
Class<?> returnType();

// Возвращает список всех типов параметров
List<Class<?>> parameterList();
```

### Преобразование типа
```java
// Создает новый MethodType с другими параметрами
MethodType changeParameterType(int num, Class<?> nptype);

// Добавляет параметры в конец
MethodType appendParameterTypes(Class<?>... ptypes);

// Удаляет часть параметров
MethodType dropParameterTypes(int start, int end);

// Проверяет совместимость типов (используется при asType())
boolean hasPrimitives();
boolean hasWrappers();
```

### Прочее
```java
boolean equals(Object x);
int hashCode();
String toString();
```

---

## 💡 Особенности и подводные камни 💡

1️⃣ 💡`MethodType` **иммутабелен** — каждый вызов `change*` возвращает новый объект.  

2️⃣ ❗❗❗Типы должны быть **точно такими же**, как в сигнатуре метода (`int` ≠ `Integer`).  
3️⃣ Используется **JVM напрямую** при связывании `MethodHandle`.  

4️⃣ Ошибки сигнатуры (`WrongMethodTypeException`) возникают при несовпадении типов в `invokeExact()`.  

5️⃣ `MethodType` может описывать **любые** методы, включая конструкторы и статические.

---
## 💎 Best Practices 💎

✅ 💡Используй `MethodType.methodType(void.class, String.class)` для читаемости.  

✅ Проверяй количество параметров перед вызовом `invokeExact`.  

✅ Не путай с `java.lang.reflect.Method.getParameterTypes()` — это отдельное API.  

✅ Сохраняй `MethodType` в переменную, чтобы не создавать лишние объекты при каждом вызове.

---
## Пример
```java
import java.lang.invoke.*;

class Example {
    public void hello(String name) {
        System.out.println("Привет, " + name);
    }


    public static void main(String[] args) throws Throwable {
        MethodHandles.Lookup lookup = MethodHandles.lookup();

        // Определяем сигнатуру метода: (String)void
        MethodType type = MethodType.methodType(
	        void.class, 
	        String.class);


        // Получаем handle на метод hello
        MethodHandle mh = lookup.findVirtual(
	        Example.class, 
	        "hello", type);

        Example instance = new Example();

        // Вызов с привязкой экземпляра
        mh.invoke(instance, "Сергей"); // → Привет, Сергей
    }
}

```
