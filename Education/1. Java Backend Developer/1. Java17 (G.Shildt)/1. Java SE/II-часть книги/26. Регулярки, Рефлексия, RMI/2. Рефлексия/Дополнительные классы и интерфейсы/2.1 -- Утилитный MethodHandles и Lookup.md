# 📕 1. Утилитный MethodHandles
Является **финальным классом** (`final`).

> [!NOTE] `MethodHandles` — это `фабрика и утилитарный класс` для работы с `MethodHandle`.
> Он предоставляет `способы получения дескрипторов` `методов`, `конструкторов` и `полей`, 
> а также внутренний класс `Lookup`, через который выполняются все операции поиска.
> 
> 💡Можно считать `MethodHandles` аналогом `java.lang.reflect` для мира `invoke`-API.

## Методы
### Получение Lookup
Используется для начала работы с `MethodHandle`.
```java
// Возвращает Lookup-объект с правами доступа текущего класса
static MethodHandles.Lookup lookup();

// Возвращает Lookup с доступом только к public-членам (аналог Class.getMethod)
static MethodHandles.Lookup publicLookup();

// Создаёт Lookup для конкретного класса с определённым уровнем доступа
static MethodHandles.Lookup privateLookupIn(
	Class<?> targetClass, 
	MethodHandles.Lookup caller);
```

### Работа с методами (через Lookup)
Эти методы доступны через `Lookup`, но перечислим их для полноты.
```java
// Поиск виртуального метода (не static)
MethodHandle findVirtual(Class<?> refc, String name, MethodType type);

// Поиск статического метода
MethodHandle findStatic(Class<?> refc, String name, MethodType type);

// Поиск конструктора
MethodHandle findConstructor(Class<?> refc, MethodType type);

// Поиск поля (геттер/сеттер)
MethodHandle findGetter(Class<?> refc, String name, Class<?> type);
MethodHandle findSetter(Class<?> refc, String name, Class<?> type);
```

### Комбинирование и адаптация
Позволяют создавать сложные цепочки вызовов из нескольких handle
```java
// Создает handle, который вызывает первый, а затем второй
static MethodHandle filterArguments(
	MethodHandle target, 
	int pos, 
	MethodHandle... filters);

// Комбинирует несколько handle в один (аналог композиции функций)
static MethodHandle foldArguments(
	MethodHandle target, 
	MethodHandle combiner);

// Подставляет один handle вместо аргумента другого
static MethodHandle filterReturnValue(
	MethodHandle target, 
	MethodHandle filter);
```

### Создание констант и заглушек
Позволяют получить handle без реальных методов
```java
// Возвращает handle, всегда возвращающий заданное значение
static MethodHandle constant(Class<?> type, Object value);

// Возвращает handle, выбрасывающий указанное исключение
static MethodHandle throwException(
	Class<?> returnType, 
	Class<? extends Throwable> exType);
```

### Прочие служебные методы
```java
// Создает handle, который возвращает объект класса
static MethodHandle identity(Class<?> type);

// Возвращает handle без аргументов, который выбрасывает исключение
static MethodHandle empty(MethodType type);
```

---
## 💡 Особенности и подводные камни 💡

1️⃣ ⭐⭐⭐Все реальные операции выполняются через `MethodHandles.Lookup`, а не напрямую через `MethodHandles`.  

2️⃣ 💡`publicLookup()` может видеть **только публичные** члены классов.  

3️⃣ ⭐`lookup()` имеет доступ **к приватным методам текущего класса**.  

4️⃣ 💡Для доступа к приватным членам других классов нужен `privateLookupIn()`.  
5️⃣ Ошибки типа `IllegalAccessException` и `NoSuchMethodException` возникают, если уровень доступа нарушен.  

6️⃣ Методы адаптации (`filter*`, `foldArguments`) позволяют **функционально комбинировать** вызовы — часто используются в JVM-интернах и фреймворках.

---
## 💎 Best Practices 💎

✅ 💡Используй `MethodHandles.lookup()` внутри класса, чтобы иметь доступ к приватным методам.  

✅ Для внешних классов — `privateLookupIn()` или `publicLookup()`.  

✅ Проверяй типы через `MethodType` перед связыванием handle.  

✅ Используй `MethodHandles.constant()` и `throwException()` при написании функциональных композиций.  

✅ 💡Не создавай Lookup глобально — держи его в пределах нужного класса (для безопасности).

# 📕 2. Вложенный класс `MethodHandles.Lookup`


> [!NOTE] `MethodHandles.Lookup` — это `вложенный статический класс` 
> Внутри `MethodHandles`, который служит `точкой входа для получения` `MethodHandle`.
> 
> Он инкапсулирует _доступный контекст_ — то есть права, с которыми можно искать и вызывать методы, конструкторы и поля.
> 
> 💡Фактически, `Lookup` — это аналог отражения (`Field`, `Method`, `Constructor`), но в более быстром и типобезопасном виде

Конструктор скрыт — экземпляры создаются только через статические фабрики `MethodHandles.lookup()`, `MethodHandles.publicLookup()` и др.

## Фабричные методы получения экземпляра
```java
// Возвращает Lookup ❗с правами доступа того класса, откуда вызван метод❗
// Имеет полный доступ ко всем членам класса.
public static MethodHandles.Lookup lookup();

// Возвращает Lookup, который имеет доступ только к public-членам всех классов.
public static MethodHandles.Lookup publicLookup();
```

## Методы
### Поиск методов, конструкторов и полей
```java
// Находит MethodHandle для вызова экземплярного метода (аналог invokeVirtual)
public MethodHandle findVirtual(
	Class<?> refc, 
	String name, 
	MethodType type);

// Находит MethodHandle для вызова статического метода
public MethodHandle findStatic(
	Class<?> refc, 
	String name, 
	MethodType type);

// Находит MethodHandle для вызова конструктора
public MethodHandle findConstructor(Class<?> refc, MethodType type);


// Находит MethodHandle для обращения к экземплярному полю
public MethodHandle findGetter(
	Class<?> refc, 
	String name, 
	Class<?> type);
	
public MethodHandle findSetter(
	Class<?> refc, 
	String name, 
	Class<?> type);

// Находит MethodHandle для обращения к статическому полю
public MethodHandle findStaticGetter(
	Class<?> refc, 
	String name, 
	Class<?> type);
	
	
public MethodHandle findStaticSetter(
	Class<?> refc, 
	String name, 
	Class<?> type);
```

### Специальные методы
```java
// Получение MethodHandle для вызова приватных методов через привилегированный Lookup
public MethodHandle findSpecial(
	Class<?> refc, 
	String name, 
	MethodType type, 
	Class<?> specialCaller);

// Проверка, имеет ли текущий Lookup нужные права
public int lookupModes();

// Возвращает класс, к которому привязан данный Lookup
public Class<?> lookupClass();
```

## 💡 Особенности и подводные камни 💡
1️⃣💡 Lookup строго контролирует доступ — вы ==НЕ СМОЖЕТЕ вызвать приватный метод другого класса==.

2️⃣ ❗❗❗Режим доступа фиксируется при создании объекта Lookup (невозможно изменить).  

3️⃣ `MethodHandles.publicLookup()` безопасен, но сильно ограничен.  

4️⃣ `MethodHandles.lookup()` может «видеть» приватные члены, если вызван из того же класса.  

5️⃣ При работе с модулями Java (JPMS) Lookup подчиняется модульным границам — нельзя обойти `module-info.java`.

---
## 💎 Best Practices 💎

✅ Используй `publicLookup()` для безопасных операций на публичных API.  

✅ Используй `lookup()` только внутри класса, если нужно получить доступ к его приватным членам.  

✅ Храни Lookup как `private static final` поле, чтобы не создавать его повторно.  

✅ Не передавай Lookup за пределы модуля, если он имеет привилегированный доступ.

# Пример
```java
import java.lang.invoke.*;

class Example {
    private void greet(String name) {
        System.out.println("Привет, " + name);
    }


    public static void main(String[] args) throws Throwable {
        // 🟢Получаем Lookup с доступом 
        // к приватным методам текущего класса
        MethodHandles.Lookup lookup = MethodHandles.lookup();

        // Описываем сигнатуру метода greet: (String)void
        MethodType type = MethodType.methodType(
	        void.class, 
	        String.class);


        // 🟢Получаем дескриптор приватного метода
        MethodHandle handle = lookup.findVirtual(
	        Example.class, 
	        "greet", type);

        Example instance = new Example();

        // Вызываем метод через handle
        handle.invoke(instance, "Сергей");
    }
}
```
