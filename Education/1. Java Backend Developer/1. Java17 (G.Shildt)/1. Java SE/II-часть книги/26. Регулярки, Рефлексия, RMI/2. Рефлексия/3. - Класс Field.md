
> [!NOTE] Класс `Field` `представляет` собой отражение (`рефлективную модель`) `поля класса` или `интерфейса`.
> Позволяет `получать информацию о поле` (тип, модификаторы, аннотации) 
> и 
> `читать/изменять` его значение ==во время выполнения==.

Используется совместно с классом `Class` для динамического анализа и манипуляций с объектами.

Реализует интерфейсы:
- `Member`
- `AnnotatedElement`
- `GenericDeclaration`

---
## Фабричные методы класса Class для создания Field:
```java
// Конструктор приватный. 
// Экземпляры Field создаются JVM и возвращаются методами класса Class.
// Например: clazz.getDeclaredField("fieldName")

// Для получения объектов Field используются фабричные методы класса Class:
public Field getField(String name)
public Field getDeclaredField(String name)
public Field[] getFields()
public Field[] getDeclaredFields()

// Пояснение:
// getField()         — возвращает только PUBLIC поля (включая унаследованные);

// getDeclaredField() — возвращает поле независимо от модификатора (только в текущем классе);
```

## Методы
### Основная информация о поле
```JAVA
// Возвращает имя поля.
public String getName()

// Возвращает тип поля.
public Class<?> getType()

// Возвращает модификаторы доступа (public, private, static и т.д.) в виде битовой маски.
public int getModifiers()

// Проверяет, является ли поле синтетическим (создано компилятором, а не программистом).
public boolean isSynthetic()

// Возвращает класс, в котором это поле объявлено.
public Class<?> getDeclaringClass()

// Возвращает тип поля с учетом дженериков.
public Type getGenericType()
```

### Работа со значениями полей
```java
// 🟢Возвращает значение поля для указанного объекта.🟢
public Object get(Object obj) throws IllegalAccessException

// Устанавливает новое значение поля для указанного объекта.
public void set(Object obj, Object value) throws IllegalAccessException

// Для примитивных типов существуют специализированные методы:
public boolean getBoolean(Object obj)

public void setBoolean(Object obj, boolean value)
public void setByte(Object obj, byte value)
public void setChar(Object obj, char value)
public void setShort(Object obj, short value)
public void setInt(Object obj, int value)
public void setLong(Object obj, long value)
public void setFloat(Object obj, float value)
public void setDouble(Object obj, double value)
```

### Управление доступом
```java
// Устанавливает флаг доступности (можно ли обращаться к приватным полям).
public void setAccessible(boolean flag) throws SecurityException

// Проверяет, имеет ли поле установленный флаг доступности.
public boolean canAccess(Object obj)
```

### Работа с аннотациями
```java
// Возвращает все аннотации, присутствующие на поле.
public Annotation[] getAnnotations()

// Возвращает конкретную аннотацию, если она есть.
public <T extends Annotation> T getAnnotation(Class<T> annotationClass)

// Проверяет, аннотировано ли поле указанной аннотацией.
public boolean isAnnotationPresent(Class<? extends Annotation> annotationClass)
```

### Сравнение и хэш-коды
```java

// Возвращает хэш-код поля.

// Возвращает строковое представление (тип, класс и имя поля).
public String toString()
```

## 💡 Особенности и подводные камни 💡

1️⃣ ⭐⭐`Field` позволяет получить и изменить **приватные** поля, если вызвать `setAccessible(true)` (до Java 9). 
❗❗В Java 9+ модульная система может блокировать это.  

2️⃣ ❗❗❗Работа с `setAccessible(true)` потенциально нарушает инкапсуляцию и может быть запрещена `SecurityManager`.  

3️⃣ ❗❗❗Если поле статическое, то при вызове `get()` и `set()` можно передавать `null` вместо объекта.  

4️⃣ 💡Типы должны строго совпадать — при попытке установить несовместимое значение будет выброшено `IllegalArgumentException`.  

5️⃣ 🧠Операции через рефлексию работают медленнее, чем прямой доступ к полям.  

6️⃣ Примитивные поля требуют использования специализированных методов `getInt`, `setInt` и т.п., иначе произойдет автоупаковка/распаковка.  

7️⃣ Рефлексия НЕ ОБХОДИТ final-модификатор на уровне JVM — изменить `final`-поле можно, но это не гарантируется везде и небезопасно.

---
## 💎 Best Practices 💎

✅ Использовать `getDeclaredField()` при необходимости доступа к приватным полям.  

✅ 💡Проверять наличие поля перед доступом к нему через `Class.getDeclaredField()`.  

✅ 💡💡Не злоупотреблять `setAccessible(true)` — использовать только в тестах или при необходимости сериализации/ORM.

✅ Кэшировать `Field`-объекты при множественном доступе для повышения производительности.  

✅ Проверять совместимость типов перед `set()` или `get()`.  

✅ Для чтения аннотаций на полях использовать `getAnnotation()`.

## Пример
```java
import java.lang.reflect.Field;

class Person {
    private String name = "Alex";
    public int age = 25;
}

public class MyMain {
	public static void main(String[] args){
		try {
			Person p = new Person();

	        // Получаем объект Class
	        Class<?> clazz = p.getClass();

	        // Получаем приватное поле "name"
	        Field nameField = clazz.getDeclaredField("name");

	        // Разрешаем доступ к приватному полю
	        nameField.setAccessible(true);

	        // Читаем значение
	        System.out.println("До: " + nameField.get(p));

	        // Изменяем значение
	        nameField.set(p, "Sergey");

	        // Проверяем изменения
	        System.out.println("После: " + nameField.get(p));

	        // Получаем публичное поле "age"
	        Field ageField = clazz.getField("age");
	        System.out.println("Возраст: " + ageField.get(p));
		} catch (Exception e) {
			e.printStackTrace();
		}
	}
}

До: Alex

После: Sergey

Возраст: 25
```
