
> [!NOTE] `HttpServletResponseWrapper` — это **обёртка для объекта HttpServletResponse**.  
> 
> Позволяет `изменять поведение HTTP-ответа` 
> (например, логировать, модифицировать заголовки, статус или тело ответа) без переписывания всего интерфейса.
> 
> Реализует интерфейс `HttpServletResponse` и наследует `jakarta.servlet.ServletResponseWrapper`.
> 

## Конструктор
```java
// Создаёт обёртку вокруг исходного HttpServletResponse.
// response — оригинальный объект ответа, который будет обёрнут.
public HttpServletResponseWrapper(HttpServletResponse response)
```

## Методы
Как и у RequestWrapper, все методы ==**делегируют** вызовы внутреннему объекту== `response`.  

Разработчик может переопределить их для изменения поведения.
### Методы доступа к оригинальному ответу
```java
// Возвращает оригинальный объект HttpServletResponse.
@Override
public HttpServletResponse getResponse()

// Устанавливает новый HttpServletResponse в качестве внутреннего.
@Override
public void setResponse(ServletResponse response)
```

### Пример переопределяемых методов
```java
// Можно переопределить для изменения статуса ответа
@Override
public void setStatus(int sc)

// Можно переопределить для добавления заголовков
@Override
public void addHeader(String name, String value)

// Можно переопределить для изменения тела ответа (через getWriter)
@Override
public PrintWriter getWriter() throws IOException
```

## 💡 Особенности и подводные камни 💡

1️⃣ ⭐⭐Часто используется в фильтрах для **логирования** или **перехвата тела ответа**. 

2️⃣ ⭐⭐⭐⭐Чтобы изменить тело ответа, нужно **переопределить `getWriter()` или `getOutputStream()`** и направить поток в буфер.  

3️⃣ ⭐⭐Не забывай вызывать `super.setStatus()` или `super.addHeader()`, чтобы не терять оригинальные данные.  

4️⃣ Используется для внедрения Content Compression, Logging, Auditing и других middleware-задач.  

5️⃣ ❗❗❗Нельзя повторно записывать ответ, если поток уже закрыт — буферизация решает проблему.

---
## 💎 Best Practices 💎

✅ ❗Используй `HttpServletResponseWrapper` для логирования или модификации тела ответа.  

✅ ⭐⭐При перезаписи тела — кэшируй `PrintWriter` в ByteArrayOutputStream.  

✅ ⭐⭐Не забывай вызывать `chain.doFilter()` с оригинальными или модифицированными объектами.  

✅ ❗❗❗Убедись, что не вызываешь `getWriter()` и `getOutputStream()` одновременно.  

✅ Применяй для реализации middleware-поведения (например, GZIP-сжатие).

## Пример
```java
@WebFilter("/*")
public class ResponseCaptureFilter extends HttpFilter {

    @Override
    protected void doFilter(
	    HttpServletRequest request,
        HttpServletResponse response,
        FilterChain chain) throws IOException, ServletException 
    {
        // Создаём буфер для захвата тела ответа
        ByteArrayOutputStream buffer = new ByteArrayOutputStream();
        PrintWriter writer = new PrintWriter(buffer);

        // Оборачиваем ответ
        HttpServletResponseWrapper wrappedResponse = 
	        new HttpServletResponseWrapper(response) 
	    {
            @Override
            public PrintWriter getWriter() {
	            //Сервлет вызовет этот метод для записи, и запишет в 
	            // mok-буфер
                return writer; 
            }
        };

        // Передаём управление дальше
        chain.doFilter(request, wrappedResponse);

        writer.flush();
        String content = buffer.toString(response.getCharacterEncoding());

        System.out.println("➡ Ответ сервера:\n" + content);

        // Отправляем оригинальный ответ пользователю
        response.getWriter().write(content);
    }
}
```
