
> [!NOTE] **`HttpServletRequest`** представляющий **`HTTP-запрос`, полученный сервером от клиента** (браузера, REST-клиента и т.п.).
> Основная цель: предоставить методы `для извлечения ДАННЫХ ЗАПРОСА:`
> - параметров, 
> - заголовков, 
> - cookies, 
> - тела, 
> - сессии, 
> - пути, 
> - метода (GET, POST и т.д.) и прочего контекста HTTP-запроса.

## Интерфейс
```java
public interface HttpServletRequest extends ServletRequest
```
📌 `HttpServletRequest` **наследует интерфейс `ServletRequest`**, добавляя HTTP-специфичные возможности.  
📌 Не реализует других интерфейсов, но тесно связан с `HttpSession`, `Cookie` и `Part`.

## Методы
### Основная информация о запросе
```java
// ❗Возвращает HTTP-метод запроса❗ (GET, POST, PUT, DELETE и т.д.)
String getMethod()


// Возвращает ❗полный URL запроса❗
StringBuffer getRequestURL()

// Возвращает URI запрашиваемого ресурса (без домена)
String getRequestURI()


// Возвращает контекстный путь веб-приложения (например, "/app")
String getContextPath()

// Возвращает путь к сервлету (например, "/user")
String getServletPath()

// Возвращает дополнительный путь после имени сервлета (path info)
String getPathInfo()



// Возвращает ❗строку запроса❗ (❗часть после "?")❗
String getQueryString()



// Возвращает ❗имя хоста клиента❗
String getRemoteHost()

// Возвращает ❗IP-адрес клиента❗
String getRemoteAddr()

// Возвращает ❗порт, с которого клиент❗ отправил запрос
int getRemotePort()



// Возвращает имя сервера, на который пришёл запрос
String getServerName()

// Возвращает порт, на который пришёл запрос (обычно 80 или 443)
int getServerPort()

// Проверяет, был ли запрос выполнен по HTTPS
boolean isSecure()
```

### Работа с заголовками
```java
// Возвращает ❗значение заголовка по имени❗
String getHeader(String name)

// Возвращает перечисление всех заголовков с данным именем
Enumeration<String> getHeaders(String name)

// Возвращает перечисление всех имён заголовков
Enumeration<String> getHeaderNames()



// Возвращает дату из заголовка (в миллисекундах UNIX-времени)
long getDateHeader(String name)

// Возвращает числовое значение из заголовка
int getIntHeader(String name)
```

### Работа с параметрами формы и запроса
```java
// Возвращает ❗значение параметра по имени❗ (например, из формы ?name=John)
String getParameter(String name)

// Возвращает массив всех значений параметра (если он повторяется)
String[] getParameterValues(String name)


// Возвращает карту всех параметров запроса
Map<String, String[]> getParameterMap()

// Возвращает перечисление всех имён параметров
Enumeration<String> getParameterNames()
```

### Работа с телом запроса
_(унаследовано из `ServletRequest` и дополнено HTTP-методами)_
```java
// Возвращает поток ❗для чтения байтов❗ (используется, например, для JSON)
ServletInputStream getInputStream() throws IOException

// Возвращает Reader ❗для чтения текста❗
BufferedReader getReader() throws IOException
```

### Работа с атрибутами (на уровне запроса)
```java
// Устанавливает объект как атрибут запроса
void setAttribute(String name, Object o)

// Возвращает объект-атрибут по имени
Object getAttribute(String name)

// Удаляет атрибут по имени
void removeAttribute(String name)

// Возвращает имена всех атрибутов запроса
Enumeration<String> getAttributeNames()
```

### Работа с cookies и сессиями
```java
// Возвращает массив всех cookies, переданных клиентом
Cookie[] getCookies()



// ❗Возвращает текущую сессию❗, если она существует, ❗или создаёт новую❗
HttpSession getSession()

// ❗Возвращает сессию❗, если она существует, или null, если не существует
HttpSession getSession(boolean create)



// Возвращает идентификатор сессии из запроса
String getRequestedSessionId()

// Проверяет, есть ли в запросе идентификатор сессии
boolean isRequestedSessionIdValid()

// Проверяет, пришёл ли идентификатор сессии через cookie
boolean isRequestedSessionIdFromCookie()

// Проверяет, пришёл ли идентификатор сессии через URL
boolean isRequestedSessionIdFromURL()
```

### Работа с multipart/form-data (загрузка файлов)
```java
// Возвращает часть (Part) по имени из multipart-запроса

// Возвращает все части multipart-запроса
Collection<Part> getParts() throws IOException, ServletException
```

### Работа с локалью и кодировкой
```java
// Возвращает предпочитаемую локаль клиента
Locale getLocale()

// Возвращает все поддерживаемые клиентом локали
Enumeration<Locale> getLocales()

// Устанавливает кодировку для тела запроса (например, UTF-8)
void setCharacterEncoding(String env) throws UnsupportedEncodingException

// Возвращает кодировку тела запроса
String getCharacterEncoding()
```

### Прочие полезные методы
```java
// Запускает тот же процесс аутентификации, который был бы запущен, если бы Запрос предназначен для ресурса, который защищен ограничением безопасности.

boolean isUserInRole(String role)

Principal getUserPrincipal()

String getContentType()

// Возвращает длину тела запроса
int getContentLength()
```

---
## 💡 Особенности и подводные камни 💡

1️⃣ Методы `getParameter()` работают **только для form-urlencoded данных**, а не для `multipart/form-data`.  

2️⃣ Если вызвать `getReader()` или `getInputStream()` несколько раз — будет `IllegalStateException`.  

3️⃣ Нельзя использовать одновременно `getReader()` и `getInputStream()` — нужно выбрать одно.  

4️⃣ 💡Метод `getSession()` **создает сессию**, если её не было — используйте `getSession(false)`, если это нежелательно.  

5️⃣ Значение `getParameter()` не может использоваться для двоичных данных (например, файлов). Для этого применяются `getParts()`.

---
## 💎 Best Practices 💎

✅ Всегда вызывайте `req.setCharacterEncoding("UTF-8")` перед чтением параметров из тела запроса.  

✅ 💡Для загрузки файлов используйте `@MultipartConfig` и `req.getParts()`.  

✅ Проверяйте `req.getMethod()` перед обработкой — разные методы подразумевают разную логику.  

✅ Не храните состояние между запросами — используйте `HttpSession` или базу данных.  

✅ Проверяйте `req.isSecure()` при работе с конфиденциальными данными.

## Пример
```java
import jakarta.servlet.*;
import jakarta.servlet.http.*;
import java.io.*;
import java.util.*;

public class UserInfoServlet extends HttpServlet {

    @Override
    protected void doGet(
	    HttpServletRequest req, 
	    HttpServletResponse resp) throws ServletException, IOException 
	{
        resp.setContentType("text/html;charset=UTF-8");
        PrintWriter out = resp.getWriter();

        // Извлекаем параметры из URL (например, ?name=Ivan&age=20)
        String name = req.getParameter("name");
        String age = req.getParameter("age");

        // Информация о клиенте
        String ip = req.getRemoteAddr();
        String method = req.getMethod();

        out.println("<html><body>");
        out.printf("<h2>Привет, %s!</h2>", name != null ? name : "Гость");
        out.printf("<p>Тебе %s лет.</p>", age != null ? age : "неизвестно");
        out.printf("<p>Твой IP: %s</p>", ip);
        out.printf("<p>Метод запроса: %s</p>", method);
        out.println("</body></html>");
    }
}
```
