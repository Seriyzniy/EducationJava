
> [!NOTE] `HttpServlet` представляет `базовый класс` для создания `HTTP-сервлетов`, 
> т.е. серверных компонентов
>- `обрабатывающих` `HTTP-запросы` (`GET`, `POST`, `PUT`, `DELETE` и т.д.) 
>- и `формирующих HTTP-ответы`.
>
>💡Основная идея: Использование более высокоуровневого класса
>вместо реализации низкоуровневого интерфейса `Servlet`, разработчик может просто **наследовать `HttpServlet` и `переопределить нужные методы`**, такие как `doGet()`, `doPost()` и др.

## Класс
```java
public abstract class HttpServlet extends GenericServlet
```
📌 `HttpServlet` — **абстрактный** класс.  
📌 Он **наследует** `GenericServlet`, который уже реализует интерфейс `Servlet`.  
📌 Таким образом, `HttpServlet` **не реализует напрямую интерфейсы**, но через наследование поддерживает `Servlet`, `ServletConfig` и `Serializable`.

## Конструктор
```java
// Создает новый объект HttpServlet.
// Конструктор не принимает параметров и вызывается контейнером сервлетов при инициализации.
// Обычно не используется напрямую разработчиком.
public HttpServlet()
```

## Методы
Класс `HttpServlet`== **унаследовал** методы== из `GenericServlet` и `Servlet`, такие как `init()`, `destroy()`, `getServletConfig()`, `getServletInfo()`, `service()`, 

но **добавил собственную HTTP-ориентированную реализацию**.
### Основные методы обработки HTTP-запросов
```java
// ❗Обрабатывает HTTP-запрос❗
// Контейнер вызывает этот метод ❗для каждого запроса❗.
// ⭐⭐Определяет тип запроса (GET, POST и т.д.) и вызывает соответствующий doXXX() метод.⭐⭐
protected void service(HttpServletRequest req, HttpServletResponse resp)
    throws ServletException, IOException



// Обрабатывает ❗GET-запросы.❗
// Используется для ПОЛУЧЕНИЯ данных от сервера (например, при загрузке страницы).
protected void doGet(
	HttpServletRequest req, 
	HttpServletResponse resp) throws ServletException, IOException

// Обрабатывает ❗POST-запросы.❗
// Используется для ОТПРАВКИ данных на сервер (например, формы, JSON и т.п.).
protected void doPost(
	HttpServletRequest req, 
	HttpServletResponse resp) throws ServletException, IOException

// Обрабатывает ❗PUT-запросы (ОБНОВЛЕНИЕ ресурса).❗
protected void doPut(
	HttpServletRequest req, 
	HttpServletResponse resp) throws ServletException, IOException

// Обрабатывает ❗DELETE-запросы (удаление ресурса).❗
protected void doDelete(
	HttpServletRequest req, 
	HttpServletResponse resp) throws ServletException, IOException



// Обрабатывает ❗OPTIONS-запросы.❗
// Используется для определения поддерживаемых сервером методов.
protected void doOptions(
	HttpServletRequest req, 
	HttpServletResponse resp) throws ServletException, IOException

// Обрабатывает ❗TRACE-запросы.❗ 
// Возвращает клиенту запрос в том виде, как он был получен сервером.
protected void doTrace(
	HttpServletRequest req, 
	HttpServletResponse resp) throws ServletException, IOException
```

> [!NOTE] 🧠Options и Trace запросы🧠
> **==OPTIONS==** — **метод HTTP, который возвращает информацию о поддерживаемых сервером HTTP-методах для конкретной сущности**. Например, может применяться для получения списка поддерживаемых методов для сущности «пользователь». 
> 
> **==TRACE==** — **метод, который возвращает запрос, отправленный на сервер**. Позволяет клиенту видеть, что действительно было отправлено и как это было преобразовано сервером. Например, может использоваться для диагностики запроса к серверу. 
> 
> Также **TRACE возвращает служебную отладочную информацию** о том, какие данные добавляют или изменяют прокси-серверы в запросе.

### Вспомогательные методы
```java
// Определяет, поддерживает ли этот сервлет метод, отличный от GET/HEAD.
// Обычно используется для контроля кэширования и HEAD-запросов.
```

## 💡 Особенности и подводные камни 💡

1️⃣ `HttpServlet` **не должен создаваться вручную** — ==экземпляры создаются контейнером== (Tomcat, Jetty, и т.д.).  
❗❗Нужно лишь НАСЛЕДОВАТЬСЯ от класса❗❗

2️⃣ 💡❗Методы `doGet`, `doPost` и др. ПО УМОЛЧАНИЮ возвращают **HTTP 405 (Method Not Allowed)**, если не переопределены.  

3️⃣ ❗❗❗Метод `service()` лучше НЕ ПЕРЕОПРЕДЕЛЯТЬ, если нет необходимости менять маршрутизацию запросов.  

4️⃣ ❗❗❗Класс ==НЕ потокобезопасен== — один экземпляр используется для всех запросов, поэтому нельзя хранить состояние в полях экземпляра.  

5️⃣ ⭐⭐При наследовании `HttpServlet` всегда нужно вызывать `super.init()` и `super.destroy()`, если они переопределяются.

---
## 💎 Best Practices 💎

✅ Всегда переопределяйте только нужные `doXXX()` методы.  

✅ Не используйте нестатические поля для хранения данных запроса. (используйте статические) 

✅ ⭐Устанавливайте `Content-Type` в ответе (`resp.setContentType("text/html;charset=UTF-8")`).  

✅ ⭐⭐Для кода ответа используйте константы `HttpServletResponse.SC_...`.  

✅ При обработке POST-запросов проверяйте кодировку (`req.setCharacterEncoding("UTF-8")`).

---
## Пример
```java
import jakarta.servlet.*;
import jakarta.servlet.http.*;
import java.io.*;

public class HelloServlet extends HttpServlet {

    @Override
    protected void doGet(
	    HttpServletRequest req, 
	    HttpServletResponse resp) throws ServletException, IOException 
	{
        // Устанавливаем MIME-тип и кодировку
        resp.setContentType("text/html;charset=UTF-8");
        
        // Пишем ответ в поток
        PrintWriter out = resp.getWriter();
        out.println("<html><body>");
        out.println("<h1>Hello world!</h1>");
        out.println("</body></html>");
    }
}
```
