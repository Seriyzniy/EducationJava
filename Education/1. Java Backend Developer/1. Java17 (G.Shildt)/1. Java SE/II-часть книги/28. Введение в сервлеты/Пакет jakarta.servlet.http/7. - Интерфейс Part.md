
> [!NOTE] Интерфейс **`Part`** представляет собой `часть HTTP-запроса`, `отправленного с типом контента`-  `multipart/form-data`.
> Каждая “`часть`” `соответствует одному элементу формы` (например, загруженному файлу или обычному полю ввода).  
> 
> 💡Используется `для работы с загрузкой файлов через HTML-формы` (`<form enctype="multipart/form-data">`).
> 
> Интерфейс не расширяет других интерфейсов, но тесно связан с `HttpServletRequest` (методы `getPart()` и `getParts()`).

## Интерфейс
```java
package jakarta.servlet.http;

import java.io.InputStream;
import java.io.IOException;
import java.util.Collection;

public interface Part {
	    
	//===========================
	// 🟢 Основные данные о части 🟢
	//===========================
    // Возвращает имя поля формы, которому соответствует эта часть
    // (атрибут name в HTML)
    String getName();

    // Возвращает имя переданного файла (если это файл)
    String getSubmittedFileName();

    // Возвращает размер содержимого части (в байтах)
    long getSize();
    
    
	//===========================
	// 🟢 Работа с содержимым 🟢
	//===========================
    // Возвращает InputStream для чтения содержимого части (например, файла)
    InputStream getInputStream() throws IOException;

    // Записывает содержимое части в указанный файл
    void write(String fileName) throws IOException;

    // Удаляет временные данные, связанные с частью (если есть)
    void delete() throws IOException;


	//===========================
	// 🟢 Работа с заголовками 🟢
	//===========================
    // Возвращает значение заголовка по имени
    String getHeader(String name);

    // Возвращает все значения заголовка по имени
    Collection<String> getHeaders(String name);

    // Возвращает все заголовки части
    Collection<String> getHeaderNames();
}
```

---
## 💡 Особенности и подводные камни 💡

1️⃣ ⭐⭐⭐**Работает только с запросами `multipart/form-data`**, иначе `getPart()` бросит `IllegalStateException`.  

2️⃣ 💡💡**Не хранит файл навсегда** — временные файлы могут быть автоматически удалены контейнером (например, Tomcat).  

3️⃣ 💡**Метод `write()`** может перезаписать существующие файлы, если не указать безопасный путь.  

4️⃣ 🧠**Размер загружаемого файла ограничен** параметрами `@MultipartConfig` (`maxFileSize`, `maxRequestSize`).  

5️⃣ **`getSubmittedFileName()` может вернуть `null`**, если это не файл, а обычное текстовое поле формы.  

6️⃣ **Важно закрывать `InputStream`**, чтобы не возникла утечка ресурсов.

---
## 💎 Best Practices 💎

✅ 💡Используй аннотацию `@MultipartConfig` над сервлетом для включения поддержки multipart-запросов.  

✅ 💡Всегда проверяй `getSubmittedFileName()` перед записью файла — поле формы может быть не файлом.  

✅ 💡Для временных файлов используй уникальные имена (`UUID`, timestamp).  

✅ После использования `delete()` — часть становится недействительной, не пытайся повторно обращаться к ней. 

✅ Проверяй тип содержимого (`getHeader("Content-Type")`) для безопасности (например, запретить `.exe`).

## Пример
```java
import jakarta.servlet.ServletException;
import jakarta.servlet.annotation.MultipartConfig;
import jakarta.servlet.http.*;
import java.io.*;
import java.nio.file.*;

@MultipartConfig(
    fileSizeThreshold = 1024 * 1024,   // 1 MB в памяти, потом на диск
    maxFileSize = 10 * 1024 * 1024,    // 10 MB максимум на файл
    maxRequestSize = 50 * 1024 * 1024  // 50 MB на весь запрос
)
public class FileUploadServlet extends HttpServlet {

    @Override
    protected void doPost(
	    HttpServletRequest req, 
	    HttpServletResponse resp) throws ServletException, IOException 
	{
        // Получаем часть формы с именем "file"
        Part filePart = req.getPart("file");
        String fileName = filePart.getSubmittedFileName();

        // Проверяем имя файла
        if (fileName == null || fileName.isEmpty()) {
            resp.getWriter().write("Файл не выбран!");
            return;
        }

        // Определяем путь для сохранения
        Path uploadDir = Paths.get("uploads");
        Files.createDirectories(uploadDir);

        // Сохраняем файл на диск
        filePart.write(uploadDir.resolve(fileName).toString());

        // Освобождаем ресурсы
        filePart.delete();

        resp.getWriter().write("Файл " + fileName + " успешно загружен!");
    }
}
```
