
> [!NOTE] **`HttpSession`** — интерфейс, предназначенный `для хранения` `данных О СОСТОЯНИИ ПОЛЬЗОВАТЕЛЯ` между HTTP-запросами.
> 
> Основная цель: 
> обеспечить `механизм сессий` в stateless-протоколе HTTP — то есть `позволить серверу “запомнить”` `пользователя` `между запросами` (например, после авторизации, при работе с корзиной покупок и т.д.).

## Интерфейс
```java
public interface HttpSession
```
📌 Не наследует другие интерфейсы.  
📌 Определяет методы для работы с атрибутами сессии, временем жизни и идентификатором.  
📌 Реализацию предоставляет контейнер сервлетов (Tomcat, Jetty и т.п.).

## Методы

### Управление атрибутами сессии
```java
// ❗Добавляет объект в сессию❗ под указанным именем
void setAttribute(String name, Object value)

// Возвращает объект, сохранённый в сессии по имени
Object getAttribute(String name)

// Удаляет атрибут из сессии
void removeAttribute(String name)


// Возвращает перечисление имён всех атрибутов, хранящихся в сессии
Enumeration<String> getAttributeNames()


//❌DEPRECATED❌

// (Старый аналог setAttribute(), поддерживается для обратной совместимости)
@Deprecated
void putValue(String name, Object value)

// (Старый аналог getAttribute())
@Deprecated
Object getValue(String name)

// (Старый аналог removeAttribute())
@Deprecated
void removeValue(String name)

// (Старый аналог getAttributeNames())
@Deprecated
String[] getValueNames()
```

### Управление жизненным циклом сессии
```JAVA
// ❗Уничтожает сессию❗ и удаляет все атрибуты❗
void invalidate()

// Проверяет, действительна ли сессия (false, если была аннулирована)
boolean isNew()


// ❗Возвращает время (в миллисекундах UNIX)❗, когда сессия была создана
long getCreationTime()

// Возвращает ❗время последнего обращения❗ к сессии
long getLastAccessedTime()


// ❗Устанавливает ❗максимальный интервал НЕАТИВНОСТИ❗ (в секундах)
// После этого времени неактивности сессия будет аннулирована
void setMaxInactiveInterval(int interval)

// ❗Возвращает установленный интервал❗ неактивности (в секундах)
int getMaxInactiveInterval()
```

### Идентификатор и контекст
```java
// Возвращает ❗уникальный идентификатор сессии❗
String getId()

// Возвращает ServletContext, к которому относится эта сессия
ServletContext getServletContext()
```

### Репликация и распределённые сессии
```java
// Возвращает объект, привязанный к этой сессии на распределённом сервере (опционально)
Object getAttribute(String name)
```
(Этот метод не выделяется отдельно в API, но сессии могут быть сериализуемыми при кластеризации контейнера.)

## 💡 Особенности и подводные камни 💡

1️⃣ ==**Сессия создаётся автоматически**== при вызове `req.getSession()` — даже если клиент не был авторизован. 
❗Чтобы этого избежать, используйте `req.getSession(false)`.  

2️⃣ ==Данные в сессии **хранятся в памяти сервера**== — не стоит сохранять туда большие объекты.  

3️⃣ 💡При кластеризации или перезапуске приложения содержимое сессии может **теряться**, если не реализована сериализация.  

4️⃣ 💡Cookie `JSESSIONID` используется для связывания клиента с сессией. Если cookies отключены, сервер добавляет `jsessionid` в URL.  

5️⃣ После вызова `invalidate()` сессию использовать **нельзя** — вызов любого метода может вызвать `IllegalStateException`.

---
## 💎 Best Practices 💎

✅ Используйте `req.getSession(false)`, если не хотите создавать сессию без необходимости.  

✅ 💡Всегда вызывайте `session.invalidate()` при выходе пользователя из системы.  

✅ ⭐Храните в сессии **только небольшие объекты** (идентификаторы, имена, флаги).  

✅ ❗❗❗Избегайте хранения `Connection`, `Thread`, `InputStream` и других неконтролируемых ресурсов.  

✅ При работе в кластере используйте сериализуемые (`Serializable`) объекты. 

✅ ⭐⭐⭐Для безопасности настройте `session-timeout` в `web.xml` или через `setMaxInactiveInterval()`.

## Пример
```java
import jakarta.servlet.*;
import jakarta.servlet.http.*;
import java.io.*;

public class LoginServlet extends HttpServlet {

    @Override
    protected void doPost(
	    HttpServletRequest req, 
	    HttpServletResponse resp) throws ServletException, IOException 
	{
        String username = req.getParameter("username");
        String password = req.getParameter("password");

        if ("admin".equals(username) && "1234".equals(password)) {
            // Получаем или создаём сессию
            HttpSession session = req.getSession();
            session.setAttribute("user", username);
            session.setMaxInactiveInterval(600); // 10 минут

            resp.sendRedirect("/home");
        } else {
            resp.sendError(HttpServletResponse.SC_UNAUTHORIZED, "Неверные данные входа");
        }
    }
}
```

## Другой пример — использование сессии для хранения состояния
```java
public class ProfileServlet extends HttpServlet {

    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp)
            throws ServletException, IOException {

        HttpSession session = req.getSession(false);
        resp.setContentType("text/html;charset=UTF-8");
        PrintWriter out = resp.getWriter();

        if (session == null || session.getAttribute("user") == null) {
            out.println("<h3>Пожалуйста, войдите в систему</h3>");
        } else {
	        // Если сессия была установлена, ТО ПОЛУЧАЕМ АТРИБУТ
            String user = (String) session.getAttribute("user");
            out.printf("<h2>Добро пожаловать, %s!</h2>", user);
        }
    }
}

```
