
> [!NOTE] `Cookie` — это класс, `представляющий HTTP-куки` (`cookie`). 
> Он используется `для хранения пар` "==имя–значение==", 
> к`оторые сервер отправляет клиенту` (браузеру), чтобы с`охранить состояние между запросами` (например, для сессий, персонализации, аналитики и т.п.).  
> 
> Не реализует интерфейсы, но является **простым POJO**, предоставляющим API для управления свойствами cookie (время жизни, домен, путь, безопасность и пр.).
> 

## Конструктор
```java
// Создаёт cookie с указанными именем и значением.
// name — имя cookie (не может содержать пробелы, запятые и другие запрещённые символы)
// value — значение cookie (может быть пустым, но не null)
public Cookie(String name, String value)
```

## Методы
### Основные методы доступа к имени и значению
```java
// ❗Возвращает имя cookie❗ (устанавливается при создании и не изменяется)
public String getName()

// Возвращает ❗текущее значение cookie❗
public String getValue()


// ❗Устанавливает новое значение cookie❗
public void setValue(String newValue)
```

### Методы для управления областью видимости
```java
// Возвращает ❗путь, для которого действует cookie❗ (например, "/app")
public String getPath()

// Устанавливает путь, при котором браузер будет отправлять cookie
public void setPath(String uri)



// Возвращает ❗домен, для которого cookie действительно❗ (например, ".example.com")
public String getDomain()

// ❗Устанавливает ❗домен, на который cookie распространяется❗
public void setDomain(String pattern)
```

### Методы для управления временем жизни
```java
// ❗Возвращает время жизни cookie❗ (в секундах)
// 0 — удалить cookie; -1 — cookie исчезает при закрытии браузера
public int getMaxAge()

// ❗Устанавливает время жизни cookie❗ в секундах
public void setMaxAge(int expiry)
```

### Методы безопасности и политики
```java
// Возвращает true, если cookie помечено как "Secure" (только по HTTPS)
public boolean getSecure()

// Устанавливает флаг Secure — cookie будет передаваться только по HTTPS
public void setSecure(boolean flag)



// Возвращает true, если cookie доступно только с HTTP (не доступно JavaScript)
public boolean isHttpOnly()

// Устанавливает флаг HttpOnly — запрещает доступ из JS
public void setHttpOnly(boolean httpOnly)
```

### Методы версии и кодировки
```java
// Возвращает версию cookie по RFC (0 — Netscape, 1 — RFC 2109)
public int getVersion()

// Устанавливает версию cookie
public void setVersion(int v)

// Возвращает комментарий cookie (опционально)
public String getComment()

// Устанавливает комментарий cookie (обычно описание назначения)
public void setComment(String purpose)
```

### Вспомогательные
```java
// Возвращает строковое представление cookie (для отладки)
public String toString()
```

## 💡 Особенности и подводные камни 💡

1️⃣ 💡Имя cookie ==нельзя менять после создания== — можно только значение.  

2️⃣ Cookie ==хранятся в браузере== и ==передаются при каждом запросе== — **не используйте для хранения больших данных**.  

3️⃣ ❗❗❗Значения cookie **НЕ ШИФРУЮТСЯ**❗ по умолчанию — храните там только безопасные токены.  

4️⃣ ❗⭐⭐`setMaxAge(0)` — удаляет cookie;❗ `-1` — cookie исчезает при закрытии браузера.  

5️⃣ При установке `Secure` без HTTPS браузер может проигнорировать cookie.

6️⃣ Разные браузеры имеют ограничения: не более 20–50 cookie на домен и ограничение по размеру (около 4KB).

---
#### ⭐⭐setSecure(true) + setHttpOnly(true) = Максимальная защита⭐⭐

|Атака|Защита|
|---|---|
|**Перехват по сети**|`Secure` - cookie не передается по HTTP|
|**XSS атаки**|`HttpOnly` - JS не может прочитать cookie|
|**MiTM атаки**|`Secure` + HTTPS шифрование|

## 💎 Best Practices 💎

✅ ❗Всегда используйте `HttpOnly` и `Secure` для сессионных cookie.  

✅ Ограничивайте `Path` и `Domain` минимально возможной областью.  

✅ Не храните чувствительные данные в cookie (логины, пароли, токены без шифрования).  

✅ ❗Явно задавайте `Max-Age`, чтобы контролировать время жизни cookie.  

✅ ⭐Используйте безопасное имя cookie (без пробелов, спецсимволов).

## Примеры возможный настроек
### Сессионная cookie:
```java
Cookie sessionCookie = new Cookie("JSESSIONID", generateSessionId());

sessionCookie.setSecure(true);      // Только HTTPS
sessionCookie.setHttpOnly(true);    // Защита от XSS
sessionCookie.setMaxAge(30 * 60);   // 30 минут
sessionCookie.setPath("/");         // Доступно для всего сайта
response.addCookie(sessionCookie);
```

### Авторизационная cookie:
```java
Cookie authCookie = new Cookie("AUTH_TOKEN", encryptedToken);

authCookie.setSecure(true);
authCookie.setHttpOnly(true); 
authCookie.setMaxAge(7 * 24 * 60 * 60); // 7 дней
authCookie.setPath("/api");
response.addCookie(authCookie);
```

### Cookie для настроек (доступна JS):
```java
// Эта cookie может быть доступна JavaScript
Cookie settingsCookie = new Cookie("USER_THEME", "dark");

settingsCookie.setSecure(true);     // Все равно только HTTPS
settingsCookie.setHttpOnly(false);  // Но доступна для JS
settingsCookie.setMaxAge(365 * 24 * 60 * 60); // 1 год
response.addCookie(settingsCookie);
```


## Пример использваония куки
Создадим Hello-cookie пример.
Пользователь будет вместе с запросом отправлять параметр `name`. 

Если куки ДЛЯ ТЕКУЩЕГО запроса с таким значением есть, то выдается приветсвие. Иначе просьба с вводом имени.

Если у пользователя окажется куки, то он сможет заходить без куки на сервер в течение минуты
```java
package servlets;

import java.io.IOException;
import java.io.PrintWriter;

import jakarta.servlet.ServletException;
import jakarta.servlet.http.Cookie;
import jakarta.servlet.http.HttpServlet;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;

public class HelloServlet extends HttpServlet {
	@Override
    protected void doGet(
	    HttpServletRequest req, 
	    HttpServletResponse resp) throws ServletException, IOException 
	{
		String userName = null;
		Cookie[] userCookies = req.getCookies();
		
		// Name is present
		// cookie is: return Hello user ...
		// cookie no: create cookie and return cookie, also say "Your registre cookie"
		
		// Name is empty
		// cookie is(so name is): return Hello user ...
		// cookie no: say, you are new user, say your name
		
		if(userCookies != null) {
			for(Cookie c : userCookies) {
				if("name".equals(c.getName())) {
					userName = c.getValue();
					break;
				}
			}
		}
		
		if(userName == null) {
			userName = req.getParameter("name");
			
			if(userName == null) {
				resp.setStatus(400);
				resp
					.getWriter()
							.println("Input your name in paramentr!");
			} else {
				Cookie cookie = new Cookie(
					"name", 
					String.valueOf(userName)
				);
				cookie.setMaxAge(60);
				
				resp.setStatus(200);
				resp.addCookie(cookie);
				resp
					.getWriter()
						.println("Your registered! Take your cookie");
			}
		} else {
			resp.setStatus(200);
			resp.getWriter().println("See you soon: " + userName);
		}
    }
}
```
