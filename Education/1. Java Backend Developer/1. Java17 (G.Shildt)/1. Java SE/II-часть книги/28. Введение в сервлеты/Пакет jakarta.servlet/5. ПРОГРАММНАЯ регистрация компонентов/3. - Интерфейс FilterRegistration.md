
> [!NOTE] Интерфейс **`FilterRegistration`** расширяет `Registration` и `представляет зарегистрированный фильтр` (Filter) в контейнере.
> 
> Он используется для получения информации о фильтре (например, к каким сервлетам или URL-паттернам он привязан),  
> а также `для программной регистрации фильтров` через `ServletContext.addFilter()` — без использования `web.xml`.

> То есть `FilterRegistration` — это "обёртка" конфигурации фильтра, с помощью которой можно задавать init-параметры, URL-мэппинги и типы диспетчеров (`REQUEST`, `FORWARD`, `INCLUDE`, `ERROR`, `ASYNC`).

---
## Интерфейс
```java
package jakarta.servlet;

import java.util.Collection;
import java.util.EnumSet;
import java.util.Map;

public interface FilterRegistration extends Registration {

    // Возвращает все URL-шаблоны, на которые замаплен данный фильтр
    Collection<String> getUrlPatternMappings();

    // Возвращает имена сервлетов, к которым привязан фильтр
    Collection<String> getServletNameMappings();

    // Вложенный интерфейс для динамической регистрации фильтра
    interface Dynamic 
	    extends FilterRegistration, Registration.Dynamic 
	{
        // Добавляет мэппинг фильтра к сервлетам по их именам
		// dispatcherTypes — типы диспетчеров (REQUEST, FORWARD, INCLUDE, ERROR, ASYNC)
		// isMatchAfter — порядок: true = после существующих фильтров, false = перед ними
        void addMappingForServletNames(
	        EnumSet<DispatcherType> dispatcherTypes,
	        boolean isMatchAfter,
            String... servletNames);

        // Добавляет мэппинг фильтра к URL-паттернам
        void addMappingForUrlPatterns(
	        EnumSet<DispatcherType> dispatcherTypes,
            boolean isMatchAfter,
            String... urlPatterns);

        // Возвращает типы диспетчеров, для которых фильтр замаплен
        Collection<String> getUrlPatternMappings();

        // Возвращает имена сервлетов, для которых фильтр замаплен
        Collection<String> getServletNameMappings();

        // 💡Определяет, поддерживает ли фильтр асинхронную обработку
        void setAsyncSupported(boolean isAsyncSupported);
    }
}
```

---
## 💡 Особенности и подводные камни 💡

1️⃣ **Расширяет `Registration`.**  
Наследует методы работы с init-параметрами (`setInitParameter()`, `getInitParameters()` и т. д.).

2️⃣ **Используется для программной ==регистрации фильтров==.**  
Обычно вызывается через `ServletContext.addFilter()` внутри `ServletContextListener` или `ServletContainerInitializer`.

3️⃣ **Мэппинги можно задавать по URL или по имени сервлета.**  
Это гибко — можно фильтровать как целые пути (`"/*"`), так и конкретные сервлеты (`"helloServlet"`).

4️⃣ 💡**Параметр `isMatchAfter` управляет порядком.**  
Если `true`, фильтр добавляется **после существующих**; если `false` — **до**.  
Это аналог порядка объявления `<filter-mapping>` в `web.xml`.

5️⃣ **Асинхронный режим.**  
Если фильтр должен участвовать в обработке `AsyncContext`, нужно явно вызвать `setAsyncSupported(true)`.

6️⃣ **Мэппинги можно добавлять только до инициализации фильтра.**  
После этого любые изменения вызывают `IllegalStateException`.

---
## 💎 Best Practices 💎

✅ Добавляй мэппинги и параметры фильтров **только на этапе инициализации контекста**, до того, как контейнер вызовет `Filter.init()`.  

✅ 💡Используй `EnumSet.of(DispatcherType.REQUEST)` вместо `null` для явности.

✅ Устанавливай `setAsyncSupported(true)`, если планируешь использовать `AsyncContext`.  

✅ Не дублируй фильтры с одинаковыми URL-мэппингами — порядок вызовов может стать непредсказуемым.  

✅ 🧠В логах контейнера проверяй, правильно ли фильтр добавлен (Tomcat пишет цепочку фильтров при запуске).

## Пример
```java
import jakarta.servlet.*;
import java.util.EnumSet;

public class MyAppInitializer implements ServletContainerInitializer {

    @Override
    public void onStartup(
	    java.util.Set<Class<?>> c, 
	    ServletContext ctx) throws ServletException 
	{
        // Создаём и регистрируем фильтр
        FilterRegistration.Dynamic loggingFilter =
                ctx.addFilter(
	                "loggingFilter", 
	                new LoggingFilter()
	            );

        // Настраиваем init-параметры фильтра
        loggingFilter.setInitParameter("level", "INFO");

        // Указываем, что фильтр поддерживает асинхронную обработку
        loggingFilter.setAsyncSupported(true);

        // Добавляем мэппинг по URL-паттернам
        loggingFilter.addMappingForUrlPatterns(
                EnumSet.of(DispatcherType.REQUEST),   // Тип запроса
                false, // Добавляем перед существующими
                "/*"   // Применяем ко всем URL
        );

        // Можно также добавить мэппинг по имени сервлета
        loggingFilter.addMappingForServletNames(
                EnumSet.of(DispatcherType.REQUEST),
                true,
                "helloServlet"
        );

        System.out.println("Фильтр 'loggingFilter' зарегистрирован программно.");
    }
}
```
