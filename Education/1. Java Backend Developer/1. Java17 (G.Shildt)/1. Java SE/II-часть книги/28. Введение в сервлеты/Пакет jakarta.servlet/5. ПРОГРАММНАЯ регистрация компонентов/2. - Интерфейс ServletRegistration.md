
> [!NOTE] Интерфейс **`ServletRegistration`** расширяет `Registration` и используется контейнером `для представления зарегистрированного сервлета`
> 💡Он позволяет `просматривать` и `изменять` `конфигурацию сервлета` (например, получить его URL-мэппинги или установить их программно).
> 
>Используется в динамической регистрации через `ServletContext.addServlet()`, например, при старте приложения или в `ServletContainerInitializer`.

## Интерфейс
```java
package jakarta.servlet;

import java.util.Collection;
import java.util.Set;

public interface ServletRegistration extends Registration {

    // Возвращает URL-мэппинги, связанные с данным сервлетом
    Set<String> getMappings();

    // Возвращает роль, с которой сервлет выполняется (если задано)
    String getRunAsRole();


    // Вложенный интерфейс для динамической регистрации сервлетов
    interface Dynamic 
	    extends ServletRegistration, Registration.Dynamic 
	{
        // Добавляет новые URL-мэппинги для сервлета
        Set<String> addMapping(String... urlPatterns);

        // Устанавливает флаг асинхронной поддержки
        void setAsyncSupported(boolean isAsyncSupported);
    }
}
```

> Проще говоря, `ServletRegistration.Dynamic` — это **живая конфигурация сервлета до его инициализации**,  
> аналог `<servlet>` + `<servlet-mapping>` из `web.xml`, только в Java-коде.

---
## 💡 Особенности и подводные камни 💡

1️⃣ **Расширяет `Registration`.**  
Наследует методы управления init-параметрами (`setInitParameter`, `getInitParameter` и др.).

2️⃣ **Используется только для сервлетов.**  
Для фильтров есть аналог — `FilterRegistration`.

3️⃣ **❗`Dynamic`❗ ==создаётся контейнером==.**  
При вызове `ServletContext.addServlet()` возвращается именно `ServletRegistration.Dynamic`.

4️⃣ **Метод `addMapping()` должен вызываться до инициализации.**  
После того как контейнер инициализировал сервлет, новые мэппинги добавить уже нельзя — будет `IllegalStateException`.

5️⃣ **Мэппинги не могут пересекаться.**  
Если другой сервлет уже обслуживает тот же URL, контейнер выбросит исключение.

6️⃣ **Методы `getMappings()` и `getRunAsRole()` доступны и после инициализации**, но изменить эти параметры уже нельзя.

---
## 💎 Best Practices 💎

✅ Используй `ServletRegistration.Dynamic` для программной регистрации сервлетов при старте приложения (например, в `ServletContextListener` или `ServletContainerInitializer`).  

✅ Добавляй init-параметры и мэппинги **до того, как контейнер вызовет `Servlet.init()`**.  

✅ ⭐⭐⭐Если нужен асинхронный сервлет — не забудь `setAsyncSupported(true)`.  

✅ Избегай пересечения URL-паттернов разных сервлетов.  

✅ Логируй результат `addMapping()` — возвращаемый `Set<String>` может содержать конфликтующие шаблоны.

## Пример
Программная регистрация сервлета без `web.xml`:
```java
import jakarta.servlet.*;
import jakarta.servlet.annotation.HandlesTypes;
import java.util.*;

public class MyAppInitializer implements ServletContainerInitializer {

    @Override
    public void onStartup(Set<Class<?>> c, ServletContext ctx) throws ServletException {

        // Регистрируем новый сервлет
        ServletRegistration.Dynamic helloReg = ctx.addServlet("helloServlet", new HelloServlet());

        // Добавляем мэппинг (URL-шаблон)
        helloReg.addMapping("/hello", "/welcome");

        // Устанавливаем init-параметры
        helloReg.setInitParameter("theme", "dark");

        // Указываем поддержку асинхронного режима
        helloReg.setAsyncSupported(true);

        System.out.println("Servlet 'helloServlet' зарегистрирован по URL /hello и /welcome");
    }
}
```
