
> [!NOTE] Интерфейс **`Registration`** — это `контракт`, который `представляет объект`, `зарегистрированный в контейнере сервлетов` (например, `Servlet` или `Filter`).
> Он используется для ==программной регистрации компонентов== 
> (через `ServletContext.addServlet()` или `addFilter()`) 
> и 
> позволяет управлять **инициализационными параметрами** и получать **метаданные** о зарегистрированном компоненте.
> 
> 💡То есть, если ты не используешь `web.xml`, а создаёшь сервлет/фильтр динамически в коде — контейнер вернёт тебе объект `Registration`, с помощью которого можно настраивать параметры.

Интерфейс **не реализует** другие интерфейсы, но сам ==является **родительским** для==:
- `ServletRegistration`
- `FilterRegistration`

---
## Интерфейс
```java
package jakarta.servlet;

import java.util.Map;
import java.util.Set;

public interface Registration {
    
    // Возвращает имя зарегистрированного компонента (сервлета или фильтра)
    String getName();

    // Возвращает имя класса компонента (FQCN)
    String getClassName();

    // Устанавливает один init-параметр
    boolean setInitParameter(String name, String value);

    // Устанавливает несколько init-параметров сразу
    Set<String> setInitParameters(Map<String, String> initParameters);

    // Возвращает значение параметра инициализации
    String getInitParameter(String name);

    // Возвращает все параметры инициализации (в виде неизменяемой Map)
    Map<String, String> getInitParameters();
}
```

---
## 💡 Особенности и подводные камни 💡

1️⃣ 💡**Программная конфигурация вместо XML.**  
Используется при динамической регистрации компонентов — например, внутри `ServletContainerInitializer` или `ServletContextListener`.

2️⃣ ❗❗❗**Параметры должны задаваться до инициализации.**  
Все `setInitParameter()` вызовы должны быть сделаны **до** того, как контейнер инициализирует компонент (иначе — `IllegalStateException`).

3️⃣ 💡**`setInitParameters()` возвращает Set конфликтующих ключей.**  
Если некоторые параметры уже были заданы ранее, метод вернёт имена конфликтующих параметров.

4️⃣ **Возвращаемые карты — неизменяемые.**  
`getInitParameters()` возвращает _immutable map_, попытка модификации вызовет `UnsupportedOperationException`.

5️⃣ **Работает одинаково для фильтров и сервлетов.**  
Разница появляется только в расширенных интерфейсах (`ServletRegistration`, `FilterRegistration`).

---
## 💎 Best Practices 💎

✅ Используй `Registration` (через `ServletRegistration` / `FilterRegistration`) для динамической настройки параметров при старте приложения, когда нет `web.xml`.  

✅ Устанавливай init-параметры **до вызова `ServletContext.addServlet()`/`addFilter()` завершения инициализации**.  

✅ Избегай дублирования имён параметров — проверяй результат `setInitParameters()`.  

✅ Для типобезопасности — храни init-параметры в константах (во избежание опечаток в строках).

## Пример использования

Пример программной регистрации фильтра и сервлета без `web.xml`:
```java
import jakarta.servlet.*;
import jakarta.servlet.annotation.HandlesTypes;
import java.util.*;

public class MyAppInitializer implements ServletContainerInitializer {

    @Override
    public void onStartup(
	    Set<Class<?>> c, 
	    ServletContext ctx) throws ServletException 
	{
        // Регистрируем сервлет
        ServletRegistration.Dynamic servletReg = 
	        ctx
		        .addServlet(
			        "helloServlet", 
			        new HelloServlet()
			    );
        servletReg.addMapping("/hello");

        // Добавляем параметры через интерфейс Registration
        servletReg.setInitParameter("greeting", "Welcome to my app!");
        servletReg.setInitParameters(Map.of("theme", "dark", "lang", "en"));

        // Регистрируем фильтр
        FilterRegistration.Dynamic filterReg = 
	        ctx
		        .addFilter(
			        "loggingFilter", 
			        new LoggingFilter()
			    );
        filterReg.addMappingForUrlPatterns(
	        EnumSet.of(DispatcherType.REQUEST), true, "/*");

        // Настраиваем init-параметры фильтра
        filterReg.setInitParameter("logLevel", "DEBUG");

        System.out.println("Components registered programmatically!");
    }
}
```
