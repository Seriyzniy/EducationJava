# 📕 1. ServletContextListener
> [!NOTE] **`ServletContextListener`** — это интерфейс наблюдателя (listener),
> который` отслеживает` `жизненный цикл всего веб-приложения` в контейнере сервлетов.
> 
> Он `уведомляет`, когда:
> - ==контекст== приложения **==создаётся==** (при старте веб-приложения),
> - контекст приложения **==уничтожается==** (при остановке приложения или контейнера).

⭐⭐Обычно используется для:
- 💡==инициализации глобальных ресурсов== (БД, ❗пула соединений❗, кеша),
- загрузки конфигурации,
    
- очистки ресурсов при завершении работы

---
## Интерфейс
```java
package jakarta.servlet;

import java.util.EventListener;

public interface ServletContextListener extends EventListener {

    /**
     * ❗Вызывается контейнером ❗при инициализации❗ (старте) контекста веб-приложения.
     * Обычно используется для загрузки конфигурации, открытия соединений и т.д.
     *
     * @param sce объект события, содержащий ссылку на ServletContext
     */
    void contextInitialized(ServletContextEvent sce);


    /**
     * Вызывается контейнером ❗перед уничтожением контекста❗ приложения.
     * Используется для освобождения ресурсов и завершения фоновых процессов.
     *
     * @param sce объект события, содержащий ссылку на ServletContext
     */
    void contextDestroyed(ServletContextEvent sce);
}
```

---
## 💡 Особенности и подводные камни 💡

1️⃣ 💡**Вызывается один раз за всё время жизни приложения.**  
— `contextInitialized()` срабатывает при старте приложения,  
`contextDestroyed()` — при его завершении.

2️⃣ 💡**Порядок вызова:**  
— Сначала все `contextInitialized()` ==всех слушателей==,  
потом инициализируются сервлеты.  
— При завершении — сначала `contextDestroyed()`, потом выгрузка сервлетов.

3️⃣ ⭐⭐**Регистрация слушателя:**
- 🔶Через аннотацию `@WebListener`
    
- 🔶Или через `web.xml`:
```xml
<listener>     
	<listener-class>com.example.AppContextListener</listener-class> </listener>
```
    

4️⃣ 💡**Работа с контекстом приложения:**  
— Через `ServletContext context = sce.getServletContext();`  
можно записывать атрибуты (`context.setAttribute("db", ds)`),  
чтобы использовать их потом в сервлетах.

5️⃣ **Нельзя использовать `request` или `session`** внутри этих методов —  
на момент вызова **ещё нет активных запросов** или **уже нет** при уничтожении.

6️⃣ 🧠**Ошибки внутри `contextInitialized()` ==могут помешать запуску приложения.==**
— Если выкинуло исключение — контейнер может не развернуть приложение.

---
## 💎 Best Practices 💎

✅ Используй `ServletContextListener` ==для инициализации **общих ресурсов**==: DataSource, кеш, пулы потоков.  

✅ ❗❗❗В `contextDestroyed()` обязательно **освобождай** ресурсы (закрывай соединения, останавливай задачи).  

✅ ⭐⭐⭐Храни глобальные объекты в `ServletContext` через `setAttribute()`.  
✅ ❗❗❗Не выполняй долгих операций синхронно в `contextInitialized()` — это может задержать старт.  

✅ 🧠❗Логику инициализации выноси в отдельные классы (например, `AppInitializer`), не перегружай listener.

## Пример использования

### Пример: инициализация пула соединений и освобождение при остановке
```java
package listeners;

import jakarta.servlet.ServletContext;
import jakarta.servlet.ServletContextEvent;
import jakarta.servlet.ServletContextListener;
import jakarta.servlet.annotation.WebListener;
import javax.sql.DataSource;
import org.apache.commons.dbcp2.BasicDataSource;

@WebListener
public class AppContextListener implements ServletContextListener {

    private BasicDataSource dataSource;

    @Override
    public void contextInitialized(ServletContextEvent sce) {
        System.out.println("🚀 Приложение запускается...");

        dataSource = new BasicDataSource();
        
        dataSource.setUrl("jdbc:postgresql://localhost:5432/mydb");
        dataSource.setUsername("user");
        dataSource.setPassword("password");
        dataSource.setInitialSize(5);
        dataSource.setMaxTotal(20);

        ServletContext context = sce.getServletContext();
        context.setAttribute("DataSource", dataSource);

        System.out.println("✅ DataSource инициализирован и сохранён в ServletContext");
    }

    @Override
    public void contextDestroyed(ServletContextEvent sce) {
        System.out.println("🧹 Приложение останавливается...");

        try {
            if (dataSource != null) {
                dataSource.close();
                System.out.println("✅ DataSource закрыт");
            }
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```

⭐⭐Теперь любой сервлет может получить пул:⭐⭐
```java
DataSource ds = 
	(DataSource) getServletContext().getAttribute("DataSource");
```

# 📕 2. ServletContextEvent

> [!NOTE] `ServletContextEvent` — это `класс события`, п`ередаваемый слушателям` типа `ServletContextListener`.  
> Он `сообщает`, что ==контекст веб-приложения== был ==создан== или ==уничтожен==, и позволяет слушателю `получить доступ` к объекту `ServletContext`.
> 
> Используется контейнером для оповещения о **жизненном цикле приложения**.
> 

## Интерфейс и сигнатура
```java
package jakarta.servlet;

public class ServletContextEvent extends java.util.EventObject {

    public ServletContextEvent(ServletContext source) {
        super(source);
    }

    public ServletContext getServletContext() {
        return (ServletContext) getSource();
    }
}
```
