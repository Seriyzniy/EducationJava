
> [!NOTE] Интерфейс `Servlet` — это **основа Java Servlet API**
> `определяющая контракт` между **сервлетом** и **контейнером сервлетов** (например, Tomcat, Jetty).  
> 
> 💡Он задаёт методы жизненного цикла сервлета: инициализацию, обработку запросов и уничтожение.
> 
> ⭐Все к`лассы-сервлеты` ==обязаны реализовать этот интерфейс== (напрямую или через наследование от `GenericServlet` или `HttpServlet`).

## Интерфейс
```java
package jakarta.servlet;

import java.io.IOException;

/**
 * Defines methods that all servlets must implement.
 */
public interface Servlet {

    // ==============================
    //  🟢 Инициализация и уничтожение 🟢
    // ==============================
    /**
     * Инициализирует сервлет.  
     * Вызывается контейнером **один раз** при создании экземпляра сервлета.
     * 
     * @param config объект конфигурации сервлета (ServletConfig)
     * @throws ServletException если произошла ошибка при инициализации
     */
    void init(ServletConfig config) throws ServletException;

    /**
     * Вызывается контейнером **один раз** при уничтожении сервлета, 
     * например, при остановке приложения или перезапуске контейнера.
     * Используется для освобождения ресурсов (потоки, соединения и т.д.)
     */
    void destroy();



    // ==============================
    //  🟢 Обработка запроса 🟢
    // ==============================
    /**
     * Основной метод обработки запросов.  
     * Контейнер вызывает его при каждом HTTP-запросе (или другом типе запроса).  
     * Обычно внутри этого метода происходит чтение запроса, формирование ответа.
     *
     * @param req  объект запроса (ServletRequest)
     * @param res  объект ответа (ServletResponse)
     * @throws ServletException при ошибке выполнения
     * @throws IOException при ошибке ввода-вывода
     */
    void service(ServletRequest req, ServletResponse res)
        throws ServletException, IOException;



    // ==============================
    // 🟢 Информация и конфигурация 🟢
    // ==============================
    /**
     * Возвращает объект конфигурации сервлета (ServletConfig),
     * предоставленный контейнером.  
     * Содержит параметры инициализации и доступ к ServletContext.
     *
     * @return объект ServletConfig, связанный с данным сервлетом
     */
    ServletConfig getServletConfig();

    /**
     * Возвращает строку с информацией о сервлете, например:
     * имя, версия, автор и т.п.
     * 
     * @return информация о сервлете
     */
    String getServletInfo();
}
```

---
## 💡 Особенности и подводные камни 💡

1️⃣ ⭐⭐⭐**Методы `init()` и `destroy()` ==вызываются контейнером==**, а не вручную. Если забыть это и вызывать их самостоятельно — можно нарушить жизненный цикл сервлета.

2️⃣ 💡**Метод `service()`** — главный для обработки запросов.  
Для HTTP-сервлетов его обычно не переопределяют, а используют `doGet()`, `doPost()` и т.п. из `HttpServlet`.

3️⃣ ⭐⭐⭐Если **инициализация** (`init()`) выбросит исключение `ServletException`, контейнер **не будет использовать** этот сервлет.

4️⃣ 🧠**`getServletConfig()`** часто возвращает `null`, если сервлет не был корректно инициализирован контейнером.

5️⃣ ❗❗❗Контейнер может **создавать только один экземпляр** сервлета (по умолчанию), обслуживающий все запросы,  
❗❗❗поэтому ==**поля класса должны быть потокобезопасными**== или неизменяемыми.

## 💎 Best Practices 💎

✅ 💡Всегда наследуйтесь от **`HttpServlet`** — это стандартный путь для веб-приложений на Java.

✅ ❗❗❗Не храните в полях сервлета данные, зависящие от пользователя (например, `HttpSession` или `Request`).

✅💡 Используйте **`init()`** для загрузки конфигурации и ресурсов (например, подключений к БД).

✅ Освобождайте ресурсы в **`destroy()`**, чтобы избежать утечек памяти.

✅ 💡Логируйте ошибки инициализации через `ServletContext.log()` для диагностики.

## Пример

> [!NOTE] **`@WebServlet("/hello")`** - аннотация `для маппинга сервлета` на URL `/hello`
> 


```java
import jakarta.servlet.*;
import jakarta.servlet.annotation.WebServlet; //Аннотация для маппинга

import java.io.IOException;
import java.io.PrintWriter;

// Пример простого сервлета, реализующего интерфейс Servlet напрямую
@WebServlet("/hello")
public class HelloServlet implements Servlet {

    private ServletConfig config;

    @Override
    public void init(ServletConfig config) throws ServletException {
        this.config = config;
        System.out.println("Servlet инициализирован");
    }

    @Override
    public ServletConfig getServletConfig() {
        return config;
    }

    @Override
    public void service(ServletRequest req, ServletResponse res)
            throws ServletException, IOException {
        res.setContentType("text/html; charset=UTF-8");
        PrintWriter out = res.getWriter();
        out.println("<h1>Привет, мир!</h1>");
    }

    @Override
    public String getServletInfo() {
        return "HelloServlet v1.0";
    }

    @Override
    public void destroy() {
        System.out.println("Servlet уничтожен");
    }
}

```
