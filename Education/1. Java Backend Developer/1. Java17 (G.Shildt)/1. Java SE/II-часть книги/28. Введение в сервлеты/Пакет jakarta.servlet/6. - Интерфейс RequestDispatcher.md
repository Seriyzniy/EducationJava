
> [!NOTE] **`RequestDispatcher`** — это интерфейс, 
> Который используется для `ВНУТРЕННЕГО перенаправления запросов` `МЕЖДУ РЕСУРСАМИ` (servlet, JSP, HTML и др.) `в ПРЕДЕЛАХ ОДНОГО веб-приложения`.
> 
> Он позволяет:
> - 🔶`передать управление другому ресурсу` (**forward**)
> - 🔶или `включить результат` его обработки `в текущий ответ` (**include**).
> 
> ⭐Используется контейнером ==для ВНУТРЕННЕГО диспетчинга== 
> (❗без нового HTTP-запроса от клиента❗).

→ ==Обычно== экземпляры ==создаются контейнером== через `ServletRequest.getRequestDispatcher()` или `ServletContext.getRequestDispatcher()`.

---
## Интерфейс
```java
package jakarta.servlet;

import java.io.IOException;

public interface RequestDispatcher {

    /**
     * Перенаправляет (внутренне) запрос и ответ другому ресурсу
     * (servlet, JSP или статическому файлу) на сервере.
     * После вызова управление не возвращается в текущий servlet.
     *
     * @param request  текущий запрос
     * @param response текущий ответ
     * @throws ServletException при ошибке на уровне сервлетов
     * @throws IOException при ошибке ввода-вывода
     */
    void forward(ServletRequest request, ServletResponse response)
            throws ServletException, IOException;

    /**
     * Включает (встраивает) вывод другого ресурса в текущий ответ.
     * В отличие от forward, выполнение продолжается после include().
     *
     * @param request  текущий запрос
     * @param response текущий ответ
     * @throws ServletException при ошибке на уровне сервлетов
     * @throws IOException при ошибке ввода-вывода
     */
    void include(ServletRequest request, ServletResponse response)
            throws ServletException, IOException;
}
```

## 💡 Особенности и подводные камни 💡

1️⃣ **`forward()` — внутренний редирект.**  
— Контейнер передаёт запрос другому ресурсу **на сервере**,  
❗клиент не получает 302 Redirect и не делает новый HTTP-запрос. ❗

— URI в браузере **не меняется**.


2️⃣ ❌**После `forward()` нельзя писать в `response`.**  
— Контейнер обычно очищает буфер ответа (`response.resetBuffer()`),  
и дальнейшие `write()` вызовут `IllegalStateException`.

3️⃣ ✅**`include()` не очищает буфер** и не завершает выполнение.  
— Вывод целевого ресурса добавляется в текущий ответ,  
и после этого код в исходном сервлете продолжает выполняться.

4️⃣ **Путь к ресурсу** зависит от метода получения:
- 🔶`request.getRequestDispatcher("page.jsp")` — относительный путь от текущего ресурса
    
- 🟢`context.getRequestDispatcher("/path")` — абсолютный путь от корня приложения (`/`)
    

5️⃣ **Нельзя делать `forward()` после того, как был отправлен заголовок.**  
— Т.е. до `response.flushBuffer()` или записи тела ответа.

6️⃣ **Параметры запроса и атрибуты сохраняются.**  
— Новый ресурс получает тот же `ServletRequest` и `ServletResponse`  
(но может добавлять атрибуты, например `javax.servlet.forward.request_uri`).

---
## 💎 Best Practices 💎

✅ 💡Используй `forward()` для внутренних переходов между сервлетами и JSP (MVC-подход).  

✅ 💡Используй `include()` для вставки общих частей (например, header/footer шаблонов).  

✅ ❗❗❗Перед `forward()` НЕ записывай** данные в поток — они будут потеряны.  

✅ ❗❗❗Всегда используй ==абсолютные пути== при вызове из `ServletContext`.  

✅ Проверяй, что `response.isCommitted()` == false перед `forward()`.  

✅ 💡💡💡Не путай `forward()` с `sendRedirect()` — второй делает **HTTP 302** и **новый запрос от клиента**.

## Пример использования
### Пример 1: `forward()` — передача управления JSP
```java
@WebServlet("/controller")
public class ControllerServlet extends HttpServlet {

    @Override
    protected void doGet(
	    HttpServletRequest req, 
	    HttpServletResponse resp) throws ServletException, IOException 
	{
        req.setAttribute("message", "Hello from Controller!");
        
        RequestDispatcher dispatcher = req
	        .getRequestDispatcher("/WEB-INF/views/page.jsp");
        
        dispatcher.forward(req, resp);
    }
}
```

📘 В браузере URL останется `/controller`,  
а фактически будет отображён результат `page.jsp`.

```java
// 1. Начало в ControllerServlet
Browser → ControllerServlet.doGet()
         ↓
	// 2. forward() передает управление
	ControllerServlet → RequestDispatcher → JSP-движок
	         ↓                  ↓               ↓
	    ЗАВЕРШАЕТСЯ      Берет те же      Обрабатывает
	                    req/resp объекты   page.jsp
					                            
	                            
// 3. JSP генерирует ответ
JSP-движок → HTML → Browser
```

### Пример 2: `include()` — вставка контента
```java
@WebServlet("/main")
public class MainPageServlet extends HttpServlet {

    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp)
            throws ServletException, IOException {

        resp.setContentType("text/html;charset=UTF-8");
        PrintWriter out = resp.getWriter();

        out.println("<html><body>");
        RequestDispatcher header = req.getRequestDispatcher("/header.html");
        header.include(req, resp);

        out.println("<h1>Main content here</h1>");

        RequestDispatcher footer = req.getRequestDispatcher("/footer.html");
        footer.include(req, resp);

        out.println("</body></html>");
    }
}
```

### **Визуализация потока данных:**
```text
PrintWriter out (буфер ответа)
│
├── out.println("<html><body>")
│
├── header.include() → добавляет "<header>Welcome...</header>"
│
├── out.println("<h1>Main content here</h1>")  
│
├── footer.include() → добавляет "<footer>Copyright...</footer>"
│
└── out.println("</body></html>")
```

#### **ФИНАЛЬНЫЙ результат:**
```html
<html><body>
<header>Welcome to my site</header>
<h1>Main content here</h1>
<footer>Copyright 2024</footer>
</body></html>
```
