
> [!NOTE] `AsyncContext` — это **интерфейс**, представляющий `контекст асинхронной обработки` в сервлетах.
> Он ==появляется, когда== сервлет или фильтр ==вызывает метод== `startAsync()` у `ServletRequest`.
> 
> Асинхронная обработка `позволяет освободить` `ПОТОК контейнера`, пока запрос выполняет долгую операцию  (например, обращение к внешнему API, БД или ожидание данных).  
> Позже другой поток завершает обработку и отправляет ответ клиенту.
> 
> ⭐Предоставляется контейнером (Tomcat, Jetty и т.д.), нельзя создать вручную.

> То есть `AsyncContext` управляет "жизненным циклом" асинхронного запроса.

## Интефрейс
```java
package jakarta.servlet;

public interface AsyncContext {

    // Возвращает объект ServletRequest, связанный с этим асинхронным контекстом.
    ServletRequest getRequest();

    // Возвращает объект ServletResponse, связанный с этим асинхронным контекстом.
    ServletResponse getResponse();

    // ❗Завершает асинхронную обработку❗ и ❗отправляет ответ❗ клиенту.
    void complete();

    // Передаёт выполнение другому потоку или сервлету.
    void dispatch();

    // Передаёт выполнение указанному сервлету или по указанному пути.
    void dispatch(String path);

    // Передаёт выполнение с указанием сервлета и контекста.
    void dispatch(ServletContext context, String path);

    // Добавляет слушателя событий асинхронного цикла (запуск, завершение, таймаут, ошибка).
    void addListener(AsyncListener listener);

    // Добавляет слушателя с явными объектами запроса и ответа.
    void addListener(AsyncListener listener, ServletRequest request, ServletResponse response);

    // Возвращает зарегистрированный слушатель.
    <T extends AsyncListener> T createListener(Class<T> clazz) throws ServletException;

    // Устанавливает таймаут асинхронной обработки в миллисекундах.
    void setTimeout(long timeout);

    // Возвращает текущий таймаут в миллисекундах.
    long getTimeout();

    // Запускает задачу асинхронно (в отдельном потоке контейнера).
    void start(Runnable run);

    // Проверяет, инициализирован ли контекст в том же запросе.
    boolean hasOriginalRequestAndResponse();
}
```

## Методы
### Доступ к запросу и ответу
```java
// Возвращает оригинальный (или обёрнутый) объект запроса.
ServletRequest getRequest();

// Возвращает объект ответа для асинхронной обработки.
ServletResponse getResponse();

// Проверяет, используются ли оригинальные request/response (без обёртки).
boolean hasOriginalRequestAndResponse();
```

### Управление жизненным циклом асинхронной обработки
```java
// ✅Завершает асинхронную обработку✅. 
// После вызова complete() контейнер отправляет ответ клиенту.
void complete();

// 💡Асинхронно передаёт выполнение стандартному сервлету💡 по текущему пути (аналог forward()).
void dispatch();

// Передаёт выполнение по указанному пути внутри того же контекста.
void dispatch(String path);

// Передаёт выполнение в другой ServletContext (например, в другой web-приложение).
void dispatch(ServletContext context, String path);
```

### ⭐ Асинхронное выполнение ⭐
```java
// 💡Запускает задачу💡 (Runnable) асинхронно в контейнере.
// Обычно используется для фоновой обработки.
void start(Runnable run);
```

### Управление временем ожидания
```java
// Устанавливает таймаут асинхронной обработки (в миллисекундах).
void setTimeout(long timeout);

// Возвращает текущий таймаут (по умолчанию — 30 секунд).
long getTimeout();
```

### Слушатели событий (мониторинг состояния)
```java
// Регистрирует слушателя событий асинхронного процесса.
void addListener(AsyncListener listener);

// Регистрирует слушателя с явными request и response.
void addListener(
	AsyncListener listener, 
	ServletRequest request, 
	ServletResponse response);

// Создаёт новый экземпляр слушателя.
<T extends AsyncListener> T createListener(Class<T> clazz) throws ServletException;
```

⭐⭐Слушатели реализуют интерфейс `AsyncListener` и позволяют реагировать на:
- `onStartAsync()` — начало обработки
- `onComplete()` — завершение
- `onError()` — ошибка
- `onTimeout()` — таймаут

## 💡 Особенности и подводные камни 💡

1️⃣ Обязательна ==ПОДДЕРЖКА АСИНХРОННОСТИ== в `web.xml` ИЛИ при помощи аннотаций
⭐⭐⭐Асинхронная обработка ==работает **только если== `asyncSupported=true`** в `@WebServlet` или `web.xml`.  

2️⃣ ⭐После вызова `startAsync()` ==контейнер **освобождает основной поток**==, но соединение остаётся открытым.  

3️⃣ ❗❗❗❗Не забудь вызвать `complete()` — иначе ответ **никогда не будет отправлен клиенту!**  

4️⃣ Методы `getRequest()` и `getResponse()` возвращают **тот же объект**, что был на момент вызова `startAsync()`.  

5️⃣ 🧠Таймаут — важен! По умолчанию 30 сек, после чего будет вызван `AsyncListener.onTimeout()`.  

6️⃣ ⭐⭐⭐Контейнер сам управляет потоками через `AsyncContext.start()`. Не стоит вручную использовать `new Thread()`!  

7️⃣ При `dispatch()` — управление передаётся снова в контейнер, как при обычном `forward()`.

---
## 💎 Best Practices 💎

✅ ❗❗❗Устанавливай `asyncSupported=true` в сервлете или `web.xml`. 

✅ Всегда вызывай `complete()` после окончания асинхронной работы.  

✅ Используй `AsyncListener` для логирования и обработки таймаутов/ошибок.  

✅ Не создавай потоки вручную — используй `start(Runnable)`.  

✅ ❗❗❗Ограничивай таймаут (`setTimeout()`), чтобы не держать соединения бесконечно.  

✅ Не пиши ничего в `response` после вызова `complete()` — поток уже закрыт.

## Пример
Асинхронная обработка в сервлете:
```java
@WebServlet(value = "/async-demo", asyncSupported = true)
public class AsyncDemoServlet extends HttpServlet {
    @Override
    protected void doGet(
	    HttpServletRequest req, 
	    HttpServletResponse resp) throws IOException 
	{
        resp.setContentType("text/plain");
        PrintWriter out = resp.getWriter();

        // Инициализируем асинхронный контекст
        AsyncContext asyncContext = req.startAsync();
        asyncContext.setTimeout(10000); // 10 секунд

        // Запускаем задачу асинхронно
        asyncContext.start(() -> {
            try {
                Thread.sleep(3000); // имитация долгой операции
                resp.getWriter().println("✅ Обработка завершена в фоне!");
            } catch (Exception e) {
                e.printStackTrace();
            } finally {
                // Завершение асинхронного цикла — важно!
                asyncContext.complete();
            }
        });
    }
}
```

Пример добавления слушателя для мониторинга событий:
```java
asyncContext.addListener(new AsyncListener() {
    @Override
    public void onComplete(AsyncEvent event) {
        System.out.println("✅ Завершено");
    }

    @Override
    public void onTimeout(AsyncEvent event) {
        System.out.println("⚠️ Таймаут");
    }

    @Override
    public void onError(AsyncEvent event) {
        System.out.println("❌ Ошибка");
    }

    @Override
    public void onStartAsync(AsyncEvent event) {
        System.out.println("▶️ Старт асинхронной обработки");
    }
});

```
