
> [!NOTE] **`WriteListener`** — интерфейс для `АСИНХРОННОЙ и неблокирующей` `ЗАПИСИ данных` в `ServletOutputStream`.
> Он позволяет `писать ответ` клиенту `по мере готовности сокета к записи`, ==НЕ блокируя поток== (в отличие от синхронного `outputStream.write()`).
> 
> Применяется, например, для потоковой передачи больших данных, SSE, WebSocket-like реализаций, стриминга JSON и т.д.

## Интерфейс
```java
package jakarta.servlet;

import java.io.IOException;

public interface WriteListener {

    /**
     * Вызывается контейнером, когда сокет готов к записи (не блокирует).
     * Используется для записи доступной порции данных в ServletOutputStream.
     */
    void onWritePossible() throws IOException;

    /**
     * Вызывается контейнером при возникновении ошибки во время записи или при закрытии соединения.
     */
    void onError(Throwable t);
}
```

## Методы
### Категория: Асинхронная запись
```java
// Вызывается контейнером, когда можно безопасно писать данные в поток (не блокируя).
void onWritePossible() throws IOException;

// Вызывается контейнером при ошибке записи (например, разрыв соединения, IOException).
void onError(Throwable t);
```

---
## 💡 Особенности и подводные камни 💡

1️⃣ ==**Методы вызываются контейнером**==, а не пользователем.  
— Контейнер следит за состоянием сокета и уведомляет, когда можно продолжить запись.  
— Это делает I/O неблокирующим, как в NIO.

2️⃣ **`onWritePossible()` может вызываться многократно**  
— Каждый раз, когда сокет снова становится доступен для записи после переполнения буфера.

3️⃣ ❗❗❗**Не вызывай `write()` в цикле без проверки `isReady()`**  
— Иначе можно заблокировать контейнерный поток.

4️⃣ ❗❗❗**Обязательно вызывать `asyncContext.complete()`**  
— Когда все данные записаны, чтобы завершить запрос.

5️⃣ **`onError()` может быть вызван в любой момент**  
— Например, при обрыве соединения клиентом или при ошибке сети.

6️⃣ ❗❗❗**Методы выполняются в контейнерных потоках (NIO poller/worker)**  
— Поэтому не выполняй долгие операции прямо в них.

---
## 💎 Best Practices 💎

✅ 💡Проверяй `output.isReady()` перед каждой записью. 

✅ ⭐⭐⭐Пиши небольшими порциями, избегай больших буферов.  

✅ Завершай асинхронный контекст (`asyncContext.complete()`) после успешной записи.  

✅ Обрабатывай ошибки в `onError()` корректно (например, логируй и закрывай ресурсы).  

✅ ⭐⭐⭐⭐Не вызывай `write()` в цикле без паузы — жди повторного `onWritePossible()` вызова.

## Пример
```java
@WebServlet(value = "/async-write", asyncSupported = true)
public class AsyncWriteServlet extends HttpServlet {

    @Override
    protected void doGet(
	    HttpServletRequest req, 
	    HttpServletResponse resp) throws IOException 
	{

        AsyncContext asyncContext = req.startAsync();
        ServletOutputStream out = resp.getOutputStream();

        out.setWriteListener(new WriteListener() {
            private int counter = 0;

            @Override
            public void onWritePossible() throws IOException {
                // Пока можно писать — записываем данные
                while (out.isReady() && counter < 5) {
                    out.write(("Chunk " + counter + "\n").getBytes());
                    counter++;
                }

                // Когда всё записано — завершаем асинхронный контекст
                if (counter >= 5) {
                    asyncContext.complete();
                }
            }

            @Override
            public void onError(Throwable t) {
                t.printStackTrace();
                asyncContext.complete();
            }
        });
    }
}
```

Может делать тоже самое что и ServletOutputStream, только асинхронно
### 🧠 Что происходит "под капотом"
| Этап                                  | Кто вызывает | Описание                                               |
| ------------------------------------- | ------------ | ------------------------------------------------------ |
| Клиент подключается                   | Контейнер    | Создаётся `HttpServletRequest` и `HttpServletResponse` |
| `req.startAsync()`                    | Приложение   | Перевод запроса в асинхронное состояние                |
| `output.setWriteListener(listener)`   | Приложение   | Регистрируется слушатель на неблокирующую запись       |
| Буфер сокета освобождается            | Контейнер    | Вызывает `listener.onWritePossible()`                  |
| Поток пишет данные (пока `isReady()`) | Приложение   | Асинхронно передаёт данные клиенту                     |
| Все данные записаны                   | Контейнер    | Закрывает сокет после `asyncContext.complete()`        |
| Ошибка записи                         | Контейнер    | Вызывает `listener.onError()`                          |
