
> [!NOTE] `AsyncListener` — интерфейс из пакета `jakarta.servlet` (ранее `javax.servlet`), 
> Предназначенный `для ОБРАБОТКИ СОБЫТИЙ` `жизненного цикла` `асинхронной обработки`.
> 

Он позволяет ==отлавливать четыре события==:
- начало (`onStartAsync`)
- завершение (`onComplete`)
	
- таймаут (`onTimeout`)
- ошибка (`onError`)

💡Используется совместно с `AsyncContext`, который уведомляет слушателя обо всех этих событиях.

---
## Интерфейс
```java
package jakarta.servlet;

import java.util.EventListener;
import java.io.IOException;

public interface AsyncListener extends EventListener {

    // Вызывается ❗при завершении❗ асинхронной обработки (успешно или с ошибкой)
    void onComplete(AsyncEvent event) throws IOException;

    // Вызывается ❗при возникновении ошибки❗ в асинхронном процессе
    void onError(AsyncEvent event) throws IOException;

    // Вызывается при ❗срабатывании таймаута❗
    void onTimeout(AsyncEvent event) throws IOException;

    // Вызывается, если внутри асинхронного запроса снова ❗вызван startAsync() ( или dispatch() )❗
    void onStartAsync(AsyncEvent event) throws IOException;
}
```

---
## 💡 Особенности и подводные камни 💡

1️⃣ `AsyncListener` ==регистрируется **до** запуска фоновой задачи== — через `asyncContext.addListener()`.  
Иначе часть событий может быть пропущена.

2️⃣ ⭐⭐Слушатель получает объект `AsyncEvent`, который содержит:
- `AsyncContext getAsyncContext()` — сам контекст,
    
- `ServletRequest getSuppliedRequest()`,
    
- `ServletResponse getSuppliedResponse()`,
    
- `Throwable getThrowable()` — при ошибке.
    

3️⃣ Все методы слушателя выполняются ==**в контейнерных потоках**==, не в твоих рабочих потоках.  
Поэтому следует избегать долгих операций внутри методов слушателя.

4️⃣ ❗❗❗❗🧠Важно: если `onTimeout()` не вызывает `complete()` или `dispatch()`, контейнер **автоматически завершит** запрос.

5️⃣ 💡`onStartAsync()` вызывается **каждый раз**, когда запрос снова переводится в асинхронный режим (например, при `dispatch()`),  
— это важно, если ты хочешь повторно зарегистрировать listener или пересоздать ресурсы.

---
## 💎 Best Practices 💎

✅ ❗❗❗Регистрируй `AsyncListener` **сразу после** вызова `startAsync()`.  (Получение контекста - см [[3. - Интерфейс ServletRequest]])

✅ В `onError()` и `onTimeout()` всегда вызывай `complete()` или `dispatch()`.  

✅ Используй `onComplete()` для логирования или освобождения ресурсов.  

✅ Не выполняй тяжёлых операций внутри методов слушателя.  

✅ Если ты логируешь время выполнения, замеряй его между `onStartAsync()` и `onComplete()`.

---
## Пример

```java
@WebServlet(value = "/async-listener", asyncSupported = true)
public class AsyncListenerExample extends HttpServlet {
    @Override
    protected void doGet(
	    HttpServletRequest req, 
	    HttpServletResponse resp) throws ServletException, IOException 
	{
        AsyncContext asyncContext = req.startAsync();
        asyncContext.setTimeout(2000);

        // ✅ Регистрируем слушатель
        asyncContext.addListener(new AsyncListener() {
            @Override
            public void onComplete(AsyncEvent event) {
                System.out.println("✅ Асинхронная операция завершена");
            }

            @Override
            public void onError(AsyncEvent event) {
                System.err.println("❌ Ошибка: " + event.getThrowable());
            }

            @Override
            public void onTimeout(AsyncEvent event) {
                System.out.println("⏰ Таймаут операции");
                event.getAsyncContext().complete();
            }

            @Override
            public void onStartAsync(AsyncEvent event) {
                System.out.println("🔁 Новый цикл асинхронной обработки");
            }
        });



        // Запуск фоновой задачи
        asyncContext.start(() -> {
            try {
                Thread.sleep(1000);
                resp.getWriter().write("Async Listener Example complete!");
            } catch (Exception e) {
                throw new RuntimeException(e);
            } finally {
                asyncContext.complete();
            }
        });
    }
}
```
