
> [!NOTE] `AsyncEvent` — это класс-событие, который `инкапсулирует информацию о состоянии` `асинхронной операции` сервлета.
> Используется контейнером ==для передачи контекста в методы== ==`AsyncListener`== (`onComplete`, `onTimeout`, `onError`, `onStartAsync`).
> 
> ❗❗❗==СОЗДАЕТСЯ КОНТЕЙНЕРОМ==❗❗❗

С его помощью слушатель получает:
- 💡ссылку на текущий `AsyncContext`,
    
- `ServletRequest` и `ServletResponse`, если нужно,
    
- а также исключение (если операция завершилась с ошибкой).

---
## Конструкторы
```java
// Создаёт событие, связанное с указанным AsyncContext.
// Обычно используется контейнером.
public AsyncEvent(AsyncContext asyncContext)

// Создаёт событие с явным указанием запроса и ответа.
// Полезно, если слушателю нужны эти объекты.
public AsyncEvent(
	AsyncContext asyncContext, 
	ServletRequest request, 
	ServletResponse response)

// Создаёт событие с ошибкой. Используется при вызове onError().
public AsyncEvent(AsyncContext asyncContext, Throwable throwable)

// Полный вариант — и AsyncContext, и Request/Response, и Throwable.
// Применяется для наиболее информативных событий об ошибках.
public AsyncEvent(
	AsyncContext asyncContext, 
	ServletRequest request,
	ServletResponse response, 
	Throwable throwable)
```

## Методы

### Доступ к контексту
```java
// Возвращает объект AsyncContext, связанный с событием.
// Через него можно управлять асинхронным запросом.
public AsyncContext getAsyncContext()
```

### Доступ к объектам запроса и ответа
```java
// 💡Возвращает объект ServletRequest💡, переданный при создании события.
public ServletRequest getSuppliedRequest()

// 💡Возвращает объект ServletResponse💡, переданный при создании события.
public ServletResponse getSuppliedResponse()
```

### Информация об ошибке
```java
// Возвращает Throwable, если событие возникло из-за исключения (например, onError()).
public Throwable getThrowable()
```

---
## 💡 Особенности и подводные камни 💡

1️⃣ ⭐⭐`AsyncEvent` ==создаётся контейнером== — вручную его почти никогда не создают.  

2️⃣ `getThrowable()` может вернуть `null`, если событие не связано с ошибкой.  

3️⃣ `getSuppliedRequest()` и `getSuppliedResponse()` могут быть `null`, если контейнер не передал их.  

4️⃣ Через `getAsyncContext()` можно, например, вызвать `dispatch()` или `complete()`,  
но **делать это в `onComplete()` нельзя** — это приведёт к IllegalStateException. 

5️⃣ 💡Все вызовы слушателя (`onError`, `onTimeout` и т.д.) и работа с `AsyncEvent` выполняются в **потоках контейнера**, а не в пользовательских.

---
## 💎 Best Practices 💎

✅ Используй `event.getThrowable()` в `onError()` для логирования и диагностики ошибок.  

✅ 💡В `onTimeout()` можно безопасно вызвать `event.getAsyncContext().complete()` или `dispatch()` — это корректный способ завершить зависшую задачу.  

✅ Проверяй `null` перед обращением к `getSuppliedRequest()` / `getSuppliedResponse()`.  

✅ ❗❗❗Никогда не запускай долгие операции внутри слушателя — это блокирует контейнерный поток.  

✅ ❗❗❗Не вызывай `complete()` в `onComplete()` — к этому моменту асинхронная операция уже завершена.

## Пример
```java
@WebServlet(value = "/async-event", asyncSupported = true)
public class AsyncEventServlet extends HttpServlet {

    @Override
    protected void doGet(
	    HttpServletRequest req, 
	    HttpServletResponse resp) throws IOException 
	{
        AsyncContext asyncContext = req.startAsync();

        // Регистрируем слушатель
        asyncContext.addListener(new AsyncListener() {
            @Override
            public void onComplete(AsyncEvent event) {
                System.out.println("✅ Async complete");
            }

            @Override
            public void onTimeout(AsyncEvent event) {
                System.out.println("⚠️ Timeout occurred");
                event.getAsyncContext().complete();
            }

            @Override
            public void onError(AsyncEvent event) {
                System.err.println("❌ Error: " + event.getThrowable());
            }

            @Override
            public void onStartAsync(AsyncEvent event) {
                System.out.println("🔄 Async restarted");
            }
        });


        // Симуляция асинхронной задачи
        asyncContext.start(() -> {
            try {
                Thread.sleep(1000);
                resp.getWriter().write("Async done!");
            } catch (Exception e) {
                throw new RuntimeException(e);
            } finally {
                asyncContext.complete();
            }
        });
    }
}
```
