
> [!NOTE] Класс **`InetAddress`** из пакета `java.net` - класс `ДЛЯ РАБОТЫ с IP-адресами` в Java.
> Он инкапсулирует как **IPv4**, так и **IPv6** адреса и связывает их с **именем хоста (hostname)**.  
> 
> Используется при разработке сетевых приложений — для подключения по имени хоста, определения IP, работы с DNS и т.д.
> 
> ⭐Реализует интерфейс `Serializable` и `Comparable<InetAddress>` (начиная с Java 1.8).

Класс **`InetAddress`** ==является **абстрактным**== и **не имеет публичных конструкторов**, а его экземпляры создаются через **фабричные методы**.

## Конструктор
_Фабричные методы для создания InetAddress_
```java
//✅ Получение InetAddress по имени хоста
public static InetAddress getByName(String host) throws UnknownHostException

//✅ Получение всех адресов для хоста
public static InetAddress[] getAllByName(String host) throws UnknownHostException

//✅ Получение InetAddress по IP (байты)
public static InetAddress getByAddress(byte[] addr) throws UnknownHostException

//✅ Получение InetAddress по имени и IP
public static InetAddress getByAddress(String host, byte[] addr) throws UnknownHostException

//✅ Получение адреса локального компьютера
public static InetAddress getLocalHost() throws UnknownHostException

//✅ Получение по любому допустимому представлению
public static InetAddress getLoopbackAddress()
```

---
## 💡 Особенности и подводные камни 💡

- 💡`InetAddress` сам по себе **не делает сетевых подключений** — он лишь представляет адрес.
    
- 💡Методы вроде `getByName()` могут быть **медленными**, если требуется DNS-запрос.
    
- `getLocalHost()` иногда возвращает не тот IP, если в `/etc/hosts` или DNS неправильно настроен hostname.
    
- ⭐Объекты `InetAddress` **кэшируются** JVM — результаты DNS-запросов могут сохраняться (управляется через `networkaddress.cache.ttl`).
    
- ❓Неявная поддержка IPv6 может вызвать ошибки при взаимодействии с IPv4-серверами — важно проверять настройки сети и `java.net.preferIPv4Stack`.

---
## 💎 Best Practices 💎
✅ Используй `getAllByName()` для балансировки или резервных IP.  
✅ Не выполняй DNS-запросы в основном потоке — оборачивай вызовы `getByName()` в отдельный поток.  
✅ Проверяй `isSiteLocalAddress()` для фильтрации внутренних IP.  
✅ Если нужно просто проверить доступность, используй `InetAddress.isReachable(timeout)` (выполняет ICMP/ping).  
✅ Используй `InetAddress.getByAddress()` для известных IP, чтобы избежать DNS-запроса.


## Пример
```java
import java.net.InetAddress;

public class InetAddressExample {
    public static void main(String[] args) {
        try {
            // Получаем InetAddress по имени хоста
            InetAddress address = InetAddress
	            .getByName("www.google.com");
	            
            System.out.println("Хост: " + address.getHostName());
            System.out.println("IP: " + address.getHostAddress());
            
            // Проверка типа адреса
            System.out.println("Локальный? " + address.isSiteLocalAddress());
            System.out.println("Loopback? " + address.isLoopbackAddress());

            // Получаем локальный адрес
            InetAddress local = InetAddress.getLocalHost();
            System.out.println("Локальный хост: " + local);

            // Пример обхода всех IP-адресов хоста
            InetAddress[] all = InetAddress.getAllByName("yahoo.com");
            for (InetAddress ip : all) {
                System.out.println("Адрес Yahoo: " + ip.getHostAddress());
            }

        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```
