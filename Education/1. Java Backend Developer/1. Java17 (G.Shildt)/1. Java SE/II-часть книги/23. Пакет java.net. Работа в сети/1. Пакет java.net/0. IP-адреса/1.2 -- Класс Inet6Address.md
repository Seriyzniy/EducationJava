
> [!NOTE] **Inet6Address** - это конкретный класс, `представляющий IPv6-адрес.` 
> 
> Наследуется от `InetAddress` и предоставляет специфичные для IPv6 методы. 
> 
> Используется для работы с 128-битными адресами (например, 2001:0db8:85a3:0000:0000:8a2e:0370:7334).
> 
> Реализует интерфейсы `Serializable` и `Comparable<InetAddress>` (унаследовано от `InetAddress`).
> 

## Конструкторы
Фабричный метод для получения `Inet6Address`
```java
// Класс Inet6Address имеет приватный конструктор.
// Объекты создаются только через статические фабричные методы.

/*
Создаёт экземпляр Inet6Address по IPv6-адресу.
@host - имя хоста (может быть null, если адрес используется без DNS)
@addr - массив из 16 байт, представляющий IPv6-адрес
@scope_id - числовой идентификатор области видимости (например, для link-local)
@scope_ifname - имя интерфейса (например, "eth0")
*/
public static Inet6Address getByAddress(
	String host, 
	byte[] addr, 
	int scope_id) throws UnknownHostException

public static Inet6Address getByAddress(
	String host, 
	byte[] addr, 
	NetworkInterface nif) throws UnknownHostException
```

## Методы
Класс **`Inet6Address`** НАСЛЕДУЕТ **все методы `InetAddress`**, включая:
- `getHostName()`,
- `getHostAddress()`,
- `isMulticastAddress()`,
- `isSiteLocalAddress()` и др.
    
Ниже перечислены **дополнительные публичные методы**, специфичные для `Inet6Address`.
### Методы работы с scope и интерфейсами
```java
/**
 * Возвращает scope_id для link-local адресов
 * @return числовой идентификатор зоны (0 для global адресов)
 */
public int getScopeId()

/**
 * Возвращает сетевой интерфейс, ассоциированный с адресом
 * @return NetworkInterface или null для global адресов
 */
public NetworkInterface getScopedInterface()

/**
 * Возвращает IPv6 адрес в текстовом формате
 * Поддерживает compressed format (сокращение нулевых групп)
 * @return строковое представление IPv6 адреса
 */
public String getHostAddress()
```

### Методы проверки типа адреса
```java
/**
 * Проверяет, является ли адрес IPv4-совместимым IPv6 адресом
 * Формат: ::x.x.x.x (первые 12 байт нулевые)
 * @return true если адрес IPv4-совместимый
 */
public boolean isIPv4CompatibleAddress()

/**
 * Проверяет, является ли адрес адресом loopback интерфейса
 * Для IPv6: ::1
 * @return true для ::1
 */
public boolean isLoopbackAddress()

/**
 * Проверяет, является ли адрес multicast адресом
 * Для IPv6: ff00::/8
 * @return true если старший байт равен 0xFF
 */
public boolean isMulticastAddress()

/**
 * Проверяет, является ли адрес link-local адресом
 * Для IPv6: fe80::/10
 * @return true для fe80::/10
 */
public boolean isLinkLocalAddress()

/**
 * Проверяет, является ли адрес site-local адресом
 * Для IPv6: fec0::/10 (устаревший, но поддерживается)
 * @return true для fec0::/10
 */
public boolean isSiteLocalAddress()

/**
 * Проверяет, является ли адрес wildcard адресом
 * Для IPv6: ::
 * @return true для :: (все байты нулевые)
 */
public boolean isAnyLocalAddress()
```

## 💡 Особенности и подводные камни 💡

- IPv6-адреса состоят из **16 байт**, а не 4, как у IPv4.
    
- 💡Адреса типа **link-local** (`fe80::/10`) требуют указания **интерфейса** (например, `fe80::1%eth0`).  
    Без scope ID такой адрес **недействителен**, и `Inet6Address` может выбросить `UnknownHostException`.
    
- При создании через `getByAddress()` важно передавать **массив ровно из 16 байт**, иначе будет ошибка.
    
- Методы `equals()` и `hashCode()` сравнивают адреса **побайтно**, включая scope ID.
    
- 💡`Inet6Address` не выполняет сетевых операций — только хранит и описывает адрес.

## 💎 Best Practices 💎

✅ Используй `NetworkInterface` при создании `Inet6Address` для link-local адресов — это предотвратит ошибки.  
✅ Избегай устаревших site-local адресов (`fec0::/10`) — они не используются в современных сетях.  
✅ При сравнении адресов учитывай, что `::1` и `0:0:0:0:0:0:0:1` — одно и то же.  
✅ Для сериализации адресов всегда используй `getHostAddress()` — он возвращает корректную текстовую форму.  
✅ Проверяй поддержку IPv6 в JVM через свойство `java.net.preferIPv6Addresses=true`.
