
> [!NOTE] Интерфейс **`java.net.CookieStore`** определяет `контракт для хранилища` `HTTP cookies`.
> Он используется классом `CookieManager``для сохранения`, `извлечения` и `удаления` `cookies`, УСТАНВЛЕВАЕМЫХ СЕРВЕРАМИ и применяемых при последующих HTTP-запросах.

## Пару слов о назначении
- 💡`CookieStore` — это **абстракция над местом хранения cookies** (в памяти, файле, базе данных и т.д.).
    
- Все cookies, полученные через `CookieManager`, сохраняются в этом хранилище.
    
- Можно реализовать собственную версию для **постоянного хранения** cookies между запусками приложения.
    
- 💡Используется автоматически при вызове `CookieManager.getCookieStore()`

---
## Интерфейс
```java
public interface CookieStore {

    // ===================== 🔹 Добавление cookies =====================

    //✅ Добавляет cookie в хранилище.
    // Параметры:
    //  uri — адрес ресурса, с которым cookie связано;
    //  cookie — экземпляр HttpCookie.
    void add(URI uri, HttpCookie cookie);



    // ===================== 🔹 Получение cookies =====================

    //✅ Возвращает список всех cookies, относящихся к указанному URI.
    // Параметры:
    //  uri — адрес ресурса (может быть null).
    // Возвращает список HttpCookie, подходящих по домену, пути и флагам.
    List<HttpCookie> get(URI uri);

    //✅ Возвращает список всех cookies, хранящихся в этом CookieStore.
    List<HttpCookie> getCookies();

    //✅ Возвращает список всех URI, для которых есть сохранённые cookies.
    List<URI> getURIs();



    // ===================== 🔹 Удаление cookies =====================

    //✅ Удаляет конкретный cookie, связанный с указанным URI.
    // Параметры:
    //  uri — адрес ресурса, с которым cookie связан;
    //  cookie — экземпляр HttpCookie для удаления.
    // Возвращает true, если cookie был найден и удалён.
    boolean remove(URI uri, HttpCookie cookie);

    //✅ Удаляет все cookies из хранилища.
    // Возвращает true, если хранилище было изменено (cookies удалены).
    boolean removeAll();
}
```

## 💡 Особенности и подводные камни 💡

1️⃣ `CookieStore` — это **интерфейс**, и в Java ==по умолчанию доступна лишь== его ==**СТАНДАРТНАЯ РЕАЛИЗАЦИЯ**== в `CookieManager` (в памяти).  

2️⃣ 💡Встроенная реализация хранит cookies в оперативной памяти,  
поэтому после завершения работы программы все cookies исчезают.  

3️⃣ 💡Метод `add()` автоматически заменяет старое cookie с тем же именем, доменом и путём.  

4️⃣ Метод `get(URI)` возвращает **только релевантные cookies**,  
учитывая домен, путь, secure-флаг и срок действия.  

5️⃣ ❗❗❗Методы возвращают **копии коллекций**, а не «живые» списки,  
поэтому прямое редактирование результата не изменяет внутреннее состояние.  

6️⃣ Все реализации должны быть **потокобезопасными**,  
так как `CookieManager` может вызываться из разных потоков одновременно.  

7️⃣ Удаление cookie через `remove()` или `removeAll()` влияет на последующую работу `CookieManager` —  
после этого такие cookies не будут добавляться в заголовки `Cookie:`.

## 💎 Best Practices 💎

✅ ❗❗❗Используй `CookieStore` через `CookieManager`:
```java
CookieManager manager = new CookieManager();
CookieStore store = manager.getCookieStore();
```
✅ 🧠Для долговременного хранения реализуй собственный класс:
```java
public class FileCookieStore implements CookieStore {
    // сохраняет cookies в файл (например, JSON)
}
```

✅ Регулярно удаляй истёкшие cookies:
```java
for (HttpCookie cookie : store.getCookies()) {
    if (cookie.hasExpired()) store.remove(null, cookie);
}
```

✅ Для отладки удобно вывести все cookies:
```java
store.getCookies().forEach(System.out::println);
```

✅ При удалении всех cookies:
```java
store.removeAll();
```

✅ Если приложение многопоточное — используй `Collections.synchronizedList(...)`  
в собственной реализации хранилища для обеспечения безопасности доступа.

## Пример
```java
import java.net.*;
import java.util.*;

public class CookieStoreDemo {
    public static void main(String[] args) throws Exception {
        // 1️⃣ Создаём CookieManager и получаем доступ к CookieStore
        CookieManager manager = new CookieManager(null, CookiePolicy.ACCEPT_ALL);
        CookieHandler.setDefault(manager);
        CookieStore store = manager.getCookieStore();

        // 2️⃣ Добавляем cookie вручную
        HttpCookie cookie = new HttpCookie("user", "Seriy");
        cookie.setDomain(".httpbin.org");
        cookie.setPath("/");
        store.add(new URI("https://httpbin.org/"), cookie);


        // 3️⃣ Делаем HTTP-запрос — CookieManager автоматически применит cookie
        URL url = new URL("https://httpbin.org/cookies");
        try (Scanner sc = new Scanner(url.openStream())) {
            System.out.println("--- Ответ сервера ---");
            while (sc.hasNextLine()) System.out.println(sc.nextLine());
        }

        // 4️⃣ Просмотр cookies в хранилище
        System.out.println("\n--- Cookies в CookieStore ---");
        for (HttpCookie c : store.getCookies())
            System.out.println(c);

        // 5️⃣ Удаляем все cookies
        store.removeAll();
        System.out.println("\nПосле очистки: " + store.getCookies().size() + " cookie(ов).");
    }
}
```
