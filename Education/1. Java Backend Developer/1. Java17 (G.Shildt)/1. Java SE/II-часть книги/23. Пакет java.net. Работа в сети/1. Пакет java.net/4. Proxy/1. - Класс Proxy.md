Proxy-сервер - посредник между пользователем и интернет-ресурсом. 
Аналог VPN-сервера, только без шифрования (но можно настроить шифрование)
![[Pasted image 20251007090711.png]]

> [!NOTE] Класс **`java.net.Proxy`** представляет `сетевой прокси-сервер`,
> через который может выполняться соединение (HTTP, HTTPS, SOCKS и т.д.).
> Он инкапсулирует сведения о типе прокси и адресе прокси-сервера и используется классами, которые поддерживают сетевые подключения — например, `URL`, `URLConnection`, `HttpClient`, `Socket`, `ServerSocket`, и т.д.
> 
> ⭐Реализует интерфейсы: `Serializable`

### Пару слов о назначении
- Класс `Proxy` позволяет явно указать, **через какой прокси** следует устанавливать соединение.
    
- Java поддерживает три основных ==типа прокси==:
    - ⭐**DIRECT** — прямое соединение без прокси;
        
    - ⭐**HTTP** — HTTP-прокси (обычно для `http://` и `https://`);
        
    - ⭐**SOCKS** — универсальный прокси для любых TCP-соединений.
        
- 💡Прокси можно задать либо программно, либо через системные свойства (`http.proxyHost`, `socksProxyHost` и др.).
    
- 💡Совместим со всеми классами, использующими `Socket` и `URLConnection`.

## Конструкторы
```java
//✅ Создаёт объект Proxy указанного типа с заданным адресом.
// Параметры:
// type    — тип прокси 
// - Proxy.Type.HTTP, 
// - Proxy.Type.SOCKS или 
// - Proxy.Type.DIRECT);
// 
// address — объект SocketAddress, представляющий адрес прокси-сервера.
public Proxy(
	Proxy.Type type, 
	SocketAddress address)
```
💬 Если тип — `DIRECT`, параметр `address` должен быть `null`.  

Этот конструктор редко вызывается вручную — чаще прокси создаётся через `ProxySelector` или используется глобальный.

### 📙 Вложенный enum Proxy.Type
```java
public static enum Proxy.Type {
    DIRECT,  // Прямое соединение без прокси
    HTTP,    // HTTP-прокси (используется для HTTP/HTTPS)
    SOCKS    // SOCKS-прокси (универсальный TCP)
}
```

## Методы
### Получение свойств прокси
```java
//✅ Возвращает тип прокси: DIRECT, HTTP или SOCKS.
public Proxy.Type type()

//✅ Возвращает адрес прокси (или null, если тип — DIRECT).
public SocketAddress address()
```

### Общие методы
```java
//✅ Возвращает строковое представление прокси (тип + адрес).
public String toString()

//✅ Сравнивает два объекта Proxy.
public boolean equals(Object obj)

//✅ Возвращает hashCode для объекта Proxy.
public int hashCode()
```

## 💡 Особенности и подводные камни 💡

1️⃣ Класс `Proxy` НЕ ОТКРЫВАЕТ СОЕДНИЕНИЯ САМ — он **только хранит настройки**, которые другие классы используют при подключении.  

2️⃣ ⭐⭐Прокси можно установить **для одного соединения** (например, `URL.openConnection(Proxy)`),  
или **глобально** — через `ProxySelector.setDefault(...)`.  

3️⃣ Тип `Proxy.Type.DIRECT` означает "без прокси" и используется по умолчанию.  
4️⃣ Для `HTTP` и `HTTPS` прокси требуется явный порт (например, `8080`).  

5️⃣ SOCKS-прокси поддерживает TCP-соединения любого типа, включая нестандартные порты.  

6️⃣ Если прокси требует авторизации — нужно использовать `Authenticator.setDefault(...)`.  

7️⃣ ⭐⭐При использовании `HttpClient` можно указать прокси прямо в билдере:
```JAVA
HttpClient.newBuilder()
	.proxy(ProxySelector.of(
		new InetSocketAddress("proxy.example.com", 8080)
	));
```

8️⃣ Если указать неверный прокси — соединение завершится `IOException`.

9️⃣ В системных свойствах Java можно задать глобальные параметры:
```java
-Dhttp.proxyHost=proxy.example.com
-Dhttp.proxyPort=8080
-DsocksProxyHost=localhost
-DsocksProxyPort=1080
```

🔟 Для обхода прокси используйте `Proxy.NO_PROXY`.

## 💎 Best Practices 💎

✅ **Создай отдельный объект `Proxy` для конкретного подключения:**
```java
Proxy proxy = new Proxy(
	Proxy.Type.HTTP,
    new InetSocketAddress("proxy.example.com", 8080)
);
```

✅ **Используй с `URL.openConnection(Proxy)`**:
```java
URL url = new URL("https://example.com");

URLConnection conn = url.openConnection(proxy);
```

✅ **Для HTTP-клиента (Java 11+):**
```java
HttpClient client = HttpClient.newBuilder()
        .proxy(ProxySelector.of(
	        new InetSocketAddress("proxy.example.com", 8080))
	    )
        .build();

```

✅ **При SOCKS-прокси (TCP):**
```java
Socket socket = new Socket(
	new Proxy(
		Proxy.Type.SOCKS,
        new InetSocketAddress("localhost", 1080))
);
```

✅ **Если прокси требует логин/пароль:**
```java
Authenticator.setDefault(new Authenticator() {
    @Override
    protected PasswordAuthentication getPasswordAuthentication() {
        return new PasswordAuthentication(
	        "user", 
	        "password".toCharArray());
    }
});
```

✅ **Выбирай динамически через `ProxySelector`:**
```java
ProxySelector selector = ProxySelector.getDefault();

List<Proxy> proxies = selector.select(new URI("https://example.com"));
```

## Пример
```java
import java.net.*;
import java.io.*;

public class ProxyDemo {
    public static void main(String[] args) throws Exception {
        // 1️⃣ Создаём HTTP-прокси
        Proxy proxy = new Proxy(
	        Proxy.Type.HTTP,
            new InetSocketAddress("proxy.example.com", 8080));


        // 2️⃣ Настраиваем аутентификацию (если требуется)
        Authenticator.setDefault(new Authenticator() {
            @Override
            protected PasswordAuthentication
             getPasswordAuthentication() {
            
                return new PasswordAuthentication(
	                "user", 
	                "pass".toCharArray());
            }
        });


        // 3️⃣ Открываем соединение через прокси
        URL url = new URL("https://httpbin.org/ip");
        URLConnection conn = url.openConnection(proxy);


        // 4️⃣ Читаем ответ сервера
        try (BufferedReader in = new BufferedReader(
                new InputStreamReader(conn.getInputStream()))) {
            String line;
            System.out.println("--- Ответ сервера ---");
            while ((line = in.readLine()) != null) {
                System.out.println(line);
            }
        }


        // 5️⃣ Пример использования ✅с HttpClient✅
        HttpClient client = HttpClient.newBuilder()
                .proxy(ProxySelector.of(
	                new InetSocketAddress("proxy.example.com", 8080))
	            )
                .build();

        HttpRequest request = HttpRequest.newBuilder()
                .uri(URI.create("https://httpbin.org/get"))
                .build();

        HttpResponse<String> response =
                client.send(
	                request, 
	                HttpResponse.BodyHandlers.ofString());

        System.out.println("\n--- Ответ через HttpClient ---");
        System.out.println(response.body());
    }
}
```
