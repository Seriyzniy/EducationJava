## 🧠 Краткая История
В современном Интернете НЕ ИСПОЛЬЗУЮТСЯ устаревшие протоколы вроде `whois`, `finger` и `FTP`. 

Речь идет о WWW - "всемирной паутине'; или веб-сети. 
Веб-сеть представляет собой несвязанную совокупность протоколов и файловых форматов более высокого уровня, объединенных в веб-браузере. 

Один из наиболее важных аспектов веб-сети заключается в том, что Тим Вернере-Ли изобрел масштабируемый способ обнаружения всех сетевых ресурсов. 
При условии надежноrо именования всеrо в веб-сети данный аспект становится очень мощной парадигмой. 

Именно для этоrо предназначен унифицированный указатель ресурса 
(==Uniform Resource Locator== - URL).
Указатель URL обеспечивает достаточно понятную форму для уникальной идентификации или адресации информации в Интернете.
## 🧠 Структура URL

Все URL имеют один и тот же базовый формат, хотя допускаются некоторые вариации. 
Вот два примера: 
`http://www.HerbSchildt.com/` и 
`http://www.HerbSchildt.com:80/index.htm`. 

Спецификация URL основана на 4 компонентах:
### Первый компонент - применяемый протокол
Протокол отделяется от остальной части URL двоеточием (:)
Распространенными протоколами являются НТТР, FTP и файловый, хотя в наши дни почти все де лается через НТТР

### Второй компонент - имя хоста или IР-адрес используемого хоста
Отделяется слева двойной косой чертой (//) и справа косой чертой (/) или дополнительно двоеточием (:).

### Третий компонент - номер порта 
Является необязательным; 
Отделяется слева от имени хоста двоеточием (:) и справа косой чертой (/)

### Четвертый компонент - фактический путь к файлу.
Большинство НТТР-серверов добавляют файл по имени` index.html` к URL, которые относятся непосредственно к ресурсу каталога. 

Таким образом, URL вида `http://www.HerbSchildt.com/` будет таким же, как `http://www.HerbSchildt.com/index.htm.`

# 📕 1. Класс URL

> [!NOTE] Класс **`java.net.URL`** (Uniform Resource Locator — _унифицированный `указатель ресурса`)_
> Используется для `представления адреса ресурса в Интернете` и работы с ним программно.
> 
> ⭐Реализует интерфейс: `Serializable`

(может быть сериализован, например, для передачи по сети или записи в файл).

Он инкапсулирует все компоненты URL (протокол, хост, порт, путь, якорь, параметры)  

Позволяет легко обращаться к ресурсам по протоколам `HTTP`, `HTTPS`, `FTP`, `file`, `jar` и т.д.

## Конструкторы
```java
//✅ Создаёт объект URL на основе полного адреса.
// Параметр:
//  spec — строка с ПОЛНЫМ URL, например "https://example.com:8080/index.html".
URL(String spec)
    throws MalformedURLException

//✅ Создаёт URL из заданного протокола, хоста и пути.
// Параметры:
//  protocol — строка с именем протокола ("http", "ftp", "file" и т.д.);
//  host — доменное имя или IP-адрес;
//  file — путь к ресурсу (может включать имя файла).
URL(String protocol, String host, String file)
    throws MalformedURLException

//✅ Создаёт URL из протокола, хоста, порта и пути.
// Параметры:
//  protocol — протокол;
//  host — домен или IP;
//  port — порт (−1 = использовать порт по умолчанию для протокола);
//  file — путь к ресурсу.
URL(String protocol, String host, int port, String file)
    throws MalformedURLException

//✅ Создаёт новый URL, разрешая относительный путь относительно базового URL.
// Параметры:
//  context — базовый URL, относительно которого будет вычисляться новый;
//  spec — строка с относительным или абсолютным URL.
URL(URL context, String spec)
    throws MalformedURLException
```

💬 Конструкторы могут выбрасывать `MalformedURLException`, если строка URL имеет неверный формат.

## Методы

### Получение компонентов URL
```java
//✅ Возвращает протокол (например, "http", "ftp").
String getProtocol()

//✅ Возвращает имя хоста (например, "www.google.com").
String getHost()

//✅ Возвращает порт. Если порт не указан — возвращает -1.
int getPort()

//✅ Возвращает путь к ресурсу (например, "/index.html").
String getFile()

//✅ Возвращает путь без параметров запроса (например, "/page").
String getPath()

//✅ Возвращает часть после символа '#' (якорь в документе).
String getRef()

//✅ Возвращает строку запроса (часть после '?', например "id=123").
String getQuery()

//✅ Возвращает пользовательскую часть (userInfo) URL, если есть, например: "user:pass".
String getUserInfo()

//✅ Возвращает объект authority (например, "user:pass@host:port").
String getAuthority()
```

### Работа с ресурсом
```java
//✅ Открывает поток для чтения ресурса, на который указывает URL.
// Эквивалентно вызову url.openConnection().getInputStream().
// Используется для простого скачивания содержимого.
InputStream openStream()
    throws IOException


//✅ Открывает соединение с ресурсом и возвращает объект URLConnection
// для получения заголовков, метаданных и потоков.
URLConnection openConnection()
    throws IOException

//✅ Открывает соединение через заданный прокси-сервер.
URLConnection openConnection(Proxy proxy)
    throws IOException
```

### Сравнение и строковое представление
```java
//✅ Сравнивает два URL по содержимому (протокол, хост, порт, путь и т.д.).
boolean equals(Object obj)

//✅ Возвращает строковое представление полного URL.
String toString()

//✅ Возвращает строку в виде «канонической формы» URL (без избыточных частей).
String toExternalForm()

//✅ Проверяет, указывают ли два URL на один и тот же ресурс (включая разрешение DNS).
boolean sameFile(URL other)
```

## 💡 Особенности и подводные камни 💡

- ❗❗❗`URL` — **не устанавливает соединение** при создании объекта.  
    Соединение открывается только при вызове `openStream()` или `openConnection()`.
    
- 💥 Если строка URL имеет неправильный формат, выбрасывается `MalformedURLException`.
    
- Поддерживает относительные ссылки при передаче базового URL (`new URL(base, relative)`).
    
- ⭐Для HTTPS требуется наличие подходящего SSL-провайдера (обычно встроен в JVM).
    
- 🧠 `URL.equals()` может вызывать DNS-резолвинг, что делает сравнение затратным.
    
- ⚠️ Класс `URL` не является потокобезопасным, если используется для изменения состояния соединения.

---
## 💎 Best Practices 💎

✅ Используй `URL` **только для представления адреса** — для сетевого взаимодействия лучше применять `URLConnection` или `HttpURLConnection`.

✅ Всегда обрабатывай `MalformedURLException` при создании URL из строки.  

✅ Для чтения данных из URL — используй буферизированные потоки:
```java
try (BufferedReader in = new BufferedReader(
         new InputStreamReader(url.openStream(), 
         StandardCharsets.UTF_8))
) {
    ...
}
```

✅ При работе с относительными ссылками всегда передавай базовый URL:
```java
URL base = new URL("https://example.com/docs/");
URL relative = new URL(base, "page.html"); // -> https://example.com/docs/page.html
```

✅ Если нужно больше контроля (заголовки, тайм-ауты, POST-данные) — используй `HttpURLConnection`.

## Пример
```java
import java.net.*;
import java.io.*;

public class URLDemo {
    public static void main(String[] args) {
        try {
            URL url = new URL("https://example.org/index.html?user=test#section1");

            // Выводим компоненты URL
            System.out.println("Протокол: " + url.getProtocol());
            System.out.println("Хост: " + url.getHost());
            System.out.println("Порт: " + url.getPort());
            System.out.println("Файл: " + url.getFile());
            System.out.println("Путь: " + url.getPath());
            System.out.println("Запрос: " + url.getQuery());
            System.out.println("Якорь: " + url.getRef());

            // Открываем поток и читаем первые строки ресурса
            try (BufferedReader in = new BufferedReader(
                    new InputStreamReader(url.openStream()))) {
                System.out.println("\n--- Содержимое страницы ---");
                for (int i = 0; i < 5; i++) {
                    String line = in.readLine();
                    if (line == null) break;
                    System.out.println(line);
                }
            }

        } catch (MalformedURLException e) {
            System.out.println("Неверный формат URL: " + e.getMessage());
        } catch (IOException e) {
            System.out.println("Ошибка при чтении: " + e.getMessage());
        }
    }
}
```
