
> [!NOTE] Класс **`java.net.http.HttpClient`** — это **новый стандартный `HTTP-клиент` Java (с версии 11)**,
> Предназначенный для `ОТПРАВКИ HTTP-запросов` и `ПОЛУЧЕНИЯ ответов`.  
> 
> Он заменяет устаревшие API (`HttpURLConnection`) и предоставляет удобные средства для работы с: 
> - 💡HTTP 1.1 и HTTP 2, 
> - 💡асинхронными запросами и 
> - 💡настройкой SSL, редиректов, прокси и т. д.
> 
> ⭐Реализует интерфейсы: `AutoCloseable`

(с версии Java 21, когда клиент получил метод `close()` для освобождения ресурсов).

### Пару слов о назначении
- `HttpClient` — высокоуровневый, ==потокобезопасный== клиент для выполнения HTTP-запросов.
    
- ❗❗❗Поддерживает как **==синхронные==**, так и **==асинхронные==** операции (через `CompletableFuture`).
    
- 🧠Реализует поддержку **HTTP/2**, автоматическое управление редиректами, прокси и SSL.
    
- Работает в связке с классами `HttpRequest` и `HttpResponse`.

## Конструктор приватный — используется **паттерн “Builder”**
_Фабричные методы_
```java
//✅Возвращает 💡новый экземпляр HttpClient со стандартными настройками💡
// Эквивалентен: HttpClient.newBuilder().build()
public static HttpClient newHttpClient()


//✅ Возвращает БИЛДЕР для 💡создания настраиваемого клиента.💡
// Можно указать версию HTTP, редиректы, прокси, SSLContext, Executor и т.д.
public static HttpClient.Builder newBuilder()
```

## Методы
### Отправка запросов (синхронная и асинхронная работа)
```java
//✅ Отправляет запрос и возвращает ответ (синхронно).
// Параметры:
//  request  — объект HttpRequest (описание запроса);
//  responseBodyHandler — обработчик тела ответа (например, BodyHandlers.ofString()).
// Возвращает HttpResponse<T> с кодом, заголовками и телом.
public <T> HttpResponse<T> send(
	HttpRequest request,
	HttpResponse.BodyHandler<T> responseBodyHandler)
	throws IOException, InterruptedException



//✅ Отправляет запрос АСИНХРОННО (не блокирует поток).
// Возвращает CompletableFuture<HttpResponse<T>>.
public <T> CompletableFuture<HttpResponse<T>> sendAsync(
	HttpRequest request, 
	HttpResponse.BodyHandler<T> responseBodyHandler)

//✅ Асинхронная отправка с отдельным обработчиком ошибок для тела.
public <T> CompletableFuture<HttpResponse<T>> sendAsync(
        HttpRequest request,
        HttpResponse.BodyHandler<T> responseBodyHandler,
        HttpResponse.PushPromiseHandler<T> pushPromiseHandler)
```

### Получение и настройка параметров клиента
```java
//✅ Возвращает версию протокола HTTP (HTTP_1_1 или HTTP_2).
public HttpClient.Version version()

//✅ Возвращает стратегию обработки редиректов (ALWAYS, NEVER, NORMAL).
public HttpClient.Redirect followRedirects()

//✅ Возвращает SSLContext, используемый этим клиентом.
public SSLContext sslContext()

//✅ Возвращает Executor, применяемый для асинхронных операций (если указан).
public Optional<Executor> executor()

//✅ Возвращает прокси-настройку (если задана через ProxySelector).
public Optional<ProxySelector> proxy()

//✅ Возвращает селектор аутентификаторов (например, для BasicAuth).
public Optional<Authenticator> authenticator()

//✅ Возвращает настройку таймаута соединения.
public Optional<Duration> connectTimeout()
```

## 📙 Вложенный интерфейс Builder

> [!NOTE] Используется `для конфигурации клиента` перед созданием.
> 

```java
public static interface HttpClient.Builder {

    //✅ Устанавливает 💡версию HTTP💡 по умолчанию (HTTP_1_1 или HTTP_2).
    HttpClient.Builder version(HttpClient.Version version);

    //✅ Определяет стратегию 💡обработки редиректов💡.
    HttpClient.Builder followRedirects(HttpClient.Redirect policy);

    //✅ Устанавливает 💡ПРОКСИ💡 (ProxySelector.of(...)).
    HttpClient.Builder proxy(ProxySelector selector);


    //✅ Устанавливает аутентификатор (например, BasicAuthenticator).
    HttpClient.Builder authenticator(Authenticator authenticator);

    //✅ Устанавливает SSLContext (например, свой truststore).
    HttpClient.Builder sslContext(SSLContext sslContext);


    //✅ Устанавливает 💡пользовательский Executor💡 для асинхронных задач.
    HttpClient.Builder executor(Executor executor);


    //✅ Задаёт таймаут соединения.
    HttpClient.Builder connectTimeout(Duration duration);

    //✅ Завершает конфигурацию и создаёт HttpClient.
    HttpClient build();
}
```

## 💡 Особенности и подводные камни 💡

1️⃣ Класс `HttpClient` появился в **Java 11** (изначально в incubator Java 9).  

2️⃣ Клиент ==**потокобезопасен**== — один экземпляр можно использовать во многих потоках.  

3️⃣ 💡Асинхронные методы (`sendAsync`) используют **CompletableFuture**, что удобно для реактивных сценариев.  

4️⃣ Поддерживает **HTTP/2**, но может автоматически понижаться до HTTP/1.1, если сервер не поддерживает HTTP/2.  

5️⃣ 💡`HttpClient` НЕ ХРАНИТ cookies по умолчанию. Для этого нужно установить `CookieHandler`:
```java
CookieHandler.setDefault(new CookieManager());
```

6️⃣ ❗❗❗По умолчанию **не следует редиректам** (`Redirect.NEVER`). Нужно включать явно.  

7️⃣ ❗❗❗Если используется HTTPS — клиент применяет **системный SSLContext**.  ❗❗❗

8️⃣ ⭐⭐⭐Не стоит создавать клиент для каждого запроса — он дорогой по ресурсам. Лучше использовать **==singleton==**.  ⭐⭐⭐

9️⃣ Клиент не поддерживает multipart-запросы напрямую — требуется ручная сборка тела.

## 💎 Best Practices 💎

✅ ⭐⭐⭐Создавай клиент **один раз** и переиспользуй:
```java
HttpClient client = HttpClient.newHttpClient();
```

✅ ⭐⭐⭐Для конфигурирования используй `builder`:
```java
HttpClient client = HttpClient.newBuilder()
        .version(HttpClient.Version.HTTP_2)
        .followRedirects(HttpClient.Redirect.NORMAL)
        .connectTimeout(Duration.ofSeconds(10))
        .build();
```

✅ 🧠🧠Для **асинхронных запросов** — `sendAsync()`, для простых — `send()`.

✅ Используй `BodyHandlers.ofString()`, `ofFile()`, `ofInputStream()` для тела ответа.

✅ ⭐⭐⭐Для JSON-запросов — вручную указывай заголовки:
```java
HttpRequest request = HttpRequest.newBuilder(uri)
    .header("Content-Type", "application/json")
    .POST(HttpRequest.BodyPublishers.ofString(json))
    .build();
```

✅ Добавь CookieManager, если нужно хранить сессии.  

✅ Для HTTPS с собственным сертификатом — настрой `sslContext()`.  

✅ ⭐⭐⭐Используй `CompletableFuture.thenApply(...)` для ==асинхронной обработки ответов==.

## Пример
```java
import java.net.http.*;
import java.net.URI;
import java.time.Duration;

public class HttpClientDemo {
    public static void main(String[] args) throws Exception {
        // 1️⃣✅ Создаём клиент
        HttpClient client = HttpClient.newBuilder()
                .version(HttpClient.Version.HTTP_2)
                .followRedirects(HttpClient.Redirect.NORMAL)
                .connectTimeout(Duration.ofSeconds(5))
                .build();

        // 2️⃣✅ Формируем запрос
        HttpRequest request = HttpRequest.newBuilder()
                .uri(URI.create("https://httpbin.org/get"))
                .header("User-Agent", "JavaHttpClientDemo/1.0")
                .GET()
                .build();


        // 3️⃣✅ Отправляем синхронно
        HttpResponse<String> response = client.send(request, HttpResponse.BodyHandlers.ofString());

        // 4️⃣✅ Выводим результат
        System.out.println("Статус: " + response.statusCode());
        System.out.println("Заголовки: " + response.headers());
        System.out.println("Тело:\n" + response.body());



        // 5️⃣✅ Асинхронный пример
        HttpRequest postReq = HttpRequest.newBuilder()
                .uri(URI.create("https://httpbin.org/post"))
                .header("Content-Type", "application/json")
                .POST(HttpRequest.BodyPublishers.ofString("{\"msg\":\"Hello\"}"))
                .build();

        client.sendAsync(postReq, HttpResponse.BodyHandlers.ofString())
                .thenApply(HttpResponse::body)
                .thenAccept(System.out::println)
                .join();
    }
}
```
