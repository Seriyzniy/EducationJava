
> [!NOTE] Класс **`java.net.http.HttpResponse<T>`** представляет `HTTP-ответ`,
> возвращаемый сервером после выполнения запроса через `HttpClient`.
> 
> Он содержит:
> - код состояния (status code), 
> - заголовки, 
> - тело ответа (в виде объекта типа `T`), 
> - URI конечного ресурса (с учётом редиректов), 
> - версию протокола HTTP (1.1 / 2).

### Пару слов о назначении
- `HttpResponse` — это ==контейнер результата HTTP-запроса==.
    
- ❗❗❗Его экземпляры создаются внутренне методом `HttpClient.send()` или `sendAsync()`.
    
- 🧠Тип `T` — это тип тела ответа, определяемый обработчиком (`BodyHandler<T>`), например:
    - `String` (через `BodyHandlers.ofString()`);
        
    - `InputStream` (`ofInputStream()`);
        
    - `Path` (`ofFile()`);
        
    - `Void` (`discarding()`).
        
- ⭐⭐`HttpResponse` можно использовать как в **синхронных**, так и в **асинхронных** вызовах.

## Интерфейс
```java
public interface HttpResponse<T> {

    // ================= 🔹 Основная информация ==================

    //✅ Возвращает статус-код HTTP (например, 200, 404, 500 и т.д.)
    int statusCode();

    //✅ Возвращает тело ответа (тип T определяется BodyHandler’ом).
    T body();

    //✅ Возвращает заголовки ответа.
    HttpHeaders headers();

    //✅ Возвращает URI, по которому был выполнен запрос (с учётом редиректов).
    URI uri();

    //✅ Возвращает версию HTTP (HTTP_1_1 или HTTP_2).
    HttpClient.Version version();

    //✅ Возвращает запрос, по которому получен этот ответ.
    HttpRequest request();

    //✅ Возвращает предшествующие ответы, если были редиректы.
    Optional<HttpResponse<T>> previousResponse();



    // ==============🔹 Асинхронное получение тела ================

    //✅ Возвращает объект SSLSession (если использовалось HTTPS).
    Optional<SSLSession> sslSession();

    //✅ Возвращает объект с информацией о соединении (HTTP/2 streams и т.д.).
    Optional<HttpResponse.ResponseInfo> responseInfo();
}
```

## 📙 Вложенные интерфейсы и классы

### 1️⃣ `HttpResponse.BodyHandler<T>`
> [!NOTE] Определяет, `КАК ОБРАБАТЫВАТЬ` `тело ответа`
> 

```java
public static interface BodyHandler<T> {
    //✅ Принимает ResponseInfo и возвращает BodySubscriber<T>,
    // который будет обрабатывать тело ответа.
    HttpResponse.BodySubscriber<T> apply(
	    HttpResponse.ResponseInfo responseInfo);
}
```

### 2️⃣`HttpResponse.BodyHandlers`
> [!NOTE]  Вспомогательный класс `с готовыми обработчиками тела`.
> 🧠Неявная реализация `BodyHandler` (в сигнатуре не указан `implements`)

```java
public static class BodyHandlers {
    //✅ Читает тело в виде строки (UTF-8 по умолчанию).
    public static BodyHandler<String> ofString();

    //✅ Читает тело и записывает его в файл.
    public static BodyHandler<Path> ofFile(Path path);

    //✅ Возвращает тело как InputStream (удобно для потоковой обработки).
    public static BodyHandler<InputStream> ofInputStream();

    //✅ Игнорирует тело (если не нужно).
    public static BodyHandler<Void> discarding();

    //✅ Возвращает тело как массив байт.
    public static BodyHandler<byte[]> ofByteArray();
}
```

###  3️⃣ `HttpResponse.BodySubscriber<T>`
> [!NOTE] Определяет, `КАК ПОЛУЧИТЬ` `тело ответ`а в виде `Publisher`/`Subscriber`
> 

```java
public static interface BodySubscriber<T> extends Flow.Subscriber<List<ByteBuffer>> {
    CompletionStage<T> getBody();
}
```
💬 ⭐⭐Обычно программист напрямую не реализует этот интерфейс,  
а ==использует готовые фабрики из== `BodyHandlers`.

### 4️⃣`HttpResponse.ResponseInfo`
> [!NOTE] Минимальные метаданные о ответе (без тела).
>

```java
public static interface ResponseInfo {
    int statusCode();       // код состояния
    HttpHeaders headers();  // заголовки
    HttpClient.Version version(); // версия протокола
}
```

## 💡 Особенности и подводные камни 💡

1️⃣ `HttpResponse` **НЕ СОЗДАЕТСЯ ВРУЧНУЮ** — только через `HttpClient.send()` или `sendAsync()`.  

2️⃣ ⭐⭐Тип `T` (тело ответа) зависит от выбранного `BodyHandler`.  

3️⃣ ❗❗❗`HttpResponse` **не буферизует тело** автоматически — поведение определяется BodyHandler’ом.  

4️⃣ ⭐⭐Для больших ответов рекомендуется `ofFile()` или `ofInputStream()` (иначе — OutOfMemoryError). 

5️⃣ Методы `headers()` и `statusCode()` всегда доступны, даже если тело не считано.  

6️⃣ Для HTTPS-доступа можно извлечь SSL-сессию: `sslSession().get()`.  

7️⃣ При редиректах важно использовать `previousResponse()` для получения истории переходов.  

8️⃣ ⭐⭐⭐Для асинхронных запросов `sendAsync()` возвращает `CompletableFuture<HttpResponse<T>>`.  

9️⃣ Заголовки возвращаются как `HttpHeaders` — безопасно, удобно и неизменяемо.  

🔟 Версия HTTP определяется автоматически — может быть HTTP/1.1 или HTTP/2 в зависимости от сервера.

## 💎 Best Practices 💎

✅ Для текстовых ответов — используй (toString):
```java
HttpResponse<String> response = client.send(
	request, 
	HttpResponse.BodyHandlers.ofString());
```

✅ ⭐⭐Для бинарных данных (файлы, PDF):
```java
client.send(
	request, 
	HttpResponse.BodyHandlers.ofFile(Paths.get("file.pdf")));
```

✅ ⭐⭐Для асинхронной загрузки (thenApply):
Можно использовать методы `ComplitableFuture`, т.к `sendAsync` возвращает именно его
```java
client.sendAsync(request, HttpResponse.BodyHandlers.ofString())
      .thenApply(HttpResponse::body)
      .thenAccept(System.out::println);
```

✅ ⭐⭐Для ПОТОКОВОЙ ОБРАБОТКИ (например, JSON-потоков):
```java
client.sendAsync(request, HttpResponse.BodyHandlers.ofInputStream())
      .thenApply(HttpResponse::body)
      .thenAccept(is -> new BufferedReader(new InputStreamReader(is))
                         .lines().forEach(System.out::println));
```

✅ Для редиректов:
```java
response.previousResponse().ifPresent(prev -> 
    System.out.println("Редирект с: " + prev.uri()));
```

✅ При HTTPS:
```java
response.sslSession().ifPresent(ssl -> 
    System.out.println("SSL protocol: " + ssl.getProtocol()));
```

## Пример
```java
import java.net.http.*;
import java.net.URI;
import java.nio.file.Path;
import java.time.Duration;

public class HttpResponseDemo {
    public static void main(String[] args) throws Exception {
        HttpClient client = HttpClient.newBuilder()
                .followRedirects(HttpClient.Redirect.NORMAL)
                .connectTimeout(Duration.ofSeconds(5))
                .build();


        // 1️⃣ Пример текстового ответа
        HttpRequest get = HttpRequest.newBuilder()
                .uri(URI.create("https://httpbin.org/get"))
                .GET()
                .build();

        HttpResponse<String> response =
                client.send(get, HttpResponse.BodyHandlers.ofString());

        System.out.println("Статус: " + response.statusCode());
        System.out.println("Версия: " + response.version());
        System.out.println("Заголовки: " + response.headers());
        System.out.println("Тело:\n" + response.body());



        // 2️⃣ Пример сохранения ответа в файл
        HttpRequest download = HttpRequest.newBuilder()
                .uri(URI.create("https://httpbin.org/image/png"))
                .build();

        HttpResponse<Path> fileResponse =
                client.send(
	                download, 
	                HttpResponse.BodyHandlers
		                .ofFile(Path.of("download.png"))
		        );

        System.out.println("\nФайл сохранён в: " + fileResponse.body());



        // 3️⃣ Асинхронный пример
        HttpRequest async = HttpRequest.newBuilder()
                .uri(URI.create("https://httpbin.org/uuid"))
                .build();

        client.sendAsync(async, HttpResponse.BodyHandlers.ofString())
              .thenApply(HttpResponse::body)
              .thenAccept(body -> System.out.println("\nUUID: " + body))
              .join();
    }
}
```
