
> [!NOTE] Класс **`Collectors`** — это финальный `служебный класс` (utility class)
> Предоставляет `набор статических фабричных методов` для `создания экземпляров `интерфейса `Collector`

Он используется для **сбора элементов Stream в различные структуры данных**, **агрегации** и **группировки результатов**.

---

## Фабричные методы
### 1. Коллекторы для сбора в коллекции
Преобразуют Stream в коллекции (List, Set, Map и др.)
```java
// Собирает элементы потока в List (ArrayList)
public static <T> Collector<T, ?, List<T>> toList()

// Собирает элементы потока в Set (HashSet)
public static <T> Collector<T, ?, Set<T>> toSet()


// Собирает элементы в коллекцию, тип которой задаётся supplier’ом
// collectionFactory — функция, создающая коллекцию (например: ArrayList::new)
public static <T, C extends Collection<T>> 
Collector<T, ?, C> 
	toCollection(
		Supplier<C> collectionFactory)

```

### 2. Коллекторы для мап (Map)
```java
// Собирает элементы в Map. Ключ и значение определяются мапперами.
// ⚠️ Если ключи дублируются — выбрасывается IllegalStateException.
public static <T, K, U> 
Collector<T, ?, Map<K, U>> 
	toMap(
        Function<? super T, ? extends K> keyMapper,
        Function<? super T, ? extends U> valueMapper)


// Разрешает коллизию ключей с помощью mergeFunction.
public static <T, K, U> 
Collector<T, ?, Map<K, U>> 
	toMap(
        Function<? super T, ? extends K> keyMapper,
        Function<? super T, ? extends U> valueMapper,
        BinaryOperator<U> mergeFunction)


// Полный контроль: можно задать стратегию разрешения коллизий и тип карты.
public static <T, K, U, M extends Map<K, U>> 
Collector<T, ?, M> 
	toMap(
        Function<? super T, ? extends K> keyMapper,
        Function<? super T, ? extends U> valueMapper,
        BinaryOperator<U> mergeFunction,
        Supplier<M> mapFactory)
```

### 3. Группировка и разделение
```java
// Группирует элементы по ключу (классификатору) в Map<K, List<T>>.
public static <T, K> 
Collector<T, ?, Map<K, List<T>>> 
	groupingBy(
        Function<? super T, ? extends K> classifier)


// Группировка + downstream-коллектор (например, counting() внутри групп).
public static <T, K, A, D> 
Collector<T, ?, Map<K, D>> 
	groupingBy(
        Function<? super T, ? extends K> classifier,
        Collector<? super T, A, D> downstream)


// Полный контроль: тип карты + вложенный коллектор.
public static <T, K, D, A, M extends Map<K, D>> 
Collector<T, ?, M> 
	groupingBy(
        Function<? super T, ? extends K> classifier,
        Supplier<M> mapFactory,
        Collector<? super T, A, D> downstream)



// Делит поток на две группы: true и false.
public static <T> 
Collector<T, ?, Map<Boolean, List<T>>> 
	partitioningBy(
        Predicate<? super T> predicate)


// Разделение + последующая операция (например counting()).
public static <T, D, A> 
Collector<T, ?, Map<Boolean, D>> 
	partitioningBy(
        Predicate<? super T> predicate,
        Collector<? super T, A, D> downstream)
```

### 4. Сводные операции и статистика
```java
// Считает количество элементов.
public static <T> Collector<T, ?, Long> counting()


// Возвращает максимальный элемент по компаратору.
public static <T> 
Collector<T, ?, Optional<T>> 
	maxBy(
		Comparator<? super T> comparator)


// Возвращает минимальный элемент по компаратору.
public static <T> 
Collector<T, ?, Optional<T>> 
	minBy(
		Comparator<? super T> comparator)



// Возвращает статистику: count, sum, min, average, max для double.
public static <T> 
Collector<T, ?, DoubleSummaryStatistics> 
	summarizingDouble(
		ToDoubleFunction<? super T> mapper)

// Аналогично, но для int.
public static <T> 
Collector<T, ?, IntSummaryStatistics> 
	summarizingInt(
		ToIntFunction<? super T> mapper)


// Аналогично, но для long.
public static <T> 
Collector<T, ?, LongSummaryStatistics> 
	summarizingLong(
		ToLongFunction<? super T> mapper)



public static <T> Collector<T, ?, Double> averagingInt(
	ToIntFunction<? super T> mapper);
	
public static <T> Collector<T, ?, Double> averagingLong(
	ToLongFunction<? super T> mapper);
	
public static <T> Collector<T, ?, Double> averagingDouble(
	ToDoubleFunction<? super T> mapper);
	
```

### 5. Сведение в строку
```java
// Объединяет CharSequence в одну строку без разделителей.
public static Collector<CharSequence, ?, String> joining()


// Объединяет элементы с разделителем.
public static 
Collector<CharSequence, ?, String> 
	joining(
		CharSequence delimiter)


// Объединяет элементы с префиксом, разделителем и суффиксом.
public static 
Collector<CharSequence, ?, String> 
	joining(
		CharSequence delimiter, 
		CharSequence prefix, 
		CharSequence suffix)
```

### 6. Преобразование значений
```java
// Применяет mapper к каждому элементу и передаёт результат в downstream коллектор.
public static <T, U, A, R> 
Collector<T, A, R> 
	mapping(
        Function<? super T, ? extends U> mapper,
        Collector<? super U, A, R> downstream)

// Аналог map + flatMap для коллектора (Java 9+)
public static <T, U, A, R> 
Collector<T, A, R> 
	flatMapping(
        Function<? super T, ? extends Stream<? extends U>> mapper,
        Collector<? super U, A, R> downstream)
```

### Прочие методы
```java
// ✅ Преобразование результата после сбора
public static <T, A, R, RR> 
Collector<T, A, RR>
	collectingAndThen(
		Collector<T, A, R> downstream,
        Function<R, RR> finisher);


// ✅ Редукция (аналог reduce)
public static <T> 
Collector<T, ?, Optional<T>>

		BinaryOperator<T> op);

Collector<T, ?, T> 
	reducing(
		T identity, 
		BinaryOperator<T> op);
		
		
public static <T, U> 
Collector<T, ?, U>
	reducing(
		U identity,
		Function<? super T, ? extends U> mapper,
        BinaryOperator<U> op);
```

## 💡 Особенности и подводные камни 💡

1️⃣ 💡`Collectors.toList()` и `Collectors.toSet()` **не гарантируют конкретную реализацию** коллекции (например, `ArrayList` или `HashSet` — под капотом, но это не обещано спецификацией).  

2️⃣ Методы `groupingBy` и `partitioningBy` создают **новую Map**, но не гарантируют потокобезопасность.  

3️⃣ 🧠При использовании `toMap` возможны **ошибки коллизий ключей** — обязательно указывай `mergeFunction`, если ключи могут повторяться.  

4️⃣ 💡`collectingAndThen` часто используется для **создания неизменяемых коллекций** (`Collectors.collectingAndThen(toList(), List::copyOf)`), но важно помнить, что внутренний коллектор всё ещё изменяемый.  

5️⃣ `reducing()` в `parallelStream()` может быть медленнее `summingInt()` или `counting()` — последние оптимизированы.

---
## 💎 Best Practices 💎

✅ 💡Используй `toList()` / `toSet()` для базовых случаев,  
но при необходимости контролировать тип коллекции — применяй `toCollection()`.

✅ Для неизменяемых коллекций — `collectingAndThen(..., List::copyOf)`.

✅ Для статистики чисел — `summarizingInt/Long/Double`.

✅ Для сложных многоуровневых группировок — комбинируй `groupingBy` + `mapping` + `collectingAndThen`.

✅ ⭐⭐При параллельных потоках — используй **потокобезопасные коллекции** или `toConcurrentMap()` (если нужна конкурентная Map).

## Примеры
```java
List<String> words = List.of("apple", "banana", "avocado", "cherry");


// ✅ Сбор в список
List<String> result = words.stream()
                           .filter(w -> w.startsWith("a"))
                           .collect(Collectors.toList());


// ✅ Преобразование в строку
String joined = words.stream()
                     .collect(Collectors.joining(", ", "[", "]"));
// → [apple, banana, avocado, cherry]

// ✅ Подсчёт количества
long count = words.stream().collect(Collectors.counting()); // 4

// ✅ Группировка по первой букве
Map<Character, List<String>> grouped =
    words.stream()
         .collect(Collectors.groupingBy(w -> w.charAt(0)));
// {a=[apple, avocado], b=[banana], c=[cherry]}

// ✅ Преобразование результата
List<String> immutable =
    words.stream()
         .collect(Collectors.collectingAndThen(Collectors.toList(),
                                               List::copyOf));
```
