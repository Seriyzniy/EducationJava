Операции, которые сводят поток к единственному значению:
- `min()`, `max()` - находят минимум/максимум
- `count()` - подсчитывает количество элементов
- `reduce()` - обобщенная операция редукции
    

**Все ==операции редукции являются терминальными.==**

## Формы метода `reduce()`
```java
// Форма 1: Возвращает Optional
Optional<T> reduce(BinaryOperator<T> accumulator)

// Форма 2: Возвращает значение типа T с начальным значением
T reduce(T identityVal, BinaryOperator<T> accumulator)
```

В обоих методах используется функциональный интерфейс `BinaryOperator`. 
В котором НАСЛЕДУТЕСЯ метод ОТ `BiFunction`
```java
T apply(T val, T val2)
```
- `val` - предыдущий результат (или `identityVal`, или первый элемент)
- `val2` - следующий элемент потока

## Примеры
```java
ArrayList<Integer> myList = new ArrayList<>();
myList.add(2); myList.add(3); myList.add(10); 
myList.add(2); myList.add(5); myList.add(2);

// 2 * 3 = 6
// 6 * 10 = 60
// 60 * 2 = 120 * 5 = 600 * 2 = 1200

// Способ 1: с Optional
Optional<Integer> productObj = myList
	.stream()
	.reduce((a,b) -> a*b);
	
if(productObj.isPresent())
    System.out.println("Произведение как Optional: " + productObj.get());


// Способ 2: с начальным значением
int product = myList
	.stream()
	.reduce(1, (a,b) -> a*b);
System.out.println("Произведение как int: " + product);


//Вывод
Произведение как Optional: 1200
Произведение как int: 1200
```

### Сложная редукция с условием
```java
// Произведение только четных значений
ArrayList<Integer> myList = new ArrayList<>();
myList.add(2); myList.add(3); myList.add(10); 
myList.add(2); myList.add(5); myList.add(2);


int evenProduct = myList
		.stream()
		.reduce(1, (a, b) -> {
	if(b % 2 == 0) return a*b; 
	else return a;
});

/*
	Только ЕСЛИ b четное
	a = 2
	b = 3 (нечетное) => return a
	
	a = 2
	b = 10 (четное) => return a*b = 20
	
	a = 20
	b = 2 (четное) => return a*b = 40
	
	a = 40
	b = 5 (нечетное) => return a
	
	a = 40
	b = 2 (четное) => return a*b = 80
*/

System.out.println(evenProduct);

// Вывод 
80
```

### Требования к аккумулирующей операции
1. **Без состояния** - не зависит от внешней информации
    
2. **Без побочных эффектов** - не изменяет источник данных
    
3. **Ассоциативность** - порядок операций не влияет на результат
    - Пример: `(10 * 2) * 7 = 10 * (2 * 7)`
