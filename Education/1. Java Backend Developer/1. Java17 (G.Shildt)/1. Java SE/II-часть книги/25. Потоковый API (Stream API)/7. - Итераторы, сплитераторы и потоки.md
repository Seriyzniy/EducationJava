Хотя ==поток не является объектом хранения== данных, ==ОДНАКО== с помощью итератора ==можно проходить по его элементам== почти так же, как в случае применения итератора для прохода по элементам коллекции. 

В потоковом API поддерживаются два типа итераторов: 
- традиционный итератор 
- сплитератор, появившийся в версии JDK 8

# 📕 1. Итераторы и потоки
##  Получение итератора для потока
Чтобы получить итератор для потока, понадобится вызвать метод `iterator()` класса `Stream`
```JAVA
// Получить итератор для элементов потока
Iterator<T> iterator()
```

### Пример использования итератора с потоком
```java
// Создание списка и получение потока
ArrayList<String> myList = new ArrayList<>();
myList.add("Alpha");
myList.add("Beta"); 
myList.add("Gamma");

// Получение потока и итератора
Stream<String> myStream = myList.stream();
Iterator<String> itr = myStream.iterator();

// Обход элементов потока
while(itr.hasNext()) {
    System.out.println(itr.next());
}

Alpha
Beta
Gamma
```

# 📕 2. Сплитераторы и потоки
Сплитератор - это альтернатива итератору, особенно когда задействована параллельная обработка

## Основные методы Spliterator
```java
// Выполняет действие с следующим элементом и перемещает итератор
boolean tryAdvance(Consumer<? super T> action)

// Выполняет действие для всех оставшихся элементов  
default void forEachRemaining(Consumer<? super T> action)

// Разделяет сплитератор на две части
Spliterator<T> trySplit()
```

### Пример использования tryAdvance() и потока
```java
ArrayList<String> myList = new ArrayList<>();
myList.add("Alpha"); myList.add("Beta"); myList.add("Gamma");


Stream<String> myStream = myList.stream();
Spliterator<String> splitItr = myStream.spliterator();

// Обход элементов
while(splitItr.tryAdvance((n) -> System.out.println(n)));
```

### Пример использования forEachRemaining()
```java
// Обработка всех оставшихся элементов без цикла
splitItr.forEachRemaining((n) -> System.out.println(n));
```

### Пример разделения сплитератора (trySplit)
```java
Spliterator<String> splitItr = myStream.spliterator();
Spliterator<String> splitItr2 = splitItr.trySplit();

if(splitItr2 != null) {
    System.out.println("Вывод из splitItr2:");
    splitItr2.forEachRemaining((n) -> System.out.println(n));
    
    System.out.println("Вывод из splitItr:");
    splitItr.forEachRemaining((n) -> System.out.println(n));
}

Вывод из splitItr2:
Alpha
Beta
Gamma

Вывод из splitItr:
Delta
Phi
Omega
```

### Преимущества Spliterator перед Iterator
1. 💡**Эффективность** - `tryAdvance()` объединяет функциональность `hasNext()` и `next()`
    
2. **Параллельная обработка** - `trySplit()` позволяет разделять данные для параллельного выполнения
    
3. **Пакетная обработка** - `forEachRemaining()` обрабатывает все элементы без цикла
    

### Особенности использования
- **Для простых случаев** - предпочтительнее использовать стандартные методы потоков
    
- **Для сложных сценариев** - сплитераторы предоставляют низкоуровневый контроль
    
- **Параллелизм** - особенно полезны при работе с большими наборами данных в параллельных потоках
