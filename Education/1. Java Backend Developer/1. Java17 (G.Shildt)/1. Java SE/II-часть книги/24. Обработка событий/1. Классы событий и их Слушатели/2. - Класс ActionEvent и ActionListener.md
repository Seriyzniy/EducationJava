
> [!NOTE] `ActionEvent` — это класс событий высокого уровня, используемый `для обозначения` `действий пользователя`
> Таких как:
> - щелчок по кнопке (`Button`);
> - выбор пункта меню (`MenuItem`);
> - нажатие клавиши Enter в `TextField`.

Он является частью **модели делегирования событий** в AWT и ==применяется там, где нужно зафиксировать факт «действия»== (action), а не конкретное физическое событие (нажатие клавиши или движение мыши).

## Иерархия
```java
java.lang.Object
   └── java.util.EventObject
        └── java.awt.AWTEvent
             └── java.awt.event.ActionEvent
```

### 🧭Назначение
`ActionEvent` используется в связке с ==интерфейсом== **`ActionListener`**, который имеет единственный метод:
```java
void actionPerformed(ActionEvent e)
```

Когда пользователь ==совершает действие== (например, кликает кнопку), компонент ==создаёт объект== `ActionEvent` и ==передаёт его слушателю==, вызвав `actionPerformed()`.

---
## Конструкторы
```java
// Создаёт новое событие действия (например, при клике кнопки)
// src — источник события (обычно компонент, например Button)
// id — 💡идентификатор события (обычно ActionEvent.ACTION_PERFORMED)
// cmd — строка команды, описывающая действие (например, текст на кнопке)
public ActionEvent(Object src, int id, String cmd)

// Расширенный конструктор с указанием клавиш-модификаторов и времени события
// src — источник
// id — 💡идентификатор (обычно ACTION_PERFORMED)
// cmd — строка команды
// when — отметка времени в миллисекундах
// modifiers — набор флагов (Ctrl, Alt, Shift и т.д.)
public ActionEvent(
	Object src, int id, 
	String cmd, long when, int modifiers)

// Конструктор с возможностью указания, является ли событие системным
// (используется редко, в системных AWT-операциях)
public ActionEvent(
	Object src, int id, 
	String cmd, int modifiers)
```

🧠 **Типичный идентификатор события:**
```java
public static final int ACTION_PERFORMED = 1001;
```

В классе ActionEvent определены четыре целочисленные константы, которые можно применять для идентификации любых модификаторов, связанных с событием действия: 
`ALT _ МАSК`, 
`CTRL _ МАSК`, 
`МЕТА_ МАSК`
`SНIFT _ МАSК.`

## Методы
Класс `ActionEvent` наследует методы от:
- `AWTEvent` (например, `getID()`, `getSource()`, `toString()`);
- `EventObject` (например, `getSource()`).
    
Здесь описаны **только его собственные публичные методы.**

### Доступ к параметрам события
```java
// 💡Возвращает строку команды, ассоциированную с действием
// Обычно совпадает с текстом кнопки или названием команды
public String getActionCommand()

// 💡Возвращает отметку времени (в миллисекундах) момента генерации события
public long getWhen()

// ⭐Возвращает флаги модификаторов клавиатуры (например, Ctrl, Alt, Shift)
// Используется для анализа сочетаний клавиш при действии
public int getModifiers()
```

### Проверка состояния клавиш
```java
// Проверяет, была ли нажата клавиша Alt при событии
public boolean isAltDown()

// Проверяет, была ли нажата клавиша Control при событии
public boolean isControlDown()

// Проверяет, была ли нажата клавиша Meta (например, Command на macOS)
public boolean isMetaDown()

// Проверяет, была ли нажата клавиша Shift
public boolean isShiftDown()
```

### Информация о событии
```java
// Возвращает строковое описание события (для отладки)
public String paramString()
```

## 💡 Особенности и подводные камни 💡
1️⃣ 💡Событие ActionEvent не несёт координат или деталей ввода.
Это событие высокого уровня, и оно описывает именно “факт действия”, а не “где и как оно произошло”.

2️⃣ getActionCommand() может возвращать null.
Например, если событие было сгенерировано не кнопкой, а программно.

3️⃣ modifiers устарел частично после Java 10,
но остаётся ради совместимости. Лучше использовать InputEvent для проверки клавиш.

4️⃣ Время (when) может быть полезно для фильтрации или дебаунса событий (например, чтобы не обрабатывать дубликаты слишком часто).

5️⃣ 💡==Один источник== может генерировать ==несколько ActionEvent== с разными командами, если это предусмотрено логикой интерфейса (например, одно меню вызывает разные действия).

---
## 💎 Best Practices 💎
✅ Используй getActionCommand() для определения, какая кнопка вызвала событие, если несколько кнопок зарегистрированы на одного слушателя.
Пример:

```java
if (e.getActionCommand().equals("Exit")) System.exit(0);
```

✅ 💡При сложных интерфейсах лучше назначать отдельный слушатель каждому компоненту, а не обрабатывать всё в одном методе.

✅ 💡Не смешивай ActionEvent с событиями ввода (KeyEvent, MouseEvent) — это разные уровни абстракции.

✅ 💡Для современных приложений (Swing, JavaFX) логика ActionEvent остаётся актуальной, т.к. используется в javax.swing.Action.

✅ При генерации события вручную (например, в тестах) указывай корректный id = ACTION_PERFORMED.

## Пример
```java
import java.awt.*;
import java.awt.event.*;

public class ActionEventDemo extends Frame implements ActionListener {
    private TextField tf;

    public ActionEventDemo() {
        setLayout(new FlowLayout());
        tf = new TextField(20);
        Button b1 = new Button("OK");
        Button b2 = new Button("Clear");

        b1.addActionListener(this);
        b2.addActionListener(this);

        add(tf);
        add(b1);
        add(b2);

        setSize(250, 100);
        setVisible(true);

        addWindowListener(new WindowAdapter() {
            public void windowClosing(WindowEvent e) {
                System.exit(0);
            }
        });
    }

	// ⭐ПРИ ВОЗНИКНОВЕНИИ СОБЫТИЯ, выолняется данный метод⭐
    @Override
    public void actionPerformed(ActionEvent e) {
        String cmd = e.getActionCommand();
        if ("OK".equals(cmd)) {
            tf.setText("Button OK pressed");
        } else if ("Clear".equals(cmd)) {
            tf.setText("");
        }
    }

    public static void main(String[] args) {
        new ActionEventDemo();
    }
}
```
🧠 При нажатии кнопок создаются объекты `ActionEvent`, которые передаются методу `actionPerformed()`.  

Метод определяет команду действия через `getActionCommand()` и выполняет соответствующую реакцию.
