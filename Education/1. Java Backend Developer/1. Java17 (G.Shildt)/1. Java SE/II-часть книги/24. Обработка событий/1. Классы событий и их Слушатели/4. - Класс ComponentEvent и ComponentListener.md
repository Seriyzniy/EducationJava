
> [!NOTE] `ComponentEvent` — это **событие низкого уровня**, 
> `Генерируемое` каждым `визуальным компонентом` AWT `при изменении его геометрии` или **состояния отображения**.
> 
> Проще говоря: `когда` окно, `панель` или `кнопка` `перемещается`, `изменяет размер`, становится видимой или скрывается — `создаётся объект` `ComponentEvent`.

## Иерархия
```java
java.lang.Object
   └── java.util.EventObject
        └── java.awt.AWTEvent
             └── java.awt.event.ComponentEvent
```

### 🧭 Назначение
`ComponentEvent` сообщает, что **геометрические параметры** или **состояние компонента** изменились.  
Это событие получают слушатели, реализующие интерфейс:
```java
public interface ComponentListener {
    void componentResized(ComponentEvent e);
    void componentMoved(ComponentEvent e);
    void componentShown(ComponentEvent e);
    void componentHidden(ComponentEvent e);
}
```
> Оно используется не для пользовательских действий, а для **внутренней реакции интерфейса** на изменение положения и размеров элементов.

---
## Конструкторы
```java
// Создаёт новое событие компонента (например, изменение размера, положения и т.д.)
// src — источник события (объект Component, например Frame, Button, Panel)
// id — идентификатор события (одна из констант: COMPONENT_MOVED, COMPONENT_RESIZED, COMPONENT_SHOWN, COMPONENT_HIDDEN)
public ComponentEvent(Component src, int id)
```
### Константы класса

|Константа|Значение|Описание|
|---|---|---|
|`COMPONENT_MOVED`|100|Компонент был перемещён|
|`COMPONENT_RESIZED`|101|Компонент изменил размер|
|`COMPONENT_SHOWN`|102|Компонент стал видимым|
|`COMPONENT_HIDDEN`|103|Компонент стал невидимым|
> Эти идентификаторы передаются в конструктор как параметр `id`.

---
## Методы
Класс `ComponentEvent` наследует методы от:
- `AWTEvent` — (`getID()`, `getSource()`, `toString()`),
- `EventObject` — (`getSource()`).
    

Ниже описаны **его собственные методы**, специфичные для компонента.

### Доступ к источнику события
```java
// Возвращает компонент, с которым связано событие
// Это источник (например, Button, Frame, TextField и т.д.)
public Component getComponent()
```

> Этот метод безопасно приводит источник события (`Object getSource()`) к типу `Component`.

### Отладочная информация
```java
// Возвращает строковое представление события
// Содержит тип события (moved, resized, shown, hidden)
public String paramString()
```

## 💡 Особенности и подводные камни 💡

1️⃣ **`ComponentEvent` — базовый класс** для ряда других событий:
- `ContainerEvent`
- `FocusEvent`
- `KeyEvent`
- `MouseEvent`
- `WindowEvent`  
    Это делает его основой всей архитектуры событий интерфейса.
    

2️⃣ 💡**Не все изменения компонента вызывают событие.**  
Например, изменение цвета (`setBackground`) не генерирует `ComponentEvent`.  
События создаются **только при изменении геометрии и видимости**.

3️⃣ 💡**Некоторые события могут происходить очень часто.**  
Например, при изменении размера окна `componentResized()` может вызываться при каждом пикселе движения.  
Поэтому такие события лучше обрабатывать эффективно.

4️⃣ 💡**Источник всегда наследник `Component`.**  
Проверять это не нужно — метод `getComponent()` гарантирует корректное приведение типа.

5️⃣ **Используется редко напрямую.**  
Чаще всего — как базовый класс для других событий или при обработке layout-менеджеров.

---

## 💎 Best Practices 💎

✅ 💡Используй `ComponentListener`, если нужно:
- адаптировать интерфейс при изменении размера окна;
- сохранять положение элементов при перемещении окна;
- отслеживать появление/скрытие компонентов.
    

✅ 💡Избегай тяжёлой отрисовки в `componentResized()` — можно вызывать `repaint()` с таймером для оптимизации.

✅ Если нужно отслеживать изменение размеров родительского окна — вешай слушатель не на `Frame`, а на внутренний `Panel`, если отрисовка там.

✅ Не забывай снимать слушателей (`removeComponentListener()`), если компонент больше не используется — иначе возможны утечки памяти.

✅ При создании собственных компонентов можно **порожать свои события** на основе `ComponentEvent`, если требуется уведомление о геометрических изменениях.

---
## Пример
```java
import java.awt.*;
import java.awt.event.*;

public class ComponentEventDemo extends Frame implements ComponentListener {
    private TextField info;

    public ComponentEventDemo() {
        setLayout(new FlowLayout());

        info = new TextField(25);
        info.setEditable(false);

        add(new Label("Измени размер или передвинь окно"));
        add(info);

        addComponentListener(this); // регистрируем слушателя

        setSize(300, 120);
        setVisible(true);

        addWindowListener(new WindowAdapter() {
            public void windowClosing(WindowEvent e) {
                System.exit(0);
            }
        });
    }

    @Override
    public void componentResized(ComponentEvent e) {
        Dimension d = getSize();
        info.setText("Размер: " + d.width + "x" + d.height);
    }

    @Override
    public void componentMoved(ComponentEvent e) {
        info.setText("Окно перемещено");
    }

    @Override
    public void componentShown(ComponentEvent e) {
        info.setText("Компонент отображён");
    }

    @Override
    public void componentHidden(ComponentEvent e) {
        info.setText("Компонент скрыт");
    }

    public static void main(String[] args) {
        new ComponentEventDemo();
    }
}
```
