
> [!NOTE] `AdjustmentEvent` — это `событие регулировки` (adjustment event)
> Возникает, когда пользователь изменяет положение **регулируемого элемента интерфейса**, такого как:
> - полоса прокрутки (`Scrollbar`),
> - или любой компонент, реализующий интерфейс `Adjustable`.

## Иерархия
```java
java.lang.Object
   └── java.util.EventObject
        └── java.awt.AWTEvent
             └── java.awt.event.AdjustmentEvent
```

### 🧭 Назначение
`AdjustmentEvent` описывает факт **изменения значения регулируемого элемента** — например, когда пользователь переместил ползунок на полосе прокрутки.

Обычно такие события обрабатываются через интерфейс:
```java
public interface AdjustmentListener {
    void adjustmentValueChanged(AdjustmentEvent e);
}
```

---
## Конструкторы
```java
// Создаёт событие регулировки (например, при перемещении Scrollbar).
// src — источник события (объект типа Adjustable, например Scrollbar)
// id — идентификатор события (обычно AdjustmentEvent.ADJUSTMENT_VALUE_CHANGED)
// type — тип регулировки (одна из констант UNIT_INCREMENT, BLOCK_DECREMENT и т.д.)
// value — новое значение регулировки (например, новая позиция полосы прокрутки)
// isAdjusting — true, если значение всё ещё изменяется (например, во время перетаскивания)
public AdjustmentEvent(
	Adjustable src, 
	int id, 
	int type, 
	int value, 
	boolean isAdjusting)


// Упрощённый конструктор без флага isAdjusting (по умолчанию false)
public AdjustmentEvent(Adjustable src, int id, int type, int value)
```

### Константы класса

Класс `AdjustmentEvent` определяет типы изменений через набор констант:

|Константа|Значение|Описание|
|---|---|---|
|`UNIT_INCREMENT`|1|перемещение на один “шаг” вперёд|
|`UNIT_DECREMENT`|2|перемещение на один “шаг” назад|
|`BLOCK_INCREMENT`|3|перемещение на “блок” вперёд (например, по странице)|
|`BLOCK_DECREMENT`|4|перемещение на “блок” назад|
|`TRACK`|5|произвольное перемещение (например, перетаскивание ползунка)|

## Методы
Класс наследует методы от `AWTEvent` (`getID()`, `getSource()`, `toString()`)  
и добавляет **свои специфические методы** для доступа к информации о регулировке.

### Доступ к свойствам события
```java
// 💡Возвращает ТИП РЕГУЛИРОВКИ (одна из констант UNIT_INCREMENT, BLOCK_DECREMENT и т.д.)
// Используется, чтобы определить, как именно пользователь изменил значение
public int getAdjustmentType()

// Возвращает новое значение регулировки
// Например, позицию полосы прокрутки после изменения
public int getValue()
```

### Работа с источником события
```java
// Возвращает источник события, приведённый к типу Adjustable
// Это, как правило, Scrollbar или другой элемент, реализующий Adjustable
public Adjustable getAdjustable()
```

### Проверка состояния регулировки
```java
// Возвращает true, если значение всё ещё изменяется (например, ползунок удерживается мышью)
// Полезно, чтобы не перерисовывать интерфейс при каждом пикселе движения
public boolean getValueIsAdjusting()
```

### Отладочная информация
```java
// Возвращает строковое представление события с указанием его типа и значения
public String paramString()
```

## 💡 Особенности и подводные камни 💡

1️⃣ **События генерируются очень часто**  
При перетаскивании ползунка мышью `AdjustmentEvent` может срабатывать десятки раз в секунду.  
Поэтому важно проверять `getValueIsAdjusting()` и реагировать **только при завершении** регулировки.

2️⃣ 💡**Тип регулировки (`getAdjustmentType()`) позволяет оптимизировать реакцию.**
Например, для `UNIT_INCREMENT` можно сдвинуть вид на одну строку, а для `BLOCK_INCREMENT` — на страницу.

3️⃣ 💡**Источник события всегда реализует `Adjustable`.**  
Обычно это `Scrollbar`, но может быть и пользовательский компонент.

4️⃣ 💡**Проверяй корректность диапазона значений (`getValue()`).**  
Иногда ползунок может вернуть значения за пределами ожидаемого диапазона, если код вручную изменяет его.

5️⃣ **В Swing `JScrollBar` и `JSlider` используют аналогичный принцип**, но событие — `ChangeEvent` (из `javax.swing.event`).

---

## 💎 Best Practices 💎

✅ Всегда проверяй:
```java
if (!e.getValueIsAdjusting()) {
    // обработка только после завершения регулировки
}
```

✅ Используй `getAdjustmentType()` для понимания, какой именно тип движения сделал пользователь — это поможет оптимизировать перерисовку интерфейса.

✅ 💡Не выполняй тяжёлую обработку (например, сложные вычисления или отрисовку) при каждом `AdjustmentEvent` — особенно во время перетаскивания.

✅ Для пользовательских компонентов можно **имитировать события регулировки** — создав `AdjustmentEvent` вручную и вызвав `adjustmentValueChanged()` у слушателя.

✅ 💡Если нужно отслеживать плавное изменение (реактивные интерфейсы), наоборот, **обрабатывай события с `isAdjusting == true`** для непрерывного обновления.

## Пример
```java
import java.awt.*;
import java.awt.event.*;

public class AdjustmentEventDemo extends Frame implements AdjustmentListener {
    private Label status;
    private Scrollbar scroll;

    public AdjustmentEventDemo() {
        setLayout(new FlowLayout());

        // Создаём полосу прокрутки
        scroll = new Scrollbar(Scrollbar.HORIZONTAL, 0, 10, 0, 100);
        scroll.addAdjustmentListener(this);

        status = new Label("Позиция: 0");
        add(scroll);
        add(status);

        setSize(300, 100);
        setVisible(true);

        addWindowListener(new WindowAdapter() {
            public void windowClosing(WindowEvent e) {
                System.exit(0);
            }
        });
    }

    // Реакция на изменение полосы прокрутки
    @Override
    public void adjustmentValueChanged(AdjustmentEvent e) {
        int value = e.getValue();
        if (!e.getValueIsAdjusting()) { // обработка после отпускания
            status.setText("Позиция: " + value);
        }
    }

    public static void main(String[] args) {
        new AdjustmentEventDemo();
    }
}

```
🧠 При перемещении ползунка создаются объекты `AdjustmentEvent`,  
которые передаются методу `adjustmentValueChanged()`.  
Метод считывает новое значение через `getValue()` и обновляет метку.
