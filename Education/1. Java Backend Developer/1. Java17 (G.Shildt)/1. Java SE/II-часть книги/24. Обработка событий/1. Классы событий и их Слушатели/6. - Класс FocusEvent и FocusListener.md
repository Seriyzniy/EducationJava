
> [!NOTE] `FocusEvent` — это `событие`, связанное `с фокусом ввода` (keyboard focus).  
> Оно генерируется, когда компонент `получает или теряет фокус` — то есть становится активным для ввода данных с клавиатуры.
> 

## Иерархия
```java
java.lang.Object
   └── java.util.EventObject
        └── java.awt.AWTEvent
             └── java.awt.event.ComponentEvent
                  └── java.awt.event.FocusEvent
```
`FocusEvent` — **наследник `ComponentEvent`**,  
то есть он связан с конкретным визуальным компонентом,  
но добавляет информацию о фокусе ввода и состоянии фокусировки.

### 🧭 Назначение
`FocusEvent` сообщает, что **компонент получил или потерял фокус**.  
Такое событие обрабатывается объектами, реализующими интерфейс:
```java
public interface FocusListener {
    void focusGained(FocusEvent e);
    void focusLost(FocusEvent e);
}
```

🧠 **Фокус** — это состояние, когда компонент (например, текстовое поле)  
получает приоритет для приёма событий клавиатуры.

## Конструкторы
```java
// Создаёт событие фокуса (например, получение или потеря фокуса).
// src — компонент, с которым связано событие (например, TextField)
// id — идентификатор события (FOCUS_GAINED или FOCUS_LOST)
// temporary — true, если событие временное (например, фокус потерян на короткое время)
public FocusEvent(Component src, int id, boolean temporary)

// Расширенный конструктор: позволяет указать, какой компонент получил фокус после потери.
// src — компонент, потерявший фокус
// id — идентификатор (FOCUS_GAINED или FOCUS_LOST)
// temporary — временное ли событие
// opposite — другой компонент, связанный с передачей фокуса
public FocusEvent(
	Component src, 
	int id, 
	boolean temporary, 
	Component opposite)
```

### Константы класса
|Константа|Значение|Описание|
|---|---|---|
|`FOCUS_GAINED`|1004|Компонент получил фокус|
|`FOCUS_LOST`|1005|Компонент потерял фокус|

## Методы
Класс наследует методы `ComponentEvent` (`getComponent()`, `paramString()`)  
и добавляет **свои собственные**, специфичные для фокуса

### Проверка состояния фокуса
```java
// Проверяет, является ли событие временным (temporary)
// true — если фокус потерян/получен временно (например, при открытии всплывающего меню)
public boolean isTemporary()
```

### Связанный компонент
```java
// Возвращает компонент, с которым связан фокус.
// Например, компонент, получивший фокус, если текущий его потерял.
public Component getOppositeComponent()
```
`getOppositeComponent()` полезен для понимания, между какими компонентами произошёл переход фокуса.

### Отладочная информация
```java
// Возвращает строковое описание события (например, focus gained/lost)
public String paramString()
```

## 💡 Особенности и подводные камни 💡

1️⃣ 💡**Фокус — эксклюзивный.**  
⭐⭐Только один компонент в окне может иметь фокус ввода в любой момент времени.  
Когда один компонент получает фокус, другой обязательно его теряет.

2️⃣ **Временные события (`isTemporary()`).**  
Иногда фокус теряется не окончательно — например, при открытии всплывающего диалога.  
Такие случаи помечаются флагом `temporary = true`, и на них обычно **не реагируют**.

3️⃣ 💡**Не все компоненты могут получать фокус.**  
Компонент должен быть "фокусируемым":
- вызывать `setFocusable(true)` (в Swing),
- или иметь флаг `enabled` и быть видимым.
    

4️⃣ ⭐**Передача фокуса может быть вызвана программно.**  
Например, `textField.requestFocus()` создаёт событие `FOCUS_GAINED`.

5️⃣ **Вызовы repaint() в обработчиках фокуса нужно использовать аккуратно**,  
чтобы избежать лишней перерисовки при частых переключениях компонентов.

## 💎 Best Practices 💎

✅ 💡При работе с несколькими текстовыми полями (например, формами)  
используй `FocusListener` для валидации данных при потере фокуса (`focusLost()`).

✅ 💡Проверяй `isTemporary()`, чтобы не реагировать на временные потери фокуса (например, при всплывающих окнах).

✅ Используй `getOppositeComponent()` для понимания, куда перешёл фокус.

✅ Если нужно, чтобы компонент **не получал фокус**, установи `setFocusable(false)` —  
это предотвратит генерацию `FocusEvent`.

✅ Для Swing-компонентов часто предпочтительнее использовать `FocusAdapter` (адаптер-класс)  
и переопределять только нужные методы.

## Пример
```java
import java.awt.*;
import java.awt.event.*;

public class FocusEventDemo extends Frame implements FocusListener {
    private TextField tf1, tf2;
    private Label info;

    public FocusEventDemo() {
        setLayout(new FlowLayout());

        tf1 = new TextField("Поле 1", 10);
        tf2 = new TextField("Поле 2", 10);
        info = new Label("Фокус пока не установлен");

        tf1.addFocusListener(this);
        tf2.addFocusListener(this);

        add(tf1);
        add(tf2);
        add(info);

        setSize(300, 120);
        setVisible(true);

        addWindowListener(new WindowAdapter() {
            public void windowClosing(WindowEvent e) {
                System.exit(0);
            }
        });
    }

    @Override
    public void focusGained(FocusEvent e) {
        info.setText("Фокус получен: " + ((TextField) e.getComponent()).getText());
    }

    @Override
    public void focusLost(FocusEvent e) {
        info.setText("Фокус потерян: " + ((TextField) e.getComponent()).getText());
    }

    public static void main(String[] args) {
        new FocusEventDemo();
    }
}
```

🧠 **Как работает:**
- Когда пользователь кликает по одному из текстовых полей — вызывается `focusGained()`.
    
- При переходе к другому — `focusLost()`.
    
- В строке состояния отображается текущее состояние фокуса.
