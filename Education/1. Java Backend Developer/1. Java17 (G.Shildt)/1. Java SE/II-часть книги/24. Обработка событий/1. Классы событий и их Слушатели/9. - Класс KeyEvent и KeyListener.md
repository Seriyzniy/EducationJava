
> [!NOTE] `KeyEvent` — это событие, генерируемое `при нажатии`, `удержании` и `отпускании` `клавиш КЛАВИАТУРЫ`.
> Оно сообщает, **какая клавиша была нажата**, **какой символ введён**, и какие **модификаторы (Shift, Ctrl, Alt)** были активны в этот момент.

## Иерархия
```JAVA
java.lang.Object
   └── java.util.EventObject
        └── java.awt.AWTEvent
             └── java.awt.event.ComponentEvent
                  └── java.awt.event.InputEvent
                       └── java.awt.event.KeyEvent
```

🧠 `KeyEvent` — это **наследник `InputEvent`**,  
поэтому он наследует всё, что касается модификаторов (`Shift`, `Ctrl`, `Alt`, `Meta`), времени (`getWhen()`) и состояния (`consume()`).

### 🧭 Назначение
`KeyEvent` сообщает о действиях клавиатуры.  
Он используется для обработки событий, поступающих от компонента, который имеет **фокус ввода**.

Для обработки клавиатурных событий служит интерфейс:
```java
public interface KeyListener {
    void keyPressed(KeyEvent e);   // клавиша нажата
    void keyReleased(KeyEvent e);  // клавиша отпущена
    void keyTyped(KeyEvent e);     // введён символ
}
```

> События `keyPressed` и `keyReleased` относятся к физическим клавишам,  
а `keyTyped` — к введённым символам (с учётом модификаторов, раскладки и т.д.).

## Конструкторы
```java
// Создаёт событие клавиатуры
// src — компонент, который имеет фокус и получил событие
// id — тип события: KEY_PRESSED, KEY_RELEASED, KEY_TYPED
// when — отметка времени (в мс)
// modifiers — битовая маска нажатых клавиш-модификаторов
// 💡keyCode — код клавиши (например, VK_ENTER, VK_A и т.д.)
// 💡keyChar — введённый символ (учитывает раскладку)
// 💡keyLocation — положение клавиши (левая, правая, цифровая и т.д.)
public KeyEvent(
	Component src, 
	int id, 
	long when, 
	int modifiers,
	int keyCode, 
	char keyChar, 
	int keyLocation)
```

### Константы класса
|Константа|Описание|
|---|---|
|`KEY_FIRST`|400|
|`KEY_LAST`|402|
|`KEY_TYPED`|400|
|`KEY_PRESSED`|401|
|`KEY_RELEASED`|402|
### Основные поля
|Поле|Тип|Описание|
|---|---|---|
|`keyCode`|`int`|Код клавиши (одна из `VK_` констант)|
|`keyChar`|`char`|Символ, соответствующий клавише|
|`keyLocation`|`int`|Расположение клавиши (левая, правая, цифровая)|

### Символьные константы клавиш
Все клавиши определены как `VK_*` константы, например:
```java
KeyEvent.VK_ENTER
KeyEvent.VK_ESCAPE
KeyEvent.VK_A
KeyEvent.VK_SPACE
KeyEvent.VK_LEFT
KeyEvent.VK_CONTROL
KeyEvent.VK_F1
```

## Методы

Класс наследует методы от `InputEvent` (`getWhen()`, `getModifiersEx()`, `isAltDown()` и т.д.),  
а также определяет **свои уникальные методы для клавиатуры**.

### Коды клавиш
```java
// Возвращает код клавиши (например, KeyEvent.VK_A, KeyEvent.VK_ENTER)
public int getKeyCode()

// Возвращает символ, введённый пользователем (учитывает раскладку и модификаторы)
public char getKeyChar()

// Возвращает положение клавиши на клавиатуре
// Например: KEY_LOCATION_LEFT, KEY_LOCATION_RIGHT, KEY_LOCATION_NUMPAD
public int getKeyLocation()
```

### Проверка символа
```java
// Проверяет, является ли клавиша "действительной" (имеет символьное значение)
public boolean isActionKey()
```
> Action-клавиши — это несимвольные клавиши: стрелки, F1-F12, PageUp, Home, End и т. д.

### Управление событием
```java
// "Поглощает" событие, предотвращая его дальнейшую обработку другими слушателями
@Override
public void consume()
```

### Статические методы утилит
```java
// Возвращает текстовое имя клавиши (например, "A", "Enter", "F5")
public static String getKeyText(int keyCode)

// Возвращает текстовое имя модификатора (например, "Ctrl", "Shift")
public static String getKeyModifiersText(int modifiers)
```

## 💡 Особенности и подводные камни 💡

1️⃣ **`KEY_TYPED` ≠ `KEY_PRESSED`.**
- `KEY_PRESSED` — событие аппаратного уровня (реакция на физическое нажатие клавиши).
    
- `KEY_TYPED` — событие символьного ввода (например, комбинация Shift+A → ‘A’).  
    Обычно `KEY_TYPED` не имеет `keyCode`, а только `keyChar`.
    

2️⃣ 💡**`keyChar` может быть `CHAR_UNDEFINED`.**  
Это случается для клавиш без символа (например, F1, Esc, стрелки).

3️⃣ **События зависят от раскладки клавиатуры.**  
Для одной и той же клавиши `keyChar` может отличаться (например, `a` ↔ `ф`).

4️⃣ **Модификаторы нужно получать через `getModifiersEx()`.**  
Старый `getModifiers()` устарел и не учитывает дополнительные клавиши.

5️⃣ **Некоторые клавиши дублируются:**  
Например, левая и правая клавиши Shift или Ctrl различаются по `getKeyLocation()`.

## 💎 Best Practices 💎

✅ Используй `keyPressed` для **управления**,  
а `keyTyped` — для **ввода текста**.

✅ Не смешивай `keyChar` и `keyCode`:
- `keyChar` — это ==символ== (например, `'A'`),
- `keyCode` — это ==физическая== клавиша (`VK_A`).
    

✅ Для горячих клавиш проверяй `getModifiersEx()` вместе с `getKeyCode()`:
```java
if (e.getKeyCode() == KeyEvent.VK_S &&
    (e.getModifiersEx() & InputEvent.CTRL_DOWN_MASK) != 0) {
    // Ctrl+S
}
```

✅ Не обрабатывай клавиатурные события в компонентах без фокуса —  
они их просто не получат.

✅ Если нужно перенаправлять фокус между элементами —  
используй `KeyEvent.consume()` после обработки, чтобы событие не ушло дальше.

## Пример
```java
import java.awt.*;
import java.awt.event.*;

public class KeyEventDemo extends Frame implements KeyListener {
    private TextField tf;
    private TextArea ta;

    public KeyEventDemo() {
        setLayout(new FlowLayout());

        tf = new TextField(20);
        ta = new TextArea(10, 30);

        tf.addKeyListener(this);

        add(new Label("Введите текст:"));
        add(tf);
        add(ta);

        setSize(400, 250);
        setVisible(true);

        addWindowListener(new WindowAdapter() {
            public void windowClosing(WindowEvent e) {
                System.exit(0);
            }
        });
    }

    @Override
    public void keyTyped(KeyEvent e) {
        ta.append("Введён символ: " + e.getKeyChar() + "\n");
    }

    @Override
    public void keyPressed(KeyEvent e) {
        ta.append("Нажата клавиша: " + KeyEvent.getKeyText(e.getKeyCode()) + "\n");
        if ((e.getModifiersEx() & InputEvent.CTRL_DOWN_MASK) != 0)
            ta.append("→ Нажат Ctrl\n");
    }

    @Override
    public void keyReleased(KeyEvent e) {
        ta.append("Отпущена клавиша: " + KeyEvent.getKeyText(e.getKeyCode()) + "\n");
    }

    public static void main(String[] args) {
        new KeyEventDemo();
    }
}

```

🧠 **Как работает:**
- Каждый ввод символа создаёт `keyTyped()`.
    
- Каждое физическое нажатие клавиши — `keyPressed()`,  
    а отпускание — `keyReleased()`.
    
- При зажатом `Ctrl` выводится дополнительная информация.
