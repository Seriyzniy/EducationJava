# 📕 1. WindowAdapter
> [!NOTE] `WindowAdapter` — это абстрактный класс, который `реализует интерфейсы`, 
> Связанные с событиями окна: `WindowListener`, `WindowFocusListener` и `WindowStateListener`.
> 

## Иерархия
```JAVA
java.lang.Object
   └── java.awt.event.WindowAdapter
```

🧠 `WindowAdapter` — это **адаптер-класс**,  
предназначенный для того, чтобы **не реализовывать все методы интерфейсов вручную**,  
а только те, которые реально нужны.

### 🧭 Назначение

В Java интерфейс `WindowListener` содержит **7 методов**.  
Если бы ты захотел обрабатывать только одно событие (например, `windowClosing()`),  
тебе пришлось бы переопределить все 7 — даже пустыми. Это неудобно.

Адаптеры решают эту проблему:  
они предоставляют ==пустую реализацию всех методов интерфейса==,  
	
а ты просто ==наследуешься== и ==переопределяешь только нужные==.

## Реализуемые интерфейсы
`WindowAdapter` охватывает все три типа событий, связанных с окнами:

|Интерфейс|Назначение|
|---|---|
|`WindowListener`|Открытие, закрытие, активация, деактивация, сворачивание, восстановление|
|`WindowFocusListener`|Получение и потеря фокуса|
|`WindowStateListener`|Изменение состояния окна (свернуто, развернуто и т. д.)|

---
## Конструктор
```java
// Конструктор без параметров (по умолчанию)
public WindowAdapter()
```
> Обычно ==не вызывается напрямую== — просто создаётся анонимный подкласс `WindowAdapter` и ==переопределяется== нужный метод.

## 🧠 Методы
Класс `WindowAdapter` реализует **все методы** трёх интерфейсов, 

но ==делает это пустыми телами== (чтобы их можно было переопределить по необходимости).

### Методы интерфейса `WindowListener`
```java
// Вызывается при открытии окна
public void windowOpened(WindowEvent e) {}

// Вызывается, когда пользователь инициирует закрытие окна
public void windowClosing(WindowEvent e) {}

// Вызывается после закрытия окна
public void windowClosed(WindowEvent e) {}

// Вызывается при сворачивании (иконфикация)
public void windowIconified(WindowEvent e) {}

// Вызывается при восстановлении из свернутого состояния
public void windowDeiconified(WindowEvent e) {}

// Вызывается, когда окно становится активным (получает фокус)
public void windowActivated(WindowEvent e) {}

// Вызывается, когда окно теряет активность (другое окно получает фокус)
public void windowDeactivated(WindowEvent e) {}
```

### Методы интерфейса `WindowFocusListener`
```java
// Вызывается, когда окно получает фокус ввода
public void windowGainedFocus(WindowEvent e) {}

// Вызывается, когда окно теряет фокус ввода
public void windowLostFocus(WindowEvent e) {}
```

### Методы интерфейса `WindowStateListener`
```java
// Вызывается при изменении состояния окна (например, свернули, развернули)
public void windowStateChanged(WindowEvent e) {}
```

## 💡 Особенности и подводные камни 💡

1️⃣ **`WindowAdapter` — это абстрактный класс**,  
но сам по себе не содержит абстрактных методов.  
Его наследуют, чтобы переопределить только нужные.

2️⃣ **Можно использовать как анонимный класс**,  
прямо при добавлении слушателя — ==самый частый вариант==:
```java
frame.addWindowListener(new WindowAdapter() {
    public void windowClosing(WindowEvent e) {
        System.exit(0);
    }
});
```
