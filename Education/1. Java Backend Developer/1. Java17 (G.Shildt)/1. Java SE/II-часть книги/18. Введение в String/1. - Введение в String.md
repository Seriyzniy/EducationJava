**Строка в Java** - это последовательность символов, представленная как объект класса `String`

> [!NOTE] 🧠`String` реализует интерфейс `CharSequence`🧠
> 

В отличие от некоторых языков, где строки - это массивы символов, в Java строки - это полноценные объекты

> [!NOTE] ⭐**Ключевая особенность**: объекты `String` являются `НЕ_ИЗМЕНЯЕМЫМИ` (immutable)⭐
> 
### Почему неизменяемость?
- **Эффективность**: неизменяемые строки можно реализовать более оптимально
    
- **Безопасность**: исключаются случайные изменения
    
- ⭐**Кэширование**: возможность кэширования хэш-кодов и пулинга строк

### Альтернативы для изменяемых строк:

- **StringBuilder** - для однопоточных приложений
    
- **StringBuffer** - для многопоточных приложений (синхронизирован)

***
# ⭐⭐ Ключевые особенности ⭐⭐

### 🔶 **Неизменяемость (immutability).**

- После создания строки её содержимое **невозможно изменить**.
- ==Любая операция «изменения»== (конкатенация, `replace`, `substring` и т.д.) ==возвращает **новый объект**.==

***
### 🔶⭐**String pool (пул строк).**

- Все ==строковые литералы попадают в пул строк== (String Intern Pool).
```java
String str = "Hello world"; // ✅ СТРОКОВЫЙ ЛИТЕРАЛ -> идет в ПУЛЛ

String str1 = new String("Hello world"); //❌❗ НЕ ЛИТЕРАЛ -> идет в КУЧУ
```

- Если ты создаёшь строку `"hello"`, JVM кладёт её в пул, и при повторном использовании того же литерала ссылка будет указывать на уже существующий объект.
    
- Разница:
```java
String a = "hello";       // из пула
String b = "hello";       // из пула, тот же объект
String c = new String("hello"); // новый объект вне пула

System.out.println(a == b); // true // Ссылаются на один и тот же объект в String Pool
System.out.println(a == c); // false // Одна ссылка на строку из кучи, а другая на String Pool
```

***
### 🔶 **Unicode и UTF-16.**
- `String` внутри использует UTF-16.
- ⭐`length()` возвращает количество `char`
```java
String s = "😊";  // Смайл занимает 2 char
System.out.println(s.length()); // 2 (!)
```

# 🛑Подводные камни🛑

### 🔶⭐**Конкатенация в цикле.** (и любая операция создает StringBuilder)

- Любая операция с изменение `String`, создает новую строку с помощью вызова `StringBuilder`
	
- Операция `str = str + "x";` в цикле СОЗДАЕТ НОВЫЕ СТРОКИ КАЖДЫЙ РАЗ → огромные накладные расходы.
    
- Для таких случаев используют `StringBuilder` или `StringBuffer` (потокобезопасный).
```java
StringBuilder sb = new StringBuilder();
for (int i = 0; i < 1000; i++) {
    sb.append("x");
}
```

***
### 🔶⭐Сравнение строк. (всегда через `equals`)
- `==` СРАВНИВАЕТ ССЫЛКИ, а не содержимое.
    
- ⭐Для содержимого ВСЕГДА ИСПОЛЬЗУЕМ `.equals()`.
```java
String a = "test";
String b = new String("test");

System.out.println(a == b);      // false // ссылки разные
System.out.println(a.equals(b)); // true // содержимое одинаковое

```

***
### 🔶⭐ **Кеширование хеш-кода.**

- `String.hashCode()` ==ВЫЧИСЛЯЕТСЯ ОДИН РАЗ И КЕШИРУЕТСЯ== (ускоряет работу `HashMap`/`HashSet`).
    
- Но если бы строки были изменяемыми, это привело бы к несогласованности → ещё одна причина неизменяемости.
***
### 🔶🧠**Интернирование строк вручную.** (Помещение в пулл)

- `String.intern()` помещает строку в пул.
    
- Может уменьшить расход памяти, но злоупотребление замедляет работу (синхронизация + нагрузка на PermGen/Metaspace).

***

### 🔶 **`substring()` до Java 7u6.**

- Раньше `substring()` хранил ссылку на **тот же массив `char[]`**, из-за чего можно было «случайно» держать в памяти огромные строки.
    
- Сейчас `substring()` создаёт **копию** подстроки → безопаснее, но чуть медленнее.
***
# 📕 1. Конструкторы класса String

#### 1. Пустой конструктор:
```java
String s = new String(); // Создает пустую строку
```

#### 2. Из массива символов:
```java
char[] chars = {'a', 'b', 'c'};

String s = new String(chars); // "abc"
```

#### 3. Из части массива символов:
```java
char[] chars = {'a', 'b', 'c', 'd', 'e', 'f'};

String s = new String(chars, 2, 3); // "cde"
// startIndex = 2, numChars = 3
```

#### 4. Копирование из другой строки:
```java
String s1 = "Java"; // ✅СТРОКОВЫЙ ЛИТЕРАЛ
String s2 = new String(s1); // Копия строки s1
```

#### 5. Из массива байтов:
```java
byte[] ascii = {65, 66, 67, 68, 69, 70}; // ASCII коды

String s1 = new String(ascii); // "ABCDEF"
String s2 = new String(ascii, 2, 3); // "CDE"
```

> [!NOTE] 🧠При создании из массива, в `String` передается КОПИЯ
> При каждом создании объекта `String` из массива содержимое последнего копируется. 
> 
> Если вы измените содержимое массива после создания строки, то объект `String` не изменится.

#### **6. Из StringBuilder/StringBuffer:**
```java
StringBuffer sb = new StringBuffer("Hello");

String s = new String(sb);
```

# 📕 2. Длина строки
Длина строки - это ==количество содержащихся в ней символов (char)==. Чтобы по лучить значение длины строки, вызовите метод `length()`:
```java
String s = "abc";
int len = s.length(); // 3
```
