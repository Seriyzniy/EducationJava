До появления запечатанных классов наследование работало в стиле "все или ничего': 
- класс открыт - от него можно наследоваться без ограничений
- класс `final` - наследоваться нельзя вовсе.

Запечатанные классы находятся между этими двумя крайностями

Начиная с версии JDK 17, разрешено объявлять ==класс, который может наследоваться только определенными подклассами.== 
Такой класс называется `запечатанным`.

***
# 📕 1. Запечатанные классы
Чтобы объявить запечатанный класс, поместите `sealed` перед объявлением, а после имени класса укажите конструкцию `permits` с разрешенными подклассами.

```java
public sealed class MySealedClass permits Alpha, Beta { }
```
- ⭐`sealed` — делает класс запечатанным.
    
- ⭐`permits` — указывает, какие именно классы могут его расширять.

> [!NOTE] ⭐⭐`permits` ОБЯЗЫВАЕТ указанные классы РЕАЛИЗОВАТЬ/РАСШИРИТЬ интерфейс/класс⭐⭐
> Если имя класса присутствует в конструкции `permits` для запечатанного класса, тогда этот класс обязан напрямую расширять запечатанный класс, иначе возникнет ошибка на этапе компиляции.

Попытка наследоваться другим классом → ошибка компиляции.

## 📙 1. Подклассы запечатанного класса

> [!NOTE] ⭐Подклассы должны быть `final`, `sealed` `non-sealed`⭐
> Они должны быть объявлены с модификатором:
> - `final` → наследование дальше запрещено;
> - `sealed` → можно продолжить ограниченное наследование; 
> - `non-sealed` → наследование разрешено всем.
>```java
>public final class Alpha extends MySealedClass { }
public sealed class Beta extends MySealedClass permits Gamma { }
public non-sealed class Delta extends MySealedClass { }
>```


Вот например `Alpha` и `Beta` - два подкласса `MySealedClass`:
```java
// Подклассы запечатанного класса
public final class Alpha extends MySealedClass {
    // ...
}

public final class Beta extends MySealedClass {
    // ...
}
```

💡 Например, вот версия `Alpha`, которая разрешает наследование подклассу `Gamma`:
```java
public sealed class Alpha extends MySealedClass permits Gamma {
	// ...
}
```
Разумеется, класс Garnma должен быть объявлен как sealed, final или non-sealed.

## 📙 2. Распаковка (распечатывание) `non-sealed`

> [!NOTE] ⭐Подкласс запечатанного класса можно "распечатать" при помощи `non-sealed`⭐
> Подкласс запечатанного класса можно распечатывать с помощью контекстно-чувствительного ключевого слова `non-sealed`, которое появилось в версии `JDK 17`. 
> 
> Оно разблокирует подкласс, позволяя любому другому классу наследоваться от него.

🧠`Основная причина` введения `non-sealed` связана с тем, чтобы `позволить суперклассу указывать ограниченный набор прямых подклассов`, которые `обеспечивают базовый уровень четко определенной функциональности`, но дать возможность свободно расширять эти подклассы.

> [!NOTE] Кроме того, объявлять как `non-sealed` ==класс, который не расширяет запечатанный класс, НЕЛЬЗЯ.==
>

# 📕 2. Правила модулей и пакетов

- ✅ Каждый разрешенный для наследования от него подкласс должен быть доступным.
	
- ✅Абстрактный класс может быть также и запечатанным.
	
- ✅Если класс в **именованном модуле** → все его разрешённые подклассы должны быть в том же модуле.
    
- Если в **неименованном модуле** → все классы должны быть в одном пакете.
    
- Можно хранить sealed-класс и его подклассы ==в одном файле== → тогда `permits` не нужен.
```java
public sealed class MySealedClass {
    // permits не требуется
}

final class Alpha extends MySealedClass { /* package-private */ }
final class Beta extends MySealedClass { /* package-private */ }
```

# 📕 3. Запечатанные интерфейсы
(тоже самое что и у классов)

Запечатанный интерфейс объявляется аналогично запечатанному классу с применением `sealed`. 

Запечатанный интерфейс использует свою конструкцию `permits` ==для УКАЗАНИЯ КЛАССОВ, которым ПОЗВОЛЯЕТСЯ ЕГО РЕАЛИЗОВЫВАТЬ==, и/или интерфейсов, которым разрешено его расширять.

Вот простой пример запечатанного интерфейса, который разрешает его реализацию только классам `Alpha` и `Beta`:
```java
public sealed interface MySealedIF permits Alpha, Beta, MyIF {
    void myMeth();
}
```

## Подинтерфейсы и классы-реализации
Класс, реализующий запечатанный интерфейс, должен быть определен как `final`, `sealed` или `non-sealed`. (тоже самое что и у классов)

```java
// Классы реализации
public non-sealed class Alpha implements MySealedIF { ... }
public final class Beta implements MySealedIF { ... }

// Расширение интерфейса
public non-sealed interface MyIF extends MySealedIF { }
```


> [!NOTE] ⭐ Все что указано в `permits` - ОБЯЗАНО РЕЛИЗОВАТЬ/НАСЛЕДОВАТЬСЯ ⭐
> Важный момент заключается в том, что любой класс, указанный в конструкции `permits` запечатанного интерфейса, обязан реализовывать интерфейс.

Как и следовало ожидать, класс может быть унаследован от запечатанного класса и реализовывать запечатанный интерфейс.


## Применение sealed

- Полный контроль над иерархией.
    
- Особенно полезно в **библиотеках и API**, где нужно чётко ограничить расширение.
    
- Хорошо сочетается с `switch` + `enum` → можно исчерпывающе обрабатывать все варианты.
