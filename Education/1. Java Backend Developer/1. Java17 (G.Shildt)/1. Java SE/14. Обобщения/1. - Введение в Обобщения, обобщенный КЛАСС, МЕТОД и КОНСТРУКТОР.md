# 📕 1. Что такое обобщения?

**Обобщения (Generics)** - это ==параметризованные типы,== добавленные в Java в JDK 5. Они позволяют создавать классы, интерфейсы и методы, которые работают с различными типами данных безопасным образом.

### Основные преимущества:
- **Безопасность типов** - ошибки обнаруживаются на этапе компиляции
    
- **Устранение приведений типов** (не нужно приводить к Object)- код становится чище и понятнее
    
- **Повторное использование кода** - один алгоритм для разных типов данных

## Простой пример обобщенного класса
```java
// Простой обобщенный класс
class Gen<T> {
    T ob; // объявить объект типа T
    
    // Конструктор
    Gen(T o) {
        ob = o;
    }
    
    // Метод возвращает объект типа T
    T getOb() {
        return ob;
    }
    
    // Показать тип T
    void showType() {
        System.out.println("Типом T является " + ob.getClass().getName());
    }
}

// Использование
class GenDemo {
    public static void main(String[] args) {
        // Для Integer
        Gen<Integer> iOb = new Gen<Integer>(88);
        iOb.showType();
        int v = iOb.getOb(); // автораспаковка
        System.out.println("Значение: " + v);
        
        // Для String
        Gen<String> strOb = new Gen<String>("Тест");
        strOb.showType();
        String str = strOb.getOb();
        System.out.println("Значение: " + str);
    }
}

Типом Т является 
java.lang.Integer значение: 88 

Типом Т является 
java.lang. String значение: Тест с обобщениями
```


> [!NOTE] **Важно**: компилятор ==не создает разные версии Gen== - происходит `стирание типов`.
> 

## 💡 Ключевые моменты:
- `T` - параметр типа (type parameter)
    
- ==Угловые скобки== `<T>` указывают ==на обобщенный тип==
    
- Аргумент типа (type argument) указывается при создании объекта

***
## 📙 1.2 Обобщения работают только со ссылочными типами

> [!NOTE] Примитивы в обобщениях НЕ ИСПОЛЬЗУЮТСЯ
> ==Параметр типа должен быть ССЫЛОЧНЫМ типом==. Примитивные типы не допускаются:
> - `Gen<int> intOb` - ошибка 
> - Используются оболочки типов: `Gen<Integer>`

## 📙 1.3. Каким образом обобщения улучшают безопасность в отношении типов?
Сравнение с необобщенным классом `NonGen` (использующим Object):

- В необобщенной версии требуются явные приведения
    
- Ошибки несоответствия типов обнаруживаются только во время выполнения
    
> [!NOTE] ⭐Обобщения преобразуют `ошибки времени выполнения` В `ошибки компиляции`
>

# 📕 2. ⭐⭐Ограниченные типы при помощи `T extends MySuperClass`⭐⭐

Обобщения позволяют ==ОГРАНИЧИВАТЬ ТИПЫ, передаваемые в качестве ОБОБЩЕНИЯ==, с помощью `extends`:
```java
class Stats<T extends Number> { ... }
```
> [!NOTE] ⭐`T` может быть только `Number` или его подклассом ⭐
> 

- Компилятор знает, что у `T` есть методы `Number` (например, `doubleValue()`)
    
- Предотвращает создание нечисловых объектов `Stats`

> [!NOTE] 💡 Можно `использовать интерфейсы` и `множественные ограничения через` `&`.
> ``` java
> class MyGen<T extends MyClass & Myinterface> {...} // 🛑Внимание🛑
> ```
### Правила МНОЖЕСТВЕННОГО ОГРАНЧИЕНИЯ при помощи `&`:
1. В списке ограничений ==**может быть только один класс**, и он **должен стоять первым**==.  
    Остальные — только интерфейсы.
```java
<T extends Number & Comparable<T> & Serializable>
```

2. Если указывается только интерфейсы — класс не нужен:
```java
<T extends Comparable<T> & Serializable>
```
### Итог
- **Множественное ограничение (`&`)** позволяет задать сразу несколько требований к параметру типа.
    
- Полезно, когда ==нужно работать с объектом, который одновременно является, например, **числом** и **сравнимым**,== или реализует несколько интерфейсов.

# 📕3. Общая форма обобщенного КЛАССА
### Синтаксис:
```java
class class_name<generic_arg> { ... }
```

### Создание экземпляра:
```java
class_name<generic_arg> var_name = 
    new class_name<generic_arg> (constructor_arg);
```

## 📙 3.1 Обобщенный класс с двумя параметрами типов

Класс `TwoGen<T, V>` демонстрирует ==множественные параметры типов==:

- ⭐Параметры разделяются ==запятыми==: `<T, V>`
    
- ⭐При создании указываются оба аргумента типа

```java
// Простой обобщенный класс с двумя параметрами типов: T и V.
class TwoGen<T, V> {
    T ob1;
    V ob2;
    
    // Передать конструктору ссылки на объекты типов T и V.
    TwoGen(T o1, V o2) {
        ob1 = o1;
        ob2 = o2;
    }
    
    // Вывести типы T и V.
    void showTypes() {
        System.out.println("Типом T является " + ob1.getClass().getName());
        System.out.println("Типом V является " + ob2.getClass().getName());
    }
    
    T getOb1() {
        return ob1;
    }
    
    V getOb2() {
        return ob2;
    }
}

// Демонстрация использования TwoGen.
class SimpGen {
    public static void main(String[] args) {
        TwoGen<Integer, String> tgObj = new TwoGen<Integer, String>(88, "Обобщения");
        
        // Вывести типы.
        tgObj.showTypes();
        
        // Получить и вывести значения.
        int v = tgObj.getOb1();
        System.out.println("значение: " + v);
        
        String str = tgObj.getOb2();
        System.out.println("значение: " + str);
    }
}


Типом T является java.lang.Integer
Типом V является java.lang.String
значение: 88
значение: Обобщения
```

# 📕 4. Создание обобщенного МЕТОДА

Обобщённый метод — это ==метод, который **сам ВВОДИТ параметр типа**== (независимо от параметров класса).

> [!NOTE] 🔑 Важное: `Genric указывается перед возвращаемым типом`.
> При этом возвращаемый тип может быть любым

> [!NOTE] `ОБОБЩЕННЫЕ методы` могут быть `в НЕОБОБЩЕННЫХ классах`. 
> Тем не менее, важно понимать, что ==обобщенные методы могут быть== 
> - как статическими, 
> - так и не статическими. 
> 
> В этом смысле нет никаких ограничений.

Параметры типов может быть несколько:
```java
static <T extends Comparable<T>, V extends T> boolean isIn(T x, V[] y)
```
Здесь первый `generic` `T` может быть любым классом, который реализует интерфейс Comparable. 
А второй `V` может быть либо таким же `T`, либо его наследником

В результате метод может безопасно выполнять поиск элемента в массиве

## Пример Обобщённый метод для массива
```java
class GenericMethodDemo {
    // Метод возвращает первый элемент массива
    public static <T> T getFirst(T[] array) {
        return array[0];
    }

    public static void main(String[] args) {
        String[] words = {"Java", "Generics", "Cool"};
        Integer[] nums = {1, 2, 3};

        System.out.println(getFirst(words)); // Java
        System.out.println(getFirst(nums));  // 1
    }
}
```
Здесь `T` выводится компилятором автоматически

## Пример 2. Метод с ограничением типа
```java
class GenericBoundDemo {
    // Метод находит максимум из двух Comparable-объектов
    public static <T extends Comparable<T> > T max(T a, T b) {
        return a.compareTo(b) > 0 ? a : b;
    }

    public static void main(String[] args) {
        System.out.println(max(10, 20));       // 20
        System.out.println(max("apple", "pear")); // pear
    }
}
```
❗❗❗Здесь `T` обязан реализовывать `Comparable<T>`.

# 📕 5. Обобщённый КОНСТРУКТОР

Точно так же, как методы, КОНСТРУКТОРУ тоже МОГУТ ВВОДИТЬ свои параметры типов.  

Это полезно, даже если **САМ КЛАСС  НЕ является обобщённым**.

```java
class GenericConstructor {
    private double value;

    // Обобщённый конструктор
    <T extends Number> GenericConstructor(T arg) {
        value = arg.doubleValue();
    }

    void showValue() {
        System.out.println("value = " + value);
    }

    public static void main(String[] args) {
        GenericConstructor gc1 = new GenericConstructor(100);    // int
        GenericConstructor gc2 = new GenericConstructor(123.45); // double
        GenericConstructor gc3 = new GenericConstructor(5.5f);   // float

        gc1.showValue();
        gc2.showValue();
        gc3.showValue();
    }
}
```

> [!NOTE] 💡 Класс может быть не обобщенным
> Здесь класс `GenericConstructor` не параметризован, но конструктор — обобщённый. Он принимает **любой `Number`** и переводит его в `double`.
