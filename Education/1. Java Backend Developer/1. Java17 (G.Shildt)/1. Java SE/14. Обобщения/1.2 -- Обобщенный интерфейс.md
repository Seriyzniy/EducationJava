В дополнение к обобщенным классам и методам также могут существовать обобщенные интерфейсы, которые определяются аналогично обобщенным классам.

## Что такое обобщённый интерфейс?
Это ==интерфейс, у которого есть **параметры типа**.  ==

Он ==позволяет описывать ОБЩИЕ алгоритмы/контракты==, которые работают с разными типами данных.

```JAVA
interface MyInterface<T> {
    T get();
    void set(T value);
}
```

***
## 💡 Основные моменты 💡

1. **Интерфейс может иметь один или несколько параметров типа**:

```java
interface Pair<K, V> {
	K getKey();
	V getValue(); 
}
```
    
2. ✅✅**РЕАЛИЗАЦИЯ интерфейса МОЖЕТ БЫТЬ обобщённой или конкретизированной**:
    - В реализации ==можно оставить generic== параметр → реализация тоже будет обобщённой.
        
    - ==Можно «зафиксировать» тип== при реализации.

3. **Можно накладывать ограничения** (`extends`) на параметр типа.

***
## Реализация обобщенного интерфейса

Если класс реализует обобщенный интерфейс, то этот класс тоже должен быть обобщенным, по крайней мере, принимая параметр типа, который передается интерфейсу

```java
// Пример обобщенного интерфейса.
// Интерфейс для нахождения минимального и максимального значений.
interface MinMax<T extends Comparable<T>> {
    T min();
    T max();
}

// Реализовать интерфейс MinMax.
class MyClass<T extends Comparable<T> > implements MinMax<T> {
    T[] vals;
    
    MyClass(T[] o) {
        vals = o;
    }
    
    // Возвратить минимальное значение в vals.
    public T min() {
        T v = vals[0];
        for(int i = 1; i < vals.length; i++)
            if(vals[i].compareTo(v) < 0)
                v = vals[i];
        return v;
    }
    
    // Возвратить максимальное значение в vals.
    public T max() {
        T v = vals[0];
        for(int i = 1; i < vals.length; i++)
            if(vals[i].compareTo(v) > 0)
                v = vals[i];
        return v;
    }
}

class GenIFDemo {
    public static void main(String[] args) {
        Integer[] inums = {3, 6, 2, 8, 6};
        Character[] chs = {'b', 'r', 'p', 'w'};
        
        MyClass<Integer> iob = new MyClass<Integer>(inums);
        MyClass<Character> cob = new MyClass<Character>(chs);
        
        System.out.println("Максимальное значение в inums: " + iob.max());
        System.out.println("Минимальное значение в inums: " + iob.min());
        System.out.println("Максимальное значение в chs: " + cob.max());
        System.out.println("Минимальное значение в chs: " + cob.min());
    }
}
```


> [!NOTE] ⭐⭐Реализующий класс должен указывать ту же границу, что и интерфейс!⭐⭐
> Поскольку для `MinMax` требуется тип, который реализует `Comparable`, ==реализующий класс== (в данном случае `MyClass`) ==должен указывать ту же самую границу.== 
> 
> Более того, после установления этой границы нет никакой необходимости указывать ее снова в конструкции implements.
> 
> ✅Разумеется, если класс реализует `специфический тип обобщенного интерфейса`, ==тогда реализующий класс НЕ ОБЯЗАН быть обобщенным:==
> ```java
> class MyClass implements MinMax<Integer> // ✅Нормально, тип Integer известен
> ```
