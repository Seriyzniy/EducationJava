`UncaughtExceptionHandler` — это **механизм в Java для обработки неперехваченных исключений**, из-за которых поток завершает работу.

```java
public interface UncaughtExceptionHandler {
    void uncaughtException(Thread t, Throwable e);
}
```

Когда поток умирает от неперехваченного исключения (`RuntimeException`, `Error`), JVM вызывает метод `uncaughtException(...)`.

По умолчанию → трассировка стека печатается в `System.err`.

Можно назначить:
- 🔶обработчик для **ОПРЕДЕЛЕННОГО потока** через `thread.setUncaughtExceptionHandler(...)`;
- 🔶обработчик по умолчанию для **ВСЕХ потоков** через `Thread.setDefaultUncaughtExceptionHandler(...)`.

## Когда это нужно
- Если нужно **логировать** такие ошибки централизованно.
    
- Если приложение должно предпринимать действия при сбое (например, перезапустить поток, уведомить мониторинг).
    
- Особенно важно в средах с «долгоживущими» потоками

## Пример: локальный обработчик для ОДНОГО потока
```java
public class UncaughtHandlerDemo {
    public static void main(String[] args) {
        Thread thread = new Thread(() -> {
            System.out.println("Поток запущен");
            throw new RuntimeException("Ошибка в потоке!");
        });

        // Устанавливаем обработчик
        thread.setUncaughtExceptionHandler((t, e) -> {
            System.err.println("Поймано исключение в потоке " + t.getName() + ": " + e);
        });

        thread.start();
    }
}

//Вывод
Поток запущен
Поймано исключение в потоке Thread-0: java.lang.RuntimeException: Ошибка в потоке!
```

## Пример: глобальный обработчик для всех потоков
```java
public class GlobalUncaughtHandlerDemo {
    public static void main(String[] args) {
        Thread.setDefaultUncaughtExceptionHandler((t, e) -> {
            System.err.println("Глобальный обработчик: поток " + t.getName() + " завершился с ошибкой: " + e);
        });

        // Первый поток
        new Thread(() -> { throw new RuntimeException("Ошибка №1"); }).start();

        // Второй поток
        new Thread(() -> { throw new RuntimeException("Ошибка №2"); }).start();
    }
}

Глобальный обработчик: поток Thread-0 завершился с ошибкой: java.lang.RuntimeException: Ошибка №1

Глобальный обработчик: поток Thread-1 завершился с ошибкой: java.lang.RuntimeException: Ошибка №2

```
