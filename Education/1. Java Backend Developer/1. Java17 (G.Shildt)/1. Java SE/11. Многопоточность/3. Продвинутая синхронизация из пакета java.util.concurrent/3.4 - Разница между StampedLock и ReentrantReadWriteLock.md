`ReentrantReadWriteLock` и `StampedLock` решают схожую задачу (_много читателей, один писатель_), но у них **разная модель работы и гарантии**.

# ReentrantReadWriteLock
- Классический **read/write lock**.
- Два вида замков:
    - `readLock()` — shared (несколько потоков могут читать параллельно).
    - `writeLock()` — exclusive (только один поток, без читателей).
        - ✅ Одновременные **чтения** разрешены.
	    - ❌ Одновременное чтение+запись запрещено (писатель блокирует всех).
	
- **Реентрантность**: поток может повторно захватывать тот же замок (читатель или писатель).
    
- **Downgrade**: писатель может захватить `readLock`, потом освободить `writeLock`.
- **Upgrade** (reader → writer) напрямую невозможен (риск deadlock).
    
- Поддерживает _fair_ и _non-fair_ режим.
    
- У `writeLock` можно создавать `Condition`.
    
- Сильно похож на `synchronized`, только с разделением «читатели/писатель».
    
**Минус:** каждое чтение = захват блокировки → накладные расходы.

# StampedLock
- Новый класс (Java 8).
    
- Основан на **штампах (stamps)** — `long` значения, возвращаемые при захвате замка.
    
- Нет реентрантности (нельзя повторно захватить тот же замок в том же потоке).
    
- Три режима:
    1. **writeLock()** — эксклюзивная блокировка (как у RWLock).
    2. **readLock()** — совместная блокировка (как у RWLock).
    3. **optimisticRead()** — **оптимистичное чтение без блокировки**:
        - возвращает stamp,
        - читаем данные,
        - ⭐ в конце проверяем `validate(stamp)` → если во время чтения не было писателей, результат валиден; иначе нужно повторить под readLock/writeLock.
            - ✅Писатели могут идти параллельно с таким «чтением».
		    - 🟠Но результат нужно валидировать (`validate(stamp)`), чтобы убедиться, что во время чтения не было записи.

**Downgrade** — есть (`writeLock → readLock`).  
**Upgrade** — тоже есть (`tryConvertToWriteLock`), но может вернуть `0`, если не получилось.

**Минус:**
- Нет reentrant.
- Нет `Condition`.
- Сложнее в использовании (надо проверять stamp).
