
> [!NOTE] **CountDownLatch** — это синхронизатор из пакета `java.util.concurrent`
> ⭐Позволяющий одному или нескольким потокам ждать, пока `определённое КОЛИЧЕСТВО СОБЫТИЙ` не произойдёт.
> 
> 🧠Он работает как "замок обратного отсчёта (==счетчик==)": 
> latch инициализируется числом `N`, и каждый вызов `countDown()` уменьшает счётчик. 
> Когда счётчик достигает нуля — все ожидающие потоки **разблокируются**.
> 
> Реализует интерфейс `java.io.Serializable`.  
> Основан на `AbstractQueuedSynchronizer (AQS)`

## Конструктор
```java
// Создает CountDownLatch с заданным числом счётчика.
// count — количество событий, которые должны произойти, прежде чем latch "откроется".
// Если count = 0, то latch сразу считается "открытым".
public CountDownLatch(int count)
```

## Методы
### Основные операции
```java
// Ожидает, пока счётчик не достигнет нуля.
// Если latch уже "открыт", возвращается немедленно.
public void await() throws InterruptedException

// Ожидает, пока счётчик не достигнет нуля, но не дольше указанного времени.
// Возвращает true, если latch "открылся" вовремя, иначе false.
// timeout — время ожидания.
// unit — единицы измерения (например, TimeUnit.SECONDS).
public boolean await(
	 long timeout, 
	 TimeUnit unit) throws InterruptedException

// Уменьшает счётчик на единицу.
// Если счётчик достигает нуля, все ожидающие потоки разблокируются.
// Если счётчик уже равен нулю, вызов не оказывает эффекта.
public void countDown()

// Возвращает текущее значение счётчика (только для мониторинга, не для синхронизации).
public long getCount()

// Возвращает строковое представление CountDownLatch, включая текущее значение счётчика.
public String toString()
```

---
## 💡 Особенности и подводные камни 💡

1️⃣ ⭐⭐⭐**==Одноразовый==**: после того как счётчик дошёл до нуля, его нельзя сбросить или переиспользовать.  

2️⃣ ⭐Не имеет метода `reset()` — для повторного использования нужно создать новый объект.  

3️⃣ 💡Если `countDown()` вызывается больше раз, чем начальное значение — "лишние" вызовы игнорируются.  

4️⃣ ❗❗❗Потоки, вызвавшие `await()`, **блокируются** до тех пор, пока счётчик не обнулится (или не произойдёт `InterruptedException`).  

5️⃣ Используется как механизм **координации потоков**, например, чтобы главный поток ждал завершения N задач.  

6️⃣ 💡Поток, который вызывает `countDown()`, **никогда не блокируется**.  

7️⃣ В основе реализации — `AbstractQueuedSynchronizer` (AQS) с логикой "разрешить проход" при достижении 0.

---

## 💎 Best Practices 💎

✅ Отлично подходит для **ожидания завершения нескольких задач** (например, потоков-помощников).  

✅ Используйте в тестах для **синхронизации потоков** (например, дождаться старта всех).  

✅ ❗❗❗Если нужен **повторно используемый барьер**, вместо него применяйте `CyclicBarrier` или `Phaser`.  

✅ Если нужно "разблокировать" все потоки по событию — `CountDownLatch(1)` подходит идеально.  

✅ Не используйте `getCount()` для логики синхронизации — это диагностический метод.

## Пример
```java
CountDownLatch latch = new CountDownLatch(5);

for (int i = 0; i < 5; i++) {
	new Thread(() -> {
		System.out.println(Thread.currentThread().getName() + " работает...");
		try { 
			Thread.sleep(3000); 
			latch.countDown(); // сообщаем о завершении
		} catch (InterruptedException ignored) {}
	}).start();
}

System.out.println("Ожидаю завершения всех потоков...");
try {
	latch.await(); // блокирует текущий поток до достижения 0
} catch (InterruptedException e) {
	e.printStackTrace();
} 
System.out.println("Все потоки завершены!");


//Вывод
Ожидаю завершения всех потоков...
Thread-4 работает...
Thread-2 работает...
Thread-1 работает...
Thread-3 работает...
Thread-0 работает...

//Через 3 секунды каждый поток уменьшит значение счетчика на 1
Все потоки завершены!

```
