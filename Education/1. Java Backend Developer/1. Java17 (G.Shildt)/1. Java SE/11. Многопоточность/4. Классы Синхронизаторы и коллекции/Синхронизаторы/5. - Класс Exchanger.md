
> [!NOTE] `Exchanger<V>` — синхронизатор, позволяющий `двум потокам` `обменяться объектами`.
> Каждый поток вызывает `exchange()`, передавая свой объект и ожидая (БЛОКИРУЯСЬ), пока другой поток также вызовет `exchange()`. 
> 
> После этого объекты обмениваются местами, и оба потока продолжают работу, получив значение от партнёра.

Используется для **организации ==двустороннего обмена== данными** между парами потоков — например, между производителем и потребителем, между двумя этапами обработки данных или между потоками, работающими над разными частями задачи.

---
## Конструкторы
```java
Exchanger();
```
Создаёт новый объект `Exchanger`.  
Никакие параметры не требуются, так как обмен происходит ==**только между ПАРАМИ потоков**.==

## Методы
## Основные методы
```java
/*
Поток вызывает этот метод, передавая объект `x`, и блокируется, пока другой поток не вызовет `exchange()` для обмена.  

После успешного обмена возвращает объект, переданный вторым потоком.  

Если поток был прерван, выбрасывается `InterruptedException`.
*/
V exchange(V x) throws InterruptedException;


/*
То же самое, что и предыдущий метод, но с возможностью задать время ожидания
- Если за указанное время партнёр не появился, выбрасывается `TimeoutException`.
- Если поток прерван, выбрасывается `InterruptedException`.
*/
V exchange(V x, long timeout, TimeUnit unit)
    throws InterruptedException, TimeoutException;
```

---
## 💡 Особенности и детали реализации 💡

- ❗❗Обмен происходит **строго попарно**. Если поток вызвал `exchange()`, но второго пока нет — он блокируется.
    
- После того как появляется второй поток, происходит атомарный обмен объектами.
    
- `Exchanger` не сохраняет очередь потоков — обмен всегда только **между текущими парами**.
    
- Реализован с помощью **spin + park/unpark** (LockSupport) для высокой производительности.
    
- Потоки могут обмениваться как большими объектами, так и сигналами (например, `null`).

## Когда использовать Exchanger

✅ Когда нужно **синхронно обменяться данными** между двумя потоками.  
✅ При реализации **передачи буфера** между потоками (один наполняет, другой очищает).  
✅ В алгоритмах, где данные обрабатываются **в двух стадиях**, и результат первой стадии передаётся второй.

## Когда НЕ стоит использовать

❌ Если требуется обмен между **множеством потоков** — лучше использовать `BlockingQueue`.  
❌ Если обмен должен происходить **асинхронно**, без ожидания партнёра.  
❌ Если обмен нужен **чаще одного раза** между многими парами потоков одновременно (могут блокироваться).

---
## Пример
```java
Exchanger<String> exchanger = new Exchanger<>();

Thread t1 = new Thread(() -> {
	try {
		String response = exchanger.exchange("Привет от T1");
		System.out.println("T1 получил: " + response);
	} catch (InterruptedException e) {}
});

Thread t2 = new Thread(() -> {
	try {
		String response = exchanger.exchange("Привет от T2");
		System.out.println("T2 получил: " + response);
	} catch (InterruptedException e) {}
});

t1.start();
t2.start();

//Вывод
T2 получил: Привет от T1
T1 получил: Привет от T2
```
