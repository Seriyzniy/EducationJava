
> [!NOTE] `Phaser` — это синхронизатор из пакета `java.util.concurrent`, который позволяет `координировать` `выполнение потоков` `по фазам`.
> 💡Он похож на `CyclicBarrier` и `CountDownLatch`, но **гораздо гибче**:
> - ⭐Позволяет динамически `регистрировать и отменять` участников (threads). 
> - ⭐Поддерживает многофазные барьеры (`многократное использование` без ручного сброса).
> 

---
## Конструктор
```java
// Создает Phaser без зарегистрированных участников (число участников — 0)
public Phaser()

// Создает Phaser с указанным числом участников
// parties — количество участников, которые должны достичь барьера, чтобы завершить фазу
public Phaser(int parties)

// Создает Phaser, имеющий родительский Phaser и заданное количество участников
// 💡parent — родительский Phaser, если требуется иерархическая синхронизация (многоуровневые барьеры)
// parties — количество участников в текущем Phaser
public Phaser(Phaser parent, int parties)
```

## Методы
### Управление регистрацией участников
```java
// 💡Регистрирует нового участника (увеличивает количество ожидаемых потоков на 1)
// Возвращает номер текущей фазы
public int register()

// Регистрирует заданное количество участников
// parties — количество новых участников
// Возвращает номер текущей фазы
public int bulkRegister(int parties)

// Отменяет регистрацию одного участника (уменьшает количество ожидаемых потоков на 1)
// Возвращает номер текущей фазы
public int arriveAndDeregister()
```

### Управление прибытием (сигнализация о завершении фазы)
```java
// 💡Сообщает о прибытии потока к барьеру, 💡но НЕ блокирует поток💡
// Возвращает номер фазы, в которой произошло прибытие
public int arrive()

// 💡Сообщает о прибытии потока 💡и ждет💡, пока все остальные участники тоже прибудут
// Возвращает номер следующей фазы
public int arriveAndAwaitAdvance()
```

### Ожидание и продвижение фаз
```java
// 💡Блокирует поток до тех пор, пока фаза с номером phase не завершится
// Возвращает номер следующей фазы
public int awaitAdvance(int phase)

// Блокирует поток до завершения указанной фазы, но с учетом прерываний
public int awaitAdvanceInterruptibly(int phase) throws InterruptedException

// Блокирует поток с таймаутом до завершения указанной фазы
// timeout — время ожидания
// unit — единицы измерения времени
public int awaitAdvanceInterruptibly(int phase, long timeout, TimeUnit unit)
        throws InterruptedException, TimeoutException
```

### Проверка состояния и управления потоком
```java
// 💡Возвращает номер текущей фазы
public int getPhase()

// Проверяет, завершен ли Phaser (true, если был завершен вручную)
public boolean isTerminated()

// Принудительно завершает Phaser, все ожидающие фазы прерываются
public void forceTermination()

// Возвращает количество зарегистрированных участников
public int getRegisteredParties()

// Возвращает количество участников, которые еще не прибыли
public int getUnarrivedParties()

// Возвращает количество участников, которые уже прибыли
public int getArrivedParties()
```

### Настройка поведения
```java
// 💡Вызывается ПРИ ЗАВЕРШЕНИИ КАЖДОЙ ФАЗЫ — можно переопределить в подклассе
// Возвращает true, если Phaser должен завершиться
protected boolean onAdvance(int phase, int registeredParties)
```
💬 Метод `onAdvance()` — ключевая точка расширения.  
Если вернуть `true`, Phaser перейдет в состояние завершения.

## 💡 **Особенности и подводные камни** 💡

1️⃣ 💡`Phaser` — **динамический барьер**: можно добавлять и убирать участников на лету, в отличие от `CyclicBarrier`.  

2️⃣ **Не использовать** `awaitAdvance()` без проверки номера фазы — иначе поток может зависнуть, если фаза уже завершена.  

3️⃣ После `forceTermination()` все методы возврата фаз возвращают **отрицательные значения**.  

4️⃣ Если все участники дерегистрированы, `Phaser` **автоматически завершает** свою работу.  

5️⃣ `onAdvance()` вызывается **автоматически** при завершении каждой фазы — можно реализовать логику сброса или завершения.

---

## 💎 **Best Practices** 💎

✅ 💡Используй `arriveAndAwaitAdvance()` для синхронного ожидания всех участников.  

✅ 💡Вызывай `arriveAndDeregister()` в конце работы потока, чтобы освободить участника.  
✅ Не вызывай `awaitAdvance()` без контекста фазы (сохраняй номер фазы заранее).  

✅ Если требуется ограничить количество фаз, переопредели `onAdvance()`.  

✅ Для иерархических систем (например, многослойные задачи) используй конструктор с `parent Phaser`.

```java
import java.util.concurrent.Phaser;
import java.util.concurrent.Semaphore;

public class MyMain {
	public static void main(String[] args) {
		Phaser phsr = new Phaser(1);
		int curPhase;

		System.out.println("Начало");

		new Thread(new MyThread(phsr, "А")).start();
		new Thread(new MyThread(phsr, "В")).start();
		new Thread(new MyThread(phsr, "С")).start();

		// Ожидать завершения всеми потоками первой стадии.
		curPhase = phsr.getPhase();
		phsr.arriveAndAwaitAdvance();
		System.out.println("Стадия " + curPhase + " завершена");

		// Ожидать завершения всеми потоками второй стадии.
		curPhase = phsr.getPhase();
		phsr.arriveAndAwaitAdvance();
		System.out.println("Стадия " + curPhase + " завершена");

		curPhase = phsr.getPhase();
		phsr.arriveAndAwaitAdvance();
		System.out.println("Стадия " + curPhase + " завершена");

		// Отменить регистрацию главного потока.
		phsr.arriveAndDeregister();

		if (phsr.isTerminated())
			System.out.println("Объект Phaser закончил работу");
	}
}

//Поток выполнения, который использует объект Phaser.
class MyThread implements Runnable {
	Phaser phsr;
	String name;

	MyThread(Phaser р, String п) {
		phsr = р;
		name = п;
		phsr.register();
	}

	public void run() {
		System.out.println("Поток " + name + " начал первую стадию");
		phsr.arriveAndAwaitAdvance(); // Сигнализировать о прибытии.

		// Организовать небольшую паузу, чтобы предотвратить беспорядочный
		// вывод. Это делается только в целях иллюстрации и не требуется
		// для корректной работы объекта Phaser.
		try {
			Thread.sleep(2000);
		} catch (InterruptedException e) {
			System.out.println(e);
		}

		System.out.println("Поток " + name + " начал вторую стадию");
		phsr.arriveAndAwaitAdvance(); // Сигнализировать о прибытии.

		// Организовать небольшую паузу, чтобы предотвратить беспорядочный
		// вывод. Это делается только в целях иллюстрации и не требуется
		// для корректной работы объекта Phaser.
		try {
			Thread.sleep(100);
		} catch (InterruptedException e) {
			System.out.println(e);
		}

		System.out.println("Поток " + name + " начал третью стадию");
		phsr.arriveAndDeregister(); // Сигнализировать о прибытии и отметить регистрацию.
	}
}

Поток С начал третью стадию
Поток А начал третью стадию
```
