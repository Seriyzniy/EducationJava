
> [!NOTE] **`ConcurrentLinkedQueue<E>`** — это **потокобезопасная неблокирующая очередь FIFO** (First-In-First-Out), 
> Реализующая интерфейсы:
> - `Queue<E>` — очередь общего назначения
> - `AbstractQueue<E>` — абстрактный базовый класс очередей;
> - `Serializable` — поддерживает сериализацию.
> 

🧠Она использует **алгоритм Майкла-Скотта (Michael & Scott Non-blocking Queue Algorithm)**, работающий на **CAS (Compare-And-Set)**, что позволяет нескольким потокам безопасно добавлять и извлекать элементы **без блокировок и без использования synchronized**.

## Конструктор
```java
// Создает пустую неблокирующую очередь.
public ConcurrentLinkedQueue()

// Создает очередь, содержащую элементы из другой коллекции.
// c — коллекция, элементы которой будут добавлены в очередь в порядке итерации.
public ConcurrentLinkedQueue(Collection<? extends E> c)
```

## Методы
### Основные операции вставки и удаления
```java
/*
offer(E e)
Добавляет элемент в конец очереди.
Не блокирует поток, использует CAS для добавления.
Возвращает true, если элемент был успешно добавлен.
Если e == null, выбрасывает NullPointerException.
*/
public boolean offer(E e)

/*
add(E e)
Эквивалентен offer(e).
Возвращает true.
*/
public boolean add(E e)

/*
poll()
Извлекает и удаляет элемент из начала очереди.
Если очередь пуста, возвращает null.
Операция неблокирующая — CAS обновляет head.
*/
public E poll()

/*
remove()
Удаляет первый элемент очереди.
Если очередь пуста, выбрасывает NoSuchElementException.
*/
public E remove()

/*
peek()
Возвращает (но не удаляет) элемент из головы очереди.
Если очередь пуста, возвращает null.
*/
public E peek()

/*
element()
Возвращает (но не удаляет) элемент из головы очереди.
Если очередь пуста, выбрасывает NoSuchElementException.
*/
public E element()
```

### Проверка состояния
```java
//
/*
isEmpty()
Возвращает true, если очередь пуста.
*/
public boolean isEmpty()

/*
size()
Возвращает примерное количество элементов в очереди.
Метод НЕ атомарный: значение может быть неточным в многопоточной среде.
*/
public int size()

/*
contains(Object o)
Проверяет наличие элемента в очереди.
Использует линейный проход.
Возвращает true, если элемент найден.
*/
public boolean contains(Object o)

/*
clear()
Удаляет все элементы из очереди.
Проходит по элементам и делает их недостижимыми (item = null).
*/
public void clear()
```

### Итерация и представление
```java
/*
iterator()
Возвращает итератор, который безопасен для многопоточности.
Итератор слабоконсистентен: не кидает ConcurrentModificationException,
но может не увидеть новые элементы, добавленные другими потоками.
*/
public Iterator<E> iterator()

/*
toArray()
Возвращает массив, содержащий элементы очереди в порядке FIFO.
*/
public Object[] toArray()

/*
toArray(T[] a)
Возвращает массив указанного типа, содержащий элементы очереди.
*/
public <T> T[] toArray(T[] a)
```

## 💡 Особенности и подводные камни 💡

1️⃣ **Неблокирующая архитектура (Lock-Free)**  
Работает без блокировок, используя CAS для обновления ссылок `head` и `tail`. Это делает её особенно эффективной при высокой конкуренции потоков.

2️⃣ **FIFO-порядок гарантирован**, но итераторы не гарантируют моментальную согласованность:  
Итератор может видеть только часть актуальных элементов, если другие потоки добавляют новые данные.

3️⃣ **Метод `size()` не точен**, т.к. вычисляется проходом по всей очереди — O(n).  
❗❗❗Не использовать его для синхронизации потоков❗❗❗

4️⃣ **Элементы не могут быть `null`** — очередь интерпретирует `null` как «нет элемента».

5️⃣ **Операции poll/offer выполняются за O(1)** в среднем.  
Отлично масштабируется при большом количестве потоков.

6️⃣ **Очередь не имеет ограничений размера**, что может привести к росту памяти при бесконтрольном добавлении элементов.

---

## 💎 Best Practices 💎

✅ Используй `ConcurrentLinkedQueue`, когда:
- требуется **многопоточная неблокирующая очередь**;
    
- потоки **активно читают и пишут одновременно**;
    
- нет необходимости в ограничении размера (неbounded queue).
    

✅ НЕ ИСПОЛЬЗУЙ, когда:
- нужна очередь с **ограничением размера** — вместо неё лучше `LinkedBlockingQueue` или `ArrayBlockingQueue`;
    
- требуется точный `size()` или строгая консистентность итерации.
    

✅ Для **производительных очередей событий** или **внутрипоточного обмена данными** — идеальный выбор.  

✅ ❗❗❗Не пытайся синхронизировать вручную — это ломает lock-free модель.

## Пример
```JAVA
import java.util.Queue;
import java.util.concurrent.ConcurrentLinkedQueue;
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;
import java.util.concurrent.TimeUnit;

// Класс задачи для обработки
class Task {
    private final int id;
    private final String data;
    
    public Task(int id, String data) {
        this.id = id;
        this.data = data;
    }
    
    public void process() {
        System.out.println(Thread.currentThread().getName() + 
                          " обрабатывает задачу " + id + ": " + data);
        try {
            // Имитация обработки
            Thread.sleep(1000);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
}

public class ConcurrentQueueExample {
    private final Queue<Task> taskQueue = new ConcurrentLinkedQueue<>();
    private final ExecutorService executor;
    private volatile boolean running = true;
    
    public ConcurrentQueueExample(int workerCount) {
        this.executor = Executors.newFixedThreadPool(workerCount);
    }
    
    // Поток-производитель (Producer)
    public void startProducer() {
        Thread producerThread = new Thread(() -> {
            int taskId = 0;
            while (running) {
                try {
                    // Создаем новую задачу
                    Task task = new Task(taskId++, "Данные-" + System.currentTimeMillis());
                    
                    // Добавляем в очередь (неблокирующая операция)
                    taskQueue.offer(task);
                    System.out.println("Добавлена задача: " + taskId);
                    
                    Thread.sleep(500); // Имитация времени между задачами
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                    break;
                }
            }
        }, "Producer-Thread");
        producerThread.start();
    }
    
    // Worker-потоки (Consumer)
    public void startWorkers(int workerCount) {
        for (int i = 0; i < workerCount; i++) {
            final int workerId = i;
            executor.submit(() -> {
                while (running || !taskQueue.isEmpty()) {
                    try {
                        // Берем задачу из очереди (неблокирующая операция)
                        Task task = taskQueue.poll();
                        
                        if (task != null) {
                            // Обрабатываем задачу
                            task.process();
                        } else {
                            // Очередь пуста - небольшая пауза
                            Thread.sleep(100);
                        }
                    } catch (InterruptedException e) {
                        Thread.currentThread().interrupt();
                        break;
                    }
                }
                System.out.println("Worker-" + workerId + " завершил работу");
            });
        }
    }
    
    public void addTask(Task task) {
        taskQueue.offer(task);
    }
    
    public void shutdown() {
        running = false;
        executor.shutdown();
        try {
            if (!executor.awaitTermination(5, TimeUnit.SECONDS)) {
                executor.shutdownNow();
            }
        } catch (InterruptedException e) {
            executor.shutdownNow();
            Thread.currentThread().interrupt();
        }
    }
    
    public static void main(String[] args) throws InterruptedException {
        ConcurrentQueueExample system = new ConcurrentQueueExample(3);
        
        // Запускаем производителя
        system.startProducer();
        
        // Запускаем worker-потоки
        system.startWorkers(3);
        
        // Добавляем несколько задач напрямую
        for (int i = 100; i < 105; i++) {
            system.addTask(new Task(i, "Прямая-задача-" + i));
        }
        
        // Даем системе поработать 10 секунд
        Thread.sleep(10000);
        
        // Останавливаем систему
        System.out.println("Завершение работы...");
        system.shutdown();
        
        // Выводим статистику
        System.out.println("Осталось задач в очереди: " + system.taskQueue.size());
    }
}

```
