
> [!NOTE] **PriorityBlockingQueue** — это неограниченная `блокирующая очередь с приоритетами`, реализующая интерфейс `BlockingQueue`.
> ⭐Элементы в ней ==упорядочены согласно их **естественному порядку**== (через `Comparable`) или через **переданный компаратор**.
> 
> 💡Очередь **не гарантирует FIFO** — порядок определяется приоритетом, а не временем добавления.
> 
> Реализует интерфейсы:  `BlockingQueue<E>`, `Serializable`.

Основана на **бинарной куче (min-heap)**, аналогично `PriorityQueue`, но с поддержкой блокирующих операций.

## Конструкторы
```java
// Создает пустую очередь с начальной емкостью 11 и естественным порядком элементов.
public PriorityBlockingQueue()

// Создает пустую очередь с указанной начальной емкостью.
public PriorityBlockingQueue(int initialCapacity)

// Создает пустую очередь с указанной начальной емкостью и компаратором для упорядочивания элементов.
// initialCapacity — начальный размер внутреннего массива.
// comparator — объект, определяющий порядок элементов (может быть null).
public PriorityBlockingQueue(int initialCapacity, Comparator<? super E> comparator)

// Создает очередь, содержащую элементы указанной коллекции.
// Порядок элементов будет определен их естественным порядком или компаратором.
public PriorityBlockingQueue(Collection<? extends E> c)
```

## Методы
### Основные операции добавления и извлечения
```java
// Добавляет элемент в очередь (аналогично offer()).
// Бросает IllegalArgumentException, если элемент не реализует Comparable и компаратор не задан.
public boolean add(E e)

// Добавляет элемент в очередь (без блокировки, всегда успешно, т.к. очередь неограничена).
public boolean offer(E e)

// Добавляет элемент, ожидая, если потребуется, (но здесь ожидание не используется, так как очередь не ограничена).
public void put(E e) throws InterruptedException

// Извлекает элемент с наивысшим приоритетом, ожидая, если очередь пуста.
public E take() throws InterruptedException

// Извлекает элемент с наивысшим приоритетом, либо null, если очередь пуста.
public E poll()

// Извлекает элемент с наивысшим приоритетом, ожидая заданное время, если очередь пуста.
public E poll(long timeout, TimeUnit unit) throws InterruptedException

// Возвращает (но не удаляет) элемент с наивысшим приоритетом, либо null, если очередь пуста.
public E peek()
```

### Операции коллекции
```java
// Возвращает количество элементов в очереди.
public int size()

// Проверяет, пуста ли очередь.
public boolean isEmpty()

// Удаляет все элементы из очереди.
public void clear()

// Удаляет один экземпляр указанного элемента, если он присутствует.
public boolean remove(Object o)

// Проверяет, содержится ли элемент в очереди.
public boolean contains(Object o)
```

### Итераторы и массивы
```java
// Возвращает массив, содержащий все элементы в порядке приоритетов.
public Object[] toArray()

// Возвращает массив указанного типа, содержащий все элементы.
public <T> T[] toArray(T[] a)

// Возвращает итератор по элементам (итератор не гарантирует порядок приоритетов).
public Iterator<E> iterator()
```

### Другие методы
```java
// Возвращает компаратор, используемый для упорядочивания элементов, или null, если используется естественный порядок.

// Добавляет все элементы из указанной коллекции.
```

## 💡 Особенности и подводные камни 💡

1️⃣ Очередь **не ограничена**, поэтому методы `put()` и `offer()` никогда не блокируют.  

2️⃣ ⭐⭐⭐Элементы должны быть **сравнимыми** — либо реализовывать `Comparable`, либо должен быть передан `Comparator`.  

3️⃣ Итератор **не гарантирует порядок** элементов по приоритету.  

4️⃣ ❗❗Очередь **не потокобезопасна для изменения элементов**, которые уже внутри и участвуют в сравнении.  

5️⃣ Приоритеты не равны "весам": если два элемента равны — их относительный порядок не определён (не FIFO).  

6️⃣ Не подходит для задач, где требуется ограничение размера (лучше `ArrayBlockingQueue` или `LinkedBlockingQueue`).

---

## 💎 Best Practices 💎

✅ Используйте, если нужно обрабатывать задачи в порядке приоритета.  

✅ Передавайте `Comparator`, если элементы не реализуют `Comparable`.  

✅ Не модифицируйте объекты после помещения в очередь — это может нарушить порядок.  

✅ Для ограничения размера можно использовать обёртку `new Semaphore(n)` для ручного контроля потока.

## Пример
```java
import java.util.concurrent.*;

public class PriorityBlockingQueueExample {
	// Внутренний класс, который помещается в очередь
    static class Task implements Comparable<Task> {
        private final int priority;
        private final String name;

        public Task(int priority, String name) {
            this.priority = priority;
            this.name = name;
        }

        @Override
        public int compareTo(Task o) {
            // Чем меньше число, тем выше приоритет
            return Integer.compare(this.priority, o.priority);
        }

        @Override
        public String toString() {
            return name + " (priority=" + priority + ")";
        }
    }


    public static void main(String[] args) throws InterruptedException {
        BlockingQueue<Task> queue = new PriorityBlockingQueue<>();

        queue.put(new Task(3, "Low priority"));
        queue.put(new Task(1, "High priority"));
        queue.put(new Task(2, "Medium priority"));

        while (!queue.isEmpty()) {
            System.out.println(queue.take());
        }
    }
}
// Порядок в зависимости от compareTo
High priority (priority=1)
Medium priority (priority=2)
Low priority (priority=3)
```
