
> [!NOTE] `ConcurrentHashMap<K,V>` — это потокобезопасная реализация `Map`,
> Оптимизированна для высокой конкурентности без блокировки всей таблицы.  
> 

⭐В отличие от `Collections.synchronizedMap()` и `Hashtable`, которые используют **глобальную блокировку**, `ConcurrentHashMap` ==применяет **тонкую гранулярную синхронизацию**== (lock striping) на уровне сегментов или бакетов (в зависимости от версии JDK).

- В **Java 7** — использовалась сегментация (Segment-based locking).
    
- В **Java 8+** — структура переработана: используется **ConcurrentHashMap Node[], CAS-операции, synchronized на узлах и tree bins (TreeNodes)**, аналогично `HashMap`.

Это делает операции `get()` **неблокирующими**, а операции `put()` и `remove()` — **минимально блокирующими**.

## Конструкторы
```java
// Создает пустую карту с начальной емкостью 16
ConcurrentHashMap()

// Создает карту с заданной начальной емкостью
ConcurrentHashMap(int initialCapacity)
// initialCapacity — начальное количество бакетов (массив Node[]).

// Создает карту с указанной емкостью и коэффициентом загрузки
ConcurrentHashMap(int initialCapacity, float loadFactor)
// loadFactor — порог заполнения (по умолчанию 0.75).

// Создает карту с указанной емкостью, коэффициентом загрузки и уровнем конкурентности (Java 7)
// concurrencyLevel — ожидаемое число одновременно модифицирующих потоков
ConcurrentHashMap(
	int initialCapacity, 
	float loadFactor, 
	int concurrencyLevel)

// Создает карту, инициализированную содержимым другой Map
ConcurrentHashMap(Map<? extends K, ? extends V> m)
```

## Методы
```java
//✅ БАЗОВЫЕ ОПЕРАЦИИ
/* 
  Возвращает значение, связанное с заданным ключом, либо null, если соответствия нет.
  Параметры:
    key - искомый ключ (не может быть null).
  Особенности:
    - Операция неблокирующая и очень быстрая (обычно O(1)). 
    - Поскольку ConcurrentHashMap не допускает null-ключей/значений, возвращаемое null однозначно означает отсутствие ключа.
    - Бросает NullPointerException при передаче null в качестве key.
*/
V get(Object key);

/* 
  Ассоциирует указанное значение с ключом. Если ключ уже присутствовал, возвращается предыдущее значение.
  Параметры:
    key   - ключ (не null).
    value - значение (не null).
  Особенности:
    - Операция атомична для конкретного ключа: вставка/замена выполняется с использованием CAS и локов в узлах по необходимости.
    - Бросает NullPointerException при null-аргументах.
    - Возвращает предыдущее значение или null, если такого не было.
*/
V put(K key, V value);

/* 
  Удаляет отображение для данного ключа (если присутствует) и возвращает прежнее значение.
  Параметры:
    key - ключ для удаления (не null).
  Особенности:
    - Удаление выполняется атомарно для этого ключа.
    - Возвращает связанное ранее значение или null, если отображение отсутствовало.
    - Бросает NullPointerException при null-ключе.
*/
V remove(Object key);

/* 
  Проверяет, присутствует ли указанный ключ в карте.
  Параметры:
    key - ключ для проверки (не null).
  Особенности:
    - Быстрый, неблокирующий (в большинстве случаев) запрос.
    - Бросает NullPointerException при null-ключе.
*/
boolean containsKey(Object key);

/* 
  Проверяет, содержится ли указанное значение среди значений карты.
  Параметры:
    value - значение для поиска (не null).
  Особенности:
    - Это операция O(n): сканирует все элементы (медленная при больших размерах).
    - Бросает NullPointerException при null-значении (ConcurrentHashMap не допускает null-значений).
    - Возвращает true, если хотя бы одно значение equals() переданному.
*/
boolean containsValue(Object value);

/* 
  Возвращает примерный размер карты как int.
  Особенности:
    - В условиях конкурентных модификаций результат может быть неточным.
    - Для точного (и более дорогого) подсчёта используйте mappingCount().
*/
int size();

/* 
  Удаляет все отображения из карты.
  Особенности:
    - Выполняет безопасную очистку структуры, может затрагивать множество бакетов.
    - Во время выполнения другие потоки могут одновременно добавлять элементы; после завершения clear карта гарантированно пуста.
*/
void clear();

/* 
  Возвращает значение по ключу или defaultValue, если ключ отсутствует.
  Параметры:
    key          - ключ (не null).
    defaultValue - значение по умолчанию (может быть null, но ConcurrentHashMap обычно не хранит null).
  Особенности:
    - Удобно для безопасного считывания без проверки на null.
*/
V getOrDefault(Object key, V defaultValue);

/* 
  Добавляет все отображения из другой m в эту карту.
  Параметры:
    m - карта-источник (не null), содержащая пары (ключи/значения не должны быть null).
  Особенности:
    - Последовательность вставок может происходить в произвольном порядке, и другие потоки могут одновременно модифицировать карту.
    - Bросает NullPointerException если m или содержащиеся в ней ключи/значения null.
*/
void putAll(Map<? extends K, ? extends V> m);



//✅ АТОМАРНЫЕ ОПЕРАЦИИ
/* 
  Вставляет значение только если для данного ключа его ещё нет.
  Параметры:
    key   - ключ (не null).
    value - значение (не null).
  Поведение:
    - Атомарно добавляет новую пару, если ключ отсутствует.
    - Возвращает предыдущее значение, если оно было, иначе null.
    - Полезно для ленивой инициализации без внешней синхронизации.
*/
V putIfAbsent(K key, V value);

/* 
  Удаляет отображение только если текущее значение точно равно указанному.
  Параметры:
    key   - ключ (не null).
    value - ожидаемое значение (не null).
  Поведение:
    - Атомарная проверка-удаление: возвращает true, если удаление произошло.
    - Бросает NullPointerException при null-аргументах.
*/
boolean remove(Object key, Object value);

/* 
  Заменяет значение только если текущее значение равно oldValue.
  Параметры:
    key      - ключ (не null).
    oldValue - ожидаемое текущее значение (не null).
    newValue - новое значение (не null).
  Поведение:
    - Атомарная операция: возвращает true при успешной замене.
    - Полезно для безблокировочного условного обновления.
*/
boolean replace(K key, V oldValue, V newValue);

/* 
  Заменяет значение для ключа, если он присутствует.
  Параметры:
    key   - ключ (не null).
    value - новое значение (не null).
  Поведение:
    - Возвращает предыдущее значение или null, если отображения не было.
*/
V replace(K key, V value);



//✅ ВЫЧИСЛИТЕЛЬНЫЕ МЕТОДЫ (Java 8+)
/* 
  Пересчитывает/устанавливает значение для ключа, используя remappingFunction, передавая текущую пару (ключ, старое значение).
  Параметры:
    key               - ключ (не null).
    remappingFunction - BiFunction(key, oldValue) -> newValue; может вернуть null, чтобы удалить отображение.
  Поведение и особенности:
    - Выполняется атомарно для данного ключа.
    - remappingFunction может быть вызвана несколько раз при конкуренции; поэтому она должна быть идемпотентной или не иметь побочных эффектов.
    - Если remappingFunction возвращает null — соответствие удаляется.
    - Бросает NPE при null-аргументах.
*/
V compute(K key, BiFunction<? super K, ? super V, ? extends V> remappingFunction);

/* 
  Если ключ отсутствует, вычисляет значение с помощью mappingFunction и вставляет его.
  Параметры:
    key             - ключ (не null).
    mappingFunction - Function(key) -> value; может вернуть null (в этом случае ничего не вставляется).
  Поведение и особенности:
    - Атомарная проверка-и-вставка.
    - mappingFunction может быть вызвана более одного раза при конкуренции; не допускает побочных эффектов.
    - Нельзя возвращать null и ожидать вставки — null трактуется как "не вставлять".
*/
V computeIfAbsent(K key, Function<? super K, ? extends V> mappingFunction);

/* 
  Если ключ присутствует, пересчитывает значение с помощью remappingFunction.
  Параметры:
    key               - ключ (не null).
    remappingFunction - BiFunction(key, oldValue) -> newValue; если возвращает null — запись удаляется.
  Поведение:
    - Атомарно для данного ключа.
    - remappingFunction может быть вызвана несколько раз при конкуренции.
*/
V computeIfPresent(K key, BiFunction<? super K, ? super V, ? extends V> remappingFunction);

/* 
  Объединяет имеющееся и новое значение: если ключ отсутствует — вставляет value; если присутствует — вызывает remappingFunction(oldValue, value).
  Параметры:
    key               - ключ (не null).
    value             - новое значение (не null) (это значение подлежит объединению).
    remappingFunction - BiFunction(oldValue, newValue) -> result; если возвращает null — запись удаляется.
  Особенности:
    - Атомарная операция; удобно для суммирования/агрегации.
    - remappingFunction может быть вызвана несколько раз под конкуренцией.
*/
V merge(K key, V value, BiFunction<? super V, ? super V, ? extends V> remappingFunction);

/* 
  Применяет функцию ко всем парам (ключ, значение) и заменяет значения на результат.
  Параметры:
    function - BiFunction(key, oldValue) -> newValue; может возвращать null, что приведёт к удалению отображения.
  Особенности:
    - Обновления выполняются в безопасном конкурентном режиме; функция может применяться параллельно к разным ключам.
    - Не гарантирует единообразное "атомарное" обновление всех записей одновременно — каждый ключ обновляется отдельно.
*/
void replaceAll(BiFunction<? super K, ? super V, ? extends V> function);

/* 
  Проходит по всем парам и применяет действие; обход слабосогласованный: итераторы не кидают ConcurrentModificationException.
  Параметры:
    action - BiConsumer(key, value).
  Особенности:
    - Может не увидеть элементы, добавленные после начала обхода, и может видеть элементы, которые были удалены в процессе.
    - Действие может выполняться в произвольном порядке.
*/
void forEach(BiConsumer<? super K, ? super V> action);



//✅ ПАРАЛЛЕЛЬНЫЕ ОПЕРАЦИИ (ForkJoinPool, Java 8+)
/* 
  Параллельный ОБХОД КАРТЫ: если оценочная численность >= parallelismThreshold, задача лестнично распараллеливается по ForkJoinPool.commonPool().
  
  Параметры:
parallelismThreshold - порог для запуска параллельной обработки (обычно число элементов).
action - BiConsumer(key, value) — действие для каждой пары.
  
  Особенности:
    - Использует общий ForkJoinPool, поэтому может конфликтовать с другими задачами в нём.
    - При маленьких размерах лучше ставить порог высоко, чтобы избежать накладных расходов.
*/
void forEach(
	long parallelismThreshold, 
	BiConsumer<? super K, ? super V> action);

/* 
  Параллельное свёртывание: трансформирует пары в промежуточные U и затем сворачивает их reducer'ом.
  Параметры:
    parallelismThreshold - порог для распараллеливания.
    transformer          - BiFunction(key, value) -> U, преобразует пару в значение типа U.
    reducer              - BiFunction(U, U) -> U, объединяет два результата.
  Особенности:
    - Возвращает итоговую агрегированную величину U или null, если карта пуста.
    - Работает параллельно в ForkJoinPool.commonPool().
*/
<U> U reduce(long parallelismThreshold,
             BiFunction<? super K, ? super V, ? extends U> transformer,
             BiFunction<? super U, ? super U, ? extends U> reducer);

/* 
  Параллельный поиск: применяет searchFunction к парам и возвращает первый ненулевой результат.
  Параметры:
    parallelismThreshold - порог для распараллеливания.
    searchFunction       - BiFunction(key, value) -> U; возвращает non-null для найденного результата.
  Особенности:
    - Поиск может быть прерван, как только найден первый подходящий результат.
    - Выполнение в ForkJoinPool.commonPool().
*/
<U> U search(long parallelismThreshold,
             BiFunction<? super K, ? super V, ? extends U> searchFunction);



//✅ ПРЕДСТАВЛЕНИЯ И СТАТИСТИКА
/* 
  Возвращает (обычно) точное количество отображений как long.
  Особенности:
    - Более надёжный способ получить количество при конкурентных модификациях, чем size().
    - Может быть более затратным по времени, но даёт long (поддерживает большие карты).
*/
long mappingCount();

/* 
  Возвращает Enumeration ключей (устаревшее API).
  Особенности:
    - Представляет snapshot-/weakly-consistent view: во время обхода карта может изменяться.
    - Рекомендуется использовать keySet() / entrySet() и итераторы вместо Enumeration.
*/
Enumeration<K> keys();

/* 
  Возвращает Enumeration значений (устаревшее API).
  Особенности:
    - См. keys().
*/
Enumeration<V> elements();

/* 
  Возвращает "живую" коллекцию значений, поддерживающую слабосогласованные итераторы.
  Особенности:
    - Коллекция отражает текущее состояние карты в момент итерации (но может не включать совсем недавние изменения).
    - Изменения в карте отражаются в view и наоборот при некоторых операциях.
*/
Collection<V> values();

/* 
  Возвращает множество ключей (view).
  Особенности:
    - Изменения в множестве отражают изменения в карте и наоборот.
    - Итераторы — слабосогласованные.
*/
Set<K> keySet();

/* 
  Возвращает множество записей (Map.Entry<K,V>) — view.
  Особенности:
    - Map.Entry может отражать изменения значений через setValue().
    - Итераторы слабосогласованы и не кидают ConcurrentModificationException.
*/
Set<Map.Entry<K,V>> entrySet();
```

## Особенности и подводные камни
- ❌ **Не допускает `null`** для ключей и значений — выбрасывает `NullPointerException`.
    
- **Методы `size()` и `isEmpty()` неточны** во время активных модификаций — используйте `mappingCount()`.
    
- ❓**Weakly-consistent итераторы** — не кидают `ConcurrentModificationException`, но могут пропустить изменения.
    
- 💡❓**`forEach()`, `reduce()`, `search()`** используют общий `ForkJoinPool.commonPool()` — может блокироваться при его загрузке.
    
- 💡**Высокая конкурентность ≠ всегда быстрее** — на малых размерах накладные расходы CAS ощутимы.

## Best Practices
- ✅ Используй `computeIfAbsent()` вместо `if (!map.containsKey()) put()`:
```java
map.computeIfAbsent(key, k -> loadValue(k));
```

- 💡Избегай вызовов `size()` в горячих потоках — лучше `mappingCount()`.
    
- ❗❗Ключи должны быть **иммутабельными** — стабильные `hashCode()` и `equals()`.
    
- Для параллельных вычислений лучше задавать разумный `parallelismThreshold`, например `1_000`.

## Пример
```java
import java.util.concurrent.*;

public class ConcurrentHashMapExample {
    public static void main(String[] args) throws InterruptedException {
        ConcurrentHashMap<String, Integer> map = 
	        new ConcurrentHashMap<>();

        // Добавление элементов
        map.put("Java", 10);
        map.putIfAbsent("C++", 7);

        // Атомарное обновление
        map.computeIfPresent("Java", (k, v) -> v + 1);

        // Многопоточное обновление
        ExecutorService pool = Executors.newFixedThreadPool(3);
        for (String lang : new String[]{"Java", "Python", "Java", "C++"}) {
            pool.submit(() -> map.merge(lang, 1, Integer::sum));
        }

        pool.shutdown();
        pool.awaitTermination(1, TimeUnit.SECONDS);

        // Безопасный обход
        map.forEach((k, v) -> System.out.println(k + " → " + v));
    }
}
```
