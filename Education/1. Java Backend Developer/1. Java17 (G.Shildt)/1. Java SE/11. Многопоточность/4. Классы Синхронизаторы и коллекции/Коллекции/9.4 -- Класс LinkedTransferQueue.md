
> [!NOTE] Класс **`LinkedTransferQueue<E>`** — это **высокопроизводительная неблокирующая очередь**
> 💡Реализует интерфейс `TransferQueue<E>`, который расширяет `BlockingQueue<E>`.
> 
> ⭐Она основана на **связанных узлах** (аналогично `LinkedBlockingQueue`), но в отличие от неё, `поддерживает` `немедленную передачу элементов` `между потоками` через специальные методы `transfer()` и `tryTransfer()`.

🧠Главная идея:  
`LinkedTransferQueue` объединяет свойства
- **очереди с буфером** (`LinkedBlockingQueue`) и 
- **очереди прямого обмена** (`SynchronousQueue`) — элемент либо передается напрямую ожидающему потребителю, либо помещается в очередь, если потребителя нет.

## Конструкторы
```java
// Создает пустую очередь для передачи элементов между потоками
public LinkedTransferQueue()

// Создает очередь, содержащую элементы из указанной коллекции
// c — коллекция, элементы которой будут добавлены в очередь
public LinkedTransferQueue(Collection<? extends E> c)
```

## Методы
Класс реализует все методы `BlockingQueue` и добавляет **новые методы передачи (`transfer`)** из интерфейса `TransferQueue`.  

Он не имеет ограничений по размеру (неограниченная очередь) и использует **немодифицируемую неблокирующую структуру (CAS)**.

### Добавление элементов
```java
// Добавляет элемент в очередь (не блокируется, всегда возвращает true)
public boolean add(E e)

// Добавляет элемент в очередь (эквивалент add(e))
public boolean offer(E e)

// Добавляет элемент, ожидая не более указанного времени (обычно немедленно возвращает true)
public boolean offer(E e, long timeout, TimeUnit unit)

// Блокирующе вставляет элемент (эквивалент offer(e))
public void put(E e)
```

### Извлечение элементов
```java
// Извлекает и удаляет элемент из головы очереди, ожидая, если очередь пуста
public E take() throws InterruptedException

// Извлекает и удаляет элемент, возвращая null, если очередь пуста
public E poll()

// Извлекает и удаляет элемент, ожидая заданное время, если очередь пуста
public E poll(long timeout, TimeUnit unit) throws InterruptedException

// Возвращает элемент из головы очереди, не удаляя его
public E peek()
```

### Методы передачи (УНИКАЛЬНЫЕ для TransferQueue)
```java
// Передает элемент другому потоку, ожидая, пока получатель его не примет
// Если нет ожидающего потребителя, вызывающий поток блокируется
public void transfer(E e) throws InterruptedException

// Пытается передать элемент немедленно, если есть ожидающий потребитель
// Возвращает true, если элемент был передан; иначе false
public boolean tryTransfer(E e)

// Пытается передать элемент в течение заданного времени
// Возвращает true, если передача состоялась; иначе false
public boolean tryTransfer(E e, long timeout, TimeUnit unit) throws InterruptedException


// Проверяет, есть ли хотя бы один поток, ожидающий получение элемента
public boolean hasWaitingConsumer()

// Возвращает количество ожидающих потребителей
public int getWaitingConsumerCount()
```

### Управление состоянием и итерация
```java
// Возвращает количество элементов в очереди (приблизительное значение)
public int size()

// Возвращает true, если очередь пуста
public boolean isEmpty()

// Удаляет все элементы из очереди
public void clear()

// Возвращает итератор по элементам (итератор слабосогласованный)
public Iterator<E> iterator()

// Возвращает массив элементов очереди
public Object[] toArray()

// Возвращает массив указанного типа с элементами очереди
public <T> T[] toArray(T[] a)
```

### Массовые операции
```java
// Перемещает все доступные элементы очереди в указанную коллекцию
public int drainTo(Collection<? super E> c)

// Перемещает не более maxElements элементов
public int drainTo(Collection<? super E> c, int maxElements)
```

## 💡 Особенности и подводные камни 💡
1️⃣ ⭐⭐**Неблокирующая архитектура (Lock-Free)** — основана на атомарных операциях CAS, а не на `ReentrantLock`.  

2️⃣ 💡**Комбинирует поведение SynchronousQueue и LinkedBlockingQueue** —  
если есть потребитель, элемент передается напрямую; иначе — добавляется в очередь.  

3️⃣ ❗❗❗**Метод `transfer()` блокирует поток**, если нет ожидающего потребителя (в отличие от обычного `put()`). 

4️⃣ ❗❗❗**Метод `tryTransfer()` позволяет немедленно проверить, есть ли ожидающий потребитель.**  

5️⃣ **Неограниченная очередь** — может расти бесконечно, если производители работают быстрее потребителей.  

6️⃣ ❗❗❗**Итератор слабосогласованный (weakly consistent)** — безопасен в многопоточности, но не гарантирует точную моментальную консистентность.  

7️⃣ **Методы `size()` и `getWaitingConsumerCount()` возвращают приблизительные значения**.  

8️⃣ **Высокая производительность** при большом количестве одновременно работающих потоков.

## 💎 Best Practices 💎

✅ 🧠Используйте `LinkedTransferQueue`, если вам нужна **максимальная производительность** при обмене между множеством потоков.  

✅ Применяйте `transfer()` для **прямой синхронной передачи** данных между потоками.  

✅ Используйте `tryTransfer()` для **асинхронной передачи**, если передача не должна блокироваться.  

✅ Подходит для систем с **динамическим балансом нагрузки** — задачи не хранятся зря, а сразу передаются свободным потокам.  

✅ Следите за ростом очереди — при большом дисбалансе между производителями и потребителями она может расти неограниченно.  

✅ Не используйте `fair` — этот параметр отсутствует, структура и так оптимизирована под конкурентный доступ.
