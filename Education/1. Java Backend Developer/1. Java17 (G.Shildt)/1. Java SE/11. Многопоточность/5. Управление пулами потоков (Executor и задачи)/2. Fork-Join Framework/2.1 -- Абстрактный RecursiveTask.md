
> [!NOTE] `RecursiveTask<V>` — это **абстрактный класс**, наследующий `ForkJoinTask<V>`.
> Он представляет собой задачу, которая `ВОЗВРАЩАЕТ РЕЗУЛЬТАТ` вычисления и используется в рамках **Fork/Join Framework**.

Типичный сценарий — реализация **алгоритма «разделяй и властвуй» (divide and conquer)**:  
задача разбивается на подзадачи (`fork`), каждая выполняется параллельно, а их результаты объединяются (`join`).

## Конструктор - АБСТРАКТНЫЙ КЛАСС
💬 Класс является **абстрактным**, поэтому напрямую экземпляры создавать нельзя.  
==Необходимо создать подкласс== и реализовать метод `compute()`.

## Методы
Поскольку `RecursiveTask` наследует все методы `ForkJoinTask`, здесь описан только его **собственный и ключевой метод**, который требуется переопределить.

### Основной метод вычислений
```java
// Метод, в котором реализуется логика задачи
// Должен возвращать результат типа V
protected abstract V compute();
```
💬 Именно этот метод определяет, **что делает задача**.  
Обычно внутри него:
1. Проверяется размер задачи.
2. 💡Если задача достаточно мала — вычисляется напрямую.
    
3. Иначе ==делится на подзадачи== (через `fork()` и `join()`).

### Унаследованные ключевые методы от ForkJoinTask

```java
// Отправить задачу на выполнение в пуле
public final ForkJoinTask<V> fork()

// Дождаться результата выполнения
public final V join()

// Выполнить задачу синхронно
public final V invoke()
```
💬 Эти методы используются в связке для организации параллельной обработки:
- `fork()` — запустить подзадачу;
- `join()` — дождаться результата;
    
- `invoke()` — запустить задачу синхронно в текущем потоке.

---
## 💡 **Особенности и подводные камни** 💡

1️⃣ `RecursiveTask` — это **ForkJoinTask с возвращаемым значением** (в отличие от `RecursiveAction`, который ничего не возвращает).  

2️⃣ 💡Используется только с **ForkJoinPool**, нельзя запускать напрямую через `Thread`.  

3️⃣ 💡Важно **сбалансировать** использование `fork()` и `join()` — иначе можно получить блокировки.  

4️⃣ ❗❗❗Если подзадача завершилась исключением, вызов `join()` пробросит `RuntimeException`.  

5️⃣ ⭐⭐⭐`compute()` должен быть **==чистым вычислением==** — БЕЗ БЛОКИРУЮЩИХ операций (I/O, sleep и т. д.).  

6️⃣ Часто используется **пороговое значение (`THRESHOLD`)** для определения, когда задачу уже не нужно делить дальше.

---

## 💎 **Best Practices** 💎

✅ Делай задачи **мелкими, но не слишком** — это ключ к эффективности Fork/Join.  

✅ Всегда переопредели `compute()` — других обязательных методов нет.  

✅ Используй `fork()` для распараллеливания и `join()` для объединения результатов.  

✅ Не забывай про **последовательное вычисление** в базовом случае (иначе рекурсия бесконечна).  

✅ Для запуска задачи используй `ForkJoinPool.invoke()` или `ForkJoinPool.commonPool().invoke()`.

## Пример
```java
import java.util.concurrent.RecursiveTask;
import java.util.concurrent.ForkJoinPool;

// Пример: рекурсивное суммирование массива
class SumTask extends RecursiveTask<Long> {
    private final long[] array;
    private final int start, end;
    private static final int THRESHOLD = 10_000; // Порог деления задачи

    public SumTask(long[] array, int start, int end) {
        this.array = array;
        this.start = start;
        this.end = end;
    }

	// ⭐⭐РАЗДЕЛЕНИЕ ЗАДАЧИ⭐⭐
    @Override
    protected Long compute() {
        int length = end - start;

        // Если задача маленькая — считаем напрямую
        if (length <= THRESHOLD) {
            long sum = 0;
            for (int i = start; i < end; i++)
                sum += array[i];
            return sum;
        }

        // Иначе делим задачу пополам
        int mid = start + length / 2;
        SumTask left = new SumTask(array, start, mid);
        SumTask right = new SumTask(array, mid, end);

        // Распараллеливаем: левую часть — в пул, правую — в текущем потоке
        left.fork();
        long rightResult = right.compute(); // текущий поток
        long leftResult = left.join();      // ждем левую часть

        // Объединяем результаты
        return leftResult + rightResult;
    }
}

public class RecursiveTaskExample {
    public static void main(String[] args) {
        long[] numbers = new long[100_000];
        for (int i = 0; i < numbers.length; i++)
            numbers[i] = i;

        ForkJoinPool pool = new ForkJoinPool();
        SumTask task = new SumTask(numbers, 0, numbers.length);

        long result = pool.invoke(task);
        System.out.println("Сумма элементов: " + result);
    }
}
```
