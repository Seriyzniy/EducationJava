
> [!NOTE] `CountedCompleter` — это **абстрактный класс**, наследующий `ForkJoinTask`, 
> Предназначенный `для реализации` `АСИНХРОННЫХ задач с зависимостями`, где результат может быть необязателен, а завершение задачи зависит от **счётчика незавершённых подзадач** (_pending count_).

В отличие от `RecursiveTask` и `RecursiveAction`, `CountedCompleter` ==НЕ требует возвращаемого значения== и ==НЕ полагается на вызов== `join()` 
— 
вместо этого используется ==механизм автоматического завершения== при достижении нулевого счётчика.

Используется в случаях, когда задачи образуют **ориентированный ациклический граф (DAG)** зависимостей или когда требуется **динамическая декомпозиция** без ручного ожидания подзадач.

## Конструкторы - АБСТРАКТНЫЙ КЛАСС
```java
// Создает CountedCompleter без родителя и с нулевым счётчиком незавершённых задач.
protected CountedCompleter()

// Создает CountedCompleter с указанным родителем.
// completer — родительская задача, которая завершится после всех своих подзадач.
protected CountedCompleter(CountedCompleter<?> completer)

// Создает CountedCompleter с указанным родителем и начальными незавершёнными задачами.
// completer — родительская задача
// pendingCount — количество подзадач, которые должны завершиться, прежде чем завершится эта
protected CountedCompleter(CountedCompleter<?> completer, int pendingCount)
```

## Методы
### Основные методы управления выполнением
```java
// 💡Абстрактный метод, который содержит основную логику выполнения задачи.
public abstract void compute()

// 💡Вызывается при завершении задачи. Можно переопределить для действий "после завершения".
public void onCompletion(CountedCompleter<?> caller)

// Вызывается, если задача завершается с исключением. Можно переопределить.
public boolean onExceptionalCompletion(
	Throwable ex, 
	CountedCompleter<?> caller)
```

### Управление счётчиком зависимостей (pending count)
```java
// Возвращает текущее количество незавершённых подзадач.
public final int getPendingCount()

// Устанавливает количество незавершённых подзадач.
public final void setPendingCount(int count)



// Уменьшает счётчик незавершённых подзадач на 1.
public final void decrementPendingCountUnlessZero()

// Добавляет указанное значение к счётчику (может быть отрицательным).
public final void addToPendingCount(int delta)



// Если счётчик достиг нуля — вызывает tryComplete().
public final boolean compareAndSetPendingCount(int expected, int count)
```

### Управление завершением задач
```java
// ⭐Отмечает задачу как завершённую, если pendingCount == 0.
// Иначе — уменьшает счётчик и завершает при достижении 0.
public final void tryComplete()

// Вызывает tryComplete() у всех родителей по цепочке completer-ов.
public final void propagateCompletion()

// Завершает задачу без проверки зависимостей.
public final void complete(T rawResult)

// Принудительно помечает задачу как завершённую с исключением.
public final void completeExceptionally(Throwable ex)
```

### Вспомогательные методы
```java
// Возвращает родительскую задачу (completer).
public final CountedCompleter<?> getCompleter()

// Возвращает корневую задачу (самую верхнюю в цепочке completer-ов).
public final CountedCompleter<?> getRoot()

// Возвращает результат (по умолчанию null). Можно переопределить.
public T getRawResult()

// Устанавливает результат (по умолчанию ничего не делает).
protected void setRawResult(T value)
```

## 💡 **Особенности и подводные камни** 💡

1️⃣ `CountedCompleter` не требует вызова `join()` — завершение контролируется автоматическим подсчётом зависимостей.  

2️⃣ 💡Идеален для **деревьев задач**, где каждая ветвь может порождать подзадачи, а завершение одной ветви не зависит от других напрямую.  

3️⃣ Ошибки в управлении `pendingCount` могут привести к "вечным" задачам, которые никогда не завершатся.  

4️⃣ Методы `tryComplete()` и `propagateCompletion()` — ключевые: неправильное использование приведёт к дедлокам или преждевременному завершению.  

5️⃣ Если нужно вернуть значение — лучше использовать `RecursiveTask`; если нет — `CountedCompleter` даёт больше гибкости.

---

## 💎 **Best Practices** 💎
✅ 💡 Вызывайте `addToPendingCount(n)` **до** разбиения задачи на подзадачи.  

✅ Каждая подзадача должна в конце вызывать `tryComplete()` для уведомления родителя.  

✅ ⭐⭐Используйте `onCompletion()` для финальной обработки результатов после завершения всех подзадач.  

✅ Не вызывайте `join()` — он не нужен и может нарушить работу механизма завершения.  

✅ Если задача не имеет зависимостей — используйте `tryComplete()` напрямую.

## Пример
Пример: параллельное суммирование массива с использованием `CountedCompleter`.
```java
import java.util.concurrent.CountedCompleter;
import java.util.concurrent.ForkJoinPool;

public class MyMain {
    static class SumTask extends CountedCompleter<Long> {
        private static final int THRESHOLD = 50;
        private final long[] array;
        private final int start, end;
        private long result;
        private SumTask left, right; // храним ссылки на подзадачи

        SumTask(CountedCompleter<?> parent, long[] array, int start, int end) {
            super(parent);
            this.array = array;
            this.start = start;
            this.end = end;
        }

        @Override
        public void compute() {
            if (end - start <= THRESHOLD) {
                // Базовый случай - вычисляем сумму напрямую
                long sum = 0;
                for (int i = start; i < end; i++) {
                    sum += array[i];
                }
                result = sum;
                tryComplete();
            } else {
                // Рекурсивный случай - разбиваем на подзадачи
                int mid = (start + end) / 2;
                left = new SumTask(this, array, start, mid);
                right = new SumTask(this, array, mid, end);
                
                setPendingCount(1); // будем ждать completion одной подзадачи
                right.fork();       // запускаем правую задачу асинхронно
                left.compute();     // выполняем левую задачу синхронно
            }
        }

        @Override
        public void onCompletion(CountedCompleter<?> caller) {
            // Агрегируем результаты только если это родительская задача
            if (left != null && right != null) {
                result = left.result + right.result;
            }
        }

        @Override
        public Long getRawResult() {
            return result;
        }
    }

    public static void main(String[] args) {
        long[] array = new long[100];
        for (int i = 0; i < array.length; i++) {
            array[i] = i + 1;
        }

        try (ForkJoinPool pool = new ForkJoinPool()) {
            SumTask root = new SumTask(null, array, 0, array.length);
            pool.invoke(root);
            System.out.println("Sum = " + root.getRawResult()); // Должно быть 5050
        }
    }
}
```
