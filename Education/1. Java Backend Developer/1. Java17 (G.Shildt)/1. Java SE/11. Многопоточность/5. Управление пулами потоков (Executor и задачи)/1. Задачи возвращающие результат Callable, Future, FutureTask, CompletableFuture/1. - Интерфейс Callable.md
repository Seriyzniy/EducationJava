
> [!NOTE] Интерфейс **`Callable<V>`** — это функциональный интерфейс из пакета `java.util.concurrent`
> Gредназначенный для `представления задачи`, которая `возвращает результат` и может `выбрасывать исключения`.
> 
> ⭐Он похож на `Runnable`, но более гибкий: `Runnable` не возвращает значения и не умеет выбрасывать checked-исключения.

Может:
- 🛑 **возвращать результат** 🛑
    
- **выбрасывать проверяемое исключение (checked exception)**

⭐Используется в **потоках, пуле потоков (`ExecutorService`) и в классах вроде `FutureTask`** для выполнения задач с результатом.

**Не реализует никаких интерфейсов** — это базовый функциональный интерфейс с единственным методом `call()`.

---
## Интерфейс
```java
@FunctionalInterface
public interface Callable<V> {

    // Выполняет задачу и возвращает результат типа V.
    // Может выбросить любое исключение (в отличие от Runnable).
    V call() throws Exception;
}
```

---
## 💡 **Особенности и подводные камни** 💡

1️⃣ ⭐`Callable` — **функциональный интерфейс**, значит, можно использовать **лямбда-выражения** и **method references**.  

2️⃣ ⭐В отличие от `Runnable`, метод `call()` **возвращает значение** и **может выбрасывать checked-исключения**.  

3️⃣ ❗❗❗Используется **вместе с `Future`/`FutureTask`/`ExecutorService`**, а не напрямую через `Thread`.  

4️⃣ Метод `call()` вызывается **внутри механизма исполнения (например, пул потоков)**, а не вручную.  

5️⃣ 💡Если `call()` выбрасывает исключение, то оно **заворачивается в `ExecutionException`**, когда ты вызываешь `Future.get()`.

---

## 💎 **Best Practices** 💎

✅ 💡Используй `Callable` вместо `Runnable`, если нужно получить результат из задачи.  

✅ 💡Оборачивай `Callable` в `FutureTask` или передавай его в `ExecutorService.submit()`.  

✅ 💡💡При работе с checked-исключениями используй `try/catch` внутри `call()`, чтобы управлять поведением.  

✅ ⭐⭐Для цепочек асинхронных вычислений переходи на `CompletableFuture`, который поддерживает композицию `Callable`-подобных действий.  

✅ Не вызывай `call()` напрямую — этот метод должен выполняться _в контексте исполнителя_, чтобы не терять смысл многопоточности.

---
## Пример
```java
import java.util.concurrent.*;

public class CallableExample {
    public static void main(String[] args) throws Exception {
        // Создаем Callable-задачу, которая возвращает значение
        Callable<Integer> task = () -> {
            System.out.println("Вычисляю...");
            Thread.sleep(1000);
            return 42;
        };

        // Используем ExecutorService для выполнения Callable
        ExecutorService executor = Executors.newSingleThreadExecutor();

        // Отправляем задачу на выполнение
        Future<Integer> future = executor.submit(task);

        // Получаем результат (блокирует поток до завершения)
        System.out.println("Результат: " + future.get());

        executor.shutdown();
    }
}
```
