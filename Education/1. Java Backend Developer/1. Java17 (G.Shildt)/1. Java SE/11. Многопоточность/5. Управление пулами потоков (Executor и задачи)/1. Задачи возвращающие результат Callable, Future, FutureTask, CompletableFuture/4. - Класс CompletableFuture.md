
> [!NOTE] `CompletableFuture<V>` — это `Future нового поколения`
> - Позволяет **`строить цепочки` асинхронных вычислений**,
> - Умеет работать без блокировки (`thenApply`, `thenAccept` и т.д.),
> - Поддерживает комбинирование нескольких задач и обработку ошибок

Он появился в Java 8 и стал основой для "==современного==" ==способа писать асинхронный код== без ручной работы с потоками.

Реализует интерфейсы:
- **`Future<V>`** — для совместимости со стандартным API асинхронных задач;
- **`CompletionStage<V>`** — определяет модель построения цепочек асинхронных операций.

---
## Конструкторы
```java
// Создает новый, изначально незавершённый CompletableFuture.
// Результат будет задан позже через complete() или completeExceptionally().
public CompletableFuture()
```
Обычно `CompletableFuture` создают не напрямую, а через **фабричные методы**, которые сразу запускают задачу.

## Фабричные методы
```java
// Создает CompletableFuture, уже завершённый с заданным значением.
public static <U> CompletableFuture<U> completedFuture(U value)
```

## Методы

### Получение результата
⛔Методы для синхронного ожидания завершения (использовать с осторожностью)
```java
// Возвращает результат (блокирует, пока задача не завершится).
public V get() throws InterruptedException, ExecutionException

// Возвращает результат с таймаутом.
public V get(long timeout, TimeUnit unit)
    throws InterruptedException, ExecutionException, TimeoutException

// Возвращает результат без checked-исключений.
public V join()

// Возвращает значение, если завершено, иначе null.
public V getNow(V valueIfAbsent)
```

### 1️⃣Создание и запуск асинхронных задач
```java
public static <U> CompletableFuture<U> supplyAsync(Supplier<U> supplier)
// Выполняет асинхронную задачу, возвращающую результат.
// supplier — лямбда, возвращающая значение. Использует общий ForkJoinPool.

public static <U> CompletableFuture<U> supplyAsync(
	Supplier<U> supplier, 
	Executor executor)
// То же, но выполняет задачу в заданном Executor’е.


// Выполняет асинхронную задачу без возвращаемого значения.
public static CompletableFuture<Void> runAsync(Runnable runnable)


// То же, но с явным указанием пула потоков.
public static CompletableFuture<Void> runAsync(
	Runnable runnable, 
	Executor executor)
```

#### Примеры
supplyAsync
```java
CompletableFuture<Integer> f = CompletableFuture.supplyAsync(() -> 42);
System.out.println(f.get()); // 42
```

runAsync
```java
CompletableFuture<Void> f = CompletableFuture.runAsync(() ->
	System.out.println("Выполняю в отдельном потоке")
);
f.join(); // ждем завершения
```

### 2️⃣Последующие действия (цепочки)
```java
// Применяет функцию к результату и возвращает новый CompletableFuture.
public <U> CompletableFuture<U> thenApply(
	Function<? super V, ? extends U> fn)

// Как thenApply, но выполняется асинхронно.
public <U> CompletableFuture<U> thenApplyAsync(
	Function<? super V, ? extends U> fn)

// Асинхронное выполнение в указанном Executor’е.
public <U> CompletableFuture<U> thenApplyAsync(
	Function<? super V, ? extends U> fn, 
	Executor executor)



// Продолжает цепочку БЕЗ ВОЗВРАТА ЗНАЧЕНИЯ.
public CompletableFuture<Void> thenAccept(
	Consumer<? super V> action)

// Как thenAccept, но асинхронно.
public CompletableFuture<Void> thenAcceptAsync(
	Consumer<? super V> action)



// Выполняет Runnable после завершения задачи (без использования результата).
public CompletableFuture<Void> thenRun(Runnable action)

```

#### Примеры
thenApply
```java
CompletableFuture<Integer> f = CompletableFuture.supplyAsync(() -> 5)
	.thenApply(x -> x * 2);

System.out.println(f.join()); // 10
```

thenAccept
```java
CompletableFuture.supplyAsync(() -> "Hello")
	.thenAccept(s -> System.out.println("Получено: " + s));
```

thenRun
```java
CompletableFuture.supplyAsync(() -> 42)
	.thenRun(() -> System.out.println("Задача завершена!"));
```

### 3️⃣Комбинирование зависимых **задач**
```java
/*
Используется, когда одна асинхронная задача зависит от другой (чтобы не было `CompletableFuture<CompletableFuture<T>>`).

Плоское связывание (аналог flatMap). Используется для последовательных зависимых задач.
*/
public <U> CompletableFuture<U> thenCompose(
	Function<? super V, ? extends CompletionStage<U>> fn)


// Ждёт 2 `CompletableFuture`, объединяет их результаты функцией.
public <U, V> CompletableFuture<V> thenCombine(
        CompletionStage<? extends U> other,
        BiFunction<? super T, ? super U, ? extends V> fn)



 
// Возвращает CompletableFuture, завершающийся, когда ВСЕ переданные ЗАДАЧИ ЗАВЕРШАТСЯ.
public static CompletableFuture<Void> allOf(
	CompletableFuture<?>... cfs)

// Возвращает CompletableFuture, завершающийся, когда ЗЩАВИРШИТСЯ ЛЮБАЯ из переданных ЗАДЛАЧА.
public static CompletableFuture<Object> anyOf(
	CompletableFuture<?>... cfs)
```

#### Примеры
thenCombine
```java
CompletableFuture<Integer> a = CompletableFuture.supplyAsync(() -> 5);
CompletableFuture<Integer> b = CompletableFuture.supplyAsync(() -> 10);

CompletableFuture<Integer> result = a.thenCombine(b, (x, y) -> x + y);

System.out.println(result.join()); // 15
```

thenCompose
```java
CompletableFuture<Integer> f = CompletableFuture.supplyAsync(() -> 5)
	.thenCompose(x -> CompletableFuture.supplyAsync(() -> x * 2));

System.out.println(f.join()); // 10
```

allOf
```java
CompletableFuture<Void> all = CompletableFuture.allOf(
	CompletableFuture.runAsync(() -> sleep(1)),
	CompletableFuture.runAsync(() -> sleep(2))
);
all.join(); // ждём обе задачи
```

anyOf
```java
CompletableFuture<Object> any = CompletableFuture.anyOf(
	CompletableFuture.supplyAsync(() -> "A"),
	CompletableFuture.supplyAsync(() -> "B")
);
System.out.println(any.join()); // A или B
```

### 4️⃣Обработка завершения (успех/ошибка)
```java
// Выполняет действие при завершении — независимо от того, успешном или с ошибкой.
public CompletableFuture<V> whenComplete(
	BiConsumer<? super V, ? super Throwable> action)


// Позволяет вернуть новый результат или обработать ошибку, если она была.
public <U> CompletableFuture<U> handle(
	BiFunction<? super V, Throwable, ? extends U> fn)


// Возвращает новый CompletableFuture с результатом функции в случае исключения.
public CompletableFuture<V> exceptionally(
	Function<Throwable, ? extends V> fn)
```

#### Примеры
exceptionally
```java
CompletableFuture<Integer> f = CompletableFuture
	.supplyAsync(() -> 1 / 0) // ошибка
	.exceptionally(ex -> {
		System.out.println("Ошибка: " + ex);
		return -1;
	});

System.out.println(f.join()); // -1
```

handle
```java
CompletableFuture<Integer> f = CompletableFuture
    .supplyAsync(() -> 1 / 0)
    .handle((res, ex) -> (ex == null) ? res : -1);

System.out.println(f.join()); // -1
```

### 5️⃣Завершение задачи вручную
```java
// Устанавливает результат выполнения, если задача ещё не завершена.
public boolean complete(V value)

// Завершает задачу с исключением.
public boolean completeExceptionally(Throwable ex)

// Принудительно завершает задачу со значением или исключением.
public void obtrudeValue(V value)
public void obtrudeException(Throwable ex)
```

#### Примеры
complete
```java
CompletableFuture<String> f = new CompletableFuture<>();
f.complete("Hello вручную");
System.out.println(f.join()); // Hello вручную
```

completeExceptionally
```java
CompletableFuture<String> f = new CompletableFuture<>();
f.completeExceptionally(new RuntimeException("Ошибка!"));

try {
    f.join();
} catch (Exception e) {
    System.out.println("Поймали: " + e.getMessage());
}
```

## 💥 ВАЖНО про CompletableFuture 💥
`thenApply` и `thenAccept` НЕ блокируют основной поток
- Когда ты вызываешь:
```java
CompletableFuture.supplyAsync(() -> 42)
    .thenApply(x -> x * 2)
    .thenAccept(System.out::println);
```
 — создаётся **цепочка действий**.
- Основной поток (например, `main`) не ждёт завершения этих операций, он сразу продолжает выполнение дальше.
    
- `thenApply` и `thenAccept` просто **регистрируют колбэки**, которые будут вызваны, когда предыдущее будущее завершится.
### Кто выполняет `thenApply` и `thenAccept`
- По умолчанию, они выполняются в **том же потоке**, который завершил предыдущее будущее (`ForkJoinPool.commonPool` или твой кастомный `Executor`).
    
- То есть если `supplyAsync` работал в пуле потоков, то и `thenApply`/`thenAccept` будут вызваны там же.

```java
public class Demo {
    public static void main(String[] args) throws InterruptedException {
        CompletableFuture.supplyAsync(() -> {
            sleep(1000);
            return "Данные";
        }).thenApply(data -> {
            System.out.println("Трансформация: " + data);
            return data.toUpperCase();
        }).thenAccept(result -> {
            System.out.println("Финальный результат: " + result);
        });

        System.out.println("Основной поток продолжает работу...");
        Thread.sleep(2000); // чтобы программа не завершилась раньше
    }

    private static void sleep(long ms) {
        try { Thread.sleep(ms); } catch (InterruptedException ignored) {}
    }
}
```

### Когда будет блокировка?
- Только если ты сам вызовешь `.get()` или `.join()`.
    
- Они останавливают текущий поток до получения результата.
