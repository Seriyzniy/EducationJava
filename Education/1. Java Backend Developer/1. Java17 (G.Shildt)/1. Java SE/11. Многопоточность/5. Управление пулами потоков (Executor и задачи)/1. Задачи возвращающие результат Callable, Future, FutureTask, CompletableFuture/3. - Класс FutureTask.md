
> [!NOTE] Класс **`FutureTask<V>`** - Представление `отложенного результата` асинхронной задачи, который можно получить позже. 
> 
> `FutureTask<V>` — это ==**класс-адаптер**==, который объединяет в себе:
> -  **`Runnable`** → можно запускать в `Thread` или через `Executor`.
> - **`Future<V>`** → можно получить результат, отменить, проверить состояние.

`FutureTask` часто применяется с `ExecutorService`, но может быть запущен и вручную (через `Thread`).

Реализует интерфейсы:
- **`RunnableFuture<V>`**, который расширяет:
    - **`Runnable`** — для возможности запуска в потоке.
    - **`Future<V>`** — для управления асинхронной задачей (получение результата, отмена и т.д.).

🧠Иными словами: _это задача + контейнер для её результата одновременно_.

## Как работает
- В конструктор передаётся `Callable<V>` или `Runnable` (с результатом `V`).
- Когда задача выполняется, результат или исключение сохраняются внутри.
- Через методы `Future` (`get`, `cancel`, `isDone`) можно получить состояние/результат.

## Конструкторы
```java
// Создает задачу, которая будет вычислять результат, возвращаемый заданным Callable.
// Callable — это функция, возвращающая значение.
public FutureTask(Callable<V> callable)


// Создает задачу, которая при выполнении Runnable вернет заданное значение result.
// Полезно, если нужно обернуть Runnable, чтобы потом получить фиксированный результат.
public FutureTask(Runnable runnable, V result)
```

## Методы
### Управление состоянием выполнения
```java
// Запускает выполнение задачи (если она еще не была запущена).
// Метод вызывается автоматически, если FutureTask передан в ExecutorService.
public void run()

// Отменяет выполнение задачи. 
// Если mayInterruptIfRunning == true — поток выполнения может быть прерван.
// Возвращает true, если отмена была успешна.
public boolean cancel(boolean mayInterruptIfRunning)

// Проверяет, была ли задача отменена.
public boolean isCancelled()

// Проверяет, завершено ли выполнение задачи (успешно, с ошибкой или отменой).
public boolean isDone()
```

### Получение результата
```java
// Возвращает результат вычисления, блокируя вызывающий поток до завершения.
// Может бросить исключения:
// - CancellationException, если задача была отменена;
// - ExecutionException, если при выполнении произошла ошибка;
// - InterruptedException, если поток ожидания был прерван.
public V get() throws InterruptedException, ExecutionException


// Возвращает результат вычисления, но ждёт не дольше заданного времени.
// Если время истекло, бросает TimeoutException.
public V get(long timeout, TimeUnit unit)
    throws InterruptedException, ExecutionException, TimeoutException
```

### Вспомогательные методы
```java
// Защищённый метод, вызывается при успешном завершении задачи.
// Можно переопределить для выполнения действия после завершения.
protected void done()

// Защищённый метод, возвращает результат (если задача завершена успешно).
// Обычно используется в наследниках FutureTask.
protected V getRawResult()

// Защищённый метод, используется для установки результата вручную.
protected void set(V v)

// Защищённый метод, используется для завершения задачи с ошибкой.
protected void setException(Throwable t)
```

## Отличия от Future
| Критерий          | Future                          | FutureTask                           |
| ----------------- | ------------------------------- | ------------------------------------ |
| **Тип**           | Интерфейс                       | Конкретный класс                     |
| **Создание**      | Через `executor.submit()`       | `new FutureTask<>(callable)`         |
| **Запуск**        | Только через Executor           | Executor или напрямую в Thread       |
| **Гибкость**      | Ограничена Executor'ом          | Высокая — разные способы запуска     |
| **Runnable**      | Нет                             | Да, реализует Runnable               |
| **Использование** | Для получения результатов задач | Когда нужен контроль над выполнением |
|                   |                                 |                                      |
## Когда использовать FutureTask
- Когда нужна **универсальность**: один объект и как `Runnable`, и как `Future`.
    
- Для ручного контроля запуска (можно стартовать позже, не сразу в `submit()`).
    
- Для ==переиспользования== — `FutureTask` ==можно хранить и отдавать РЕЗУЛЬТАТ ВЫПОЛНЕНИЯ другим== компонентам, которые потом вызовут `get()`.

## 💡 **Особенности и подводные камни** 💡

1️⃣ **==Одноразовость==** — `FutureTask` можно выполнить только один раз. Повторный вызов `run()` не приведёт к повторному вычислению.  

```java
FutureTask<Integer> ftask = new FutureTask<>(new Callable<Integer>() {
	@Override
	public Integer call() throws Exception {
		Random rand = new Random();
		System.out.println("Running task");
		
		return rand.nextInt(1000);
	}
	
});

Thread t = new Thread(ftask);
t.start();

System.out.println(ftask.get());


Thread t1 = new Thread(ftask);
t1.start();

System.out.println(ftask.get());

//Вывод
Running task
678
678
```

2️⃣ **Состояния задачи:**
- _NEW_ — задача не выполнена;
    
- _COMPLETING_ → _NORMAL_ — завершена успешно;
    
- _EXCEPTIONAL_ — завершена с ошибкой;
    
- _CANCELLED_ / _INTERRUPTING_ / _INTERRUPTED_ — отменена.  

3️⃣ **Потокобезопасен** — можно безопасно вызывать методы из разных потоков.  

4️⃣ **Синхронизация внутри** — реализация использует атомарные операции (CAS) и `LockSupport.park/unpark` для блокировок ожидания.  

5️⃣ **Отмена не всегда мгновенна** — если задача уже выполняется, отмена лишь ставит флаг и может прервать поток, но выполнение может продолжиться.  

6️⃣ **get() блокирует поток** — до завершения, поэтому при необходимости используйте версию с таймаутом.

---
## 💎 **Best Practices** 💎

✅ Используйте `FutureTask` для обертки `Callable`, если нужно вручную управлять выполнением и результатом.  

✅ 💡Передавайте `FutureTask` в `ExecutorService` для автоматического запуска.  

✅ Для периодических или отложенных задач лучше использовать `ScheduledExecutorService`.  

✅ Проверяйте `isDone()` или используйте `get(timeout)` вместо бесконечного `get()`, чтобы избежать зависаний.  

✅ При необходимости выполнять действия после завершения задачи — переопределите метод `done()`.

## Пример
```java
import java.util.concurrent.*;

public class FutureTaskExample {
    public static void main(String[] args) throws Exception {
        // Создаем Callable-задачу
        Callable<Integer> task = () -> {
            Thread.sleep(1000);
            return 42;
        };

        // Оборачиваем в FutureTask
        FutureTask<Integer> futureTask = new FutureTask<>(task);

        // Запускаем вручную в отдельном потоке
        Thread t = new Thread(futureTask);
        t.start();

        // Ожидаем результат
        System.out.println("Результат: " + futureTask.get());

        // Проверяем состояние
        System.out.println("Задача завершена? " + futureTask.isDone());
    }
}
```
