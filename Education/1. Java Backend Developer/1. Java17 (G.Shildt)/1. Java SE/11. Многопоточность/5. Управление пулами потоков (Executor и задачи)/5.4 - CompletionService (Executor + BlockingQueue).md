CompletionService — это довольно мощная штука, которая совмещает Executor и BlockingQueue. 

> [!NOTE] Позволяет получать результат выполнения потока НЕ по мере их запуска, а ПО МЕРЕ готовности результата любого из потоков 
> То есть если поток 1 стартанул раньше 2-го, но долго вычисляет результат, то при помощи `CopletionService` можно получить результат любого другого потока, который завершил выполнение
> 

Вы можете ==отправлять ему задачи для выполнения в виде экземпляров== Callable и ==использовать методы== подобные тем, что имеются у очередей - методы take и poll - ==для получения завершенных результатов==, упакованных в экземпляры интерфейса Future, по мере их доступности

# 📕1. Что такое `CompletionService`- Интерфейс
```java
public interface CompletionService<V> {
    Future<V> submit(Callable<V> task);
    Future<V> submit(Runnable task, V result);
    Future<V> take() throws InterruptedException;
    Future<V> poll();
    Future<V> poll(long timeout, TimeUnit unit) throws InterruptedException;
}
```
- ⭐⭐⭐Основная реализация: **`ExecutorCompletionService`**.⭐⭐⭐
    
- Внутри он использует:
    - `Executor` для выполнения задач.
    - `BlockingQueue<Future<V>>` для хранения завершённых задач.

***
## 📙Основные методы
##### 1. `submit(Callable<V> task)`
Отправляет задачу на выполнение

##### 2. `submit(Runnable task, V result)`
Отправляет Runnable задачу с указанием результата

##### 3. `take()`
Блокирует поток до появления завершенной задачи, возвращает Future

##### 4. `poll()`
Проверяет наличие завершенных задач без блокировки

##### 5. `poll(long timeout, TimeUnit unit)`
Ожидает завершенную задачу в течение указанного времени

***
# 📕2. Как работает
1. Ты отдаёшь задачи через `submit`.
2. Они выполняются в пуле потоков (`ExecutorService`).
    
3. Когда задача заканчивается — её `Future` автоматически кладётся во внутреннюю очередь.
    
4. Ты можешь забирать завершённые задачи через:
    - `take()` — блокирующий, ждёт пока появится результат.
        
    - `poll()` — неблокирующий, вернёт `null`, если результатов нет.
        
    - `poll(timeout, unit)` — ждёт ограниченное время.

🛑Таким образом, ==**обработка идёт в порядке завершения**==, а не в порядке запуска.

***
## Пример

### 🟠 Без `CompletionService` (обычный `ExecutorService`)
```java
ExecutorService executor = Executors.newFixedThreadPool(3);

List<Future<String>> futures = new ArrayList<>();
for (int i = 1; i <= 3; i++) {
    int id = i;
    futures.add(executor.submit(() -> {
        Thread.sleep(1000L * id); // разные задержки
        return "Task " + id;
    }));
}

// Обрабатываем результаты в порядке запуска
for (Future<String> f : futures) {
    System.out.println(f.get()); // блокируемся на каждом
}
executor.shutdown();
```
⛔ Минус: если `Task 3` закончит первой, ==мы всё равно будем ждать== `Task 1`, потом `Task 2`.

### 🟢 С `CompletionService`
```java
//🛑 Способ объявления 🛑
ExecutorService executor = Executors.newFixedThreadPool(3);
CompletionService<String> cs = new ExecutorCompletionService<>(executor);

// Отправляем задачи
for (int i = 1; i <= 3; i++) {
    int id = i;
    cs.submit(() -> {
        Thread.sleep(1000L * id);
        return "Task " + id;
    });
}

// Забираем результаты в порядке завершения
for (int i = 0; i < 3; i++) {
    Future<String> f = cs.take(); // ждём ближайший готовый
    System.out.println(f.get());
}
executor.shutdown();

```
