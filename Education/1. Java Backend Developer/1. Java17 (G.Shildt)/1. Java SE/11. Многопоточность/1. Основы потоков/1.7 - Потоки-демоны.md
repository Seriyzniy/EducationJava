**Daemon thread** — это поток, предназначенный для ==**фоновой работы**==, который ==**автоматически завершается при завершении всех пользовательских потоков**.==

> [!NOTE] 🛑Выполнение программы завершается, даже если есть потоки-демоны 🛑
> Они **не мешают завершению JVM**: как только **все обычные (недаемонские) потоки завершаются**, JVM автоматически завершает работу, даже если есть живые демоны.
> 
> ☣При создании нового потока внутри потока-демона, его статус наследуется

Чтобы сделать поток потоком-демоном используется метод `setDaemon()`
```java
Thread t = new Thread(() -> {
    while (true) {
        System.out.println("Фоновая задача...");
        try { Thread.sleep(1000); } catch (InterruptedException e) {}
    }
});
t.setDaemon(true); // Устанавливаем как daemon
t.start();
```
> [!NOTE] ⚠️ `setDaemon(true)` можно ==вызывать **только до запуска потока**== (`start()`), иначе — `IllegalThreadStateException`.
> 

После этого, когда поток `main` завершит свою работу, все потоки-демоны ТАКЖЕ ЗАВЕРШАТ работу.

***
# 💡 Подводные камни 💡

## 1. ❗ Потоки-демоны не завершают работу корректно

Если в демоне идёт запись в файл, лог или работа с БД, **JVM может завершить его на полуслове**.
```java
Thread t = new Thread(() -> {
    try (PrintWriter out = new PrintWriter("log.txt")) {
        out.println("Начинаем логирование");
        Thread.sleep(5000); // JVM может завершиться до этого
        out.println("Завершили логирование");
    } catch (Exception e) {}
});
t.setDaemon(true);
t.start();
```

## 2. ❗ Не используйте демоны для критичной логики
- Демоны не гарантируют завершения `finally` блоков.
    
- Не подходят для транзакций, сохранения данных, операций с сетью, безопасности.
    
## 3. ❗ Нельзя изменить daemon-флаг после `start()`

***
# 💡 Лучшие практики 💡

- ✅ Использовать демоны для **необязательных фоновых задач**.
    
- ❌ Не использовать демоны для **записи в БД, логов, критичных операций**.
    
- ✅ Использовать **`ExecutorService`** с явным управлением, если требуется надёжность.
    
- ✅ Проверять, завершилась ли работа демона, если это важно.
