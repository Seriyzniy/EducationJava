# 1. Как работают исключения в потоках

- ==Исключение всегда **живёт внутри того потока, где оно произошло**.==
    
- ==Оно **не может "проброситься" вверх в родительский поток**== через обычный `try/catch`, потому что у каждого потока **свой стек вызовов**.
    
- Родительский поток вообще может не знать, что у дочернего случилась ошибка — если только мы сами не сделаем механизм передачи.

***
# 2. Как можно уведомить "родителя"
Есть три основных способа:
## 1. **`UncaughtExceptionHandler`**

- Поток умирает → JVM вызывает `UncaughtExceptionHandler`.
    
- Это "глобальный сторож", который может логировать и реагировать.  
    Но! Он **не вернёт исключение в родительский поток** — только уведомит.

```java
Thread t = new Thread(() -> {
    throw new RuntimeException("Ошибка в потоке");
});
t.setUncaughtExceptionHandler((th, ex) ->
    System.out.println("Поймали в handler: " + ex.getMessage())
);
t.start();
```

## 2. **Через `Future` / `ExecutorService`**

- Самый надёжный способ.
    
- ==Исключение **запаковывается в `ExecutionException`** ==и возвращается в родительский поток, когда он вызывает `get()`.
```java
ExecutorService ex = Executors.newSingleThreadExecutor();

Future<Integer> f = ex.submit(() -> {
    throw new RuntimeException("Ошибка в задаче");
});

//main
try {
    f.get(); // бросит ExecutionException
} catch (ExecutionException e) {
    System.out.println("Поймано в main: " + e.getCause());
}
```
✅ Тут мы действительно "уведомляем" родителя через API.

## 3. **`CompletableFuture`**

- Более современный способ, с явной обработкой ошибок в цепочках.
    
- Ошибки не теряются и могут быть обработаны асинхронно.
```java
CompletableFuture<Void> cf = CompletableFuture.runAsync(() -> {
    throw new RuntimeException("Ошибка в потоке");
});

cf.exceptionally(ex -> {
    System.out.println("Ошибка перехвачена: " + ex);
    return null;
});
```
