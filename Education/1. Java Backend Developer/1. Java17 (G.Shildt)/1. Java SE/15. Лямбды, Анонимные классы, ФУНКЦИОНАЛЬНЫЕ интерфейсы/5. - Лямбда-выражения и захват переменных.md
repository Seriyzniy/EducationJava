Переменные, определенные в объемлющей области действия лямбда-выражения, доступны внутри лямбда-выражения

> [!NOTE] Что может использовать?
> Лямбда-выражение может задействовать 
> 1. `Переменную экземпляра` 
> 2. `Static переменную`, определенную в объемлющем классе. 
> 3. Имеет доступ к ссылке `this` (явно и неявно), которая ссылается на вызывающий экземпляр класса, включающего лямбда-выражение. (может вызывать методы объемлющего класса)

Тем не менее, когда в лямбда-выражении ==используется ЛОКАЛЬНАЯ переменная из его объемлющей области видимости==, то возникает **особая ситуация**, называемая `ЗАХВАТОМ ПЕРЕМЕННОЙ.`

Такая переменная становится - фактически финальной

> [!NOTE] `Фактически финальная переменная` -  переменная, `значение которой не меняется` после ее первого присваивания
> Явно объявлять такую переменную как final нет никакой необходимости, хотя ошибкой не будет

```java
interface MyFunc {
    int func(int n);
}

class VarCapture {
    public static void main(String[] args) {
        int num = 10; // фактически финальная переменная
        
        MyFunc myLambda = (n) -> {
            // ✅⭐Можно использовать, НО нельзя изменять
            int v = num + n;
            
            // ❌Ошибка!
            // num++; 
            
            return v;
        };
        
        // num = 9; // Тоже ошибка!
    }
}
```
Однако ==если бы значение== `num` ==изменилось== внутри лямбда-выражения или за его пределами, то переменная `num` утратит свой статус фактически финальной, ==что вызовет ошибку и программа не скомпилируется.==


> [!NOTE] ⭐Лямбда `может изменять переменные экземпляра`, НО `НЕ МОЖЕТ изменять ЛОКАЛЬНЫЕ`
> Важно подчеркнуть, что лямбда-выражение может использовать и модифицировать переменную экземпляра из вызывающего его класса. 
> 
> Оно просто не может работать с локальной переменной из своей объемлющей области видимости, если только эта переменная не является фактически финальной.

## Использование в качестве параметра лямбды ИМЯ ЛОКАЛЬНОЙ переменной
```java
String str = "Hello world!";

//❌ Ошибка компиляции
Consumer<String> c = str -> System.out.println(str); 
```
Но **в Java ==нельзя использовать имя параметра, совпадающее с именем локальной переменной== в охватывающей области**.  
→ То есть параметр `str` «затеняет» локальную переменную `str`, и компилятор запрещает это.
