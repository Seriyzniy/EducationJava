
> [!NOTE] `Function<T, R>`
> Функция, принимающая 
> - один аргумент типа `T` 
> - возвращающая результат типа `R`. 
> 
> Базовый функциональный интерфейс в Java.

## Абстрактный метод `R apply(T t)`
```java
// Применяет функцию к аргументу
R apply(T t);
```
## Интерфейс
```java
@FunctionalInterface
public interface Function<T, R> {
    // Применяет функцию к аргументу
    R apply(T t);

    // Создаёт композицию функций: сначала before, потом this
    default <V> Function<V, R> compose(Function<? super V, ? extends T> before);

    // Создаёт композицию функций: сначала this, потом after
    default <V> Function<T, V> andThen(Function<? super R, ? extends V> after);

    // Функция-идентичность: возвращает свой аргумент
    static <T> Function<T, T> identity();
}
```

## ✅ Best Practices ✅

- Используйте `Function.identity()` вместо лямбды `t -> t`.
    
- Для длинных цепочек лучше выносить шаги в отдельные функции для читаемости.


## Пример
```java
Function<String, Integer> length = String::length;
System.out.println(length.apply("Java")); // 4

Function<String, String> upper = String::toUpperCase;
Function<String, String> trim = String::trim;
Function<String, String> pipeline = trim.andThen(upper);

System.out.println(pipeline.apply("  hello ")); // "HELLO"

```
