Чтобы избежать **автобоксинга** (Integer → int, Double → double и т.д.), ввели специализированные интерфейсы.

- `IntFunction<R>` → принимает `int`, возвращает `R`.
    
- `ToIntFunction<T>` → принимает `T`, возвращает `int`.
    
- `IntUnaryOperator` → принимает `int`, возвращает `int`.
    
- `IntBinaryOperator` → принимает два `int`, возвращает `int`.
    
- Аналоги есть для `long`, `double`.

```java
IntFunction<String> intToStr = i -> "Value: " + i;
System.out.println(intToStr.apply(42)); // "Value: 42"

ToIntFunction<String> strLength = String::length;
System.out.println(strLength.applyAsInt("Java")); // 4

IntBinaryOperator sum = Integer::sum;
System.out.println(sum.applyAsInt(5, 7)); // 12
```
