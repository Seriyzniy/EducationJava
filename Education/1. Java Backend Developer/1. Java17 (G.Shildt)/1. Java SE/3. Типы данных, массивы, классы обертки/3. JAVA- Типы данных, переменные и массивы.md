# 📕1. Примитивные типы данных

| Тип       | Размер (байт)         | Минимум                          | Максимум                         | Особенности                                                                                       |     |
| --------- | --------------------- | -------------------------------- | -------------------------------- | ------------------------------------------------------------------------------------------------- | --- |
| `byte`    | 1 (8 бит)             | -128                             | 127                              | Наименьший целочисленный тип; <br>Часто используется для массивов байтов, например, чтение файлов |     |
| `short`   | 2 (16 бит)            | -32,768                          | 32,767                           | Экономичнее `int`, но используется редко                                                          |     |
| `int`     | 4 (32 бит)            | -2<sup>31</sup> = -2,147,483,648 | 2<sup>31</sup>−1 = 2,147,483,647 | Тип по умолчанию для целых чисел                                                                  |     |
| `long`    | 8 (64 бит)            | -2<sup>63</sup> ≈ -9.2×10¹⁸      | 2<sup>63</sup>−1 ≈ 9.2×10¹⁸      | Используется для больших целых значений; литералы требуют `L` на конце: `1234567890L`             |     |
| `float`   | 4 (32 бит)            | ~1.4×10<sup>-45</sup>            | ~3.4×10<sup>38</sup>             | Одинарная точность, может иметь ошибки округления                                                 |     |
| `double`  | 8 (64 бит)            | ~4.9×10<sup>-324</sup>           | ~1.8×10<sup>308</sup>            | Двойная точность; тип по умолчанию для вещественных чисел                                         |     |
| `char`    | 2 (16 бит)            | 0                                | 65,535                           | Представляет **один символ Unicode**, например `'A'` или `'Ж'`                                    |     |
| `boolean` | зависит от реализации | `false`                          | `true`                           | Реально может занимать 1 байт или больше в массивах; используется для логических флагов           |     |
### 💡 Примечания:
🔹 В **Java ==нет беззнаковых типов**== целых чисел (`byte`, `short`, `int`, `long` всегда _signed_).
🔹 Если нужны **беззнаковые операции**, начиная с Java 8 доступны методы вроде `Byte.toUnsignedInt()`, `Integer.toUnsignedLong()` и др.

- `boolean` **не имеет чётко определённого размера** в спецификации Java — это зависит от JVM.
- `char` в Java — **не `byte`**, а 16-битный символ Unicode.
- Для **научных и финансовых расчётов** лучше использовать `BigDecimal`, чтобы избежать ошибок округления `float` и `double`.

# 📕 2. Целочисленные литералы
В Java есть ==**четыре системы счисления**==, которые можно использовать для задания литералов:

> [!NOTE] Литералы по умолчанию
> - Целочисленные литералы создают значение типа `int` по умолчанию
> - Литералы с плавающей точкой создают значение типа `double` по умолчанию

|Система|Префикс|Пример|Комментарий|
|---|---|---|---|
|**Десятичная** (по умолчанию)|_(нет)_|`42`, `-7`, `0`|Обычные числа|
|**Двоичная**|`0b` или `0B`|`0b1010`|С Java 7|
|**Восьмеричная**|`0`|`012`|`012` = `10` в десятичной|
|**Шестнадцатеричная**|`0x` или `0X`|`0xFF`|`0xFF` = `255`|
#### Суффиксы

| Суффикс     | Тип     | Пример                    | Примечание                                                              |
| ----------- | ------- | ------------------------- | ----------------------------------------------------------------------- |
| _(нет)_     | `int`   | `123`                     | Литерал типа `int`                                                      |
| `L`         | `long`  | `123L` или `0xFFFF_FFFFL` | Используй `L`, а не `l`, чтобы избежать путаницы                        |
| `f` или `F` | `float` | `3.14f`, `2e3F`           | Литерал типа `float`, обязателен при присваивании значения типа `float` |

# 📕 3. Выведение типов локальных переменных (Java 10)
Ключевое слово `var` позволяет **не указывать явно тип переменной**, если компилятор может его **однозначно вывести** из инициализатора.
```java
var number = 10;         // int
var name = "Java";       // String
var list = new ArrayList<String>(); // ArrayList<String>
```

## ❗ Нельзя использовать `var`:
- Для **параметров методов**
- Для **поля класса**
- Без инициализации
- С `null`, без уточнения типа
