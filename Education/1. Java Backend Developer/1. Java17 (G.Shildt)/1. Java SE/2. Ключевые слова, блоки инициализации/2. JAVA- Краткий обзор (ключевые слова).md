# 📕 2. Ключевые слова

## 🔺`abstract`

### Что делает:
Объявляет **абстрактный класс** или **абстрактный метод**, которые нельзя использовать напрямую.
### Где используется:
- Когда нужен **шаблон поведения**, но реализация откладывается до подклассов.
- В паттерне **Template Method**, **Strategy**.
### 💡 Особенности:
- Абстрактный класс **не может быть создан с помощью `new`**.
- Абстрактный метод **не имеет тела**.
- **Обязательно** ==помечать класс с абстрактными методами как== `abstract`

```java
abstract class Animal {
    abstract void makeSound(); // подкласс обязан реализовать
}

class Dog extends Animal {
    @Override
    void makeSound() {
        System.out.println("Bark");
    }
}
```

## 🔺 `assert`

### Что делает:
Добавляет **утверждение**, которое проверяется во время выполнения. ==Используется для отладки.==
### Где используется:
- Проверка **предусловий, постусловий, инвариантов**.
### 💡 Особенности:
- Работает **только при включении флага `-ea`** (`enable assertions`).
- Может быть **выключено в продакшене**.
```java
int age = -5;
assert age > 0 : "Возраст должен быть положительным!";

//----ИЛИ----

public void withdraw(int amount) {
    assert balance >= 0 : "Баланс не может быть отрицательным!";
}
```

## 🔺 `default`

### Что делает:
- В ==интерфейсах: реализация метода по умолчанию==.
- В `switch`: блок по умолчанию.
### 💡 Особенности:
- Позволяет **расширять интерфейсы** без нарушения обратной совместимости.
```java
interface MyInterface {
    default void greet() {
        System.out.println("Hello from default!");
    }
}
```

## 🔺`instanceof`

### Что делает:
Проверяет, **является ли объект экземпляром указанного класса или интерфейса**.
### 💡 Особенности:
- С Java 16 добавлена поддержка **pattern matching**.

if (obj instanceof String s) { // Присваивание obj строке s, если obj относится к типу String

}

## 🔺`sealed` (Java 15+)

### Что делает:
Ограничивает **наследование**. 
Ключевое слово **`sealed`** позволяет строго контролировать, какие классы или интерфейсы могут наследоваться (или реализовываться).
##### 🔑 **Цель:**
- Запретить произвольное расширение класса/интерфейса.
- Чётко определить иерархию типов (например, для `Shape` разрешены только `Circle` и `Square`).

###  Как работает?

1. **Объявление `sealed`-класса/интерфейса** с указанием разрешённых наследников (`permits`).
2. **Наследники должны быть** (хотя бы одно условие):
    - `final` — нельзя наследовать дальше.
    - `sealed` — можно наследовать, но только указанным классам.
    - `non-sealed` — снимает ограничения (как обычный класс).

```java
// 1. sealed-интерфейс с явным указанием разрешённых реализаций
public sealed interface Shape permits Circle, Square, Rectangle {
    double area();
}

// 2. Допустимые реализации:
final class Circle implements Shape {  // final — запрещает дальнейшее наследование
    @Override
    public double area() { 
        return Math.PI * radius * radius; 
    }
}

sealed class Square implements Shape permits ColoredSquare {  // sealed — можно наследовать, но только ColoredSquare
    @Override
    public double area() { 
        return side * side; 
    }
}

non-sealed class Rectangle implements Shape {  // non-sealed — можно свободно наследовать
    @Override
    public double area() { 
        return width * height; 
    }
}

// 3. Дополнительный разрешённый наследник Square
final class ColoredSquare extends Square {
    private String color;
}
```
## 🔺`non-sealed` (Java 17)
### Что делает:
Разрешает **наследование без ограничений** в контексте `sealed`-иерархии.

## 🔺 `permits` (Java 15+)

### Что делает:
Указывает **разрешённые подклассы sealed-типа**.

## 🔺 `provides` (JPMS)
### Что делает:
Директива **`provides`** в модуле (`module-info.java`) ==связывает **реализацию** интерфейса== (или абстрактного класса) с его **объявлением как сервиса (SPI — Service Provider Interface)**.

👉 **Простыми словами:**
- **Интерфейс** объявляется как сервис (`uses`).
- **Класс-реализация** указывается как его провайдер (`provides ... with`).
- JVM автоматически находит и загружает все реализации сервиса через **`ServiceLoader`**.

### Как это работает?
```java
// Пример: сервис для шифрования
public interface Encryptor {
    String encrypt(String data);
}

public class AesEncryptor implements Encryptor {
    @Override
    public String encrypt(String data) {
        return "[AES] " + data;  // Упрощённая реализация
    }
}
```

**Объявление в модуле (`module-info.java`)**:
```java
module my.encryption.provider {
    // Предоставляем реализацию Encryptor
    provides com.example.Encryptor with com.example.AesEncryptor;
}
```

**Модуль, который использует сервис**:
```java
module my.app {
    // Зависимость от модуля с интерфейсом Encryptor
    requires my.encryption.api;
    // Указываем, что нужен сервис Encryptor
    uses com.example.Encryptor;
}
```

**Загрузка сервиса в коде**:
```java
ServiceLoader<Encryptor> loader = ServiceLoader.load(Encryptor.class);

for (Encryptor encryptor : loader) {
    System.out.println(encryptor.encrypt("Hello"));
}
// Вывод: [AES] Hello
```

## 🔺 `record` (Java 14+, stable с Java 16)

### Что делает:
`record` — это новый тип класса в Java, предназначенный ==для создания **неизменяемых (immutable)** объектов-контейнеров данных==.  
Он автоматически генерирует стандартные методы (`equals()`, `hashCode()`, `toString()`), конструктор и геттеры.

### 💡Основные особенности `record`

#### 1. **Автоматически генерируемые методы**
При объявлении `record` Java **автоматически** создаёт:  
✅ **Конструктор** — принимает все поля.  
✅ **Геттеры** — методы доступа к полям (но без префикса `get`!).  
✅ **`equals()` и `hashCode()`** — сравнивают записи по содержимому.  
✅ **`toString()`** — выводит все поля в удобном формате.

#### 2. **Неизменяемость (immutable)**
Все поля `record` — **`final`**, их нельзя изменить после создания.

### 📑Примеры

```java
// Объявление record (аналог класса)
public record Person(String name, int age) { }

Person person = new Person("Алексей", 30);
System.out.println(person.name()); // "Алексей" (геттер)
System.out.println(person);       // Person[name=Алексей, age=30]
```

 **Record с дополнительными методами**
```java
public record Person(String name, int age) {
    // Дополнительный метод
    public String greet() {
        return "Привет, я " + name + "! Мне " + age + " лет.";
    }

    // Статическое поле
    public static final int MAX_AGE = 120;

    // Кастомный конструктор (например, для валидации)
    public Person {
        if (age < 0) throw new IllegalArgumentException("Возраст не может быть отрицательным!");
    }
}
```

## 🔺 `requires` (JPMS)

### Что делает:
Директива **`requires`** в файле `module-info.java` указывает, что текущий модуль **зависит от другого модуля**. Без этой зависимости модуль не сможет использовать его классы.

```java
module com.myapp {
    requires java.sql;          // Зависимость от стандартного модуля JDBC
    requires org.apache.logging.log4j; // Зависимость от внешней библиотеки
}
```

## 🔺`strictfp`  (Strict Floating-Point)

### Что делает:
Гарантирует, что **все вычисления с `float`/`double` строго IEEE 754**.
С `strictfp` результаты будут **идентичны** везде (за счёт отказа от оптимизаций).

```java
public strictfp class MathLib {
    double compute(double a, double b) {
        return a / b;
    }
}
```

###  Где применяется?
Его можно использовать для:
1. **Классов**:
```java
public strictfp class Calculator {
    // Все методы класса будут strictfp
}
```

2. **Методов**:
```java
public class MathUtils {
    public strictfp double calculate() {
        return 0.1 + 0.2; // Гарантированно 0.30000000000000004
    }
}
```

3. **Интерфейсов** (все их реализации будут `strictfp`):
```java
public strictfp interface FloatingPointMath {
    double compute();
}
```

## 🔺 `transient`

### Что делает:
Ключевое слово **`transient`** помечает поле класса так, чтобы оно **игнорировалось при сериализации** (преобразовании объекта в байтовый поток).

При десериализации такие поля получают **значения по умолчанию** (`null`, `0`, `false`).

```java
public class User implements Serializable {
    private String username;
    private transient String password; // Не сериализуется!

    public User(String username, String password) {
        this.username = username;
        this.password = password;
    }

    @Override
    public String toString() {
        return "User: " + username + ", Password: " + password;
    }
}


public class Main {
    public static void main(String[] args) throws IOException, ClassNotFoundException {
        User user = new User("admin", "secret123");

        // Сериализация (запись в файл)
        try (ObjectOutputStream out = new ObjectOutputStream(
	        new FileOutputStream("user.dat"))) 
	    {
            out.writeObject(user);
        }

        // Десериализация (чтение из файла)
        try (ObjectInputStream in = new ObjectInputStream(
	        new FileInputStream("user.dat"))) 
	    {
            User restoredUser = (User) in.readObject();
            System.out.println(restoredUser); // User: admin, Password: null
        }
    }
}
```

## 🔺 `yield` (Java 14+)

### Что делает:
Возвращает значение из **`switch`-выражения с блоком**.
Ключевое слово **`yield`** используется в **switch-выражениях** (не путать с `switch`-операторами) для возврата значения из конкретной ветки.

👉 **Простыми словами:**
- Это аналог `return`, но только внутри `switch`.
- Позволяет использовать `switch` как **выражение** (которое возвращает значение), а не просто как инструкцию.

### ДО `yield`
```java
String dayType;
switch (day) {
    case "MON", "TUE", "WED", "THU", "FRI" -> dayType = "Рабочий день";
    case "SAT", "SUN" -> dayType = "Выходной";
    default -> dayType = "Неизвестно";
}
```

### С `yield`
```java
String dayType = switch (day) {
    case "MON", "TUE", "WED", "THU", "FRI" -> {
        System.out.println("Это будни!");
        yield "Рабочий день";  // Возвращаем значение
    }
    case "SAT", "SUN" -> "Выходной";  // Краткая форма (без yield)
    default -> {
        yield "Неизвестно";
    }
};
```

👉 Если блок кода в `case` обёрнут в `{ }`, нужно использовать `yield`.  
👉 В краткой форме (`-> "Выходной"`) `yield` не нужен.

## 🔺 `native`

### Что делает:

Указывает, что метод **реализован на другом языке** (обычно C/C++ через JNI **Java Native Interface**).
### Как это работает?
1. **Объявление `native`-метода** в Java:
```java
public native void process();  // Метод без тела (реализация в нативном коде)
```

2. **Загрузка нативной библиотеки** (`.dll`, `.so`, `.dylib`):
```java
static {
    System.loadLibrary("nativeLib");  // Загружает библиотеку "nativeLib.dll" (Windows) или "libnativeLib.so" (Linux)
}
```

3. **Реализация метода на C/C++** (через JNI):
```cpp
#include <jni.h>

JNIEXPORT void JNICALL Java_ClassName_process(JNIEnv *env, jobject obj) {
    printf("Вызван нативный метод!\n");
}
```
