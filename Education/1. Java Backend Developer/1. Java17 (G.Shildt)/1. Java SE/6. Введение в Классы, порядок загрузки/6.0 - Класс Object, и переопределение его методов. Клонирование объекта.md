# 📕 1. Краткий список публичных/защищённых методов `Object` (основные)

- `public final Class<?> getClass()`  
    Возвращает runtime-class объекта. `final` — нельзя переопределить.
    
- ⭐`public int hashCode()`  
    Возвращает хэш-код по умолчанию (идентичность объекта). Часто переопределяют.
    
- ⭐`public boolean equals(Object obj)`  
    Сравнение объектов по «эквивалентности». Часто переопределяют.
    
- `public String toString()`  
    Строковое представление объекта. Часто переопределяют.
    
- ⭐ `protected Object clone() throws CloneNotSupportedException`  
    Клонирование (по умолчанию вызывает native `clone`). По умолчанию `protected` и требует реализации `Cloneable`, если нужен публичный `clone()` — переопределять с осторожностью.
    
- `protected void finalize() throws Throwable` _(==устаревший, deprecated==)_  
    Финализация перед сборкой мусора — не рекомендуется переопределять (deprecated).
    
- `public final void wait() throws InterruptedException`  
    `public final void wait(long timeout)`  
    `public final void wait(long timeout, int nanos)`  
    Ожидание на мониторе объекта — вызываются **только** при захваченном (synchronized) мониторе объекта. `final` — нельзя переопределить.
    
- `public final void notify()`  
    `public final void notifyAll()`  
    Разблокируют один/все потоки, ожидающие на мониторе. Тоже требуют владения монитором и не переопределяются.

# 📕 2. Правильное переопределение `equals` — контракт и шаблон

**Контракт `equals` (важно!)**
- Рефлексивность: `x.equals(x)` — всегда `true`.
    
- Симметричность: `x.equals(y)` ⇔ `y.equals(x)`.
    
- Транзитивность: если `x.equals(y)` и `y.equals(z)` ⇒ `x.equals(z)`.
    
- Консистентность: повторные вызовы дают тот же результат (если поля не менялись).
    
- `x.equals(null)` должно возвращать `false`.
    

**Шаблон (безопасный, рекомендуемый для немодифицируемых/не-наследуемых классов):**
```java
@Override
public boolean equals(Object o) {
	// 1. Проверка на ссылочное равенство
    if (this == o) return true;             
    
	// 2. Проверка на null и совместимость типов  
    if (o == null || getClass() != o.getClass()) return false; 
    
    // 3. Приведение типа
    MyClass other = (MyClass) o;
    
    // 4. Сравнение полей по порядку (сначала примитивы, потом объекты)
    return Objects.equals(field1, other.field1)
           && field2 == other.field2
           && Double.compare(field3, other.field3) == 0
           && Arrays.equals(arrayField, other.arrayField);
}
```

Пояснения:
- `getClass()` vs `instanceof`:
    - `getClass()` даёт _строгую_ проверку — объекты разных подклассов не равны. Рекомендуется для value-типов.
        
    - `instanceof` допускает равенство с подклассами — подходит, если вы хотите полиморфное равенство (редкий вариант). `instanceof` легче нарушить симметричность при наследовании → осторожно.
        
- Для чисел с плавающей точкой используйте `Double.compare`/`Float.compare`.
- Для массивов — `Arrays.equals`/`Arrays.deepEquals`.

# 📕3. Правильное переопределение `hashCode`

**Правило:** если `a.equals(b)` → `a.hashCode() == b.hashCode()`. Обратное не обязательно.

- 🔶Использовать `Objects.hash(...)` (удобно, но немного медленнее):
```java
@Override
public int hashCode() {
    return Objects.hash(field1, field2, field3);
}
```

- 🔶Для производительности (особенно в hot-коде) — комбинировать аккуратно:
```java
@Override
public int hashCode() {
    int result = 17;
    result = 31 * result + (field1 == null ? 0 : field1.hashCode());
    result = 31 * result + Integer.hashCode(field2);
    long bits = Double.doubleToLongBits(field3);
    result = 31 * result + (int)(bits ^ (bits >>> 32));
    return result;
}
```

⭐⭐**Кэширование `hashCode`:**
- Подходит для **неизменяемых** объектов (immutable). Кэширующий подход: вычислить один раз и сохранить в `final` или `volatile` поле. Не делайте это для изменяемых объектов.
