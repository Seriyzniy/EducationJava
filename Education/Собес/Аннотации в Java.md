Аннотация - способ предоставления ВСПОМОГАТЕЛЬНОЙ информации и МАРКИРОВКИ объектов

Для определения СВОЕЙ аннотации нужно использовать следующий синтаксис:
```java
@interface MySuperAnnotation{

}
```

Для того чтобы аннотация могла принимать данные, их необходимо объявить
==При этом ПЕРЕМЕННАЯ объявляется как фукнция!==
```java
@interface MySuperAnnotation{
	String myStr()
}
```
***

==Срок действия аннотаций==
- **`RetentionPolicy.SOURCE`**: Аннотация ==доступна только на уровне исходного кода== и не сохраняется в скомпилированном классе. Это значит, что она будет удалена на стадии компиляции.
    
- **`RetentionPolicy.CLASS`**: Аннотация ==сохраняется в скомпилированном классе==, но не будет доступна во время выполнения программы. Это поведение по умолчанию для аннотаций, если не указано иное.
    
- **`RetentionPolicy.RUNTIME`**: Аннотация ==сохраняется в скомпилированном классе и доступна во время выполнения программы==. Это позволяет считывать аннотацию с помощью рефлексии.

***

==К аннотациям можно применять другие аннотации==. 
### Аннотация `@Retention()`
Так например аннотация `@Retention` используется для описания того, как будет использоваться аннотация
### Аннотация `@Target()`
Используется для указания того, к каким элементам программы можно применять аннотацию

- **`ElementType.TYPE`** — Аннотация может быть применена к **классам**, **интерфейсам**, **перечислениям**
	
- **`ElementType.FIELD`** — Аннотация может быть применена к **поля**.
	
- **`ElementType.METHOD`** — Аннотация может быть применена к **методам**
	
- **`ElementType.PARAMETER`** — Аннотация может быть применена к **параметрам методов**.
	
- **`ElementType.CONSTRUCTOR`** — Аннотация может быть применена к **конструкторам**.
	
- **`ElementType.ANNOTATION_TYPE`** — Аннотация может быть применена к **другим аннотациям**.

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.SOURCE)
@interface MySuperAnnotation{

}
```
