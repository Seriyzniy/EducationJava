# 📕1. Виды блоков инициализации в Java

## 1. Статический блок инициализации (`static {...}`)
- Выполняется **один раз при загрузке класса** (перед первым использованием класса).
    
- Используется для инициализации **статических полей** и выполнения любой логики, которая должна быть выполнена только один раз.
    
- Выполняется **в порядке объявления в коде**, вместе со статическими полями.
```java
public class Example {
    static int x;
    static {
        x = 10;
        System.out.println("Статический блок: x = " + x);
    }
}
```
> [!NOTE] Static-блок: имеет доступ только к `static`-полям/методам.
> 

## 2. Нестатический (instance) блок инициализации (`{...}`)
- ==Выполняется **каждый раз при создании объекта**== (`new`).
    
- Срабатывает ==**после инициализации полей по умолчанию**, НО **до конструктора**==.
    
- Если в классе несколько таких блоков — они выполняются **сверху вниз в порядке объявления**.

```java
public class Example {
    int y;

    { 
        y = 20;
        System.out.println("Нестатический блок: y = " + y);
    }

    public Example() {
        System.out.println("Конструктор: y = " + y);
    }
}
```
> [!NOTE] Instance-блок: имеет доступ КО ВСЕМ ПОЛЯМ И МЕТОДАМ, включая `this`
> 

***
# 📕2. Что можно делать внутри блоков инициализации?

- В обоих блоках (`static` и обычных) ==можно писать **любой код Java**==: присваивания, вызовы методов, условия, циклы, даже `try-catch`.
    
- ==**Можно инициализировать как статические, так и нестатические поля==.**  
    ⚠️ Но это считается ==**плохим стилем**==, если:
    - в НЕстатическом блоке** инициализируются **статические поля**; 
	    (🛑Это допустимо, т.к. статическая переменная существует одна на весь класс.🛑)
	    
    - в **статическом блоке** используется нестатическое поле (так вообще не скомпилируется).

***

# 📕3. Когда реально использовать блоки инициализации?
- 🔶**Static-блок**:
    - сложная инициализация статических полей (например, загрузка драйвера JDBC: `Class.forName("com.mysql.jdbc.Driver")`),
        
    - логика, требующая try-catch при инициализации статических полей.
        
- 🔶 **Instance-блок**:
    - редко используется. Иногда применяют для общей логики, которая должна выполняться **перед любым конструктором**.
        
    - но чаще такую логику выносят в конструктор или отдельный `init()`-метод → это понятнее.
