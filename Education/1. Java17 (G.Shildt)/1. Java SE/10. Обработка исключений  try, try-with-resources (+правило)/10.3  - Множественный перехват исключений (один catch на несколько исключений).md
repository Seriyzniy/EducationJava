Начиная с JDK 7 в Java появилась возможность МНОЖЕСТВЕННОГО ПЕРЕХВАТА исключений (когда один блок `catch` может обработать сразу два исключений)

Такой механизм удобен, если два разных исключения при перехвате обрабатываются одинаково

> [!NOTE] 🛑Использования множественного перехвата🛑
> Чтобы использовать множественных перехват, нужно ОБЪЕДИНИТЬ ВСЕ ТИПЫ исключений в конструкции `catch` при помощи операции `ИЛИ`
> 

```java
public static void main(String[] args) {
	try {
		int[] numbers = {1, 2, 3};
		System.out.println(numbers[5]);  // Может вызвать ArrayIndexOutOfBoundsException
		
		String text = null;
		System.out.println(text.length());  // Может вызвать NullPointerException
		
	} catch (ArrayIndexOutOfBoundsException | NullPointerException e) {
		System.out.println("Произошла ошибка: " + e.getClass().getSimpleName());
		System.out.println("Сообщение: " + e.getMessage());
	}
}
```


> [!NOTE] 🛑Объект исключения = finall при множественном перехвате🛑
> При множественном перехвате исключений в одном `catch` блоке переменная исключения (`e` в `catch (Ex1 | Ex2 e)`) является **фактически `final`**, начиная с Java 8. Это значит, что её **нельзя изменить** внутри блока `catch`

# 💡 **Важные моменты** 💡

✔ **Можно ловить только родственные исключения** (например, оба — `RuntimeException`).  
✔ **Нельзя ловить исключения из одной иерархии** (например, `Exception | ArithmeticException` — ошибка, т.к. `ArithmeticException` уже входит в `Exception`).
