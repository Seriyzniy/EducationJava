# 📕1. Использование нескольких `catch` 
В блоке `catch` обрабатываются исключения указанного типа, а также любой из его подклассов. 

> [!NOTE] 🛑 Порядок обработки исключений 🛑
> ==СНАЧАЛА КОНКРЕТНЫЕ== исключения, а ==ЗАТЕМ более ОБЩИЕ==
> 
> Если сделать наоборот (общий, а потом точный) - ошибка компиляции (НЕДОСТИЖИМЫЙ КОД)

```java
try {
    // ...
} catch (IOException e) { //Более точное исключение
    // ...
} catch (Exception e) {  //Более общее исключение
    // ...
}
```

# 🛑🛑🛑 ДРУГОЙ способ ПРОБРАСЫВАНИЯ исключения 🛑🛑🛑
```java
public class MyMain {
    public static void main(String[] args) {
    	try {
    		CallerExceptuon.generatedException();
    	} catch (Exception e) {
    		System.out.println(e.getMessage());;
    	}
        
    }
}

class CallerExceptuon {
	public static double generatedException() {
		double result = 0;
		try {
			result = 1 / 0;
		} catch (Exception e) {
			throw new ArithmeticException("Hahah");
		}
		return result;
	}
}

//Вывод:
Hahah
```

# 📕 2. Вложенный `try` 
Вложенный `try` — это просто `try` внутри другого `try`, `catch` или `finally`.

## 💡 Особенности и поведение 💡
### 🔸 Правило 1: **Каждое исключение обрабатывается тем `catch`, который ближе** (ближе в своем блоке try/catch)

==Если исключение возникло во **ВНУТРЕННЕМ== `try`**, то:
- сначала проверяются **внутренние `catch`-блоки**;
    
- если не найдено подходящего `catch`, исключение **"вылетает наружу"** — в внешний `catch`.

```java
try {
    try {
        int a = 5 / 0; // ArithmeticException
    } catch (NullPointerException e) {
        System.out.println("Внутренний catch");
    }
} catch (ArithmeticException e) {
    System.out.println("Внешний catch"); // 👉 сюда попадем
}
```

> [!NOTE] 🛑Исключительная ситуация внутри метода, который вызывается внутри try🛑
> Схожая ситуация происходит с функциями. Сначала ошибка пытается обработаться внутри своего блока try/catch, если тип исключения в catch не найден, то ошибка передается во внешний блок кода откуда вызывается метод

### 🔸 Правило 2: Исключение во внешнем `try` НЕ передаётся во внутренний `try`
```java
try {
    int[] arr = null;
    int x = arr[0]; // ❗ NullPointerException НЕ ПЕРЕДАЕТСЯ ВО ВНУТРЕННИЙ
    
    try {
        System.out.println("Внутренний блок");
    } catch (Exception e) {
        System.out.println("Внутренний catch");
    }
} catch (NullPointerException e) {
    System.out.println("Внешний catch"); // 👉 сюда попадем
}
```

# 📕 3. Операторы  `throw` и `throws` 
## Оператор `throw`
Оператор `throw` используется для **явного выбрасывания исключения** в коде.
При этом пользовательское исключение должно быть наследником `Throwable`
```java
public class Example {
    public static void main(String[] args) {
        int age = 15;
        if (age < 18) {
            throw new ArithmeticException("Доступ запрещён – пользователь младше 18 лет!");
        } else {
            System.out.println("Доступ разрешён");
        }
    }
}
```

## Оператор `throws`
Ключевое слово `throws` используется в сигнатуре метода, чтобы **предупредить, что метод может выбросить исключение**, но не обрабатывает его сам.
```java
public class Example {
    public static void readFile() throws IOException {
        FileReader file = new FileReader("test.txt");
        // если файла нет, будет IOException
    }

    public static void main(String[] args) {
        try {
            readFile();
        } catch (IOException e) {
            System.out.println("Ошибка: " + e.getMessage());
        }
    }
}
```

# 📕4. Оператор `finally`

> [!NOTE] 🛑Блок `finally`🛑
> `finally` — это блок кода, ==который **всегда выполняется** после завершения== `try` и `catch`, **даже если в них был `return`, `break`, `continue`, `throw`, или ошибка**.
> 
> Если вызвать System.exit(0) - то finally не сработает, так как метод завершает работу JVM

## 🛑Возврат результата НЕ в `finally` 🛑
> [!NOTE] Делать `return` внутри `finally` АНТИПАТТЕРН!!!

### ✅Правильный подход в возврату результата 
- 🔶Возврат после блока try/catch/finally
```JAVA
public String readFile() {
    String result = null;
    Exception exception = null;
    
    try {
        result = Files.readString(Path.of("file.txt"));
    } catch (IOException e) {
        exception = e;
        System.err.println("Ошибка чтения: " + e.getMessage());
    } finally {
        // Очистка ресурсов, но НЕ возврат!
        closeResources();
    }
    
    if (exception != null) {
        return "default"; // или бросить исключение
    }
    return result;
}
```

- 🔶 Возврат внутри `try`
```java
public String readFile() throws IOException {
    try {
        return Files.readString(Path.of("file.txt"));// Возврат
    } finally {
        // Очистка ресурсов, но результат уже возвращен из try
        closeResources();
    }
}
```

- 🔶Использование `try with resources`
```java
public String readFile() {
    try (BufferedReader reader = new BufferedReader(new FileReader("file.txt"))) {
        return reader.readLine(); // Возврат из try
    } catch (IOException e) {
        System.err.println("Ошибка чтения: " + e.getMessage());
        return "default";
    }
    // finally не нужен - ресурсы закрываются автоматически
}
```
## ⚠️Особенности
### 🔶`finally` и циклы
```java
for (int i = 0; i < 3; i++) {
    try {
        System.out.println("i = " + i);
        continue;
    } finally {
        System.out.println("finally на итерации " + i);
    }
}
```
> [!NOTE] 🛑На каждой итерации цикла `finally` всё равно выполняется.🛑

### 🔶 `return` в `finally` — подавляет исключения
```java
public static void test() {
    try {
        throw new RuntimeException("Исключение в try");
    } finally {
        return; // ❌ исключение игнорируется
    }
}
```

### 🔶 `finally` может выбросить другое исключение и скрыть исходное
```java
try {
    throw new IOException("исходное");
} finally {
    throw new NullPointerException("новое");
}
```
> [!NOTE] ❗ `IOException` будет **ПОДАВЛЕНО**, и будет выброшено только `NullPointerException`.
>

# 💡Получение ПОДАВЛЕННЫХ исключений 
📌 С Java 7 ==suppressed== исключения можно получить:
```java
try {
    // ...
} catch (Exception e) {
    for (Throwable suppressed : e.getSuppressed()) { //Перебор ранее выбрашенных исключений
        suppressed.printStackTrace();
    }
}
```

# 💡**Ключевые правила**💡
1. **Никогда не возвращайте значение из блока finally**
    
2. **Используйте finally только для очистки ресурсов**
    
3. **Возвращайте результат из try или catch блоков**
    
4. **Для возврата разных результатов используйте переменные**
    
5. **Предпочитайте try-with-resources для автоматического закрытия**
