В Java есть несколько способов отправить запросы к сторонним REST API. Вот основные классы и подходы, которые вы можете использовать:

***
### **1. Использование `HttpURLConnection`**

Это встроенный в Java класс, который можно использовать для отправки HTTP-запросов. Это базовый способ, требующий больше кода для обработки запросов и ответов.

> [!NOTE] Класс `URL` из пакета `java.net`
> - Позволяет ==представлять адрес ресурса== (например, веб-страницы, API, файла).
> - ==Предоставляет методы для извлечения частей URL==, таких как протокол, хост, порт, путь и параметры.
> - Может быть использован ==для создания соединения с ресурсом== (например, через `HttpURLConnection`).


> [!NOTE] Класс `HttpURLConnection` из пакета `java.net`
> - **Поддержка HTTP-методов**: `GET`, `POST`, `PUT`, `DELETE`, `HEAD`, и др.
> - **Работа с заголовками**: Возможность добавлять заголовки запроса и получать заголовки ответа.
> - **Работа с телом запроса/ответа**: Поддерживает чтение и запись данных в запросе или ответе.
> - **Поддержка HTTPS**: Автоматически работает с HTTPS-соединениями.
> - **Таймауты**: Настройка времени ожидания соединения и чтения данных.
> - **Низкоуровневый API**: Требует больше кода, чем современные аналоги, такие как `HttpClient` (Java 11+) или OkHttp.
> - `HttpURLConnection` ==наследует от== `URLConnection`

```java
import java.io.*;
import java.net.HttpURLConnection;
import java.net.URL;

public class HttpClientExample {
    public static void main(String[] args) throws IOException {
    
        URL url = new URL("https://api.example.com/resource");
        HttpURLConnection connection = (HttpURLConnection) url.openConnection();
        connection.setRequestMethod("GET");
        
        int responseCode = connection.getResponseCode();
        if (responseCode == HttpURLConnection.HTTP_OK) {
            BufferedReader in = new BufferedReader(new InputStreamReader(connection.getInputStream()));
            String inputLine;
            StringBuilder response = new StringBuilder();
            while ((inputLine = in.readLine()) != null) {
                response.append(inputLine);
            }
            in.close();
            System.out.println(response);
        } else {
            System.out.println("Request failed: " + responseCode);
        }
    }
}
```

### **2. Использование `HttpClient` (Java 11 и выше)**
`HttpClient` — это современный API для отправки HTTP-запросов, который появился в Java 11.
Это мощный инструмент, который заменяет устаревший `HttpURLConnection` и предлагает более удобный, гибкий и лаконичный способ работы с HTTP/HTTPS-запросами.

> [!NOTE] Класс `HttpClient` из пакета `java.net.http`
> Для создания HTTP-запроса используется метод `newBuilder`
> 
> Поддерживает:
> - Асинхронные и синхронные запросы
> - HTTP/2:
> - Автоматическое управление потоками для обработки запросов.
> - Легко настраивается: таймауты, редиректы, авторизация и т. д.
> - Возможность настройки прокси-серверов.
> - API высокого уровня
 
> [!NOTE] **Основные классы, связанные с `HttpClient`**
> `HttpClient`:- Основной класс для отправки запросов.
> `HttpRequest`: - Представляет HTTP-запрос. Используется для настройки URL, метода, заголовков, тела и других параметров.
> `HttpResponse`:- Представляет ответ от сервера. Содержит код состояния, заголовки и тело ответа.
> `BodyHandlers` и `BodyPublishers`:- Утилиты для обработки тела запросов и ответов.

```java
import java.net.http.*;
import java.net.URI;
import java.io.IOException;

public class HttpClientExample {
    public static void main(String[] args) throws IOException, InterruptedException {
        HttpClient client = HttpClient.newHttpClient();
        HttpRequest request = HttpRequest.newBuilder()
                .uri(URI.create("https://api.example.com/resource"))
                .GET()
                .build();

        HttpResponse<String> response = client.send(request, HttpResponse.BodyHandlers.ofString());
        System.out.println(response.body());
    }
}

```

### **3. Использование Spring WebClient (Spring Framework)**

Если вы используете Spring, `WebClient` — это удобный способ отправки запросов.

```java
import org.springframework.web.reactive.function.client.WebClient;

public class WebClientExample {
    public static void main(String[] args) {
        WebClient client = WebClient.create("https://api.example.com");

        String response = client.get()
                .uri("/resource")
                .retrieve()
                .bodyToMono(String.class)
                .block();

        System.out.println(response);
    }
}
```
#### **HttpEntity**

Класс `HttpEntity` представляет сущность HTTP-запроса или ответа. Он может содержать:

- **Заголовки (`HttpHeaders`)**
- **Тело (payload)**

Используется для передачи данных в HTTP-запросе (POST, PUT) или для работы с ответами.

#### Основные особенности:

- Может быть полезен, если вам нужно указать заголовки и тело запроса.
- Используется с классом `RestTemplate` или другими инструментами Spring Web.

---
#### **HttpHeaders**

Класс `HttpHeaders` позволяет удобно управлять заголовками HTTP-запросов. Это обертка над картой (`Map`), которая упрощает добавление и извлечение заголовков.

```java
import org.springframework.http.*;
import org.springframework.web.client.RestTemplate;

public class RestTemplateExample {
    public static void main(String[] args) {
        // Создаем заголовки
        HttpHeaders headers = new HttpHeaders();
        headers.setContentType(MediaType.APPLICATION_JSON);
        headers.set("Authorization", "Bearer your-token");

        // Тело запроса (payload)
        String requestBody = """
            {
                "key": "value"
            }
        """;

        // Создаем HttpEntity с телом и заголовками
        HttpEntity<String> requestEntity = new HttpEntity<>(requestBody, headers);

        // Отправляем запрос через RestTemplate
        RestTemplate restTemplate = new RestTemplate();
        String url = "https://api.example.com/resource";

        ResponseEntity<String> response = restTemplate.exchange(
                url, HttpMethod.POST, requestEntity, String.class);

        // Выводим ответ
        System.out.println("Response code: " + response.getStatusCode());
        System.out.println("Response body: " + response.getBody());
    }
}
```

> [!NOTE] Класс `RestTemplate`
> Класс **`RestTemplate`** используется в Spring Framework и предназначен для упрощения взаимодействия с RESTful веб-сервисами. 
> 
> Метод **`exchange()`** поддерживает все HTTP-методы: `GET`, `POST`, `PUT`, `DELETE`, `PATCH`, `OPTIONS`, и `HEAD`.
> Является одним из самых универсальных методов для выполнения HTTP-запросов.
>

```java
public <T> ResponseEntity<T> exchange( 
	String url, 
	HttpMethod method, 
	@Nullable HttpEntity<?> requestEntity, 
	Class<T> responseType, 
	Object... uriVariables )
```
