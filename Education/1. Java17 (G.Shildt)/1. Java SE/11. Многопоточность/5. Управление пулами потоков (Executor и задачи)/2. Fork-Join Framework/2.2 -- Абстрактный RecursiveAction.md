
> [!NOTE] `RecursiveAction` — это **абстрактный подкласс** `ForkJoinTask<Void>`, 
> Предназначенный для задач,**==НЕ ВОЗВРАЩАЮЩИХ РЕЗУЛЬТАТ== вычисления**.
> 

Используется в **Fork/Join Framework** для реализации параллельных алгоритмов типа _divide and conquer_,  
==когда нужно просто _выполнить действие_==, но не нужно ничего возвращать (например, сортировка, изменение массива, обработка данных и т.д.).

## Конструктор
💬 Класс **абстрактный**, поэтому нельзя создавать объекты напрямую.  
Нужно унаследоваться и реализовать метод `compute()`.

## Методы
### Основной метод
```java
// Метод, в котором определяется основная логика задачи.
// Не возвращает результата.
protected abstract void compute();
```
💬 Именно здесь описывается логика деления задачи и её выполнения.  

Как правило:
1. Проверяется, достаточно ли мала задача.
2. Если да — выполняется напрямую.
    
3. Если нет — ==делится на подзадачи==, которые запускаются через `fork()` и `join()`.

### Унаследованные ключевые методы от ForkJoinTask
```java
// Запускает задачу в пуле
public final ForkJoinTask<Void> fork()

// Ждет завершения задачи
public final Void join()

// Выполняет задачу синхронно в текущем потоке
public final Void invoke()

// Выполняет две задачи параллельно и ждет их завершения
public static void invokeAll(ForkJoinTask<?> t1, ForkJoinTask<?> t2)

// Выполняет коллекцию задач параллельно
public static <T extends ForkJoinTask<?>> Collection<T> 
	invokeAll(Collection<T> tasks)
```

💬 `RecursiveAction` поддерживает тот же механизм управления, что и `RecursiveTask`,  
но без возврата результата (`Void` вместо `V`).

---
## 💡 **Особенности и подводные камни** 💡

1️⃣ Класс `RecursiveAction` подходит для **параллельных действий без результата** (например, изменение элементов массива).  

2️⃣ Обязательно **переопредели `compute()`** — без него задача не имеет смысла.

3️⃣ 💡 Используй **==порог деления== (threshold)**, чтобы избежать бесконечной рекурсии и избыточного создания подзадач.  

4️⃣ Методы `fork()` и `join()` нужно использовать **осознанно** — несбалансированное использование может привести к снижению производительности.  

5️⃣ ❗❗❗Не выполняй в `compute()` блокирующих операций (I/O, sleep, synchronized), иначе весь пул потоков может “зависнуть”.  

6️⃣ 💡Исключения, возникшие в `compute()`, будут проброшены через `join()` или `invoke()`.

---

## 💎 **Best Practices** 💎

✅ 💡Используй `RecursiveAction`, если нужно **изменить общий ресурс**, а не вернуть значение.  

✅ ⭐⭐⭐Выбирай `THRESHOLD` опытным путём: он сильно влияет на производительность.  

✅ Для “мелких” задач лучше выполнить их напрямую, чем делить дальше.  

✅ Используй `ForkJoinPool.commonPool()` для большинства задач — он автоматически масштабируется под количество ядер.  

✅ Избегай побочных эффектов между подзадачами (изменение общих переменных без синхронизации).

## Пример
```java
import java.util.concurrent.RecursiveAction;
import java.util.concurrent.ForkJoinPool;

// Пример: умножение всех элементов массива на 2
class MultiplyTask extends RecursiveAction {
    private final int[] array;
    private final int start, end;
    private static final int THRESHOLD = 10_000;

    public MultiplyTask(int[] array, int start, int end) {
        this.array = array;
        this.start = start;
        this.end = end;
    }

    @Override
    protected void compute() {
        int length = end - start;

        // Если задача маленькая — выполняем напрямую
        if (length <= THRESHOLD) {
            for (int i = start; i < end; i++) {
                array[i] *= 2;
            }
        } else {
            // Делим задачу пополам
            int mid = start + length / 2;
            MultiplyTask left = new MultiplyTask(array, start, mid);
            MultiplyTask right = new MultiplyTask(array, mid, end);

            // Запускаем подзадачи параллельно
            invokeAll(left, right);
        }
    }
}

public class RecursiveActionExample {
    public static void main(String[] args) {
        int[] data = new int[100_000];
        for (int i = 0; i < data.length; i++) data[i] = i;

        ForkJoinPool pool = new ForkJoinPool();
        MultiplyTask task = new MultiplyTask(data, 0, data.length);

        pool.invoke(task);

        System.out.println("Первый элемент после обработки: " + data[0]);
        System.out.println("Последний элемент после обработки: " + data[data.length - 1]);
    }
}
```
