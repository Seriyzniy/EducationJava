
> [!NOTE] `ForkJoinPool` — это специализированный пул потоков,
> Оптимизированный ==для выполнения большого числа мелких параллельных задач==, которые могут `рекурсивно порождать` `подзадачи`.
> 
> Ключевые моменты:
> 1. ==**Демоны по умолчанию**==: Рабочие потоки ForkJoinPool создаются как демоны
> 2. **Поведение при завершении**: Когда все обычные (non-daemon) потоки приложения завершаются, потоки ForkJoinPool автоматически прекращают работу


Он лежит в основе _Fork/Join Framework_ и реализует _work-stealing_ алгоритм (воровство работы), позволяющий эффективно использовать все доступные процессорные ядра.


Класс реализует интерфейсы:
- `Executor` — для запуска задач.
- `ExecutorService` — для управления жизненным циклом пула.
    
- `ForkJoinPool.ForkJoinWorkerThreadFactory` (вложенный интерфейс) используется для кастомизации создаваемых потоков.

## Конструкторы
```java
// Создает пул с числом рабочих потоков, равным количеству доступных процессорных ядер.
// Эквивалентен ForkJoinPool(Runtime.getRuntime().availableProcessors()).
public ForkJoinPool()

// Создает пул с указанным числом потоков.
// parallelism — уровень параллелизма (обычно ≈ числу ядер процессора).
public ForkJoinPool(int parallelism)

// Создает пул с детализированной конфигурацией.
// parallelism — уровень параллелизма (целевое количество рабочих потоков)
// factory — фабрика для создания пользовательских рабочих потоков (может быть null для использования фабрики по умолчанию)
// handler — обработчик неперехваченных исключений в рабочих потоках (может быть null)
// asyncMode — режим очереди задач: true = FIFO (подходит для задач, которые никогда не выполняют join), 
//             false = LIFO (по умолчанию, оптимально для вычислительных задач с разделением)
public ForkJoinPool(int parallelism,
                    ForkJoinWorkerThreadFactory factory,
                    Thread.UncaughtExceptionHandler handler,
                    boolean asyncMode)


// Расширенный конструктор (Java 9+)
// включает возможность настройки политики блокировок и ограничений.
// parallelism — уровень параллелизма (💡целевое количество рабочих потоков)
// factory — 💡фабрика для создания пользовательских рабочих потоков
// handler — обработчик неперехваченных исключений в рабочих потоках
// asyncMode — режим очереди задач: true для FIFO, false для LIFO
// corePoolSize — минимальное количество рабочих потоков, которые не будут завершены по таймауту (только для Java 9+)
// maximumPoolSize — максимальное количество рабочих потоков (только для Java 9+)
// minimumRunnable — минимальное количество рабочих потоков, которые могут работать одновременно без блокировок (только для Java 9+)
// 💡saturate — предикат, вызываемый когда пул не может создать новый рабочий поток (только для Java 9+)
// keepAliveTime — время простоя потока перед завершением (только для Java 9+)
// unit — единица измерения времени для keepAliveTime (только для Java 9+)
public ForkJoinPool(int parallelism,
                    ForkJoinWorkerThreadFactory factory,
                    Thread.UncaughtExceptionHandler handler,
                    boolean asyncMode,
                    int corePoolSize,
                    int maximumPoolSize,
                    int minimumRunnable,
                    Predicate<? super ForkJoinPool> saturate,
                    long keepAliveTime,
                    TimeUnit unit)
```

## Методы
### Управление задачами
```java
/**
 * Выполняет переданную задачу и возвращает ее результат после завершения.
 * 
 * Метод блокирует текущий поток до тех пор, пока задача не будет выполнена.
 * Если задача уже выполнена, метод немедленно возвращает результат.
 * 
 * @param <T> тип результата, возвращаемого задачей
 * @param task задача для выполнения (должна быть наследником ForkJoinTask)
 * @return результат выполнения задачи
 * 
 * @throws NullPointerException если переданная задача равна null
 * @throws RejectedExecutionException если задача не может быть принята к выполнению
 * @throws RuntimeException если во время выполнения задачи произошло исключение
 */
public <T> T invoke(ForkJoinTask<T> task)


// Отправляет задачу в пул для выполнения (немедленно или отложенно).
public <T> ForkJoinTask<T> submit(ForkJoinTask<T> task)

// Выполняет задачу Callable и возвращает Future для получения результата.
public <T> Future<T> submit(Callable<T> task)

// Выполняет задачу Runnable и возвращает Future.
public Future<?> submit(Runnable task)



// Выполняет задачу без возврата результата, НЕ ДОЖИДАЯСЬ ЗАВЕРШЕНИЯ
public void execute(ForkJoinTask<?> task)

// Ожидает завершения всех задач, запущенных в этом пуле.
public void awaitQuiescence(long timeout, TimeUnit unit)
```

## 💡 **Особенности и подводные камни** 💡

1️⃣ `ForkJoinPool` ==оптимизирован для мелких, короткоживущих задач== — не стоит использовать его для долгих блокирующих операций.  

2️⃣ ⭐⭐Использует стратегию _work-stealing_: потоки «==воруют» задачи у других==, ==если== у них ==своя очередь пуста==.  

3️⃣ При блокировке потока (например, ожидание ввода/вывода) производительность резко падает. Лучше использовать `ManagedBlocker`.  

4️⃣ Если вызвать `join()` или `invoke()` внутри той же задачи — возможна дедлоковая ситуация при неправильной декомпозиции.  

5️⃣ `commonPool()` — общий для всех задач в JVM, поэтому чрезмерная нагрузка в одном месте может повлиять на другое.

---

## 💎 **Best Practices** 💎

✅ ❗❗❗Используйте `ForkJoinPool` для CPU-bound задач, а не для I/O.  

✅ 💡Делите задачи так, чтобы они были примерно равны по трудоёмкости.  

✅ Для больших проектов создавайте собственные пулы, не используйте `commonPool()` без необходимости.  

✅ ⭐⭐При необходимости блокировки используйте `ForkJoinPool.ManagedBlocker`.  

✅ Используйте `invoke()` для синхронного ожидания результата, если задача — корневая.

## Пример
```java
import java.util.concurrent.*;

public class ForkJoinExample {
    static class SumTask extends RecursiveTask<Long> {
        private final long[] arr;
        private final int start, end;
        private static final int THRESHOLD = 1000;

        SumTask(long[] arr, int start, int end) {
            this.arr = arr;
            this.start = start;
            this.end = end;
        }

        @Override
        protected Long compute() {
            if (end - start <= THRESHOLD) {
                long sum = 0;
                for (int i = start; i < end; i++) sum += arr[i];
                return sum;
            }
            int mid = (start + end) / 2;
            SumTask left = new SumTask(arr, start, mid);
            SumTask right = new SumTask(arr, mid, end);
            left.fork(); // асинхронно
            return right.compute() + left.join();
        }
    }

    public static void main(String[] args) {
        long[] numbers = new long[10_000];
        for (int i = 0; i < numbers.length; i++) numbers[i] = i + 1;

        ForkJoinPool pool = new ForkJoinPool();
        long result = pool.invoke(new SumTask(numbers, 0, numbers.length));
        System.out.println("Sum = " + result);

        pool.shutdown();
    }
}
```
