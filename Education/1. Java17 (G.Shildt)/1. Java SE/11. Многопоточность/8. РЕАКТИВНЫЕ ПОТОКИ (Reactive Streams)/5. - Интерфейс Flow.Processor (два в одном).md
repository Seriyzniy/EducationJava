
> [!NOTE] `Flow.Processor<T, R>` — это интерфейс, который `расширяет сразу два интерфейса`:
> - `Flow.Subscriber<T>` — он принимает данные типа `T`
> - `Flow.Publisher<R>` — и публикует (передаёт дальше) данные типа `R`

Таким образом, `Processor` — это ==реактивный преобразователь== (middleware), который может **обрабатывать, фильтровать, буферизовать, трансформировать** поток данных перед тем, как он дойдёт до конечного подписчика.

## Пример создания
```java
// Интерфейс не имеет конструкторов.
// Реализуется вручную или через готовые реализации (например, SubmissionPublisher как Processor).
Flow.Processor<T, R> processor = new Flow.Processor<>() {
    @Override
    public void subscribe(Flow.Subscriber<? super R> subscriber) {
        // Регистрация подписчика (Publisher часть)
    }

    @Override
    public void onSubscribe(Flow.Subscription subscription) {
        // Получение подписки (Subscriber часть)
    }

    @Override
    public void onNext(T item) {
        // Получение нового элемента (Subscriber часть)
    }

    @Override
    public void onError(Throwable throwable) {
        // Обработка ошибок
    }

    @Override
    public void onComplete() {
        // Завершение потока
    }
};
```

---
## 💡 Особенности и подводные камни 💡

1️⃣ **Processor — связующее звено.**  
Он может принимать данные от одного Publisher и передавать преобразованные элементы дальше в цепочку (другому Subscriber).

2️⃣ 💡 **Типы данных могут отличаться.**  
Например, `Processor<String, Integer>` может принимать строки и публиковать длину каждой строки.

3️⃣ 💡**Поток данных проходит через него асинхронно.**  
Обычно `Processor` реализует внутренний буфер или очередь, чтобы избежать блокировок.

4️⃣ **Backpressure передаётся вниз по цепочке.**  
Если нижестоящий подписчик запрашивает меньше элементов, `Processor` должен это учитывать.

5️⃣ ❗❗❗**Поток должен быть потокобезопасен.**  
Между получением (`onNext`) и публикацией (`subscribe`) может участвовать несколько потоков.

6️⃣ ❗❗❗**Нельзя нарушать контракты обеих сторон:**❗❗❗
- Сначала `onSubscribe()`, потом `onNext()`,
    
- И только после всех данных `onComplete()` или `onError()`.
    

7️⃣ После вызова `onError()` или `onComplete()` никаких новых элементов передаваться не должно.

---

## 💎 Best Practices 💎

✅ Используй `Processor`, если нужно добавить ==промежуточную обработку данных== (например, фильтрацию, агрегацию или преобразование).

✅ Всегда обеспечивай корректное управление **backpressure** между `Subscription` (верхний поток) и `Subscriber` (нижний поток).

✅ Внутри `onNext()` не выполняй блокирующие операции — используй очереди или ExecutorService.

✅ Обрабатывай ошибки (`onError`) корректно: завершай нижестоящий поток и освобождай ресурсы.

✅ 💡Если ты реализуешь собственный Processor, **передавай события в следующего подписчика строго в порядке** и **не вызывай onNext() после onComplete()/onError()**.

## Пример
Пример показывает, как создать простой `Processor`, который принимает строки, преобразует их в верхний регистр и передаёт дальше подписчику.
```java
import java.util.concurrent.Flow;
import java.util.concurrent.SubmissionPublisher;

public class ProcessorExample {
    public static void main(String[] args) throws InterruptedException {
        SubmissionPublisher<String> publisher = new SubmissionPublisher<>();
        UppercaseProcessor processor = new UppercaseProcessor();
        Flow.Subscriber<String> subscriber = new PrintSubscriber();

        // ⭐⭐Цепочка: Publisher -> Processor -> Subscriber⭐⭐
        publisher.subscribe(processor);
        processor.subscribe(subscriber);

        // Отправляем данные
        publisher.submit("reactive");
        publisher.submit("streams");
        publisher.submit("in java");

        publisher.close();
        Thread.sleep(1000);
    }

    // Преобразователь строк в верхний регистр
    static class UppercaseProcessor extends SubmissionPublisher<String>
            implements Flow.Processor<String, String> {

        private Flow.Subscription subscription;

        @Override
        public void onSubscribe(Flow.Subscription subscription) {
            this.subscription = subscription;
            subscription.request(1);
        }

        @Override
        public void onNext(String item) {
            submit(item.toUpperCase()); // публикуем преобразованный элемент
            subscription.request(1);
        }

        @Override
        public void onError(Throwable throwable) {
            throwable.printStackTrace();
            closeExceptionally(throwable);
        }

        @Override
        public void onComplete() {
            close();
        }
    }

    // Подписчик, который просто выводит данные
    static class PrintSubscriber implements Flow.Subscriber<String> {
        private Flow.Subscription subscription;

        @Override
        public void onSubscribe(Flow.Subscription subscription) {
            this.subscription = subscription;
            subscription.request(1);
        }

        @Override
        public void onNext(String item) {
            System.out.println("Получено: " + item);
            subscription.request(1);
        }

        @Override
        public void onError(Throwable throwable) {
            System.out.println("Ошибка: " + throwable.getMessage());
        }

        @Override
        public void onComplete() {
            System.out.println("Поток завершён!");
        }
    }
}


//Вывод
Получено: REACTIVE
Получено: STREAMS
Получено: IN JAVA
Поток завершён!

```
