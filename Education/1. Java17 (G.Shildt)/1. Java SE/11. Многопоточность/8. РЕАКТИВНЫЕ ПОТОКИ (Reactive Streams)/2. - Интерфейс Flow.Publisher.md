
> [!NOTE] `Flow.Publisher<T>` — это `функциональный интерфейс` из пакета `java.util.concurrent`, 
> Представляющий `издателя данных` (_publisher_) в реактивном потоке.  
> 
> Он определяет **источник данных**, который может **отправлять элементы** подписчикам (`Subscriber`) **асинхронно** и **по запросу (backpressure)**.
> 

## Пример создания
```java
// Интерфейс не имеет конструкторов
// Экземпляр создаётся путём реализации интерфейса Publisher
Flow.Publisher<T> publisher = new Flow.Publisher<>() {
    @Override
    public void subscribe(Flow.Subscriber<? super T> subscriber) {
        // Реализация
    }
};
```

## Методы
Интерфейс определяет **один единственный метод** — `subscribe(...)`.  
По сути, это **контракт между издателем и подписчиком**.

```java
// Регистрирует подписчика для получения данных.
// Publisher должен вызвать subscriber.onSubscribe(...) и передать Subscription,
// через которое будет осуществляться управление потоком данных.
void subscribe(Flow.Subscriber<? super T> subscriber);

```
🟦 **Параметры:**
- `subscriber` — объект, реализующий интерфейс `Flow.Subscriber`, который будет получать элементы, сигналы ошибок и уведомления о завершении.
    


🟩 **Контракт:**  
1️⃣ `subscribe()` вызывается **издателем** для регистрации подписчика.  
2️⃣ ❗❗❗Издатель **обязан** вызвать `subscriber.onSubscribe(Subscription)` **ровно один раз**.  
3️⃣ После этого подписчик управляет потоком через `Subscription.request(n)` — запрашивая данные порциями.  
4️⃣ Если `subscribe()` вызывается повторно тем же подписчиком — это **недопустимо** (Publisher должен выбросить `IllegalStateException`).

---
## 💡 Особенности и подводные камни 💡

1️⃣ `Publisher` — это **не источник данных сам по себе**, а _спецификация_, описывающая, _как_ нужно отправлять данные.  
Для использования требуется конкретная реализация (например, `SubmissionPublisher`).

2️⃣ 💡Поток данных **асинхронный**: `Publisher` может производить элементы в отдельном потоке.

3️⃣ ⭐⭐Важна поддержка **backpressure** — подписчик сам регулирует скорость получения данных с помощью `request(n)`.

4️⃣ ❗❗❗**Publisher не должен блокировать поток**, пока подписчик не запросит элементы.

5️⃣ 💡При ошибке передачи данных вызывается `Subscriber.onError(Throwable)` и взаимодействие завершается.

6️⃣ Все взаимодействия (`onNext`, `onError`, `onComplete`) должны происходить **последовательно** и быть **потокобезопасными**.

---

## 💎 Best Practices 💎

✅ Используй **`SubmissionPublisher`** как стандартную реализацию `Flow.Publisher`, если не нужно писать собственный Publisher.

✅ ⭐⭐Соблюдай **контракт Reactive Streams** (сторона `Subscriber`):
- Сначала `onSubscribe()`,
    
- Затем `onNext()` _n_ раз,
    
- Потом `onComplete()` или `onError()`.
    

✅ Реализуй асинхронность с помощью **ExecutorService** или `ForkJoinPool`.

✅ Не допускай выброса непроверяемых исключений из `subscribe()` — это нарушает поток данных.

✅ Если создаёшь свой Publisher — обязательно обеспечь **корректную реализацию backpressure**.


## Пример
```java
import java.util.concurrent.Flow;
import java.util.concurrent.SubmissionPublisher;

public class PublisherExample {
    public static void main(String[] args) throws InterruptedException {
        // Создаём готовую реализацию издателя
        SubmissionPublisher<String> publisher = new SubmissionPublisher<>();

        // Подписчик
        Flow.Subscriber<String> subscriber = new Flow.Subscriber<>() {
            private Flow.Subscription subscription;

            @Override
            public void onSubscribe(Flow.Subscription subscription) {
                this.subscription = subscription;
                System.out.println("Подписка получена");
                subscription.request(1); // Запрашиваем первый элемент
            }

            @Override
            public void onNext(String item) {
                System.out.println("Получено: " + item);
                subscription.request(1); // Запрашиваем следующий элемент
            }

            @Override
            public void onError(Throwable throwable) {
                System.out.println("Ошибка: " + throwable.getMessage());
            }

            @Override
            public void onComplete() {
                System.out.println("Поток завершён");
            }
        };

        // Подписываем подписчика
        publisher.subscribe(subscriber);

        // Публикуем данные
        publisher.submit("Привет");
        publisher.submit("Реактивный");
        publisher.submit("Мир");

        // Закрываем издателя
        publisher.close();

        // Даём время потоку обработать
        Thread.sleep(1000);
    }
}

//Вывод
Подписка получена
Получено: Привет
Получено: Реактивный
Получено: Мир
Поток завершён
```
