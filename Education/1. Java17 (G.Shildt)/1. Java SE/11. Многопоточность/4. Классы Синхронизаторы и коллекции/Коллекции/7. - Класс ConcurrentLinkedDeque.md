
> [!NOTE] **ConcurrentLinkedDeque** — неблокирующая (lock-free) двусторонняя очередь
> Реализует интерфейсы :`Deque<E>`, `Queue<E>`, `Collection<E>`, `Iterable<E>`, `Serializable`.

🧠Основана на **связном списке узлов (Node)**, где каждый элемент связан с предыдущим и следующим, обеспечивая доступ с обеих сторон.  
В отличие от `ConcurrentLinkedQueue`, этот класс позволяет вставку и удаление элементов **как с головы, так и с хвоста**.  

🧠Использует **атомарные операции (CAS)** для обеспечения потокобезопасности без синхронизации и блокировок.

## Конструкторы
```java
// Создаёт пустую двустороннюю очередь.
public ConcurrentLinkedDeque()

// Создаёт двустороннюю очередь, содержащую элементы из заданной коллекции.
// Элементы добавляются в порядке итерации коллекции.
// collection — исходная коллекция, чьи элементы будут скопированы.
public ConcurrentLinkedDeque(Collection<? extends E> collection)
```

## Методы
### Основные операции вставки и удаления
```java
/* Добавляет элемент в начало дека.
 * Бросает NullPointerException, если элемент равен null.
 */
public void addFirst(E e)

/* Добавляет элемент в конец дека.
 * Бросает NullPointerException, если элемент равен null.
 */
public void addLast(E e)

/* Возвращает и удаляет первый элемент дека.
 * Если дека пуста — возвращает null.
 */
public E pollFirst()

/* Возвращает и удаляет последний элемент дека.
 * Если дека пуста — возвращает null.
 */
public E pollLast()

/* Возвращает, но не удаляет первый элемент дека.
 * Если дека пуста — возвращает null.
 */
public E peekFirst()

/* Возвращает, но не удаляет последний элемент дека.
 * Если дека пуста — возвращает null.
 */
public E peekLast()
```

### Методы добавления и вставки
```java
/* Добавляет элемент в конец дека.
 * Возвращает true, если вставка успешна.
 */
public boolean add(E e)

/* Добавляет элемент в конец дека (аналог add).
 * Возвращает true, если вставка успешна.
 */
public boolean offer(E e)

/* Добавляет элемент в начало дека.
 * Возвращает true, если вставка успешна.
 */
public boolean offerFirst(E e)

/* Добавляет элемент в конец дека.
 * Возвращает true, если вставка успешна.
 */
public boolean offerLast(E e)
```

### Методы удаления
```java
/* Удаляет первый элемент дека и возвращает его.
 * Если пуст — возвращает null.
 */
public E removeFirst()

/* Удаляет последний элемент дека и возвращает его.
 * Если пуст — возвращает null.
 */
public E removeLast()

/* Удаляет первый вхождением элемента, равного переданному.
 * Возвращает true, если элемент был найден и удалён.
 */
public boolean removeFirstOccurrence(Object o)

/* Удаляет последнее вхождение элемента, равного переданному.
 * Возвращает true, если элемент был найден и удалён.
 */
public boolean removeLastOccurrence(Object o)
```

### Методы доступа
```java
/* Возвращает первый элемент без удаления.
 * Если дека пуста — бросает NoSuchElementException.
 */
public E getFirst()

/* Возвращает последний элемент без удаления.
 * Если дека пуста — бросает NoSuchElementException.
 */
public E getLast()

/* Возвращает первый элемент без удаления.
 * Если дека пуста — возвращает null.
 */
public E peek()

/* Возвращает последний элемент без удаления.
 * Если дека пуста — возвращает null.
 */
public E peekLast()
```

### Методы очистки и состояния
```java
/* Удаляет все элементы из дека.
 * После вызова становится пустым.
 */
public void clear()

/* Возвращает true, если дека не содержит элементов. */
public boolean isEmpty()

/* Возвращает количество элементов.
 * ⚠️ O(n) операция, так как требует полного обхода дека.
 */
public int size()
```

### Итераторы
```java
/* Возвращает итератор, проходящий от головы к хвосту.
 * Итератор слабоконсистентен — отражает состояние на момент создания.
 */
public Iterator<E> iterator()

/* Возвращает итератор, проходящий от хвоста к голове.
 * Также слабоконсистентен.

public Iterator<E> descendingIterator()
```

---
## 💡 Особенности и подводные камни 💡

1️⃣ **Без блокировок (lock-free)** — все операции основаны на CAS, что обеспечивает высокую производительность при большом числе потоков.  

2️⃣ **Слабая согласованность итераторов** — они не бросают `ConcurrentModificationException`, но могут не видеть новейшие изменения.  

3️⃣ **Метод size() — O(n)**, т.к. требуется обход всех узлов. Не использовать в горячем коде.  

4️⃣ **Элементы не могут быть null** — добавление `null` вызовет `NullPointerException`.  

5️⃣ **FIFO/LIFO поддержка** — можно использовать как очередь (FIFO) или стек (LIFO).  

6️⃣ **Удаление середины** не поддерживается эффективно — операции оптимизированы под начало и конец.

---
## 💎 Best Practices 💎

✅ Используй **ConcurrentLinkedDeque**, когда нужна неблокирующая двусторонняя очередь (например, планировщики задач).  

✅ Избегай частого вызова `size()` — лучше проверяй `isEmpty()` или используй счётчики.  

✅ Не изменяй структуру дека внутри итератора — он слабосогласованный и может пропустить элементы.  

✅ Подходит для **высоконагруженных сценариев**, где важны скорость и отсутствие блокировок.  

✅ Для односторонней очереди — предпочтительнее `ConcurrentLinkedQueue`.
