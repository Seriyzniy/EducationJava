# 📕1. Синхронизированные коллекции
Это ==**обычные коллекции== (например, `ArrayList`, `HashMap`)**, ==к которым **добавлена синхронизация**==. Они обеспечивают **взаимный доступ по `synchronized`**

- просто обёртка: ==**все методы синхронизированы целиком**== 🛑🛑🛑
- Это значит: ==только **один поток** может работать с Map в любой момент времени==
## Как получить:
```java
List<String> list = Collections.synchronizedList(new ArrayList<>());
Map<String, Integer> map = Collections.synchronizedMap(new HashMap<>());
Set<String> set = Collections.synchronizedSet(new HashSet<>());
```
## ⚠ Недостатки:
- **Медленные** при высокой конкуренции (все операции блокируют друг друга)
- **Итерация не потокобезопасна** — ты сам должен синхронизироваться:
```java
synchronized (list) {
    for (String item : list) {
        // безопасно
    }
}
```

# 📕 2. Параллельные коллекции (concurrent collections)
Это **специально спроектированные структуры**, которые ==**могут работать одновременно с несколькими потоками** **БЕЗ ПОЛНОЙ БЛОКИРОВКИ**==. 

Они используют:
- `CAS`
- `volatile`
- `ReentrantLock`
- `Lock striping` и др.

## 💡 Основные классы в пакете `java.util.concurrent`:💡

| Коллекция               | Интерфейс                | Особенности                                                           |
| ----------------------- | ------------------------ | --------------------------------------------------------------------- |
| `ConcurrentHashMap`     | `ConcurrentMap`          | Ключи и значения потокобезопасны, без блокировки всех потоков         |
| `CopyOnWriteArrayList`  | `List`                   | Все операции чтения безопасны, но записи создают копию                |
| `CopyOnWriteArraySet`   | `Set`                    | То же самое, основано на `CopyOnWriteArrayList`                       |
| `ConcurrentLinkedQueue` | `Queue`                  | Lock-free очередь (FIFO)                                              |
| `ConcurrentLinkedDeque` | `Deque`                  | Lock-free двусторонняя очередь                                        |
| `LinkedBlockingQueue`   | `BlockingQueue`          | Очередь с ограничением по размеру, блокирует при пустоте/переполнении |
| `ArrayBlockingQueue`    | `BlockingQueue`          | Быстрая очередь фиксированного размера                                |
| `PriorityBlockingQueue` | `BlockingQueue`          | Очередь с приоритетом и блокировкой                                   |
| `DelayQueue`            | `BlockingQueue`          | Очередь для отложенных задач                                          |
| `SynchronousQueue`      | `BlockingQueue`          | Каждая вставка должна быть сразу извлечена (нет хранилища)            |
| `ConcurrentSkipListMap` | `ConcurrentNavigableMap` | Потокобезопасный сортированный `Map`, основан на skip list            |
| `ConcurrentSkipListSet` | `NavigableSet`           | То же самое, только `Set`                                             |
