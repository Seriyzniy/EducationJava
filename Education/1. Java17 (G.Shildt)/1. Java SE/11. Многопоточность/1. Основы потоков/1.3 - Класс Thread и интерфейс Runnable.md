# 1. Класс Thread
## 1.1 Конструктор класса
```java
public class Thread implements Runnable {
    // 1. Без аргументов
	public Thread() 
	
	// 2. Runnable
	public Thread(Runnable target)
	
	// 3. Runnable + имя
	public Thread(Runnable target, String name)
	
	// 4. Имя
	public Thread(String name)
	
	// 5. ThreadGroup + Runnable
	public Thread(ThreadGroup group, Runnable target)
}
```
`ThreadGroup` — это класс из Java (java.lang.ThreadGroup), который используется для **объединения потоков в группы**.

> [!NOTE] ThreadGroup - устарел
> Сейчас принято управлять потоками через `ExecutorService`, `ForkJoinPool`, `CompletableFuture` и другие API.
> 
> В официальной документации Oracle сказано, что `ThreadGroup` считается **устаревшим механизмом управления потоками**

## Методы класса Thread
```java
// 🟢=== ОСНОВНЫЕ МЕТОДЫ ===🟢
// Запускает новый поток: вызывает native start0(), 
// который создаёт системный поток и потом вызывает run().
public synchronized void start() { /* запускает поток */ }

// Точка входа потока. По умолчанию вызывает run() у target (Runnable).
// Обычно переопределяют в наследнике или передают Runnable в конструктор.
@Override
public void run() { if (target != null) target.run(); }

// ❌ Устаревший способ убить поток насильно (опасно: ресурсы могут остаться захвачены).
@Deprecated
public final void stop() { /* устаревшее завершение потока */ }

// Устанавливает флаг прерывания потока (не убивает сразу).
public void interrupt() { /* прерывает поток */ }

// Проверяет, был ли прерван текущий поток, и СБРАСЫВАЕТ флаг прерывания.
public static boolean interrupted() { 
    return currentThread().isInterrupted(true); 
}

// Проверяет флаг прерывания данного потока, НЕ сбрасывая его.
public boolean isInterrupted() { return isInterrupted(false); }


// 🟢=== ОЖИДАНИЕ И ПАУЗЫ ===🟢
// Засыпает поток на millis миллисекунд.
// Не освобождает монитор! Может выбросить InterruptedException.
public static native void sleep(long millis) throws InterruptedException;

// То же самое, но с точностью до наносекунд.
public static void sleep(long millis, int nanos) throws InterruptedException { /* точная версия */ }

// Ждёт завершения другого потока (блокируется).
public final void join() throws InterruptedException { join(0); }

// Ждёт завершения другого потока ограниченное время (миллисекунды).
public final synchronized void join(long millis) throws InterruptedException { /* ожидание */ }

// Ждёт завершения другого потока ограниченное время (миллисек + наносекунды).
public final synchronized void join(long millis, int nanos) throws InterruptedException { /* точная версия */ }


// 🟢 === СОСТОЯНИЕ === 🟢
// true — если поток был запущен (start()), но ещё не TERMINATED.
public final native boolean isAlive(); 

// Возвращает текущее состояние потока (NEW, RUNNABLE, BLOCKED и т.д.).
public State getState() { return State.values()[getState0()]; }
private native int getState0();


// 🟢 === ПРИОРИТЕТ === 🟢
// Устанавливает приоритет планирования (от MIN_PRIORITY до MAX_PRIORITY).
public final void setPriority(int newPriority) { /* установить приоритет */ }

// Возвращает текущий приоритет.
public final int getPriority() { return priority; }


// 🟢 === ДЕМОН ===🟢
// Устанавливает поток демоном (должен быть вызван до start()).
// Демон не мешает завершению JVM.
public final void setDaemon(boolean on) { daemon = on; }

// Проверяет, является ли поток демоном.
public final boolean isDaemon() { return daemon; }


// 🟢 === СТАТИЧЕСКИЕ МЕТОДЫ === 🟢 
// Возвращает объект Thread текущего потока.
public static native Thread currentThread();

// Просит планировщик уступить CPU другим потокам (hint, может проигнорироваться).
public static native void yield();

// Количество активных потоков в текущей группе.
public static int activeCount() { 
    return currentThread().getThreadGroup().activeCount(); 
}


// 🟢 === ОТЛАДКА ===🟢 
// Имя потока.
public final String getName() { return name; }

// Устанавливает имя потока.
public final void setName(String name) { this.name = name; }

// Возвращает уникальный идентификатор потока (tid).
public long getId() { return tid; }

// Строковое представление: Thread[имя, приоритет, группа].
public String toString() { 
    return "Thread[" + getName() + "," + getPriority() + "," + getThreadGroup().getName() + "]"; 
}

// Возвращает stack trace для всех потоков (диагностика).
public static native Map<Thread, StackTraceElement[]> getAllStackTraces();


// 🟢 === ПОЛЯ === 🟢
// name — имя потока (может меняться)
// priority — приоритет
// target — Runnable для исполнения
// tid — уникальный ID
// daemon — true если поток-демон
private volatile String name;
private int priority;
private Thread target;
private long tid;
private boolean daemon = false;

// Счётчик для генерации новых имён потоков.
private static long threadSeqNumber = 0;
private static synchronized long nextThreadNum() { return ++threadSeqNumber; }


// 🟢 === СОСТОЯНИЯ ПОТОКА === 🟢 
public enum State {
    NEW,           // создан, но start() ещё не вызывался
    RUNNABLE,      // готов к выполнению или выполняется
    BLOCKED,       // ждёт монитор (synchronized)
    WAITING,       // вызван wait() без таймаута
    TIMED_WAITING, // вызван sleep(), join() или wait(timeout)
    TERMINATED     // завершён
}

```

```java
@FunctionalInterface
public interface Runnable {
    //Запускает выполнение кода в отдельном потоке.
    void run();
}
```
