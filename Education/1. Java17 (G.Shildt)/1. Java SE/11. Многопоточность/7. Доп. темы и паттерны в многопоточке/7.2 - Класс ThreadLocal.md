`ThreadLocal<T>` — это ==**переменная==, у которой ==у каждого потока будет своё собственное значение**.==

Это значит:
- ==Каждый поток== **автоматически ==получает собственную копию переменной**==
- Эта копия **недоступна другим потокам**
- Используется как **контекст потока** или **локальное хранилище** данных

Пример 1
```java
public class MyClass {
    private static final ThreadLocal<Integer> local = ThreadLocal.withInitial(() -> 0);

    public static void main(String[] args) {
        Runnable task = () -> {
            Integer value = local.get(); // каждый поток получит своё значение
            System.out.println(Thread.currentThread().getName() + ": " + value);
            local.set(value + 1);
        };

        Thread t1 = new Thread(task, "Thread-1");
        Thread t2 = new Thread(task, "Thread-2");
        t1.start();
        t2.start();
    }
}
```

Пример 2
```java
private static ThreadLocal<Connection> connectionHolder 
	= new ThreadLocal<Connection>() { 
		public Connection initialValue() { 
			return DriverManager.getConnection(DB_URL); 
		} 
	}; 

public static Connection getConnection() { 
	return connectionHolder.get(); 
}
```
Вот что происходит по шагам:
1. Поток вызывает `getConnection()`
2. Внутри вызывается `connectionHolder.get()`
3. Если **у этого потока ещё нет Connection**, вызывается `initialValue()`
4. Возвращается `DriverManager.getConnection(...)`
5. Это значение кэшируется **внутри потока** в `ThreadLocalMap`

Это просто ==**более "старый" способ задания начального значения== `ThreadLocal`** через **переопределение метода `initialValue()`**.

🛑🛑 ==С Java 8 можно использовать фабричный метод== `ThreadLocal.withInitial()` — это **более читаемо и ==предпочтительнее==**:
```java
private static ThreadLocal<Connection> connectionHolder = 
    ThreadLocal.withInitial(() -> DriverManager.getConnection(DB_URL));

```
#### Внутреннее устройство (кратко)
- Каждый поток (`Thread`) содержит поле `ThreadLocalMap`
- Когда ты вызываешь `threadLocal.set(...)`, значение записывается в этот map
- Ключ — это `ThreadLocal`-объект, значение — объект потока
***
# 📕 1. Когда и зачем использовать `ThreadLocal`

###### ✅ Используется когда:
1. ❌ **Нельзя** делить переменную между потоками
2. ✅ Нужно сохранить **независимое состояние потока**
3. ✅ Нужно сохранить **данные запроса** в веб-сервере (например, текущий пользователь)
4. ✅ Объект **не потокобезопасен**, но создание каждого раза — дорого

***
# 📕 2. Ограничения и подводные камни

## 🔶1. Утечки памяти (особенно в thread pool)

Если поток не умирает (например, в `ExecutorService`), а значение в `ThreadLocal` остаётся → **объект не будет собран GC**.

> ❗ **Обязательно** использовать `remove()` при завершении работы потока.

```java
try {
    local.set(data);
    // ...работа
} finally {
    local.remove(); // предотвращает утечку
}
```

## 🔶2. Невидимость для других потоков
- Нет никакой "магии": это не синхронизация, не `volatile`
- Это просто изолированное хранилище
- Значение нельзя передать из одного потока в другой напрямую

## 🔶3. Плохо масштабируется для разделения общего состояния

- Если тебе нужно, чтобы все потоки делили одно состояние — `ThreadLocal` не подходит
    
- ==Это **НЕ альтернатива synchronized или ReentrantLock**==
