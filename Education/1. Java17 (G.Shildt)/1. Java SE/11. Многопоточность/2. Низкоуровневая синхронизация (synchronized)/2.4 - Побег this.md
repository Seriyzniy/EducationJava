
> [!NOTE] Что значит "побег `this` (this escape)"?
> **Побег `this`** — это ситуация, при которой ==ссылка на объект (т.е. `this`) становится **доступной другим потокам до того, как конструктор== этого объекта ==завершит свою работу**.==
> 
> Проще говоря: объект ещё не полностью построен, а кто-то уже может им пользоваться.
##### ❗ Почему это плохо?
Java **не гарантирует**, что поля объекта будут инициализированы и видны другим потокам в корректном состоянии **до завершения конструктора**.

Если другой поток получит ссылку на объект до того, как он полностью инициализирован:
- Он может увидеть **"грязные данные"**
    
- Может возникнуть **race condition**
    
- Может нарушиться **согласованность объекта**

##### Пример побега
```java
public class Bad {
    public Bad() {
        new Thread(() -> {
            doSomething(); // Поток начнет работу, пока объект ещё строится
        }).start();
    }

    void doSomething() {
        // обращение к this
    }
}
```
 > ❗❗❗Поток может начать выполнение, пока объект ещё **не инициализирован**.

***
# Решение
### ✔ 1. **Не публикуй `this` из конструктора**
Не передавай `this` в другие компоненты, слушатели, обработчики или потоки в теле конструктора.

### ✔ 2. **Используй фабрику / инициализатор**

Раздели создание и регистрацию:
```java
public class Safe {
    private Safe() {
        // только конструирование
    }

    public static Safe newInstance(EventSource source) {
        Safe safe = new Safe();
        source.registerListener(safe.createListener());
        return safe;
    }

    public EventListener createListener() {
        return new EventListener() {
            public void onEvent(Event e) {
                // безопасно, объект уже построен
            }
        };
    }
}
```

### ✔ 3. **Не запускай поток сразу из конструктора**

Вместо этого:
```java
public class Worker {
    private final Thread thread;

    public Worker() {
        thread = new Thread(() -> doWork());
    }

    public void start() {
        thread.start();
    }

    private void doWork() {
        // ...
    }
}
```
