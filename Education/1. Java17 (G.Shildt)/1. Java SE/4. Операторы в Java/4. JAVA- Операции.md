# 📕1. Побитовые операции сдвига
В Java есть **три оператора сдвига**:

| Оператор | Название                         | Что делает                          |
| -------- | -------------------------------- | ----------------------------------- |
| `<<`     | Сдвиг влево                      | Умножает на `2^n`                   |
| `>>`     | Сдвиг вправо с сохранением знака | Делит на `2^n` с округлением вниз   |
| `>>>`    | Беззнаковый сдвиг вправо         | Делит на `2^n`, вставляя нули слева |
***
## 📙 Сдвиг влево `<<`

Сдвиг всех битов числа влево на `n` позиций. 
==**Каждый сдвиг влево на 1 бит** ≈ **умножение на 2**.==
```java
int x = 3;        // 00000000 00000000 00000000 00000011
int result = x << 2;
// Сдвигаем на 2 бита влево:
```
> [!NOTE] Математически ==УМНОЖАЕМ== на 2 в степени битов:
> 3\*2^2 = 3 * 4 = 12

***
## 📙 Сдвиг вправо `>>` (с сохранением знака)
```java
int x = -8;        // 11111111 11111111 11111111 11111000 (доп. код)
int result = x >> 2;
// После сдвига:
//x >> 2 = 11111111 11111111 11111111 11111110 (т.е. -2)
```
> [!NOTE] Математически ==ДЕЛИМ== на 2 в степени кол-ва битов (округляем вниз):
> -8 >> 2 == -8 / 2^2 == -2 (округляется вниз)


> [!NOTE] ВАЖНО!
> При сдвиге вправо, "открытые" при сдвиге биты, ==заполняются предыдущим содержимым старшего бита==

***
## 📙 Беззнаковый сдвиг вправо `>>>`

Работает как `>>`, но ==**всегда вставляет нули слева**==, даже если число отрицательное. Используется для работы с **битами, а не со знаками**.
```java
int x = -8;        // 11111111 11111111 11111111 11111000
int result = x >>> 2;
// Сдвиг:
//x >>> 2 = 00111111 11111111 11111111 11111110
```
