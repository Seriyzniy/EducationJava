# 📕 1. Расширения оператора switch
Начиная с JDK 14, оператор `switch` был значительно улучшен. Добавлены четыре основные возможности:

1. **Выражения `switch`** — `switch` может возвращать значение.
    
2. **Оператор `yield`** — используется для возврата значения из `switch`.
    
3. **Списки констант в `case`** — несколько значений в одном `case`.
    
4. **Оператор `case` со стрелкой (`->`)** — компактный синтаксис без сквозного выполнения.

### Пример старого `switch`
В зависимости от сгенерированного числа, мы хотим определять его приоритет:
```java
switch(eventCode) {
    case 1000:
    case 1205:
    case 8900:
        priorityLevel = 1;
        break;
    case 2000:
    case 6010:
    case 9128:
        priorityLevel = 2;
        break;
    default:
        priorityLevel = 0;
}
```
❌**Проблема:** Много шаблонного кода, легко забыть `break`, что приводит к ошибкам.

***
## 📙 1. Использование списка констант case

Вместо укладки `case` теперь ==можно писать их через запятую:==
```java
case 1000, 1205, 8900:
    priorityLevel = 1;
    break;
```
➡️ Код становится компактнее, читаемость выше.  
➡️ Но это по-прежнему «старый» `switch`, просто удобнее.

Перепишем полный ОПЕРАТОР `switch` для использования списка констант case:
```java
// В этом операторе switch применяется список констант case.
switch (eventCode) {
    case 1000, 1205, 8900:
        priorityLevel = 1;
        break;
        
    case 2000, 6010, 9128:
        priorityLevel = 2;
        break;
        
    case 1002, 7023, 9300:
        priorityLevel = 3;
        break;
        
    default:
        priorityLevel = 0; // нормальный приоритет
        break;
}
```

## 📙2. Появление ВЫРАЖЕНИЕ switch и оператора yield

Главное новшество: `switch` теперь может быть **==выражением==**, то есть ==возвращать значение.==

==Предоставить значение выражения== switch можно ==с помощью нового оператора== `yield`, который имеет следующий общий вид:
```java
yield youtrValue;
```
> [!NOTE] ⭐`yield` - `НЕМЕДЛЕННО ЗАВЕРШАЕТ switch` и возвращает значение
> 

```java
int priorityLevel = switch(eventCode) {
    case 1000, 1205, 8900: yield 1;
    case 2000, 6010, 9128: yield 2;
    default: yield 0;
    
}; //⭐Точка с запятой обязательна!!⭐
```
> [!NOTE] ⭐ Каждый оператор `case` ДОЛЖЕН ВОЗВРАЩАТЬ ЗНАЧЕНИЕ
> С применением выражения `switch` связан один важный момент: 
> - каждый оператор case (плюс default) должен производить значение (если только не генерируется исключение).

> [!NOTE] ВЫРАЖЕНИЕ `switch` - ==должно быть **ИСЧЕРПЫВАЮЩИМ**== 
> - если тип перечислимый — все `enum`-значения покрыты, 
> - либо есть `default` (для `int` и прочих значений).
>

## 📙 3. Появление стрелки (`->`) в операторе case 

Наряду с тем, что ==применение== `yield` в предыдущей программе - совершенно допустимый способ указать значение для выражения switch, он ==не является единственным==. 

Во многих ситуациях ==более простой способ предоставления значения== предусматривает использование новой формы `case`, в которой ==двоеточие заменяется стрелкой==

Вместо `:` можно писать `->`.
```java
int priorityLevel = switch(eventCode) {
    case 1000, 1205, 8900 -> 1;
    case 2000, 6010, 9128 -> 2;
    case 1002, 7023, 9300 -> 3;
    default -> 0;
    
};
```

### Блоки в `case ->`
Если нужно больше логики, можно писать:
```java
case 9300 -> {
    stopNow = true;
    System.out.println("Опасность");
    yield 3;
}

```
- ==Если== используется ==блок==, то для возврата значения ==обязателен== `yield`.
- В блоке можно делать побочные действия + возвращать итог.

### `switch` как оператор со стрелками
Можно использовать и **без выражения** (старый `switch`, но без fall-through):
```java
switch(direction) {
    case 'L' -> { System.out.println("Повернуть налево"); left++; }
    case 'R' -> { System.out.println("Повернуть направо"); right++; }
}

```
➡️ Такой `switch` НЕ ВОЗВРАЩАЕТ ЗНАЧЕНИЕ, но зато компактный и безопасный.
### Отличия от старого синтаксиса:

1. **Нет fall-through** — break не нужен и не разрешён.
    
2. Целью `->` может быть:
    - выражение (короткая форма),
        
    - блок кода `{ }` с `yield`,
        
    - `throw`.
        
3. Нельзя смешивать `:` и `->` в одном switch.

➡️ Читаемость выше, кода меньше, ошибки типа «забыл break» исчезают.
