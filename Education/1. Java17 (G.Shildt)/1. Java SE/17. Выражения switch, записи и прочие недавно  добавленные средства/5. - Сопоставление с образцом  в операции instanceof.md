# 📕 1. Сопоставление с образцом (`pattern matching`) в `instanceof`
## 📙1. Традиционный `instanceof` (до JDK 16)

1. Проверка, принадлежит ли объект к типу, или можно ли привести к нему.
2. Но если проверка успешна, то **обязательно нужно было явное приведение**:
```java
if (myOb instanceof Integer) {
    Integer iObj = (Integer) myOb; // Явное приведение
    System.out.println("Целое: " + iObj);
}
```
❌Минус: много шаблонного кода → проверка + приведение.

## 📙 2. Новый `instanceof` с шаблоном (с JDK 16)

Форма:
```java
ссылка instanceof Тип имяПеременной
```

Пример:
```java
Number myOb = Integer.valueOf(9);

if (myOb instanceof Integer iObj) {
    System.out.println("iObj указывает на число: " + iObj);
}
```
🔑 Объяснение:
- Проверяется, можно ли привести `myOb` к `Integer`.
- Если да → автоматически создаётся новая переменная `iObj` типа `Integer`.
    Если нет-> то шаблонная переменная создаваться не будет.
- Эта переменная доступна только внутри области действия `if`.

➡️ Нет явного каста, код короче и безопаснее.

## 📙 3. Область видимости шаблонной переменной

- ==Переменная доступна **только там, ГДЕ ПРОВЕРКА УЖЕ ГАРАНТИРОВАННО ПРОШЛА**.==
    
- Внутри `if` блоков, но **не в `else`**.
    
- Создаётся только при успешном совпадении с шаблоном.

# 📕 2. Шаблонные переменные в логических выражениях "И"
Операция `instanceof` может использоваться в лоrическом выражении "И': 
⭐Однако необходимо помнить о том, что шаблонная переменная попадает в область видимости только после своеrо создания.

Пример:
```java
if ((myOb instanceof Integer iObj) && (iObj >= 0)) {
    // ✅сюда попадём, ТОЛЬКО ЕСЛИ myOb — Integer и iObj неотрицателен✅
}
```

Почему работает?
- `&&` — **короткозамкнутая операция**: правая часть вычисляется только если левая true. (см теорию из книги про ускоренное вычисление булевых переменных)
    
- Значит, `iObj` уже создана и доступна во второй половине условия.

# 📕 3. Сопоставление с образцом в других операторах
`instanceof` МОЖЕТ ПРИМЕНЯТЬСЯ НЕ ТОЛЬКО В `if( )`

Его можно использовать в циклах:
```java
Object[] someObjs = {
    "Альфа", "Бета", "Омега", Integer.valueOf(10)
};

int i;
for (i = 0; (someObjs[i] instanceof String str) && (i < someObjs.length); i++) {
    System.out.println("Обработка " + str);
}
System.out.println("Количество строк: " + i);

```

То же самое можно переписать через `while`.

⚠️ В `do-while` использовать бессмысленно: переменная-шаблон будет недоступна в теле цикла, пока не выполнится проверка.
