Рассмотрим пример того, что хранит в себе ссылка типа базового интерфейса, которая ссылается на класс реализацию

Определим следующие данные
```java
interface Data{  
}  
  
class DataProduct implements Data{  
    private int id;  
    private int price;  
  
    public DataProduct(int id, int price) {  
        this.id = id;  
        this.price = price;  
    }  
  
    @Override  
    public String toString() {  
        return "DataProduct{" +  
                "id=" + id +  
                ", price=" + price +  
                '}';  
    }  
}  
  
class DataUser implements Data{  
    private int id;  
    private String username;   
  
    public DataUser(int id, String username) {  
        this.id = id;  
        this.username = username;  
    }  
  
    @Override  
    public String toString() {  
        return "DataUser{" +  
                "id=" + id +  
                ", username='" + username + '\'' +  
                '}';  
    }  
}
```

То есть, у нас есть некий интерфейс Data, от которого по разному реализуются два класса, хранящие разные данные.

Что будет хранить в себе ссылка интерфейса Data, если ей присвоить объект класса?
```java
public class Test {  
    public static void main(String[] args) {  
		
		// Создание двух классов
        DataProduct dataProduct = new DataProduct(5,500);  
        DataUser dataUser = new DataUser(7,"Tom");  
  
		// Ссылка интерфейса ссылается на dataUser
        Data data = dataUser;  
        System.out.println(
	        "Interface data link to User = " + data.toString());  
  
		// Ссылка интерфейса ссылается на dataProduct
        data = dataProduct;  
        System.out.println(
	        "Interface data link to Product = " + data.toString());  
    }  
}

Interface data link to User = DataUser{id=7, username='Tom'}

Interface data link to Product = DataProduct{id=5, price=500}
```

Как видим, ссылка data ХРАНИТ в себе данные двух разных объектов

Почему так вышло?

Когда мы пишем
```java
Data data = new DataProduct(5, 500);
```
происходит ровно следующее:

### ✔ 1. Объект создаётся как **DataProduct**
При выполнении `new DataProduct(5, 500)` ==в памяти создаётся объект== **класса DataProduct**:
```css
+---------------------------+
|   Объект DataProduct      |
|---------------------------|
| id = 5                    |
| price = 500               |
+---------------------------+
```
**Объект существует полностью** и содержит ВСЕ свои поля.


### ✔ 2. Переменная `data` — это **ссылка типа интерфейса**, а НЕ объект
У тебя две разные вещи:
- **Тип переменной**: `Data`
	
- **Тип объекта в куче**: `DataProduct`


==Тип переменной определяет _что ты можешь вызвать(МЕТОД)== через эту переменную_, 
НО **не то**, какие данные лежат внутри объекта.

То есть, ссылка data ссылается на объект DataProduct, который содержит данные
```css
data ───────────────────────► (объект типа DataProduct)
```

Переменная `data` — просто "линза", через которую ты смотришь на объект.  

==Она ограничивает доступ только методами интерфейса== (если бы они там были).

---
# ПРАВИЛО
1. **Интерфейс определяет только контракт методов.**
    
2. **Переменная интерфейсного типа ограничивает, какие методы компилятор разрешит вызвать.**
    
3. **Объект в памяти всегда остаётся объектом своего класса и содержит все поля и методы этого класса.**
    
4. **Тип переменной не изменяет структуру объекта — он влияет только на то, что видно при обращении к объекту.**
    
5. **При вызове методов всегда используется реальный тип объекта (полиморфизм), независимо от типа ссылки.**
