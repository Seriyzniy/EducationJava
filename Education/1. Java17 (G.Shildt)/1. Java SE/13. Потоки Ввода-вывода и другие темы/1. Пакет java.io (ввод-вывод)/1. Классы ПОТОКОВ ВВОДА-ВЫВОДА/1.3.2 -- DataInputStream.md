`DataInputStream` — это ==**обёртка== над `InputStream`**, которая ==позволяет читать **примитивные типы данных** (int, long, float, double и т.д.) и строки в формате **модифицированной UTF-8**.  ==

Используется совместно с `DataOutputStream` для организации _портабельного бинарного формата_, независимого от платформы (Java сама приводит байты к big-endian).

> [!NOTE] НЕТ БУФЕРА
> ⚠️ Важно: `DataInputStream` **не добавляет буферизации**, поэтому почти всегда его используют в паре с `BufferedInputStream`.

## Конструкторы
```java
public DataInputStream(InputStream in)
// in — любой поток ввода, откуда читаем данные.
// ⭐⭐Обычно оборачивают FileInputStream или BufferedInputStream.⭐⭐
```
## Методы
```java
// Читает ровно 1 байт, возвращает int (0–255).
// -1 если конец потока (EOF).
public final int read() throws IOException

// Читает в массив b[] данные.
// Возвращает количество реально прочитанных байт или -1 при EOF.
public final int read(byte[] b, int off, int len) throws IOException

// Читает ровно len байтов в массив b[].
// Если EOF наступил раньше — выбрасывает EOFException.
public final void readFully(byte[] b) throws IOException
public final void readFully(byte[] b, int off, int len) throws IOException

// Пропускает n байтов, возвращает фактическое число пропущенных.
// Может быть меньше n, если достигнут EOF.
public final int skipBytes(int n) throws IOException


// ✅ ЧТЕНИЕ БАЙТОВ И ПРЕОБРАЗОВАНИЕ В ПРИМИТИВ ✅
// Читает boolean (1 байт). 0 → false, любое ≠ 0 → true.
public final boolean readBoolean() throws IOException

// Читает 1 байт и возвращает его как signed byte (-128..127).
public final byte readByte() throws IOException

// Читает 1 байт и возвращает как unsigned (0..255).
public final int readUnsignedByte() throws IOException

// Читает 2 байта, возвращает short (-32768..32767).
// Использует big-endian.
public final short readShort() throws IOException

// Читает 2 байта как unsigned short (0..65535).
public final int readUnsignedShort() throws IOException

// Читает 2 байта, возвращает char (UTF-16 код символа).
public final char readChar() throws IOException

// Читает 4 байта, возвращает int.
public final int readInt() throws IOException

// Читает 8 байт, возвращает long.
public final long readLong() throws IOException

// Читает 4 байта, интерпретирует как IEEE 754 float.
public final float readFloat() throws IOException

// Читает 8 байт, интерпретирует как IEEE 754 double.
public final double readDouble() throws IOException


// Читает строку в "модифицированной UTF-8".
// Сначала читает 2 байта (длина), затем строку.
// Не совместимо с обычным UTF-8.
public final String readUTF() throws IOException
public static final String readUTF(DataInput in) throws IOException
```

***
## 💡 Особенности 💡

- `readFully(...)` гарантированно читает указанное число байтов или кидает `EOFException`. В отличие от `read(...)`, который может вернуть меньше.
    
- `readUTF()` использует **модифицированный UTF-8** (не путать с обычным), что делает бинарный формат совместимым между JVM, но не всегда с другими языками.
    
- Все примитивы читаются в **big-endian** порядке — стандарт в Java.

***
## Пример
```java
import java.io.*;

public class DataInputStreamExample {
    public static void main(String[] args) {
        File file = new File("data.bin");

        try (DataOutputStream dos = new DataOutputStream(new FileOutputStream(file))) {
            // Записываем данные
            dos.writeInt(42);
            dos.writeUTF("Hello");
            dos.writeDouble(3.14);
        } catch (IOException e) {
            e.printStackTrace();
        }

        try (DataInputStream dis = new DataInputStream(
                new BufferedInputStream(new FileInputStream(file)))) {
            // Читаем в том же порядке
            int number = dis.readInt();
            String text = dis.readUTF();
            double value = dis.readDouble();

            System.out.println("int = " + number);
            System.out.println("String = " + text);
            System.out.println("double = " + value);
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

## Best-Practices
- ✅ **Всегда** оборачивай `DataInputStream` в `BufferedInputStream` для производительности.
    
- ✅ Используй `readFully()` вместо `read()`, если тебе нужно гарантированно получить весь блок данных.
    
- ✅ Следи за порядком записи/чтения: `writeInt → readInt`, `writeUTF → readUTF`. Несоответствие приведёт к `EOFException` или мусору.
    
- ❌ Не используй `readUTF()` для обмена с системами на других языках (лучше пиши длину строки и обычный UTF-8 байтами).
    
- ✅ Закрывай ресурсы через try-with-resources.
