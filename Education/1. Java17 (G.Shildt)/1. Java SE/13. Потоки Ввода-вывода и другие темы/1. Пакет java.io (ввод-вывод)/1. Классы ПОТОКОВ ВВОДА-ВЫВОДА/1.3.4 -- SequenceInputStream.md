`SequenceInputStream` — это специальный ==поток, который **объединяет несколько входных потоков (`InputStream`) в один логический**.==

- Когда читаем данные → сначала читается первый поток до конца.
    
- Когда он исчерпался (возвратил `-1`), `SequenceInputStream` автоматически переключается на следующий.
    
- Так продолжается, пока не будут прочитаны все потоки.
    

> [!NOTE] ⭐НЕСКОЛЬКО отдельных ПОТОКОВ ЧИТАЕТ КАК ОДИН целый⭐
> Используется, когда нужно представить **несколько файлов/буферов/источников** как один непрерывный поток (например, при склеивании файлов).

## Конструкторы
```java
// Создаёт последовательный поток из двух InputStream.
// Сначала читается s1, затем s2.
public SequenceInputStream(InputStream s1, InputStream s2)

// ⭐Создаёт поток из "произвольного числа" потоков.
// elements — это Enumeration<InputStream> (старый стиль, до коллекций).
// Каждый элемент — отдельный InputStream.
public SequenceInputStream(Enumeration<? extends InputStream> elements)
```

## Методы
```java
// Читает один байт (0–255).
// Возвращает -1, если все потоки закончились.
// Автоматически переключается на следующий поток, когда текущий исчерпан.
public int read() throws IOException

// Читает до len байтов в массив b[], начиная с позиции off.
// Возвращает количество реально прочитанных байтов или -1 при EOF.
// Работает точно так же, как read(), но с массивом.
public int read(byte[] b, int off, int len) throws IOException

// Закрывает текущий поток и все остальные, когда они будут достигнуты.
// После вызова close() дальнейшее чтение невозможно.
public void close() throws IOException
```

***
## 💡💡 Особенности 💡💡
- ⚡ Работает **лениво**: переключается на следующий поток только после того, как предыдущий вернёт `-1`.
    
- ⚡ `Enumeration` — это старый API, ==можно обернуть== `Vector<InputStream>` ==или использовать== ⭐`Collections.enumeration(...)`.⭐
    
- ❌ Не поддерживает `mark/reset`.
    
- ==❌ Не потокобезопасен.==

***
## Пример
```java
import java.io.*;
import java.util.*;


public static void main(String[] args) {
	try {
		// ✅ Два источника
		ByteArrayInputStream b1 = new ByteArrayInputStream("Hello ".getBytes());
		ByteArrayInputStream b2 = new ByteArrayInputStream("World!".getBytes());

		// Объединение через 1-й конструктор
		try (SequenceInputStream sis = new SequenceInputStream(b1, b2)) {
			int ch;
			while ((ch = sis.read()) != -1) {
				System.out.print((char) ch);
			}
		}

		System.out.println();

		// ✅ Объединение через Enumeration
		ByteArrayInputStream b3 = new ByteArrayInputStream("Java ".getBytes());
		ByteArrayInputStream b4 = new ByteArrayInputStream("Stream".getBytes());
		
		// Объединение в вектор
		Vector<InputStream> v = new Vector<>();
		v.add(b3);
		v.add(b4);

		try (SequenceInputStream sis = new SequenceInputStream(v.elements())) {
			int ch;
			while ((ch = sis.read()) != -1) {
				System.out.print((char) ch);
			}
		}

	} catch (IOException e) {
		e.printStackTrace();
	}
}

Hello World!
Java Stream
```

## Best-Practices

- ✅ Используй `SequenceInputStream`,== если нужно склеить **несколько файлов** или потоков в один ==(например, объединить логи, части файла, куски бинарных данных).
    
- ✅ ⭐⭐Если есть список потоков (`List<InputStream>`), можно легко обернуть его в `Enumeration` через `Collections.enumeration(list)`.⭐⭐
    
- ❌ ==Не стоит использовать для **очень большого числа потоков**== — закрытие происходит по мере чтения, но `Enumeration` может хранить длинный список.
    
- ✅ Для современных приложений лучше рассмотреть NIO (`Files.newInputStream`, `Channels`, `Stream API`) — они гибче.
