`FileReader` — это **символьный поток ввода**, который предназначен для чтения текста из файла.  

Он является **специализированным наследником** `InputStreamReader`, который создаёт поток чтения символов из файла с использованием **кодировки по умолчанию** системы.

> [!NOTE] ⭐ПРОСТОЙ СПОСОБ читать текстовый файл с кодировкой по умолчанию
> 📌 Это упрощённый способ читать текстовые файлы без необходимости вручную создавать `FileInputStream` и `InputStreamReader`.  
> 
> Но стоит помнить: `FileReader` ==**не позволяет указывать кодировку**== — он всегда использует кодировку по умолчанию JVM.
> 
> ⭐⭐⭐Один вызов `read()` ==может вернуть целый символ, который состоит из нескольких байт.==

***
## Конструкторы
```java
// Создаёт FileReader для файла по имени
// fileName — путь к файлу
public FileReader(String fileName) throws FileNotFoundException

// Создаёт FileReader для объекта File
public FileReader(File file) throws FileNotFoundException
```

## Методы
`FileReader` наследует все методы от `InputStreamReader`:
```java
// Читает один символ
public int read() throws IOException

// Читает массив символов
public int read(char[] cbuf, int off, int len) throws IOException

// Проверяет готов ли поток к чтению
public boolean ready() throws IOException

// Пропускает символы
public long skip(long n) throws IOException

// Закрывает поток
public void close() throws IOException

// Проверяет поддержку mark/reset
public boolean markSupported()
```

***
## Особенности
- ⭐Работает **только с текстовыми файлами**.
    
- Кодировка — **по умолчанию** системы, что может привести к проблемам при переносе файлов между системами с разными кодировками.
    
- Удобен для простого чтения текстовых файлов без необходимости писать лишний код.
    
- ⭐Не подходит для бинарных файлов.
    
- ⭐Если нужен контроль над кодировкой — лучше использовать `InputStreamReader` поверх `FileInputStream`.

***
## Пример
```java
import java.io.*;

public class FileReaderExample {
    public static void main(String[] args) {
        try (FileReader fr = new FileReader("example.txt")) {
            int ch;
            while ((ch = fr.read()) != -1) {
                System.out.print((char) ch);
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

***
## Best-Practices

- ✅ Использовать `FileReader` для быстрого чтения небольших текстовых файлов, когда кодировка по умолчанию подходит.
    
- ✅ Закрывать поток через try-with-resources.
    
- ✅ Для больших файлов — оборачивать в `BufferedReader` для повышения производительности.
    
- ⚠️ Не использовать для бинарных файлов.
    
- ⚠️ Если требуется специфическая кодировка — использовать `InputStreamReader` вместо `FileReader`.
