
`PipedReader` — это символьный поток ввода, который используется **в паре с `PipedWriter`** для организации канала передачи данных между потоками внутри одной программы.  

Обычно применяется, когда нужно, чтобы один поток **писал символы**, а другой поток их **читал**.

## Конструкторы
```java
PipedReader()
// Создает не подключённый PipedReader. 
// Его нужно связать с PipedWriter с помощью connect().

PipedReader(int pipeSize)
// Создает канал с буфером заданного размера (по умолчанию 1024 символа).
// pipeSize — размер буфера.

PipedReader(PipedWriter src)
// 💡Создает PipedReader и сразу соединяет его с указанным PipedWriter.
// src — поток для записи.

PipedReader(PipedWriter src, int pipeSize)
// 💡Создает PipedReader, соединенный с PipedWriter, и задает размер буфера.
// src — поток для записи.
// pipeSize — размер буфера.
```

## Методы
```java
int read()
// Читает один символ. Возвращает -1, если поток закрыт и данные закончились.

int read(char[] cbuf, int off, int len)
// Читает символы в массив cbuf, начиная с позиции off, не более len символов.
// Возвращает количество реально прочитанных символов или -1 при конце потока.

void close()
// Закрывает поток и освобождает ресурсы.
```

***
## Особенности

- ✅Работает **только в связке** с `PipedWriter`.
    
- Используется **для межпоточного обмена** данными.
    
- **Буфер** по умолчанию — 1024 символа (можно задать свой).
    
- `read()` блокируется, если буфер пуст.
- `write()` в `PipedWriter` блокируется, если буфер полон.
    
- ❌Нельзя использовать в одном и том же потоке для записи и чтения — приведёт к `IOException`.
    
- ❌Если не вызвать `writer.close()`, то `reader.read()` может бесконечно ждать данных.

***
## Пример
```java
import java.io.*;

public class PipedReaderExample {
    public static void main(String[] args) throws Exception {
	    // ✅Создание потока-канала на запись и чтение
        PipedWriter writer = new PipedWriter();
        PipedReader reader = new PipedReader(writer);

        Thread producer = new Thread(() -> {
            try {
                String msg = "Привет из PipedWriter!";
                for (char c : msg.toCharArray()) {
                    writer.write(c);
                    Thread.sleep(100); // имитация поступления данных
                }
                writer.close(); // важно закрыть поток
            } catch (Exception e) {
                e.printStackTrace();
            }
        });

        Thread consumer = new Thread(() -> {
            try {
                int data;
                while ((data = reader.read()) != -1) {
                    System.out.print((char) data);
                }
            } catch (Exception e) {
                e.printStackTrace();
            }
        });

        producer.start();
        consumer.start();
    }
}

```

***
## Best-Practices

- ✅ Использовать **разные потоки** для записи и чтения.
    
- ✅ Всегда закрывать `PipedWriter`, чтобы `PipedReader` корректно завершил чтение.
    
- ✅ Подумать о размере буфера: если данных много, лучше увеличить `pipeSize`.
    
- ⚠️ Не использовать в однопоточном коде — приведёт к зависанию.
    
- ⚠️ ⭐⭐⭐В новых проектах часто проще использовать **`BlockingQueue`**, которая более гибкая и удобная для межпоточного обмена.⭐⭐
