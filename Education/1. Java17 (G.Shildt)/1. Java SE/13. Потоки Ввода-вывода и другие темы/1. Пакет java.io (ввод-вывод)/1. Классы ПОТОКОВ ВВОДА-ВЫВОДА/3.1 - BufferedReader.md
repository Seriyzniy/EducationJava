`BufferedReader` — это ==**декоратор для чтения символов**==, который оборачивает другой `Reader` ==и добавляет **буферизацию**==.  

Главная цель — уменьшить количество обращений к низкоуровневому источнику символов (например, файл, сокет) и повысить производительность.

💡 Особенности:
- Буфер по умолчанию — 8192 символа.
    
- ⭐==Позволяет читать текст **ПОСТРОЧНО** с помощью метода== `readLine()`.
    
- Работает только с символами (`Reader` → символьный поток).
    
- Используется поверх любых других `Reader` (например, `FileReader`, `InputStreamReader`).

***
## Конструкторы
```java
// Оборачивает заданный Reader с 💡буфером размером по умолчанию (8192 символа)
public BufferedReader(Reader in)

// Оборачивает заданный Reader 💡с указанным размером буфера
// sz — размер буфера в символах
public BufferedReader(Reader in, int sz)
```

## Методы
```java
// Читает один символ и возвращает его как int (0–65535), или -1 при EOF.
public int read() throws IOException

// Читает массив символов.
public int read(char[] cbuf, int off, int len) throws IOException

// ✅Читает СИМВОЛЫ ДО КОНЦА СТРОКИ✅, возвращает строку без символа перевода строки.
// Возвращает null при EOF.
public String readLine() throws IOException


// Пропускает указанный символов
public long skip(long n) throws IOException

// Проверяет, готов ли поток к чтению (не блокирует)
public boolean ready() throws IOException

// Закрывает поток и освобождает ресурсы
public void close() throws IOException

// Сбрасывает буфер чтения
public void mark(int readAheadLimit) throws IOException

// Возвращает true, если поддерживается mark/reset
public boolean markSupported()

// Возвращает поток в место, где было сделано mark()
public void reset() throws IOException
```

***
## 💡 Особенности💡

- **Буферизация** → меньше обращений к низкоуровневому источнику.
    
- ✅`readLine()` полезен для парсинга текстовых файлов.
    
- `mark()` и `reset()` позволяют вернуться к ранее прочитанным символам (если `markSupported()` = true).
    
- ✅Поддерживает разные кодировки через обёртки типа `InputStreamReader`.
    
- ❌ Не синхронизирован (работа в многопотоке требует внешней синхронизации).

***
## Пример
```java
import java.io.*;

public class BufferedReaderExample {
    public static void main(String[] args) {
        try (BufferedReader br = new BufferedReader(
                new FileReader("example.txt"))) {

            String line;
            while ((line = br.readLine()) != null) {
                System.out.println(line);
            }

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```
📄 Этот код читает файл `example.txt` построчно и печатает его содержимое.

***
## Best-Practices

- ✅ Всегда оборачивать низкоуровневые `Reader` в `BufferedReader`, если нужен построчный ввод или высокая производительность.
    
- ✅ Закрывать поток через try-with-resources.
    
- ✅ Использовать `readLine()` для удобного чтения текстовых файлов.
    
- ⚠️ Не забывать про кодировку — лучше использовать `InputStreamReader` с явной кодировкой.
    
- ⚠️ Проверять `markSupported()` перед использованием `mark()` и `reset()`.
    
- ⚠️ Не использовать `BufferedReader` напрямую для бинарных данных — он предназначен для текста.
