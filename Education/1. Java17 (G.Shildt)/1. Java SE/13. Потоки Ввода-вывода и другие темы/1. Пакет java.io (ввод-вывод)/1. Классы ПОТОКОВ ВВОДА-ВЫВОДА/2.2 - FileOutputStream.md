`FileOutputStream` — это ==**прямой поток вывода В ФАЙЛ**==.  

Используется, когда нужно:
- ==записывать данные в файл== (в бинарном виде);
- создавать новые файлы или перезаписывать существующие;
- дописывать данные в конец файла (append mode).
    
📌 В отличие от `ByteArrayOutputStream`, данные не буферизуются в памяти (если сам не добавишь `BufferedOutputStream`), а пишутся напрямую в файловую систему.

***
## Конструкторы
```java
// Создаёт поток, связанный с файлом по имени.
// Если файла нет — создаётся новый.
// Если файл существует — содержимое затирается.
public FileOutputStream(String name) throws FileNotFoundException

// То же самое, но через объект File.
public FileOutputStream(File file) throws FileNotFoundException

// ✅Открывает файл для записи.
// Если append = true → запись идёт в конец файла.
// Если append = false → файл перезаписывается.
public FileOutputStream(String name, boolean append) throws FileNotFoundException
public FileOutputStream(File file, boolean append) throws FileNotFoundException

// Создаёт поток для записи в указанный дескриптор (нечасто используется).
public FileOutputStream(FileDescriptor fdObj)
```

## Методы
```java
// Записывает один байт (младшие 8 бит int).
// param b — байт для записи
public void write(int b) throws IOException

// Записывает весь массив байтов.
// param b — массив данных
public void write(byte[] b) throws IOException

// Записывает часть массива байтов.
// param b — массив данных
// param off — смещение в массиве
// param len — сколько байтов писать
public void write(byte[] b, int off, int len) throws IOException

// ✅Принудительно сбрасывает буфер ОС на диск.
//  ✅⭐Гарантирует, что данные реально записаны в файл.⭐
public void flush() throws IOException

// Закрывает поток и освобождает файловый дескриптор.
public void close() throws IOException

// Возвращает файловый дескриптор.
// 🧠Может быть полезно для низкоуровневой работы.
public final FileDescriptor getFD() throws IOException
```

***
## 💡 Особенности 💡
- ✅ При создании может **создать новый файл** или **стереть существующий** (если не append).
    
- ✅ В append-режиме новые данные добавляются в конец файла.
    
- ✅💡 Без `flush()` (💡или `close()`💡) данные могут остаться в буфере ОС и не попасть в файл сразу.
    
- ❌ Поток небезопасен для многопоточной записи — нужна синхронизация сверху.
    
- ✅При работе лучше всегда использовать **try-with-resources**, чтобы гарантировать закрытие.

***
## Пример
```java
import java.io.*;

public class FileOutputStreamExample {
    public static void main(String[] args) {
        // Запись в файл (перезапись, если файл существует)
        try (FileOutputStream fos = new FileOutputStream("output.txt")) {
            String text = "Hello, FileOutputStream!\n";
            fos.write(text.getBytes("UTF-8")); // пишем строку в UTF-8
            fos.flush(); // сброс данных на диск
        } catch (IOException e) {
            e.printStackTrace();
        }

        // Запись с дозаписью (append = true)
        try (FileOutputStream fos = new FileOutputStream("output.txt", true)) {
            fos.write("Appending new line!\n".getBytes("UTF-8"));
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

## Best-Practices
- ✅ Используй `try-with-resources`, чтобы гарантировать закрытие.
    
- ✅ Всегда явно указывай кодировку при `getBytes("UTF-8")`.
    
- ✅ Для **частой записи маленькими кусками** лучше оборачивать в `BufferedOutputStream` для повышения производительности.
    
- ❌ Не использовать напрямую в многопоточной среде без синхронизации.
    
> [!NOTE] `flush` для немедленной записи
> ❌ ⭐⭐Не забывать про `flush()`, если данные нужно немедленно сбросить (например, при логгировании).
