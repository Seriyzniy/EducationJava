`InputStreamReader` — это **мост между байтовыми потоками и символьными потоками**.  

> [!NOTE] Из байта в символ
> Он читает байты из заданного `InputStream` и преобразует их в символы, используя указанную кодировку (Charset). 
> 
> 📌 Это ключевой класс, когда нужно читать текстовые данные из байтового источника (файл, сеть, сокет), так как он заботится о декодировании байтов в символы.
> 
> Они **декодируют** байты в символы, используя алгоритмы конкретной кодировки.
> 
> ⭐⭐⭐Один вызов `read()` ==может вернуть целый символ, который состоит из нескольких байт.==

Особенности:
- Работает поверх `InputStream`.
    
- Поддерживает указание кодировки (Charset), иначе используется системная кодировка по умолчанию.
    
- Позволяет читать данные как символы (`Reader`).
    
- Критически важен для работы с текстом в разных кодировках (UTF-8, ISO-8859-1, Windows-1251 и т.д.).

***
## Конструкторы
```java
// Создаёт InputStreamReader с кодировкой по умолчанию.
public InputStreamReader(InputStream in)

// Создаёт InputStreamReader с указанной кодировкой.
// charsetName — имя кодировки, например "UTF-8", "Windows-1251"
public InputStreamReader(InputStream in, String charsetName) throws UnsupportedEncodingException

// Создаёт InputStreamReader с указанной кодировкой Charset.
public InputStreamReader(InputStream in, java.nio.charset.Charset cs)
```

## Методы
```java
// Читает один символ (возвращает int, 0–65535) или -1 при EOF.
public int read() throws IOException

// Читает символы в массив.
// cbuf — массив для записи символов
// off — смещение в массиве
// len — количество символов для чтения
public int read(char[] cbuf, int off, int len) throws IOException

// Проверяет готов ли поток к чтению (не блокирует)
public boolean ready() throws IOException

// Пропускает указанное количество символов
public long skip(long n) throws IOException

// Закрывает поток и освобождает ресурсы
public void close() throws IOException

// Проверяет поддержку mark/reset
public boolean markSupported()
```

***
## ⭐ Особенности ⭐

- ⭐Преобразует **байты в символы** с учётом кодировки.
    
- Если кодировка указана неверно → будет `UnsupportedEncodingException`.
    
- Кодировка влияет на то, как символы преобразуются из байтов.
    
- ⭐Поддерживает только одностороннее преобразование (байты → символы).
    
- ⭐Очень важен при работе с файлами и потоками в сетевых приложениях.
    
- ⭐⭐Для обратного преобразования используется `OutputStreamWriter`.
***
## Пример
```java
import java.io.*;

public class InputStreamReaderExample {
    public static void main(String[] args) {
        try (InputStreamReader isr = new InputStreamReader(
                new FileInputStream("example.txt"), "UTF-8")) {

            char[] buffer = new char[1024];
            int numChars;
            while ((numChars = isr.read(buffer)) != -1) {
                System.out.print(new String(buffer, 0, numChars));
            }

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```
📄 Этот код читает файл `example.txt`, используя кодировку UTF-8, и выводит его содержимое в консоль.

## Best-Practices

- ✅ Явно указывать кодировку при создании `InputStreamReader`, чтобы избежать проблем на разных системах.
    
- ✅ Оборачивать в `BufferedReader` для повышения производительности и удобного чтения строк.
    
- ✅ Использовать try-with-resources для автоматического закрытия потока.
    
- ✅ ⭐Для больших файлов читать частями через буфер, а не символ за символом.
    
- ⚠️ Не использовать без явной кодировки, если файл/поток имеет специфическую кодировку.
    
- ⚠️ ⭐Не использовать напрямую для бинарных данных — это класс для текстовых данных. 
⭐Если ты передаёшь ==**чистые бинарные данные** (например, картинку, видео,== архив) в `InputStreamReader`, он **попробует их декодировать как текст**
