# 📕 1. Основа Ввода-Вывода
### Потоки данных (Streams)
==Ввод-вывод== в программах на Java ==выполняется через потоки данных==. 

> [!NOTE] Поток
> Поток данных (stream) - это абстракция, которая либо производит, либо потребляет информацию. 

Поток связан с физическим устройством посредством системы ввода-вывода Java. 
==Все потоки ведут себя одинаково==, даже если фактические физические устройства, с которыми они связаны, различаются.


Существует два типа потоков:
1. 🔶**Байтовые потоки** (классы `InputStream`/`OutputStream`) - для работы с двоичными данными
    
2. 🔶**Символьные потоки** (классы `Reader`/`Writer`) - для работы с текстом, используют Unicode

> [!NOTE] 🧠На самом низком уровне все операции ввода-вывода по-прежнему ориентированы на байты

### Предопределенные БАЙТОВЫЕ потоки 
```java
System.in → стандартный ввод (обычно клавиатура, тип `InputStream`).

System.out → стандартный вывод (обычно консоль, тип `PrintStream`).

System.err → вывод ошибок.
```
- `Systern.in` представляет собой объект типа `InputStream`
- `System.out` и `System.err` - объекты типа` PrintStream`. 

Это потоки байтовых данных, хотя обычно они применяются для чтения символов из консоли и записи символов на консоль.
## 📙 1.1 Классы потоков БАЙТовых данных
Потоки байтовых данных определяются с применением двух иерархий классов. Вверху находятся два абстрактных класса: `InputStream` и `Outputstream`. 

Каждый из них имеет несколько конкретных подклассов, которые справляются с отличиями между разными устройствами

Помните, что для использования классов потоков должен быть импортирован пакет `java.io`.

| Класс                    | Назначение                                                                                     |
| ------------------------ | ---------------------------------------------------------------------------------------------- |
| ⭐**InputStream**         | Абстрактный базовый класс для всех потоков ввода байтов.                                       |
| ⭐**OutputStream**        | Абстрактный базовый класс для всех потоков вывода байтов.                                      |
| ==--------------------== |                                                                                                |
| ✔BufferedInputStream     | Буферизованный поток ввода — ускоряет чтение, так как использует внутренний буфер.             |
| ✔BufferedOutputStream    | Буферизованный поток вывода — ускоряет запись, используя внутренний буфер.                     |
| ==--------------------== |                                                                                                |
| ✔ByteArrayInputStream    | Читает байты из массива в памяти.                                                              |
| ✔ByteArrayOutputStream   | Записывает байты в массив в памяти.                                                            |
| ==--------------------== |                                                                                                |
| ✔DataInputStream         | Позволяет читать примитивные типы данных Java (int, double, char и т. д.) из байтового потока. |
| ✔DataOutputStream        | Позволяет записывать примитивные типы данных Java в байтовый поток.                            |
| ==--------------------== |                                                                                                |
| ✔FileInputStream         | Читает байты из файла.                                                                         |
| ✔FileOutputStream        | Записывает байты в файл.                                                                       |
| ==--------------------== |                                                                                                |
| FilterInputStream        | Абстрактный класс-надстройка над `InputStream` (родитель для "фильтрующих" потоков).           |
| FilterOutputStream       | Абстрактный класс-надстройка над `OutputStream`.                                               |
| ==--------------------== |                                                                                                |
| ✔ObjectInputStream       | Читает объекты из потока (десериализация).                                                     |
| ✔ObjectOutputStream      | Записывает объекты в поток (сериализация).                                                     |
| ==--------------------== |                                                                                                |
| 💡PipedInputStream*      | Входной канал, используется для связи с `PipedOutputStream` (межпоточный обмен данными).       |
| 💡PipedOutputStream      | Выходной канал, используется для связи с `PipedInputStream`.                                   |
| ==--------------------== |                                                                                                |
| PrintStream              | Вывод с удобными методами `print()` и `println()`.                                             |
| ==--------------------== |                                                                                                |
| PushbackInputStream      | Позволяет "откатить" байты назад в поток (например, при парсинге).                             |
| SequenceInputStream      | Объединяет несколько потоков ввода в один (читаются последовательно).                          |

> [!NOTE] ⭐ Два главных метода Input/OutputStream⭐
> В абстрактных классах InputStream и OutputStream определено несколько ключевых методов, реализуемых другими классами потоков. 
> 
> Двумя наиболее важными из них являются 
> - `read()` - чтение  байтов данных
> - `write()`- запись байтов данных.
