## 📙 2. Закрытие ресурсов
1. ✅ Традиционный  
```java
try { 
	... 
} finally { 
	close(); 
}
```
При этом `close()` также может бросать `IOException`, поэтому код может быть громоздким.

3. ✅ Современный _try-with-resources_ 
```java
try (Resource r = ...) {
	 ... 
}
``` 
где ресурс реализует `AutoCloseable`. 
Рекомендуется применять в новом коде — автоматическое закрытие, короче и безопаснее.
см [[10.4  - Try с ресурсами]]

# 📕 3. Исключения ввода-вывода

В обработке ввода-вывода важную роль играют два исключения.
- 🔶`IOException` — основной базовый тип для ошибок ввода-вывода.
    
- 🔶`FileNotFoundException` — частный случай при открытии файла (подкласс `IOException`).

❓❗Дополнительное исключение
- `SecurityException` — при нарушениях политик безопасности (если есть `SecurityManager`; в современных JDK он устаревает, но в некоторых окружениях ещё встречается). Для работы с консолью могут возникать `IOError` (подкласс `Error`).
