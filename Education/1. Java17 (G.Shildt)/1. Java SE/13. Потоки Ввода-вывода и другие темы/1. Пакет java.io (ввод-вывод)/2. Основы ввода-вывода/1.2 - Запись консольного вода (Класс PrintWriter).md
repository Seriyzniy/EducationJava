Консольный вывод проще всего обеспечить с помощью описанных ранее методов `print()` и `println ()` , которые используются в большинстве примеров книги. 
# 1. Класс PrintWriter
Упомянутые методы определены в классе `PrintStream` (тип объекта, на который ссылается System. out).

Поскольку класс `PrintStream` представляет собой выходной поток, производный от `OutputStream`, он также реализует низкоуровневый метод `write()`
```java
void write(int byteval)
```
> [!NOTE] 🧠В аргументе `int` но записываются младшие 8 бит
> 

==В реальных программах на Java рекомендуется осуществлять запись в консоль посредством потока PrintWriter== - одного из символьных классов. 
Использование символьного класса для консольного вывода упрощает интернационализацию программы.

#### Конструкторы
1. 🔶Через `Writer`
```java
PrintWriter(Writer out)
PrintWriter(Writer out, boolean autoFlush)

//✅Exmaple
PrintWriter pw = new PrintWriter(new FileWriter("out.txt"), true);
pw.println("Hello, file!"); //⭐Записывает в поток, указанный в параметре
```
Принимает **любой символьный поток** (например, `BufferedWriter` или `FileWriter`).

2. 🔶Через `OutputStream`
```java
PrintWriter(OutputStream out)
PrintWriter(OutputStream out, boolean autoFlush)

//✅Exmaple
PrintWriter pw = new PrintWriter(System.out, true);
pw.println("Привет, консоль!");
```
Принимает **байтовый поток** (`FileOutputStream`, `Socket.getOutputStream()` и т.д.).

3. 🔶Через `File`
```java
PrintWriter(File file) 
	throws FileNotFoundException

PrintWriter(File file, String charsetName) 
	throws FileNotFoundException, UnsupportedEncodingException

//✅Exmaple
PrintWriter pw = new PrintWriter(new File("log.txt"), "UTF-8");
pw.println("Запись в файл с UTF-8");
pw.close();
```
- Открывает файл для записи.
- Можно указать **кодировку** (например, `"UTF-8"`).
- Если файла нет → будет создан новый.

> [!NOTE] 🧠Параметры
> Здесь `outputStream` является объектом типа OutputStream, 
> а `flushingOn` определяет, ==будет ли поток вывода очищаться при каждом вызове метода== `println()` (среди прочих). 
> 
> Если в аргументе `flushingOn` указано значение `true`, тогда очистка выполняется автоматически, а если `false`, то очистка автоматически не происходит.
