
> [!NOTE] Интерфейсы `FilenameFilter` и `FileFilter`
> Оба интерфейса — это ==**фильтры при получении списка файлов/каталогов**== через `File.list(...)` и `File.listFiles(...)`. 
> 
> Они ==позволяют отсечь ненужные элементы== прямо на уровне API, вместо того чтобы вручную пробегать по всему массиву.

Разница:
- 🟢**`FilenameFilter`** работает с _именами файлов_ (строки) и знает ещё про родительский каталог.
    
- 🟠**`FileFilter`** работает сразу с объектами `File`.

`FilenameFilter` исторически появился раньше (JDK 1.0), `FileFilter` — в JDK 1.2 как более удобная и объектно-ориентированная альтернатива.


*** 
## Интерфейс `FilenameFilter`
```java
@FunctionalInterface
public interface FilenameFilter {
    /**
     * dir   каталог, в котором выполняется фильтрация
     * name  имя файла или подкаталога внутри dir
     * @return true если файл должен быть включён в результат, false если исключить
     */
    boolean accept(File dir, String name);
}
```

## Интерфейс `FileFilter`
```java
@FunctionalInterface
public interface FileFilter {
    /**
     * pathname File-объект (файл или каталог), для которого вызывается фильтр
     * @return true если оставить, false если исключить
     */
    boolean accept(File pathname);
}
```

***

## Особенности и подводные камни
- **Возврат значения — критичен.** Если забыть вернуть `true`, результат всегда пустой (типичный баг новичков).
    
- ❗❗**Разница в параметрах:**
    - `FilenameFilter` ==не даёт напрямую доступа к объекту== `File` (только имя и родитель). Если нужно узнать размер/тип файла, придётся создать новый `File(dir, name)` → лишняя работа.
        
    - `FileFilter` ==сразу даёт== `File` → проще.
        
- **Лямбды в Java 8+.** Оба интерфейса функциональные, значит можно передавать `lambda` вместо анонимных классов.
    
- **Кодировки и регистр.** Сравнение имён (`name.endsWith(".txt")`) зависит от файловой системы: Windows регистронезависимая, Unix/Linux — чувствительна к регистру.
    
- ❗❗ **`list()` vs `listFiles()`.**
    - 💡 `list(FilenameFilter)` возвращает `String[]`.💡
        
    - 💡`listFiles(FileFilter)` возвращает `File[]`. 💡
        Выбирайте исходя из того, что вам нужно.
        
- Если фильтр возвращает `false` для всего — метод вернёт пустой массив, а не `null`. Но `list()` может вернуть `null`, если сам объект `File` не указывает на каталог.
