# 📕 1.  Интерфейс DataInput


> [!NOTE] `java.io.DataInput`
> Интерфейс для ==**чтения примитивов и строк** из бинарного потока== в машинно-независимом формате.  
> 
> 💡Основная реализация — `DataInputStream`.

То есть: `DataOutputStream.writeInt(42)` → потом `DataInputStream.readInt()` даст то же самое, независимо от платформы (всегда Big Endian).

## Интерфейс DataInput
```java
public interface DataInput {
    // Читает один байт и возвращает boolean (0 = false, ≠0 = true)
    boolean readBoolean() throws IOException;

    // Читает один байт как signed byte (-128..127)
    byte readByte() throws IOException;

    // Читает один байт и возвращает как char (0..255)
    char readChar() throws IOException;

    // Читает 2 байта (Big Endian)
    short readShort() throws IOException;

    // Читает 2 байта беззнаково (0..65535)
    int readUnsignedShort() throws IOException;

    // Читает 1 байт беззнаково (0..255)
    int readUnsignedByte() throws IOException;

    // Читает 4 байта как int
    int readInt() throws IOException;

    // Читает 8 байт как long
    long readLong() throws IOException;

    // Читает 4 байта IEEE 754
    float readFloat() throws IOException;

    // Читает 8 байт IEEE 754
    double readDouble() throws IOException;

    // Читает строку из "modified UTF-8"
    String readUTF() throws IOException;

    // Читает заданное количество байтов в массив
    void readFully(byte[] b) throws IOException;

    // Читает в часть массива
    void readFully(byte[] b, int off, int len) throws IOException;

    // Пропускает n байтов
    int skipBytes(int n) throws IOException;

    // Читает строку устаревшим способом (зависит от кодировки) — не рекомендуется
    @Deprecated
    String readLine() throws IOException;
}
```

## 💡 Особенности и подводные камни DataInput 💡

1. ❗❗**Формат должен совпадать с DataOutput.**  
    Если ты написал через `writeInt` → читаешь через `readInt`.  
    Иначе байты «съедутся» и дальше будет мусор.
    
2. ❗❗**readUTF — только для парных writeUTF.**  
    Это _modified UTF-8_. Обычный UTF-8 файл так не прочитаешь.
    
3. 🧠**readLine устарел.** Не использовать — лучше `BufferedReader.readLine()`.
    
4. **EOF.** Если байтов не хватает — будет `EOFException`. Это нормально, нужно обрабатывать.
    
5. ❗❗**readFully vs обычное чтение.**  
    `readFully` ждёт, пока не заполнит массив (или кинет EOF). `read(byte[])` может вернуть меньше байтов.

## Пример
```java
import java.io.*;

public class DataInputExample {
    public static void main(String[] args) throws IOException {
        try (DataOutputStream out = new DataOutputStream(
	        new FileOutputStream("data.bin"))
	    ) {
            out.writeInt(42);
            out.writeDouble(3.14);
            out.writeUTF("Привет");
        }

        try (DataInputStream in = new DataInputStream(
	        new FileInputStream("data.bin"))
	    ) {
            int n = in.readInt();
            double d = in.readDouble();
            String s = in.readUTF();
            System.out.println(n + " " + d + " " + s);
        }
    }
}
```

# 📕 2.  Интерфейс ObjectInput


> [!NOTE] `java.io.ObjectInput` расширяет `DataInput` 
> Добавляет ==возможность **читать объекты**==, `сериализованные` через `ObjectOutput`.  
> 💡Основная реализация — `ObjectInputStream`.

## Интерфейс ObjectInput
```java
public interface ObjectInput extends DataInput, AutoCloseable {
    // Читает объект (он должен быть Serializable/Externalizable)
    Object readObject() throws ClassNotFoundException, IOException;

    // Закрывает поток
    void close() throws IOException;

    // Пропускает n байтов
    int skipBytes(int n) throws IOException;

    // Доступные байты
    int available() throws IOException;
}
```

## Особенности и подводные камни ObjectInput

1. **readObject — потенциально опасен.**  
    Можно получить RCE (Remote Code Execution), ==если десериализовать данные откуда-то снаружи== (атака через gadget chains). Всегда фильтруйте допустимые классы (`ObjectInputFilter` в Java 9+).
    
2. ❗❗**Граф объектов.**  
    При чтении восстанавливается не только сам объект, но и весь его граф ссылок.
    
3. **serialVersionUID.**  
    Если он не совпадает между версией класса и сериализованными данными → `InvalidClassException`.
	
4. ❗❗**Конструкторы не вызываются.**  
	При десериализации поля восстанавливаются напрямую, мимо конструктора. Инварианты могут нарушиться.

## Пример
```java
import java.io.*;

class User implements Serializable {
    private static final long serialVersionUID = 1L;
    String name;
    int age;
    User(String n, int a) { name = n; age = a; }
}

public class ObjectInputExample {
    public static void main(String[] args) throws Exception {
        try (ObjectOutputStream out = new ObjectOutputStream(
	        new FileOutputStream("user.bin"))
	    ) {
            out.writeObject(new User("Alice", 25));
        }

        try (ObjectInputStream in = new ObjectInputStream(
	        new FileInputStream("user.bin"))
	    ) {
            User u = (User) in.readObject();
            System.out.println(u.name + " " + u.age); // Alice 25
        }
    }
}
```
