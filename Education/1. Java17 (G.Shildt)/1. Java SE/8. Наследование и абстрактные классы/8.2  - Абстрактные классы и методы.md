Иногда может случиться ситуация, когда базовый класс не имеет определенного смысла, а лишь определяет сигнатуру методов, которые реализуют классы наследники

Для этого используются `абстрактные методы`

# 📕 1. Ключевое слово `abstract`
Модификатор `abstract` гарантирует, что классы наследники реализуют методы суперкласса, помеченные как абстрактные.

Абстрактные методы не содержат реализации

> [!NOTE] Объяви класс абстрактным, если есть хотя бы один абстрактный метод
> ==Любой класс==, содержащий ==1 или несколько абстрактных методов==, должен быть ==объявлен как абстрактный==
> 
> При этом, они могут иметь и обычные методы с реализацией

> [!NOTE] Объект абстрактного класса и его конструктор
> При этом, создать ОБЪЕКТ АБСТРАКТОГО класса НЕВОЗМОЖНО
> 
> Однако у него ==может быть конструктор==, который можно **использовать в классах наследниках** ==для инициализации полей==
```java
abstract class A {
	static int x;
	
    A (int a) {
	    System.out.println("Inside constructor A");
	    x = a;
    }
    
    abstract void sayHello();
}

class B extends A {
    B () {
    	super(5); //Вызов конструктора абстрактнго класса для инициализации полей
        System.out.println("Inside constructor B");
    }

	@Override
	void sayHello() {
		System.out.println(x);
	}
}

// Main
public class MyMain {
	public static void main(String[] args) {
		A a = new B();
		a.sayHello();
	}
}

//Вывод
Inside constructor A
Inside constructor B
5
```
