
> [!NOTE] `BinaryOperator<T>`
> Частный случай `BiFunction<T,T,T>`. 
> - Принимает `два аргумента ОДНОГО типа` и возвращает тот же тип.

## Интерфейс
```java
@FunctionalInterface
public interface BinaryOperator<T> extends BiFunction<T, T, T> {
    // ⭐Наследует apply(T t1, T t2)

    // Возвращает оператор, выбирающий минимальное значение по компаратору
    static <T> BinaryOperator<T> minBy(Comparator<? super T> comparator);

    // Возвращает оператор, выбирающий максимальное значение по компаратору
    static <T> BinaryOperator<T> maxBy(Comparator<? super T> comparator);
}
```

## 💡 Особенности 💡
- Применяется для операций свёртки (`reduce` в Stream API).
    
- Часто используют `Integer::sum`, `Math::max`, `Math::min`.

## Пример
Метод `apply` наследуется от `BiFunction`
```java
BinaryOperator<Integer> sum = Integer::sum;
System.out.println(sum.apply(10, 20)); // 30

BinaryOperator<Integer> min = BinaryOperator.minBy(Integer::compareTo);
System.out.println(min.apply(5, 8)); // 5
```
