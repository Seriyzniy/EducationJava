
> [!NOTE] Тип параметра метода должен относится к типу функи-го интерфейса
> Чтобы лямбда-выражение можно было передавать как аргумент, ==тип параметра, получающего аргумент в форме лямбда==-выражения, ==должен относиться к типу функционального интерфейса==, который совместим с лямбда-выражением.

Лямбда-выражения можно передавать в методы:
```java
interface StringFunc {
    String func(String n);
}

class LambdasAsArgumentsDemo {
	//Метод принимающий в аргумент лямбду
    static String stringOp(StringFunc sf, String s) {
        return sf.func(s);
    }
    
    public static void main(String[] args) {
        String inStr = "Hello World";
        
        // Простое лямбда-выражение
        String outStr = stringOp((str) -> str.toUpperCase(), inStr);
        
        // Блочное лямбда-выражение
        outStr = stringOp((str) -> {
            String result = "";
            for(int i = 0; i < str.length(); i++)
                if(str.charAt(i) != ' ')
                    result += str.charAt(i);
            return result;
        }, inStr);
    }
}
```
