
Лямбда-выражение - синтаксический сахар анонимного класса, а ПОД КАПОТОМ - `Функциональный объект`
```java
interface ISum {
	public int sum(int a, int b);
}

class MyClass{
	public static void main(String[] args){
		ISum isum1 = new ISum(){
			@Ovveride
			public int sum(int a, int b){
				return a + b;
			}	
		};
		
		//Лямба-выражние позволяет сократить объявление анонимного класса
		ISum isum2 = (a, b) -> a + b;
		
		// Компилятор:
		// 1. ISum isum2 == ISum (true)
		// Совпадает ли ссылка isum2 с типом функционального интерфейса?
		
		// 2. ISum.sum(int a, int b) == (int a, int b) 
		// Проверяет, есть ли в лямба два аргумента (a, b), как в интерфейсе
		// и подставляет к лямбде (int a, int b)
		
		// 3. int ISum.sum(...) == a + b
		// Сверяет возвращаемое зачение в лямбде и в методе интерфейса
		
		// ✅⭐Лямбда может быть многострочной НО ТОГДА ОБЯЗАТЕЛЬНО return
		ISum isum3 = (a, b) -> {
			if (a == 0) return 0; 
			return a + b;
		}
	}
}
```

# 📕 1. Как работают анонимные классы
```java
Consumer<String> myI = new Consumer<>() {
    @Override
    public void accept(String target) {
        System.out.println("hello");
    }
};
```

==Компилятор **создаёт новый анонимный класс** ==(в байткоде он будет называться примерно `MyMain$1.class`).  
Поэтому при компиляции ты видишь:
- `MyMain.class` — основной класс;
    
- `MyMain$1.class` — сгенерированный анонимный класс.
    
То есть реально создаётся **новый .class файл**, который хранит байткод с `accept`.

# 📕 2. Как работают лямбды (начиная с Java 8)
```java
Consumer<String> myI = System.out::println;
// или
Consumer<String> myI = s -> System.out.println(s);
```

Здесь компилятор **НЕ ГЕНЕРИРУЕТ НОВЫЙ .class файл**, а использует механизм `invokedynamic` из JVM (появился в Java 7).

![[Pasted image 20250925172305.png]]

Что происходит:
1. В байткоде твоего `MyMain.class` появляется специальная 💡инструкция `invokedynamic`.
    
2. Эта инструкция указывает на фабричный метод (`LambdaMetafactory.metafactory`), который во время **загрузки класса в JVM** создаёт объект-функцию.
    
3. Для оптимизации JIT может:
    
    - сгенерировать **маленький synthetic-класс** прямо в памяти (но не как отдельный `.class` файл на диске),
        
    - или даже обойтись без отдельного класса, используя динамические прокси через `MethodHandle`.
        

Именно поэтому на диске появляется только **один .class файл** (`MyMain.class`), хотя в рантайме объект-функция действительно существует.
