Как демонстрировалось в предшествующих примерах, можно получать поток из коллекции (что на самом деле делается часто), 
но 
временами желательно обратное: ==получить коллекцию из потока==. 

Для выполнения такого действия обычно используется метод `collect()`

# 📕1. Основные формы метода collect()
```java
// Форма 1: С использованием Collector
<R, A> R collect(Collector<? super T, A, R> collectorFunc)
```

В `R` указывается тип результата, 
в `T` - тип элемента вызывающего потока, 
в `А` - внутренний накопленный тип, 
а в `CollectorFunc` - способ работы процесса накопления. 

Метод `collect()` является заключительной операцией.

## Интерфейсе Collector
```java
/**
 * Интерфейс для реализации операций сбора (reduction) элементов потока
 * в изменяемый контейнер-результат
 *
 * @param <T> тип входных элементов
 * @param <A> тип изменяемого аккумулятора (промежуточный тип накопления)
 * @param <R> тип конечного результата
 */
public interface Collector<T, A, R> {
    
    /**
     * Возвращает ПОСТАВЩИК (factory) ДЛЯ СОЗДАНИЯ новых экземпляров аккумулятора
     * Вызывается для создания пустого контейнера перед началом накопления
     * 
     * @return функция, создающая новый пустой аккумулятор
     */
    Supplier<A> supplier();
    
    /**
     * Возвращает функцию ДЛЯ ДОБАВЛЕНИЯ ЭЛЕМЕНТА в аккумулятор
     * Вызывается для каждого элемента потока в процессе накопления
     * 
     * @return функция, добавляющая элемент в аккумулятор
     */
    BiConsumer<A, T> accumulator();
    
    /**
     * Возвращает функцию для объединения двух аккумуляторов
     * 💡Используется в параллельных операциях для слияния частичных результатов
     * 
     * @return функция, объединяющая два аккумулятора в один
     */
    BinaryOperator<A> combiner();
    
    /**
     * Возвращает функцию финального преобразования аккумулятора в результат
     * Вызывается после обработки всех элементов для получения конечного значения
     * 
     * @return функция, преобразующая аккумулятор в конечный результат
     */
    Function<A, R> finisher();
    
    /**
     * Возвращает характеристики коллектора, определяющие его поведение
     * 
     * @return набор характеристик (UNORDERED, CONCURRENT, IDENTITY_FINISH)
     */
    Set<Characteristics> characteristics();

    /**
     * Фабричный метод для создания коллектора с полным набором функций
     * 
     * @param supplier поставщик аккумулятора
     * @param accumulator аккумуляторная функция
     * @param combiner комбинирующая функция
     * @param finisher финализатор
     * @param characteristics характеристики коллектора
     * @return созданный коллектор
     */
    static <T, A, R> Collector<T, A, R> of(
        Supplier<A> supplier,
        BiConsumer<A, T> accumulator,
        BinaryOperator<A> combiner,
        Function<A, R> finisher,
        Characteristics... characteristics
    ) {
        // реализация фабричного метода
    }

    /**
     * Фабричный метод для создания коллектора с тождественным финализатором
     * (когда аккумулятор可以直接作为结果使用)
     * 
     * @param supplier поставщик аккумулятора
     * @param accumulator аккумуляторная функция
     * @param combiner комбинирующая функция
     * @param characteristics характеристики коллектора
     * @return созданный коллектор
     */
    static <T, R> Collector<T, R, R> of(
        Supplier<R> supplier,
        BiConsumer<R, T> accumulator,
        BinaryOperator<R> combiner,
        Characteristics... characteristics
    ) {
        // реализация с Function.identity() в качестве finisher
    }
}
```

```java
/**
 * Характеристики, определяющие поведение коллектора
 */
enum Characteristics {
    /**
     * Указывает, что коллектор не сохраняет порядок элементов
     * Позволяет оптимизировать параллельное выполнение
     */
    UNORDERED,
    
    /**
     * Указывает, что аккумулятор может безопасно использоваться конкурентно из нескольких потоков
     */
    CONCURRENT,
    
    /**
     * Указывает, что функция finisher является тождественной и может быть опущена
     */
    IDENTITY_FINISH
}
```
## Использование предопределенных коллекторов (Collectors)
```java
// Сбор в List
static <T> Collector<T, ?, List<T>> toList()

// Сбор в Set  
static <T> Collector<T, ?, Set<T>> toSet()
```

## Пример: преобразование потока объектов в коллекции
```java
class NamePhoneEmail {
	String name;
	String phonenum;
	String email;

	NamePhoneEmail(String n, String p, String e) {
		name = n;
		phonenum = p;
		email = e;
	}
}

class NamePhone {
	String name;
	String phonenum;

	NamePhone(String n, String p) {
		name = n;
		phonenum = p;
	}
}


public static void main(String[] args) throws InterruptedException {

	ArrayList<NamePhoneEmail> myList = new ArrayList<>();

	myList.add(new NamePhoneEmail(
		"Larry", "555-5555", "Larry@HerbSchildt.com"));

	myList.add(new NamePhoneEmail(
		"James", "555-4444", "James@HerbSchildt.com"));

	myList.add(new NamePhoneEmail(
		"Mary", "555-3333", "Mary@HerbSchildt.com"));

	myList.stream()
		.forEach((e) -> System.out.println(e.name + " " + e.email + " " + e.phonenum));

	/// Исходный поток объектов NamePhone
	Stream<NamePhone> nameAndPhone = myList.stream()
		.map((a) -> new NamePhone(a.name, a.phonenum));


	// Сбор в List
	List<NamePhone> npList = nameAndPhone.collect(Collectors.toList());
	
	// ❗❗❗ ИЛИ ❗❗❗
	// После сбора в List, поток уже закрыт 
	// Сбор в Set
	
	//Set<NamePhone> npSet = nameAndPhone.collect(Collectors.toSet());
	
}
```

# 📕2. Расширенная форма collect() с явным указанием коллекции
```java
// Форма 2: С явным указанием функций
<R> R collect(Supplier<R> target, 
              BiConsumer<R, ? super T> accumulator,
              BiConsumer<R, R> combiner)
```

```java
// Создание конкретного типа коллекции (LinkedList)
LinkedList<NamePhone> npList = nameAndPhone
	.collect(
		// поставщик коллекции
	    () -> new LinkedList<>(), 
	     // добавление элемента
	    (list, element) -> list.add(element),
	    // объединение коллекций 
	    (listA, listB) -> listA.addAll(listB)  
);

// Альтернатива с ссылками на методы
HashSet<NamePhone> npSet = nameAndPhone.collect(
    HashSet::new,
    HashSet::add, 
    HashSet::addAll
);
```

## Особенности функций в collect()

- **Supplier** - создает новую коллекцию (`() -> new LinkedList<>()`)
    
- **Accumulator** - добавляет элемент в коллекцию (`(list, element) -> list.add(element)`)
    
- **Combiner** - объединяет две коллекции (`(listA, listB) -> listA.addAll(listB)`)
