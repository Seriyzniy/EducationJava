# 📕 1. parallelStream VS stream.parallel
## 1. `parallelStream()` метод `Collection`

Метод `parallelStream()` — это **==метод интерфейса== `Collection`**, который создаёт **параллельный поток** напрямую из коллекции.  

То есть, операции над элементами будут выполняться **в нескольких потоках**, если это возможно.
```java
List<Integer> numbers = List.of(1, 2, 3, 4, 5);

// создаётся сразу параллельный поток
numbers.parallelStream()
       .map(n -> n * n)
       .forEach(System.out::println);
```

## 2. `stream().parallel()` методы `Stream`

Метод `stream()` создаёт **последовательный поток**,  

а вызов `parallel()` **переводит этот поток в параллельный режим**
```java
List<Integer> numbers = List.of(1, 2, 3, 4, 5);

// сначала последовательный, затем переключаем в параллельный режим
numbers.stream()
       .parallel()
       .map(n -> n * n)
       .forEach(System.out::println);
```

## Разница

> [!NOTE] ❗❗НЕТ ГАРАНТИИ ПОРЯДКА❗❗
> Оба вывода будут непредсказуемо перемешаны, так как параллельные потоки **не гарантируют порядок** при `forEach()`.


|Критерий|`parallelStream()`|`stream().parallel()`|
|---|---|---|
|**Источник**|Работает только с коллекциями (`Collection` API)|Работает с любым `Stream`|
|**Создание**|Сразу создаёт параллельный поток|Сначала создаётся последовательный поток, потом переведен в параллельный|
|**Читаемость**|Коротко, удобно, когда с самого начала нужен параллельный поток|Гибко — можно начать с последовательного потока и по условию сделать параллельным|
|**Производительность**|Почти идентична|Почти идентична|
|**Используется когда**|Известно, что нужен параллельный поток сразу|Когда решение о параллельности принимается динамически|

```java
List<Integer> numbers = IntStream.rangeClosed(1, 10).boxed().toList();

// способ 1 — сразу параллельный
numbers.parallelStream()
       .map(n -> n * n)
       .forEach(System.out::println);

// способ 2 — сначала обычный поток, потом переключение
numbers.stream()
       .parallel()
       .map(n -> n * n)
       .forEach(System.out::println);
```

# 📕 2. Последовательное vs Параллельное исполнение в Stream API

## 1. Последовательное исполнение (`stream()`)

```java
  ┌──────────────┐
  │  Источник    │  → map → filter → reduce → Результат
  └──────────────┘
```

🔹 Поток создаётся **в одном основном потоке выполнения** (обычно — main).  

🔹 Каждый элемент обрабатывается **поочерёдно**, **в порядке следования в источнике**.
```java
List<Integer> numbers = List.of(1, 2, 3, 4, 5);

numbers.stream()
       .map(n -> n * n)
       .forEach(System.out::println);
```

Все операции (`map`, `filter`, `forEach`) выполняются **в одном потоке**, последовательно —  то есть `1→2→3→4→5`.

## 2. Параллельное исполнение (`parallelStream()` или `stream().parallel()`)

```java
┌──────────────┐
│ Источник     │
└─────┬────────┘
      │ разделение (split)
 ┌────┴─────┐
 │ Fork1     │ → map → filter → ...
 │ Fork2     │ → map → filter → ...
 │ Fork3     │ → map → filter → ...
 └────┬──────┘
      │ объединение (combine)
      ▼
   Результат
```

🔹 Поток создаётся **параллельным** — ==элементы== коллекции ==**делятся на подзадачи**== и обрабатываются **в нескольких потоках одновременно**.  

🔹 💡Используется **ForkJoinPool.commonPool** — общий пул потоков из класса `java.util.concurrent`.
```java
List<Integer> numbers = List.of(1, 2, 3, 4, 5);

numbers.parallelStream()
       .map(n -> n * n)
       .forEach(System.out::println);
```
Элементы обрабатываются **в разных потоках** и **в произвольном порядке**.  
Порядок вывода может быть: `4, 9, 1, 25, 16` — потому что результат зависит от того, какой поток закончит первым.
