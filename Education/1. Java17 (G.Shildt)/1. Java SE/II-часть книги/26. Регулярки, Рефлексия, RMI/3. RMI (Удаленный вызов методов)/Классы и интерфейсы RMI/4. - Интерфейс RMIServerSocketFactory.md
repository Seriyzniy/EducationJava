
> [!NOTE] `RMIServerSocketFactory` — это **интерфейс** из пакета `java.rmi.server`
> который определяет `фабрику` для `создания серверных сокетов`, используемых RMI `при приёме входящих соединений` от клиентов.
> 
> Его основная цель — ==кастомизация сетевого взаимодействия== `на стороне сервера`, например:
> - использование SSL (через `SSLServerSocketFactory`);
> - задание нестандартных портов или IP-интерфейсов;
> - контроль над параметрами сокета (timeout, reuse address и т.д.); 
> - реализация протокольного фильтра (например, ограничение IP клиентов).

Этот интерфейс реализуют разработчики, когда нужно, чтобы RMI-сервер использовал **особый тип сокета** для приёма соединений.

## Интерфейс
```java
public interface RMIServerSocketFactory
```

## Методы

### Создание серверного сокета
```java
ServerSocket createServerSocket(int port) throws IOException;
```
- **Назначение:** ==создаёт серверный сокет==, который ==RMI будет использовать== для приёма входящих соединений.
    
- **Параметры:**
    - `port` — порт, на котором сервер будет слушать входящие соединения. Если `port == 0`, система выберет свободный порт автоматически.
        
- 💡**Возвращает:** объект `java.net.ServerSocket`, который RMI использует для приёма соединений.
    
- **Исключения:** `IOException` — если при создании сокета произошла ошибка.

---
## 💡 Особенности и подводные камни 💡

1️⃣ ⭐⭐⭐**Сериализация обязательна.**  
RMI передает фабрику серверных сокетов клиенту как часть дескриптора удаленного объекта,  
чтобы клиент знал, с каким типом сокета он должен взаимодействовать (через соответствующий `RMIClientSocketFactory`).

2️⃣ ⭐⭐⭐**Пара фабрик (client/server) должна быть согласована.**  
Если сервер использует нестандартный `RMIServerSocketFactory`, то клиент должен иметь совместимый `RMIClientSocketFactory`.  
Иначе клиент не сможет установить соединение.

3️⃣ ⭐⭐⭐**equals() и hashCode() обязательны к переопределению.**  
RMI использует эти методы, чтобы сравнивать фабрики при экспорте объектов.  
Например, два объекта с одинаковыми фабриками будут слушать один и тот же порт.

4️⃣ 💡**Контроль над портами и безопасностью.**  
Можно ограничить подключение только с определённых IP-адресов или использовать защищённые соединения (например, SSL).

5️⃣ ❗❗❗**Используется при экспорте объекта.**  
Указывается как параметр в `UnicastRemoteObject.exportObject(...)` наряду с `RMIClientSocketFactory`.

---
## 💎 Best Practices 💎

✅ Реализуй `equals()` и `hashCode()` корректно (иначе RMI создаст лишние слушающие сокеты).  

✅ ❗❗❗Настраивай параметры `ServerSocket` (например, `setReuseAddress(true)` или `setSoTimeout(...)`).  

✅ Используй `SSLServerSocketFactory` для защищённых соединений.  

✅ ⭐⭐Убедись, что выбранный порт открыт в firewall.  

✅ Если создаёшь несколько экспортов — переиспользуй фабрику, а не создавай каждый раз новую.

## Пример
```java
import java.io.IOException;
import java.net.InetAddress;
import java.net.ServerSocket;
import java.rmi.server.RMIServerSocketFactory;
import java.rmi.server.UnicastRemoteObject;

public class CustomServerSocketFactory implements RMIServerSocketFactory, java.io.Serializable {

    private InetAddress bindAddress;
    private int backlog;

    // Пояснение:
    // Конструктор задаёт IP-адрес для привязки и размер очереди подключений
    public CustomServerSocketFactory(InetAddress bindAddress, int backlog) {
        this.bindAddress = bindAddress;
        this.backlog = backlog;
    }

    // Пояснение:
    // ⭐⭐Создаёт серверный сокет, привязанный к конкретному IP и с заданной очередью
    @Override
    public ServerSocket createServerSocket(int port) throws IOException 
    {
        return new ServerSocket(port, backlog, bindAddress);
    }

    // Пояснение:
    // Сравнение фабрик по параметрам
    @Override
    public boolean equals(Object obj) {
        if (this == obj) return true;
        if (!(obj instanceof CustomServerSocketFactory)) return false;
        CustomServerSocketFactory other = (CustomServerSocketFactory) obj;
        return backlog == other.backlog && bindAddress.equals(other.bindAddress);
    }

    // Пояснение:
    // Возвращает хэш по IP и очереди
    @Override
    public int hashCode() {
        return bindAddress.hashCode() ^ backlog;
    }
}
```

🔶Использование на **сервере**:
```java
RMIClientSocketFactory csf = new CustomClientSocketFactory(5000);

RMIServerSocketFactory ssf = new CustomServerSocketFactory(InetAddress.getByName("0.0.0.0"), 50);

MyRemoteImpl obj = new MyRemoteImpl();

// Экспорт удалённого объекта с кастомными фабриками
UnicastRemoteObject.exportObject(obj, 0, csf, ssf);

// Регистрация в реестре
Naming.rebind("rmi://localhost/MyService", obj);
```

🔶 Использование на **клиенте**:
```java
MyRemote stub = (MyRemote) Naming.lookup("rmi://server/MyService");

stub.remoteMethod();
```
