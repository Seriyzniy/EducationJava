
> [!NOTE] Класс `java.rmi.registry.LocateRegistry` 
> Используется для 
> - нахождения, 
> - создания и получения ссылки на удалённый реестр RMI (`Registry`)
> 
> Этот класс **не реализует интерфейсов** и `содержит` `только` `статические фабричные методы` для взаимодействия с RMI-реестром.

## Фабричные методы
```java
// ⭐Получает ссылку на удалённый реестр⭐ по имени хоста и порту
public static Registry getRegistry(
	String host, 
	int port) throws RemoteException

// Упрощённая версия: получает ссылку на реестр на локальной машине по указанному порту
public static Registry getRegistry(int port) throws RemoteException

// Упрощённая версия: получает ссылку на реестр на локальной машине по умолчанию (порт 1099)
public static Registry getRegistry() throws RemoteException



// 💡Создаёт и экспортирует💡 локальный реестр на указанном порту
public static Registry createRegistry(int port) throws RemoteException

// Создаёт и экспортирует локальный реестр на указанном порту с заданной фабрикой сокетов
public static Registry createRegistry(
	int port, 
	RMIClientSocketFactory csf, 
	RMIServerSocketFactory ssf) throws RemoteException
```

- **`getRegistry`** — возвращает **stub** (прокси) реестра, не гарантируя, что реестр действительно существует. Проверка происходит только при вызове методов на объекте `Registry`.
    
- 💡**`createRegistry`** — **создаёт настоящий экземпляр реестра** (в памяти) и **экспортирует его**, делая доступным для удалённых клиентов.
    
- ⭐⭐**`csf` / `ssf`** — фабрики клиентских и серверных сокетов, применяются ==при необходимости зашифрованного== или кастомного сетевого взаимодействия.

## Методы
Класс НЕ СОДЕРЖИТ **нестатических публичных методов**, так как его ==назначение — служить точкой входа к объекту== `Registry`.

---
## 💡 Особенности и подводные камни 💡

1️⃣ **`getRegistry()` НЕ ПРОВЕРЯЕТ подключение.**  
Возвращает объект-заглушку, даже если реестр не запущен. Ошибка проявится только при обращении к методам `lookup()` или `list()`.

2️⃣ 💡**Порт по умолчанию — 1099.**  
Если реестр создаётся или ищется без указания порта, используется именно он.

3️⃣ ❗❗❗**Реестр можно создать только один раз на один порт.**  
Повторный вызов `createRegistry(1099)` приведёт к `ExportException`.

4️⃣ ❗❗❗**Безопасность.**  
Для работы RMI требуется, чтобы политика безопасности (`java.security.policy`) позволяла экспорт и связывание удалённых объектов.

5️⃣ 🧠❗**Доступ из-за NAT или FireWall может блокироваться.**  
RMI использует TCP-порты, поэтому в продакшене часто добавляют SSH-туннелирование или порт-форвардинг.

---
## 💎 Best Practices 💎

✅ ⭐⭐⭐При создании реестра в коде сервера ВСЕГДА ИСПОЛЬЗОВАТЬ `createRegistry()`, а не `rmiregistry` из консоли — это упрощает запуск.  

✅ 💡Проверять доступность реестра через `list()` после `getRegistry()`.  

✅ Для надёжности задавать порт явно.  

✅ 🧠Избегать "магического" порта 1099 в продакшене — использовать значение из конфигурации.  

✅ ⭐❗Для шифрования трафика применяйте `RMIClientSocketFactory` / `RMIServerSocketFactory`.

## Пример
```java
import java.rmi.registry.LocateRegistry;
import java.rmi.registry.Registry;

public class RMIServer {
    public static void main(String[] args) {
        try {
            // Создание реестра на порту 1099
            Registry registry = LocateRegistry.createRegistry(1099);

            // Регистрация удалённого объекта
            MyRemoteImpl obj = new MyRemoteImpl();
            registry.rebind("MyService", obj);

            System.out.println("RMI Registry запущен и объект зарегистрирован!");
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

// Клиентская часть
public class RMIClient {
    public static void main(String[] args) {
        try {
            // Получение ссылки на реестр
            Registry registry = LocateRegistry.getRegistry("localhost", 1099);

            // Поиск объекта
            MyRemote stub = (MyRemote) registry.lookup("MyService");
            System.out.println(stub.sayHello());
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

```
