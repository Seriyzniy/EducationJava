
> [!NOTE] Интерфейс `java.rmi.registry.Registry` — это ==центральный интерфейс== RMI, который представляет **удалённый объект-реестр**.  
> 
> Реестр RMI выступает как **служба именования**, где `хранятся пары`
>  `имя → удалённый объект (stub)`.
> 
> 🔶Клиенты используют этот интерфейс, чтобы **искать удалённые объекты по имени**, а 🔸серверы — чтобы **регистрировать их**.
> 
> 💡`Registry` **наследуется от `Remote`**, что делает все его методы **удалёнными** (вызываемыми через сеть).

## Интерфейс
```java
public interface Registry extends Remote {
    
    // Константа порта по умолчанию
    public static final int REGISTRY_PORT = 1099;
    
    // ================================
    // 🟢 Работа с именами в реестре 🟢
    // ================================
    
    // ⭐⭐Возвращает удалённый объект, связанный с указанным именем.
    // name — строка, под которой зарегистрирован удалённый объект.
    // Возвращает Remote stub.
    // Бросает NotBoundException, если имя не связано с объектом.
    public Remote lookup(String name)
        throws RemoteException, NotBoundException, AccessException;
    
    // 💡 Связывает указанное имя с удалённым объектом.
    // Если имя уже связано, выбрасывается AlreadyBoundException.
    public void bind(String name, Remote obj)
        throws RemoteException, AlreadyBoundException, AccessException;
    
    // Аналогично bind, но перезаписывает существующее связывание.
    public void rebind(String name, Remote obj)
        throws RemoteException, AccessException;
    
    // Удаляет связывание имени с объектом.
    // Бросает NotBoundException, если имя не существует.
    public void unbind(String name)
        throws RemoteException, NotBoundException, AccessException;
    
    
    // ======================
    // 🟢 Служебные методы 🟢
    // ======================
    
    // Возвращает список всех зарегистрированных имён в реестре.
    public String[] list()
        throws RemoteException, AccessException;
}
```

---
## 💡 Особенности и подводные камни 💡

1️⃣ 💡**Все методы — удалённые.**  
==Каждый вызов происходит через сеть==, поэтому нужно быть готовым к `RemoteException`.

2️⃣ 💡**`lookup` возвращает `Remote` stub**, который клиент может привести к нужному интерфейсу.

3️⃣ **`bind` и `rebind` различаются:**
- `bind` не перезаписывает — выбросит `AlreadyBoundException`,
    
- `rebind` всегда заменяет старое связывание новым.
    

4️⃣ 🧠**`list()` возвращает только имена**, а не сами объекты. Чтобы получить объект, нужно вызвать `lookup()` по каждому имени.

5️⃣ 💡**Безопасность:** при неправильной конфигурации политики безопасности (`java.security.policy`) возможен `AccessException`.

6️⃣ 🧠**Реестр сам по себе — удалённый объект**, реализуется классом `RegistryImpl` (скрыт внутри JDK), создаётся через `LocateRegistry.createRegistry()`.

---

## 💎 Best Practices 💎

✅ На стороне сервера:
- ⭐⭐Используй `rebind()` вместо `bind()`, чтобы избежать `AlreadyBoundException` при повторном запуске сервера.
    
- ❗❗❗После `createRegistry()` НЕ ЗАПУСКАЙ отдельный процесс `rmiregistry` — он уже встроен.
    

✅ На стороне клиента:
- ⭐⭐После `LocateRegistry.getRegistry(host, port)` проверь доступность реестра через `list()` или `lookup()` внутри блока `try-catch`.
    
- Приводи результат `lookup()` к нужному интерфейсу, например:
    `MyService stub = (MyService) registry.lookup("MyService");`
    

✅ 💡Всегда обрабатывай `RemoteException`, `NotBoundException` и `AccessException`.

✅ ❗❗❗Не полагайся на порт 1099 в продакшене — параметризуй его.

✅ Если нужно обеспечить ==безопасность==, ==используй фабрики сокетов== с SSL (`RMISSLClientSocketFactory`).

## Пример
```java
import java.rmi.registry.LocateRegistry;
import java.rmi.registry.Registry;
import java.rmi.Remote;
import java.rmi.RemoteException;

// Интерфейс удалённого объекта
public interface HelloService extends Remote {
    String sayHello(String name) throws RemoteException;
}

// Реализация сервиса
public class HelloServiceImpl implements HelloService {
    public String sayHello(String name) throws RemoteException {
        return "Привет, " + name + "!";
    }
}

// Сервер
public class Server {
    public static void main(String[] args) {
        try {
            // Создаём локальный реестр
            Registry registry = LocateRegistry.createRegistry(1099);

            // ❗❗❗Регистрируем удалённый объект под именем "Hello"
            HelloService service = new HelloServiceImpl();
            registry.rebind("Hello", service);

            System.out.println("Сервер запущен. Объект 'Hello' зарегистрирован!");
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

// Клиент
public class Client {
    public static void main(String[] args) {
        try {
            // Получаем ссылку на реестр
            Registry registry = LocateRegistry.getRegistry("localhost", 1099);

            // ❗❗❗Получаем объект по имени
            HelloService service = (HelloService) registry.lookup("Hello");

            // Вызываем удалённый метод
            String response = service.sayHello("Сергей");
            System.out.println(response);
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

```
