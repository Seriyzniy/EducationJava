
> [!NOTE] **`Method`** - объектное отражение (`рефлективную модель`) `метода класса` или `интерфейса`.  
> Позволяет `получать информацию о методах` (имя, параметры, возвращаемый тип, аннотации, модификаторы) и ==вызвать их во время выполнения==.
> 

Реализует интерфейсы:
- `GenericDeclaration`
- `Member`
- `AnnotatedElement`

---
## Фабричные методы для создания
```java
// Конструктор приватный. 
// Экземпляры Method создаются JVM и возвращаются методами класса Class.
// Например: clazz.getDeclaredMethod("methodName", paramTypes...)


// 💡Для получения объектов Method используются фабричные методы класса Class:
public Method getMethod(String name, Class<?>... parameterTypes)
public Method getDeclaredMethod(String name, Class<?>... parameterTypes)
public Method[] getMethods()
public Method[] getDeclaredMethods()

// Пояснение:
// getMethod()         — возвращает только публичные методы (включая унаследованные);
// getDeclaredMethod() — возвращает метод независимо от модификатора (только из текущего класса).
```

## Методы
### Основная информация о методе
```java
// Возвращает имя метода.
public String getName()

// Возвращает возвращаемый тип метода.
public Class<?> getReturnType()


// Возвращает типы параметров метода.
public Class<?>[] getParameterTypes()

// Возвращает количество параметров.
public int getParameterCount()

// Возвращает информацию о параметрах (с именами, если они доступны).
public Parameter[] getParameters()



// Возвращает типы исключений, объявленные методом.
public Class<?>[] getExceptionTypes()

// Возвращает модификаторы метода (public, static, final и т.д.)
public int getModifiers()

// Возвращает класс, в котором этот метод объявлен.
public Class<?> getDeclaringClass()



// Проверяет, является ли метод varargs (с переменным числом аргументов).
public boolean isVarArgs()

// 💡Проверяет, является ли метод bridge-методом (созданным компилятором для дженериков).
public boolean isBridge()

// 💡Проверяет, является ли метод синтетическим (созданным компилятором).
public boolean isSynthetic()
```

### Вызов метода
```java
// ⭐⭐Вызывает метод на указанном объекте с заданными аргументами.
public Object invoke(Object obj, Object... args)
    throws IllegalAccessException, IllegalArgumentException, InvocationTargetException

// Пояснение:
// obj — объект, на котором вызывается метод (или null, если метод статический);
// args — аргументы метода.
// Возвращает результат вызова или null, если метод void.

// Проверяет, можно ли получить доступ к методу для данного объекта.
public boolean canAccess(Object obj)

// Устанавливает флаг доступности (для приватных методов).
public void setAccessible(boolean flag)
```

### Работа с дженериками
```java
// Возвращает информацию о параметризованных типах возвращаемого значения.
public Type getGenericReturnType()

// Возвращает типы параметров метода с учетом дженериков.
public Type[] getGenericParameterTypes()

// Возвращает типы исключений, объявленных методом, с учетом дженериков.
public Type[] getGenericExceptionTypes()

// Возвращает параметры типа, объявленные методом (например, <T>).
public TypeVariable<Method>[] getTypeParameters()
```

### Работа с аннотациями
```java
// Возвращает все аннотации метода.
public Annotation[] getAnnotations()

// Возвращает все аннотации, примененные к параметрам метода.
public Annotation[][] getParameterAnnotations()

// Возвращает конкретную аннотацию по классу.
public <T extends Annotation> T getAnnotation(Class<T> annotationClass)

// Проверяет, аннотирован ли метод указанной аннотацией.
public boolean isAnnotationPresent(Class<? extends Annotation> annotationClass)
```

### Сравнение и строковое представление
```java
// Проверяет равенство двух объектов Method.
public boolean equals(Object obj)

// Возвращает хэш-код метода.
public int hashCode()

// Возвращает строковое представление метода (с типами, именем и параметрами).
public String toString()
```

## 💡 Особенности и подводные камни 💡

1️⃣ 💡`invoke()` может выбросить **InvocationTargetException**, если вызываемый метод сам выбрасывает исключение — это «обертка» над исходным исключением.  

2️⃣ 💡Для вызова **приватных** методов необходимо вызвать `setAccessible(true)`.

3️⃣ Если метод **статический**, объект (`obj`) при вызове `invoke()` может быть `null`.  

4️⃣ 💡При вызове метода с неправильными типами аргументов выбрасывается `IllegalArgumentException`.  

5️⃣ В Java 9+ доступ к приватным методам может быть ограничен модульной системой (JPMS).  

6️⃣ Метод `isBridge()` и `isSynthetic()` важны при работе с обобщениями и компиляторными вставками.  

7️⃣ Рефлексия медленнее прямого вызова метода, особенно при частом использовании.

---
## 💎 Best Practices 💎

✅ 💡Проверять модификаторы (`getModifiers()`) перед вызовом метода через `invoke()`.  

✅ Кэшировать объект `Method`, если вызов производится часто.  

✅ 💡Избегать использования `setAccessible(true)` в продакшене.  

✅ Оборачивать вызовы `invoke()` в `try-catch`, чтобы корректно обработать `InvocationTargetException`.  

✅ ⭐⭐Использовать `MethodHandle` (из `java.lang.invoke`) для более быстрого вызова при необходимости высокой производительности.  

✅ Использовать аннотации и фильтровать методы через `isAnnotationPresent()` при разработке фреймворков.

## Пример
```java
class Calculator {
    private int add(int a, int b) {
        return a + b;
    }

    public void printHello(String name) {
        System.out.println("Hello, " + name + "!");
    }
}

public class MyMain {
	public static void main(String[] args){
		try {
			Calculator calc = new Calculator();
	        Class<?> clazz = calc.getClass();

	        // Получаем приватный метод "add"
	        Method addMethod = clazz.getDeclaredMethod("add", int.class, int.class);

	        // Разрешаем доступ
	        addMethod.setAccessible(true);

	        // Вызываем метод и получаем результат
	        Object result = addMethod.invoke(calc, 6, 10);
	        System.out.println("Результат add: " + result);

	        // Получаем публичный метод "printHello"
	        Method helloMethod = clazz.getMethod("printHello", String.class);
	        helloMethod.invoke(calc, "Sergey");
		} catch (Exception e) {
			e.printStackTrace();
		}
	}
}

Результат add: 16
Hello, Sergey!
```
