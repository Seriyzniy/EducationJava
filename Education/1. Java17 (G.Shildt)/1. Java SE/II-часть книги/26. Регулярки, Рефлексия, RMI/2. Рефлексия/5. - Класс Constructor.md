
> [!NOTE] Класс **`Constructor`** `представляет` `конструктор класса` во время выполнения (runtime)
> Он является частью пакета **Reflection API** и `используется для` динамического `создания объектов`, 
> ❗❗даже если тип НЕИЗВЕСТЕН во время компиляции.

```java
Object
  ↳ AccessibleObject
       ↳ Executable
            ↳ Constructor<T>
```

Реализует интерфейсы:
- `GenericDeclaration` — позволяет получать информацию о параметрах типа (например, `<T>`)
    
- `Member` — общий интерфейс для методов, конструкторов и полей класса

---
## Фабричные методы
Конструктор protected, используются фабричные методы
```java
// 1. Получение всех публичных конструкторов класса
// Параметр: clazz - класс, конструкторы которого нужно получить
Constructor<?>[] getConstructors(Class<?> clazz)

// 2. Получение всех конструкторов класса (включая приватные)
// Параметр: clazz - класс, конструкторы которого нужно получить  
Constructor<?>[] getDeclaredConstructors(Class<?> clazz)

// 3. Получение конкретного публичного конструктора по типам параметров
// Параметры:
//   clazz - класс, конструктор которого нужно получить
//   parameterTypes - типы параметров конструктора (Class<?>...)
Constructor<T> getConstructor(Class<T> clazz, Class<?>... parameterTypes)

// 4. Получение любого конструктора класса (включая приватные) по типам параметров
// Параметры:
//   clazz - класс, конструктор которого нужно получить
//   parameterTypes - типы параметров конструктора (Class<?>...)
Constructor<T> getDeclaredConstructor(Class<T> clazz, Class<?>... parameterTypes)

// Возвращает конструктор, внутри которого был объявлен данный класс
// (для локальных или анонимных классов, объявленных внутри конструктора)
// Возвращает: объект Constructor или null, если класс не был объявлен внутри конструктора
Constructor<?> getEnclosingConstructor() throws SecurityException
```

### Пример с getEnclosingConstructor() 
```java
public class Outer {
    public Outer() {
        // Локальный класс внутри конструктора
        class LocalClass {}
        LocalClass local = new LocalClass();
        
        // Получение конструктора, содержащего этот класс
        Constructor<?> enclosingConstructor = local
	        .getClass()
	        .getEnclosingConstructor();
    }
}
```

## Методы

Класс `Constructor<T>` наследует методы от:
- `AccessibleObject` (например, `setAccessible(true)` — для открытия доступа);
    
- `Executable` (например, `getParameterCount()`, `getAnnotatedReturnType()` и др.).


Ниже приведены **только собственные публичные методы класса `Constructor`**, не унаследованные.

### Получение информации о конструкторе
```java
public Class<T> getDeclaringClass()
// Возвращает класс, которому принадлежит данный конструктор.

public String getName()
// Возвращает имя конструктора (совпадает с именем класса).

public int getModifiers()
// Возвращает модификаторы доступа (public, private, protected и т.д.) в виде битовой маски.
// Для расшифровки можно использовать класс Modifier.

public boolean isVarArgs()
// Возвращает true, если конструктор принимает переменное количество аргументов (...).

public TypeVariable<Constructor<T>>[] getTypeParameters()
// Возвращает массив параметров типа (generic-параметров), если они есть.
```

### Создание экземпляра
```java
public T newInstance(Object... initargs)
    throws InstantiationException,
           IllegalAccessException,
           IllegalArgumentException,
           InvocationTargetException
// Создаёт новый экземпляр класса, используя данный конструктор.
// Аргументы initargs должны соответствовать типам параметров конструктора.
//
// Исключения:
// - InstantiationException — если класс абстрактный
// - IllegalAccessException — если конструктор недоступен
// - InvocationTargetException — если конструктор выбросил исключение
```
### Получение типов и исключений
```java
public Class<?>[] getParameterTypes()
// Возвращает типы параметров конструктора (без учёта generic-информации).

public Type[] getGenericParameterTypes()
// Возвращает типы параметров с учётом generic-информации.

public Class<?>[] getExceptionTypes()
// Возвращает массив типов исключений, которые может выбросить конструктор.
```

### Строковое представление
```java
public String toString()
// Возвращает строковое представление конструктора без generic-информации.

public String toGenericString()
// Возвращает строковое представление конструктора с generic-параметрами.
```

### Сравнение и хэширование
```java
public boolean equals(Object obj)
// Сравнивает два конструктора по классу и списку параметров.

public int hashCode()
// Возвращает хэш-код, основанный на классе и параметрах конструктора.
```

## 💡 Особенности и подводные камни 💡

1️⃣ 💡**`newInstance()` медленнее, чем `new`**, т.к. выполняет проверки доступа и создаёт объект через Reflection.  

2️⃣ Чтобы вызывать **приватные конструкторы**, нужно вызвать `setAccessible(true)` — без этого выбросится `IllegalAccessException`.  

3️⃣ Если конструктор выбрасывает исключение, оно **оборачивается в `InvocationTargetException`**.  

4️⃣ При работе с varargs-конструкторами нужно передавать **массив последним аргументом**.  

5️⃣ 💡Рефлексия нарушает инкапсуляцию и может вызвать **SecurityException** в средах с ограниченным доступом (например, Java EE, Android).

---
## 💎 Best Practices 💎
✅ 💡Используй `getDeclaredConstructor()` вместо `getConstructor()`, если нужно получить **непубличные конструкторы**.  

✅ Всегда **обрабатывай `InvocationTargetException`**, так как он скрывает реальную причину ошибки.  

✅ 💡Не злоупотребляй `setAccessible(true)` — это может нарушить безопасность и оптимизации JVM.  

✅ Кэшируй объекты `Constructor`, если они используются многократно — это повысит производительность.  

✅ При работе с generic-конструкторами используй `getGenericParameterTypes()`, а не `getParameterTypes()`.

## Пример
```java
import java.lang.reflect.Constructor;

class User {
    private String name;
    private int age;

    // Конструктор с параметрами
    public User(String name, int age) {
        this.name = name;
        this.age = age;
    }

    void print() {
        System.out.println(name + ", возраст " + age);
    }
}


public class MyMain {
	public static void main(String[] args){
		try {
			// Получаем объект Class
	        Class<User> clazz = User.class;

	        // Получаем конструктор с параметрами String, int
	        Constructor<User> constructor = clazz.getConstructor(String.class, int.class);

	        // Создаём объект через Reflection
	        User u = constructor.newInstance("Сергей", 22);

	        // Вызываем метод объекта
	        u.print(); // Сергей, возраст 22
		} catch (Exception e) {
			e.printStackTrace();
		}
	}
}

Сергей, возраст 22
```
