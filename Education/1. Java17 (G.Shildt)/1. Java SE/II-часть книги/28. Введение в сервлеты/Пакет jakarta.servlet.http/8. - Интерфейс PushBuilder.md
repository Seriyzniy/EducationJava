
> [!NOTE] `PushBuilder` — интерфейс из пакета `jakarta.servlet.http`, появившийся в Servlet API 4.0. 
> Он `предназначен` для **HTTP/2 Server Push** — `механизма`, ==позволяющего серверу "заранее" отправлять клиенту ресурсы== (CSS, JS, изображения и т. д.) до того, как браузер сам их запросит.  
> 
> Не реализует других интерфейсов (чистый интерфейс).
> 

## Методы
### Настройка параметров запроса
```java
// Устанавливает HTTP-метод для push-запроса (по умолчанию GET)
PushBuilder method(String method)

// Устанавливает строку запроса (query string)
PushBuilder queryString(String queryString)

// Устанавливает идентификатор сессии (если нужно передать её вручную)
PushBuilder sessionId(String sessionId)

// Устанавливает или добавляет HTTP-заголовок к push-запросу
PushBuilder setHeader(String name, String value)
PushBuilder addHeader(String name, String value)

// Удаляет HTTP-заголовок
PushBuilder removeHeader(String name)

// ⭐Устанавливает путь (URI) ресурса⭐, который будет отправлен клиенту
PushBuilder path(String path)
```

### Получение текущих параметров
```java
// Возвращает HTTP-метод, установленный для push-запроса
String getMethod()

// Возвращает строку запроса
String getQueryString()

// Возвращает идентификатор сессии
String getSessionId()

// Возвращает значение указанного заголовка
String getHeader(String name)

// Возвращает все имена заголовков
Set<String> getHeaderNames()

// Возвращает путь push-запроса
String getPath()
```

### 🚀 Отправка ресурса
```java
// Отправляет (пушит) указанный ресурс клиенту
void push()
```

---
## 💡 Особенности и подводные камни 💡

1️⃣ 💡**Работает только с HTTP/2.** Если клиент или сервер не поддерживает HTTP/2, `PushBuilder` будет `null`.  

2️⃣ ==**Получить экземпляр== можно ==только из запроса==.** 
⭐⭐Используется `HttpServletRequest.newPushBuilder()`.  

3️⃣ ❗❗❗**Push не гарантирует доставку.** Клиент может отказаться от получения ресурса.  

4️⃣❗❗❗**Заголовки копируются.** 
При создании `PushBuilder` часть заголовков текущего запроса копируется (например, `Accept`, `Accept-Encoding`).  

5️⃣ **Не следует "пушить" слишком много.** Избыточный push может ухудшить производительность (перегрузка канала, дублирование ресурсов в кэше клиента).  

6️⃣ ❗❗❗**`push()` не возвращает статус.** Ошибки пуша не сигнализируются напрямую — нужно анализировать поведение клиента.

---
## 💎 Best Practices 💎

✅ ❗❗❗Проверяйте поддержку HTTP/2 перед использованием (`request.newPushBuilder() != null`).  

✅ 💡Пушьте только действительно необходимые ресурсы (например, CSS или JS, которые точно понадобятся).  

✅ Устанавливайте корректные заголовки кеширования (`Cache-Control`, `ETag`).  
✅ Не пушьте большие файлы (видео, изображения в высоком разрешении).  

✅ Используйте `path()` с относительным путем, связанным с текущим контекстом.

## Пример
```java
@WebServlet("/index")
public class IndexServlet extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp)
            throws IOException {
        PushBuilder pushBuilder = req.newPushBuilder();
        if (pushBuilder != null) {
            // Пушим ресурсы до того, как браузер сам их запросит
            pushBuilder.path("style.css").push();
            pushBuilder.path("script.js").push();
        }

        resp.setContentType("text/html;charset=UTF-8");
        try (PrintWriter out = resp.getWriter()) {
            out.println("<html><head>");
            out.println("<link rel='stylesheet' href='style.css'>");
            out.println("<script src='script.js'></script>");
            out.println("</head><body>");
            out.println("<h1>HTTP/2 Push Demo</h1>");
            out.println("</body></html>");
        }
    }
}
```
