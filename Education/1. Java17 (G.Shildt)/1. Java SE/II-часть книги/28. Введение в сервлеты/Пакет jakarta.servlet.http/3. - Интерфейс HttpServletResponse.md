
> [!NOTE] **`HttpServletResponse`** — это часть Jakarta Servlet API, отвечающая за `формирование` `HTTP-ответа`, отправляемого сервером клиенту (браузеру, REST-клиенту и т.д.).
> 
> Основная задача: ==предоставить методы для== установки:
> - **статуса ответа**, 
> - **заголовков**, 
> - **куки**, 
> - **типа содержимого** и для **записи тела ответа**.

## Интерфейс
```java
public interface HttpServletResponse extends ServletResponse
```
📌 `HttpServletResponse` **наследует интерфейс `ServletResponse`**, который задает базовые методы для управления выходным потоком, кодировкой, MIME-типом и буферизацией.  

📌 Расширяет его функциональность, добавляя **HTTP-специфичные методы**.

## Методы
### Управление статусом ответа
```java
// ❗Устанавливает код состояния HTTP❗ (например, 200, 404, 500)
void setStatus(int sc)

// Устанавливает код состояния и причину (устаревший метод, заменен на setStatus(int))
@Deprecated
void setStatus(int sc, String sm)

// Возвращает текущий код состояния ответа
int getStatus()
```

### Управление заголовками
```java
// ❗Устанавливает значение заголовка❗ (перезаписывает❗ старое значение)
void setHeader(String name, String value)

// ❗Добавляет значение заголовка❗ (❗НЕ перезаписывает❗)
void addHeader(String name, String value)



// Проверяет, установлен ли указанный заголовок
boolean containsHeader(String name)



// ❗Устанавливает дату❗ в заголовке (в формате миллисекунд UNIX-времени)
void setDateHeader(String name, long date)

// Добавляет дату в заголовке
void addDateHeader(String name, long date)



// Устанавливает ❗числовое значение❗ в заголовке
void setIntHeader(String name, int value)

// ❗Добавляет числовое значение❗ в заголовке
void addIntHeader(String name, int value)
```

### Управление перенаправлениями и кэшированием
```java
// ❗Отправляет клиенту ответ-перенаправление (HTTP 302)❗
// Автоматически устанавливает заголовок "Location" и завершает обработку
void sendRedirect(String location) throws IOException


// Основная форма - только код ошибки
void sendError(int sc) throws IOException;

// С сообщением об ошибке
void sendError(int sc, String msg) throws IOException;



// Кодирует URL, добавляя информацию о сессии, если cookies недоступны
String encodeRedirectURL(String url)

// Аналог encodeRedirectURL(), устаревший метод (для совместимости)
@Deprecated
String encodeRedirectUrl(String url)

// Кодирует обычный URL для поддержки сессий
String encodeURL(String url)

// Аналог encodeURL(), устаревший метод (для совместимости)
@Deprecated
String encodeUrl(String url)
```

### Работа с телом ответа
_(унаследованы из `ServletResponse`, но часто используются вместе с `HttpServletResponse`)_
```java
// ❗Возвращает Writer❗ для текстового вывода (например, HTML, JSON)
PrintWriter getWriter() throws IOException

// ❗Возвращает OutputStream❗ для бинарного вывода (например, файлов)
ServletOutputStream getOutputStream() throws IOException
```

### Управление MIME-типом и кодировкой
_(унаследованы из `ServletResponse`)_
```java
// 💡Устанавливает MIME-тип содержимого (например, "text/html", "application/json")
void setContentType(String type)

// Возвращает MIME-тип содержимого
String getContentType()

// Устанавливает кодировку ответа (например, "UTF-8")
void setCharacterEncoding(String charset)

// Возвращает текущую кодировку
String getCharacterEncoding()
```

# Константы статусов
## **Успешные ответы (2xx)**

|Константа|Значение|Описание|
|---|---|---|
|`SC_OK`|200|Успешный запрос|
|`SC_CREATED`|201|Ресурс создан|
|`SC_ACCEPTED`|202|Запрос принят на обработку|
|`SC_NO_CONTENT`|204|Успешно, но нет содержимого|

## **Перенаправления (3xx)**

|Константа|Значение|Описание|
|---|---|---|
|`SC_MOVED_PERMANENTLY`|301|Постоянное перенаправление|
|`SC_FOUND`|302|Временное перенаправление|
|`SC_SEE_OTHER`|303|Смотри другой ресурс|
|`SC_NOT_MODIFIED`|304|Контент не изменился|

## **Ошибки клиента (4xx)**

| Константа                   | Значение | Описание                     |
| --------------------------- | -------- | ---------------------------- |
| `SC_BAD_REQUEST`            | 400      | Неверный запрос              |
| `SC_UNAUTHORIZED`           | 401      | **Требуется аутентификация** |
| `SC_FORBIDDEN`              | 403      | Доступ запрещен              |
| `SC_NOT_FOUND`              | 404      | Ресурс не найден             |
| `SC_METHOD_NOT_ALLOWED`     | 405      | Метод не поддерживается      |
| `SC_REQUEST_TIMEOUT`        | 408      | Таймаут запроса              |
| `SC_CONFLICT`               | 409      | Конфликт ресурсов            |
| `SC_GONE`                   | 410      | Ресурс удален                |
| `SC_PAYLOAD_TOO_LARGE`      | 413      | Слишком большой запрос       |
| `SC_UNSUPPORTED_MEDIA_TYPE` | 415      | Неподдерживаемый тип данных  |

## **Ошибки сервера (5xx)**

|Константа|Значение|Описание|
|---|---|---|
|`SC_INTERNAL_SERVER_ERROR`|500|Внутренняя ошибка сервера|
|`SC_NOT_IMPLEMENTED`|501|Не реализовано|
|`SC_BAD_GATEWAY`|502|Ошибка шлюза|
|`SC_SERVICE_UNAVAILABLE`|503|Сервис недоступен|
|`SC_GATEWAY_TIMEOUT`|504|Таймаут шлюза|

## 💡 Особенности и подводные камни 💡

1️⃣ ❗❗❗После вызова `sendRedirect()` или `sendError()` нельзя писать в тело ответа — соединение закрывается.  

2️⃣ ⭐⭐Все ==заголовки и статус== нужно ==устанавливать== **ДО ПЕРВОГО ВЫВОДА В ПОТОК** (`getWriter()` / `getOutputStream()`)**.  

3️⃣ 🧠Методы `encodeURL()` и `encodeRedirectURL()` нужны **для поддержки сессий без cookie**.  

4️⃣ Нельзя использовать одновременно `getWriter()` и `getOutputStream()` — это вызовет `IllegalStateException`. 

5️⃣ После `flushBuffer()` нельзя изменять заголовки и код состояния — они уже отправлены клиенту.

## 💎 Best Practices 💎

✅ ⭐⭐Всегда задавайте `Content-Type` и кодировку перед записью данных.  

✅ ⭐⭐⭐⭐Используйте `resp.sendError()` ВМЕСТО РУЧНОЙ установки СТАТУСА и ТЕКСТА ==ОШИБКИ==.  

```java
// С кастомным сообщением
response
	.sendError(
		HttpServletResponse.SC_FORBIDDEN, 
		"Access denied for this resource"
	);
```


✅ 💡💡Для редиректов всегда применяйте `sendRedirect()` — он автоматически формирует правильный заголовок.  

✅ ⭐⭐⭐⭐Для REST API предпочтительно использовать `resp.setStatus(HttpServletResponse.SC_...)` и JSON-ответы.  

✅ 💡💡Проверяйте, не был ли ответ уже коммитнут (`resp.isCommitted()`) перед изменением заголовков.

## Пример
```java
import jakarta.servlet.*;
import jakarta.servlet.http.*;
import java.io.*;

public class LoginServlet extends HttpServlet {

    @Override
    protected void doPost(
	    HttpServletRequest req, 
	    HttpServletResponse resp) throws ServletException, IOException 
	{
        String user = req.getParameter("username");
        String pass = req.getParameter("password");

        if ("admin".equals(user) && "1234".equals(pass)) {
            resp.sendRedirect("/home");
        } else {
            // Устанавливаем статус ошибки и отправляем сообщение
            resp.setContentType("text/html;charset=UTF-8");
            resp.setStatus(HttpServletResponse.SC_UNAUTHORIZED);

            PrintWriter out = resp.getWriter();
            out.println("<html><body>");
            out.println("<h2>Ошибка авторизации!</h2>");
            out.println("</body></html>");
        }
    }
}
```
