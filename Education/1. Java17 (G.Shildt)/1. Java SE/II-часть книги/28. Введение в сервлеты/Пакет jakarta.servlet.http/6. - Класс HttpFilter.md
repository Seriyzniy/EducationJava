
> [!NOTE] `HttpFilter` — это **базовый класс для фильтров**, 
> предназначенных для работы именно с HTTP-запросами и ответами. 
> 
> Он упрощает написание фильтров, ==наследуя общий функционал== от `jakarta.servlet.GenericFilter` 
> и 
> ❗❗==предоставляя типизированный== `doFilter()` с параметрами `HttpServletRequest` и `HttpServletResponse`.  
> 
> Реализует интерфейс `jakarta.servlet.Filter` (напрямую через `GenericFilter`).
> 
> Используется для пред- и пост-обработки HTTP-запросов: 
> - логирования, 
> - аутентификации, 
> - кэширования, 
> - добавления заголовков, 
> - переадресации и т.д.
> 

## Конструктор
```java
// Конструктор без параметров. Обычно не требует переопределения.
public HttpFilter()
```

## Методы
`HttpFilter` наследует методы от `GenericFilter`, но добавляет собственную реализацию `doFilter()` для работы именно с HTTP-сущностями.

### Основной метод фильтрации
```java
// Главный метод фильтра, вызывается контейнером при каждом HTTP-запросе.
// request  — объект запроса (HttpServletRequest)
// response — объект ответа (HttpServletResponse)
// chain    — цепочка фильтров (FilterChain), позволяет передать управление дальше
@Override
public void doFilter(
	ServletRequest request, 
	ServletResponse response, 
	FilterChain chain) throws IOException, ServletException
```

⭐⭐⭐Перегруженный вариант (==удобнее== переопределять)
```java
// Упрощённый метод, который принимает именно HTTP-запрос и HTTP-ответ.
// Обычно этот метод переопределяется вместо предыдущего.
protected void doFilter(
	HttpServletRequest request,
    HttpServletResponse response,
    FilterChain chain) throws IOException, ServletException
```

**Как это работает:**  
Контейнер вызывает `doFilter(ServletRequest, ServletResponse, FilterChain)`, а внутри `HttpFilter` выполняется проверка — если запрос действительно HTTP, он вызывает защищённый метод `doFilter(HttpServletRequest, HttpServletResponse, FilterChain)`. 

Таким образом, разработчик работает с типизированными объектами без ручного приведения типов.

### Методы жизненного цикла (унаследованные)
```java
// Инициализация фильтра при запуске приложения.
// config — объект с параметрами фильтра из web.xml или аннотации.
@Override
public void init(FilterConfig config) throws ServletException

// Освобождение ресурсов при завершении работы фильтра.
@Override
public void destroy()
```

## 💡 Особенности и подводные камни 💡

1️⃣ `HttpFilter` появился в **Servlet API 4.0** (Jakarta Servlet 4+) — до этого фильтры приходилось писать вручную, приводя `ServletRequest` к `HttpServletRequest`.  

2️⃣ 💡Если переопределяешь `doFilter(HttpServletRequest, HttpServletResponse, FilterChain)`, **не забудь вызвать** `chain.doFilter(request, response)`, иначе запрос не продолжится.  

3️⃣ В отличие от `Servlet`, фильтр **не формирует ответ** — он обрабатывает или изменяет данные «на пути» к Servlet.  

4️⃣ 💡Можно использовать `@WebFilter` для автоматической регистрации без `web.xml`.  

5️⃣ 💡Следи за порядком вызова фильтров — он зависит от порядка декларации в `web.xml` или аннотаций.

---
## 💎 Best Practices 💎

✅ Используй `HttpFilter`, если фильтр работает **только с HTTP**, чтобы избежать ручных приведений типов.  

✅ Не изменяй тело запроса/ответа без необходимости — это может привести к ошибкам.  

✅ Всегда вызывай `chain.doFilter(...)`, если фильтр не должен блокировать запрос.  

✅ Реализуй `destroy()` для освобождения ресурсов (например, закрытия логеров).  

✅ ⭐⭐Логируй время обработки запроса — это помогает выявлять узкие места.

## Пример ИЗМЕРЕНИЕ ВРЕМЕНИ ВЫПОЛНЕНИЯ
```java
package com.example.filter;

import jakarta.servlet.FilterChain;
import jakarta.servlet.ServletException;
import jakarta.servlet.annotation.WebFilter;
import jakarta.servlet.http.HttpFilter;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;
import java.io.IOException;

@WebFilter("/*") // Фильтр применяется ко всем URL
public class LoggingFilter extends HttpFilter {

    @Override
    protected void doFilter(
	    HttpServletRequest request,
		HttpServletResponse response,
        FilterChain chain) throws IOException, ServletException 
    {
        long start = System.currentTimeMillis();
        String uri = request.getRequestURI();

        System.out.println("➡ Запрос: " + uri);

        // Передаём управление дальше по цепочке (к следующему фильтру или сервлету)
        chain.doFilter(request, response);

        long duration = System.currentTimeMillis() - start;
        System.out.println("⬅ Ответ: " + uri + " (" + duration + " мс)");
    }
}
```

**Что делает пример:**
- Логирует каждый запрос (URI и время обработки).
    
- Работает с типизированными `HttpServletRequest` и `HttpServletResponse` — без приведения.
    
- Срабатывает до и после обработки основным сервлетом.
