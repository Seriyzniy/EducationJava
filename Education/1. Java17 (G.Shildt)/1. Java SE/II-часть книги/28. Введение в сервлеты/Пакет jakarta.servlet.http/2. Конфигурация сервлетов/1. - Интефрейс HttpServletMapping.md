
> [!NOTE] `HttpServletMapping` — это интерфейс, появившийся в **Servlet API 4.0**. 
> Он `описывает`, как именно HTTP-запрос был сопоставлен (`mapped`) `с сервлетом`.  
> 
> Позволяет получить информацию:
> - о шаблоне URL, 
> - типе сопоставления (например, `EXACT`, `PATH`, `EXTENSION`, `DEFAULT`), 
> - имени сервлета и оставшейся части пути (path info).  
> 
> ⭐⭐⭐Используется, когда нужно понять, **==почему и по какому правилу== был ==вызван текущий сервлет==**.
> 

## Интерфейс
```java
package jakarta.servlet.http;

public interface HttpServletMapping {
	// Возвращает значение, соответствующее совпавшему фрагменту URL.
	// 💡Например, при URL "/app/data/list" и шаблоне "/data/*" вернёт "data".
    String getMatchValue();
    
    // Возвращает шаблон сопоставления, по которому сервлет был найден.
	// Например: "/data/*", "*.jsp" или "/exact/path".
    String getPattern();
    
    // Возвращает имя сервлета, определённое в web.xml или аннотации @WebServlet.
    String getServletName();
    
    
    // Возвращает тип сопоставления (EXACT, PATH, EXTENSION, DEFAULT).
	
	// EXAMPLE: MappingMatch.EXACT — точное совпадение, 
	// PATH — по пути, 
	// EXTENSION — по расширению.
    jakarta.servlet.http.MappingMatch getMappingMatch();
}
```

---
## 💡 Особенности и подводные камни 💡

1️⃣ Интерфейс доступен с **Servlet API 4.0+**, т.е. начиная с Jakarta EE 8.  

2️⃣ ⭐⭐⭐Получить объект `HttpServletMapping` можно только через `HttpServletRequest.getHttpServletMapping()`.  

3️⃣ ⭐⭐🧠Используется в основном для **отладки маршрутизации** и **динамического поведения сервлета** (например, один сервлет обслуживает несколько шаблонов).  

4️⃣ 🧠Полезен при построении REST API, когда один сервлет обрабатывает разные endpoint’ы, и нужно понять, по какому пути он вызван.  

5️⃣ Тип `MappingMatch` — это enum (`EXACT`, `DEFAULT`, `EXTENSION`, `PATH`) и позволяет чётко понимать логику выбора сервлета контейнером.

---
## 💎 Best Practices 💎

✅ 💡Используй `HttpServletMapping` для логирования или маршрутизации внутри одного сервлета.  

✅ ❗❗❗Не хардкоди URL-шаблоны — лучше получать их через `getPattern()`. 

✅ Проверяй `getMappingMatch()`, чтобы понимать, сработало ли точное совпадение или шаблонное.  

✅ 💡Используй в административных панелях или middleware для отладки маршрутов.

## Пример
```java
package com.example;

import jakarta.servlet.ServletException;
import jakarta.servlet.annotation.WebServlet;
import jakarta.servlet.http.HttpServlet;
import jakarta.servlet.http.HttpServletMapping;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;
import java.io.IOException;

@WebServlet(urlPatterns = {"/api/*", "/user/*"})
public class MappingInfoServlet extends HttpServlet {

    @Override
    protected void doGet(
	    HttpServletRequest request, 
	    HttpServletResponse response) throws ServletException, IOException 
	{
        response.setContentType("text/html;charset=UTF-8");

        // Получаем объект сопоставления
        HttpServletMapping mapping = request.getHttpServletMapping();


        response.getWriter()
	        .println("<h3>Информация о маршрутизации:</h3>");
	        
        response.getWriter().println("<ul>");
        response.getWriter()
	        .println("<li>Servlet name: " 
		        + mapping.getServletName() + "</li>"
		    );
        
        response.getWriter()
	        .println("<li>Pattern: " + mapping.getPattern() + "</li>");
        
        response.getWriter()
	        .println("<li>Match value: " 
		        + mapping.getMatchValue() + "</li>");
        
        response.getWriter()
	        .println("<li>Mapping type: " 
		        + mapping.getMappingMatch() + "</li>");
    
	    response.getWriter().println("</ul>");
    }
}
```
