
> [!NOTE] `HttpServletRequestWrapper` — это **декоратор (`обёртка`)** для объекта `HttpServletRequest`.
> Используется `для расширения или изменения поведения` `стандартного` `запроса` 
> (например, переопределить заголовки, параметры, тело запроса и т.п.) 
> без необходимости писать собственную реализацию интерфейса `HttpServletRequest`.
> 
> Реализует интерфейс `HttpServletRequest` и наследует `jakarta.servlet.ServletRequestWrapper`.

## Конструктор
```java
// Создаёт обёртку вокруг существующего запроса.
// request — оригинальный HttpServletRequest, который будет обёрнут.
public HttpServletRequestWrapper(HttpServletRequest request)
```

## Методы
Наследует все методы `HttpServletRequest`, но ==по умолчанию **делегирует вызовы оригинальному объекту**== (который передаётся в конструктор).  

Разработчик может переопределить нужные методы, чтобы изменить поведение.

### Методы доступа к оригинальному запросу
```java
// Возвращает исходный HttpServletRequest, который был передан в конструктор.
@Override
public HttpServletRequest getRequest()

// Устанавливает новый HttpServletRequest в качестве "внутреннего" запроса.
@Override
public void setRequest(ServletRequest request)
```


### Пример переопределяемых методов
```java
// Можно переопределить для изменения параметров запроса
@Override
public String getParameter(String name)

// Можно переопределить для изменения заголовков
@Override
public String getHeader(String name)
```

## 💡 Особенности и подводные камни 💡

1️⃣ ❗Это **паттерн "Декоратор"** — добавляет логику поверх стандартного объекта, не меняя его реализацию.  

2️⃣ 💡Очень полезен в фильтрах (`Filter`), когда нужно изменить запрос до передачи в `Servlet`.  

3️⃣ ❗Все методы по умолчанию просто **делегируют** вызовы оригинальному `request`.  

4️⃣ ⭐⭐⭐Можно переопределить методы чтения тела (`getReader()`, 
`getInputStream()`), но нужно аккуратно работать с потоками (т.к. тело можно прочитать только один раз).  

5️⃣ Нельзя просто "добавить" параметр — для этого придётся хранить дополнительную Map внутри обёртки.

---
## 💎 Best Practices 💎

✅ Используй `HttpServletRequestWrapper`, если нужно модифицировать запрос (например, добавить параметр, изменить заголовок).  

✅ ❗❗Не читай тело запроса более одного раза. Если нужно — кэшируй его.  

✅ При работе с JSON-телом запроса создай BufferedReader-копию, чтобы повторно использовать.  

✅ ❗❗❗Не забывай вызывать `super()` в конструкторе.  

✅ ❗❗❗Используй в сочетании с фильтрами (`Filter`).

## Пример
```java
@WebFilter("/*")
public class HeaderAddingFilter extends HttpFilter {

    @Override
    protected void doFilter(
	    HttpServletRequest request,
        HttpServletResponse response,
        FilterChain chain) throws IOException, ServletException 
    {
        // Оборачиваем запрос в наш кастомный wrapper
        HttpServletRequest wrapped = 
	        new HttpServletRequestWrapper(request) 
	    {
            @Override
            public String getHeader(String name) {
                if ("X-Custom".equalsIgnoreCase(name)) {
                    return "Added-By-Filter";
                }
                return super.getHeader(name);
            }
        };

        chain.doFilter(wrapped, response);
    }
}
```
