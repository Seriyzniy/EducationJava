Он — **аналогичный декоратор**, но для **ответа сервера (response)**, и используется в тех же целях, что и `ServletRequestWrapper`, только для ВЫХОДНЫХ данных.


> [!NOTE] `ServletResponseWrapper` — это **обёртка (декоратор)** вокруг объекта `ServletResponse`.
> Позволяет `расширять, изменять или перехватывать` `поведение` стандартного `объекта ответа`, НЕ ИЗМЕНЯЯ его исходный код.

Чаще всего используется:
- для **логирования ответа** (HTTP-кода, заголовков, тела);
- 💡для **сжатия** (`GZIP`) перед отправкой;
- для **буферизации** и анализа данных перед отдачей клиенту;
- для **изменения контента** (например, внедрение баннера, подмена текста и т. д.).

## Конструктор
```java
// Создаёт новую обёртку вокруг указанного объекта ServletResponse.
// Параметр response — исходный объект ServletResponse, который будет обёрнут.
// Если response == null, выбрасывается IllegalArgumentException.
public ServletResponseWrapper(ServletResponse response)
```

## Методы
Класс реализует интерфейс `ServletResponse`.  

==Все методы== по умолчанию просто ==делегируются внутреннему объекту== `response`.

### Работа с вложенным объектом ответа
```java
// Возвращает обёрнутый объект ServletResponse.
// Можно использовать для прямого доступа к оригинальному объекту.
public ServletResponse getResponse()

// Позволяет заменить текущий объект ответа на другой.
// Полезно, если нужно "переобернуть" ответ на лету.
public void setResponse(ServletResponse response)
```

### Делегирующие методы
Все остальные методы интерфейса `ServletResponse` делегируются оригинальному объекту.  

Вот несколько типичных примеров:
```java
// Делегирует вызов setContentType() оригинальному response
@Override
public void setContentType(String type) {
    response.setContentType(type);
}

// Делегирует вызов getOutputStream() оригинальному response
@Override
public ServletOutputStream getOutputStream() throws IOException {
    return response.getOutputStream();
}

// Делегирует вызов getWriter() оригинальному response
@Override
public PrintWriter getWriter() throws IOException {
    return response.getWriter();
}
```

Ты можешь **наследоваться от `ServletResponseWrapper`** и переопределять любые методы, чтобы изменить их поведение.

---
## 💡 Особенности и подводные камни 💡

1️⃣ `ServletResponseWrapper` реализует **паттерн "Декоратор"**, как и `ServletRequestWrapper`.  

2️⃣ 💡На практике ==чаще используется его наследник== — **`HttpServletResponseWrapper`**, который добавляет методы работы с HTTP (заголовки, коды, cookies и т.д.).  

3️⃣ Если переопределяешь `getWriter()` или `getOutputStream()`, будь осторожен — их нельзя вызывать **оба** в одном ответе.  

4️⃣ 💡Часто требуется **буферизация вывода** — т.е. записывать данные сначала в `ByteArrayOutputStream`, чтобы потом анализировать или модифицировать контент.  

5️⃣ ❗❗❗`setResponse()` может быть опасным — не замещай объект на несовместимый тип.

---
## 💎 Best Practices 💎

✅ 💡Используй `HttpServletResponseWrapper`, если работаешь с HTTP.  

✅ ⭐Переопредели `getOutputStream()` или `getWriter()`, если нужно перехватывать или фильтровать вывод.  

✅ Для логирования тела ответа — буферизуй поток в памяти (`ByteArrayOutputStream`).  

✅ ⭐⭐Всегда вызывай `super.flushBuffer()` в конце, если меняешь поток, чтобы всё корректно сбросилось.  

✅ ❗❗❗Не забывай передавать обёрнутый объект дальше в `FilterChain`.

## Пример
Пример фильтра, который логирует тело ответа сервера:

```java
public class LoggingResponseWrapper extends HttpServletResponseWrapper {

    private ByteArrayOutputStream capture;
    private ServletOutputStream output;
    private PrintWriter writer;

    public LoggingResponseWrapper(HttpServletResponse response) {
        super(response);
        capture = new ByteArrayOutputStream();
    }

    @Override
    public ServletOutputStream getOutputStream() {
        if (writer != null) {
            throw new IllegalStateException(
	            "getWriter() already called"
	        );
        }
        if (output == null) {
            output = new ServletOutputStream() {
                @Override
                public void write(int b) {
                    capture.write(b);
                }
                @Override
                public boolean isReady() { return true; }
                @Override
                public void setWriteListener(WriteListener writeListener) {}
            };
        }
        return output;
    }

    @Override
    public PrintWriter getWriter() {
        if (output != null) {
            throw new IllegalStateException("getOutputStream() already called");
        }
        if (writer == null) {
            writer = new PrintWriter(new OutputStreamWriter(capture, getCharacterEncoding()));
        }
        return writer;
    }

    public byte[] getCapturedAsBytes() throws IOException {
        if (writer != null) writer.flush();
        if (output != null) output.flush();
        return capture.toByteArray();
    }

    public String getCapturedAsString() throws IOException {
        return new String(getCapturedAsBytes(), getCharacterEncoding());
    }
}
```

И фильтр, использующий его:
```java
@WebFilter("/*")
public class ResponseLoggingFilter implements Filter {
    @Override
    public void doFilter(
	    ServletRequest req, 
	    ServletResponse res, 
	    FilterChain chain) throws IOException, ServletException 
	{
        HttpServletResponse response = (HttpServletResponse) res;
        
        LoggingResponseWrapper wrapped = 
	        new LoggingResponseWrapper(response);


        chain.doFilter(req, wrapped);


        // Теперь можно проанализировать тело перед отправкой клиенту
        String content = wrapped.getCapturedAsString();
        System.out.println("Response body:\n" + content);

        // Отправляем тело клиенту вручную
        ServletOutputStream out = response.getOutputStream();
        out.write(wrapped.getCapturedAsBytes());
        out.flush();
    }
}
```
