
> [!NOTE] `ServletRequestWrapper` — это **декоратор** (wrapper, "`обёртка`") для объекта `ServletRequest`
> Он предназначен для того, чтобы `расширять или изменять` `поведение` оригинального `запроса` `БЕЗ ИЗМЕНЕНИЯ` его исходного `кода`.

🔹 Класс реализует интерфейс **`ServletRequest`**,  
🔹 и обычно используется в фильтрах (`Filter`), где нужно:
- ❓изменить параметры запроса,
- переопределить тело запроса (`InputStream`),
- внедрить кэширование тела для повторного чтения,
- замаскировать, фильтровать или логировать данные.

---
## Конструктор
```java
// ⭐Создаёт новый объект-обёртку⭐ вокруг указанного запроса.
// Параметр request — исходный объект ServletRequest, который будет обёрнут.
// Если request == null, выбрасывается IllegalArgumentException.
public ServletRequestWrapper(ServletRequest request)
```

## Методы
`ServletRequestWrapper` реализует интерфейс `ServletRequest`,  
и **по умолчанию ==все методы просто делегируют вызовы внутреннему объекту== `request`**.  

Ниже описаны только собственные и наиболее важные методы класса.
### Работа с вложенным запросом
```java
// 💡Возвращает оригинальный объект ServletRequest💡, который был обёрнут.
// Полезно, если нужно получить доступ к исходным данным запроса.
public ServletRequest getRequest()

// 💡Заменяет текущий обёрнутый объек💡т запроса на новый.
// Можно использовать, чтобы "переобернуть" другой объект запроса в процессе.
public void setRequest(ServletRequest request)
```

### Делегирование (все остальные методы)
Все остальные методы `ServletRequest` (такие как `getParameter()`, `getInputStream()`, `getReader()`, `getAttribute()`, `getLocale()`, и т.д.)  
**вызовами ==просто перенаправляются к оригинальному== `ServletRequest`.**

Например:
```java
// Делегирует вызов методу getParameter() оригинального запроса
@Override
public String getParameter(String name) {
    return request.getParameter(name);
}

// Делегирует вызов методу getInputStream() оригинального запроса
@Override
public ServletInputStream getInputStream() throws IOException {
    return request.getInputStream();
}
```

Таким образом, чтобы изменить поведение,  
ты **наследуешься** от `ServletRequestWrapper` и **переопределяешь нужные методы**

---
## 💡 Особенности и подводные камни 💡

1️⃣ `ServletRequestWrapper` — **паттерн "Декоратор"**. Все методы делегируют оригинальному объекту.  

2️⃣ 💡Используется в **фильтрах** для модификации запроса **до попадания в сервлет**

3️⃣ ❗❗❗При переопределении `getInputStream()` или `getReader()` важно **сохранить совместимость** с остальными методами (например, чтобы не вызвать `getReader()` после `getInputStream()`).  

4️⃣ 💡Если ты переопределяешь `getParameter*()`, убедись, что возвращаешь те же данные, что и тело (`InputStream`) — иначе могут быть рассинхронизации.  

5️⃣ `setRequest()` может сломать совместимость, если передать объект, который не реализует все ожидаемые методы `ServletRequest`.

---
## 💎 Best Practices 💎

✅ Используй `ServletRequestWrapper` **в фильтрах** для безопасной модификации запроса.  

✅ ⭐⭐Переопредели **только нужные методы**, а остальные оставь делегированными. 

✅ Для повторного чтения тела запроса — кэшируй `InputStream` в байтовом массиве.  

✅ Не меняй оригинальный объект напрямую — лучше работать только с обёрткой.  

✅ Не вызывай одновременно `getReader()` и `getInputStream()`, если переопределяешь их — выбери один источник данных.

## Пример использования

Пример фильтра, который оборачивает запрос, чтобы логировать и сохранять тело для повторного чтения:
```java
public class CachedBodyRequestWrapper extends ServletRequestWrapper {

    private byte[] cachedBody;

    public CachedBodyRequestWrapper(HttpServletRequest request) throws IOException {
        super(request);
        try (InputStream inputStream = request.getInputStream()) {
            this.cachedBody = inputStream.readAllBytes();
        }
    }

    @Override
    public ServletInputStream getInputStream() {
        ByteArrayInputStream bais = new ByteArrayInputStream(cachedBody);
        return new ServletInputStream() {
            @Override
            public int read() {
                return bais.read();
            }

            @Override
            public boolean isFinished() {
                return bais.available() == 0;
            }

            @Override
            public boolean isReady() {
                return true;
            }

            @Override
            public void setReadListener(ReadListener readListener) {}
        };
    }

    public byte[] getCachedBody() {
        return cachedBody;
    }
}
```

И использование в фильтре:
```java
@WebFilter("/*")
public class LoggingFilter implements Filter {
    @Override
    public void doFilter(
	    ServletRequest request, 
	    ServletResponse response, 
	    FilterChain chain) throws IOException, ServletException 
	{
        CachedBodyRequestWrapper wrapped = 
	        new CachedBodyRequestWrapper((HttpServletRequest) request);

        System.out.println("Request body: " 
	        + new String(wrapped.getCachedBody()));

        chain.doFilter(wrapped, response); // Передаём уже обёрнутый запрос
    }
}
```
