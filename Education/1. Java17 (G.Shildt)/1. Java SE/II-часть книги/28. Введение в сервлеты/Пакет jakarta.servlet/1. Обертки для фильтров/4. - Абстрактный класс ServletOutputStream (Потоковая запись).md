
> [!NOTE] `ServletOutputStream` — это абстрактный класс который предназначен для `ЗАПИСИ данных в тело HTTP-ответа` (response body);
> Управляется контейнером (Tomcat, Jetty и т.д.) — ты `НЕ СОЗДАЕШЬ ВРУЧНУЮ`, а получаешь из `ServletResponse`;
> 
> Добавляет поддержку **неблокирующего ввода/вывода (non-blocking I/O)** через `WriteListener`.
> 
> ⭐Наследуется от `java.io.OutputStream`;

(Для асинхроншины см [[5. - Интерфейс WriteListener]])
## Абстрактный класс
```java
package jakarta.servlet;

import java.io.IOException;
import java.io.OutputStream;

public abstract class ServletOutputStream extends OutputStream {

    /**
     * Проверяет, готов ли поток к записи данных без блокировки.
     * Возвращает true, если можно вызывать write() прямо сейчас.
     */
    public abstract boolean isReady();

    /**
     * Устанавливает WriteListener для асинхронной (неблокирующей) записи.
     * После установки контейнер будет вызывать методы onWritePossible() и onError().
     *
     * @param writeListener слушатель событий записи
     */
    public abstract void setWriteListener(WriteListener writeListener);
}
```
Все обычные методы `write(...)`, `flush()`, `close()` наследуются от `OutputStream`.

## Методы

### 1️⃣ Управление асинхронной записью (Servlet 3.1+)
```java
// Проверяет, готов ли поток к записи (без блокировки)
boolean isReady()

// Регистрирует слушателя событий асинхронной записи
void setWriteListener(WriteListener writeListener)
```

💬 Эти методы нужны, если ты используешь ==**асинхронный режим сервлета==**  
(`@WebServlet(asyncSupported = true)`) и хочешь писать данные в ответ **неблокирующе** — например, ==по мере их поступления от базы==, файла, или другого потока.

### 2️⃣ Методы записи (унаследованы от OutputStream)
```java
// Записывает один байт
void write(int b) throws IOException

// Записывает часть массива байтов
void write(byte[] b, int off, int len) throws IOException

// Принудительно сбрасывает буфер в клиент
void flush() throws IOException

// Закрывает поток и освобождает ресурсы
void close() throws IOException
```

💬 Эти методы используются для стандартной блокирующей записи.  
Например, когда ты пишешь текст или бинарные данные в тело ответа (PDF, изображение, CSV и т.д.).

## 💡 Особенности и подводные камни 💡

1️⃣ 💡 **Нельзя создавать вручную.**  
`ServletOutputStream` возвращается только через `response.getOutputStream()` — контейнер управляет им сам.

2️⃣ ❗❗❗**Конфликт с `getWriter()`.**  
Если ты вызвал `response.getWriter()`, **нельзя вызывать `getOutputStream()`** (и наоборот) — выбросится `IllegalStateException`.  
→ `Writer` — для **текста**, `OutputStream` — для **байтов**.

3️⃣ 💡**Управление буфером.**  
Контейнер буферизует ответ, пока ты не вызовешь `flush()` или не закроешь поток.

4️⃣ 💡**Асинхронная запись требует проверки `isReady()`.**  
Если поток не готов, запись должна быть отложена (иначе будет блокировка или ошибка).

5️⃣ ❗❗❗**Нельзя писать после commit.**  
После того как контейнер отправил заголовки (HTTP-статус, length, content-type) — писать уже нельзя.

6️⃣ ⭐⭐⭐**Не подходит для больших файлов напрямую.**  
Для больших бинарных данных (видео, архивы) лучше использовать `Files.copy()` или `NIO`-стриминг с chunked encoding.
```java
Path file = Path.of("/opt/reports/large-report.pdf");

try (ServletOutputStream out = resp.getOutputStream()) {
	// 📦 Копируем файл прямо в HTTP-ответ
	Files.copy(file, out);
}
```

---
## 💎 Best Practices 💎

✅ 💡Используй `PrintWriter` (через `getWriter()`) для текста и `ServletOutputStream` для бинарных данных.  

✅ Перед записью обязательно указывай `Content-Type` и, по возможности, `Content-Length`.  

✅ ❗❗❗Не смешивай `Writer` и `OutputStream` в одном ответе.  

✅ 💡Для асинхронных операций — всегда проверяй `isReady()` перед `write()`.  

✅ После записи всегда вызывать `flush()` (особенно если используешь chunked transfer).

## Пример
### Простой пример (скачивание файла)

В web.xml
```java
<servlet>
    <servlet-name>HelloServlet</servlet-name>
    <servlet-class>servlets.HelloServlet</servlet-class>
    <!-- Включаем поддержку асинхронных операций для др примеров-->
    <async-supported>true</async-supported>
</servlet>

<servlet-mapping>
    <servlet-name>HelloServlet</servlet-name>
    <url-pattern>/async-read</url-pattern>
</servlet-mapping>
```

```java
package servlets;

import java.io.IOException;
import java.io.InputStream;

import jakarta.servlet.ServletOutputStream;
import jakarta.servlet.http.HttpServlet;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;

public class HelloServlet extends HttpServlet {
	
	@Override
    protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws IOException {
        resp.setContentType("application/pdf");
        resp.setHeader("Content-Disposition", "attachment; filename=\"report.pdf\"");

        try (ServletOutputStream out = resp.getOutputStream();
             InputStream in = getServletContext().getResourceAsStream("/WEB-INF/files/report.pdf")) 
        {
             
            byte[] buffer = new byte[8192];
            int bytesRead;
            while ((bytesRead = in.read(buffer)) != -1) {
                out.write(buffer, 0, bytesRead);
            }
            out.flush();
        }
    }
}
```

Также скопируем любой файл `pdf` в папку `/WEB-INF/files/` и назовем `report.pdf`
