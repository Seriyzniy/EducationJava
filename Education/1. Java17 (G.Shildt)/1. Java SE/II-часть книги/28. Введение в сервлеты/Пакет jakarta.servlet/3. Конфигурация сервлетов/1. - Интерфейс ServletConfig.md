
> [!NOTE] `ServletConfig` служит `для передачи КОНФИГУРАЦИОННОЙ информации`
> сервлету при его инициализации.  
> 
> `Экземпляр` этого интерфейса ==СОЗДАЕТСЯ КОНТЕЙНЕРОМ сервлетов== (например, Tomcat) и передаётся методу `init(ServletConfig config)`.

Он хранит:
- 💡==параметры==💡 инициализации (`<init-param>` из `web.xml` или аннотаций),
- ссылку на контекст приложения (`ServletContext`),
- имя сервлета.
    

> Таким образом, `ServletConfig` — это **мост между контейнером и конкретным экземпляром сервлета**, позволяющий получить сведения, специфичные для данного сервлета.

---
## Интерфейс
```java
package jakarta.servlet;

import java.util.Enumeration;

/**
 * Provides servlet configuration information to a servlet, including
 * initialization parameters and a reference to the ServletContext.
 */
public interface ServletConfig {

    // ==============================
    // 🟢 Основная информация о сервлете 🟢
    // ==============================
    /**
     * Возвращает имя сервлета, заданное в web.xml или аннотацией @WebServlet.
     *
     * @return имя сервлета
     */
    String getServletName();



    // ==============================
    // 🟢 Параметры инициализации 🟢
    // ==============================
    /**
     * Возвращает значение параметра инициализации по его имени.
     * 💡Параметры задаются в web.xml (внутри <init-param>).
     *
     * @param name имя параметра
     * @return значение параметра или null, если параметр не найден
     */
    String getInitParameter(String name);

    /**
     * Возвращает имена всех параметров инициализации, доступных этому сервлету.
     *
     * @return Enumeration имён параметров
     */
    Enumeration<String> getInitParameterNames();



    // ==============================
    //  🟢 Контекст приложения 🟢
    // ==============================
    /**
     * ❗Возвращает объект ServletContext❗, общий для всего веб-приложения.
     * Через него можно получить доступ к ресурсам, логированию и т.д.
     *
     * @return ServletContext, связанный с данным сервлетом
     */
    ServletContext getServletContext();
}
```

---
## 💡 Особенности и подводные камни 💡

1️⃣ **Объект `ServletConfig` ==создаётся контейнером==**, а не вручную — разработчик лишь получает его через `init()`.

2️⃣ ⭐⭐**==Параметры== инициализации (`<init-param>`) ==относятся только к одному сервлету==**, 
в отличие от параметров контекста (`<context-param>`), которые ОБЩИЕ ДЛЯ ВСЕХ.

3️⃣ **Методы `getInitParameter()` и `getInitParameterNames()` возвращают данные только ==для текущего сервлета==**, а не для всего приложения.

4️⃣ 🧠При использовании аннотаций (`@WebServlet`) можно задавать `initParams` прямо в коде, и контейнер передаст их в `ServletConfig`.

5️⃣ Если попытаться вызвать методы `ServletConfig` до вызова `init()`, объект может быть `null` (например, при ручном создании сервлета в тестах).

---
## 💎 Best Practices 💎

✅ Используйте `getServletContext()` для доступа к общим ресурсам приложения.

✅ Храните ссылку на `ServletConfig` в поле сервлета, если параметры часто используются.

✅ Проверяйте наличие параметра перед использованием:
```JAVA
String param = config.getInitParameter("threshold");

if (param != null) {
    // использовать значение
}
```

✅ ❗❗❗Не путайте `ServletConfig` (==локальные== параметры сервлета) 
и `ServletContext` (==глобальные== параметры приложения).

✅ ❗❗Если параметров немного, удобнее работать с аннотацией `@WebInitParam`, чем с web.xml.

## Пример
### В `web.xml`
```xml
<servlet>
    <servlet-name>HelloServlet</servlet-name>
    <servlet-class>com.example.HelloServlet</servlet-class>
    
    <init-param>
        <param-name>greeting</param-name>
        <param-value>Elizabeth</param-value>
    </init-param>
</servlet>
```

### В коде сервлета
```java
package servlets;

import java.io.IOException;
import java.io.PrintWriter;

import jakarta.servlet.GenericServlet;
import jakarta.servlet.Servlet;
import jakarta.servlet.ServletConfig;
import jakarta.servlet.ServletException;
import jakarta.servlet.ServletRequest;
import jakarta.servlet.ServletResponse;
import jakarta.servlet.annotation.WebServlet;

@WebServlet(name = "HelloServlet", urlPatterns = "/hello")
public class HelloServlet implements Servlet  {
	
	private ServletConfig config;
	
	@Override
	public void init(ServletConfig arg0) throws ServletException {
		this.config = arg0;
        System.out.println("Initialization: " + config.getServletName());
	}

	@Override
	public void service(
		ServletRequest arg0, 
		ServletResponse arg1) throws ServletException, IOException 
	{
		arg1.setContentType("text/html; charset=UTF-8");
        PrintWriter out = arg1.getWriter();

        String greeting = config.getInitParameter("greeting");
        if (greeting == null) greeting = "Zdarova!";
        out.println("<h1>" + greeting + "</h1>");
		
	}

	
	@Override
	public void destroy() {
		// TODO Auto-generated method stub
		
	}

	@Override
	public ServletConfig getServletConfig() {
		return config;
	}

	@Override
	public String getServletInfo() {
		return "HelloServlet v1.0";
	}
}
```
