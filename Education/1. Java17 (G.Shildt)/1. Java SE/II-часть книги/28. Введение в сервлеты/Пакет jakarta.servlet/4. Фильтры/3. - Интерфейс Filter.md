
> [!NOTE] Интерфейс **`Filter`** — часть Servlet API, 
> используемая для `предобработки` и `постобработки` `HTTP-запросов и ответов`.
> 
> Фильтр может:
> - ⭐`изменять запрос или ответ` (например, добавлять заголовки, логировать, ❗проверять авторизацию❗);
> - `блокировать` дальнейшую `обработку` (например, при ошибке авторизации);
> - `передавать` управление дальше `по цепочке фильтро`в.

Фильтры часто используются для 
- **аутентификации, 
- логирования, 
- сжатия, 
- кэширования, 
- CORS, 
- XSS-фильтрации** и др.

---
## Интерфейс
```java'
package jakarta.servlet;

import java.io.IOException;

public interface Filter {
	// Вызывается контейнером ❗один раз при инициализации фильтра❗
	// Можно считать этот метод аналогом init() у сервлета.
    void init(FilterConfig filterConfig) throws ServletException;
    
    // ⭐⭐Основной метод фильтра.⭐⭐
	// ❗Выполняется для каждого запроса❗, проходящего через фильтр.
    void doFilter(
	    ServletRequest request, 
	    ServletResponse response, 
	    FilterChain chain) throws IOException, ServletException;
    
    // ❗Вызывается контейнером при остановке приложения❗ или уничтожении фильтра.
    void destroy();
}
```

```java
void doFilter(
	ServletRequest request, 
	ServletResponse response, 
	FilterChain chain) throws IOException, ServletException;
```
В нём можно:
- изменить `request` или `response` (например, обернуть в wrapper);
    
- прервать обработку (не вызывать `chain.doFilter()`);
    
- вызвать `chain.doFilter(request, response)` для передачи дальше (следующему фильтру или сервлету).

> [!NOTE] ⭐Метод `chain.doFilter()` ❗`БЛОКИРУЕТСЯ`❗ на время обработки запроса сервлетом⭐
>

## 💡 Особенности и подводные камни 💡

1️⃣ ⭐⭐**Фильтры НЕ ЗАВИСЯТ от сервлетов.**  
Они работают "до" и "после" сервлета, не зная, какой именно сервлет обслуживает запрос.

2️⃣ 💡💡**Обертки (`ServletRequestWrapper` / `ServletResponseWrapper`).**  
Если нужно модифицировать тело запроса или ответ — оберни оригинальные объекты и передай их в `chain.doFilter()`.

3️⃣ **Порядок выполнения фильтров** определяется:
- порядком `<filter-mapping>` в `web.xml`,
- либо аннотацией `@WebFilter` (контейнер решает порядок сам).

4️⃣ **Фильтры должны быть статичными.**  
❗НЕ ХРАНЯТ СОСТОЯНИЕ запроса в полях класса❗ — иначе получишь гонки потоков.

5️⃣ ❗❗❗❗⭐**Важно вызывать `chain.doFilter()`!**  
Если забыть, запрос "зависнет" и дальше не пойдёт.

6️⃣ **Фильтры не могут напрямую управлять жизненным циклом сервлетов.**  
Они работают _вокруг_ вызова сервлета, но не контролируют его создание.

7️⃣ **Ловля исключений.**  
❗❗Если фильтр выбросит `ServletException` или `IOException`, ==контейнер прекратит цепочку== и вызовет error-handling.

---
## 💎 Best Practices 💎

✅ 💡Используй **обертки (wrappers)** для безопасного изменения запроса/ответа.  
✅ 💡Всегда вызывай `chain.doFilter()` (если только фильтр не должен прервать цепочку).  

✅ Делай фильтры **потокобезопасными** — не используй поля для хранения состояния запроса.  

✅ 💡Для логирования и метрик — используй `try-finally`, чтобы гарантировать выполнение после `chain.doFilter()`.  

✅ ⭐Разделяй ответственность — один фильтр = одна задача (логирование, аутентификация, CORS и т.п.).

## Пример
Напишем фильтр, КОТОРЫЙ НЕ ПРИНИМАЕТ ЗАПРОСЫ, БЕЗ ПАРАМЕТРА `name`
Если параметра нет, ТО НА ЭТАПЕ ФИЛЬТРАЦИИ вернется ответ
```java
package servlets;

import java.io.IOException;
import java.util.Optional;

import jakarta.servlet.Filter;
import jakarta.servlet.FilterChain;
import jakarta.servlet.ServletException;
import jakarta.servlet.ServletOutputStream;
import jakarta.servlet.ServletRequest;
import jakarta.servlet.ServletResponse;

public class MyFilter implements Filter{

	@Override
	public void doFilter(
			ServletRequest req, 
			ServletResponse resp, 
			FilterChain filterChain) throws IOException, ServletException 
	{
		System.out.println("Log: In filter");
		
		//Получение имени если есть
		Optional<String> userName = Optional
			.ofNullable(req.getParameter("name"));
		
		//Если имени нет, то выводим сообщение пользователю
		if(userName.isEmpty()) {
			resp.setContentType("text/plain");
			
			ServletOutputStream writerResponce = resp.getOutputStream();
			
			writerResponce.println("Ebat loshara, input your name");
			return; // ❗❗ОБЯЗАТЕЛЬНО! Иначе ответ не отрпавится❗❗
		}
		//Если параметр есть, значит пускаем дальше по цепочке
		filterChain.doFilter(req, resp);
	}
}
```


Сервлет, просто читаем тело запроса, и записывает в ответ
```java
package servlets;

import java.io.IOException;

import jakarta.servlet.AsyncContext;
import jakarta.servlet.ReadListener;
import jakarta.servlet.ServletException;
import jakarta.servlet.ServletInputStream;
import jakarta.servlet.http.HttpServlet;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;

public class HelloServlet extends HttpServlet {
	
	@Override
    protected void doPost(
	    HttpServletRequest req, 
	    HttpServletResponse resp) throws ServletException, IOException 
	{
        AsyncContext asyncContext = req.startAsync();

        ServletInputStream input = req.getInputStream();


        input.setReadListener(new ReadListener() {
            private StringBuilder buffer = new StringBuilder();

            @Override
            public void onDataAvailable() throws IOException {
                byte[] data = new byte[1024];
                int len;
                while (
	                input.isReady() && (len = input.read(data)) != -1
	            ) {
                    buffer.append(new String(data, 0, len));
                }
            }

            @Override
            public void onAllDataRead() throws IOException {
                HttpServletResponse response = 
	                (HttpServletResponse) asyncContext.getResponse();
                
                response.setContentType("text/plain;charset=UTF-8");
                response.getWriter().write("Received: " + buffer);
                asyncContext.complete();
            }

            @Override
            public void onError(Throwable t) {
                t.printStackTrace();
                asyncContext.complete();
            }
        });
    }
}
```
