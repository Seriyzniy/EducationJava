## Асинхронность и синхронность
🔶**Синхронность по умолчанию.** Все фильтры и сервлеты выполняются в **одном потоке** запроса

🔶**Если фильтр или сервлет поддерживает асинхронную обработку** 
(`<async-supported>true</async-supported>`),  
тогда вызов 
`chain.doFilter()` 
может не блокироваться — если внутри сервлета был вызван `request.startAsync()`.
> [!NOTE] но это требует, чтобы **все** фильтры в цепочке поддерживали `async-supported=true`.

# 📕 1. Фильтры
> [!NOTE] Фильтр
> Фильтр — это объект, который может преобразовывать заголовок и содержимое (или и то, и другое) запроса или ответа. 
> 
> Фильтры отличаются от веб-компонентов тем, что фильтры обычно сами по себе не создают ответ, а наоборот `перехватывает HTTP-запрос` `перед тем, как он попадёт в сервлет` (или JSP), и/`или ответ` — перед тем, как он `уйдёт` клиенту.

Применение фильтров включает:
- аутентификацию, 
- ведение журнала, 
- преобразование изображений, 
- сжатие данных, 
- шифрование, 
- токенизацию потоков, 
- преобразования XML 
- и так далее.

Вы можете настроить ==фильтрацию== веб-ресурса ==по цепочке== ==(Filter Chain)== из нуля, одного или нескольких фильтров в определенном порядке. 

Эта цепочка указывается при развертывании веб-приложения, содержащего компонент, и создается при загрузке компонента веб-контейнером.

Каждый фильтр решает:
- обработать запрос полностью самостоятельно,
    
- изменить запрос/ответ и передать дальше,
    
- или прервать цепочку (например, при отказе в доступе).

# 📕 2. Программирование индивидуальных запросов и ответов 

Фильтр, изменяющий ответ, обычно должен захватывать ответ до того, как он будет возвращен клиенту. 

Для этого вы передаете поток замены сервлету, который генерирует ответ. 

==Заменяющий поток НЕ ПОЗВОЛЯЕТ СЕРВЛЕТУ ЗАКРЫТЬ исходный поток== ответа после его завершения и позволяет фильтру изменять ответ сервлета.

Для этого используются ОБОЛОЧКИ
- оболочки запросов/ответов
- оболочки для ПОТОКОВ чтения/записи тела

## 📙1. Использование оберток
Фильтры (и иногда сервлеты) могут захотеть:
- **модифицировать запрос** перед тем, как он дойдёт до сервлета;
- **перехватить или изменить ответ**, прежде чем он уйдёт клиенту.

❗❗Однако `HttpServletRequest` и `HttpServletResponse` — это **интерфейсы, реализуемые контейнером**, и вы не можете просто "изменить" их поведение напрямую.  

Здесь и приходят на помощь **классы-обёртки** — они действуют как **декораторы**, позволяя переопределить нужные методы и делегировать остальные.
(см [[1. - Декоратор ServletRequestWrapper]] [[2. - Декоратор ServletResponseWrapper]])

|Класс|Назначение|
|---|---|
|`jakarta.servlet.ServletRequestWrapper`|Базовая обёртка для `ServletRequest`|
|`jakarta.servlet.http.HttpServletRequestWrapper`|Упрощает работу с HTTP-запросами|
|`jakarta.servlet.ServletResponseWrapper`|Базовая обёртка для `ServletResponse`|
|`jakarta.servlet.http.HttpServletResponseWrapper`|Расширяет функциональность HTTP-ответа|

## Принцип работы обертки
Каждая обёртка **содержит оригинальный объект запроса/ответа** и **переопределяет** (при необходимости) отдельные методы, чтобы изменить поведение.

📌 Примерная структура:
```java
public class MyRequestWrapper extends HttpServletRequestWrapper {
    public MyRequestWrapper(HttpServletRequest request) {
        super(request); // сохраняем оригинальный запрос
    }

    @Override
    public String getParameter(String name) {
        String value = super.getParameter(name);
        
        // например, убираем пробелы
        return value == null ? null : value.trim(); 
    }
}
```

## 🧠 Где и зачем используются

### 1️⃣ В фильтрах — для изменения входных данных (request)

Например:
- Заменить тело POST-запроса перед тем, как он попадёт в сервлет (например, логирование, XSS-фильтрация, JSON-трансформация).
    
- Добавить или изменить параметры (`getParameter`).
    
- Изменить заголовки (`getHeader`).
    
- Подменить `InputStream` для чтения тела.

```java
@WebFilter("/*")
public class TrimFilter implements Filter {
    @Override
    public void doFilter(
	    ServletRequest request, 
	    ServletResponse response,
	    FilterChain chain) throws IOException, ServletException 
	{
        HttpServletRequest httpRequest = (HttpServletRequest) request;
        
        // ⭐ Использование ОБЕРТКИ запроса с почищенными пробелами⭐
        chain.doFilter(new TrimmedRequestWrapper(httpRequest), response);
    }
}
```

### 2️⃣ В фильтрах — для изменения выходных данных (response)
Например:
- Перехватить вывод, изменить HTML перед отправкой клиенту.
    
- Добавить заголовки.
    
- Сжать содержимое.
    
- Сохранить ответ в лог.
    

Пример — фильтр, который логирует тело ответа:
```java
public class LoggingResponseWrapper extends HttpServletResponseWrapper {
    private ByteArrayOutputStream output = new ByteArrayOutputStream();
    private PrintWriter writer = new PrintWriter(output);

    public LoggingResponseWrapper(HttpServletResponse response) {
	    // Получаем в параметре исходный ОТВЕТ
	    // и передаем его в конструктор обертки
        super(response);
    }

    @Override
    public PrintWriter getWriter() {
        return writer; // подменяем writer
    }

    public String getContent() {
        writer.flush();
        return output.toString(StandardCharsets.UTF_8);
    }
}
```

### 3️⃣ Для добавления новых "виртуальных" параметров
Например, можно сделать фильтр, который добавляет параметр `"userId"` в каждый запрос, если он есть в JWT.
```java
@Override
public String getParameter(String name) {
    if ("userId".equals(name)) {
        return extractUserIdFromToken();
    }
    return super.getParameter(name);
}
```

### 4️⃣ Для реализации кэширования или постобработки контента
Например:
- Обёртка ответа сохраняет данные в буфере;
    
- Фильтр анализирует/модифицирует эти данные;
- Потом отправляет их пользователю.

## 📙2. Использование кастомных ПОТОКОВ
Тема **кастомных `ServletOutputStream` / `ServletInputStream`** напрямую связана с **обёртками** (`ServletRequestWrapper`, `ServletResponseWrapper`), 

потому что именно **через обёртки** можно подменить стандартные потоки ввода/вывода контейнера.

### ❗❗❗Где появляются потоки❗❗❗
У `HttpServletRequest` и `HttpServletResponse` есть методы:
```java
ServletInputStream getInputStream()
ServletOutputStream getOutputStream()
```

Они ==дают ДОСТУП к "ТЕЛУ" запроса и ответа== (body):
- **`ServletInputStream`** — читает тело запроса (например, JSON POST);
- **`ServletOutputStream`** — пишет тело ответа клиенту.

Эти потоки **реализуются контейнером** (Tomcat, Jetty и т.п.), и обычно программист их просто использует.

### ⚙️ Зачем нужны кастомные реализации
Иногда нужно **вмешаться в работу этих потоков**:
- перехватить или изменить содержимое;
- кэшировать данные, проходящие через поток;
    
- анализировать или фильтровать их (например, XSS, JSON-валидация);
    
- производить сжатие (GZIP);
    
- реализовать логирование тела запроса/ответа.

Именно для этого создаются **кастомные подклассы** `ServletInputStream` / `ServletOutputStream`,  
которые можно **подставить** вместо стандартных — обычно через **обёртку**.

### 🧠 Где и как используются

#### 🔸 1. В кастомных `HttpServletRequestWrapper`
Позволяет подменить тело запроса.

==По умолчанию== ТЕЛО (`InputStream`) МОЖНО ==ПРОЧИТАТЬ ОДИН РАЗ== — контейнер не даёт повторно его использовать.  

Если фильтру нужно прочитать тело (например, логировать JSON), а потом чтобы сервлет тоже его читал —  
❗❗❗нужно **сохранить тело** и отдать копию из кастомного `ServletInputStream`.

##### Обертка ЗАПРОСА
```java
public class CachedBodyRequestWrapper extends HttpServletRequestWrapper {
    private final byte[] cachedBody;

    public CachedBodyRequestWrapper(HttpServletRequest request) throws IOException {
        super(request);
        // читаем и кешируем тело запроса
        cachedBody = request.getInputStream().readAllBytes();
    }

    @Override
    public ServletInputStream getInputStream() {
        // возвращаем новый поток поверх сохранённого тела
        return new CachedBodyInputStream(cachedBody);
    }

    // ⭐⭐кастомный ServletInputStream⭐⭐
    private static class CachedBodyInputStream extends ServletInputStream {
        private final ByteArrayInputStream inputStream;

        public CachedBodyInputStream(byte[] body) {
            this.inputStream = new ByteArrayInputStream(body);
        }

        @Override
        public int read() {
            return inputStream.read();
        }

        @Override
        public boolean isFinished() {
            return inputStream.available() == 0;
        }

        @Override
        public boolean isReady() {
            return true;
        }

        @Override
        public void setReadListener(ReadListener readListener) {}
    }
}
```
📍 **Где применяется:**  
в фильтрах логирования, XSS-фильтрации, JSON-парсинга, подписи данных, валидации входящих запросов.

---
#### 🔸 2. В кастомных `HttpServletResponseWrapper`
Позволяет перехватить тело ответа.

Аналогично, фильтр может захотеть:
- изменить HTML-ответ;
- посчитать длину данных;
- вставить заголовки;
- применить компрессию или шифрование.
    

Для этого создаётся обёртка, подменяющая `getOutputStream()`.
##### Обертка ОТВЕТА
```java
public class CachedBodyResponseWrapper 
	extends HttpServletResponseWrapper 
{
    private final ByteArrayOutputStream buffer = 
	    new ByteArrayOutputStream();
	    
    private final ServletOutputStream outputStream = 
	    new CachedBodyOutputStream(buffer);
	    
    private PrintWriter writer = new PrintWriter(outputStream, true);



    public CachedBodyResponseWrapper(HttpServletResponse response) {
        super(response);
    }

    @Override
    public ServletOutputStream getOutputStream() {
	    // ⭐ Возвращаем КАСТОМНЫЙ ПОТОК
        return outputStream;
    }

    @Override
    public PrintWriter getWriter() {
        return writer;
    }

    public byte[] getCachedBody() {
        writer.flush();
        return buffer.toByteArray();
    }


    // ⭐ кастомный ServletOutputStream ⭐
    private static class CachedBodyOutputStream 
	    extends ServletOutputStream 
	{
        private final OutputStream stream;

        public CachedBodyOutputStream(OutputStream stream) {
            this.stream = stream;
        }

        @Override
        public void write(int b) throws IOException {
            stream.write(b);
        }

        @Override
        public boolean isReady() {
            return true;
        }

        @Override
        public void setWriteListener(WriteListener writeListener) {}
    }
}
```
📍 **Где применяется:**  
в фильтрах логирования ответа, компрессии (GZIP), постобработки HTML/JSON/XML, добавления подписей, внедрения статистики.

### Пример использования в фильтре
```java
@WebFilter("/*")
public class ResponseLoggingFilter implements Filter {
    @Override
    public void doFilter(
	    ServletRequest req, 
	    ServletResponse res, 
	    FilterChain chain) throws IOException, ServletException 
	{
        HttpServletResponse httpRes = (HttpServletResponse) res;
        
        //Обертка НАД ОТВЕТОМ
        CachedBodyResponseWrapper responseWrapper = 
	        new CachedBodyResponseWrapper(httpRes);

        chain.doFilter(req, responseWrapper); // обработка сервлетом


        // теперь можем анализировать тело ответа
        String content = new String(responseWrapper.getCachedBody(), StandardCharsets.UTF_8);
        System.out.println("Response body: " + content);

        // возвращаем оригинальный ответ клиенту
        ServletOutputStream out = httpRes.getOutputStream();
        out.write(responseWrapper.getCachedBody());
        out.flush();
    }
}
```
