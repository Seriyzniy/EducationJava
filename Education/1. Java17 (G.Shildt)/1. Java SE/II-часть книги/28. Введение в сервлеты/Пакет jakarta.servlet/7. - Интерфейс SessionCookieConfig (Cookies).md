
> [!NOTE] Интерфейс **`SessionCookieConfig`** предоставляет `API для настройки cookie`, 
> которые контейнер использует для идентификации HTTP-сессий (обычно `JSESSIONID`).
> 

С его помощью можно изменить:
- имя cookie (по умолчанию `JSESSIONID`),
- домен, путь, флаг безопасности (`Secure`),
- флаг `HttpOnly`,
- время жизни cookie.

Получить объект `SessionCookieConfig` можно через 
```java
ServletContext.getSessionCookieConfig()
```  
и настроить до первого вызова `getSession()` в приложении.

> 📘 Используется для тонкой настройки безопасности и поведения сессий в контейнере.

---
## Методы
### Настройка идентификационных свойств cookie
```java
// Устанавливает имя cookie для сессии (по умолчанию "JSESSIONID")
void setName(String name);

// Устанавливает домен, в рамках которого действует cookie
void setDomain(String domain);

// Устанавливает путь (Path) — ограничивает область действия cookie
void setPath(String path);
```

### Настройка безопасности cookie
```java
// Устанавливает, что cookie отправляется только по HTTPS
void setSecure(boolean secure);

// Устанавливает флаг HttpOnly — cookie недоступно из JavaScript
void setHttpOnly(boolean httpOnly);
```

### Настройка времени жизни cookie
```java
// Устанавливает срок жизни cookie в секундах (0 — удалить при закрытии браузера)
void setMaxAge(int expiry);
```

### Прочие параметры
```java
// Добавляет комментарий (редко используется)
void setComment(String comment);
```

## 💡 Особенности и подводные камни 💡

1️⃣ ❗❗**Настраивается только до первого вызова `HttpSession`.**  
После создания первой сессии настройки игнорируются — контейнер уже “запомнил” параметры cookie.

2️⃣ ==**Используется контейнером==, а не напрямую разработчиком.**  
Т.е. вы не создаёте cookie вручную — контейнер сам применяет конфигурацию при генерации `Set-Cookie`.

3️⃣ 💡**Флаг `HttpOnly`** — защищает от XSS-атак, предотвращая доступ JS к сессионным cookie.

4️⃣ 💡**Флаг `Secure`** — обязателен при работе по HTTPS. Без него браузер может отправить cookie и по незащищённому HTTP.

5️⃣ 🧠**`setMaxAge(0)` ≠ logout.**  
Это лишь время хранения cookie, но не уничтожение серверной сессии (`session.invalidate()` нужно вызвать отдельно).

6️⃣ **Влияет на все сессии в контексте.**  
Настройки `SessionCookieConfig` глобальны для всего приложения (ServletContext).

---
## 💎 Best Practices 💎

✅ 💡Настраивай `SessionCookieConfig` **внутри `ServletContextListener.contextInitialized()`** — до первого запроса.  

✅ Обязательно включай:
```java
config.setHttpOnly(true); 
config.setSecure(true);
```
— для безопасных приложений.  

✅ Используй осмысленные пути (`setPath("/")`) и домены (`setDomain(".example.com")`) при работе с несколькими поддоменами.  

✅ Не меняй `JSESSIONID` на нестандартное имя без необходимости — это может нарушить интеграцию с балансировщиками.  

✅ 🧠Для тестовых стендов можно установить `setMaxAge(-1)` (cookie живёт до закрытия браузера).

## Пример
```java
import jakarta.servlet.*;

public class MyAppInitializer implements ServletContextListener {

    @Override
    public void contextInitialized(ServletContextEvent sce) {
        ServletContext ctx = sce.getServletContext();

        SessionCookieConfig config = ctx.getSessionCookieConfig();

        // Настройка cookie для сессий
        config.setName("MYSESSIONID");
        config.setHttpOnly(true);
        config.setSecure(true);
        config.setPath("/");
        config.setMaxAge(3600); // 1 час
        config.setComment("Session cookie for MyApp");

        System.out.println("Session cookie настроен: " + config.getName());
    }

    @Override
    public void contextDestroyed(ServletContextEvent sce) {
        // ничего
    }
}
```
