Пакет `jakarta.servlet` содержит несколько интерфейсов и классов, образующих инфраструктуру, в которой функционируют сервлеты


> [!NOTE] `Базовые интерфейсы` Servlet и  ServletRequest и ServletResponse
> Наиболее важным из них является `Servlet`.
> ⭐ `Все сервлеты должны реализовы вать интерфейс Servlet` или расширять реализующий его класс. 
> 
> Интерфейсы `ServletRequest` и `ServletResponse` тоже очень важны.

## Основные ИНТЕФРЕЙСЫ в пакете jakarta.servlet

| Интерфейс        | Назначение                                                                                                     |
| ---------------- | -------------------------------------------------------------------------------------------------------------- |
| ⭐**Servlet**     | Объявляет `методы жизненного цикла` сервлета                                                                   |
|                  |                                                                                                                |
| 🧠ServletConfig  | Позволяет сервлетам `ПОЛУЧАТЬ` `параметры инициализации`                                                       |
| 🧠ServletContext | Позволяет сервлетам `регистрировать события в журнале` и `получать доступ к информации`, касающейся `их среды` |
|                  |                                                                                                                |
| ServletRequest   | Используется `для чтения данных` из клиентского `запроса`                                                      |
| ServletResponse  | Используется `для записи данных` в клиентский `ответ`                                                          |
|                  |                                                                                                                |

## Основные КЛАССЫ в пакете jakarta.servlet

| Класс                  | Назначение                                               |
| ---------------------- | -------------------------------------------------------- |
| GenericServlet         | Реализует интерфейсы Servlet и ServletConfig             |
|                        |                                                          |
| ServletInputStream     | Инкапсулирует поток ввода для чтения запросов от клиента |
| ServletOutputStream    | Инкапсулирует поток вывода для записи ответов клиенту    |
|                        |                                                          |
| 🚨ServletException     | Указывает на то, что в сервлете произошла ошибка         |
| 🚨UnavailableException | Указывает на то, что сервлет недоступен                  |

# ПОЛНЫЙ СПИСОК ИНТЕРФЕЙСОВ И КЛАССОВ
## **Интерфейсы (Interface Summary)**
| Интерфейс                         | Описание                                                                                   |
| --------------------------------- | ------------------------------------------------------------------------------------------ |
| `AsyncContext` ✅                  | Предоставляет контекст для асинхронной обработки запросов                                  |
| `AsyncListener`✅                  | Слушатель событий, связанных с AsyncContext                                                |
| `Filter` ✅                        | Объект, выполняющий задачи фильтрации запросов к ресурсу и/или ответов от ресурса          |
| `FilterChain`✅                    | Объект, предоставляющий представление в цепочку вызовов фильтрованного запроса для ресурса |
| `FilterConfig`✅                   | Объект конфигурации фильтра, используемый для передачи информации при инициализации        |
| `FilterRegistration`✅             | Интерфейс для дополнительной конфигурации Filter                                           |
| `FilterRegistration.Dynamic`✅     | Интерфейс для дополнительной конфигурации Filter, зарегистрированного через addFilter      |
| `ReadListener` ✅                  | Получает уведомления о событиях чтения при использовании неблокирующего IO                 |
| `Registration`✅                   | Общий интерфейс для регистрации Filter и Servlet                                           |
| `Registration.Dynamic`            | Интерфейс для дополнительной конфигурации Servlet или Filter                               |
| `RequestDispatcher`✅              | Объект, который получает запросы от клиента и отправляет их на ресурсы сервера             |
| `Servlet` ✅                       | Определяет методы, которые должны реализовывать все сервлеты                               |
| `ServletConfig`✅                  | Объект конфигурации сервлета, используемый для передачи информации при инициализации       |
| `ServletContainerInitializer`     | Инициализаторы контейнера сервлетов, регистрируемые через META-INF/services                |
| `ServletContext`✅                 | Определяет методы для взаимодействия сервлета с его контейнером                            |
| `ServletContextAttributeListener` | Получает уведомления об изменениях атрибутов контекста сервлета                            |
| `ServletContextListener`✅         | Получает уведомления об изменениях контекста сервлета веб-приложения                       |
| `ServletRegistration`✅            | Интерфейс для дополнительной конфигурации Servlet                                          |
| `ServletRegistration.Dynamic`✅    | Интерфейс для дополнительной конфигурации Servlet, зарегистрированного через addServlet    |
| `ServletRequest` ✅                | Предоставляет информацию о клиентском запросе сервлету                                     |
| `ServletRequestAttributeListener` | Уведомления об изменениях атрибутов запроса                                                |
| `ServletRequestListener`          | Уведомления о запросах, поступающих в веб-компонент и выходящих из него                    |
| `ServletResponse`✅                | Помогает сервлету отправить ответ клиенту                                                  |
| `SessionCookieConfig`✅            | Конфигурирует сессионные куки, используемые веб-приложением                                |
| `SingleThreadModel`               | **Устарело** (начиная с Servlet API 2.4)                                                   |
| `WriteListener`✅                  | Получает уведомления о событиях записи при использовании неблокирующего IO                 |
