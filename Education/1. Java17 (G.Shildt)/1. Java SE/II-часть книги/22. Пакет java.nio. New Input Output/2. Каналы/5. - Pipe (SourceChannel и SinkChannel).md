# 📕 1. Pipe
> [!NOTE] `Pipe` — это `abstract` класс из пакета `java.nio.channels`, представляющий ==ОДНОНАПРАВЛЕННЫЙ канал связи== внутри одного процесса.
> Он состоит из двух частей:
> - **`Pipe.SinkChannel`** — «вход» трубы, в который **записываются** данные, 
> - **`Pipe.SourceChannel`** — «выход», из которого **читаются** данные.

📌 `Pipe` — это альтернатива `PipedInputStream` / `PipedOutputStream`, но работает на уровне NIO и поддерживает неблокирующий режим.

## Конструктор
У `Pipe` **нет публичного конструктора**, объект создаётся фабричным методом:
```java
// Создаёт новую трубу (Pipe) с двумя связанными концами
public static Pipe open() throws IOException;
```

## Методы
```java
// ✅ Получение каналов
// ❗❗❗ Возвращает канал для чтения (источник)
public abstract Pipe.SourceChannel source();

// ❗❗❗ Возвращает канал для записи (приёмник)
public abstract Pipe.SinkChannel sink();
```

# 📕2. Вложенные классы
## 1. Pipe.SourceChannel 

> [!NOTE] `SourceChannel` — это **внутренний класс**, наследующий `AbstractSelectableChannel`  
> Также реализует `ReadableByteChannel` и `ScatteringByteChannel`.
> 
> 💡Он используется ==для **чтения данных**==, записанных в `SinkChannel`.

```java
public abstract static class Pipe.SourceChannel
        extends AbstractSelectableChannel
        implements ReadableByteChannel, ScatteringByteChannel {
	
	// ❗❗❗НАСЛЕДУЕМЫЙ МЕТОД❗❗❗
    // ✅ Чтение данных
    // Читает данные из трубы в буфер
    public abstract int read(ByteBuffer dst) throws IOException;
	
	// ❗❗❗НАСЛЕДУЕМЫЙ МЕТОД❗❗❗
    // Чтение сразу в несколько буферов (scatter read)
    public abstract long read(
	    ByteBuffer[] dsts, 
	    int offset, 
	    int length) throws IOException;


    // ✅ Поддержка Selector
    // Возвращает маску поддерживаемых операций (OP_READ)
    @Override
    public final int validOps() {
        return SelectionKey.OP_READ;
    }
}
```

## 2. Pipe.SinkChannel

> [!NOTE] `SinkChannel` — это **внутренний класс**, наследующий `AbstractSelectableChannel`  
> Также реализует `WritableByteChannel` и `GatheringByteChannel`.
> 
> 💡Он используется ==для **записи данных** в трубу==, откуда их можно прочитать через `SourceChannel`

```java
public abstract static class Pipe.SinkChannel
        extends AbstractSelectableChannel
        implements WritableByteChannel, GatheringByteChannel {
	
	// ❗❗❗НАСЛЕДУЕМЫЙ МЕТОД❗❗❗
    // ✅ Запись данных
    // Записывает содержимое буфера в трубу
    public abstract int write(ByteBuffer src) throws IOException;
	
	// ❗❗❗НАСЛЕДУЕМЫЙ МЕТОД❗❗❗
    // Запись из нескольких буферов (gather write)
    public abstract long write(
	    ByteBuffer[] srcs, 
	    int offset, 
	    int length) throws IOException;

    // ✅ Поддержка Selector
    // Возвращает маску поддерживаемых операций (OP_WRITE)
    @Override
    public final int validOps() {
        return SelectionKey.OP_WRITE;
    }
}

```

***
## 💡 Особенности и подводные камни 💡
1. 💡**Только внутри одного процесса**
    - В отличие от `SocketChannel`, `Pipe` не использует сеть — данные передаются внутри процесса JVM.
        
2. **Ограниченный буфер**
    - Внутри трубы хранится ограниченное количество данных (зависит от реализации ОС).
        
    - Если буфер заполнен — `write()` блокируется (в blocking режиме) или возвращает `0` (в non-blocking).
        
3. **Non-blocking режим**
    - Оба конца (`SourceChannel` и `SinkChannel`) можно перевести в неблокирующий режим и использовать с `Selector`.
        
4. 💡**Двунаправленность**
    - `Pipe` — ==однонаправленный== (sink → source).
        
    - ❗❗Если нужен ==двунаправленный== обмен, нужно создать ==две трубы==.
        
5. ❗❗**Безопасность**
    - Можно безопасно использовать для обмена между потоками внутри JVM, без синхронизации.

## ✅ Best Practices ✅

- 🧠Используй `Pipe` для связи потоков продюсер/консьюмер без очередей.
    
- Для интеграции с `Selector` (например, сигнализация между потоками).
    
- Не забывай вызывать `flip()` перед чтением данных и `clear()` после.
    
- ⚠️ Не путай с `PipedInputStream` — `Pipe` работает на уровне каналов и более производителен.
    
-  ❗❗Для двунаправленного обмена — создай две трубы (A→B и B→A).

## Пример
Простой обмен между потоками
```java
import java.nio.ByteBuffer;
import java.nio.channels.Pipe;

public class PipeExample {
    public static void main(String[] args) throws Exception {
        Pipe pipe = Pipe.open();

        Pipe.SinkChannel sink = pipe.sink();
        Pipe.SourceChannel source = pipe.source();

        // Поток записи
        new Thread(() -> {
            try {
                ByteBuffer buffer = ByteBuffer.allocate(64);
                buffer.put("Hello from SinkChannel!".getBytes());
                buffer.flip();
                sink.write(buffer);
                sink.close();
            } catch (Exception e) {
                e.printStackTrace();
            }
        }).start();

        // Поток чтения
        ByteBuffer buffer = ByteBuffer.allocate(64);
        source.read(buffer);
        buffer.flip();
        System.out.println("Received: " + new String(buffer.array(), 0, buffer.limit()));

        source.close();
    }
}

```

Использование с Selector (non-blocking)
```java
import java.nio.*;
import java.nio.channels.*;
import java.util.Iterator;

public class PipeSelectorExample {
    public static void main(String[] args) throws Exception {
        Pipe pipe = Pipe.open();

        Pipe.SourceChannel source = pipe.source();
        Pipe.SinkChannel sink = pipe.sink();

        source.configureBlocking(false);
        Selector selector = Selector.open();
        source.register(selector, SelectionKey.OP_READ);

        // Поток записи
        new Thread(() -> {
            try {
                Thread.sleep(500);
                ByteBuffer buf = ByteBuffer.wrap("Signal via Pipe".getBytes());
                sink.write(buf);
            } catch (Exception e) {
                e.printStackTrace();
            }
        }).start();

        // Ждём сигнал
        selector.select();
        Iterator<SelectionKey> it = selector.selectedKeys().iterator();
        while (it.hasNext()) {
            SelectionKey key = it.next();
            it.remove();

            if (key.isReadable()) {
                ByteBuffer buf = ByteBuffer.allocate(64);
                source.read(buf);
                buf.flip();
                System.out.println("Selector received: " + new String(buf.array(), 0, buf.limit()));
            }
        }

        source.close();
        sink.close();
        selector.close();
    }
}

```
