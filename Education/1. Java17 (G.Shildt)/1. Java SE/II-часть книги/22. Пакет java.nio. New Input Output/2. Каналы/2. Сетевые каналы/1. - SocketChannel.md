
> [!NOTE] `SocketChannel` — это **абстрактный класс**, наследующий `AbstractSelectableChannel` и реализующий `ByteChannel`, `ScatteringByteChannel`, `GatheringByteChannel`.
> Он работает с **TCP** (stream-oriented) соединениями: клиентские сокеты.

## Конструктор
У `SocketChannel` **нет публичных конструкторов**.  
Создание происходит через фабричные методы:

```java
// Открытие нового канала (без подключения)
public static SocketChannel open() throws IOException;

// Открытие и сразу подключение к удалённому адресу
public static SocketChannel open(SocketAddress remote) throws IOException;
```

## 💡 Особенности и подводные камни 💡

1. **Blocking vs Non-blocking**
    - ==По умолчанию== канал в ==blocking-режиме.==
        
    - В non-blocking режиме `read()` может вернуть `0`, а `write()` — записать меньше, чем есть в буфере.
        
    - Для асинхронного ввода-вывода используется с `Selector`.
        
2. 💡**Подключение**
    - В blocking режиме `connect()` сразу устанавливает соединение.
        
    - В non-blocking режиме `connect()` возвращает `false`, а соединение завершается позже через `finishConnect()`.
        
3. 💡**Совместимость с java.net.Socket**
    - `SocketChannel` ==оборачивает обычный== `Socket`.
        
    - Через `socket()` можно настраивать параметры TCP (timeout, TCP_NODELAY, linger и др.).
        
4. **Чтение/запись через буферы**
    - Работает напрямую с `ByteBuffer`.
        
    - Поддерживает scatter/gather операции.
        
5. **Ограничения**
    - Только TCP (stream-oriented). Для UDP нужен `DatagramChannel`.

##  ✅ Best Practices ✅

- ⚡ Для неблокирующего режима всегда проверяй возвращаемые значения (`0`, `-1`).
    
- 🛠 Используй `Selector` для масштабируемых серверов (множество клиентов на один поток).
    
- 📌 После записи данных проверяй `buffer.hasRemaining()`, так как `write()` может записать не всё.
    
- 🔒 При работе с несколькими потоками синхронизируй доступ к каналу.
    
- 🔄 Для graceful shutdown — используй `shutdownOutput()` перед `close()`.
    
- 🧩 Для настройки TCP используй `socket().setOption(...)`.

## Пример
Клиент в blocking-режиме
```java
import java.net.InetSocketAddress;
import java.nio.ByteBuffer;
import java.nio.channels.SocketChannel;

public class BlockingSocketChannelExample {
    public static void main(String[] args) throws Exception {
        try (SocketChannel socketChannel = SocketChannel.open(new InetSocketAddress("example.com", 80))) {
            // Отправляем HTTP-запрос
            String request = "GET / HTTP/1.1\r\nHost: example.com\r\n\r\n";
            ByteBuffer buffer = ByteBuffer.wrap(request.getBytes());
            while (buffer.hasRemaining()) {
                socketChannel.write(buffer);
            }

            // Читаем ответ
            buffer.clear();
            int bytesRead = socketChannel.read(buffer);
            buffer.flip();
            while (buffer.hasRemaining()) {
                System.out.print((char) buffer.get());
            }
        }
    }
}
```

Клиент в non-blocking-режиме с finishConnect
```java
import java.net.InetSocketAddress;
import java.nio.ByteBuffer;
import java.nio.channels.SocketChannel;

public class NonBlockingSocketChannelExample {
    public static void main(String[] args) throws Exception {
        SocketChannel socketChannel = SocketChannel.open();
        socketChannel.configureBlocking(false);

        if (!socketChannel.connect(new InetSocketAddress("example.com", 80))) {
            while (!socketChannel.finishConnect()) {
                System.out.println("Подключаемся...");
                Thread.sleep(50);
            }
        }

        ByteBuffer buffer = ByteBuffer.wrap("Hello server!".getBytes());
        while (buffer.hasRemaining()) {
            socketChannel.write(buffer);
        }

        socketChannel.close();
    }
}
```
