
> [!NOTE] Классы каналов определены в пакете `java.nio.channels`. 
> `Канал` представляет собой `открытое подключение с источником` или адресатом `ввода-вывода`. 
> 
> ❗❗Все классы каналов РЕАЛИЗУЮТ интерфейс `Channel`
> 
> 🧠`Channel`, который расширяет интерфейсы `Closeable` и `AutoCloseable`. 
> 

## Получение канала через getChannel()
Один из ==способов получения канала== предусматривает ==вызов метода==`getChannel()` на объекте, поддерживающем каналы. 

Например, метод `getChannel()` поддерживается следующими классами ввода-вывода:
`DatagramSocket` 
`RandomAccessFile` 
`FileinputStream` 
`ServerSocket` 
`FileOutputStream` 
`Socket`

Конкретный ==тип возвращаемого канала зависит от типа объекта==, на котором вызывается метод `getChannel()` . 

Скажем, при вызове 
- На `FileinputStream`, `FileOutputStream` или `RandomAccessFile` метод `getChannel()` возвращает канал типа `FileChannel`, 
- А при вызове на `java.net.Socket` метод `getChannel()` воз вращает `SocketChannel`.


Еще один способ получения канала предполагает применение одного из статических методов, определенных в классе `Files`. 

Например, с использованием `Files` можно получить байтовый канал, вызвав метод `newByteChannel()`. Он возвращает объект реализации `SeekableByteChannel`, который представляет собой интерфейс, реализованный `FileChannel`.

> [!NOTE] 💡 Суть каналов 💡
> Каналы вроде `FileChannel` и `SocketChannel` ==поддерживают различные методы== `read()` и `write()` , позволяющие выполнять операции ввода-вывода через канал. 
> 


# 📕 1. Интерфейс Channel


> [!NOTE] `Channel` — это ОСНОВНОЙ интерфейс пакета `java.nio.channels`.  
> 
> Он ==представляет== собой ==**открытое подключение** к объекту ввода-вывода==: файл, сокет, сетевой порт, устройство.  
> 

💡 Идея: `Channel` похож на старые `InputStream`/`OutputStream`, но:
- работает с **буферами** (`Buffer`) вместо массивов байтов,
    
- ❗❗❗поддерживает **двунаправленность** (можно читать и писать через один объект),
    
- ❗❗❗может быть **асинхронным или неблокирующим** (в отличие от `Stream`).

```java
package java.nio.channels;

public interface Channel extends Closeable {
    
    // Проверка: открыт ли канал
    public boolean isOpen();

    // Закрытие канала (освобождение ресурсов, закрытие файлового дескриптора/сокета)
    public void close() throws IOException;
}
```

## 💡 Особенности и подводные камни 💡
1. **Базовый контракт**  
    Интерфейс `Channel` сам по себе маленький: всего два метода (`isOpen`, `close`).  
    ❗❗❗Все "настоящие" возможности дают его **==подинтерфейсы==**:
    - `ReadableByteChannel` (метод `int read(ByteBuffer dst)`),
        
    - `WritableByteChannel` (метод `int write(ByteBuffer src)`),
        
    - `SeekableByteChannel` (методы для `position()`, `truncate()`, `size()`),
        
    - `GatheringByteChannel` и `ScatteringByteChannel` (запись/чтение из набора буферов).
        
2. **Закрытие**
    - `Channel` расширяет `Closeable`, поэтому его можно использовать с `try-with-resources`.
        
    - Если не закрыть — утечёт файловый дескриптор.
        
    - После закрытия `isOpen()` возвращает `false`, и дальнейшие операции вызывают `ClosedChannelException`.
        
3. **Потокобезопасность**
    - Большинство каналов ==НЕ ПОТОКОБЕЗОПАСНЫ==.
        
    - Для работы из нескольких потоков нужно либо синхронизировать, либо использовать разные каналы.
        
4. ❗❗**Блокирующий vs неблокирующий режим**
    - Например, `SocketChannel` может быть в blocking или non-blocking режиме.
        
    - В blocking режиме `read()` и `write()` ведут себя как в обычных стримах.
        
    - В non-blocking режиме могут возвращать `0` или меньше ожидаемого количества байтов.

## ✅Best Practices ✅

- Используй `try-with-resources` для каналов (`FileChannel`, `SocketChannel` и т.д.) → это гарантирует `close()`.
    
- 💡Разделяй каналы по назначению:
    - 🔸`FileChannel` для работы с файлами,
        
    - 🔸`SocketChannel` для TCP,
        
    - 🔸`DatagramChannel` для UDP.
        
- ❓❗При больших файлах — используй `FileChannel.transferTo/transferFrom` или `map` (mmap).
    
- ⚠️ Проверяй `isOpen()` перед работой с каналом (особенно в пулах соединений).
    
- ❓❗Для неблокирующих операций — комбинируй с `Selector`.

## Пример с Sokcet
```java
import java.net.InetSocketAddress;
import java.nio.ByteBuffer;
import java.nio.channels.SocketChannel;

public class SocketChannelExample {
    public static void main(String[] args) throws Exception {
        try (SocketChannel socketChannel = SocketChannel.open(
	        new InetSocketAddress("example.com", 80))
	    ) {
            String request = "GET / HTTP/1.1\r\nHost: example.com\r\n\r\n";
            
            ByteBuffer buffer = ByteBuffer.wrap(request.getBytes());

            // Отправляем запрос
            socketChannel.write(buffer);

            // Читаем ответ
            buffer.clear();
            int bytesRead = socketChannel.read(buffer);
            buffer.flip();
            while (buffer.hasRemaining()) {
                System.out.print((char) buffer.get());
            }
        }
    }
}
```
