
> [!NOTE] `AsynchronousChannelGroup` — это **абстрактный класс** из пакета `java.nio.channels`.  
> Он ==УПРАВЛЯЕТ **ГРУППОЙ асинхронных каналов**==, определяя:
> - какие потоки обрабатывают I/O-события;
> - сколько потоков выделяется;
> - когда завершать работу группы.
> 
> Все асинхронные каналы (`AsynchronousSocketChannel`, `AsynchronousServerSocketChannel`, `AsynchronousFileChannel`) могут работать **в рамках одной группы**, используя её **thread pool** для выполнения операций.
> 

## Назначение
`AsynchronousChannelGroup` — это ядро асинхронной модели NIO.2.  

Она позволяет:
- объединять каналы, чтобы они **делили один пул потоков**;
- управлять количеством активных потоков и их завершением;
    
- явно **закрывать** или **ожидать завершения** всех операций;

## Конструкторы
Класс абстрактный, поэтому объектов напрямую не создают.  

Используются фабричные методы `withThreadPool()`, `withFixedThreadPool()`, `withCachedThreadPool()`:
```JAVA
// Создаёт группу на основе существующего ExecutorService (гибко)
public static AsynchronousChannelGroup withThreadPool(
	ExecutorService executor) throws IOException;

// Создаёт группу с фиксированным числом потоков (рекомендуется для серверов)
public static AsynchronousChannelGroup withFixedThreadPool(
        int nThreads, 
        ThreadFactory threadFactory) throws IOException;

// Создаёт группу с динамическим пулом потоков
public static AsynchronousChannelGroup withCachedThreadPool(
        ExecutorService executor, 
        int initialSize) throws IOException;

```
📌**Параметры:**📌
- `executor` — пул потоков (`ExecutorService`), где выполняются операции I/O;
    
- `threadFactory` — создаёт потоки (для именования, приоритета и т.п.);
    
- `nThreads` — число потоков (ограничивает параллельные I/O операции);
    
- `initialSize` — минимальное число активных потоков.

## Методы
```java
// ✅ Связь с каналами
/**
 * Возвращает группу, к которой принадлежит данный канал.
 * @param channel — любой AsynchronousChannel
 * @return группа, управляющая этим каналом
 */
public static AsynchronousChannelGroup currentGroup(
	AsynchronousChannel channel);

/**
 * Проверяет, привязан ли канал к этой группе.
 * @param channel — канал для проверки
 * @return true, если канал принадлежит этой группе
 */
public abstract boolean contains(AsynchronousChannel channel);



// ✅ Управление завершением
/**
 * Инициирует упорядоченное завершение группы.
 * Новые операции не принимаются, но выполняются незавершённые.
 */
public abstract void shutdown();

/**
 * Принудительно завершает группу.
 * Все незавершённые операции будут отменены.
 */
public abstract void shutdownNow();

/**
 * Проверяет, была ли группа завершена (shutdown).
 * @return true, если shutdown() уже вызван
 */
public abstract boolean isShutdown();

/**
 * Проверяет, завершена ли группа полностью (все операции окончены).
 * @return true, если все каналы закрыты и операции завершены
 */
public abstract boolean isTerminated();

/**
 * Блокирует поток до завершения группы или таймаута.
 * @param timeout — время ожидания
 * @param unit — единица измерения времени
 * @return true, если завершилась до таймаута
 */
public abstract boolean awaitTermination(long timeout, TimeUnit unit) throws InterruptedException;



// ✅ Мониторинг и статистика
/**
 * Возвращает количество каналов, зарегистрированных в группе.
 * @return текущее число активных каналов
 */
public abstract int channelCount();

/**
 * Возвращает максимальное количество каналов, которое может содержать группа.
 * @return лимит каналов или -1, если ограничения нет
 */
public abstract int getMaxChannels();

/**
 * Проверяет, доступна ли группа для регистрации новых каналов.
 * @return true, если группа активна и принимает новые каналы
 */
public abstract boolean isAvailable();



// ✅ Управление ресурсами
/**
 * Устанавливает обработчик для неперехваченных исключений в группе.
 * @param handler — обработчик исключений
 */
public abstract void setUncaughtExceptionHandler(Thread.UncaughtExceptionHandler handler);

/**
 * Возвращает текущий обработчик неперехваченных исключений.
 * @return установленный обработчик или null
 */
public abstract Thread.UncaughtExceptionHandler getUncaughtExceptionHandler();



// ✅ Служебные
/**
 * Возвращает связанный ExecutorService.
 * Можно использовать для статистики, мониторинга или кастомной конфигурации.
 */
public abstract ExecutorService executor();

/**
 * Возвращает имя группы для идентификации в логах и мониторинге.
 * @return имя группы или null, если имя не задано
 */
public abstract String getName();

/**
 * Возвращает время создания группы.
 * @return timestamp создания группы в миллисекундах
 */
public abstract long getCreationTime();
```

## 💡 Особенности и подводные камни 💡

### 1. Необходимость групп
- Без явного `AsynchronousChannelGroup` каналы используют **глобальную системную группу**.
    
- Это удобно для простых приложений, но непредсказуемо при высокой нагрузке.

### 2. Совместное использование потоков
- Все каналы в одной группе используют **один пул потоков**.
    
- Это снижает накладные расходы и позволяет ограничить параллелизм.

### 🧠 3. Управление жизненным циклом
- `shutdown()` запрещает новые операции, но ждёт завершения текущих.
    
- `shutdownNow()` — прерывает все незавершённые операции.
    
### 🧠 4. Группы для разных задач
- Можно создать **несколько групп**, например:
    - одна группа — для сетевых каналов,
        
    - другая — для файловых операций.  
        Это даёт гибкость и изоляцию нагрузки.
        

### ❗❗❗ 5. Потенциальные deadlock-и
- Если поток, выполняющий CompletionHandler, сам блокируется (`get()`, `join()`),  
    может заблокировать пул — используйте асинхронную логику до конца.
    

### 🧠 6. Планирование
- Каждая операция `read()`, `write()`, `accept()` — задача, исполняемая пулом.
    
- Чем больше каналов и нагрузка — тем важнее правильно выбрать количество потоков.

## ✅Best Practices ✅
- Для сетевых серверов — создавай **фиксированную группу**:
```java
AsynchronousChannelGroup group = AsynchronousChannelGroup
	.withFixedThreadPool(
		Runtime.getRuntime().availableProcessors(), 
		Executors.defaultThreadFactory()
	);
```

- Используй **одну группу на сервер** — она справится с тысячами клиентов.
    
- 🔒 Всегда **закрывай** группу (`shutdown()`) при завершении работы приложения.
    
- Для независимых подсистем — используй разные группы.


## Пример 1 — Асинхронный сервер с собственной группой потоков
```java
import java.net.*;
import java.nio.*;
import java.nio.channels.*;
import java.util.concurrent.*;


public static void main(String[] args) throws Exception {
	// Создаём группу с фиксированным пулом потоков
	AsynchronousChannelGroup group = AsynchronousChannelGroup
		.withFixedThreadPool(
			4, 
			Executors.defaultThreadFactory()
		);

	// Привязываем сервер к группе
	AsynchronousServerSocketChannel server =
			AsynchronousServerSocketChannel.open(group)
					.bind(new InetSocketAddress(9000));

	System.out.println("Server started with custom thread group...");

	server.accept(null, new CompletionHandler<>() {
		@Override
		public void completed(
			AsynchronousSocketChannel client, 
			Object att) 
		{
			server.accept(null, this); // продолжаем слушать

			ByteBuffer buf = ByteBuffer.allocate(128);
			client.read(
				buf, 
				buf, 
				new CompletionHandler<Integer, ByteBuffer>() 
			{
				@Override
				public void completed(Integer res, ByteBuffer b) {
					if (res == -1) {
						try { 
							client.close(); 
						} catch (Exception ignored) {}
						return;
					}
					b.flip();
					client.write(b, b, 
						new CompletionHandler<Integer, ByteBuffer>() 
					{
						@Override
						public void completed(Integer r, ByteBuffer bb)
						{
							bb.clear();
							client.read(bb, bb, this);
						}

						@Override
						public void failed(
							Throwable exc, 
							ByteBuffer bb) 
						{
							try { 
								client.close(); 
							} catch (Exception ignored) {}
						}
					});
				}

				@Override
				public void failed(Throwable exc, ByteBuffer b) {
					try { client.close(); } catch (Exception ignored) {}
				}
			});
		}

		@Override
		public void failed(Throwable exc, Object att) {
			System.err.println("Accept failed: " + exc);
		}
	});

	// Работаем, пока группа не завершится
	group.awaitTermination(Long.MAX_VALUE, TimeUnit.SECONDS);
}
```
