
> [!NOTE] Интерфейс `Path` - **==абстракция пути== к файлу или каталогу** в файловой системе.
> Он заменяет устаревший класс `java.io.File` и является частью современного API `NIO.2`, предоставляя **платформонезависимую**, безопасную и удобную работу с путями, файлами и каталогами.


> [!NOTE] 🧠Замена `File` на `Path🧠`
> При обновлении унаследованного кода, где задействован класс `File`, определенный в `java.io`, экземпляр `File` можно преобразовать в экземпляр реализации `Path`, вызвав метод `toPath ()` на объекте `File`. 
> 
> Кроме того, экземпляр `File` можно получить, вызвав метод `toFile()`, который определен в интерфейсе `Path`.


Основные возможности:
- построение и разбор путей;
- извлечение компонентов пути (имя, родитель, корень);
- нормализация и разрешение относительных путей;
- проверка относительности и эквивалентности путей;
    
- конвертация в `URI`, `File`, `AbsolutePath` и т.д.

## Интерфейс
```java
package java.nio.file;

public interface Path 
    extends Comparable<Path>, Iterable<Path>, Watchable {

    // ✅ СТАТИЧЕСКИЕ ФАБРИЧНЫЕ МЕТОДЫ (Java 11+)
    // Создает Path из одной или нескольких строк
    // @param first - первая часть пути
    // @param more - дополнительные части пути
    // @return новый объект Path
    static Path of(String first, String... more) {
        return FileSystems.getDefault().getPath(first, more);
    }
    
    // Создает Path из URI
    // @param uri - URI для преобразования в путь
    // @return новый объект Path
    // @throws IllegalArgumentException если URI не может быть преобразован в Path
    static Path of(URI uri) {
        return FileSystems.getDefault().provider().getPath(uri);
    }



    // ✅ Получение БАЗОВОЙ ИНФОРМАЦИИ о пути
    // Возвращает имя файла или каталога (последний элемент пути)
    // @return объект Path, представляющий имя файла/каталога, либо null, если путь пустой
    Path getFileName();

    // Возвращает корневой элемент пути (например, "C:\" или "/")
    // @return объект Path корня, либо null, если путь относительный
    Path getRoot();

    // Возвращает родительский путь (путь без последнего элемента)
    // @return объект Path родителя, либо null, если путь не имеет родителя
    Path getParent();

    // Возвращает количество элементов в пути (например, "a/b/c" → 3)
    int getNameCount();

    // Возвращает элемент пути по индексу (0 — первый)
    // @param index - индекс элемента (от 0 до getNameCount()-1)
    // @return Path, представляющий указанный элемент
    Path getName(int index);

    // Возвращает подпуть от начального до конечного индекса
    // @param beginIndex - начальный индекс (включительно)
    // @param endIndex - конечный индекс (исключительно)
    // @return новый Path, представляющий подпуть
    Path subpath(int beginIndex, int endIndex);



    // ✅ ИТЕРАЦИЯ и СРАВНЕНИЕ
    // Возвращает итератор по элементам пути (по частям: папкам, файлам)
    // @return Iterator<Path>
    @Override
    Iterator<Path> iterator();

    // Сравнивает путь с другим лексикографически
    // @param other - другой путь для сравнения
    // @return отрицательное, ноль или положительное число
    @Override
    int compareTo(Path other);



    // ✅ Работа с ОТНОСИТЕЛЬНЫМИ и АБСОЛЮТНЫМИ путями
    // Проверяет, является ли путь абсолютным
    // @return true, если путь начинается с корня
    boolean isAbsolute();

    // Возвращает абсолютное представление пути (без разрешения символических ссылок)
    // @return абсолютный Path
    Path toAbsolutePath();

    // Возвращает нормализованный путь (убирает ".", "..", лишние разделители)
    // @return нормализованный путь
    Path normalize();

    // Разрешает (добавляет) другой путь относительно текущего
    // @param other - путь, который нужно добавить
    // @return новый объединённый Path
    Path resolve(Path other);

    // Разрешает другой путь, заданный строкой
    // @param other - строка пути
    // @return новый объединённый Path
    Path resolve(String other);

    // Разрешает относительный путь между двумя путями
    // @param other - целевой путь
    // @return относительный путь от текущего до other
    Path relativize(Path other);

    // Разрешает строку как родственную к текущему пути
    // @param other - строка родственного пути
    // @return новый Path
    Path resolveSibling(Path other);

    // Разрешает строку как родственную к текущему пути
    // @param other - строка родственного пути
    // @return новый Path
    Path resolveSibling(String other);



    // ✅ СРАВНЕНИЕ и проверка ЭКВИВАЛЕНТНОСТИ
    // Проверяет, начинается ли путь с указанного префикса
    // @param other - путь для проверки
    // @return true, если текущий путь начинается с данного
    boolean startsWith(Path other);

    // Проверяет, начинается ли путь с указанной строки
    // @param other - строка для проверки
    // @return true, если текущий путь начинается с данной строки
    boolean startsWith(String other);

    // Проверяет, заканчивается ли путь указанным суффиксом
    // @param other - путь для проверки
    // @return true, если текущий путь оканчивается данным
    boolean endsWith(Path other);

    // Проверяет, заканчивается ли путь указанной строкой
    // @param other - строка для проверки
    // @return true, если текущий путь оканчивается данной строкой
    boolean endsWith(String other);

    // Проверяет, эквивалентны ли два пути, игнорируя регистр и особенности ОС
    // @param other - другой путь
    // @return true, если пути логически равны
    boolean equals(Object other);

    // Вычисляет хэш-код пути
    // @return hashCode
    int hashCode();



    // ✅ ПРЕОБРАЗОВАНИЯ
    // Преобразует путь в строку
    // @return строковое представление пути
    @Override
    String toString();

    // Преобразует путь в объект File (для совместимости со старым API)
    // @return File-объект
    java.io.File toFile();

    // Преобразует путь в URI (универсальный идентификатор)
    // @return объект URI
    java.net.URI toUri();

    // Преобразует путь в объект FileSystem
    // @return файловая система, к которой относится путь
    FileSystem getFileSystem();

    // Преобразует путь в реальный путь на диске (разрешая символические ссылки)
    // @param options - параметры (например, LinkOption.NOFOLLOW_LINKS)
    // @return Path, представляющий реальное местоположение
    Path toRealPath(LinkOption... options) throws IOException;



    // ✅ Работа с НАБЛЮДЕНИЕМ ЗА ИЗМЕНЕНИЕМ файлов
    // Регистрирует путь в WatchService для отслеживания изменений
    // @param watcher - объект WatchService
    // @param events - типы событий (CREATE, DELETE, MODIFY)
    // @return ключ регистрации
    @Override
    WatchKey register(
        WatchService watcher, 
        WatchEvent.Kind<?>... events) throws IOException;

    // Регистрирует путь с дополнительными модификаторами
    // @param watcher - сервис наблюдения
    // @param events - типы событий
    // @param modifiers - модификаторы (например, SensitivityWatchEventModifier)
    // @return ключ регистрации
    WatchKey register(
        WatchService watcher, 
        WatchEvent.Kind<?>[] events, 
        WatchEvent.Modifier... modifiers) throws IOException;
}
```

> [!NOTE] Интерфейс `Watchable` описывает объект, изменения которого можно отслеживать.
> 


## 💡Особенности и подводные камни💡
1. **`Path` не хранит физический файл**, а ==только его представление в виде логического пути==.  
    Файл может существовать или не существовать — это проверяется через `Files.exists(path)`.
    
2. **Относительные пути** не имеют корня (`getRoot()` возвращает `null`).  
    Чтобы сделать путь абсолютным, нужно вызвать `toAbsolutePath()`.
    
3. **Нормализация (`normalize()`)** не обращается к файловой системе, а просто убирает лишние `.` и `..`.
    
4. **Символические ссылки**: `toRealPath()` может вернуть другой путь, если файл является симлинком.
    
5. 💡**Кроссплатформенность**: Разделители (`/` и `\`) автоматически адаптируются под ОС,  
    но строки путей, записанные вручную, могут привести к ошибкам.

---
## ✅ Best Practices

- Для создания пути используйте `Paths.get(...)` или `Path.of(...)`, а не `new File(...)`:
```java
Path path = Path.of("data", "config.json");
```

- Для проверки существования файла:
```java
if (Files.exists(path)) { ... }
```

- ⭐Для обхода дерева каталогов используйте `Files.walk(path)`.
    
- ⭐Для ==сравнения путей== используйте `equals()` или `Files.isSameFile(path1, path2)`,  
    а не сравнение строк — строковое представление может различаться.
    
- При работе с относительными путями вызывайте `normalize()` и `toAbsolutePath()` для предсказуемого поведения.

## Пример
```java
import java.nio.file.*;

public class PathExample {
    public static void main(String[] args) {
        Path path = Path.of("data", "reports", "2025", "summary.txt");

        System.out.println("Путь: " + path);
        System.out.println("Имя файла: " + path.getFileName());
        System.out.println("Родитель: " + path.getParent());
        System.out.println("Корень: " + path.getRoot());
        System.out.println("Абсолютный путь: " + path.toAbsolutePath());
        System.out.println("Количество элементов: " + path.getNameCount());

        // Разрешение относительного пути
        Path newPath = path.resolveSibling("archive.zip");
        System.out.println("Новый путь: " + newPath);

        // Относительный путь между двумя путями
        Path other = Path.of("data", "reports");
        System.out.println("Относительный путь: " + other.relativize(path));
    }
}
```
