
> [!NOTE] `SelectionKey` — это **абстрактный класс** из пакета `java.nio.channels`
> Представляет ==**регистрацию канала в селекторе**== и состояние этого канала в рамках регистрации.

Другими словами, когда ты делаешь:
```java
SelectionKey key = channel.register(
	selector, 
	SelectionKey.OP_READ);
```

— ты ==создаёшь== `SelectionKey`, который будет:
- хранить интересующие операции (`interestOps`);
- отслеживать готовые операции (`readyOps`);
    
- 💡хранить ссылку на сам `channel`;
    
- знать, в каком `selector` он зарегистрирован;
    
- содержать “привязанное” пользовательское состояние (`attachment`).

## Конструктор
Класс абстрактный, поэтому напрямую экземпляры не создаются.
Они создаются внутри реализации Selector при регистрации канала.

Экземпляр `SelectionKey`==возвращается вызовом== `channel.register(selector, ops)`  и живёт, пока канал не будет отменён или закрыт.

```java
// ✅ Общие методы
/**
 * Возвращает канал, связанный с этим ключом.
 * @return зарегистрированный канал
 */
public abstract SelectableChannel channel();

/**
 * Возвращает селектор, с которым связан этот ключ.
 * @return селектор, управляющий этим ключом
 */
public abstract Selector selector();

/**
 * Проверяет, действителен ли еще этот ключ.
 * @return true, если ключ действителен
 */
public abstract boolean isValid();

/**
 * Отменяет регистрацию канала в селекторе.
 * Ключ становится недействительным.
 */
public abstract void cancel();



// ✅ Операции интересов (Interest Operations)
/**
 * Возвращает текущий набор операций, которые готовы к выполнению.
 * @return битовая маска готовых операций
 */
public abstract int readyOps();

/**
 * Возвращает текущий набор операций, за которыми следит селектор.
 * @return битовая маска операций интереса
 */
public abstract int interestOps();

/**
 * Устанавливает новый набор операций интереса.
 * @param ops битовая маска новых операций интереса
 * @return этот SelectionKey
 * @throws IllegalArgumentException если указаны недопустимые операции
 */
public abstract SelectionKey interestOps(int ops);

/**
 * Добавляет операции к текущему набору интересов.
 * @param ops битовая маска операций для добавления
 * @return этот SelectionKey
 */
public final SelectionKey interestOpsOr(int ops) {
	int current = interestOps();
	return interestOps(current | ops);
}

/**
 * Удаляет операции из текущего набора интересов.
 * @param ops битовая маска операций для удаления
 * @return этот SelectionKey
 */
public final SelectionKey interestOpsAnd(int ops) {
	int current = interestOps();
	return interestOps(current & ~ops);
}



// ✅ ❗❗ПРОВЕРКА ГОТОВНОСТИ❗❗ операций
/**
 * Проверяет, готов ли канал к операции чтения.
 * @return true, если операция READ готова
 */
public final boolean isReadable() {
	return (readyOps() & OP_READ) != 0;
}

/**
 * Проверяет, готов ли канал к операции записи.
 * @return true, если операция WRITE готова
 */
public final boolean isWritable() {
	return (readyOps() & OP_WRITE) != 0;
}

/**
 * Проверяет, готов ли канал к операции подключения.
 * @return true, если операция CONNECT готова
 */
public final boolean isConnectable() {
	return (readyOps() & OP_CONNECT) != 0;
}

/**
 * Проверяет, готов ли канал к операции принятия подключения.
 * @return true, если операция ACCEPT готова
 */
public final boolean isAcceptable() {
	return (readyOps() & OP_ACCEPT) != 0;
}



// ✅ ПРИКРЕПЛЕННЫЕ объекты (Attachment)
/**
 * ВОЗВРАЩАЕТ прикрепленный объект, или null если объект не установлен.
 * @return прикрепленный объект
 */
public final Object attachment() {
	return attachment;
}

/**
 * ❗❗❗ПРИКРЕПЛЯЕТ объект к этому ключу.
 * @param ob объект для прикрепления (может быть null)
 * @return ранее прикрепленный объект, или null если его не было
 */
public final Object attach(Object ob) {
	Object old = this.attachment;
	this.attachment = ob;
	return old;
}



// ✅ КОНСТАНТЫ операций
/**
 * Операция интереса для ЧТЕНИЯ.
 */
public static final int OP_READ = 1 << 0;

/**
 * Операция интереса для ЗАПИСИ.
 */
public static final int OP_WRITE = 1 << 2;

/**
 * Операция интереса для ПОДКЛЮЧЕНИЯ (только для клиентских сокетов).
 */
public static final int OP_CONNECT = 1 << 3;

/**
 * Операция интереса для ПРИНЯТИЯ ПОДКЛЮЧЕНИЙ (только для серверных сокетов).
 */
public static final int OP_ACCEPT = 1 << 4;



// ✅ Дополнительные методы (Java 11+)
/**
 * Возвращает событие, которое привело к готовности ключа.
 * @return событие готовности
 * @since 11
 */
public SelectorProvider provider() {
	return channel().provider();
}

/**
 * Ожидает указанное время, пока не произойдет событие интереса.
 * @param timeout время ожидания в миллисекундах
 * @return true, если событие произошло до таймаута
 * @throws IOException если произошла ошибка ввода-вывода
 * @since 16
 */
public boolean waitForReadyOps(long timeout) throws IOException {
	// Реализация зависит от конкретной платформы
	return selector().select(timeout) > 0 && isValid();
}

/**
 * Ожидает неопределенное время, пока не произойдет событие интереса.
 * @throws IOException если произошла ошибка ввода-вывода
 * @since 16
 */
public void waitForReadyOps() throws IOException {
	selector().select();
}
```

## 💡 Особенности и подводные камни 💡

### 1. Один ключ на одну регистрацию
Если ты регистрируешь один и тот же канал в нескольких селекторах —  
он получит **разные `SelectionKey`** (по одному на регистрацию).  
Но в пределах одного селектора — **один канал = один ключ**.

### 2. Отмена не мгновенная
Вызов `key.cancel()` только **помечает** ключ как недействительный,  
а физически удаляется он при следующем `select()`.

### 4. InterestOps и readyOps не одно и то же
- 🟠`interestOps` — _что тебе интересно_;
- 🟢`readyOps` — _что реально готово сейчас_.
    
❗❗Типичная ошибка новичков — пытаться вызвать `read()` без проверки `isReadable()`.

### 5. Потокобезопасность
- `interestOps()` можно вызывать из других потоков,  
    если при этом вызвать `selector.wakeup()` — чтобы “разбудить” селектор и применить изменения.
    
- Но нельзя параллельно менять `selectedKeys()` и `interestOps()` без wakeup().

---
## ✅ Best Practices ✅

✅ Храни состояние клиента прямо в `attachment()` (например, `Session`, `ByteBuffer`, DTO).  
✅ Перед `read()` и `write()` **всегда проверяй** `isReadable()` / `isWritable()`.  
✅ Не забывай вызывать `key.cancel()` при разрыве соединения.  
✅ Не меняй `interestOps()` из другого потока без `selector.wakeup()`.  
✅ Используй `interestOps(0)` для временной "заморозки" активности канала (throttling).  
✅ Не забывай, что `readyOps()` обновляется только после вызова `select()`.

## Пример использования
```java
import java.io.IOException;
import java.net.InetSocketAddress;
import java.nio.*;
import java.nio.channels.*;
import java.util.*;

public class SelectionKeyExample {
    public static void main(String[] args) throws IOException {
        Selector selector = Selector.open();
        
        ServerSocketChannel server = ServerSocketChannel.open();
        server.bind(new InetSocketAddress(8080));
        server.configureBlocking(false);
        
        server.register(selector, SelectionKey.OP_ACCEPT);

        while (true) {
            selector.select();
            Iterator<SelectionKey> iter = selector
	            .selectedKeys()
	            .iterator();

            while (iter.hasNext()) {
                SelectionKey key = iter.next();
                iter.remove();

                if (key.isAcceptable()) {
                    ServerSocketChannel srv = 
	                    (ServerSocketChannel) key.channel();
	                    
                    SocketChannel client = srv.accept();
                    client.configureBlocking(false);
                    
                    SelectionKey clientKey = client
	                    .register(selector, SelectionKey.OP_READ);
	                
	                // attach буфер
                    clientKey.attach(ByteBuffer.allocate(1024)); 
                }

                if (key.isReadable()) {
                    SocketChannel client = 
	                    (SocketChannel) key.channel();
	                    
                    ByteBuffer buf = (ByteBuffer) key.attachment();
                    int read = client.read(buf);
                    if (read == -1) {
                        key.cancel();
                        client.close();
                        continue;
                    }
                    buf.flip();
                    client.write(buf);
                    buf.clear();
                }
            }
        }
    }
}
```
