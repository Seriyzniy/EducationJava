Вместо того, чтобы читать в один буфер, а потом руками делить данные,  или писать из одного большого массива, собирая всё вручную, NIO позволяет **передать массив буферов**, и ОС сама распределит байты по ним (или соберёт обратно).

> [!NOTE] `Scatter/Gather I/O`
>  Механизм NIO, который позволяет **==работать сразу с НЕСКОЛЬКИМИ БУФЕРАМИ== в ОДНОЙ ОПЕРАЦИИ чтения/записи**.
>  - 🔸**Scatter read** — “разбрасывает” (scatter) входящие данные по нескольким буферам.
>  - 🔸**Gather write** — “собирает” (gather) данные из нескольких буферов и отправляет их одной операцией.

⭐⭐Работает только с каналами, реализующие интерфейсы⭐⭐
## Где это доступно
Эти методы определены в интерфейсах:
```JAVA
interface ScatteringByteChannel extends ReadableByteChannel {
    long read(ByteBuffer[] dsts) throws IOException;
    long read(
	    ByteBuffer[] dsts, 
	    int offset, 
	    int length) throws IOException;
}


interface GatheringByteChannel extends WritableByteChannel {
    long write(ByteBuffer[] srcs) throws IOException;
    long write(
	    ByteBuffer[] srcs, 
	    int offset, 
	    int length) throws IOException;
}
```
📌 И реализованы, например, в:
- `FileChannel`
- `SocketChannel`
- `DatagramChannel`

___
# 📕 1. Scatter Read💡
При чтении данных из канала — они “раскидываются” по массиву `ByteBuffer[]`.
```java
ByteBuffer header = ByteBuffer.allocate(128);
ByteBuffer body   = ByteBuffer.allocate(1024);

ByteBuffer[] buffers = { header, body };
long bytesRead = channel.read(buffers);
```
Что произойдёт:
- 🔸Сначала заполняется `header` (до `limit`).
    
- ✅Когда `header` заполнен, NIO начинает писать в `body`.
    
- Всё одной системной операцией чтения (`readv()` в Unix/Linux).

💡 Таким образом, можно логически разделить структуру данных (например, заголовок и тело) без копирования.

---
# 📕 2. Gather Write
При записи данные из нескольких буферов объединяются и пишутся одной операцией.
```java
ByteBuffer header = ByteBuffer.wrap("HEADER\n".getBytes());
ByteBuffer body   = ByteBuffer.wrap("DATA_CONTENT".getBytes());

ByteBuffer[] buffers = { header, body };
long bytesWritten = channel.write(buffers);
```
Что произойдёт:
- ❗❗❗ОС “соберёт” (gather) данные из `header` и `body`  
    и отправит их как один непрерывный поток байтов —  
    без промежуточных копий или конкатенаций в памяти.

---
## Почему это круто
### 1️⃣ Меньше системных вызовов
Каждая операция `read()` / `write()` = системный вызов.  
Scatter/Gather уменьшает их количество:  
один вызов → несколько буферов сразу.

### 2️⃣ Zero-Copy подход
Нет лишних копий между буферами, ты работаешь прямо с `ByteBuffer`,  
и ОС заполняет / считывает их напрямую (особенно с `DirectByteBuffer`).

### 3️⃣ ⭐Удобство при сложных форматах данных ⭐
Например, у тебя структура пакета:
```less
| HEADER (fixed size 128) | BODY (variable) |
```
С `Scatter` ты можешь считать обе части одним вызовом,  
НЕ ПАРСЯ вручную смещения.

### 4️⃣ Оптимизация для сетевых протоколов
- Заголовок (metadata) — в одном буфере,
    
- payload — в другом.  
    Можно отправлять обе части одним `write()` без слияния.

---
## ⚠️ Подводные камни

1️⃣ ❗❗❗**Порядок буферов имеет значение.**  
Чтение и запись происходят строго последовательно по массиву `ByteBuffer[]`.  
Ошибёшься в порядке — данные собьются.

2️⃣ ❗❗❗**Лимиты и позиции должны быть правильно установлены.**
- Перед записью — `position` указывает, откуда писать.
    
- Перед чтением — `position` указывает, куда писать (и `limit` ограничивает размер).
    

3️⃣ ==**Не все каналы поддерживают Scatter/Gather.**==  
Только те, которые реализуют `ScatteringByteChannel` / `GatheringByteChannel`.

4️⃣ **Direct-буферы эффективнее.**  
Так как scatter/gather используют системные вызовы `readv()`/`writev()`,  
`DirectByteBuffer` позволяет ОС работать напрямую без копий из heap.

---
## 💎 Best Practices 💎

✅ Используй **`ByteBuffer[]`** для структурированных данных (заголовок + тело, фрейм + метаданные).  
✅ Для максимальной производительности — **DirectByteBuffer**.  
✅ Проверяй `bytesRead` / `bytesWritten`, они могут быть меньше ожидаемого (partial I/O).  
✅ После `read()` не забудь `flip()` каждый буфер, перед `write()` — сбросить `position`.  
✅ Не пытайся использовать scatter/gather на блокирующих потоках ввода — смысла мало.  
✅ В сетевых протоколах (HTTP, FTP, RPC) — очень эффективен при формировании ответов.

## Пример 1 — Чтение файла с разделением на заголовок и тело
```java
import java.io.IOException;
import java.nio.*;
import java.nio.channels.*;
import java.nio.file.*;

public class ScatterReadExample {
    public static void main(String[] args) throws IOException {
        try (FileChannel channel = FileChannel
	        .open(
		        Path.of("data.bin"), 
		        StandardOpenOption.READ)
		) {
            ByteBuffer header = ByteBuffer.allocate(128);
            ByteBuffer body   = ByteBuffer.allocate(1024);

            ByteBuffer[] buffers = { header, body };
            long bytesRead = channel.read(buffers);

            header.flip();
            body.flip();

            System.out.println("Header bytes: " + header.remaining());
            System.out.println("Body bytes: " + body.remaining());
        }
    }
}
```
🧠 Здесь:
- `header` заполнится первыми 128 байтами,
- `body` — остальными,
    
- всё за один `read()` (через `readv()` в ядре ОС).

## Пример 2 — Отправка HTTP-ответа без конкатенации строк
```java
import java.net.*;
import java.nio.*;
import java.nio.channels.*;

public class GatherWriteExample {
    public static void main(String[] args) throws Exception {
        try (ServerSocketChannel server = ServerSocketChannel.open()) {
            server.bind(new InetSocketAddress(8080));
            server.configureBlocking(true);

            System.out.println("Server started...");
            try (SocketChannel client = server.accept()) {
                ByteBuffer header = ByteBuffer.wrap("HTTP/1.1 200 OK\r\nContent-Type: text/plain\r\n\r\n".getBytes());
                ByteBuffer body   = ByteBuffer.wrap("Hello from NIO Gather!\n".getBytes());

				// ✅✅ГЛАВНЫЙ ПРИКОЛ✅✅
                ByteBuffer[] buffers = { header, body };
                client.write(buffers); // ✅обе части одним вызовом
            }
        }
    }
}
```
