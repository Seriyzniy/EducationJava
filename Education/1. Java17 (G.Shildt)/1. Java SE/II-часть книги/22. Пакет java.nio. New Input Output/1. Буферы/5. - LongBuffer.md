
> [!NOTE] `LongBuffer` — это специализированный буфер для хранения **64-битных целых (`long`)**.
> - Наследует `Buffer`.
> - Реализует: `Comparable<LongBuffer>`.
> - Используется для эффективной ==работы с большими массивами чисел==.
> - Может быть создан напрямую (`allocate`, `wrap`) или как **view buffer** над `ByteBuffer` (`asLongBuffer()`).

## Конструкторы - нет конструкторов
У `LongBuffer` **нет публичных конструкторов**. Создание идёт через статические фабрики:
```java
// Создаёт новый буфер на куче (backed array long[])
public static LongBuffer allocate(int capacity)

// Оборачивает существующий массив long[]
public static LongBuffer wrap(long[] array)

// Оборачивает часть массива
public static LongBuffer wrap(long[] array, int offset, int length)
```

## Особенности и подводные камни

1. **View buffer**  
    Обычно `LongBuffer` создают через `ByteBuffer.asLongBuffer()`.  
    Здесь важен порядок байтов (`ByteOrder`). Если не выставить явно, будет **BIG_ENDIAN** по умолчанию.
```java
ByteBuffer bb = ByteBuffer.allocate(32).order(ByteOrder.LITTLE_ENDIAN);
LongBuffer lb = bb.asLongBuffer();
```

2. **compact()**  
    Очень полезен для стриминга данных: остаток переносится в начало, и буфер готов к новой дозаписи.
    
3. **slice()/duplicate()**  
    Разделяют данные с оригиналом, но имеют своё состояние (`position/limit`).  
    ⚠️ Изменения в содержимом будут видны всем view.
    
4. 💡💡**Размеры**  
    Каждый `long` занимает **8 байт**, поэтому при работе через `ByteBuffer` надо помнить, 
    - что capacity в байтах, 
    - а в `LongBuffer` — ==в long'ах.==  
    Пример:
```java
ByteBuffer bb = ByteBuffer.allocate(16);

System.out.println(bb.capacity()); // 16 байт

System.out.println(bb.asLongBuffer().capacity()); // 2 long
```

## ✅ Best Practices ✅ 

- ⚡ Для работы с большими бинарными файлами (индексы, offsets) — используй `MappedByteBuffer` → `asLongBuffer()`. Это быстрее, чем вручную читать по 8 байт.
    
- 🛠 При сериализации чисел в сеть или файл **явно выставляй ByteOrder**.
    
- 🧹 Для потоковой обработки (например, бинарный лог) — комбинируй `compact()` + `flip()`.

## Пример
```java
import java.nio.*;

public class LongBufferExample {
    public static void main(String[] args) {
        // --- Прямое создание ---
        LongBuffer lb = LongBuffer.allocate(4);

        lb.put(100L).put(200L).put(300L);
        lb.flip();

        while (lb.hasRemaining()) {
            System.out.println("Value: " + lb.get());
        }

        // --- View Buffer через ByteBuffer ---
        ByteBuffer bb = ByteBuffer
	        .allocate(32)
	        .order(ByteOrder.LITTLE_ENDIAN);
	        
        LongBuffer view = bb.asLongBuffer();

        view.put(11111111111L).put(22222222222L);
        view.flip();

        while (view.hasRemaining()) {
            System.out.println("From view: " + view.get());
        }
    }
}
```
