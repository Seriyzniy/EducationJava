
> [!NOTE] `ByteBuffer` — это конкретная `реализация буфера` для хранения байтов (`byte`).  
> Используется чаще всего: 
> - работа с файлами (`FileChannel`), 
> - сокетами (`SocketChannel`, `DatagramChannel`),
> - маппингом (`MappedByteBuffer`),
> - кодировкой (`CharsetEncoder/Decoder`).
> 
> ⭐**Главная роль:** обеспечивать ==эффективное ЧТЕНИЕ/ЗАПИСЬ данных В ВИДЕ МАССИВА БАЙТ== с управлением `position/limit/capacity`.


## Конструкторы - нет конструкторов
`ByteBuffer` **не имеет публичных конструкторов**.  
Создание идёт через фабричные методы класса:
```JAVA
// 💡Создаёт новый heap-буфер заданной ёмкости (данные в массиве byte[])
public static ByteBuffer allocate(int capacity) 
// capacity – максимальное количество байтов

// Создаёт новый direct-буфер (💡❗❗данные вне heap, в off-heap памяти❗❗💡)
public static ByteBuffer allocateDirect(int capacity) 
// Используется для I/O, быстрее при работе с каналами

// 💡Создаёт буфер-обёртку💡 над существующим массивом byte[]
public static ByteBuffer wrap(byte[] array) 
// Все изменения в буфере отражаются в массиве

// 💡Обёртка над ПОДДИАПАЗОНОМ массива💡
public static ByteBuffer wrap(byte[] array, int offset, int length) 
// offset – начальная позиция в массиве
// length – сколько байтов использовать
```

## 💡Особенности и подводные камни💡
1. **Heap vs Direct**
    - `allocate()` → обычный heap-буфер (GC управляет памятью, `hasArray() == true`).
        
    - `allocateDirect()` → direct-буфер (off-heap). Быстрее при I/O, но дороже при выделении и освобождении, нет `.array()`.
        
2. **wrap() — обёртка над массивом**  
    Изменения в массиве → видны в буфере и наоборот. Но смещение (`arrayOffset()`) может смещать базовый индекс.
    
3. **View Buffers**  
    `asIntBuffer()` и др. позволяют читать байты как примитивы.  
    ⚠️ Учитывай **byte order** (по умолчанию BIG_ENDIAN). Для корректности вызови `order(ByteOrder.nativeOrder())`.

## ✅Best Practices✅

- 💡❗Используй `allocateDirect()` для частого I/O (сетевые приложения, работа с каналами).
    
- Пулируй direct-буферы (дорого создавать часто).
    
- 🔄 Правило:
    - `put()` → `flip()` → `get()`
        
    - `get()` → `clear()` → `put()`
        
-  Проверяй `remaining()` перед чтением/записью массивов.
    
- ⚠️Для сериализации структур — аккуратно с `ByteOrder`. Устанавливай явно `buffer.order(ByteOrder.LITTLE_ENDIAN)` при работе с бинарными протоколами.
    
- Для больших файлов — подумай о `MappedByteBuffer` через `FileChannel.map()`.

## Пример
Чтение файла через `FileChannel` и `ByteBuffer`
```java
import java.io.IOException;
import java.nio.ByteBuffer;
import java.nio.channels.FileChannel;
import java.nio.file.Path;
import java.nio.file.StandardOpenOption;

public class ByteBufferExample {
    public static void main(String[] args) {
        Path path = Path.of("example.txt");

        try (FileChannel channel = FileChannel.open(
	        path, 
	        StandardOpenOption.READ)
	    ) {
		    // heap buffer на 64 байта
            ByteBuffer buffer = ByteBuffer.allocate(64); 

            while (channel.read(buffer) != -1) {
                buffer.flip(); // 💡переключаемся на чтение💡
                
                while (buffer.hasRemaining()) {
                    byte b = buffer.get();
                    System.out.print((char) b); // вывод как символ
                }
                buffer.clear(); // снова готов к записи
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```

Работа с примитивами
```java
ByteBuffer buf = ByteBuffer
	.allocate(16)
	.order(java.nio.ByteOrder.LITTLE_ENDIAN);
	
buf.putInt(42);
buf.putFloat(3.14f);

buf.flip();
System.out.println(buf.getInt());   // 42
System.out.println(buf.getFloat()); // 3.14

```
