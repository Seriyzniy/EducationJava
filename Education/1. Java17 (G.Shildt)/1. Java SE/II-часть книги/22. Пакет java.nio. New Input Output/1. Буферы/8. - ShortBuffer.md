
> [!NOTE] Буфер для хранения `short` (16-битные целые).
> Часто используется при работе с аудиоданными (PCM16), сетевыми бинарными протоколами и low-level форматами.

## Конструкторы - нет конструкторов
```java
public static ShortBuffer allocate(int capacity)
public static ShortBuffer wrap(short[] array)
public static ShortBuffer wrap(short[] array, int offset, int length)
```

## Методы
```java

public final int capacity()

public final ShortBuffer reset()

// сбрасывает позицию в начало (position=0)
public final ShortBuffer rewind()

// возвращает количество оставшихся элементов (limit - position)
public final int remaining()

// проверяет, остались ли элементы в буфере
public final boolean hasRemaining()

// проверяет, является ли буфер доступным только для чтения
public abstract boolean isReadOnly()

// проверяет, поддерживает ли буфер доступ к базовому массиву
public abstract boolean hasArray()

// возвращает базовый массив буфера типа short[]
public abstract short[] array()

// возвращает смещение в базовом массиве
public abstract int arrayOffset()

// проверяет, является ли буфер прямым (direct)
public abstract boolean isDirect()



// ✅ Чтение
// читает short с текущей позиции и увеличивает позицию

// читает short по указанному индексу (не изменяет позицию)
public abstract short get(int index)

// копирует оставшиеся элементы в указанный массив
public ShortBuffer get(short[] dst)

// копирует часть элементов в указанный массив с заданного смещения
public ShortBuffer get(short[] dst, int offset, int length)



// ✅ Запись
// записывает short в текущую позицию и увеличивает позицию
public abstract ShortBuffer put(short s)

// записывает short по указанному индексу (не изменяет позицию)
public abstract ShortBuffer put(int index, short s)

// записывает все элементы из указанного массива
public ShortBuffer put(short[] src)

// записывает часть элементов из массива с заданного смещения
public ShortBuffer put(short[] src, int offset, int length)

// копирует все оставшиеся элементы из другого ShortBuffer
public ShortBuffer put(ShortBuffer src)



// ✅ Манипуляции
// создает вид буфера от текущей позиции до лимита

// создает полную копию буфера с общими данными
public abstract ShortBuffer duplicate()

// создает буфер только для чтения с теми же данными
public abstract ShortBuffer asReadOnlyBuffer()

// уплотняет буфер, перемещая оставшиеся элементы в начало

// ✅ Сравнение и утилиты
// сравнивает два буфера лексикографически
public int compareTo(ShortBuffer that)

// проверяет равенство содержимого буферов
public boolean equals(Object ob)

// возвращает хэш-код на основе содержимого буфера
public int hashCode()

// возвращает строковое представление состояния буфера
public String toString()
```

## 💡 Особенности и подводные камни 💡
- Каждый short = **2 байта**.
    
- При работе через `ByteBuffer.asShortBuffer()` учитывай `ByteOrder`.
    
- Полезен для работы с аудиопотоками и бинарными протоколами.
    

## ✅ Best Practices ✅
- Для потоковых аудиоданных — лучше direct + native order.
    
- Используй `compact()` для сохранения непрочитанных сэмплов.
    
- Для межплатформенной работы — всегда указывай порядок байт.

## Пример
```java
ShortBuffer sb = ShortBuffer.allocate(4);
sb.put((short)100).put((short)200).flip();

while (sb.hasRemaining()) {
    System.out.println(sb.get());
}
```
