
> [!NOTE] Класс **`java.net.CookieManager`** — это **стандартная реализация** абстрактного класса `CookieHandler`
> Он обеспечивает **автоматическую `обработку HTTP cookies`**: 
> - принимает cookies из ответов сервера, 
> - сохраняет их в хранилище (`CookieStore`)  и 
> - добавляет обратно к последующим запросам — как браузер.
> 
> ⭐Реализует интерфейсы: `Serializable`

### Пару слов о назначении
- `CookieManager` — это **центральный компонент Java Cookie API**, реализующий «память» для HTTP-сессий.
    
- После установки как глобального обработчика (`CookieHandler.setDefault(manager)`)  
    он работает **прозрачно** для всех соединений `HttpURLConnection`.
    
- Управляет cookies согласно заданной **политике (`CookiePolicy`)**  
    и хранит их в указанном **хранилище (`CookieStore`)**.

## Конструкторы
```java
//✅ Создаёт CookieManager с политикой по умолчанию (ACCEPT_ORIGINAL_SERVER)
// и внутренним хранилищем InMemoryCookieStore.
public CookieManager()

//✅ Создаёт CookieManager с указанным хранилищем cookies и политикой.
// Параметры:
//  store  — экземпляр CookieStore (например, в памяти или на диске);
//  policy — политика обработки cookies (ACCEPT_ALL, ACCEPT_NONE и т.д.).
public CookieManager(CookieStore store, CookiePolicy policy)
```
💬 Если `store == null`, создаётся стандартное внутренняя реализация (в памяти).  
Если `policy == null`, используется политика `CookiePolicy.ACCEPT_ORIGINAL_SERVER`.

## Методы
### Получение и установка cookie
```java
//✅ Возвращает список заголовков, которые нужно добавить к запросу (Cookie: ...).
// Вызывается внутренне при отправке HTTP-запроса.
// Параметры:
//  uri — адрес ресурса;
//  requestHeaders — заголовки исходящего запроса.
// Возвращает Map<String, List<String>> с cookie-заголовками.
public Map<String, List<String>> get(
	URI uri, 
	Map<String, List<String>> requestHeaders) throws IOException

//✅ Обрабатывает полученные cookie из ответа сервера (Set-Cookie, Set-Cookie2).
// Параметры:
//  uri — адрес ресурса;
//  responseHeaders — заголовки ответа.
// Добавляет cookies в хранилище в зависимости от политики.
public void put(
	URI uri, 
	Map<String, List<String>> responseHeaders) throws IOException
```
💬 Методы `get()` и `put()` реализуют логику,  
которая позволяет `HttpURLConnection` автоматически работать с cookies:
- 🟢при отправке запроса → добавляются подходящие cookies;
- 🟠при получении ответа → сохраняются новые cookies в `CookieStore`.

### Управление хранилищем и политикой
```java
//✅ Возвращает текущее хранилище cookies (CookieStore).
public CookieStore getCookieStore()

//✅ Возвращает текущую политику обработки cookies.
public CookiePolicy getCookiePolicy()

//✅ Устанавливает новую политику обработки cookies.
public void setCookiePolicy(CookiePolicy policy)
```

## 💡 Особенности и подводные камни 💡
- 💡`CookieManager` — **готовая реализация**: не нужно писать свой обработчик cookie.
    
- ❗❗❗Все cookies сохраняются в `CookieStore` и автоматически подставляются в запросы.
    
- 🧠Политика определяет, какие `cookies` сохраняются (по домену, протоколу и т.п.).
    
- ⭐⭐Если не установить глобально через `CookieHandler.setDefault(...)`,  
    `HttpURLConnection` **НЕ БУДЕТ ИСПОЛЬЗОВАТЬ** этот менеджер.
    
- Встроенный `InMemoryCookieStore` НЕ СОХРАНЯЕТ cookies между запусками приложения.
    
- ⚠️ При большом количестве cookies возможен рост памяти — периодически очищай хранилище.
    
- 🧠 Потокобезопасность зависит от реализации `CookieStore`:  
    стандартная реализация синхронизирована и безопасна для многопоточности.

## 💎 Best Practices 💎

✅ **Устанавливай `CookieManager` один раз при старте приложения:**
```java
CookieManager manager = new CookieManager();
CookieHandler.setDefault(manager);
```

✅ **Принимай все cookies (например, для REST-клиентов):**
```java
CookieManager manager = new CookieManager(
	null, 
	CookiePolicy.ACCEPT_ALL);
	
CookieHandler.setDefault(manager);
```

✅ ❗❗❗**Храни cookies между сессиями** (реализуй собственный `CookieStore`, ==сохраняющий данные в файл или БД==).

✅ **Просматривай содержимое CookieStore:**
```java
CookieStore store = manager.getCookieStore();
for (HttpCookie cookie : store.getCookies()) {
    System.out.println(cookie);
}
```

✅ **Очищай хранилище при необходимости:**
```java
manager.getCookieStore().removeAll();
```

## Пример
```java
import java.net.*;
import java.io.*;
import java.util.*;

public class CookieManagerDemo {
    public static void main(String[] args) throws Exception {
        // 1️⃣ Создаём CookieManager с политикой принять все cookies
        CookieManager manager = new CookieManager(
	        null, 
	        CookiePolicy.ACCEPT_ALL);

        // 2️⃣ Устанавливаем как глобальный обработчик
        CookieHandler.setDefault(manager);


        // 3️⃣ Первый запрос — сервер установит cookies
        URL url = new URL("https://httpbin.org/cookies/set?session=java123");
        try (InputStream in = url.openStream()) {
            in.readAllBytes();
        }

        // 4️⃣ Второй запрос — cookies автоматически отправятся обратно
        URL url2 = new URL("https://httpbin.org/cookies");
        try (BufferedReader reader = new BufferedReader(
                new InputStreamReader(url2.openStream()))) {
            System.out.println("--- Ответ сервера ---");
            reader.lines().forEach(System.out::println);
        }

        // 5️⃣ Просмотр текущих cookies
        System.out.println("\n--- Сохранённые cookies ---");
        CookieStore store = manager.getCookieStore();
        for (HttpCookie cookie : store.getCookies())
            System.out.println(cookie);
    }
}

--- Ответ сервера ---
{
  "cookies": {
    "session": "java123"
  }
}

--- Сохранённые cookies ---
session=java123
```

При вводе ` https://httpbin.org/cookies/set?session=java123 ` в браузер, откроется сайт с json
