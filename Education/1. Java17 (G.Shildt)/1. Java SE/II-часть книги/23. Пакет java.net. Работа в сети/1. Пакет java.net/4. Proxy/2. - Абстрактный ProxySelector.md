
> [!NOTE] Класс **`java.net.ProxySelector`** определяет `механизм выбора` `прокси-сервера`
> для определённого URI (например, в зависимости от домена, протокола или IP-адреса).

## Пару слов о назначении
- `ProxySelector` используется для **централизованного управления** тем, какие прокси применяются при сетевых соединениях.
    
- Он решает, через какой прокси (или напрямую) нужно подключаться к конкретному URI.
    
- Позволяет:
    - задать **глобальный прокси-селектор** для всего приложения,
        
    - или использовать **кастомную реализацию**, которая выбирает прокси динамически (например, по домену).
        
- Все соединения, создаваемые через `HttpClient`, `URLConnection`, `Socket` и т.п., используют `ProxySelector` для определения прокси по умолчанию.

---
## Конструктор
Конструктор защищённый, используется ==только при наследовании==.

## Методы
### Статические методы (работа с глобальным селектором)
```java
//✅ Возвращает текущий ProxySelector по умолчанию.
// Если не установлен явно, возвращает реализацию по умолчанию,
// которая использует системные настройки прокси.
public static ProxySelector getDefault()

//✅ Устанавливает глобальный ProxySelector для всех подключений.
// Параметр:
//  ps — новый экземпляр ProxySelector (или null для отключения).
public static void setDefault(ProxySelector ps)
```
💬 ❗❗❗После вызова `setDefault(...)` все сетевые вызовы (URL, HttpClient и т.д.)  будут использовать ваш селектор для выбора прокси.❗❗❗

### Абстрактные методы (==реализуются подклассами==)
```java
//✅ Определяет список прокси для указанного URI.
// Параметр:
//  uri — адрес ресурса (например, https://example.com)
// Возвращает список прокси, в порядке приоритета.
// Если прямое соединение, возвращает List.of(Proxy.NO_PROXY).
public abstract List<Proxy> select(URI uri);

//✅ Сообщает о неудаче при подключении к прокси.
// Параметры:
//  uri — адрес, к которому пытались подключиться;
//  sa — адрес прокси;
//  ioe — исключение, вызвавшее сбой.
public abstract void connectFailed(
	URI uri, 
	SocketAddress sa, 
	IOException ioe);

```

### Вспомогательные реализации
Один и тот же прокси, для всех URI
```java
//✅ Создаёт ProxySelector, который всегда возвращает один и тот же прокси для всех URI.
// Параметр:
//  sa — адрес прокси (например, InetSocketAddress).
public static ProxySelector of(InetSocketAddress sa)
```

## 💡 Особенности и подводные камни 💡

1️⃣ ⭐⭐`ProxySelector` — **абстрактный класс**, его нельзя создать напрямую.  
==Чтобы задать собственную логику, нужно унаследовать== и ==переопределить== `select()` и `connectFailed()`.

2️⃣ ⭐Метод `select()` должен возвращать список `Proxy` объектов,  
даже если это один `Proxy.NO_PROXY`.

3️⃣ Если `ProxySelector` возвращает `null` или пустой список — соединение не будет выполнено.

4️⃣ ⭐При ошибках подключения (`connectFailed`) можно реализовать механизм "fallback" —  
например, при недоступности HTTP-прокси переходить на SOCKS.

5️⃣ 💡`ProxySelector.of(...)` удобен для простых случаев, когда нужен **фиксированный прокси**.

6️⃣ ⭐⭐Глобальный `ProxySelector` влияет на **все сетевые вызовы в JVM**,  
включая библиотеки и сторонние клиенты (например, JDBC, REST API).⭐⭐

7️⃣ Реализация по умолчанию использует системные настройки (из окружения или JVM-параметров).

8️⃣ ⭐⭐❗❗`HttpClient` всегда использует `ProxySelector`, даже если явно не задан.

9️⃣ Для совместимости с корпоративными сетями можно подменять `ProxySelector` в рантайме.

## 💎 Best Practices 💎

✅ **Для простого фиксированного прокси:**
```java
HttpClient client = HttpClient.newBuilder()
        .proxy(
	        ProxySelector.of(
		        new InetSocketAddress("proxy.example.com", 8080))
		)
        .build();
```

✅ ❗❗**Для глобального применения:**❗❗
```java
ProxySelector.setDefault(
	ProxySelector.of(new InetSocketAddress("corp-proxy", 3128)));
```

✅ **Для прямых подключений (без прокси):**
```java
HttpClient client = HttpClient.newBuilder()
        .proxy(ProxySelector.of(null))
        .build();
```

✅ ❗❗**Для кастомного выбора по домену:**❗❗
```java
ProxySelector customSelector = new ProxySelector() {
    @Override
    public List<Proxy> select(URI uri) {
        if (uri.getHost().endsWith(".internal.company.com")) {
            return List.of(Proxy.NO_PROXY);
        } else {
            return List.of(new Proxy(Proxy.Type.HTTP,
                    new InetSocketAddress("proxy.company.com", 8080)));
        }
    }

    @Override
    public void connectFailed(
	    URI uri, 
	    SocketAddress sa, 
	    IOException ioe) 
	{
        System.err.println("Ошибка подключения к прокси " + sa + " для " + uri + ": " + ioe);
    }
};


ProxySelector.setDefault(customSelector);
```

✅ **Для логирования неудачных соединений:**
```java
@Override
public void connectFailed(
	URI uri, 
	SocketAddress sa, 
	IOException ioe) 
{
    System.err.printf("Прокси %s недоступен для %s: %s%n", sa, uri, ioe.getMessage());
}
```

✅ **Для временного переопределения только в одном `HttpClient`:**
```java
HttpClient client = HttpClient.newBuilder()
        .proxy(customSelector)
        .build();
```

## Пример использования
```java
import java.net.*;
import java.io.*;
import java.util.*;

public class ProxySelectorDemo {
    public static void main(String[] args) throws Exception {
        // 1️⃣ Создаём собственный ProxySelector
        ProxySelector customSelector = new ProxySelector() {
            @Override
            public List<Proxy> select(URI uri) {
                if (uri.getHost().contains("google")) {
                    System.out.println("Используем прокси для " + uri);
                    return List.of(new Proxy(Proxy.Type.HTTP,
                            new InetSocketAddress(
	                            "proxy.example.com", 
	                            8080))
	                        );
                } else {
                    System.out.println("➡ Прямое соединение с " + uri);
                    return List.of(Proxy.NO_PROXY);
                }
            }

            @Override
            public void connectFailed(
	            URI uri, 
	            SocketAddress sa, 
	            IOException ioe) 
	        {
                System.err.println("❌ Ошибка подключения к прокси " + sa + ": " + ioe);
            }
        };


        // 2️⃣ Устанавливаем глобальный ProxySelector
        ProxySelector.setDefault(customSelector);


        // 3️⃣ Открываем соединение (через прокси или напрямую)
        URL url = new URL("https://www.google.com/");
        
        // ⭐⭐автоматически использует ProxySelector⭐⭐
        URLConnection conn = url.openConnection(); 


        try (BufferedReader reader = new BufferedReader(
                new InputStreamReader(conn.getInputStream()))) 
            {
            System.out.println("--- Ответ сервера ---");
            reader.lines().limit(5).forEach(System.out::println);
        }

        // 4️⃣ Проверяем, что глобальный селектор установлен
        System.out.println("\nТекущий ProxySelector: " + ProxySelector.getDefault().getClass().getName());
    }
}
```
