
> [!NOTE] Класс **`java.net.DatagramSocket`** используется для `ОТПРАВКИ И ПРИЁМА` `UDP-пакетов` (дейтаграмм).
> В отличие от `Socket` (TCP), он НЕ УСТАНАВЛИВАЕТ ПОСТОЯННОГО соединения — каждая передача — это **независимый пакет** данных.
> 
> ⭐Реализует интерфейс: `AutoCloseable`

Подходит для:
- передачи небольших сообщений без подтверждения доставки;
- задач, где важна скорость (игры, потоковое аудио/видео, датчики).

## Конструкторы
```java
//✅ Создаёт сокет и автоматически привязывает его к любому доступному локальному порту.
// Используется, если порт не важен (например, клиентская сторона).
DatagramSocket()
    throws SocketException

//✅ Создаёт сокет, привязанный к конкретному локальному порту.
// Параметр:
//  port — номер порта, к которому нужно привязать сокет.
// Если порт занят или недоступен → SocketException.
DatagramSocket(int port)
    throws SocketException

//✅ Создаёт сокет, привязанный к указанному адресу и порту.
// Параметры:
//  port — локальный порт;
//  ipAddress — локальный IP-адрес сетевого интерфейса, через который будет идти обмен.
DatagramSocket(int port, InetAddress ipAddress)
    throws SocketException

//✅ Создаёт сокет, привязанный к объекту SocketAddress (например, InetSocketAddress).
// Удобно, когда адрес формируется динамически.
DatagramSocket(SocketAddress address)
    throws SocketException
```
🔹 В отличие от `Socket`, здесь НЕТ УСТАНОВЛЕНИЯ СОЕДИНЕНИЯ.  
Создание `DatagramSocket` — это ==просто привязка к порту== (и опционально — адресу).
## Методы
### Отправка и приём пакетов
```java
//✅ Отправляет пакет данных (объект DatagramPacket).
// Пакет должен содержать адрес и порт получателя.
void send(DatagramPacket packet)
    throws IOException

//✅ Принимает пакет данных (блокирующий метод).
// Полученные данные записываются в буфер, переданный в DatagramPacket.
// Метод блокируется, пока не придёт дейтаграмма или не сработает тайм-аут.
void receive(DatagramPacket packet)
    throws IOException
```
💬 Обе операции используют объекты `DatagramPacket`, которые инкапсулируют ==массив байт== и ==сетевой адрес== получателя/отправителя.

### Информация о сокете
```java
//✅ Возвращает адрес удалённого узла, если сокет был «логически подключён» методом connect().
InetAddress getInetAddress()

//✅ Возвращает локальный порт, к которому привязан сокет.
int getLocalPort()

//✅ Возвращает удалённый порт, если сокет подключён через connect().
int getPort()
```

### Управление состоянием
```java
//✅ Проверяет, привязан ли сокет к адресу.
boolean isBound()

//✅ Проверяет, подключён ли сокет (через connect()).
boolean isConnected()

//✅ Закрывает сокет и освобождает системные ресурсы.
void close()
```

### Настройки и тайм-ауты
```java
//✅ Устанавливает тайм-аут ожидания приёма (в миллисекундах).
// Если за указанное время не пришла дейтаграмма → SocketTimeoutException.
void setSoTimeout(int millis)
    throws SocketException

//✅ Возвращает текущий тайм-аут ожидания.
int getSoTimeout()
    throws SocketException
```

## 💡 Особенности и подводные камни 💡

- UDP — **ненадёжный протокол**: пакеты могут теряться, приходить в другом порядке или дублироваться.
    
- ❗❗❗Метод `receive()` **блокирует поток**, пока не получен пакет или не сработал тайм-аут.
    
- ❗❗❗Размер пакета ограничен (~64 КБ, включая заголовки IP/UDP).
    
- 💡 Если не указать порт вручную, ОС выберет свободный автоматически.
    
- 🧠Для «логического подключения» можно вызвать `connect()`, чтобы зафиксировать адрес получателя и отбрасывать другие пакеты.
    
- ⚠️ Некоторые брандмауэры и антивирусы могут блокировать UDP-трафик.

---
## 💎 Best Practices 💎

✅ Используй **буферы достаточного размера**, чтобы вместить ожидаемые сообщения:
```java
byte[] buf = new byte[1024];
```

✅ Применяй **`setSoTimeout()`**, чтобы программа не блокировалась бесконечно при ожидании:
```java
socket.setSoTimeout(5000);
```

✅ Для многопользовательских приложений (игры, датчики) используй **многопоточность**:  
отдельный поток на приём, отдельный на отправку.

✅ Всегда **закрывай сокет** (`close()` или `try-with-resources`).

✅ При передаче строк — кодируй их в байты с явной кодировкой (`UTF-8`), чтобы избежать проблем с локалью.

## Пример
```java
import java.net.*;

public class DatagramExample {
    public static void main(String[] args) {
        int serverPort = 5000;
        int clientPort = 5001;
        byte[] buffer = new byte[1024];

        // 🔹 Сервер
        new Thread(() -> {
            try (DatagramSocket server = 
	            new DatagramSocket(serverPort)) 
	        {
                System.out.println("Сервер ждёт сообщения...");
                DatagramPacket packet = 
	                new DatagramPacket(buffer, buffer.length);
                
                server.receive(packet);
                
                String msg = new String(
	                packet.getData(), 
	                0, 
	                packet.getLength());
                
                System.out.println("Сервер получил: " + msg);
            } catch (Exception e) {
                e.printStackTrace();
            }
        }).start();

        // 🔹 Клиент
        try (DatagramSocket client = new DatagramSocket(clientPort)) {
            String message = "Привет, сервер!";
            byte[] data = message.getBytes();
            InetAddress address = InetAddress.getLocalHost();

            DatagramPacket packet =
                new DatagramPacket(
	                data, 
	                data.length, 
	                address, 
	                serverPort);

            client.send(packet);
            System.out.println("Клиент отправил сообщение.");
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```
