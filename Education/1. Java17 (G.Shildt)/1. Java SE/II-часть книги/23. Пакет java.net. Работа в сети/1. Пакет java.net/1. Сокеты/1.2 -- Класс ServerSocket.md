
> [!NOTE] Класс **`java.net.ServerSocket`** используется для создания `СЕРВЕРНЫХ TCP/IP-СОКЕТОВ,`
> Которые `ПРОСЛУШИВАЮТ` порт и `ПРИНИМАЮТ` входящие подключения от клиентов.
> 
> ⭐Реализует интерфейс: `AutoCloseable`

Он служит "точкой входа" для установления соединений и возвращает обычные объекты `Socket` при каждом подключении клиента.

`ServerSocket` — это серверная сторона TCP-соединения (==пара к классу== `Socket`).

## Конструкторы
```JAVA
//✅ Создаёт серверный сокет на указанном порту.
// Параметр:
//  port — номер порта, который будет прослушиваться (0–65535).
// Очередь подключений по умолчанию — 50.
// Если порт уже занят → IOException.
ServerSocket(int port)
    throws IOException

//✅ Создаёт серверный сокет с указанным портом и максимальной длиной очереди ожидания.
// Параметры:
//  port — номер порта;
//  backlog — максимальное число ожидающих клиентов (обычно 50).
ServerSocket(int port, int backlog)
    throws IOException

//✅ Создаёт серверный сокет на указанном порту, с очередью backlog,
// и привязывает его к конкретному сетевому адресу (IP).
// Параметры:
//  port — номер порта;
//  backlog — максимальное число подключений в очереди;
//  localAddress — IP-адрес сетевого интерфейса, на котором сервер будет слушать.
ServerSocket(int port, int backlog, InetAddress localAddress)
    throws IOException
```

> [!NOTE] Регистрация в ОС
> 🔹 При создании серверного сокета происходит **регистрация в системе** - ОС начинает прослушивать указанный порт, ожидая подключения клиентов.

## Методы
### 🔹 Подключение клиентов
```java
//✅ Ожидает подключения клиента.
// Это блокирующий метод — программа остановится здесь, пока кто-то не подключится.
// Возвращает объект Socket для связи с подключившимся клиентом.
Socket accept()
    throws IOException
```
💬 После вызова `accept()` ==сервер ПОЛУЧАЕТ **новый объект== `Socket`**, через который можно читать и писать данные клиенту.

### 🔹 Управление жизненным циклом
```java
//✅ Закрывает серверный сокет и освобождает занятый порт.
// После этого сервер больше не принимает подключения.
void close()
    throws IOException

//✅ Проверяет, закрыт ли сокет.
boolean isClosed()

//✅ Проверяет, связан ли сокет с каким-либо портом.
boolean isBound()
```

### 🔹 Настройки сокета
```java
//✅ Устанавливает тайм-аут ожидания для метода accept() (в миллисекундах).
// Если в течение указанного времени не подключится клиент, будет выброшено SocketTimeoutException.
void setSoTimeout(int timeout)
    throws SocketException

//✅ Возвращает текущий тайм-аут ожидания (в миллисекундах).
int getSoTimeout()
    throws IOException

//✅ Возвращает локальный порт, который слушает данный сервер.
int getLocalPort()

//✅ Возвращает локальный адрес (InetAddress), к которому привязан серверный сокет.
InetAddress getInetAddress()
```

---
## 💡 Особенности и подводные камни 💡

- ❗❗❗`accept()` — **блокирующий метод**. Сервер “зависнет”, пока клиент не подключится.  
    Чтобы сервер мог обслуживать несколько клиентов — нужно использовать **многопоточность**.
    (или см [[2.1.3 --- Абстрактный AsynchronousServerSocketChannel]])
    
- 💡 `ServerSocket` **НЕ обрабатывает данные напрямую** — для каждого клиента создаётся отдельный `Socket`.
    
- 💡Если порт уже занят другой программой — выбрасывается `BindException`.
    
- ⏱️ Тайм-аут через `setSoTimeout()` полезен для graceful shutdown (чтобы сервер не зависал).
    
- После вызова `close()` сервер больше не принимает соединения, даже если он снова войдёт в цикл `accept()`.
    
- 💥 Важно всегда освобождать порт, иначе при повторном запуске сервер не сможет стартовать.

---
## 💎 Best Practices 💎

✅ Используй **`try-with-resources`** для автоматического закрытия сокета:
```java
try (ServerSocket server = new ServerSocket(8080)) {
    // ...
}
```

✅ Для каждого клиента создавай **отдельный поток** или используй **пул потоков**:
```java
Socket client = server.accept();
new Thread(() -> handleClient(client)).start();
```

✅ 💡Проверяй, свободен ли порт перед запуском (или используй `port = 0` для автоподбора).

✅ Используй `setSoTimeout()` для ограничения времени ожидания клиентов.

✅ При тестировании используй **локальный адрес `InetAddress.getLocalHost()`**.

## Пример
```java
import java.io.*;
import java.net.*;

public class SimpleServer {
    public static void main(String[] args) {
        // try-with-resources автоматически закроет серверный сокет
        try (ServerSocket server = new ServerSocket(5000)) {
            System.out.println(
	            "Сервер запущен. Ожидание подключения...");

            // accept() — блокируется до подключения клиента
            Socket client = server.accept();
            System.out.println(
	            "Клиент подключён: " + client.getInetAddress());

            // Получаем потоки
            BufferedReader in = new BufferedReader(
	            new InputStreamReader(client.getInputStream()));
	            
            PrintWriter out = new PrintWriter(
	            client.getOutputStream(), true);

            // Обмен данными
            String line;
            while ((line = in.readLine()) != null) {
                System.out.println("Получено: " + line);
                out.println("Эхо: " + line);
            }

            client.close();
            System.out.println("Клиент отключился.");
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```
