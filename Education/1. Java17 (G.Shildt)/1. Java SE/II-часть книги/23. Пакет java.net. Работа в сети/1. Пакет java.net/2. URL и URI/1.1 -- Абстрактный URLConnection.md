
> [!NOTE] Класс **`java.net.URLConnection`** представляет `активное сетевое соединение`
> МЕЖДУ `приложением` и `ресурсом`, на который указывает объект `URL`
> 
> ⭐Реализует интерфейс: `AutoCloseable`

Он позволяет:
- устанавливать соединение с удалённым сервером,
    
- читать и (при поддержке протокола) отправлять данные,
    
- получать метаинформацию о ресурсе (заголовки, дату, тип контента, длину и др.),
    
- настраивать параметры соединения (тайм-ауты, кэширование, перенаправления и т.п.).

### Реализации URLConnection
`URLConnection` — это **абстрактный класс**, от которого наследуются конкретные реализации, например:
- `HttpURLConnection` — для HTTP/HTTPS;
    
- `JarURLConnection` — для работы с JAR-архивами.

## Фабричные методы получения экземпляра
```java
//✅ Возвращает объект URLConnection для данного URL.
// Соединение не открывается автоматически — это лишь объект, который можно настроить.
URLConnection openConnection()
    throws IOException

//✅ То же самое, но с использованием заданного прокси-сервера.
URLConnection openConnection(Proxy proxy)
    throws IOException
```

❗❗❗Объект `URLConnection` создаётся вызовом этих методов у `URL`,  
например:
```java
URL url = new URL("https://example.com");
URLConnection conn = url.openConnection();
```


## Методы
### Управление соединением
```java
//✅ Открывает физическое соединение с ресурсом.
// Обычно вызывается неявно при первом доступе к потокам ввода/вывода.
void connect()
    throws IOException

//✅ Проверяет, установлено ли соединение (неявно через внутренние поля).
boolean getDoInput()

//✅ Устанавливает флаг, разрешающий чтение данных из соединения.
// По умолчанию true.
void setDoInput(boolean doInput)

//✅ Устанавливает флаг, разрешающий отправку данных (POST-запросы и т.п.).
void setDoOutput(boolean doOutput)

//✅ Возвращает true, если соединение разрешает отправку данных.
boolean getDoOutput()
```

### Потоки ввода/вывода
```java
//✅ Возвращает поток для чтения данных с ресурса (входящий поток).
// Обычно используется для загрузки содержимого страницы, файла и т.д.
InputStream getInputStream()
    throws IOException

//✅ Возвращает поток для записи данных на сервер (если поддерживается протоколом, например HTTP POST).
OutputStream getOutputStream()
    throws IOException
```
💬 Эти методы ==**открывают соединение автоматически**==, если оно ещё не было установлено.

### Метаданные ресурса
```java
//✅ Возвращает дату последней модификации ресурса (в миллисекундах с 1970 г.).
long getLastModified()

//✅ Возвращает дату публикации (если указана сервером).
long getDate()

//✅ Возвращает дату истечения срока годности ресурса (для кэширования).
long getExpiration()

//✅ Возвращает MIME-тип содержимого (например, "text/html" или "image/png").
String getContentType()

//✅ Возвращает длину содержимого в байтах, если сервер передаёт этот заголовок.
// Если неизвестно — возвращает -1.
int getContentLength()

//✅ Возвращает длину содержимого как long (для больших файлов, >2 ГБ).
long getContentLengthLong()
```

### Работа с заголовками
```java
//✅ Возвращает значение заголовка по имени.
// Например, "Content-Type" или "Server".
String getHeaderField(String name)

//✅ Возвращает карту всех заголовков (Map<String, List<String>>).
Map<String, List<String>> getHeaderFields()

//✅ Возвращает имя заголовка по индексу (для итерации).
String getHeaderFieldKey(int n)

//✅ Возвращает значение заголовка по индексу.
String getHeaderField(int n)
```

### Настройки соединения
```java
//✅ Устанавливает тайм-аут ожидания подключения (в миллисекундах).
void setConnectTimeout(int timeout)

//✅ Возвращает текущий тайм-аут подключения.
int getConnectTimeout()

//✅ Устанавливает тайм-аут ожидания чтения данных из потока.
void setReadTimeout(int timeout)

//✅ Возвращает текущий тайм-аут чтения.
int getReadTimeout()

//✅ Устанавливает пользовательский агент (или другие свойства HTTP).
void setRequestProperty(String key, String value)

//✅ Возвращает значение свойства запроса.
String getRequestProperty(String key)
```

## 💡 Особенности и подводные камни 💡

- `URLConnection` — **абстрактный**: напрямую экземпляры обычно не создаются, 
    используется `URL.openConnection()`.
    
- 💡Фактическое соединение устанавливается **только при обращении к потокам** или вызове `connect()`.
    
- ⏱️ Если не задать тайм-ауты, программа может **зависнуть навсегда** при плохом соединении.
    
- 💡 Поведение зависит от протокола (HTTP, FTP, file и др.) — часть методов может быть неактуальна.
    
- 🧠 Методы `getHeaderField()` и `getContentType()` полезны для анализа ответа перед чтением данных.
    
- 🧠При HTTPS соединении автоматически используется встроенный SSL.
    
- ❗❗❗`setDoOutput(true)` необходимо вызывать **до connect()**, иначе будет `IllegalStateException`. 

---

## 💎 Best Practices 💎

✅ **Устанавливай тайм-ауты**:
```java
conn.setConnectTimeout(5000);
conn.setReadTimeout(5000);
```

✅ **Используй try-with-resources** при работе с потоками ввода/вывода.  

✅ **Вызывай `connect()` явно**, если хочешь проверить успешность соединения до начала чтения.  

✅ ❗❗❗**Проверяй заголовки** перед чтением (например, `Content-Type` и `Content-Length`).  

✅ ❗❗❗Для HTTP-протокола используй **`HttpURLConnection`**, чтобы управлять кодами ответа и методами запроса (GET, POST и т.д.).  

✅ Не забудь закрыть потоки, иначе соединение зависнет и не освободит ресурсы.

## Пример
```java
import java.io.*;
import java.net.*;
import java.util.*;

public class URLConnectionDemo {
    public static void main(String[] args) {
        try {
            URL url = new URL("https://www.example.com");
            URLConnection conn = url.openConnection();

            // Настройка соединения
            conn.setConnectTimeout(3000);
            conn.setReadTimeout(3000);

            // Получаем метаданные
            System.out.println("Дата: " + new Date(conn.getDate()));
            System.out.println("Тип контента: " + conn.getContentType());
            System.out.println("Длина контента: " + conn.getContentLengthLong());

            // Заголовки
            System.out.println("\n--- Заголовки ---");
            conn.getHeaderFields().forEach((k, v) -> System.out.println(k + ": " + v));

            // Чтение содержимого (первые несколько строк)
            try (BufferedReader reader =
                     new BufferedReader(
	                     new InputStreamReader(conn.getInputStream()))\
	        ) {
                System.out.println("\n--- Содержимое ---");
                for (int i = 0; i < 5; i++) {
                    String line = reader.readLine();
                    if (line == null) break;
                    System.out.println(line);
                }
            }

        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```
