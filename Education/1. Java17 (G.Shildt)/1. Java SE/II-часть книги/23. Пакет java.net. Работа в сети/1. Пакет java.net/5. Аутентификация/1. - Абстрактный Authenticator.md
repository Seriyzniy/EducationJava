# 📕 1. Абстрактный Authenticator

> [!NOTE] Класс **`java.net.Authenticator`** предоставляет `механизм` централизованной `аутентификации`
> для сетевых соединений, требующих логин и пароль - например, при доступе к HTTP/HTTPS-ресурсам или при использовании прокси-сервера

## Пару слов о назначении
- `Authenticator` — это ==базовый класс==, который ==можно расширить==,  
    чтобы автоматически предоставлять имя пользователя и пароль при запросе аутентификации.
    
- Он используется:
    - при соединении через **HTTP или HTTPS** (401 Unauthorized),
        
    - при соединении через **Proxy** (407 Proxy Authentication Required),
        
    - при доступе к ресурсам FTP.
        
- Система вызывает `Authenticator` автоматически,  
    когда сервер или прокси требует авторизацию.

---
## Конструктор Authenticator
Защищённый конструктор, используется только при наследовании.

### 📙 Вложенный enum RequestorType
```java
public static enum Authenticator.RequestorType {
    PROXY,   // Авторизация прокси-сервера (HTTP 407)
    SERVER   // Авторизация веб-сервера (HTTP 401)
}
```

## Методы
### Статические методы
```java
//✅ Устанавливает глобальный Authenticator для всех сетевых вызовов.
// Параметр:
//  a — экземпляр Authenticator (или null для отключения).
public static void setDefault(Authenticator a)

//✅ Возвращает текущий глобальный Authenticator.
public static Authenticator getDefault()
```

💬 ❗❗Обычно `Authenticator` устанавливается один раз в приложении:
```java
Authenticator.setDefault(new MyAuthenticator());
```

### ⭐Основной защищённый метод⭐
```java
//✅ Запрашивает логин и пароль при попытке аутентификации.
// Этот метод вызывается системой автоматически, когда сервер или прокси
// требует авторизацию (например, код ответа HTTP 401 или 407).
// Возвращает объект PasswordAuthentication или null, если аутентификация невозможна.
protected PasswordAuthentication getPasswordAuthentication()
```

### Методы для получения контекста (вызываются внутри getPasswordAuthentication)
Эти методы позволяют понять, **кто запрашивает авторизацию**,  
чтобы принять решение о предоставлении пароля.

```java
//✅ Возвращает тип запроса (RequestorType.SERVER или RequestorType.PROXY).
protected final Authenticator.RequestorType getRequestorType()

//✅ Возвращает адрес хоста, запрашивающего авторизацию.
protected final InetAddress getRequestingSite()

//✅ Возвращает порт удалённого сервера или прокси.
protected final int getRequestingPort()

//✅ Возвращает имя хоста, запрашивающего авторизацию.
protected final String getRequestingHost()

//✅ Возвращает протокол (http, https, ftp).
protected final String getRequestingProtocol()

//✅ Возвращает prompt-сообщение от сервера (обычно realm).
protected final String getRequestingPrompt()

//✅ Возвращает схему аутентификации (например, "Basic", "Digest", "NTLM").
protected final String getRequestingScheme()

//✅ Возвращает URL запрашиваемого ресурса (если известен).
protected final URL getRequestingURL()
```

## 💡 Особенности и подводные камни 💡

1️⃣ `Authenticator` **не вызывается напрямую** — он ==срабатывает автоматически==,  
когда `URLConnection`, `HttpClient`, `Socket` или другие сетевые классы  
сталкиваются с запросом аутентификации.

2️⃣ 💡Метод `getPasswordAuthentication()` выполняется **в том же потоке**,  
что и запрос, поэтому он должен быть **быстрым** (без долгих блокировок).

3️⃣ ⭐Для HTTPS и HTTP различаются `RequestorType`:
- `SERVER` → ответ 401 (Unauthorized)
- `PROXY` → ответ 407 (Proxy Authentication Required)
    

4️⃣ ⭐Объект `PasswordAuthentication` содержит логин и пароль в виде массива `char[]`,  
чтобы их можно было безопасно очистить из памяти.

5️⃣ Если `Authenticator` возвращает `null`,  
соединение будет разорвано с ошибкой авторизации.

6️⃣🧠Можно реализовать `Authenticator`, который выдает логин/пароль **по хосту, схеме или порту**.

7️⃣ ❗❗❗Глобальный `Authenticator` применяется ко всем соединениям в JVM.  
Если нужно изолировать — можно реализовать ручную аутентификацию через `HttpRequest.Builder.header("Authorization", ...)`.

8️⃣ При работе с `HttpClient` (Java 11+) `Authenticator` используется через `Authenticator` из `java.net.Authenticator`,  
но можно задать и через `Authenticator`-объект в билдере:
```java
HttpClient.newBuilder().authenticator(Authenticator.getDefault());
```

9️⃣ ❗❗❗Не путай `Authenticator` (из `java.net`) и `javax.mail.Authenticator` — это разные классы.❗❗❗

## 💎 Best Practices 💎

✅ **Создавай собственный подкласс Authenticator** и переопредели `getPasswordAuthentication()`:
```java
Authenticator.setDefault(new Authenticator() {
    @Override
    protected PasswordAuthentication getPasswordAuthentication() {
        return new PasswordAuthentication(
	        "user", 
	        "password".toCharArray());
    }
});
```

✅ **Используй контекстные методы**, чтобы фильтровать запросы:
```java
if (getRequestorType() == RequestorType.PROXY) ...
if ("https".equalsIgnoreCase(getRequestingProtocol())) ...
```

✅ 💡**Для безопасного хранения паролей** используй шифрованные файлы,  
а не жёстко закодированные строки.

✅ **Для одного HttpClient можно задать Authenticator локально:**
```java
HttpClient client = HttpClient.newBuilder()
        .authenticator(new Authenticator() {
            @Override
            protected PasswordAuthentication getPasswordAuthentication() {
                return new PasswordAuthentication(
	                "apiuser", 
	                "apipass".toCharArray());
            }
        })
        .build();
```

✅ **Для разных доменов — разные учётные данные:**
```java
@Override
protected PasswordAuthentication getPasswordAuthentication() {
    if (getRequestingHost().contains("internal.company.com")) {
        return new PasswordAuthentication("intranet", "secret".toCharArray());
    }
    return null;
}
```

✅ **Для работы с Proxy:**
```java
if (getRequestorType() == RequestorType.PROXY) {
    return new PasswordAuthentication(
	    "proxyUser", 
	    "proxyPass".toCharArray());
}
```

## Пример
```java
import java.net.*;
import java.io.*;

public class AuthenticatorDemo {
    public static void main(String[] args) throws Exception {
    
        // 1️⃣ ⭐⭐Устанавливаем глобальный аутентификатор
        Authenticator.setDefault(new Authenticator() {
            @Override
            protected PasswordAuthentication getPasswordAuthentication() 
            {
                System.out.println(
	                "Запрошена аутентификация для: " +
	                 getRequestingHost());
	                 
                if (getRequestorType() == RequestorType.PROXY) {
                    return new PasswordAuthentication(
	                    "proxyUser", 
	                    "proxyPass".toCharArray());
                } else {
                    return new PasswordAuthentication(
	                    "user", 
	                    "password".toCharArray());
                }
            }
        });

        // 2️⃣ Настраиваем прокси
        Proxy proxy = new Proxy(Proxy.Type.HTTP, new InetSocketAddress("proxy.example.com", 8080));

        // 3️⃣ Делаем запрос через прокси (требует авторизацию)
        URL url = new URL("https://httpbin.org/basic-auth/user/password");
        URLConnection conn = url.openConnection(proxy);

        // 4️⃣ Читаем ответ
        try (BufferedReader reader = new BufferedReader(
                new InputStreamReader(conn.getInputStream()))) {
            System.out.println("--- Ответ сервера ---");
            reader.lines().forEach(System.out::println);
        }
    }
}
```
