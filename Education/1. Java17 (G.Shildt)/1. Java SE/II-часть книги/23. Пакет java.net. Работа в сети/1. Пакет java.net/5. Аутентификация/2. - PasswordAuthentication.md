
> [!NOTE] Класс `java.net.PasswordAuthentication`- `неизменяемый контейнер`  для пары `логин–пароль`
> имя и пароль, необходимые для прохождения аутентификации при сетевом подключении.

## Пару слов о назначении
- 💡Используется `Authenticator` для **возврата логина и пароля** в ответ на запрос сервера.
    
- Объект `PasswordAuthentication` является **транспортной обёрткой**,  
    которую Java передаёт из `Authenticator` обратно в сетевой стек.
    
- ❗❗❗Данные пароля хранятся не в `String`, а в `char[]` — это повышает безопасность,  
    ==так как массив== ==можно обнулить== после использования.

---
## Конструктор
```java
//✅ Создаёт объект PasswordAuthentication с указанным логином и паролем.
// Параметры:
//  userName — имя пользователя (строка, не может быть null);
//  password — пароль в виде массива символов (char[]), может быть null.
//  Массив пароля копируется внутрь объекта для безопасности.
public PasswordAuthentication(
	String userName, 
	char[] password)
```

💬 Этот объект затем возвращается из `Authenticator.getPasswordAuthentication()`  
и используется сетевыми классами (`HttpURLConnection`, `HttpClient`, `Socket`, `Proxy`)  
==для формирования ЗАГОЛОВКОВ== `Authorization` или `Proxy-Authorization`.

## 💡 Особенности и подводные камни 💡

1️⃣ **Безопасность прежде всего:**  
пароль хранится в `char[]`, чтобы можно было обнулить содержимое вручную (`Arrays.fill(...)`).  
Это предотвращает попадание пароля в пул строк.

2️⃣ **Мутируемость:**  
Массив `char[]` возвращается напрямую — если его изменить, изменится и внутреннее состояние.  
Поэтому правильнее делать копию при чтении или записи.

3️⃣ **Использование только в рамках `Authenticator`:**  
Обычно `PasswordAuthentication` создаётся и возвращается исключительно внутри  
`getPasswordAuthentication()` и нигде больше.

4️⃣ **Нет встроенного шифрования:**  
Пароль хранится в памяти как обычные символы.  
Шифрование или хеширование нужно реализовывать отдельно, если требуется.

5️⃣ **Thread-safety:**  
Класс не потокобезопасен, но обычно используется только в текущем потоке авторизации.

6️⃣ **Поддержка всех схем:**  
Используется как для `Basic`, `Digest`, `NTLM`, `Kerberos`, так и для `Proxy Authentication`.

---
## 💎 Best Practices 💎

✅ Создавай объект **только внутри `Authenticator`**:
```java
@Override
protected PasswordAuthentication getPasswordAuthentication() {
    return new PasswordAuthentication("user", "secret".toCharArray());
}
```

✅ **Не сохраняй** объект `PasswordAuthentication` в полях класса  
и не логируй его содержимое.

✅ Если извлекаешь пароль:
```java
char[] pass = pa.getPassword();
try {
    // использовать пароль
} finally {
    Arrays.fill(pass, '\0'); // очистить из памяти
}
```

✅ **Не передавай `String password`** —  
строка остаётся в пуле и не может быть очищена.

✅ Для чувствительных операций (`DB`, `HTTP Basic`, `FTP`)  
всегда используй `char[]`.

## Пример
```java
import java.net.*;
import java.io.*;
import java.util.Arrays;

public class PasswordAuthDemo {
    public static void main(String[] args) throws Exception {
        // 1️⃣ Устанавливаем глобальный Authenticator
        Authenticator.setDefault(new Authenticator() {
            @Override
            protected PasswordAuthentication getPasswordAuthentication() {
                System.out.println("
	                Аутентификация для: " + getRequestingHost());
	                
                return new PasswordAuthentication(
	                "admin", 
	                "12345".toCharArray());
            }
        });

        // 2️⃣ Делаем запрос к защищённому ресурсу
        URL url = new URL("https://httpbin.org/basic-auth/admin/12345");
        HttpURLConnection conn = 
	        (HttpURLConnection) url.openConnection();


        // 3️⃣ Читаем ответ
        System.out.println("HTTP статус: " + conn.getResponseCode());
        try (BufferedReader reader = new BufferedReader(
                new InputStreamReader(conn.getInputStream()))) {
            reader.lines().forEach(System.out::println);
        }

        // 4️⃣ Работаем с PasswordAuthentication вручную
        PasswordAuthentication pa =
                new PasswordAuthentication("user1", "password1".toCharArray());

        System.out.println("\nИмя пользователя: " + pa.getUserName());
        char[] pwd = pa.getPassword();
        System.out.println("Пароль (длина): " + pwd.length);

        // 5️⃣ Очищаем пароль
        Arrays.fill(pwd, '\0');
        System.out.println("Пароль очищен из памяти.");
    }
}
```
