
> [!NOTE] `ContainerEvent` — это `событие` `добавления или удаления компонента` `из контейнера`.
> Контейнером считается любой элемент, способный содержать другие компоненты — например, `Panel`, `Frame`, `Window`, `Applet` и т.д.
> 
> ⭐⭐Иными словами: 
> Когда ты вызываешь `add()` или `remove()` для компонента, **генерируется объект `ContainerEvent`**, который сообщает слушателям о структурном изменении интерфейса.

## Иерархия
```java
java.lang.Object
   └── java.util.EventObject
        └── java.awt.AWTEvent
             └── java.awt.event.ComponentEvent
                  └── java.awt.event.ContainerEvent

```
`ContainerEvent` — **потомок `ComponentEvent`**,  
поэтому наследует его свойства (например, `getComponent()`) и добавляет новые поля, связанные с контейнером.

### 🧭 Назначение
`ContainerEvent` сообщает, что **структура контейнера изменилась**
- в него добавили новый компонент (`COMPONENT_ADDED`),
- или из него удалили компонент (`COMPONENT_REMOVED`).
    

Такие события получают объекты, реализующие интерфейс:
```java
public interface ContainerListener {
    void componentAdded(ContainerEvent e);
    void componentRemoved(ContainerEvent e);
}
```

---
## Конструкторы
```java
// Создаёт событие контейнера (например, при добавлении или удалении компонента)
// 💡src — источник события (💡объект типа Container, например Panel💡 или Frame)
// id — идентификатор события (COMPONENT_ADDED или COMPONENT_REMOVED)
// 💡child — компонент, который был добавлен или удалён
public ContainerEvent(
	Component src, 
	int id, 
	Component child)
```

### Константы класса
|Константа|Значение|Описание|
|---|---|---|
|`COMPONENT_ADDED`|300|Компонент добавлен в контейнер|
|`COMPONENT_REMOVED`|301|Компонент удалён из контейнера|

---

## Методы
Класс наследует все методы `ComponentEvent` (`getComponent()`, `paramString()`),  
но добавляет **два собственных метода** для доступа к контейнеру и дочернему компоненту.

### Работа с контейнерами и вложенными элементами
```java
// Возвращает контейнер, в котором произошло событие
// Источник события (например, Panel или Frame)
public Container getContainer()

// Возвращает компонент, который был добавлен или удалён
// Например, кнопку Button, которую вставили в Panel
public Component getChild()
```

### Отладочная информация
```java
// Возвращает строковое описание события, включая тип (added/removed)
// и сведения о дочернем компоненте
public String paramString()
```

## 💡 Особенности и подводные камни 💡

1️⃣ **Наследует все свойства `ComponentEvent`.**  
То есть `ContainerEvent` — частный случай событий компонента,  
но с дополнительной информацией о дочернем элементе.

2️⃣ 💡**Генерируется автоматически при вызове `add()` или `remove()`.**  
Ручное создание таких событий обычно не требуется.

3️⃣ 💡**Срабатывает только при фактическом изменении контейнера.**  
Если добавить компонент в тот же контейнер, где он уже есть, событие не будет создано.

4️⃣ 💡**Порядок событий:**  
При добавлении компонента сначала вызывается `addNotify()`, затем `ContainerEvent`.

5️⃣ 💡**Не все контейнеры порождают события.**  
Некоторые нативные или нестандартные контейнеры могут не поддерживать `ContainerListener`.

---

## 💎 Best Practices 💎

✅ Используй `ContainerListener`, если тебе нужно:
- отслеживать динамическое создание/удаление элементов интерфейса;
- вести лог изменений контейнера;
- отлаживать layout-менеджеры или собственные контейнерные компоненты.
    

✅ 💡Не выполняй тяжёлые операции (перерисовка, пересчёт) прямо в `componentAdded()` —  
💡лучше ставить их в очередь на выполнение (через `EventQueue.invokeLater()`).

✅ Для проверки, какой именно элемент добавлен или удалён, используй `getChild()` —  
это надёжнее, чем анализировать весь контейнер заново.

✅ При создании **собственных контейнеров** (наследников `Container`) можно переопределять методы `addImpl()` и `remove()` для контроля событий.

✅ Помни, что `ContainerEvent` **не информирует о видимости** — для этого существуют `ComponentEvent` и `HierarchyEvent`.

## Пример
```java
import java.awt.*;
import java.awt.event.*;

public class ContainerEventDemo extends Frame implements ContainerListener {
    private TextArea log;
    private Panel panel;

    public ContainerEventDemo() {
        setLayout(new BorderLayout());

        panel = new Panel();
        panel.setLayout(new FlowLayout());
        panel.addContainerListener(this); // регистрируем слушателя

        Button addBtn = new Button("Добавить");
        Button removeBtn = new Button("Удалить");
        log = new TextArea(8, 40);

        addBtn.addActionListener(e -> {
            Button newButton = new Button("Новая кнопка");
            panel.add(newButton);
            panel.validate();
        });

        removeBtn.addActionListener(e -> {
            if (panel.getComponentCount() > 0) {
                panel.remove(0);
                panel.validate();
                panel.repaint();
            }
        });

        add(panel, BorderLayout.CENTER);
        add(addBtn, BorderLayout.NORTH);
        add(removeBtn, BorderLayout.SOUTH);
        add(log, BorderLayout.EAST);

        setSize(500, 250);
        setVisible(true);

        addWindowListener(new WindowAdapter() {
            public void windowClosing(WindowEvent e) {
                System.exit(0);
            }
        });
    }

    @Override
    public void componentAdded(ContainerEvent e) {
        log.append("Добавлен компонент: " + e.getChild().getClass().getSimpleName() + "\n");
    }

    @Override
    public void componentRemoved(ContainerEvent e) {
        log.append("Удалён компонент: " + e.getChild().getClass().getSimpleName() + "\n");
    }

    public static void main(String[] args) {
        new ContainerEventDemo();
    }
}

```

🧠 **Как работает:**
- При нажатии кнопки “Добавить” создаётся новая кнопка и вставляется в панель.  
    Это вызывает событие `ContainerEvent.COMPONENT_ADDED`.
    
- При нажатии “Удалить” — удаляется первый компонент панели и вызывается `COMPONENT_REMOVED`.
    
- Методы `componentAdded()` и `componentRemoved()` фиксируют эти изменения в текстовом поле.
