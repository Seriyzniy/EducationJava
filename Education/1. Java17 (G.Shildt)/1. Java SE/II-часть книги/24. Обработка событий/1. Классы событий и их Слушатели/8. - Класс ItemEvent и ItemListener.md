
> [!NOTE] `ItemEvent` — это `событие` `выбора или отмены выбора элемента` `в компоненте`, который `поддерживает выбор` (item selection).
> Оно генерируется, когда пользователь **выбирает или снимает выбор** с элемента, например: 
> - отмечает или снимает галочку в `Checkbox`,
> - выбирает пункт в `Choice`,
> - изменяет состояние `CheckboxMenuItem`, 
> - меняет выбранный элемент в `List`.
> 

## Иерархия
```java
java.lang.Object
   └── java.util.EventObject
        └── java.awt.AWTEvent
             └── java.awt.event.ItemEvent
```

🧠 В отличие от `ActionEvent`, который сигнализирует о действии (например, нажатие кнопки),  
`ItemEvent` сигнализирует ==об **изменении состояния выбора элемента**.==

### 🧭 Назначение
`ItemEvent` используется для ==**реакции на изменение состояния элемента управления**==,  
например, чтобы обновить интерфейс при выборе опции пользователем.

Для обработки таких событий используется интерфейс:
```java
public interface ItemListener {
    void itemStateChanged(ItemEvent e);
}
```
> `ItemEvent` — это источник информации для `itemStateChanged()`.

## Конструкторы
```java
// Создаёт событие изменения состояния элемента выбора
// src — источник события (компонент, например Checkbox или Choice)
// id — идентификатор события (обычно ItemEvent.ITEM_STATE_CHANGED)
// item — объект, который был выбран или отменён (например, текст пункта)
// stateChange — состояние: SELECTED или DESELECTED
public ItemEvent(
	ItemSelectable src, 
	int id, 
	Object item, 
	int stateChange)
```

### Константы класса
| Константа            | Значение | Описание                                                       |
| -------------------- | -------- | -------------------------------------------------------------- |
| `ITEM_FIRST`         | 701      | Первый идентификатор для событий выбора                        |
| `ITEM_LAST`          | 701      | Последний идентификатор (одинаков с ITEM_FIRST, т.к. тип один) |
| `ITEM_STATE_CHANGED` | 701      | Компонент изменил состояние выбора                             |
| `SELECTED`           | 1        | Элемент был выбран                                             |
| `DESELECTED`         | 2        | Элемент был снят с выбора                                      |

---
## Методы
Класс `ItemEvent` наследует от `AWTEvent` методы `getSource()`, `getID()`, `toString()`,  
а также добавляет **собственные методы**, специфичные для работы с состоянием выбора.

### Работа с источником события
```java
// 💡Возвращает источник события в виде интерфейса ItemSelectable
// Это может быть Checkbox, Choice, List и т.д.
public ItemSelectable getItemSelectable()
```

### Работа с элементом выбора
```java
// Возвращает объект (элемент), с которым связано событие
// Например, текст пункта в списке или Checkbox
public Object getItem()
```

### Проверка состояния выбора
```java
// Возвращает состояние элемента: SELECTED или DESELECTED
public int getStateChange()

// Удобная форма проверки
// true, если элемент был выбран
public boolean isSelected() {
    return getStateChange() == SELECTED;
}
```

### Отладочная информация
```java
// Возвращает строковое описание события (например, item selected/deselected)
public String paramString()
```

## 💡 Особенности и подводные камни 💡

1️⃣💡 **ItemEvent описывает именно изменение выбора**,  
💡а не действие (например, нажатие кнопки).  
Если нужно отреагировать на “нажатие” — используй `ActionEvent`.

2️⃣ **Метод `getItem()` возвращает `Object`.**  
Это может быть:
- строка (в `Choice`),
- сам объект (в `List`),
- `CheckboxMenuItem`.  
    Поэтому обычно нужно приводить тип к ожидаемому.
    

3️⃣ 💡**`ItemSelectable` — это интерфейс**, который реализуют компоненты, поддерживающие выбор.  
К ним относятся:  
`Checkbox`, `Choice`, `List`, `CheckboxMenuItem`.

4️⃣ 💡**Выпадающие списки (`Choice`) генерируют `ItemEvent` только при реальном изменении.**  
Повторное выбор того же элемента не вызовет новое событие.

5️⃣ **Порядок событий может зависеть от платформы.**  
Например, при переключении между пунктами `Choice` может сначала прийти `DESELECTED` старого, затем `SELECTED` нового элемента.

## 💎 Best Practices 💎

✅ 💡Используй `ItemListener`, если нужно отслеживать **изменения состояния выбора** (а не клики).

✅ Проверяй `getStateChange()`:
```java
if (e.getStateChange() == ItemEvent.SELECTED) {
    // элемент выбран
} else {
    // элемент снят
}
```

✅ Приводи `getItem()` к нужному типу, если важно знать, **какой элемент** был выбран.

✅ При работе со списками (`List`) можно получить индекс выбранного элемента через сам компонент,  
а не через `ItemEvent`, если нужно больше информации.

✅ Для `Checkbox` и `CheckboxMenuItem` можно дублировать логику выбора и в `ActionListener`,  
если нужно реагировать и на действие, и на состояние.

## Пример
```java
import java.awt.*;
import java.awt.event.*;

public class ItemEventDemo extends Frame implements ItemListener {
    private Checkbox javaCB, pythonCB, cCB;
    private Label status;

    public ItemEventDemo() {
        setLayout(new FlowLayout());

        javaCB = new Checkbox("Java");
        pythonCB = new Checkbox("Python");
        cCB = new Checkbox("C++");

        javaCB.addItemListener(this);
        pythonCB.addItemListener(this);
        cCB.addItemListener(this);

        status = new Label("Выберите язык");
        add(javaCB);
        add(pythonCB);
        add(cCB);
        add(status);

        setSize(250, 150);
        setVisible(true);

        addWindowListener(new WindowAdapter() {
            public void windowClosing(WindowEvent e) {
                System.exit(0);
            }
        });
    }

    @Override
    public void itemStateChanged(ItemEvent e) {
        Checkbox src = (Checkbox) e.getItemSelectable();
        String state = (e.getStateChange() == ItemEvent.SELECTED) ? "выбран" : "снят";
        status.setText(src.getLabel() + " " + state);
    }

    public static void main(String[] args) {
        new ItemEventDemo();
    }
}

```
