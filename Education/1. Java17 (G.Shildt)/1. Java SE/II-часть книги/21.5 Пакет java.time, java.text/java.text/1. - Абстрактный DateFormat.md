
> [!NOTE] Класс `DateFormat` — это **абстрактный** класс, предназначенный `для форматирования и разбора дат и времени` в зависимости от локали.
> Он обеспечивает общий API для преобразования объектов `Date` в строки и обратно.  
> 
> Реализует интерфейс `Cloneable` и `Serializable`.

см Аналоги из `java.time.format*`
## Конструктор
```java
// Конструктор отсутствует (protected)
// DateFormat — абстрактный класс, поэтому его нельзя напрямую создать.

// Для получения экземпляра используются фабричные методы:
public static final DateFormat getInstance()
// Возвращает стандартный формат даты и времени по умолчанию для текущей локали.

public static final DateFormat getDateInstance()
// Возвращает формат только для даты (без времени) в стандартном стиле и текущей локали.

public static final DateFormat getDateInstance(int style)
// Возвращает формат для даты с указанным стилем (FULL, LONG, MEDIUM, SHORT).

public static final DateFormat getDateInstance(int style, Locale locale)
// Возвращает формат даты с заданным стилем и локалью.

public static final DateFormat getTimeInstance()
// Возвращает формат только для времени (без даты).

public static final DateFormat getTimeInstance(int style)
// Возвращает формат для времени с указанным стилем.

public static final DateFormat getTimeInstance(int style, Locale locale)
// Возвращает формат времени с заданным стилем и локалью.

public static final DateFormat getDateTimeInstance()
// Возвращает формат для даты и времени в стандартном стиле.

public static final DateFormat getDateTimeInstance(int dateStyle, int timeStyle)
// Возвращает формат даты и времени с указанными стилями.

public static final DateFormat getDateTimeInstance(int dateStyle, int timeStyle, Locale locale)
// Возвращает формат даты и времени с заданными стилями и локалью.
```

## Методы

### Форматирование
```java
// Преобразует объект Date в строку в соответствии с установленным шаблоном.
public final String format(Date date)
```

### Разбор (парсинг)
```java
// Преобразует строку в объект Date, согласно установленному формату.
public Date parse(String source) throws ParseException
```

### Настройка
```java
// Устанавливает временную зону.
public void setTimeZone(TimeZone zone)

// Возвращает текущую временную зону.
public TimeZone getTimeZone()

// Устанавливает форматирование в режим строгой проверки (lenient = false)
public void setLenient(boolean lenient)

// Проверяет, включен ли "гибкий" режим разбора даты.
public boolean isLenient()
```

---

|Константа|Назначение|Пример (для `Locale.US`)|Пример (для `Locale.FRANCE`)|
|---|---|---|---|
|**FULL**|Полный формат, включает название дня недели и месяца.|Tuesday, October 15, 2025|mardi 15 octobre 2025|
|**LONG**|Почти полный — полное название месяца, но без дня недели.|October 15, 2025|15 octobre 2025|
|**MEDIUM**|Средний — короткое название месяца, числовой год.|Oct 15, 2025|15 oct. 2025|
|**SHORT**|Короткий — полностью числовое представление.|10/15/25|15/10/25|


---
## 💡 Особенности и подводные камни 💡

1️⃣ `DateFormat` не является потокобезопасным (не стоит использовать один экземпляр в нескольких потоках).  

2️⃣ Использует **локаль по умолчанию**, если явно не указать другую.  

3️⃣ При `setLenient(true)` парсер может корректировать некорректные даты (например, 32 января → 1 февраля).  

4️⃣ Методы `getDateInstance()` и т.п. возвращают разные шаблоны в зависимости от локали.  

5️⃣ Лучше использовать `java.time.form`

## Пример
```java
import java.text.DateFormat;
import java.util.Date;
import java.util.Locale;

public class Example {
    public static void main(String[] args) {
        Date now = new Date();
        DateFormat df = DateFormat.getDateTimeInstance(DateFormat.LONG, DateFormat.SHORT, Locale.US);

        String formatted = df.format(now);
        System.out.println("Formatted: " + formatted);

        try {
            Date parsed = df.parse(formatted);
            System.out.println("Parsed: " + parsed);
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

```
