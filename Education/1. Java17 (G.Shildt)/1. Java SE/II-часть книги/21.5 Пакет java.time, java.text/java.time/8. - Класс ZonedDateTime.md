
> [!NOTE] Представляет `дату, время и часовой пояс` (`ZoneId`).
> Иммутабельный, потокобезопасный.  
> Реализует:  `Temporal`, `ChronoZonedDateTime<LocalDate>`, `Serializable`.

## Фабричные методы:
```java
public static ZonedDateTime now();
public static ZonedDateTime now(ZoneId zone);
public static ZonedDateTime of(
	LocalDate date, 
	LocalTime time, 
	ZoneId zone);
	
public static ZonedDateTime of(LocalDateTime dateTime, ZoneId zone);
public static ZonedDateTime ofInstant(Instant instant, ZoneId zone);
```

## Методы
### Получение данных

public ZoneId getZone();

---
## 💡 Особенности и подводные камни 💡
1️⃣ Содержит информацию о часовой зоне.  

2️⃣ Корректно учитывает переход на летнее/зимнее время.  

3️⃣ Потокобезопасен.  

4️⃣ Для хранения в БД лучше конвертировать в `Instant`.

---

## 💎 Best Practices 💎
✅ Используй при работе с реальными часовыми поясами.  

✅ Для глобального UTC — `Instant`.  

✅ Для локального отображения — `ZonedDateTime`.

---
## Пример
```java
ZonedDateTime moscow = ZonedDateTime.now(ZoneId.of("Europe/Moscow"));
ZonedDateTime tokyo = moscow.withZoneSameInstant(ZoneId.of("Asia/Tokyo"));
System.out.println("Москва: " + moscow);
System.out.println("Токио: " + tokyo);
```
