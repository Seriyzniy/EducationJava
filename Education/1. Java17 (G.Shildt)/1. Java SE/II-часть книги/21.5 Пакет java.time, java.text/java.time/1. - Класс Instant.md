
> [!NOTE] Представляет `точку во времени` (timestamp) — количество секунд и наносекунд с начала эпохи `1970-01-01T00:00:00Z`.
>Используется как аналог `System.currentTimeMillis()`, но с наносекундной точностью.

Иммутабельный, потокобезопасный, используется для хранения timestamp'ов.
## Класс
```java
public final class Instant
        implements Temporal, TemporalAdjuster, Comparable<Instant>, Serializable {
    // ...
}
```
## Конструктор - приватный
/Конструктор приватный — экземпляры ==создаются через фабричные методы==.
```java
private Instant(long seconds, int nanos);
```

## Фабричные методы:
```java
// Текущий момент времени (UTC)
public static Instant now();

// Создание из секунд с начала эпохи
public static Instant ofEpochSecond(long epochSecond);

// Создание из секунд и наносекунд
public static Instant ofEpochSecond(long epochSecond, long nanoAdjustment);

// Создание из миллисекунд с начала эпохи
public static Instant ofEpochMilli(long epochMilli);
```

## Методы
### Получение информации
```java
// Возвращает секунды эпохи (с 1970-01-01T00:00:00Z)
public long getEpochSecond();

// Возвращает наносекундную часть
public int getNano();

// Преобразует в миллисекунды эпохи
public long toEpochMilli();
```

### Арифметика
```java
// Прибавляет длительность
public Instant plus(Duration duration);

// Вычитает длительность
public Instant minus(Duration duration);

// Добавляет секунды
public Instant plusSeconds(long seconds);

// Добавляет наносекунды
public Instant plusNanos(long nanos);
```

### Сравнение
```java
public boolean isBefore(Instant other);
public boolean isAfter(Instant other);
public boolean equals(Object obj);
public int compareTo(Instant other);
```

---
## 💡 Особенности и подводные камни 💡
1️⃣ ⭐Всегда хранит момент в **UTC**, без смещения.  

2️⃣ ⭐Не зависит от часового пояса пользователя.  

3️⃣ Идеален для хранения в БД и журналировании.  

4️⃣ Иммутабельный и потокобезопасный.  

5️⃣ ⭐⭐При использовании с локальным временем нужно указать зону (`ZoneId` или `ZoneOffset`).

---

## 💎 Best Practices 💎
✅ Используй `Instant.now()` для регистрации событий.  

✅⭐ Храни `Instant` в БД, а отображай пользователю `ZonedDateTime`.  

✅ Для измерения времени выполнения — используй `Duration.between(start, end)`.

---
## Пример
```java
Instant start = Instant.now();
Thread.sleep(500);
Instant end = Instant.now();

System.out.println("Разница: " + Duration.between(start, end).toMillis() + " мс");
```
