
> [!NOTE] Определяет в`ременной интервал` `в секундах` и наносекундах.
> 💡Используется для разницы между `Instant`, `LocalTime` и т.п.  
> Иммутабельный и потокобезопасный.  
> 
> Реализует `TemporalAmount`, `Comparable<Duration>`, `Serializable`.

## Конструктор - private
Приватный — экземпляры ==создаются через фабричные методы.==
```java
private Duration(long seconds, int nanos);
```

### Фабричные методы:
```java
// Длительность между двумя моментами времени
public static Duration between(Temporal startInclusive, Temporal endExclusive);

// Создание фиксированной длительности
public static Duration ofDays(long days);
public static Duration ofHours(long hours);
public static Duration ofMinutes(long minutes);
public static Duration ofSeconds(long seconds);
public static Duration ofMillis(long millis);
public static Duration ofNanos(long nanos);
```

## 💡 Особенности и подводные камни 💡
1️⃣ Не знает о датах — только секунды и наносекунды.  

2️⃣ Можно использовать отрицательные значения.  

3️⃣ Не подходит для календарных периодов (для этого есть `Period`).  

4️⃣ Используется с `Instant` и `LocalTime`.  

5️⃣ Потокобезопасный.

---

## 💎 Best Practices 💎
✅ Для измерения производительности: `Duration.between(start, end)`  

✅ Для периодов “час/минута/секунда” — `Duration`, а не `Period`.  

✅ Избегай больших значений — `Duration` не умеет “годы/месяцы”.

---
## Пример
```java
Instant start = Instant.now();
Thread.sleep(1000);
Instant end = Instant.now();

Duration d = Duration.between(start, end);
System.out.println("Прошло: " + d.toMillis() + " мс");
```
