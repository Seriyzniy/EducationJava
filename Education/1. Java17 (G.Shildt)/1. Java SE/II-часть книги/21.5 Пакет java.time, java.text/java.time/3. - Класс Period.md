
> [!NOTE] Представляет `интервал между датами` `в годах`, месяцах и днях
> 💡Работает только с `LocalDate`, не учитывает время суток.  
> Иммутабельный и потокобезопасный.  
> 
> Реализует `ChronoPeriod`, `Serializable`.

## Конструктор - private
Приватный — создается фабричными методами
```java
private Period(int years, int months, int days);
```

### Фабричные методы:
```java
public static Period ofDays(int days);
public static Period ofMonths(int months);
public static Period ofYears(int years);

// Разница между двумя датами
public static Period between(LocalDate startDate, LocalDate endDate);
```

## Методы
### Доступ к данным
```java
public int getDays();
public int getMonths();
public int getYears();
```

### Арифметика
```java
public Period plus(Period other);
public Period minus(Period other);
public Period multipliedBy(int scalar);
```

---
## 💡 Особенности и подводные камни 💡

1️⃣ ⭐Не содержит времени — только календарные даты.  

2️⃣ Может быть отрицательным (если start > end).  

3️⃣ ⭐⭐Не совместим с `Instant` и `Duration`.  

4️⃣ Учитывает календарные правила (високосные годы и т.д.).  

5️⃣ Потокобезопасен.

---

## 💎 Best Practices 💎
✅ 💡Используй `Period` для бизнес-интервалов (например, “через 3 месяца”).  

✅ Для измерения в секундах — `Duration`, не `Period`.  

✅ ⭐Не пытайся преобразовать `Period` в миллисекунды.

---
## Пример
```java
LocalDate birth = LocalDate.of(2000, 5, 1);
LocalDate now = LocalDate.now();
Period age = Period.between(birth, now);

System.out.println("Возраст: " + age.getYears() + " лет, " + age.getMonths() + " мес.");
```
