
> [!NOTE] **`Properties`**
> специализированный подкласс `Hashtable<Object,Object>`, предназначенный **для хранения пар "ключ-значение" типа `String`**.  
> 
> Обычно используется для **загрузки и сохранения конфигураций** из `.properties` файлов (например, `config.properties`, `application.properties`, и т.д.).

## Конструкторы
```java
// Создаёт пустой объект Properties без предков (родительских свойств).
public Properties()

// Создаёт объект Properties, который наследует значения по умолчанию
// из другого объекта Properties (defaults).
public Properties(Properties defaults)
```
🔹 `defaults` — это “родительский” набор свойств.  
Если ключ не найден в текущем объекте, он будет искаться в `defaults`.


## Методы

### Работа с данными
```java
// Получает значение по ключу (возвращает null, если не найдено).
public String getProperty(String key)

// Получает значение по ключу или возвращает значение по умолчанию.
public String getProperty(String key, String defaultValue)


// Устанавливает свойство (ключ и значение должны быть строками).
public Object setProperty(String key, String value)


// Проверяет, есть ли ключ.
public boolean containsKey(Object key)

// Удаляет свойство по ключу.
public synchronized Object remove(Object key)
```

### Загрузка и сохранение
```java
// Загружает свойства из потока ввода (InputStream) в формате ISO 8859-1.
// Каждая строка имеет формат "ключ=значение".
public synchronized void load(InputStream inStream)

// Загружает свойства из Reader (поддерживает Unicode напрямую).
public synchronized void load(Reader reader)



// Сохраняет свойства в поток вывода (OutputStream) в формате ISO 8859-1.
public synchronized void store(OutputStream out, String comments)

// Сохраняет свойства в Writer (UTF-8, поддержка Unicode).
public synchronized void store(Writer writer, String comments)
```
💡 `comments` — это необязательная строка, которая будет добавлена в начале файла как комментарий (с символом `#`).

### Работа с XML
```java
// Сохраняет свойства в XML-файл (кодировка UTF-8 по умолчанию).
public void storeToXML(OutputStream os, String comment)
public void storeToXML(
	OutputStream os, 
	String comment, 
	String encoding
)


// Загружает свойства из XML-файла.
public void loadFromXML(InputStream in)
```

### Работа с перечислением (Enumeration)
```java
// Возвращает перечисление всех ключей (Enumeration<Object>).
public Enumeration<?> propertyNames()

// Возвращает все свойства, включая унаследованные из defaults.
public Set<String> stringPropertyNames()
```

## 💡 Особенности и подводные камни 💡

1️⃣ ❗❗❗`Properties` наследует `Hashtable`, но **использует только строки** (`String` → `String`).  
Однако по сигнатуре методов допускаются `Object`, поэтому возможны ошибки, если вставить нестроковые типы.

2️⃣ 💡При `load(InputStream)` используется **кодировка ISO-8859-1**, поэтому ==НЕ-ASCII символы нужно экранировать== `\uXXXX`,  
или использовать `Reader`/`load(Reader)` для корректного Unicode.

3️⃣ ❗❗❗`store()` **перезаписывает весь файл** — нет частичного обновления.

4️⃣ Потоковые методы (`load`, `store`) **синхронизированы**, но общие операции с картой нет — при многопоточности стоит использовать внешнюю синхронизацию.

5️⃣ `Properties` не гарантирует порядок ключей (поскольку наследует от `Hashtable`).  
Для упорядоченного хранения лучше использовать `LinkedHashMap` или `TreeMap`.

---
## 💎 Best Practices 💎

✅ Используйте `try-with-resources` при работе с файлами (`InputStream`, `OutputStream`).  

✅ Для чтения UTF-8 файлов используйте `load(Reader)` вместо `load(InputStream)`. 

✅ Храните ключи в едином стиле (например, `db.url`, `db.user`, `db.password`).  

✅ Не храните чувствительные данные в `.properties` файлах (пароли, токены).  

✅ Для переопределения свойств используйте цепочку `defaults`.

## Пример
```java
import java.io.*;
import java.util.Properties;

public class PropertiesExample {
    public static void main(String[] args) {
        Properties props = new Properties();

        // Загрузка конфигурации из файла
        try (FileReader reader = new 
	        FileReader("config.properties")
	    ) {
            props.load(reader);
        } catch (IOException e) {
            e.printStackTrace();
        }

        // Получение значений
        String host = props.getProperty("db.host", "localhost");
        String port = props.getProperty("db.port", "5432");
        System.out.println("Host: " + host + ", Port: " + port);

        // Добавление нового свойства
        props.setProperty("app.mode", "production");

        // Сохранение обратно в файл
        try (FileWriter writer = new 
	        FileWriter("config.properties")
	    ) {
            props.store(writer, "App configuration updated");
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
```
