
> [!NOTE] Контракт
> «Я источник символов, из меня можно читать».


Интерфейс `Readable` указывает, что объект можно применять в качестве источника символов. В нем определен один метод по имени `read()`:
```java
int read(CharBuffer buf) throws IOException
```

Метод `read()` считывает символы в `buf`. Он возвращает количество про читанных символов или -1, если был достиrнут конец файла.

## Кто реализует:
- `Reader` (например, `FileReader`, `BufferedReader`)
    
- `Scanner`

## Комментарий:
1. `Readable` мало кто юзает напрямую, но он встроен в экосистему: `Scanner` работает именно с `Readable`.
    
2. ⭐Если ты делаешь свой источник данных (например, читаешь из сети), можно реализовать `Readable`, и ==все стандартные инструменты будут работать== с ним «из коробки».

***
# Пример
```java
import java.io.IOException;
import java.nio.CharBuffer;

class StringReadable implements Readable {
    private final String text;
    private int index = 0;

    public StringReadable(String text) {
        this.text = text;
    }

    @Override
    public int read(CharBuffer cb) throws IOException {
        if (index >= text.length()) return -1; // конец данных
        cb.append(text.charAt(index++));
        return 1;
    }
}

public class TestReadable {
    public static void main(String[] args) throws IOException {
        Readable r = new StringReadable("Java");
        
        CharBuffer buf = CharBuffer.allocate(10);
        
        while (r.read(buf) != -1) {}
        
        buf.flip();
        System.out.println(buf); // Java
    }
}

```
