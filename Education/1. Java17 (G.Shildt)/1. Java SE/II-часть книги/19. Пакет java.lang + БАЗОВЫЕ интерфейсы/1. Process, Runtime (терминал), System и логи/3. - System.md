# 📕1. Использование currentTimeМillis() для хронометража выполнения программы

Одно из особенно интересных применений класса System связано с использованием метода `currentTimeMillis()` для определения, ==сколько времени занимает выполнение различных частей программы==

Для хронометража раздела программы ==сохраните такое значение прямо перед началом== интересующего раздела. 
Немедленно ==после завершения раздела снова вызовите== `currentTimeMillis()`. 

Затраченное время вычисляется как ==разность времени окончания и времени начала==, что демонстрируется ниже в программе:
```java
public class SystemDemo {
    public static void main(String[] args) {
        // ⏳Время
        long start = System.nanoTime();
        for (int i = 0; i < 1_000_000; i++);
        long end = System.nanoTime();
        System.out.println("Время выполнения: " + (end - start) + " нс");

        // Свойства JVM
        System.out.println("Java версия: " + System.getProperty("java.version"));
        System.out.println("ОС: " + System.getProperty("os.name"));

        // Переменные окружения
        System.out.println("PATH = " + System.getenv("PATH"));

        // Копирование массива
        int[] src = {1, 2, 3, 4};
        int[] dest = new int[4];
        System.arraycopy(src, 0, dest, 0, src.length);
        System.out.println("dest[2] = " + dest[2]); // 3

        // Завершение программы с кодом 0
        System.exit(0);
    }
}
```

### Пример получения определенного свойства
```java
public class SystemPropertiesExample {
    public static void main(String[] args) {
        // Получение отдельных свойств
        String javaVersion = System.getProperty("java.version");
        String osName = System.getProperty("os.name");
        String userHome = System.getProperty("user.home");
        
        System.out.println("Java Version: " + javaVersion);
        System.out.println("OS: " + osName);
        System.out.println("User Home: " + userHome);
        
        // Получение всех свойств
        System.out.println("\nAll System Properties:");
        System.getProperties().forEach((key, value) -> {
            System.out.println(key + " = " + value);
        });
        
        // Установка пользовательского свойства
        System.setProperty("my.custom.property", "customValue");
        String customProp = System.getProperty("my.custom.property");
        System.out.println("Custom Property: " + customProp);
    }
}
```

# 📕 4. System. Logger и System. LoggerFinder 
Интерфейс `System.Logger` и класс `System.LoggerFinder` поддерживают журнал программы. 

## Взаимосвязь
- `System.getLogger()` → вызывает `LoggerFinder` → возвращает конкретный `Logger`.
    
- Пользователь работает только с `System.Logger`.
    
- 💡Реализация (например, JUL, Log4j2, SLF4J) может быть подключена через `LoggerFinder`.

## 📙1. System.Logger`

Это **унифицированный API для логирования**, встроенный в Java.  

Он был добавлен как ==облегчённая альтернатива== `java.util.logging.Logger` (JUL), чтобы JVM могла предоставлять **единый интерфейс для логов**, а реализацию можно было подменять


> [!NOTE] ⭐`System.Logger` - ==ИНТЕРФЕЙС==, который можно реализовать⭐
>```java
> public interface System.Logger
> ```

==Получить его можно через:==
```java
System.Logger logger = System.getLogger("MyLogger");
```
### Уровни логирования (`System.Logger.Level`):
- `ALL`
- `TRACE`
- `DEBUG`
    
- `INFO`
- `WARNING`
- `ERROR`
    
- `OFF`

### Методы:
```java
void log(System.Logger.Level level, String msg);
void log(System.Logger.Level level, String msg, Throwable thrown);
void log(System.Logger.Level level, Supplier<String> msgSupplier);
void log(System.Logger.Level level, Supplier<String> msgSupplier, Throwable thrown);

```

📌 То есть можно писать:
```java
System.Logger logger = System.getLogger("demo");

logger.log(System.Logger.Level.INFO, "Привет, лог!");
```

## 📙2.  System.LoggerFinder

Это **SPI (Service Provider Interface)** — ==механизм==, который ==позволяет заменить дефолтный логгер на свою реализацию==.

### Основные моменты:
- Абстрактный класс: `public abstract class System.LoggerFinder`.
    
- 🧠JVM при вызове `System.getLogger()` делегирует создание логгера именно через `LoggerFinder`.
    
- 💡По умолчанию Java использует **`java.util.logging` (JUL)**.
    
- Можно написать свою реализацию и зарегистрировать её через `META-INF/services/java.lang.System$LoggerFinder`.

### Ключевые методы
```java
public abstract System.Logger getLogger(String name, Module module);
```
📌 JVM вызывает этот метод, когда кто-то делает `System.getLogger("...")`

```java
// ✅ 1. Кастомный логгер
class MyFileLogger implements Logger {
    private final String name;
    private final PrintWriter writer;

    MyFileLogger(String name) {
        this.name = name;
        try {
            this.writer = new PrintWriter(new FileWriter("myapp.log", true), true);
        } catch (IOException e) {
            throw new RuntimeException("Не удалось открыть лог-файл", e);
        }
    }

    @Override
    public String getName() {
        return name;
    }

    @Override
    public boolean isLoggable(Level level) {
        return true; // всегда логируем
    }

    @Override
    public void log(Level level, ResourceBundle bundle, String msg, Object... params) {
        writer.printf("[%s] %s: %s%n", level, name, String.format(msg, params));
    }

    @Override
    public void log(Level level, ResourceBundle bundle, String msg, Throwable thrown) {
        writer.printf("[%s] %s: %s - %s%n", level, name, msg, thrown);
        thrown.printStackTrace(writer);
    }
}

// ✅ 2. Провайдер логгеров
public class MyLoggerFinder extends LoggerFinder {
    @Override
    public Logger getLogger(String name, Module module) {
        return new MyFileLogger(name);
    }
}
```
### Регистрация в SPI

Чтобы JVM узнал о нашей реализации, нужно создать файл:
```bash
META-INF/services/java.lang.System$LoggerFinder
```
Содержимое файла:
```nginx
MyLoggerFinder
```
