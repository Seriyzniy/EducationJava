До появления **Collections Framework** (JDK 1.2) в пакете `java.util` существовали свои классы и интерфейсы для работы с коллекциями.
    
После появления новых коллекций они были **переделаны**, чтобы поддерживать современные интерфейсы (`List`, `Map`, `Iterator` и т.д.).
    
Но часть старых API осталась ради обратной совместимости — ==их называют **унаследованными (legacy) классами и интерфейсами**==.

## Унаследованные классы
В пакете `java.util` остались такие классы:

- 🔸**Dictionary** (устаревший аналог `Map`)
    
- 🔸**Hashtable** (старый аналог `HashMap`, но синхронизирован)
    
- 🔸**Properties** (наследник `Hashtable`, работает с `String`-ключами и значениями, часто используется для конфигураций)
    
- 🔸**Vector** (старый аналог `ArrayList`, но синхронизирован)
    
- 🔸**Stack** (наследник `Vector`, реализует стек LIFO — сейчас лучше `ArrayDeque`)

## Важное отличие
- 💡Все **унаследованные коллекции СИНХРОНИЗИРОВАНЫ** (thread-safe).
    
- 💡А современные (`ArrayList`, `HashMap`, `TreeMap`, `HashSet`) — **НЕсинхронизированные**.
    
- Поэтому иногда унаследованные коллекции удобны в многопоточной среде, хотя обычно лучше использовать `Collections.synchronizedXXX()` или современные конкурентные коллекции (`ConcurrentHashMap`, `CopyOnWriteArrayList`).

***
⚠️ Рекомендация: в современном коде стоит использовать **новые коллекции** (`ArrayList`, `HashMap`, `ArrayDeque` и т.п.), а унаследованные — только если приходится поддерживать старый код.

***
# 📕 1. Интepфeйc Enumeration
**Enumeration** — старый аналог `Iterator`.
## Методы
```java
boolean hasMoreElements()
E nextElement()
```

- - С JDK 9 появился метод `asIterator()` для преобразования `Enumeration` → `Iterator`.
    - Используется до сих пор в `Vector`, `Properties`, `Hashtable`.

👉 В современном коде лучше использовать `Iterator` или цикл `for-each`.

***
# 📕 2. Класс Vector
Реализует **динамический массив**, как `ArrayList`.
    
Отличия:
- синхронизирован;
	
- имеет старые методы (`addElement()`, `elementAt()`, `firstElement()`, `lastElement()` и т.д.), дублирующие `List`-методы.

Сейчас рекомендуется `ArrayList`, а `Vector` оставили для обратной совместимости.

***
# 📕3. Класс Stack

Подкласс `Vector`.
Реализует классический стек (LIFO).

Методы:
- `push(E e)` — добавить;
	
- `pop()` — извлечь и удалить;
	
- `peek()` — посмотреть верхний элемент без удаления;
	
- `empty()`, `search()`.
        
Недостаток: основан на `Vector`.

Современная альтернатива — **`ArrayDeque`**.

---
# 📕 4. Абстрактный класс Dictionary

Абстрактный класс — устаревший предок ассоциативных контейнеров.
    
Позже полностью заменён на **`Map`**.
    
Сейчас почти не используется напрямую.

---
# 📕 5. Класс Hashtable

Реализация `Dictionary`, после JDK 1.2 переписана под интерфейс `Map`.
    
Отличия от `HashMap`:
- синхронизирован;
	
- **не допускает `null`** ни в ключах, ни в значениях;
	
- немного медленнее.
        
В современном коде лучше `HashMap` или `ConcurrentHashMap`.

---

# 📕 6. ⭐Класс Properties

Наследник `Hashtable<String, String>`.
    
Используется для хранения конфигураций в формате **ключ=значение**.
    
Методы:
- `getProperty(String key)` / `setProperty(String key, String value)`;
	
- `load(InputStream)` / `store(OutputStream)` для работы с файлами;
	
- поддерживает **XML-формат** (`loadFromXML`, `storeToXML`).
        
До сих пор активно применяется (например, для настроек `System.getProperties()` или конфигов приложений).

## Использование методов store() и load()
Один из наиболее полезных аспектов класса `Properties` связан с тем, что информацию, содержащуюся в объекте `Properties`, ==можно легко сохранять и загружать с диска==посредством методов `store()` и `load()` . 

Объект Properties можно в любой момент записать в поток и прочитать его обратно.

```java
/*Простая база данных телефонных номеров, которая использует список свойств.*/
import java.io.*;
import java.util.*;

class Phonebook {
    public static void main(String[] args) throws IOException {
        Properties ht = new Properties();
        
        BufferedReader br = new BufferedReader(
            new InputStreamReader(System.in, System.console().charset()));
        
        String name, number;
        FileInputStream fin = null;
        boolean changed = false;

        // Попробовать открыть файл phonebook.dat.
        try {
            fin = new FileInputStream("phonebook.dat");
        } catch (FileNotFoundException e) {
            // Игнорировать отсутствие файла.
        }

        /* Если файл phonebook.dat существует, тогда загрузить из него номера телефонов. */
        try {
            if (fin != null) {
                ht.load(fin);
                fin.close();
            }
        } catch (IOException e) {
            System.out.println("Ошибка при чтении файла.");
        }

        // Предоставить пользователю возможность вводить новые имена и номера.
        do {
            System.out.println("Введите новое имя" +
                " (quit для завершения): ");
            name = br.readLine();
            if (name.equals("quit")) continue;

            System.out.println("Введите номер: ");
            number = br.readLine();

            ht.setProperty(name, number);
            changed = true;
        } while (!name.equals("quit"));

        // Если данные телефонного справочника изменились, тогда сохранить его.
        if (changed) {
            FileOutputStream fout = new FileOutputStream("phonebook.dat");
            ht.store(fout, "Телефонный справочник");
            fout.close();
        }

        // Искать номера по заданному имени.
        do {
            System.out.println("Введите интересующее имя" +
                " (quit для завершения): ");
            name = br.readLine();
            if (name.equals("quit")) continue;
            
            number = (String) ht.get(name);
            System.out.println(number);
        } while (!name.equals("quit"));
    }
}
```
