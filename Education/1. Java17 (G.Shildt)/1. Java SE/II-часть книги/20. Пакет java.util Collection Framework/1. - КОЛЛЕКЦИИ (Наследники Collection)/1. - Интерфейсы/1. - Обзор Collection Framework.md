# Введение
Collections Framework — это унифицированная архитектура для представления и манипуляциирования коллекциями объектов. 

До его появления в Java 1.2 использовались разрозненные классы вроде `Vector`, `Hashtable`, `Stack`, которые не имели общей архитектуры.

**Ключевые преимушения:**
- **Стандартизация**: Единый способ работы с разными типами коллекций
    
- **Производительность**: Оптимизированные реализации основных структур данных
    
- **Интероперабельность**: Разные коллекции могут легко взаимодействовать
    
- **Расширяемость**: Возможность создавать собственные реализации
    
- **Интеграция с массивами**: Плавный переход между массивами и коллекциями


> [!NOTE] ⭐ГОТОВЫЕ алгоритмы в классе `CollectionS`⭐
> Алгоритмы - еще одна важная часть механизма коллекций. 
> Алгоритмы работают с коллекциями и определяются как `статические методы` в классе `Collections`.


> [!NOTE] 🧠Интерфейс `Iterator`
> С инфраструктурой Collections Framework тесно связан еще один эле мент - интерфейс `Iterator`. 
> Итератор предлагает универсальный стандартизированный способ доступа к элементам внутри коллекции по одному за раз.
> 
> Поскольку каждая коллекция поддерживает итератор, `доступ к элементам любого класса коллекции можно получить с помощью` `методов`, определенных `в интерфейсе Iterator`

> [!NOTE] 🧠Интерфейс `Spliterator`
> В версии JDK 8 добавлен еще один тип итератора, называемый сплитератором (spliterator), или разделяющим итератором
> Сплитераторы представляют собой итераторы, которые обеспечивают поддержку параллельной итерации


> [!NOTE] `Карты` (ключ-значение) `НЕ ЯВЛЯЮТСЯ КОЛЛЕКЦИЯМИ`, но работать с ней можно как с коллекцией
