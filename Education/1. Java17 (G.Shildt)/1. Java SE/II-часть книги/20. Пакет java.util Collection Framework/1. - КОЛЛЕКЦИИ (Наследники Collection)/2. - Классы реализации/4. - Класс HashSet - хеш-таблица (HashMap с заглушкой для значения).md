
> [!NOTE] Класс `HashSet` - `ХЕШ-ТАБЛИЦА` с уникальными элементами
> Расширяет `AbstractSet` и реализует интерфейс `Set`. 
> Он создает коллекцию, в которой для хранения данных применяется хеш-таблица.
> 
> 🧠 ПОД КАПОТОМ используется `HashMap`


> [!NOTE] 🧠Ключ преобразуется в хеш-код🧠
> `Преобразование` ключа в его хеш-код выполняется `автоматически` - вам никогда не придется видеть сам хеш-код

```java
public class HashSet<E>
    extends AbstractSet<E>
    implements Set<E>, Cloneable, java.io.Serializable
{ ... }
```
### Характеристики:
- Порядок хранения элементов **не гарантируется** (может меняться даже при одинаковом наборе элементов).
    
- ⭐⭐В основе — `HashMap<E,Object>`, где:⭐⭐
    - ✅ключ = элемент множества,
    - ❌значение = константа-заглушка (`PRESENT`).
        
- Допускает `null` (только один).
    
- Основные операции (`add`, `remove`, `contains`) работают за **O(1)** в среднем, но в худшем случае могут быть **O(n)** (например, при плохой `hashCode()`).

***
## Конструкторы
```java
// Создаёт пустой HashSet с начальной ёмкостью = 16 и коэффициентом загрузки = 0.75
public HashSet()

// Создаёт пустой HashSet с указанной начальной ёмкостью
// initialCapacity — размер внутренней таблицы хешей
public HashSet(int initialCapacity)

// Создаёт пустой HashSet с указанной ёмкостью и коэффициентом загрузки
// 🧠loadFactor — определяет, когда увеличивать ёмкость (по умолчанию 0.75)
public HashSet(int initialCapacity, float loadFactor)

// Создаёт HashSet, содержащий элементы другой коллекции
public HashSet(Collection<? extends E> c)
```

## Методы
Никаких дополнительных методов кроме тех, которые предоставлены суперклассами и интерфейсами, в классе HashSet не определено.
```java
// ==== Основные методы ====
// ✅Добавление✅
// Добавляет элемент в множество.
// Возвращает true, если элемент был добавлен (его ещё не было).
public boolean add(E e)

// Добавляет все элементы из коллекции c.
// Возвращает true, если множество изменилось.
public boolean addAll(Collection<? extends E> c)

// ✅Удаление✅
// Удаляет элемент из множества.
// Возвращает true, если элемент был найден и удалён.
public boolean remove(Object o)

// Удаляет все элементы, содержащиеся в коллекции c.
public boolean removeAll(Collection<?> c)

// ✅Очистка✅
// Удаляет все элементы множества.
public void clear()

// ✅Проверка содержимого✅
// Проверяет, содержится ли элемент.
public boolean contains(Object o)

// Проверяет, содержатся ли все элементы коллекции c.
public boolean containsAll(Collection<?> c)

// Оставляет только те элементы, которые есть в коллекции c.
public boolean retainAll(Collection<?> c)

// ✅Инфа о коллекции✅
// Возвращает количество элементов.
public int size()

// Проверка на пустоту.
public boolean isEmpty()

// ✅Итератор✅
// Возвращает итератор (порядок обхода непредсказуем).
public Iterator<E> iterator()

// ✅Преобразование в массив✅
// Возвращает массив Object[] с элементами.
public Object[] toArray()

// Возвращает массив T[] с элементами.
public <T> T[] toArray(T[] a)

// ==== Специфические ====
// Повторяет добавление, но при этом гарантирует, что в множестве нет дубликатов.
public boolean equals(Object o)

// Хеш множества (зависит от элементов, но не от порядка).
public int hashCode()
```

## 💡 Особенности и подводные камни💡

1. 🔶**Основан на HashMap**
    - Каждый элемент хранится как ключ.
        
    - Значение — константа `PRESENT`.
        
2. 🔶**hashCode & equals**
    - Корректность работы `HashSet` **целиком зависит от реализации `hashCode()` и `equals()`**.
        
    - Если их нарушить → возможны дубликаты или "потеря" элементов.
        
3. 🔶**Порядок элементов**
    - Не гарантируется.
        
    - Для "упорядоченного" множества лучше использовать:
        
        - `LinkedHashSet` (сохраняет порядок вставки),
            
        - `TreeSet` (сортировка).
            
4. 🔶**Производительность**
    - `loadFactor = 0.75` — оптимальный компромисс.
        
    - 🧠При переполнении (кол-во элементов > capacity * loadFactor) размер таблицы удваивается. (80 > 100*0.75 => 80 > 75 => capacity = 200)
        
5. 🔶**Null**
    - Допускается **ровно один `null`**.
        
6. 🔶**Итерация**
    - Итерация по `HashSet` **O(n)**, но порядок произвольный.

***
## ✅ Best Practices ✅
- Всегда правильно переопределяй **equals() и hashCode()**, если элементы — собственные классы.
    
- Если нужен **порядок вставки** — используй `LinkedHashSet`.
    
- Если нужна **сортировка** — используй `TreeSet`.
    
- Для **частого поиска/проверки уникальности** `HashSet` — оптимален.
    
- Если ожидается много элементов — указывай `initialCapacity`, чтобы избежать перераспределений.

## Пример
```java
import java.util.*;

public class HashSetExample {
    public static void main(String[] args) {
        // Создаём HashSet
        HashSet<String> set = new HashSet<>();

        // Добавление элементов
        set.add("Java");
        set.add("C++");
        set.add("Python");
        set.add("Java"); // дубликат, проигнорируется

        // Проверка наличия
        System.out.println("Есть ли Python? " + set.contains("Python"));

        // Удаление
        set.remove("C++");

        // Размер
        System.out.println("Размер: " + set.size());

        // Итерация
        for (String lang : set) {
            System.out.println(lang);
        }

        // Преобразование в массив
        String[] arr = set.toArray(new String[0]);
        System.out.println("Массив: " + Arrays.toString(arr));

        // Демонстрация уникальности
        set.add(null);
        set.add(null); // второй раз не добавится
        System.out.println("HashSet с null: " + set);
    }
}
```
