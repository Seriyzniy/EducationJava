
> [!NOTE] **`LinkedHashSet`** — это "прокачанный" `HashSet`, у которого добавилось **сохранение порядка**.
> `LinkedHashSet<E>` — это **множество с сохранением порядка вставки**.

```java
public class LinkedHashSet<E>
    extends HashSet<E>
    implements SequencedSet<E>, Cloneable, java.io.Serializable 
{ ... }
```

В основе лежит:
- **`LinkedHashMap<E,Object>`**, где:
    - ✅ключ (`key`) = элемент множества,
    - ❌значение (`value`) = константа-заглушка (`PRESENT`).
        
- 🧠Каждая запись (`Entry`) дополнительно хранит ссылки `before` и `after`, образуя **двусвязный список** для сохранения порядка.

Таким образом, `LinkedHashSet` сочетает свойства `HashSet` (уникальность, быстрые операции) и `LinkedList` (упорядоченность элементов).

***
## Конструкторы
```java
// Создаёт пустой LinkedHashSet с ёмкостью = 16 и loadFactor = 0.75
public LinkedHashSet()

// Создаёт пустой LinkedHashSet с указанной ёмкостью.
public LinkedHashSet(int initialCapacity)

// Создаёт пустой LinkedHashSet с указанной ёмкостью и коэффициентом загрузки.
public LinkedHashSet(int initialCapacity, float loadFactor)

// Создаёт LinkedHashSet, содержащий элементы другой коллекции.
// Элементы будут вставлены в порядке обхода коллекции.
public LinkedHashSet(Collection<? extends E> c)
```

## Методы
Методы такие же, как у `HashSet`, потому что `LinkedHashSet` **не добавляет новых**, только переопределяет поведение итерации (сохраняется порядок вставки).

***
## 💡Особенности и подводные камни💡 
- 🔶**Порядок элементов**
    - В отличие от `HashSet`, **порядок вставки сохраняется**.
        
    - Можно итерироваться по множеству и получать элементы в том же порядке, в котором они были добавлены.
        
- 🔶**Стоимость по памяти**
    - `LinkedHashSet` хранит дополнительную двусвязную цепочку (`before`/`after`) → расходует больше памяти, чем `HashSet`.
        
- 🔶**Скорость**
    - Операции `add`, `remove`, `contains` остаются **O(1)** в среднем.
        
    - Но итерация немного медленнее, чем у `HashSet`, из-за связанного списка.
        
- 🔶**Null**
    - Допускается один `null`, как и в `HashSet`.
        
- 🔶**Сравнение с TreeSet**
    - `TreeSet` сортирует элементы по `Comparator` или `Comparable`, а `LinkedHashSet` лишь сохраняет порядок вставки.

***
## ✅Best Practices✅
- Используй `LinkedHashSet`, когда нужно **множество с уникальными элементами и сохранением порядка вставки**.
    
- Если порядок **не важен** — лучше `HashSet` (экономия памяти).
    
- Если нужен **сортированный порядок** — используй `TreeSet`.
    
- 🧠Хорошо подходит для:
    - хранения **истории** (например, посещённых URL),
        
    - реализации **LRU-кэшей** (через переопределённый `removeEldestEntry` в `LinkedHashMap`).

***
## Пример
```java
import java.util.*;

public class LinkedHashSetExample {
    public static void main(String[] args) {
        LinkedHashSet<String> set = new LinkedHashSet<>();

        // Добавление элементов
        set.add("Java");
        set.add("C++");
        set.add("Python");
        set.add("Go");
        set.add("Java"); // дубликат не добавится

        // ⭐⭐Итерация (порядок вставки сохраняется)⭐⭐
        System.out.println("Элементы в порядке вставки:");
        for (String lang : set) {
            System.out.println(lang);
        }

        // Проверка наличия
        System.out.println("Содержит Python? " + set.contains("Python"));

        // Удаление
        set.remove("C++");

        System.out.println("После удаления:");
        for (String lang : set) {
            System.out.println(lang);
        }

        // ✅Добавление null
        set.add(null);
        set.add(null); // ❌второй раз не добавится
        System.out.println("С null: " + set);
    }
}
```
