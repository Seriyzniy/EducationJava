
> [!NOTE] `Comparator<T>` — это `интерфейс` из пакета `java.util`, который используется `для задания логики сравнения` объектов.
> 

# 📕1. Интерфейс и методы

```java
@FunctionalInterface
public interface Comparator<T> {
	// 💡 Здесь obj 1 и obj 2 - сравниваемые объекты.
	// return 1, если obj1 > obj2
	// return 0, если obj1 == obj2
	// return -1, если obj1 < obj2
    int compare(T obj1, T obj2);

	// 💡 В obj указывается объект, подлежащий проверке на равенство
	// true -> obj и this являются экземплярами реализации Comparator
	// и использую одно и то же упорядочивание
	// false в противном случае
	// 🧠ПЕРЕОПРЕДЕЛЯТЬ необязательно
    boolean equals(Object obj);

    // ✅ Методы по умолчанию (default): ✅
    // Возвращает компаратор с обратным порядком
    default Comparator<T> reversed() { ... }
    
    // Позволяет "цепочкой" добавлять вторичные критерии сортировки.
    // 💡 Возвращает компаратор, который выполняет второе сравнение, когда результат первого сравнения указывает, что сравниваемые объекты равны.
    default Comparator<T> thenComparing(Comparator<? super T> other) {...}
    
    default <U> Comparator<T> 
	    thenComparing(
            Function<? super T, ? extends U> keyExtractor,
            Comparator<? super U> keyComparator) { ... }
            
    default <U extends Comparable<? super U>> Comparator<T> 
	    thenComparing(
            Function<? super T, ? extends U> keyExtractor) { ... }


    // ✅ Статические фабричные методы:✅ 
    // Получить компаратор, использующий естественный порядок
    public static <T extends Comparable<? super T>> Comparator<T> 
	    naturalOrder() { ... }
	    
	    
    // Сортировка в обратном порядке
    public static <T extends Comparable<? super T>> Comparator<T> 
	    reverseOrder() { ... }
    
    
    // Создание компаратора по извлечённому ключу (через `Function`).
    public static  <T, U extends Comparable<? super U>> Comparator<T> 
	    comparing(
            Function<? super T, ? extends U> keyExtractor) { ... }
            
    public static <T, U> Comparator<T> 
		comparing(
            Function<? super T, ? extends U> keyExtractor,
            Comparator<? super U> keyComparator) { ... }
            
    // Возвращает компаратор, который трактует значения null как 
    // меньшие/большие, чем другие значения.    
    public static <T> Comparator<T> 
	    nullsFirst(Comparator<? super T> comparator) { ... }
    
    public static <T> Comparator<T> 
	    nullsLast(Comparator<? super T> comparator) { ... }
}
```

# 📕 2. Собственная реализация `Comparator`

> [!NOTE] Реализовывать `equals` не обязательно
> 
## 📙1. Реализация через класс
Ты можешь создать отдельный класс, который реализует интерфейс `Comparator<T>` и переопределяет метод `compare`:
```java
import java.util.*;

// Класс с данными
class Person {
    String name;
    int age;
    Person(String name, int age) { this.name = name; this.age = age; }
    @Override
    public String toString() { return name + " (" + age + ")"; }
}

// 🟢 Реализация Comparator через отдельный класс
class AgeComparator implements Comparator<Person> {
    @Override
    public int compare(Person p1, Person p2) { //🟢ПЕРЕОПРЕДЕЛЯЕМ
        // Сравнение по возрасту
        return Integer.compare(p1.age, p2.age);
    }
}

// Использование
public class Test {
    public static void main(String[] args) {
        List<Person> people = Arrays.asList(
            new Person("Alex", 30),
            new Person("Bob", 25),
            new Person("Charlie", 40)
        );

        Collections.sort(people, new AgeComparator());
        System.out.println(people); 
        // [Bob (25), Alex (30), Charlie (40)]
    }
}
```

## 📙 2. Реализация через анонимный класс
```java
Collections.sort(people, new Comparator<Person>() {
    @Override
    public int compare(Person p1, Person p2) {
        return p1.name.compareTo(p2.name); // сортировка по имени
    }
});
```

## 📙3. Реализация через лямбду (короткая форма)
Так как `Comparator` — это **функциональный интерфейс** (1 абстрактный метод), его можно реализовать лямбдой:
```java
Collections.sort(people, (p1, p2) -> p1.name.compareTo(p2.name));
```

⭐или ещё короче через `Comparator.comparing`:
```java
people.sort(Comparator.comparing(p -> p.name));
```

## ☢ Подводные камни при ручной реализации

- **Не возвращай только `-1`, `0`, `1`** — используй `Integer.compare`, чтобы не было ошибок при больших числах.
    
- **Убедись в транзитивности**: если `a > b` и `b > c`, то `a > c`.
    
- Обрабатывай `null`, если такие значения могут встречаться.

# 📕 3. Спец. методы для примитивов
В интерфейсе Comparator также добавляются следующие специализиро ванные версии методов последующего сравнения для примитивных типов: 
```java
default Comparator thenComparingDouble(ToDoubleFunction<? super T> getKey) 
default Comparator thenComparingInt(ToIntFunction<? super T> getKey) 
default Comparator thenComparingLong(ToLongFunction<? super T> getKey)
```
Во всех методах `getKey` ссылается на функцию, которая получает следующий ключ для сравнения.

🧠Все эти ссылки `To...Function` - функциональные интерфейсы
```java
@FunctionalInterface
public interface ToIntFunction<T> {
    int applyAsInt(T value);
}
```

***
# ✅ Best Practices ✅
- Используй **`Comparator.comparing(...)`** вместо ручной реализации `compare()` для читаемости.
    
- Если нужно сортировать по нескольким полям — используй **`thenComparing(...)`**.

***
## Пример
```java
import java.util.*;

class Person {
    String name;
    int age;
    
    Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
    
    @Override
    public String toString() {
        return name + " (" + age + ")";
    }
}

public class ComparatorExample {
    public static void main(String[] args) {
        List<Person> people = Arrays.asList(
            new Person("Alex", 30),
            new Person("Bob", 25),
            new Person("Charlie", 25),
            new Person("Dave", 40)
        );

        // Сортировка по возрасту
        people.sort(Comparator.comparingInt(p -> p.age));

        // Дополнительная сортировка по имени при равном возрасте
        people.sort(
            Comparator.comparingInt((Person p) -> p.age)
                      .thenComparing(p -> p.name)
        );

        System.out.println(people);
        // [Bob (25), Charlie (25), Alex (30), Dave (40)]
    }
}
```

### Почему работает именно для `String`?

Потому что `String` в Java **реализует интерфейс `Comparable<String>`**:
```java
public final class String
    implements java.io.Serializable, Comparable<String>, CharSequence {
    ...
    public int compareTo(String anotherString) { ... }
}
```
То есть у `String` уже есть определённый "естественный порядок" сравнения — **лексикографический** (по Unicode-кодам символов слева направо).
