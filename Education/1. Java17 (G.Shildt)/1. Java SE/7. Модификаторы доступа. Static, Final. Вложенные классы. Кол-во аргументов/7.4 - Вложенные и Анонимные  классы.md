В Java ==вложенные классы== (nested classes) — это ==классы, определённые **внутри другого класса**==

> [!NOTE] Доступ к внешним данным объемлющего класса
> ==Вложенный класс имеет доступ к членам (даже к private) внешнего класса==
> 
> ОДНАКО внешний класс, НЕ имеет доступа к членам класса внутреннего класса

# 📕 Виды вложенных классов

| Вид класса                               | Статический? | Есть доступ к членам внешнего класса? | Где используется?                               |
| ---------------------------------------- | ------------ | ------------------------------------- | ----------------------------------------------- |
| ==Статический== вложенный                | ✅ Да         | ❌ Нет (только к `static`)             | Вспомогательная логика                          |
| ==НЕстатический== вложенный (внутренний) | ❌ Нет        | ✅ Да                                  | Когда нужен доступ к состоянию внешнего объекта |
|                                          |              |                                       |                                                 |
| **Local Class**                          | ❌ Нет        | ✅ Да, если final/эфф. final           | В методах                                       |
| **Anonymous Class**                      | ❌ Нет        | ✅ Да                                  | Для однократного использования                  |
|                                          |              |                                       |                                                 |
***
## 📙 1. Static Nested Class — **статический вложенный класс**
###### 💡 Особенности:
- ==Может обращаться **только к== `static` членам** внешнего класса.
- ==**Не имеет ссылки на объект== `Outer`**.
- Создаётся ==без== объекта внешнего класса:

```java
class Outer {
    static class StaticNested {
        void display() {
            System.out.println("Static nested class");
        }
    }
}


Outer.StaticNested nested = new Outer.StaticNested();
```

***
## 📙 2. Inner Class — **нестатический вложенный класс**
📌 Особенности:
- ==Имеет **неявную ссылку на объект внешнего класса**==.
- ==Может обращаться ко всем полям/методам внешнего объекта==.
- Создаётся ==через== объект внешнего класса:
```java
class Outer {
    int x = 10;

    class Inner {
        void show() {
            System.out.println("x = " + x); // доступ к полям внешнего объекта
        }
    }
}

Outer outer1 = new Outer();
Outer.Inner inner = outer1.new Inner();
```

> [!NOTE] Ссылка на внешний класс
> В **нестатическом внутреннем классе** Java автоматически хранится **ссылка на объект внешнего класса**, к которому он принадлежит
> ```java
> ИмяВнешнегоКласса.this
> ```

***
## 📙 3. Local Class — **локальный класс внутри метода**
📌 Особенности:
- ==Видим только внутри метода.==
- Может обращаться **только к `final` или _эффективно final_ переменным метода**.
- Полезен для ограниченной логики, например, в обработчиках событий.
```java
void myMethod() {
    final int local = 100;

    class Local {
        void print() {
            System.out.println("Local var = " + local);
        }
    }

    Local l = new Local();
    l.print();
}
```

***
## 📙 4. Anonymous Class — **анонимный класс**
📌 Особенности:
- Не имеет имени.
- ==Используется для реализации интерфейса или подкласса **на месте**.==
- Очень часто ==используется в обработчиках событий и потоках==.
```java
Runnable r = new Runnable() {
    @Override
    public void run() {
        System.out.println("Anonymous class running");
    }
};
```
