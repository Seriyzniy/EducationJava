# 1. Мусор в Java
Мусор - объекты в памяти, которые больше не используются (на них больше нет ссылок)

```java
var mapper = new ObjectMapper();
// Выделение памяти, mapper добавлен в кучу

return mapper.writeValueAsString(serializableobject);
// mapper больше не используется, но все еще находится в куче
```

# 2. Теория поколений
Каждый вид GC использует теорию поколений при организации работы с памятью

🧠Категории объектов по сроку жизни
• 🟢Краткосрочные, используемые сразу после создания
	
• 🟠Со средним временем жизни, переходящие по методам
	
• 🔴 Долгожители, существующие все время выполнения

##### Разделение кучи
С учетом такого разделения, ==куча условно делится на== :
1. Младшее поколение (young generation)
2. Старшее поколение (old generation)

Соответственно, ==процессы сборки мусора== разделяются на
##### 🔹 Minor GC - для младшего поколения:
- Срабатывает в Young Generation (в Eden)
- **Быстро и часто**
- Выжившие объекты перемещаются в Survivor → затем в Old
##### 🔹 Major GC (или Full GC) - для обоих поколений:
- Затрагивает Old Generation (и иногда весь heap)
- **Медленно и редко**
- Может вызвать **Stop-The-World** паузу

# 3. Основные этапы работы GC

### 📌 Основные фазы:

1. **Маркировка (Marking)** — определяет, какие объекты **достижимы** (живы)
    
2. **Удаление (Sweeping)** — удаляет **недостижимые** объекты
    
3. **Компактирование (Compacting)** — сжимает память, чтобы избежать фрагментации
    
4. **Промоция (Promotion)** — выжившие объекты из Eden перемещаются в Survivor, затем в Old

## Как GC понимает что объект "мертвый"
После создания объекта при помощи оператора `new`, в стеке метода создаётся ссылка на объект, а сам объект размещается в куче, где состоит из заголовка (`Mark Word` и `Klass Pointer`) и полей экземпляра.

Когда область памяти (например, Eden или Old Generation) заполняется, JVM запускает сборку мусора.  
GC начинает обход с **GC Roots** — это локальные ==переменные в стеках== активных потоков, статические поля, JNI-ссылки и др.

GC переходит по этим ссылкам и **рекурсивно обходит все reachable объекты**, помечая их как **достижимые**.  
Объекты, **до которых GC не может добраться** от GC Roots, считаются **недостижимыми** и подлежат удалению.

# 4. Критерии для выбора GC
Вам быстро, дешево или качественно?

• ⏳Максимальная задержка - максимальное ==время==, на которое сборщик ==приостанавливает== выполнение программы ==для выполнения одной сборки==. 
Такие остановки называются stop the-world (или STW).

• Пропускная способность - отношение общего времени работы программы к общему времени простоя, вызванного сборкой мусора, на длительном промежутке времени.

• 🛠Потребляемые ресурсы - объем ресурсов процессора и/или дополнительной памяти, потребляемых сборщиком.
