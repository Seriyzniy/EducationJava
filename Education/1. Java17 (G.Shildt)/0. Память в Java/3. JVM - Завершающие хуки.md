## Что это такое

**Shutdown hook** — это специальный поток, который можно зарегистрировать в JVM, и он будет выполнен при завершении работы виртуальной машины.

Регистрируется через:
```java
Runtime.getRuntime().addShutdownHook(Thread hook)
```

Можно УДАЛИТЬ hook перед его запуском:
```java
Runtime.getRuntime().removeShutdownHook(Thread hook)
```
## Когда они запускаются
Хуки запускаются при следующих событиях:
1. **Нормальное завершение программы** — все потоки `main` и `non-daemon` закончили работу.
    
2. **Вызов `System.exit(int)`**.
    
3. **Внешний сигнал ОС** — например, `SIGTERM` (обычное завершение процесса в Linux/Unix).

⚠️ **Не запускаются**, если JVM убита «жёстко»:
- `kill -9` (SIGKILL),
    
- аварийный сбой JVM (native crash),
    
- выключение питания.

## Как они работают
- Каждый hook — это **поток**, выполняющийся параллельно с другими хуками.
    
- Хуки выполняются **до** фактического завершения JVM.
    
- JVM ждёт завершения всех хуков. Если hook зависнет → JVM зависнет при завершении.
    
- Порядок выполнения хуков **не гарантируется**.

### Пример
```java
public class ShutdownHookDemo {
    public static void main(String[] args) {
        Runtime.getRuntime().addShutdownHook(new Thread(() -> {
            System.out.println("Выполняем очистку ресурсов перед завершением...");
        }));

        System.out.println("Программа работает...");
        try {
            Thread.sleep(2000);
        } catch (InterruptedException ignored) {}
        System.out.println("Выходим через System.exit()");
        System.exit(0);
    }
}

Программа работает...
Выходим через System.exit()
Выполняем очистку ресурсов перед завершением...

```

## Для чего используют shutdown hooks
1. **Освобождение ресурсов** (закрытие файлов, сокетов, соединений с БД).
    
2. **Сохранение состояния** (сериализация данных в файл, дампы).
    
3. **Логирование причины завершения**.
    
4. **Остановка фоновых потоков и служб** (корректный shutdown серверов, пула потоков).
