# 1. Создай объект конфигурации и проверь метаданные по умолчанию
Иногда в билетах специально снимают флажки в метаданных прикладных объектов. 

Поэтому для проверки **создай прикладной объект** *который хочешь проверить*, **и проверь все флажки**

# 2. Всегда смотри ошибки через "Подробно"
Ошибки при отладке лучше смотреть через подробно, там более детальная инфа

# 3. Используй шаблоны Ctrl + Q при написании кода
Пишешь 
```
Процедура|
```
Где | жмешь Ctrl + Q, и по шаблону пишется процедура (название, параметры, и "КонецПроцедуры"). И так же в других базовых конструкциях

## Удобство для Сообщение Сообщить
Пишешь `Сообщ` Ctrl+Q, и появляется шаблон, с 
- Создание сообщения
- Заполнением текста, поля
- Вызов метода `сообщить()`

## 4.1 Использование "двух точек" в SQL - НЕЯВНОЕ СОЕДИНЕНИЕ
Если при составлении SQL запроса, получается обращение к данным, через другие данные, то это НЕЯВНОЕ СОЕДИНЕНИЕ

Например
![[Pasted image 20260308151809.png]]

Так как мы достаем данные из ***табличной части, которая содержит ссылку на таблицу "родителя"***, то если мы обращаемся через ссылку на родителя, к данным родителя, ТО ЭТО ==НЕЯВНОЕ СОЕДИНЕНИЕ==
```
РасходнаяНакладнаяСписокНоменклатуры.Ссылка.Склад КАК Склад
```
Это **ДВА неявных соединения**:
1. Табличная часть → Документ (обратились к родителю ТЧ)
    
2. Документ → Справочник "Склады" (от родителя обратились к Справочнику)


При этом, запись
```
РасходнаяНакладнаяСписокНоменклатуры.Номенклатура.Представление КАК НоменклатураПредставление
```
КОРРЕКТНА, так как здесь лишь одна связь

Это **ОДНО неявное соединение**:
1. Табличная часть → Справочник "Номенклатура"
