## **Зачем нужен регистр накопления**
Анализировать документы напрямую для получения итоговых данных — плохое решение по двум причинам:
1. **Сложность изменений.** При добавлении новых видов документов (например, перемещение материалов) придется переделывать все отчеты, которые анализируют исходные документы.
    
2. **Низкая производительность.** Отчеты, работающие напрямую с документами, будут работать медленно.


**Решение:** использовать **регистры** — специальные объекты конфигурации, которые накапливают данные в удобном для анализа виде. 
Документы поставляют в них информацию, а отчеты и алгоритмы уже читают готовые, быстро доступные данные из регистров. Это упрощает развитие конфигурации и ускоряет работу.

> [!IMPORTANT] Регистр содержит аккумулированную информацию

## **Что такое регистр накопления**

> [!IMPORTANT] Регистр - данные в разрезе определенных реквизитов
> Объект конфигурации `Регистр накопления` предназначен **для накопления числовых данных** ==В РАЗРЕЗЕ НЕСКОЛЬКИХ ИЗМЕРЕНИЙ==. 

На его основе платформа создаёт в базе данных:
- таблицы движений (записи, поступающие от документов) 
- и таблицу итогов (актуальные остатки на текущий момент).
в которых будут накапливаться данные, поставляемые различными объектами базы данных.

![[Pasted image 20260215085452.png]]
### **Ключевые особенности:**
- Не предназначен для ручного редактирования пользователем — *изменения вносятся* через алгоритмы других объектов (***обычно при проведении документов***).
    
- ==Состоит из== 
	- **измерений** (разрезы, например, товар, склад), 
		- *НА ПЕРЕСЕЧЕНИИ ЧЕГО* сохраняем данные
	- **ресурсов** (числовые данные, например, количество, сумма)
		- *КАКИЕ ДАННЫЕ* НАКАПЛИВАЕМ
	- и **реквизитов** (дополнительная информация о движении).
    
- Каждое движение (запись) содержит значения измерений, приращения ресурсов, направление (приход/расход) и ссылку на **регистратор** — документ, вызвавший это движение.

Таким образом, регистры накопления позволяют эффективно хранить и быстро получать итоговые данные (остатки, обороты), агрегированные по нужным разрезам. 

### ВИД РЕГИСТРА
![[Pasted image 20260210210129.png]]

### Очень важная деталь:  
**остатки не хранятся напрямую**, они _вычисляются_ по движениям (с оптимизациями внутри СУБД).

Поэтому:
- нельзя «поправить остаток» напрямую;
- нельзя обойти документ;
- всё идёт через движения.

# 1. **Добавление регистра накопления «ОстаткиМатериалов»**
Теперь, когда мы знаем, для чего предназначены регистры накопления, посмотрим, как можно их использовать в нашем примере. 

Прежде всего, нас интересует информация о том, сколько и каких материалов есть у нас на складах. Для накопления такой информации мы создадим регистр `ОстаткиМатериалов`.


Чтобы отслеживать количество материалов на складах, создадим регистр накопления **ОстаткиМатериалов**.

**Создание в конфигураторе:**
- Ветвь _Регистры накопления_ → кнопка _Добавить_.
    
- Имя: _ОстаткиМатериалов_.
    
- Расширенное представление списка: _Движения по регистру Остатки материалов_.
![[Pasted image 20260215090609.png]]

**Подсистемы:**  
Отмечаем _Учет материалов_, _Оказание услуг_, _Бухгалтерия_.

## **Структура регистра (закладка Данные):**
- **Измерения** (разрезы аналитики):
    - _Материал_ (тип _СправочникСсылка.Номенклатура_)
        
    - _Склад_ (тип _СправочникСсылка.Склады_)
        
- **Ресурс** (накапливаемое значение):
    - _Количество_ (Число, длина 15, точность 3)
![[Pasted image 20260215090844.png]]

Если запустить 1С:Предприятие сейчас, будет ошибка: «Ни один из документов не является регистратором». Это нормально — регистр пока не связан с документами, которые будут создавать в нём движения.

На следующем шаге сформируем движения регистра при проведении документов _ПриходнаяНакладная_ и _ОказаниеУслуг_.

## **Движения документа**
**Движения** — это записи в регистрах, которые создаются ==при проведении документа== и отражают изменения в учёте.

### **Настройка движений для документа «ПриходнаяНакладная»:**
1. Открыть редактирование документа _ПриходнаяНакладная_ → закладка _Движения_.
    
2. Раскрыть список _Регистры накопления_ → отметить регистр _ОстаткиМатериалов_.
![[Pasted image 20260215091204.png]]

3. Нажать **Конструктор движений**.
![[Pasted image 20260215091236.png]]

### **В конструкторе:**
Конструктор устроен просто. В списке `Регистры` перечислены регистры, в которых документ может создавать движения. В нашем случае там пока один регистр `ОстаткиМатериалов`. 

В списке `Реквизиты документа` должны находиться исходные данные для создания движений – реквизиты документа `ПриходнаяНакладная`. 

А в **таблице** `Поле – Выражение` ==должны быть заданы формулы==, по которым будут вычисляться значения измерений и ресурсов регистра при записи движений (рис. 6.8). 
![[Pasted image 20260215091536.png]]

Обратите внимание, что по умолчанию конструктор предлагает нам создавать движения прихода (***Тип движения регистра – Приход***, символ + рядом с названием регистра) по регистру `ОстаткиМатериалов`. 
Это нас вполне устраивает, ведь документ `ПриходнаяНакладная` и должен приходовать материалы.

В **поле выбора** `Табличная часть` ***выберем табличную часть нашего документа*** – `Материалы`. Список реквизитов документа, который уже заполнен реквизитами шапки документа, автоматически дополнится реквизитами нашей табличной части.
![[Pasted image 20260215091749.png]]

Итак, установим следующие настройки в конструкторе
- Тип движения: _Приход_ (по умолчанию).
- Табличная часть: _Материалы_.
- Кнопка _Заполнить выражения_ — автоматически устанавливаются соответствия:
    - _Материал_ ← Материал (из строки табличной части)
    - _Склад_ ← Склад (из шапки документа)
    - _Количество_ ← Количество (из строки табличной части)
![[Pasted image 20260215091845.png]]


Нажать _ОК_ — конструктор **создаст в модуле документа процедуру** _ОбработкаПроведения()_ с готовым кодом формирования движений.
![[Pasted image 20260215092007.png]]

#### **Что происходит в коде:**
##### Свойство `Движения` у Документа - коллекция регистров
Объект встроенного языка `ДокументОбъект` **имеет свойство** `Движения`.
**Оно возвращает объект** `КоллекцияДвижений`, содержащий коллекцию наборов записей регистров, по которым этот документ может формировать движения. 
> Через свойство `Движения` можно обратиться к коллекции Регистров

К конкретному набору записей этой коллекции можно обратиться, указав через точку имя регистра, которому принадлежит этот набор записей. Например, `Движения.ОстаткиМатериалов`. 

##### Методы самого регистра
Затем через точку можно использовать различные методы набора записей регистра, например, `Движения.ОстаткиМатериалов.Добавить()`. Метод `Добавить()` добавляет новую запись в набор записей.
> Метод `Добавить()` у регистра, позволяет записать запись в регистр

В первой строке процедуры мы устанавливаем свойство `Записывать` набора записей регистра в значение `Истина`. 
> То есть в явном виде указываем, что после завершения **обработки проведения** платформа должна будет записать этот набор записей в базу данных. 

##### Цикл обработки каждой записи
Внутри обработчика расположен цикл Для Каждого … Из … Цикл. Он предназначен для перебора строк табличной части нашего документа. 

В цикле обращение к табличной части документа происходит по имени (Материалы). Переменная `ТекСтрокаМатериалы` содержит объект с данными текущей строки табличной части документа. Эта переменная создается в начале цикла и меняется по мере его прохождения. 

В первой строке тела цикла, используя метод `Добавить()`, мы добавляем к набору записей, который создает наш документ в регистре, новую запись. 
> Тем самым мы создаем объект `РегистрНа копленияЗапись` и сохраняем его в переменной Движение. 

Используя этот объект, мы можем обратиться к полям этой записи, указав имя поля через точку от этой переменной (например, Движение.Количество).



В результате документ _ПриходнаяНакладная_ становится **регистратором** для регистра _ОстаткиМатериалов_ (это видно на закладке _Регистраторы_ регистра).
