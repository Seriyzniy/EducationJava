# 3. Вывод данных по всем дням в выбранном периоде 
Следующий отчет, который мы добавим, будет называться Выручка мастеров. 
Он будет содержать информацию о том, какая выручка была получена ООО «На все руки мастер» благодаря работе каждого из мастеров, с детализацией по всем дням в выбранном периоде и разворотом по клиентам, обслуженным в каждый из дней
![[Pasted image 20260223195240.png]]

На примере этого отчета мы проиллюстрируем, **как строить многоуровневые группировки в запросе** и как обходить все даты в выбранном периоде. 

Также продемонстрируем настройку отдельных элементов структуры отчета, научимся выводить данные в диаграмму и создавать несколько вариантов отчета в конфигураторе

---
## **В конфигураторе:**
1. Создать новый отчет **ВыручкаМастеров** → открыть конструктор схемы компоновки данных.
    
2. Добавить _Набор данных – запрос_ → запустить конструктор запроса.
    
3. Источник данных: виртуальная таблица регистра накопления _Продажи.Обороты_.

### **Параметры виртуальной таблицы:**
- Выделить таблицу → нажать _Параметры виртуальной таблицы_.
![[Pasted image 20260223195827.png]]

- Установить **Периодичность = День** — данные будут сгруппированы по дням.
![[Pasted image 20260223195901.png]]

### **Выбранные поля:**
- _ПродажиОбороты.Мастер_
    
- _ПродажиОбороты.Период_
    
- _ПродажиОбороты.Клиент_
    
- _ПродажиОбороты.ВыручкаОборот_
    

### **Псевдонимы:**  
На закладке _Объединения/Псевдонимы_ для поля _ПродажиОбороты.ВыручкаОборот_ задать псевдоним **Выручка**.

Далее будут настраиваться многоуровневые группировки и вывод данных в диаграмму.

### 🔥Анализ текста запроса🔥
Нажмем ОК и рассмотрим текст запроса, сформированный конструк тором (листинг 13.10). 

Листинг 13.10. Текст запроса 
```
ВЫБРАТЬ 
	ПродажиОбороты.Мастер КАК Мастер, 
	ПродажиОбороты.Период КАК Период, 
	ПродажиОбороты.Клиент КАК Клиент, 
	ПродажиОбороты.ВыручкаОборот КАК Выручка 
ИЗ 
	РегистрНакопления.Продажи.Обороты( , , День, ) КАК ПродажиОбороты 
```

В части описания запроса обратите внимание, что **у источника данных задана периодичность** выбираемых данных – День (листинг 13.11).
Листинг 13.11. Задание периодичности виртуальной таблицы 
```
ИЗ 
	РегистрНакопления.Продажи.Обороты( , , День, ) КАК ПродажиОбороты 
```
> **Именно благодаря этому** у нас появляется **ВОЗМОЖНОСТЬ ОПИСАТЬ среди выбранных полей поле `Период`**.

---
### **Ресурсы**
На закладке _Ресурсы_ нажать кнопку подбора (`>>`) — конструктор автоматически выберет ресурс _Выручка_.
![[Pasted image 20260223200412.png]]
### **Параметры**
На закладке _Параметры_:
- Для _НачалоПериода_ и _КонецПериода_ в поле _Тип_ установить состав даты = _Дата_ (без времени).
    
- Для _КонецПериода_ задать выражение: `КонецПериода(&КонецПериода, "День")`.
![[Pasted image 20260223200547.png]]

### **Настройки (структура отчета)**
Создаем иерархию группировок:
1. Корневой элемент _Отчет_ → добавить группировку по полю **Мастер**.
    
2. В группировку _Мастер_ добавить вложенную группировку по полю **Период**.
    
3. В группировку _Период_ добавить вложенную группировку **Детальные записи** (без поля).

4. На закладке _Выбранные поля_ для детальных записей выбрать поля _Клиент_ и _Выручка_ 
	>(поля _Мастер_ и _Период_ выводятся автоматически как группировочные).
![[Pasted image 20260223200837.png]]

---
#### **Другие настройки:**
##### _Расположение полей группировок_ = _Отдельно и только в итогах_
(каждая группировка **в отдельной области слева направо**, наименование только в самой группировке).
![[Pasted image 20260223200924.png]]

По умолчанию поля группировок в отчете располагаются вертикально друг под другом
![[Pasted image 20260223201122.png]]

Установка этого свойства в значение `Отдельно и только в итогах` означает, что **каждая группировка будет располагаться в отдельной области отчета слева направо** и ее наименование будет выводиться только в данной группировке
![[Pasted image 20260223201215.png]]

---
##### _Расположение общих итогов по вертикали_ = _Начало_
(общие итоги отображаются вверху, перед строками группировки).
![[Pasted image 20260223201002.png]]

---
##### _Заголовок_ = _Выручка мастеров_.
![[Pasted image 20260223201022.png]]

---
##### Параметры _Начало периода_ и _Конец периода_ 
включить в быстрые пользовательские настройки (будут отображаться в форме отчета).

---

**Подсистемы:**  
Закрыть конструктор, на закладке _Подсистемы_ отметить _Оказание услуг_ и _Расчет зарплаты_. Ссылка на отчет появится в панели функций этих разделов.

## Проверка отчета «Выручка мастеров» в 1С:Предприятии
В разделах _Оказание услуг_ и _Расчет зарплаты_ в подменю _Отчеты_ появилась команда _Выручка мастеров_.

При выполнении команды открывается форма отчета с быстрыми настройками периода. Зададим, например, период с 12.10.2022 по 16.10.2022 и сформируем отчет.

Результат отчета отображается в соответствии с настроенной структурой:
- группировка по мастерам;
    
- внутри — по дням;
    
- детализация по клиентам и суммам выручки.

![[Pasted image 20260223201714.png]]

> Если период не важен, пользователь может снять флажок использования параметров — отчет сформируется по всем записям регистра _Продажи_ (в данном примере результат будет тем же).

## Вывод всех дат в выбранном периоде
> [!NOTE] `СтандартныйПериод` в параметрах запроса и с доступностью пользователю
> Используй `СТАНДАРТНЫЙ ПЕРИОД` как параметр и все

Если вы помните, в начале раздела мы говорили, что этот отчет должен показывать данные с детализацией по всем дням в выбранном периоде. 

У нас же отображаются только те дни, для которых существуют ненулевые записи в таблице регистра накопления Продажи. 

Для детализации данных в отчете система компоновки данных позволяет указывать для группировок дополнение периодов с заданной периодичностью в указанном интервале. 

Поэтому сейчас мы изменим настройки отчета таким образом, чтобы в отчет попадала каждая дата из периода, за который сформирован отчет.
### В режиме «Конфигуратор»
1. Открыть схему компоновки данных отчёта _ВыручкаМастеров_ → закладка _Настройки_.
    
2. В структуре отчёта выделить группировку **Период**.
    
3. В средней части окна нажать кнопку с именем этой группировки (_Период_), чтобы перейти к её настройкам.
	![[Pasted image 20260224195413.png]]

4. На закладке **Поля группировки** для поля _Период_ установить **Тип дополнения = День** — это укажет, что для этой группировки существующие записи с ненулевым значением ресурса будут дополняться записями для каждого из дней.
	![[Pasted image 20260224195520.png]]

5. В полях **Начальная дата периода** и **Конечная дата периода** вместо фиксированных дат нужно указать параметры отчёта:
    - Очистить поле, нажать кнопку выбора типа → выбрать **Поле компоновки данных**.
        ![[Pasted image 20260224200305.png]]
        ![[Pasted image 20260224200329.png]]
    
	- В открывшемся окне выбрать параметр **НачалоПериода**.
		![[Pasted image 20260224200533.png]]
    
    - Аналогично для конечной даты выбрать параметр **КонецПериода**.
        ![[Pasted image 20260224200640.png]]

### **В 1С:Предприятии:**
Сформировать отчёт за период, например, с 12.10.2022 по 16.10.2022. Теперь в отчёте отображаются **все дни** этого периода, в том числе и те, по которым не было оказано услуг.

## Новый вариант отчета — диаграмма
Для наглядного анализа работы мастеров (например, для начисления зарплаты) создадим другой вариант отчета _ВыручкаМастеров_, представляющий данные в виде диаграммы.

### **Диаграмма в системе компоновки данных:**
Диаграмма предназначена для размещения в таблицах и формах диаграмм и графиков различного вида.

Логически она состоит из:
- **Точек** — *объекты или моменты*, для которых получаются значения;
    
- **Серий** — *характеристики*, значения которых *анализируются*;
    
- **Значений** — ***данные на пересечении серии и точки***.
	![[Pasted image 20260224201238.png]]

Например, в диаграмме продаж по месяцам:
- точки — месяцы;
    
- серии — виды номенклатуры;
    
- значения — обороты продаж.

Как объект, диаграмма имеет три области оформления: 
- **область построения**, 
- **область заголовка**, 
- **область легенды**.
![[Pasted image 20260224201457.png]]

В следующем варианте отчёта диаграмма будет добавлена как отдельный элемент структуры настроек схемы компоновки данных.

### **В конфигураторе:**
1. Открыть схему компоновки данных отчета _ВыручкаМастеров_ → закладка _Настройки_.
    
2. В левой части (==список вариантов==) нажать _Добавить_:
    - Имя варианта: _ОбъемВыручки_
        
    - Представление: _Объем выручки_
    ![[Pasted image 20260224201734.png]]
    
3. Структура и настройки очистились — теперь они относятся к новому варианту. Данные для отчета остались те же (общий запрос).


#### **Создание диаграммы:**
- Корневой элемент _Отчет_ → контекстное меню → добавить **Диаграмму**.
    ![[Pasted image 20260224201853.png]]
    
- В ветке _Точки_ добавить группировку по полю **Мастер** (серии не задаем — ==для измерительной диаграммы достаточно точек==).
    ![[Pasted image 20260224202135.png]]
    
- На закладке _Выбранные поля_ выбрать ресурс **Выручка** (обязательно, иначе ошибка).
    ![[Pasted image 20260224202252.png]]

**Настройка диаграммы (закладка _Другие настройки_):**
- Тип диаграммы = **Измерительная**.
    ![[Pasted image 20260224202349.png]]

- В свойстве _Информационные интервалы значений_ добавить три интервала:
    - Плохо, Хорошо, Отлично — с указанием границ и цветов.
      ![[Pasted image 20260224202746.png]]
    
**Пользовательские настройки:**
- Параметры _Начало периода_ и _Конец периода_:
    - Включить в пользовательские настройки.
        
    - Режим редактирования = _Быстрый доступ_.
        
    - Признак использования в отчете **не включать**.
        
- _Важно:_ пользовательские настройки для каждого варианта отчета задаются отдельно.

---
### **Проверка нового варианта отчета в 1С:Предприятии**
В разделе _Расчет зарплаты_ выполнить команду _Отчеты → Выручка мастеров_.

В открывшейся форме отчета нажать кнопку **Выбрать вариант**. В списке теперь доступны два варианта:
- **Основной** (табличный)
    
- **Объем выручки** (диаграмма)

Выбрать вариант _Объем выручки_ → нажать _Выбрать_ → _Сформировать_.

![[Pasted image 20260224203234.png]]

**Результат:**
- Те же данные, что и в основном варианте, представлены в виде **измерительной диаграммы**.
    
- На диаграмме наглядно видна доля каждого мастера в общей выручке.
    
- При наведении на стрелку диаграммы появляется подсказка.


**Отчетный период:**  
В данном примере период не задавался, поэтому в отчет попали все данные. В реальной работе, когда документов много, следует указывать период (например, месяц), чтобы анализировать данные за нужный интервал.

**Переключение между вариантами:**
- Для просмотра детализации по дням и клиентам достаточно выбрать **Основной** вариант и снова сформировать отчет.
    

Таким образом, разные варианты одного отчета позволяют представить одни и те же данные в наиболее удобной форме для различных задач.
