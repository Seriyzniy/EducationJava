# 3. Вывод данных по всем дням в выбранном периоде 
Следующий отчет, который мы добавим, будет называться Выручка мастеров. 
Он будет содержать информацию о том, какая выручка была получена ООО «На все руки мастер» благодаря работе каждого из мастеров, с детализацией по всем дням в выбранном периоде и разворотом по клиентам, обслуженным в каждый из дней
![[Pasted image 20260223195240.png]]

На примере этого отчета мы проиллюстрируем, **как строить многоуровневые группировки в запросе** и как обходить все даты в выбранном периоде. 

Также продемонстрируем настройку отдельных элементов структуры отчета, научимся выводить данные в диаграмму и создавать несколько вариантов отчета в конфигураторе

---
## **В конфигураторе:**
1. Создать новый отчет **ВыручкаМастеров** → открыть конструктор схемы компоновки данных.
    
2. Добавить _Набор данных – запрос_ → запустить конструктор запроса.
    
3. Источник данных: виртуальная таблица регистра накопления _Продажи.Обороты_.

### **Параметры виртуальной таблицы:**
- Выделить таблицу → нажать _Параметры виртуальной таблицы_.
![[Pasted image 20260223195827.png]]

- Установить **Периодичность = День** — данные будут сгруппированы по дням.
![[Pasted image 20260223195901.png]]

### **Выбранные поля:**
- _ПродажиОбороты.Мастер_
    
- _ПродажиОбороты.Период_
    
- _ПродажиОбороты.Клиент_
    
- _ПродажиОбороты.ВыручкаОборот_
    

### **Псевдонимы:**  
На закладке _Объединения/Псевдонимы_ для поля _ПродажиОбороты.ВыручкаОборот_ задать псевдоним **Выручка**.

Далее будут настраиваться многоуровневые группировки и вывод данных в диаграмму.

### 🔥Анализ текста запроса🔥
Нажмем ОК и рассмотрим текст запроса, сформированный конструк тором (листинг 13.10). 

Листинг 13.10. Текст запроса 
```
ВЫБРАТЬ 
	ПродажиОбороты.Мастер КАК Мастер, 
	ПродажиОбороты.Период КАК Период, 
	ПродажиОбороты.Клиент КАК Клиент, 
	ПродажиОбороты.ВыручкаОборот КАК Выручка 
ИЗ 
	РегистрНакопления.Продажи.Обороты( , , День, ) КАК ПродажиОбороты 
```

В части описания запроса обратите внимание, что **у источника данных задана периодичность** выбираемых данных – День (листинг 13.11).
Листинг 13.11. Задание периодичности виртуальной таблицы 
```
ИЗ 
	РегистрНакопления.Продажи.Обороты( , , День, ) КАК ПродажиОбороты 
```
> **Именно благодаря этому** у нас появляется **ВОЗМОЖНОСТЬ ОПИСАТЬ среди выбранных полей поле `Период`**.

---
### **Ресурсы**
На закладке _Ресурсы_ нажать кнопку подбора (`>>`) — конструктор автоматически выберет ресурс _Выручка_.
![[Pasted image 20260223200412.png]]
### **Параметры**
На закладке _Параметры_:
- Для _НачалоПериода_ и _КонецПериода_ в поле _Тип_ установить состав даты = _Дата_ (без времени).
    
- Для _КонецПериода_ задать выражение: `КонецПериода(&КонецПериода, "День")`.
![[Pasted image 20260223200547.png]]

### **Настройки (структура отчета)**
Создаем иерархию группировок:
1. Корневой элемент _Отчет_ → добавить группировку по полю **Мастер**.
    
2. В группировку _Мастер_ добавить вложенную группировку по полю **Период**.
    
3. В группировку _Период_ добавить вложенную группировку **Детальные записи** (без поля).

4. На закладке _Выбранные поля_ для детальных записей выбрать поля _Клиент_ и _Выручка_ 
	>(поля _Мастер_ и _Период_ выводятся автоматически как группировочные).
![[Pasted image 20260223200837.png]]

---
#### **Другие настройки:**
##### _Расположение полей группировок_ = _Отдельно и только в итогах_
(каждая группировка **в отдельной области слева направо**, наименование только в самой группировке).
![[Pasted image 20260223200924.png]]

По умолчанию поля группировок в отчете располагаются вертикально друг под другом
![[Pasted image 20260223201122.png]]

Установка этого свойства в значение `Отдельно и только в итогах` означает, что **каждая группировка будет располагаться в отдельной области отчета слева направо** и ее наименование будет выводиться только в данной группировке
![[Pasted image 20260223201215.png]]

---
##### _Расположение общих итогов по вертикали_ = _Начало_
(общие итоги отображаются вверху, перед строками группировки).
![[Pasted image 20260223201002.png]]

---
##### _Заголовок_ = _Выручка мастеров_.
![[Pasted image 20260223201022.png]]

---
##### Параметры _Начало периода_ и _Конец периода_ 
включить в быстрые пользовательские настройки (будут отображаться в форме отчета).

---

**Подсистемы:**  
Закрыть конструктор, на закладке _Подсистемы_ отметить _Оказание услуг_ и _Расчет зарплаты_. Ссылка на отчет появится в панели функций этих разделов.

---
## Проверка отчета «Выручка мастеров» в 1С:Предприятии
В разделах _Оказание услуг_ и _Расчет зарплаты_ в подменю _Отчеты_ появилась команда _Выручка мастеров_.

При выполнении команды открывается форма отчета с быстрыми настройками периода. Зададим, например, период с 12.10.2022 по 16.10.2022 и сформируем отчет.

Результат отчета отображается в соответствии с настроенной структурой:
- группировка по мастерам;
    
- внутри — по дням;
    
- детализация по клиентам и суммам выручки.

![[Pasted image 20260223201714.png]]

> Если период не важен, пользователь может снять флажок использования параметров — отчет сформируется по всем записям регистра _Продажи_ (в данном примере результат будет тем же).
