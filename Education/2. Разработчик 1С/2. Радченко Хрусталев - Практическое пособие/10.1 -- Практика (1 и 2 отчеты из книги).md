# 1. Выбор данных из одной таблицы
**Создание отчета "Реестр документов оказание услуги"**

Создадим отчет с помощью системы компоновки данных, который выводит список всех документов _ОказаниеУслуги_, упорядоченных по дате и номеру.

## **В конфигураторе:**
1. Добавить новый объект конфигурации **Отчет**.
    - Имя: _РеестрДокументовОказаниеУслуги_
        
    - Расширенное представление: _Список оказанных услуг_
![[Pasted image 20260223164338.png]]

2. Нажать кнопку **Открыть схему компоновки данных** (или кнопку с лупой).
    
3. В конструкторе макета нажать **Готово**.
    
4. В конструкторе схемы компоновки данных создать **Набор данных – запрос** и нажать **Конструктор запроса**.
![[Pasted image 20260223164559.png]]
![[Pasted image 20260223164725.png]]

### **Составление запроса:**
- **Источник данных:** объектная (ссылочная) таблица документа _ОказаниеУслуги_.
    
- **Выбрать поля:** _Склад_, _Мастер_, _Клиент_, _Ссылка_.
![[Pasted image 20260223164810.png]]

**Псевдонимы полей (закладка Объединения/Псевдонимы):**
- Для поля _Ссылка_ установить псевдоним **Документ** (так удобнее в отчете).
![[Pasted image 20260223164854.png]]

> СОВЕТ 
> ***Имена полей лучше изменять в запросе,*** так как в этом случае в схему компоновки данных они перенесутся сразу в три колонки: Поле, Путь и Заголовок, и не нужно будет лишний раз их изменять.


**Порядок записей (закладка Порядок):**
- Упорядочить результат по полю _Документ_.

---
### Настройки отчета
На закладке **Настройки** создаются ==стандартные НАСТРОЙКИ, ОПРЕДЕЛЯЮЩИЕ ВЫВОД ИНФОРМАЦИИ в отчет==. 

#### Структура ОТЧЕТА
Иерархическая структура отчета может содержать три основных элемента:
- **Группировка** — для вывода в виде линейного отчета;
    
- **Таблица** — для вывода в виде таблицы;
    
- **Диаграмма** — для вывода в виде диаграммы.


**Добавление группировки:**
1. В дереве структуры отчета выделить корневой элемент **Отчет**.
    
2. Вызвать контекстное меню → **Новая группировка** (или нажать Ins).
    
3. В окне выбора поля группировки просто нажать **ОК** — это означает, что в группировке будут выводиться `детальные записи`.
    
4. В структуре отчета появится группировка **Детальные записи**.
![[Pasted image 20260223171439.png]]

**Выбор полей для отчета:**  
На закладке **Выбранные поля** перенести из списка доступных полей нужные:
- _Документ_
    
- _Склад_
    
- _Мастер_
    
- _Клиент_

![[Pasted image 20260223171536.png]]

_Добавление полей:_ перетаскиванием, двойным щелчком или кнопкой _Добавить_. Порядок полей можно изменить кнопками _Вверх/Вниз_ или перетаскиванием.

**Завершение настройки:**
- Окно настроек должно соответствовать рисунку.
    
- Закрыть конструктор схемы компоновки данных.


**Подсистемы:**  
В окне редактирования отчета на закладке **Подсистемы** отметить подсистему _Оказание услуг_. Ссылка на отчет автоматически появится в панели функций этого раздела (подменю _Отчеты_).

![[Pasted image 20260223171828.png]]

Таким образом, отчет демонстрирует использование конструктора схемы компоновки данных и основных конструкций языка запросов.

# 2. Выбор данных из двух таблиц
**Отчет «Рейтинг услуг»**
Отчет показывает, какие услуги принесли наибольшую прибыль за указанный период.

## **В конфигураторе:**
1. Создать новый отчет _РейтингУслуг_ → открыть конструктор схемы компоновки данных.
    
2. Добавить _Набор данных – запрос_ → запустить конструктор запроса.

### **Запрос:**
- **Источники:** таблица _Номенклатура_ (переименовать в _спрНоменклатура_) и виртуальная таблица _Продажи.Обороты_.
    
- **Поля:** _спрНоменклатура.Ссылка_, _ПродажиОбороты.ВыручкаОборот_.
![[Pasted image 20260223172726.png]]

### **Связи (закладка Связи):**
- Связь по полю _Номенклатура_.
    
- Тип связи — **Левое соединение**: флажок _Все_ у таблицы _спрНоменклатура_ (все услуги), снять флажок у _ПродажиОбороты_ (только те, что есть в оборотах). В результат попадут все услуги, а обороты — только для тех, что оказывались в периоде.
![[Pasted image 20260223172748.png]]
### **Условия (закладка Условия):**
- _спрНоменклатура.ЭтоГруппа = ЛОЖЬ_ — исключить группы.
    
- Перетащить _ВидНоменклатуры_ — автоматически создастся условие _ВидНоменклатуры = &ВидНоменклатуры_. При выполнении отчёта в параметр будет передаваться значение _Услуга_.
![[Pasted image 20260223172821.png]]  

> _СОВЕТ:_ 
> - **отбор** лучше делать **в запросе** (если данные точно не понадобятся), 
> - а **сортировку и группировку** — **в настройках отчёта** для большей гибкости.

Создание запроса завершено. Нажать **ОК** для возврата в конструктор схемы компоновки данных.

### Ресурсы 
В нашем отчете мы хотим видеть итоговые значения выручки для каждой услуги. Для этого нам нужно определить поля ресурсов отчета.

> [!NOTE] 
> **Ресурсы** в системе компоновки данных — это ==поля, значения которых вычисляются на основе детальных записей== *внутри группировки*. 
> 
> Они представляют собой групповые или общие итоги отчета.

**Настройка ресурсов:**
1. Перейти на закладку **Ресурсы**.
    
2. Нажать кнопку **Подобрать доступные** (или аналогичную) — конструктор автоматически выберет все поля, по которым можно вычислять итоги.
    
3. В нашем случае доступен единственный ресурс — **Выручка**.
    
4. Платформа автоматически предложит для него агрегатную функцию **Сумма** (именно это нам и нужно).

![[Pasted image 20260223174455.png]]

Таким образом, для каждой услуги будет рассчитана итоговая сумма выручки.

### Параметры
*Пользователя обычно интересуют* данные *за определенный период*, поэтому в отчете используются параметры, задающие начало и конец отчетного периода. 
> Параметры задают условия отбора записей.

**Закладка Параметры** в схеме компоновки данных:
- Здесь отображаются все параметры — как явно указанные в запросе (_ВидНоменклатуры_), так и параметры виртуальных таблиц (_НачалоПериода_, _КонецПериода_), которые система компоновки данных добавила автоматически, проанализировав текст запроса.
#### **Настройка параметров периода:**
По умолчанию параметры даты содержат время с точностью до секунды, что неудобно для пользователя. 

Учитываем две особенности:
1. **Избавляем пользователя от ввода времени.**  
    Для параметра _НачалоПериода_ изменяем тип данных: устанавливаем состав даты = _Дата_ (без времени).
![[Pasted image 20260223174933.png]]

2. **Корректируем окончание периода.**  
    Если пользователь задаст период с 01.06 по 11.06, расчет будет выполнен с начала 01.06 (00:00:00) по начало 11.06 (00:00:00) — данные за 11 июня не войдут в отчет.  
    
    Решение:
    - Для параметра _КонецПериода_ ***устанавливаем флажок*** **Ограничение доступности** (пользователь его не увидит)
    ![[Pasted image 20260223175131.png]]
        
    - Добавляем новый параметр **ДатаОкончания** (тип _Дата_, состав даты = _Дата_), доступный пользователю.
        
    - В выражении для _КонецПериода_ прописываем формулу на языке выражений СКД:  
        `КонецПериода(&ДатаОкончания, "День")`  
        — теперь _КонецПериода_ будет автоматически установлен на конец указанного пользователем дня.
    ![[Pasted image 20260223175455.png]]

> _ПРИМЕЧАНИЕ_ 
> Подробное описание языка выражений системы компоновки данных содержится во встроенной справке системы в разделе 
> `Справка > Содержание справки > Система компоновки данных > Язык выражений системы компоновки данных`.

**Настройка параметра ВидНоменклатуры:**
- Параметр должен быть скрыт от пользователя (флажок _Ограничение доступности_ установлен по умолчанию).
    
- В ячейке _Значение_ выбираем нужное значение перечисления — _Услуга_. Таким образом, отчёт всегда формируется только по услугам.
![[Pasted image 20260223184337.png]]

### Настройки отчета
**Структура отчета (закладка Настройки):**
- Добавить **группировку** (поле группировки не указывать — будут выводиться детальные записи).
    
- На закладке **Выбранные поля** указать поля _Услуга_ и _Выручка_.
![[Pasted image 20260223184509.png]]
 

**Другие настройки:**
- На закладке **Другие настройки** задать заголовок отчета — _Рейтинг услуг_.
    

_Примечание:_ при изменении параметров, требующих выбора значения, нужно дважды щелкнуть в поле _Значение_, нажать кнопку выбора и выбрать нужный вариант. Флажок использования значения появится автоматически (можно снять или установить вручную).

#### **Быстрые пользовательские настройки**
Чтобы пользователь мог задавать отчетный период перед формированием отчета, параметры _Дата начала_ и _Дата окончания_ должны быть включены в пользовательские настройки и находиться непосредственно в форме отчета.

На закладке **Параметры**:
- Выделить параметр → нажать **Свойства элемента пользовательских настроек**.
![[Pasted image 20260223184725.png]]

- Установить флажок **Включать в пользовательские настройки**.
![[Pasted image 20260223184745.png]]

- **Режим редактирования** = _Быстрый доступ_ (настройка отображается прямо в форме отчета).

> _ПОЯСНЕНИЕ:_
> - _Включать в пользовательские настройки_ — **настройка доступна в отдельном окне** (меню _Еще → Настройки_).
> 
> - _Быстрый доступ_ — **настройка автоматически выводится в форму отчета** для постоянного использования.
    

**НАЧАЛЬНЫЕ значения параметров даты:**
- Для параметра _Дата начала_ установить _Начало этого месяца_.
    
- Для _Дата окончания_ — _Начало этого дня_.  
    При каждом открытии отчета период будет по умолчанию показывать данные с начала текущего месяца по сегодняшний день.

Таким образом, при выполнении отчета даты начала и окончания отчетного периода будут динамически меняться и показывать период с начала текущего месяца по сегодняшнее число, и *пользователю, возможно, не придется менять их вручную.*

![[Pasted image 20260223185045.png]]

**Подсистемы:**  
Закрыть конструктор, в окне редактирования отчета на закладке _Подсистемы_ отметить _Оказание услуг_ и _Бухгалтерия_. Ссылка на отчет появится в панели функций этих разделов.

## **Проверка отчета «Рейтинг услуг» в 1С:Предприятии**
В разделах _Оказание услуг_ и _Бухгалтерия_ в подменю _Отчеты_ появилась команда _Рейтинг услуг_.

При открытии отчета отображаются параметры периода (по умолчанию — с начала месяца по текущую дату). При необходимости даты можно изменить через календарь.

После нажатия _Сформировать_:
- Выручка из всех трёх документов _Оказание услуги_ попадает в отчёт, так как их даты входят в период.
    
- Вверху выводится заданный заголовок и параметры периода.
    
- При прокрутке шапка отчёта остаётся зафиксированной (автоматически). Фиксацией строк и столбцов можно управлять параметрами _ФиксацияСлева_ и _ФиксацияСверху_.
    
- Названия услуг отображаются с видом номенклатуры согласно заданному представлению ссылок (см [[8. - Перечисления и изменение Основного представления справочника (наименования)]]).
 ![[Pasted image 20260223190608.png]]

**Проверка границы периода:**  
При изменении даты окончания на 14.10.2022 данные за 14 октября (документ № 1) попадают в отчёт. Благодаря функции _КонецПериода()_ последнее число периода включается корректно.


**Особенность:**  
Из-за левого соединения таблицы номенклатуры с регистром продаж в отчёте отображаются все услуги, даже те, по которым не было продаж.

## **Настройки в конфигураторе и в режиме 1С:Предприятие**
На примере отчёта «Рейтинг услуг» покажем создание настроек ==Условное оформление== и ==Отбор==. 

Действия можно выполнять как в конфигураторе, так и в режиме 1С:Предприятие (команда _Еще → Изменить вариант..._), где открывается окно настроек, похожее на закладку _Настройки_ в схеме компоновки данных.

### **Разница между настройками:**
- **Стандартные настройки** (создаются в конфигураторе) — сохраняются в схеме компоновки, являются частью конфигурации. Все пользователи видят отчёт в таком виде.
    
- **Настройки в 1С:Предприятии** — не являются частью конфигурации, доступны только конкретному пользователю, который их выполнил.

> _Примечание:_ 
> Возможность изменения варианта отчёта в 1С:Предприятии предназначена не для рядовых пользователей (для них — быстрые и пользовательские настройки), а для разработчиков, администраторов или опытных пользователей.
> 
> Настройки, сделанные в 1С:Предприятии, «перекрывают» стандартные. Вернуться к ним можно командой _Еще → Установить стандартные настройки_.

В данном случае мы настраиваем отчёт для всех пользователей (в конфигураторе). Но при необходимости аналогичные действия можно выполнить прямо в рабочей базе, не меняя конфигурацию.

## Условное оформление отчета «Рейтинг услуг»

### **В конфигураторе:**
На закладке _Настройки_ схемы компоновки данных перейти к нижней закладке _Условное оформление_ и добавить новое условие.
1. **Оформление:** нажать кнопку выбора в поле _Оформление_ → установить красный цвет текста.
![[Pasted image 20260223191540.png]]

2. **Условие:** нажать кнопку выбора в поле _Условие_ → добавить новый элемент отбора:
    - Левое значение: _Выручка_
        
    - Вид сравнения: _Меньше_
        
    - Правое значение: *700* (В НАШЕМ СЛУЧАЕ установим не 700 а 2100 для наглядности)
![[Pasted image 20260223191458.png]]

2. **Оформляемые поля:** можно оставить пустым (тогда выделяется вся строка красным) или выбрать поля _Услуга_ и _Выручка_.
    
3. **Представление:** задать как _Непопулярная услуга_ (это название увидит пользователь в настройках).

**Добавление в пользовательские настройки:**
- Нажать кнопку _Свойства элемента пользовательских настроек_.
![[Pasted image 20260223191847.png]]

- Установить флажок _Включать в пользовательские настройки_.
    
- Режим редактирования: _Обычный_ (настройка будет доступна в отдельном окне, не в форме отчета).

### **В 1С:Предприятии:**

## Пользовательские настройки отчета

### **В конфигураторе:**
На закладке _Настройки_ схемы компоновки данных нажать кнопку **Свойства элемента пользовательских настроек** (вверху командной панели). 

![[Pasted image 20260223193636.png]]

В открывшемся окне:
- Установить признак использования для настроек **Отбор** и **Условное оформление**.
    
- Режим редактирования для обеих настроек установить в значение _Обычный_ (доступны в отдельном окне по команде _Настройки…_).

![[Pasted image 20260223193729.png]]

---
### **Настройка отбора:**  

На закладке _Отбор_ (внизу окна настроек) раскрыть поле _Услуга_, дважды щелкнуть на поле _Родитель_ — оно перенесется в список условий отбора. 
> Это позволит пользователю отбирать услуги по группам.

![[Pasted image 20260223193924.png]]

---
### **В 1С:Предприятии:**
Открыть отчет, нажать _Настройки…_ В окне пользовательских настроек появились закладки _Отбор_ и _Условное оформление_.

![[Pasted image 20260223194235.png]]

**Работа с отбором:**
- На закладке _Отбор_ в колонке _Значение_ нажать кнопку выбора, ввести первые символы нужной группы (например, «Стиральные машины») и выбрать её из списка.
![[Pasted image 20260223194340.png]]

- Нажать _Завершить редактирование_, затем _Сформировать_. В отчете останутся только услуги из выбранной группы, в заголовке отобразится информация об отборе.
![[Pasted image 20260223194428.png]]

---
**Дополнительные возможности пользователя:**
- При закрытии отчета настройки запоминаются для текущего пользователя.
    
- Можно изменить условие отбора или создать новые настройки условного оформления.
    
- Команда _Еще → Изменить состав настроек…_ позволяет перенести настройки между быстрым доступом (в форме отчета) и обычным (в окне _Настройки…_).
![[Pasted image 20260223194642.png]]
![[Pasted image 20260223194656.png]]

> **Важно:** если пользователю не нужна гибкость, настройки лучше задать жестко в конфигураторе, оставив пользователю только включение/выключение.

Для дальнейших примеров в конфигураторе нужно снять признак использования у настройки отбора.
