***Для размещения стандартных команд*** *открытия списков и создания новых* объектов конфигурации в интерфейсе «1С:Предприятия» **существует общий стандартный алгоритм**, который мы сейчас объясним на примере справочников. 
Но это спра ведливо и для документов, планов счетов и т. п.

В «1С:Предприятии» *команды для открытия списка и создания новых* элементов ***добавляются в интерфейс автоматически для тех подсистем***, *где отображается объект*. 
Но команда создания **по умолчанию невидима**, так как создание новых элементов требуется реже, чем просмотр списка. Её видимость включают только там, где это необходимо по логике работы.

В нашей конфигурации:
- Для подсистемы **Оказание услуг** (работает менеджер) включим видимость команды создания нового клиента:  
    _Подсистема ОказаниеУслуг → кнопка Командный интерфейс → группа Панель действий.Создать → команда Клиент: создать → включить видимость_.

![[Pasted image 20260214124915.png]]

- Для подсистемы **Бухгалтерия** команду создания не включаем — бухгалтеры могут создавать клиентов через список, но основная работа с клиентами ведётся в разделе оказания услуг.

Запускаем «1С:Предприятие» в режиме отладки, подтверждаем обновление конфигурации и принимаем изменения.
