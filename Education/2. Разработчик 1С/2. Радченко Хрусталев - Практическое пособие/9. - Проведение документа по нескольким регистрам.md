Это занятие будет посвящено тому, как один и тот же документ может поставлять информацию в различные регистры конфигурации и для чего может понадобиться такая возможность. 

На этом занятии мы создадим еще один регистр накопления нашей конфигурации и изменим процедуру проведения документов так, чтобы они записывали необходимые данные как в один, так и в другой регистр. 

Также мы подготовим базу для изучения следующей главы.

---
## Зачем нужно проведение документа по нескольким регистрам
Ранее мы учитывали только **количество** материалов (регистр _ОстаткиМатериалов_). 

Этого недостаточно — нужно знать:
- сколько денег потрачено на закупку материалов;
- какова стоимость материальных запасов в денежном выражении.


Руководство хочет вести **суммовой учёт по средней стоимости**:
- при закупке материалы учитываются по цене приобретения;
    
- при расходе списываются по средней стоимости (рассчитывается как общая сумма закупок материала, делённая на общее количество этого материала).


**Решение:**  
Создадим новый регистр накопления _СтоимостьМатериалов_ для хранения суммовых данных. 

Теперь документы _ПриходнаяНакладная_ и _ОказаниеУслуги_ будут формировать движения **в двух регистрах** одновременно:
- _ОстаткиМатериалов_ — для количества.
- _СтоимостьМатериалов_ — для суммы.

# 2. Проведение документа «Оказание услуги» по двум регистрам
В процедуру проведения документа _ОказаниеУслуги_ нужно внести изменения согласно требованию руководства: при списании материалов должна указываться стоимость, рассчитанная по текущей конъюнктуре, а не только цена.

Для этого необходимо:
1. Добавить в табличную часть _ПереченьНоменклатуры_ новый реквизит _Стоимость_ (число, 15.2).
    
2. Изменить процедуру _ОбработкаПроведения_, добавив формирование движений в регистр _СтоимостьМатериалов_ с типом **Расход**.
    
3. В режиме 1С:Предприятие перепровести все документы _Оказание услуги_ для применения нового алгоритма.

## **Добавление реквизита «Стоимость» в документ «Оказание услуги»**

### **В конфигураторе:**
1. Открыть документ _ОказаниеУслуги_ → закладка _Данные_.
    
2. В табличной части _ПереченьНоменклатуры_ добавить новый реквизит:
    - Имя: _Стоимость_
    - Тип: _Число_, длина 15, точность 2, неотрицательное
        
3. Открыть форму документа _ФормаДокумента_:
    - На закладке _Реквизиты_ раскрыть _Объект_ → найти в табличной части реквизит _Стоимость_
        
    - Перетащить его мышью в окно элементов формы (левое верхнее), разместить после поля _Номенклатура_


### **Изменение процедуры проведения:**
1. В документе _ОказаниеУслуги_ на закладке _Движения_ отметить регистр _СтоимостьМатериалов_.
> Чтобы не потерять внесенные изменения (проверку на то что видНоменклатуры=Материал), внесем изменения вручную
    
2. На закладке _Прочее_ открыть _Модуль объекта_.
    
3. В процедуре _ОбработкаПроведения_ добавить:
    - В начале: `Движения.СтоимостьМатериалов.Записывать = Истина;`
        
    - Внутри цикла (после создания движения по количеству) добавить блок для регистра стоимости:
```
// Регистр СтоимостьМатериалов Расход
Движение = Движения.СтоимостьМатериалов.Добавить();
Движение.ВидДвижения = ВидДвиженияНакопления.Расход;
Движение.Период = Дата;
Движение.Материал = ТекСтрокаПереченьНоменклатуры.Номенклатура;
Движение.Стоимость = ТекСтрокаПереченьНоменклатуры.Количество * ТекСтрокаПереченьНоменклатуры.Стоимость;
```
