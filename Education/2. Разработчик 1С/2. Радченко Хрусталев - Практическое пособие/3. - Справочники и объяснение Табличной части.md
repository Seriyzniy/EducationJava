## **Что это:**  
Объект конфигурации **для работы со списками данных** (сотрудники, товары, клиенты и т.д.). На его основе платформа создаёт в базе данных таблицы для хранения информации.

---
## **Элементы и реквизиты:**  
==Справочник состоит из элементов(конкретный объект, одна строка данных)== 
Каждый *элемент может содержать дополнительную информацию* — для этого разработчик создаёт **реквизиты** (подчинённые объекты конфигурации). 

> В базе данных справочник хранится как таблица: строки — элементы, столбцы — реквизиты.
![[Pasted image 20260211203738.png]]

У каждого справочника **по умолчанию есть стандартные реквизиты**: `Код` и `Наименование`. Их доступность зависит от свойств справочника (иерархия, подчинение и т.д.).
![[Pasted image 20260211203805.png]]

![[Pasted image 20260211203925.png]]

---
## **Табличные части:**  
Если у разных элементов справочника отличается количество однотипной информации (например, состав семьи сотрудника), используются **табличные части**. 

> Для них *платформа автоматически создаёт подчинённые таблицы в базе данных.*

Так, например, каждый элемент справочника Сотрудники может содержать информацию о составе семьи сотрудника. Для одного сотрудника это будет только супруга, а у другого семья может состоять из супруги, сына и дочери.
![[Pasted image 20260211204211.png]]

Причем система скрывает от разработчика всю «техническую» часть, связанную с хранением данных: в базе данных для справочника создаются несколько таблиц, эти таблицы связываются по уникаль ному полю (Ссылка), поля таблиц имеют определенные типы и т. д. 

Все это система делает сама. Нам лишь нужно добавить в объект конфигурации Справочник подчиненный ему объект ТабличнаяЧасть.

---
## **Иерархия:**  
Элементы можно группировать. 

Виды иерархии:
- иерархия групп и элементов (группы как папки);
- иерархия элементов (один элемент выступает родителем для других).

### Иерархия групп и элементов
Например, в справочнике `Бытовая техника` могут быть созданы следующие группы: 
- Холодильники, 
- Телевизоры, 
- Стиральные машины 
- и т. д. 

**Возможность создания таких групп** в справочнике **задается свойством** `Иерархический` объекта конфигурации `Справочник`. 
![[Pasted image 20260211204553.png]]

В этом случае элемент справочника, представляющий собой группу, будет являться родителем для всех элементов и групп, входящих в эту группу. Такой вид иерархии называется иерархией групп и элементов
![[Pasted image 20260211204617.png]]

### Иерархия элементов
Возможен и другой вид иерархии – иерархия элементов. В этом случае в качестве родителя выступает не группа элементов справочника, а непосредственно один из его элементов. 

Например, такой вид иерархии можно использовать при создании справочника Подразделения, когда одно подразделение является родителем для нескольких других, входящих в его состав
![[Pasted image 20260211204716.png]]

## **Подчинение:**  
Элементы одного справочника могут быть подчинены элементам или группам другого справочника (владельцу).

В системе «1С:Предприятие» это достигается путем указания списка владельцев справочника для каждого объекта конфигурации Спра вочник.
![[Pasted image 20260211204810.png]]

Например, справочник Контак тныеЛица может быть подчинен справочнику Партнеры. Тогда для каждого партнера мы сможем указать его сотрудников, с которыми нужно контактировать.
![[Pasted image 20260211204944.png]]

Как только добавили владельца для справочника, то в добавление к стандартный реквизитам, появляется реквизит Владелец
![[Pasted image 20260211205329.png]]

И в добавок, подчиненный справочник появляется как вкладка для владельца при его создании
![[Pasted image 20260218174647.png]]

## **Предопределённые элементы:**  
Порой возникают ситуации, когда необходимо, чтобы в справочнике некоторые элементы существовали всегда, независимо от действий пользователя.

Создаются в конфигураторе, всегда присутствуют в справочнике независимо от действий пользователя, доступны из встроенного языка. 
В интерфейсе помечены специальной пиктограммой.
![[Pasted image 20260211205956.png]]

## Формы справочника
![[Pasted image 20260211210624.png]]

Для разных действий со справочником нужны разные способы его отображения: 
- для выбора элемента — список, 
- для редактирования — форма с реквизитами.

![[Pasted image 20260211210459.png]]

Система может автоматически сгенерировать все необходимые формы. Максимум для справочника требуется пять форм.

### **Виды форм и их назначение:**
![[Pasted image 20260211210802.png]]
- **Форма элемента** — для создания и редактирования элемента справочника.
    
- **Форма группы** — для создания и редактирования группы (содержит меньше информации, чем форма элемента).
    
- **Форма списка** — для отображения всех элементов справочника.
    
- **Форма выбора** — для выбора элемента из справочника в каком-либо поле (упрощенная форма списка).
    
- **Форма выбора группы** — для выбора группы справочника.


> В разных местах конфигуратора формы называются немного по-разному (например, «Основная форма объекта» и «Форма элемента справочника»), но речь идет об одном и том же.

Любая форма создается в конфигураторе с помощью подчиненного объекта конфигурации **Форма**. На основе его описания платформа в нужный момент создает программный объект «Форма», с которым работает пользователь.

**Назначение формы:** визуализировать данные из базы в удобном виде и описывать алгоритмы работы с этими данными.

# 1. Простой Справочник

## 1.1 **Представления объекта конфигурации**
Эти свойства определяют, как названия объекта будут выглядеть в интерфейсе.

**Представление объекта** — название в единственном числе. Используется:
- в команде создания нового элемента (например, _Клиент: создать_);
- в заголовке формы объекта (если не задано расширенное едставление);
    
- в представлении ссылки на объект.

Нужно задавать, если синоним объекта во множественном числе (например, синоним _Клиенты_, а для создания нужен _Клиент_).


**Расширенное представление объекта** — заголовок формы объекта. Если не задано, используется Представление объекта.


**Представление списка** — название во множественном числе для команды открытия списка (например, _Клиенты_). Нужно задавать, если синоним в единственном числе (например, документ _Приходная накладная_, а для списка — _Приходные накладные_).


**Расширенное представление списка** — заголовок формы списка. Если не задано, используется Представление списка.

_Зачем это нужно:_ чтобы в интерфейсе не было некрасивых повторений вроде «Список клиентов, Список сотрудников» — лучше просто «Клиенты, Сотрудники».

## 1.2 **Код и наименование справочника**
На закладке «Данные» задаются:
- **Длина кода** — определяет максимальное *количество элементов в справочнике.* Платформа может автоматически контролировать уникальность кодов и поддерживать автонумерацию. Для нашей конфигурации оставим длину 9 символов (коды от 1 до 999999999).
    
- **Длина наименования** — увеличим до 50 символов (вместо 25).

### Изменение синонима стандартного реквизита
Чтобы в интерфейсе вместо «Наименование» отображалось «Ф. И. О.» (для справочника сотрудников), ***изменим синоним стандартного реквизита***:
- Кнопка` «Стандартные реквизиты» → выделить «Наименование» → контекстное меню → «Свойства»`.
    
- В палитре свойств установить синоним: **Ф. И. О.**

Это изменит отображение во всех автоматически сгенерированных формах.

> Также обратите внимание: 
> Свойство «Проверка заполнения» по умолчанию стоит «Выдавать ошибку» — если реквизит не заполнен, система сообщит об ошибке.

# 3. Иерархический справочник
Справочник Номенклатура будет содержать информацию об услугах, которые оказывает ООО «На все руки мастер», и о тех материалах, которые при этом могут быть использованы. 

Этот справочник не будет сложным. Единственная особенность, которой он будет обладать, – наличие иерархической структуры. Для того чтобы справочником было удобно пользоваться, мы сгруппируем услуги в одну группу, а материалы – в другую. 

Кроме этого, поскольку ООО «На все руки мастер» оказывает самые разные услуги, они также будут логически собраны в несколько групп. То же самое можно сказать и про материалы

**Создание справочника:**
- Имя: _Номенклатура_ (синоним заполнится автоматически).
- Представление объекта и списка не задаём — будет использоваться синоним «Номенклатура».
![[Pasted image 20260214132809.png]]

**Подсистемы:**  
Справочник должен быть доступен в разделах _Учет материалов_, _Оказание услуг_ и _Бухгалтерия_ — отмечаем их.

**Иерархия:**  
На закладке _Иерархия_ устанавливаем флаг **Иерархический справочник**.
![[Pasted image 20260214132940.png]]

**Данные:**
- Длина наименования: **100 символов**.
- Для стандартного реквизита _Родитель_ меняем синоним на _Группа номенклатуры_ (через кнопку «Стандартные реквизиты»).
![[Pasted image 20260214133210.png]]

**Настройка командного интерфейса (через «Все подсистемы»):**
- Для подсистем _Учет материалов_ и _Оказание услуг_: включить видимость команды _Номенклатура: создать_ (Панель действий.Создать). Команда открытия списка уже видна по умолчанию.
    
- Для _Бухгалтерии_ команду создания не добавляем.
![[Pasted image 20260214133404.png]]
![[Pasted image 20260214133511.png]]

Запускаем «1С:Предприятие», подтверждаем обновление, принимаем изменения. Теперь можно заполнять справочник, создавая группы и перенося элементы.
