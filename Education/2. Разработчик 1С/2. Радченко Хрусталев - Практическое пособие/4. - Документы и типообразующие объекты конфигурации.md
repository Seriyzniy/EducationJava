## Что такое Документ?
Объект конфигурации **Документ** предназначен для фиксации хозяйственных операций или событий (накладные, приказы, платежные поручения и т.д.). На его основе платформа создаёт в базе данных таблицы для хранения документов.

### **Ключевая особенность — проведение:**
- **Проведённый документ** влияет на состояние учёта.
    
- **Непроведённый** — лишь черновик, учёт не меняет.  

Документ всегда привязан к конкретному моменту времени (реквизит _Дата_), что позволяет отражать последовательность событий. Система может отслеживать изменения, внесённые задним числом, и при необходимости перепроводить связанные документы.

### **Структура документа:**
Большинство реквизитов разработчик создает самостоятельно, однако у каждого объекта конфигурации Документ по умолчанию существуют стандартные реквизиты.

- **Стандартные реквизиты** (есть всегда): _Дата_ (определяет положение на оси времени) и _Номер_.
![[Pasted image 20260214143620.png]]

- **Реквизиты** — дополнительная информация, одинаковая для всех документов данного вида (например, поставщик, склад). Создаются разработчиком.
    
- **Табличные части** — для информации, одинаковой по структуре, но разной по количеству (например, список товаров в накладной). Для них создаются отдельные таблицы в базе данных.
![[Pasted image 20260214143635.png]]

В базе данных каждый документ — отдельная запись в основной таблице.

## **Типы данных и типообразующие объекты конфигурации**
В системе 1С существуют:
- **Примитивные типы** — определены изначально и доступны всегда: _Число, Строка, Дата, Булево_.
    
- **Типы, определяемые конфигурацией** — появляются только после создания определённых объектов конфигурации (справочников, документов и т.д.). Такие объекты называются **типообразующими**.

Например, после создания справочника _Склады_ становятся доступны новые типы:
- _СправочникСсылка.Склады_
- _СправочникОбъект.Склады_
- _СправочникМенеджер.Склады_
- _СправочникВыборка.Склады_

Эти типы существуют только в данной конфигурации и позволяют, например, хранить в реквизите ссылку на конкретный элемент справочника.

При создании документов нам понадобятся типы _СправочникСсылка.Склады_ и _СправочникСсылка.Номенклатура_, появившиеся после создания соответствующих справочников.

## **Справочники и документы: разница в именовании**
В типовых конфигурациях:
- Объекты в ветке **Справочники** обычно именуются во **множественном** числе (например, _Товары_).
    
- Объекты в ветке **Документы** — в **единственном** числе (например, _ПриходнаяНакладная_).

Может показаться, что справочник — это хранилище множества элементов, а документ — шаблон одного документа. ***Но это не так***.

**На самом деле:**
- Создавая справочник _Товары_ (мн. ч.), мы получаем хранилище для множества элементов, каждый из которых описывает один товар (ед. ч.).
    
- Создавая документ _ПриходнаяНакладная_ (ед. ч.), мы также получаем хранилище для множества записей, каждая из которых описывает одну конкретную накладную.

Концептуально логичнее было бы именовать документы во множественном числе (_ПриходныеНакладные_), подчёркивая, что это набор документов одного вида. Однако психологически пользователю привычнее видеть в ветке _Документы_ перечисление видов документов в единственном числе. Поэтому так сложилось исторически.

## **Автоматический пересчет суммы в строках документа**
Чтобы сумма в строках документа вычислялась автоматически при изменении количества или цены, нужно создать собственную форму документа и описать алгоритм на встроенном языке.

### **Создание формы документа в конфигураторе:**
1. Открыть редактирование документа _ПриходнаяНакладная_ → закладка _Формы_.
    
2. Нажать кнопку _Добавить_ (или кнопку с лупой) — откроется конструктор форм.
    
3. Выбрать тип _Форма документа_ → _Готово_ (оставив предложенные настройки).
![[Pasted image 20260214150717.png]]

В результате:
- В дереве объектов появится форма _ФормаДокумента_.
- Откроется редактор форм, где можно редактировать структуру формы.
![[Pasted image 20260214150755.png]]

### **Редактор форм:**
- Разработчик не рисует форму вручную, а задаёт структуру элементов (закладка _Элементы_). Система сама располагает их.
![[Pasted image 20260214150841.png]]

- В дереве элементов видны реквизиты табличной части: _МатериалыКоличество_, _МатериалыЦена_, _МатериалыСумма_.
    
- Можно менять порядок элементов перетаскиванием — изменения сразу отображаются в макете формы.

**Задача:** при изменении количества или цены автоматически вычислять сумму (Сумма = Количество × Цена). Для этого нужно написать код на встроенном языке, который будет срабатывать при изменении этих полей.

## **Обработчик события «ПриИзменении»**
У системы есть «стандартное поведение», но разработчик может вмешаться в него через **события**. Для поля _Количество_ табличной части создадим обработчик события _ПриИзменении_, чтобы автоматически пересчитывать сумму.

### **Создание обработчика:**
- В редакторе форм выделить поле _МатериалыКоличество_ → открыть палитру свойств.
    
- Найти событие _ПриИзменении_ → нажать кнопку с лупой.
![[Pasted image 20260214151040.png]]

- В окне выбора оставить _Создать на клиенте_ → система создаст шаблон процедуры в модуле формы и откроет его.
![[Pasted image 20260214151054.png]]
![[Pasted image 20260214151123.png]]
### **Код обработчика:**
```text
СтрокаТабличнойЧасти = Элементы.Материалы.ТекущиеДанные;
СтрокаТабличнойЧасти.Сумма = СтрокаТабличнойЧасти.Количество * СтрокаТабличнойЧасти.Цена;
```

![[Pasted image 20260214151236.png]]
**Что происходит:**
- _Элементы.Материалы_ — обращение к табличной части формы.
    
- _ТекущиеДанные_ — получает данные текущей ==редактируемой строки== как объект (переменная _СтрокаТабличнойЧасти_).
    
- Через этот объект доступны значения колонок: _Количество_, _Цена_, _Сумма_.
    
- Вычисляем произведение и записываем в поле _Сумма_.

Теперь при изменении количества сумма будет пересчитываться автоматически. 

Аналогичный обработчик нужно создать и для поля _Цена_.
> Или же можно ссослаться на тот же обработчик, который мы написали выше

## **Одна процедура для обработки нескольких событий**
Чтобы не дублировать код для полей _Количество_ и _Цена_ (и для возможного использования в других документах), вынесем расчёт суммы в **общий модуль**.

### **Создание общего модуля:**
- В дереве конфигурации: _Общие → Общие модули_ → кнопка _Добавить_.
    
- Имя: _РаботаСДокументами_.
    
- В свойствах: установить флажок _Клиент (управляемое приложение)_, снять флажок _Сервер_.
![[Pasted image 20260214151904.png]]

### **Код в общем модуле:**
```text
Процедура РассчитатьСумму(СтрокаТабличнойЧасти) Экспорт
    СтрокаТабличнойЧасти.Сумма = СтрокаТабличнойЧасти.Количество * СтрокаТабличнойЧасти.Цена;
КонецПроцедуры
```

> [!IMPORTANT] Ключевое слово `Экспорт`
> Ключевое слово _Экспорт_ делает процедуру доступной из других модулей.

![[Pasted image 20260214151945.png]]

### **Изменение обработчика для поля _Количество_:**
- Оставляем получение текущей строки.
- Вместо прямого расчёта вызываем процедуру из общего модуля:
```text
&НаКлиенте
Процедура МатериалыКоличествоПриИзменении(Элемент)
    СтрокаТабличнойЧасти = Элементы.Материалы.ТекущиеДанные;
    РаботаСДокументами.РассчитатьСумму(СтрокаТабличнойЧасти);
КонецПроцедуры
```
![[Pasted image 20260214152152.png]]
### **Для поля _Цена_:**  
Создаём отдельный обработчик _ПриИзменении_ (стандарт требует для каждого события свой обработчик). В нём вызываем ту же процедуру:

```text
&НаКлиенте
Процедура МатериалыЦенаПриИзменении(Элемент)
    СтрокаТабличнойЧасти = Элементы.Материалы.ТекущиеДанные;
    РаботаСДокументами.РассчитатьСумму(СтрокаТабличнойЧасти);
КонецПроцедуры
```

Теперь сумма пересчитывается при изменении количества **или** цены, а код не дублируется.

**Форма документа:**
- Создаётся основная форма.
    
- Для полей _Количество_ и _Цена_ создаются обработчики события _ПриИзменении_ (на клиенте), вызывающие процедуру _РассчитатьСумму_ из общего модуля _РаботаСДокументами_.
> Мы же просто создадим один обработчик, переименуем его в `ЦенаИКоличествоПриИзменении(Элемент)` и привяжем его к обоим полям
    

**Командный интерфейс:**  
В подсистеме _Оказание услуг_ включается видимость команды создания документа (_Панель действий.Создать → Оказание услуги: создать_).
