==Все данные в базе данных== «1С:Предприятия» делятся на две категории: 
- **объектные** 
- и **необъектные**. 

Различия в их природе определяют разные способы работы с ними.


К объектным данным относятся:
- справочники, документы;
    
- планы видов характеристик, планы счетов, планы видов расчета;
    
- бизнес-процессы, задачи, планы обмена.


**Объектные данные состоят из отдельных объектов**, каждый из которых:
- ***обладает внутренним уникальным идентификатором***;
    
- представляет ***совокупность значений как единое целое***;
    
- имеет самостоятельную значимость (например, элемент справочника "Физические лица" остаётся тем же лицом даже при изменении его реквизитов).

> Удаление объекта изменяет состояние базы данных. 
> 
> Создание нового объекта с теми же значениями реквизитов даёт другое состояние, так как объект получает новый уникальный идентификатор.

> [!WARNING] Отличительная особенность - `Ссылка`
> **Каждая таблица содержит поле** `Ссылка` с внутренним идентификатором объекта. 
> 
> Таким образом, **объект** ***представляет собой*** **совокупность записи основной таблицы и связанных с ней строк табличных частей.**


> [!NOTE] Версия объекта и оптимистическая блокировка
> ***Основная таблица также содержит*** ==обязательное поле== с текущей **версией объекта**, изменяющееся при каждой записи. 
> 
> Это поле обеспечивает оптимистическую блокировку.


> *Для работы с объектными данными* во встроенном языке существуют два основных типа: 
> - **ссылка**(чтение) 
> - и **объект**(чтение+запись).

---
## 1.1 Ссылка
Значение ссылочного типа (например, `СправочникСсылка.<имя>`) используется для однозначной идентификации объекта базы данных. 
> Фактически это внутренний идентификатор, хранящийся в поле _Ссылка_.

Ссылки могут:
- выбираться в полях ввода;
    
- храниться в полях других таблиц;
    
- сравниваться между собой.

![[Pasted image 20260301190812.png]]

Ссылки на объекты разных типов никогда не равны. Ссылки на один и тот же объект одного типа всегда равны, независимо от способа получения.

**Пустая ссылка** — значение, которому не соответствует ни один объект в базе (идентификатор `00000000-0000-0000-0000-000000000000`). 

***Пустые ссылки на объекты разных типов НЕ РАВНЫ МЕЖДУ СОБОЙ***.
![[Pasted image 20260301191032.png]]

Ссылка позволяет обращаться к свойствам объекта и получать сам объект (при этом выполняется чтение из базы данных).

## 1.2 Представление ссылочных значений
Для удобства работы в интерфейсе используется механизм формирования представлений ссылочных значений. 

Система самостоятельно определяет правила формирования по умолчанию, но разработчик может их изменять.

- Для справочников, планов видов характеристик, счетов, видов расчёта и узлов планов обмена представление может быть задано в виде **кода** или **наименования**.
	![[Pasted image 20260301191156.png]]
    
- Для задач можно выбрать представление в виде **номера** или **наименования**.
	![[Pasted image 20260301191224.png]]
	
- ***Для документов и бизнес-процессов*** **представление фиксировано** *и формируется как совокупность*: 
	- синоним, 
	- номер и дата 
	- (например, «Приходная накладная 000000003 от 03.05.2011 15:35:27»).

## 1.3 Контроль ссылочной целостности
Платформа предоставляет возможность контролировать ссылочную целостность базы данных, но это не обязательно. 

***База данных может содержать неразрешимые ссылки*** (**ссылки на несуществующие объекты), и это не является ошибкой**. 

> Необходимость контроля определяется логикой прикладного решения.

### 1.3.1 Удаление объектов
**Интерактивное удаление** может выполняться двумя способами:

- **Непосредственное удаление** — без проверок ссылочной целостности, восстановление невозможно. Регулируется правом _Интерактивное удаление_.
    
- **Удаление с контролем** — в два этапа:
    1. ==Установка пометки удаления== для объектов.
        
    2. Запуск процедуры удаления помеченных объектов, которая сначала ищет ссылки на удаляемые объекты и удаляет только те, на которые ссылки отсутствуют.


**Программное удаление** также может использовать контроль или обходиться без него:

- метод объекта **Удалить()** — непосредственное удаление;
    
- метод **УстановитьПометкуУдаления()** — установка пометки;
    
- функция **НайтиПомеченныеНаУдаление()** — поиск помеченных объектов;
    
- процедура **УдалитьОбъекты()** — удаление с контролем или без (с контролем выполняется в монопольном режиме);
    
- функция **НайтиПоСсылкам()** — поиск ссылок на удаляемые объекты.

---
### 1.3.2 Особенности пометки удаления
==Таблицы объектных данных содержат специальное поле== `ПометкаУдаления`. 

Значение может быть установлено двумя способами:

**Интерактивно или через МЕТОД `УстановитьПометкуУдаления()`:**
- выполняются дополнительные действия (для справочника — пометка подчинённых элементов; для документа — отмена проведения);
    
- вызывается событие _ПередЗаписью_.
    

**Непосредственной УСТАНОВКОЙ СВОЙСТВА `ПометкаУдаления` и записью:**
- дополнительные действия не выполняются;
    
- требуется учитывать особенности (например, документ не может быть одновременно проведён и помечен на удаление — предварительно нужно отменить проведение).
