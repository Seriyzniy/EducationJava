Тип данных представляет собой перечень возможных типов значений, которые могут быть использованы тем или иным элементом системы. 

Типы данных применяются при описании:
- объектов конфигурации (реквизиты, измерения, ресурсы и др.);
    
- реквизитов формы;
    
- элементов управления в форме и табличном документе;
    
- некоторых объектов встроенного языка (список значений, колонки таблицы значений и дерева значений и др.).


Тип данных позволяет перечислить допустимые типы значений, а для примитивных типов — задать квалификаторы, описывающие ограничения.

При визуальном конструировании тип данных описывается через палитру свойств или специальный диалог. 

> [!NOTE] Описание типа данных программно
> Во встроенном языке для этого используется объект `ОписаниеТипов`.

---
# 1. Числовые типы данных
При описании числового типа можно указать:
- допустимый знак числа (любой или неотрицательный);
    
- общую разрядность числа;
    
- разрядность дробной части.

## 1.1. Визуальное описание
При редактировании объектов конфигурации (например, реквизита формы) числовой тип настраивается в диалоге редактирования типа данных.
![[Pasted image 20260302180621.png]]
## 1.2. Программное описание
Для описания числового типа средствами встроенного языка используется объект **КвалификаторыЧисла**:
```java
КвалификаторЧисла = Новый КвалификаторыЧисла(
	5, 
	2, 
	ДопустимыйЗнак.Неотрицательный);
	
ОписаниеТипа = Новый ОписаниеТипов("Число", КвалификаторЧисла);


ТаблицаСкидок.Колонки.Добавить("Скидка", ОписаниеТипа);
```

В примере создаётся тип "Число" с общей разрядностью 5, дробной частью 2 и знаком "неотрицательный". Затем этот тип используется для колонки таблицы значений.

---
# 2. Строковые типы данных
При описании строкового типа можно указать:
- вариант ограничения длины (фиксированная или переменная);
    
- длину строки (при фиксированной длине).


Строки различаются по длине:
- **ограниченной длины** — длина задана (больше нуля);
    
- **неограниченной длины** — длина равна нулю.

## 2.1. Особенности ограниченных строк
- При присвоении значения, превышающего ограничение, строка обрезается до указанной длины.
    
- Строки **фиксированной длины** дополняются пробелами справа до заданной длины.
    
- Строки **переменной длины** хранятся без дополнения пробелами.

---
## 2.2. Визуальное описание
При редактировании объектов конфигурации строковый тип настраивается в диалоге редактирования типа данных.
![[Pasted image 20260302181116.png]]
## 2.3. Программное описание
Для описания строкового типа используется объект **КвалификаторыСтроки**:
```java
КвалификаторСтроки = Новый КвалификаторыСтроки(
	5, 
	ДопустимаяДлина.Переменная);
ОписаниеТипа = Новый ОписаниеТипов("Строка", КвалификаторСтроки);

ТаблицаСкидок.Колонки.Добавить("Артикул", ОписаниеТипа);
```

В примере создаётся тип "Строка" переменной длины с максимумом 5 символов, который затем используется для колонки таблицы значений.

# 3. Типы данных, описывающие значения типа «Дата»
При описании допустимых значений типа `Дата` можно указать, какие части даты будут использоваться:
- **дата и время**;
    
- **только дата**;
    
- **только время**.

## 3.1. Визуальное описание
При редактировании объектов конфигурации (например, реквизита формы) тип данных «Дата» настраивается в диалоге редактирования типа данных, где выбирается нужный состав частей даты.
![[Pasted image 20260302181358.png]]

## 3.2. Программное описание
Для описания типа данных «Дата» средствами встроенного языка используется объект **КвалификаторыДаты**:

```java
КвалификаторДаты = Новый КвалификаторыДаты(ЧастиДаты.Дата);
ОписаниеТипаДата = Новый ОписаниеТипов("Дата", КвалификаторДаты);

ТаблицаКурсВалют.Колонки.Добавить("Дата", ОписаниеТипаДата);
```

В примере:
- создаётся квалификатор, устанавливающий использование только части **Дата** (без времени);
    
- на его основе создаётся объект `ОписаниеТипов` для типа `Дата` с указанным ограничением;
    
- полученное описание используется для колонки таблицы значений.

В результате в колонке `Дата` могут храниться значения типа `Дата`, у которых время всегда равно 00:00:00.

# 4. Составной тип данных
До сих пор рассматривались типы данных, содержащие описание только одного типа значений. В общем случае тип данных может содержать перечень нескольких типов значений — такой тип называется **составным**.

Благодаря составным типам в базе данных могут храниться реквизиты, значения которых имеют разный тип в разные моменты времени, но в каждый конкретный момент хранится значение только одного из перечисленных типов.

**Пример**
Например, документ _Внутренний заказ_ может иметь реквизит _Заказчик_ составного типа: `СправочникСсылка.Подразделения`, `СправочникСсылка.Склады`. В нём могут храниться как ссылки на подразделения, так и ссылки на склады.

## 4.1. Описание составного типа
### 4.1.1. Визуальное описание
При визуальном конструировании для описания составного типа используется окно редактирования типа данных, где разработчик перечисляет типы значений и наборы типов, входящие в составной тип, и при необходимости задаёт квалификаторы.

### 4.1.2. Программное описание
Составной тип данных может быть описан средствами встроенного языка:
```java
МассивТипов = Новый Массив;
МассивТипов.Добавить(Тип("Строка"));
МассивТипов.Добавить(Тип("Число"));
МассивТипов.Добавить(Тип("СправочникСсылка.Номенклатура"));

ОписаниеСоставногоТипа = Новый ОписаниеТипов(МассивТипов);


НовыйРеквизит = Новый РеквизитФормы(
	"РеквизитСоставногоТипа", 
	ОписаниеСоставногоТипа);
	
МассивРеквизитов = Новый Массив;
МассивРеквизитов.Добавить(НовыйРеквизит);

ИзменитьРеквизиты(МассивРеквизитов);
```

---
## 4.2. Значение по умолчанию
Для реквизита составного типа значением ==по умолчанию является== **Неопределено**. 

Поэтому тип `Неопределено` всегда входит в составной тип данных наряду с прочими типами. 
> Представление значения `Неопределено` — пустая строка.

При отображении в поле ввода при нажатии кнопки выбора открывается окно для выбора одного из возможных типов значений, после чего можно ввести конкретное значение выбранного типа.

---
## 4.3. Преобразование значений
### 4.3.1. Неявное преобразование
При присвоении значения реквизиту составного типа:
- Если присваиваемое значение соответствует одному из типов составного типа, реквизит принимает этот тип и значение.
    
- Если не соответствует, система пытается привести присваиваемое значение к одному из типов составного типа.
    
- Если приведение невозможно, реквизиту присваивается значение по умолчанию — `Неопределено`.

---
### 4.3.2. Явное преобразование
Для явного преобразования используется метод **ПривестиЗначение()** объекта `ОписаниеТипов`:

```java
ОписаниеСоставногоТипа = Новый ОписаниеТипов(МассивТипов);
ПриводимоеЗначение = Справочники.Номенклатура.НайтиПоКоду("000000001");

Результат = ОписаниеСоставногоТипа.ПривестиЗначение(ПриводимоеЗначение);
```

Метод приводит значение произвольного типа к значению, соответствующему составному типу данных. Если исходный тип не входит в составной, система приводит его к одному из входящих типов (например, объект справочника может быть приведён к строке).

---
# 5. Наборы типов
При выборе типа данных реквизита система предоставляет возможность выбирать не только конкретные типы, но и **наборы типов**. 

Примерами наборов типов являются `ЛюбаяСсылка`, `СправочникСсылка`, `Характеристика.<имя>` и другие.

Наборы типов, как и составной тип данных, содержат перечень типов, определённых в прикладном решении. Однако, в отличие от составного типа, этот перечень ==формируется== **системой автоматически** на основе анализа объектов конфигурации.

## 5.1. Пример работы Набора Типов
Рассмотрим на примере ссылок на справочники. Пусть в прикладном решении есть справочники _Номенклатура_ и _Контрагенты_.
- **Составной тип данных**: можно определить реквизит, включающий типы `СправочникСсылка.Номенклатура` и `СправочникСсылка.Контрагенты`.
    
- **Набор типов**: можно определить реквизит, содержащий набор типов `СправочникСсылка`.
    

В обоих случаях реквизит сможет хранить ссылки как на _Номенклатуру_, так и на _Контрагенты_.

Если добавить в конфигурацию новый справочник _Цены_:
- в реквизите составного типа по-прежнему можно хранить только ссылки на _Номенклатуру_ и _Контрагенты_;
    
- в реквизите, описанном как набор типов `СправочникСсылка`, можно хранить ссылку на **любой** справочник, включая новый _Цены_.

## 5.2. Преобразование наборов типов
При запуске прикладного решения набор типов обычно преобразуется системой в составной тип, содержащий все типы, входящие в этот набор. 

Поэтому новый справочник автоматически попадает в набор.

Однако набор типов не всегда преобразуется в составной тип:
- Если в набор входит единственный тип значений, набор преобразуется непосредственно в этот тип (а не в составной тип с одним элементом).

### 5.2.1. Пример с планом видов характеристик
Пусть план видов характеристик _Свойства_ имеет единственный тип значений в свойстве _ТипЗначенияХарактеристик_. Тогда набор типов `Характеристика.Свойства` преобразуется системой не в составной тип, а в тот единственный тип значений, который указан для плана видов характеристик.

Эта особенность важна, например, при проверке реквизита на заполненность:
- если `Характеристика.Свойства` преобразуется в составной тип, проверять нужно на значение `Неопределено`;
    
- если преобразуется в конкретный тип, проверять нужно на значение по умолчанию этого типа.
