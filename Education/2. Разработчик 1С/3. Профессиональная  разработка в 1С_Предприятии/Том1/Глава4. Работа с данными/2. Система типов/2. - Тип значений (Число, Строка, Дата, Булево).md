# 1. Числовой тип (Число)
Числовой тип предназначен для представления десятичных чисел. Максимальная разрядность числа, сохраняемого в базе данных, составляет 32 знака, включая десятичную точку. В оперативной памяти разрядность не ограничена.

## 1.1. Литералы числового типа
Литерал числа записывается набором цифр, может начинаться с «+» или «-», разделитель целой и дробной части — точка (например: `3874`, `-1475.25`).

---
## 1.2. Значение по умолчанию = 0
==Значение по умолчанию== для числового типа — **0**. 

В интерфейсе представление нуля зависит от элемента управления:
- **В колонке таблицы** по умолчанию отображается как пустое место. 
	Можно задать ***собственное представление через свойство*** `Формат` с параметром `ЧН` (например, `ЧН='---'`), при этом концевые пробелы обрезаются.
	![[Pasted image 20260302162221.png]]
```
Элементы.ТЗЗначение.Формат = "ЧН='---'"; Элементы.ТЗЗначение.ФорматРедактирования = "ЧН="---'";
```

- **В поле ввода** отображение зависит от разрядности дробной части: при нулевой разрядности — «0», при ненулевой — «0,000…» (с соответствующим количеством знаков). Собственное представление задать нельзя.

---
## 1.3. Операции с числами
**Арифметические операции:**
- Сложение (`+`), вычитание (`-`), умножение (`*`), деление (`/`), остаток от деления (`%`), унарный минус (`-`).
    ![[Pasted image 20260302162422.png]]

**Логические операции:**
- Больше (`>`), больше или равно (`>=`), равно (`=`), не равно (`<>`), меньше (`<`), меньше или равно (`<=`).
	![[Pasted image 20260302162434.png]]
---
## 1.4. Преобразование числовых значений

### 1.4.1. Неявное преобразование
- **К строке:** выполняется автоматически при использовании в строковом контексте (например, в `Сообщить()`). Результат зависит от региональных настроек.
    
- **К булеву:** любое число, кроме 0, преобразуется в `Истина`, 0 — в `Ложь`. Используется в логических выражениях и операциях (`НЕ`, `И`, `ИЛИ`).
    
- **В сравнениях с булевым операндом:** булево значение сначала преобразуется в число (`Истина` → 1, `Ложь` → 0), затем выполняется сравнение.

---
### 1.4.2. Явное преобразование
- **В булево:** функция `Булево(Число)` — возвращает `Истина` для любого ненулевого числа, `Ложь` для 0.
    
- **В строку:** функция `Строка(Число)` — возвращает строковое представление числа согласно региональным настройкам.

---
# 2. Строковый тип (Строка)
Строковый тип предназначен для представления строк в формате Unicode произвольной длины.

## 2.1. Литералы строкового типа
Литералы строк заключаются в кавычки. Если внутри строки требуется указать символ кавычки, он удваивается.

**Примеры:**
- `"книга"` → книга
    
- `"фирма ""Ваш сад"""` → фирма "Ваш сад"

**Многострочные литералы** могут задаваться двумя способами:
- Строки, каждая в своих кавычках, располагаются последовательно (между ними допускаются только пробелы, переводы строк и комментарии).
    ![[Pasted image 20260302163108.png]]
    
- ==С использованием символа== `|` в начале каждой строки внутри одних кавычек.
    ![[Pasted image 20260302163121.png]]

---
## 2.2. Значение по умолчанию = "" (пустая строка)
Значение по умолчанию — пустая строка (`""`), которая не отображается. В отладчике для наглядности может показываться как `"`.

---
## 2.3. Операции со строками
**Конкатенация:** оператор `+` — соединение строк.

**Логические операции:** сравнение строк (`>`, `>=`, `<`, `<=`, `=`, `<>`).
![[Pasted image 20260302163039.png]]

### 2.3.1. Особенности сравнения
Сравнение строк выполняется в Unicode с учётом региональных установок информационной базы (параметр _Язык/Страна_).

Для разных языков порядок сортировки может различаться. 

Результат сравнения зависит не только от самих строк, но и от региональных настроек.

---
## 2.4. Преобразование строк
### 2.4.1. Неявное преобразование - JavaScript
**К числу:** строка преобразуется, если содержит числовое представление. 

Например, `"25"` в выражении `35 + "25"` даст `60`. При невозможности преобразования возникает исключение.

**К булеву:** преобразуются строки, соответствующие литералам булевых значений (`Истина`, `Ложь`, `True`, `False`), а также их представления на языке интерфейса платформы. 

Например, для литовского языка строка `"Ne"` преобразуется в `Ложь`.

### 2.4.2. Явное преобразование
**В число:** функция `Число(Строка)` — преобразует строку, содержащую корректное числовое представление (допускается как точка, так и запятая в качестве разделителя).

**В дату:** функция `Дата(Строка)` — преобразует строку, содержащую части даты в порядке: год, месяц, день, час, минута, секунда (например, `"20110415174534"`).

---
# 3. Тип Дата
Тип `Дата` предназначен для представления даты и времени с точностью до секунды. Минимальное значение — 1 января 0001 года 00:00:00.

## 3.1. Литералы даты
**Во встроенном языке:**
- Последовательность цифр в одинарных кавычках: `'20110415224533'` (год, месяц, день, час, минута, секунда). Части даты могут разделяться произвольными символами. Час, минута и секунда могут опускаться (полагаются равными нулю).
    ![[Pasted image 20260302163728.png]]

**В языке запросов:**
- Ключевое слово `ДАТАВРЕМЯ(год, месяц, день, час, минута, секунда)`. Последние три параметра необязательны.
    ![[Pasted image 20260302163751.png]]

---
## 3.2. Значение по умолчанию - 1 января 0001 года
Значение по умолчанию — 1 января 0001 года 00:00:00.

**Отображение в интерфейсе:**
- **В поле ввода:** отображается как символы-разделители даты в соответствии с региональными установками и составом даты (дата и время, только дата, только время). 
  Можно задать собственное представление через параметр `ДП` в `ФорматРедактирования`.
  ![[Pasted image 20260302164215.png]]
    ![[Pasted image 20260302164203.png]]
    
- **В колонке таблицы:** по умолчанию отображается как пустое место. Можно задать собственное представление через параметр `ДП` в свойствах `Формат` и `ФорматРедактирования` колонки.

> [!NOTE] Максимальная дата
> 3999 год, 31.12.3999

---
## 3.3. Операции с датами

**Арифметические операции:**
- `Дата + Число` — дата, увеличенная на указанное количество секунд.
    
- `Дата - Дата` — разница между датами в секундах.
    
- `Дата - Число` — дата, уменьшенная на указанное количество секунд.
	![[Pasted image 20260302164242.png]]

**Логические операции:**
- Сравнение дат: `>`, `>=`, `=`, `<>`, `<`, `<=`.
	![[Pasted image 20260302164250.png]]

---
## 3.4. Преобразование дат
**Неявное преобразование** выполняется при необходимости, например, ***при выводе даты в сообщении — происходит автоматическое преобразование к строке*** согласно региональным установкам.

**Явное преобразование** возможно ==только в строку== с помощью функции `Строка(Дата)`. Результат зависит от региональных установок сеанса.

# 4. Булев тип (Булево)

Тип `Булево` предназначен для представления логических величин и имеет только два значения: **Истина** и **Ложь**.

## 4.1. Литералы булева типа

|Значение|Литералы|
|---|---|
|Истина|`Истина` или `True`|
|Ложь|`Ложь` или `False`|

---
## 4.2. Значение по умолчанию
Значением по умолчанию является **Ложь**.

---
## 4.3. Операции с булевыми значениями
**Операции сравнения:** `>`, `>=`, `=`, `<>`, `<`, `<=`. При сравнении считается, что `Истина` больше `Ложь`.

**Булевы операции:**
- `И` — истина, только если оба операнда истинны.
    
- `ИЛИ` — ложь, только если оба операнда ложны.
    
- `НЕ` — инверсия значения.
    

---
## 4.4. Преобразование булевых значений

### 4.4.1. Неявное преобразование
- **К числу:** `Истина` → `1`, `Ложь` → `0`.
    
- **К строке:** выполняется в соответствии с языком интерфейса платформы (например, для русского языка `Истина` → `"Да"`, `Ложь` → `"Нет"`).
    

### 4.4.2. Явное преобразование
- **В число:** функция `Число(Булево)` — `Истина` → `1`, `Ложь` → `0`.
    
- **В строку:** функция `Строка(Булево)` — возвращает строковое представление согласно языку интерфейса.
