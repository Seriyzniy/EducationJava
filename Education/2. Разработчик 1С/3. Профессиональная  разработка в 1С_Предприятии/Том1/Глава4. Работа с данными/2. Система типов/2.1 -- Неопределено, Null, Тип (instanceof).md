# 1. Значение «Неопределено» - пустота в значении
Значение типа `Неопределено` применяется, ==когда необходимо использовать «ПУСТОЕ ЗНАЧЕНИЕ»==, не принадлежащее ни к одному другому типу данных. 

Тип `Неопределено` имеет единственное значение — **Неопределено**.

Это значение используется, например, как значение составного типа по умолчанию, поэтому составной тип всегда содержит тип `Неопределено`.

Представлением значения `Неопределено` является пустая строка.

---
## 1.1. Литерал
Значение задаётся литералом **`Неопределено`**.

---
## 1.2. Операции
Для значения `Неопределено` определены **только операции сравнения** на равенство и неравенство со значениями других типов. 

Таким образом, всегда можно проверить, равна ли переменная `Неопределено`.

---
## 1.3. Преобразование
Значение `Неопределено` может быть преобразовано только в строку:
- **неявно** — например, при выводе;
    
- **явно** — функцией `Строка(Неопределено)`.

Результатом преобразования всегда является пустая строка.

# 2. Null - пустота в БД
Тип `Null` имеет единственное значение — **Null**. 

Оно используется системой для обозначения **отсутствующего значения** ==при работе с базой данных.==

## 2.1. Ситуации возникновения
Значение `Null` может возникать в различных случаях:
- при соединении нескольких таблиц в запросах, когда для одной из соединяемых таблиц нет записей, удовлетворяющих условиям;
    
- при обращении к реквизитам объектов, которые не используются. Например, у иерархического справочника реквизит, доступный только для элементов, при обращении к нему у группы будет возвращать `Null`.

---
## 2.2. Литерал
Значение задаётся литералом **`Null`**. Представлением значения является пустая строка.

---
## 2.3. Операции
Для значения `Null` определены **операции сравнения** на равенство и неравенство со значениями других типов. 

Таким образом, всегда можно проверить, равна ли переменная `Null`

---
## 2.4. Преобразование
Значение `Null` может быть преобразовано только в строку:

- **неявно** — например, при выводе;
    
- **явно** — функцией `Строка(Null)`.
    

Результатом преобразования всегда является пустая строка.

# 3. Тип - instanceof из Java

> [!IMPORTANT] ОПРЕДЕЛЕНИЕ ТИПОВ
> Значения типа `Тип` используются для идентификации различных типов значений и их сравнения.

## 3.1. Получение значения типа
Значения этого типа возвращаются двумя системными функциями:
- **Тип(ИмяТипа)** — возвращает значение типа, соответствующее ==переданному имени.== 
  Например: `Тип("Строка")`, `Тип("СправочникСсылка.Номенклатура")`.
    
- **ТипЗнч(Значение)** — возвращает тип ==переданного значения==. Например: `ТипЗнч("произвольная строка")`, `ТипЗнч(387.67)`.


> Также значения этого типа могут быть получены из объекта `ОписаниеТипов` методом `Типы()` в виде массива.

---
## 3.2. Операции
Для значений типа `Тип` определены **операции сравнения** на равенство и неравенство. 

> [!NOTE] Это позволяет проверять принадлежность произвольного значения заданному типу
> 

```java
Если ТипЗнч(ПроверяемаяПеременная) = Тип("Строка") Тогда
    // переменная имеет тип Строка
КонецЕсли;
```

---
## 3.3. Преобразование
Значения типа `Тип` могут быть ==преобразованы только в строку==:
- **неявно** — при выводе;
    
- **явно** — функцией `Строка(ЗначениеТипа)`.


Результат преобразования зависит от языка интерфейса платформы:
- `Тип("Строка")` → для русского языка: `"Строка"`, для украинского: `"Рядок"`, для латышского: `"Rinda"`;
    
- `Тип("СправочникСсылка.Номенклатура")` → для русского: `"Справочник ссылка: Номенклатура"`, для казахского: `"Анықтама сілтеме: Номенклатура"` и т.д.
