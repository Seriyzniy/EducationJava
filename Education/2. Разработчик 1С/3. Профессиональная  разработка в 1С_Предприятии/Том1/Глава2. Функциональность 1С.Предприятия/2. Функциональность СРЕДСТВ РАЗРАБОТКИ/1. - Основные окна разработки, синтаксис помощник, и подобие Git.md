# 1. Объекты конфигурации и инструменты для их редактирования
## 1.1 Дерево объектов конфигурации
**Дерево объектов конфигурации** — основной инструмент. Представляет состав конфигурации в виде древовидной структуры, каждая ветвь описывает определённую составляющую. Позволяет выполнять различные действия над объектами.
![[Pasted image 20260226184842.png]]

---
## 1.2 Окно редактирования объекта конфигурации
**Окно редактирования объекта конфигурации** — обеспечивает последовательную установку свойств. Удобно для начинающих, так как позволяет сначала задать свойства, влияющие на наличие или отсутствие других свойств.
![[Pasted image 20260226184947.png]]

---
## 1.3 Палитра свойств
**Палитра свойств** — даёт быстрый доступ к информации, связанной с объектом, и возможность изменять свойства. Удобна для опытных разработчиков и при работе с одноимёнными свойствами разных объектов.
![[Pasted image 20260226185032.png]]

---
## 1.4 Окно «Дополнительно»
**Окно «Дополнительно»** — отображает в едином списке свойства объекта, относящиеся к разным объектам конфигурации. 
![[Pasted image 20260226185111.png]]

Здесь можно:
- настраивать вхождение команд в командный интерфейс;
    
- просматривать и изменять включение объекта в подсистемы;
    
- редактировать связи с функциональными опциями;
    
- настраивать права и выполнять другие действия.

## 1.5 Конструкторы и редакторы
### Конструкторы
**Конструкторы** автоматизируют создание часто используемых элементов, модификация которых может быть формализована. С их помощью можно быстрее и легче создавать то, что можно создать и вручную, но конструктор упрощает процесс.

> Визуальный интерфейс для написания запросов, создания форм и макетов и прочего

**Пример:** конструктор запросов позволяет визуально выбирать таблицы, поля и связи — после нажатия ОК создаётся синтаксически верный текст запроса.

**Основные конструкторы:**
- конструктор запроса;
    
- конструктор запроса с обработкой результата;
    
- конструктор движений регистров;
    
- конструктор печати;
    
- конструктор ввода на основании;
    
- конструктор форм;
    
- конструктор макета;
    
- конструктор форматной строки;
    
- конструктор строк на разных языках;
    
- конструктор агрегатов;
    
- конструктор схемы компоновки данных;
    
- конструктор настроек компоновки данных;
    
- конструктор макета оформления компоновки данных;
    
- конструктор ограничения доступа к данным.

### Редакторы - изменение того, что создал конструктор, при помощи UI
**Редакторы** предназначены для изменения элементов, модификацию которых формализовать затруднительно или не имеет смысла.

**Пример:** конструктор форм создаёт типовую форму, а редактор форм позволяет придать ей нужный вид, расположить элементы управления, настроить привязки в соответствии с конкретным назначением.

**Основные редакторы:**
- редактор форм;
    
- редактор текстов и модулей;
    
- редактор командного интерфейса конфигурации;
    
- редактор рабочей области рабочего стола;
    
- редактор командного интерфейса рабочего стола;
    
- редактор командного интерфейса;
    
- редактор «Все подсистемы»;
    
- редактор роли;
    
- редактор «Все роли»;
    
- редактор «Все ограничения доступа»;
    
- редактор табличных документов;
    
- редактор графической схемы;
    
- редактор картинок;
    
- HTML-редактор.

# 2. Синтакс-помощник
Синтакс-помощник предоставляет справку по:
- конструкциям встроенного языка;
    
- свойствам, методам и событиям объектов встроенного языка (как платформенных, так и создаваемых наследованием от прототипов объектов конфигурации);
    
- составу таблиц — источников данных в языке запросов.

![[Pasted image 20260226185711.png]]

**Режимы использования:**
- Самостоятельно — для изучения возможностей языка.
    
- В связке с редактором текстов и модулей:
    - из **текста модуля можно перейти к описанию свойства** (*если выделить нужный фрагмент кода и Ctrl+F1*), метода или функции в синтакс-помощнике (например, сочетанием клавиш);
        
    - конструкции из синтакс-помощника можно переносить в текст модуля с помощью мыши.

# 3. Отладчик и режим замера производительности

**Отладчик** позволяет анализировать работу прикладного решения:
- останавливать выполнение в указанном месте кода;
    
- просматривать значения переменных и состояние объектов в конкретные моменты;
    
- выполнять код по шагам.


**Режим замера производительности** используется для анализа эффективности участков кода:
- измеряется время выполнения каждой строки (абсолютное и относительное, в процентах к общему времени);
    
- позволяет выявить операторы, занимающие больше всего времени, и оптимизировать алгоритм.

# 4. Проверка конфигурации - ПОДОБИЕ Git
Режим выявляет ошибки, не критичные для работы в целом, но способные снизить производительность или вызвать сбои в специальных режимах. 

Использование необязательно, но полезно:
- перед поставкой заказчику;
    
- перед выпуском тиражного решения;
    
- после массового удаления объектов;
    
- после объединения конфигураций.

## 4.1 Сравнение/объединение конфигураций

> [!IMPORTANT] Git решение в 1С по решению конфликтов конфигов
> 

Механизм позволяет:
- детально анализировать отличия между версиями прикладного решения;
    
- выполнять переход от одной версии к другой.

Разработчик может задавать правила объединения для каждого элемента:
- взять из новой версии;
    
- оставить без изменений;
    
- объединить (сохранить оба варианта).


После нажатия _Выполнить_ платформа автоматически объединяет конфигурации по заданным правилам. Это исключает ручную модификацию (и связанные с ней ошибки) и значительно ускоряет перенос изменений.
