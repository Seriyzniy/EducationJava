Средства администрирования, входящие в состав интерфейса разработки и администрирования, позволяют управлять составом пользователей информационной базы, осуществлять регламентные операции по ее обслуживанию и выполнять другие административные действия (рис. 2.9). 

Перечислим основные средства, используемые для администрирования информационных баз «1С:Предприятия».

# 1. Список пользователей
Платформа позволяет **вести список пользователей**, которым разрешена работа с данной информационной базой. 

Этот список не является частью прикладного решения, а создается отдельно, в каждой конкретной организации, где используется решение.

Возможности работы со списком пользователей:
- добавление и удаление пользователей;
    
- назначение различных видов аутентификации;
    
- определение набора доступных ролей;
    
- выбор режима запуска приложения;
    
- установка языка пользователя и других параметров.

---
# 2. Механизмы аутентификации
Механизм аутентификации позволяет определить, какой именно пользователь из списка подключается к прикладному решению. Эта информация используется для выбора соответствующего командного интерфейса, разрешения доступа к данным и т.д.

## Система поддерживает два вида аутентификации:
**Аутентификация средствами «1С:Предприятия»** — в конфигураторе для каждого пользователя задаётся пароль. При начале работы пользователь выбирает или вводит имя и соответствующий пароль.

**Аутентификация средствами операционной системы** — в конфигураторе пользователю ***сопоставляется учётная запись ОС***. Система определяет, от имени какого пользователя ОС выполняется подключение, и находит соответствующего пользователя 1С. 
Диалог аутентификации 1С при этом не отображается (если не указан специальный параметр командной строки).

---
# 3. Механизм заданий
Механизм заданий предназначен для выполнения ***административных действий по расписанию.*** Он включает фоновые и регламентные задания. Каждое задание — **последовательность действий на встроенном языке,** ==описанная в общих модулях.==

**Фоновые задания** позволяют запускать выполнение процедур общих модулей асинхронно, без ожидания завершения.

**Регламентные задания** обеспечивают автоматический вызов процедур общих модулей по расписанию. Они описываются на этапе конфигурирования и являются частью прикладного решения.

> Фоновые задания реализуются средствами встроенного языка.

---
# 4. Система прав доступа
Для каждого прикладного решения можно описать **наборы прав**, соответствующие должностям **пользователей** или виду деятельности. Структура прав определяется конкретным прикладным решением.

## Группы прав
Все права делятся на две группы:

**Основные права** описывают *действия над элементами данных* или системой в целом. Они проверяются всегда, независимо от способа обращения к данным.

**Интерактивные права** описывают ***действия, выполняемые пользователем интерактивно.*** Они проверяются только ***при выполнении интерактивных операций стандартными способами***. В клиент-серверном варианте все проверки прав (кроме интерактивных) выполняются на сервере.

---
# 5. Ограничение прав на уровне записей и полей
Д***ля действий над объектами базы*** данных **могут задаваться дополнительные условия на данные** (ограничение доступа к данным). 

К таким действиям относятся:
- чтение (получение записей или их фрагментов);
- добавление новых записей;
- изменение существующих записей;
- удаление записей.

**Ограничение доступа представляет собой условие на языке, являющемся подмножеством языка запросов.** Оно применяется для каждой записи, над которой выполняется операция. При просмотре списков и формировании отчётов можно обеспечить отображение только тех данных, доступ к которым разрешён пользователю.

Аналогичные условия могут задаваться и для таблиц, не имеющих объектной природы (регистров).

---
# 6. Журнал регистрации - логирование по уровням
Для хранения информации о событиях в информационной базе **ведётся журнал регистрации**. 

Администратор может настроить уровень событий, отображаемых в журнале: 
- ошибки, 
- предупреждения, 
- информация, 
- примечания.

Просмотр журнала возможен с помощью фильтра по множеству критериев. Например, можно отобрать информацию о том, какие документы изменял определённый пользователь в заданный промежуток времени.

Журнал может быть полностью или частично выгружен в формат XML для программного анализа.

---
# 7. Загрузка/выгрузка информационной базы
Команды загрузки/выгрузки позволяют сохранить информационную базу в файл на диске или загрузить базу из файла. Используются, например, для переноса базы из файлового варианта в клиент-серверный и обратно.

---
# 8. Утилита администрирования кластера серверов
Утилита используется при работе в клиент-серверном варианте и позволяет выполнять следующие действия:
- создание, изменение и удаление кластеров серверов;
    
- модификация существующих кластеров: создание, изменение параметров и удаление рабочих серверов, рабочих процессов кластера, информационных баз, входящих в состав кластера;
    
- управление списками администраторов центральных серверов кластеров и списками администраторов кластеров;
    
- мониторинг соединений пользователей с информационными базами и служебных соединений;
    
- отключение пользователей от информационной базы;
    
- мониторинг объектных блокировок «1С:Предприятия» и блокировок клиентского соединения;
    
- оперативный анализ транзакционных блокировок системы управления базами данных;
    
- управление блокировкой установки соединений пользователей с информационной базой;
    
- управление блокировкой выполнения регламентных заданий.

Утилита представляет собой подключаемый модуль MMC (Microsoft Management Console) и может использоваться на компьютерах с соответствующим ПО.

# 9. «1С:Корпоративный инструментальный пакет» (1С:КИП)
Продукт предназначен для повышения производительности, масштабируемости и надежности информационных систем на платформе «1С:Предприятие». 

> Используется **для нагрузочного тестирования** и сопровождения крупных корпоративных систем.

## Инструменты
Включает следующие инструменты:
**1С:Центр управления производительностью** — инструмент ***мониторинга и анализа производительности***. Позволяет оценивать производительность системы, собирать техническую информацию о проблемах и анализировать её для дальнейшей оптимизации.

**1С:Тест-центр** — инструмент автоматизации многопользовательских ***нагрузочных испытаний***. Позволяет моделировать работу предприятия без участия реальных пользователей, оценивая применимость, производительность и масштабируемость системы в реальных условиях.

**1С:Центр контроля качества** — инструмент для обеспечения эффективной и надёжной работы системы за счёт своевременного и правильного технического обслуживания. Особенно важен для внедрений с высокими требованиями к производительности и надёжности.
