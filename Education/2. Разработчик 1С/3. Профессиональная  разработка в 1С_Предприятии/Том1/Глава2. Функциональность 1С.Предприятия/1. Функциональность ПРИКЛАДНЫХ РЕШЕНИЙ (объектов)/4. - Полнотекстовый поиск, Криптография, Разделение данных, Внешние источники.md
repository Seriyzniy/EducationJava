# 1. Механизм полнотекстового поиска
Механизм позволяет пользователю ***быстро находить нужную информацию в базе*** данных и справочной системе. Например, можно найти все вхождения заданного слова или сочетания слов, при этом найденные фрагменты выделяются цветом.

**Принцип работы:**
- Основан на **полнотекстовом индексе** и **средствах выполнения поиска**.
    
- Поиск осуществляется по данным, содержащимся в индексе.
    
- Индекс создаётся однократно и периодически обновляется (интерактивно или программно).

![[Pasted image 20260226183819.png]]

**Какие объекты участвуют:**  
Планы обмена, справочники, документы, планы видов характеристик, планы счетов, планы видов расчета, регистры (сведений, накопления, бухгалтерии, расчета), бизнес-процессы и задачи.

**Настройка:**
- Возможно выборочное включение объектов и их реквизитов в поиск.
    
- Можно ограничить область поиска только указанными объектами.
    

**Особенности:**
- Поиск выполняется с учётом прав пользователя (недоступная информация не находится).
    
- Результаты возвращаются порциями, отсортированными по релевантности — наиболее вероятные данные попадают в начало.
    
- Поддерживается нечёткий поиск (с порогом нечёткости), поиск с учётом синонимов (русский, английский, украинский), использование дополнительных словарей.

# 2. Механизм криптографии
Механизм позволяет прикладным решениям **выполнять криптографические операции с данными информационной базы**: зашифровывание/расшифровывание, формирование и проверку электронной цифровой подписи. Это обеспечивает высокую конфиденциальность и аутентичность информации.

**Особенности реализации:**
- Не содержит собственных алгоритмов криптографии.
    
- Предоставляет объекты для взаимодействия с внешними криптопровайдерами.
    
- В Windows используется интерфейс **CryptoAPI** — поддерживаются любые криптопровайдеры, совместимые с этим интерфейсом.
    
- В Linux реализовано непосредственное взаимодействие с установленными компонентами.

# 3. Механизм разделения данных
Механизм позволяет ***хранить в одной информационной базе*** **данные нескольких независимых организаций**, ***используя при этом одну и ту же конфигурацию.***

> Подозрение на возможное разделение данных в БД при помощи схем или пространств имен

**Режимы разделения:**
- **Полное разделение** — данные разных организаций полностью изолированы.
    
- **Частичное разделение** — часть данных может быть общей для всех (например, справочник банков), а часть — индивидуальной (хозяйственная и финансовая деятельность).

**Преимущества:**
- Значительно упрощает администрирование и обновление прикладных решений.
    
- Особенно полезно, когда одно решение используется для ведения учёта множества организаций (например, аутсорсинговые бухгалтерские услуги).

# 4. Внешние источники данных
Платформа позволяет работать **с внешними источниками данных** ==через ODBC (OpenDataBaseConnectivity==. 
> Это необходимо для использования средств 1С с уже существующими на предприятии базами данных или для получения данных из внешних БД.

**Особенности:**
- Описываются в конфигураторе в отдельной ветви объектов (таблицы, поля, связи).
    
- Для СУБД, поддерживаемых 1С (SQL Server, PostgreSQL, IBM DB2, Oracle), доступны полные возможности платформы (запросы, отчёты и т.д.).
    
- Для других СУБД через ODBC возможности могут быть ограничены стандартами самой СУБД.


**Поддержка в механизмах платформы:**  
Запросы, отчёты на СКД, динамические списки, управляемые формы, встроенный язык.

**Ограничения:**
- ***Данные внешних источников доступны только для чтения.***
    
- Реквизиты объектов конфигурации могут ссылаться на объектные таблицы внешних источников (ссылочный тип).
    

**Подключение:**
- ***Параметры соединения задаются в параметрах сеанса***, пользователя или общих параметрах.
    
- *Управление подключением/отключением и настройка параметров* выполняется **через стандартную обработку** `Управление внешними источниками данных`.
    
- Подключение происходит автоматически при обращении к данным внешнего источника (например, в запросе).
