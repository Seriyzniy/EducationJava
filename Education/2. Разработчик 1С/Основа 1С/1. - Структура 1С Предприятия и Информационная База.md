# 1. Структура 1С:Предприятия

> [!NOTE] Что такое 1С:Предприятие
> **«1С:Предприятие»** — универсальная система для автоматизации бизнеса. 

Её ключевая особенность — **конфигурируемость**, т.е. возможность адаптации к разным отраслям и задачам предприятия.
## Две основные части системы 1С:Предприятие
Система состоит из двух частей:
1. **Платформа** — «движок», обеспечивающий работу.
    
2. **Конфигурация** — прикладное решение, описывающее конкретную область учёта (бухгалтерия, торговля, зарплата).

> [!NOTE] Конфигурация в 1С
> ==Конфигурация в 1С== — это комплекс объектов, настроек и программного кода, который определяет, как работает конкретное прикладное решение. 

**Аналогия:**  
Платформа — это **проигрыватель**, конфигурация — **компакт-диск**.
- Без диска проигрыватель бесполезен.
- Без проигрывателя диск не прослушать.

Для работы всегда нужны **обе части**: одна платформа + одна конфигурация.
- Платформа без конфигурации не решает бизнес-задач.
- Конфигурация без платформы не может функционировать.

---
## **Конфигурируемость и** 
==Конфигурация и прикладное решение== — это синонимы. Разница только в контексте:
- **Конфигурация** — термин для разработчика.
- **Прикладное решение** — понятный термин для пользователя.

В качестве примера существующих прикладных решений можно перечислить следующие типовые решения: 
- «1С:Бухгалтерия 8», 
- «1С:Управление небольшой фирмой 8»,
- «1С:Управление торговлей 8», 
- «1С:Зарплата и управление персоналом 8», 
- «1С:Управление производственным предприятием 8», «1С:Налогоплательщик 8», 
- «1С:Документооборот 8», 
- «1С:Консолидация 8».

### Возможность изменять типовое решение
**Типовые решения** универсальны, но **могут не полностью отвечать нуждам конкретного предприятия.** 

Благодаря **конфигурируемости** платформы можно:
- Изменять существующую конфигурацию.
- Создавать свою конфигурацию с нуля.

**Аналогия:** Как проигрыватель компакт-дисков позволяет редактировать треки и записывать свои мелодии, платформа «1С» даёт инструменты для настройки и создания конфигураций под конкретные задачи.

---
## Два режима работы
Система «1С:Предприятие» имеет два основных режима работы:
1. **«1С:Предприятие»** — для работы пользователей: ввод данных, обработка, получение отчётов.
    
2. **«Конфигуратор»** — для разработчиков и администраторов: создание и изменение конфигураций.
    

Эта книга в основном посвящена работе в режиме **«Конфигуратор»**. Режим «1С:Предприятие» будет использоваться только для проверки результатов.

# 2. Информационная база
После установки 1С:Предприятия, мы сталкиваемся с информационной базой
![[Pasted image 20260209165320.png]]

> [!NOTE] Информационная База
> **Информационная база 1С** — это ==ХРАНИЛИЩЕ ДАННЫХ + ПРИКЛАДНАЯ ЛОГИКА конкретного приложения 1С.==

Проще:
> **Информационная база = данные + структура + поведение системы**

Когда ты запускаешь **1С:Предприятие**, тебе предлагают **создать или выбрать ИБ**, потому что:
- без ИБ 1С не знает, **какое приложение запускать**
- и **где хранятся данные**

## Что внутри информационной базы
ИБ — это **не только таблицы**, как в обычной БД.

Внутри ИБ есть:
### 1️⃣ Конфигурация (метаданные)
Это описание **что за система вообще**:
- справочники (Контрагенты, Номенклатура…)
    
- документы (Поступление, Реализация…)
    
- регистры (накопления, сведений, бухгалтерии)
    
- роли, формы, отчёты
    
- модули с кодом на языке 1С

📌 Это **аналог схемы БД + бизнес-логики**, но сильно богаче.

---
### 2️⃣ Данные
Это уже **конкретные записи**:
- элементы справочников
    
- проведённые документы
    
- записи регистров
    
- пользователи, настройки и т.д.

#### Информационная база = реляционная БД?
**Ответ: и да, и нет.**
##### ✔ Да
Физически данные **хранятся в СУБД**:
- PostgreSQL  
- MS SQL Server
- Oracle
- файловый вариант (собственный формат)

И внутри — **таблицы, индексы, транзакции**.

---
### 3️⃣ Служебная информация
Всякая внутренняя кухня 1С:
- версии конфигурации
    
- блокировки
    
- кэш
    
- служебные таблицы платформы

---
## 1. Добавление Информационной Базы (ИБ)
Для выполнения примеров из книги нужна ИБ для разработки новой конфигурации, а не созданная из шаблона.

**Последовательность действий:**
1. **Запустите «1С:Предприятие»**.
    
2. В окне списка информационных баз нажмите **«Добавить»**.

3. Выберите **«Создание новой информационной базы»** → **Далее**.
![[Pasted image 20260209170951.png]]

4. Выберите **«Создание информационной базы без конфигурации...»** → **Далее**.
![[Pasted image 20260209171023.png]]

5. Укажите **наименование** ИБ и выберите тип расположения **«На данном компьютере...»** → **Далее**.
![[Pasted image 20260209171114.png]]

6. Укажите **каталог** для размещения ИБ (язык по умолчанию — Русский) → **Далее**.
![[Pasted image 20260209171237.png]]

![[Pasted image 20260209171316.png]]

7. Нажмите **«Готово»**.

Созданная пустая информационная база появится в списке для запуска.
![[Pasted image 20260209171553.png]]

### Внутри ТА ЖЕ БД что и обычная реляционная БД
#### База данных с расширением `1CD`
Если перейти по пути созданной Информационной базы, то можно заметить файл с расширением `1CD` - ==это файл есть сама БД==
![[Pasted image 20260209171835.png]]

А остальные файлы не имеют особого значения. И при переносе БД, следует переносить именно файл с расширением `1CD`

#### Папка журнала действий (аналог журнала транзакций) `1Cv8Log`
А в папке 1Cv8Log находится журнал все действий пользователя

## 2. Удаление Информационной Базы
Что бы удалить базу, ее необходимо удалить ее папку! Удалить из окна программы 1С нельзя - тогда база удалится только из списка

Поэтому, необходимо скопировать путь Базы из програмы
![[Pasted image 20260209174937.png]]

И **по этому пути удалить папку с базой**, только после чего в программе 1С выбрать базу, и нажать Удалить, тогда база удалится и из списка доступных баз
