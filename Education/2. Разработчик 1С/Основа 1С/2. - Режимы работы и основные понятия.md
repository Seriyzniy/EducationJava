# 1. Режимы работы с 1С:Предприятием
У системы 1С:Предприятие есть два режима работы
- `Конфигуратор` - для разработчика
- `Платформа (1C:Предприятие)` - для обычного пользователя

![[Pasted image 20260209172507.png]]

## Запуск в режиме Платформы
Если запустить в базовом режиме, то откроется пустое окно, с некоторыми предустановленными кнопками
![[Pasted image 20260209172700.png]]

## Запуск в режиме конфигуратора
В основном, в режиме конфигуратора мы будем работать с вкладкой `Конфигурация`, которая открывается не сразу

Чтобы ее открыть необходимо
1) Перейти во вкладку Конфигурация
2) Нажать на кнопку "Открыть конфигурацию"
![[Pasted image 20260209172926.png]]

И тогда откроется окно слева
![[Pasted image 20260209172959.png]]

### Предустановленные конфигурации
Конфигурации можно создавать самим, а можно использовать готовые.

В данном случае, у нас нет лицензии на конкретную конфигурацию, а также мы не работаем в какой-либо компании, которая бы уже имела определенную конфигурацию и лицензию к ней.

Поэтому при установке учебной версии, компания 1С предоставила нам бесплатно несколько типовых конфигураций

![[Pasted image 20260209173444.png]]

### Установка ГОТОВОЙ конфигурацией
Для того, чтобы установить определенную конфигурацию, нам необходимо скачать эту конфигурацию (или другим способом получить), и после чего запустить установочный файл (setup.exe)
![[Pasted image 20260209173627.png]]

И там просто протыкать "Далее"

И тогда, при создании НОВОЙ информационной базы, у нас появится возможность выбрать базу из шаблона
![[Pasted image 20260209173914.png]]

Мы установим ДЕМО версию базы (с уже заготовленными данными), и запустим ее
> При запуске потребуют войти в учетку, но там есть предустановленные Антон и Ирина (просто слева нужно нажать на треугольник для выбора)

В результате мы увидим программу, в которой приведена отчетность и множество других вкладок "Операции", "Календарь", "Шаблон операций" и т.д
![[Pasted image 20260209174333.png]]

Если зайти в эту конфигурацию в режиме Конфигуратора, то можно увидеть множество настроек
![[Pasted image 20260209174555.png]]

# 2. Основные понятия
После установки 1С:Предприятия, и его открытии в режиме конфигуратора, для конфигурации будут доступны следующие элементы из категорий:
![[Pasted image 20250103203454.png]]

Объяснение будет в виде аналогии с языком программирования Java. Смотри следующие конспекты [[2.1 -- Вкладка Общее и ее содержимое]]




---

- **Общие**  
    В 1С это ==элементы, которые применяются в различных частях конфигурации==: общие модули, общие формы, общие реквизиты и т.д.  
    **Аналогия:** в ООП это может быть базовый класс или библиотека, которая используется везде.
    
- **Константы**  
    Это глобальные параметры, которые остаются неизменными (или редко меняются) и доступны везде.  
    **Аналогия:** статические поля в классе, или глобальные переменные.
    
- **Справочники**  
    Это ==структуры для хранения данных, которые используются многократно==. 
    Например, справочник "Сотрудники" может содержать записи обо всех сотрудниках компании.  
    **Аналогия:** классы с коллекцией объектов, например `List<Employee>` в Java или C++.
    
- **Документы**  
    ==Документы в 1С используются для фиксации событий или операций==. Например, "Поступление товаров".  
    **Аналогия:** объекты, представляющие транзакции или действия, например `Transaction` в классическом программировании.
    
- **Журналы документов**  
    Это списки документов одного или нескольких типов для упрощения работы с ними.  
    **Аналогия:** коллекции объектов типа "документ" с фильтрацией и группировкой.
    
- **Перечисления**  
    Это фиксированный набор значений. Например, "Типы оплаты: Наличные, Безналичные".  
    **Аналогия:** `enum` в Java и C++.
    
- **Отчеты**  
    ==Это механизм для генерации итоговой информации== (вывод данных в заданной структуре).  
    **Аналогия:** классы или модули, формирующие отчет на основе данных.
    
- **Обработки**  
    ==Внешние или внутренние программы для выполнения каких-либо действий==, например, загрузка данных из файла.  
    **Аналогия:** утилиты, вспомогательные классы или скрипты.
    
- **Планы видов характеристик**  
    Это механизм для описания дополнительной аналитики. Например, план "Характеристики товаров".  
    **Аналогия:** добавление метаданных к объектам (например, через аннотации или словари).
    
- **Планы счетов**  
    Используются для ведения бухгалтерского учета. ==Это структура, описывающая счета==.  
    **Аналогия:** классы или структуры данных для представления бухгалтерской логики.
    
- **Планы видов расчета**  
    ==Применяются для расчета зарплат, бонусов ==и т.д.  
    **Аналогия:** методы или классы для вычислений, связанных с оплатой труда.
    
- **Регистры сведений**  
    ==Для хранения и анализа информации, не имеющей прямого отношения к бухгалтерии==.  
    **Аналогия:** базы данных или коллекции, содержащие вспомогательные данные.
    
- **Регистры накопления**  
    Для учета количественной или суммовой информации (например, склад).  
    **Аналогия:** таблицы базы данных для учета остатков.
    
- **Регистры бухгалтерии**  
    Предназначены для отражения бухгалтерских операций.  
    **Аналогия:** специализированные таблицы учета или классы.
    
- **Регистры расчета**  
    Для расчета и хранения данных, связанных с зарплатой.  
    **Аналогия:** классы для хранения сложных вычисляемых данных.
    
- **Задачи**  
    Это часть функционала 1С для управления бизнес-процессами и постановки задач пользователям.  
    **Аналогия:** элементы workflow или планировщики задач.
    
- **Внешние источники данных**  
    Позволяют подключать внешние базы данных или файлы.  
    **Аналогия:** подключения к внешним API или базам данных в программировании.


**"Подсистема"** — ==это инструмент для объединения и структурирования функциональности== внутри конфигурации. Она помогает группировать объекты (документы, справочники, отчеты и т.д.) по смысловым или логическим блокам, чтобы упростить работу пользователей и разработчиков.

Подсистема в 1С ==похожа на **модуль или пакет (package)**== в языке программирования ==Java==
