## 2. Приведение типов TODO
Главное, что нужно запомнить: **1С — язык с динамической типизацией, но с контролем типов**. 

Это значит, что переменная не имеет жёстко закреплённого за ней типа. Тип переменной определяется тем значением, которое в неё записано в данный момент, и может меняться по ходу выполнения программы

Однако, когда мы говорим о реквизитах справочников, документов и других объектов метаданных, их типы задаются жёстко при создании в конфигураторе. 
Платформа будет строго следить, чтобы в реквизит не попало значение неподходящего типа.
> `Реквизиты` имеют СТРОГО зафиксированный тип, который проверяется платформой

Исходя из этой особенности, можно выделить два основных сценария: **преобразование** (конвертация) значений и **приведение** (уточнение) типа.

### Преобразование типов (конвертация)
Это процесс, когда вы берете значение одного типа и получаете из него новое значение другого типа. 
> Например, из строки "123" получаете число 123. 

В 1С для этого есть специальные функции-конструкторы. Важно понимать, что платформа пытается выполнить преобразование, и если это невозможно, возникнет ошибка.

- **Число()** — пытается преобразовать переданное значение в число.
    - `Число("15.5")` → `15.5`
    - `Число("15,5")` → `15.5` (запятая тоже работает)
    - `Число("abc")` → **ошибка**
    
- **Строка()** — преобразует значение в строку. Пожалуй, самый безопасный и часто используемый способ.
    - `Строка(153)` → `"153"`
    - `Строка(ТекущаяДата())` → `"2023.10.26 14:05:30"`
    - `Строка(Справочники.Номенклатура.НайтиПоКоду("0001"))` → получим ***строковое представление ссылки*** (обычно наименование).
        
- **Дата()** — преобразует строку или число в дату.
    - `Дата("2023.10.26")` → `'2023.10.26 00:00:00'`
    - `Дата("20231026143025")` → `'2023.10.26 14:30:25'`
        
- **Булево()** — преобразует значение в булево (`Истина` или `Ложь`).
    - `Булево(1)` → `Истина`
    - `Булево(0)` → `Ложь`
    - `Булево("Истина")` → `Истина` (работает со строками)

# 4. Базовые операторы в 1С 

## **Основные операторы сравнения в 1С**

|**Оператор**|**Описание**|**Пример**|
|---|---|---|
|`=`|Равно|`Если А = 10 Тогда`|
|`<>`|Не равно|`Если А <> 10 Тогда`|
|`>`|Больше|`Если А > 10 Тогда`|
|`<`|Меньше|`Если А < 10 Тогда`|
|`>=`|Больше или равно|`Если А >= 10 Тогда`|
|`<=`|Меньше или равно|`Если А <= 10 Тогда`|
***
## **Основные логические операторы в 1С**

| **Оператор** | **Описание**          | **Пример**                                   |
| ------------ | --------------------- | -------------------------------------------- |
| **`И`**      | Логическое "И" (AND)  | Истина, если оба условия истинны             |
| **`ИЛИ`**    | Логическое "ИЛИ" (OR) | Истина, если хотя бы одно из условий истинно |
| **`НЕ`**     | Логическое "НЕ" (NOT) | Инвертирует значение условия                 |

# 5. Базовые конструкции

## 1. Условные операторы
```bsl
Если Условие1 Тогда
      // Код для Условие1
ИначеЕсли Условие2 Тогда
      // Код для Условие2
Иначе
      // Код для всех остальных случаев
КонецЕсли;

```

Пример
```bsl
Если Число > 20 Тогда
    Сообщить("Число больше 20");
ИначеЕсли Число > 10 Тогда
    Сообщить("Число больше 10, но меньше или равно 20");
Иначе
    Сообщить("Число меньше или равно 10");
КонецЕсли;
```

## 2. Циклы
##### Конструкция `Для ... ПО ... Цикл`

```bsl
Для переменная = Начало По предел_число Цикл
    // Код цикла
КонецЦикла;
```

Пример
```bsl
Для Число = 1 По 5 Цикл
    Сообщить("Число: " + Число);
КонецЦикла;
```

***
##### Конструкция `Для Каждого ... ИЗ ... Цикл`
```bsl
Для Каждого Элемент Из Коллекция Цикл
    // Код цикла
КонецЦикла;
```

Пример:
```bsl
МойМассив = Новый Массив;

МойМассив.Добавить("Первый");
МойМассив.Добавить("Второй");
МойМассив.Добавить("Третий");

Для Каждого МойЭлемент Из МойМассив Цикл
    Сообщить(МойЭлемент);
КонецЦикла;
```

***
##### Конструкция `Пока`
```bsl
Пока Условие Цикл
    // Код цикла
КонецЦикла;
```

Пример
```bsl
Перем Число;
Число = 5;

Пока Число > 0 Цикл
    Сообщить("Число: " + Число);
    Число = Число - 1;
КонецЦикла;
```

## 3. Операторы работы с ошибками

##### Конструкция `Попытка ... Исключение`

Попытка

# 6. Процедуры и функции

### Процедура
Используется для выполнения действий без возврата значения.

```bsl
Процедура ИмяПроцедуры(Параметры)
    // Код процедуры
КонецПроцедуры
```

Пример
```bsl
Процедура Приветствие()
    Сообщить("Привет, мир!");
КонецПроцедуры
```

***
### Функция
Возвращает значение в месте вызова.
```
Функция ИмяФункции(Параметры)
    // Код функции
    Возврат Значение;
КонецФункции
```

Пример
```
Функция Сложить(Число1, Число2)
    Возврат Число1 + Число2;
КонецФункции
```

***
## **Вызов процедуры или функции**

Для вызова процедур и функций используется их имя.
```
Приветствие(); // Вызов процедуры

Результат = Сложить(5, 10);
Сообщить("Результат: " + Результат);
```

# 7. Оператор "new"

> [!NOTE] Аналог оператора `new`
> В 1С для создания более сложных типов данных используется оператор `Новый`
> ```bsl
> Перем МойМассив; 
> МойМассив = Новый Массив;
> ```
> 
> При использовании оператора `Новый` создавать переменную с помощью `Перем` НЕ НУЖНО.
> ```bsl
> МойМассив = Новый Массив;
> ```
### Сравнение с Java

|**Особенности**|**1С (`Новый`)**|**Java (`new`)**|
|---|---|---|
|**Цель**|Создает объект определенного типа.|Создает объект класса или массива.|
|**Указание типа**|Используется встроенный тип (например, `Массив`, `Структура`).|Используется пользовательский класс или встроенный тип.|
|**Типизация**|Динамическая (не нужно явно указывать тип переменной).|Статическая (тип объекта должен быть задан).|
|**Использование**|Работает с объектами метаданных 1С и встроенными типами.|Работает с пользовательскими классами и массивами.|
### Типы данных, которые можно создать с помощью `Новый` в 1С

1. Массив
```bsl
 МойМассив = Новый Массив;
```

2. Структура
```bsl
МояСтруктура = Новый Структура("Имя", "Возраст");
```

3. СписокЗначений
```bsl
МойСписок = Новый СписокЗначений; 
МойСписок.Добавить("Элемент 1");
```

4. ТаблицаЗначений
```bsl
МояТаблица = Новый ТаблицаЗначений; 
МояТаблица.Колонки.Добавить("Имя");
```

5. Соответствие
```bsl
МоеСоответствие = Новый Соответствие;
МоеСоответствие.Вставить("Ключ", "Значение");
```

6. ДвоичныеДанные
```bsl
Данные = Новый ДвоичныеДанные(ПолучитьФайл("путь_к_файлу"));
```

7. **Объекты метаданных** (например, Справочники, Документы и т.д.)
```bsl
Объект = Справочники.Номенклатура.СоздатьЭлемент();
```

# 8. **Способы работы с коллекциями**
Многие объекты встроенного языка 1С являются коллекциями. Существует два основных способа доступа к их элементам:

1. **Перебор в цикле** (для любой коллекции):
```text
Для Каждого Элемент Из Коллекция Цикл
	// действия с Элемент
КонецЦикла;
```
    
2. **Прямой доступ к элементу** (без перебора):
- **По имени** — для именованных коллекций (например, справочники, документы): 
```text
Справочники.Сотрудники
Справочники["Сотрудники"]
```

- **По индексу** — для неименованных коллекций (например, строки табличной части, колонки таблицы значений). Индекс первого элемента — 0:
```text
ТабличнаяЧасть[0]  // первая строка
```
    

Некоторые коллекции поддерживают оба способа (например, колонки таблицы значений — можно обращаться и по имени, и по индексу).
