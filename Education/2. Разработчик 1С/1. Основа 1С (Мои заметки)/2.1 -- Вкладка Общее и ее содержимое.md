# 1. Общее - Инфраструктура приложения (платформы)
Это **глобальные объекты**, которые:
- не принадлежат конкретному справочнику или документу
- доступны **из любой части системы**
- ==ФОРМИРУЮТ ИНФРАСТРУКТУРУ приложения==

Можно думать так:
> **«Общее» = application-level слой**

## 1. Подсистемы - `package в Java`
==Логическое разбиение функциональности системы==.
- используются для навигации в интерфейсе
- помогают структурировать конфигурацию
- не хранят данные

**Пример:**  
Подсистема «Продажи», «Закупки», «Склад».

**Аналогия с Java:**
- `package`
- `module` (в смысле логического модуля, не JPMS)

📌 **Важно:** подсистемы — это **не код**, а **структурирование**.

---
## 2. Общие модули - специальные классы помощники (`утилитарные классы`)
Глобальные модули с кодом 1С.
- функции
- процедуры
- бизнес-логика
- утилиты

Могут быть:
- серверные
- клиентские
- общие

**Аналогия с Java:**
- `@Service`
- `@Component`
- 🔥 `Utils` / `Helper` классы
- shared business logic

📌 В 1С **очень много логики живёт именно тут**.

---
## 3. Параметры сеанса - переменные живущие в рамках сеанса пользователя
==Переменные==, живущие **в рамках пользовательского сеанса**.
- доступны из любого модуля
- индивидуальны для пользователя
- не хранятся в БД


**Пример:**  
Текущая организация, язык интерфейса, признак режима работы.

**Аналогия с Java:**
- `HttpSession`
- `SecurityContext`
- ThreadLocal-контекст запроса

## 6. Планы обмена - Способы взаимодействия между Информационными Базами
Механизм **распределённых систем**.
- обмен данными между ИБ
- репликация
- синхронизация

**Аналогия с Java:**
- Data replication
- 🔥Kafka / CDC / integration sync
- distributed systems sync

### **Аналогия с Java/Enterprise-системами:**

| **1С**                    | **Java/Enterprise**               | **Назначение**                           |
| ------------------------- | --------------------------------- | ---------------------------------------- |
| План обмена               | CDC (Change Data Capture) + Kafka | Отслеживание и распространение изменений |
| Узлы плана обмена         | Kafka topics / потребители        | Адресация сообщений                      |
| Регистрация изменений     | Debezium / логическая репликация  | Захват изменений                         |
| Выгрузка/загрузка данных  | ETL-процессы (Apache NiFi)        | Перенос данных                           |
| Распределенная транзакция | Saga-паттерн / 2PC                | Согласованность данных                   |

**Планы обмена в 1С** — это механизм создания распределенных систем для синхронного или асинхронного обмена данными между несколькими информационными базами (ИБ). Это enterprise-уровень репликации данных.

### **Суть простыми словами:**
Это "инфраструктура" для построения **распределенной системы 1С**, где несколько баз работают как единое целое:
- Головной офис ↔ филиалы
    
- 1С в магазине ↔ 1С на складе
    
- Продажи ↔ Бухгалтерия ↔ Склад (каждая в своей базе)

## 10. Функциональные опции - вкл/выкл определенной опции (компонента, выполняющего опцию)
Фиче-флаги внутри 1С.
- включать/выключать функциональность
- без переписывания кода

Это "переключатели", которые показывают или скрывают части интерфейса и логики в зависимости от настроек.

### **Как это работает:**
#### **1. Создание опции (в Конфигураторе)**
- Во вкладке **«Общие» → «Функциональные опции»** `создаете новую опцию`, например:
    - **«ИспользоватьСкладскойУчет»**
    - **«ВключитьМноговалютность»**
    - **«РаботаСКлиентскимиКартами»**

#### **2. Настройка видимости**
==Для каждого объекта метаданных== (справочник, документ, форма, реквизит) указываете:
- **«Функциональная опция»** — какую опцию он реализует
    
- **«Модуль управления видимостью»** — код, который динамически определяет, показывать ли элемент

**Пример:** Реквизит «БонуснаяКарта» в документе «Продажа» будет виден только если включена опция **«РаботаСКлиентскимиКартами»**.

#### **3. Включение/выключение**
Делается двумя способами:
- **Глобально** — в настройках конфигурации для всех пользователей
    
- **Персонально** — через права ролей или интерфейс «1С:Предприятие»

## 15. Общие команды
**Общие команды в 1С — это глобальные действия интерфейса, доступные пользователю в разных разделах программы.** 

Это элементы навигации, вызовы отчетов, обработок или часто используемых операций.

### **Суть простыми словами:**
Это =="кнопки" или пункты меню==, которые:
- Появляются в разных местах интерфейса (панель действий, главное меню, контекстные меню)
    
- Выполняют конкретное действие (создать документ, открыть отчет, запустить обработку)
    
- Могут быть доступны при разных условиях

---
## 16. Группы команд
**Группы команд в 1С** — это логические контейнеры для объединения связанных общих команд в иерархические меню и панели.

### **Суть простыми словами:**
Это =="папки" или "подменю" в интерфейсе==, которые группируют команды по смыслу. 

Например:
- Группа **«Отчеты»** → команды «Отчет по продажам», «Оборотно-сальдовая», «Анализ остатков»
    
- Группа **«Документы»** → команды «Создать приход», «Создать расход», «Провести все»

## 19. XDTO-пакеты
**XDTO-пакеты в 1С** — это механизм для описания структурированных данных (==Data Transfer Objects - DTO==) для обмена с внешними системами через XML, SOAP, REST и другие протоколы.

### **Суть простыми словами:**
Это "контракты" или "схемы данных", которые определяют:
- Какие данные будут передаваться
    
- В каком формате (XML, JSON)
    
- Какого типа поля (строка, число, дата)

Например, для интеграции с интернет-магазином вы создаете XDTO-пакет "ЗаказИзИнтернетМагазина", где описываете поля: номер заказа, дата, товары, клиент.

## 20. Web-сервисы - API-ручки SOAP
**Web-сервисы в 1С** — это ==реализация SOAP-протокола== для создания и потребления веб-сервисов, аналогичная JAX-WS в Java.

### **Суть простыми словами:**
Это способ "выставить наружу" методы вашей 1С-базы, чтобы другие программы могли их вызывать через интернет по стандартному протоколу SOAP/XML. 

Например:
- CRM-система отправляет заказ в 1С
    
- Мобильное приложение получает остатки товаров
    
- Банк загружает выписку по счетам

---
## 21. HTTP-сервисы 
**HTTP-сервисы в 1С** — это механизм для создания RESTful API, аналогичный Spring MVC контроллерам в Java. Это основной способ интеграции 1С с современными системами.

### **Суть простыми словами:**
Это "веб-точки входа" в вашу 1С-базу, которые принимают HTTP-запросы (GET, POST, PUT, DELETE) и возвращают ответы (JSON, XML, HTML). 

Например:
- `GET /api/products` → список товаров в JSON
- `POST /api/orders` → создание нового заказа
- `GET /api/customers/{id}` → информация о клиенте

---
## 22. WS-ссылки - подключение НАРУЖУ
**WS-ссылки в 1С** — это клиент для подключения к внешним SOAP-веб-сервисам и вызова их методов. 

Это аналог SOAP-клиента в Java (JAX-WS) или RestTemplate для SOAP-протокола.

### **Суть простыми словами:**
Это "подключение" из вашей 1С-базы к внешнему сервису, которое позволяет:
- Получать курсы валют из Центробанка
    
- Отправлять данные в CRM-систему
    
- Проверять контрагентов по базе ФНС
    
- Получать информацию о товарах от поставщика
