
# 1. Модули
Модули — это «хранилища» текста программы на встроенном языке. Они срабатывают в определённые моменты (события). 

> [!NOTE] Менеджеры
> Для каждого прикладного объекта существует менеджер, предназначенный для управления этим объектом как объектом конфигурации.

Основные виды:

**1. Модуль управляемого приложения**  
Выполняется при старте и завершении работы в тонком и веб-клиенте. 
Содержит процедуры/функции, доступные всей конфигурации (кроме внешнего соединения).
![[Pasted image 20260214154021.png]]


**2. Общие модули**  
Содержат процедуры и функции, вызываемые из других модулей. Сам модуль не исполняется — исполняются его экспортируемые методы при вызове.
![[Pasted image 20260214154031.png]]


**3. Модули объектов**  
Принадлежат конкретным объектам (справочники, документы). Срабатывают при создании объекта, его записи, удалении и т.д.
![[Pasted image 20260214154218.png]]

| Модуль               | Аналогия                                                   | Суть                                                                                       |
| -------------------- | ---------------------------------------------------------- | ------------------------------------------------------------------------------------------ |
| **Модуль объекта**   | **Конкретный ресторан** (например, "Ресторан на Тверской") | Знает про себя: свой адрес, свой коллектив, свои текущие запасы продуктов                  |
| **Модуль менеджера** | **Управляющая компания** всей сети                         | Знает про все рестораны: стандарты сети, правила открытия новых, общие маркетинговые акции |


**4. Модули форм**  
Есть у каждой формы. Содержат обработчики событий элементов формы. Исполняются при открытии формы.
![[Pasted image 20260214154255.png]]


**5. Модуль сеанса**  
Выполняется при старте системы ***до загрузки конфигурации***. Используется для инициализации параметров сеанса.
![[Pasted image 20260214154305.png]]


**6. Модуль внешнего соединения**  

> [!NOTE] Менеджеры
> Для каждого прикладного объекта существует менеджер, предназначенный для управления этим объектом как объектом конфигурации.


Содержит функции, доступные при подключении через внешнее соединение (например, из других приложений).

**7. Модуль менеджера**  
Расширяет функциональность менеджера объекта (например, справочника). Позволяет создавать собственные методы для всего объекта конфигурации, не привязанные к конкретному элементу.

|Модуль|Аналогия|Суть|
|---|---|---|
|**Модуль объекта**|**Конкретный ресторан** (например, "Ресторан на Тверской")|Знает про себя: свой адрес, свой коллектив, свои текущие запасы продуктов|
|**Модуль менеджера**|**Управляющая компания** всей сети|Знает про все рестораны: стандарты сети, правила открытия новых, общие маркетинговые акции|

Чтобы открыть модуль менеджера, нужно в окне редактирования объекта конфигурации перейти на закладку Прочее и нажать кнопку Модуль менеджера (рис. 5.6). Или, выделив нужный объект в дереве объектов конфигурации, вызвать из контекстного меню команду Открыть модуль менеджера.

![[Pasted image 20260214154531.png]]

**8. Модуль команды**  
Как в самой конфигурации, так и у многих прикладных объектов, могут существовать подчиненные объекты конфигурации – `Команды`.

У каждой команды существует *модуль команды*, в котором можно написать предопределенную процедуру `ОбработкаКоманды()` для выполнения этой команды. 

![[Pasted image 20260214154648.png]]

# 2. Контекст модуля формы
Контекст модуля определяет, какие объекты, переменные и функции доступны в этом модуле во время выполнения. Контекст модуля формы складывается из нескольких составляющих:

## **1. Локальный контекст самого модуля формы**  
Переменные, процедуры и функции, объявленные в этом же модуле. Доступны напрямую по имени.

## **2. Реквизиты формы**  
К реквизитам формы можно обращаться напрямую по имени.
![[Pasted image 20260214154922.png]]


## **3. Свойства и методы объекта «УправляемаяФорма»** (ФОРМА) 
Свойства и методы объекта встроенного языка УправляемаяФорма описаны в синтакс-помощнике: `Интерфейс (управляемый) > УправляемаяФорма`
![[Pasted image 20260214155146.png]]

Например, _Заголовок_ или _Закрыть()_ — доступны напрямую.
![[Pasted image 20260214155046.png]]


## **4. Расширение формы по основному реквизиту**  
Один из реквизитов формы может быть основным, в списке реквизитов он выделяется жирным. Как правило, основной реквизит формы содержит данные того объекта, который отображается в форме. 

Например, если это форма справоч ника, то основной реквизит будет содержать данные объекта `СправочникОбъект.<имя>`
![[Pasted image 20260214155308.png]]

А если это форма документа, то основной реквизит формы будет содержать данные объекта `ДокументОбъект.<имя>`
![[Pasted image 20260214155334.png]]

### Тип реквизита - "ненастоящий"
Тут следует пояснить, почему в списке реквизитов тип основного реквизита указан в скобках: `(ДокументОбъект.Документ1)`. 

Дело в том, что это «ненастоящий» тип реквизита формы. Настоящий тип в данном случае будет `ДанныеФормыСтруктура`
![[Pasted image 20260214155518.png]]

Но `ДанныеФормыСтруктура` – это универсальный тип, который может содержать данные различных прикладных объектов. 

Поэтому, чтобы в редакторе формы можно было «ориентироваться», данные какого же именно прикладного объекта отображает эта форма, в колонке Тип редактора показывается не тип реквизита формы `(ДанныеФормыСтруктура)`, а тип того объекта, данные которого содержатся в этом реквизите. 

И этот «ненастоящий» тип показывается в скобках.

### Контекст объекта добавляется к контексту формы
В зависимости от того, какой тип имеет объект, данные которого содержатся в основном реквизите, к контексту программного объекта `УправляемаяФорма` добавляется контекст соответствующего расширения. 

Например, если основной реквизит – `СправочникОбъект.<имя>`, то в модуле формы становятся доступны свойства, методы объекта встроенного языка ***Расширение управляемой формы для справочника*** (синтакс-помощник – Интерфейс (управляемый) > Управляемая форма > Расширение справочника)

> В версии 8.3.27 это вроде бы находится во вкладе помощника - `Прикладные объекты`


**5. Глобальный контекст**  
- Встроенные функции (например, _ТекущаяДата()_), глобальные общие модули (доступны по имени), 
- неглобальные общие модули (доступны через точку от имени модуля).

**6. Экспортируемые процедуры и функции модуля управляемого приложения**  
Доступны напрямую по имени.

# 3. **Форма как программный объект**
Форма доступна из других частей конфигурации как объект встроенного языка **УправляемаяФорма**. 

Помимо стандартных свойств и методов, у неё могут быть дополнительные процедуры и функции, определённые разработчиком и объявленные с ключевым словом _Экспорт_.

**Пример:**  
В модуле формы документа есть экспортная процедура:

```text
&НаКлиенте
Процедура МатериалыКоличествоПриИзменении(Элемент)
    // тело процедуры
КонецПроцедуры
```

Из другого места её можно вызвать так:

```text
Форма = ПолучитьФорму("Документ.ПриходнаяНакладная.Форма.ФормаДокумента");

Форма.МатериалыКоличествоПриИзменении();
```

Таким образом, форма предоставляет свой интерфейс для взаимодействия с внешним кодом.

> [!NOTE] 
>  Также стоит отметить, что доступ к элементам на форме можно получить через
> ```
> ЭтаФорма.Элементы.
> //Или
> Элементы
> ```
> А доступ к ТЕКУЩЕМУ ИСПОЛЬЗУЕМОМОУ ОБЪЕКТУ
> ```
> ЭтаФорма.ТекущийОбъект
> ```

# 4. **Процедуры-обработчики событий в модуле формы**
В модуле формы можно описывать процедуры, которые будут вызываться при наступлении определённых событий формы (например, открытие или закрытие окна).

**Важно:** ***имена процедур могут быть произвольными***. 
Недостаточно просто написать процедуру с подходящим названием — её нужно **явно связать** с событием в конфигураторе (через палитру свойств формы).

## ТОП СОВЕТ - Создать событие через свойство "События", а потом переименовать и пересвязать свойство
Чтобы избежать ошибок, лучше создавать обработчики автоматически:
- нажатием кнопки с лупой в палитре свойств события;
- выбором события из выпадающего списка в модуле формы.

В этом случае система сама создаст процедуру с правильным именем и установит связь с событием.
