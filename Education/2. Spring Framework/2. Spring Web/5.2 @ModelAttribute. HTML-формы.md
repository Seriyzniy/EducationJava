# 1. HTML-формы классические
Для того чтобы пользователь мог отправить данные НЕ через ПАРАМЕТРЫ запроса, а с помощью удобных форм, необходимо написать соответствующий HTML-код
![[Pasted image 20241117125309.png]]

```html
<form method="POST" action="/people">
	<label for="name">Name</label>
	<input name="name" type="text" id="name"/>
	
	<br>
	<label for="surname">Surname</label>
	<input name="surname" type="text" id="surname"/>
	
	<br>
	<label for="email">Email</label>
	<input name="email" type="text" id="email"/>
	
	<br>
	<input type="submit" value="Add Person"/>
</form>
```

> [!NOTE] Код HTML-формы
> - тег `method` указывает ==HTTP-метод== будет использоваться ==при отправке== данных ==с этой формы==
> - тег `action` указывает ==адрес==, куда будут отправляться данные с формы
> - тег `input` создает поле для ==ввода==
> - тег `label` определяет ==текст, для input-поля== для ввода ПЕРЕМЕННОЙ
> - тег `br` вставляет пустую строку
> 
> Тип `type="submit"` используется для создания кнопки, по нажатию которой отправятся данные. ==В виде URL-запроса==

# 2. HTML-формы Thymleaf
```html
<form th:method="POST" th:action="@{/people}" th:object="${person}">
	<div>
		<label for="name">Name</label>
		<input type="text" th:field="*{name}" id="name"/>
		
		<br/>
		<label for="surname">Surname</label>
		<input type="text" th:field="*{surname}" id="surname"/>
		
		<br>
		<label for="email">Email</label>
		<input type="text" th:field="*{email}" id="email"/>
		
		<br>
		<input type="submit" value="Add Person"/>
	</div>
</form>
```

> [!NOTE] HTML-форма Thymleaf
> Такая ==HTML-форма принимает== на вход ==объект==, для которого эта форма создана
> 
> Для этого ==метод контроллера== должен передавать пользователю html-страницу и объект, который нужно "создать" через эту форму
> ```java
> @GetMapping("/new")
> public String newPerson(Model model) {
> 	model.addAttribute( "person", new Person());Атрибут используется в ФОРМЕ
> 	return "people/new";
> }

# 3. Принятие данных с HTML-формы Контроллером
```java
@PostMapping()
public String create(@RequestParam("name") String name, @RequestParam("surname") 
			 String surname, @RequestParam("email") String email, Model model) 
{
	Person person = new Person();
	person.setName(name);
	person.setSurname(surname);
	person.setEmail(email);
	// Добавляем человека в БД
	model.addAttribute( s: "person", person);
	return "success Page";
}
```
Также можно использовать и объект `HttpServletRequest`для принятия данных

# 4. Аннотация `@ModelAttribute`

> [!NOTE] `@ModelAttribute`
> Аннотация может аннотировать:
> - Метод (создает пару ==ключ-значение:== headerMessage="Welcome to our website!")
>  ```java
> @ModelAttribute("headerMessage")
> public String populateHeaderMessage() {
> 	return "Welcome to our website!";
> }
> ```
> - Аргумент метода
> ```java
> @PostMapping()
> public String create(@ModelAttribute("person") Person person) {
> 	personDAO.save(person);
> 	return "redirect:/people";
> }
> ```

#### 4.1 Аннотирование МЕТОДА
![[Pasted image 20241117133630.png]]

> [!NOTE] Аннотация в МЕТОДЕ
> Создаст `пару ключ-значение` для `ВСЕХ моделей` данного `контроллера` со значением по умолчанию
> 
> Можно создавать ==любые пары== `ключ-значение`, не только строковые

#### 4.2 Аннотация в ПАРАМЕТРЕ метода
![[Pasted image 20241117135217.png]]


> [!NOTE] Аннотация в ПАРАМЕТРЕ метода
> Ранее мы создавали контроллер для POST запроса, который берет параметры из HTML-формы и по этим параметрам:
> - Создает пустой объект класса `Person`
> - Устанавливает значения, полученные из формы, в поля объекта
> - Добавляет полученный объект в атрибут модели
> 
> ==Все это== теперь ==делает аннотация== `@ModelAttribute`, ==если передать ее в качестве параметра== метода
> 
> Если в HTML-форме не передавать параметры, то поскольку `Person` это класс, то установятся значения по умолчанию
