### 1.2 Конфигурация Java-кодом
Чтобы не использовать файл `web.xml` будем использовать `Java-класс`

Как описывалось в практике ==1.1 Практика (кофиг с XML и Java-код)==, класс заменяющий файл `web.xml` должен 
- либо реализовать интерфейс `WebApplicationInitializer` 
- либо унаследовать абстрактный класс, который уже реализовал этот интерфейс - `AbstractAnnotationConfigDispatcherServletInitializer`

Для классов конфигурации (заменяющий web.xml и файл с контекстом), создадим в папке `src/main/java` пакет `myConfig`

###### Класс-конфигуратор (контекст)
Теперь реализуем класс-конфигуратор (контекст)
```java
//Класс заменяющий контекст.xml
@Configuration
@ComponentScan("myCrudApp")
@EnableWebMvc
@RequestMapping("/crud")
public class AppContext implements WebMvcConfigurer{
	private final ApplicationContext appContext;
	@Autowired
	public AppContext(ApplicationContext context) {
		appContext = context;
	}
	@Bean
	public SpringResourceTemplateResolver templateResolver() {
		SpringResourceTemplateResolver templateResolver = new 
		SpringResourceTemplateResolver();
	
		templateResolver.setApplicationContext(appContext);
		templateResolver.setPrefix("/WEB-INF/views/");
		templateResolver.setSuffix(".html");
		return templateResolver;
	}
	@Bean
	public SpringTemplateEngine templateEngine() {
		SpringTemplateEngine templateEngine = new SpringTemplateEngine();
		templateEngine.setTemplateResolver(templateResolver());
		templateEngine.setEnableSpringELCompiler(true);
		return templateEngine;
	}
	@Override
	public void configureViewResolvers(ViewResolverRegistry registry) {
		ThymeleafViewResolver resolver = new ThymeleafViewResolver();
		resolver.setTemplateEngine(templateEngine());
		registry.viewResolver(resolver);
	}
}
```
#### 1.`templateResolver()`
Этот метод создает и настраивает `SpringResourceTemplateResolver`, который ==отвечает за нахождение и обработку шаблонов== (HTML файлов). Вот что делает этот метод:
- **`setApplicationContext(applicationContext)`**: Связывает `templateResolver` с контекстом Spring, что позволяет ему находить ресурсы в приложении.
#### 2.`templateEngine()`
Этот метод ==создает и настраивает== `SpringTemplateEngine`, который является ==ядром движка Thymeleaf==. Он отвечает за обработку шаблонов, используя `templateResolver` для их поиска.
- **`setTemplateResolver(templateResolver())`**: Связывает движок с ранее настроенным `templateResolver`.
- **`setEnableSpringELCompiler(true)`**: Включает компилятор выражений Spring EL, что может улучшить производительность при работе с выражениями внутри шаблонов.
#### 3.`configureViewResolvers(ViewResolverRegistry registry)`
Этот метод ==настраивает резолвер представлений==, который Spring MVC использует для определения представления, возвращаемого контроллером.
- **Создание `ThymeleafViewResolver`**: `ThymeleafViewResolver` — это адаптер между Spring MVC и Thymeleaf, который преобразует логические имена представлений (например, строки, возвращаемые контроллерами) в конечные представления с использованием движка Thymeleaf.
- **`resolver.setTemplateEngine(templateEngine())`**: Связывает `ThymeleafViewResolver` с настроенным движком `templateEngine`, чтобы тот мог использовать шаблоны для рендеринга.
- **`registry.viewResolver(resolver)`**: Регистрирует резолвер представлений в Spring MVC.

# 3. Реализация DAO-класса

Реализуем класс, отвечающий за работу с данными (получение ВСЕХ данных, получение одной записи данных).


> [!NOTE] DAO-класс
> Обычно этот класс используется для работы с БД
> Он отвечает за СОЗДАНИЕ, ЧТЕНИЕ, ОБНОВЛЕНИЕ и УДАЛЕНИЕ данных из БД
> 
> Сейчас упростим задачу и будем брать данные НЕ из БД, а из списка

```java
@Component
public class PersonDAO {
	private List<Person> personsList;
	
	public PersonDAO() {
		personsList.add(new Person(1,"Alisa"));
		personsList.add(new Person(2,"Bob"));
		personsList.add(new Person(3,"Clark"));
		personsList.add(new Person(4,"Din"));
		personsList.add(new Person(5,"Elisabet"));
	}
	
	public List<Person> getList(){
		return personsList;
	}
	
	public Person getPerson(int id) {
		return personsList.stream().filter(person -> person.getId() == id).findAny().orElse(null);
	}
}
```

Здесь мы имитируем запись в таблицу БД, и добавляем `ID` и `Name` пользователя

Для получения одной записи из БД используется `лямбда-выражение`

# 4. Реализация контроллера

Теперь реализуем контроллер, у которого будет два метода. По одному URL-адресу будет выводится весь список Person, а по другому лишь отдельный

```java
@Controller
@RequestMapping("/people")
public class PeopleController {
	private PersonDAO myPersonDAO;
	@Autowired
	public PeopleController(PersonDAO PersonDAO) {
		this.myPersonDAO = PersonDAO;
	}
	
	@GetMapping()
	public String showAll(Model myModel) {
		myModel.addAttribute("allpeople", myPersonDAO.getList());
		return "people/showall";
	}
	
	@GetMapping("/{id}")
	public String showOne(@PathVariable("id") int id, Model myModel) {
		myModel.addAttribute("oneperson", myPersonDAO.getPerson(id));
		return "people/showone";
	}
}
```

> [!NOTE] `@Autowired`
> По конвенциям `@Autowired` лучше использовать в методах/конструкторах, чем в переменной
> 

Все наши представления будут лежать в папке `people`
