# 1. Проблемы с добавлением Tomcat

#### Шаг 1
Для разработки WEB приложений необходима версия Eclipse EE, или необходимо установить WTP (Web Tools Platform) как расширение к основной версии Eclipse

Это расширение позволит разрабатывать WEB приложения и позволит работать с сервером Apache Tomcat

Это можно сделать через кладку Help->Instal new software и ввести адрес Официального сайта Eclipse и выбрать расширение WTP ( http://download.eclipse.org/webtools/repository/latest/ ).
![[Pasted image 20250206131918.png]]

Также на всякий случай можно установить через Marketplace внутри Eclipse и также найти WTP

#### Шаг 2
После этого появится вкладка на Window->Show view->Other...  и внутри Servers->Server

Для добавление сервера необходимо заранее его скачать также с официального сайта

Теперь если открыть кладку Server можно будет добавить новый сервер.

> [!NOTE] ВЕРСИИ TOMCAT
> Следует внимательно выбирать версию Tomcat и версию Spring, поскольку для:
> 
> - Spring 6+ и Tomcat 10+  необходимо использовать библиотеку и зависимости для сервлетов от `jakatra`
> 	
> - Spring 5 и ниже и Tomcat 9 и ниже используют зависимости от `javax`

#### Другая возможная проблема
Убедитесь в наличии установленного плагина в `pom.xml`
```xml
<plugin>
	<groupId>org.apache.maven.plugins</groupId>
	<artifactId>maven-war-plugin</artifactId>
	<version>3.3.1</version>
	<configuration>
	<warSourceDirectory>src/main/webapp</warSourceDirectory>
	</configuration>
</plugin>
```

# 2. Важная папка `target/w2e-wtp`

Когда вы создаете Maven проект в Eclipse, используемый для веб-приложений, Eclipse автоматически создаёт дополнительную структуру папок, связанную с ==**Maven Integration for Eclipse (m2e)** и **Web Tools Platform (WTP)**. ==

Эта структура помогает Eclipse и Tomcat корректно работать с веб-ресурсами, такими как статические файлы, манифесты, и предоставляет настройки для развертывания.

В частности:
- `target/m2e-wtp/web-resources/` содержит папку с веб-ресурсами, которые будут развернуты в сервер Tomcat.
- `META-INF/maven/` содержит информацию о проекте, такую как координаты Maven (группа, артефакт, версия).

> [!NOTE] ВАЖНО ПИПЕЦ КАК
>Если эта структура отсутствует, значит, проект, возможно, не настроен или не собран должным образом для работы с Tomcat через Eclipse.
## Шаги для исправления:

> [!NOTE] Maven команды ВАЖНО ПИПЕЦ КАК
> Run as -> Maven clean - удаляет старые конфигурационные файлы проекта в папке target
> Run as -> Maven instal - пересоберет проект 
> Maven ->Update Project вкладка Force Update ...

##### 0. Случайный фикс

ПКМ на проект -> Configure -> Convert to Maven project
##### 1. Убедитесь, что у вас настроен проект как Maven Web Project:
В Eclipse проект должен быть настроен как веб-проект с типом `war`. Проверьте файл `pom.xml`:
![[Pasted image 20241111183621.png]]
##### 2.Проверьте настройки Maven в Eclipse:
Убедитесь, что ваш проект правильно настроен для использования **Maven** в Eclipse:
- Откройте проект в **Project Explorer**.
- Щелкните правой кнопкой мыши по проекту, выберите **Maven** и убедитесь, что выбраны опции **Update Project** и **Enable Maven Nature** (должен стоять `Disable`). Это может помочь восстановить нужные Maven настройки.
##### 3.Проверьте настройки для `m2e-wtp`:
Если в вашем проекте нет структуры `target/m2e-wtp/`, возможно, нужно добавить **Maven Integration for Eclipse (m2e)** и **Web Tools Platform (WTP)**.

Для этого:
- Откройте в Eclipse **Help** -> **Eclipse Marketplace**.
- Убедитесь, что установлены следующие плагины:
    - **Maven Integration for Eclipse (m2e)**.
    - **Web Tools Platform (WTP)** (если еще не установлен).
##### 4.Выполните повторную сборку проекта:
После того как убедились, что все плагины и настройки правильные, выполните команду **Maven clean install**:

- Щелкните правой кнопкой мыши по проекту в **Project Explorer**.
- Выберите **Run As** -> **Maven clean install**. (==можно по отдельности==)

Это очистит проект и пересоберет его, создав нужные папки и файлы в `target`. После этого структура в папке `target` должна выглядеть как в рабочем проекте, включая папку `m2e-wtp`.

##### 5. Развертывание проекта на сервере
Убедитесь, что ваш проект корректно добавлен в сервер Tomcat в Eclipse:
- Откройте вкладку **Servers** в Eclipse.
- Щелкните правой кнопкой мыши по серверу Tomcat и выберите **Add and Remove**.
- Убедитесь, что ваш проект находится в списке **Configured**. Если его нет, добавьте его вручную.

# 3. Сервер не видит проект для развертывания
##### 1. Project Facets
Если проект собран не правильно (что возможно) необходимо подключить ==Dinamic Web Module ВЕРСИИ 4.0==

Для этого:
- Перейти в свойства проекта
- Выбрать `Projects Facets`
- Выбрать `Dinamic Web Module` версии 4.0
- Также стоит выбрать `java` версии 1.8

# 4. Другие возможные правки
##### 1. Папки src/main
> [!NOTE] Папка `src/main/java`
> Папка `src/main/java` должна быть ==включена в путь сборки как корневая папка== (ресурсная папка)

Для этого:
ПКМ по проекту -> Build Path ->Use as Sourse Folder

>[!NOTE] Папка `src/main/webapp`
> Папка `src/main/webapp` НЕ НУЖНО помечать как Sourse Folder

##### 2. Проблемы с шаблонизатором Tymleaf

> [!NOTE] Смотри ссылку на xmlns
> Важно проверить ссылку на схему xml, должна быть ссылка с `https` а не `http`
