Используя проект с контроллером с двумя методами (sayHello, sayBye), изменим его так чтобы один из методов контроллеров принимал и обрабатывал параметры
## Изменение метода sayHello() при помощи HttpServletRequest
Добавим следующий код:
```java
@Controller
public class MyWelcomeController {
	@GetMapping("/hello")
	public String sayHello(HttpServletRequest request) {
		String nameUser = request.getParameter("name");
		System.out.println("Connect from " + nameUser);
		
		return "myfirst/hellopage";
	}
	. . .
}
```
В параметрах метода указали объект класса `HttpServletRequest`, с помощью которого получим параметр запроса по имени параметра `name` и положим значение параметра в переменную `nameUser`. Это имя выведем в консоль

## Изменение метода sayHello() при помощи `@RequestParam`
```java
@Controller
public class MyWelcomeController {
	@GetMapping("/hello")
	public String sayHello(@RequestParam("name") String nameUser) {
		System.out.println("Connected user: " + nameUser);
		
		return "myfirst/hellopage";
	}
	...
}
```
Эффект тот же что и в предыдущем случае

> [!NOTE] БЕЗ ПАРАМЕТРА
> Ошибка в браузере что нет параметра
> 
> Чтобы ошибки не было необходимо в аннотации после "name" через запятую добавить required=false (`@RequestParam("name", required=false)`)
