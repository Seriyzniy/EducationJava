*Пакетные сообщения* (созданные производителем) **автоматически распаковываются** (de-batched) **контейнерами слушателей** (==используя заголовок сообщения== `springBatchFormat`). 

Если producer отправил **пакет**:
- контейнер слушателя **автоматически распаковывает (de-batching)** пакет,
    
- и **передаёт сообщения слушателю по одному**, как будто они пришли отдельно.

> Отклонение любого сообщения из пакета приводит к отклонению всего пакета. Для получения дополнительной информации о пакетировании см. "[Batching](https://docs.spring.io/spring-amqp/reference/amqp/sending-messages.html#template-batching)".

## 1. Автоматический сбор одиночных сообщений в пакет
Начиная с версии 2.2, `SimpleMessageListenerContainer` **может использоваться для создания пакетов на стороне потребителя** (где производитель отправлял отдельные сообщения).

```mathematica
producer → message, message, message

consumer → получает List<Message>
```

Установите свойство контейнера
- `consumerBatchEnabled` в `true`, чтобы включить эту функцию. 
- Свойство `deBatchingEnabled` также должно быть `true`, чтобы контейнер отвечал за обработку пакетов обоих типов. 


Реализуйте интерфейс 
- `BatchMessageListener` 
- или `ChannelAwareBatchMessageListener`, 

когда `consumerBatchEnabled` имеет значение `true`. 


Начиная с версии 2.2.7, и `SimpleMessageListenerContainer`, и `DirectMessageListenerContainer` могут распаковывать пакеты, созданные производителем, в виде `List<Message>`. 
> См. "@RabbitListener с пакетированием ([@RabbitListener with Batching](https://docs.spring.io/spring-amqp/reference/amqp/receiving-messages/batch.html))" для получения информации об использовании этой функции с `@RabbitListener`.
