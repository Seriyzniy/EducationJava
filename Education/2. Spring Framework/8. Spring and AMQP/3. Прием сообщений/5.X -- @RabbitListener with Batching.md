По умолчанию при получении **пакетных сообщений** контейнер выполняет **распаковку (de-batching)** и вызывает слушатель **по одному сообщению**.  

Начиная с **Spring AMQP 2.2**, можно настроить слушатель так, чтобы ==ПОЛУЧАТЬ ВЕСЬ ПАКЕТ ЗА ОДИН ВЫЗОВ==.

Для этого в `RabbitListenerContainerFactory` нужно включить режим пакетного слушателя и принимать payload как `List` или `Collection`
```java
@Bean
public SimpleRabbitListenerContainerFactory rabbitListenerContainerFactory() {

    SimpleRabbitListenerContainerFactory factory = 
	    new SimpleRabbitListenerContainerFactory();
	    
    factory.setConnectionFactory(connectionFactory());
    factory.setBatchListener(true);
    
    return factory;
}

@RabbitListener(queues = "batch.1")
public void listen1(List<Thing> in) {
    ...
}

// или

@RabbitListener(queues = "batch.2")
public void listen2(List<Message<Thing>> in) {
    ...
}
```

Установка **`batchListener = true`** автоматически отключает **`deBatchingEnabled`** у контейнера (если только `consumerBatchEnabled` не равен `true`). 
> **В этом режиме распаковка фактически переносится из контейнера в адаптер слушателя**, который формирует список и передаёт его в метод.


Фабрика с включённым `batchListener` **не поддерживает multi-method listener**.

Также с версии **2.2**, ==при обработке пакета по одному сообщению== **последнее сообщение содержит заголовок** **`AmqpHeaders.LAST_IN_BATCH`** со значением `true`. 

Его можно получить так:
```java
@Header(AmqpHeaders.LAST_IN_BATCH) boolean last
```

Заголовок маппится из `MessageProperties.isLastInBatch()`, а **`AmqpHeaders.BATCH_SIZE`** содержит размер пакета.

## Сборка сообщений в пакет
В `SimpleMessageListenerContainer` есть свойство **`consumerBatchEnabled`**

Когда оно равно `true`, контейнер **сам собирает пакет сообщений** размером до `batchSize`. 

Если новые сообщения не приходят и истекает `receiveTimeout`, доставляется **частичный пакет**.

Пакеты, созданные producer’ом, при этом **распаковываются и добавляются** в consumer-batch, поэтому фактическое число доставленных сообщений может превышать `batchSize`. 

> [!WARNING] 
> При включённом `consumerBatchEnabled` свойство **`deBatchingEnabled` должно быть `true`**, и фабрика контейнеров гарантирует это автоматически.

Пример настройки фабрики контейнеров:
```java
@Bean
public SimpleRabbitListenerContainerFactory consumerBatchContainerFactory() {
    SimpleRabbitListenerContainerFactory factory = 
	    new SimpleRabbitListenerContainerFactory();
	    
    factory.setConnectionFactory(rabbitConnectionFactory());
    factory.setConsumerTagStrategy(consumerTagStrategy());
    
    factory.setBatchListener(true); // BatchMessageListenerAdapter
    
    factory.setBatchSize(2);
    factory.setConsumerBatchEnabled(true);
    
    return factory;
}
```

Использование с `@RabbitListener`:
```java
@RabbitListener(
	queues = "batch.1", 
	containerFactory = "consumerBatchContainerFactory")
public void consumerBatch1(List<Message> amqpMessages) {
    ...
}

@RabbitListener(queues = "batch.2", containerFactory = "consumerBatchContainerFactory")
public void consumerBatch2(List<org.springframework.messaging.Message<Invoice>> messages) {
    ...
}

@RabbitListener(queues = "batch.3", containerFactory = "consumerBatchContainerFactory")
public void consumerBatch3(List<Invoice> strings) {
    ...
}
```

- В первом случае метод получает **сырые `org.springframework.amqp.core.Message`**. 
	
- Во втором — **`org.springframework.messaging.Message<?>`** с конвертированной полезной нагрузкой и заголовками. 
	
- В третьем — **только конвертированные payload’ы**, без доступа к заголовкам и свойствам. Дополнительно можно принимать параметр `Channel` (обычно при `MANUAL` ack), но без `delivery_tag` это имеет ограниченный смысл.

В Spring Boot есть свойства конфигурации для **`consumerBatchEnabled`** и **`batchSize`**, но не для `batchListener`. Начиная с версии **3.0**, установка `consumerBatchEnabled = true` автоматически включает `batchListener`, и метод слушателя **обязан быть пакетным**. Также с версии 3.0 методы могут принимать `Collection<?>` или `List<?>`.

Пакетные слушатели **не поддерживают синхронные ответы**, так как нет однозначной корреляции между сообщениями в пакете и единичным ответом. При этом **асинхронные возвращаемые типы** остаются поддержанными.
