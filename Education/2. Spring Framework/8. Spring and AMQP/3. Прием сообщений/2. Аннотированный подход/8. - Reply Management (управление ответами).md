Существующая поддержка в `MessageListenerAdapter` *уже позволяет вашему методу иметь тип возвращаемого значения, отличный от* `void`. 
В этом случае **результат вызова инкапсулируется в сообщение**, отправляемое по адресу, указанному в заголовке `ReplyToAddress` исходного сообщения, или на адрес по умолчанию, настроенный в слушателе. В
> ы можете установить этот адрес по умолчанию, используя аннотацию `@SendTo` из абстракции messaging.

Предположим, наш метод `processOrder` **теперь должен возвращать** `OrderStatus`. Мы можем написать его следующим образом, чтобы автоматически отправлять ответ:
```java
@RabbitListener(destination = "myQueue")
@SendTo("status")
public OrderStatus processOrder(Order order) {
    // обработка заказа
    return status;
}
```

Если вам нужно **установить дополнительные заголовки независимым от транспорта способом**, вы можете вернуть `Message`, примерно так:
```java
@RabbitListener(destination = "myQueue")
@SendTo("status")
public Message<OrderStatus> processOrder(Order order) {
    // обработка заказа
    return MessageBuilder
        .withPayload(status)
        .setHeader("code", 1234)
        .build();
}
```

## Предобработка перед отправкой ответа
### MessagePostProcessor через свойство beforeSendReplyMessagePostProcessors

В качестве альтернативы **вы можете использовать** `MessagePostProcessor` **в СВОЙСТВЕ фабрики** контейнера `beforeSendReplyMessagePostProcessors`, чтобы добавить больше заголовков. 

Начиная с версии 2.2.3, вызванный bean/метод становится доступным в ответном сообщении, что может быть использовано в пост-процессоре сообщений **для передачи информации обратно вызывающей стороне**:
```java
factory.setBeforeSendReplyPostProcessors(msg -> {
    msg
	    .getMessageProperties()
	    .setHeader(
		    "calledBean", 
		    msg
			    .getMessageProperties()
			    .getTargetBean().getClass().getSimpleName()
		);
		
    msg
	    .getMessageProperties()
	    .setHeader(
		    "calledMethod",
            msg
	            .getMessageProperties()
	            .getTargetMethod().getName()
	    );
    return m;
});
```

### ReplyPostProcessor
Начиная с версии 2.2.5, вы можете настроить `ReplyPostProcessor` **для изменения ответного сообщения перед его отправкой**; 
он **вызывается после того**, как заголовок `correlationId` был настроен для соответствия запросу.

```java
@RabbitListener(
	queues = "test.header", 
	group = "testGroup", 
	replyPostProcessor = "echoCustomHeader")
public String capitalizeWithHeader(String in) {
    return in.toUpperCase();
}


@Bean
public ReplyPostProcessor echoCustomHeader() {
    return (req, resp) -> {
        resp
	        .getMessageProperties()
	        .setHeader(
		        "myHeader", 
		        req
			        .getMessageProperties()
			        .getHeader("myHeader")
			);
        return resp;
    };
}
```

### Пост-процессор на фабрике
Начиная с версии 3.0, вы можете **настраивать пост-процессор на фабрике контейнеров**, а не в аннотации.
```java
factory.setReplyPostProcessorProvider(id -> (req, resp) -> {
    resp
	    .getMessageProperties()
	    .setHeader(
		    "myHeader",
		    req
			    .getMessageProperties()
			    .getHeader("myHeader")
		);
    return resp;
});
```
Параметр `id` — это идентификатор слушателя.

## Аннотация @SendTo
Значение `@SendTo` **интерпретируется** как пара 
- **обменника** ответа (reply exchange) 
- и **ключа маршрутизации**, 
==следующая шаблону== `exchange/routingKey`, где одна из частей может быть опущена. 

Допустимые значения:
- `thing1/thing2`: Обменник ответа (`replyTo exchange`) и ключ маршрутизации.
    
- `thing1/`: Обменник ответа и ключ маршрутизации по умолчанию (пустой).
    
- `thing2` или `/thing2`: Ключ маршрутизации ответа и обменник по умолчанию (пустой).
    
- `/` или пустое значение: Обменник ответа по умолчанию и ключ маршрутизации по умолчанию.

Также можно использовать `@SendTo` без атрибута `value`. Этот случай эквивалентен пустому шаблону `sendTo`. `@SendTo` используется только если входящее сообщение не имеет свойства `replyToAddress`.

### SpEL в @SendTo
Начиная с версии 1.5, значение `@SendTo` может быть выражением SpEL, вычисляемым при инициализации бина, как показано в следующем примере:
```java
@RabbitListener(queues = "test.sendTo.spel")
@SendTo("#{spelReplyTo}")
public String capitalizeWithSendToSpel(String foo) {
    return foo.toUpperCase();
}

...
@Bean
public String spelReplyTo() {
    return "test.sendTo.reply.spel";
}
```

Выражение должно вычисляться в `String`, который может быть простым именем очереди (отправляется в обменник по умолчанию) или иметь форму `exchange/routingKey`, как обсуждалось до этого примера.

> Выражение `#{…​}` вычисляется один раз во время инициализации.

### Вычисление SpEL во время выполнения
Начиная с версии 1.6, `@SendTo` **может быть выражением** SpEL, **которое вычисляется во время выполнения** на основе запроса и ответа, как показано в следующем примере:
```java
@RabbitListener(queues = "test.sendTo.spel")
@SendTo("!{'some.reply.queue.with.' + result.queueName}")
public Bar capitalizeWithSendToSpel(Foo foo) {
    return processTheFooAndReturnABar(foo);
}
```

### Динамическая маршрутизация
Для **динамической маршрутизации** ответов ==ОТПРАВИТЕЛЬ сообщения должен ВКЛЮЧИТЬ СВОЙСТВО сообщения== `reply_to` или использовать альтернативное выражение SpEL времени выполнения 

Динамический характер выражения SpEL времени выполнения **обозначается разделителями** `!{…​}`. 

Объект контекста вычисления `#root` для выражения *имеет три свойства:*
1. `request`: Объект запроса `o.s.amqp.core.Message`.
    
2. `source`: `o.s.messaging.Message<?>` после конвертации.
    
3. `result`: Результат метода.


*Контекст имеет доступ к свойствам через map*, стандартный конвертер типов и распознаватель бинов (bean resolver), который позволяет ссылаться на другие бины в контексте (например, `@someBeanName.determineReplyQ(request, result)`).

Таким образом, `#{…​}` **вычисляется один раз во время инициализации**, где объект `#root` — это контекст приложения. 

На бины ссылаются по их именам. `!{…​}` в**ычисляется во время выполнения для каждого сообщения**, где корневой объект имеет перечисленные выше свойства. 
> На бины ссылаются по их именам с префиксом `@`.

Начиная с версии 2.1, также поддерживаются простые плейсхолдеры свойств (например, `${some.reply.to}`). 

В более ранних версиях можно использовать следующий обходной путь, как показано в примере:
```java
@RabbitListener(queues = "foo")
@SendTo("#{environment['my.send.to']}")
public String listen(Message in) {
    ...
    return ...
}
```
