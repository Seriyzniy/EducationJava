**Когда вы используете атрибут** `queues`, вы *можете указать*, что **связанный КОНТЕЙНЕР может ПРОСЛУШИВАТЬ НЕСКОЛЬКО ОЧЕРЕДЕЙ**. 

Вы можете использовать аннотацию `@Header`, чтобы сделать имя очереди, из которой было получено сообщение, доступным для POJO-метода. 

Следующий пример показывает, как это сделать:
```JAVA
@Component
public class MyService {

    @RabbitListener(queues = { "queue1", "queue2" } )
    public void processOrder(
	    String data, 
	    @Header(AmqpHeaders.CONSUMER_QUEUE) String queue) 
	{
        ...
    }

}
```

Начиная с версии 1.5, вы можете **вынести имена очередей наружу, используя плейсхолдеры свойств** (property placeholders) и SpEL. Следующий пример показывает, как это сделать:
```java
@Component
public class MyService {

    @RabbitListener(queues = "#{'${property.with.comma.delimited.queue.names}'.split(',')}" )
    public void processOrder(
	    String data, 
	    @Header(AmqpHeaders.CONSUMER_QUEUE) String queue) 
	{
        ...
    }

}
```
**До версии 1.5** таким способом можно было указать только одну очередь. Для каждой очереди требовалось отдельное свойство.
