Фабрики контейнеров слушателей были введены для поддержки `@RabbitListener` и регистрации контейнеров в `RabbitListenerEndpointRegistry`, как обсуждалось в разделе "[Programmatic Endpoint Registration :: Spring AMQP](https://docs.spring.io/spring-amqp/reference/amqp/receiving-messages/async-annotation-driven/registration.html)".

Начиная с версии 2.1, они **могут использоваться для создания любого контейнера слушателя** — даже контейнера без слушателя (например, для использования в Spring Integration). 
> Конечно, слушатель должен быть добавлен до запуска контейнера.

Есть ==два способа== создать такие контейнеры:
1. Использовать `SimpleRabbitListenerEndpoint`
2. Добавить слушателя после создания

Следующий пример показывает, как использовать `SimpleRabbitListenerEndpoint` для создания контейнера слушателя:
```java
@Bean
public SimpleMessageListenerContainer factoryCreatedContainerSimpleListener (
        SimpleRabbitListenerContainerFactory rabbitListenerContainerFactory) 
{
    SimpleRabbitListenerEndpoint endpoint = 
	    new SimpleRabbitListenerEndpoint();
	    
    endpoint.setQueueNames("queue.1");
    endpoint.setMessageListener(message -> {
        ...
    });
    return rabbitListenerContainerFactory
		    .createListenerContainer(endpoint);
}
```


Следующий пример показывает, как добавить слушателя **после создания**:
```java
@Bean
public SimpleMessageListenerContainer factoryCreatedContainerNoListener
(
SimpleRabbitListenerContainerFactory rabbitListenerContainerFactory
) {
    SimpleMessageListenerContainer container = 
	    rabbitListenerContainerFactory.createListenerContainer();
	    
    container.setMessageListener(message -> {
        ...
    });
    container.setQueueNames("test.no.listener.yet");
    return container;
}
```

В любом случае слушателем также может быть `ChannelAwareMessageListener`, поскольку теперь это подинтерфейс `MessageListener`.

Эти техники полезны, если вы хотите ==создать несколько контейнеров с похожими свойствами== или использовать предварительно настроенную фабрику контейнеров, такую как предоставленная автоконфигурацией Spring Boot, или и то, и другое.

Контейнеры, созданные таким образом, являются обычными экземплярами `@Bean` и не регистрируются в `RabbitListenerEndpointRegistry`.
