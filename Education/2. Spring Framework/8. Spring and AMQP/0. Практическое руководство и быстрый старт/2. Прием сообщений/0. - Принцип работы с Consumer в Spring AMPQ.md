# 0. Концепция Consumer в Spring AMQP

## 1. Что такое Consumer в AMQP/RabbitMQ?
**Consumer** – это клиент, который:
- Подключается к брокеру сообщений (RabbitMQ)
- Читает сообщения из очередей (queues)
- Может подтверждать получение сообщений (acknowledgment)
- Работает в рамках подписок (subscriptions)


**На низком уровне (чистый RabbitMQ API):**
```java
ConnectionFactory factory = new ConnectionFactory();
factory.setHost("localhost");

Connection connection = factory.newConnection();

Channel channel = connection.createChannel();
channel.queueDeclare("queue", false, false, false, null);

DeliverCallback deliverCallback = (consumerTag, delivery) -> {
    String message = new String(delivery.getBody(), "UTF-8");
    processMessage(message);
};

channel.basicConsume("queue", true, deliverCallback, consumerTag -> {});
```

## 2. Listener Container в Spring AMQP
> [!NOTE] Обертка над Consumer  
> Spring AMQP не заменяет Consumer, а ОБЕРТЫВАЕТ его!

### Принцип работы
#### 1. Spring создаёт соединения через ConnectionFactory
```java
@Bean
public ConnectionFactory connectionFactory() {
    CachingConnectionFactory factory = 
	    new CachingConnectionFactory("localhost");
	    
    factory.setUsername("guest");
    factory.setPassword("guest");
    
    return factory;
}
```

#### 2. Container управляет жизненным циклом Consumer:
Два вида контейнеров
- SimpleMessageListenerContainer
- DirectMessageListenerContainer
(см [[2.1 -- Choosing a Container (Simple vs Direct)]])

Они могут использоваться
- напрямую
- через `@RabbitListener` и ФАБРИКУ контейнеров `...RabbitListenerContainerFactory`

```java
@Bean
public SimpleRabbitListenerContainerFactory rabbitListenerContainerFactory() 
{
    SimpleRabbitListenerContainerFactory factory = 
	    new SimpleRabbitListenerContainerFactory();
	    
    factory.setConnectionFactory(connectionFactory());
    factory.setConcurrentConsumers(3); // Создаёт 3 потока-потребителя!
    
    return factory;
}
```

#### 3. Вы пишете только бизнес-логику
```java
@Component
public class MyBusinessLogic {

    @RabbitListener(queues = "my-queue")
    public void handleMessage(String message) {
        // Spring сам создал Consumer, подписался на очередь,
        // получает сообщения и вызывает этот метод
        System.out.println("Received: " + message);
    }
}
```

# 📕 1. Конфигурация Consumer

## 1. Настройка ConnectionFactory
Минимальные параметры:
- Адрес брокера (host, port)
- Учётные данные (username, password)
- Виртуальный хост (virtual host)

```java
@Bean
public ConnectionFactory connectionFactory() {
    CachingConnectionFactory factory = new CachingConnectionFactory();
    
    factory.setHost("localhost");
    factory.setPort(5672);
    factory.setUsername("guest");
    factory.setPassword("guest");
    
    return factory;
}
```

## 2. Способы конфигурации контейнера и слушателей

### Способ 1: Ручное создание контейнера (низкоуровневый)
```java
@Bean
public SimpleMessageListenerContainer messageListenerContainer(
        ConnectionFactory connectionFactory,
        MessageListenerAdapter listenerAdapter) 
{
    SimpleMessageListenerContainer container = 
        new SimpleMessageListenerContainer();
        
    container.setConnectionFactory(connectionFactory);
    container.setQueueNames("my-queue");
    container.setMessageListener(listenerAdapter);
    container.setConcurrentConsumers(3);
    
    return container;
}

@Bean
public MessageListenerAdapter listenerAdapter(Receiver receiver) {
    return new MessageListenerAdapter(receiver, "receiveMessage");
}
```

### Способ 2: @RabbitListener (высокоуровневый, удобный)

> [!NOTE] НЕОБХОДИМАЯ аннотация `@EnableRabbit`  
> Чтобы `@RabbitListener` работал, добавь `@EnableRabbit` в конфигурацию

```java
@Bean
public SimpleRabbitListenerContainerFactory rabbitListenerContainerFactory
(
	ConnectionFactory connectionFactory
) {
    SimpleRabbitListenerContainerFactory factory = 
        new SimpleRabbitListenerContainerFactory();
        
    factory.setConnectionFactory(connectionFactory);
    factory.setConcurrentConsumers(3);
    factory.setPrefetchCount(10); // Количество сообщений "в работе"
    
    return factory;
}

@Component
public class MessageReceiver {

    @RabbitListener(queues = "my-queue")
    public void receiveMessage(String message) {
        System.out.println("Received: " + message);
    }
}
```

В аннотации @RabitListener можно также декларировать компоненты, а также указывать метод обработки ошибок
(см конспекты в папке `8. Spring and AMQP/3.Прием сообщений/2. Аннотированный подход`)

# 📕 2. Принятие сообщений
В аннотированном подходе, мы вещаем аннотацию `@RabbitListener` над методом, который будет принимать сообщения. 

Когда COnsumer подключится к RabbitMQ - то брокер будет PUSH-ить сообщения в COnsumer. Spring полученное сообщение будет преобразовывать в объект указанный в сигнатуре метода

## Допустимые сигнатуры методов с @RabbitListener
### 1. Простое сообщение
```java
@RabbitListener(queues = "queue")
public void receive(String message) {
    // Обработка
}
```

### 2. С объектом Message
```java
@RabbitListener(queues = "queue")
public void receive(Message message) {
    byte[] body = message.getBody();
    MessageProperties properties = message.getMessageProperties();
    // Доступ ко всем свойствам
}
```

### 3. С аннотациями для заголовков
```java
@RabbitListener(queues = "queue")
public void receive(
    @Payload String body,
    @Header("custom-header") String customHeader,
    @Header(AmqpHeaders.CONSUMER_TAG) String consumerTag,
    @Header(AmqpHeaders.DELIVERY_TAG) Long deliveryTag) 
{
    // Обработка с доступом к заголовкам
}
```

### 4. С Channel для ручного ack/nack
```java
@RabbitListener(queues = "queue")
public void receive(
    String message, 
    Channel channel,
    @Header(AmqpHeaders.DELIVERY_TAG) long tag) throws IOException 
{
    try {
        // Обработка
        channel.basicAck(tag, false); // Подтверждение
    } catch (Exception e) {
        channel.basicNack(tag, false, true); // Возврат в очередь
    }
}
```

#### Режимы подтверждения (Acknowledge Mode)

|Режим|Описание|
|---|---|
|AUTO|Spring подтверждает автоматически после успешной обработки|
|MANUAL|Ручное подтверждение через Channel.basicAck()|
|NONE|Без подтверждения (autoack)|

Настройка:
```java
factory.setAcknowledgeMode(AcknowledgeMode.MANUAL);
```

    factory.setMessageConverter(messageConverter);

## Пакетная обработка (Batch)
```java
@Bean
public SimpleRabbitListenerContainerFactory batchFactory() {
    SimpleRabbitListenerContainerFactory factory = 
        new SimpleRabbitListenerContainerFactory();
        
    factory.setConnectionFactory(connectionFactory());
    
    factory.setBatchListener(true); // Включаем пакетный режим
    factory.setBatchSize(10); // Размер пакета
    
    return factory;
}

@RabbitListener(
	queues = "queue", 
	containerFactory = "batchFactory")
public void receiveBatch(List<Message> messages) {
    for (Message message : messages) {
        // Обработка каждого сообщения в пакете
    }
}
```

# 📕 3. Расширенные возможности

## Ретраи и обработка ошибок
```java
@Bean
public SimpleRabbitListenerContainerFactory rabbitListenerContainerFactory() {
    SimpleRabbitListenerContainerFactory factory = 
        new SimpleRabbitListenerContainerFactory();
    
    // Настройка retry
    RetryInterceptorBuilder<?, ?> builder = 
        RetryInterceptorBuilder.stateless()
            .maxAttempts(3)
            .backOffOptions(1000, 2.0, 10000);
    
    factory.setAdviceChain(builder.build());
    
    // Настройка обработки ошибок
    factory.setErrorHandler(new ConditionalRejectingErrorHandler(
        new DefaultExceptionStrategy()));
    
    return factory;
}
```

## Динамическое управление подписками
```java
@RabbitListener(id = "myListener", queues = "queue")
public void listen(
    String msg, 
    @Header(AmqpHeaders.CONSUMER_TAG) String consumerTag) {
    // Можно управлять подпиской
}

// Где-то в коде:
@Autowired
private RabbitListenerEndpointRegistry registry;

public void stopListener() {
    MessageListenerContainer container = 
        registry.getListenerContainer("myListener");
    container.stop();
}

public void startListener() {
    MessageListenerContainer container = 
        registry.getListenerContainer("myListener");
    container.start();
}
```

## Фильтрация сообщений
```java
@Bean
public SimpleRabbitListenerContainerFactory rabbitListenerContainerFactory() {
    SimpleRabbitListenerContainerFactory factory = 
        new SimpleRabbitListenerContainerFactory();
    
    // Добавляем фильтр
    factory.setAfterReceivePostProcessors(message -> {
        // Проверяем заголовки или тело сообщения
        String type = message.getMessageProperties()
            .getHeader("message-type");
        return "valid".equals(type) ? message : null; // null = отбросить
    });
    
    return factory;
}
```
