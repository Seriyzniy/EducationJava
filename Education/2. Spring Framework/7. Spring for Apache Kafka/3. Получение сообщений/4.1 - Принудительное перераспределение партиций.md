С версии **3.1.2** Spring Kafka позволяет вызвать принудительное перераспределение (enforced rebalance) на Kafka Consumer через контейнер слушателя.

- **Что происходит:** вызов `enforceRebalance()` просто уведомляет потребителя о необходимости перераспределения; фактический ребаланс произойдёт при следующем `poll()`.
    
- Если ребаланс уже идёт, вызов **не выполняется (NO-OP)**.
    
- Перед повторным вызовом нужно дождаться завершения текущего ребаланса.

## Пример
```java
@KafkaListener(id = "my.id", topics = "my-topic")
void listen(ConsumerRecord<String, String> record) {
    System.out.println("From KafkaListener: " + record);
}

@Bean
public ApplicationRunner runner(KafkaListenerEndpointRegistry registry) {
    return args -> {
        MessageListenerContainer container = 
	        registry.getListenerContainer("my.id");
        
        System.out.println("Enforcing a rebalance");
        Thread.sleep(5000);
        
        // 🔥инициирует ребаланс при следующем poll()
        container.enforceRebalance(); 
        
        
        Thread.sleep(5000);
    };
}
```
- Доступ к контейнеру слушателя осуществляется через **`KafkaListenerEndpointRegistry`**.
    
- Вызов `enforceRebalance()` делегируется на внутренний Kafka Consumer.
