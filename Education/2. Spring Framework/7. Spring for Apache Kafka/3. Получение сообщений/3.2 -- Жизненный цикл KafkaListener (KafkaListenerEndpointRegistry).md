## 1. Регистрация контейнеров
- **Контейнеры**, создаваемые для `@KafkaListener`, **не являются обычными бинами**.
    
- Они регистрируются в инфраструктурном бине **`KafkaListenerEndpointRegistry`**, который управляет их жизненным циклом.
    
- Если `autoStartup=true`, контейнеры запускаются автоматически при старте приложения.
```java
@KafkaListener(
	id = "myContainer", 
	topics = "myTopic", 
	autoStartup = "false")
public void listen(String msg) { ... }

@Autowired
private KafkaListenerEndpointRegistry registry;

// Запуск контейнера вручную
registry.getListenerContainer("myContainer").start();
```

## 2. Получение контейнеров
- **Все контейнеры:** `registry.getListenerContainers()`
    
- **По ID:** `registry.getListenerContainer("id")`
    
- **Фильтрация по ID или состоянию:** с версии 3.2 можно использовать `getListenerContainersMatching()`:
```java
// По префиксу ID
Collection<MessageListenerContainer> filtered = 
    registry.getListenerContainersMatching(
	    id -> id.startsWith("productListener"));

// По регулярному выражению
Collection<MessageListenerContainer> regexFiltered = 
    registry.getListenerContainersMatching(myPattern::matches);

// По набору ID
Collection<MessageListenerContainer> setFiltered = 
    registry.getListenerContainersMatching(myIdSet::contains);

// По ID и состоянию контейнера
Collection<MessageListenerContainer> advanced = 
    registry.getListenerContainersMatching((id, container) -> 
        id.startsWith("prefix") && container.isRunning()
    );
```

## 3. Особенности запуска
- Контейнеры, зарегистрированные после инициализации контекста, запускаются **независимо от `autoStartup`**, чтобы соответствовать контракту SmartLifecycle.
	
- С версии 2.8.7 можно отключить это поведение через `registry.setAlwaysStartAfterRefresh(false)`.

## 4. Важно
- Контейнеры, объявленные как обычные Spring-бины, **не управляются registry**.
	
- Registry управляет только контейнерами, созданными через `@KafkaListener`.
