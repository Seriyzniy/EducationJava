С версии **2.0**, если метод, аннотированный `@KafkaListener`, возвращает результат и дополнительно аннотирован `@SendTo`, **результат отправляется в указанный топик**.

(Результат МЕТОДА слушателя, попадает в SendTo)

## **Формы `@SendTo`**
1. **Статический топик:**
```java
@SendTo("someTopic") // отправка в конкретный топик
```

2. **Выражение SpEL на этапе инициализации контекста:**
```java
@SendTo("#{myBean.replyTopic}")
``` 

3. **Выражение SpEL на этапе выполнения:**
```java
@SendTo("!{request.value()}")
``` 
- `request` — входящий `ConsumerRecord` или `ConsumerRecords` (для batch)
    
- `source` — объект `Message<?>`, преобразованный из запроса
    
- `result` — возвращаемое значение метода
    

4. **Без параметров `@SendTo`:**
- Используется топик из заголовка `kafka_replyTopic` 
(`!{source.headers['kafka_replyTopic']}`)

---
# Примеры
```java
@KafkaListener(topics = "annotated21")
@SendTo("!{request.value()}") // runtime SpEL
public String replyingListener(String in) { ... }

@KafkaListener(topics = "${some.property:annotated22}")
@SendTo("#{myBean.replyTopic}") // config time SpEL
public Collection<String> replyingBatchListener(List<String> in) { ... }

@KafkaListener(
	topics = "annotated23", 
	errorHandler = "replyErrorHandler")
@SendTo("annotated23reply") // статический топик
public String replyingListenerWithErrorHandler(String in) { ... }
```

---
# 1. Фабрика контейнера и KafkaTemplate
Чтобы `@SendTo` работал, **фабрика контейнера слушателя должна иметь replyTemplate**, которым будет `KafkaTemplate` (❗❗не `ReplyingKafkaTemplate`❗❗).

```java
@Bean
public ConcurrentKafkaListenerContainerFactory<Integer, String> kafkaListenerContainerFactory() {
    ConcurrentKafkaListenerContainerFactory<Integer, String> 
    factory =
        new ConcurrentKafkaListenerContainerFactory<>();
    
    factory.setConsumerFactory(cf());
    
    // KafkaTemplate для отправки ответа
    factory.setReplyTemplate(template()); 
    return factory;
}
```

## **Возврат `Message<?>`**
Если метод возвращает 
`Message<?>` или `Collection<Message<?>>`, 

**сам метод отвечает за заголовки**:
```java
@KafkaListener(id = "messageReturned", topics = "someTopic")
public Message<?> listen(String in,
        @Header(KafkaHeaders.REPLY_TOPIC) byte[] replyTo,
        @Header(KafkaHeaders.CORRELATION_ID) byte[] correlation) {

    return MessageBuilder.withPayload(in.toUpperCase())
            .setHeader(KafkaHeaders.TOPIC, replyTo)
            .setHeader(KafkaHeaders.KEY, 42)
            .setHeader(KafkaHeaders.CORRELATION_ID, correlation)
            .build();
}
```

## **Обработка ошибок и `@SendTo`**
Даже если метод **не возвращает результат**, `@SendTo` можно использовать с `errorHandler`, чтобы отправить информацию о неудаче в отдельный топик:

```java
@KafkaListener(
	id = "voidListenerWithReplyingErrorHandler", 
	topics = "someTopic",
    errorHandler = "voidSendToErrorHandler")
@SendTo("failures")
public void voidListenerWithReplyingErrorHandler(String in) {
    throw new RuntimeException("fail");
}

@Bean
public KafkaListenerErrorHandler voidSendToErrorHandler() {
    return (m, e) -> {
        return ... // данные об ошибке и входное сообщение
    };
}
```

## **Особенности для коллекций**
- Если метод возвращает `Iterable`, по умолчанию создается отдельный `ProducerRecord` на элемент.
    
- С версии 2.3.5 можно отключить разделение через `splitIterables = false`, тогда вся коллекция отправляется как один `ProducerRecord`.
    
- Для `Iterable<Message<?>>` `splitIterables` игнорируется, каждый `Message` отправляется отдельно.
