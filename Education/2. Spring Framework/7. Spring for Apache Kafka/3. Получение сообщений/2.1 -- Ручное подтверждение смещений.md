При использовании `AckMode.MANUAL` или `AckMode.MANUAL_IMMEDIATE` подтверждения должны выполняться **строго по порядку**, так как Kafka не отслеживает каждую запись отдельно, а только закоммиченное смещение для каждой группы/раздела.

**Новый режим (версия 2.8+):**  
Свойство `asyncAcks` позволяет подтверждать записи из одного poll-запроса **в любом порядке**. 

Контейнер откладывает неупорядоченные коммиты до получения всех подтверждений. 
Потребитель приостанавливается (новые записи не доставляются), пока все смещения из предыдущего poll-запроса не будут подтверждены.

**⚠️ Важные ограничения:**
- Увеличивает вероятность **дублирующих доставок** при сбоях
    
- **Несовместимо с `nack()`** (негативными подтверждениями)
