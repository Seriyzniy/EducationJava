Вы можете **получать сообщения**:
- настроив `MessageListenerContainer` и предоставив прослушиватель сообщений 
- или используя аннотацию @KafkaListener.

# 1. Концепция Consumer в Spring Kafka

## 1. Что такое Consumer в Kafka?
**Consumer** - это клиент Kafka, который:
- Подключается к Kafka-кластеру
- Читает сообщения из топиков
- Отслеживает свою позицию (offset) в каждом partition
- Участвует в rebalance при изменении количества потребителей в группе

**На низком уровне (чистый Kafka API):**
```java
// Вручную создаем и управляем consumer
Properties props = new Properties();
props.put(
	"bootstrap.servers", "localhost:9092");
props.put(
	"group.id", "test");

KafkaConsumer<String, String> consumer = 
	new KafkaConsumer<>(props);

consumer.subscribe(Arrays.asList("topic"));

while (true) {
    ConsumerRecords<String, String> records = 
	    consumer.poll(Duration.ofMillis(100));
    
    for (ConsumerRecord<String, String> record : records) {
        // Обрабатываем каждое сообщение
        processRecord(record);
    }
    // Вручную коммитим offsets
    consumer.commitSync();
}
```


## 2. Listener/Container в Spring Kafka
> [!NOTE] Обертка над Consumer
> Spring Kafka не заменяет Consumer, а ОБЕРТЫВАЕТ его!
### Аналогия:
- **Consumer** = Двигатель автомобиля
    
- **Listener Container** = Весь автомобиль с коробкой передач, рулем, тормозами
    
- **Message Listener** = Водитель, который только говорит "куда ехать"

### Принцип работы
##### 1. Spring создает Consumer через ConsumerFactory
```java
@Bean
public ConsumerFactory<String, String> consumerFactory() {
    // Spring создает настоящие KafkaConsumer-ы
    return new DefaultKafkaConsumerFactory<>(consumerConfigs());
}
```

#### 2. 🔥Container управляет жизненным циклом🔥 Consumer:
```java
@Bean
public ConcurrentKafkaListenerContainerFactory<String, String> containerFactory() 
{
    ConcurrentKafkaListenerContainerFactory<String, String> 
	    factory = 
		    new ConcurrentKafkaListenerContainerFactory<>();
    
    factory.setConsumerFactory(consumerFactory());
    factory.setConcurrency(3); // Создает 3 Consumer-а!
    return factory;
}
```

#### 3. Вы пишете только бизнес-логику
```java
@Component
public class MyBusinessLogic {
    
    @KafkaListener(
	    topics = "my-topic", 
	    containerFactory = "containerFactory")
    public void handleMessage(String message) {
        // Spring сам создал Consumer, подписался на топик,
        // получает сообщения и вызывает этот метод
        System.out.println("Received: " + message);
    }
}
```

### Под капотом
Когда Spring видит `@KafkaListener`:
1. **Создает реальные KafkaConsumer-ы** через вашу ConsumerFactory
    
2. 🔥**Запускает контейнер**🔥, который в фоновом потоке:
    - Вызывает `consumer.poll()`
    - Обрабатывает rebalance
    - Управляет коммитами offsets
    - Обрабатывает ошибки
        
3. **Когда приходят сообщения** → вызывает ваш метод `handleMessage()`

# 2. Практическое руководство по приёму сообщений
## **1️⃣ Настройка свойств потребителя (Consumer Properties)**
Прежде чем создать фабрику потребителей, нужно определить **основные свойства Kafka Consumer**:
- `bootstrap.servers` — адрес брокера Kafka
    
- `key.deserializer` и `value.deserializer` — десериализация ключа и значения
    
- `group.id` — группа потребителей
    
- `auto.offset.reset` — поведение при отсутствии смещения


**Пример кода:**
```java
@Bean
public Map<String, Object> consumerConfigs() {
    Map<String, Object> props = new HashMap<>();
    props.put(
	    ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, 
	    "localhost:9092");
    props.put(
	    ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG, 
		IntegerDeserializer.class);
    props.put(
	    ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG, 
	    StringDeserializer.class);
    props.put(
	    ConsumerConfig.GROUP_ID_CONFIG, "myGroup");
    props.put(
	    ConsumerConfig.AUTO_OFFSET_RESET_CONFIG, "earliest");
    return props;
}
```

## **2️⃣ Создание ConsumerFactory**
`ConsumerFactory` отвечает за создание экземпляров `KafkaConsumer` с указанными свойствами.

**Пример кода:**
```java
@Bean
public ConsumerFactory<Integer, String> consumerFactory() {
    return new DefaultKafkaConsumerFactory<>(consumerConfigs());
}
```

## **3️⃣ Настройка фабрики контейнера слушателя (KafkaListenerContainerFactory)**
`KafkaListenerContainerFactory` создаёт **контейнеры слушателя**,которые управляют потребителями и их потоками.

- Для параллельной обработки используется `ConcurrentKafkaListenerContainerFactory`
    
- Можно настроить:
    - количество потоков (`concurrency`)
    - таймаут `pollTimeout`
    - batch-листенеры (`batchListener = true`)
    - AckMode для ручного подтверждения

**Пример кода:**
```java
@Bean
public KafkaListenerContainerFactory<ConcurrentMessageListenerContainer<Integer, String>> kafkaListenerContainerFactory() 
{
    ConcurrentKafkaListenerContainerFactory<Integer, String> 
	    factory =
            new ConcurrentKafkaListenerContainerFactory<>();
    
    factory.setConsumerFactory(consumerFactory());
    factory.setConcurrency(3); // три потока потребителей
    factory.getContainerProperties().setPollTimeout(3000); // таймаут poll
    factory
	    .getContainerProperties()
	    .setAckMode(ContainerProperties.AckMode.MANUAL); // ручное подтверждение
    
    return factory;
}
```

## **4️⃣ Создание слушателей сообщений (@KafkaListener)**
Аннотация `@KafkaListener` связывает метод бина с Kafka-топиком:
```java
@Component
public class MyListener {
    @KafkaListener(id = "listener1", topics = "myTopic", containerFactory = "kafkaListenerContainerFactory")
    public void listen(String message, Acknowledgment ack) {
        System.out.println("Получено сообщение: " + message);
        ack.acknowledge(); // ручное подтверждение смещения
    }
}
```
