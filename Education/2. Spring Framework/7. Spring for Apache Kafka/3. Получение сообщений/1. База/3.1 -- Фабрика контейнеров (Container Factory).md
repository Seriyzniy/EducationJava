Как обсуждалось в разделе "Аннотация @KafkaListener", `ConcurrentKafkaListenerContainerFactory` **используется для создания контейнеров** *для аннотированных методов*.

## Создание контейнеров вручную:
Начиная с версии 2.2, вы можете использовать ту же фабрику для создания любого `ConcurrentMessageListenerContainer`. 

Это может быть полезно, если вы хотите создать **несколько КОНТЕЙНЕРОВ с похожими свойствами** или вы хотите использовать некоторую внешне настроенную фабрику, такую как предоставленная автоконфигурацией Spring Boot. 

После создания контейнера вы можете дополнительно изменить его свойства, многие из которых устанавливаются с помощью `container.getContainerProperties()`. 

Следующий пример настраивает `ConcurrentMessageListenerContainer`:
```java
@Bean
public 
ConcurrentMessageListenerContainer<String, String> myContainer(
	ConcurrentKafkaListenerContainerFactory<String, String> 
	factory) 
{
    ConcurrentMessageListenerContainer<String, String> container 
    = factory
	    .createContainer("topic1", "topic2");
    
    container.setMessageListener(m -> { ... } );
    return container;
}
```

ConcurrentKafkaListenerContainerFactory<K, V>` — это дженерик-класс из Spring Kafka, который параметризуется:
- **K** — тип _ключа сообщения_
- **V** — тип _значения сообщения_

Контейнеры, созданные таким способом, не добавляются в реестр конечных точек (endpoint registry). Они должны быть созданы как определения `@Bean`, чтобы они были зарегистрированы в контексте приложения.

## ContainerCustomizer - настройка каждого контейнера
Начиная с версии 2.3.4, вы можете добавить `ContainerCustomizer` в фабрику для дальнейшей настройки каждого контейнера после его создания и конфигурации.
```java
@Bean
public KafkaListenerContainerFactory<?> kafkaListenerContainerFactory() {
    ConcurrentKafkaListenerContainerFactory<Integer, String> 
    factory =
	    new ConcurrentKafkaListenerContainerFactory<>();
    ...
    
    factory.setContainerCustomizer(container -> { /* настроить контейнер */ });
    return factory;
}
```

## ContainerPostProcessor - настройка отдельного слушателя
Начиная с версии 3.1, также возможно применять тот же вид настройки на отдельном слушателе, указав имя бина `ContainerPostProcessor` в аннотации `KafkaListener`.
```java
@Bean
public ContainerPostProcessor<
	String, 
	String, 
	AbstractMessageListenerContainer<String, String>> customContainerPostProcessor() 
{
    return container -> { /* настроить контейнер */ };
}

...

@KafkaListener(
	..., 
	containerPostProcessor="customContainerPostProcessor", 
	...)
public void listen(...) {
    ...
}
```
