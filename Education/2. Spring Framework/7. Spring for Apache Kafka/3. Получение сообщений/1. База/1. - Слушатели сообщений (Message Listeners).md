Когда вы используете контейнер слушателя сообщений, вы должны предоставить слушатель для получения данных. 
> По сути в слушателе **мы говорим, как обрабатывать сообщения**
> А контейнер управляет жизненным циклом Cosumer (подписывается на топик, выполнят poll и прочее)


В настоящее время существует **8 поддерживаемых интерфейсов для слушателей сообщений**. 

Следующий листинг показывает эти интерфейсы:
```java
public interface MessageListener<K, V> { 
    void onMessage(ConsumerRecord<K, V> data);
}

public interface AcknowledgingMessageListener<K, V> { 
    void onMessage(
	    ConsumerRecord<K, V> data, 
	    Acknowledgment acknowledgment);
}

public interface ConsumerAwareMessageListener<K, V> extends MessageListener<K, V> { 
    void onMessage(
	    ConsumerRecord<K, V> data, 
	    Consumer<?, ?> consumer);
}

public interface AcknowledgingConsumerAwareMessageListener<K, V> extends MessageListener<K, V> { 
    void onMessage(
	    ConsumerRecord<K, V> data, 
	    Acknowledgment acknowledgment, 
	    Consumer<?, ?> consumer);
}


public interface BatchMessageListener<K, V> { 
    void onMessage(List<ConsumerRecord<K, V>> data);
}

public interface BatchAcknowledgingMessageListener<K, V> { 
    void onMessage(
	    List<ConsumerRecord<K, V>> data, 
	    Acknowledgment acknowledgment);
}

public interface BatchConsumerAwareMessageListener<K, V> extends BatchMessageListener<K, V> { 
    void onMessage(
	    List<ConsumerRecord<K, V>> data, 
	    Consumer<?, ?> consumer);
}

public interface BatchAcknowledgingConsumerAwareMessageListener<K, V> extends BatchMessageListener<K, V> { 
    void onMessage(
	    List<ConsumerRecord<K, V>> data, 
	    Acknowledgment acknowledgment, 
	    Consumer<?, ?> consumer);
}
```

**Описание каждого интерфейса:**
1. **`MessageListener<K, V>`** - Используйте этот интерфейс **для обработки отдельных экземпляров** `ConsumerRecord`, полученных из операции `poll()` потребителя Kafka, *при использовании авто-подтверждения* (auto-commit) *или одного из методов подтверждения*, управляемых контейнером.
    
2. **`AcknowledgingMessageListener<K, V>`** - Используйте этот интерфейс для обработки отдельных экземпляров `ConsumerRecord`, полученных из операции `poll()` потребителя Kafka, *при использовании одного из методов ручного подтверждения*.
    
3. **`ConsumerAwareMessageListener<K, V>`** - Используйте этот интерфейс для обработки отдельных экземпляров `ConsumerRecord`, полученных из операции `poll()` потребителя Kafka,* при использовании авто-подтверждения или одного из методов подтверждения, управляемых контейнером*. Предоставляется доступ к объекту `Consumer`.
    
4. **`AcknowledgingConsumerAwareMessageListener<K, V>`** - Используйте этот интерфейс для обработки отдельных экземпляров `ConsumerRecord`, полученных из операции `poll()` потребителя Kafka, *при использовании одного из методов ручного подтверждения*. Предоставляется доступ к объекту `Consumer`.
    
5. **`BatchMessageListener<K, V>`** - Используйте этот интерфейс для обработки всех экземпляров `ConsumerRecord`, полученных из операции `poll()` потребителя Kafka, при использовании авто-подтверждения или одного из методов подтверждения, управляемых контейнером. `AckMode.RECORD` не поддерживается, когда вы используете этот интерфейс, поскольку слушателю предоставляется полная пачка.
    
6. **`BatchAcknowledgingMessageListener<K, V>`** - Используйте этот интерфейс для обработки всех экземпляров `ConsumerRecord`, полученных из операции `poll()` потребителя Kafka, при использовании одного из методов ручного подтверждения.
    
7. **`BatchConsumerAwareMessageListener<K, V>`** - Используйте этот интерфейс для обработки всех экземпляров `ConsumerRecord`, полученных из операции `poll()` потребителя Kafka, при использовании авто-подтверждения или одного из методов подтверждения, управляемых контейнером. `AckMode.RECORD` не поддерживается, когда вы используете этот интерфейс, поскольку слушателю предоставляется полная пачка. Предоставляется доступ к объекту `Consumer`.
    
8. **`BatchAcknowledgingConsumerAwareMessageListener<K, V>`** - Используйте этот интерфейс для обработки всех экземпляров `ConsumerRecord`, полученных из операции `poll()` потребителя Kafka, при использовании одного из методов ручного подтверждения. Предоставляется доступ к объекту `Consumer`.
    

**Важные замечания:**
- Объект `Consumer` не является потокобезопасным. Вы должны вызывать его методы только в том потоке, который вызывает слушатель.
    
- Вы не должны выполнять любые методы `Consumer<?, ?>`, которые влияют на позиции потребителя или подтвержденные смещения (committed offsets) в вашем слушателе; контейнер должен управлять такой информацией.
    

--- 
# 💡 Практический совет:

**Не зацикливайтесь на запоминании всех этих интерфейсов!** На практике в 95% случаев вы будете использовать **`@KafkaListener`**, который автоматически обрабатывает все эти низкоуровневые детали.

Эти интерфейсы полезно знать для понимания того, как работает Spring Kafka под капотом, но для повседневной разработки `@KafkaListener` гораздо удобнее.
