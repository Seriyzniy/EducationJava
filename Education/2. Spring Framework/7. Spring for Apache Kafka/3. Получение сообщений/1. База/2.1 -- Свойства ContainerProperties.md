# Примеры
## ContainerProperties
### 1. ContainerProperties - через фабрику контейнеров
```java
@Bean
public ConcurrentKafkaListenerContainerFactory<String, String> kafkaListenerContainerFactory() {
    ConcurrentKafkaListenerContainerFactory<String, String> 
    factory = 
        new ConcurrentKafkaListenerContainerFactory<>();
    factory.setConsumerFactory(consumerFactory());
    
    // Настройка ContainerProperties
	factory
		.getContainerProperties()
		.setAckMode(ContainerProperties.AckMode.BATCH);
		
    factory.getContainerProperties().setPollTimeout(3000);
    factory.getContainerProperties().setMissingTopicsFatal(true);
    factory.getContainerProperties().setSyncCommits(true);
    factory.getContainerProperties().setIdleEventInterval(30000L);
    factory.getContainerProperties().setMicrometerEnabled(true);
    factory.getContainerProperties().setLogContainerConfig(true);
    
    return factory;
}
```

### 2. ContainerProperties - при создании контейнера вручную
```java
@Bean
public KafkaMessageListenerContainer<String, String> manualContainer() {
    ContainerProperties containerProps = new ContainerProperties("my-topic");
    
    // Прямая настройка всех параметров
    containerProps
	    .setAckMode(
		    ContainerProperties.AckMode.MANUAL_IMMEDIATE);
    containerProps.setPollTimeout(5000);
    containerProps.setAckCount(100);
    containerProps.setAckTime(10000L);
    containerProps.setIdleBetweenPolls(1000L);
    containerProps.setMonitorInterval(60L);
    containerProps.setNoPollThreshold(4.0f);
    containerProps.setShutdownTimeout(15000L);
    containerProps.setCommitLogLevel(LogIfLevelEnabled.Level.INFO);
    
    return new KafkaMessageListenerContainer<>(consumerFactory(), containerProps);
}
```

## AbstractMessageListenerContainer - через фабрику
```java
@Bean
public ConcurrentKafkaListenerContainerFactory<String, String> advancedFactory() {
    ConcurrentKafkaListenerContainerFactory<String, String> 
    factory = 
        new ConcurrentKafkaListenerContainerFactory<>();
    factory.setConsumerFactory(consumerFactory());
    
    // Настройка обработчиков ошибок и интерцепторов
    factory.setCommonErrorHandler(new DefaultErrorHandler());
    factory.setBatchErrorHandler(new BatchLoggingErrorHandler());
    
    // Record Interceptor
    factory.setRecordInterceptor(record -> {
        System.out.println("Processing record: " + record.key());
        return record;
    });
    
    // Batch Interceptor  
    factory.setBatchInterceptor(records -> {
        System.out.println("Processing batch of " + records.count() + " records");
        return records;
    });
    
    return factory;
}
```

## KafkaMessageListenerContainer - прямое создание
```java
@Bean
public KafkaMessageListenerContainer<String, String> singleThreadContainer() {
    ContainerProperties containerProps = new 
	    ContainerProperties("single-topic");
    
    KafkaMessageListenerContainer<String, String> container = 
        new KafkaMessageListenerContainer<>(consumerFactory(), containerProps);
    
    // Уникальный clientId для этого контейнера
    container.setClientIdSuffix("single-processor");
    
    return container;
}
```

## ConcurrentMessageListenerContainer - многопоточность
```java
@Bean
public ConcurrentMessageListenerContainer<String, String> concurrentContainer() {
    ContainerProperties containerProps = 
	    new ContainerProperties("concurrent-topic");
    
    ConcurrentMessageListenerContainer<String, String> 
    container = 
        new ConcurrentMessageListenerContainer<>
	        (consumerFactory(), containerProps);
    
    // Настройка многопоточности
    container.setConcurrency(3);
    container.setAlwaysClientIdSuffix(false); // Отключаем суффиксы для client.id
    
    return container;
}
```

## Через @KafkaListener аннотацию
```java
@Component
public class AnnotatedListener {
    
    // Основные параметры в аннотации
    @KafkaListener(
        id = "annotated-listener",
        topics = "annotated-topic",
        groupId = "annotated-group",
        concurrency = "2",
        autoStartup = "false",
        clientIdPrefix = "annotated",
        properties = {
            "max.poll.interval.ms:300000",
            "max.poll.records:500"
        }
    )
    public void handleMessage(String message) {
        // Обработка сообщения
    }
    
    // Использование разных фабрик
    @KafkaListener(
        id = "batch-listener", 
        topics = "batch-topic",
        containerFactory = "batchFactory"
    )
    public void handleBatch(List<String> messages) {
        // Обработка пачки
    }
}
```

```java
spring:

      poll-timeout: 3000
      concurrency: 3
      missing-topics-fatal: true

    consumer:
      auto-offset-reset: earliest
      enable-auto-commit: false
      max-poll-records: 500
      properties:
        max.poll.interval.ms: 300000
```

# ContainerProperties Properties
## ackCount
**Значение по умолчанию**: 1  
**Описание**: Количество записей перед коммитом ожидающих смещений, когда ackMode установлен в COUNT или COUNT_TIME

## adviceChain
**Значение по умолчанию**: null  
**Описание**: Цепочка объектов Advice (например, MethodInterceptor around advice), оборачивающая слушатель сообщений, вызывается по порядку

## ackMode
**Значение по умолчанию**: BATCH  
**Описание**: Управляет тем, как часто коммитятся смещения - см. Committing Offsets

## ackTime
**Значение по умолчанию**: 5000  
**Описание**: Время в миллисекундах, после которого ожидающие смещения коммитятся, когда ackMode установлен в TIME или COUNT_TIME

## assignmentCommitOption
**Значение по умолчанию**: LATEST_ONLY_NO_TX  
**Описание**: Следует ли коммитить начальную позицию при назначении; по умолчанию начальное смещение будет закоммичено только если ConsumerConfig.AUTO_OFFSET_RESET_CONFIG установлен в latest, и это не будет выполняться в транзакции, даже если присутствует менеджер транзакций

## asyncAcks
**Значение по умолчанию**: false  
**Описание**: Включает неупорядоченные коммиты; потребитель приостанавливается и коммиты откладываются до заполнения пропусков

## authExceptionRetryInterval
**Значение по умолчанию**: null  
**Описание**: Когда не null, Duration для сна между опросами, когда Kafka клиент выбрасывает AuthenticationException или AuthorizationException. Когда null, такие исключения считаются фатальными и контейнер остановится

## batchRecoverAfterRollback
**Значение по умолчанию**: false  
**Описание**: Установить в true для включения восстановления пачек после отката

## clientId
**Значение по умолчанию**: (empty string)  
**Описание**: Префикс для свойства client.id потребителя. Переопределяет свойство client.id фабрики потребителя; в concurrent контейнере добавляется суффикс -n для каждого экземпляра потребителя

## checkDeserExWhenKeyNull
**Значение по умолчанию**: false  
**Описание**: Установить в true для постоянной проверки заголовка DeserializationException при получении null ключа. Полезно когда код потребителя не может определить, что настроен ErrorHandlingDeserializer

## checkDeserExWhenValueNull
**Значение по умолчанию**: false  
**Описание**: Установить в true для постоянной проверки заголовка DeserializationException при получении null значения. Полезно когда код потребителя не может определить, что настроен ErrorHandlingDeserializer

## commitCallback
**Значение по умолчанию**: null  
**Описание**: Когда присутствует и syncCommits false, callback вызывается после завершения коммита

## commitLogLevel
**Значение по умолчанию**: DEBUG  
**Описание**: Уровень логирования для логов, относящихся к коммиту смещений

## consumerRebalanceListener
**Значение по умолчанию**: null  
**Описание**: Слушатель ребалансировки; см. Rebalancing Listeners

## commitRetries
**Значение по умолчанию**: 3  
**Описание**: Установить количество повторных попыток при RetriableCommitFailedException при использовании syncCommits установленного в true. По умолчанию 3 (всего 4 попытки)

## consumerStartTimeout
**Значение по умолчанию**: 30s  
**Описание**: Время ожидания старта потребителя перед логированием ошибки; это может случиться если, например, вы используете исполнитель задач с недостаточным количеством потоков

## deliveryAttemptHeader
**Значение по умолчанию**: false  
**Описание**: См. Delivery Attempts Header

## eosMode
**Значение по умолчанию**: V2  
**Описание**: Режим Exactly Once Semantics; см. Exactly Once Semantics

## fixTxOffsets
**Значение по умолчанию**: false  
**Описание**: При потреблении записей, произведенных transactional продюсером, и когда потребитель позиционирован в конце раздела, lag может некорректно сообщаться как больше нуля

## groupId
**Значение по умолчанию**: null  
**Описание**: Переопределяет свойство group.id потребителя; автоматически устанавливается @KafkaListener id или groupId свойством

## idleBeforeDataMultiplier
**Значение по умолчанию**: 5.0  
**Описание**: Множитель для idleEventInterval, который применяется до получения любых записей. После получения записи множитель больше не применяется

## idleBetweenPolls
**Значение по умолчанию**: 0  
**Описание**: Используется для замедления доставок путем усыпления потока между опросами. Время обработки пачки записей плюс это значение должно быть меньше [max.poll.interval.ms](https://max.poll.interval.ms/) свойства потребителя

## idleEventInterval
**Значение по умолчанию**: null  
**Описание**: Когда установлено, включает публикацию ListenerContainerIdleEvents, см. Application Events and Detecting Idle and Non-Responsive Consumers

## idlePartitionEventInterval
**Значение по умолчанию**: null  
**Описание**: Когда установлено, включает публикацию ListenerContainerIdlePartitionEvents, см. Application Events and Detecting Idle and Non-Responsive Consumers

## kafkaConsumerProperties
**Значение по умолчанию**: None  
**Описание**: Используется для переопределения любых произвольных свойств потребителя, настроенных в фабрике потребителя

## kafkaAwareTransactionManager
**Значение по умолчанию**: null  
**Описание**: См. Transactions

## listenerTaskExecutor
**Значение по умолчанию**: SimpleAsyncTaskExecutor  
**Описание**: Исполнитель задач для запуска потоков потребителя. Исполнитель по умолчанию создает потоки с именами `<name>-C-n`

## logContainerConfig
**Значение по умолчанию**: false  
**Описание**: Установить в true для логирования на уровне INFO всех свойств контейнера

## messageListener
**Значение по умолчанию**: null  
**Описание**: Слушатель сообщений

## micrometerEnabled
**Значение по умолчанию**: true  
**Описание**: Вести или нет таймеры Micrometer для потоков потребителя

## micrometerTags
**Значение по умолчанию**: empty  
**Описание**: Map статических тегов для добавления к метрикам micrometer

## micrometerTagsProvider
**Значение по умолчанию**: null  
**Описание**: Функция, предоставляющая динамические теги на основе записи потребителя

## missingTopicsFatal
**Значение по умолчанию**: false  
**Описание**: Когда true предотвращает старт контейнера если настроенные топики отсутствуют на брокере

## monitorInterval
**Значение по умолчанию**: 30s  
**Описание**: Как часто проверять состояние потоков потребителя для NonResponsiveConsumerEvent

## noPollThreshold
**Значение по умолчанию**: 3.0  
**Описание**: Умножается на pollTimeOut для определения публикации NonResponsiveConsumerEvent

## observationConvention
**Значение по умолчанию**: null  
**Описание**: Когда установлено, добавляет динамические теги к таймерам и трассировкам на основе информации в записях потребителя

## observationEnabled
**Значение по умолчанию**: false  
**Описание**: Установить в true для включения наблюдения через Micrometer

## offsetAndMetadataProvider
**Значение по умолчанию**: null  
**Описание**: Провайдер для OffsetAndMetadata; по умолчанию провайдер создает offset и metadata с пустыми метаданными

## onlyLogRecordMetadata
**Значение по умолчанию**: false  
**Описание**: Установить в false для логирования полной записи потребителя вместо только topic-partition@offset

## pauseImmediate
**Значение по умолчанию**: false  
**Описание**: Когда контейнер приостановлен, остановить обработку после текущей записи вместо обработки всех записей из предыдущего опроса

## pollTimeout
**Значение по умолчанию**: 5000  
**Описание**: Таймаут передаваемый в Consumer.poll() в миллисекундах

## pollTimeoutWhilePaused
**Значение по умолчанию**: 100  
**Описание**: Таймаут передаваемый в Consumer.poll() (в миллисекундах) когда контейнер в приостановленном состоянии

## restartAfterAuthExceptions
**Значение по умолчанию**: false  
**Описание**: True для перезапуска контейнера если он остановлен из-за authorization/authentication исключений

## scheduler
**Значение по умолчанию**: ThreadPoolTaskScheduler  
**Описание**: Планировщик для запуска задачи мониторинга потребителя

## shutdownTimeout
**Значение по умолчанию**: 10000  
**Описание**: Максимальное время в ms для блокировки метода stop() до остановки всех потребителей и перед публикацией события остановки контейнера

## stopContainerWhenFenced
**Значение по умолчанию**: false  
**Описание**: Остановить контейнер слушателя если выброшено ProducerFencedException

## stopImmediate
**Значение по умолчанию**: false  
**Описание**: Когда контейнер остановлен, остановить обработку после текущей записи вместо обработки всех записей из предыдущего опроса

## subBatchPerPartition
**Значение по умолчанию**: false  
**Описание**: При использовании batch слушателя, если true, слушатель вызывается с результатами опроса разделенными на подпачки по одной на раздел

## syncCommitTimeout
**Значение по умолчанию**: null  
**Описание**: Таймаут для использования когда syncCommits true. Когда не установлен, контейнер попытается определить свойство [default.api.timeout.ms](https://default.api.timeout.ms/) потребителя и использовать его

## syncCommits
**Значение по умолчанию**: true  
**Описание**: Использовать синхронные или асинхронные коммиты для смещений; см. commitCallback

## topics/topicPattern/topicPartitions
**Значение по умолчанию**: n/a  
**Описание**: Настроенные топики, шаблон топика или явно назначенные топики/разделы. Взаимоисключающие; хотя бы один должен быть предоставлен

## transactionManager
**Значение по умолчанию**: null  
**Описание**: Устарело с версии 3.2

# ConcurrentMessageListenerContainer Properties

## alwaysClientIdSuffix
**Значение по умолчанию**: true  
**Описание**: Установить в false для подавления добавления суффикса к свойству client.id потребителя, когда параллелизм только 1

## assignedPartitions
**Значение по умолчанию**: (read only)  
**Описание**: Агрегат разделов, в настоящее время назначенных дочерним KafkaMessageListenerContainer этого контейнера (явно или нет)

## concurrency
**Значение по умолчанию**: 1  
**Описание**: Количество дочерних KafkaMessageListenerContainer для управления

## containerPaused
**Значение по умолчанию**: n/a  
**Описание**: True если пауза запрошена и все потребители дочерних контейнеров фактически приостановлены

## containers
**Значение по умолчанию**: n/a  
**Описание**: Ссылка на все дочерние KafkaMessageListenerContainer
