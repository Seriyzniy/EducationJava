Начиная с версии 2.5, вы можете использовать `RoutingKafkaTemplate` **для выбора продюсера во время выполнения (runtime)**, основываясь на имени топика назначения.

> [!NOTE] ❗ ОГРАНИЧЕНИЯ ❗
> Маршрутизирующий шаблон **НЕ ПОДДЕРЖИВАЕТ ТРАНЗАКЦИИ и** , `execute`, `flush` или операции `metrics`, потому что топик для этих операций неизвестен.  

Шаблон требует карту (map) экземпляров `java.util.regex.Pattern` в `ProducerFactory<Object, Object>`. Эта **карта должна быть упорядоченной** (например, `LinkedHashMap`), потому что она просматривается по порядку; вам следует добавлять более специфичные шаблоны в начало.

Следующее простое Spring Boot приложение предоставляет пример того, как использовать один и тот же шаблон для отправки в разные топики, каждый из которых использует разный сериализатор значения.

```java
@SpringBootApplication
public class Application {

    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }

    @Bean
    public RoutingKafkaTemplate routingTemplate(
	    GenericApplicationContext context,
	    ProducerFactory<Object, Object> pf) 
	{
        // 1️⃣Клонируем PF (ProducerFactory) 
        Map<String, Object> configs = 
	        new HashMap<>(pf.getConfigurationProperties());
	    
	    // 2️⃣с другим Serializer, 
        configs.put(
	        ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, 
	        ByteArraySerializer.class);
	        
        DefaultKafkaProducerFactory<Object, Object> bytesPF = 
	        new DefaultKafkaProducerFactory<>(configs);
         
        // 3️⃣регистрируем в Spring для корректного завершения
        context.registerBean(
	        "bytesPF", 
	        DefaultKafkaProducerFactory.class, 
	        () -> bytesPF);


        Map<Pattern, ProducerFactory<Object, Object>> map = 
	        new LinkedHashMap<>();
	    // Для топиков, совпадающих с "two", используем bytesPF
        map.put(Pattern.compile("two"), bytesPF);
        
        // Дефолтный PF с StringSerializer для всех остальных топиков
        map.put(Pattern.compile(".+"), pf); 
        return new RoutingKafkaTemplate(map);
    }

    @Bean
    public ApplicationRunner runner(RoutingKafkaTemplate routingTemplate) {
        return args -> {
	        // Отправит как String
            routingTemplate.send("one", "thing1");
            
            // Отправит как byte[]
            routingTemplate.send("two", "thing2".getBytes());
        };
    }
}
```

> Соответствующие `@KafkaListener` для этого примера показаны в разделе "Annotation Properties".
> (см [[Аннотации Spring Kafka]])

Для другой техники достижения похожих результатов, но с дополнительной возможностью отправлять разные типы в один и тот же топик, смотрите "Delegating Serializer and Deserializer". 
(см [[Delegating Serializer and Deserializer]])
