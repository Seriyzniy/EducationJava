## **1️⃣ Настройка свойств продюсера (Producer Properties)**
Для создания Kafka-продюсера нужны основные свойства:
- `bootstrap.servers` — адрес брокера Kafka
- `key.serializer` и `value.serializer` — сериализация ключа и значения
- `acks` — подтверждение брокером
- `retries` — количество повторов при сбое


**Пример кода:**
```java
@Bean
public Map<String, Object> producerConfigs() {
    Map<String, Object> props = new HashMap<>();
    props.put(
	    ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, 
	    "localhost:9092");
    props.put(
	    ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, 
	    IntegerSerializer.class);
    props.put(
	    ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, 
	    StringSerializer.class);
    props.put(ProducerConfig.ACKS_CONFIG, "all"); // ждём подтверждения от всех реплик
    props.put(ProducerConfig.RETRIES_CONFIG, 3);
    return props;
}
```


## **2️⃣ Создание ProducerFactory**
`ProducerFactory` отвечает за создание экземпляров `KafkaProducer` с указанными свойствами.
```java
@Bean
public ProducerFactory<Integer, String> producerFactory() {
    return new DefaultKafkaProducerFactory<>(producerConfigs());
}
```

## **3️⃣ Создание KafkaTemplate**
`KafkaTemplate` — ==основной класс Spring Kafka для отправки== сообщений. Он оборачивает продюсер и упрощает API.

```java
@Bean
public KafkaTemplate<Integer, String> kafkaTemplate() {
    return new KafkaTemplate<>(producerFactory());
}
```
- Можно использовать для синхронной и асинхронной отправки сообщений
    
- Позволяет задавать key, value и topic

### Отправка сообщений

#### Простая отправка без key**
```java
@Autowired
private KafkaTemplate<Integer, String> kafkaTemplate;

public void sendMessage(String message) {
    kafkaTemplate.send("myTopic", message);
}
```

#### Отправка с ключом
```java
public void sendMessageWithKey(Integer key, String message) {
    kafkaTemplate.send("myTopic", key, message);
}
```
- Key определяет **разделение на партиции** в Kafka
- Одни и те же ключи будут попадать в одну партицию

### Асинхронная обработка результата (ListenableFuture)
`send()` возвращает `ListenableFuture<SendResult<K,V>>`. Можно добавить callback для обработки успешной отправки или ошибки:

```java
public void sendMessageAsync(Integer key, String message) {
    kafkaTemplate
	    .send("myTopic", key, message)
	    .addCallback(
	        result -> System.out.println(
		        "Сообщение отправлено: " + 
		        result.getRecordMetadata()
		    ),
	        ex -> System.err.println(\
		        "Ошибка при отправке: " + ex.getMessage()
		)
    );
}
```

`SendResult.getRecordMetadata()` содержит: topic, partition, offset, timestamp

### ## **Транзакции (опционально)**
Если нужно отправлять несколько сообщений атомарно:
```java
@Bean
public KafkaTemplate<Integer, String> kafkaTransactionalTemplate() 
{
    KafkaTemplate<Integer, String> template = 
	    new KafkaTemplate<>(producerFactory());
    // Установили ID для Prodcuer чтобы использовать транзакции
    template.setTransactionIdPrefix("tx-"); 
    return template;
}

public void sendTransactional(List<String> messages) 
{
	kafkaTransactionalTemplate()
		.executeInTransaction(kafkaTemplate -> {
	        messages.forEach(msg ->
		        kafkaTemplate.send("myTopic", msg)
		    );
        return null;
    });
}
```
