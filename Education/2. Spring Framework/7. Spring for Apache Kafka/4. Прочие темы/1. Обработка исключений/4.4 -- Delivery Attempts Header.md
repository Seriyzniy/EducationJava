
**Delivery Attempts Header** — заголовок количества попыток доставки
    
**Listener Info Header** — заголовок информации о слушателе

# 1. Delivery Attempts Header
Это заголовок **kafka_deliveryAttempt** (KafkaHeaders.DELIVERY_ATTEMPT), который Spring Kafka может добавлять к записи при повторной попытке обработки.


Используется для:
- логгирования
- метрик
- маршрутизации (например, в DLT после N попыток)
- специфичных действий на основе номера попытки

Значение — **целое число (1, 2, 3, …)**  
В raw ConsumerRecord доступно как byte\[4].

## Используется
Работает, если применяется:
- **DefaultErrorHandler**
    
- **DefaultAfterRollbackProcessor**

оба должны реализовывать **DeliveryAttemptAware** (они реализуют).

## Включение заголовка
По умолчанию **выключено**, чтобы избежать лишних аллокаций.

Нужно:
```java
containerProps.setDeliveryAttemptHeader(true);
```

После этого, при retry, Spring будет увеличивать deliveryAttempt.

## Получение значение заголовка
#### ✔ В KafkaListener
```java
@KafkaListener(...)
public void listen(
	String msg,
   @Header(KafkaHeaders.DELIVERY_ATTEMPT) int attempt) {
}
```

#### ✔ Через raw ConsumerRecord
```java
int attempt = ByteBuffer.wrap(
    record
	    .headers()
		.lastHeader(KafkaHeaders.DELIVERY_ATTEMPT).value()
).getInt();
```

# 2. Delivery Attempt Header в Batch Listener (3.3+)
В batch-слушателях заголовок работает иначе:  
он должен быть внедрён **через RetryListener**, потому что batch обрабатывается пачкой.

Для включения:
```java
DefaultErrorHandler errorHandler =
    new DefaultErrorHandler(new FixedBackOff(1, 10));

// Включение слушателя с включенным заголовком по умолчанию
errorHandler.setRetryListeners(
	new DeliveryAttemptAwareRetryListener());

factory.setCommonErrorHandler(errorHandler);
```

Теперь при каждом неуспехе:
- DeliveryAttemptAwareRetryListener добавляет заголовок в ConsumerRecord
    
- и вы сможете получать число попыток при batch-потреблении

# 3. Listener Info Header
Специальный заголовок `KafkaHeaders.LISTENER_INFO`, который показывает:
- в **каком слушателе**
- и **в каком контейнере**

было обработано сообщение.

Используется для:
- логгирования
- отладки
- фильтров и интерсепторов
- маршрутизации

## Как включить Listener Info Header
#### ✔ В @KafkaListener
```java
@KafkaListener(
    id = "something",
    topics = "topic",
    info = "this is the something listener"
)
public void listen(...,
    @Header(KafkaHeaders.LISTENER_INFO) String listenerInfo) {
}
```

#### ✔ В ContainerProperties
```java
containerProps.setListenerInfo("my-listener-1");
```

---
# 4. Listener Info для batch (2.8.6+)
При batch-потреблении заголовок:
- копируется в **каждый элемент пачки**
- доступен отдельно как String-параметр


Пример:
```java
@KafkaListener(
	id = "list2", 
	topics = "topic", 
	containerFactory = "batchFactory",
    info = "info for batch")
public void listen(
        List<Thing> list,
        @Header(KafkaHeaders.LISTENER_INFO) String info) {
}
```

Если фильтр сделал пачку пустой → требуется:
```java
@Header(name = KafkaHeaders.LISTENER_INFO, required = false)
```
