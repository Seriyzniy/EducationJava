> [!NOTE] ❗❗ ОБЯЗАТЕЛЬНАЯ АННОТАЦИЯ ❗❗
> **@EnableTransactionManagement** над классом конфигурации или над классом запуска приложения
> 
> **@EnableTransactionManagement** — интерфейс аннотации в Spring Framework, который **включает возможность управления транзакциями на основе аннотаций**


> [!NOTE] Чтобы ОТКАТ работал
> Чтобы ОТКАТ транзакции сработал правильно, ВНУТРИ блока кода с транзакцией, НЕ ДОЛЖНО ПЕРЕХВАТЫВАТЬСЯ ИСКЛЮЧЕНИЕ
> 
> То есть, ИСКЛЮЧЕНИЕ из-за которого падает транзакция, ДОЛЖНА ПРОКИДЫВАТЬСЯ НАРУЖУ, а не обрабатываться внутри метода/блока кода с транзакцией (иначе считается что исключение обработано, и можно делать коммит транзакции) 


# (1) KafkaTransactionManager

## 1. Конфигурация Spring Boot: самый простой способ
В **application.yml**:
```yaml
spring:
  kafka:
    producer:
      bootstrap-servers: localhost:9092
      transaction-id-prefix: tx-  # <-- включает транзакционные продюсеры
```

Spring Boot автоматически:
✔ **создаст транзакционный ProducerFactory**  
✔ создаст `KafkaTransactionManager`  
✔ подключит его к контейнеру слушателя (если есть)

Теперь можно autowire менеджер:
```java
@Configuration
public class KafkaTxConfig {

    @Bean
    public KafkaTemplate<String, String> kafkaTemplate(
	    // Spring сам подставит фабрику
        ProducerFactory<String, String> pf
    ) {
		
		// Создание шаблона на основе фабрики
        KafkaTemplate<String, String> template = 
	        new KafkaTemplate<>(pf);
	        
        template.setTransactionalIdPrefix("tx-");
        return template;
    }
}
```

И использовать:
```java
@Service
public class MyService {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public MyService(
	    KafkaTemplate<String, String> kafkaTemplate
	) {
        this.kafkaTemplate = kafkaTemplate;
    }

    @Transactional  // <-- работает с KafkaTransactionManager
    public void sendMessage() {
        kafkaTemplate.send("topic", "hello");
    }
}
```

## 2) Полный пример вручную (без Spring Boot)
### 1. Конфигурация ProducerFactory (включаем транзакции):
```java
@Bean
public ProducerFactory<String, String> producerFactory() {
    Map<String, Object> props = new HashMap<>();
    props.put(
	    ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, 
	    "localhost:9092");
    props.put(
	    ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, 
	    StringSerializer.class);
    props.put(
	    ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, 
	    StringSerializer.class);

    DefaultKafkaProducerFactory<String, String> factory =
            new DefaultKafkaProducerFactory<>(props);

    factory.setTransactionIdPrefix("tx-"); // <-- важный момент!
    return factory;
}
```

### 2. KafkaTemplate
```java
@Bean
public KafkaTemplate<String, String> kafkaTemplate(
        ProducerFactory<String, String> pf) {
    return new KafkaTemplate<>(pf);
}
```

### 3. KafkaTransactionManager
```java
@Bean
public KafkaTransactionManager<String, String> kafkaTransactionManager(
    ProducerFactory<String, String> pf
) {
    return new KafkaTransactionManager<>(pf);
}
```

### 4) Использование @Transactional
```java
@Service
public class TxService {

    private final KafkaTemplate<String, String> kafkaTemplate;

    public TxService(KafkaTemplate<String, String> kafkaTemplate) {
        this.kafkaTemplate = kafkaTemplate;
    }

    @Transactional
    public void process() {
        kafkaTemplate.send("topic1", "value1");
        kafkaTemplate.send("topic2", "value2");
        // если тут исключение — обе записи откатятся
    }
}
```

# 🔥 Важный момент
Вышеуказанный `@Transactional` работает, только если **KafkaTransactionManager — ЕДИНСТВЕННЫЙ менеджер транзакций** в контексте.

Если у вас есть БД:
- Spring выберет **DataSourceTransactionManager** по умолчанию
- Kafka не войдёт в транзакцию
    

**Решение:** использовать квалификатор:
```java
@Transactional("kafkaTransactionManager")
public void kafkaTxMethod() { ... }
```
или сделать набор транзакций «БД → Kafka» / «Kafka → БД».

# (2) Использование транзакций, инициированных потребителем
## 1) Spring Boot конфигурация (application.yml)
```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    consumer:
      group-id: my-group
      auto-offset-reset: earliest
      enable-auto-commit: false
      isolation-level: read_committed   # важно для транзакций
    producer:
      transactional-id-prefix: tx-  # включает транзакции для producer
```

- `transactional-id-prefix` автоматически включает **KafkaTransactionManager** в Spring Boot
    
- `isolation-level: read_committed` → consumer будет видеть только коммитнутые записи


Spring Boot автоматически создаст **KafkaTransactionManager**, *если transactional-id-prefix задан*.

---
## 2. Ручная Java-конфигурация (без Spring Boot auto-config)

```java
@Configuration
@EnableKafka
public class KafkaConfig {

    @Bean
    public ProducerFactory<String, String> producerFactory() {
        Map<String, Object> props = new HashMap<>();
        props.put(
	        ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, 
	        "localhost:9092");
        props.put(
	        ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, 
	        StringSerializer.class);
        props.put(
	        ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, 
		    StringSerializer.class);
        props.put(
	        // 🔥включает транзакции
	        ProducerConfig.TRANSACTIONAL_ID_CONFIG, "tx-"); 
        return new DefaultKafkaProducerFactory<>(props);
    }

    @Bean
    public KafkaTransactionManager<String, String> 
    kafkaTransactionManager(
        ProducerFactory<String, String> pf
    ) {
        return new KafkaTransactionManager<>(pf);
    }

    @Bean
    public KafkaTemplate<String, String> 
    kafkaTemplate(ProducerFactory<String, String> pf) {
        return new KafkaTemplate<>(pf);
    }


    @Bean
    public ConsumerFactory<String, String> consumerFactory() {
        Map<String, Object> props = new HashMap<>();
        props.put(
	        ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, 
	        "localhost:9092");
        props.put(
	        ConsumerConfig.GROUP_ID_CONFIG, 
	        "my-group");
        props.put(
	        ConsumerConfig.ENABLE_AUTO_COMMIT_CONFIG, 
	        false);
        props.put(
	        ConsumerConfig.ISOLATION_LEVEL_CONFIG, 
	        "read_committed");
        props.put(
	        ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG, 
	        StringDeserializer.class);
        props.put(
	        ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG, 
	        StringDeserializer.class);
	        
        return new DefaultKafkaConsumerFactory<>(props);
    }

	// Фабрика для контейнера слушателей
    @Bean
    public 
	    ConcurrentKafkaListenerContainerFactory<String, String> 
	    kafkaListenerContainerFactory(
            ConsumerFactory<String, String> cf,
            KafkaTransactionManager<String, String> ktm
        ) {
        ConcurrentKafkaListenerContainerFactory<String, String> 
        factory =
	        new ConcurrentKafkaListenerContainerFactory<>();
        factory.setConsumerFactory(cf);
        
        // привязываем transaction manager
        factory.setTransactionManager(ktm); 
        factory
	        .getContainerProperties()
	        .setTransactionManager(ktm);
        factory
	        .getContainerProperties()
	        .setAckMode(ContainerProperties.AckMode.RECORD);
        return factory;
    }
}
```

✅ Обратите внимание:
- Контейнер слушателя (`ConcurrentKafkaListenerContainerFactory`) **привязан к KafkaTransactionManager**
    
- Offset commit + producer send будут происходить **атомарно**

## Пример Listener с транзакцией
```java
@Service
public class MyKafkaListener {

    private final KafkaTemplate<String, String> kafkaTemplate;
    private final JdbcTemplate jdbcTemplate;

    public MyKafkaListener(
	    KafkaTemplate<String, String> kafkaTemplate,
        JdbcTemplate jdbcTemplate
    ) {
        this.kafkaTemplate = kafkaTemplate;
        this.jdbcTemplate = jdbcTemplate;
    }

    @KafkaListener(
	    topics = "input-topic", 
	    containerFactory = "kafkaListenerContainerFactory")
    @Transactional  // включает DB transaction внутри listener
    public void listen(String message) {
        // 1️⃣ Обработка данных
        jdbcTemplate.update(
	        "INSERT INTO processed_data (msg) VALUES (?)", 
	        message);

        // 2️⃣ Отправка в другой топик в рамках той же Kafka транзакции
        kafkaTemplate.send(
	        "output-topic", 
	        "Processed: " + message);

        // 🔥offset commit + producer send произойдет автоматически контейнером
    }
}
```
