В **Spring Kafka** `MessageConverter` — это интерфейс, отвечающий за преобразование сообщений между форматами **KafkaRecord ⇄ Spring Message**, а также **(ключ/значение ProducerRecord ⇄ Java-объекты)**.  
Они позволяют автоматически сериализовать и десериализовать данные, превращать их в POJO, строки, JSON и т.п.

Spring Kafka может работать на двух уровнях:
## **1. Низкоуровневый уровень (Kafka Serializer/Deserializer)**
Здесь ты сам указываешь:
- `StringSerializer`
    
- `ByteArraySerializer`
    
- `JsonSerializer` (Spring или Confluent)
- и т.п.

Но это работает **только с payload** и **не работает с Message**, т.е. Spring Messaging не задействован.  
Kafka считает, что работает только с `byte[]`.

---
## **2. Уровень Spring Messaging (MessageConverter)**
Используется, когда ты применяешь:
- `KafkaTemplate.send(Message<?>)`
    
- `@KafkaListener`
    
- Spring Integration Kafka

Здесь появляется **MessageConverter**, который:
 ● Преобразует `Message<?>` → `ProducerRecord`
 ● Преобразует `ConsumerRecord` → `Message<?>`

И именно здесь работают:
- `MessagingMessageConverter`
    
- `JsonMessageConverter`
    
- `JacksonJsonMessageConverter`
    
- `ByteArrayJacksonJsonMessageConverter`
    
- `StringJacksonJsonMessageConverter`
    
- и т.п

---
# Spring Messaging Message Conversion
Хотя API сериализаторов и десериализаторов Kafka довольно простой, при работе на уровне Spring Messaging (например, с `@KafkaListener` или Spring Integration Kafka) иногда требуется больше гибкости.

Чтобы упростить преобразование в/из объектов `org.springframework.messaging.Message`, Spring Kafka предоставляет абстракцию `MessageConverter`, включая реализацию `MessagingMessageConverter` и её специализации `JacksonJsonMessageConverter`.

> [!NOTE] MessagingMessageConverter использует JacksonJsonMessageConverter
> То есть, **любые заголовки и payload, которые не подходят для Kafka напрямую → сериализуются Jackson в JSON**

Можно внедрить MessageConverter:
- напрямую в KafkaTemplate
- через `AbstractKafkaListenerContainerFactory` в @KafkaListener

```java
@Bean
public KafkaListenerContainerFactory<?> kafkaJsonListenerContainerFactory() {
    ConcurrentKafkaListenerContainerFactory<Integer, String> 
    factory =
        new ConcurrentKafkaListenerContainerFactory<>();
    
    factory.setConsumerFactory(consumerFactory());
	    
    factory.setRecordMessageConverter(
	    new JsonMessageConverter());
    return factory;
}
...

@KafkaListener(
	topics = "jsonData",
    containerFactory = "kafkaJsonListenerContainerFactory")
public void jsonListener(Cat cat) {
...
}
```
Если используется Spring Boot, достаточно объявить конвертер как `@Bean`, и auto-configuration подключит его автоматически.

## _@KafkaListener тип параметра помогает MessageConverter_
Когда метод имеет параметр:
```java
@KafkaListener
public void listen(Cat cat)
```

Spring передаёт тип `Cat` в конвертер, чтобы облегчить преобразование.

Это работает **только** при методном уровне аннотации.  
Если аннотация стоит на классе, payload должен быть конвертирован заранее — т.к. тип используется для выбора @KafkaHandler.

## JSON конвертеры на стороне consumer
`JacksonJsonMessageConverter` может обрабатывать:
- `byte[]`
- `Bytes`
- `String`

Поэтому обычно используется совместно с:
- `ByteArrayDeserializer`
    
- `BytesDeserializer`
    
- `StringDeserializer`

Использование byte\[] или Bytes эффективнее — нет лишнего преобразования в строку.

---
## Конвертеры для producer

Если используешь:
- Spring Integration
- или `KafkaTemplate.send(Message<?>)`

тогда нужно согласовать MessageConverter и Kafka Serializer:

|Конвертер|Serializer|
|---|---|
|`StringJacksonJsonMessageConverter`|`StringSerializer`|
|`BytesJacksonJsonMessageConverter`|`BytesSerializer`|
|`ByteArrayJacksonJsonMessageConverter`|`ByteArraySerializer`|

Снова — byte\[] или Bytes эффективнее.

С версии 2.3 есть универсальный `StringOrBytesSerializer`, который умеет в строку, byte[], Bytes.

- можно учитывать Content-Type
    
- автоматом конвертировать JSON в объекты


# Пример
## 1. Базовая конфигурация Consumer
```java
@Configuration
@EnableKafka
public class KafkaConsumerConfig {

    @Bean
    public ConsumerFactory<String, Object> consumerFactory() {
        Map<String, Object> props = new HashMap<>();
        props.put(
	        ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, 
	        "localhost:9092");
        props.put(
	        ConsumerConfig.GROUP_ID_CONFIG, "my-group");
        props.put(
	        ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG, 
	        StringDeserializer.class);
        props.put(
	        ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG, 
	        JsonDeserializer.class);
        props.put(
	        JsonDeserializer.TRUSTED_PACKAGES, 
	        "com.example.dto");
        
        return new DefaultKafkaConsumerFactory<>(props);
    }

    @Bean
    public ConcurrentKafkaListenerContainerFactory<String, Object> kafkaListenerContainerFactory() {
        ConcurrentKafkaListenerContainerFactory<String, Object> 
        factory = 
            new ConcurrentKafkaListenerContainerFactory<>();
        factory.setConsumerFactory(consumerFactory());
        
        // 🔥Установка MessagingMessageConverter🔥
        factory.setMessageConverter(messageConverter());
        
        factory.setConcurrency(3);
        return factory;
    }

    @Bean
    public MessagingMessageConverter messageConverter() {
        MessagingMessageConverter converter = 
	        new MessagingMessageConverter();
        
        // 🔥Настройка JsonKafkaHeaderMapper для работы с заголовками
        JsonKafkaHeaderMapper headerMapper = 
	        new JsonKafkaHeaderMapper();
        headerMapper.addTrustedPackages("com.example.dto");
        
        converter.setHeaderMapper(headerMapper);
        return converter;
    }
}
```

## 2. Базовая конфигурация Producer
```java
@Configuration
public class KafkaProducerConfig {

    @Bean
    public ProducerFactory<String, Object> producerFactory() {
        Map<String, Object> props = new HashMap<>();
        props.put(
	        ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, 
	        "localhost:9092");
        props.put(
	        ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, 
	        StringSerializer.class);
        props.put(
	        ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, 
	        JsonSerializer.class);
        props.put(
	        JsonSerializer.ADD_TYPE_INFO_HEADERS, false);
        
        return new DefaultKafkaProducerFactory<>(props);
    }

    @Bean
    public KafkaTemplate<String, Object> kafkaTemplate() {
        KafkaTemplate<String, Object> template = 
	        new KafkaTemplate<>(producerFactory());
        
        // 🔥Установка MessagingMessageConverter🔥 для отправки сообщений
        template.setMessageConverter(messageConverter());
        
        return template;
    }

    @Bean
    public MessagingMessageConverter messageConverter() {
        MessagingMessageConverter converter = 
	        new MessagingMessageConverter();
        
        JsonKafkaHeaderMapper headerMapper = 
	        new JsonKafkaHeaderMapper();
        headerMapper.addTrustedPackages("com.example.dto");
        
        converter.setHeaderMapper(headerMapper);
        return converter;
    }
}
```

## 3. Расширенная конфигурация с кастомными настройками
```java
@Configuration
@EnableKafka
public class AdvancedKafkaConfig {

    // Consumer Factory с ручной настройкой десериализаторов
    @Bean
    public ConsumerFactory<String, UserDto> userConsumerFactory() {
        Map<String, Object> props = new HashMap<>();
        props.put(
	        ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, 
	        "localhost:9092");
        props.put(
	        ConsumerConfig.GROUP_ID_CONFIG, 
	        "user-group");
        
        // Ручное создание десериализаторов
        StringDeserializer keyDeserializer = 
	        new StringDeserializer();
	        
        JsonDeserializer<UserDto> valueDeserializer = 
	        new JsonDeserializer<>(UserDto.class);
        valueDeserializer.addTrustedPackages("com.example.dto");
        
        return new DefaultKafkaConsumerFactory<>(
            props, keyDeserializer, valueDeserializer
        );
    }

    // Producer Factory с ручной настройкой сериализаторов
    @Bean
    public ProducerFactory<String, UserDto> userProducerFactory() {
        Map<String, Object> props = new HashMap<>();
        props.put(
	        ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, 
	        "localhost:9092");
        
        StringSerializer keySerializer = new StringSerializer();
        
        JsonSerializer<UserDto> valueSerializer = 
	        new JsonSerializer<>();
        valueSerializer.configure(props, false);
        
        return new DefaultKafkaProducerFactory<>(
            props, keySerializer, valueSerializer
        );
    }

    // 🔥Кастомный MessageConverter🔥 с расширенными настройками
    @Bean
    public MessagingMessageConverter customMessageConverter() {
        MessagingMessageConverter converter = 
	        new MessagingMessageConverter();
        
        JsonKafkaHeaderMapper headerMapper = 
	        new JsonKafkaHeaderMapper();
        
        // Настройка паттернов для заголовков
        headerMapper.setEncodeStrings(false); // Не кодировать строки в JSON
        headerMapper.addTrustedPackages("*"); // Доверять всем пакетам
        headerMapper.setRawMappedHeaders(createRawHeadersMap());
        
        converter.setHeaderMapper(headerMapper);
        return converter;
    }

    private Map<String, Boolean> createRawHeadersMap() {
        Map<String, Boolean> rawHeaders = new HashMap<>();
        rawHeaders.put("correlation-id", true);  // Обрабатывать как сырые байты
        rawHeaders.put("trace-id", true);
        return rawHeaders;
    }

    // Listener Container Factory с кастомным конвертером
    @Bean
    public ConcurrentKafkaListenerContainerFactory<String, UserDto> userKafkaListenerContainerFactory() {
        ConcurrentKafkaListenerContainerFactory<String, UserDto> factory = 
            new ConcurrentKafkaListenerContainerFactory<>();
        factory.setConsumerFactory(userConsumerFactory());
        factory.setMessageConverter(customMessageConverter());
        
        // Дополнительные настройки
        factory.getContainerProperties().setAckMode(ContainerProperties.AckMode.MANUAL_IMMEDIATE);
        factory.setConcurrency(3);
        factory.setBatchListener(true); // Для batch обработки
        
        return factory;
    }

    @Bean
    public KafkaTemplate<String, UserDto> userKafkaTemplate() {
        KafkaTemplate<String, UserDto> template = 
            new KafkaTemplate<>(userProducerFactory());
        template.setMessageConverter(customMessageConverter());
        return template;
    }
}
```

# **1. RecordMessageConverter**
Используется **KafkaListenerContainerFactory** для преобразования `ConsumerRecord<?, ?>` → объект аргумента метода `@KafkaListener`.

Ниже показаны реализации
## 1. StringJsonMessageConverter
**Самый распространённый.**

**Для чего:**
- Конвертирует JSON **строки** в POJO и обратно.
- Формат Kafka payload: `String` 
- Тип в Java-коде: `MyObject myObj`
    

**Использование:**  
Payload в Kafka — **строка JSON**.  
Пример: `"{"id":10,"name":"Banana"}"`

Spring автоматически сделает:  
`String → POJO (через Jackson ObjectMapper)`

### Кофнигурация
```java
@Configuration
public class KafkaConfig {

    @Bean
    public ConcurrentKafkaListenerContainerFactory<String, Object> kafkaListenerContainerFactory(
            ConsumerFactory<String, Object> consumerFactory) {

        ConcurrentKafkaListenerContainerFactory<String, Object> 
        factory =
			new ConcurrentKafkaListenerContainerFactory<>();
        factory.setConsumerFactory(consumerFactory);

        factory.setMessageConverter(
	        new StringJsonMessageConverter());

        return factory;
    }
}
```
### Использование (`@KafkaListener`)
```java
public record Order(int id, String item) {}

@KafkaListener(topics = "orders")
public void handle(Order order) {
    System.out.println("Order received: " + order);
}
```

Kafka payload:
```json
{"id":10,"item":"Banana"}
```
Spring → Order POJO.

**То же, что StringJsonMessageConverter, но для `byte[]` JSON.**

Если приложение получает **байтовый JSON от сторонней системы**, лучше использовать его.

### Конфигурация
```java

```

### Listener
```java
@KafkaListener(topics = "orders")

    System.out.println(order);
}
```

## 7.StringMessageConverter
Самый простой.

**Для чего:**
- Конвертация `byte[]` или `Bytes` → `String`.
- Поддерживает только строки, без JSON.

Используется в простых сервисах логгирования, пересылки строк.

### Конфигурация
```java
factory.setMessageConverter(new BytesMessageConverter());
```
### Listener
```java
@KafkaListener(topics = "binary")
public void handle(Bytes data) {
    System.out.println(data);
}
```

## 8.ByteArrayMessageConverter
> Если используешь тип `org.apache.kafka.common.utils.Bytes`.

Конвертирует payload Kafka в `byte[]`.

Если у тебя бинарный протокол (Protobuf, Avro, custom), но ты не хочешь, чтобы Spring что-то парсил — ставишь его.

### Конфигурация
```java
factory.setMessageConverter(new BytesMessageConverter());
```
### Listener
```java
@KafkaListener(topics = "binary")
public void handle(Bytes data) {
    System.out.println(data);
}
```

## 9.BytesMessageConverter
То же самое, но для типа `org.apache.kafka.common.utils.Bytes`.

Используется, когда:

- Kafka producer пишет с key/value типа `Bytes`
    
- Драйвер Kafka Streams генерирует такие сообщения


Используется для `@KafkaListener(batch = true)`.
Обычно комбинируется с JSON-конвертером.

### Конфигурация
```java
@Bean
public ConcurrentKafkaListenerContainerFactory<String, Object> batchFactory(
        ConsumerFactory<String, Object> consumerFactory) {

    var factory = 
	    new ConcurrentKafkaListenerContainerFactory
		<String, Object>();
    factory.setConsumerFactory(consumerFactory);

    factory.setBatchListener(true);

	// 🔥BatchMessageConverter
    factory.setBatchMessageConverter(
		new BatchMessagingMessageConverter(
			new StringJsonMessageConverter())
    );

    return factory;
}
```
### Listener
```java
@KafkaListener(
	topics = "orders", 
	containerFactory = "batchFactory")
public void handle(List<Order> orders) {
    orders.forEach(System.out::println);
}
```

## 10. 🔥 MessagingMessageConverter 🔥 
Базовый общий конвертер:
- Преобразует `ConsumerRecord` в `Message<?>`
    
- Прокидывает headers
    
- Поддерживает многослойную структуру 

Обычно используется **внутри** других конвертеров.

==Это базовый конвертер==.  
Ты используешь его, когда хочешь получать `Message<?>`.

### Конфигурация
```java
factory.setMessageConverter(new MessagingMessageConverter());
```
### Listener
```java
@KafkaListener(topics = "orders")
public void handle(Message<?> message) {
    System.out.println("Headers = " + message.getHeaders());
    System.out.println("Payload = " + message.getPayload());
}
```

## 11. CompositeMessageConverter
Позволяет цеплять несколько конвертеров.

Используется редко, в сложных сценариях:
- разные топики → разные форматы
- поддержка нескольких payload типов

Для случаев, когда нужно применять цепочку конвертеров.

### Конфигурация
```java
@Bean
public RecordMessageConverter compositeConverter() {
    List<MessagingMessageConverter> converters = List.of(
        new StringJsonMessageConverter(),
        new StringMessageConverter()
    );

    return new CompositeMessageConverter((List) converters);
}

@Bean
public ConcurrentKafkaListenerContainerFactory<String, Object> kafkaListenerContainerFactory
(
	ConsumerFactory<String, Object> consumerFactory
) {
    var factory = 
	    new ConcurrentKafkaListenerContainerFactory
	    <String, Object>();
    factory.setConsumerFactory(consumerFactory);
    
    factory.setMessageConverter(compositeConverter());
    return factory;
}
```
### Listener
```java
@KafkaListener(topics = "mixed-topic")
public void handle(Object payload) {
    System.out.println(payload.getClass());
}
```

Если пришёл JSON → POJO  
Если строка → String  
И т.д.

# Обзор конверторов Producer
---
Отдельная категория — это то, что включается в `KafkaTemplate`.

## **KafkaHeaderMapper + MessagingMessageConverter**
Когда ты посылаешь:
```java
kafkaTemplate.send("topic", new MyObject());
```

Spring используют MessageConverter, чтобы:
- сериализовать объект в JSON / String / byte[]
- сформировать Kafka headers

==Те же JSON-конвертеры применяются и для producer==, если KafkaTemplate работает с типом `<String, Object>`.

## Использование JSON-конвертера для отправки POJO
### 1) Конфигурация KafkaTemplate

**Важно:** он работает только если `value-serializer` = `StringSerializer`.

```java
@Bean
public KafkaTemplate<String, Object> kafkaTemplate(ProducerFactory<String, Object> pf) {
    KafkaTemplate<String, Object> template = 
	    new KafkaTemplate<>(pf);
	    
    template.setMessageConverter(
	    new StringJsonMessageConverter());
    return template;
}
```

### 2) Отправка
```java
record User(int id, String name) {}

@Autowired
private KafkaTemplate<String, Object> template;

public void sendUser() {
    template.send("users", new User(1, "Alice"));
}
```

Kafka получит JSON строку.
