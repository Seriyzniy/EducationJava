Spring Kafka может работать на двух уровнях:
## **1. Низкоуровневый уровень (Kafka Serializer/Deserializer)**
Здесь ты сам указываешь:
- `StringSerializer`
    
- `ByteArraySerializer`
    
- `JsonSerializer` (Spring или Confluent)
- и т.п.

Но это работает **только с payload** и **не работает с Message**, т.е. Spring Messaging не задействован.  
Kafka считает, что работает только с `byte[]`.

---
## **2. Уровень Spring Messaging (MessageConverter)**
Используется, когда ты применяешь:
- `KafkaTemplate.send(Message<?>)`
    
- `@KafkaListener`
    
- Spring Integration Kafka

Здесь появляется **MessageConverter**, который:
 ● Преобразует `Message<?>` → `ProducerRecord`
 ● Преобразует `ConsumerRecord` → `Message<?>`

И именно здесь работают:
- `MessagingMessageConverter`
    
- `JsonMessageConverter`
    
- `JacksonJsonMessageConverter`
    
- `ByteArrayJacksonJsonMessageConverter`
    
- `StringJacksonJsonMessageConverter`
    
- и т.п

---
# Spring Messaging Message Conversion
Хотя API сериализаторов и десериализаторов Kafka довольно простой, при работе на уровне Spring Messaging (например, с `@KafkaListener` или Spring Integration Kafka) иногда требуется больше гибкости.

Чтобы упростить преобразование в/из объектов `org.springframework.messaging.Message`, Spring Kafka предоставляет абстракцию `MessageConverter`, включая реализацию `MessagingMessageConverter` и её специализации `JacksonJsonMessageConverter`.

Можно внедрить MessageConverter:
- напрямую в KafkaTemplate
- через `AbstractKafkaListenerContainerFactory` в @KafkaListener

```java
@Bean
public KafkaListenerContainerFactory<?> kafkaJsonListenerContainerFactory() {
    ConcurrentKafkaListenerContainerFactory<Integer, String> 
    factory =
        new ConcurrentKafkaListenerContainerFactory<>();
    
    factory.setConsumerFactory(consumerFactory());
	    
    factory.setRecordMessageConverter(
	    new JsonMessageConverter());
    return factory;
}
...

@KafkaListener(
	topics = "jsonData",
    containerFactory = "kafkaJsonListenerContainerFactory")
public void jsonListener(Cat cat) {
...
}
```
Если используется Spring Boot, достаточно объявить конвертер как `@Bean`, и auto-configuration подключит его автоматически.

## _@KafkaListener тип параметра помогает MessageConverter_
Когда метод имеет параметр:
```java
@KafkaListener
public void listen(Cat cat)
```

Spring передаёт тип `Cat` в конвертер, чтобы облегчить преобразование.

Это работает **только** при методном уровне аннотации.  
Если аннотация стоит на классе, payload должен быть конвертирован заранее — т.к. тип используется для выбора @KafkaHandler.

## JSON конвертеры на стороне consumer
`JacksonJsonMessageConverter` может обрабатывать:
- `byte[]`
- `Bytes`
- `String`

Поэтому обычно используется совместно с:
- `ByteArrayDeserializer`
    
- `BytesDeserializer`
    
- `StringDeserializer`

Использование byte\[] или Bytes эффективнее — нет лишнего преобразования в строку.

---
## Конвертеры для producer

Если используешь:
- Spring Integration
- или `KafkaTemplate.send(Message<?>)`

тогда нужно согласовать MessageConverter и Kafka Serializer:

|Конвертер|Serializer|
|---|---|
|`StringJacksonJsonMessageConverter`|`StringSerializer`|
|`BytesJacksonJsonMessageConverter`|`BytesSerializer`|
|`ByteArrayJacksonJsonMessageConverter`|`ByteArraySerializer`|

Снова — byte\[] или Bytes эффективнее.

С версии 2.3 есть универсальный `StringOrBytesSerializer`, который умеет в строку, byte[], Bytes.

## SmartMessageConverter (с версии 2.7.1)
Теперь можно делегировать конвертацию Spring Messaging конвертеру:
- можно учитывать Content-Type
    
- автоматом конвертировать JSON в объекты


Примеры:
```java
template.setMessagingConverter(mySmartConverter);
```

```java
@KafkaListener(
	id = "withSmartConverter", 
	topics = "someTopic",
    contentTypeConverter = "mySmartConverter")
public void smart(Thing thing) {
}

```
