Когда Producer отправляет сообщение, при этом возможны три группы ошибок:
1. **Ошибки сериализации**
    
2. **Ошибки отправки на брокер (сетевые, партиционирование, брокер недоступен)**
    
3. **Ошибки подтверждения (acks) — Timeout, Retries, `NotEnoughReplicasException` и т.д.**

## 1. Места появления ошибок
Ошибки возможны на двух этапах:
### При формировании записи
Это происходит ДО отправки, при создании `ProducerRecord` и сериализации:
- NPE(NullPointerException) в ключе/значении
- ошибка сериализатора
- ошибка маппера заголовков (`KafkaHeaderMapper`)
- ошибка MessageConverter (если используется Message)

### При отправке сообщения в Kafka
Это происходит ПОСЛЕ вызова `Producer.send()`:
- недоступен брокер
- партиция недоступна
    
- нехватка реплик (`acks=all`)
- истек таймаут `delivery.timeout.ms`
    
- ошибка идемпотентности
    
- ошибка транзакции (если используется транзакционный продюсер)

## 2. Способы обработки ошибок
Spring Kafka + KafkaTemplate предоставляет три способа:
- Способ 1 — Callback (`ListenableFuture.addCallback`)
- Способ 2 — Error Handling Serializer
- Способ 3 — KafkaTemplate.setProducerListener(...)

### **Способ 1 — Callback (ListenableFuture.addCallback)**
Самый базовый, используется всегда.

Задача: Перехватить успешную отправку или ошибку на уровне каждой операции `send()`.

#### Используемые классы
- `ListenableFuture<SendResult<K,V>>`
    
- `KafkaSendCallback`
    
- `ListenableFutureCallback`
    
- (иногда) `SendResult`

#### Порядок работы
- Отправляем сообщение → получаем `ListenableFuture`.
- Вызываем `.addCallback()`.
- Реализуем логику успеха / ошибки.

#### Пример
```java
ListenableFuture<SendResult<String, UserInfo>> future =
        kafkaTemplate.send("topic-spring", userInfo);

future.addCallback(new ListenableFutureCallback<>() {
    @Override
    public void onSuccess(SendResult<String, UserInfo> result) {
        System.out.println("SUCCESS: " + result.getRecordMetadata());
    }

    @Override
    public void onFailure(Throwable ex) {
        System.err.println("ERROR while sending: " + ex.getMessage());
    }
});
```

### **Способ 2 — Error Handling Serializer**          (в DefaultProducerFactory)
Spring позволяет **«обернуть» сериализаторы** так, чтобы сериализационные ошибки не ломали Producer.

**Если ошибка произошла в сериализаторе** — она **не попадает в Callback**, потому что Callback вызывается ТОЛЬКО после отправки.

По умолчанию сериализация → исключение прямо в `send()`.

Spring даёт механизм «ошибочного сериализатора»
#### Используемые классы
- `ErrorHandlingSerializer`
    
- `ErrorHandlingDeserializer`
    
- `DefaultKafkaProducerFactory.setValueSerializer(...)`

#### Пример использования
```java
props.put(ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG,
          ErrorHandlingSerializer.class);
          
props.put(ErrorHandlingSerializer.VALUE_SERIALIZER_CLASS,
          CustomUserInfoSerializer.class);
```

Теперь сериализационная ошибка попадает в Callback:
```java
future.addCallback(...); // теперь onFailure вызовется
```

### **Способ 3 — KafkaTemplate.setProducerListener(...)**
**Глобальный слушатель** успешных отправок и ошибок.

Если нужно логировать или обрабатывать все отправки глобально — используется `ProducerListener`.

Задача: Получить уведомление об успешной/неудачной отправке ГЛОБАЛЬНО, для всех send().

#### Основные классы
- `ProducerListener<K, V>`
    
- `KafkaTemplate.setProducerListener(...)`
    
- Реализация: `LoggingProducerListener` (готовая)

#### Порядок работы
1. Создаёшь свой класс, реализующий `ProducerListener`.
    
2. Передаёшь в `kafkaTemplate.setProducerListener(...)`.

#### Пример
```java
@Bean
public KafkaTemplate<String, UserInfo> kafkaTemplate(
        ProducerFactory<String, UserInfo> pf) {

    KafkaTemplate<String, UserInfo> template = new KafkaTemplate<>(pf);


    template.setProducerListener(new ProducerListener<>() {
        @Override
        public void onSuccess(
	        ProducerRecord<String, UserInfo> record,
			RecordMetadata metadata) 
		{
            System.out.println("GLOBAL SUCCESS: " + metadata);
        }

        @Override
        public void onError(
	        ProducerRecord<String, UserInfo> record,
			RecordMetadata metadata,
            Exception exception) 
        {
            System.err.println(
	            "GLOBAL ERROR: " + exception.getMessage());
        }
    });

    return template;
}
```

## Ошибки при отсутствии партиции / ретраях / таймаутах
Эти ошибки — стандартные Kafka Producer ошибки:
- `TimeoutException`
    
- `RetriableException`
    
- `NetworkException`
    
- `NotEnoughReplicasException`
    
- `NotLeaderForPartitionException`
    
- `RecordTooLargeException`
    
- `SerializationException`

> Все они ловятся в `.onFailure()`.

Если делаешь синхронный `send().get()`:
```java
try {
    kafkaTemplate.send("topic", value).get(10, TimeUnit.SECONDS);
} catch (ExecutionException e) {
    Throwable cause = e.getCause(); // Тут реальная ошибка Kafka
}
```
