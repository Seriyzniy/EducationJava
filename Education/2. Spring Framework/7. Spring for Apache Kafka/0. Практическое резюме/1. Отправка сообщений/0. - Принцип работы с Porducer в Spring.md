# 📕1. Конфигурация Producer
## **1️⃣ Настройка свойств продюсера (Producer Properties)**
Для создания Kafka-продюсера нужны основные свойства:
- `bootstrap.servers` — адрес брокера Kafka
- `key.serializer` и `value.serializer` — сериализация ключа и значения
	>В параметры передаются классы, которые реализуют `Serializer` из Kafka API
	
- `acks` — подтверждение брокером
- `retries` — количество повторов при сбое

(см [[1. - АРХИТЕКТУРА Kafka API, KafkaStreams]])

**Пример кода:**
```java
@Bean
public Map<String, Object> producerConfigs() {
    Map<String, Object> props = new HashMap<>();
    props.put(
	    ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, 
	    "localhost:9092");
    props.put(
	    ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, 
	    IntegerSerializer.class);
    props.put(
	    ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, 
	    StringSerializer.class);
    props.put(ProducerConfig.ACKS_CONFIG, "all"); // ждём подтверждения от всех реплик
    props.put(ProducerConfig.RETRIES_CONFIG, 3);
    return props;
}
```


## **2️⃣ Создание ProducerFactory**
`ProducerFactory` отвечает за создание экземпляров `KafkaProducer` с указанными свойствами.
```java
@Bean
public ProducerFactory<Integer, String> producerFactory() {
    return new DefaultKafkaProducerFactory<>(producerConfigs());
}
```

## **3️⃣ Создание KafkaTemplate**
`KafkaTemplate` — ==основной класс Spring Kafka для отправки== сообщений. Он оборачивает продюсер и упрощает API.

```java
@Bean
public KafkaTemplate<Integer, String> kafkaTemplate() {
    return new KafkaTemplate<>(producerFactory());
}
```
- Можно использовать для синхронной и асинхронной отправки сообщений
    
- Позволяет задавать key, value и topic

# 📕2. Процесс БАЗОВОЙ отправки сообщения
## 1. Типы сообщений
После конфигурации Producer, мы можем отправлять сообщение в виде:
- ProducerRecord используя Kafka API
- Message из Spring Messaging
- **кортеж из параметров** (topic, key, data)

### ProducerRecord
ProducerRecord — это **родной** объект Kafka из пакета `org.apache.kafka.clients.producer`.

Используем, когда нужно:
- указывать ключ
- указывать партицию
- добавлять заголовки
- управлять timestamp сообщения
- полностью контролировать формат сообщения

#### Примеры создания ProducerRecord
##### Простая запись: только топик + value
```java
ProducerRecord<String, UserInfo> record =
        new ProducerRecord<>("users-topic", userInfo);
```

##### С указанием ключа
```java
ProducerRecord<String, UserInfo> record =
        new ProducerRecord<>("users-topic", "user123", userInfo);
```

##### С указанием партиции
```java
ProducerRecord<String, UserInfo> record =
        new ProducerRecord<>("users-topic", 1, "user123", userInfo);
```

##### С добавлением заголовков
```java
ProducerRecord<String, UserInfo> record =
        new ProducerRecord<>("users-topic", "user123", userInfo);

record.headers().add("source", "auth-service".getBytes());
record.headers().add(new RecordHeader("trace-id", "abc123".getBytes()));
```

##### С указанием собственного timestamp
```java
ProducerRecord<String, UserInfo> record =
        new ProducerRecord<>(
	        "users-topic", 
	        null, 
	        System.currentTimeMillis(),
	        "user123", 
	        userInfo);
```

### Message
Однако, прямиком Message отправить в Kafka мы не можем. Нам нужно конвертировать Message в ProdcuerRecord. 

Spring Message — это удобный способ обогащать сообщения заголовками без ручного управления байтами.

Используется вместе с **MessagingMessageConverter**.

#### MessagingMessageConverter
Spring Kafka преобразует `Message<?>` → ProducerRecord через `MessagingMessageConverter`.

KafkaTemplate вызывает этот конвертер, если вы вызываете:
```java
kafkaTemplate.send(Message<?> message)
```

##### MessagingMessageConverter по умолчанию
Если вы **не создали кастомный конвертер**, KafkaTemplate использует:
```java
new MessagingMessageConverter()
```
Через конвертер задается:
- маппинг заголовков
- сериализация payload через ProducerFactory → Serializer


Этот конвертер содержит методы
 → `toMessage()`
Преобразует ConsumerRecord → Message (для Listener)

→ `fromMessage()`
Преобразует Message → ProducerRecord  
(вызывается KafkaTemplate перед отправкой)


Какие **мапперы заголовков** есть по умолчанию?
Когда Spring Kafka видит **Jackson** на classpath:
- Заголовки сериализуются/десериализуются **через JsonSerializer**
- И используется **JsonHeaderMapper**

Без Jackson:
- заголовки конвертируются как строки → bytes
- используется **DefaultKafkaHeaderMapper**

##### Кастомный MessagingMessageConverter
Задается в KafkaTemplate при помощи 
```java
template.setMessageConverter(converter);
```

Пример, конвертер с кастомным преобразованием заголовков
```java
@Bean
public KafkaTemplate<String, UserInfo> kafkaTemplate(
        ProducerFactory<String, UserInfo> producerFactory) {

    KafkaTemplate<String, UserInfo> template =
            new KafkaTemplate<>(producerFactory);

    MessagingMessageConverter converter =
            new MessagingMessageConverter();

    converter.setHeaderMapper(new KafkaHeaderMapper() {
        @Override
        public void fromHeaders(
	        MessageHeaders headers, 
	        Headers target) 
	    {
            headers.forEach((key, value) -> {
                target.add(key, value.toString().getBytes());
            });
        }

        @Override
        public void toHeaders(
	        Headers source, 
	        Map<String, Object> target) { }
    });

    template.setMessageConverter(converter);

    return template;
}

```

#### Message с указанием топика (обязательно!) и заголовками
```java
Message<UserInfo> msg = MessageBuilder
        .withPayload(userInfo)
        .setHeader(KafkaHeaders.TOPIC, "users-topic")
        .setHeader("source", "auth-service")
        .setHeader("trace-id", UUID.randomUUID().toString())
        .build();
```

## 2. Сериализаторы сообщений
Конечно, **чтобы отправить сообщение**, необходимо **написать класс для сериализации ТЕЛА** сообщения. Этот класс затем передается в как параметр конфигурацию Producer.

Чтобы Kafka использовал этот сериализатор, необходимо, чтобы реализация реалзовывала класс `Serializer` из Kafka API.

```java
public interface Serializer<T>
```
(Например для кастомного сериализатора можно использовать Jackson ObjectMapper)
```java
public class CustomUserInfoSerializer implements Serializer<UserInfo> {
    private final ObjectMapper mapper = new ObjectMapper();

    @Override
    public byte[] serialize(String topic, UserInfo userInfo) {
        try {
            return mapper.writeValueAsBytes(userInfo);
        } catch (Exception e) {
            throw new SerializationException(
	            "Error serializing UserInfo", e);
        }
    }
}

```

При этом, есть и готовые реализации

### Готовые сериализаторы Kafka
|Сериализатор|Описание|
|---|---|
|**StringSerializer**|сериализация строк в UTF-8|
|**ByteArraySerializer**|отправка сырого массива байтов|
|**IntegerSerializer**|сериализация int|
|**LongSerializer**|сериализация long|
|**DoubleSerializer**|сериализация double|
|**BytesSerializer**|сериализация Bytes-класса Kafka|

### Готовые сериализаторы Spring Kafka
| Сериализатор                | Описание                                                   |
| --------------------------- | ---------------------------------------------------------- |
| **JsonSerializer**          | сериализует объект в JSON (Jackson)                        |
| **JsonSerde**               | для Kafka Streams                                          |
| **DelegatingSerializer**    | выбирает сериализатор по заголовку type                    |
| **ErrorHandlingSerializer** | оборачивает кастомный сериализатор ==с обработкой ошибок== |

## 3. Заголовки сообщений
Как известно, к сообщению можно добавить заголовок.
Kafka хранит заголовки в виде массива байт

### ProducerRecord
Для ProducerRecord есть конструктор, который принимает заголовки в виде коллекции `Iterable<Header>`.

Чтобы создать эту коллекцию можно использовать любую коллекцию которая реалзиует Iterable - например List.

Для добавления заголовков можно
- явно передавать ключ-значение в байтах (сериализатор для заголовков при создании заголовка)

- через RecordHeader, в конструктор которого передаем так же ключ и значение в виде массива байт

(см [[2. - Проект с Kafka + Spring]])

### Message (Spring)
Для добавления заголовков в Message от Spring Messaging можно использовать `MessageBuilder`, у которого есть метод setHeader(), в котором также передаются ключ-значение
```java
Message<UserInfo> msg = MessageBuilder
        .withPayload(userInfo)
        .setHeader("trace-id", "999999")
        .setHeader("source", "backend")
        .setHeader(KafkaHeaders.TOPIC, "users-topic")
        .build();
```

#### Особенность работы с заголовками используя Message
Поскольку Kafka может работать только с потокам байт, то нужно как-то конвертировать Message в байты.

Для этого в Spring есть интерфейс KafkaHeaderMapper
```java
public interface KafkaHeaderMapper {

    void fromHeaders(MessageHeaders headers, Headers target);

    void toHeaders(Headers source, Map<String, Object> target);

}
```

В этих методах мы указываем как конвертировать заголовки при отправке сообщений и при приеме.

Чтобы установить такой кастомный маппер заголовков нужно его передать в маппер сообщений
```java
@Bean  
public KafkaTemplate<String, UserInfo> kafkaTemplate(  
		ProducerFactory<String, UserInfo> producerFactory){  

	KafkaTemplate<String, UserInfo> template = 
		new KafkaTemplate<>(producerFactory);  

	MessagingMessageConverter converter = 
		new MessagingMessageConverter(); 
		
	// Описание класса, 🔥для конвертации заголовков🔥 
	converter.setHeaderMapper(new KafkaHeaderMapper() {  
		@Override  
		public void fromHeaders(
			MessageHeaders headers, 
			Headers target) 
		{  
			for(Map.Entry<String, Object> entry:
			headers.entrySet()
			){ 
				target.add(
					entry.getKey(), 
					entry.getValue()
						.toString()
						.getBytes());  
			}  
		}  

		@Override  
		public void toHeaders(
			Headers source, 
			Map<String, Object> target) { }  
	});  

	template.setMessageConverter(converter);  

	return template;
```

## 4. Отправка данных
Для отправки данных используется метод 
```java
ListenableFuture<SendResult<K, V>> send(...)
```
, который асинхронно отправляет данные.

Как уже известно, сообщением может быть Message, ProducerRecord. Оно успешно отправится если настроен сериализатор тела сообщения.

### Добавление ключа и партиции при отправке сообщения
В базовом варианте, можно указать партицию и ключ сообщения

#### Указание ключа
```java
kafkaTemplate.send("users-topic", "user123", userInfo);
```

#### Указание партиции
```java
kafkaTemplate.send("users-topic", 1, "user123", userInfo);
```

### Доабвления Callback 
Также в метод send можно добавить Callback, который выполнит код, когда от Kafka будет получени ответ - ACK. 

#### Spring Kafka
```java
kafkaTemplate.send("users-topic", userInfo)
        .addCallback(
            result -> System.out.println("Sent OK: " + result),
            ex -> System.out.println("Error: " + ex.getMessage())
        );
```

#### Kafka
```java
ProducerRecord<String, UserInfo> record =
        new ProducerRecord<>("users-topic", userInfo);

kafkaTemplate.send(record, (metadata, exception) -> {
    if (exception == null) {
        System.out.println("Sent to " + metadata.topic());
    }
});
```
### Синхронная отправка
Так как метод send возвращает ListenableFuture, то можно вызывать на нем метод get() тем самым сделав отправку синхронной
```java
SendResult<String, UserInfo> result =
        kafkaTemplate.send("users-topic", userInfo).get();

```
