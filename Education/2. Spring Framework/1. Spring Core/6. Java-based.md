# @Configuration

> [!NOTE] @Configuration
> Помечает класс, который будет использоваться как класс-конфигурации

```java
package ru.studyspring.myproject;

import org.springframework.context.annotation.Configuration;

@Configuration
public class SpringConfig {
}
```

Пустой класс с аннотацией `@Configuration` ==равен по функциональности== пустому XML файлу
```xml
<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
	xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
	xmlns:context="http://www.springframework.org/schema/context"
	xsi:schemaLocation="http://www.springframework.org/schema/beans
	http://www.springframework.org/schema/beans/spring-beans.xsd
	http://www.springframework.org/schema/context
	http://www.springframework.org/schema/context/spring-context.xsd">
</beans>
```

## Аннотации для тегов
Для каждого тега в xml файле есть аннотация

Для этого:
```xml
<context:component-scan base-package="ru.studyspring.myproject"/>
```
Существует это:
```java
@Configuration
@ComponentScan("ru.studyspring.myproject") //Аналог тега <context:component-scan>
public class SpringConfig {
}
```

==! ! !== Также для тега `<bean>` существует аннотация
```xml
<bean id="musicPlayer"
	  class="ru.studyspring.myproject.MusicPlayer"/>
</bean>
```
Заменяется на:
```java
@Configuration
@ComponentScan("ru.studyspring.myproject")
public class SpringConfig {
	@Bean                              // Аннотация @Bean
	public MusicPlayer musicPlayer() {
		return new MusicPlayer();
	}
}
```

Теперь ==ДЛЯ ИСПОЛЬЗОВАНИЯ== конфигурационного ==КЛАССА== необходимо создать объект класса `AnnotationConfigApplicationContext` и передать его конструктору наш конфиг класс
```java
public class ApplicationRunner {
	public static void main(String[] args) {
		AnnotationConfigApplicationContext context = new  AnnotationConfigApplicationContext(SpringConfig.class);
	
	MusicPlayer music = context.getBean("musicPlayer", MusicPlayer.class);
	music.PlaingMusic();
	}
}
```
# Ручное внедрение зависимостей (без @Autowired)

![[Pasted image 20241021161210.png]]
Рассмотрим на примере Java-кода
Уберем в классе MusicPlayer аннотацию @Autowired
```java
@Component
public class MusicPlayer {
	private GenreMusic music;
	
	public MusicPlayer(ClassicMusic music) {
		this.music = music;
	}
	public void PlaingMusic() { System.out.println("Playing: " + music.plaing()); }
}
```

Теперь конфигурационный класс выглядит так:
```java
@Configuration
@ComponentScan("ru.studyspring.myproject")
public class SpringConfig {
	@Bean
	public ClassicMusic classicMusic() {
		return new ClassicMusic();
	}
	@Bean
	public MusicPlayer musicPlayer() {
		return new MusicPlayer(classicMusic());
	}
}
```

# @Bean
По умолчанию у бинов  `scope = SINGLETON`

* Это означает что тело `@Bean` методов по умолчанию вызывается только один раз, а все последующие вызовы Spring прерывает и возвращает уже имеющийся бин из контекста
