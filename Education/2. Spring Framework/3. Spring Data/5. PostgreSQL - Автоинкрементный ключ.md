Ранее в нашем приложении мы проставляли `id` вручную и он всегда был один для всех полей. Так делать нельзя, `id` должен быть уникальным для каждой записи

***
# 1. Последовательность (Sequence) в PostgreSQL

Для того чтобы создать автоинкрементное поле в PostgreSQL, необходимо создать специальный объект `Sequence`

> [!NOTE] Объект `Sequence`
> Данный объект служит для создания уникальных чисел. Своего рода СЧЕТЧИК

Создается последовательность следующей командой:
```sql
create sequence first_sequence
```

# 3. Стандартный способ создания автоинкремента 

Начиная с 10 версии PostgreSQL, автоинкрементное поле создается по другому

> [!NOTE] Новый синтаксис `GENERATED BY DEFAULT AS IDENTITY`
> Используется для совместимости с другими БД, и для соответствия стандартам SQL
> ```sql
> CREATE TABLE Persons(
> 	id int GENERATED BY DEFAULT AS IDENTITY,
> 	p_name varchar
> 	age int,
> 	email varchar
> )
> ```

Данный тип данных делает то же самое что и `SERIAL`, отличается синтаксис
