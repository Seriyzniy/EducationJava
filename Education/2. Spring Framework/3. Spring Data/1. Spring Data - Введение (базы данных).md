# 1. Объединение Java-приложения и БД

Существует ==3 способа связать Java-приложение и БД==
![[Pasted image 20241123174504.png]]
#### 1. JDBC API
- Самый низкоуровневый, здесь необходимо ВРУЧНУЮ делать все запросы к БД и ВРУЧНУЮ переводить Java-объекты в строки таблицы и наоборот

#### 2. JdbcTemplate
- Тонкая обертка вокруг JDBC API. ==Часть Spring Framework==. 
- Предоставляет некоторые абстракции, берет часть дел на себя

#### 3. Hibernate
- Самый высокий уровень абстракции. Практически не пишем вручную запросы к БД. 
- Автоматически переводит Java-объекты в строки таблицы и наоборот.
- Может автоматически создавать таблицы в БД на основании наших Java-классов. Этот функционал называется ORM (Object-Relational Mapping)

# 2. Подключение к БД
Для подключения к ==БД==, необходимо ее сначала ==запустить== (локально или удаленно)

- У работающей БД есть свой адрес (как и у сервера	`localhost:8080/hello-world`)
- Адрес БД:
	- url=`jdbc:postgresql://localhost:5432/db_name`
		Порт у PostgreSQL обычно 5432
	- username=`postgres`
	- password=
	
- Используется JDBC драйвер, чтобы подключиться к БД с указанным адресом
