> [!NOTE] **Apache Kafka**
> **Apache Kafka** — это **распределённая платформа потоковой передачи сообщений (distributed streaming platform)**
> 
> Предназначенная для:
> - публикации, 
> - хранения, 
> - обработки 
> - подписки на потоки событий (event streams) **в реальном времени**, с гарантией надёжной доставки и высокой производительностью.

## Если более кратко то:
==Kafka== — это **система обмена сообщениями** (message broker) нового поколения, которая сочетает в себе:
- 💡функции **очереди сообщений** (как RabbitMQ);
- 💡 и функции **журнала событий / commit log** (как база данных событий).

Допустим мы хотим передать некоторые данные 3-х категорий. 
У нас есть производители этих данных и потребители. 

Однако напрямую передавать данные от producer к consumer достаточно сложно (проблема регистрации новых потребителей, гарантия доставки и прочее)

Использование Kafka упрощает эту задачу. Каждое ==сообщение Kafka кладет в определенный "ящик"==, а данные из этих "ящиков" читают потребители. 

## Основные свойства Kafka:
| Свойство                 | Описание                                                                                                                                      |
| ------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------- |
| **Распределённая**       | Работает в кластере из нескольких брокеров; данные реплицируются для отказоустойчивости.<br>(==может использоваться на нескольких серверах==) |
| **Устойчивая (durable)** | Сообщения записываются на диск и могут храниться заданное время (retention).                                                                  |
| **Масштабируемая**       | Потоки делятся на _partition’ы_ — можно обрабатывать миллионы сообщений в секунду.                                                            |
| **Pub/Sub модель**       | Поддерживает publish–subscribe и потоковую обработку.                                                                                         |
| **Реальное время**       | Данные читаются и обрабатываются сразу после публикации.                                                                                      |
|                          |                                                                                                                                               |

# 📕1. Основные компоненты
| Компонент                                   | Описание                                                                                                     |
| ------------------------------------------- | ------------------------------------------------------------------------------------------------------------ |
| **Producer** (производитель)                | Отправляет сообщения (events) в Kafka — в определённый _topic_.                                              |
| **Consumer** (потребитель)                  | Приложение, которое читает сообщения из топиков.                                                             |
| **Consumer Group** (группа потребителей)    | Набор потребителей, совместно читающих данные из топика, распределяя партиции между собой.                   |
| -------------------                         |                                                                                                              |
| **Broker** (брокер)                         | Отдельный сервер Kafka, который хранит сообщения и обрабатывает запросы от producers и consumers.            |
| **ZooKeeper / KRaft** (контроллер кластера) | Компонент для управления метаданными кластера (в новых версиях заменён на KRaft).                            |
| -------------------                         |                                                                                                              |
| **Topic** (топик)                           | Логическая категория или поток сообщений. Например: `orders`, `payments`, `logs`.                            |
| **Partition** (раздел)                      | Подраздел внутри топика, куда физически записываются сообщения — обеспечивает масштабирование и параллелизм. |
| **Offset**                                  | Номер позиции сообщения внутри партиции — уникальный и монотонно возрастающий.                               |
| -------------------                         |                                                                                                              |
| **Kafka Connect**                           | Компонент для интеграции Kafka с внешними системами (БД, API, файлы, Elasticsearch и т.п.).                  |
| **Kafka Streams / ksqlDB**                  | Модуль для обработки потоков данных прямо внутри Kafka.                                                      |
