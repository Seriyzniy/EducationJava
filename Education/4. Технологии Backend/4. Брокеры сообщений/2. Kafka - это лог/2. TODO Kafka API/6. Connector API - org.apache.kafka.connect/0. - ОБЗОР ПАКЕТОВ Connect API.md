
## Иерархия по назначению

```bash
org.apache.kafka.connect
 ├── connector    # Базовые интерфейсы Connector и Task
 │    └── policy  # Политики безопасности и валидации конфигов
 ├── source       # API для Source коннекторов (внешний источник → Kafka)
 ├── sink         # API для Sink коннекторов (Kafka → внешний приёмник)
 ├── data         # Модель данных (Schema, Struct)
 ├── transforms   # Single Message Transformations (SMT)
 ├── header       # Работа с заголовками записей
 ├── errors       # Исключения и обработка ошибок
 ├── components   # Метаданные компонентов (версионирование, capabilities)
 ├── health       # Метрики и статус коннекторов
 ├── storage      # Хранение offset'ов и конфигурации
 ├── rest         # REST API Connect Worker
 └── util         # Утилиты и служебные классы
```

## 🟢 Основные пакеты (используются в реальной разработке)

| Пакет                                 | Назначение                                                                                                                                                |
| ------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `org.apache.kafka.connect.source`     | API для реализации **Source Connector’ов** — получение данных из внешних систем (БД, файлов, REST, и т.д.) и запись их в Kafka.                           |
| `org.apache.kafka.connect.sink`       | API для реализации **Sink Connector’ов** — отправка данных из Kafka во внешние системы (БД, хранилища, API и т.п.).                                       |
| `org.apache.kafka.connect.data`       | Определяет **модель данных** Connect: `Schema`, `Struct`, `Field`, `SchemaBuilder` — используется во всех коннекторах.                                    |
| `org.apache.kafka.connect.transforms` | API для написания **Single Message Transformations (SMT)** — промежуточных преобразований сообщений без изменения коннектора.                             |
| `org.apache.kafka.connect.errors`     | Базовые исключения Connect API (`ConnectException`, `DataException`, `RetriableException`) — используются при обработке ошибок в собственных коннекторах. |
| `org.apache.kafka.connect.header`     | API для работы с **заголовками записей** (headers), например при трансформации сообщений.                                                                 |
| `org.apache.kafka.connect.connector`  | Базовые интерфейсы для коннекторов: `Connector`, `Task`, `ConnectorContext`, `ConnectorConfig`. Основа любого Source/Sink коннектора.                     |

## 🟡 Служебные / конфигурационные пакеты (используются редко вручную)

|Пакет|Назначение|
|---|---|
|`org.apache.kafka.connect.components` ⚙️|Определяет **компоненты Connect** (например, `Versioned`, `Verifiable`), помогает при introspection и версионировании коннекторов.|
|`org.apache.kafka.connect.connector.policy` ⚙️|Политики безопасности конфигурации (например, `AllConnectorClientConfigOverridePolicy`), применяются в кластерном режиме.|
|`org.apache.kafka.connect.health` ⚙️|API для **мониторинга и health check** коннекторов. Используется внутри Connect runtime и REST API.|
|`org.apache.kafka.connect.storage` ⚙️|Управление **offset’ами, конфигурацией и статусами задач**. Используется рантаймом Connect для хранения состояния в топиках `_connect-configs`, `_connect-status`, `_connect-offsets`.|
|`org.apache.kafka.connect.util` ⚙️|Набор утилит (вспомогательные классы для потоков, сериализации, валидации). В продакшене **не используется напрямую**.|

## 🔴 Вспомогательные/системные пакеты (в основном “под капотом”)

|Пакет|Назначение|
|---|---|
|`org.apache.kafka.connect.rest` ⚙️|REST API сервиса **Connect Worker** — эндпоинты для управления коннекторами, задачами и статусами. Не используется при разработке коннекторов.|
|`org.apache.kafka.connect.health` ⚙️|(см. выше) Метрики и статусные объекты `ConnectClusterState`, `ConnectorState`, `TaskState`.|
|`org.apache.kafka.connect.storage` ⚙️|Слой хранения состояния (offsetы, конфиги, статусы). Пример: `OffsetStorageReader`, `Converter`, `ConverterConfig`.|
|`org.apache.kafka.connect.util` ⚙️|Служебные классы (`ConnectorUtils`, `ReflectionUtils`, `TopicAdmin`) — вспомогательные механизмы Connect runtime.|
