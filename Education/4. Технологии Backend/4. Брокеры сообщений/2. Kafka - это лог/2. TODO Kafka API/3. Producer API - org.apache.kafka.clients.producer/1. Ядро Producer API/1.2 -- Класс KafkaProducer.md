
> [!NOTE] `KafkaProducer` 
> это **основная реализация интерфейса `Producer`**, используемая для **отправки сообщений (records)** в брокеры Kafka.

Он отвечает за:
- сериализацию ключей и значений,
- выбор партиции,
- буферизацию записей перед отправкой,
    
- асинхронную доставку с подтверждениями (ACK),
    
- транзакции и Exactly-Once Delivery (EOS).


Архитектурно внутри `KafkaProducer` работает несколько ключевых компонентов:
- **`RecordAccumulator`** — буферизация сообщений по партициям;
    
- **`Sender`** — отдельный поток, который отправляет батчи сообщений на брокеры;
    
- **`Metadata`** — информация о брокерах, топиках и партициях;
    
- **`Serializer`** — для преобразования `K` и `V` в байты.

---
## Конструктор
```java
// Создаёт экземпляр продюсера с указанными настройками
public KafkaProducer(Map<String, Object> configs);

// Создаёт продюсер с настройками и пользовательскими сериализаторами ключа и значения
public KafkaProducer(
	Map<String, Object> configs, 
	Serializer<K> keySerializer, 
	Serializer<V> valueSerializer);

// Создаёт продюсер с настройками в виде Properties
public KafkaProducer(Properties properties);

// Создаёт продюсер с Properties и пользовательскими сериализаторами
public KafkaProducer(
	Properties properties, 
	Serializer<K> keySerializer, 
	Serializer<V> valueSerializer);
```

**Комментарии (перевод оригинальных):**
- Конструкторы создают потокобезопасный экземпляр продюсера, который можно использовать из нескольких потоков.
    
- `configs` или `properties` содержат обязательные параметры, такие как:
    - `bootstrap.servers` — список брокеров Kafka;
        
    - `key.serializer`, `value.serializer` — классы сериализаторов;
        
    - `acks`, `retries`, `batch.size`, `linger.ms` и др.
        
- Если сериализаторы не переданы, они извлекаются из конфигурации.

## Методы

### Отправка сообщений
```java
// ❗❗Асинхронно отправляет запись❗❗ в Kafka и ❗возвращает Future❗ для получения метаданных
public Future<RecordMetadata> send(ProducerRecord<K, V> record);

// Асинхронно отправляет запись и вызывает callback при завершении операции
public Future<RecordMetadata> send(
	ProducerRecord<K, V> record, 
	Callback callback);
```
(💡на самом деле он может блокироваться из-за не получения метаданных см [[6. - Отправка данных, Producer (at most-least-exactly once)]])

**Комментарий:**  
Сообщения сначала сериализуются, добавляются в буфер (`RecordAccumulator`), а затем асинхронно отправляются потоком `Sender`.  
Ошибки могут произойти **позже**, при вызове callback или получении Future.

### Управление буферами и завершением
```java
// Принудительно отправляет все накопленные в буфере сообщения
public void flush();

// Безопасно закрывает продюсер и все его внутренние потоки
public void close();

// Безопасно закрывает продюсер, ожидая завершения всех операций в течение заданного таймаута
public void close(Duration timeout);
```

**Комментарий:**
- `flush()` блокирует поток до отправки всех сообщений.
    
- `close()` — завершает работу `Sender`, очищает очереди и закрывает соединения.
    
- `close(Duration)` позволяет задать время ожидания, после которого оставшиеся записи будут отброшены.

### Работа с транзакциями
```java
// ❗❗Инициализация транзакций (выполняется один раз)❗❗
public void initTransactions();

// ❗❗Начало новой транзакции❗❗
public void beginTransaction();


// ❗❗Коммит (фиксация) текущей транзакции❗❗
public void commitTransaction();

// ❗❗Откат (rollback) текущей транзакции❗❗
public void abortTransaction();


// Отправка смещений потребителя в рамках текущей транзакции
public void sendOffsetsToTransaction(
	Map<TopicPartition, OffsetAndMetadata> offsets, 
	String consumerGroupId);

// Отправка смещений вместе с метаданными группы
public void sendOffsetsToTransaction(
	Map<TopicPartition, OffsetAndMetadata> offsets,
	ConsumerGroupMetadata groupMetadata);
```

**Комментарий:**
- Используется для реализации **Exactly-Once Semantics (EOS)**.
    
- `initTransactions()` выполняет handshake с координатором транзакций.
    
- `sendOffsetsToTransaction()` используется для атомного коммита смещений потребителя и записей продюсера в одной транзакции.

### Метаданные и вспомогательные операции
```java
// Возвращает список всех открытых транзакций (для внутреннего использования)
public Optional<Long> producerId();

// Получите полный набор внутренних метрик, поддерживаемых производителем.
public Metrics metrics();
```
**Комментарий:**
- `metrics()` предоставляет набор метрик (задержки, размер буфера, количество ретраев и т.д.), полезных для мониторинга.

## 💡 Особенности и подводные камни 💡

1️⃣ **Потокобезопасность:** `KafkaProducer` можно безопасно использовать из нескольких потоков.  

2️⃣ 💡**Асинхронность:** метод `send()` НЕ БЛОКИРУЕТ ПОТОК — все сообщения идут в фоновом `Sender`.  

3️⃣ 💡**Ограниченный буфер:** если буфер (`buffer.memory`) заполнен, `send()` может блокироваться до `max.block.ms`. 

4️⃣ 💡**Idempotence:** при `enable.idempotence=true` гарантируется, что сообщение не будет записано дважды.

5️⃣ ❗❗**Транзакции:** работают только при наличии `transactional.id` и требуют инициализации.  

6️⃣ ❗❗❗❗**Batching:** сообщения в один и тот же partition буферизуются и отправляются батчами для эффективности.

7️⃣ ❗❗❗❗**Ретраи:** при сбое отправки происходит автоматическая повторная попытка (`retries`).

---
## 💎 Best Practices 💎

✅ ❗❗❗❗Используй один экземпляр `KafkaProducer` на приложение — создание новых экземпляров дорого.  

✅ 💡Настраивай `linger.ms` и `batch.size` для оптимального баланса между задержкой и производительностью. 

✅ Включай `acks=all` для надёжности.  

✅ 💡При Exactly-Once доставке включай `enable.idempotence=true` и задавай `transactional.id`.  

✅ 💡Всегда вызывай `flush()` перед `close()`, чтобы не потерять данные.  

✅ 🧠Используй `Callback` для логирования ошибок при отправке.

## Пример
```java
import org.apache.kafka.clients.producer.*;
import java.util.*;

public class KafkaProducerExample {
    public static void main(String[] args) {
        // Конфигурация продюсера
        Properties props = new Properties();
        props.put("bootstrap.servers", "localhost:9092");
        props.put("acks", "all");
        props.put("key.serializer", "org.apache.kafka.common.serialization.StringSerializer");
        props.put("value.serializer", "org.apache.kafka.common.serialization.StringSerializer");
        props.put("linger.ms", "5");

        // Создаём продюсер
        KafkaProducer<String, String> producer = new KafkaProducer<>(props);

        try {
            for (int i = 0; i < 10; i++) {
                ProducerRecord<String, String> record =
                    new ProducerRecord<>(
	                    "demo-topic", 
	                    "key-" + i, 
	                    "Message " + i);

                // Асинхронная отправка с callback
                producer.send(record, (metadata, exception) -> {
                    if (exception == null)
                        System.out.printf(
	                        "Отправлено в %s-%d offset=%d%n",
                            metadata.topic(), 
                            metadata.partition(),
                            metadata.offset());
                    else
                        exception.printStackTrace();
                });
            }
        } finally {
            // Отправляем всё накопленное и закрываем продюсер
            producer.flush();
            producer.close();
        }
    }
}
```
