# 📕 1. Kafka API 
![[Pasted image 20251114210802.png]]

## 1. Admin API
Admin API позволяет управлять Kafka программно.
### Основной класс:
```java
org.apache.kafka.clients.admin.AdminClient
```

### Настройка клиента:
Для получения настроек Admin используется класс AdminClientConfig
```java
Properties props = new Properties(); props.put(
	AdminClientConfig.BOOTSTRAP_SERVERS_CONFIG, 
	"localhost:9092");  

AdminClient admin = AdminClient.create(props);
```
(см больше настроек в [[1. - Конфигурация Broker]])
### Создание топика:
Используется класс:

```java
org.apache.kafka.clients.admin.NewTopic
```

```java
NewTopic topic = new NewTopic(
	"orders", 
	3, 
	(short) 2); 

admin.createTopics(List.of(topic));
```

Admin API позволяет также удалять топики, изменять конфиги и получать информацию о кластере.

## 2. Producer API
Producer — компонент, который отправляет сообщения в Kafka.

### Основной класс продюсера
```java
org.apache.kafka.clients.producer.KafkaProducer
```

### Минимальные настройки:
Для получения настроек Producer используется класс ProducerConfig
```java
Properties props = new Properties(); props.put(
	ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, 
	"localhost:9092"); 
props.put(
		ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, 
		StringSerializer.class.getName()); 
props.put(
	ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, 
	StringSerializer.class.getName());
```
(см больше настроек в [[4. - Параметры Producer]])

### Создание продюсера:
```java
KafkaProducer<String, String> producer = new KafkaProducer<>(props);
````

---
### Отправка сообщения

Сообщение описывается объектом:
```java
org.apache.kafka.clients.producer.ProducerRecord
```

Пример:
```java
ProducerRecord<String, String> record = new ProducerRecord<>(
	"orders", 
	"order-1", 
	"payload");
```

### Отправка:
```java
producer.send(record);
```

send() может работать:
- асинхронно (обычно так)
- синхронно — через `.get()` на возвращаемом Future

### Что важно знать о продюсере**
- он буферизует сообщения
    
- может сжимать данные (`compression.type`)
    
- управляет батчами (`batch.size`, `linger.ms`)
    
- требует вызова `producer.close()` после завершения работы

## Consumer API
Consumer получает сообщения из Kafka.

### Основной класс
```java
org.apache.kafka.clients.consumer.KafkaConsumer
```

### Минимальная конфигурация:
```java
Properties props = new Properties(); 
props.put(
	ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, 
	"localhost:9092"); 
props.put(
	ConsumerConfig.GROUP_ID_CONFIG, 
	"orders-group"); 
props.put(
	ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG, 
	StringDeserializer.class.getName()); 
props.put(
	ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG, 
	StringDeserializer.class.getName());
```
(см больше настроек в [[3. - Параметры Consumer]])

### Создание:
```java
KafkaConsumer<String, String> consumer = new KafkaConsumer<>(props);
```

### Подписка на топики**
```java
consumer.subscribe(List.of("orders"));
```

### Чтение сообщений**
Основной метод:
```java
ConsumerRecords<String, String> records = 
	consumer.poll(Duration.ofMillis(100));
```

Consumer получает сразу набор сообщений (batched fetch), распределённых по партициям.

### Коммиты offset

Offset — позиция последнего обработанного сообщения.

Коммиты могут быть:

#### Автоматические:
```properties
enable.auto.commit=true
```

#### Ручные:
```java
consumer.commitSync(); 
consumer.commitAsync();
```

Offset'ы сохраняются в служебный топик `__consumer_offsets`.

---
## Работа в Consumer Group**
Для работы группы достаточно:
- `group.id`
- одинаковая подписка (`subscribe()`)
    

Kafka сама выполнит:
- распределение партиций
    
- выбор координатора группы
    
- выполнение JoinGroup/Rebalance

# 📕2. Kafka Streams
![[Pasted image 20251114232626.png]]

## StreamsBuilder — описание топологии
Используется класс:

```java
org.apache.kafka.streams.StreamsBuilder
```

Пример простой топологии (word count):
```java
StreamsBuilder builder = new StreamsBuilder();  

KStream<String, String> textLines = builder.stream("input-topic");  

KTable<String, Long> wordCounts = textLines
	.flatMapValues(value -> Arrays.asList(value.split(" ")))
	.groupBy((key, word) -> word)
	.count();  

wordCounts.toStream().to("output-topic");
```

Streams DSL похож на работу с коллекциями (Java Streams) или SQL.

### KStream
Это **непрерывный поток событий**, аналог ConsumerRecords:
- каждое событие обрабатывается отдельно
    
- порядок — важен только внутри партиции
    
- поддерживает трансформации, ветвление, объединение
    

### KTable
Это **логическое состояние**, представляющее последнюю версию каждого ключа:
- формируется из changelog-топика
    
- обновляет значения по ключу
    
- идеально подходит для join или агрегаций
    

### State Store
Когда выполняются агрегации, окна или `transform()`, создаются:
- RocksDB (на диске)
    
- In-memory stores
    

State store связан с KTable и агрегаторами.

### Output KStream → topic
Результирующий поток записывается в выходной Kafka-топик через:
```java
.to("output")
```

## Настройка Kafka Streams
Streams-приложение запускается с помощью класса:

```java
org.apache.kafka.streams.KafkaStreams
```

Для него требуется объект `StreamsConfig`:
```java
Properties props = new Properties();

props.put(
	StreamsConfig.APPLICATION_ID_CONFIG, 
	"wordcount-app");
props.put(
	StreamsConfig.BOOTSTRAP_SERVERS_CONFIG, 
	"localhost:9092");
props.put(
	StreamsConfig.DEFAULT_KEY_SERDE_CLASS_CONFIG, 
	Serdes.String().getClass());
props.put(
	StreamsConfig.DEFAULT_VALUE_SERDE_CLASS_CONFIG, 
	Serdes.String().getClass());
```

#### Основные обязательные параметры:
- **application.id**  
    Уникальное имя Streams приложения. Используется для state stores и для consumer groups.
    
- **bootstrap.servers**  
    Адрес брокеров.
    
- **default key/value serde**  
    Как сериализуются ключи и значения.

### Запуск Streams-приложения

```java
KafkaStreams streams = new KafkaStreams(builder.build(), props);

streams.start();
```

Приложение становится:
- consumer (читает input-topics),
    
- processor,
    
- producer (записывает output-topics).


---
### Ошибки и отказоустойчивость

Streams-приложение устойчиво:
- автоматически обрабатывает consumer rebalance
    
- восстанавливает local state store из changelog топиков
    
- перезапускается при ошибках
    
- можно использовать **uncaught exception handler**

```java
streams.setUncaughtExceptionHandler((thread, exception) ->
	StreamsUncaughtExceptionHandler
		.StreamThreadExceptionResponse.SHUTDOWN_CLIENT
);
```

### Завершение работы
```java
streams.close();
```
