
> [!NOTE] `DescribeConfigsResult`
> это **финальный (final)** класс из пакета `org.apache.kafka.clients.admin`.  
> 
> Он инкапсулирует ==результаты асинхронного запроса== `describeConfigs()`, возвращаемого из `AdminClient`.
> 

📌 Основная цель:
> **Представить результаты описания конфигураций ресурсов Kafka** (например, брокеров, топиков, пользователей) через **асинхронный API** с использованием `KafkaFuture`.

Возвращается методом:
```java
Admin.describeConfigs(
	Collection<ConfigResource> resources, 
	DescribeConfigsOptions options)
```
🔹 Каждый запрошенный `ConfigResource` (например, топик или брокер) будет иметь собственный `KafkaFuture<Config>` с конфигурацией.

---
## Методы

### Получение результатов по каждому ресурсу
```java
// Возвращает отображение ресурс → future с результатом его конфигурации.
// Позволяет работать с результатами асинхронно и индивидуально по каждому ресурсу.
public Map<ConfigResource, KafkaFuture<Config>> values()
```

💬 Пример: можно пройтись по map и вызвать `future.whenComplete(...)` для каждого топика или брокера.

### Получение единого future для всех результатов
```java
// Возвращает KafkaFuture, который завершится успешно,
// если все конфигурации были успешно получены.
// Если хотя бы одна операция завершится ошибкой — future завершится с исключением.
public KafkaFuture<Map<ConfigResource, Config>> all()
```
💬 Этот метод удобен, если нужно дождаться **всех конфигураций сразу**,  
например, при старте приложения или массовом мониторинге.

---
## 💡 Особенности и подводные камни 💡

1️⃣ ⭐`DescribeConfigsResult` — это **асинхронный контейнер**: сами данные доступны только через `.get()` или колбэки `KafkaFuture`.  

2️⃣ 💡 Если один из `ConfigResource` возвращает ошибку (например, нет прав доступа), — `all()` выбросит исключение💡 и не вернёт частичный результат.  

3️⃣ Если нужно получить **частичные результаты**, используйте `values()` и обрабатывайте каждое `KafkaFuture` отдельно.  

4️⃣ ❗❗❗Метод `.all()` может долго блокировать поток при больших кластерах — его лучше вызывать асинхронно. 

5️⃣ ❗❗❗`KafkaFuture` в Kafka **не тот же самый**, что `CompletableFuture` — но поддерживает методы `thenApply`, `whenComplete`, `exceptionally`,❗❗❗ что позволяет строить неблокирующие цепочки.

---
## 💎 Best Practices 💎

✅ 💡Используй `values()` при необходимости гибко обрабатывать несколько ресурсов (например, 10 брокеров).  
✅ 💡Не вызывай `all().get()` в UI-потоке или при большом количестве ресурсов — лучше использовать `whenComplete()`. 

✅ Для отладки удобно логировать `ConfigResource` вместе с результатом `Config` через `entry.source()` и `entry.value()`. 

✅ При запросе нескольких ресурсов обязательно задавай `timeoutMs()` в `DescribeConfigsOptions`, чтобы избежать зависаний.  

✅ Если нужно получить только один ресурс — всё равно лучше вызывать через `values().get(resource)` (это надёжнее, чем `all()`).

## Пример
```java
import org.apache.kafka.clients.admin.*;
import org.apache.kafka.common.config.ConfigResource;

import java.util.*;
import java.util.concurrent.ExecutionException;

public class DescribeConfigsResultExample {
    public static void main(String[] args) throws ExecutionException, InterruptedException {
        Properties props = new Properties();
        props.put(
	        AdminClientConfig.BOOTSTRAP_SERVERS_CONFIG, 
	        "localhost:9092");

        try (AdminClient admin = AdminClient.create(props)) {
            // Указываем ресурс, чьи конфигурации хотим получить
            ConfigResource topicResource = new ConfigResource(
	            ConfigResource.Type.TOPIC, 
	            "logs"
	        );

            // Настройки операции
            DescribeConfigsOptions options = new DescribeConfigsOptions()
                    .includeSynonyms(true)
                    .timeoutMs(4000);

            // ⭐Отправляем запрос
            DescribeConfigsResult result = 
	            admin.describeConfigs(
		            Collections.singleton(topicResource),
		            options
		        );


            // ✅Вариант 1 — получить все сразу
            Map<ConfigResource, Config> configs = 
	            result.all().get();
            
            configs.forEach((resource, config) -> {
                System.out.println("Ресурс: " + resource);
                config.entries().forEach(entry ->
                        System.out.printf(
	                        "  %s = %s (источник: %s)\n",
                                entry.name(),
                                entry.value(),
                                entry.source()));
            });


            // ✅Вариант 2 — обработать каждое future отдельно
            result.values().forEach((resource, future) ->
                future.whenComplete((config, ex) -> {
                    if (ex == null) {
                        System.out.println(
	                        "Конфигурация для ресурса " 
	                        + resource + ":"
	                    );
                        config.entries().forEach(e ->
							System.out.println(
								"  " + e.name() + 
								" = " + e.value()));
                    } else {
                        System.err.println(
                        "Ошибка при получении конфигурации " +
                        resource + ": " + ex.getMessage());
                    }
                })
            );
        }
    }
}
```
