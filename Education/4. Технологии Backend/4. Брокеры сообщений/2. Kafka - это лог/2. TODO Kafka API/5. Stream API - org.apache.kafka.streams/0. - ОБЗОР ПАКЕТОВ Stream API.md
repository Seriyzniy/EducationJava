## ✅ Реально используемые пакеты
Эти пакеты — **публичный API Kafka Streams**. Они стабилизированы, документированы и предназначены для прямого использования в приложениях.


> [!NOTE] **DSL** 
> это **специализированный язык**, который позволяет **описывать потоковые преобразования данных простыми** и интуитивно понятными **операциями**, без погружения в низкоуровневые детали
> 


| Пакет                | Назначение                                                                                  | Используется вручную                                           |
| -------------------- | ------------------------------------------------------------------------------------------- | -------------------------------------------------------------- |
| **`streams`** ✅      | Управление жизненным циклом приложения <br>и базовая настройка.                             | ✅ Всегда                                                       |
| `streams.kstream`✅   | DSL API — декларативное построение потоков и таблиц.                                        | ✅ Очень часто                                                  |
| **`streams.state`**  | Работа с состоянием (<br>`StateStore`, <br>`QueryableStore`, <br>`Stores`).                 | ✅ При использовании агрегатов, join’ов, интерактивных запросов |
| **`streams.errors`** | Обработка исключений<br>`DeserializationExceptionHandler`<br>`ProductionExceptionHandler`). | 🟡 Иногда — для продакшн-настроек                              |
| **`streams.test`**   | Тестирование (`TopologyTestDriver`, `TestInputTopic`).                                      | 🟡 В тестах, не в продакшне                                    |


## 🧠 Пакеты, которые работают "под капотом"
Следующие пакеты — это **низкоуровневые внутренние механизмы** Kafka Streams.
Ты их не используешь напрямую, если только не пишешь кастомные процессоры или экспериментальные расширения.

| Пакет                   | Назначение                                                                                  | Использовать вручную                                  |
| ----------------------- | ------------------------------------------------------------------------------------------- | ----------------------------------------------------- |
| `streams.processor`     | Низкоуровневый Processor API (старый). Управляет обработкой записей, временем и состоянием. | ⚠️ Только если DSL не хватает (редко в продакшн-коде) |
| `streams.processor.api` | Новый типобезопасный Processor API (в Kafka ≥ 3.0). Более современная альтернатива старому. | ⚠️ Используется для кастомных узлов; редко            |
| `streams.topology`      | Новый API (Kafka 4.x) для декларативного описания топологий.                                | 🚧 Пока экспериментальный, обычно не трогают          |
