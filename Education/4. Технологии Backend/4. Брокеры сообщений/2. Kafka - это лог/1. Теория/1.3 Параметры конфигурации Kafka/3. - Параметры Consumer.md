# 1. Параметры из книги и лекций
**fetch.min.bytes** - минимальный объем данных для возврата потребителю. Уменьшает нагрузку при малой активности.


**[fetch.max.wait.ms](https://fetch.max.wait.ms/)** - максимальное время ожидания данных (по умолчанию 500 мс). 
💡Вместе с fetch.min.bytes определяет баланс между задержкой и эффективностью.

**max.partition.fetch.bytes** - максимальный объем данных на раздел за один запрос (по умолчанию 1 МБ). 
💡(ЛИМИТ, СКОЛЬКО МАКСИМУМ МОГУ ПОЛУЧИТЬ ДАННЫХ ИЗ ОДНОЙ ПАРТИЦИИ)

Влияет на потребление памяти и время обработки.
Это значит, что при возврате методом KafkaConsumer.poll() объекта ConsumerRecords, объект записи будет занимать не более max.partition.fetch.bytes на каждый назначенный потребителю раздел.

Так что если в теме 20 разделов, а потребителей —5, каждому из них понадобится 4 Мбайт доступной для объекта ConsumerRecords памяти.


**[session.timeout.ms](https://session.timeout.ms/)** - время бездействия до исключения потребителя из группы (по умолчанию 3 с). Связан с [heartbeat.interval.ms](https://heartbeat.interval.ms/) (обычно 1/3 от [session.timeout.ms](https://session.timeout.ms/)).


**auto.offset.reset** - определяет поведение потребителя при начале чтения раздела, для которого у него зафиксированное смещение отсутствует или стало некорректным
- `latest` - чтение новых сообщений (по умолчанию)
    
- `earliest` - чтение с начала раздела
    


**enable.auto.commit** - автоматическая фиксация смещений (по умолчанию true). При false требуется ручное управление.


**partition.assignment.strategy** - стратегия распределения разделов:
- `Range` - последовательные подмножества разделов (по умолчанию)
    
- `RoundRobin` - циклическое распределение
  


**client.id** - идентификатор для мониторинга и квот.


**max.poll.records** - максимальное количество записей за один вызов poll(). Позволяет контролировать объем обработки.


**receive.buffer.bytes/send.buffer.bytes** - размеры TCP-буферов для оптимизации сетевого взаимодействия, особенно при работе между ЦОД.
Если значение этих параметров равно –1, будут использоваться значения по умолчанию операционной системы

### **Ключевые рекомендации:**
- Настройка [session.timeout.ms](https://session.timeout.ms/) и [heartbeat.interval.ms](https://heartbeat.interval.ms/) для баланса между быстрым обнаружением сбоев и стабильностью
    
- Выбор стратегии распределения разделов в зависимости от паттерна подписки
    
- Контроль объема данных за один poll() для своевременной отправки heartbeat


# 2. Другие параметры
## Минимальные настройки ConsumerConfig
```java
Properties props = new Properties();

props.put(
	ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, 
	"localhost:9092"
);

props.put(
	ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG,
	StringDeserializer.class.getName());
	
props.put(
	ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG, 
	StringDeserializer.class.getName());

props.put(
	ConsumerConfig.GROUP_ID_CONFIG, 
	"my-consumer-group");


// Автоматически коммитить оффсеты (можно отключить для ручного контроля)
props.put(ConsumerConfig.ENABLE_AUTO_COMMIT_CONFIG, "true");

// Интервал автокоммита
props.put(ConsumerConfig.AUTO_COMMIT_INTERVAL_MS_CONFIG, "5000");

// Начинать чтение с начала, если оффсет не найден
props.put(ConsumerConfig.AUTO_OFFSET_RESET_CONFIG, "earliest");
```

# Параметры конфигурации properties

**Пояснения**:
- `enable.auto.commit=false` даёт контроль над тем, когда оффсеты фиксировать — важно для надёжности.
    
- `max.poll.interval.ms` нужно увеличить, если обработка каждого сообщения занимает время, иначе происходит ребаланс.
    
- `isolation.level=read_committed` требуется, если вы читаете из топика, где продюсер использовал транзакции.
    
- `fetch.min.bytes`, `fetch.max.wait.ms` влияют на задержку vs пропускную способность.
    
- `partition.assignment.strategy` можно настроить под вашу топологию, особенно при ручном расширении/сжатии группы.****

## Базовые настройки подключения
```yaml
# Список брокеров Kafka (через запятую)
bootstrap.servers=localhost:9092

# Класс десериализации ключа сообщения
key.deserializer=org.apache.kafka.common.serialization.StringDeserializer

# Класс десериализации значения сообщения
value.deserializer=org.apache.kafka.common.serialization.StringDeserializer

# Уникальный идентификатор группы потребителей
group.id=my-consumer-group

# Автоматически создавать группу, если не существует
group.instance.id=

# Идентификатор клиента (для логов и мониторинга)
client.id=my-consumer-client

# Идентификатор стойки (rack) — помогает брокеру выбирать ближайшего лидера
client.rack=
```

## Управление оффсетами
```yaml
# Включить/выключить автоматический коммит оффсетов
enable.auto.commit=true

# Интервал автокоммита (в мс)
auto.commit.interval.ms=5000

# Поведение при отсутствии оффсета (при первом запуске)
# earliest — с начала лога, latest — с конца, none — бросает ошибку
auto.offset.reset=latest
```


## Управление процессом чтения (poll)
```yaml
# Максимальное количество сообщений за один poll()
max.poll.records=500

# Максимальный объём данных, получаемых из одной партиции за poll (байты)
max.partition.fetch.bytes=1048576  # 1 MB

# Минимальный размер данных, которые брокер должен отправить в ответ на fetch
fetch.min.bytes=1

# Максимальное время ожидания fetch-запроса (если нет данных)
fetch.max.wait.ms=500

# Максимальный общий размер данных, которые может хранить consumer (байты)
fetch.max.bytes=52428800  # 50 MB
```

## Таймауты и heartbeat'ы
```yaml
# Интервал heartbeat (сигнал живости для координатора группы)
heartbeat.interval.ms=3000

# Время до "выпадения" из группы, если нет heartbeat
session.timeout.ms=45000

# Максимальное время обработки одной партии (до ребаланса)
max.poll.interval.ms=300000  # 5 мин

# Таймаут для ребалансировки
rebalance.timeout.ms=60000

# Максимальное время ожидания при poll() до возврата пустого результата
request.timeout.ms=305000
```

## 🔥Изоляция транзакций
```yaml
# read_uncommitted — читать все сообщения
# read_committed — читать только завершённые транзакции
isolation.level=read_uncommitted
```

## Контроль партиций и ребалансировки
```yaml
# Стратегия распределения партиций
# RangeAssignor (по диапазонам), RoundRobinAssignor (по кругу), StickyAssignor (по "прилипанию")
partition.assignment.strategy=org.apache.kafka.clients.consumer.RangeAssignor

# Минимальное время сессии группы
group.min.session.timeout.ms=6000

# Максимальное время сессии группы
group.max.session.timeout.ms=1800000
```

## Безопасность
```yaml
# Протокол безопасности: PLAINTEXT, SASL_PLAINTEXT, SASL_SSL, SSL
security.protocol=PLAINTEXT

# Настройки SSL
ssl.truststore.location=
ssl.truststore.password=

# Механизм SASL
sasl.mechanism=PLAIN

# Конфигурация JAAS
sasl.jaas.config=com.sun.security.auth.module.Krb5LoginModule required ...
```

## Управление потоком (Flow Control)
```yaml
# Максимальное количество запросов к брокеру одновременно
max.in.flight.requests.per.connection=5

# Задержка перед повторной попыткой (в мс)
retry.backoff.ms=100

# Таймаут соединения с брокером
connections.max.idle.ms=540000
```

## Контроль логирования и метрик
```yaml
# Уровень записи метрик: INFO, DEBUG, TRACE
metrics.recording.level=INFO

# Интервал обновления метрик (в мс)
metrics.sample.window.ms=30000

# Количество сэмплов для усреднения метрик
metrics.num.samples=2
```

## Расширенные параметры
```yaml
# Задержка перед повторным подключением к координатору группы (мс)
reconnect.backoff.ms=50

# Максимальное значение reconnect.backoff.ms
reconnect.backoff.max.ms=1000

# Максимальное время жизни метаданных брокеров
metadata.max.age.ms=300000  # 5 мин
```

## Управление потоками
```yaml
# Количество потоков для операций ввода-вывода

num.network.threads=3

# Количество потоков для fetch-запросов
num.fetch.threads=1
```
