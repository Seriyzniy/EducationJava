# 📕1. Zookeeper
ZooKeeper — это **внешний координационный сервис**, который:

### 🟦 1. Управляет метаданными Kafka
ZooKeeper хранит:
- список брокеров
- список топиков
- список партиций
- лидеров партиций
    
- ISR ("in-sync replicas")
    
- ACL
- конфигурации брокеров

### 🟧 2. Выбирает контроллер Kafka
Контроллер — это _один из брокеров_, который:
- следит за состоянием всех брокеров
- назначает лидеров партиций
- обрабатывает failover

Контроллером становится тот брокер, который первым записался в ZK ноду `/controller`.

### 🟩 3. Хранит данные конфигурации
Каждый параметр Kafka может быть переопределён через ZooKeeper.

## Как устроен ZooKeeper внутри
ZooKeeper — это:
- распределённая файловая система типа UNIX (/path/to/node)
- каждый узел называется **znode**
- запись атомарная
    
- работает через протокол Zab (аналог консенсуса)

### Типы нод:
| Тип            | Назначение                                                                  |
| -------------- | --------------------------------------------------------------------------- |
| **persistent** | постоянная (используется Kafka для топиков, конфигов)                       |
| **ephemeral**  | исчезает при отключении клиента (Kafka использует для регистрации брокеров) |
| **sequential** | создаётся с авто-номером (например: `/controller_epoch_00000001`)           |

## Кворум ZooKeeper?
У ZooKeeper тоже есть внутренние порты (но это НЕ Kafka listeners):

### Стандартные порты ZooKeeper
| Порт     | Назначение                                 |
| -------- | ------------------------------------------ |
| **2181** | Клиентские подключения (Kafka, приложения) |
| **2888** | Репликация между серверами кворума         |
| **3888** | Выбор лидера                               |

### Параметры кворума
#### server.N 
Это главный параметр, который формирует кворум.
```ini
server.<id>=<hostname>:<quorum_port>:<election_port>
```

Пример
```ini
server.1=zoo1:2888:3888
server.2=zoo2:2888:3888
server.3=zoo3:2888:3888
```

|Часть|Назначение|
|---|---|
|**server.1**|ID сервера (должен совпадать с myid на ноде)|
|**zoo1**|Хост машины|
|**2888**|Порт репликации между серверами (кворумная связь)|
|**3888**|Порт выборов лидера|

# 📕 2. Основные параметры конфигурации ZooKeeper

## clientPort
Порт, на котором ZooKeeper **принимает соединения клиентов**.

Стандарт:
```ini
clientPort=2181
```

🔥Значение важно для Kafka:
```ini
KAFKA_ZOOKEEPER_CONNECT=zookeeper:2181
```

---
## tickTime
**Базовая единица времени в ZooKeeper.**

По умолчанию:
```ini
tickTime=2000  # 2 секунды
```
Используется для:
- heartbeat'ов
- timeout'ов сессии

---
## initLimit
Сколько `tickTime` можно тратить ==на начальную синхронизацию== follower → leader.

```ini
initLimit=10
```

Это значит:
```ini
10 * tickTime = 20 секунд
```

---
## syncLimit
Сколько времени follower может отставать от leader.
```ini
syncLimit=5  # 5 * 2 = 10 сек
```

Если follower превышает — исключается из кворума.

---
## dataDir
Где ZooKeeper хранит снимки (snapshots):
```ini
dataDir=/var/lib/zookeeper
```

---
## dataLogDir
Отдельная директория для журналов транзакций:
```ini
dataLogDir=/var/log/zookeeper
```
> Разносить обязательно в продакшене.

# Важные параметры ZooKeeper (Security)

### ✔ authProvider.x
Устанавливает авторизацию (например SASL):
```ini
authProvider.1=org.apache.zookeeper.server.auth.SASLAuthenticationProvider
```

### ✔ requireClientAuthScheme
Требовать авторизацию:
```ini
requireClientAuthScheme=sasl
```

### ✔ jaasLoginRenew
Период обновления Kerberos-токенов.
