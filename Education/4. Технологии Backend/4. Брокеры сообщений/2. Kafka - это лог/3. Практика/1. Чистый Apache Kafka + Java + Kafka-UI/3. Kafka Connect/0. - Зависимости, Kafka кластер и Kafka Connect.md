# 1. Kafka кластер
Как уже было описано в конспекте "Принципы работы с Kafka Conncet", нам необходимо:
- Источник данных (БД)
- Кластер Kafka
- Kafka Connect (коннектор)
- Конфигурационный файл коннектора 

В данном случае нам необходимо выполнить последние 3 пункта, поскольку БД Postgres установлен локально на компьютере

# 2. Kafka Connect
При работе с Kafka Connect Kafka кластер работает как и обычно. Нам нужно лишь добавить контейнер, который с коннектером.

Использовать мы будем Debezium. Для этого нам необходимо ДОБАВИТЬ в предыдущий Docker-compose следующий сервис:
```yaml
debezium-connector:  
  image: debezium/connect:2.7.3.Final  
  container_name: debezium-kafka-connect  
  ports:  
    - "8083:8083"  
  environment:  
    BOOTSTRAP_SERVERS: "kafka-bc-container-1:19092,kafka-b-container-2:19092,kafka-b-container-3:19092"  
    OFFSET_STORAGE_TOPIC: "connector-offset-lsn"  
    CONNECT_STATUS_STORAGE_TOPIC: "connector-status"  
    CONFIG_STORAGE_TOPIC: "connector-config"  
    GROUP_ID: 1
```

> [!NOTE] Указание портов
> В качестве `BOOTSTRAP_SERVERS` нужно указать порты, НА КОТОРЫХ слушают контейнеры Kafka, которые мы ранее описали, а это 19092
> (9092 мы установили в качестве слушателя для клиентов)

# 3. Подключение зависимостей
Для работы с Kafka Connect нам нужно, чтобы создавались топики указанные в Debezium.

Это можно сделать 
- вручную, через CLI или KafkaAdmin в коде \
- автоматически, если настроен параметр `auto.create.topics.enable=true` в /opt/kafka/config/server.properties
- автоматически, если установить ПЕРЕМЕННУЮ ОКРУЖЕНЯ в ДРУГОЙ образе для брокеров, например `confluentinc/cp-kafka:7.6.0` есть переменная `KAFKA_AUTO_CREATE_TOPICS_ENABLE`

> Мы же запустили оригинальный образ, и методом "тыка", определили, что свойство установлено в true 
> `auto.create.topics.enable=true`

Поскольку, теперь нам не нужен Producer, а лишь класс, который будет сохранять данные в БД, то нам нужно подключить ДРАЙВЕР для Postgres

А также зависимость Kafka-Clients, поскольку нам все еще нужен Consumer

И чтобы Consumer мог десериализовать данные из Kafka, нам нужен Jackson

```xml
<dependencies>
	<dependency>
		<groupId>org.apache.kafka</groupId>
		<artifactId>kafka-clients</artifactId>
		<version>4.1.0</version>
	</dependency>

	<dependency>
		<groupId>org.postgresql</groupId>
		<artifactId>postgresql</artifactId>
		<version>42.7.8</version>
	</dependency>

</dependencies>
```
