# 1. Запуск через Docker-compose
Опишем Docker-compose. При этом, добавим kafka-ui, для наглядности

## Поиск образа для kafka-ui
В DockerHub нашел наиболее популярный образ для Kafka-ui
![[Pasted image 20251116103423.png]]
Его и будем использовать

Для его конфигурации нужно [Веб-интерфейс для Kafka](https://itzm.tech/p/kafka_ui/?ysclid=mi1ei1vkzd968091306):
```yaml
environment:
      KAFKA_CLUSTERS_0_NAME: "local"
      KAFKA_CLUSTERS_0_BOOTSTRAPSERVERS: "kafka:9092" # замени на адрес твоего брокера
```

> Чтобы контейнер Kafka-ui УВИДЕЛ Kafka контейнер, **они должны быть в одной сети**.
> Поэтому опишем Docker-compose, который ПО УМОЛЧАНИЮ создает сеть для сервисов


## Docker-compose
```docker-compose
version: '3.8'  
  
services:  
  kafka:  
    image: apache/kafka:latest  
    container_name: kafka  
    environment:  
      KAFKA_NODE_ID: 1  
      KAFKA_PROCESS_ROLES: broker,controller  
      KAFKA_LISTENERS: MY_HOST://0.0.0.0:9092,EXTERNAL://0.0.0.0:29092,CONTROLLER://0.0.0.0:9093  
      KAFKA_ADVERTISED_LISTENERS: MY_HOST://localhost:9092,EXTERNAL://kafka:29092,CONTROLLER://kafka:9093  
      KAFKA_LISTENER_SECURITY_PROTOCOL_MAP: MY_HOST:PLAINTEXT,EXTERNAL:PLAINTEXT,CONTROLLER:PLAINTEXT  
      KAFKA_INTER_BROKER_LISTENER_NAME: MY_HOST  
  
      KAFKA_CONTROLLER_LISTENER_NAMES: CONTROLLER  
      KAFKA_CONTROLLER_QUORUM_VOTERS: 1@kafka:9093  
  
      KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR: 1  
      KAFKA_TRANSACTION_STATE_LOG_REPLICATION_FACTOR: 1  
      KAFKA_TRANSACTION_STATE_LOG_MIN_ISR: 1  
  
    ports:  
      - "9092:9092"    # для хоста (Java клиент)  
      - "29092:29092"  # для Docker (Kafka-UI)  
      - "9093:9093"    # контроллер (необязательно наружу)  
  
  kafka-ui:  
    image: provectuslabs/kafka-ui:latest  
    container_name: kafka-ui  
    ports:  
      - "8080:8080"  
    environment:  
      KAFKA_CLUSTERS_0_NAME: "local"  
      KAFKA_CLUSTERS_0_BOOTSTRAPSERVERS: "kafka:29092"
```

(см про устройство KRaft и его параметры[[0. - KRaft]])

Мы указываем на каких портах принимать:
- запросы клиентов
- запросы от других Docker-контейнеров

### ❗❗❗ ОЧЕНЬ ВАЖНО ❗❗❗
Поскольку, мы запускаем одного брокера, то НЕОБХОДИМО УКАЗАТЬ эти параметры
```ini
KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR: 1  
KAFKA_TRANSACTION_STATE_LOG_REPLICATION_FACTOR: 1  
KAFKA_TRANSACTION_STATE_LOG_MIN_ISR: 1 
```

Иначе, без этой конфигурации топик `__consumers_offsets`, НЕ БУДЕТ СОЗДАН, и Consumer будет БЕСКОНЕЧНО ОТРПАВЛЯТЬ ЗАПРОС НА СОЗДАНИЕ ЭТОГО ТОПИКА при вызове методы poll()

#### Почему не создастся?
Потому что ПО УМОЛЧАНИЮ количество реплик для `__consumers_offsets` = 3
```ini
replication factor = 3
offsets.topic.num.partitions=50
```

А у нас всего один брокер! И Kafka не может запихнуть все 3 реплики на одного брокера - из-за ОГРАНИЧЕНИЙ ОТКАЗОУСТОЙЧИВОСТИ
(см [[5. - Репликация партиций, балансировка и надежность]])
