# Описание проекта
Ссимулируем систему обработки заказов. У нас будет 3 топика
- Пользователи (клиенты)
- Покупки клиентов

Producer будет составлять данные и отправлять в два топика. Тем самым симулируя работу покупателя, который покупает товар

Consumer используя Kafka Streams должен:
- **Соединить (KStream → KTable) покупки с пользователями**, чтобы обогащать информацию о покупке данными пользователя.

- **Сгруппировать покупки по странам**.

- **Посчитать число покупок пользователей из каждой страны**.

- Результат ==отправить в топик==: `purchase-stats-by-country`

> [!NOTE] Запись в БД
> НЕЛЬЗЯ ИСПОЛЬЗОВАТЬ длительные операции в Kafka Streams
> см [[3.1 -- Компоненты]]

# 0. Добавление файла .properties
Чтобы не хардкодить названия топиков, адреса серверов и прочего прямо в код, добавим файл .properties, где будут указаны все настройки

Для этого, в папке resources добавим файл `kafka-config.properties`
```css
kafka.bootstrap-servers=localhost:29092,localhost:39092,localhost:49092  
kafka.topic-users=topic-users  
kafka.topic-products=topic-product  
kafka.topic-result=topic-product-stats-by-country
```

Считывать файл будем при помощи Properties через метод `load()` в параметры которого нужно передать любой класс из java.io для чтения - InputStream, Reader

# 1. Определение данных
Теперь определим данные, которые будут отправляться.

Из описания проекта, нам нужно 3 топика
- Топик для данных пользователя
- Топик для заказов пользователя
- Топик для вывода результатов

Теперь опишем данные DTO для каждого топика

> [!NOTE] ХИТРОСТЬ
> Так как нам придется отправлять данные в топики, то придется писать ДВА разных KafkaProducer, с разными сериализаторами! 
> (Поскольку разные типы данных отправляются в 2 разных топика)
> 
> Кошмар
> НО, можно схитрить! **Было бы здорово написать один сериализатор**, который **бы подходил для двух типов данных**, и использовать его в настройке KafkaProducer
> 
> То есть, нужно каким-то образом сделать два класса, одного типа - ИНТЕРФЕЙСЫ
> (см [[9.2 -- Ссылка интерфейса ссылается на класс реализацию + Правило]])

Используя наследование мы **определим** БАЗОВЫЙ тип для двух типов данных - по сути **маркерный интерфейс**.
Именно **для него и будем писать сериализатор**, и использовать в KafkaProducer

Назовем такой маркерный интерфейс `Data`

А от этого базового маркерного интерфейса реализуем два типа данных
- `DataUser`
- `DataProduct`

А в класс сериализатор будет принимать любой объект типа `Data`, и сериализовывать его.

Будет работать? - ДА

Поскольку объект в памяти создастся (мы будем создавать объекты данных `DataUser`, `DataProduct`) и передавать ссылки на эти объекты в метод сериализации. 

При помощи нисходящего преобразования ссылка на конкретный объект преобразуется в ссылку на `Data`, ПРИ ЭТОМ ДАННЫЕ, которые лежат в объекте останутся

# 2. Описание класса де/сериализации
Основываясь на идее описанной выше, напишем классы де/сериализации.
(см [[0. - ОБЗОР Common Serialization API]])

Так как мы добавили в заголовок Instant из java.time, то нам необходимо подключить еще одну зависимость от Jackson - `jackson-datatype-jsr310`. 
Потому что Jackson не умеет работать со временем

Поэтому в pom.xml добавим записимость
```xml
<dependency>  
    <groupId>com.fasterxml.jackson.datatype</groupId>  
    <artifactId>jackson-datatype-jsr310</artifactId>  
    <version>2.20.1</version> <!-- или актуальная версия -->  
</dependency>
```

И также необходимо будет настроить ObjectMapper
```java
objectMapper.registerModule(new JavaTimeModule())
```

Но поскольку, objectMapper мы будем использовать в разных классах, нам придется каждый раз его настраивать `registerModule(new JavaTimeModule())`

Но чтобы этого не делать, напишем класс обертку над ObjectMapper, и **сделаем его singletone**
### CustomObjectMapper
```java
package util;  
  
import com.fasterxml.jackson.databind.ObjectMapper;  
import com.fasterxml.jackson.datatype.jsr310.JavaTimeModule;  
  
public class CustomObjectMapper {  
    private static final ObjectMapper INSTANCE = new ObjectMapper();  
  
    private CustomObjectMapper() {}  
    static {  
        INSTANCE.registerModule(new JavaTimeModule());  
    }  
  
    public static ObjectMapper instance() {  
        return INSTANCE;  
    }  
}
```

## 1. Класс сериализации
Для сериализации будем использовать библиотеку Jackson, которая сериализает объект в JSON

```java
package util;  
  
import com.fasterxml.jackson.core.JsonProcessingException;  
import com.fasterxml.jackson.databind.ObjectMapper;  
import model.Data;  
import org.apache.kafka.common.serialization.Serializer;  
  
public class DataSerializer implements Serializer<Data> {  
    private final ObjectMapper objectMapper = 
	    CustomObjectMapper.instance();  
  
    @Override  
    public byte[] serialize(String s, Data data) {  
        try {  
            return objectMapper.writeValueAsBytes(data); 
             
        } catch (JsonProcessingException e) {  
            System.out.println("Exception on serializer User");  
            throw new RuntimeException(e);  
        }  
    }  
}
```

Здесь метод `serialize(String s, Data data)` принимает ссылку на маркерный интерфейс, и поэтому под этот интерфейс теперь подходят и `DataUser` и `DataProduct`

А ссылка на типа `Data data` будет указывать на конкретный объект в памяти, и поэтому Jackson сериализует именно объект

## 2. Класс десереализации
Теперь опишем класс десериализации для Consumer

Мы уже знаем, что с сообщением будет передаваться заголовок `HeaderDataType`, который будет хранить поле с типом payload (сообщения)

Нам необходимо **перед тем как десереализовать сообщение**, **прочитать заголовок**, и на основе его, **определить тип тела** сообщения

При десериализации при помощи Jackson используется метод `readValue()`, который принимает
- байты сообщения
- тип класса, в который нужно преобразовать байты

Чтобы десериализовать байты в `DataUser` и `DataProduct`, нам нужно получить объект `Class` этих классов.

Это можно сделать при помощи метода `Class.forName()`, а имя у нас есть в заголовке

```JAVA
package util;  
  
import com.fasterxml.jackson.databind.ObjectMapper;  
import model.Data;  
import model.HeaderDataType;  
import org.apache.kafka.common.header.Headers;  
import org.apache.kafka.common.serialization.Deserializer;  
  
import java.io.IOException;  
  
public class DataDeserializer implements Deserializer<Data> {  
    private final ObjectMapper objectMapper = 
	    CustomObjectMapper.instance();  
  
    @Override  
    public Data deserialize(String s, byte[] bytes) {  
        return null;  
    }  
  
    @Override  
    public Data deserialize(String topic, Headers headers, byte[] data) {  
        try {  
	        // Получаем байты заголовка
            byte[] headerDataTypeByte =
	            headers.lastHeader("DataType").value(); 
	        
	        // Десериализуем заголовок
            HeaderDataType headerDataType = 
	            objectMapper.readValue(
		            headerDataTypeByte, 
		            HeaderDataType.class
		        );  
			  
			//Получаем объект Class по имени типа класса из заголовка
            Class<?> clazz = 
	            Class.forName(headerDataType.getClassName());  
  
			// Десериализуем тело сообщения
            return (Data)objectMapper.readValue(data, clazz);  
            
        } catch (IOException e) {  
            System.out.println("Exception on deserializing data");  
            throw new RuntimeException(e);  
        } catch (ClassNotFoundException e) {  
            System.out.println("Exception on found CLASS for name");  
            throw new RuntimeException(e);  
        }  
    }  
}
```

> [!NOTE] ВАЖНО
> В интерфейсе `Deserializer` есть два метода десериализации
> ```java
> public interface Deserializer<T> extends Closeable {
> 	T deserialize(String var1, byte[] var2);
> 	
> 	default T deserialize(
> 		String topic, 
> 		Headers headers, 
> 		byte[] data) 
> 	{ 
> 		return (T)this.deserialize(topic, data);
> 	}
> }
> ```
> Kafka по умолчанию ПЫТАЕТСЯ ВЫЗВАТЬ метод с Headers, но если он не определен, вызывает обычный метод (видно из кода)

# 3. Написания Admin
Теперь опишем класс `CustomKafkaAdmin` который будет создавать топики

Для чтения свойств из файла `kafka-config.properties` как уже говорилось используем класс `Properties`

> [!NOTE] Нужно указать полный путь до файла с `.properties`
> Так как используем обычный `Reader`, который читает файла из корня проекта, то необходимо явно указать путь до файла
> 
> Или же использовать 
> - ConfigLoader
> - Через класс CustomKafkaAdmin получить ClassLoader и затем `.getResourceAsStream("kafka-config.properties")`


```java
package admin;  
  
import org.apache.kafka.clients.admin.AdminClient;  
import org.apache.kafka.clients.admin.AdminClientConfig;  
import org.apache.kafka.clients.admin.NewTopic;  
  
import java.io.FileReader;  
import java.io.IOException;  
import java.util.Collections;  
import java.util.Properties;  
  
public class CustomKafkaAdmin {  
    private final Properties props = new Properties();  
  
    public CustomKafkaAdmin(String bootstrapServers) {  
        props.put(  
            AdminClientConfig.BOOTSTRAP_SERVERS_CONFIG,  
            bootstrapServers  
        );  
    }  
  
    public void createTopic(String topicName, int partitions, int replication) {  
        try(AdminClient adminClient = AdminClient.create(props)) {  
            adminClient.createTopics(  
                    Collections.singleton(  
                            new NewTopic(  
                                    topicName,  
                                    partitions,  
                                    (short)replication  
                            )  
                    )  
            );  
        } catch (Exception e) {  
            System.out.println(
	            "Exception on Kafka Admin, in create topic: " + 
	            e.getMessage());  
        }  
    }  
  
    public static void main(String[] args) {  
        try(FileReader fileReader = new 
	        FileReader("src/main/resources/kafka-config.properties")
        
        ) {  
            Properties props = new Properties();  
            props.load(fileReader);  
  
            CustomKafkaAdmin admin = 
	            new CustomKafkaAdmin(
		            props.getProperty("kafka.bootstrap-servers"));  
  
            admin.createTopic(
	            props.getProperty("kafka.topic-users"), 2, 2);  
            
            admin.createTopic(
	            props.getProperty("kafka.topic-products"), 2, 2);  
            
            admin.createTopic(
	            props.getProperty("kafka.topic-result"), 2, 2);  
  
        } catch (IOException e) {  
            throw new RuntimeException(e);  
        }  
    }  
}
```

# 4. Описание Producer
Теперь опишем Producer.

Как и обсуждалось ранее, у нас будет ОДИН KafkaProducer для отправки данных в топики.

Для этого в параметре указываем наш универсальный сериализатор
```java
props.put(  
	ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG,  
	DataSerializer.class  
);  
```

Теперь мы можем отправлять данные в два топика. Для этого напишем два метода, в каждом из которых создадим:
- заголовок с полем = имя класса (через Class)
- временную метку Instant


```JAVA
package producer;  
  
import com.fasterxml.jackson.core.JsonProcessingException;  
import com.fasterxml.jackson.databind.ObjectMapper;  
import model.Data;  
import model.DataProduct;  
import model.HeaderDataType;  
import model.DataUser;  
import org.apache.kafka.clients.producer.KafkaProducer;  
import org.apache.kafka.clients.producer.ProducerConfig;  
import org.apache.kafka.clients.producer.ProducerRecord;  
import org.apache.kafka.common.header.Header;  
import org.apache.kafka.common.header.internals.RecordHeader;  
import org.apache.kafka.common.serialization.StringSerializer;  
import util.CustomObjectMapper;  
import util.DataSerializer;  
  
import java.io.FileReader;  
import java.io.IOException;  
import java.time.Instant;  
import java.util.ArrayList;  
import java.util.List;  
import java.util.Properties;  
import java.util.concurrent.TimeUnit;  
  
public class CustomKafkaProducer {  
    private final Properties props = new Properties();  
    private final ObjectMapper objectMapper = 
	    CustomObjectMapper.instance();  
  
    public static void main(String[] args) {  
        try(FileReader fileReader = 
	        new FileReader(
		        "src/main/resources/kafka-config.properties")
		){  
			// Чтение свойств из файла
            Properties props = new Properties();  
            props.load(fileReader);  
			  
			// Создание продюсера
            CustomKafkaProducer kafkaProducer = new 
	            CustomKafkaProducer(
		            props.getProperty("kafka.bootstrap-servers"));  
			
			// Подготовка данных для отправки
            DataUser dataUser = 
	            new DataUser(
		            1, 
		            "User-1", 
		            "RU"
		        );  
            DataProduct dataProduct = 
	            new DataProduct(
		            101, 
		            1, 
		            "Iphone", 
		            1
		        );  
  
			// Отправка данных
            kafkaProducer.sendUserRecord(
	            props.getProperty("kafka.topic-users"), 
	            dataUser);  
            
            kafkaProducer.sendProductRecord(
	            props.getProperty("kafka.topic-products"), 
	            dataProduct);  
  
        } catch (IOException e) {  
            throw new RuntimeException(e);  
        }  
  
    }  
  
    public CustomKafkaProducer(String bootstrapServers) {  
        props.put(  
            ProducerConfig.BOOTSTRAP_SERVERS_CONFIG,  
            bootstrapServers  
        );  
        props.put(  
            ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG,  
            StringSerializer.class  
        );  
        props.put(  
            ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG,  
            DataSerializer.class  
        );  
    }  
  
    public void sendUserRecord (String topic, DataUser user) {
	    // Создание заголовка  
        HeaderDataType header = 
	        new HeaderDataType(
		        user.getClass().getName(), 
		        Instant.now()
		    );  
  
        try (KafkaProducer<String, Data> producer = 
	        new KafkaProducer<>(props)
	    ){  
		    // Добавление заголовка в список
            List<Header> producerHeaders = new ArrayList<>();  
            producerHeaders.add(  
				new RecordHeader(  
					"DataType",  
					objectMapper.writeValueAsBytes(header)  
				)  
            );  
			
			// Составление сообщения с заголовками
            ProducerRecord<String, Data> record =  
				new ProducerRecord<>(  
					topic,  
					null,  
					"users",  
					user,  
					producerHeaders);  

            producer.send(record).get(10, TimeUnit.SECONDS);  
  
        } catch (JsonProcessingException e) {  
            System.out.println(
	            "Exception on serializer header on send record");  
            throw new RuntimeException(e);  
        } catch (Exception e) {  
            System.out.println("Exception on send record");  
            throw new RuntimeException(e);  
        }  
    }  
  
    public void sendProductRecord (String topic, DataProduct product) { 
        // Создание заголовка  
        HeaderDataType header = 
	        new HeaderDataType(
		        product.getClass().getName(), 
		        Instant.now()
		    );  
  
        try (KafkaProducer<String, Data> producer = 
	        new KafkaProducer<>(props)
	    ){  
		    // Добавление заголовка в список
            List<Header> producerHeaders = new ArrayList<>();  
            producerHeaders.add(  
                    new RecordHeader(  
                            "DataType",  
                            objectMapper.writeValueAsBytes(header)  
                    )  
            );  
			  
			// Составление сообщения с заголовками
            ProducerRecord<String, Data> record =  
                    new ProducerRecord<>(  
                            topic,  
                            null,  
                            "products",  
                            product,  
                            producerHeaders);  
  
            producer.send(record).get(10, TimeUnit.SECONDS);  
  
        } catch (JsonProcessingException e) {  
            System.out.println(
	            "Exception on serializer header on send record");  
            throw new RuntimeException(e);  
        } catch (Exception e) {  
            System.out.println("Exception on send record");  
            throw new RuntimeException(e);  
        }  
    }  
}
```

# 5. Описание Consumer
Чтобы проверить, что данные приходят так как и задумывалось, напишем ПРОСТОЙ Consumer (без Kafka Streams)

Consumer будет принимать данные, десериализовывает их, и выводит содержимое при помощи переопределенного метода toString
```java
package consumer;  
  
import com.fasterxml.jackson.core.exc.StreamReadException;  
import com.fasterxml.jackson.databind.DatabindException;  
import com.fasterxml.jackson.databind.ObjectMapper;  
import model.Data;  
import model.HeaderDataType;  
import org.apache.kafka.clients.consumer.ConsumerConfig;  
import org.apache.kafka.clients.consumer.ConsumerRecord;  
import org.apache.kafka.clients.consumer.ConsumerRecords;  
import org.apache.kafka.clients.consumer.KafkaConsumer;  
import org.apache.kafka.common.TopicPartition;  
import org.apache.kafka.common.header.Header;  
import org.apache.kafka.common.serialization.StringDeserializer;  
import util.CustomObjectMapper;  
import util.DataDeserializer;  
  
import java.io.FileNotFoundException;  
import java.io.FileReader;  
import java.io.IOException;  
import java.time.Duration;  
import java.util.List;  
import java.util.Properties;  
  
public class CustomKafkaConsumer {  
    private final Properties props = new Properties();  
    private final ObjectMapper objectMapper = CustomObjectMapper.instance();  
  
    public static void main(String[] args) {  
        try(FileReader fileReader = 
	        new FileReader(
		        "src/main/resources/kafka-config.properties")
		){  
            Properties props = new Properties();  
            props.load(fileReader);  
  
            CustomKafkaConsumer kafkaConsumer = 
	            new CustomKafkaConsumer(
		            props.getProperty("kafka.bootstrap-servers"));  
  
            kafkaConsumer.consume(  
                    props.getProperty("kafka.topic-users"),  
                    props.getProperty("kafka.topic-products"));  
  
        } catch (FileNotFoundException e) {  
            System.out.println("File not found in consumer");  
            throw new RuntimeException(e);  
        } catch (IOException e) {  
            throw new RuntimeException(e);  
        }  
    }  
  
    public CustomKafkaConsumer(String bootstrapServers) {  
        props.put(  
            ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG,  
            bootstrapServers  
        );  
        props.put(  
            ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG,  
            StringDeserializer.class  
        );  
        props.put(  
            ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG,  
            DataDeserializer.class  
        );  
        props.put(  
            ConsumerConfig.GROUP_ID_CONFIG,  
            "group-1"  
        );  
        props.put(  
            ConsumerConfig.AUTO_OFFSET_RESET_CONFIG,  
            "earliest"  
        );  
    }  
  
    public void consume(String userTopic, String productTopic) {  
        try (KafkaConsumer<String, Data> consumer = 
	        new KafkaConsumer<>(props)
	    ) {  
            consumer.subscribe(List.of(userTopic, productTopic));  
  
            while(true){  
                ConsumerRecords<String, Data> records = 
	                consumer.poll(Duration.ofSeconds(10));  
  
                for(TopicPartition topicPartition : 
	                records.partitions()
	            ){  
                    System.out.println(
	                    "Consumer from : " + topicPartition.topic());  
  
                    for(ConsumerRecord<String, Data> record : 
	                    records.records(topicPartition)
	                ){  
                        System.out.println(  
                            record.value().toString()  
                        );  
                    }  
                    System.out.println();  
                }  
            }  
        }  
    }  
}
```

> Чтобы проверить, что данные правильно принимают свой тип, можно убрать из классов `DataUser` `DataProduct` метод `toString()`, и тогда при запуске кода, они будут выдавать имя класса - тип к которому они принадлежат
