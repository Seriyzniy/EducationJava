# 📕 1. Класс MyAdmin

Как видно из схемы работы с Kafka API, вся работа с администрированием Kafka, выполняется через AdminClietn (можно также и через Admin, KafkaProducer)

```JAVA
package admin;

import org.apache.kafka.clients.admin.AdminClient;
import org.apache.kafka.clients.admin.AdminClientConfig;
import org.apache.kafka.clients.admin.NewTopic;

import java.util.Collections;
import java.util.Properties;

public class MyAdminClient {
    public static void main(String[] args) {
        Properties props = new Properties();
        props.put(
	        AdminClientConfig.BOOTSTRAP_SERVERS_CONFIG, 
	        "localhost:9092");

        try (AdminClient admin = AdminClient.create(props)){
            NewTopic topic = new NewTopic("mytopic", 1, (short) 1);

            admin.createTopics(Collections.singleton(topic));

        } catch (Exception e){
            System.out.println("Exception on Admin API");
        }
    }
}
```

# 📕 2. Класс MyPorducer
Теперь опишем простого Producer, который отправит одно сообщение

Как помним из теории, МИНИМАЛЬНО, нам нужно указать в параметрах
- адрес брокера
- сериализатор ключа
- сериализатор сообщения

```java
package producer;

import org.apache.kafka.clients.producer.KafkaProducer;
import org.apache.kafka.clients.producer.ProducerConfig;
import org.apache.kafka.clients.producer.ProducerRecord;
import org.apache.kafka.common.serialization.StringSerializer;

import java.util.Properties;

public class MyProducer {
    private final KafkaProducer<String, String> producer;

    public MyProducer() {
        Properties props = new Properties();
        props.put(
                ProducerConfig.BOOTSTRAP_SERVERS_CONFIG,
                "localhost:9092");
        props.put(
                ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG,
                StringSerializer.class.getName());
        props.put(
                ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG,
                StringSerializer.class.getName());

        producer = new KafkaProducer<>(props);
    }

    public void produceMessaging(){
        try(producer){
            producer.send(new ProducerRecord<>(
                    "mytopic",
                    "Hello from Producer"));
            System.out.println("[INFO] Message sent");

        } catch (Exception e){
            System.out.println("Exception on produce message");
        }
    }


    public static void main(String[] args) {
        MyProducer myProducer = new MyProducer();
        myProducer.produceMessaging();
    }
}
```

# 📕 3. Класс MyConsumer
Теперь опишем Consumer.

Здесь установим такие параметры
- адрес брокера
- десериализатор ключа
- десериализатор сообщения
- group.id (так как consumer всегда находится в какой-то группе)
- поведение при отсутствии смещения (чтобы мождно было запускать по-отдельности Prodcuer и Consumer)

```java
package consumer;

import org.apache.kafka.clients.consumer.ConsumerConfig;
import org.apache.kafka.clients.consumer.ConsumerRecord;
import org.apache.kafka.clients.consumer.ConsumerRecords;
import org.apache.kafka.clients.consumer.KafkaConsumer;
import org.apache.kafka.common.serialization.StringDeserializer;

import java.time.Duration;
import java.util.Collections;
import java.util.Properties;

public class MyConsumer {
    private final KafkaConsumer<String, String> consumer;

    public MyConsumer() {
        Properties props = new Properties();
        props.put(
            ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG,
            "localhost:9092"
        );
        props.put(
            ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG,
            StringDeserializer.class);
        props.put(
            ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG,
            StringDeserializer.class
        );
        props.put(
	        ConsumerConfig.GROUP_ID_CONFIG, 
	        "group-1");
        props.put(
	        ConsumerConfig.AUTO_OFFSET_RESET_CONFIG, 
	        "earliest");

        consumer = new KafkaConsumer<>(props);
    }

    public void consumeMessages() {
        try(consumer){
	        // 1️⃣ Подписка на топик
            consumer.subscribe(Collections.singleton("mytopic"));
            System.out.println(
	            "[INFO] Subscribed to topic: mytopic");

			// 2️⃣ Чтение данных
            while(true){
                ConsumerRecords<String, String> records =
                        consumer.poll(Duration.ofMillis(1000));
                System.out.println(
	                "[INFO] Polled records: " +
		            records.count());

                for(ConsumerRecord<String, String> record :
	                 records)
	            {
                    System.out.println(
	                    "Got message: " + record.value());
                }
                Thread.sleep(2000);
            }


        } catch (Exception e){
            System.out.println("Exception on consume message");
        }
    }

    public static void main(String[] args) {
        MyConsumer myConsumer = new MyConsumer();
        myConsumer.consumeMessages();
    }
}
```

### Запуск
> [!NOTE] Conusmer НЕ СЧИТЫВАЕТ сообщения
> Такое может произойти, если до этого уже запускал Consumer.
> Группа, с которой был запущен Consumer уже была зарегистрирована, 


При этом, сообщение читается только после 2-го запуска метода poll (сначала происходит балансировка партиций, а только потом чтение)

(запустил через консоль, при этом нужно указать -cp и директорию с подключенными jar файлами `Downloads\*`)
