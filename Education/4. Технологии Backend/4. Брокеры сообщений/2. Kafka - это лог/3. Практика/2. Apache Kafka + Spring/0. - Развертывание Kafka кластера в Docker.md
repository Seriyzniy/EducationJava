# Нода 1
## Слушатели
```yaml
KAFKA_LISTENERS:  
	MY_HOST://:9092,  
	MY_INTERNAL_HOST://:19092,   
	CONTROLLER://:9093
```

Здесь указываем какие слушатели у нас есть и на какие интерфейсы они слушаеют
- MY_HOST://:9092 - для клиентов
- MY_INTERNAL_HOST://:19092 - для других контейнеров, и kafka-ui
- CONTROLLER://:9093 - для контроллера (нужен при KRaft)

## Протоколы слушаетелей
```yaml
KAFKA_LISTENER_SECURITY_PROTOCOL_MAP:  
	MY_HOST:PLAINTEXT,  
	MY_INTERNAL_HOST:PLAINTEXT,  
	CONTROLLER:PLAINTEXT  
```

Здесь мапим протоколы для каждого слушателя

## Доступ к слушателям
```yaml
KAFKA_ADVERTISED_LISTENERS:  
	MY_HOST://localhost:29092,  
	MY_INTERNAL_HOST://kafka-bc-container-1:19092 
``` 

Здесь указываем как подключиться к определенному слушателю
(Container не указываем, так как он внутренний, и наружу его не прокидываем)

## Проброс портов
Чтобы можно было достучаться до интефрейса MY_HOST (который слушает клиентов), необходимо пробросить порт наружу и СВЯЗАТЬ с портом, который слушает MY_HOST (9092)
```yaml
ports:  
  - "29092:9092" 
```

## Указание слушателей для внутреннего взаимодействия и контроллера
```yaml
KAFKA_INTER_BROKER_LISTENER_NAME: MY_INTERNAL_HOST  
KAFKA_CONTROLLER_LISTENER_NAMES: CONTROLLER 
``` 

Здесь указываем какой слушатель используется внутри, для взаимодействия между брокерами
И указываем какой слушатель является контроллером

## Кворум контроллеров
```yaml
KAFKA_CONTROLLER_QUORUM_VOTERS: 1@kafka-bc-container-1:9093
```

Указываем ноды с ролью брокер-контроллер (или же просто контроллер)
