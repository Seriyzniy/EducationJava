# 1. Создание KafkaAdmin
## 1. Класс конфигурации Kafka Admin
Воспользуемся KafkaAdmin, из Spring Kafka
(см [[1. - Конфигурация топиков (KafkaAdmin)]])

Нам нужны свойства
- Для подключения к брокеру (см структуру работы с Kafka API)


И опишем класс, для создания топиков
```java
package admin.config;  
  
import org.apache.kafka.clients.admin.AdminClientConfig;  
import org.apache.kafka.clients.admin.NewTopic;  
import org.springframework.context.annotation.Bean;  
import org.springframework.context.annotation.Configuration;  
import org.springframework.kafka.config.TopicBuilder;  
import org.springframework.kafka.core.KafkaAdmin;  
  
import java.util.HashMap;  
import java.util.Map;  
  
@Configuration  
public class KafkaConfig {  
    @Bean  
    public KafkaAdmin kafkaAdmin() {  
        Map<String,Object> configProps = new HashMap<>();  
        configProps.put(  
            AdminClientConfig.BOOTSTRAP_SERVERS_CONFIG,  
                "localhost:29092,localhost:39092,localhost:49092");  
  
        return new KafkaAdmin(configProps);  
    }  
  
    @Bean  
    public NewTopic topic(){  
        return TopicBuilder.name("topic-spring-1")  
                .partitions(1)  
                .replicas(1)  
                .build();  
    }  
}
```

Также повесим над методами создания нужных нам классов аннотацию `@Bean`, чтобы Spring увидел эти бины и создал их при инициализации контекста
(см [[6. Java-based (Configuration, Bean)]])

## 1. Класс сериализации сообщения
Так как мы хотим передавать КАСТОМНОЕ СООБЩЕНИЕ (не просто строку, а объект), то нам нужно в параметры конфигурации Producer передать КАСТОМНЫЙ СЕРИАЛИЗАТОР
(см [[1. - Интерфейс Serializer]])

Это можно сделать при помощи интерфейса Apache Kafka - Serializer. Реализация интерфейса подставляется в
```java
props.put(  
	ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG,  
	CustomUserInfoSerializer.class  
); 
```

Теперь, обозначив важность этого класса, опишем его
```java
package util;  
  
//import com.fasterxml.jackson.core.JsonProcessingException;  
//import com.fasterxml.jackson.databind.ObjectMapper;  
import model.UserInfo;  
import org.apache.kafka.common.serialization.Serializer;  
  
public class CustomUserInfoSerializer implements Serializer<UserInfo> {  
    //private ObjectMapper mapper = new ObjectMapper();  
  
    @Override  
    public byte[] serialize(String s, UserInfo userInfo) {

    }  
}
```

## 2. Класс конфигурации Producer
Как уже знаем из [[2. - Отправка сообщений (Kafka Template)]]
то нам нужно настроить
- Конфигурационные параметры для обычного Producer
- Фабрику по созданию Producer
- KafkaTemplate

Опишем класс конфигурации всего этого
```java
package producer.config;  
  
import model.CustomHeader;  
import model.UserInfo;  
import org.apache.kafka.clients.producer.ProducerConfig;  
import org.apache.kafka.common.header.Headers;  
import org.apache.kafka.common.serialization.StringSerializer;  
import org.springframework.context.annotation.Bean;  
import org.springframework.context.annotation.Configuration;  
import org.springframework.kafka.core.DefaultKafkaProducerFactory;  
import org.springframework.kafka.core.KafkaTemplate;  
import org.springframework.kafka.core.ProducerFactory;  
import org.springframework.kafka.support.KafkaHeaderMapper;  
import org.springframework.kafka.support.converter.MessagingMessageConverter;  
import org.springframework.messaging.MessageHeaders;  
import util.CustomUserInfoSerializer;  
  
import java.util.HashMap;  
import java.util.Map;  
  
@Configuration  
public class KafkaProducerConfig {  
    private Map<String, Object> producerConfig(){  
        Map<String, Object> props = new HashMap<String, Object>();  
        props.put(  
            ProducerConfig.BOOTSTRAP_SERVERS_CONFIG,  
                "localhost:29092,localhost:39092,localhost:49092"  
        );  
        props.put(  
            ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG,  
            StringSerializer.class  
        );  
        // 🔥Кастомный сериализатор сообщения
        props.put(  
            ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG,  
            CustomUserInfoSerializer.class  
        );  
        return props;  
    }  
  
    @Bean  
    public ProducerFactory<String, UserInfo> producerFactory(){  
        return new DefaultKafkaProducerFactory<>(producerConfig());  
    }  
  
    @Bean  
    public KafkaTemplate<String, UserInfo> kafkaTemplate(  
            ProducerFactory<String, UserInfo> producerFactory){  
  
        KafkaTemplate<String, UserInfo> template = 
	        new KafkaTemplate<>(producerFactory);  
  
        MessagingMessageConverter converter = 
	        new MessagingMessageConverter(); 
	        
	    // Описание класса, 🔥для конвертации заголовков🔥 
        converter.setHeaderMapper(new KafkaHeaderMapper() {  
            @Override  
            public void fromHeaders(
	            MessageHeaders headers, 
	            Headers target) 
	        {  
                for(Map.Entry<String, Object> entry: headers.entrySet()){ 
                    target.add(
	                    entry.getKey(), 
	                    entry.getValue()
		                    .toString()
		                    .getBytes());  
                }  
            }  
  
            @Override  
            public void toHeaders(
	            Headers source, 
	            Map<String, Object> target) { }  
        });  
  
        template.setMessageConverter(converter);  
  
        return template;  
    }  
}
```

### Конвертация заголовков Spring Kafka -> Apache Kafka
Так как мы при отправке сообщения будем ОТПРАВЛЯТЬ КАСТОМНЫЙ СЛОЖНЫЙ ЗАГОЛОВОК, то там необходимо указать, **как КОНВЕРТИРОВАТЬ заголовки из MessageHeaders** (используемые в MessageBuilder) **в заголовки Headers Apache Kafka**

Для этого в Spring Kafka есть интерфейс KafkaHeaderMapper
(см [Message Headers :: Spring Kafka](https://docs.spring.io/spring-kafka/reference/kafka/headers.html))

У этого интерфейса есть два важных метода
```java
public interface KafkaHeaderMapper {
	//Spring Kafka MessageHeaders -> Apache Kafka Headers
    void fromHeaders(MessageHeaders headers, Headers target);

	//Apache Kafka Headers -> Spring Kafka MessageHeaders
    void toHeaders(Headers source, Map<String, Object> target);
}
```

Конвертор хедеров настраивается в `MessagingMessageCOnvertor`
```java
MessagingMessageConverter converter = 
	new MessagingMessageConverter();

// Описание класса, 🔥для конвертации заголовков🔥 
converter.setHeaderMapper(new KafkaHeaderMapper() {  
	@Override  
	public void fromHeaders(
		MessageHeaders headers, 
		Headers target) 
	{  
		// Перебираем все заголовки, которые добавили в MessageBuilder
		// и добавляем в Headers Apache Kafka
		for(Map.Entry<String, Object> entry: headers.entrySet()){
			target.add(
				entry.getKey(), 
				entry.getValue()
					.toString()
					.getBytes());  
		}  
	}  

	@Override  
	public void toHeaders(
		Headers source, 
		Map<String, Object> target) { }  
}); 
```

## 3. Класс для отправки данных
После конфигурации Producer, опишем класс для отправки данных с КАСТОМНЫМ заголовком
```java
package producer.service;  
  
import model.CustomHeader;  
import model.UserInfo;  
import org.springframework.kafka.core.KafkaTemplate;  
import org.springframework.kafka.support.KafkaHeaders;  
import org.springframework.messaging.support.MessageBuilder;  
import org.springframework.stereotype.Component;  
  
import java.util.UUID;  
import java.util.concurrent.TimeUnit;  
  
@Component  
public class ProducerService {  
    private final KafkaTemplate<String, UserInfo> kafkaTemplate;  
  
    public ProducerService(KafkaTemplate<String, UserInfo> kafkaTemplate) 
    {  
        this.kafkaTemplate = kafkaTemplate;  
    }  
  
    public void produceMessage(){  
        System.out.println("[PRODUCER] In produceMessage");  
  
        try {  
            kafkaTemplate.send(MessageBuilder  
                    .withPayload(new UserInfo("Tom Kruz", 10))  
                    .setHeader(KafkaHeaders.TOPIC, "topic-spring-1")  
                    .setHeader("Simple_Header", "Header 1") 
                    // 🔥КАСТОМНЫЙ сложный заголовок🔥
                    .setHeader(  
                            "Custom_Header",  
                            new CustomHeader(  
								"Inner Header",  
								UUID.randomUUID(),  
								17))  
                    .build())  
                .get(10, TimeUnit.SECONDS);  
  
            System.out.println("Message sent (sync).");  
        } catch (Exception e) {  
            e.printStackTrace();  
        }  
    }  
}
```

И поскольку, при отправке сообщения происходят дополнительные действия (см [[6. - Отправка данных, Producer (at most-least-exactly once)]])

то чтобы JVM не завершился раньше, чем сообщения действительно отправятся, БУДЕМ ЖДАТЬ ответа от брокера Kafka при помощи вызова метода get() - этот метод вызывается на CompletablFuture 
(см [[4. - Класс CompletableFuture]])

### Пояснения принципа отправки
После того, как мы запустим приложение, создадутся бины(в результате создания которых мы получим KafkaTemplate).

Затем, мы получим бин класса отправки сообщения, и явно вызовем метод отправки сообщения.

В методе отправки, мы создаем сообщение при помощи MessageBuilder, и вызываем метод send().

Но поскольку мы работаем с API Spring Kafka, а не с API Apache Kafka напрямую, то мы не можем отправить этот созданный Message.
Поэтому, Spring вызываем MessaggeConvertor, который мы указали в конфигурации KafkaTemplate.

Далее Spring увидит, что в конверторе, так же настроен KafkaHeaderMapper, и он понимает, как преобразовывать заголовки Spring Kafka, в заголовки Apache Kafka

## 4. Класс запуска приложения
```java
package producer;  
  
import org.springframework.context.ApplicationContext;  
import org.springframework.context.annotation.AnnotationConfigApplicationContext;  
import org.springframework.context.annotation.ComponentScan;  
import org.springframework.context.annotation.Configuration;  
import producer.service.ProducerService;  
  
@Configuration  
@ComponentScan({"producer.config", "producer.service"})  
public class KafkaProducerApp {  
    public static void main(String[] args) {  
        ApplicationContext context =  
                new AnnotationConfigApplicationContext(KafkaProducerApp.class);  
  
        ProducerService producerService = 
	        (ProducerService)context.getBean("producerService"); 
	         
        producerService.produceMessage();  
    }  
}
```

Так как мы не используем MVC, то дернуть метод на отправку сообщения после запуска приложения мы не можем.

Поэтому, мы описали класс для отправки данных при помощи аннотации Component, чтобы класс запуска приложения увидел его и создал бин.

Этот бин мы получаем из контекста и вызываем метод
