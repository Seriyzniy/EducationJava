
> [!NOTE] `Deserializer<T>`
> Интерфейс `Deserializer<T>` используется для ==преобразования массива байтов (byte[]) обратно в Java-объект==.  
> 
> Это противоположность `Serializer<T>`, и он необходим **потребителю (Kafka Consumer)** для чтения данных из топика Kafka.
> 
> Kafka получает из брокера только байты — и именно `Deserializer` отвечает за их перевод в объекты типа `T`.
> 
> Реализует интерфейс **`java.io.Closeable`**, что позволяет закрывать ресурсы после использования.

## Интерфейс
```java
package org.apache.kafka.common.serialization;

import java.util.Map;

public interface Deserializer<T> extends Closeable {

    void configure(Map<String, ?> configs, boolean isKey);

    T deserialize(String topic, byte[] data);

    default T deserialize(
	    String topic, 
	    Headers headers, 
	    byte[] data) 
	{
        return deserialize(topic, data);
    }

    default void close() {}
}
```

## Методы
### Настройка
```java
// Configure this class.
void configure(Map<String, ?> configs, boolean isKey);
```
**Перевод:**  
Настраивает экземпляр десериализатора.  
В параметре `isKey` указывается, используется ли десериализатор для ключа (`true`) или для значения (`false`).

### Десериализация
```java
// Преобразует массив байтов в объект типа `T`.  
// Если `data == null`, метод обычно должен вернуть `null`.
T deserialize(String topic, byte[] data);

// То же самое, что и предыдущий метод, но с учётом заголовков Kafka (`Headers`).  
// Можно использовать, например, для версионирования сообщений или кастомной логики по типу контента.
default T deserialize(
	String topic, 
	Headers headers, 
	byte[] data) 
{
    return deserialize(topic, data);
}
```

## 💡 Особенности и подводные камни 💡

1️⃣ **Потокобезопасность** — `Deserializer` может вызываться из разных потоков, поэтому ==реализация должна быть потокобезопасной==.  

2️⃣ ❗❗❗**null-handling** — Kafka может передать `null` в `data`, если сообщение было записано с `null`-значением, поэтому нужно корректно это обрабатывать.  

3️⃣ ❗❗❗**Headers** — большинство встроенных десериализаторов игнорируют `Headers`, но при кастомной логике это может быть важно (например, если разные форматы данных в одном топике).  

4️⃣ ❗❗❗**Неверная кодировка** — при десериализации строк (StringDeserializer) стоит явно указывать кодировку (UTF-8 и т.п.), чтобы избежать проблем с locale.  

5️⃣ 💡**Ошибка парсинга** — некорректный формат данных может вызвать `SerializationException`, лучше ловить и логировать ошибки.

---
## 💎 Best Practices 💎

✅ ⭐⭐Используйте встроенные десериализаторы (`StringDeserializer`, `IntegerDeserializer`, `LongDeserializer`, и т.п.) — они уже оптимизированы.  

✅ ❗❗При кастомной десериализации обрабатывайте `null`-значения явно.  

✅ 💡Если вы используете формат JSON/Avro/Protobuf — создавайте отдельный класс-десериализатор, не смешивайте логику в Consumer. 

✅ Реализуйте `close()` и освобождайте ресурсы (например, `ObjectMapper` или `SchemaRegistryClient`).  

✅ Используйте версию `deserialize(topic, headers, data)`, если ваши данные включают метаинформацию в заголовках.

## Пример 
Пример простого десериализатора, который преобразует JSON → объект `User` через Jackson:
```java
import com.fasterxml.jackson.databind.ObjectMapper;
import org.apache.kafka.common.serialization.Deserializer;

import java.util.Map;

public class UserDeserializer implements Deserializer<User> {
    private final ObjectMapper objectMapper = new ObjectMapper();

    @Override
    public void configure(
	    Map<String, ?> configs, 
	    boolean isKey) 
	{
        // можно использовать конфигурации, если нужно
    }

    @Override
    public User deserialize(String topic, byte[] data) {
        if (data == null) return null;
        try {
            return objectMapper.readValue(data, User.class);
        } catch (Exception e) {
            throw new RuntimeException(
	            "Ошибка десериализации", 
	            e
	        );
        }
    }

    @Override
    public void close() {
        // закрытие ресурсов при необходимости
    }
}
```

Использование:
```java
Properties props = new Properties();
props.put(
	ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG, 
	StringDeserializer.class);

// ⭐Кастномный ДЕСИРАЛИЗАТОР	
props.put(
	ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG, 
	UserDeserializer.class);
```
