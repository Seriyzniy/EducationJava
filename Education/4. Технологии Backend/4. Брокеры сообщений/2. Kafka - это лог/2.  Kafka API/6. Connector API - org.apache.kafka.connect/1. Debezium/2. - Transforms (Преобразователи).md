Коннекторы можно настраивать с помощью преобразователей для упрощения изменений на уровне отдельных сообщений. 

Debezium предоставляет несколько [преобразований отдельных сообщений](https://kafka.apache.org/documentation/#connect_transforms) (SMT - single message transformations), которые **можно использовать ДЛЯ ИЗМЕНЕНИЯ ЗАПИСЕЙ ПЕРЕД ИХ ОТПРАВКОЙ в ​​Apache Kafka** (применяя их к коннекторам Debezium) или при их считывании из Kafka коннектором-приемником.

### ⚠️ Важные моменты
- ❗❗❗SMT применяются **после чтения данных коннектором, до сериализации**.❗❗❗
    
- Если используешь несколько SMT, порядок их важен: они применяются слева направо (`transforms=a,b,c`).
    
- В Kafka Connect **конфигурация JSON просто сериализует результат SMT**, так что без SMT поле `id` останется в JSON.

## Таблица SMT (Single Message Transformations) Debezium

|**Трансформация**|**Описание (перевод)**|
|---|---|
|**Topic Routing**|Перенаправляет записи в другие топики на основе регулярного выражения, применённого к исходному имени топика.|
|**Content-Based Routing**|Перенаправляет выбранные события в другие топики на основе содержимого события.|
|**New Record State Extraction**|Извлекает плоскую структуру имён полей и значений из событий изменений Debezium. Это упрощает обработку для sink-коннекторов, которые не умеют работать со сложной структурой событий Debezium.|
|**MongoDB New Document State Extraction**|Специальный для MongoDB аналог New Record State Extraction SMT.|
|**Outbox Event Router**|Предоставляет способ безопасного и надёжного обмена данными между несколькими (микро)сервисами.|
|**MongoDB Outbox Event Router**|Версия Outbox Event Router SMT для MongoDB.|
|**Message Filtering**|Применяет фильтр к событиям изменений, которые генерируют коннекторы — на основе содержимого события. Позволяет передавать дальше только те записи, которые действительно нужны.|
|**HeaderToValue**|Перемещает или копирует заголовки сообщения (headers) в значение записи (value).|
|**Partition Routing**|Перенаправляет записи в определённые партиции на основе выбранных полей полезной нагрузки (payload fields).|
|**Timezone Converter**|Конвертирует поля временных меток Debezium и Kafka Connect в событиях в указанную временную зону.|
|**TimescaleDB Integration**|Маршрутизирует и обогащает сообщения, которые PostgreSQL-коннектор Debezium получает от TimescaleDB.|
|**Convert CloudEvents to Saveable Form**|Преобразует значения записей, десериализованных Debezium CloudEvents converter, в структуру, подходящую для Debezium JDBC sink-коннектора.|
|**Vitess Use Local VGTID**|Уменьшает размер VGTID, которые генерирует коннектор Vitess. Сохраняется только VGTID для шарда, в котором произошли изменения; остальные VGTID удаляются. Используется только с коннектором Debezium для Vitess.|
|**Vitess Remove Field**|Специальная SMT для Vitess: удаляет определённые поля для уменьшения размера записей (например, избыточные поля VGTID).|
|**Vitess Filter Transaction Topic Records**|Фильтрует все сообщения, которые коннектор Vitess записывает в трансакционный топик (если включена передача трансакционных метаданных). Используется только с Debezium Vitess.|
