**SMT ConvertCloudEventToSaveableForm** используется совместно с `CloudEventsConverter` и `JdbcSinkConnector`.  
Его задача — **преобразовать CloudEvent в форму, подходящую для сохранения в базе данных**.

==Преобразование выполняется на основе отображения (mapping) ==между полями CloudEvent и столбцами целевой таблицы в БД:  
**указывает, какое поле CloudEvent соответствует какому столбцу**.

SMT создаёт новую структуру и новую схему, содержащую поля с именами столбцов целевой таблицы и значениями из исходного CloudEvent.  
Если CloudEvent содержит поле `data` в виде структуры, оно «сплющивается», то есть преобразуется в строку JSON.

**Чтобы выполнить преобразование, SMT должен знать тип сериализатора**, используемый для сериализации и десериализации CloudEvents (`json` или `avro`).  

> Параметр `serializer.type` должен иметь то же значение, что и `value.converter.serializer.type`.

### **Пример**
Чтобы преобразовать CloudEvent в форму, подходящую для `JdbcSinkConnector`, **необходимо настроить SMT** `ConvertCloudEventToSaveableForm` **в конфигурации коннектора** Kafka Connect.

Например, чтобы преобразовать CloudEvents, сериализованные и десериализованные с помощью JSON, при условии, что в целевой таблице есть столбцы `id`, `source`, `type` и `payload`, используется следующая конфигурация:
```json
"connector.class": "io.debezium.connector.jdbc.JdbcSinkConnector",
...
"transforms": "convertCloudEvent",

"transforms.convertCloudEvent.type": "io.debezium.connector.jdbc.transforms.ConvertCloudEventToSaveableForm",

"transforms.convertCloudEvent.fields.mapping": "id,source,type,data:payload",

"transforms.convertCloudEvent.serializer.type": "json",

"value.converter": "io.debezium.converters.CloudEventsConverter",
"value.converter.serializer.type": "json",
"value.converter.data.serializer.type": "json"
...
```

### **Пример 1. Результат применения SMT ConvertCloudEventToSaveableForm**
#### **Значение _до_ обработки SMT:**
```json
{
	"id": "624e6565-99ee-4fdb-9228-653138c3a7b3",
	"source": "/debezium/postgresql/book",
	"specversion": "1.0",
	"type": "BookCreated",
	"time": "2023-11-11T07:11:01.825Z",
	"datacontenttype": "application/json",
	"data": {
		"id": 4,
		"name": "1984",
		"publicationYear": 1949
	}
}
```

Значение _после_ обработки SMT:
```json
{
	"id": "624e6565-99ee-4fdb-9228-653138c3a7b3",
	"source": "/debezium/postgresql/book",
	"type": "BookCreated",
	"payload": "{\"id\": 4, \"name\": \"1984\", \"publicationYear\": 1949}"
}
```

# Конфигурационные параметры
## **fields.mapping**
**Описание (перевод):**  
Список пар, разделённых запятыми, где каждая пара содержит имя поля CloudEvent и имя соответствующего столбца целевой таблицы.

**Тип данных:** list  
**Значение по умолчанию:** отсутствует  
**Валидные значения:** непустой список  
**Важность:** high

---
## **serializer.type**
**Описание (перевод):**  
Тип сериализатора, который используется для сериализации и десериализации CloudEvents. Должен совпадать со значением `value.converter.serializer.type`.

**Тип данных:** string  
**Значение по умолчанию:** отсутствует  
**Валидные значения:** `json`, `avro`  
**Важность:** high

---
## **schema.cloudevents.name**
**Описание (перевод):**  
Имя схемы CloudEvents, под которым схема зарегистрирована в Schema Registry.

**Тип данных:** string  
**Значение по умолчанию:** отсутствует  
**Валидные значения:** —  
**Важность:** low
