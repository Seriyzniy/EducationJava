## 1. Основные параметры коннектора
Эти параметры задают базовую конфигурацию самого коннектора.

### name
Уникальное имя коннектора. Повторная регистрация с тем же именем завершится ошибкой.  
**Значение по умолчанию**: Нет значения по умолчанию

### connector.class
Имя Java-класса коннектора. Для PostgreSQL всегда использовать `io.debezium.connector.postgresql.PostgresConnector`.  
**Значение по умолчанию**: Нет значения по умолчанию

### tasks.max
Максимальное количество задач. PostgreSQL коннектор всегда использует одну задачу.  
**Значение по умолчанию**: 1

### topic.prefix
Префикс топика Kafka для данного сервера или кластера PostgreSQL. Должен быть уникальным и содержать только буквы, цифры, дефисы, точки и подчёркивания.  
**Значение по умолчанию**: Нет значения по умолчанию

## 2. Параметры подключения к PostgreSQL
Настройки подключения к базе данных и параметры SSL.

### database.hostname
IP-адрес или имя хоста сервера PostgreSQL.  
**Значение по умолчанию**: Нет значения по умолчанию

### database.port
Порт PostgreSQL сервера.  
**Значение по умолчанию**: 5432

### database.user
Имя пользователя PostgreSQL для подключения.  
**Значение по умолчанию**: Нет значения по умолчанию

### database.password
Пароль пользователя PostgreSQL.  
**Значение по умолчанию**: Нет значения по умолчанию

### database.dbname
Имя базы данных для потоковой передачи изменений.  
**Значение по умолчанию**: Нет значения по умолчанию

### database.sslmode
Использование SSL: `disable`, `allow`, `prefer`, `require`, `verify-ca`, `verify-full`.  
**Значение по умолчанию**: prefer

### database.sslcert
Путь к SSL-сертификату клиента.  
**Значение по умолчанию**: Нет значения по умолчанию

### database.sslkey
Путь к приватному ключу клиента.  
**Значение по умолчанию**: Нет значения по умолчанию

### database.sslpassword
Пароль для приватного ключа.  
**Значение по умолчанию**: Нет значения по умолчанию

### database.sslrootcert
Путь к корневому сертификату для проверки сервера.  
**Значение по умолчанию**: Нет значения по умолчанию

### database.tcpKeepAlive
Включение TCP keep-alive для проверки соединения.  
**Значение по умолчанию**: true

## 3. Параметры логической декодировки
Настройки слота, плагина и публикации для потоковой передачи изменений.

### [plugin.name](https://plugin.name/)
Имя плагина логической декодировки PostgreSQL (`decoderbufs` или `pgoutput`).  
**Значение по умолчанию**: decoderbufs

### [slot.name](https://slot.name/)
Имя логического репликационного слота.  
**Значение по умолчанию**: debezium

### slot.drop.on.stop
Удалять ли слот при корректной остановке коннектора.  
**Значение по умолчанию**: false

### [publication.name](https://publication.name/)
Имя публикации PostgreSQL для `pgoutput`.  
**Значение по умолчанию**: dbz_publication

### publication.autocreate.mode
Режим автосоздания публикации: `all_tables`, `disabled`, `filtered`, `no_tables`.  
**Значение по умолчанию**: all_tables

## 6. Настройки формата данных и типов
Управление представлением типов данных, точностью и обработкой специальных типов.

**Значение по умолчанию**: adaptive

### decimal.handling.mode
Обработка DECIMAL и NUMERIC: `precise`, `double`, `string`.  
**Значение по умолчанию**: precise

### hstore.handling.mode
Обработка HSTORE колонок: `map` или `json`.  
**Значение по умолчанию**: json

### interval.handling.mode
Обработка интервалов: `numeric` или `string`.  
**Значение по умолчанию**: numeric

### binary.handling.mode
Представление бинарных колонок: `bytes`, `base64`, `base64-url-safe`, `hex`.  
**Значение по умолчанию**: bytes

### money.fraction.digits
Количество десятичных знаков для типа money при использовании `BigDecimal`.  
**Значение по умолчанию**: 2
