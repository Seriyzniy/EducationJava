
> [!NOTE] Класс `KafkaConsumer<K, V>` 
> Основной компонент для **чтения сообщений из топиков Kafka**.  
> 
> Он реализует интерфейс `Consumer<K, V>`, предоставляя полную функциональность: подписку, получение, коммиты, паузы и управление смещениями.
> 

## Конструктор
```java
// Создаёт нового потребителя Kafka на основе переданных свойств.
// Свойства конфигурации указываются через java.util.Properties и включают:
// - 💡bootstrap.servers — список брокеров Kafka;
// - 💡key.deserializer / 💡value.deserializer — классы десериализаторов ключа и значения;
// - 💡group.id — идентификатор группы потребителей;
// - enable.auto.commit — автофиксация смещений;
// - auto.offset.reset — стратегия чтения ("earliest" / "latest").
// K и V — типы ключа и значения сообщений.
public KafkaConsumer(Properties properties)


// Создаёт потребителя с указанными десериализаторами ключа и значения.
// Позволяет передавать конфигурацию и десериализаторы напрямую.
public KafkaConsumer(
	Properties properties, 
	Deserializer<K> keyDeserializer, 
	Deserializer<V> valueDeserializer)
	

// Более общий конструктор, принимающий Map конфигурации вместо Properties.
public KafkaConsumer(Map<String, Object> configs)


// Конструктор с явными десериализаторами и Map конфигурации.
// Наиболее гибкий вариант.
public KafkaConsumer(
	Map<String, Object> configs, 
	Deserializer<K> keyDeserializer, 
	Deserializer<V> valueDeserializer)
```

## Методы
Класс **наследует все методы интерфейса `Consumer`**, а также реализует дополнительные служебные и закрывающие методы.

### Подписка и отписка
```java
// Подписывает потребителя на список топиков.
// После вызова Kafka сама назначает партиции этому потребителю.
public void subscribe(Collection<String> topics)

// Подписывает потребителя на топики, имена которых соответствуют заданному регулярному выражению.
// Например, можно подписаться на все топики, начинающиеся с "logs-".
public void subscribe(Pattern pattern, ConsumerRebalanceListener listener)

// Отписывает потребителя от всех топиков.
public void unsubscribe()
```

💬**Пояснение:**  
Используется для динамического назначения партиций.  
Если вызывается `subscribe()`, Kafka автоматически распределяет партиции между участниками группы.

### Получение сообщений
```java
// Получает данные из топиков или партиций, заданных ранее через subscribe() или assign().
// timeout — максимальное время ожидания данных в миллисекундах.
public ConsumerRecords<K, V> poll(long timeout)


// Современный вариант poll, использующий java.time.Duration вместо миллисекунд.
// Возвращает пакет записей, полученных за это время.
public ConsumerRecords<K, V> poll(Duration timeout)
```

💬**Пояснение:**  
Главный метод чтения сообщений.  
Вызывается в бесконечном цикле.  
Если данных нет — поток блокируется до истечения таймаута.  
Все вызовы `poll()` должны выполняться **в одном потоке**.

### Пауза и возобновление чтения
```java
// Приостанавливает получение данных из указанных партиций.
public void pause(Collection<TopicPartition> partitions)

// Возобновляет получение данных из ранее приостановленных партиций.
public void resume(Collection<TopicPartition> partitions)

// Возвращает список партиций, которые сейчас находятся в состоянии паузы.
public Set<TopicPartition> paused()
```
💬**Пояснение**:
Позволяет временно «заморозить» обработку некоторых партиций, не изменяя подписку.  
Например, если система перегружена и нужно дать ей «отдохнуть».

### Управление смещениями и позициями
```java
// Синхронно фиксирует смещения, возвращённые последним вызовом poll().
// Блокирует поток до подтверждения фиксации брокером.
public void commitSync()

// Синхронно фиксирует указанные смещения.
public void commitSync(Map<TopicPartition, OffsetAndMetadata> offsets)


// Асинхронно фиксирует смещения, возвращённые последним poll().
// Не блокирует поток, но не гарантирует доставку при сбое.
public void commitAsync()

// Асинхронно фиксирует смещения с передачей callback-функции.
// Callback вызывается при успешной или неудачной фиксации.
public void commitAsync(OffsetCommitCallback callback)


// Перемещает указатель чтения для заданной партиции на конкретное смещение.
public void seek(TopicPartition partition, long offset)

// Перемещает указатель чтения на начало всех указанных партиций.
public void seekToBeginning(Collection<TopicPartition> partitions)

// Перемещает указатель чтения на конец всех указанных партиций.
public void seekToEnd(Collection<TopicPartition> partitions)

```
💬**Пояснение:**
- `commitSync()` гарантирует запись в `__consumer_offsets`, но может замедлить цикл обработки.
    
- `commitAsync()` — быстрее, но возможна потеря фиксации при сбое.
    
- Обычно используют `commitAsync()` в цикле и `commitSync()` при завершении приложения.

### Ручное управление партициями
```java
// Назначает конкретные партиции вручную.
// Используется вместо subscribe(), если не требуется балансировка.
public void assign(Collection<TopicPartition> partitions)

// Возвращает набор партиций, назначенных этому потребителю.
public Set<TopicPartition> assignment()
```
💬**Пояснение:**  
Используется, если приложение само решает, какие партиции обрабатывать.  
Полезно при ручном контроле потоков или распределении нагрузки.

### Получение метаданных
```java
/**
 * Возвращает множество топиков, на которые в данный момент подписан потребитель.
 * Для ручного назначения партиций (assign()) возвращает пустое множество.
 */
public Set<String> subscription()

/**
 * Возвращает метаданные партиций для указанного топика.
 * Включает информацию о лидере, репликах, ISR для каждой партиции.
 * Может выполняться без подписки на топик.
 */
public List<PartitionInfo> partitionsFor(String topic)

/**
 * Возвращает метаданные всех топиков в кластере.
 * Включает внутренние топики (например, __consumer_offsets).
 * Для выполнения требуется подключение к брокерам.
 */
public Map<String, List<PartitionInfo>> listTopics()

/**
 * Возвращает метрики производительности потребителя.
 * Метрики обновляются в реальном времени (потребление, латенси, ошибки).
 */
public Map<MetricName, ? extends Metric> metrics()

/**
 * Возвращает последние зафиксированные смещения для указанных партиций.
 * Если смещение не найдено, возвращает null для соответствующей партиции.
 * Требует подключения к брокерам для запроса информации из __consumer_offsets.
 */
public Map<TopicPartition, OffsetAndMetadata> committed(Set<TopicPartition> partitions)
```

### Завершение работы
```java
// Закрывает потребителя, дожидаясь завершения всех операций с таймаутом по умолчанию.
public void close()

// Закрывает потребителя, дожидаясь завершения в течение указанного времени.
public void close(Duration timeout)
```

---
## 💡 Особенности и подводные камни 💡

1️⃣ 💡`KafkaConsumer` **не потокобезопасен** — вызывать методы можно только из одного потока.  

2️⃣ 💡`poll()` должен вызываться регулярно (по умолчанию не реже, чем каждые 5 минут), иначе координатор исключит клиента из группы.  

3️⃣ `commitAsync()` не гарантирует запись в `__consumer_offsets`, поэтому при сбое возможна потеря фиксации.  

4️⃣ `enable.auto.commit=true` может привести к **повторной обработке** или **пропуску сообщений**.  

5️⃣ `assign()` и `subscribe()` — взаимоисключающие режимы (нельзя использовать одновременно). 

6️⃣ Закрытие (`close()`) обязательно, иначе heartbeat-поток (главный поток) продолжит работать, и сессия «повиснет».

## 💎 Best Practices 💎

✅ Используй `poll(Duration.ofMillis(100))` в цикле без долгих блокировок.  

✅ Делай `commitAsync()` в цикле и `commitSync()` перед завершением приложения.  

✅ Отключай автофиксацию смещений (`enable.auto.commit=false`).  

✅ 💡Всегда закрывай consumer в блоке `finally` или через try-with-resources.  

✅ 💡Логику обработки сообщений выполняй в отдельном потоке, чтобы не блокировать polling.

```java
Properties props = new Properties();
props.put("bootstrap.servers", "localhost:9092");
props.put("group.id", "test-group");
props.put("enable.auto.commit", "false");
props.put("key.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");
props.put("value.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");

try (KafkaConsumer<String, String> consumer = new KafkaConsumer<>(props)
) {
    consumer.subscribe(List.of("my-topic"));
    
    while (true) {
        ConsumerRecords<String, String> records = 
	        consumer.poll(Duration.ofMillis(100));
	        
        for (ConsumerRecord<String, String> record : records) {
            System.out.printf(
                "topic=%s, partition=%d,"
                "offset=%d, key=%s, value=%s%n",
                record.topic(), 
                record.partition(), 
                record.offset(), 
                record.key(), 
                record.value()
            );
        }
        consumer.commitAsync();
    }
}
```
