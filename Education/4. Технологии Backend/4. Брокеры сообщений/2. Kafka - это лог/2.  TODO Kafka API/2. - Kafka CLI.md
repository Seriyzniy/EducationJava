## Управление кластером и метаданными

### Утилита `kafka-server-start.sh`
==Запуск сервера== Kafka в режиме брокера или контроллера KRaft
```bash
kafka-server-start.sh [config-file]
```

- `config-file` - путь к файлу конфигурации сервера Kafka
    

### Утилита `kafka-server-stop.sh`
==Остановка работающего сервера== Kafka с возможностью выбора по роли или node-id
```bash
kafka-server-stop.sh [--process-role] [--node-id]
```
- `--process-role` - остановка процессов по роли (broker/controller)
    
- `--node-id` - остановка конкретного узла по ID

### Утилита `kafka-storage.sh`
==Управление хранилищем== Kafka - генерация UUID кластера и форматирование хранилища
```bash
kafka-storage.sh format -t [cluster-id] -c [config]
kafka-storage.sh random-uuid
```

- `format` - форматирование хранилища
    
- `random-uuid` - генерация UUID кластера
    
- `-t` - идентификатор кластера
    
- `-c` - файл конфигурации
    

### Утилита `kafka-cluster.sh`
Получение ==идентификатора кластера== Kafka

```bash
kafka-cluster.sh cluster-id --bootstrap-server [server]
```
- `cluster-id` - получение идентификатора кластера
    
- `--bootstrap-server` - адрес брокера для подключения
    

### Утилита `kafka-metadata-quorum.sh`
Запрос состояния кворума метаданных для кластеров в режиме KRaft

```bash
kafka-metadata-quorum.sh --bootstrap-server [server] describe --status
```

- `--bootstrap-server` - адрес брокера для подключения
    
- `describe` - описание состояния кворума
    
- `--status` - вывод статуса кворума

## Управление топиками и партициями

### Утилита `kafka-topics.sh`
Создание, удаление, описание и ==изменение топиков Kafka==

```bash
kafka-topics.sh --bootstrap-server [server] 
	--topic [topic] 
	--create 
	--partitions [N] 
	--replication-factor [N]

kafka-topics.sh --list --bootstrap-server [server]

kafka-topics.sh --describe --bootstrap-server [server] --topic [topic]

kafka-topics.sh --alter --bootstrap-server [server] --topic [topic] --partitions [N]
```
- `--create` - создание топика
    
- `--list` - список всех топиков
    
- `--describe` - детальная информация о топике
    
- `--alter` - изменение топика
    
- `--delete` - удаление топика
    
- `--topic` - название топика
    
- `--partitions` - количество партиций
    
- `--replication-factor` - фактор репликации
    
- `--config` - дополнительные настройки топика
    
- `--under-replicated-partitions` - показ отстающих партиций
    
- `--unavailable-partitions` - показ недоступных партиций
    

### Утилита `kafka-configs.sh`
Изменение и просмотр конфигурации топиков, брокеров, пользователей и контроллеров

```bash
kafka-configs.sh --bootstrap-server [server] 
	--entity-type [type] 
	--entity-name [name] 
	--alter 
	--add-config [config]

kafka-configs.sh --describe --bootstrap-server [server] 
	--entity-type [type]
```
- `--alter` - изменение конфигурации
    
- `--describe` - просмотр конфигурации
    
- `--add-config` - добавление конфигурации
    
- `--delete-config` - удаление конфигурации
    
- `--entity-type` - тип сущности (topics/brokers/users)
    
- `--entity-name` - имя сущности
    
- `--entity-default` - настройки по умолчанию
    
- `--bootstrap-controller` - подключение к контроллеру KRaft
    

### Утилита `kafka-reassign-partitions.sh`
Перераспределение партиций между брокерами для балансировки нагрузки

```bash
kafka-reassign-partitions.sh 
	--bootstrap-server [server] 
	--reassignment-json-file [file] 
	--execute
	
kafka-reassign-partitions.sh 
	--verify 
	--bootstrap-server [server] 
	--reassignment-json-file [file]
```
- `--reassignment-json-file` - JSON файл с планом перераспределения
    
- `--execute` - выполнение перераспределения
    
- `--verify` - проверка статуса перераспределения
    
- `--generate` - генерация плана перераспределения
    

### Утилита `kafka-get-offsets.sh`
Получение оффсетов топика для определенного времени

```bash
kafka-get-offsets.sh 
	--bootstrap-server [server] 
	--topic [topic] 
	--time [timestamp]
```
- `--topic` - название топика
    
- `--time` - временная метка (-1=latest, -2=earliest)

## Работа с сообщениями

### Утилита `kafka-console-producer.sh`
Консольный производитель для отправки сообщений в топики Kafka

```bash
kafka-console-producer.sh 
	--bootstrap-server [server] 
	--topic [topic] 
	--property [key=value]
```

- `--topic` - топик для отправки сообщений
    
- `--property` - дополнительные свойства
    
- `--key-separator` - разделитель ключа и значения
    
- `--producer.config` - файл конфигурации продюсера
    

### Утилита `kafka-console-consumer.sh`
Консольный потребитель для чтения сообщений из топиков Kafka

```bash
kafka-console-consumer.sh 
	--bootstrap-server [server] 
	--topic [topic] 
	--from-beginning 
	--group [group]
```

- `--topic` - топик для чтения
    
- `--from-beginning` - чтение с начала топика
    
- `--group` - группа потребителей
    
- `--property` - дополнительные свойства
    
- `--formatter` - форматтер вывода сообщений
    
- `--offset` - чтение с определенного оффсета

## Управление потребителями и группами

### Утилита `kafka-consumer-groups.sh`
Управление группами потребителей - просмотр, сброс оффсетов, мониторинг отставаний

```bash
kafka-consumer-groups.sh 
	--list 
	--bootstrap-server [server]
	
kafka-consumer-groups.sh 
	--describe 
	--bootstrap-server [server] 
	--group [group]

kafka-consumer-groups.sh 
	--reset-offsets 
	--bootstrap-server [server] 
	--group [group] 
	--topic [topic] 
	--to-earliest
```
- `--list` - список всех групп потребителей
    
- `--describe` - детальная информация о группе
    
- `--reset-offsets` - сброс оффсетов
    
- `--group` - название группы
    
- `--to-earliest` - сброс к началу топика
    
- `--to-latest` - сброс к концу топика
    
- `--to-datetime` - сброс к определенной дате
    
- `--shift-by` - сдвиг оффсетов на N сообщений
    
- `--members` - информация о членах группы
    
- `--state` - состояние группы


### Утилита `kafka-groups.sh`
Просмотр всех типов групп в кластере Kafka

```bash
kafka-groups.sh --bootstrap-server [server] --list
```
- `--list` - список всех типов групп

kafka-broker-api-versions.sh

kafka-log-dirs.sh --bootstrap-server [server] --describe

## Безопасность

### Утилита `kafka-acls.sh`
Управление Access Control Lists (ACL) для контроля доступа к ресурсам Kafka

```bash
kafka-acls.sh 
	--bootstrap-server [server] 
	--add 
	--allow-principal [principal] 
	--operation [operation] 
	--topic [topic]
```
- `--add` - добавление ACL
    
- `--remove` - удаление ACL
    
- `--list` - список ACL
    
- `--allow-principal` - разрешенный принципал
    
- `--deny-principal` - запрещенный принципал
    
- `--operation` - операция (Read/Write/Create и т.д.)
    
- `--topic` - топик для применения ACL

## Тестирование производительности

### Утилита `kafka-producer-perf-test.sh`
Тестирование производительности продюсера - измерение пропускной способности и задержек

```bash
kafka-producer-perf-test.sh 
	--topic [topic] 
	--num-records [N] 
	--record-size [bytes] 
	--throughput [N] 
	--producer-props [props]
```
- `--topic` - топик для тестирования
    
- `--num-records` - количество сообщений
    
- `--record-size` - размер сообщения в байтах
    
- `--throughput` - ограничение пропускной способности
    
- `--producer-props` - свойства продюсера
    

### Утилита `kafka-consumer-perf-test.sh`
Тестирование производительности потребителя - измерение скорости потребления сообщений

```bash
kafka-consumer-perf-test.sh 
	--bootstrap-server [server] 
	--topic [topic] 
	--messages [N]
```
- `--topic` - топик для тестирования
    
- `--messages` - количество сообщений для потребления
    

### Утилита `kafka-e2e-latency.sh`
Измерение сквозной задержки (end-to-end latency) между отправкой и получением сообщений

```bash
kafka-e2e-latency.sh 
	[broker-list] 
	[topic] 
	[num-messages] 
	[producer-acks] 
	[message-size-bytes]
```
- `broker-list` - список брокеров
    
- `topic` - топик для тестирования
    
- `num-messages` - количество сообщений
    
- `producer-acks` - настройка acks продюсера
    
- `message-size-bytes` - размер сообщения в байтах

## Управление Connect

### Утилита `connect-distributed.sh`
Запуск Kafka Connect в распределенном режиме для высокой доступности

```bash
connect-distributed.sh [config-file]
```
- `config-file` - файл конфигурации Connect
    

### Утилита `connect-standalone.sh`
Запуск Kafka Connect в standalone режиме для разработки и тестирования

```bash
connect-standalone.sh 
	[worker-config] 
	[connector-configs...]
```
- `worker-config` - конфигурация воркера
    
- `connector-configs` - конфигурации коннекторов
