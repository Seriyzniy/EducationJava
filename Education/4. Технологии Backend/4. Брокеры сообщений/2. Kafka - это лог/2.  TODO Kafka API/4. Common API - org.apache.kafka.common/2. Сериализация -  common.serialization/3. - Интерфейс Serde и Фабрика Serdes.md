
# 1. Интерфейс Serde
> [!NOTE] `Serde<T>`
> **объединяющий интерфейс**, который ==связывает== `Serializer<T>` и `Deserializer<T>` в одну сущность.  
> 
> 🧠Название происходит от слов **SER**ializer + **DE**serializer.
> 
> Он используется в первую очередь в **Kafka Streams API**, чтобы упростить работу с сериализацией и десериализацией ключей и значений в потоковых топологиях.
> 
> Благодаря `Serde` Kafka Streams знает, **как преобразовывать** данные из Java-объектов в байты и обратно — при записи в топики и чтении из них.

## Интерфейс
```java
package org.apache.kafka.common.serialization;

import java.io.Closeable;
import java.util.Map;

public interface Serde<T> extends Closeable {

    void configure(Map<String, ?> configs, boolean isKey);

    Serializer<T> serializer();

    Deserializer<T> deserializer();

    @Override
    void close();
}
```

## Методы
### Настройка
```java
// Configure this class.
void configure(Map<String, ?> configs, boolean isKey);
```

**Перевод:**  
Настраивает `Serde`, включая его `Serializer` и `Deserializer`.
Параметр `isKey` указывает, используется ли `Serde` для ключа (`true`) или для значения (`false`).

### Получение сериализатора и десериализатора
```java
// Return the serializer for the specified type.
Serializer<T> serializer();

// Return the deserializer for the specified type.
Deserializer<T> deserializer();
```
**Перевод:**  
Предоставляют доступ к экземплярам сериализатора и десериализатора, связанных с данным `Serde`.  
Обычно Kafka Streams использует эти методы автоматически под капотом.

---
## Особенности и подводные камни 💡

1️⃣ **Используется в Kafka Streams** — ==напрямую редко используется в обычных Producer/Consumer==, но критически важен для Streams API.  

2️⃣ **Оборачивает оба компонента** — единая точка управления для `Serializer` и `Deserializer`. 

3️⃣ ❗❗**Thread-safety** — Kafka Streams может использовать один `Serde` из разных потоков, поэтому реализация должна быть потокобезопасной.  

4️⃣ ❗❗**Настройка через `configure()`** — важно передавать одинаковые параметры сериализатору и десериализатору, если требуется согласованность.  

5️⃣ 💡**Можно создавать обёртки (WrapperSerde)** — например, комбинировать разные сериализаторы и десериализаторы вручную.

---
## 💎 Best Practices 💎

✅ ⭐⭐⭐Используй встроенные `Serdes` (из класса `Serdes`) для стандартных типов (`String`, `Long`, `Integer`, `Double`, `UUID` и т.п.)  

✅ При работе с кастомными типами — создай собственный `Serde`, объединив `Serializer` и `Deserializer`.  

✅ Корректно реализуй `close()` — освобождай ресурсы обоих компонентов.  

✅ Передавай `Serde` в Streams API, а не отдельные сериализаторы — Kafka Streams сам выберет нужную часть (key/value).  

✅ ⭐⭐⭐Для JSON, Avro, Protobuf и других форматов — используй специализированные библиотеки (например, Confluent `SpecificAvroSerde`).

---
## Пример использования
Пример собственного `Serde` для класса `User`:
```java
import org.apache.kafka.common.serialization.Serde;
import org.apache.kafka.common.serialization.Serializer;
import org.apache.kafka.common.serialization.Deserializer;

import java.util.Map;

public class UserSerde implements Serde<User> {
    private final Serializer<User> serializer = new UserSerializer();
    private final Deserializer<User> deserializer = new UserDeserializer();

    @Override
    public void configure(Map<String, ?> configs, boolean isKey) {
        serializer.configure(configs, isKey);
        deserializer.configure(configs, isKey);
    }

    @Override
    public Serializer<User> serializer() {
        return serializer;
    }

    @Override
    public Deserializer<User> deserializer() {
        return deserializer;
    }

    @Override
    public void close() {
        serializer.close();
        deserializer.close();
    }
}
```

Использование:
```java
StreamsBuilder builder = new StreamsBuilder();
KStream<String, User> stream = builder.stream(
	"input-topic",
	Consumed.with(
		Serdes.String(), 
		new UserSerde()
	)
);
```

# 2. Класс `Serdes`


> [!NOTE] `Serdes` — это **фабрика стандартных `Serde`** для самых распространённых типов данных.  
> 
> Он избавляет от необходимости вручную создавать пары сериализатор/десериализатор.
> 

Почти все реализации `Serde` в этом классе являются **статическими вложенными классами** (например, `Serdes.StringSerde`, `Serdes.LongSerde`, `Serdes.ByteArraySerde` и т.д.).

```java
package org.apache.kafka.common.serialization;

public class Serdes {
    public static Serde<Void> Void() { ... }
    public static Serde<String> String() { ... }
    public static Serde<Short> Short() { ... }
    public static Serde<Integer> Integer() { ... }
    public static Serde<Long> Long() { ... }
    public static Serde<Float> Float() { ... }
    public static Serde<Double> Double() { ... }
    public static Serde<byte[]> ByteArray() { ... }
    public static Serde<ByteBuffer> ByteBuffer() { ... }
    public static Serde<Bytes> Bytes() { ... }
    public static Serde<UUID> UUID() { ... }
    public static <T> Serde<List<T>> List(Serde<T> serde) { ... }
}
```
