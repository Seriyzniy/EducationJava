# Смысл **пакета**
Ты **почти никогда не создаёшь**:
- `Cluster`, `Node`, `PartitionInfo`, `TopicCollection`, `ClusterResource` — всё это Kafka сама создаёт, обновляет и хранит в `MetadataCache`.
    

Ты **иногда создаёшь вручную**:
- `TopicPartition` (при ручном назначении партиций);
    
- `KafkaFuture` (если работаешь с AdminClient);
    
- `MetricName` (если подключаешь свои метрики).


Это **“общий” (common) слой** библиотеки Kafka —  
всё, что используется и продюсером, и консюмером, и админ-клиентом, и даже серверной частью брокера.

Kafka устроена модульно:
```bash
org.apache.kafka.clients
 ├── producer
 ├── consumer
 ├── admin
 └── common  ← базовые общие модели и типы
```

То есть все клиенты “сидят” поверх `org.apache.kafka.common`  
Когда ты вызываешь методы вроде:
```java
producer.send(record);
consumer.poll();
adminClient.describeTopics();
```
всё, что происходит дальше — это:
1. Формируется `TopicPartition`, `Cluster`, `Node`, `PartitionInfo` и др.
    
2. Они используются для построения сетевых запросов (`MetadataRequest`, `ProduceRequest`, `FetchRequest`).
    
3. Результаты этих запросов возвращаются в виде объектов из `org.apache.kafka.common`.

# 1.⭐Основные классы⭐ и интерфейсы (ядро пакета)

| Имя класса/интерфейса                                          | Описание                                                                                                                                  |
| -------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| **Cluster** ✅                                                  | Неизменяемое представление части узлов, топиков и партиций в кластере Kafka. Используется клиентами (продюсер, консьюмер) для метаданных. |
| **Node**<br>(то же самое что и CLuster - работает под капотом) | Представляет брокер Kafka (узел кластера), включая его ID, хост, порт и прочие параметры.                                                 |
| **PartitionInfo**                                              | Описывает состояние отдельной партиции — лидер, реплики, ISR и т.п.                                                                       |
| **TopicPartition**✅                                            | Комбинация имени топика и номера партиции. Ключевая структура для маршрутизации сообщений.                                                |
| **TopicPartitionInfo**                                         | Содержит информацию о лидерстве, репликах и ISR для конкретной партиции.                                                                  |
| **ClusterResource**                                            | Инкапсулирует метаданные о кластере Kafka, включая идентификатор кластера.                                                                |
| **ClusterResourceListener**                                    | Интерфейс обратного вызова для получения уведомлений об изменениях метаданных кластера.                                                   |

# 2.Конфигурационные и управляемые интерфейсы

|Имя класса/интерфейса|Описание|
|---|---|
|**Configurable**|Интерфейс для классов, создаваемых через рефлексию и нуждающихся в параметрах конфигурации.|
|**Reconfigurable**|Интерфейс для компонентов, поддерживающих **динамическую переконфигурацию** без перезапуска.|

# 3. Метаданные топиков и партиций

| Имя класса/интерфейса                   | Описание                                                                           |
| --------------------------------------- | ---------------------------------------------------------------------------------- |
| **TopicCollection**                     | Представляет коллекцию топиков, используемую в административных операциях Kafka.   |
| **TopicCollection.TopicIdCollection**   | Коллекция топиков, определённых по их **topic ID** (UUID).                         |
| **TopicCollection.TopicNameCollection** | Коллекция топиков, определённых по их **имени**.                                   |
| **TopicIdPartition**🟢                  | Сочетание **topic ID** и **partition number** — уникальный идентификатор партиции. |
| **TopicPartitionReplica**               | Описывает реплику партиции — топик, номер партиции и ID брокера.                   |

---

# 4. Управление группами и состояниями

|Имя класса/интерфейса|Описание|
|---|---|
|**ClassicGroupState**|Состояния классических групп потребителей (до введения новых типов групп).|
|**GroupState**|Отражает текущее состояние группы (active, dead, preparing rebalance и т.д.).|
|**GroupType**|Определяет тип группы (например, классическая, consumer group, coordinator и т.п.).|
|**ConsumerGroupState**|**Устаревший класс**, описывающий состояние группы потребителей. (Deprecated since 4.0)|

# 8. Асинхронность и Futures

| Имя класса/интерфейса           | Описание                                                                                         |
| ------------------------------- | ------------------------------------------------------------------------------------------------ |
| `KafkaFuture<T>`✅               | Гибкий future-объект, поддерживающий **цепочки вызовов и асинхронные шаблоны программирования**. |
| `KafkaFuture.BaseFunction<A,B>` | Функция, принимающая объекты типа A и возвращающая объект типа B.                                |
| **KafkaFuture.BiConsumer<A,B>** | Потребитель двух объектов разных типов, используемый в асинхронных колбэках.                     |
