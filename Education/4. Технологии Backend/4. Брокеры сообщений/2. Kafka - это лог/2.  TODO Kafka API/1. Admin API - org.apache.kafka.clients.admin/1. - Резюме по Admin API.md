**Kafka Admin API** — это часть _Kafka Clients API_, предназначенная для **программного управления кластерами Kafka**.  

Она позволяет не только читать метаданные, но и **создавать, изменять и удалять топики, ACL, конфигурации брокеров и т. д.**

> [!NOTE] AdminClient - main класс
> Главный класс — **`AdminClient`** 
> (фабричный метод `Admin.create(Properties)`).

## **Работа с топиками**
(Это наиболее часто используемая часть)

|Операция|Метод|Описание|
|---|---|---|
|Создание топика|`createTopics()`|Создаёт новые топики с заданными параметрами.|
|Удаление топика|`deleteTopics()`|Удаляет указанные топики.|
|Изменение числа реплик / партиций|`createPartitions()`|Увеличивает количество партиций (уменьшить нельзя).|
|Получение списка топиков|`listTopics()`|Возвращает список всех топиков (включая внутренние, если указано).|
|Получение метаданных топика|`describeTopics()`|Возвращает информацию о партициях, лидерах и репликах.|

### Пример: создать топик
```java
try (Admin admin = Admin.create(Map.of("bootstrap.servers", "localhost:9092"))) {
    NewTopic topic = new NewTopic("my-topic", 3, (short) 1);
    admin.createTopics(List.of(topic)).all().get();
}
```

### Пример: получение информации о топике
```java
admin.describeTopics(List.of("my-topic"))
     .all()
     .get()
     .forEach((name, desc) -> System.out.println(desc.partitions()));
```

## Конфигурации брокеров и топиков
Можно читать и изменять **динамические конфиги**, не перезапуская брокеры.

|Операция|Метод|Описание|
|---|---|---|
|Чтение конфигурации|`describeConfigs()`|Получает текущие параметры для `ConfigResource` (топика или брокера).|
|Изменение конфигурации|`alterConfigs()` или `incrementalAlterConfigs()`|Изменяет конфигурации топиков/брокеров.|

### Пример: чтение конфигурации топика
```java
ConfigResource resource = new 
	ConfigResource(ConfigResource.Type.TOPIC, "my-topic");
	
Map<ConfigResource, Config> configs = 
	admin
		.describeConfigs(List.of(resource))
        .all()
        .get();
        
configs.forEach((res, conf) -> 
	conf.entries().forEach(System.out::println));
```

### Пример: изменение конфигурации топика
```java
ConfigResource resource = new 
	ConfigResource(ConfigResource.Type.TOPIC, "my-topic");
	
ConfigEntry entry = new ConfigEntry("cleanup.policy", "compact");
Config config = new Config(List.of(entry));

admin.alterConfigs(Map.of(resource, config)).all().get();
```

## Управление ACL (доступом)
|Операция|Метод|Описание|
|---|---|---|
|Создание ACL|`createAcls()`|Добавляет новые ACL правила.|
|Удаление ACL|`deleteAcls()`|Удаляет правила.|
|Просмотр ACL|`describeAcls()`|Выводит список текущих ACL.|

## Метаданные и состояние кластера

|Операция|Метод|Описание|
|---|---|---|
|Получить информацию о брокерах|`describeCluster()`|Возвращает контроллера, ID брокеров, их адреса.|
|Проверить доступные узлы|`describeCluster().nodes()`|Список брокеров в кластере.|
|Получить ID контроллера|`describeCluster().controller()`|Кто сейчас является контроллером.|

### Пример:
```java
DescribeClusterResult cluster = admin.describeCluster();

System.out.println("Cluster ID: " + cluster.clusterId().get());
System.out.println("Controller: " + cluster.controller().get());

cluster.nodes().get().forEach(System.out::println);
```

## Работа с Consumer Group
|Операция|Метод|Описание|
|---|---|---|
|Получить список групп|`listConsumerGroups()`|Возвращает список всех Consumer Groups.|
|Получить информацию по группе|`describeConsumerGroups()`|Состав участников, назначенные партиции и статус.|
|Удалить группу|`deleteConsumerGroups()`|Удаляет указанную группу.|
|Получить оффсеты группы|`listConsumerGroupOffsets()`|Отображает текущие оффсеты партиций.|
|Изменить оффсеты группы|`alterConsumerGroupOffsets()`|Ручное смещение оффсетов.|
