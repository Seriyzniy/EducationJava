Kafka включает ==пять основных API==:
- ✅**Producer API** — позволяет приложениям отправлять потоки данных в топики кластера Kafka.
    
- ✅**Consumer API** — позволяет приложениям читать потоки данных из топиков кластера Kafka.
    
- **Streams API** — позволяет преобразовывать потоки данных из входных топиков в выходные.
    
- **Connect API** — используется для создания коннекторов, которые постоянно загружают данные из **внешних систем** в Kafka или выгружают их из Kafka в другие системы.
    
- ✅**Admin API** — предоставляет средства для управления и мониторинга топиков, брокеров и других объектов Kafka.

Kafka предоставляет все свои возможности через **языко-независимый протокол**, поэтому существуют клиенты на множестве языков программирования.  
Однако официально поддерживаются только **Java-клиенты** — остальные реализованы как независимые open-source проекты.


## 🥇 **КРИТИЧЕСКИ ВАЖНЫЕ** (используются постоянно)

| Пакет                                    | Для чего нужен                                               |
| ---------------------------------------- | ------------------------------------------------------------ |
| `org.apache.kafka.clients.admin` ✅       | **Управление кластером** - создание топиков, настройка ACL   |
| `org.apache.kafka.clients.consumer`✅     | **Потребление сообщений** - основной Consumer API            |
| `org.apache.kafka.clients.producer`✅     | **Отправка сообщений** - основной Producer API               |
| `org.apache.kafka.common`✅               | **Общие утилиты** - базовые классы, исключения               |
| `org.apache.kafka.common.serialization`✅ | **Сериализация** - String, Integer, JSON, Avro сериализаторы |
