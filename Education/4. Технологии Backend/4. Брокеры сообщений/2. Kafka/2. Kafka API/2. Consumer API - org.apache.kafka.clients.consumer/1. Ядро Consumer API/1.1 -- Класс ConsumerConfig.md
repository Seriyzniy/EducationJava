
> [!NOTE] `ConsumerConfig`
> это **класс-конфигурация потребителя (consumer)** из пакета `org.apache.kafka.clients.consumer`.  
> 
> Он **наследуется от `AbstractConfig`** и предоставляет **набор стандартных ключей** и **методы для валидации, получения и документирования** конфигурационных параметров.

⭐⭐Используется при создании `KafkaConsumer` (обычно через `Properties`).  

Каждый параметр (например, `bootstrap.servers`, `group.id`, `enable.auto.commit`) имеет **строковый ключ**, значение и тип (String, int, boolean и т. д.).

## Конструктор
```java
// Создаёт новый экземпляр ConsumerConfig, используя предоставленные настройки.
// Параметры:
// - props — свойства (ключ-значение), определяющие параметры конфигурации потребителя
public ConsumerConfig(Map<?, ?> props)
```

## Методы
Класс наследует методы от `AbstractConfig`, но также определяет **множество статических констант** (названия параметров конфигурации) и **методы доступа**.

### Константы конфигурации
```java
// ❗❗Список адресов Kafka-брокеров❗❗, к которым подключается потребитель
public static final String BOOTSTRAP_SERVERS_CONFIG = "bootstrap.servers";

// Уникальный идентификатор группы потребителей
public static final String GROUP_ID_CONFIG = "group.id";

// Включает или выключает ❗❗автокоммит смещений❗❗
public static final String ENABLE_AUTO_COMMIT_CONFIG = "enable.auto.commit";

// ❗❗Интервал между автокоммитами (если включён)❗❗
public static final String AUTO_COMMIT_INTERVAL_MS_CONFIG = "auto.commit.interval.ms";

// Указывает, что делать при отсутствии смещения в Kafka (earliest / latest / none)
public static final String AUTO_OFFSET_RESET_CONFIG = "auto.offset.reset";

// ❗❗Таймаут ожидания получения данных в poll()❗❗
public static final String FETCH_MAX_WAIT_MS_CONFIG = "fetch.max.wait.ms";

// ❗❗Максимальный размер данных,❗❗ которые потребитель может получить за один запрос
public static final String FETCH_MAX_BYTES_CONFIG = "fetch.max.bytes";

// Интервал heartbeat-сообщений координатору группы
public static final String HEARTBEAT_INTERVAL_MS_CONFIG = "heartbeat.interval.ms";

// ❗❗Таймаут сессии для детектирования "мертвого" потребителя❗❗
public static final String SESSION_TIMEOUT_MS_CONFIG = "session.timeout.ms";

// Максимальное время обработки сообщения до ребаланса
public static final String MAX_POLL_INTERVAL_MS_CONFIG = "max.poll.interval.ms";

// ❗❗Максимальное количество записей за один вызов poll()❗❗
public static final String MAX_POLL_RECORDS_CONFIG = "max.poll.records";

// Идентификатор клиента (для логов и метрик)
public static final String CLIENT_ID_CONFIG = "client.id";

// ❗❗Настройки сериализаторов❗❗
public static final String KEY_DESERIALIZER_CLASS_CONFIG = "key.deserializer";
public static final String VALUE_DESERIALIZER_CLASS_CONFIG = "value.deserializer";
```
💬Эти константы — **ключи**, которые указываются в `Properties`, передаваемых в `KafkaConsumer`.

### Методы доступа и утилиты
```java
// Возвращает объект с описанием всех доступных конфигураций
public static ConfigDef configDef()

// Возвращает ConfigDef — структуру, описывающую все параметры ConsumerConfig (имя, тип, значение по умолчанию)
public static ConfigDef getConfigDef()

// Возвращает документацию по каждому параметру конфигурации (используется в CLI и тестах)
public static Map<String, Object> postProcessParsedConfig(Map<String, Object> parsedValues)
```

### Методы родителя (AbstractConfig)
(наследуются, но часто используются напрямую через ConsumerConfig)
```java
// Возвращает значение конфигурации по ключу
public Object get(String key)

// Возвращает строковое значение по ключу
public String getString(String key)

// Возвращает логическое значение по ключу
public boolean getBoolean(String key)

// Возвращает целочисленное значение по ключу
public int getInt(String key)
```

## 💡 Особенности и подводные камни 💡

1️⃣ `ConsumerConfig` — **immutable** после создания, менять параметры “на лету” нельзя.  

2️⃣ 💡Не все параметры валидируются на этапе создания — некоторые ошибки проявятся при подключении.💡

3️⃣ 💡Параметры `session.timeout.ms`, `heartbeat.interval.ms` и `max.poll.interval.ms` **влияют на стабильность группы**💡 — неправильные значения могут привести к ребалансам. 

4️⃣ `enable.auto.commit` = true по умолчанию, что может вызвать потерю сообщений при неправильной обработке.  

5️⃣ 💡Параметр `auto.offset.reset` управляет, с какого места начинать чтение, если offset не найден. 
💡Ошибка выбора (`latest` вместо `earliest`) может привести к “пропущенным” сообщениям.💡

---

## 💎 Best Practices 💎

✅ 💡**Явно указывай `group.id`**, иначе Consumer не войдёт в группу.  

✅ 🧠Используй `enable.auto.commit=false` и управляй коммитами вручную при критичной логике.  

✅ 💡Поддерживай разумное значение `max.poll.records` (обычно 100–500) для равномерной нагрузки.

✅ Для отладки логов используй `client.id` — он отображается в брокере.  

✅ Не ставь слишком маленький `session.timeout.ms`, иначе потребитель будет часто выпадать из группы.

## Пример
```java
import org.apache.kafka.clients.consumer.ConsumerConfig;
import org.apache.kafka.clients.consumer.KafkaConsumer;

import java.util.Properties;

public class ConsumerExample {
    public static void main(String[] args) {
        Properties props = new Properties();

        // Адреса брокеров
        props.put(
	        ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, 
	        "localhost:9092");

        // Уникальная группа потребителей
        props.put(
	        ConsumerConfig.GROUP_ID_CONFIG, 
	        "order-processor-group");

        // Отключаем автокоммит, будем фиксировать вручную
        props.put(
	        ConsumerConfig.ENABLE_AUTO_COMMIT_CONFIG, 
	        "false");

        // Начинать с самого начала, если offset отсутствует
        props.put(
	        ConsumerConfig.AUTO_OFFSET_RESET_CONFIG, 
	        "earliest");

        // Десериализация ключа и значения
        props.put(
	        ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG, 
	"org.apache.kafka.common.serialization.StringDeserializer");
	
  props.put(ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG, "org.apache.kafka.common.serialization.StringDeserializer");


        KafkaConsumer<String, String> consumer = new KafkaConsumer<>(props);
    }
}
```
