# 📕 1. Класс **ConsumerRecord<K, V>**


> [!NOTE] `ConsumerRecord` — это **единичная запись**, полученная потребителем из Kafka.  
> Каждая запись содержит данные о **топике**, **разделе (partition)**, **смещении (offset)**, **временной метке**, **ключе** и **значении сообщения**.  
> 
> Используется как базовый элемент внутри `ConsumerRecords`.
> 

## Конструктор
```java
// Создаёт новый объект ConsumerRecord, представляющий одно сообщение из Kafka.
// Параметры:
// - topic — имя топика, из которого получено сообщение
// - partition — номер раздела топика
// - offset — смещение (позиция) сообщения в разделе

// - timestamp — временная метка (может быть временем создания или логгирования)
// - timestampType — тип временной метки (CREATE_TIME или LOG_APPEND_TIME)

// - checksum — контрольная сумма CRC32 (устаревшее поле, начиная с Kafka 0.11)

// - serializedKeySize — размер сериализованного ключа в байтах
// - serializedValueSize — размер сериализованного значения в байтах

// - key — ключ сообщения
// - value — значение сообщения
public ConsumerRecord(String topic,
                      int partition,
                      long offset,
                      long timestamp,
                      TimestampType timestampType,
                      long checksum,
                      int serializedKeySize,
                      int serializedValueSize,
                      K key,
                      V value)
```

## Методы

### Доступ к метаданным сообщения
```java
// Возвращает имя топика, из которого пришло сообщение
public String topic()

// Возвращает номер раздела (partition)
public int partition()

// Возвращает смещение (offset) в разделе
public long offset()

// Возвращает временную метку сообщения (в миллисекундах)
public long timestamp()

// Возвращает тип временной метки (например, CREATE_TIME)
public TimestampType timestampType()

// Возвращает контрольную сумму (CRC32)
// ⚠️ Устаревшее поле, не используется в новых версиях Kafka
public long checksum()
```

### Работа с данными (ключ и значение)
```java
// Возвращает ключ сообщения (может быть null)
public K key()

// Возвращает значение сообщения (payload)
public V value()
```

### Размеры сериализованных данных
```java
// Возвращает размер сериализованного ключа в байтах
public int serializedKeySize()

// Возвращает размер сериализованного значения в байтах
public int serializedValueSize()
```

---
## 💡 Особенности и подводные камни 💡

1️⃣ ❗❗❗Класс **иммутабельный** — все поля `final`. Это делает его безопасным для многопоточности.  

2️⃣ 💡Поле `checksum` больше не используется, оставлено для обратной совместимости.  

3️⃣ 💡Поле `timestamp` зависит от настроек продюсера (`CreateTime` или `LogAppendTime`).  

4️⃣ Если сериализация не указана, `key()` и `value()` могут вернуть `null`.  

5️⃣ Методы `topic()`, `partition()`, `offset()` часто используются в логировании и метриках.

---
## 💎 Best Practices 💎

✅ 💡Используй `topic()` и `partition()` для трассировки проблем в конкретном разделе.  

✅ Не изменяй объект `ConsumerRecord` — он должен оставаться неизменным.  

✅ При парсинге сообщений — проверяй `key()` и `value()` на `null`.  

✅ В логах указывай `(topic, partition, offset)` для точного дебага.

---
## Пример
```java
ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(1000));

for (ConsumerRecord<String, String> record : records) {
    System.out.println("Key: " + record.key());
    System.out.println("Value: " + record.value());
    System.out.println("Topic: " + record.topic());
    System.out.println("Partition: " + record.partition());
    System.out.println("Offset: " + record.offset());
    System.out.println("Timestamp: " + new Date(record.timestamp()));
    System.out.println("---");
}
```

# 📕2. Класс ConsumerRecords

> [!NOTE] `ConsumerRecords`
> это **коллекция записей (`ConsumerRecord`)**, возвращаемая методом `KafkaConsumer.poll()`
> 
> Содержит все сообщения, полученные за один вызов `poll()`
> 
> Обеспечивает итерацию, группировку по топикам и разделам.
> 

## Конструктор
```java
// Создаёт объект ConsumerRecords, содержащий набор сообщений, сгруппированных по топику и разделу.
// Параметр:
// - records — карта, где ключ — TopicPartition, а значение — список ConsumerRecord для него
public ConsumerRecords(
	Map<TopicPartition, List<ConsumerRecord<K, V>>> records)
```

## Методы

### Общие методы доступа
```java
// Возвращает все записи как неизменяемую Map: ключ — TopicPartition, значение — список записей
public Map<TopicPartition, List<ConsumerRecord<K, V>>> records()

// 💡Возвращает список всех записей для указанного топика💡
public List<ConsumerRecord<K, V>> records(String topic)

// Возвращает список всех записей для конкретного раздела
public List<ConsumerRecord<K, V>> records(
	TopicPartition partition)
```

### Итерация и количество
```java
// Возвращает количество всех записей (во всех разделах)
public int count()

// Проверяет, есть ли вообще записи (true, если пусто)
public boolean isEmpty()

// Возвращает итератор по всем ConsumerRecord<K,V>
public Iterator<ConsumerRecord<K, V>> iterator()
```

---
## 💡 Особенности и подводные камни 💡

1️⃣ 💡Класс **НЕ потокобезопасен**, его нельзя изменять после создания. 

2️⃣ ❗❗❗Итерация по всем записям идёт в порядке их получения в `poll()`.  

3️⃣ ⭐⭐⭐Часто записи группируются по `TopicPartition`, что важно при ручном коммите offset'ов.  

4️⃣ `records(String topic)` возвращает все разделы данного топика.  

5️⃣ 💡Используется как временный контейнер — обычно живёт в пределах одного цикла `poll()`.

---
## 💎 Best Practices 💎

✅ ❗❗❗Всегда обрабатывай **все записи** из `ConsumerRecords` до следующего `poll()`.  

✅ Не храни объект `ConsumerRecords` дольше, чем длится цикл обработки — Kafka удаляет старые смещения.

✅ Для ручного коммита используй `records.last()` для получения последнего offset. 

✅ При парсинге сообщений проверяй, не пустой ли `ConsumerRecords`.

---
## Пример: Обработка по партициям
```java
ConsumerRecords<String, String> records = 
	consumer.poll(Duration.ofMillis(1000));

//Перебераем сообщения по партициям 
for (TopicPartition partition : records.partitions()) {
    List<ConsumerRecord<String, String>> partitionRecords = 
	    records.records(partition);
    
    System.out.println("Processing partition: " + partition);
    for (ConsumerRecord<String, String> record : partitionRecords) {
        System.out.println("  Offset " + record.offset() + ": " + record.value());
    }
}
```
