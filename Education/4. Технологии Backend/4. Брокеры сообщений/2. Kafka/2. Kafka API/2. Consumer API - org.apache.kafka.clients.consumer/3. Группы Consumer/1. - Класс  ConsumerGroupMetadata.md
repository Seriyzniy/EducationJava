
> [!NOTE] `ConsumerGroupMetadata` 
> это **класс-обёртка** из пакета `org.apache.kafka.clients.consumer`, 
> 
> представляющий **метаданные о группе потребителей**.
> 

Он хранит идентификатор группы, поколение (generation), идентификатор участника (member id) и тип присоединения (protocol type). 

Эти данные нужны для:
- внутренней идентификации consumer-а в группе,
    
- передачи метаданных при **commitSync(Map<TopicPartition, OffsetAndMetadata>, ConsumerGroupMetadata)**,
    
- точного согласования смещений между несколькими экземплярами consumer-ов в одной группе.
    

> Впервые класс появился в Kafka 2.5, чтобы сделать commit-операции **более согласованными** и позволить использовать **расширенные протоколы ребалансировки**.

## Конструктор
```java
// Создаёт метаданные для группы потребителей с указанным идентификатором.
// Используется, когда достаточно знать только имя группы.
public ConsumerGroupMetadata(String groupId)

// Создаёт полные метаданные о группе потребителей.
// Параметры:
// - groupId — уникальный идентификатор группы потребителей
// - generationId — номер поколения (generation), увеличивается при каждом ребалансе
// - memberId — уникальный идентификатор участника внутри группы
// - groupInstanceId — опциональный постоянный идентификатор участника (для static membership)
// - groupMetadataVersion — версия метаданных протокола (для совместимости)
public ConsumerGroupMetadata(
	String groupId, 
	int generationId, 
	String memberId, 
	Optional<String> groupInstanceId, 
	String groupMetadataVersion)
```

## Методы

### Методы доступа
```java
// Возвращает идентификатор группы потребителей.
public String groupId()

// Возвращает идентификатор участника (member.id), назначаемый координатором.
public String memberId()

// Возвращает уникальный идентификатор экземпляра участника (если используется static membership).
public Optional<String> groupInstanceId()

// Возвращает номер поколения (generation), который увеличивается при каждом ребалансе.
public int generationId()

// Возвращает тип протокола группы (обычно "consumer").
public String protocolType()
```

---
## Особенности и подводные камни 💡

1️⃣ ❗❗❗**Используется** ==при ручных коммитах==.  
Если ты вызываешь `commitSync(offsets, groupMetadata)`, то Kafka фиксирует оффсеты именно **от имени этой группы**.

2️⃣ **Позволяет "привязать" commit к конкретному поколению.**
Это предотвращает запись устаревших оффсетов от предыдущих consumer-ов (предыдущего поколения).

3️⃣ **Static membership** (постоянный идентификатор участника) снижает частоту ребалансов.  
При наличии `groupInstanceId`, Kafka может распознать "вернувшегося" участника.

4️⃣ **generationId** помогает обнаруживать "устаревших" участников.  
Если consumer отстал, координатор может отклонить commit с неправильным поколением.

5️⃣ **Используется внутри Kafka internals.**  
Большинство разработчиков взаимодействуют с ним только при advanced-настройках commit’ов.

---
## 💎 Best Practices 💎

✅ Используй `ConsumerGroupMetadata`, если реализуешь **собственную обработку commit’ов** или интеграцию с внешними системами.  

✅ При статических членствах (`group.instance.id`) передавай значение в метаданных, чтобы избежать ребалансов.  

✅ ❗❗❗При логировании сохраняй `groupId`, `memberId` и `generationId` — это сильно помогает при отладке ребалансировок.

✅ 💡Не создавай `ConsumerGroupMetadata` вручную, если не уверен — его можно получить напрямую из consumer-а (`consumer.groupMetadata()`).

## Пример
```java
import org.apache.kafka.clients.consumer.*;
import org.apache.kafka.common.*;
import java.time.Duration;
import java.util.*;

public class GroupMetadataExample {
    public static void main(String[] args) {
        Properties props = new Properties();
        props.put(
	        ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, 
	        "localhost:9092");
        props.put(
	        ConsumerConfig.GROUP_ID_CONFIG, 
	        "order-group");
        props.put(
	        ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG, 
	        StringDeserializer.class.getName());
        props.put(
	        ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG, 
		    StringDeserializer.class.getName());
        props.put(
	        ConsumerConfig.ENABLE_AUTO_COMMIT_CONFIG, 
	        "false");


        try (KafkaConsumer<String, String> consumer = new KafkaConsumer<>(props)
        ) {
		    consumer.subscribe(
			    Collections.singletonList("orders"));
            
            while (true) {
                ConsumerRecords<String, String> records = 
	                consumer.poll(Duration.ofMillis(500));
	                
                for (ConsumerRecord<String, String> record : 
	                records
	            ) {
                    System.out.printf(
	                    "Получено: topic=%s offset=%d key=%s"+ 
	                    "value=%s%n",
                        record.topic(), 
                        record.offset(), 
                        record.key(), 
                        record.value());
                }

                // Получаем метаданные текущей группы
                ConsumerGroupMetadata metadata = 
	                consumer.groupMetadata();
                
                System.out.println(
	                "Текущие метаданные группы: " + metadata);

                // Фиксируем оффсеты с явным указанием метаданных группы
                consumer.commitSync(
	                Collections.emptyMap(), 
	                metadata);
            }
        }
    }
}

```
