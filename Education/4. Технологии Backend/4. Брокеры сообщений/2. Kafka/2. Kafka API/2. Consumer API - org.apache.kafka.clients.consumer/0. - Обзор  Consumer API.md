# 📕1. Основные
## **Основные классы и интерфейсы (ядро API Kafka Consumer)**

| Интерфейс        | Описание                                                                                                                    |
| ---------------- | --------------------------------------------------------------------------------------------------------------------------- |
| `Consumer<K,V>`✅ | Определяет API для чтения сообщений из Kafka, управления подписками, смещениями, партициями и жизненным циклом потребителя. |


| Класс / Интерфейс       | Описание                                                                                                                                                  |
| ----------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `KafkaConsumer<K,V>`✅   | Главный клиент для чтения сообщений из Kafka. Поддерживает подписку на темы, чтение записей, фиксацию смещений и управление жизненным циклом потребителя. |
| `ConsumerRecord<K,V>`✅  | Представляет одно сообщение, полученное из Kafka. Содержит ключ, значение, смещение, партицию, топик и временную метку.                                   |
| `ConsumerRecords<K,V>`✅ | Контейнер для набора `ConsumerRecord`, возвращаемого методом `poll()`. Позволяет итерироваться по всем сообщениям или по партициям.                       |
| `ConsumerConfig`✅       | Содержит ключи и описание всех параметров конфигурации потребителя. Используется при создании `KafkaConsumer`.                                            |
| `OffsetAndMetadata`✅    | Хранит смещение и дополнительную метаинформацию при ручной фиксации (commit) оффсета.                                                                     |
| `OffsetAndTimestamp`✅   | Контейнер, возвращаемый при поиске смещений по времени (`offsetsForTimes`). Содержит смещение и соответствующий timestamp.                                |

# 📕2. Дополнительные

## Интерфейсы обратных вызовов (callbacks, listeners)

| Интерфейс                    | Описание                                                                                                                              |
| ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| `ConsumerRebalanceListener`✅ | Вызывается при перераспределении партиций между потребителями группы. Позволяет корректно обработать ревокацию и назначение партиций. |
| `OffsetCommitCallback`✅      | Колбэк, вызываемый после завершения асинхронного коммита смещений (`commitAsync`). Позволяет обработать ошибки коммита.               |
| `ConsumerInterceptor<K, V>`  | Позволяет перехватывать и модифицировать записи, получаемые потребителем (например, для логирования или метрик).                      |

## Распределение партиций (Partition Assignment)

| Класс / Интерфейс                                       | Описание                                                                                         |
| ------------------------------------------------------- | ------------------------------------------------------------------------------------------------ |
| `RangeAssignor`                                         | Реализация assignor, распределяющая партиции "по диапазонам" в пределах каждого топика.          |
| `RoundRobinAssignor`                                    | Реализация assignor, равномерно распределяющая партиции по всем потребителям.                    |
| `StickyAssignor`                                        | Ставит целью минимизировать перемещение партиций между потребителями при ребалансировке.         |
| `CooperativeStickyAssignor`                             | Улучшенная версия StickyAssignor, поддерживающая "мягкие" (cooperative) ревоки партиций.         |
|                                                         |                                                                                                  |
| `ConsumerPartitionAssignor`<br>(ниже внутренние классы) | Интерфейс для реализации пользовательских алгоритмов распределения партиций между потребителями. |
| Внутренний класс<br>`.Assignment`                       | Представляет назначенные партиции конкретному потребителю.                                       |
| Внутренний класс<br>`.GroupAssignment`                  | Определяет итоговое распределение партиций между всеми участниками группы.                       |
| Внутренний класс<br>`.GroupSubscription`                | Содержит подписки всех участников группы.                                                        |
| Внутренний класс<br>`RebalanceProtocol`                 | Определяет протокол ребалансировки (EAGER или COOPERATIVE).                                      |
| Внутренний класс<br>`.Subscription`                     | Хранит подписку одного потребителя (на какие темы и с какими параметрами).                       |

## Метаданные и управление группами

| Класс                                       | Описание                                                                               |
| ------------------------------------------- | -------------------------------------------------------------------------------------- |
| `ConsumerGroupMetadata`✅                    | Содержит метаданные группы потребителей (ID группы, generationId, memberId).           |
| `GroupProtocol`                             | Представляет используемый протокол взаимодействия группы потребителей с координатором. |
| **`CloseOptions`**                          | Опции закрытия `KafkaConsumer` (например, поведение при выходе из группы).             |
| **`CloseOptions.GroupMembershipOperation`** | Enum, определяющий, нужно ли сохранять членство в группе при закрытии.                 |

## ⚠️ Исключения

|Класс|Описание|
|---|---|
|**`CommitFailedException`**|Возникает, если фиксация смещения (`commitSync`) не удалась по невосстанавливаемой причине.|
|**`RetriableCommitFailedException`**|Ошибка фиксации смещений, которую можно повторить.|
|**`NoOffsetForPartitionException`**|Нет сохранённого смещения для партиции, и политика сброса оффсета (`auto.offset.reset`) не определена.|
|**`OffsetOutOfRangeException`**|Смещение вышло за допустимые границы (слишком старое или слишком новое).|
|**`InvalidOffsetException`**|Указано недопустимое смещение.|
|**`LogTruncationException`**|Происходит при "грязной" смене лидера, когда часть лога была усечена и данные потеряны.|

## Дополнительные классы (реже используемые)

|Класс|Описание|
|---|---|
|**`AcknowledgementCommitCallback`**|Интерфейс колбэка для обработки подтверждения (acknowledge) сообщений.|
|**`AcknowledgeType`**|Enum, определяющий тип подтверждения для `KafkaShareConsumer` (успешно/ошибка).|
|**`KafkaShareConsumer<K, V>`**|Клиент для чтения сообщений с использованием **shared consumer groups** (новая модель в Kafka 4.0+).|
|**`ShareConsumer<K, V>`**|Альтернативный API потребителя, поддерживающий совместное чтение сообщений несколькими экземплярами.|
|**`SubscriptionPattern`**|Представляет регулярное выражение (Google RE2/J), используемое при подписке на темы по шаблону.|
