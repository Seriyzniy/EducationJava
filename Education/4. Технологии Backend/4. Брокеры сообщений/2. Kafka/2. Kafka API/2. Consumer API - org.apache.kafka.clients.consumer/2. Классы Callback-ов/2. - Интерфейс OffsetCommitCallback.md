
> [!NOTE] `OffsetCommitCallback`
> то **интерфейс** из пакета  `org.apache.kafka.clients.consumer`,
> 
> используемый для обработки **результатов асинхронного коммита оффсетов** (смещений).

Когда ты вызываешь:
```java
consumer.commitAsync(callback);
```

Kafka отправляет commit-запрос координатору группы в фоне.  

Метод возвращается сразу, **не блокируя поток**,  
а когда commit завершается (успешно или с ошибкой), Kafka вызывает **колбэк** — реализацию `OffsetCommitCallback`.

## Создание интерфейса
Для его использования **нужно создать класс, реализующий интерфейс**, ИЛИ воспользоваться **лямбда-выражением**.
```java
OffsetCommitCallback callback = (offsets, exception) -> {
    if (exception != null) {
        System.err.println(
	        "Ошибка при коммите: " + exception.getMessage());
    } else {
        System.out.println(
	        "Оффсеты успешно зафиксированы: " + offsets);
    }
};
```

## Методы
Интерфейс содержит **единственный метод**:
### Callback метод
```java
// Вызывается после завершения асинхронного коммита оффсетов.
// Параметры:
//  - offsets: карта TopicPartition → OffsetAndMetadata, отражающая ❗❗оффсеты, которые пытались зафиксировать.❗❗
//  - exception: если не null — commit завершился с ошибкой (например, из-за смены координатора группы).
void onComplete(
	Map<TopicPartition, OffsetAndMetadata> offsets, 
	Exception exception);
```

## 💡 Особенности и подводные камни 💡

1️⃣ ❗❗❗**Асинхронный commit НЕ гарантирует успех.**  
Колбэк может быть вызван с `exception`, если произошёл сбой или consumer потерял связь с координатором.  
Коммиты могут быть **потеряны**, если consumer завершился до получения ответа.

2️⃣ **Асинхронный commit — "fire-and-forget".**  
Основной поток не ждёт завершения операции.  
Это увеличивает производительность, но снижает гарантию доставки.

3️⃣ 💡**Колбэк вызывается в клиентском I/O потоке.**  
Не выполняй в нём долгие операции (вроде записи в БД), иначе задержишь сетевые операции клиента Kafka.

4️⃣ ❗❗❗**Возможна "гонка" между коммитами.**  
Если коммиты выполняются слишком часто, колбэк может получить ответ от "старого" коммита позже, чем от "нового" —  
и перезаписать более свежие оффсеты старыми.

5️⃣ ❗❗❗**commitAsync() не следует вызывать после commitSync().**
Асинхронный commit может завершиться позже и "откатить" оффсет на старое значение.

---
## 💎 Best Practices 💎

✅ ❗❗❗Используй `OffsetCommitCallback` **только** если нужен ==контроль ошибок== при `commitAsync()`. 

✅ ❗❗❗Проверяй `exception` в `onComplete()` и логируй ошибки. 

✅ 💡Не делай тяжёлых операций внутри колбэка.  

✅ Если после commitAsync нужно гарантировать успешное завершение —  вызывай `commitSync()` перед закрытием consumer-а.  

✅ 🧠В логах помечай оффсеты и партиции для отладки: это поможет при расследовании сбоев.

## Пример
```java
import org.apache.kafka.clients.consumer.*;
import org.apache.kafka.common.TopicPartition;
import java.time.Duration;
import java.util.*;

public class AsyncCommitExample {
    public static void main(String[] args) {
        Properties props = new Properties();
        props.put(
	        ConsumerConfig.BOOTSTRAP_SERVERS_CONFIG, 
	        "localhost:9092");
        props.put(
	        ConsumerConfig.GROUP_ID_CONFIG, 
	        "example-group");
        props.put(
	        ConsumerConfig.KEY_DESERIALIZER_CLASS_CONFIG, 
	        StringDeserializer.class.getName());
        props.put(
	        ConsumerConfig.VALUE_DESERIALIZER_CLASS_CONFIG, 
	        StringDeserializer.class.getName());
        props.put(
	        ConsumerConfig.ENABLE_AUTO_COMMIT_CONFIG, 
	        "false"); // ручное управление оффсетами


        try (KafkaConsumer<String, String> consumer = new 
	        KafkaConsumer<>(props)
	    ) {
	        consumer.subscribe(
		        Collections.singletonList("orders")
		    );

            while (true) {
                ConsumerRecords<String, String> records = 
	                consumer.poll(Duration.ofMillis(100));

                for (ConsumerRecord<String, String> record : 
	                records
	            ) {
                    System.out.printf(
	                    "Получено сообщение: topic=%s" + 
	                    "partition=%d" +
	                    "offset=%d key=%s value=%s%n",
                        record.topic(), 
                        record.partition(), 
                        record.offset(), 
                        record.key(), 
                        record.value());
                }

                // Асинхронная фиксация оффсетов
                // с обработкой результата
                consumer.commitAsync(new OffsetCommitCallback() {
                    @Override
                    public void onComplete(
	                    Map<TopicPartition, OffsetAndMetadata> 
		                offsets, 
	                    Exception exception
	                ) {
                        if (exception != null) {
                            System.err.println(
	                            "Ошибка при коммите: " + 
	                            exception.getMessage());
                        } else {
                            System.out.println(
	                            "Оффсеты успешно зафиксированы:" 
	                            " + offsets);"
                        }
                    }
                });
            }
        }
    }
}
```
