
> [!NOTE] `ProducerRecord<K, V>`
> это неизменяемый объект (immutable), ==представляющий одно сообщение==, отправляемое в Kafka.  
> 
> 💡Каждый экземпляр описывает **куда (топик, раздел)** и **что (ключ, значение, заголовки)** нужно отправить.  
> 
> ❗❗Используется как основной аргумент метода `KafkaProducer.send()`.

## Конструкторы
```java
// Создает запись для заданного топика и значения.
// Ключ и разделение (partition) будут определены автоматически.
public ProducerRecord(String topic, V value)

// Создает запись с указанным топиком, ключом и значением.
// Партиция выбирается автоматически на основе ключа.
public ProducerRecord(String topic, K key, V value)

// Создает запись с указанным топиком, партицией, ключом и значением.
public ProducerRecord(
	String topic, 
	Integer partition, 
	K key, 
	V value)

// Создает запись с заданным временем метки (timestamp), партицией, ключом и значением.
// Время используется при записи в Kafka как timestamp события.
public ProducerRecord(
	String topic, 
	Integer partition, 
	Long timestamp, 
	K key, 
	V value)

// Создает запись с дополнительными заголовками (Headers).
// Это самый полный конструктор.
public ProducerRecord(
	String topic, 
	Integer partition, 
	Long timestamp, 
	K key, 
	V value, 
	Iterable<Header> headers)
```

### 💬 Пояснения к параметрам:
- **`topic`** — имя топика, в который отправляется сообщение. Не может быть `null`.
    
- **`partition`** — номер партиции (может быть `null`, тогда выберет `Partitioner`).
    
- **`timestamp`** — временная метка сообщения (в миллисекундах с начала эпохи). Если `null`, брокер поставит своё время.
    
- **`key`** — ключ сообщения (используется для выбора партиции). Может быть `null`.
- **`value`** — основное содержимое сообщения. Может быть `null`.
    
- ⭐⭐**`headers`** — дополнительные метаданные в виде пар «ключ-значение» (опционально).

## Методы

### Основные геттеры
```java
// Возвращает имя топика, в который будет отправлено сообщение.
public String topic()

// Возвращает номер партиции, если был задан пользователем, иначе null.
public Integer partition()

// Возвращает временную метку сообщения (может быть null).
public Long timestamp()

// Возвращает ключ сообщения.
public K key()

// Возвращает значение сообщения.
public V value()

// Возвращает коллекцию заголовков сообщения (Headers API).
public Iterable<Header> headers()
```

## 💡 Особенности и подводные камни 💡

1️⃣ ❗❗❗`ProducerRecord` — **immutable**: после создания его поля нельзя изменить, что гарантирует потокобезопасность. 

2️⃣ ⭐⭐Если партиция не указана, она вычисляется через `Partitioner` (по ключу или round-robin).  

3️⃣ ❗❗❗Если `key == null`, сообщения распределяются равномерно по всем партициям.  

4️⃣ ❗❗❗Если `timestamp == null`, брокер подставляет текущее системное время (event-time ≠ ingest-time). 

5️⃣ 🧠Заголовки (`Headers`) можно использовать для передачи метаданных (например, ID запроса, тип контента и т. д.).  

6️⃣ Не рекомендуется создавать новый `ProducerRecord` на каждый байт данных — лучше переиспользовать сериализаторы, но не сами записи.  

7️⃣ Значения `topic` и `value` **НЕ МОГУТ быть null** одновременно — иначе выбросится `NullPointerException`.

---
## 💎 Best Practices 💎

✅ Всегда задавайте **ключ**, если хотите обеспечить порядок сообщений по одному источнику.  

✅ Используйте **timestamp**, если работаете с event-time обработкой (например, в Kafka Streams).  

✅ 💡Добавляйте **headers** для трассировки или межсервисных корреляций.  

✅ 💡Избегайте передачи слишком больших сообщений (>1MB) — Kafka не оптимизирована под это. 

✅ Используйте `toString()` при логировании — формат уже содержит всю диагностическую информацию.

## Пример
```java
import org.apache.kafka.clients.producer.*;

public class ProducerRecordExample {
    public static void main(String[] args) {
        // Настройка продюсера
        Properties props = new Properties();
        props.put(
	        ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, 
	        "localhost:9092");
        props.put(
	        ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, "org.apache.kafka.common.serialization.StringSerializer");
        props.put(
	        ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, "org.apache.kafka.common.serialization.StringSerializer");

        // Создание продюсера
        KafkaProducer<String, String> producer = new KafkaProducer<>(props);

        // 1️⃣Создание записи
        ProducerRecord<String, String> record = new ProducerRecord<>(
                "my-topic",           // Топик
                0,                    // Партиция
                System.currentTimeMillis(), // Время
                "user123",            // Ключ
                "Hello Kafka!",       // Значение
                List.of(new RecordHeader("source", "service-A".getBytes())) // Заголовки
        );

        // 2️⃣Отправка записи асинхронно с колбэком
        producer.send(record, (metadata, exception) -> {
            if (exception != null) {
                System.err.println(
	                "Ошибка при отправке: " +
	                exception.getMessage());
            } else {
                System.out.printf(
	                "Сообщение отправлено в %s-%d, offset=%d%n",
                    metadata.topic(), 
                    metadata.partition(), 
                    metadata.offset());
            }
        });

        // Закрытие продюсера
        producer.close();
    }
}
```
