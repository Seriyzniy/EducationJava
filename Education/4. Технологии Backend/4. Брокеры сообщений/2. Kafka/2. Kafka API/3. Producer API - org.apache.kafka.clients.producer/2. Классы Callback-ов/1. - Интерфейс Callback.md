
> [!NOTE] `Callback`
> это **функциональный интерфейс**, используемый продюсером Kafka ==для обработки результатов асинхронной отправки== сообщений.  
> 
> Когда вы вызываете `producer.send(record, callback)`, Kafka выполнит метод `onCompletion()` этого интерфейса,
> передав туда результат (`RecordMetadata`) и исключение (`Exception`), если оно возникло.
> 

Таким образом, интерфейс нужен для **реакции на успешную или неуспешную отправку сообщения** — например, логирования, метрик, повторной отправки или пользовательской обработки ошибок.

---
## Интерфейс
```java
package org.apache.kafka.clients.producer;

@FunctionalInterface
public interface Callback {

    /**
     * Метод, вызываемый, когда запись отправлена на сервер.
     *
     * Этот метод вызывается, когда сервер подтверждает получение записи, или если произошла ошибка при её отправке.
     * 
     * - 💡metadata метаданные записи💡, содержащие информацию о топике, партиции и смещении.
     * Если произошла ошибка, это значение может быть null.
     * 
     * - exception исключение, возникшее при отправке записи.
     * Если операция завершилась успешно, это значение будет null.
     */
    void onCompletion(
	    RecordMetadata metadata, 
	    Exception exception);
}
```

---
## 💡 Особенности и подводные камни 💡

1️⃣ 💡**Асинхронный вызов:**  
Метод `onCompletion()` вызывается в одном из потоков `KafkaProducer`, **не в основном** потоке приложения.  
Следовательно, **не стоит выполнять в нём тяжёлые или блокирующие операции** (например, I/O, БД).

2️⃣ 💡**Возможность вызова при ошибке:**  
Если при отправке произошла ошибка (например, `TimeoutException`, `SerializationException`), то:
- второй аргумент (`exception`) будет не `null`, 
- а `RecordMetadata` — наоборот, `null`.

3️⃣ **Порядок выполнения:**  
Kafka ==не гарантирует порядок выполнения callback’ов== относительно отправленных сообщений,  
особенно при использовании **нескольких потоков** или **повторных попыток (retries)**.

4️⃣ ❗❗❗**Риски при повторных отправках:**  
При включённом `retries > 0` один и тот же callback может быть вызван повторно,  
если сообщение было переотправлено — это нужно учитывать при подсчёте метрик или логировании.

5️⃣ **Потокобезопасность:**  
`onCompletion()` может вызываться параллельно для разных сообщений,  
поэтому **нельзя изменять в нём общие mutable-поля без синхронизации**.

## 💎 Best Practices 💎

✅ ❗❗**Минимизируй логику внутри callback:**  
Делай только лёгкие операции — запись в лог, инкремент метрики, постановка события в очередь и т.д.

✅ **Обрабатывай ошибки явно:**  
Не игнорируй `exception`, иначе можно «потерять» сообщения.  
Пример:
```java
if (exception != null) {
    log.error("Ошибка при отправке: ", exception);
}
```

✅ 💡**Используй для метрик:**  
Отсюда можно получать `metadata.topic()`, `metadata.partition()`, `metadata.offset()` — удобно для мониторинга производительности.

✅ ❗❗❗**Не блокируй callback:**  
Если нужно выполнить что-то долгосрочное (например, HTTP-запрос),  
отправь задачу в отдельный пул потоков, не блокируя Kafka I/O потоки.

✅ 💡**Можно использовать лямбды (Java 8+):**  
Интерфейс — функциональный, поэтому можно писать коротко:
```java
producer.send(record, (metadata, e) -> { ... });
```

## Пример
```java
import org.apache.kafka.clients.producer.*;

import java.util.Properties;

public class CallbackExample {
    public static void main(String[] args) {
        Properties props = new Properties();
        props.put(
	        ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, 
	        "localhost:9092");
        props.put(
	        ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, "org.apache.kafka.common.serialization.StringSerializer");
        props.put(
	        ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, "org.apache.kafka.common.serialization.StringSerializer");

        try (KafkaProducer<String, String> producer = new KafkaProducer<>(props)) {

            ProducerRecord<String, String> record =
                    new ProducerRecord<>(
	                    "payments", 
	                    "order-1001", 
	                    "processed");

            // ⭐⭐Асинхронная отправка с колбэком
            producer.send(record, new Callback() {
                @Override
                public void onCompletion(RecordMetadata metadata, Exception exception) {
                    if (exception == null) {
                        System.out.printf(
	                        "Успешно отправлено в %s-%d,"+ 
	                        "offset=%d%n",
                            metadata.topic(), 
                            metadata.partition(),
                             metadata.offset());
                    } else {
                        System.err.printf(
	                        "Ошибка при отправке сообщения:"+ 
	                        "%s%n", 
	                        exception.getMessage());
                    }
                }
            });
        }
    }
}
```
