
> [!NOTE] **`KafkaFuture<T>`** — это _обобщённый класс_, ==представляющий результат асинхронной операции== в Kafka.
> 
> Он используется для методов `AdminClient`, которые выполняются асинхронно (например, `createTopics`, `describeTopics`, `deleteRecords` и др.).
> 
> Этот класс — аналог `java.util.concurrent.CompletableFuture`, но реализован **самостоятельно** и без внешних зависимостей, чтобы быть совместимым с различными версиями JDK.

---
## Конструктор - private
У класса **приватный конструктор**, потому что `KafkaFuture` создаётся **через фабричные методы**. 

Пользователь вручную его не создаёт — Kafka создаёт и возвращает `KafkaFuture` при вызове асинхронных методов.

### Фабричные методы
```java
// Создаёт уже завершённое успешно будущее с указанным значением
public static <U> KafkaFuture<U> completedFuture(U value);

// Создаёт новое пустое будущее, которое можно завершить вручную
public static <U> KafkaFuture<U> completedFuture();

// Создаёт новое будущее с ошибкой (exceptional completion)
public static <U> KafkaFuture<U> failedFuture(
	Throwable throwable);
```

## Методы

### Управление завершением
```java
// Блокирует текущий поток, пока результат не будет готов
public T get() throws InterruptedException, ExecutionException;

// Блокирует поток на указанный таймаут
public T get(long timeout, TimeUnit unit)
    throws InterruptedException, ExecutionException, TimeoutException;

// Проверяет, завершено ли будущее
public boolean isDone();

// Проверяет, было ли будущее отменено
public boolean isCancelled();

// Отменяет выполнение задачи (если возможно)
public boolean cancel(boolean mayInterruptIfRunning);
```

### Обработка результата (цепочки и колбэки)
```java
// Применяет функцию к результату и возвращает новое KafkaFuture
public <R> KafkaFuture<R> thenApply(
	BaseFunction<T, R> function);

// Добавляет колбэк, вызываемый при завершении
public void whenComplete(
	BiConsumer<? super T, ? super Throwable> action);

// Применяет асинхронную функцию, возвращающую другое KafkaFuture
public <R> KafkaFuture<R> thenCompose(
	BaseFunction<T, KafkaFuture<R>> function);
```

### Принудительное завершение (вручную)
```java
// Устанавливает успешный результат вручную
public void complete(T value);

// Завершает будущее с ошибкой
public void completeExceptionally(Throwable throwable);
```

### Вспомогательные статические методы
```java
// Возвращает KafkaFuture, завершённое после всех futures в списке
public static KafkaFuture<Void> allOf(
	KafkaFuture<?>... futures);

// Возвращает KafkaFuture, завершённое после первого успешно завершённого future
public static <T> KafkaFuture<T> anyOf(
	KafkaFuture<? extends T>... futures);
```

## 💡 Особенности и подводные камни 💡

1️⃣ `KafkaFuture` — **не часть стандартной Java**, а собственная реализация Future для Kafka (без `CompletableFuture`).  

2️⃣ ❗❗❗Он **НЕ использует ForkJoinPool**, то есть все колбэки (`thenApply`, `whenComplete`) выполняются в **том же потоке**, где завершается future.  

3️⃣ ❗❗❗Методы `get()` — **блокирующие**. В асинхронных приложениях (например, Spring Boot Web) лучше использовать колбэки.  

4️⃣ Если KafkaFuture завершается с ошибкой, `get()` выбросит `ExecutionException`, внутри которого будет реальное исключение Kafka. 

5️⃣ Не предназначен для внешнего создания — обычно создаётся **Kafka AdminClient’ом**.

---
## 💎 Best Practices 💎

✅ ❗❗❗Используй `thenApply()` или `whenComplete()` для обработки результата без блокировки. 

✅ ❗❗❗Избегай `get()` в асинхронных потоках (может блокировать IO).  

✅ 💡Проверяй `isDone()` или используй `allOf()` для синхронизации нескольких асинхронных вызовов.  

✅ При отладке логируй результат через `whenComplete()` — удобно видеть ошибки Admin API.  

✅ Используй `failedFuture()` для тестов или имитации ошибок.

## Пример
```java
AdminClient admin = AdminClient.create(props);

// ⭐Асинхронное создание топика
NewTopic topic = new NewTopic("example", 1, (short) 1);
KafkaFuture<Void> future = 
	admin.createTopics(Collections.singleton(topic)).all();

// ⭐Асинхронная обработка результата
future.whenComplete((result, exception) -> {
    if (exception == null) {
        System.out.println("Топик успешно создан!");
    } else {
        System.err.println(
	        "Ошибка при создании топика: " +
	        exception.getMessage());
    }
});

// ⭐Или блокирующий вариант (не рекомендуется в продакшене)
future.get();
```
