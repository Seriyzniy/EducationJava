
> [!NOTE] `Serializer<T>` 
> это ==основной интерфейс Kafka==, который отвечает за ==ПРЕОБРАЗОВАНИЕ ОБЪЕКТОВ Java в массив байтов== (`byte[]`) перед отправкой в Kafka.  
> 
> Используется продюсером (`KafkaProducer`), чтобы сериализовать **ключи и значения** сообщений в бинарный формат, пригодный для передачи по сети и записи в топики.
> 
> Реализуется множеством стандартных сериализаторов (`StringSerializer`, `IntegerSerializer`, `ByteArraySerializer`, и т.д.), а также может быть реализован пользователем для своих типов.
> 

## Интерфейс
```java
package org.apache.kafka.common.serialization;

public interface Serializer<T> extends Closeable {
```

> Интерфейс **параметризован** типом `<T>` — это тип объекта, который вы хотите сериализовать (например, `String`, `User`, `Integer` и т.п.).

---
## Методы
### Жизненный цикл сериализатора
```java
// Настраивает сериализатор. Вызывается один раз до начала сериализации.
// Аргументы:
//  - configs — карта параметров конфигурации
//  - isKey — true, если сериализатор используется для ключей сообщений, false — для значений
void configure(
	Map<String, ?> configs, 
	boolean isKey);
	
	

// Закрывает сериализатор и освобождает ресурсы.
// Метод должен быть идемпотентным (может вызываться несколько раз).
void close();
```

### Сериализация данных
```java
// Сериализует объект `data` в массив байтов.
// Аргументы:
//  - topic — имя топика (может использоваться для выбора схемы сериализации)
//  - data — объект для сериализации (может быть null)
byte[] serialize(
	String topic, 
	T data);


// Расширенная версия метода serialize, учитывающая метаданные заголовков Kafka.
// Аргументы:
//  - topic — имя топика
//  - headers — заголовки сообщения (могут использоваться для выбора стратегии сериализации)
//  - data — объект для сериализации
byte[] serialize(
	String topic, 
	Headers headers, 
	T data);
```

## 💡 Особенности и подводные камни 💡

1️⃣ **Kafka вызывает `configure()` один раз**, при инициализации продюсера. Все сериализаторы должны быть готовы к многопоточному доступу после этого.  

2️⃣ Метод `serialize()` может вызываться **в разных потоках**, поэтому реализация должна быть **потокобезопасной**. 

3️⃣ 💡Если `serialize()` возвращает `null`, Kafka отправит “пустое” значение (т.е. ключ/значение будет NULL).

4️⃣ ❗❗❗Метод `close()` вызывается автоматически при закрытии `KafkaProducer`.  

5️⃣ ❗❗❗Если сериализация выбрасывает исключение, **KafkaProducer бросит `SerializationException`**, и ❗❗❗сообщение не будет отправлено❗❗❗

6️⃣ 🧠При использовании `Headers` можно реализовать контекстно-зависимую сериализацию (например, разные форматы для разных типов сообщений).

---
## 💎 Best Practices 💎

✅ 💡Используй стандартные сериализаторы (`StringSerializer`, `LongSerializer`) для базовых типов. 

✅ ❗❗❗Для своих типов — реализуй `Serializer<T>` и **добавь** `Deserializer<T>`, чтобы составить `Serde<T>`.  

✅ ❗❗❗Не храни состояние внутри сериализатора — Kafka может использовать его параллельно. 

✅ ❗❗❗Не выполняй тяжёлые операции (например, подключение к БД) в `serialize()` — делай это в `configure()`

✅ Если сериализуешь JSON/Avro/Protobuf — указывай схему и версию, чтобы избежать несовместимости. 

✅ При ошибках сериализации логируй `topic` и `data.getClass()` — это ускоряет диагностику.

## Пример
```java
import org.apache.kafka.common.serialization.Serializer;
import java.nio.charset.StandardCharsets;
import java.util.Map;

// Пример собственного сериализатора для класса User
public class UserSerializer implements Serializer<User> {

    @Override
    public void configure(
	    Map<String, ?> configs, 
	    boolean isKey) 
	{
        // Можно прочитать кастомные параметры (например, кодировку)
    }

    @Override
    public byte[] serialize(String topic, User data) {
        if (data == null) return null;
        String serialized = data.getId() + "," + data.getName();
        return serialized.getBytes(StandardCharsets.UTF_8);
    }

    @Override
    public void close() {
        // Освобождение ресурсов, если нужно
    }
}
```

Использование с продюсером:
```java
Properties props = new Properties();

props.put(
	ProducerConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
props.put(
	ProducerConfig.KEY_SERIALIZER_CLASS_CONFIG, 
	StringSerializer.class.getName());

// КАСТОМНЫЙ сериализатор ЗНАЧЕНИЯ
props.put(
	ProducerConfig.VALUE_SERIALIZER_CLASS_CONFIG, 
	UserSerializer.class.getName());

KafkaProducer<String, User> producer = new KafkaProducer<>(props);

producer.send(
	new ProducerRecord<>(
		"users", 
		"user-key", 
		new User(1, "Alice")));

producer.close();
```
