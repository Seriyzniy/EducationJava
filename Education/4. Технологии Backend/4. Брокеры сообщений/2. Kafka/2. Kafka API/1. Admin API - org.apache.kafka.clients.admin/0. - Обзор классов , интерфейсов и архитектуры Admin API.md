# 📕1. Архитектура пакета Admin API
Почти все операции, выполняемые через `AdminClient`, имеют **три компонента**

- **Options** — это _“что и как мы хотим запросить”_.  
    Они _не выполняют логику_ и _не хранят состояние брокера_

- **Result** — это _“обещание получить результат”_, т.е. объект с `KafkaFuture`, который можно асинхронно ожидать.  
    Kafka не возвращает «чистый результат» сразу, потому что операции могут выполняться на нескольких брокерах, с задержкой, через сеть и т.п.

- **Entity/Model-классы** — уже содержат реальные данные (например, `ConfigEntry`, `TopicPartitionInfo`, `OffsetSpec` и т.д.), но **они создаются внутри `Result`** после выполнения запроса.


> 👉 **Options / Result не “конфигурируют Kafka”**, они лишь **конфигурируют запросы к Kafka** и **представляют их результаты**.

# 📕2. Классы и Интерфейсы
## 📙Интерфейсы

| Название интерфейса | Описание                                                                                                                                                              |
| ------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Admin** ✅         | Основной интерфейс для выполнения административных операций в Kafka (создание топиков, просмотр брокеров, управление конфигурациями и другие административные задачи) |

## 📙Классы
### Основные

| Название класса                                                         | Описание                                                                                                                                 |
| ----------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| **KafkaAdminClient**✅<br>(не используется, обычно через Admin.create()) | **Стандартная реализация интерфейса Admin**, предоставляющая клиент для административных операций с Kafka-кластером                      |
| **AdminClientConfig**✅                                                  | Класс **конфигурации для клиента Admin**, содержащий ключи и настройки для подключения и работы с Kafka-кластером                        |
|                                                                         |                                                                                                                                          |
| **NewTopic**✅                                                           | Класс, описывающий **параметры создаваемого топика**, включая имя, количество партиций, фактор репликации и другие настройки             |
| **CreateTopicsOptions**✅                                                | Класс, содержащий **опции для операции создания топиков**, такие как таймауты и флаги валидации                                          |
| **CreateTopicsResult**✅                                                 | Класс, представляющий **результат операции создания топиков**, содержащий информацию об успешности операции и возможные ошибки           |
|                                                                         |                                                                                                                                          |
| **DescribeConfigsOptions**✅                                             | Класс, содержащий опции для операции получения конфигураций ресурсов Kafka (топиков, брокеров)                                           |
| **DescribeConfigsResult**✅                                              | Класс, представляющий результат операции получения конфигураций, содержащий запрошенные конфигурационные данные                          |
| **AlterConfigOp**                                                       | Класс, описывающий изменения конфигурации для incremental операций, включая имя параметра, значение и тип операции (SET, DELETE, APPEND) |
| **ListOffsetsOptions**                                                  | Класс, содержащий опции для запроса информации о смещениях (offsets) в топиках и партициях                                               |
| **ListOffsetsResult**                                                   | Класс, представляющий результат запроса информации о смещениях, содержащий данные о позициях в партициях                                 |
| **TopicDescription**                                                    | Класс, содержащий подробную информацию о топике, включая список партиций, информацию о лидерах, репликах и их распределении              |
