
> [!NOTE] `CreateTopicsResult`
> **final-класс** из пакета `org.apache.kafka.clients.admin`, представляющий **асинхронный результат** операции создания топиков в Kafka.

Он **инкапсулирует** внутренние `KafkaFuture` объекты для каждого топика, что позволяет:
- 🔶дожидаться завершения операции (`all()` или `values()`),
- 🔶обрабатывать ошибки для конкретных топиков,
- 🔶работать в асинхронном стиле без блокировок.

📌 Возвращается методом:
```java
Admin.createTopics
	(Collection<NewTopic> newTopics, 
	CreateTopicsOptions options)
```

---
## Конструктор - private
```java
// Конструктор используется внутри Kafka Admin API и недоступен напрямую.
// Объекты CreateTopicsResult создаются фабричным методом AdminClient#createTopics().
private CreateTopicsResult
	(Map<String, KafkaFuture<Void>> futures)
```
**Пояснение:**
- `futures` — отображение (map) из имени топика в `KafkaFuture<Void>`, который завершится при успешном создании темы или завершится с ошибкой, если операция не удалась.

---
## Методы

### Получение результатов по конкретным топикам
```java
// Возвращает отображение: 💡имя топика → KafkaFuture,💡 соответствующий результату создания конкретного топика.
// 💡Позволяет обработать результат каждого топика отдельно.
public Map<String, KafkaFuture<Void>> values()

// Возвращает KafkaFuture для конкретного топика по его имени.
// Можно использовать для точечной проверки или асинхронной обработки.
public KafkaFuture<Void> values(String topicName)
```

### Ожидание завершения всех операций
```java
// ⭐Возвращает общий KafkaFuture, который завершится успешно, если все топики были успешно созданы.⭐
// Если хотя бы один топик не создался — future завершится с ошибкой.
public KafkaFuture<Void> all()
```
💬 **Комментарий:**  
Этот метод удобен, **если нужно просто дождаться окончания всей операции**, не проверяя каждый топик по отдельности.

---
## 💡 Особенности и подводные камни 💡

1️⃣ `CreateTopicsResult` — **асинхронный объект**. ==Само создание топиков может занимать время==, особенно в больших кластерах.  

2️⃣ 💡Вызов `all().get()` **блокирует поток** до завершения операции — используйте осторожно в высоконагруженных потоках.  

3️⃣ Ошибки (например, `TopicExistsException`, `InvalidReplicationFactorException`) выбрасываются **при вызове `.get()`**, а не при вызове `createTopics()`.  

4️⃣ Даже если `ignoreIfExists(true)` включён, Kafka всё равно может вернуть **частичные ошибки** — например, если часть конфигурации темы несовместима.  

5️⃣ 💡`KafkaFuture` не является стандартным `CompletableFuture`, но поддерживает схожие методы (`whenComplete`, `thenApply`, `exceptionally`).

---
## 💎 Best Practices 💎

✅ 💡Используй `all().get()` только для небольших наборов топиков или при инициализации кластера.  

✅ 💡Для масштабных операций — обрабатывай `values()` по отдельности, чтобы не блокировать сразу всё.  

✅ Если создаёшь темы при старте приложения — добавляй `ignoreIfExists(true)` в `CreateTopicsOptions`.  

✅ ⭐Используй `KafkaFuture.whenComplete(...)` для неблокирующего ожидания результата. ⭐ 

✅ 🧠При отладке полезно логировать `CreateTopicsResult#values()` до вызова `.get()` — это поможет выявить зависшие futures.

## Пример
```java
import org.apache.kafka.clients.admin.*;
import java.util.*;
import java.util.concurrent.ExecutionException;

public class CreateTopicsResultExample {
    public static void main(String[] args) throws ExecutionException, InterruptedException {
        Properties props = new Properties();
        props.put(
	        AdminClientConfig.BOOTSTRAP_SERVERS_CONFIG, 
	        "localhost:9092"
	    );

        try (AdminClient admin = AdminClient.create(props)) {
            // Создаём описание топика
            NewTopic newTopic = new NewTopic(
	            "logs", 
	            2, 
	            (short) 1
	        );

            // Настройки операции
            CreateTopicsOptions options = new CreateTopicsOptions()
                    .ignoreIfExists(true)
                    .timeoutMs(5000);

            // Запуск операции
            CreateTopicsResult result = 
	            admin
		            .createTopics(List.of(newTopic), options);


            // ✅ Вариант 1 — дождаться создания всех топиков
            result.all().get();
            System.out.println(
	            "Все топики успешно созданы!"
	        );


            // ✅ Вариант 2 — обработать по каждому топику
            result.values().forEach((topic, future) -> {
                future.whenComplete((v, ex) -> {
                    if (ex == null)
                        System.out.println(
	                        "Топик " + topic + 
	                        " создан успешно");
                    else
                        System.err.println("Ошибка при создании топика " + topic + ": " + ex.getMessage());
                });
            });
        }
    }
}
```
