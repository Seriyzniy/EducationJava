
> [!NOTE] `NewTopic` - модельный класс (data class)
> представляющий **описание нового топика**, который должен быть создан в Kafka через `AdminClient`.

Используется вместе с методом:
```java
Admin.createTopics(Collection<NewTopic> newTopics)
```

`NewTopic` инкапсулирует все необходимые параметры для создания темы:
- имя топика,
- количество партиций, 
- фактор репликации,
- карту размещения партиций вручную (если нужно).


**Реализует интерфейс:**  
🧩 `org.apache.kafka.common.Configurable` (не напрямую, но поддерживает установку `configs`)  
— фактически, `NewTopic` — это POJO, не реализующий интерфейсы.

## Конструкторы
```java
// ⭐Конструктор №1: стандартный⭐ (имя, партиции, репликация)
// Используется чаще всего для создания нового топика с автоматическим размещением.
public NewTopic(
	String name, 
	int numPartitions, 
	short replicationFactor) 
{
    this.name = Objects.requireNonNull(name, "name");
    this.numPartitions = Optional.of(numPartitions);
    this.replicationFactor = Optional.of(replicationFactor);
    this.replicasAssignments = Optional.empty();
    this.configs = Optional.empty();
}

// Конструктор №2: создание топика с ручным распределением партиций по брокерам.
// 💡Ключ — номер партиции, значение — список ID брокеров,💡 где будут храниться реплики.
public NewTopic(
	String name, 
	Map<Integer, List<Integer>> replicasAssignments) 
{
    this.name = Objects.requireNonNull(name, "name");
    this.replicasAssignments = 
	    Optional.of(
		    Collections.unmodifiableMap(
			    new LinkedHashMap<>(replicasAssignments)
			)
		);
    this.numPartitions = Optional.empty();
    this.replicationFactor = Optional.empty();
    this.configs = Optional.empty();
}
```

## Методы
Класс `NewTopic` — это **простая DTO-модель**, поэтому содержит в основном **геттеры**, **сеттеры (chainable)** и методы для настройки конфигураций.

### Основные методы
```java
// Возвращает имя топика
public String name()

// Возвращает количество партиций (если указано)
public Optional<Integer> numPartitions()

// Возвращает фактор репликации (если указан)
public Optional<Short> replicationFactor()

// Возвращает карту ручного размещения партиций (если указано)
public Optional<Map<Integer, List<Integer>>> replicasAssignments()
```

### Конфигурация топика
```java
// Добавляет конфигурацию топика (ключ -> значение)
public NewTopic configs(Map<String, String> configs) {
    this.configs = 
	    Optional.of(
		    Collections.unmodifiableMap(
			    new LinkedHashMap<>(configs)
			)
		);
    return this;
}

// Возвращает конфигурации топика
public Optional<Map<String, String>> configs()
```

Пример настроек через configs:
```java
Map<String, String> configs = new HashMap<>();
configs.put("cleanup.policy", "compact");
configs.put("retention.ms", "86400000");

NewTopic topic = new NewTopic(
	"example", 
	3, 
	(short) 1
).configs(configs);
```

## 💡 Особенности и подводные камни 💡

1️⃣ ❗❗❗Если не заданы `numPartitions` и `replicationFactor`, Kafka **не создаст топик автоматически** — одно из значений обязательно.  

2️⃣ Поле `replicasAssignments` **взаимоисключающе** с `numPartitions`/`replicationFactor`.  

3️⃣ 💡Метод `configs()` не влияет на уже созданные топики — он применяется **только при создании**. 

4️⃣ 💡Kafka не проверяет значения `configs` на корректность до отправки запроса — ошибки будут на стороне брокера. 

5️⃣ **Репликация (`replicationFactor`) не может быть больше** числа **доступных брокеров**.

---
## 💎 Best Practices 💎

✅ Используй **`Optional` поля осознанно** — не нужно задавать сразу и `replicasAssignments`, и `numPartitions`.  

✅ ⭐Для типовых задач достаточно конструктора `(name, numPartitions, replicationFactor)`.  

✅ ⭐Для продакшена всегда указывай **уникальные имена⭐ топиков** — Kafka не создаёт дубликаты.

✅ Проверяй конфиги через `describeTopics()` после создания, чтобы убедиться, что все настройки применились.  

✅ 💡Для создания большого числа топиков делай **batch-запросы** (`AdminClient#createTopics(Collection<NewTopic>)`).

## Пример
```java
import org.apache.kafka.clients.admin.*;
import java.util.*;
import java.util.concurrent.ExecutionException;

public class CreateTopicExample {
    public static void main(String[] args) throws ExecutionException, InterruptedException {
        Properties props = new Properties();
        props.put(
	        AdminClientConfig.BOOTSTRAP_SERVERS_CONFIG, 
	        "localhost:9092"
	    );

        try (Admin admin = Admin.create(props)) {
            // Конфигурации для нового топика
            Map<String, String> topicConfig = new HashMap<>();
            topicConfig.put("cleanup.policy", "delete");
            topicConfig.put("retention.ms", "3600000");

            // Создание объекта NewTopic
            NewTopic newTopic = new NewTopic(
	            "my-topic", 
	            3, 
	            (short) 1
	        ).configs(topicConfig);

            // Отправка запроса на создание
            CreateTopicsResult result = 
	            admin
		            .createTopics(
			            Collections.singleton(newTopic)
			        );

            // Проверка результата
            result.all().get();
            System.out.println("Топик успешно создан!");
        }
    }
}
```
