
> [!NOTE] `DescribeConfigsOptions`
> **final-класс**, наследуемый от базового `AbstractOptions<DescribeConfigsOptions>`.  
> 
> 💡Он используется **для настройки поведения операции** `describeConfigs()` в `AdminClient`. (ПРИ ПОЛУЧЕНИИ КОНФИГУРАЦИИ топиков, брокеров и т.д)

📌 Основное назначение:
> Определяет, **как именно** и **что именно следует извлекать при описании конфигураций** (в частности, — включать ли нестатические, динамические или "чувствительные" параметры).

## Конструктор
```java
// Создаёт объект настроек описания конфигурации с параметрами по умолчанию.
public DescribeConfigsOptions()
```

**Пояснение:**
- По умолчанию включает только **эффективные** (активные) конфигурации,
- **не включает** "чувствительные" параметры (например, пароли),
- **не обрабатывает** нестандартные (временные или pending) изменения.

Никаких других конструкторов у этого класса нет — параметры настраиваются через **цепочку сеттеров** (fluent API).

## Методы
Поскольку класс наследует `AbstractOptions`, он также **унаследует** методы для задания таймаута (`timeoutMs()` и т.д.), но ниже — только методы, **добавленные в `DescribeConfigsOptions`**.

### Настройка включения нестатических конфигураций
```java
// Указывает, нужно ли включать неэффективные (нестатические) конфигурации.
// Например, временные значения, которые будут применены после перезапуска брокера.
public DescribeConfigsOptions includeSynonyms(
	boolean includeSynonyms)
	
// Возвращает true, если должны быть включены синонимы конфигураций.
public boolean includeSynonyms()
```

💬 **Пояснение:**  
Kafka хранит конфигурации на разных уровнях (по умолчанию, на уровне брокера, на уровне топика и т.д.).  
Если `includeSynonyms(true)` — метод вернёт все эти уровни (например, `default.config`, `topic.config`, `broker.config`).

> Говорит Kafka:  
> "Покажи мне все возможные источники этого конфигурационного параметра (topic-level, broker-level, default-level)."

### Управление отображением чувствительных параметров
```java
// Указывает, нужно ли включать ❗❗чувствительные❗❗ (sensitive) параметры, ❗❗например пароли❗❗.
public DescribeConfigsOptions includeSensitive(
	boolean includeSensitive)

// Возвращает true, если чувствительные параметры должны быть включены в ответ.
public boolean includeSensitive()
```

> Говорит Kafka:  
> "Верни мне также значения, помеченные как sensitive (❗❗например, пароли, ключи, токены❗❗)."

### Таймаут операции (унаследован от `AbstractOptions`)
```java
// Устанавливает таймаут в миллисекундах.
public DescribeConfigsOptions timeoutMs(Integer timeoutMs)

// Возвращает установленное значение таймаута.
public Integer timeoutMs()
```

> Говорит Kafka Admin API:
> "Если брокеры не ответят в течение 5000 мс — прекращай ждать и бросай `TimeoutException`."

---
## 💡 Особенности и подводные камни 💡

1️⃣ ❗❗❗`DescribeConfigsOptions` управляет **только параметрами запроса**, а не самим результатом.  

2️⃣ 💡Если `includeSensitive(true)` используется без соответствующих прав ACL — будет `AuthorizationException`.

3️⃣ 💡`includeSynonyms(true)` может вернуть **множество дублирующих ключей**, если значение переопределено на разных уровнях (например, topic и broker).  

4️⃣ ❗❗❗Операция `describeConfigs()` выполняется асинхронно — реальный результат доступен через `KafkaFuture`.  

5️⃣ ⭐⭐Если описываются несколько ресурсов (например, Topic и Broker), результат будет мапой `ConfigResource → Config`.

---
## 💎 Best Practices 💎

✅ Используй `includeSynonyms(true)` при отладке или аудите конфигураций.  

✅ ❗❗❗Не включай `includeSensitive(true)` в production-логах. 

✅ Устанавливай разумный таймаут через `.timeoutMs(5000)` — иначе запрос может зависнуть при медленных брокерах. 

✅ Работай через `KafkaFuture.whenComplete()` для асинхронной обработки результата.  

✅ Используй `ConfigResource` с корректным типом (`ConfigResource.Type.TOPIC`, `BROKER` и т.д.).

## Пример
```java
import org.apache.kafka.clients.admin.*;
import org.apache.kafka.common.config.ConfigResource;

import java.util.*;
import java.util.concurrent.ExecutionException;

public class DescribeConfigsExample {
    public static void main(String[] args) throws ExecutionException, InterruptedException {
        Properties props = new Properties();
        props.put(
	        AdminClientConfig.BOOTSTRAP_SERVERS_CONFIG, 
	        "localhost:9092");

        try (AdminClient admin = AdminClient.create(props)) {
            // Описываем конфигурацию конкретного топика
            ConfigResource topicResource = new ConfigResource(
	            ConfigResource.Type.TOPIC, 
	            "logs");

            // Настройки операции
            DescribeConfigsOptions options = new DescribeConfigsOptions()
                    .includeSynonyms(true)
                    .includeSensitive(false)
                    .timeoutMs(5000);

            // Запускаем операцию
            DescribeConfigsResult result = 
				admin.describeConfigs(
					Collections.singleton(topicResource), 
					options
				);

            // Получаем результат (блокирующий)
            Map<ConfigResource, Config> configs = 
	            result.all().get();

            // Выводим конфигурации
            configs.forEach((resource, config) -> {
                System.out.println(
	                "Конфигурация ресурса: " + resource
	            );
                config.entries().forEach(entry -> {
                    System.out.printf(
	                    "  %s = %s (источник: %s)\n",
						entry.name(),
						entry.value(),
						entry.source());
                });
            });
        }
    }
}
```
