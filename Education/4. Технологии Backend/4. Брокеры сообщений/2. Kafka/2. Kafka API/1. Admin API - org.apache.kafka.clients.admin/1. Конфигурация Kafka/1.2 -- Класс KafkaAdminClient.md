> [!NOTE] `KafkaAdminClient`
> Это **реализация интерфейса Admin**, предоставляющая клиентский API для административного управления кластером Kafka. (БЕЗ ДОП МЕТОДОВ)
> 
> Через этот класс можно **создавать топики, управлять ACL, пользователями, конфигурацией, квотами, группами, транзакциями и т.д.**
> 
> 💡Используется в продакшн-коде редко напрямую💡, так как ==обычно применяется **фабричный метод== `Admin.create()`, который внутри создаёт экземпляр `KafkaAdminClient`.

## Конструктор
```java
// Конструктор приватный, т.к. создание экземпляра KafkaAdminClient контролируется фабричными методами.
// Это гарантирует корректную инициализацию внутренних ресурсов (сетевые клиенты, таймауты, thread pool).
private KafkaAdminClient(
	AdminClientConfig config,
    String clientId,
    Time time,
    KafkaClient kafkaClient,
    Metadata metadata,
    Metrics metrics,
    TimeoutProcessorFactory timeoutProcessorFactory) {}
```

### Фабричные методы создания экземпляра
```java
// Создаёт новый экземпляр AdminClient (внутри — KafkaAdminClient).
// Параметр config — карта конфигураций (bootstrap.servers, security и т.д.)
public static Admin create(Map<String, Object> config)

// Перегрузка метода create — позволяет использовать готовый AdminClientConfig.
public static Admin create(Properties properties)
```
- `create()` — стандартный способ получить экземпляр клиента.
- Внутри вызывается приватный конструктор `KafkaAdminClient`. 
- Возвращает объект интерфейса `Admin`, но фактически это `KafkaAdminClient`.

## Методы
`KafkaAdminClient` — это **конкретная реализация интерфейса Admin**, и **все его публичные методы — это реализации методов интерфейса**.
