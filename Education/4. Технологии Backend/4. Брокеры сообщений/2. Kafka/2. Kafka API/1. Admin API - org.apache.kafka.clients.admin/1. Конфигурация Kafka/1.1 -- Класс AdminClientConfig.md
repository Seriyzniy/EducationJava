
> [!NOTE] `AdminClientConfig` — это **конфигурационный класс**  клиента Admin.  
> 
> Он наследуется от базового `AbstractConfig` (через `AdminConfig`) и отвечает за:
> - чтение, 
> - проверку, 
> - валидацию и 
> - хранение параметров конфигурации администратора Kafka.
> 
> ⭐Чаще ==используются ЕГО ПОЛЯ==, чем методы⭐


## Конструктор
```java
// Основной публичный конструктор.
// Используется для создания конфигурации AdminClient из Map или Properties.
public AdminClientConfig(Map<?, ?> originals) {
    super(CONFIG, originals);
}

// Вспомогательный конструктор с указанием "дополнительных" настроек и опции doLog.
public AdminClientConfig(Map<?, ?> originals, boolean doLog) {
    super(CONFIG, originals, doLog);
}

// Приватное статическое поле CONFIG (ConfigDef) содержит схему всех доступных параметров
// (их имена, типы, значения по умолчанию, документацию и т.п.)
private static final ConfigDef CONFIG = new ConfigDef()
    // Внутри определяется набор ключей конфигурации — bootstrap.servers, client.id и др.
    ...
;
```

---
## Основные параметры конфигурации (ключевые поля)
`AdminClientConfig` определяет **набор констант (public static final String)** — имена всех поддерживаемых настроек.

### Базовые сетевые настройки
```java
public static final String BOOTSTRAP_SERVERS_CONFIG = "bootstrap.servers"; 
// Список брокеров для подключения (host:port)

public static final String CLIENT_ID_CONFIG = "client.id";
// Идентификатор клиента (по умолчанию — adminclient-<UUID>)

public static final String SECURITY_PROTOCOL_CONFIG = "security.protocol";
// Протокол безопасности (PLAINTEXT, SSL, SASL_PLAINTEXT, SASL_SSL)
```

### Таймауты и ожидания
```java
public static final String REQUEST_TIMEOUT_MS_CONFIG = "request.timeout.ms";
// Максимальное время ожидания ответа брокера (по умолчанию 120000 мс)

public static final String DEFAULT_API_TIMEOUT_MS_CONFIG = "default.api.timeout.ms";
// Таймаут по умолчанию для API-вызовов (если не указан отдельно)

public static final String RETRIES_CONFIG = "retries";
// Количество повторных попыток при временных ошибках
```

### Безопасность
```java
public static final String SASL_MECHANISM_CONFIG = "sasl.mechanism";
// Тип механизма SASL (PLAIN, SCRAM-SHA-256, GSSAPI и др.)

public static final String SSL_KEYSTORE_LOCATION_CONFIG = "ssl.keystore.location";
// Путь к keystore (если SSL используется)

public static final String SSL_TRUSTSTORE_PASSWORD_CONFIG = "ssl.truststore.password";
// Пароль к truststore
```

### Потоки и производительность
```java
public static final String MAX_IN_FLIGHT_REQUESTS_PER_CONNECTION = "max.in.flight.requests.per.connection";
// Максимум незавершённых запросов в одном соединении

public static final String METADATA_MAX_AGE_CONFIG = "metadata.max.age.ms";
// Как часто обновлять метаданные кластера
```

## Методы
Класс `AdminClientConfig` **наследует все методы** от `AbstractConfig`, но добавляет несколько **своих** — в основном для получения и описания конфигурации.

### Доступ к параметрам
```java
// Возвращает значение свойства по имени (с приведением к типу)
public <T> T getConfiguredInstance(String key, Class<T> t)

// Возвращает список всех конфигурационных параметров
public static ConfigDef configDef()
```

### Методы-наследники (из AbstractConfig)
```java
// Проверяет наличие параметра
public boolean hasKey(String key)

// Возвращает значение параметра (в типизированном виде)
public <T> T get(String key)

// Возвращает карту всех настроек
public Map<String, Object> originals()
```

## 💡 Особенности и подводные камни 💡

1️⃣ `AdminClientConfig` **не валидирует наличие всех обязательных параметров сразу** — например, `bootstrap.servers` проверяется позже, при создании `KafkaAdminClient`.  

2️⃣ Некоторые параметры **унаследованы** от `CommonClientConfigs` (KafkaProducer, KafkaConsumer), поэтому могут показаться “лишними”.  

3️⃣ 💡Параметры безопасности (`SASL_`, `SSL_`) могут конфликтовать, если заданы неверно — например, указание `SASL_SSL` без truststore приведёт к `ConfigException`.  

4️⃣ Если не задать `client.id`, KafkaAdminClient сам сгенерирует UUID, что может осложнить отладку.  

5️⃣ Некоторые параметры (`retries`, `request.timeout.ms`) влияют на внутренние retry-механизмы — важно не ставить слишком малые значения.

## 💎 Best Practices 💎

✅ ⭐**Минимальный набор для подключения:**⭐
```
bootstrap.servers=host1:9092,host2:9092 
client.id=admin-client 
request.timeout.ms=30000
```

✅ 💡**Безопасное подключение:**  
Указывать явно `security.protocol` и SSL/SASL параметры, если кластер защищён.

✅ ⭐**Контролируй таймауты:**  
`default.api.timeout.ms` должен быть разумным (30–60 сек), чтобы операции вроде `describeCluster()` не падали на медленных сетях.

✅ ❗❗❗**Не смешивай конфигурации разных клиентов:**  
Не передавай сюда параметры от `ProducerConfig` или `ConsumerConfig`, если они не указаны в документации `AdminClientConfig`.

✅ **Храни конфиги централизованно** (например, через `.properties` файл) — удобно для CI/CD.

## Пример
```java
import org.apache.kafka.clients.admin.*;
import java.util.*;

public class AdminConfigExample {
    public static void main(String[] args) {
        Map<String, Object> config = new HashMap<>();
        config.put(
	        AdminClientConfig.BOOTSTRAP_SERVERS_CONFIG,
	        "localhost:9092"
	    );
        config.put(
	        AdminClientConfig.CLIENT_ID_CONFIG, 
	        "admin-demo"
	    );
        config.put(
	        AdminClientConfig.REQUEST_TIMEOUT_MS_CONFIG,
	        "20000"
	    );

        // ⭐Создаём конфигурацию⭐
        AdminClientConfig adminConfig = new AdminClientConfig(config);

        // ⭐Используем её для создания Admin-клиента⭐
        try (Admin admin = Admin.create(config)) {
            System.out.println(
	            "Cluster ID: " + 
	            admin.describeCluster().clusterId().get()
	        );
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

```
