# 1. Exchange
Маршрутизатор сообщений. Создается с параметрами `exchangeDeclare`

**Ключевые параметры:**
- `exchange`: Имя обменника (строка).
    
- `type`: Тип обменника:
    - `direct` (по умолчанию): Сообщение идет в очередь, чей `routing_key` точно совпадает.
        
    - `fanout`: Рассылка во **все** привязанные очереди (игнорирует `routing_key`).
        
    - `topic`: Паттернная маршрутизация по `routing_key` (с поддержкой `*` и `#`).
        
    - `headers`: Маршрутизация по заголовкам из свойства `headers` (игнорирует `routing_key`).
        
- `durable`: `true` — обменник переживет перезагрузку брокера. `false` — временный.
    
- `auto_delete`: `true` — обменник удалится автоматически, когда ни одна очередь не будет к нему привязана.
    
- `internal`: `true` — обменник нельзя использовать для прямой публикации, только для связки с другими обменниками (binding).
    
- `arguments`: Дополнительные аргументы (Map). Например, `alternate-exchange` — обменник для "потерянных" сообщений.

# 2. Queue (Очередь)
Контейнер для сообщений. Создается с параметрами `queueDeclare`.

**Ключевые параметры:**
- `queue`: Имя очереди (строка). Если пустая строка `""`, брокер создаст уникальное имя и вернет его.
    
- `durable`: `true` — очередь (и **persistent** сообщения в ней) переживет перезагрузку брокера. Важно: durable очередь ≠ persistent сообщения. Нужно задать оба параметра.
    
- `exclusive`: `true` — очередь используется только одним соединением и удалится при его закрытии. Подходит для временных приватных очередей (например, для callback в RPC).
    
- `auto_delete`: `true` — очередь удалится, когда от нее отпишется последний потребитель.
    
- `arguments`: Самый мощный параметр. Map с дополнительными настройками:
    - `"x-message-ttl"`: Время жизни сообщений в очереди (в мс), например `60000`. Сообщение удалится, если не будет потреблено за это время.
        
    - `"x-expires"`: Время жизни самой очереди (в мс), если она не используется.
        
    - `"x-max-length"`: Максимальное количество сообщений в очереди. Старые будут вытесняться (dead-lettered).
        
    - `"x-max-length-bytes"`: Максимальный общий размер сообщений в очереди.
        
    - `"x-overflow"`: Поведение при переполнении (`reject-publish` или `reject-publish-dlx`).
        
    - `"x-dead-letter-exchange"`: Имя обменника, куда будут отправляться "мертвые" сообщения (отклоненные, просроченные, вытесненные).
        
    - `"x-dead-letter-routing-key"`: Ключ маршрутизации для "мертвых" сообщений.
        
    - `"x-single-active-consumer"`: `true` — позволяет иметь только одного активного потребителя, остальные будут в режиме standby (HA для конкурентных потребителей).
        
    - `"x-queue-mode"`: `"lazy"` — сообщения хранятся сразу на диске, минимизируя использование RAM (для очень длинных очередей).

# 3. Consumer (Потребитель)
Приложение, которое получает и обрабатывает сообщения. Настраивается в `basicConsume`.

**Основные параметры подписки:**
- `queue`: Имя очереди, из которой потреблять.
    
- `autoAck`: Флаг (булевый).
    - `true` (**автоматическое подтверждение**): Сообщение считается успешно доставленным сразу после отправки потребителю. Риск потери при падении процесса-потребителя.
        
    - `false` (**ручное подтверждение**): Потребитель должен явно отправить брокеру `basicAck` (подтверждение) или `basicNack` (отказ). Это гарантирует доставку "at-least-once".
        
- `consumerTag`: Уникальный идентификатор потребителя (строка). Можно задать свой или позволить брокеру сгенерировать.
    
- `noLocal`: `true` — соединение не будет получать сообщения, опубликованные **им же самим**. Актуально не для всех клиентов.
    
- `exclusive`: `true` — доступ к очереди будет только у этого потребителя.
    
- `arguments`: Дополнительные аргументы подписки.
    
- `callback` (DeliveryCallback): Объект, метод которого вызывается при получении сообщения. Содержит тело (`body`) и envelope с метаданными (`deliveryTag`, `routingKey`, `redeliver` и др.).


**Методы работы в режиме ручного подтверждения (autoAck=false):**
- `basicAck(deliveryTag, multiple)`: Подтвердить обработку конкретного сообщения (`deliveryTag`). Если `multiple=true`, подтверждаются все сообщения с тегами до указанного включительно.
    
- `basicNack(deliveryTag, multiple, requeue)`: Явно отказаться от сообщения.
    - `requeue=true` — сообщение вернется в очередь (риск циклов).
        
    - `requeue=false` — сообщение будет удалено или отправлено в Dead Letter Exchange.
        
- `basicReject(deliveryTag, requeue)`: Упрощенный `basicNack` только для одного сообщения.
    
- `basicQos(prefetchCount, prefetchSize, global)`: **Критически важная настройка** для распределения нагрузки.
    - `prefetchCount` — максимальное количество неподтвержденных сообщений, которые брокер отправит этому потребителю _за раз_. Ставьте `1` для строгого чередования или небольшое число (например, `10-100`) для балансировки.
        
    - `global` — применяется ли лимит на все соединение или только на канал.
