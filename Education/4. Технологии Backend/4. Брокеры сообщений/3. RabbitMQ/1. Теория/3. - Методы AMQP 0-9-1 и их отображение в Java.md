# 1. _class_ `basic.*` (работа с сообщениями)
Это самый важный набор методов — именно они управляют publish/consume.
## 👉 Публикация / получение
- **basic.publish** — отправка сообщения брокеру
    
- **basic.get** — синхронное получение сообщения (pull-модель)
    
- **basic.consume** — подписка на очередь (push-модель)
    
- **basic.cancel** — отмена подписки

## 👉 Подтверждения доставки
- **basic.ack** — положительное подтверждение
    
- **basic.nack** — отрицательное подтверждение (с requeue)
    
- **basic.reject** — отрицательное подтверждение (старый вариант nack, с requeue)

## 👉 Управление потоком сообщений
- **basic.qos** — настройка prefetch
    
- **basic.recover** — восстановить недоставленные сообщения
    
- **basic.recover-async** — асинхронное восстановление

## 👉 Гарантии публикации (publisher confirms)
- **basic.return** — сообщение вернулось издателю (если mandatory=1)
    
- **basic.deliver** — доставка сообщения потребителю
    
- **basic.get-ok** / **basic.get-empty** — результат команды basic.get

# 2. _class_ `queue.*` (операции с очередями)
Работа с очередями как объектами.
- **queue.declare** — создать очередь
    
- **queue.bind** — привязать очередь к exchange
    
- **queue.unbind** — отвязать
    
- **queue.purge** — очистить очередь
    
- **queue.delete** — удалить очередь
    
- **queue.declare-ok**, **queue.bind-ok**, … — ответы брокера
    

---
# 3. _class_ `exchange.*` (операции с exchange)
Работа с обменниками.
- **exchange.declare**
    
- **exchange.delete**
    
- **exchange.bind**
    
- **exchange.unbind**
    
- соответствующие `*-ok` методы
    

---
# 4. _class_ `channel.*` (жизненный цикл канала)
Работа с AMQP-каналом внутри соединения.
- **channel.open**
    
- **channel.close**
    
- **channel.flow** — устаревший механизм управления потоком
    
- ответы: **channel.open-ok**, **channel.close-ok**

- **connection.secure**
    
- **connection.secure-ok**
    
- **connection.tune**
    
- **connection.tune-ok**
    
- **connection.open**
    
- **connection.open-ok**

---
# 6. _class_ `confirm.*` (Publisher Confirms)
Используются, когда канал переключён в режим подтверждений публикаций.
- **confirm.select** — включить publisher confirms
    
- **confirm.select-ok**
    
- **basic.ack** / **basic.nack** — для подтверждения публикации брокером
    

(Confirm API встроен в basic-методы, но логически — отдельная подсистема.)

---
# 7. _class_ `tx.*` (транзакции)
Очень редко используются (и не рекомендуются в RabbitMQ из-за тормозов).
- **tx.select**
    
- **tx.select-ok**
    
- **tx.commit**
    
- **tx.commit-ok**
    
- **tx.rollback**
    
- **tx.rollback-ok**
    

---
# 8. _class_ `access.*` (устаревшее)

В современных RabbitMQ не используется.

# 9. ОТОБРАЖЕНИЕ В JAVA
**Методы AMQP (`basic.nack`, `basic.ack`, `basic.reject`, `exchange.declare` и т.д.) существуют в Java-клиенте**, но **не один-в-один названиями**, ==а представлены как методы класса== `Channel` в RabbitMQ Java Client.

То есть в Java ты не вызываешь строку `"basic.nack"`, а вызываешь метод:
```java
channel.basicNack(tag, multiple, requeue);
```

## Как AMQP-методы отображаются в Java-клиенте

### ✔ Методы для подтверждений
|AMQP метод|Java метод|
|---|---|
|`basic.ack`|`channel.basicAck(deliveryTag, multiple)`|
|`basic.nack`|`channel.basicNack(deliveryTag, multiple, requeue)`|
|`basic.reject`|`channel.basicReject(deliveryTag, requeue)`|

---
### ✔ Методы потребления

| AMQP метод      | Java метод / поведение                      |
| --------------- | ------------------------------------------- |
| `basic.consume` | `channel.basicConsume(queue, autoAck, ...)` |
| `basic.cancel`  | `channel.basicCancel(consumerTag)`          |
| `basic.get`     | `channel.basicGet(queue, autoAck)`          |

---
### ✔ Методы публикации

|AMQP метод|Java метод|
|---|---|
|`basic.publish`|`channel.basicPublish(exchange, routingKey, props, body)`|
|`confirm.select`|`channel.confirmSelect()`|
|`tx.select`|`channel.txSelect()`|
|`tx.commit`|`channel.txCommit()`|
|`tx.rollback`|`channel.txRollback()`|

---
### ✔ Методы очередей

|AMQP метод|Java метод|
|---|---|
|`queue.declare`|`channel.queueDeclare(...)`|
|`queue.bind`|`channel.queueBind(queue, exchange, key)`|
|`queue.unbind`|`channel.queueUnbind(queue, exchange, key)`|
|`queue.delete`|`channel.queueDelete(queue)`|
|`queue.purge`|`channel.queuePurge(queue)`|

---
### ✔ Методы обменников

|AMQP метод|Java метод|
|---|---|
|`exchange.declare`|`channel.exchangeDeclare(...)`|
|`exchange.bind`|`channel.exchangeBind(dst, src, key)`|
|`exchange.unbind`|`channel.exchangeUnbind(dst, src, key)`|
|`exchange.delete`|`channel.exchangeDelete(name)`|
