**Издатели (publishers) могут ожидать два типа ошибок:**
1. **Сетевое I/O исключение** — из-за неудачной записи или таймаута; может возникнуть сразу или с задержкой. При блокировке соединения из-за сигнала тревоги публикация продолжится только после его снятия.
    
2. **Таймаут доставки подтверждения** — возникает, если разработчик установил таймаут; он не должен быть меньше эффективного таймаута heartbeat.

> Замечание: «исключение» здесь — общий термин; на некоторых языках ошибки обрабатываются иначе.

В основном, логика обработки ошибок возлагается на разработчика. Ему решать, как обрабатывать ошибки

## **Эффекты сигналов тревоги ресурсов (Effects of Resource Alarms)**
**Resource Alarm** — это сигнал, который активирует RabbitMQ, когда узел кластера испытывает нехватку ресурсов, например:
- переполнена память
- заполнен диск

Когда такой сигнал активен, узел **блокирует публикацию сообщений** на всех соединениях, чтобы предотвратить потерю данных или аварийное завершение работы.

### Что происходит с соединениями
1. Все соединения, которые пытаются **публиковать сообщения**, блокируются.
    
2. На заблокированном соединении **данные не читаются и не обрабатываются**.
    
3. После снятия тревоги обработка трафика **возобновляется автоматически**.

### Уведомление клиентов
- Клиенты AMQP 0-9-1 получают уведомления, когда соединение **заблокировано** и когда **разблокировано**.
    
- Любые попытки записи на заблокированное соединение **заканчиваются ошибкой I/O или таймаутом**.

# 1. Практическое применение
В RabbitMQ Java Client обработка исключений/событий делится на два уровня:
### 1. **Уровень Connection**
Тут регистрируются слушатели глобальных событий:
- `ShutdownListener` — если соединение/канал закрылись ошибкой
    
- `BlockedListener` — если соединение заблокировано/разблокировано (Resource Alarm)
    

### 2. **Уровень Channel**
Тут регистрируются:
- `ConfirmCallback` — обработчики publisher confirms (`ack`/`nack`)
    
- `ReturnListener` — сообщения, что не смогли быть маршрутизированы (`mandatory`)
    
- `ShutdownListener` — если канал закрылся по ошибке

## Примеры
### Создание соединения с обработкой Block / Unblock
```java
ConnectionFactory factory = new ConnectionFactory();
factory.setHost("localhost");

Connection connection = factory.newConnection();

// Слушатель блокировок (resource alarms)
connection.addBlockedListener(new BlockedListener() {
    @Override
    public void handleBlocked(String reason) {
        System.out.println("CONNECTION BLOCKED: " + reason);
    }

    @Override
    public void handleUnblocked() {
        System.out.println("CONNECTION UNBLOCKED");
    }
});
```

### Создание канала с обработкой подтверждений (acks/nacks)
```java
Channel channel = connection.createChannel();

// Включаем publisher confirms
channel.confirmSelect();

// Map для отслеживания сообщений
ConcurrentNavigableMap<Long, String> outstanding = new ConcurrentSkipListMap<>();

// Обработчик позитивных подтверждений
ConfirmCallback ackCallback = (seqNo, multiple) -> {
    outstanding.headMap(seqNo, true).clear();
};

// Обработчик негативных подтверждений
ConfirmCallback nackCallback = (seqNo, multiple) -> {
    String msg = outstanding.remove(seqNo);
    System.out.println("NACK for msg: " + msg);
    // здесь можно выполнить retry
};

channel.addConfirmListener(ackCallback, nackCallback);
```

### ReturnListener (если mandatory=true)
```java
channel.addReturnListener(
	(replyCode, replyText, exchange, routingKey, properties, body) -> {
	    System.out.println("RETURNED MSG: " + new String(body));
});
```

### Обработка ошибок записи (I/O exceptions)
Публикация оборачивается в try/catch:
```java
try {
    long seqNo = channel.getNextPublishSeqNo();
    outstanding.put(seqNo, "my-message-data");

    channel.basicPublish(
            "ex", 
            "key",
            true,  // mandatory
            MessageProperties.PERSISTENT_TEXT_PLAIN,
            "Hello".getBytes()
    );

} catch (IOException io) {
    System.out.println("I/O error while publishing: " + io.getMessage());
    // тут логика retry / reconnect
}
```

## Что ты ОБЯЗАН обработать в Publisher
|Событие|Как обрабатывается|
|---|---|
|**Publisher confirms ack/nack**|`channel.addConfirmListener()`|
|**Resource alarms (blocked/unblocked)**|`connection.addBlockedListener()`|
|**Delivery failure (mandatory)**|`channel.addReturnListener()`|
|**Сбой канала/соединения**|`addShutdownListener()` на `Channel` и `Connection`|
|**I/O ошибки записи**|try/catch вокруг `basicPublish`|
|**Timeout ожидания подтверждений**|`waitForConfirmsOrDie(timeout)`|
