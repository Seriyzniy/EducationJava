# 1. Свойства сообщений
## 1. AMQP 0-9-1 
### Свойства маршутизации
Следующие свойства относятся к деталям **доставки и маршрутизации**

Они не являются свойствами сообщения как такового и **устанавливаются RabbitMQ во время маршрутизации и доставки**:

|Свойство|Тип|Описание|
|---|---|---|
|**Delivery tag**|Положительное целое число|**Идентификатор доставки**, см. раздел "Подтверждения" (Confirms).|
|**Redelivered**|Логическое значение (Boolean)|Устанавливается в **`true`**, если это сообщение было доставлено ранее и возвращено в очередь (requeued).|
|**Exchange**|Строка (String)|**Обменник**, который маршрутизировал это сообщение.|
|**Routing key**|Строка (String)|**Ключ маршрутизации**, использованный издателем.|
|**Consumer tag**|Строка (String)|**Идентификатор потребителя (подписки)**.|

Многие из них необязательные. 

Их устанавливает Producer при отправке сообщений

| Свойство             | Тип                                | Описание                                                                                                                                                                                     | Обязательное? |
| -------------------- | ---------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------- |
| **Delivery mode**    | Перечисление (1 или 2)             | **2** для "`persistent`" (сохраняемые), <br>**1** для "`transient`" (временные). <br><br>Некоторые клиентские библиотеки представляют это свойство как логическое значение или перечисление. | **Да**        |

#### Тип сообщений
**type** — произвольная строка, указываемая издателем для обозначения рода сообщения.

- **RabbitMQ не проверяет и не использует это поле**; оно служит для приложений и плагинов.
    
- **Обычно используют точечную нотацию**: `orders.created`, `logs.line`, `profiles.image.changed`.
    
- Если Consumer получает сообщение с неизвестным типом — рекомендуется логировать событие для диагностики.


#### **Тип содержимого и кодировка**
Поля **content-type*** *(MIME media type)* и **content-encoding** сообщают Consumer, как десериализовать и декодировать payload.

- RabbitMQ их не использует — они предназначены для приложений и плагинов.
    
- Примеры: `content-type = application/json`, `content-encoding = gzip`.
    
- Несколько кодировок можно перечислять через запятую.
