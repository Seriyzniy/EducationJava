В AMQP 0-9-1 есть manual ack.  
`basic.reject` может отклонять **только одно** сообщение.  

RabbitMQ расширяет это, добавляя **basic.nack**, который умеет:

**1) Отклонять сообщения массово**
Если `multiple = true` в методе `basic.nack`, то отклоняются **все неподтверждённые сообщения до указанного `delivery_tag`** метода `basic.nack`.  
Это аналог массового `basic.ack`, только в обратную сторону.

**2) Делать всё то же, что `basic.reject`**
- отклонение одного сообщения
- указание: **requeue или drop**

**3) Работает для любых потребителей**
- долгоживущих (long-running)
- polling (`basic.get`)

**4) Поведение requeue**
Если сообщение возвращается в очередь:
- оно пытается попасть **на исходную позицию**
    
- но если очередь параллельно доставляет другим потребителям, то сообщение попадёт **как можно ближе к началу**, но точное место не гарантируется

## Примеры
Этот пример для клиента Java отклоняет **одно** сообщение, полученное через опрос (`basic.get`), с запросом к брокеру повторно поставить его в очередь:

```java
GetResponse gr = channel.basicGet("some.queue", false);

channel.basicNack(gr.getEnvelope().getDeliveryTag(), false, true);
```

Этот пример отклоняет **два** сообщения с помощью одного вызова брокеру (второй аргумент в `basicNack` — это флаг `multiple`):
```java
GetResponse gr1 = channel.basicGet("some.queue", false);
GetResponse gr2 = channel.basicGet("some.queue", false);

channel.basicNack(gr2.getEnvelope().getDeliveryTag(), true, true);
```
