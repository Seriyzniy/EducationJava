# **Категория 1: Основное API (Используется постоянно)**
Эти пакеты — основа любого приложения, работающего с RabbitMQ. С них нужно начинать.

| Пакет                  | Описание на русском                                                                                                       | Ключевые элементы (для понимания)                                                                                   |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------- |
| `com.rabbitmq.client`✅ | **Основное клиентское API:** классы и интерфейсы, представляющие AMQP-соединения, каналы и дескрипторы фреймов протокола. | `ConnectionFactory`, <br>`Connection`,<br>`Channel`,<br>`Consumer`,<br>`DefaultConsumer`,<br>`AMQP.BasicProperties` |

---
# **Категория 2: Внутренние реализации и расширенные функции**
Эти пакеты **важны для настройки, оптимизации и понимания внутреннего устройства**. 
> Прямо с них начинать не нужно, но к ним часто обращаются.

| ПодПакеты<br>`com.rabbitmq.client.` | Описание на русском                                                                                                  | Когда используется?                                                                         |
| ----------------------------------- | -------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------- |
| **`impl`**                          | **Реализации интерфейсов**, указанных в основном API, и вспомогательные классы.                                      | При глубоком анализе или необходимости кастомизации поведения, заложенного в API.           |
| **`impl.nio`**                      | **NIO сетевой коннектор.** Альтернатива стандартному (блокирующему) вводу-выводу для большого количества соединений. | Для снижения нагрузки на потоки при высокой нагрузке (`NioParams`, `NioConnectionFactory`). |
| **`impl.recovery`**                 | **Реализация восстановления соединения и топологии.** Основа для автоматического восстановления после сбоя сети.     | Всегда, когда включен Automatic Recovery (включен по умолчанию в современных версиях).      |

# **Категория 3: Наблюдаемость (Observability)**
Пакеты **для мониторинга и трассировки** работы клиента. 

Становятся критичными в production-средах.

| ПодПакеты <br>`com.rabbitmq.client.` | Описание на русском                                                                                                 | Когда используется?                                                                                  |
| ------------------------------------ | ------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------- |
| **`observation`**                    | **Абстракции для наблюдаемости.** Интерфейсы для интеграции с системами трассировки (например, Micrometer Tracing). | Для кастомизированной трассировки операций RabbitMQ.                                                 |
| **`observation.micrometer`**         | **Реализация наблюдаемости на основе Micrometer.** Готовая интеграция с экосистемой Micrometer.                     | Для автоматического сбора метрик и трассировок в приложениях на Spring Boot Actuator или Micrometer. |

# **Категория 4: Вспомогательные утилиты и инструменты**
Специализированные **пакеты для конкретных задач**. ==Используются по мере необходимости.==

| Пакет                            | Описание на русском                                                                       | Когда используется?                                                                                                                  |
| -------------------------------- | ----------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| **`com.rabbitmq.tools`**         | **Неосновные утилиты и инструменты администрирования.**                                   | Для однократных операций (например, `tracer` для логирования фреймов) или утилитарных классов.                                       |
| **`com.rabbitmq.tools.json`**    | **JSON reader/writer и вспомогательные классы.**                                          | ==При работе с сообщениями в формате JSON (==внутренние нужды клиента).                                                              |
| **`com.rabbitmq.tools.jsonrpc`** | **Клиентские и серверные классы JSON-RPC** для поддержки JSON-RPC поверх AMQP-транспорта. | ==Для реализации RPC== поверх RabbitMQ с использованием устаревшего JSON-RPC протокола (==не путать с обычным RPC-паттерном AMQP==). |
| **`com.rabbitmq.utility`**       | **Пакет вспомогательных классов**, в основном используемых в коде реализации.             | Редко, в основном внутренними разработчиками библиотеки.                                                                             |
