
> [!NOTE] Класс `ConnectionFactory`
> **удобный фабричный класс** для конфигурации и создания **соединений (`Connection`) с узлом RabbitMQ**. 
> 
> ==Все параметры==, влияющие на соединение (хост, порт, виртуальный хост, креды, SSL, таймауты и т. д.), ==конфигурируются на экземпляре== `ConnectionFactory`, а затем через него создаются подключения к брокеру.

**Реализует интерфейсы:**  
✔️ `Cloneable` — позволяет клонировать объекты для использования в нескольких местах с сохранением конфигурации.

Эти **значения можно изменять через сеттеры** перед созданием соединения. [RabbitMQ](https://www.rabbitmq.com/client-libraries/java-api-guide?utm_source=chatgpt.com)

## Настройка и создание соединений
```java

        throws IOException, TimeoutException

        throws IOException, TimeoutException

        throws IOException, TimeoutException
```

💬 **Перевод:**  
Методы создают _новое соединение с RabbitMQ_.

- С этими параметрами можно указать **имя соединения**, которое будет видно в логах и UI, или собственный `ExecutorService`, управляющий потоками.

- включают проверку имени сервера при TLS.

# Особенности и подводные камни 💡
1️⃣ **`ConnectionFactory` не создает соединения в конструкторе** — _только конфигурирует параметры_. Соединение создаётся при вызове `newConnection()`. [RabbitMQ](https://www.rabbitmq.com/client-libraries/java-api-guide?utm_source=chatgpt.com) 

2️⃣ Метод `newConnection(...)` может бросать **IOException** и **TimeoutException** — это стандартные ошибки сетевых/протокольных проблем. [RabbitMQ](https://rabbitmq.github.io/rabbitmq-java-client/api/current/com/rabbitmq/client/ConnectionFactory.html?utm_source=chatgpt.com)  

3️⃣ Установка SSL через `useSslProtocol(...)` без проверки хоста удобна для разработки, но **опасна в проде** (нет проверки подлинности сервера). [RabbitMQ](https://rabbitmq.github.io/rabbitmq-java-client/api/current/com/rabbitmq/client/ConnectionFactory.html?utm_source=chatgpt.com)  

4️⃣ Свойство `clientProperties` может быть полезно для **оперативного мониторинга**, но не должно использоваться для бизнес-логики. [RabbitMQ](https://rabbitmq.github.io/rabbitmq-java-client/api/current/com/rabbitmq/client/ConnectionFactory.html?utm_source=chatgpt.com)  

5️⃣ **TLS через SocketFactory** действует только для блокирующего IO; в NIO/Netty нужен иной API.

6️⃣ Наличие overload'ов `newConnection` с `ExecutorService` позволяет контролировать потоки работы, но требует правильного завершения executor-ов. [RabbitMQ](https://rabbitmq.github.io/rabbitmq-java-client/api/current/com/rabbitmq/client/ConnectionFactory.html?utm_source=chatgpt.com)

# 💎 Best Practices 💎

✅ Создавать **один `ConnectionFactory` на приложение** и использовать его многократно. [RabbitMQ](https://www.rabbitmq.com/client-libraries/java-api-guide?utm_source=chatgpt.com) 

✅ Перед `newConnection()` всегда настраивать нужные параметры (TLS, имя, креды). [RabbitMQ](https://www.rabbitmq.com/client-libraries/java-api-guide?utm_source=chatgpt.com)  

✅ Использовать `newConnection(String connectionName)` для понятного идентификатора в логах и UI. [RabbitMQ](https://www.rabbitmq.com/client-libraries/java-api-guide?utm_source=chatgpt.com)  

✅ В проде предпочитать `useSslProtocol(SSLContext)` + `enableHostnameVerification()` для безопасности. [RabbitMQ](https://rabbitmq.github.io/rabbitmq-java-client/api/current/com/rabbitmq/client/ConnectionFactory.html?utm_source=chatgpt.com)  

✅ Если нужно много соединений, управлять thread-pool через `ExecutorService`.

# Пример использования
```java
ConnectionFactory factory = new ConnectionFactory();
factory.setHost("rabbitmq.local");
factory.setUsername("appUser");
factory.setPassword("secret");
factory.setVirtualHost("vhost1");

// Включить TLS с проверкой имени сервера
SSLContext sslContext = SSLContext.getInstance("TLSv1.2");
sslContext.init(null, null, null);
factory.useSslProtocol(sslContext);
factory.enableHostnameVerification();

// Имя соединения полезно для логов/мониторинга
Connection connection = factory.newConnection("order-service-conn1");
Channel channel = connection.createChannel();
// ... работа с каналом
```
