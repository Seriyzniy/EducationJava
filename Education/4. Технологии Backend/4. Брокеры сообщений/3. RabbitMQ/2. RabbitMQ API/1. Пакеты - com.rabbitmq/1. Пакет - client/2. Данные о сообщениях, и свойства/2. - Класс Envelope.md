
> [!NOTE] Класс `Envelope`
> **Envelope** - это неизменяемый (immutable) класс, который инкапсулирует метаданные о доставке сообщения. 
> 
> Он содержит информацию о том, как сообщение было доставлено: через какой обменник, с каким ключом маршрутизации, и уникальный идентификатор доставки (delivery tag).

**Зачем нужен:** Для получения метаданных о доставленном сообщении. Содержит критически важную информацию, такую как `deliveryTag`, который используется для подтверждения (ack) или отклонения (nack/reject) сообщения.

**Не реализует никаких интерфейсов** - это простой POJO (Plain Old Java Object) класс.

# Конструктор
```java
/**
 * Construct an {@link Envelope} with the given parameters.
 *
 * @param deliveryTag the delivery tag
 * @param redeliver true if this is a redelivery
 * @param exchange the exchange used for the publishing
 * @param routingKey the routing key used for the publishing
 */
public Envelope(long deliveryTag, 
                boolean redeliver, 
                String exchange, 
                String routingKey) {
    this.deliveryTag = deliveryTag;
    this.redeliver = redeliver;
    this.exchange = exchange;
    this.routingKey = routingKey;
}
// Создает новый Envelope с указанными параметрами: тегом доставки, флагом повторной доставки, 
// именем обменника и ключом маршрутизации.
```
💬**Комментарий:** Конструктор публичный, но на практике пользователи редко создают экземпляры `Envelope` самостоятельно. 

==Обычно они получаются из объектов== `Delivery`, создаваемых RabbitMQ клиентом. Класс спроектирован как immutable - все поля объявлены как `final`, что гарантирует потокобезопасность.

## **Поля класса (приватные и final):**
- `private final long deliveryTag` - уникальный идентификатор доставки в пределах канала
    
- `private final boolean redeliver` - флаг повторной доставки (сообщение было доставлено ранее)
    
- `private final String exchange` - имя обменника, через который было отправлено сообщение
    
- `private final String routingKey` - ключ маршрутизации, использованный при отправке

# Методы
## Получение метаданных доставки
```java
/**
 * Get the delivery tag included in this envelope.
 *
 * @return the delivery tag
 * @see com.rabbitmq.client.AMQP.Basic.Ack
 * @see com.rabbitmq.client.AMQP.Basic.Nack
 * @see com.rabbitmq.client.AMQP.Basic.Reject
 */
public long getDeliveryTag() {
    return deliveryTag;
}
// Возвращает тег доставки, включенный в этот конверт.

/**
 * Get the redelivery flag included in this envelope.
 * @see com.rabbitmq.client.AMQP.BasicProperties#getPriority()
 */
public boolean isRedeliver() {
    return redeliver;
}
// Возвращает флаг повторной доставки, включенный в этот конверт.

/**
 * Get the name of the exchange included in this envelope.
 */
public String getExchange() {
    return exchange;
}
// Возвращает имя обменника, включенное в этот конверт.

/**
 * Get the routing key included in this envelope.
 */
public String getRoutingKey() {
    return routingKey;
}
// Возвращает ключ маршрутизации, включенный в этот конверт.
```
💬**Небольшой комментарий по методам:** Все методы являются простыми геттерами без побочных эффектов. Класс является pure data holder (контейнером для данных).

# 💡 Особенности и подводные камни 💡

1️⃣ **DeliveryTag уникален только в пределах канала:** Тег доставки гарантированно уникален только в контексте одного канала. Нельзя использовать `deliveryTag` из одного канала в другом канале для подтверждения сообщения.

2️⃣ **DeliveryTag монотонно увеличивается:** В пределах одного канала теги доставки всегда увеличиваются. Это можно использовать для определения порядка доставки сообщений:
```java
// Сообщение с большим deliveryTag было доставлено позже
if (envelope1.getDeliveryTag() > envelope2.getDeliveryTag()) {
    // envelope1 доставлен позже
}
```

3️⃣ **Redeliver флаг указывает на возможные дубликаты:** Если `isRedeliver()` возвращает `true`, это означает, что сообщение уже доставлялось ранее, но не было подтверждено. Это важно для идемпотентной обработки:
```java
if (envelope.isRedeliver()) {
    // Проверяем, не обрабатывали ли мы это сообщение ранее
    if (alreadyProcessed(envelope.getDeliveryTag())) {
        channel.basicAck(envelope.getDeliveryTag(), false);
        return; // Пропускаем повторную обработку
    }
}
```

4️⃣ **Exchange может быть пустой строкой:** Для сообщений, отправленных в обменник по умолчанию (с пустым именем), `getExchange()` вернет пустую строку `""`, а не `null`:
```java
// Неправильно
if (envelope.getExchange() == null) { ... }

// Правильно
if (envelope.getExchange().isEmpty()) { ... }
// или
if ("".equals(envelope.getExchange())) { ... }
```

5️⃣ **RoutingKey может отличаться от binding key:** Ключ маршрутизации в конверте - это тот ключ, который был указан при публикации, а не ключ привязки очереди. Они могут отличаться при использовании wildcards в topic exchange:
```java
// Очередь привязана с ключом "stock.#"
// Сообщение опубликовано с ключом "stock.nyse.ibm"
// В конверте будет routingKey = "stock.nyse.ibm"
```

6️⃣ **Envelope не содержит информации о очереди:** В конверте нет информации о том, из какой очереди было получено сообщение. Эта информация доступна только в момент регистрации потребителя.

7️⃣ **Неизменяемость (Immutability):** Объект `Envelope` неизменяем после создания. Это гарантирует безопасность при передаче между потоками и предотвращает случайную модификацию метаданных.

# 💎 Best Practices 💎
✅ **Всегда извлекайте deliveryTag для подтверждений:**
```java
DeliverCallback deliverCallback = (consumerTag, delivery) -> {

    long deliveryTag = delivery.getEnvelope().getDeliveryTag();
    
    try {
        // Обработка сообщения
        processMessage(delivery.getBody());
        channel.basicAck(deliveryTag, false);
    } catch (Exception e) {
        // При ошибке - отклоняем с возвратом в очередь
        channel.basicNack(deliveryTag, false, true);
    }
};
```

✅ **Проверяйте флаг redeliver для идемпотентной обработки:**
```java
if (envelope.isRedeliver()) {
    logger.warn("Message redelivered: deliveryTag={}, exchange={}", 
                envelope.getDeliveryTag(), envelope.getExchange());
    // Реализуйте логику предотвращения дублирующей обработки
}
```

✅ **Логируйте полную информацию для отладки:**
```java
logger.debug("Message envelope: tag={}, redeliver={}, exchange={}, routingKey={}",
    envelope.getDeliveryTag(),
    envelope.isRedeliver(),
    envelope.getExchange(),
    envelope.getRoutingKey());
```

✅ **Используйте exchange и routingKey для маршрутизации логики:**
```java
String exchange = envelope.getExchange();
String routingKey = envelope.getRoutingKey();

if ("order-events".equals(exchange)) {
    switch (routingKey) {
        case "order.created":
            handleOrderCreated(delivery);
            break;
        case "order.cancelled":
            handleOrderCancelled(delivery);
            break;
    }
}
```

✅ **Создавайте удобные обертки для работы с Envelope:**
```java
public class MessageMetadata {
    private final Envelope envelope;
    
    public MessageMetadata(Envelope envelope) {
        this.envelope = envelope;
    }
    
    public boolean isFromDefaultExchange() {
        return envelope.getExchange().isEmpty();
    }
    
    public boolean isTopicMessage() {
        return envelope.getRoutingKey().contains(".");
    }
    
    // Другие полезные методы
}
```

✅ **Валидируйте метаданные при обработке:**
```java
private void validateEnvelope(Envelope envelope) {
    if (envelope.getDeliveryTag() <= 0) {
        throw new IllegalArgumentException("Invalid delivery tag: " + envelope.getDeliveryTag());
    }
    
    if (envelope.getExchange() == null) {
        throw new IllegalArgumentException("Exchange cannot be null");
    }
    
    if (envelope.getRoutingKey() == null) {
        throw new IllegalArgumentException("Routing key cannot be null");
    }
}
```

✅ **Используйте в мониторинге и метриках:**
```java
// Сбор метрик по обменникам
meterRegistry.counter("messages.received", 
    "exchange", envelope.getExchange(),
    "redelivered", String.valueOf(envelope.isRedeliver())
).increment();

// Трассировка
tracer.nextSpan()
    .name("process.message")
    .tag("delivery.tag", String.valueOf(envelope.getDeliveryTag()))
    .tag("exchange", envelope.getExchange())
    .start();
```

# Пример использования
```java
// Пример комплексной обработки с использованием Envelope
public class MessageProcessor {
    private final Channel channel;
    private final ObjectMapper objectMapper;
    private final Set<Long> processedDeliveryTags = ConcurrentHashMap.newKeySet();
    
    public MessageProcessor(Channel channel) {
        this.channel = channel;
        this.objectMapper = new ObjectMapper();
    }
    
    public DeliverCallback createDeliverCallback() {
        return (consumerTag, delivery) -> {
            Envelope envelope = delivery.getEnvelope();
            
            // 1. Логирование метаданных
            logMessageMetadata(envelope);
            
            // 2. Проверка на повторную доставку
            if (envelope.isRedeliver()) {
                handleRedelivery(envelope, delivery);
                return;
            }
            
            // 3. Извлечение deliveryTag для последующего подтверждения
            long deliveryTag = envelope.getDeliveryTag();
            
            try {
                // 4. Валидация метаданных
                validateEnvelope(envelope);
                
                // 5. Маршрутизация по exchange и routingKey
                String handlerKey = envelope.getExchange() + ":" + envelope.getRoutingKey();
                
                switch (handlerKey) {
                    case "orders:order.created":
                        processOrderCreated(delivery);
                        markAsProcessed(deliveryTag);
                        channel.basicAck(deliveryTag, false);
                        break;
                        
                    case "notifications:user.*":
                        processUserNotification(delivery, envelope.getRoutingKey());
                        markAsProcessed(deliveryTag);
                        channel.basicAck(deliveryTag, false);
                        break;
                        
                    case "dlq:":
                        processDeadLetter(delivery);
                        // Сообщения из DLQ не возвращаем обратно
                        channel.basicAck(deliveryTag, false);
                        break;
                        
                    default:
                        logger.warn("Unhandled message: exchange={}, routingKey={}", 
                                   envelope.getExchange(), envelope.getRoutingKey());
                        // Отклоняем с запретом возврата в очередь
                        channel.basicReject(deliveryTag, false);
                }
                
            } catch (ValidationException e) {
                logger.error("Invalid message envelope: {}", e.getMessage());
                // Отклоняем с возвратом в очередь для повторной попытки
                channel.basicNack(deliveryTag, false, true);
                
            } catch (ProcessingException e) {
                logger.error("Failed to process message: tag={}", deliveryTag, e);
                // Временная ошибка - возвращаем в очередь
                channel.basicNack(deliveryTag, false, true);
                
            } catch (FatalException e) {
                logger.error("Fatal error processing message: tag={}", deliveryTag, e);
                // Фатальная ошибка - не возвращаем в очередь
                channel.basicReject(deliveryTag, false);
            }
        };
    }
    
    private void logMessageMetadata(Envelope envelope) {
        logger.info("Message received - Tag: {}, Redeliver: {}, Exchange: '{}', RoutingKey: '{}'",
            envelope.getDeliveryTag(),
            envelope.isRedeliver(),
            envelope.getExchange(),
            envelope.getRoutingKey());
        
        // Метрики
        metricsCollector.recordMessageReceived(
            envelope.getExchange(),
            envelope.getRoutingKey(),
            envelope.isRedeliver()
        );
    }
    
    private void handleRedelivery(Envelope envelope, Delivery delivery) {
        long deliveryTag = envelope.getDeliveryTag();
        
        // Проверяем, не обрабатывали ли мы это сообщение ранее
        if (processedDeliveryTags.contains(deliveryTag)) {
            logger.info("Skipping already processed redelivery: tag={}", deliveryTag);
            channel.basicAck(deliveryTag, false);
            return;
        }
        
        logger.warn("Processing redelivered message: tag={}", deliveryTag);
        
        // Для redelivered сообщений можно применить особую логику:
        // - Увеличить таймауты
        // - Записать в отдельный лог
        // - Отправить уведомление
    }
    
    private void validateEnvelope(Envelope envelope) throws ValidationException {
        if (envelope.getDeliveryTag() <= 0) {
            throw new ValidationException("Invalid delivery tag: " + envelope.getDeliveryTag());
        }
        
        // Exchange должен быть указан (кроме default exchange)
        if (envelope.getExchange() == null) {
            throw new ValidationException("Exchange is null");
        }
        
        // RoutingKey должен быть указан
        if (envelope.getRoutingKey() == null) {
            throw new ValidationException("Routing key is null");
        }
    }
    
    private void markAsProcessed(long deliveryTag) {
        // Храним только последние N тегов для предотвращения утечки памяти
        if (processedDeliveryTags.size() > 10000) {
            // Удаляем старые теги (можно реализовать более сложную логику)
            processedDeliveryTags.clear();
        }
        processedDeliveryTags.add(deliveryTag);
    }
    
    // Методы обработки бизнес-логики
    private void processOrderCreated(Delivery delivery) throws ProcessingException {
        // ... обработка создания заказа
    }
    
    private void processUserNotification(Delivery delivery, String routingKey) throws ProcessingException {
        // ... обработка уведомлений пользователя
        // routingKey может быть "user.created", "user.updated" и т.д.
        String[] parts = routingKey.split("\\.");
        if (parts.length == 2) {
            String action = parts[1]; // "created", "updated"
            // Обработка в зависимости от действия
        }
    }
    
    private void processDeadLetter(Delivery delivery) {
        // Обработка сообщений из Dead Letter Queue
        logger.error("Processing dead letter message: {}", 
                    new String(delivery.getBody(), StandardCharsets.UTF_8));
    }
}

// Использование в приложении
public class Application {
    public static void main(String[] args) throws Exception {
        ConnectionFactory factory = new ConnectionFactory();
        factory.setHost("localhost");
        
        try (Connection connection = factory.newConnection();
             Channel channel = connection.createChannel()) {
            
            MessageProcessor processor = new MessageProcessor(channel);
            
            // Настройка QoS
            channel.basicQos(10);
            
            // Регистрация обработчика
            String consumerTag = channel.basicConsume(
                "orders-queue",
                false, // autoAck = false
                processor.createDeliverCallback(),
                cancelTag -> logger.info("Consumer cancelled: {}", cancelTag)
            );
            
            // Работа приложения
            Thread.sleep(30000);
            
            // Отмена потребителя
            channel.basicCancel(consumerTag);
        }
    }
}
```
