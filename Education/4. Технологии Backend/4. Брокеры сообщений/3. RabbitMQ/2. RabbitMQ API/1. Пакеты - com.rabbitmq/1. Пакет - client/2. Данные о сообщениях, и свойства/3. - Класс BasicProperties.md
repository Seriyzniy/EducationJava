
> [!NOTE] Класс `BasicProperties`
> **BasicProperties** - это класс, представляющий свойства сообщения AMQP. 
> 
> ==Он инкапсулирует все МЕТАДАННЫЕ сообщения==, такие как заголовки, тип контента, идентификатор сообщения, timestamp и другие атрибуты. Это один из самых важных классов для настройки поведения сообщений в RabbitMQ.

**Зачем нужен:** Для задания метаданных сообщения при публикации и получения этих метаданных при потреблении.
> Свойства управляют такими аспектами как персистентность сообщений, приоритет, срок жизни (TTL), маршрутизация и многое другое.

**Класс является вложенным (nested) классом:** `AMQP.BasicProperties`  
**Имеет вложенный Builder класс:** `AMQP.BasicProperties.Builder` для удобного создания объектов.

# Конструктор
```java
/**
 * Constructor that takes all possible values.
 *
 * @param contentType content type
 * @param contentEncoding content encoding
 * @param headers message header field table
 * @param deliveryMode delivery mode (1: non-persistent, 2: persistent)
 * @param priority message priority, 0 to 9
 * @param correlationId application correlation identifier
 * @param replyTo address to reply to
 * @param expiration message expiration specification
 * @param messageId application message identifier
 * @param timestamp message timestamp
 * @param type message type name
 * @param userId creating user id
 * @param appId creating application id
 * @param clusterId intra-cluster routing identifier (deprecated)
 */
public BasicProperties(String contentType,
                       String contentEncoding,
                       Map<String,Object> headers,
                       Integer deliveryMode,
                       Integer priority,
                       String correlationId,
                       String replyTo,
                       String expiration,
                       String messageId,
                       Date timestamp,
                       String type,
                       String userId,
                       String appId,
                       String clusterId) {
    this.contentType = contentType;
    this.contentEncoding = contentEncoding;
    this.headers = headers;
    this.deliveryMode = deliveryMode;
    this.priority = priority;
    this.correlationId = correlationId;
    this.replyTo = replyTo;
    this.expiration = expiration;
    this.messageId = messageId;
    this.timestamp = timestamp;
    this.type = type;
    this.userId = userId;
    this.appId = appId;
    this.clusterId = clusterId;
}
// Создает новый BasicProperties со всеми возможными значениями.
```
💬**Комментарий:** Конструктор принимает все возможные параметры, многие из которых могут быть `null`. 
На практике для создания объектов `BasicProperties` ==почти всегда используется вложенный== `Builder` класс, который предоставляет fluid API для удобного конструирования.
## **Поля класса** (все приватные и final в иммутабельной версии):

- `contentType`, `contentEncoding` - тип и кодировка контента
    
- `headers` - дополнительные заголовки (Map<String, Object>)
    
- `deliveryMode` - режим доставки (1 - non-persistent, 2 - persistent)
    
- `priority` - приоритет сообщения (0-9)
    
- `correlationId`, `replyTo` - для RPC паттерна
    
- `expiration` - TTL в миллисекундах (строка)
    
- `messageId`, `timestamp`, `type`, `userId`, `appId`, `clusterId` - различные идентификаторы

# 💡 Особенности и подводные камни 💡

1️⃣ **Иммутабельность (начиная с определенной версии):** В современных версиях RabbitMQ Java Client класс `BasicProperties` является иммутабельным. Это означает, что после создания объект нельзя изменить:

```java
// ❌ НЕ РАБОТАЕТ - класс иммутабельный
properties.setContentType("application/json");


// ✅ ПРАВИЛЬНО - создаем новый объект через Builder
BasicProperties newProps = properties.builder()
    .contentType("application/json")
    .build();
```

2️⃣ **Null значения:** Все поля могут быть `null`. При получении значений всегда проверяйте на `null`:

```java
// ❌ ОПАСНО - может вызвать NPE
String contentType = properties.getContentType().toLowerCase();


// ✅ БЕЗОПАСНО
if (properties.getContentType() != null) {
    String contentType = properties.getContentType().toLowerCase();
}
```

3️⃣ 🔥**Delivery Mode критически важен для persistence:**🔥
- `1` - non-persistent (сообщение теряется при перезагрузке брокера)
    
- `2` - persistent (сообщение сохраняется на диск)

```java
// Персистентное сообщение
BasicProperties props = new BasicProperties.Builder()
    .deliveryMode(2)  // ← ВАЖНО!
    .build();
```

4️⃣ **Expiration - строка, а не число:** Поле `expiration` хранится как строка, содержащая количество миллисекунд:
```java
// Устанавливаем TTL 5 минут (300000 мс)
BasicProperties props = new BasicProperties.Builder()
    .expiration("300000")  // ← СТРОКА!
    .build();
```

5️⃣ **Headers - могут содержать сложные типы:** Заголовки поддерживают различные типы данных:
```java
Map<String, Object> headers = new HashMap<>();

headers.put("string-header", "value");
headers.put("int-header", 42);
headers.put("boolean-header", true);
headers.put("list-header", Arrays.asList("a", "b", "c"));
headers.put("map-header", Collections.singletonMap("key", "value"));
```

6️⃣ **Priority работает только с приоритетными очередями:** Приоритет сообщения учитывается только если очередь объявлена с поддержкой приоритетов:
```java
Map<String, Object> args = new HashMap<>();
args.put("x-max-priority", 10);  // Максимальный приоритет 10
channel.queueDeclare("priority-queue", true, false, false, args);

// Сообщение с приоритетом
BasicProperties props = new BasicProperties.Builder()
    .priority(5)  // Будет учтен только в priority-queue
    .build();
```

7️⃣ **Timestamp - Date объект, но передается как long:** Хотя поле имеет тип `Date`, при сериализации в AMQP передается timestamp в секундах:
```java
// RabbitMQ преобразует Date.getTime() / 1000
Date now = new Date();  // Текущее время с миллисекундами
// В AMQP будет передано now.getTime() / 1000 (секунды)
```

8️⃣ **ContentType vs ContentEncoding:** Частая путаница:
- `contentType` - тип данных (application/json, text/plain, etc.)
    
- `contentEncoding` - кодировка (utf-8, gzip, etc.)

# 💎 Best Practices 💎

✅ Всегда ==используйте Builder для создания BasicProperties==:
```java
// Простой случай
BasicProperties props = new BasicProperties.Builder()
    .contentType("application/json")
    .deliveryMode(2)
    .timestamp(new Date())
    .build();

// Сложный случай с заголовками
Map<String, Object> headers = new HashMap<>();
headers.put("trace-id", UUID.randomUUID().toString());
headers.put("retry-count", 0);

BasicProperties props = new BasicProperties.Builder()
    .contentType("application/json")
    .contentEncoding("utf-8")
    .deliveryMode(2)
    .priority(5)
    
    .headers(headers)
    
    .messageId(UUID.randomUUID().toString())
    .timestamp(new Date())
    .appId("order-service")
    .build();
```

✅ **Используйте заголовки для сквозной трассировки:**
```java

    if (this.getHeaders() != null) {
        headers.putAll(this.getHeaders());
    }
    headers.put("X-Trace-Id", traceId);
    headers.put("X-Span-Id", spanId);
    
    return this.builder()
        .headers(headers)
        .build();
}
```

✅ **Обрабатывайте expiration безопасно:**
```java
public Long getExpirationMillis(AMQP.BasicProperties properties) {
    if (properties.getExpiration() == null) {
        return null;
    }
    try {
        return Long.parseLong(properties.getExpiration());
    } catch (NumberFormatException e) {
        logger.warn("Invalid expiration format: {}", properties.getExpiration());
        return null;
    }
}
```


```java
// Пример комплексного использования BasicProperties
public class MessagePublisher {
    private final Channel channel;
    private final ObjectMapper objectMapper;
    private final Tracer tracer;
    
    public MessagePublisher(Channel channel, Tracer tracer) {
        this.channel = channel;
        this.objectMapper = new ObjectMapper();
        this.tracer = tracer;
    }
    
    public void publishOrderEvent(Order order, String eventType) throws IOException {
        // 1. Подготовка тела сообщения
        String messageBody = objectMapper.writeValueAsString(order);
        
        // 2. Создание trace context
        Span span = tracer.currentSpan();
        Map<String, Object> headers = new HashMap<>();
        
        if (span != null) {
            headers.put("traceparent", span.context().traceIdString());
            headers.put("tracestate", span.context().traceStateString());
        }
        
        // 3. Добавление бизнес-заголовков
        headers.put("order-id", order.getId());
        headers.put("customer-id", order.getCustomerId());
        headers.put("event-type", eventType);
        headers.put("retry-count", 0);
        
        // 4. Создание свойств сообщения
        BasicProperties properties = new BasicProperties.Builder()
            .contentType("application/json")
            .contentEncoding("utf-8")
            .deliveryMode(2)  // Persistent
            .priority(order.isPriority() ? 5 : 1)
            .messageId(UUID.randomUUID().toString())
            .timestamp(new Date())
            .appId("order-service")
            .type(eventType)  // "order.created", "order.updated"
            .headers(headers)
            // Устанавливаем TTL 1 час для обработки
            .expiration("3600000")
            .build();
        
        // 5. Публикация с подтверждением
        channel.confirmSelect();
        
        String routingKey = "order." + eventType.toLowerCase();
        channel.basicPublish(
            "order-events",  // exchange
            routingKey,      // routing key
            true,            // mandatory
            properties,
            messageBody.getBytes(StandardCharsets.UTF_8)
        );
        
        // 6. Ожидание подтверждения от брокера
        if (!channel.waitForConfirms(5000)) {
            throw new IOException("Message not confirmed by broker");
        }
        
        // 7. Логирование
        logger.info(
	        "Published order event: id={}, type={}, messageId={}", 
            order.getId(), eventType, properties.getMessageId());
    }
    
    public void publishRpcRequest(String serviceName, Object request) throws IOException {
        // 1. Подготовка correlationId
        String correlationId = UUID.randomUUID().toString();
        String replyQueueName = "rpc-replies-" + UUID.randomUUID().toString();
        
        // 2. Создание временной очереди для ответов
        channel.queueDeclare(replyQueueName, false, true, true, null);
        
        // 3. Подготовка тела запроса
        String requestBody = objectMapper.writeValueAsString(request);
        
        // 4. Создание свойств для RPC
        BasicProperties props = new BasicProperties.Builder()
            .contentType("application/json")
            .deliveryMode(2)
            .correlationId(correlationId)
            .replyTo(replyQueueName)
            .messageId(UUID.randomUUID().toString())
            .timestamp(new Date())
            .expiration("30000")  // 30 секунд timeout
            .build();
        
        // 5. Публикация запроса
        channel.basicPublish(
            "",  // default exchange
            serviceName + "-requests",
            true,
            props,
            requestBody.getBytes(StandardCharsets.UTF_8)
        );
        
        // 6. Настройка потребителя для ответа
        DeliverCallback deliverCallback = (consumerTag, delivery) -> {
            AMQP.BasicProperties replyProps = delivery.getProperties();
            
            if (replyProps != null && correlationId.equals(replyProps.getCorrelationId())) {
                // Обработка ответа
                String response = new String(delivery.getBody(), StandardCharsets.UTF_8);
                handleRpcResponse(correlationId, response);
                
                // Подтверждение и закрытие временной очереди
                channel.basicAck(delivery.getEnvelope().getDeliveryTag(), false);
                channel.queueDelete(replyQueueName);
            }
        };
        
        channel.basicConsume(replyQueueName, false, deliverCallback, null);
    }
    
    public BasicProperties createDeadLetterProperties(
	    String originalExchange, 
        String originalRoutingKey,
        String reason) 
    {
        Map<String, Object> headers = new HashMap<>();
        headers.put("x-death", Arrays.asList(
            Map.of(
                "reason", reason,
                "exchange", originalExchange,
                "routing-keys", Arrays.asList(originalRoutingKey),
                "time", new Date()
            )
        ));
        
        return new BasicProperties.Builder()
            .contentType("application/json")
            .deliveryMode(2)
            .headers(headers)
            .type("dead-letter")
            .timestamp(new Date())
            .build();
    }
}

// Пример использования в потребителе
public class MessageConsumer {
    public DeliverCallback createMessageHandler() {
        return (consumerTag, delivery) -> {
            AMQP.BasicProperties properties = delivery.getProperties();
            Envelope envelope = delivery.getEnvelope();
            
            // 1. Логирование метаданных
            logMessageProperties(properties);
            
            // 2. Проверка на dead letter
            if (isDeadLetter(properties)) {
                handleDeadLetter(delivery);
                return;
            }
            
            // 3. Валидация свойств
            validateProperties(properties);
            
            // 4. Обработка на основе типа контента
            String contentType = properties.getContentType();
            String body = new String(delivery.getBody(), StandardCharsets.UTF_8);
            
            if ("application/json".equals(contentType)) {
                processJsonMessage(body, properties);
            } else if ("text/plain".equals(contentType)) {
                processTextMessage(body, properties);
            } else {
                logger.warn("Unsupported content type: {}", contentType);
            }
            
            // 5. Подтверждение обработки
            channel.basicAck(envelope.getDeliveryTag(), false);
            
            // 6. Обновление метрик
            updateMetrics(properties);
        };
    }
    
    private void logMessageProperties(AMQP.BasicProperties properties) {
        if (properties == null) {
            logger.debug("Message has no properties");
            return;
        }
        
        logger.info("Message properties: contentType={}, messageId={}, timestamp={}, headers={}",
                   properties.getContentType(),
                   properties.getMessageId(),
                   properties.getTimestamp(),
                   properties.getHeaders());
    }
    
    private boolean isDeadLetter(AMQP.BasicProperties properties) {
        if (properties == null || properties.getHeaders() == null) {
            return false;
        }
        return properties.getHeaders().containsKey("x-death");
    }
    
    private void validateProperties(AMQP.BasicProperties properties) {
        if (properties == null) {
            throw new ValidationException("Message properties cannot be null");
        }
        
        // Проверка TTL
        String expiration = properties.getExpiration();
        if (expiration != null) {
            try {
                long ttl = Long.parseLong(expiration);
                if (ttl <= 0) {
                    logger.warn("Message has expired TTL: {}", ttl);
                }
            } catch (NumberFormatException e) {
                logger.warn("Invalid expiration format: {}", expiration);
            }
        }
        
        // Проверка обязательных заголовков для некоторых типов сообщений
        if ("order.created".equals(properties.getType())) {
            Map<String, Object> headers = properties.getHeaders();
            if (headers == null || !headers.containsKey("order-id")) {
                throw new ValidationException("Order message must have order-id header");
            }
        }
    }
    
    private void updateMetrics(AMQP.BasicProperties properties) {
        metricsCollector.recordMessageProcessed(
            properties.getContentType(),
            properties.getType(),
            properties.getTimestamp() != null ? properties.getTimestamp().getTime() : null
        );
    }
```
