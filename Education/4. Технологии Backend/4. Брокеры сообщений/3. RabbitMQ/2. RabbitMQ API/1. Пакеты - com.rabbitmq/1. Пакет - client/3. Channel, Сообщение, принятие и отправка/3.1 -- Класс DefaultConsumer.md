
> [!NOTE] Класс `DefaultConsumer`
> `DefaultConsumer` — удобный **базовый класс-реализация интерфейса `Consumer`**, предназначенный для создания объектов-потребителей (consumers) сообщений от брокера RabbitMQ. 
> 
> Он реализует интерфейс `Consumer` и предоставляет **реализации всех его методов, ОБЫЧНО НИЧЕГО НЕ ДЕЛАЮЩИЕ** (no-op). 
> 
> Разработчик обычно **переопределяет только нужные методы**, например, `handleDelivery(...)`, чтобы обрабатывать входящие сообщения.

### **Реализует интерфейсы:**  
-  `Consumer` — через implements. [RabbitMQ](https://rabbitmq.github.io/rabbitmq-java-client/api/current/com/rabbitmq/client/DefaultConsumer.html?utm_source=chatgpt.com)
    
- **Не реализует** `AutoCloseable` или `Closeable` напрямую — управление временем жизни потребителя происходит через каналы/подписки. [RabbitMQ](https://rabbitmq.github.io/rabbitmq-java-client/api/current/com/rabbitmq/client/DefaultConsumer.html?utm_source=chatgpt.com)
    

### **Известные реализации/подклассы:**
- `QueueingConsumer` — устаревший класс, расширяет `DefaultConsumer` (и сам реализует `Consumer`). [RabbitMQ](https://rabbitmq.github.io/rabbitmq-java-client/api/4.x.x/com/rabbitmq/client/QueueingConsumer.html?utm_source=chatgpt.com)
    

Обычно `DefaultConsumer` используется как **удобная основа для собственных реализаций**.

# Конструктор
```java
// Construct a new instance and record its association to the passed-in channel.
public DefaultConsumer(Channel channel)
```
💬 **Комментарий:**  
Создаёт новый экземпляр `DefaultConsumer` и связывает его с указанным `Channel`. 
Этот канал будет тем, из которого будут обрабатываться сообщения и к которому будет относиться потребительский тэг.

# Методы
## Основные методы доступа
```java
// Retrieve the channel.
public Channel getChannel()

// Retrieve the consumer tag.
public String getConsumerTag()
```
💬 Эти методы позволяют получить канал, к которому привязан потребитель (`getChannel()`), и **текущий consumer tag** (`getConsumerTag()`) — уникальный идентификатор потребителя в рамках канала.

## "Реализация" методов интерфейса Consumer
```java
// Stores the most recently passed-in consumerTag – semantically, there should be only one.
public void handleConsumeOk(String consumerTag)

// No-op implementation of Consumer.handleCancelOk(java.lang.String).
public void handleCancelOk(String consumerTag)


// No-op implementation of Consumer.handleCancel(String)
public void handleCancel(String consumerTag) throws IOException



// No-op implementation of Consumer.handleShutdownSignal(java.lang.String, com.rabbitmq.client.ShutdownSignalException).
public void handleShutdownSignal(String consumerTag, ShutdownSignalException sig)

// No-op implementation of Consumer.handleRecoverOk(java.lang.String).
public void handleRecoverOk(String consumerTag)

// No-op implementation of Consumer.handleDelivery(...)
public void handleDelivery(String consumerTag,
                           Envelope envelope,
                           AMQP.BasicProperties properties,
                           byte[] body)
                    throws IOException
```
💬 _Что здесь происходит:_  
Эти методы являются **реализациями интерфейса `Consumer`**, но в `DefaultConsumer` они по умолчанию **ничего не делают** (no-op), за исключением `handleConsumeOk`, который **сохраняет consumer tag**. Разработчик может переопределить любые из них, чтобы обработать события жизненного цикла потребителя.

# 💡 Особенности и подводные камни 💡
1️⃣ **No-op по умолчанию:** если вы не переопределите `handleDelivery(...)`, сообщения **не будут обрабатываться по-умолчанию** при подписке с использованием `DefaultConsumer`. [RabbitMQ](https://rabbitmq.github.io/rabbitmq-java-client/api/current/com/rabbitmq/client/DefaultConsumer.html?utm_source=chatgpt.com)  

2️⃣ **Сохранение consumer tag:** `handleConsumeOk` сохраняет consumer tag, что полезно для дальнейшего управления потребителем. [RabbitMQ](https://rabbitmq.github.io/rabbitmq-java-client/api/current/com/rabbitmq/client/DefaultConsumer.html?utm_source=chatgpt.com)  

3️⃣ **Исключения внутри callback’ов:** если в переопределённых методах возникает ошибка (например, в `handleDelivery(...)`), это может **привести к закрытию канала/соединения**, если ошибки не обрабатываются. [RabbitMQ](https://www.rabbitmq.com/client-libraries/java-api-guide?utm_source=chatgpt.com)  

4️⃣ **Работа в отдельном потоке разгрузки:** вызовы методов происходят **в отдельном dispatch-потоке**, а не в основном сетевом или рабочем потоке, что позволяет безопасно вызывать методы `Channel` внутри callback’ов. [RabbitMQ](https://rabbitmq.github.io/rabbitmq-java-client/api/4.x.x/com/rabbitmq/client/Consumer.html?utm_source=chatgpt.com)  

5️⃣ **Не автоматически закрывается:** `DefaultConsumer` **не управляет временем жизни** сам по себе — отмена и отписка происходят через вызовы `Channel.basicCancel(...)`. [RabbitMQ](https://www.rabbitmq.com/client-libraries/java-api-guide?utm_source=chatgpt.com)

# 💎 Best Practices 💎
✅ **Переопределяйте только нужные методы**, чаще всего `handleDelivery(...)`. [RabbitMQ](https://rabbitmq.github.io/rabbitmq-java-client/api/current/com/rabbitmq/client/DefaultConsumer.html?utm_source=chatgpt.com)  

✅ **Поймайте исключения внутри обработчиков**, чтобы не допустить неожиданного закрытия канала. [RabbitMQ](https://www.rabbitmq.com/client-libraries/java-api-guide?utm_source=chatgpt.com)  

✅ Используйте отдельные классы/объекты для разной логики обработки, если потребители работают с разными очередями. [RabbitMQ](https://www.rabbitmq.com/client-libraries/java-api-guide?utm_source=chatgpt.com)  

✅ Для простых случаев, когда нужен только обработчик доставки, можно использовать **lambda-ориентированные API** (`DeliverCallback`), но `DefaultConsumer` все равно остается основой. [RabbitMQ](https://rabbitmq.github.io/rabbitmq-java-client/api/current/com/rabbitmq/client/package-summary.html?utm_source=chatgpt.com)  

✅ Всегда знаете **consumer tag**, если он вам нужен для отмены подписки.

# Пример использования
```java
Channel channel = connection.createChannel();

Consumer myConsumer = new DefaultConsumer(channel) {
    @Override
    public void handleDelivery(String consumerTag,
                               Envelope envelope,
                               AMQP.BasicProperties properties,
                               byte[] body)
            throws IOException 
    {
        String message = new String(body, StandardCharsets.UTF_8);
        System.out.println("Received message: " + message);
        
        // подтверждаем вручную
        channel.basicAck(envelope.getDeliveryTag(), false);
    }
};

channel.basicConsume("my.queue", false, myConsumer);
```
💬 В этом примере потребитель **переопределяет только `handleDelivery(...)`**, чтобы обрабатывать входящие сообщения и подтверждать их (`basicAck`).
