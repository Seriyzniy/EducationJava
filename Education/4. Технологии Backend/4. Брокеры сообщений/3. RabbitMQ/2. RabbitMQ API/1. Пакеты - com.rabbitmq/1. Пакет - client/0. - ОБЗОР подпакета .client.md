# Таблица 2: Callback-интерфейсы и обработчики событий

Интерфейсы для получения уведомлений о различных событиях: доставка сообщений, отмена потребителя, блокировка подключения и т.д.
> ДОБАВЛЯЮТСЯ ЧЕРЕЗ `Channel` в слушателей

| Класс                            | Описание на русском                                                                                                                                 |
| -------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ConfirmCallback`                | Реализуйте этот интерфейс, чтобы получать уведомления о событиях подтверждения доставки (Publisher Confirms).                                       |
| `ReturnCallback`                 | Реализуйте этот интерфейс, чтобы получать уведомления о неудачных доставках, когда `basicPublish` вызывается с флагами "mandatory" или "immediate". |
| `CancelCallback`                 | Callback-интерфейс для получения уведомлений об отмене потребителя.                                                                                 |
| `ConsumerShutdownSignalCallback` | Callback-интерфейс для получения уведомлений, когда либо потребительский канал, либо базовое соединение были закрыты.                               |
| `BlockedListener`                | Реализуйте этот интерфейс, чтобы получать уведомления о событиях блокировки и разблокировки подключения.                                            |
| `BlockedCallback`                | Реализуйте этот интерфейс, чтобы получать уведомления о событиях блокировки подключения.                                                            |
| `UnblockedCallback`              | Реализуйте этот интерфейс, чтобы получать уведомления о событиях разблокировки подключения.                                                         |
| `ShutdownListener`               | `ShutdownListener` получает информацию о завершении работы подключений и каналов.                                                                   |
| `RecoveryListener`               | `RecoveryListener` получает уведомления об успешном завершении автоматического восстановления подключения.                                          |
| `ExceptionHandler`               | Интерфейс ==для объекта обработки исключений.==                                                                                                     |
| `TrafficListener`                | Контракт для логирования исходящих и входящих команд.                                                                                               |

# Таблица 3: Восстановление подключения (Recovery)
Классы для работы с автоматическим восстановлением подключения после сбоев.

|Класс|Описание на русском|
|---|---|
|`Recoverable`|Предоставляет способ регистрации callback-ов для восстановления (сетевого, AMQP 0-9-1) подключения.|
|`RecoverableConnection`|Удобный интерфейс при работе с автоматически восстанавливаемыми подключениями.|
|`RecoverableChannel`|Удобный интерфейс при работе с автоматически восстанавливаемыми каналами.|
|`RecoveryDelayHandler`|`RecoveryDelayHandler` используется, чтобы указать автоматическому восстановлению, как долго ожидать между попытками переподключения.|
|`RecoveryDelayHandler.ExponentialBackoffDelayHandler`|Реализация задержки восстановления с экспоненциальным откатом (backoff), использующая последовательность Фибоначчи (по умолчанию) для увеличения времени задержки после каждой неудачной попытки.|

# Таблица 4: Адресация и разрешение хостов (Address Resolution)
Классы для настройки подключения к кластеру RabbitMQ.

|Класс|Описание на русском|
|---|---|
|`Address`|Представление сетевых адресов, т.е. пар хост/порт, с некоторыми утилитарными функциями для разбора строк адресов.|
|`AddressResolver`|Стратегия (интерфейс) для получения потенциальных серверов для подключения.|
|`ListAddressResolver`|Простая реализация `AddressResolver`, возвращающая фиксированный список адресов.|
|`DnsSrvRecordAddressResolver`|`AddressResolver`, который разрешает адреса с помощью DNS SRV запроса.|
|`DnsRecordIpAddressResolver`|`AddressResolver`, который разрешает IP-адреса из DNS записей.|

# Таблица 5: AMQP-модель и низкоуровневые дескрипторы
Классы, представляющие AMQP-методы, команды и их строителей (Builders). Используются для тонкого контроля над протоколом.

| Класс                 | Описание на русском                                                                                                                 |
| --------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| `AMQP`                | Главный класс, содержащий вложенные классы, соответствующие классам AMQP (например, `Basic`, `Channel`, `Exchange`).                |
| `AMQP.Basic`          | Вложенный класс, содержащий методы и строителей для операций базового класса AMQP (`publish`, `consume`, `ack`, `nack` и т.д.).     |
| `AMQP.Channel`        | Вложенный класс, содержащий методы и строителей для операций управления каналом (`open`, `close`, `flow`).                          |
| `AMQP.Connection`     | Вложенный класс, содержащий методы и строителей для операций управления подключением (`start`, `tune`, `open`, `close`, `blocked`). |
| `AMQP.Exchange`       | Вложенный класс, содержащий методы и строителей для операций с обменниками (`declare`, `delete`, `bind`, `unbind`).                 |
| `AMQP.Queue`          | Вложенный класс, содержащий методы и строителей для операций с очередями (`declare`, `delete`, `bind`, `unbind`, `purge`).          |
| `AMQP.Tx`             | Вложенный класс, содержащий методы и строителей для транзакционных операций (`select`, `commit`, `rollback`).                       |
| `AMQP.Confirm`        | Вложенный класс, содержащий методы и строителей для операций подтверждения доставки (publisher confirms).                           |
| `Command`             | Интерфейс для контейнера AMQP-метода с аргументами, опциональным заголовком контента и телом.                                       |
| `Method`              | Публичный интерфейс для объектов, представляющих метод AMQP 0-9-1.                                                                  |
| `ContentHeader`       | Публичный API для абстрактных объектов заголовка контента AMQP.                                                                     |
| `MessageProperties`🔴 | Класс-контейнер констант с полезными статическими экземплярами `AMQContentHeader`.                                                  |

# Таблица 8: Исключения

|Класс|Описание на русском|
|---|---|
|`AlreadyClosedException`|Выбрасывается, когда приложение пытается выполнить действие на уже закрытом подключении/канале.|
|`AuthenticationFailureException`|Выбрасывается, когда брокер отказывает в доступе из-за сбоя аутентификации.|
|`ShutdownSignalException`|Инкапсулирует условие завершения работы подключения к брокеру AMQP.|
|`PossibleAuthenticationFailureException`|Выбрасывается, когда вероятной причиной является сбой аутентификации.|
|`ChannelContinuationTimeoutException`|Исключение, выбрасываемое, когда канал превышает время ожидания продолжения (continuation) во время RPC вызова.|
|`MalformedFrameException`|Инкапсулирует ошибку формата фрейма на уровне протокола.|
|`MissedHeartbeatException`|Инкапсулирует исключение, указывающее, что подключение пропустило слишком много сердцебиений и закрывается.|
|`ProtocolVersionMismatchException`|Выбрасывается, чтобы указать, что сервер не поддерживает запрошенную версию протокола сразу после открытия TCP-сокета.|
|`TopologyRecoveryException`|Указывает на исключение, возникшее во время восстановления топологии.|
|`UnexpectedFrameError`|Выбрасывается, когда парсер команд встречает неожиданный тип фрейма.|
|`UnexpectedMethodError`|Указывает, что был передан объект `Method`, который не ожидался.|
|`UnknownClassOrMethodId`|Выбрасывается, когда обработчики протокола обнаруживают неизвестный номер класса или метода.|
|`UnroutableRpcRequestException`|Исключение, выбрасываемое, когда RPC-запрос не маршрутизируется ни в одну очередь.|
