Теперь рассмотрим 2 пример из гайда в RabbitMQ. 

Там говорится про случай, когда сообщение из одной очереди обрабатывают несколько Consumer (в нашем случае 2 консюмера)

![[Pasted image 20251213164643.png]]

В этом примере, **сообщения будут отправляться консюмерам ПО ОЧЕРЕДИ (Round Robbin)** - ПОВЕДЕНИЕ ПО УМОЛЧАНИЮ RabbitMQ

Но **может возникнуть ситуация**, когда один из **Consumer будет долго выполнять задачу** (обрабатывать сообщение), а другой будет быстро обрабатывать

**В результате**, тот консюмер, который долго обрабатывает сообщения, **будет получать больше сообщений, чем он может обработать.**

Это происходит **из-за того, что RabbitMQ по умолчанию раскидывает сообщения по очереди** (Round Robbin), и ==не смотрит на загрузку каждого Consumer== (по количеству ACK). Он просто шлет сообщения.

Чтобы это решить, нужно установить параметр `prefetch=1`, чтобы максимальное сообщений, которые могут быть не подтверждены, было 1.

В результате, Rabbit увидит, что на канале с каким-либо Consumer'ом установлен этот параметр, и не будет отправлять ему сообщение, а отправит тому, кто свободен

# 1. Описание Producer
Чтобы ссимулировать работу для консюмера, нам Prodcuer будет отправлять сообщения с определенным количеством точек.

А Consumer, в зависимости от количества точек, будет "засыпать" на то же количество секунд (3 точки - 3 секунды сна)

Теперь опишем Producer
```java
package rabbitmq.example2.producer;  
  
import com.rabbitmq.client.Channel;  
import com.rabbitmq.client.Connection;  
import com.rabbitmq.client.ConnectionFactory;  
  
import java.io.IOException;  
import java.util.Scanner;  
import java.util.concurrent.TimeoutException;  
  
public class RabbitProducer {  
    private final static String QUEUE_NAME = "example2-task-queue";  
    private final static String EXCHANGE_NAME = "example2-task-exchange";  
    private final Connection connection;  
  
    public static void main(String[] args) {  
        ConnectionFactory factory = new ConnectionFactory();  
        factory.setHost("localhost");  
  
        RabbitProducer rabbitProducer = new RabbitProducer(factory);  
  
        Scanner scanner = new Scanner(System.in);  
        while (true) {  
            System.out.print("Input message with few dot. \n>> ");  
            String input = scanner.nextLine();  
            rabbitProducer.produceMessage(input);  
        }  
    }  
    public RabbitProducer(ConnectionFactory connectionFactory) {  
        try {  
            this.connection = 
	            connectionFactory.newConnection("localhost");  
  
            try (Channel channel = connection.createChannel()) { 
             
                channel.exchangeDeclare(
	                EXCHANGE_NAME, 
	                "direct");  
	                
                channel.queueDeclare(
	                QUEUE_NAME, 
	                false, 
	                false, 
	                false, 
	                null);  
	                
                channel.queueBind(
	                QUEUE_NAME, 
	                EXCHANGE_NAME, 
	                "task");  
            }  
        } catch (IOException e) {  
            throw new RuntimeException(e);  
        } catch (TimeoutException e) {  
            System.out.println(
	            "[Producer] Connection CONNECTION timed out");  
            throw new RuntimeException(e);  
        }  
    }  
  
    public void produceMessage(String message) {  
        try(Channel channel = connection.createChannel()) { 
         
            channel.basicPublish(  
                EXCHANGE_NAME,  
                "task",  
                null,  
                message.getBytes());  
  
            System.out.println("[x] Sent '" + message + "'\n");  
        } catch (IOException e) {  
            throw new RuntimeException(e);
              
        } catch (TimeoutException e) {  
            System.out.println(
	            "[Producer] Connection CHANNEL timed out");  
            throw new RuntimeException(e);  
        }  
    }  
}
```


> [!NOTE] ДЕКЛАРИРОВАНИЕ обменника и очереди на `Consumer`
> Очередь и обменник можно было бы определить на Consumer, при этом явно не указывая имя очереди
> ```java
> String queueName = channel.queueDeclare().getQueue();
> ```

# 2. Описание Consumer
Теперь опишем класс Consumer

Как уже говорили выше, наш консюмер будет засыпать на то же количество секунд, сколько и точек в сообщении.

Для этого опишем метод
```java
private void doWork(String message) throws InterruptedException {  
    for(char ch : message.toCharArray()){  
        if(ch == '.') Thread.sleep(1000);  
    }  
}
```

Теперь опишем метод обработки сообщения
```java
public void consumeMessageWithChannel() {  
    System.out.println("Consuming message ... ");  
  
    DeliverCallback deliverCallback = (consumerTag, delivery) -> {  
        Envelope envelope = delivery.getEnvelope();  
        
        String message = new String(
	        delivery.getBody(), 
	        StandardCharsets.UTF_8);  
  
        System.out.println("Consumer Tag: " + consumerTag +  
                "\nExchange: " + envelope.getExchange() +  
                "\nRouting Ket: " + envelope.getRoutingKey()+  
                "\nDelivery Tag" + envelope.getDeliveryTag() +  
                "\nMessage: " + message + "\n");  
  
        try{  
	        // СИМУЛЯЦИЯ РАБОТЫ
            doWork(message);  
        } catch (Exception e){  
            System.out.println(e.getMessage());  
            throw new RuntimeException(e);  
        }  
    };  
  
    try {  
        Channel channel = connection.createChannel();  
  
        boolean autoAck = true;  
        channel.basicConsume(
	        QUEUE_NAME, 
	        autoAck, 
	        deliverCallback, 
	        consumerTag -> {}
	    );  
    } catch (IOException e) {  
        throw new RuntimeException(e);  
    }  
}
```
В методе basicConsume необходимо еще передать и `CanselCallback`, так как только с ним есть перегрузка метода, использующая `DeliverCallback`

В результате получим такой код
```java
package rabbitmq.example2.consumer;  
import com.rabbitmq.client.*;  
  
import java.io.IOException;  
import java.nio.charset.StandardCharsets;  
import java.util.concurrent.TimeoutException;  
  
public class RabbitConsumer {  
    private final static String QUEUE_NAME = "example2-task-queue";  
    private final Connection connection;  
  
    public static void main(String[] args) {  
        ConnectionFactory factory = new ConnectionFactory();  
        factory.setHost("localhost");  
  
        RabbitConsumer consumer = new RabbitConsumer(factory);  
        consumer.consumeMessageWithChannel();  
    }  
  
    public RabbitConsumer(ConnectionFactory connectionFactory) {  
        try {  
            this.connection = connectionFactory.newConnection();  
        } catch (IOException e) {  
            throw new RuntimeException(e);  
        } catch (TimeoutException e) {  
            System.out.println("Connection timed out");  
            throw new RuntimeException(e);  
        }  
    }  
  
    public void consumeMessageWithChannel() {  
        System.out.println("Consuming message ... ");  
  
        DeliverCallback deliverCallback = (consumerTag, delivery) -> {  
            Envelope envelope = delivery.getEnvelope();  
            
            String message = new String(
	            delivery.getBody(), 
	            StandardCharsets.UTF_8);  
  
            System.out.println("Consumer Tag: " + consumerTag +  
                    "\nExchange: " + envelope.getExchange() +  
                    "\nRouting Ket: " + envelope.getRoutingKey()+  
                    "\nDelivery Tag" + envelope.getDeliveryTag() +  
                    "\nMessage: " + message + "\n");  
  
            try{  
                doWork(message);  
            } catch (Exception e){  
                System.out.println(e.getMessage());  
                throw new RuntimeException(e);  
            }  
        };  
  
        try {  
            Channel channel = connection.createChannel();  
  
            boolean autoAck = true;  
            channel.basicConsume(
	            QUEUE_NAME, 
	            autoAck, 
	            deliverCallback, 
	            consumerTag -> {}
	        );  
        } catch (IOException e) {  
            throw new RuntimeException(e);  
        }  
    }  
  
    private void doWork(String message) throws InterruptedException {  
        for(char ch : message.toCharArray()){  
            if(ch == '.') Thread.sleep(1000);  
        }  
    }  
}
```

# 4. УЛУЧШЕНИЯ
Можно добавить в Consumer
- `prefetch=1` через `channel.basicQos(1)`
- Ручной коммит сообщения в CallBack - `DeliverCallback`, в блоке finally
```java
public void consumeMessageWithChannel() {  
    System.out.println("Consuming message ... ");  
  
    try {  
        Channel channel = connection.createChannel();  
        channel.basicQos(1);  
  
        DeliverCallback deliverCallback = (consumerTag, delivery) -> {  
            Envelope envelope = delivery.getEnvelope();  
            String message = new String(
	            delivery.getBody(), 
	            StandardCharsets.UTF_8);  
  
            System.out.println("Consumer Tag: " + consumerTag +  
                    "\nExchange: " + envelope.getExchange() +  
                    "\nRouting Ket: " + envelope.getRoutingKey()+  
                    "\nDelivery Tag" + envelope.getDeliveryTag() +  
                    "\nMessage: " + message + "\n");  
  
            try{  
                doWork(message);  
            } catch (Exception e){  
                System.out.println(e.getMessage());  
                throw new RuntimeException(e);  
            } finally {  
                channel.basicAck(envelope.getDeliveryTag(), false);  
            }  
        };  
          
        boolean autoAck = false;  // ПОМЕНЯЛ АВТОКОММИТ на false
        channel.basicConsume(
	        QUEUE_NAME, 
	        autoAck, 
	        deliverCallback, 
	        consumerTag -> {}
	    );  
    } catch (IOException e) {  
        throw new RuntimeException(e);  
    }  
}
```

И тогда не будет той проблемы, из-за которой один из Consumer будет "тонуть" в сообщениях и не успевать их обрабатывать

## Проверка
Теперь, будет отправлять сообщения первому Консюмеру (который запущен первым), сообщения с большим количеством точек, чтобы он дольше "спал", а второму вообще без точек

И посмотрим, как будет Rabbit "ломать" Round Robbin
![[Pasted image 20251213173358.png]]

1) Как видим, мы отправили длинное сообщение, и оно попало в 1-го консюмера. Он начал его долго обрабатывать.

2) Затем мы отправили маленькое сообщение, и оно попало ко второму консюмеру, он его быстро обработал и отправил ACK

Теперь как видим, следующе сообщения ДОЛЖНО БЫЛО опять отправиться на 1-го консюмера
3) Отмечено ЗЕЛЕНЫМ сообщение, которое должно было попасть туда же куда и КРАСНОЕ (к первому консюмеру), но оно попало ко второму, там как он вернул ACK, и у него 0 сообщений без подтверждения (свободен), а первого все еще 1 сообщение без подтверждение (МАКСИМУМ из возможного `prefetch=1`)
