# 💡1. Инициализация и настройка💡 

## Команда: `git init`
```bash
git init [путь]

# Создаёт новый локальный репозиторий Git в текущей или указанной директории
```

## Команда `git config`
```bash
git config [опции] [ключКонфига] [значениеКлюча]
# Настройка параметров Git (глобально или для конкретного репо)

--global         # применить настройку глобально (для всех репозиториев)
--local          # только для текущего репозитория
--system         # системный уровень
--list           # вывести все настройки
user.name        # имя пользователя
user.email       # email пользователя
core.editor      # задать текстовый редактор по умолчанию
```

# 📂 2. Работа с файлами и индексом (staging area)

## 💡 Команда: `git status` - состояние индекса и рабочего каталога
```bash
git status

# Показывает 💡состояние рабочего каталога и индекса💡
```

## 💡 Команда: `git add` - добавление файлов в индекс
```bash
git add [файл | . | -A] 
# Добавляет файлы в индекс (staging area)  


.      # добавить все изменённые файлы 
-A     # добавить все файлы, включая удалённые 
-p     # добавить интерактивно (по кускам)
```

## Команда: `git restore`
```bash
git restore [файл] 
# Отменяет изменения в файлах (возвращает их к HEAD)  

--staged [файл]  # убрать файл из индекса (аналог git reset HEAD <файл>)
```

## Команда: `git rm`
```bash
git rm [файл] 
# УДАЛЯЕТ файл из рабочего каталога и из индекса  

--cached  # удалить только из индекса, оставить файл на диске
```

## Команда: `git mv`
```bash
git mv old new 
# Переименовывает файл с отслеживанием изменений
```

# 🧾 3. Коммиты и история

## 💡 Команда: `git commit` - создание коммита

```bash
git commit -m "сообщение" 
# Создаёт новый коммит из изменений в индексе  

-a       # автоматически добавить все изменённые файлы 
--amend  # изменить последний коммит 
--no-edit # оставить старое сообщение при amend
```

## 💡 Команда: `git log` - история коммитов
```bash
git log [опции] 
# Показывает историю коммитов  

--oneline    # один коммит в строке 
--graph      # дерево веток 
--decorate   # показать имена веток и тегов 
--stat       # статистика по файлам 
-p           # показать diff каждого коммита
```

## Команда: `git show`
```bash
git show [commit] 
# Показывает содержимое коммита (diff + метаданные)
```

## 💡Команда: `git diff` - различия между файлами
```bash
git diff [опции] 
# Показывает различия между файлами  

git diff                # рабочий каталог vs индекс 
git diff --staged       # индекс vs HEAD 
git diff commit1 commit2 # сравнение любых двух коммитов 

--name-only             # только имена файлов 
--stat                  # только статистику изменений
```

# 🌿 4. Ветки и слияния

## 💡Команда: `git branch` - УПРАВЛЕНИЕ ветками (см ключи) и СОЗДАНИЕ
```bash
git branch [имя_ветки] 
# Управление ветками  


git branch myvetka 
# Создаст ветку myvetka


# Без аргументов — показать список веток 
-d <ветка>  # удалить ветку (текущую нельзя удалить)
-D <ветка>  # принудительно удалить 
-m <новое>  # переименовать ❗❗ТЕКУЩУЮ❗❗ ветку 
-a          # показать все ветки (включая удалённые) 
-r          # только удалённые ветки

-f          # напрямую 💡ПРИКРЕПИТЬ ВЕТКУ к комиту💡
git checkout -f main HEAD~3
```

## 💡Команда: `git checkout / git switch` - переключение между ветками и СОЗДАНИЕ
Перемещает файлы из репозитория определенной версии в рабочую директорию
```bash
git checkout [ветка | commit | commit_hash] 
# Переключение между ветками или коммитами  

-b <имя>  # ⭐⭐СОЗДАТЬ⭐⭐ новую ветку и сразу перейти в неё  


# Современный вариант: 
git switch [ветка] 
git switch -c [новая_ветка]   # ⭐⭐СОЗДАТЬ и переключиться⭐⭐
```

## 💡Команда: `git merge` - объединение изменений (подтягивает другую ветку к себе)
```bash
git merge [ветка] 
# Объединяет изменения ИЗ УКАЗАННОЙ ветки В ТЕКУЩУЮ  

--no-ff     # всегда создавать коммит слияния 
--squash    # объединить коммиты без merge-коммита 
--abort     # отменить слияние при конфликте
-s          # ВЫБОР СТРАТЕГИИ (ours, theirs ...)
```

## 💡Команда: `git rebase` - объединение изменений (заталкивает себя поверх другой ветки)
```bash
git rebase [ветка | commit] 
git rebase ПОВЕРХ_ЧЕГО ЧТО
# Переносит ❗❗ТЕКУЩИЕ коммиты ОТ РОДИТЕЛЯ❗❗ ПОВЕРХ УКАЗАННОЙ ветки  

--continue  # продолжить после конфликта 
--abort     # отменить rebase 
-i          # интерактивный режим (reorder, squash и т.д.)
```

# 🌐 5. Работа с удалёнными репозиториями

## 💡Команда: `git remote` - УПРАВЛЕНИЕ подключенными репозиториями (см ключи)
```bash
git remote [опции] 
# Управление подключёнными репозиториями  

-v                 # показать адреса 
add origin <url>   # добавить удалённый репозиторий 
remove <имя>       # удалить rename old new     # переименовать
```

## 💡Команда: `git fetch` - загрузка данных их удаленного репозитория (не сливает)
```bash
git fetch [origin] [ветка] 
# ЗАГРУЖАЕТ данные из удалённого репозитория, но не сливает

--prune удаляет "мертвые" ссылки на ветки, которых больше нет на сервере
```

## ⭐⭐Команда: `git pull` - загрузить и объединить
```bash
git pull [origin] [ветка] 
# fetch + merge (или rebase, если задано)  

--rebase  # подтянуть изменения с переписыванием коммитов
```

## ⭐⭐Команда: `git push` - отправить коммиты на удаленный репозиторий
```bash
git push [origin] [ветка] 
# Отправляет коммиты на сервер  

-u --set-upstream origin branch  # установить связь ветки с удалённой 
--force (-f)                  # перезаписать удалённую историю 
--tags                        # отправить все теги
```

# 🏷️ 6. Теги (tags)

## Команда: `git tag`

```bash
git tag [имя_тега] [имя_коммита] 
# Создать тег на коммит

-a <имя> -m "сообщение"  # аннотированный тег 
-d <имя>                 # удалить тег 
-l                       # показать список тегов
```

## Команда: `git push/pull` для тегов
```bash
git push origin <tag>     
# отправить конкретный тег 

git push origin --tags    
# отправить все теги 

git push origin --delete tag <tag> 
# удалить тег на сервере
```

# ♻ 7. Откат и исправление ошибок

## Команда: `git reset`

```bash
git reset [commit] 
# Перемещает указатель HEAD и изменяет индекс/файлы  

--soft   # только HEAD (сохраняет индекс и файлы) 
--mixed  # HEAD + индекс (по умолчанию) 
--hard   # 💡HEAD + индекс + файлы (откат полностью)💡
```

## Команда: `git revert`

```bash
git revert [commit] 
# Создаёт новый коммит, отменяющий указанный
```

## Команда: `git stash`

```bash
git stash [опции] 
# Временно сохраняет незакоммиченные изменения  

list     # показать список сохранений 
pop      # восстановить и удалить последнее 
apply    # восстановить, но не удалять 
drop     # удалить сохранение 
clear    # очистить все stash
```

```bash
git show-branch
```

```bash
git reflog
```
