
> [!NOTE] `.gitignore` — это **текстовый файл**, который указывает Git,  
> **какие файлы и каталоги нужно игнорировать** при добавлении (`git add`) и коммите.
> Git не будет **отслеживать** файлы, пути которых **соответствуют шаблонам** в `.gitignore`.
> 


💡`.gitignore` влияет **только на НЕОТСЛЕЖИВАЕМЫЕ файлы**.  
❗❗❗Если файл **уже был добавлен в репозиторий**, то добавление его в `.gitignore` **не остановит** отслеживание.

## Как Git применяет `.gitignore`
1. Когда ты выполняешь `git status` или `git add .`,  
    Git проверяет все файлы в каталоге.
    
2. Если путь совпадает с шаблоном из `.gitignore`,  
    файл **не попадает** в список для добавления.
    
3. Git пропускает эти файлы, как будто их “не существует”.

## Где может находиться `.gitignore`

|Расположение|Применение|
|---|---|
|`.gitignore` в корне|Глобальные правила для всего репозитория|
|`.gitignore` в подкаталогах|Локальные правила, действующие **только внутри** папки|
|Глобальный `.gitignore`|Настраивается один раз для пользователя на всей системе|
|`.git/info/exclude`|Исключения **только для текущего репо** (не коммитятся)|

# 📕 1. Синтаксис `.gitignore`
### 1️⃣ Комментарии
```bash
# Это комментарий
```

### 2️⃣ Игнорирование файла
```bash
secret.txt
```

### 3️⃣ Игнорирование всех файлов с расширением
```bash
*.log 
*.tmp
```

### 4️⃣ Игнорирование каталога
```bash
/build/
```
> Слэш `/` в начале означает “относительно корня репозитория”.  
> Без `/` — шаблон применяется ко всем подпапкам.

### 5️⃣ Игнорирование по шаблону
```
*.class 
*.py[cod]
```

Игнорирует:
- любые файлы с расширением .class
- файлы с любым именем и расширением py, pyc, pyo, pyd

### 6️⃣ Инверсия правила (отмена игнора)
```bash
*.log 
!important.log
```
> `!` — ❗означает “НЕ игнорировать” (исключение из игнора).❗

### 7️⃣ Игнорировать всё, кроме указанных

```bash
* 
!.gitignore
!/src/
```
