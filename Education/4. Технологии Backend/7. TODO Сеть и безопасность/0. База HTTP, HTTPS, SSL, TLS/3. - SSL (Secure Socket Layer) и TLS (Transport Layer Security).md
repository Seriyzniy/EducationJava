# История разработки
## Netscape
Разработка этих протоколов научилась в 1994 году в компании Netscape, которая разрабатывала браузер.

В 1994 году был создал SSL 1.0, который не был опубликован, из-за того что были найдены уязвимости. Тоже самое произошло с SSL 2.0

После этого началась разработка SSL 3.0 который стал стандартом и опубликован в RFC 6101.

## Другие компании и появление TLS
Тем временем, пока велась разработка SSL 3.0, другие компании как Microsoft, переосмыслили SSL 2.0 и решили выкатить свою версию SSL для браузера Internet Explorer.

Так поступали и другие компании для своих браузеров таких как Opera и другие.

Такая проприетарная разработка велась до 1999 года, пока не решили, что лучше все это дело стандатизировать, и передали разработку инженерному совету интернета IETF (Internet Engineering Task Forse), который разработал новую версию SSL основанную на SSL 3.0 и назвал TLS 1.0

# 1. SSL (Secure Socket Layer)
**Secure Sockets Layer** (SSL) - это криптографический протокол, обеспечивающий безопасное общение пользователя и сервера по небезопасной сети. 

Впервые был представлен публике в 1995 году, однако с 2015 года признан полностью устаревшим. 

На основе спецификации SSL 3.0 в 1996 был разработан TLS 1.0.

> [!NOTE] 
> Аббревиатура SSL используется в названиях в качестве указания семейства протоколов (OpenSSL, LibreSSL)

### OSI
Располагается между транспортным уровнем и уровнем программы-клиента (Прикладным уровнем) (FTP, HTTP и т.п.) ([подробнее про уровни телекоммуникаций](https://satoms.com/7-layers-of-the-osi-model-explained/)).

Транспортный уровень:
- Передача данных между процессами

Сеансовый уровень:
- Восстановление сеанса шифрования, который был установлен ранее для повышения производительности

Уровень представления:
- ***Представление данных в зашифрованном виде*** при передаче посети
