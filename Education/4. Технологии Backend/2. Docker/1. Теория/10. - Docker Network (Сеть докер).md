
> [!NOTE] Docker Network
> Это механизм, который обеспечивает `сетевое взаимодействие контейнеров` `между собой` и `с внешним миром`.  
> 
> 💡Он позволяет Docker-контейнерам:
> - 💡обмениваться данными по IP-адресам и DNS-именам,
> - быть изолированными от других приложений,
> - 💡управлять тем, кто может "видеть" кого.
> 
> Проще говоря, ==это виртуальная сеть внутри Docker==, которую ты можешь настраивать и использовать как обычную сеть в Linux.


> [!NOTE] `СЕТЕВОЙ ИНТЕРЙЕФС` `Docker0` при установке Docker
> При установке на сервер Docker, мы получаем СЕТЕВОЙ ИНТЕРЙЕФС `Docker0`

А при ==ЗАПУСКЕ== Docker становятся доступны ==НЕСКОЛЬКО ТИПОВ сетей==
- bridge (по умолчанию)
- host
- none
- macvlan
- ipvlan
- overlay
# 📕 1. Тип сети bridge (По умолчанию)

> [!NOTE] По умолчанию, при запуске контейнер попадает в тип сети `Bridge` (Мост)
> Контейнеры в этой сети, ИМЕЮТ ДОСТУП НАРУЖУ (в интернет)
> - могут подключаться в другим серверам
> - могут получить доступ к интеренту
> Чтобы подключиться К КОНТЕЙНЕРУ ИЗВНЕ, нужно СОЗДАТЬ МОСТ при помощи Port Mapping
> ```bash
> docker run -p 8080:80 контейнер
> ```
> ПО УМОЛЧАНИЮ, могут общаться между собой ==ТОЛЬКО по IP адресам.==
> 
> 
> Чтобы общаться можно было по именам, необходимо создать сеть
> ```bash
> docker network create NAME
> ```

- Контейнеры в одной `bridge`-сети могут общаться.
    
- Можно создать пользовательскую `bridge`-сеть, чтобы задать собственные настройки.

## Создание контейнера в сети

```bash
# Создание сети
docker network create --drive bridge mynet1

# Запуск контейнера внутри сети mynet1
docker run --network mynet1 nginx 
```

> [!NOTE] Если создать собственную сеть bridge - ОБЩАТЬСЯ ==МОЖНО ПО ИМЕНАМ==
> 

Контейнеры внутри этой сети получают свои IP адреса. Контейнеры могут общаться между собой по именам (через параметр --net указываем доменное имя контейнера)

> [!NOTE] 💡 МЕЖДУ СОБОЙ разные сети ОБЩАТЬСЯ НЕ МОГУТ💡
> СЕТИ внутри Докера - изолированы

# 📕 2. Тип сети host

> [!NOTE] Host
> Контейнер использует **сетевой стек хоста** (без изоляции). (`ПОЛУЧАЕЮТ IP-адрес ХОСТА`)
> ```bash
> docker run nginx --network host
> ```
> 💡Теперь Nginx слушает **тот же порт**, что и хост.
> 
> ИЛИ
> Создание сети host
> ```bash
> # Создание сети
> docker network create --drive host
> ```

- Контейнер ==делит сетевые интерфейсы с хостом.==
    
- ❗❗❗Нет отдельного IP для контейнера.
    
- Подходит для случаев, где важна производительность сети (например, при тестировании).
	
- Контейнеры могут общаться между собой

> [!NOTE] ❗❗❗МОЖНО СОЗДАТЬ ТОЛЬКО ОДНУ сеть типа `host`❗❗❗
>

# 📕 3. Тип сети none

> [!NOTE] Полная изоляция — контейнер ==без сети==.
> - Нет доступа к внешнему миру.
> - Контейнеры НЕ МОГУТ ОБЩАТЬСЯ МЕЖДУ СОБОЙ
> - Используется для безопасных или оффлайн-задач.
> ```bash
> docker run --network none ubuntu
> ```
> 

> [!NOTE] ❗❗❗МОЖНО СОЗДАТЬ ТОЛЬКО ОДНУ сеть типа `none`❗❗❗
>

# 📕 4. **macvlan**

> Контейнер получает **свой MAC-адрес и IP**, как полноценное устройство в локальной сети.

- Позволяет контейнерам быть видимыми в той же сети, что и хост.
    
- Полезно, если контейнеру нужен прямой доступ к физической сети.

---
# 5. ipvlan
> Контейнеры получают ТОТ ЖЕ MAC адрес ЧТО И СЕРВЕР, НО СВОИ СОБСТВЕННЫЕ IP-адреса


---
# 6. **overlay**

> Используется в **Docker Swarm** для объединения контейнеров на разных хостах.

- Позволяет контейнерам на **разных машинах** общаться между собой.
    
- Требует включённого Swarm.
