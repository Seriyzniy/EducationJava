# 📕 1. Docker Repository (репозиторий)
Вручную перемещать образы с компьютера на сервер НЕ УДОБНО.

Для этого есть Docker Repository (аналог GitHub, только для образов Docker)

> [!NOTE] `Docker Repository` (репозиторий) - Это `набор версий` одного `образа`.
> 
## Что хранится в репозитории:
Один и тот же образ в разных версиях (тегах).  

Например:
На Docker Hub есть репозиторий **nginx**
```makefile
nginx:latest
nginx:1.25
nginx:1.25-alpine
```
💡Все они — **один репозиторий (`nginx`)**, просто разные версии (теги).

У каждой версии — свой уникальный набор слоёв (`layers`).

> [!NOTE] 🧠`alpine` - минимальный размер образа🧠
> 

## Именование образов
```less
[registry]/[user_or_org]/[repository]:[tag]
```

Примеры:

|Пример|Что значит|
|---|---|
|`nginx:latest`|репозиторий `nginx` на Docker Hub (по умолчанию), тег `latest`|
|`myuser/app:1.0`|пользовательский репозиторий `app`, тег `1.0`|
|`registry.company.com/backend/api:2.3`|репозиторий `backend/api` на **своём реестре** компании|

# 📕 2. Docker Registry (реестр)

> [!NOTE] `Docker Registry` (реестр)-  Это `сервер-хранилище`, `где находятся репозитории` образов.
> 

## Что делает Registry:
- Принимает образы (`docker push`)
- Отдаёт образы (`docker pull`)
    
- Хранит метаданные и слои (`layers`)
    
- Может быть публичным или приватным

## Примеры реестров:
| Название                                               | Тип                             | Описание                          |
| ------------------------------------------------------ | ------------------------------- | --------------------------------- |
| **Docker Hub**                                         | Публичный                       | Главный публичный реестр Docker   |
|                                                        |                                 |                                   |
| **GitHub Container Registry** (`ghcr.io`)              | Публичный/приватный             | Для хранения контейнеров в GitHub |
| **GitLab Container Registry**                          | Приватный                       | Интегрируется с GitLab CI/CD      |
|                                                        |                                 |                                   |
| **Harbor**, **AWS ECR**, **Azure ACR**, **Google GCR** | Приватные корпоративные решения |                                   |
