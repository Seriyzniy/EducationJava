
> [!NOTE] ⭐ `Dockerfile` — это текстовый сценарий (`инструкция`), в котором описано, `КАК СОБРАТЬ Docker-ОБРАЗ`.  ⭐
> Он `содержит команды`, которые `Docker` `выполняет пошагово` — от базового образа до готового приложения.
> 


> [!NOTE] ❗❗❗ Важные заметки ❗❗❗
> 1) Создавать лучше в папке с проектом
> 2) ==Название ОБЯЗАТЕЛЬНО== `Dockrefile` (Dockerfile с большой буквы)
> 3) Похоже нужно создавать файл ВНУТРИ src (в Eclipse), иначе ошибки которые можно просто закрыть)


## Слои в Docker FIle
Dockerfile описывает:
- какой **базовый образ** взять (например, `ubuntu`, `openjdk`, `node`);
- какие **зависимости** установить (*БИБЛИОТЕКИ*);
- какие **файлы** скопировать внутрь образа (*ФАЙЛ ПРИЛОЖЕНИЯ*);
- какую **команду** запускать при старте контейнера (*КОМАНДА ЗАПУСКА ПРИЛОЖЕНИЯ).


На его основе Docker создаёт **многоуровневый образ** (каждая инструкция — отдельный слой).

## Основные инструкции Dockerfile
| Инструкция   | Назначение                                                                                                                                                           |
| ------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ⭐`FROM`      | Указывает базовый образ (например, `FROM ubuntu:22.04`). <br>💡Указывает на основе КАКОГО Image собираем собственный образ 💡<br>❗❗❗Это ==всегда первая строка.==❗❗❗ |
|              |                                                                                                                                                                      |
| `RUN`        | Выполняет `команду внутри контейнера` ==на ЭТАПЕ СБОРКИ==<br>(например, `RUN apt-get install -y curl`).                                                              |
| ⭐`COPY`      | `Копирует файлы` `с хоста` `в контейнер` <br>(`COPY . /app`).                                                                                                        |
| `ADD`        | Как `COPY`, но также `умеет распаковывать архивы и скачивать файлы по URL.`                                                                                          |
|              |                                                                                                                                                                      |
| ⭐`WORKDIR`   | Задает ==рабочую директорию для последующих инструкций== <br>(`WORKDIR /app`).                                                                                       |
|              |                                                                                                                                                                      |
| `CMD`        | Определяет `команду`, которая выполняется ==при СТАРТЕ КОНТЕЙНЕРА== <br>(`CMD ["java", "-jar", "app.jar"]`).                                                         |
| `ENTRYPOINT` | Задает команду, к которой можно добавлять аргументы при запуске <br>(`docker run`).                                                                                  |
|              |                                                                                                                                                                      |
| `EXPOSE`     | Указывает, `какие порты контейнер слушает` <br>(например, `EXPOSE 8080`).                                                                                            |
| `ENV`        | Определяет переменные окружения <br>(`ENV APP_ENV=prod`).                                                                                                            |
| `VOLUME`     | Создает точку монтирования для хранения данных вне контейнера.                                                                                                       |
| `LABEL`      | Добавляет метаданные к образу (автор, версия и т.д.).                                                                                                                |

## Используй ГОТОВЫЕ базовые Image
К примеру для запуска контейнера с приложением Java необходимо
- Базовый слой Linux
- OpenJDK нужной версии Java (допустим 17)
- Само приложение Java
- Команда запуска

При написании Docker File пришлось бы вручную прописывать один и те же зависимости
- базовый слой
- OpenJDK 17

> [!NOTE] ⭐ Развертывание приложения ПОВЕРХ ГОТОВЫХ ОБРАЗОВ ⭐
> Однако, есть готовые образы, с уже все необходимым для запуска приложения Java в контейнере.
На DockerHub можно просто скачать Image `OpenJDK` нужной версии, ==И К НЕМУ ДОБАВИТЬ НАШЕ ПРИЛОЖЕНИЕ==
