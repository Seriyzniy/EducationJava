
> [!NOTE] Переменные окружения
> Это `ключи и значения`, которые `передаются контейнеру при запуске` и становятся `доступными внутри него`, как обычные переменные среды Linux.
> 

Они используются для:
- 💡настройки приложения (например, `DB_URL`, `PORT`, `MODE=prod`);
- 💡передачи конфигурации без изменения образа;
- 💡безопасного хранения паролей, логинов, токенов;
    
- настройки поведения Docker-контейнера (например, `TZ`, `JAVA_OPTS` и др).

---
# 📕 1. Как задать переменные окружения?

## ✅ 1. Через флаг `-e` (или `--env`)
Самый прямой способ:
```bash
docker run -e APP_MODE=prod -e PORT=8080 myapp_image
```

📘 Внутри контейнера теперь есть переменные:
```bash
echo $APP_MODE   # prod
echo $PORT       # 8080
```

> [!NOTE] 💡МОЖНО придумать свои собственные переменные окружения и сохранить их в контейнере💡
>

---
## ✅ 2. Через `.env` файл
Docker может читать переменные из текстового файла `.env`:
```ini
# .env
APP_MODE=prod
DB_USER=admin
DB_PASS=1234
```

Запуск:
```bash
docker run --env-file .env myapp
```

📘 Внутри контейнера:
```bash
echo $DB_USER  # admin
```

---
## ✅ 3. В `Dockerfile` (через `ENV`)
Можно “==зашить==” переменные ==в образ== (будут постоянными для всех контейнеров из этого образа):
```dockerfile
FROM openjdk:17

ENV APP_HOME=/usr/src/app
ENV APP_MODE=prod

WORKDIR $APP_HOME
COPY . .
CMD ["java", "-jar", "app.jar"]
```

📘 Внутри контейнера:
```bash
echo $APP_HOME  # /usr/src/app
```

> [!NOTE] ⚠️ Минус — эти значения становятся **частью образа** (видны при `docker inspect`).

---
## ✅ 4. В `docker-compose.yml`
В `docker-compose` переменные задаются в разделе `environment:`  
или через подключение `.env` файла.

```yaml
version: "3.8"
services:
  web:
    image: nginx
    ports:
      - "8080:80"
    environment: 💡💡
      - MODE=prod
      - API_URL=http://backend:5000
  db:
    image: postgres
    environment:
      POSTGRES_USER: admin
      POSTGRES_PASSWORD: 1234
```

Или подключить отдельный `.env` файл:
```yaml
env_file:
  - .env
```

---
# 📕 2. Проверка переменных в контейнере
Посмотреть все переменные окружения внутри запущенного контейнера:
```bash
docker exec mycontainer env
```

Или одну конкретную:
```bash
docker exec mycontainer printenv APP_MODE
```

# 📕 3. Пример использование переменных окружения
Для примера работы с переменными окружения, мы установим образ сервера My-SQL

## Скачивание и запуск MySql. И установка переменной окружения при запуске
Для этого перейдем в DockerHub, и найдем там MySQL.
В первой ссылке будет образ MYSQL.
А чуть ниже команда bash для скачивания образа и запуска контейнера
```bash
$ docker run --name some-mysql -e MYSQL_ROOT_PASSWORD=my-secret-pw -d mysql:tag
```

Изменим ее
```bash
docker run --name MySqlImage -e MYSQL_ROOT_PASSWORD=pass123 -d mysql:latest
```

Запустим терминал внутри контейнера
