## **Docker Client (docker)** — клиент
Это **командная утилита**, с которой ты общаешься:
```bash
docker run hello-world
docker ps
docker build .
```

Клиент просто отправляет HTTP-запросы (REST API) к демону Docker — **по Unix-сокету или TCP**.

То есть команда `docker run` на самом деле ≈ «отправь запрос демону: создай и запусти контейнер».

## **Docker Daemon (dockerd)** — сервер
Это **фоновый процесс**, который делает всю “грязную работу”:
- создаёт и запускает контейнеры;
- управляет образами (images);
- следит за сетями и хранилищами (volumes);
- взаимодействует с Linux kernel (через namespaces, cgroups).

📍 Работает обычно как служба (`systemd service`) на хост-машине.

---

| Компонент            | Описание                                                                                                                                                              | Аналогия / Роль                          |
| -------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------- |
| **Client**           | Интерфейс, с которым взаимодействует пользователь — CLI-команда `docker` или GUI (Docker Desktop). Отправляет команды демону Docker через API.                        | «Пульт управления»                       |
| **Daemon (dockerd)** | Фоновый `процесс`, который `принимает команды` от клиента и `управляет контейнерами`, образами, сетями и томами.                                                      | «Двигатель» или «сервер Docker»          |
| **Host**             | `Машина` (физическая или виртуальная), `где запущен Docker Daemon`. <br>Предоставляет ресурсы: CPU, RAM, сеть и файловую систему.                                     | «Компьютер, где всё работает»            |
|                      |                                                                                                                                                                       |                                          |
| **Image**            | Шаблон, из которого создаются контейнеры. Содержит файловую систему (слои) и инструкции для запуска приложения.                                                       | «Снимок готовой системы»                 |
| **Container**        | Запущенный экземпляр образа в изолированной среде. Имеет своё пространство процессов, сеть и файловую систему.                                                        | «Рабочий процесс / приложение»           |
| **Repository**       | Набор связанных образов (разных версий одного приложения). Например: `postgres:14`, `postgres:latest`.                                                                | «Каталог версий образа»                  |
| **Registry**         | Хранилище, где размещаются репозитории образов. Может быть публичным или приватным.                                                                                   | «Хранилище образов»                      |
|                      |                                                                                                                                                                       |                                          |
| **Docker Hub**       | Публичный Docker Registry по умолчанию — официальный сайт, где размещаются образы (например, `nginx`, `mysql`).                                                       | «GitHub для Docker-образов»              |
| **Dockerfile**       | Текстовый файл с инструкциями, как собрать образ: какие базовые слои использовать, что установить, какие файлы скопировать и как запускать приложение.                | «Рецепт создания образа»                 |
| **Docker Compose**   | Инструмент для описания и запуска **многоконтейнерных приложений** через YAML-файл (`docker-compose.yml`). Позволяет связать контейнеры, сети и тома в одном проекте. | «Оркестратор для нескольких контейнеров» |
