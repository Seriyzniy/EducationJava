# 📕 1. Контейнеры
## Команда: `docker run`

Создаёт и запускает ==новый контейнер== из образа. КАЖДЫЙ РАЗ НОВЫЙ КОНТЕЙНЕР СОЗДАЕТ
```bash
docker run [ОПЦИИ] <image> [КОМАНДА]
```

```bash
# Ключи
-i, --interactive     — держит stdin открытым (интерактивный режим)
-t, --tty             — подключает псевдотерминал (tty)

-d, --detach          — 💡запускает контейнер в фоне💡
--name <имя>          — задаёт имя контейнера
--rm                  — удалить контейнер после остановки

-p <хост:контейнер>   — ⭐⭐⭐ проброс портов ⭐⭐⭐

-v <путь_хоста:путь_контейнера> — подключает том или директорию
-e, --env KEY=VALUE   — 💡задаёт переменные окружения💡
--network <сеть>      — подключает контейнер к сети
--cpus=N              — ограничение по CPU
--memory, -m 512m     — ограничение по оперативной памяти
```

Примеры:
```bash
# Запустить интерактивную сессию Ubuntu
docker run -it ubuntu bash

# ⭐⭐⭐Запустить nginx с пробросом порта ⭐⭐⭐
docker run -d -p 8080:80 --name web nginx

# Удалить контейнер после завершения
docker run --rm ubuntu echo "Hello"

# Передать переменные окружения
docker run -e MODE=prod -e DEBUG=false myapp
```


> [!NOTE] ⭐Можно попасть внутрь контейнера при помощи `docker run -it container_name`⭐
> После этого можно работать с терминалом ВНУТРИ КОНТЕЙНЕРА


## Команда: `docker exec`
Выполняет команду в уже запущенном контейнере.
```bash
docker exec [ОПЦИИ] <container> <command>
```

```bash
-it           — интерактивный терминал
-d            — выполнить в фоне
-e KEY=VALUE  — задать переменные окружения
```

Примеры:
```bash
# Войти в контейнер
💡 docker exec -it web bash 💡

# Выполнить команду внутри контейнера
💡 docker exec myapp ls /app 💡

# Выполнить команду в фоне
docker exec -d myapp touch /tmp/test.txt
```

## 🛑Команды`docker stop` / `docker start` / `docker restart`
Управление ==состоянием контейнеров==.
```bash
docker stop <container_name> / <container_id>
docker start <container_name> / <container_id>
docker restart <container_name> / <container_id>
```

Примеры
```bash
docker stop web
docker start web
docker restart web
```

## Команда `docker rm`
Удаляет контейнеры.
```bash
docker rm [ОПЦИИ] <container>...
```

```bash
-f, --force   — принудительно удалить (даже если работает)
-v, --volumes — удалить связанные тома
```

Примеры:
```bash
docker rm myapp
docker rm -f old_web
docker rm -v mydb
```

## Команда `docker ps`
Показывает список контейнеров.
```bash
docker ps [ОПЦИИ]
```

```bash
-a, --all      — показать все, включая остановленные
-q, --quiet    — вывести только ID
-f, --filter   — фильтрация (по статусу, имени и т. д.)
--format       — форматированный вывод
```

Примеры:
```bash
docker ps
docker ps -a
docker ps -f "status=exited"
docker ps --format "{{.Names}} - {{.Status}}"
```

## Команда `docker logs`
Показывает логи контейнера.
```bash
docker logs [ОПЦИИ] <container>
```

```bash
-f, --follow   — следить за логами в реальном времени
--tail=N       — последние N строк
```

Примеры
```bash
docker logs web
docker logs -f web
docker logs --tail=20 web
```

# 📕 2. Образы
##  Команда `docker build`
Создаёт новый образ из Dockerfile.

```bash
docker build [ОПЦИИ] <путь_до_Dockerfile>
```

```bash
-t, --tag name:tag     — присвоить имя и тег 
-f, --file Dockerfile  — указать путь к Dockerfile 
--no-cache             — не использовать кеш 
--build-arg KEY=VALUE  — передать аргументы в Dockerfile (ARG) 
--pull                 — всегда тянуть свежий базовый образ
```

**Примеры:**
```bash
docker build -t myapp:latest . 
docker build -f Dockerfile.prod -t myapp:prod . 
docker build --no-cache -t app:v2 . 
docker build --build-arg VERSION=1.2.0 .
```

---
## Команда `docker pull`
Загружает образ из Docker Hub или другого реестра.
```bash
docker pull <имя>:<тег>
```

**Примеры:**
```bash
docker pull ubuntu:22.04 docker pull nginx
```

---
## Команда `docker push`
Отправляет локальный образ в реестр.
```
docker push <имя>:<тег>
```

**Примеры:**
```bash
docker push myrepo/myapp:1.0
```

---

## Команда `docker rmi`
Удаляет образы.

```bash
docker rmi [ОПЦИИ] <image>...
```

```bash
-f, --force   — принудительно удалить
```

**Примеры:**
```bash
docker rmi ubuntu:22.04 docker rmi -f myoldapp
```

---

##  Команда `docker images`
Показывает список локальных образов.

```bash
docker images [ОПЦИИ]
```

```bash
-a, --all      — показать промежуточные слои 
-q, --quiet    — вывести только ID 
-f, --filter   — фильтрация
```

**Примеры:**
```bash
docker images docker images -q docker images -f "dangling=true"
```

# 🌐 3. Сети (Networks)
## Команда `docker network`
Управление сетями контейнеров.
```bash
docker network [COMMAND]
```

**Подкоманды:**
```bash
create <name>   — создать сеть 
ls              — список сетей 
inspect <name>  — подробная информация 
rm <name>       — удалить сеть 
connect/disconnect — 💡подключить контейнер💡
```

**Примеры:**

```bash
docker network create mynet docker network ls docker network connect mynet web docker network inspect bridge
```

## Команда `docker volume`

Управление хранилищами данных контейнеров.

```bash
docker volume [COMMAND]
```

**Подкоманды:**

```bash
create <name>   — создать том 
ls              — список томов 
inspect <name>  — информация о томе 
rm <name>       — удалить том 
prune           — удалить неиспользуемые тома
```

**Примеры:**

```bash
docker volume create mydata docker volume ls docker volume rm olddata docker volume prune
```

# 📕 5. Системные команды

---

## 🧹 Команда `docker system prune`

Очищает неиспользуемые образы, контейнеры и сети.

```bash
docker system prune [ОПЦИИ]
```

```bash
-a, --all       — удалить всё, включая образы без тегов 
-f, --force     — не спрашивать подтверждение
```

**Примеры:**

```bash
docker system prune -a
```

# 📕 6. Docker Compose

---

## 🏗️ Команда `docker compose up`

Создаёт и запускает контейнеры из `docker-compose.yml`.

```bash
docker compose up [ОПЦИИ]
```

```bash
-d, --detach          — запуск в фоне 
--build               — пересобрать образы 
--force-recreate      — пересоздать контейнеры 
--remove-orphans      — удалить неописанные контейнеры
```

**Примеры:**

```bash
docker compose up docker compose up -d docker compose up --build
```

---

## Команда `docker compose down`

Останавливает и удаляет все контейнеры проекта.

```bash
docker compose down [ОПЦИИ]
```

```bash
-v, --volumes   — удалить тома
```

**Примеры:**

```bash
docker compose down docker compose down -v
```

---

## Команда `docker compose logs`

Показывает логи всех сервисов.

```bash
docker compose logs [ОПЦИИ]
```

```bash
-f, --follow  — следить за логами в реальном времени
```

**Примеры:**

```bash
docker compose logs docker compose logs -f
```

---

## Команда `docker compose exec`

Выполняет команду внутри контейнера сервиса.

```bash
docker compose exec [ОПЦИИ] <service> <command>
```

```bash
-it  — интерактивный терминал
```

**Примеры:**

```bash
docker compose exec web bash docker compose exec db psql -U user
```
