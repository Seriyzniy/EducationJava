## Проблема удаление данных при остановки/удалении контейнера

Допустим в контейнере мы запустили сервер MySQL
По умолчанию сервер MySQL на Linux хранит данные в каталоге `/var/lib/mysql`

![[Pasted image 20251022160109.png]]

Если мы вызываем команды (то есть удалим контейнер или захотим обновить контейнер, то все равно сначала нужно удалить старый и установить новый)
- docker stop myssql
- docker rmi mysql

❗❗❗То каталог `/var/lib/mysql` УДАЛИТСЯ

> [!NOTE] ❗❗❗Такая проблема возникает с ЛЮБЫЙ КОНТЕЙНЕРОМ, который хоть как-то сохраняет данные в памяти❗❗❗
> 

## Решение
Для того, чтобы не потерять данные, которые были изменены в контейнере, НУЖНО СОХРАНИТЬ ДАННЫЕ НА СЕРВЕРЕ (Хосте где запущены контейнеры)

> [!NOTE] ⭐⭐Если задан `Volume`, то при создании контейнера, `данные будут считываться из ДИРЕКТОРИИ СЕРВЕРА`, а не контейнера⭐⭐
>

# 📕 1. Volumes(Томы) в Docker

> [!NOTE] `Mapping томов` — это механизм, который позволяет `связывать файлы и каталоги контейнера` С `хостовой системой`.  
> 
> То есть ты можешь `«примонтировать» папку` с твоего `компьютера` `ВНУТРЬ` `контейнера` — и данные будут доступны **с обеих сторон**.
> 

## Типы томов в Docker

| Тип                                  | Пример                                    | Где хранится                     | Описание                                 |
| ------------------------------------ | ----------------------------------------- | -------------------------------- | ---------------------------------------- |
| **Anonymous Volume (анонимный)**     | `docker run -v /app/data`                 | `/var/lib/docker/volumes/...`    | Docker сам создаёт и управляет томом     |
| **Named Volume (именованный)**       | `docker run -v mydata:/app/data`          | `/var/lib/docker/volumes/mydata` | У тебя есть доступ к этому тому по имени |
| **Bind Mount (связывание с хостом)** | `docker run -v /home/user/data:/app/data` | конкретная папка на хосте        | Прямое подключение твоей локальной папки |

### 1. Связанный с хостом
![[Pasted image 20251022162744.png]]

```bash
docker run -v /opt/mysql_data:/var/lib/mysql mysql
```
- `/opt/mysql_data` - директория сервера
- `/var/lib/mysql` - директория контейнера

### 2. Анонимный том
```bash
docker run -v /var/lib/mysql mysql
```

Тоже создается директория на хосте, НО В ПУТИ для конкретного контейнера УКАЗЫВАЕТСЯ ХЕШ

![[Pasted image 20251022162823.png]]

### 3. Именованный том
![[Pasted image 20251022163053.png]]

```bash
docker run -v имя_каталога:/var/lib/mysql mysql
```
