# 1. Общий синтаксис
```bash
kubectl [команда] [тип-ресурса] [имя] ❗❗[флаги]❗❗
```
- `[команда]` — действие (`get`, `apply`, `delete` и т.д.)
- `[тип-ресурса]` — например, `pods`, `deployments`, `services`
- `[имя]` — конкретный объект (можно опустить для всех)
- `[флаги]` — дополнительные параметры (`-n`, `-o`, `--all` и т.п.)

# 2. Создание и применение

## Команда: kubectl apply
```bash
kubectl apply -f файл.yaml
```

**Назначение**: Создание или обновление ресурсов в кластере из файлов конфигурации. 
Команда сравнивает текущее состояние с желаемым и вносит необходимые изменения. Это предпочтительный способ управления ресурсами в production.

Все ключи команды:
- -f, --filename - путь к файлу или директории
    
- -k, --kustomize - путь к kustomization.yaml
    
- --prune - удалить ресурсы, которых больше нет в конфигурации
    
- --prune-whitelist - разрешенные типы ресурсов для удаления

Пример использования
```bash
kubectl apply -f deployment.yaml
kubectl apply -k ./kustomize-dir
```


## Команда: kubectl create
```bash
kubectl create -f файл.yaml
```

**Назначение**: Создание новых ресурсов в кластере. 
В отличие от apply, ==создает ресурсы только если они не существуют.== Подходит для инициализации ресурсов, но не для последующих обновлений.

Все ключи команды:
- -f, --filename - путь к файлу
    
- --save-config - сохранить конфигурацию в аннотации ресурса
    
- --dry-run - симуляция выполнения
    

Пример использования
```bash
kubectl create -f pod.yaml
kubectl create deployment nginx --image=nginx
```

# 3. Просмотр и получение информации

## Команда: kubectl get

```bash
kubectl get [тип_ресурса]
```

**Назначение**: Получение основной информации о ресурсах в кластере. 
Показывает базовые данные (имя, статус, возраст) и позволяет быстро оценить состояние системы.

Все ключи команды:
- -A, --all-namespaces - все пространства имен
    
- -l, --selector - фильтр по меткам
    
- -w, --watch - режим наблюдения
    
- --show-labels - показать метки
    
- -o, --output - формат вывода (wide, yaml, json, name)
    

Пример использования
```bash
kubectl get pods
kubectl get pods -A
kubectl get pods -l app=nginx
kubectl get pods -o wide
```

## Команда: kubectl describe

```bash
kubectl describe [тип_ресурса] [имя]
```

**Назначение**: Получение детальной информации о конкретном ресурсе. Показывает события, конфигурацию, статусы контейнеров и другую отладочную информацию.

Все ключи команды:
- -A, --all-namespaces - все пространства имен
    
- -l, --selector - фильтр по меткам
    
- -f, --filename - путь к файлу
    

Пример использования
```bash
kubectl describe pod my-pod
kubectl describe service my-service
```

# 4. Удаление

## Команда: kubectl delete
```bash
kubectl delete [тип_ресурса] [имя]
```

**Назначение**:Удаление ресурсов из кластера. Поддерживает graceful shutdown, позволяя приложениям корректно завершить работу перед удалением.

Все ключи команды:
- -f, --filename - путь к файлу
    
- --all - удалить все ресурсы указанного типа
    
- -l, --selector - фильтр по меткам
    
- --now - немедленное удаление
    
- --grace-period - период graceful deletion
    

Пример использования
```bash
kubectl delete pod my-pod
kubectl delete -f deployment.yaml
kubectl delete pods --all
```

# 5. Отладка и логи

## Команда: kubectl logs
```bash
kubectl logs [pod] [контейнер]
```

**Назначение**: Просмотр логов контейнеров. Критически важна для отладки приложений, анализа ошибок и мониторинга работы контейнеров.

Все ключи команды:
- -f, --follow - следить за логами в реальном времени
    
- --tail - количество последних строк
    
- --previous - логи предыдущего контейнера
    
- -c, --container - указать контейнер
    
- --timestamps - показывать временные метки
    

Пример использования
```bash
kubectl logs my-pod
kubectl logs my-pod -f
kubectl logs my-pod --tail=100
```

## Команда: kubectl exec
```bash
kubectl exec [pod] [команда]
```

**Назначение**: Выполнение команд внутри запущенных контейнеров. 
Позволяет интерактивно работать с контейнерами, проводить отладку и диагностику.

Все ключи команды:
- -i, --stdin - передавать stdin
    
- -t, --tty - allocate a TTY
    
- -c, --container - указать контейнер
    
- -- bash/sh - запустить shell
    

Пример использования

```bash
kubectl exec -it my-pod -- bash
kubectl exec my-pod -- ls /app
```

# 6. Модификация работающих ресурсов

## Команда: kubectl edit
```bash
kubectl edit [тип_ресурса] [имя]
```

**Назначение**: Интерактивное редактирование ресурсов непосредственно в кластере. Открывает конфигурацию ресурса в текстовом редакторе для быстрых изменений.

Все ключи команды:
- --output-version - версия API
    
- --record - записать команду в аннотацию
    
- --windows-line-endings - использовать Windows line endings
    

Пример использования

```bash
kubectl edit deployment nginx
kubectl edit pod my-pod
```

## Команда: kubectl patch
```bash
kubectl patch [тип_ресурса] [имя] -p 'патч'
```

**Назначение**: Частичное обновление ресурсов с помощью JSON или YAML патчей. Эффективна для точечных изменений без полного пересоздания ресурса.

Все ключи команды:
- -p, --patch - JSON или YAML патч
    
- --type - тип патча (strategic, merge, json)
    
- --local - локальная операция
    

Пример использования

```bash
kubectl patch pod valid-pod -p '{"spec":{"containers":[{"name":"kubernetes-serve-hostname","image":"new image"}]}}'
```

# 7.Масштабирование

## Команда: kubectl scale

```bash
kubectl scale [тип_ресурса] [имя] --replicas=число
```

**Назначение**: ==Изменение количества реплик== для ресурсов, поддерживающих масштабирование (Deployments, StatefulSets, ReplicaSets). Позволяет быстро адаптировать приложение к нагрузке.

Все ключи команды:
- --replicas - количество реплик
    
- --current-replicas - текущее количество реплик для проверки
    
- --timeout - таймаут операции
    

Пример использования
```bash
kubectl scale deployment nginx --replicas=3
kubectl scale --replicas=5 rc/nginx
```

# 8. Конфигурация и контекст

## Команда: kubectl config

```bash
kubectl config [подкоманда]
```

**Назначение**: Управление конфигурацией kubectl. 
Позволяет переключаться между кластерами, просматривать настройки и управлять учетными данными.

Все подкоманды:
- current-context - текущий контекст
	
- get-contexts - список контекстов
    
- use-context - переключить контекст
    
- set-cluster - настроить кластер
    
- set-credentials - настроить учетные данные
    
- view - показать конфигурацию
    

Пример использования
```bash
kubectl config get-contexts
kubectl config use-context my-cluster
kubectl config view
```

# 9. Порталы и доступ

## Команда: kubectl port-forward

```bash
kubectl port-forward [pod] [локальный_порт]:[удаленный_порт]
```

**Назначение**: Проброс портов из локальной машины к подам или сервисам в кластере. Обеспечивает временный доступ к приложениям для тестирования и отладки.

Все ключи команды:
- --address - адрес для прослушивания
    
- --pod-running-timeout - таймаут ожидания запуска пода
    

Пример использования

```bash
kubectl port-forward my-pod 8080:80
kubectl port-forward service/my-service 8080:80
```

## Команда: kubectl proxy
```bash
kubectl proxy [порт]
```

**Назначение**: Создание прокси-сервера к Kubernetes API. Предоставляет безопасный доступ к API через локальный прокси с автоматической аутентификацией.

Все ключи команды:
- --port - порт для прослушивания
    
- --address - адрес для привязки
    
- --api-prefix - префикс API
    
- --www - статические файлы
    

Пример использования

```bash
kubectl proxy
kubectl proxy --port=8001
```

# 10. Управление нодами

## Команда: kubectl cordon

```bash
kubectl cordon [node]
```

**Назначение**: Помечение ноды как недоступной для планирования новых подов. Используется при обслуживании нод для предотвращения размещения новых workload.

Все ключи команды:
- --selector - фильтр по меткам
    

Пример использования

``` bash
kubectl cordon node-1
```

## Команда: kubectl uncordon

```bash
kubectl uncordon [node]
```

**Назначение**: Снятие ограничения с ноды, разрешение планирования новых подов. Используется после завершения обслуживания ноды.

Все ключи команды:
- --selector - фильтр по меткам
    

Пример использования

```bash
kubectl uncordon node-1
```

## Команда: kubectl drain

```bash
kubectl drain [node]
```

**Назначение**: Подготовка ноды к обслуживанию путем graceful удаления всех подов с переносом их на другие ноды. Критически важна для безопасного обслуживания кластера.

Все ключи команды:
- --ignore-daemonsets - игнорировать DaemonSets
    
- --delete-emptydir-data - удалить EmptyDir данные
    
- --force - принудительное выполнение
    
- --grace-period - период graceful deletion
    

Пример использования
```bash
kubectl drain node-1 --ignore-daemonsets
```

# 11.Полезные утилиты

## Команда: kubectl explain

```bash
kubectl explain [ресурс]
```

**Назначение**: Получение документации по ресурсам Kubernetes. Показывает структуру и описание полей для ресурсов, помогая в написании конфигураций.

Все ключи команды:
- --recursive - рекурсивное объяснение
    
- --api-version - версия API
    

Пример использования

```bash
kubectl explain pod.spec.containers
```

## Команда: kubectl api-resources
```bash
kubectl api-resources
```

**Назначение**: Просмотр списка всех доступных ресурсов API. Помогает узнать, какие типы ресурсов поддерживаются в кластере и их short names.

Все ключи команды:
- --namespaced - только namespaced ресурсы
    
- --verbs - фильтр по операциям
    
- -o, --output - формат вывода
    

Пример использования

```bash
kubectl api-resources --namespaced=true
```

## Команда: kubectl version
```bash
kubectl version
```

**Назначение**: Проверка версий клиента и сервера Kubernetes. Важна для диагностики проблем совместимости.

Все ключи команды:
- --client - только клиентская версия
    
- --output - формат вывода
    
- --short - краткий вывод
    

Пример использования

```bash
kubectl version --client
```

# 12. Поиск и фильтрация

## Команда: kubectl top
```bash
kubectl top [node|pod]
```

**Назначение**: Просмотр метрик использования ресурсов (CPU, memory) для нод и подов. Требует установленного Metrics Server в кластере.

Все ключи команды:
- --selector - фильтр по меткам
    
- --containers - показать контейнеры
    
- --all-namespaces - все пространства имен
    

Пример использования
```bash
kubectl top nodes
kubectl top pods -A
```

## Команда: kubectl wait
```bash
kubectl wait [ресурс] [условие]
```

**Назначение**: Ожидание выполнения определенных условий для ресурсов. Полезна в скриптах для синхронизации операций.

Все ключи команды:
- --for - условие ожидания
    
- --timeout - таймаут ожидания
    
- -A, --all-namespaces - все пространства имен
    

Пример использования
```bash
kubectl wait --for=condition=Ready pod/my-pod
kubectl wait --for=delete pod/my-pod
```
