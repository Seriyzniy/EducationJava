# 1. Установка и настройка

## Команда: helm repo

**Назначение**: Управление репозиториями chart'ов - добавление, обновление, просмотр и удаление источников chart'ов

```bash
helm repo [команда] [репозиторий] [url]
```

Все подкоманды:
- add - добавить репозиторий
    
- list - список репозиториев
    
- update - обновить информацию о chart'ах
    
- remove - удалить репозиторий
    

Пример использования

```bash
helm repo add bitnami https://charts.bitnami.com/bitnami
helm repo update
helm repo list
```

## Команда: helm search

**Назначение**: Поиск chart'ов в репозиториях и в Helm Hub для обнаружения доступных приложений

```bash
helm search [команда] [термин] [флаги]
```

Все подкоманды:
- repo - поиск в подключенных репозиториях
    
- hub - поиск в Helm Hub  
    Все ключи:
    
- -l, --versions - показать все версии
    
- --version - конкретная версия
    

Пример использования

```bash
helm search repo nginx
helm search hub database
helm search repo bitnami/nginx --versions
```

# 2. Установка и управление релизами

## Команда: helm install

**Назначение**: ==Установка нового релиза chart'а в кластер== Kubernetes с возможностью кастомизации параметров

```bash
helm install [имя] [chart] [флаги]
```

Все ключи:
- -f, --values - ❗❗❗файлы values.yaml❗❗❗ (можно создать свои)
    
- --set - установить значения через командную строку
    
- --dry-run - симуляция установки
    
- --wait - ждать завершения rollout'а
    
- --timeout - таймаут операции
    
- --create-namespace - создать namespace если не существует
    

Пример использования
```bash
helm install my-nginx bitnami/nginx

helm install my-app ./my-chart -f values.yaml --set replicaCount=3
```

## Команда: helm upgrade

**Назначение**: Обновление существующего релиза до новой версии или конфигурации

```bash
helm upgrade [релиз] [chart] [флаги]
```


Все ключи:
- -f, --values - файлы values.yaml
    
- --set - установить значения через командную строку
    
- --install - установить если релиз не существует
    
- --wait - ждать завершения rollout'а
    
- --timeout - таймаут операции
    
- --force - принудительное обновление
    
- --debug - отладочный вывод
    
- --dry-run - симуляция обновления без применения изменений
    
- --history-max - ограничить историю релизов

Пример использования

```bash
helm upgrade my-nginx bitnami/nginx --set service.type=LoadBalancer

helm upgrade my-app ./my-chart -f new-values.yaml

helm upgrade my-release . --debug --dry-run
```

## Команда: helm rollback

**Назначение**: Откат релиза к предыдущей ревизии при возникновении проблем

```bash
helm rollback [релиз] [ревизия] [флаги]
```

Все ключи:
- --wait - ждать завершения rollout'а
    
- --timeout - таймаут операции
    
- --force - принудительный откат
    

Пример использования

```bash
helm rollback my-nginx 1
helm rollback my-app 2 --wait
```

## Команда: helm uninstall

**Назначение**: Удаление релиза из кластера Kubernetes

```bash
helm uninstall [релиз] [флаги]
```
Все ключи:

- --keep-history - сохранить историю
    
- --no-hooks - пропустить hooks
    
- --timeout - таймаут операции
    

Пример использования

```bash
helm uninstall my-nginx
helm uninstall my-app --keep-history
```

# 3. Просмотр информации

## Команда: helm list

**Назначение**: Просмотр списка установленных релизов в кластере

```bash
helm list [флаги]
```

Все ключи:
- -A, --all-namespaces - все пространства имен
    
- -f, --filter - фильтр по имени
    
- -o, --output - формат вывода (table, json, yaml)
    
- --deployed - только deployed релизы
    

Пример использования

```bash
helm list
helm list -A
helm list --filter "nginx" -o json
```
## Команда: helm status

**Назначение**: Просмотр детального статуса конкретного релиза

```bash
helm status [релиз] [флаги]
```
Все ключи:
- -o, --output - формат вывода (table, json, yaml)
    
- --show-desc - показать описание
    
- --revision - конкретная ревизия
    

Пример использования

```bash
helm status my-nginx
helm status my-app -o yaml
```

## Команда: helm history

**Назначение**: Просмотр истории изменений и ревизий релиза

```bash
helm history [релиз] [флаги]
```

Все ключи:
- -o, --output - формат вывода
    
- --max - максимальное количество записей
    

Пример использования
```bash
helm history my-nginx
helm history my-app --max=10
```

# 4. Работа с chart'ами

## Команда: helm create

**Назначение**: Создание шаблона нового chart'а с базовой структурой
```bash
helm create [имя] [флаги]
```

Все ключи:
- --starter - использовать starter chart
    

Пример использования

```bash
helm create my-chart

helm create my-app --starter=webapp
```

## Команда: helm template

**Назначение**: Локальный рендеринг chart'а без установки в кластер для отладки
```bash
helm template [имя] [chart] [флаги]
```

Все ключи:
- -f, --values - файлы values.yaml
    
- --set - установить значения через командную строку
    
- --output-dir - директория для вывода
    
- --validate - валидация chart'а
    

Пример использования

```bash
helm template my-app ./my-chart
helm template my-nginx bitnami/nginx -f values.yaml
```

## Команда: helm package

**Назначение**: Упаковка chart'а в архив .tgz для распространения

```bash
helm package [путь] [флаги]
```
Все ключи:
- --destination - директория для сохранения
    
- --version - версия пакета
    
- --app-version - версия приложения
    

Пример использования
```bash
helm package ./my-chart
helm package ./my-app --version=1.2.3
```

## Команда: helm pull

**Назначение**: Загрузка chart'а из репозитория для локального использования

```bash
helm pull [chart] [флаги]
```

Все ключи:

- --version - конкретная версия
    
- --destination - директория для сохранения
    
- --untar - распаковать архив
    

Пример использования

```bash
helm pull bitnami/nginx
helm pull bitnami/mysql --version=9.0.0 --untar
```

# 5. Проверка и тестирование

## Команда: helm lint

**Назначение**: Проверка chart'а на ошибки и соответствие best practices

```bash
helm lint [путь] [флаги]
```

Все ключи:
- --strict - строгая проверка
    
- --quiet - только ошибки
    
- --with-subcharts - проверять subcharts
    

Пример использования
```bash
helm lint ./my-chart

helm lint ./my-app --strict
```

## Команда: helm test

**Назначение**: Запуск тестов для установленного релиза для проверки работоспособности
```bash
helm test [релиз] [флаги]
```

Все ключи:
- --timeout - таймаут операции
    
- --logs - показать логи тестов
    
- --filter - фильтр тестов
    

Пример использования
```bash
helm test my-nginx
helm test my-app --logs
```

## Команда: helm get

**Назначение**: Получение различной информации о установленном релизе

```bash
helm get [команда] [релиз]
```

Все подкоманды:
- values - values релиза
    
- manifest - манифест ресурсов
    
- hooks - hooks релиза
    
- notes - notes релиза
    
- all - вся информация
    

Пример использования

```bash
helm get values my-nginx
helm get manifest my-app
helm get all my-release`
```

# 6. Управление зависимостями

## Команда: helm dependency

**Назначение**: Управление зависимостями chart'а - обновление и просмотр

```bash
helm dependency [команда] [chart]
```

Все подкоманды:
- update - обновить зависимости
    
- list - список зависимостей
    
- build - собрать зависимости
    

Пример использования
```bash
helm dependency update ./my-chart
helm dependency list ./my-app
```

# 7. Плагины

## Команда: helm plugin

**Назначение**: Управление плагинами Helm - установка, обновление и удаление

```bash
helm plugin [команда] [имя]
```

Все подкоманды:
- install - установить плагин
    
- list - список плагинов
    
- uninstall - удалить плагин
    
- update - обновить плагин
    

Пример использования

```bash
helm plugin install https://github.com/helm/helm-diff
helm plugin list
```
