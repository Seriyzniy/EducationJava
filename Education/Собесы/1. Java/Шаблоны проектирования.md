**1. ==Одиночка (Singleton):**== Обеспечивает создание ==ТОЛЬКО ОДНОГО== экземпляра класса и предоставляет глобальную точку доступа к этому экземпляру.

```java
class Singleton {  
    private static Singleton instance;  
  
    private Singleton() {}  
  
    public static Singleton getInstance() {  
        if (instance == null) {  
            instance = new Singleton();  
        }  
        return instance;  
    }  
}
```

**3. ==Фабричный метод==**: Позволяет создавать объекты разного типа, при помощи одного метода

Допустим, вы разрабатываете систему для отображения диалогов. В зависимости от операционной системы (Windows, macOS, Linux) вам нужно создавать соответствующие кнопки. Создавать кнопки через фабричный метод позволяет упростить добавление новых типов кнопок без изменения основного кода.

```java
public interface Button {
    void render();
    void onClick();
}

// Конкретный продукт для Windows
public class WindowsButton implements Button {
    @Override
    public void render() {
        System.out.println("Рисую кнопку Windows");
    }

    @Override
    public void onClick() {
        System.out.println("Клик по кнопке Windows");
    }
}

// Конкретный продукт для macOS
public class MacOSButton implements Button {
    @Override
    public void render() {
        System.out.println("Рисую кнопку macOS");
    }

    @Override
    public void onClick() {
        System.out.println("Клик по кнопке macOS");
    }
}
```

```java
// Абстрактная фабрика
public abstract class Dialog {
    // Фабричный метод
    public abstract Button createButton();

    // Логика работы с продуктом
    public void renderDialog() {
        Button button = createButton();
        button.render();
        button.onClick();
    }
}

// Конкретный создатель для Windows
public class WindowsDialog extends Dialog {
    @Override
    public Button createButton() {
        return new WindowsButton();
    }
}

// Конкретный создатель для macOS
public class MacOSDialog extends Dialog {
    @Override
    public Button createButton() {
        return new MacOSButton();
    }
}
```

```java
public class Main {
    public static void main(String[] args) {
        Dialog dialog;

        // Выбираем создателя на основе ОС
        String os = System.getProperty("os.name").toLowerCase();
        if (os.contains("win")) {
            dialog = new WindowsDialog();
        } else if (os.contains("mac")) {
            dialog = new MacOSDialog();
        } else {
            throw new UnsupportedOperationException("Неизвестная ОС");
        }

        // Отображаем диалог
        dialog.renderDialog();
    }
}

```

==**4. Adapter**== преобразовать интерфейс одного класса в интерфейс, ожидаемый клиентом. Это позволяет классам с несовместимыми интерфейсами работать вместе.

```java
// Целевой интерфейс (Target)
interface DataBase {
    void insert();
}

// Адаптируемый класс (Adaptee)
class MyProgramm {
    public void dataBaseInsert() {
        System.out.println("Вставка в БД");
    }
}

// Адаптер (Adapter) через наследование
class Adapter extends MyProgramm implements DataBase {
    @Override
    public void insert() {
        // Преобразуем вызов
        dataBaseInsert();
    }
}

// Клиентский код
public class Main {
    public static void main(String[] args) {
        DataBase adapter = new Adapter(); // Используем адаптер
        adapter.insert(); // Вызываем метод через адаптер
    }
}
```

**6. ==Фасад**==  предоставляет упрощённый интерфейс для работы с группой сложных подсистем.

Он помогает скрыть сложность системы, предоставляя одну точку входа (интерфейс), через которую можно взаимодействовать с несколькими компонентами.

По сути, мы пишем отдельный класс (Фасад), который имеет один или несколько простых методов. Эти методы/метод вызывают необходимые методы сложных объевтов

Например:
Нам нужно написать кинотеатр, пользователь должен просто включить фильм, ПРИ ЭТОМ НЕ задумываясь как включить проектор, как настроить звуковую систему и т.д.

Пользователь просто будет использовать метод `watchFilm(String filmName)`, а этот метод уже выполнит все необходимое
