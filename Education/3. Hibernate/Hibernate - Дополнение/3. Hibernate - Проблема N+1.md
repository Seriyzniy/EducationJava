> [!NOTE] Определение ТИПА ЗАГРУЗКИ В HQL
> Для решения проблемы нужно объединить две таблицы и СРАЗУ ЗАГРУЗИТЬ данные о товарах соответствующим людям
> Для НЕ ЛЕНИВОЙ загрузки в ЗАПРОСЕ указываем КЛЮЧЕВОЕ слово `FETCH`

После этого в БД поступит 1 запрос, в результате на выходе получим данные из объединенных таблиц. 
Эти данные сохраним в список людей, для каждого из который данные о товарах будут ЗАГРУЖЕННЫ СРАЗУ
