# 1. Создание таблиц в БД

##### Создадим таблицу Клиентов
```sql
CREATE TABLE Clients(
	id int PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
	c_name varchar(25) NOT NULL,
	c_age int CHECK(c_age < 100 AND c_age > 0)
```

# 2. Создание классов сущностей в Java
Создадим класс `Client`
```java
@Entity
@Table(name = "Clients")
public class Client {
	@Id
	@Column(name = "id")
	@GeneratedValue(strategy = GenerationType.IDENTITY)
	private int client_id;
	
	@Column(name = "c_name")
	private String name;
	
	@Column(name = "c_age")
	private int age;
	
	public Client() {};
	public Client(String name, int age) {
		this.name = name;
		this.age = age;
	}
	
	public int getAge() {
		return age;
	}
	public int getId() {
		return client_id;
	}
	public String getName() {
		return name;
	}
	public void setId(int id) {
		this.client_id = id;
	}
	public void setAge(int age) {
		this.age = age;
	}
	public void setName(String name) {
		this.name = name;
	}
	public String toString() {
		return this.client_id + ", " +this.name + ", " + this.age;
	}
}
```

##### ВСЕ товары ОПРЕДЕЛЕННОГО человека
Напишем код, который выводит список всех товаров для клиента 3. 

Для этого получим из таблицы БД объект класса `Client`, а затем получим при помощи метода список товаров для объекта и выведем его
> [!NOTE] Hibernate и методы `get`
> При выполнении метода `getList()`, Hibernate САМ составит запрос к БД и получит список записей

В результате получим
```java
public class App {
	public static void main(String[] args) {
		Configuration myConfigHiber = new Configuration();
		myConfigHiber.addAnnotatedClass(Client.class);
		myConfigHiber.addAnnotatedClass(Item.class);
		
		SessionFactory mySessionFact = myConfigHiber.buildSessionFactory();
		Session mySession = mySessionFact.getCurrentSession();
		
		try {
			mySession.beginTransaction();
			
			Client client1 = mySession.get(Client.class, 3);
			System.out.println(client1);
			
			List<Item> items = client1.getListItems();
			for(Item i : items) {
				System.out.println(i);
			}
			
			mySession.getTransaction().commit();
		}finally {
			mySessionFact.close();
		}
	}
}
```

##### Добавление ТОВАРОВ для определенного ЧЕЛОВЕКА
Для начала достанем человека из БД, и для него добавим новый товар.
Затем создадим новый товар, и УКАЖЕМ в качестве владельца нашего клиента

> [!NOTE] СВЯЗЬ
> Может возникнуть вопрос, если связь двусторонняя, нужно ли для полученного клиента, в его СПИСОК ТОВАРОВ добавлять только что созданный НОВЫЙ ТОВАР.
> 
> `Hibernate` делает это все АВТОМАТИЧЕСКИ
> 
> ==ОГРОМНОЕ НО==
> `Hibernate` созданные объекты КЭШИРУЕТ, и поэтому, каждый раз когда мы получаем объект, `Hibernate` достает его из КЭША.
> Поэтому ЗАПИСИ в БД могут отличаться от КЭША
> 
> - Если интересует лишь то ЧТО БУДЕТ СОХРАНЕНО В БД - то устанавливать двусторонюю связь ВРУЧНУЮ НЕ НУЖНО
>- Если же планируем работать с объектом, то НУЖНО  
>

```java
try {
	mySession.beginTransaction();
	
	Client client = mySession.get(Client.class, 3);
	Item item1 = new Item(client, "Audi");
	
	mySession.save(item1);
	
	mySession.getTransaction().commit();
}finally {
	mySessionFact.close();
}
```
