## 2. Создание класса контроллера
Контроллеры основные игроки в Spring MVC. Их задача обрабатывать HTTP запросы и либо передавать результат в представление для преобразования в HTML, либо записывать из непосредственно в тело ответа (RESTful).

Для приложения TacoCloud нам понадобится простой контроллер выполняющий следующие действия:
- принимает и обрабатывает HTTP-запросы GET с путем  /design
	
- Составляет список ингредиентов
	
- Передает запрос и ингредиенты в шаблон представления, который будет преобразован в HTML и отправлен в браузер.

```java
package ru.point.Controllers;

import java.util.Arrays;
import java.util.List;
import java.util.stream.Collectors;
  
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.ModelAttribute;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.SessionAttributes;
  
import lombok.extern.slf4j.Slf4j;

import ru.point.Data.ClientPointPita;
import ru.point.Data.Ingredient;
import ru.point.Data.Ingredient.Type;
import ru.point.Data.Pita;
 
@Slf4j
@Controller
@RequestMapping("/design")
@SessionAttributes("tacoOrder")
public class PitaIngredientController {
	@ModelAttribute
	public void addIngedientsToModel(Model model) {
		List<Ingredient> ingredients = Arrays.asList(
			new Ingredient("FLTO", "Flour Tortilla", Type.WRAP),
			new Ingredient("COTO", "Corn Tortilla", Type.WRAP),
			new Ingredient("CARN", "Carnitas", Type.PROTEIN),
			new Ingredient("GRBF", "Ground Beef", Type.PROTEIN),
			new Ingredient("TMTO", "Tomatoes", Type.VEGGIES),
			new Ingredient("CHED", "Cheddar", Type.CHEESE),
			new Ingredient("SLSA", "Salsa", Type.SAUCE)
		);
		Type[] types = Ingredient.Type.values();
		for(Type type : types) {
			model.addAttribute(type.toString().toLowerCase(), filterByType(ingredients, type));
		}
	}
	
	@ModelAttribute(name = "clientPointPita")
	public ClientPointPita client() {
		return new ClientPointPita();
	}
	@ModelAttribute(name = "pita")
	public Pita taco() {
		return new Pita();
	}
	@GetMapping
	public String showDesignForm() {
		return "design";
	}
	private Iterable<Ingredient> filterByType(List<Ingredient> ingredients, Type type)
	{
		return ingredients
			.stream()
			.filter(x -> x.getType().equals(type))
			.collect(Collectors.toList());
	}
}
```

### ModelAtribute и SessinAtributes
Аннотация `@ModelAttribute` позволяет добавить объект в модель, чтобы он был доступен для использования в представлении (например, JSP, Thymeleaf).

Если `@ModelAttribute` используется с методом, Spring вызовет этот метод перед выполнением любого метода контроллера, чтобы гарантировать, что объект существует и доступен для обработки запроса.

> Если объект с указанным именем уже существует в `SessionAttributes`, он будет использован. Если его нет, метод создаст новый объект и добавит его в модель.

**`@ModelAttribute(name = "clientPointPita")`**: Аннотация на методе делает так, чтобы объект `TacoOrder` либо загружался из сессии (если он там есть), либо создавался и добавлялся в модель, если его нет.

**`@SessionAttributes("clientPointPita")`**: Аннотация на уровне класса указывает Spring сохранить объект `TacoOrder` в сессии. Это означает, что объект будет доступен между запросами в течение одной сессии пользователя.
### RequestMapping
Когда `@RequestMapping("/design")` применяется к классу, это означает, что все запросы, которые начинаются с `/design`, будут направлены на методы внутри этого контроллера.
Например, запрос `GET /design` или `POST /design/create` будет обработан этим контроллером.

### Iterable
`Iterable` в Java — это интерфейс, который представляет коллекцию объектов, по которой можно проходить с помощью цикла `for-each`. Этот интерфейс определяет стандартный способ перебора элементов коллекции и является суперинтерфейсом.
- `ingredients.stream()` — создается поток (`Stream`) из списка `ingredients`. Поток позволяет удобно работать с коллекцией данных (например, фильтровать, сортировать, преобразовывать).
    
- `.filter(x -> x.getType().equals(type))` — используется метод `filter`, чтобы отобрать только те элементы, которые соответствуют заданному условию. Здесь:
    - `x` — каждый объект `Ingredient` из списка.
    - `x.getType().equals(type)` — проверяется, совпадает ли тип ингредиента с типом, переданным в метод.
    - Если условие `x.getType().equals(type)` возвращает `true`, объект `x` будет добавлен в результирующий поток. Если `false`, то нет.
- `.collect(Collectors.toList())` — все элементы, которые прошли через фильтр, собираются в список (`List`). Этот список возвращается методом.

 Стрелочка `->` в Java — это лямбда-выражение, которое появилось в версии Java 8. Лямбда-выражения позволяют создавать анонимные функции, которые можно использовать для краткого описания поведения в коде

Основу лямбда-выражения составляет лямбда-оператор, который представляет стрелку ->. Этот оператор разделяет лямбда-выражение на две части: левая часть содержит список параметров выражения, а правая собственно представляет тело лямбда-выражения, где выполняются все действия.
